# mini-sprint #9

## planning
* **Wins**:
  + Implemented a streaming-native, async inference layer
  + Designed a unified gateway for interoperable LLM provider APIs
  + Resolved technical debt in Python and TypeScript stacks
* **Blockers**:
  + Integration issues with Spark micro-batches
  + Limited high-throughput testing for LLM provider APIs
* **Next sprint goals**:
  + Implement async inference layer integration with Spark (2h)
  + Develop high-throughput testing framework for LLM provider APIs (3h)
  + Refactor Python and TypeScript stacks for improved performance (2h)
* **Risks**:
  + Technical debt accumulation due to rapid feature development
  + Insufficient testing and validation of new features
  + Potential performance issues with async inference layer and LLM provider APIs

## retro
* **What went well**:
  + Implemented a streaming-native, async inference layer to decouple Spark micro-batches.
  + Developed a unified gateway for interoperable LLM provider APIs, enabling high-throughput.
  + Utilized multiple backend technologies, including Python and TypeScript.
* **What didn't**:
  + Multiple duplicate ADRs were created for the same unified gateway concept.
  + Lack of diversity in backend technologies, with most ADRs using TypeScript.
  + No clear indication of testing or validation for the implemented solutions.
* **Process improvements**:
  + Implement a duplicate detection system for ADRs to prevent redundant entries.
  + Encourage experimentation with a wider range of backend technologies.
  + Develop a testing and validation framework for new solutions.
* **Quality score**: 6/10
  + Justification: While the team has made progress in implementing new solutions, the lack of testing and validation, as well as the duplication of effort, detract from the overall quality. Additionally, the reliance on a single backend technology (TypeScript) may limit the scalability and flexibility of the solutions.
