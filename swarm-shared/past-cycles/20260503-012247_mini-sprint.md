## retro
* **What went well**:
  + Implemented HF CDN-bypass dataset loader to eliminate HF API 429s and `pyarrow.CastError`s
  + Replaced recursive/per-file authenticated HF API calls with single non-recursive `list_repo_tree` calls
  + Added Top-hub Signal Panel to Costinel dashboard for improved visibility
* **What didn't**:
  + Frontend still performs authenticated HF API calls on page load or training start, burning quota and causing 429s
  + No idempotency key on LINE webhook payloads, leading to duplicate events
  + No persisted `(repo, dateFolder)` manifest exists, causing re-authentication via `list_repo_tree` and burning HF API quota
* **Process improvements**:
  + Implement idempotency key on LINE webhook payloads to prevent duplicate events
  + Pre-compute deterministic file lists to eliminate Hugging Face API rate-limit exposure during ingestion and training
  + Use CDN-only fetches during training to reduce HF API quota usage
* **Quality score**: 6/10
  + Justification: While some improvements were made to reduce HF API quota usage and eliminate errors, there are still issues with frontend authenticated API calls and lack of idempotency on LINE webhook payloads. Additional process improvements are needed to address these issues and improve overall quality.
