# airship / design

## Highest-Value Incremental Improvement (≤2h)

**Embed a CDN-only file manifest into Surrogate-1 training pipeline** so Lightning Studio training runs with **zero HF API calls during data loading**, eliminating 429 rate-limit risk and quota burn during iteration.

### Why this ships fastest and unlocks the most value
- Removes the primary cause of training flakiness (HF API 429) during active development.
- Enables rapid script iteration in Lightning Studio without quota anxiety (reuse running studios, no idle-timeout churn).
- Uses only file-list-once + CDN fetch pattern (no model changes, no infra spin-up).
- Fits in <2h: single manifest generation script + small train.py patch.

---

## Implementation Plan

1. **Generate CDN manifest (Mac orchestration)**
   - Run once per date folder after rate-limit window clears.
   - Use `list_repo_tree(path, recursive=False)` per folder (non-recursive to avoid 100x pagination).
   - Save `batches/mirror-merged/{date}/manifest.json` with `{ "files": ["file1.parquet", ...], "base_url": "https://huggingface.co/datasets/{repo}/resolve/main/batches/mirror-merged/{date}/" }`.

2. **Embed manifest in training payload**
   - Commit `manifest.json` alongside training script (small, stable).
   - In `train.py`, load manifest and construct URLs for direct CDN download via `hf_hub_download`-style or raw `requests`/`urllib`.
   - Parse only `{prompt, response}` at load time; drop all other columns.

3. **Lightning Studio reuse + resilience**
   - Before `.run()`, list `Teamspace.studios` and reuse a running studio with matching name.
   - On idle-stop, restart with `target.start(machine=Machine.L40S)` (or fallback L40S if H200 unavailable).
   - Ensure training script itself makes **zero** `list_repo_files`/`load_dataset(streaming=True)` calls.

4. **Validation**
   - Dry-run manifest generation locally.
   - Run a 1-epoch smoke train in Studio and confirm:
     - No HF API calls in data loader (check logs for `huggingface.co/api`).
     - Dataset loads via CDN URLs.
     - Only projected columns (`prompt`, `response`) in memory.

## Code Snippets

### 1) Manifest generator (run on Mac) — `scripts/generate_cdn_manifest.py`

```python
#!/usr/bin/env python3
"""
Generate CDN-only manifest for Surrogate-1 training.
Run from Mac (or any orchestration node) after HF API window clears.
"""
import json
import os
from datetime import datetime
from huggingface_hub import HfApi

REPO = "axentx/surrogate-1-mirror"
DATE_FOLDER = datetime.utcnow().strftime("%Y-%m-%d")  # or pass as arg
BASE_PATH = f"batches/mirror-merged/{DATE_FOLDER}"
OUT_PATH = f"batches/mirror-merged/{DATE_FOLDER}/manifest.json"

api = HfApi()

def main() -> None:
    # Non-recursive per-folder to avoid heavy pagination
    entries = api.list_repo_tree(
        repo_id=REPO,
        path=BASE_PATH,
        repo_type="dataset",
        recursive=False,
    )

    files = [e.path for e in entries if e.path.lower().endswith(".parquet")]
    if not files:
        raise RuntimeError(f"No parquet files found in {BASE_PATH}")

    manifest = {
        "repo": REPO,
        "date": DATE_FOLDER,
        "base_url": f"https://huggingface.co/datasets/{REPO}/resolve/main/{BASE_PATH}/",
        "files": sorted(files),
    }

    os.makedirs(os.path.dirname(OUT_PATH), exist_ok=True)
    with open(OUT_PATH, "w") as f:
        json.dump(manifest, f, indent=2)

    print(f"Manifest written to {OUT_PATH} ({len(files)} files)")

if __name__ == "__main__":
    main()
```

### 2) Lightweight CDN dataset loader — `surrogate/train_data.py`

```python
import json
import pyarrow.parquet as pq
import requests
from io import BytesIO
from typing import List, Dict

class CDNParquetLoader:
    """Load projected {prompt, response} from parquet via CDN (zero HF API)."""

    def __init__(self, manifest_path: str):
        with open(manifest_path) as f:
            self.manifest = json.load(f)
        self.base_url = self.manifest["base_url"]

    def _fetch_parquet(self, filename: str) -> bytes:
        url = f"{self.base_url}{filename}"
        resp = requests.get(url, timeout=60)
        resp.raise_for_status()
        return resp.content

    def load_all(self) -> List[Dict[str, str]]:
        examples = []
        for fname in self.manifest["files"]:
            data = self._fetch_parquet(fname)
            table = pq.read_table(BytesIO(data), columns=["prompt", "response"])
            df = table.to_pandas()
            # Drop rows with missing values and convert to records
            df = df.dropna(subset=["prompt", "response"])
            examples.extend(df.to_dict(orient="records"))
        return examples
```

### 3) Lightning Studio reuse + resilient run — `scripts/launch_train.py`

```bash
#!/usr/bin/env bash
# Mac orchestration launcher for Surrogate-1 training.
# Ensures Studio reuse and restarts on idle-stop.

set -euo pipefail
export SHELL=/bin/bash

MANIFEST_PATH="batches/mirror-merged/$(date -u +%Y-%m-%d)/manifest.json"
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"

# Generate manifest if missing (safe to re-run; lightweight)
if [[ ! -f "$MANIFEST_PATH" ]]; then
  echo "Manifest missing, generating..."
  python3 "$PROJECT_ROOT/scripts/generate_cdn_manifest.py"
fi

python3 - <<PY
import lightning as L
from lightning.fabric.plugins import LightningContainer
from pathlib import Path

MANIFEST_PATH = Path("$MANIFEST_PATH")
PROJECT_ROOT = Path("$PROJECT_ROOT")

# Reuse running studio if available
teamspace = L.Teamspace()
studio_name = "surrogate-1-train"
studio = None
for s in teamspace.studios:
    if s.name == studio_name and s.status == "running":
        studio = s
        print(f"Reusing running studio: {studio_name}")
        break

if studio is None:
    print(f"Starting new studio: {studio_name}")
    studio = teamspace.create_studio(
        name=studio_name,
        machine="L40S",  # fallback if H200 unavailable
        cloud="lightning-public-prod",
    )

# Ensure running
if studio.status != "running":
    studio.start(machine="L40S")

# Run training (zero HF API calls during data load)
run = studio.run(
    str(PROJECT_ROOT / "surrogate" / "train.py"),
    requirements=[
        "pyarrow",
        "requests",
        "lightning",
        "huggingface_hub",
    ],
)
print(f"Run started: {run}")
PY
```
