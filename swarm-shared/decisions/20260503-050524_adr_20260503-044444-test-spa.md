## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployed on CF Workers with Supabase Postgres; supports single-account read-only discovery for Lambda and Cloud Functions; lists duplicate schedules (same cron+target), drifted schedules (cron changed since last run), and orphaned schedules (target missing); exports CSV/JSON report and stores audit events; no automated remediation, no UI, free-tier only.

## Open questions
- Which IAM scopes are minimally required to list/describe Lambda EventBridge rules and Cloud Functions triggers across customer accounts (cross-account vs single-account first)?
- How will we detect 'orphaned' schedules (e.g., target function deleted) given eventual consistency and cross-region boundaries?
- Will customers prefer read-only auditor mode or opt-in automated retirement of stale schedules (and what approval workflow is needed)?
- What retention and privacy policy applies to payload_hash and schedule metadata (PII in tags)?
