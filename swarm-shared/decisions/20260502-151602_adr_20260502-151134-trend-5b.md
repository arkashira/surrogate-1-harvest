## v0.1 scope
v0.1 in 2 weeks: deterministic workflow engine with research→signal→risk→execution pipeline; Postgres-backed immutable run state and audit trail; one broker connector (Alpaca sandbox) and one data vendor (IEX Cloud free); three guardrails (max position, sector cap, kill-switch); Cloudflare Queues for durable dispatch and replay; local DuckDB audit export; CI tests with recorded traces; deploy to HF Space (backend) + Supabase (Postgres) + Cloudflare (Queues/KV); NSM tracking for weighted successful runs.

## Open questions
- What is the maximum acceptable end-to-end latency (research→execution) to declare sub-second met? Target p95?
- Which brokers/OMS/EMS and data vendors must be supported in v0.1 and what auth flows do they require (OAuth, API keys, mTLS)?
- What compliance rules are non-negotiable for launch (e.g., pre-trade checks, kill-switch semantics, record retention period)?
- How should deterministic replay be triggered and scoped (replay by time window, by signal, or by workflow definition)?
- Where will secrets reside in dev/test/prod (Supabase Vault, Cloudflare Secrets, or local envelope-encrypted files)?
