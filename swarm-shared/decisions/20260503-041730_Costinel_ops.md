# Costinel / ops

## Final Synthesis — CDN-First Top-Hub Signal Panel (Production-Ready, <2h)

**Goal**: Add a resilient, zero-runtime-HF-API “Top-Hub Signal” panel to Costinel that surfaces the most-connected hub (e.g., “MOC”) using CDN-first baked data and robust fallbacks. Follows “Sense + Signal — ไม่ Execute” (read-only signals, no mutations).

---

### Why this approach (merged rationale)
- Avoids HF API rate limits and commit caps by baking data at build/deploy time.
- Aligns with `#knowledge-rag #graph #hub` pattern: review most-connected hub before planning.
- Uses CDN for static assets; no runtime HF API calls.
- Prioritizes graceful degradation: UI never breaks if CDN or baked data is missing.

---

### Scope (what ships)
1. **Bake step** (CI or local script):  
   - Single `list_repo_tree` call for `knowledge-rag/top-hub/` (non-recursive) → produce `public/signals/top-hub-files.json`.  
   - Generate `public/signals/top-hub.json` containing:  
     `{ hub, score, cdnUrl, lastUpdated, files[], relatedDocs[], fallback? }`

2. **Frontend panel** (`TopHubSignalPanel`):  
   - Fetches `/signals/top-hub.json` (CDN/static).  
   - Shows hub name, score, last updated, link to file via CDN, and top 3 related docs.  
   - Dual fallback:  
     - Runtime fetch fails → use bundled fallback JSON (imported at build).  
     - No data → neutral empty state (no UI errors).

3. **Ops script** (`scripts/bake-top-hub.sh`):  
   - Idempotent bash script using HF CDN + optional HF API for tree (with retry/backoff).  
   - Shebang `#!/usr/bin/env bash`, `set -euo pipefail`, designed for `bash scripts/bake-top-hub.sh "$@"`.

4. **Zero runtime HF API** — all data is static or CDN-fetched.

### Final code (merged + corrected)

#### 1) Bake script — `scripts/bake-top-hub.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

# Config
REPO="AXENTX/knowledge-rag"
FOLDER="top-hub"
OUT_DIR="public/signals"
FILES_JSON="${OUT_DIR}/top-hub-files.json"
SIGNAL_JSON="${OUT_DIR}/top-hub.json"
HF_API_BASE="https://huggingface.co/api"
HF_CDN_BASE="https://huggingface.co/datasets"

mkdir -p "${OUT_DIR}"

# 1) List files (non-recursive) — single API call
echo "Listing ${REPO}/${FOLDER} ..."
TREE=$(curl -sSf "${HF_API_BASE}/repos/${REPO}/tree?path=${FOLDER}&recursive=false" 2>/dev/null || echo "[]")

# Save file list
echo "${TREE}" | jq '[ .[] | select(.type=="file") | {path, size, lfs} ]' > "${FILES_JSON}"

# 2) Pick latest by filename pattern (expect yyyy-mm-dd or similar)
LATEST_FILE=$(echo "${TREE}" | jq -r '[ .[] | select(.type=="file") | .path ] | sort | last')
if [[ -z "${LATEST_FILE}" || "${LATEST_FILE}" == "null" ]]; then
  echo "No files found, using fallback."
  cat > "${SIGNAL_JSON}" <<'EOF'
{
  "hub": "MOC",
  "score": 0,
  "cdnUrl": null,
  "lastUpdated": null,
  "files": [],
  "relatedDocs": [],
  "fallback": true
}
EOF
  exit 0
fi

# 3) Build CDN URL (datasets repo resolve)
CDN_URL="${HF_CDN_BASE}/${REPO}/resolve/main/${FOLDER}/${LATEST_FILE}"

# 4) Try to extract metadata from filename/content (best-effort)
# Expect filename like "top-hub-2026-04-27-moc-98.json" or similar
BASENAME=$(basename "${LATEST_FILE}" .json)
HUB_GUESS=$(echo "${BASENAME}" | grep -o '[a-zA-Z]\+' | tail -1 || echo "MOC")
SCORE_GUESS=$(echo "${BASENAME}" | grep -o '[0-9]\+' | tail -1 || echo "0")
UPDATED_AT=$(echo "${BASENAME}" | grep -o '[0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\}' | head -1 || date +%F)

# 5) Produce top-hub.json (with relatedDocs placeholder — extend if real metadata exists)
cat > "${SIGNAL_JSON}" <<EOF
{
  "hub": "${HUB_GUESS}",
  "score": ${SCORE_GUESS},
  "cdnUrl": "${CDN_URL}",
  "lastUpdated": "${UPDATED_AT}",
  "files": $(echo "${TREE}" | jq '[ .[] | select(.type=="file") | .path ]'),
  "relatedDocs": [],
  "fallback": false
}
EOF

echo "Baked ${SIGNAL_JSON} -> hub=${HUB_GUESS}, score=${SCORE_GUESS}"
```

Make executable:
```bash
chmod +x scripts/bake-top-hub.sh
```
