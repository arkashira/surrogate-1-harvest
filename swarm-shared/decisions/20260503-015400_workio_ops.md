# workio / ops

## 1. Diagnosis

- **No idempotency key enforcement on LINE webhook** — retries with same `X-Line-Signature` + payload can create duplicate punches because uniqueness is enforced only at the app level (`findOne → update/insert`), not at the DB level.
- **Race condition in punch upsert** — concurrent webhook deliveries (LINE retries + user rapid taps) can interleave `findOne` calls and both proceed to `insert`, producing two rows for the same punch event.
- **Missing unique constraint on tenant+user+date+punch_type+session** — without a DB constraint, duplicates can exist and reports/rollups can silently double-count time.
- **No idempotency table for LINE event IDs** — LINE Messaging API includes `webhookEventId` in the event envelope; not storing it means we cannot detect replays across different signature payloads (e.g., if LINE changes signature format or retries with canonicalized payload).
- **Ops observability gap** — duplicate punches are only detectable retroactively via reports; no alerting or dedupe dashboard for ops to intervene quickly.

## 2. Proposed change

- **File**: `workio/server/src/db/schema.sql` (add constraint + idempotency table)  
- **File**: `workio/server/src/routes/line/webhook.ts` (add atomic upsert using constraint + store `webhookEventId`)  
- **Scope**: add DB-level unique constraint on `(tenant_id, user_id, punch_date, punch_type, session_id)` and a unique index on `webhook_event_id`; convert punch upsert to single `INSERT ... ON CONFLICT DO NOTHING` (or update only status fields) so duplicates are rejected atomically.

### 3.2 Webhook handler (`workio/server/src/routes/line/webhook.ts`)

```typescript
import { Request, Response } from 'express';
import { pool } from '../../db';
import crypto from 'crypto';

// Verify LINE signature (keep existing)
function verifyLineSignature(rawBody: string, signature: string): boolean {
  const channelSecret = process.env.LINE_CHANNEL_SECRET || '';
  const hash = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody, 'utf8')
    .digest('base64');
  return hash === signature;
}

export async function lineWebhook(req: Request, res: Response) {
  const signature = req.get('X-Line-Signature') || '';
  const rawBody = JSON.stringify(req.body);
  if (!verifyLineSignature(rawBody, signature)) {
    return res.status(401).send('Invalid signature');
  }

  const events = req.body.events || [];
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    for (const event of events) {
      const webhookEventId = event.webhookEventId;
      if (!webhookEventId) continue;

      // Idempotency: skip if event already processed
      const exists = await client.query(
        'SELECT 1 FROM line_webhook_events WHERE webhook_event_id = $1',
        [webhookEventId]
      );
      if (exists.rows.length > 0) continue;

      // Record event first (best-effort; if punch insert fails we still avoid replay)
      await client.query(
        'INSERT INTO line_webhook_events (webhook_event_id, payload) VALUES ($1, $2)',
        [webhookEventId, event]
      );

      // Only handle message events for clock in/out (simplified)
      if (event.type === 'message' && event.message.type === 'text') {
        const userId = event.source.userId;
        const text = event.message.text.trim().toLowerCase();
        const punchType = text === 'in' || text === 'clock in' ? 'in' : text === 'out' || text === 'clock out' ? 'out' : null;
        if (!punchType) continue;

        const tenantId = 1; // derive from DB or mapping; placeholder
        const punchDate = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
        const sessionId = webhookEventId; // or derive from business session

        // Atomic upsert: insert only if unique constraint not violated
        // If you need to update fields on conflict (e.g., updated_at), use DO UPDATE
        await client.query(
          `INSERT INTO punches (tenant_id, user_id, punch_date, punch_type, session_id, created_at, updated_at)
           VALUES ($1, $2, $3, $4, $5, NOW(), NOW())
           ON CONFLICT (tenant_id, user_id, punch_date, punch_type, session_id)
           DO NOTHING`,
          [tenantId, userId, punchDate, punchType, sessionId]
        );
      }
    }

    await client.query('COMMIT');
    res.status(200).send('OK');
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing error:', err);
    res.status(500).send('Server error');
  } finally {
    client.release();
  }
}
```

## 4. Verification

1. **Schema check** — confirm constraint exists:
   ```bash
   psql workio -c "\d punches"
   # should show UNIQUE CONSTRAINT for (tenant_id, user_id, punch_date, punch_type, session_id)
   ```

2. **Idempotency table** — confirm it’s present:
   ```bash
   psql workio -c "SELECT * FROM line_webhook_events LIMIT 1;"
   ```

3. **Replay test** — simulate duplicate webhook delivery:
   - Send a valid LINE webhook payload twice (same `webhookEventId`) via curl or Postman.
   - Verify only one punch row is created:
     ```bash
     psql workio -c "SELECT count(*) FROM punches WHERE session_id = '<test-session>';"
     # expected: 1
     ```

4. **Concurrency test** — fire two requests in parallel for same tenant/user/date/type/session:
   ```bash
   hey -n 10 -c 5 -m POST -H "Content-Type: application/json" -H "X-Line-Signature: <sig>" -d '{"events":[{"webhookEventId":"test-dup-123","type":"message","source":{"userId":"U123"},"message":{"type":"text","text":"in"}}]}' http://localhost:3000/webhook/line
   ```
   - Confirm only one row inserted and no errors.
