## v0.1 scope
v0.1 ships a single-platform (Linux x86_64) CLI stealth client that: establishes a tun0 interface via boringtun; routes all traffic through a local shadowsocks client with pluggable obfuscation (simple XOR/TLS camouflage) to a fixed outbound proxy; enforces a fail-closed killswitch that blocks non-VPN interfaces via iptables while active; emits minimal, opt-in, anonymized session events to Supabase Postgres; and accepts signed config bundles from Cloudflare Workers KV for endpoint/obfs updates — all deployable as a static binary with zero server provisioning.

## Open questions
- Which cover protocols (TLS 1.3 mimicry, QUIC, or domain-fronting) provide the best DPI evasion vs. performance trade-off for our target regions?
- How will clients bootstrap endpoint hints without leaking DNS queries (DoH/DoT vs. baked-in IP/DoH fallback)?
- What telemetry granularity is acceptable for WAU-ZB while preserving deniability and minimizing client-side storage/identifiers?
- How do we distribute and rotate signed config bundles through Workers KV/Durable Objects while preventing replay/rollback attacks?
- Which platform-specific TUN/packet-filter APIs require special handling to remain stealthy (Windows WFP, macOS utun, Linux netfilter)?
