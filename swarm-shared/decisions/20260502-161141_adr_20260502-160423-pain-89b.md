## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI agent (Linux/macOS/Windows) that creates randomized local SOCKS5 ports, obfuscates TLS fingerprints, tunnels traffic through a CF Worker reflector, hides TUN artifacts when possible, and enforces a local kill-switch that blocks non-stealth egress; includes encrypted SQLite session store and minimal encrypted telemetry to compute WAU-ZB.

## Open questions
- Which DPI signatures are highest risk in target regions (SNI, JA3, packet length/time, TUN interface presence)?
- Can we reliably run a TUN device in userspace across Windows/macOS/Linux without elevated privileges?
- What kill-switch granularity is required (per-app, system-wide, IPv4/IPv6) and how to enforce it client-side?
- How will we distribute/rotate reflector endpoints (CF workers) to avoid single-point blocking?
- What telemetry can be collected without deanonymizing users or leaking session metadata?
