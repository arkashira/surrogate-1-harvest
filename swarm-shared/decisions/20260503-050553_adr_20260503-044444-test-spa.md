## Deployment

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI endpoints that connect to one AWS account and one GCP project, list schedules, compute next-run, detect duplicates (identical cron+target) and orphans (schedule with no matching deployed function), expose a Supabase-backed Postgres for inventory and audit events, and output a simple JSON report.

## Open questions
- How will we authenticate/authorize cross-account AWS and cross-project GCP reads (least-privilege roles vs. user-provided tokens)?
- What is the sync cadence (ad-hoc API vs. periodic jobs) and how do we avoid provider rate limits?
- What constitutes a duplicate (identical cron+target, identical cron across targets, or time-colliding schedules)?
- How will customers retire stale schedules (advisory report only or automated safe-disable with approval flow)?
