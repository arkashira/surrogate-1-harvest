# airship / ops

## Implementation Plan — <2h Quality Increment

**Highest-value incremental improvement**: Harden the `airship discover` orchestrator so it is **fully deterministic, CDN-safe, and zero-runtime** by:
- Adding a strict Bash shebang and POSIX-safe guards (no runtime deps beyond `bash`, `curl`, `jq`, `find`)
- Pre-listing HF dataset file paths once and embedding them as a JSON manifest (avoid recursive `list_repo_files` during runtime)
- Using HF CDN direct downloads (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) with zero Authorization headers to bypass API rate limits
- Making the script idempotent and safe for cron (exit codes, temp file cleanup, lockfile)
- Ensuring executable bit and invoking via `bash` in crontab with `SHELL=/bin/bash`

**Why this is highest value**:
- Eliminates HF API 429 failures during discovery/ingest
- Removes runtime schema surprises (project only `{prompt,response}` at parse time)
- Makes `airship discover` safe for unattended ops (cron, CI, Airflow)
- Costs nothing, ships in <2h, applies all past lessons (HF CDN bypass, pre-list, no streaming, Mac=orchestrator-only)

---

### Concrete steps (all in `/opt/axentx/airship`)

1. Create hardened wrapper: `scripts/airship-discover.sh`
2. Add lockfile + temp cleanup + strict mode
3. Use single `list_repo_tree` (non-recursive) for one date folder → save `file-list.json`
4. Train/ingest script reads `file-list.json` and downloads via CDN URLs only
5. Make executable and add cron stub with `SHELL=/bin/bash`

#!/usr/bin/env bash
# airship-discover.sh
# Deterministic, CDN-safe, zero-runtime discovery for HF datasets.
# Usage: bash scripts/airship-discover.sh <dataset_repo> <date_folder> <out_dir>
# Crontab:
#   SHELL=/bin/bash
#   0 2 * * * /opt/axentx/airship/scripts/airship-discover.sh myorg/surrogate-data 2026-05-02 /opt/axentx/airship/data/mirror >> /var/log/airship-discover.log 2>&1

readonly HF_TOKEN="${HF_TOKEN:-}"  # optional; only used for private repos
readonly LOCKFILE="/tmp/airship-discover.lock"
readonly MAX_AGE_SECONDS=86400

log() {
  echo "[$(date -u +"%Y-%m-%dT%H:%M:%SZ")] [$SCRIPT_NAME] $*"
}

  log "ERROR: $*"
  exit 1
}

cleanup() {
  rm -f "$LOCKFILE"

DATASET_REPO="${1:-}"
DATE_FOLDER="${2:-}"
OUT_DIR="${3:-}"

  AUTH_HEADER=()

# Ensure non-empty

fi

mv "${MANIFEST}.tmp" "$MANIFEST"
log "Saved manifest with $(wc -l < "$MANIFEST") files to $MANIFEST"

# 2) Download via CDN (no auth, bypasses API rate limits)
# CDN tier has much higher limits. Public datasets require no Authorization header.
DOWNLOAD_DIR="${OUT_DIR}/downloads"
mkdir -p "$DOWNLOAD_DIR"

log "Downloading files via CDN to $DOWNLOAD_DIR"
FAILED=0
while IFS= read -r relpath; do
  # Skip empty
  [[ -z "$relpath" ]] && continue
  url="${HF_API}/datasets/${DATASET_REPO}/resolve/main/${DATE_FOLDER}/${relpath}"
  outpath="${DOWNLOAD_DIR}/${relpath}"
  mkdir -p "$(dirname "$outpath")"
  if curl -fsSL --retry 3 --retry-delay 5 -o "$outpath" "$url"; then
    log "OK: ${relpath}"
  else
    log "FAILED: ${relpath} (HTTP $?)"
    rm -f "$outpath"
    ((FAILED++)) || true
  fi
done < "$MANIFEST"

if (( FAILED > 0 )); then
  fail "$FAILED file(s) failed to download"
fi
