## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres + edge functions) + Hugging Face Space (free-tier Redis/worker fallback)

## v0.1 scope
v0.1 in 2 weeks: deterministic research→signal→risk→execution workflow engine with Postgres-backed immutable audit trail, Redis-backed signal/risk cache, mock broker/OMS connectors, compliance guardrails (hard limits on notional/position), replayable runs via Cloudflare Queues (or asyncio replay), and API endpoints to submit workflows and fetch audit events; no UI, free-tier deployable to CF Workers + Supabase + HF Space.

## Open questions
- What are the deterministic replay and idempotency guarantees required across connectors (broker/OMS) that may not support at-least-once safely?
- Which compliance checks must be blocking vs non-blocking and what SLAs (e.g., sub-500ms) apply to guardrail evaluation?
- How will secrets/credentials for brokers and data vendors be provisioned and rotated on free-tier infra (Workers/Supabase/HF)?
- What is the canonical data model for normalized signals across vendors to ensure deterministic merges and replay?
- Which subset of broker/OMS connectors will be in-scope for v0.1 to limit integration surface?
