# Costinel / ops

## Implementation Plan — Costinel “Top-Hub Signal” Card (read-only)

**Scope**: Frontend-only card that identifies the most-connected hub from the knowledge-rag graph and surfaces the top hub with 3 contextual signals.  
**Timebox**: <2h (frontend-only, zero backend changes).  
**Tags**: #knowledge-rag #graph #hub #frontend

---

### 1. Goal
- Add a read-only dashboard card: “Top-Hub Signal”
- Identify most-connected hub (e.g., “MOC”) from knowledge-rag graph
- Show hub name + 3 contextual signals (short summaries or doc links)
- Zero backend changes — use static JSON + client-side ranking (or lightweight fetch to existing `/api/knowledge-rag/top-hub` if already available)

---

### 2. Implementation Steps (≤2h)

1. **Check existing API surface**  
   - Look for `/api/knowledge-rag/top-hub` or `/api/graph/hubs`.  
   - If exists: use it. If not: add static JSON under `public/data/top-hub.json` (fallback) and plan backend later.

2. **Create card component**  
   - Location: `src/components/cards/TopHubSignalCard.tsx` (or `.jsx` if TS not used).  
   - Props: `hub`, `signals: Array<{title, summary, href}>`.

3. **Fetch + rank logic (client)**  
   - Fetch hub list (or use static JSON).  
   - Rank by `connections` (or `degree`) descending.  
   - Pick top hub and map top 3 related signals.

4. **Add to dashboard layout**  
   - Insert into main dashboard grid (likely `src/pages/Dashboard.tsx` or equivalent).  
   - Ensure responsive (mobile-first).

5. **Styling & UX**  
   - Use existing design tokens (colors, spacing).  
   - Add subtle icon (hub/network).  
   - Make signals clickable (open in new tab if external).

6. **Testing**  
   - Smoke test in dev.  
   - Verify no console errors.  
   - Ensure graceful fallback when data missing.

#### `src/components/cards/TopHubSignalCard.tsx`
```tsx
import React, { useEffect, useState } from 'react';
import './TopHubSignalCard.css';

interface Signal {
  title: string;
  summary: string;
  href: string;
}

interface Hub {
  id: string;
  label: string;
  type: string;
  connections: number;
}

interface TopHubData {
  hub: Hub;
  signals: Signal[];
}

const TopHubSignalCard: React.FC = () => {
  const [data, setData] = useState<TopHubData | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchData = async () => {
      try {
        // Try API first; fallback to static JSON
        const res = await fetch('/api/knowledge-rag/top-hub').catch(() => null);
        if (res?.ok) {
          const json = await res.json();
          setData(json);
        } else {
          const fallback = await fetch('/data/top-hub.json');
          const json = await fallback.json();
          setData(json);
        }
      } catch (err) {
        console.warn('TopHubSignalCard: failed to load data', err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  if (loading) return <div className="top-hub-card loading">Loading hub signal…</div>;
  if (!data) return null;

  return (
    <div className="top-hub-card">
      <div className="top-hub-header">
        <span className="top-hub-icon" aria-hidden="true">◎</span>
        <h3>Top-Hub Signal</h3>
      </div>
      <div className="top-hub-body">
        <div className="hub-name">{data.hub.label}</div>
        <div className="hub-meta">{data.hub.connections} connections</div>
        <ul className="signals-list">
          {data.signals.map((s, i) => (
            <li key={i} className="signal-item">
              <a href={s.href} target="_blank" rel="noopener noreferrer" className="signal-link">
                <div className="signal-title">{s.title}</div>
                <div className="signal-summary">{s.summary}</div>
              </a>
            </li>
          ))}
        </ul>
      </div>
    </div>
  );
};

export default TopHubSignalCard;
```
