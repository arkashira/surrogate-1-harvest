## Deployment
cf workers (free tier) + Supabase (free tier) — deploy API as worker, Postgres via Supabase, optional HF space for lightweight UI if added later

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP (FastAPI worker on CF) that connects to one AWS account and one GCP project via read-only credentials, lists all schedules, detects duplicates (same cron targeting different functions), detects orphans (schedules referencing deleted functions), and exposes a simple JSON report endpoint and CSV export; stores results in Supabase Postgres; no UI, no auth beyond bearer token; manual credential provisioning via env vars.

## Open questions
- How will customers grant least-privilege read-only access to Lambda and Cloud Functions (IAM roles vs service account keys)?
- What is the sync cadence that balances freshness vs rate limits/costs (hourly vs daily)?
- How do we define and detect 'drift' across providers (expression change, timezone change, target function change)?
- What retention period do we need for Schedule and Finding records on the free tier?
- Do we need to support Terraform/CloudFormation state reconciliation as an additional signal?
