# workio / backend

## Final Synthesis — One Correct, Actionable Plan

**Core goal:** Make punch writes safe under retries, enforce valid state transitions, keep LINE webhooks fast, and preserve auditability/compliance.

---

### 1. Diagnosis (merged & resolved)

- **Duplicate punches from retries**  
  → Require an idempotency key and enforce it server-side with a uniqueness constraint.

- **Invalid state transitions (clock-in while already in, clock-out without prior clock-in)**  
  → Server must validate last punch per user/tenant and reject invalid transitions.

- **Partial failures between punch write and LINE reply**  
  → Commit punch first; send LINE reply after commit (fire-and-forget with retries). Never couple external calls inside the DB transaction.

- **Missing location enforcement and audit trail**  
  → Store raw lat/long + verification result with each punch. Add immutable punch history (append-only) and soft-delete for edits; do not overwrite rows.

- **Synchronous geocoding/location checks blocking webhook**  
  → Do lightweight validation synchronously (radius check against a cached site list). Offload heavy geocoding/reverse-geocode to a background job.

CREATE TABLE IF NOT EXISTS idempotency_keys (
  key TEXT PRIMARY KEY,
  payload_hash TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_idempotency_created ON idempotency_keys(created_at);

-- 2) Punch history (append-only for audit/compliance)
CREATE TABLE IF NOT EXISTS punches (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  tenant_id TEXT NOT NULL,
  type TEXT NOT NULL CHECK (type IN ('clock_in', 'clock_out')),
  latitude NUMERIC,
  longitude NUMERIC,
  location_verified BOOLEAN DEFAULT FALSE,
  location_note TEXT,
  idempotency_key TEXT UNIQUE NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  created_by TEXT,
  deleted_at TIMESTAMPTZ
);
CREATE INDEX idx_punches_user_tenant ON punches(user_id, tenant_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_punches_created ON punches(created_at) WHERE deleted_at IS NULL;

-- 3) Sites for location radius checks (lightweight sync validation)
CREATE TABLE IF NOT EXISTS sites (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id TEXT NOT NULL,
  name TEXT NOT NULL,
  latitude NUMERIC NOT NULL,
  longitude NUMERIC NOT NULL,
  radius_meters NUMERIC NOT NULL DEFAULT 100,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

**Idempotency helper**
```ts
// server/src/lib/idempotency.ts
import { Pool } from 'pg';
import crypto from 'crypto';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function registerIdempotent(key: string, payload: any): Promise<boolean> {
  const hash = crypto.createHash('sha256').update(JSON.stringify(payload)).digest('hex');
  const res = await pool.query(
    'INSERT INTO idempotency_keys(key, payload_hash) VALUES($1,$2) ON CONFLICT (key) DO NOTHING RETURNING 1',
    [key, hash]
  );
  return res.rowCount > 0;
}
```

import { Pool } from 'pg';

import { registerIdempotent } from '../lib/idempotency';

const router = express.Router();

function distance(lat1: number, lon1: number, lat2: number, lon2: number): number {
  const R = 6371000;
  const toRad = (x: number) => (x * Math.PI) / 180;
  const dLat = toRad(lat2 - lat1);
  const dLon = toRad(lon2 - lon1);
  const a =
    Math.sin(dLat / 2) ** 2 +
    Math.cos(toRad(lat1)) * Math.cos(toRad(lat2)) * Math.sin(dLon / 2) ** 2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
}
