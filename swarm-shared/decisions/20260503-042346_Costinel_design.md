# Costinel / design

## Final Synthesis — CDN-First Top-Hub Signal Panel (<2h)

**Chosen approach:** Build-time fetch from HF CDN + static asset fallback.  
**Why:** Zero runtime HF API calls (no 429/limits), deterministic deploy, fits Costinel “sense + signal” philosophy, and can be shipped in <2h.

---

### 1) Canonical data contract (merge best fields)
Use a strict, minimal schema that supports UI, provenance, and future extensibility.

```ts
// src/types/hub.ts
export interface HubSignal {
  slug: string;            // e.g. "moc"
  title: string;           // e.g. "Multi-Org Cost governance"
  connections?: number;    // optional graph degree
  score?: number;          // optional relevance/confidence
  summary: string;
  related?: string[];      // optional tags/topics
  updatedAt: string;       // ISO date
  url: string;             // canonical link
  source?: string;         // e.g. "knowledge-rag"
}
```

**Rationale:** Combines Candidate 1 (`connections`, explicit `url`) with Candidate 2 (`score`, `related`, `source`) while keeping required fields minimal (`slug`, `title`, `summary`, `updatedAt`, `url`).

---

### 2) Build-time fetch (robust, CI-friendly)

```bash
#!/usr/bin/env bash
# scripts/fetch-top-hub.sh
# Downloads baked top-hub.json from HF CDN into public/data/
# Usage: npm run fetch:top-hub  (or prebuild)

set -euo pipefail

REPO="axentx/knowledge-rag"
PATH_IN_REPO="baked/top-hub.json"
OUT_DIR="public/data"
OUT_FILE="${OUT_DIR}/top-hub.json"
CDN_URL="https://huggingface.co/datasets/${REPO}/resolve/main/${PATH_IN_REPO}"

mkdir -p "${OUT_DIR}"

echo "Fetching top-hub signal from CDN: ${CDN_URL}"
if curl -fsSL --retry 3 --retry-delay 5 --max-time 15 "${CDN_URL}" -o "${OUT_FILE}.tmp"; then
  # Validate required fields
  if jq -e '.slug and .title and .summary and .updatedAt and .url' "${OUT_FILE}.tmp" > /dev/null 2>&1; then
    mv "${OUT_FILE}.tmp" "${OUT_FILE}"
    echo "Saved to ${OUT_FILE}"
    exit 0
  else
    echo "Invalid payload — missing required fields"
    rm -f "${OUT_FILE}.tmp"
    exit 1
  fi
else
  echo "Failed to fetch top-hub signal from CDN"
  rm -f "${OUT_FILE}.tmp"
  exit 1
fi
```

**Make executable:** `chmod +x scripts/fetch-top-hub.sh`  
**CI/Build integration:** add `npm run fetch:top-hub` to `prebuild` or CI job.

### 3) Resilient React panel (no runtime HF API)

```tsx
// src/components/TopHubSignalPanel.tsx
import React, { useEffect, useState } from "react";
import { HubSignal } from "../types/hub";

const FALLBACK: HubSignal = {
  slug: "moc",
  title: "MOC — Most Connected Hub",
  summary: "Signal unavailable — using default hub context.",
  updatedAt: new Date().toISOString(),
  url: "https://github.com/axentx/knowledge-rag",
  source: "fallback",
};

export const TopHubSignalPanel: React.FC = () => {
  const [signal, setSignal] = useState<HubSignal | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    let mounted = true;
    setLoading(true);

    fetch("/data/top-hub.json", { cache: "no-store" })
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then((data: HubSignal) => {
        if (mounted) {
          setSignal(data);
          setError(null);
        }
      })
      .catch((err) => {
        if (mounted) {
          console.warn("Top-hub CDN fetch failed, using fallback:", err);
          setSignal(FALLBACK);
          setError("Using fallback signal");
        }
      })
      .finally(() => {
        if (mounted) setLoading(false);
      });

    return () => {
      mounted = false;
    };
  }, []);

  if (loading) {
    return (
      <div className="rounded border border-gray-200 bg-gray-50 p-4">
        <div className="h-4 w-32 animate-pulse rounded bg-gray-300" />
        <div className="mt-2 h-3 w-24 animate-pulse rounded bg-gray-300" />
      </div>
    );
  }

  if (!signal) return null;

  return (
    <div className="rounded border border-blue-200 bg-blue-50 p-4 shadow-sm">
      <div className="flex items-start justify-between">
        <div>
          <h4 className="text-sm font-semibold text-blue-900">Top Hub Signal</h4>
          <p className="mt-1 text-sm font-medium text-blue-800">{signal.title}</p>
          <p className="mt-1 text-xs text-blue-700">{signal.summary}</p>
          <div className="mt-2 flex flex-wrap items-center gap-3 text-xs text-blue-600">
            {typeof signal.connections === "number" && (
              <span>{signal.connections} connections</span>
            )}
            {typeof signal.score === "number" && (
              <span>Score {signal.score.toFixed(2)}</span>
            )}
            <span>Updated {new Date(signal.updatedAt).toLocaleDateString()}</span>
          </div>
          {signal.related && signal.related.length > 0 && (
            <div className="mt-2 flex flex-wrap gap-1">
              {signal.related.map((tag) => (
                <span
                  key={tag}
                  className="rounded bg-blue-100 px-1.5 py-0.5 text-xs text-blue-800"
                >
                  {tag}
                </span>
              ))}
            </div>
          )}
        </div>
        <a
          href={signal.url}
          target="_blank"
          rel="noopener noreferrer"
          className="text-xs text-blue-700 underline hover:text-blue-900"
        >
          View
        </a>
      </div>
      {error && <div className="mt-2 text-xs text-amber-700">{error}</div>}
    </div>
  );
};
```

**Key improvements vs candidates:**
- Uses both `connections` and `score` when present.
- Shows `related` tags if available.
- Keeps resilient fallback (Candidate 1) and explicit source/provenance (Candidate 2).
- No runtime HF API — only static `/data/top-hub.json`.
