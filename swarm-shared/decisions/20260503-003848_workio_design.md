# workio / design

## 1. Diagnosis

- **Missing idempotency key on LINE webhook**: LINE retries on network blips/5xx/slow client can create duplicate punches; no request-level deduplication exists.
- **No DB-level guard against multiple open punches**: Absence of partial unique index (`user_id`, `clock_out_at IS NULL`) permits concurrent or retried requests to open multiple active punches.
- **Non-atomic clock-in flow**: Read-then-insert allows race conditions between duplicate/retried webhook deliveries; no transaction or `ON CONFLICT` handling.
- **No idempotency table for webhook events**: Without storing `line_delivery_id`/`event_id`, retries cannot be detected and safely short-circuited.
- **Unclear punch state machine**: Missing explicit state (e.g., `pending`, `clocked_in`, `clocked_out`) makes duplicate/out-of-order event handling brittle.

## 2. Proposed change

- **File**: `workio/server/src/db/schema.sql` (add constraint + idempotency table)  
- **File**: `workio/server/src/routes/webhook/line.ts` (add idempotency check + atomic upsert)  
- **Scope**: Add partial unique index on `punches(user_id) WHERE clock_out_at IS NULL`, add `webhook_events` table with `event_id` unique key, and wrap punch creation in an upsert that is idempotent on `line_delivery_id`.

## 3. Implementation

```sql
-- workio/server/src/db/schema.sql
-- Add idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS webhook_events (
  id              SERIAL PRIMARY KEY,
  event_id        TEXT NOT NULL,
  source_type     TEXT NOT NULL,        -- 'line'
  source_id       TEXT NOT NULL,        -- channel/user id for partitioning
  payload_hash    TEXT,                 -- optional integrity check
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT uq_webhook_event UNIQUE (source_type, event_id)
);

-- Ensure one open punch per user (partial unique index)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
  ON punches (user_id)
  WHERE clock_out_at IS NULL;
```

```ts
// workio/server/src/routes/webhook/line.ts
import { Router } from 'express';
import { pool } from '../../db';
import crypto from 'crypto';

const router = Router();

async function upsertPunch(userId: number, lineEventId: string, clockInAt: Date, location?: { lat: number; lng: number }) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency: skip if event already processed
    const exists = await client.query(
      `SELECT 1 FROM webhook_events WHERE source_type = $1 AND event_id = $2`,
      ['line', lineEventId]
    );
    if (exists.rows.length > 0) {
      await client.query('ROLLBACK');
      return { created: false, reason: 'duplicate_event' };
    }

    // Record event first (avoid races)
    await client.query(
      `INSERT INTO webhook_events (event_id, source_type, source_id, payload_hash)
       VALUES ($1, $2, $3, $4)`,
      [lineEventId, 'line', String(userId), crypto.createHash('sha256').update(JSON.stringify({ userId, clockInAt, location })).digest('hex')]
    );

    // Atomic upsert: insert punch only if no open punch exists for user
    // If a row exists with clock_out_at IS NULL, do nothing (or optionally update location)
    const res = await client.query(
      `INSERT INTO punches (user_id, clock_in_at, clock_out_at, latitude, longitude)
       VALUES ($1, $2, NULL, $3, $4)
       ON CONFLICT (user_id)
       WHERE clock_out_at IS NULL
       DO NOTHING
       RETURNING id, clock_in_at`,
      [userId, clockInAt, location?.lat ?? null, location?.lng ?? null]
    );

    await client.query('COMMIT');

    if (res.rowCount === 0) {
      // There was already an open punch; treat as idempotent success (no double punch)
      return { created: false, reason: 'already_clocked_in' };
    }

    return { created: true, punchId: res.rows[0].id, clockInAt: res.rows[0].clock_in_at };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}

router.post('/line', async (req, res) => {
  try {
    const { events } = req.body;
    if (!Array.isArray(events)) return res.status(400).json({ error: 'invalid_payload' });

    const results = [];
    for (const ev of events) {
      if (ev.type !== 'message' || ev.message?.type !== 'text') continue;

      // Example: parse userId from source.userId and clock-in intent from message
      const userId = await resolveUserIdFromLineUid(ev.source.userId); // implement mapping
      const location = ev.message?.location
        ? { lat: ev.message.location.latitude, lng: ev.message.location.longitude }
        : undefined;

      const result = await upsertPunch(userId, ev.eventId, new Date(ev.timestamp), location);
      results.push({ eventId: ev.eventId, result });
    }

    res.json({ ok: true, results });
  } catch (err) {
    console.error(err);
    res.status(500).json({ error: 'internal' });
  }
});

export default router;
```

## 4. Verification

- **DB constraint check**: Run `\d punches` and confirm partial unique index exists; run `\d webhook_events` and confirm unique constraint on `(source_type, event_id)`.
- **Duplicate event rejection**: Send same LINE webhook payload twice (same `eventId`); second request must return `{ created: false, reason: 'duplicate_event' }` and not create a second punch row.
- **Concurrent punch prevention**: Simulate two simultaneous clock-in requests for same user (different `eventId`s); only one punch row with `clock_out_at IS NULL` should exist (use `pgbench` or parallel curl). Verify no integrity error leaks to client (handled gracefully).
- **Idempotent retry after failure**: Insert event into `webhook_events`, then retry same event; confirm no new punch row and response indicates duplicate.
- **End-to-end LINE retry simulation**: Use ngrok + LINE console to replay webhook with same `eventId`; confirm single punch row and no duplicate notifications.
