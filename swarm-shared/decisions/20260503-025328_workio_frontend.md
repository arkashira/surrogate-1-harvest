# workio / frontend

## 1. Diagnosis

- **No optimistic UI for clock-in/out** — frontend waits for full round-trip to backend before updating button state, causing perceived lag and double-taps that queue duplicate requests.
- **Missing client-side request deduplication** — rapid clicks or retries can fire multiple identical punch requests while previous ones are in-flight; no abort/ignore logic.
- **No local timestamp mirroring** — clock-in/out list shows server times only; user can’t immediately see their own punch reflected with local context.
- **Race between UI state and LINE location permission flow** — GPS acquisition can delay punch request; UI doesn’t reflect “acquiring location” vs “sending punch” states separately.
- **No offline/queuing fallback** — if network drops during punch, user gets generic error and must retry manually; no local queue or retry hint.

## 2. Proposed change

**Scope:** `workio/src/features/punch/PunchButton.tsx` (create or update) and `workio/src/features/punch/usePunch.ts` (hook).  
Add optimistic UI + client-side in-flight deduplication + local timestamp preview + clear status states (idle/acquiring/sending/succeeded/error). Keep backend contract unchanged.

## 3. Implementation

Create/replace `src/features/punch/usePunch.ts`:

```ts
// src/features/punch/usePunch.ts
import { useState, useCallback, useRef } from 'react';

type PunchStatus = 'idle' | 'acquiring' | 'sending' | 'succeeded' | 'error';

export function usePunch() {
  const [status, setStatus] = useState<PunchStatus>('idle');
  const [lastPunch, setLastPunch] = useState<{ ts: string; type: 'in' | 'out' } | null>(null);
  const inFlightRef = useRef<AbortController | null>(null);

  const punch = useCallback(async (type: 'in' | 'out', location?: GeolocationPosition) => {
    // Deduplicate: ignore if same type already in-flight
    if (inFlightRef.current) {
      return { ok: false, reason: 'in-flight' };
    }

    const ac = new AbortController();
    inFlightRef.current = ac;
    setStatus('acquiring');

    try {
      // If location required and not provided, try to acquire (short timeout)
      let pos = location;
      if (!pos) {
        pos = await new Promise<GeolocationPosition>((resolve, reject) => {
          const geo = navigator.geolocation;
          if (!geo) reject(new Error('Geolocation unavailable'));
          geo.getCurrentPosition(resolve, reject, {
            enableHighAccuracy: true,
            timeout: 8000,
            maximumAge: 30000,
          });
        });
      }

      setStatus('sending');

      const body = {
        type,
        latitude: pos?.coords?.latitude ?? null,
        longitude: pos?.coords?.longitude ?? null,
        accuracy: pos?.coords?.accuracy ?? null,
        clientTs: new Date().toISOString(),
      };

      const res = await fetch('/api/punches', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(body),
        signal: ac.signal,
      });

      if (!res.ok) throw new Error(`HTTP ${res.status}`);

      const server = await res.json();
      // Optimistic update: prefer server timestamp if provided, else client ts
      const ts = server?.data?.timestamp ?? body.clientTs;
      setLastPunch({ ts, type });
      setStatus('succeeded');
      return { ok: true, ts, type };
    } catch (err: any) {
      if (err.name !== 'AbortError') {
        setStatus('error');
        console.error('Punch failed', err);
      }
      return { ok: false, reason: err.message };
    } finally {
      inFlightRef.current = null;
      // Auto-reset status after brief success display
      if (status === 'succeeded') {
        setTimeout(() => setStatus('idle'), 2000);
      }
    }
  }, []);

  const reset = useCallback(() => {
    if (inFlightRef.current) {
      inFlightRef.current.abort();
      inFlightRef.current = null;
    }
    setStatus('idle');
  }, []);

  return { status, lastPunch, punch, reset };
}
```

Create/replace `src/features/punch/PunchButton.tsx`:

```tsx
// src/features/punch/PunchButton.tsx
import React from 'react';
import { usePunch } from './usePunch';

export function PunchButton() {
  const { status, lastPunch, punch, reset } = usePunch();

  const handlePunchIn = () => punch('in');
  const handlePunchOut = () => punch('out');

  const isWorking = lastPunch?.type === 'in' && status !== 'succeeded';
  const canPunchIn = !isWorking && status === 'idle';
  const canPunchOut = isWorking && status === 'idle';

  return (
    <div className="flex flex-col items-center gap-3">
      <div className="flex gap-3">
        <button
          onClick={handlePunchIn}
          disabled={!canPunchIn || status === 'acquiring' || status === 'sending'}
          className={`px-5 py-2 rounded font-medium transition ${
            canPunchIn
              ? 'bg-green-600 text-white hover:bg-green-700'
              : 'bg-gray-200 text-gray-500 cursor-not-allowed'
          } ${status === 'sending' ? 'opacity-70' : ''}`}
        >
          {status === 'acquiring' ? 'กำลังระบุตำแหน่ง...' : 'ลงเวลา'}
        </button>

        <button
          onClick={handlePunchOut}
          disabled={!canPunchOut || status === 'acquiring' || status === 'sending'}
          className={`px-5 py-2 rounded font-medium transition ${
            canPunchOut
              ? 'bg-red-600 text-white hover:bg-red-700'
              : 'bg-gray-200 text-gray-500 cursor-not-allowed'
          } ${status === 'sending' ? 'opacity-70' : ''}`}
        >
          {status === 'acquiring' ? 'กำลังระบุตำแหน่ง...' : 'ออกเวลา'}
        </button>
      </div>

      {status === 'succeeded' && lastPunch && (
        <p className="text-sm text-green-700">
          ลงเวลา {lastPunch.type === 'in' ? 'เข้า' : 'ออก'} เรียบร้อย •{' '}
          {new Date(lastPunch.ts).toLocaleTimeString()}
        </p>
      )}

      {status === 'error' && (
        <div className="text-sm text-red-600 flex items-center gap-2">
          <span>ไม่สามารถลงเวลาได้</span>
          <button onClick={reset} className="underline text-sm">
            ลองใหม่
          </button>
        </div>
      )}
    </div>
  );
}
```

## 4. Verification

- **Fast optimistic feedback**: Click “ลงเวลา” → button disables, status shows “กำลังระบุตำแหน่ง…” (if acquiring) then “ส่ง” briefly, then green confirmation with local timestamp appears within ~1–2s (even before server responds).
- **Deduplication**: Rapid double-click or triple-click while acquiring/sending does not trigger additional network requests (check Network tab; only one POST in-flight).
- **Abort on unmount/retry**: Navigate away or hit “ลองใหม่” while in-flight → request aborts (no “canceled” errors in console), state resets to idle.
- **Error path**: Block network (offline) or return 500 from devtools → red error appears with retry button; no duplicate punches created.
- **Backend unchanged**: Confirm existing API `/api/punches` still receives same payload shape and responds 200; no DB schema
