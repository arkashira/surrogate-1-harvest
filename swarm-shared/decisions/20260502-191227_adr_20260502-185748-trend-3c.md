## Deployment
cf workers (free tier) + Supabase (free) + HF space (CPU-small for optional hosted analyzer); primary local CLI runs anywhere

## v0.1 scope
v0.1 in 2 weeks: CLI-only local analyzer for Python repos using tree-sitter + ruff + Semgrep OSS rules; produces JSON report and terminal summary; local SQLite cache for findings; optional upload of anonymized metadata to Supabase (user opt-in); no paid features, auth optional for cloud sync.

## Open questions
- Will users accept optional cloud upload for cross-repo semantic search, or must all embeddings remain strictly local?
- Which language analyzers are required in v0.1 (Python/JS/Go) and can we reuse tree-sitter grammars exclusively?
- How will licensing/compliance handle Semgrep rules vs. custom proprietary rulesets?
- What is the performance target for cold analysis on a 10k-file repo on a laptop CPU (time/RAM)?
- Do we ship a local SQLite cache per repo or centralize metadata in Supabase from day one?
