## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-instrument (e.g. SPY) research→signal→risk→execution pipeline with deterministic state machine, Supastore audit log, Cloudflare Queues for stage decoupling, and a replay harness that can reprocess a day's runs end-to-end; target median signal-to-execution latency <500ms in local/perf tests and expose health/latency metrics.

## Open questions
- Which brokers/data vendors will be supported first and what auth/connectivity model (OAuth, API keys, FIX) is acceptable within free-tier constraints?
- What is the deterministic replay SLA (time window, data retention) required for compliance forensics on the free tier?
- How will clock synchronization and timestamp precision be enforced across research→signal→risk→execution to guarantee <500ms median latency?
- What constitutes a 'trade' for the NSM (paper vs live) and how will success be measured in v0.1?
- Edge vs region placement for Workers: where must stateful steps (risk/execution) run to minimize latency to brokers?
