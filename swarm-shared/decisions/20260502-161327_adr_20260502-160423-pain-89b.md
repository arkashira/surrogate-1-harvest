## Deployment
cf workers

## v0.1 scope
v0.1 ships a single Rust CLI client (static binary) that: establishes a local SOCKS5 listener hidden behind shadowsocks_aead; creates a TUN via boringtun with traffic encapsulated in obfuscated streams; enforces a process-level kill-switch that blocks non-tunneled traffic if stealth path fails; stores ephemeral session state in SQLite and submits minimal, pseudonymous telemetry to Supabase via Cloudflare Workers; uses Workers KV to fetch revocation/allow-list and config bundles on launch. No server management required by users; all coordination via CF Workers + Supabase free tier.

## Open questions
- Which obfuscation primitives minimize DPI fingerprint while maximizing throughput on mobile and desktop (e.g., meek HTTP/2 vs. QUIC vs. utp_mux)?
- How will we distribute and rotate ephemeral ports/bind strategies to avoid port-scan detection without centralized server control?
- What telemetry schema and sampling rate balances WAU-ZB measurement with client privacy and minimal storage?
- How do we implement a reliable client-side kill-switch across platforms (TUN/SOCKS5) that is fail-closed and non-bypassable?
- Which revocation propagation latency is acceptable for Workers KV vs. Durable Objects, and how do we handle offline clients?
