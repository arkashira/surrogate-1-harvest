# Costinel / discovery

## Implementation Plan — Costinel Top-Hub Signal Card (Read-Only)

**Scope**: Read-only frontend card (≤2h)  
**Principle**: “Sense + Signal — ไม่ Execute” (strictly no writes, no runtime mutations, no self-execution)  
**Goal**: Surface the most-connected hub (e.g., “MOC”) with contextual insights from the knowledge-rag graph.

### 1) High-value incremental improvement
Add a persistent, read-only “Top-Hub Signal” card to the dashboard that:
- Identifies the most-connected hub node from the knowledge-rag graph (cached, read-only lookup).
- Shows a short insight blurb and related docs count.
- Links to the hub detail view (read-only).
- Refreshes only via explicit user action or cache expiry (no background writes/auto-mutations).

### 2) Concrete implementation steps (≤2h)

#### A) Add read-only hub resolver (backend) — 20 min
- Create `src/server/routes/hubs.ts` (or extend existing analytics route).
- Expose `GET /api/hubs/top` returning `{ slug, title, insight, relatedDocsCount, updatedAt }`.
- Source data from a static JSON snapshot generated by knowledge-rag (e.g., `data/knowledge-rag/top-hubs.json`) to avoid runtime graph queries and ensure read-only behavior.
- No DB writes; cache headers `max-age=300`.

```ts
// src/server/routes/hubs.ts
import { Router } from 'express';
import fs from 'fs';
import path from 'path';

const router = Router();
const SNAPSHOT = path.join(process.cwd(), 'data/knowledge-rag/top-hubs.json');

router.get('/api/hubs/top', (req, res) => {
  try {
    const raw = fs.readFileSync(SNAPSHOT, 'utf8');
    const hubs = JSON.parse(raw);
    const top = hubs[0] || null;
    res.set('Cache-Control', 'public, max-age=300');
    res.json(top ? {
      slug: top.slug,
      title: top.title,
      insight: top.insight || 'High-impact hub with cross-domain signals.',
      relatedDocsCount: top.relatedDocsCount || 0,
      updatedAt: top.updatedAt || new Date().toISOString(),
    } : {});
  } catch (err) {
    res.status(503).json({ error: 'Hub snapshot unavailable' });
  }
});

export default router;
```

#### C) Add TopHubSignalCard component (frontend) — 45 min
- Create `src/components/TopHubSignalCard.tsx`.
- Fetch `/api/hubs/top` client-side (SWR or react-query) with `revalidateOnMount` only (no polling).
- Render card with icon, title, insight excerpt, and link to `/hubs/{slug}` (read-only route).
- Keep UI minimal and non-interactive beyond navigation.

```tsx
// src/components/TopHubSignalCard.tsx
import useSWR from 'swr';
import Link from 'next/link';

const fetcher = (url: string) => fetch(url).then((r) => r.json());

export default function TopHubSignalCard() {
  const { data, error } = useSWR('/api/hubs/top', fetcher, {
    revalidateOnMount: true,
    revalidateOnFocus: false,
    refreshInterval: 0,
  });

  if (error) return null;
  if (!data?.slug) return null;

  return (
    <div className="rounded-lg border bg-card p-4 shadow-sm">
      <div className="flex items-start justify-between">
        <div>
          <h3 className="text-sm font-semibold text-foreground">Top Hub</h3>
          <p className="text-lg font-medium">{data.title}</p>
          <p className="mt-1 text-sm text-muted-foreground line-clamp-2">
            {data.insight}
          </p>
          <div className="mt-2 flex items-center gap-3 text-xs text-muted-foreground">
            <span>{data.relatedDocsCount} related docs</span>
            <span>Updated {new Date(data.updatedAt).toLocaleDateString()}</span>
          </div>
        </div>
        <Link
          href={`/hubs/${encodeURIComponent(data.slug)}`}
          className="text-sm text-primary hover:underline"
        >
          View
        </Link>
      </div>
    </div>
  );
}
```
