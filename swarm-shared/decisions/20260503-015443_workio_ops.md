# workio / ops

## 1. Diagnosis

- **No idempotency guard on LINE webhook replays** — `X-Line-Signature` is verified but not persisted; identical payload retries create duplicate clock-in/out rows.
- **Non-atomic read-then-write upsert** — `findOne` → conditional `update`/`insert` races under concurrent replays or double-tap by users.
- **Missing unique constraint** — DB allows multiple active clock-ins per user (no `(userId, tenantId, date, type)` uniqueness), enabling invalid state.
- **No replay dedup window** — retries within minutes (LINE retry policy) are treated as new events rather than idempotent repeats.
- **Clock-in/out state derived on read** — business logic computes “latest punch” from non-unique rows instead of enforcing single active row.

## 2. Proposed change

- **File scope**: `workio/server/src/services/lineWebhookService.ts` (or equivalent handler) and `workio/server/src/db/schema.sql`.
- **Changes**:
  - Add `line_msg_idempotency` table (or extend punches with `line_delivery_id` + unique constraint).
  - Add unique constraint on active clock-in rows (`userId, tenantId, date, type` where `type='in'` and `clocked_out_at IS NULL`).
  - Replace `findOne` → upsert with atomic `INSERT ... ON CONFLICT DO NOTHING` (idempotent by `line_delivery_id`), then state transition via single `UPDATE ... WHERE clocked_out_at IS NULL`.

## 3. Implementation

### 3.1 DB schema (run once)

```sql
-- workio/server/src/db/schema.sql
-- Idempotency table for LINE delivery replay protection
CREATE TABLE IF NOT EXISTS line_msg_idempotency (
  line_delivery_id VARCHAR(255) PRIMARY KEY,
  tenant_id        VARCHAR(255) NOT NULL,
  user_id          VARCHAR(255) NOT NULL,
  event_type       VARCHAR(50)  NOT NULL,
  payload_hash     VARCHAR(64)  NOT NULL,
  created_at       TIMESTAMPTZ  DEFAULT NOW()
);

-- Ensure at most one active clock-in per user/tenant/date
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_active_clock_in
ON punches (tenant_id, user_id, DATE(clocked_in_at))
WHERE clocked_out_at IS NULL;
```

### 3.2 Idempotent handler

```ts
// workio/server/src/services/lineWebhookService.ts
import { db } from '../db';
import crypto from 'crypto';

function payloadHash(body: any): string {
  return crypto.createHash('sha256').update(JSON.stringify(body)).digest('hex');
}

export async function handleClockEvent(
  tenantId: string,
  userId: string,
  lineDeliveryId: string,
  body: any
) {
  const hash = payloadHash(body);

  // Atomic idempotency insert (safe under concurrent replays)
  const idempotent = await db.oneOrNone(
    `INSERT INTO line_msg_idempotency (line_delivery_id, tenant_id, user_id, event_type, payload_hash)
     VALUES ($1, $2, $3, $4, $5)
     ON CONFLICT (line_delivery_id) DO NOTHING
     RETURNING line_delivery_id`,
    [lineDeliveryId, tenantId, userId, 'clock', hash]
  );

  // If not inserted, this is a replay — skip processing
  if (!idempotent) {
    return { status: 'duplicate', message: 'already processed' };
  }

  // Atomic state transition: close previous active punch if exists, insert new punch
  await db.tx(async (t: any) => {
    // Close any open punch for this user/tenant (should be at most one due to partial unique index)
    await t.none(
      `UPDATE punches
       SET clocked_out_at = NOW(), updated_at = NOW()
       WHERE tenant_id = $1 AND user_id = $2 AND clocked_out_at IS NULL`,
      [tenantId, userId]
    );

    // Insert new clock-in
    await t.none(
      `INSERT INTO punches (tenant_id, user_id, clocked_in_at, clocked_out_at, source, line_delivery_id, created_at, updated_at)
       VALUES ($1, $2, NOW(), NULL, 'line', $3, NOW(), NOW())`,
      [tenantId, userId, lineDeliveryId]
    );
  });

  return { status: 'ok', message: 'clocked in' };
}
```

1. **Schema applied**: run `psql workio < server/src/db/schema.sql` and confirm:
   - `\d line_msg_idempotency` exists with PK on `line_delivery_id`.
   - Partial index `idx_one_active_clock_in` exists.

2. **Idempotency test** (local or staging):
   - Send identical LINE webhook payload twice within seconds.
   - Expect: first returns `{"status":"ok"}`, second returns `{"status":"duplicate"}`.
   - Assert only one punch row created and `clocked_out_at IS NULL`.

3. **Race test**:
   - Fire 10 concurrent identical requests (curl in parallel).
   - Assert exactly one punch row with `clocked_out_at IS NULL` and no constraint violations.

4. **Active punch uniqueness**:
   - Manually insert an active punch (`clocked_out_at IS NULL`).
   - Attempt to insert another via handler — should close previous and create new one (or reject cleanly depending on desired policy).

5. **Log check**:
   - Confirm handler logs `duplicate` for replays and no duplicate punches appear in dashboard/reports.
