# workio / design

### Final Synthesized Solution  
*(Best parts combined; contradictions resolved in favor of correctness + concrete actionability)*

---

## 1. Diagnosis (consensus)

- **No storage-level uniqueness** for active punches → duplicates on `(employee_id, punch_type, date)`.
- **No idempotency key** → LINE webhook and client retries create duplicates.
- **Non-atomic read-then-insert** (check-then-act) → race window.
- **No transactional upsert** → blind `INSERT` loses under concurrency.
- **Missing audit columns** → no trail for corrections/voids.

---

## 2. Proposed change (single plan)

**Scope**
- File: `server/src/db/schema.sql` (or `/opt/axentx/workio/server/src/db/schema.sql` — use project-relative path).
- Add:
  - `idempotency_key`, `status`, `updated_at` to `punches`.
  - Unique **partial index** for one active punch per `(employee_id, punch_type, date)`.
  - Unique index on `idempotency_key` (for safe retries).
  - Auto-maintained `updated_at` via trigger.
- Update punch write path to use **transactional upsert + idempotency check**.

## 3. Implementation

### 3.1 Schema changes (SQL)

```sql
-- server/src/db/schema.sql
-- Apply to existing punches table

-- Add idempotency and audit columns (safe to rerun)
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255) NULL,
  ADD COLUMN IF NOT EXISTS status VARCHAR(50) NOT NULL DEFAULT 'active',
  ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW();

-- Unique active punch per employee per punch_type per date
-- Allows multiple rows only when status != 'active' (voided/cancelled)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_unique_active
  ON punches (employee_id, punch_type, date)
  WHERE status = 'active';

-- Idempotency index (supports safe retries)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key
  ON punches (idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- Keep updated_at current on row updates
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

DROP TRIGGER IF EXISTS trg_punches_set_updated_at ON punches;
CREATE TRIGGER trg_punches_set_updated_at
  BEFORE UPDATE ON punches
  FOR EACH ROW
  EXECUTE FUNCTION set_updated_at();
```

Apply:
```bash
psql workio < server/src/db/schema.sql
```

### 3.2 Punch write path (Node/Express + pg)

File: `server/src/routes/punches.js` (or `.ts`)

```js
// POST /api/punches
// Requires: employee_id, punch_type, date, idempotency_key (recommended)
async function createPunch(req, res) {
  const { employee_id, punch_type, date, idempotency_key } = req.body;
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency fast-path
    if (idempotency_key) {
      const found = await client.query(
        'SELECT id, status FROM punches WHERE idempotency_key = $1',
        [idempotency_key]
      );
      if (found.rows.length > 0) {
        await client.query('ROLLBACK');
        // Return existing punch; do not create new row
        return res.status(200).json({ id: found.rows[0].id, duplicate: true });
      }
    }

    // 2) Atomic insert with unique constraint protection
    const result = await client.query(
      `INSERT INTO punches (employee_id, punch_type, date, idempotency_key, status)
       VALUES ($1, $2, $3, $4, 'active')
       ON CONFLICT DO NOTHING
       RETURNING id`,
      [employee_id, punch_type, date, idempotency_key]
    );

    if (result.rowCount === 0) {
      // Conflict: duplicate active punch for same (employee_id, punch_type, date)
      await client.query('ROLLBACK');
      return res.status(409).json({ error: 'Duplicate active punch for this date and type' });
    }

    await client.query('COMMIT');
    return res.status(201).json({ id: result.rows[0].id });
  } catch (err) {
    await client.query('ROLLBACK');
    return res.status(500).json({ error: err.message });
  } finally {
    client.release();
  }
}
```

**Key correctness choices**
- Use **partial unique index** (`WHERE status = 'active'`) so voided/cancelled punches don’t block new active ones.
- **Idempotency check inside the same transaction** before insert to avoid TOCTOU races.
- `ON CONFLICT DO NOTHING` + rowCount check ensures atomic upsert semantics.
- Return `200` with `duplicate: true` for idempotent retries; `409` for true duplicates.

## 5. Summary of resolved contradictions

- **Unique constraint form**: Use **partial unique index** (`WHERE status = 'active'`) instead of a table-wide `UNIQUE` constraint — allows voided/cancelled punches while preventing duplicate active punches.
- **Idempotency placement**: Idempotency key stored in `punches` with a unique index (not a separate table) — simpler, fewer joins, and sufficient for this domain.
- **Atomicity**: Idempotency check + insert performed inside a single transaction to close TOCTOU races.
- **Auditability**: Added `status` and `updated_at` for traceability and safe soft-state management.
