# workio / ops

## 1. Diagnosis
- **No idempotency at DB level**: LINE retries with same `X-Line-Signature`/event can create duplicate punch rows.
- **Non-atomic find-then-upsert**: Clock-in/out path uses `findOne → if exists → update/else insert` under concurrent LINE retries → race conditions.
- **No dedupe key on pending punches**: `usePunchQueue` uses random `uuidv4()` per attempt — retries create multiple pending entries for same logical punch.
- **Missing unique constraint**: `punches(employee_id, punch_time, punch_type)` allows duplicates within same second.
- **No transactional rollback on failure**: partial writes can leave inconsistent state when webhook processing fails mid-flow.

## 2. Proposed change
- **Scope**: `server/src/db/schema.sql` (add constraint + index) + `server/src/services/punchService.ts` (atomic upsert + idempotency key) + `src/hooks/usePunchQueue.ts` (idempotency key derived from event).
- **Goal**: make punch creation idempotent and atomic at DB level; eliminate duplicate rows from LINE retries.

## 3. Implementation

### 3.1 DB schema — add idempotency constraint and index
```sql
-- server/src/db/schema.sql
-- Add idempotency key column if not exists
ALTER TABLE punches ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255);
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key ON punches(idempotency_key) WHERE idempotency_key IS NOT NULL;

-- Prevent duplicate punches for same employee+time+type (within same second)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_unique_per_second
ON punches (employee_id, DATE_TRUNC('second', punch_time), punch_type);
```

### 3.2 Punch service — atomic upsert with idempotency key
```ts
// server/src/services/punchService.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export type PunchType = 'IN' | 'OUT';

export async function createOrUpdatePunch({
  employeeId,
  punchTime,
  punchType,
  latitude,
  longitude,
  idempotencyKey,
}: {
  employeeId: number;
  punchTime: Date;
  punchType: PunchType;
  latitude?: number;
  longitude?: number;
  idempotencyKey: string;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Try idempotency key first
    const existingByIdempotency = await client.query(
      'SELECT id FROM punches WHERE idempotency_key = $1',
      [idempotencyKey]
    );
    if (existingByIdempotency.rows.length > 0) {
      await client.query('ROLLBACK');
      return existingByIdempotency.rows[0];
    }

    // Atomic upsert by unique time+type
    const upsert = await client.query(
      `INSERT INTO punches (employee_id, punch_time, punch_type, latitude, longitude, idempotency_key)
       VALUES ($1, $2, $3, $4, $5, $6)
       ON CONFLICT (employee_id, DATE_TRUNC('second', punch_time), punch_type)
       DO UPDATE SET latitude = EXCLUDED.latitude, longitude = EXCLUDED.longitude, idempotency_key = EXCLUDED.idempotency_key
       RETURNING *`,
      [employeeId, punchTime, punchType, latitude, longitude, idempotencyKey]
    );

    await client.query('COMMIT');
    return upsert.rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 3.3 Hook — derive stable idempotency key from LINE event
```tsx
// src/hooks/usePunchQueue.ts
import { useCallback, useRef } from 'react';
import { createPunch } from '../api/punchApi';

type PunchType = 'IN' | 'OUT';

export function usePunchQueue() {
  const inflight = useRef<Set<string>>(new Set());

  const enqueue = useCallback(async (event: any, punchType: PunchType) => {
    // Deterministic idempotency key from LINE event
    const idempotencyKey = event.idempotencyKey || `${event.source.userId}-${event.timestamp}-${event.type}`;
    if (inflight.current.has(idempotencyKey)) return;
    inflight.current.add(idempotencyKey);

    try {
      await createPunch({
        punchType,
        latitude: event.latitude,
        longitude: event.longitude,
        idempotencyKey,
      });
    } finally {
      inflight.current.delete(idempotencyKey);
    }
  }, []);

  return { enqueue };
}
```

### 3.4 API layer — pass idempotency key to backend
```ts
// src/api/punchApi.ts
export async function createPunch(payload: {
  punchType: 'IN' | 'OUT';
  latitude?: number;
  longitude?: number;
  idempotencyKey: string;
}) {
  const res = await fetch('/api/punches', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload),
  });
  if (!res.ok) throw new Error('Punch failed');
  return res.json();
}
```

- **Manual**: simulate LINE retry by sending same webhook payload twice (same `timestamp` + `userId`). Confirm only one row created in `punches` and second request returns existing row.
- **SQL check**: `SELECT employee_id, DATE_TRUNC('second', punch_time), punch_type, COUNT(*) FROM punches GROUP BY 1,2,3 HAVING COUNT(*) > 1;` — should return zero rows.
- **Idempotency key check**: `SELECT idempotency_key, COUNT(*) FROM punches WHERE idempotency_key IS NOT NULL GROUP BY 1 HAVING COUNT(*) > 1;` — should return zero rows.
- **Hook test**: call `enqueue` twice rapidly with same synthetic event; confirm `inflight` prevents duplicate POST and only one network request is made.
