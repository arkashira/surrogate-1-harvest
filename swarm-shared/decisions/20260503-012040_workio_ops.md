# workio / ops

## 1. Diagnosis

- LINE webhook retries are not idempotent: duplicate `events` with same `webhookEventId` can create double punches or double leave/OT records.
- Punch creation uses `find-then-insert/update` on `punches` without DB uniqueness, so concurrent/close requests can create two open punches for the same employee.
- No transactional boundary between punch write and LINE reply / notification; partial failure can leave user state inconsistent.
- Clock-in/out endpoint accepts stale/out-of-order timestamps (e.g., clock-in after an existing open punch) without server-side time monotonicity checks.
- Ops observability is minimal: no structured request IDs, no webhook delivery log, and no metrics for duplicates/races.

## 2. Proposed change

- **Files**:  
  - `workio/server/src/routes/webhook/line.ts` (webhook handler)  
  - `workio/server/src/services/punch.service.ts` (punch logic)  
  - `workio/server/src/db/schema.sql` (add constraints + idempotency table)  
  - `workio/server/src/middleware/request-id.ts` (correlation IDs)
- **Scope**: add idempotency key enforcement for LINE webhooks, enforce single open punch via DB constraint + upsert, and wrap punch creation in transaction with LINE reply inside same tx scope where possible (or best-effort after-commit with retry).

## 3. Implementation

### 3.1 DB schema changes (`schema.sql`)

-- Idempotency keys for LINE webhook events (unique per event)
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  event_id      VARCHAR(255) PRIMARY KEY,
  received_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  handled_at    TIMESTAMPTZ,
  payload_hash  BYTEA,
  UNIQUE (event_id)
);

-- Ensure at most one open punch per employee
-- Closed punches have clock_out NOT NULL; open punches have clock_out NULL.
-- Use partial unique index to enforce single open punch.
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_employee
ON punches (employee_id)
WHERE clock_out IS NULL;

### 3.2 Request ID middleware (`middleware/request-id.ts`)

```ts
import { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';

export function requestIdMiddleware(
  req: Request,
  _res: Response,
  next: NextFunction
) {
  const id = req.headers['x-request-id'] as string || crypto.randomUUID();
  (req as any).requestId = id;
  next();
}

### 3.3 Punch service (`services/punch.service.ts`)

```ts
import { PoolClient } from 'pg';
import { pool } from '../db';

export type PunchInput = {
  employeeId: number;
  clockIn: Date;
  clockOut?: Date | null;
  latitude?: number;
  longitude?: number;
  imageUrl?: string;
};

export async function upsertPunchWithIdempotency(
  input: PunchInput,
  client?: PoolClient
): Promise<{ punchId: number; wasCreated: boolean }> {
  const db = client || pool;
  // Use ON CONFLICT on the partial index via exclusion pattern:
  // We enforce single open punch at DB level; try insert first, if violates
  // unique open punch, update the existing open punch (only if still open).
  const result = await db.query(
    `
    INSERT INTO punches (employee_id, clock_in, clock_out, latitude, longitude, image_url)
    VALUES ($1, $2, $3, $4, $5, $6)
    ON CONFLICT (employee_id)
    WHERE clock_out IS NULL
    DO UPDATE SET
      clock_in = EXCLUDED.clock_in,
      latitude = EXCLUDED.latitude,
      longitude = EXCLUDED.longitude,
      image_url = EXCLUDED.image_url
    RETURNING id, (xmax = 0) AS was_created
    `,
    [
      input.employeeId,
      input.clockIn,
      input.clockOut || null,
      input.latitude || null,
      input.longitude || null,
      input.imageUrl || null,
    ]
  );

  const row = result.rows[0];
  return { punchId: row.id, wasCreated: Boolean(row.was_created) };
}

export async function handleClockInOut(
  employeeId: number,
  now: Date,
  location?: { latitude: number; longitude: number },
  imageUrl?: string
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Check for existing open punch
    const open = await client.query(
      `SELECT id, clock_in FROM punches WHERE employee_id = $1 AND clock_out IS NULL FOR UPDATE`,
      [employeeId]
    );

    if (open.rows.length > 0) {
      // Clock-out existing open punch
      await client.query(
        `UPDATE punches SET clock_out = $1 WHERE id = $2`,
        [now, open.rows[0].id]
      );
      await client.query('COMMIT');
      return { action: 'clock_out', punchId: open.rows[0].id, time: now };
    }

    // Clock-in new punch
    const { punchId } = await upsertPunchWithIdempotency(
      {
        employeeId,
        clockIn: now,
        clockOut: null,
        latitude: location?.latitude,
        longitude: location?.longitude,
        imageUrl,
      },
      client
    );
    await client.query('COMMIT');
    return { action: 'clock_in', punchId, time: now };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 3.4 LINE webhook handler (`routes/webhook/line.ts`)

```ts
import express from 'express';
import crypto from 'crypto';
import { pool } from '../../db';
import { handleClockInOut } from '../../services/punch.service';

const router = express.Router();

function verifySignature(body: string, signature: string): boolean {
  const channelSecret = process.env.LINE_CHANNEL_SECRET || '';
  const hash = crypto
    .createHmac('sha256', channelSecret)
    .update(body)
    .digest('base64');
  return hash === signature;
}
