## Implementation Plan (≤2h)

**Highest-value incremental improvement**: Embed a CDN-only file manifest into Surrogate-1 training so Lightning Studio runs with **zero HF API calls during data loading**, eliminating 429s and quota risk.

**Why this wins**:  
- Directly applies the HF CDN bypass pattern (THE KEY INSIGHT 2026-04-29)  
- Single-file change with immediate payoff (no infra, no retraining)  
- Fits in <2h: manifest generation + lightweight dataloader swap

### Steps

1. **Generate manifest once on Mac** (or any dev box) after rate-limit window clears  
   - Use `list_repo_tree(path, recursive=False)` per date folder under the surrogate-1 dataset repo  
   - Save `train_files.json` listing only `*.parquet` paths (no dirs, no non-parquet)  
   - Commit `train_files.json` into repo (or bake into docker image)

2. **Add CDN-only dataloader** in `surrogate/training/data.py`  
   - Read `train_files.json`  
   - Build URLs: `f"https://huggingface.co/datasets/{repo}/resolve/main/{path}"`  
   - Stream parquet via `pyarrow.parquet.ParquetDataset` or `polars.scan_parquet` (zero HF API)  
   - Project to `{prompt, response}` at parse time (schema hygiene per surrogate-1 pattern)

3. **Update training script** (`train.py` or Lightning `DataModule`)  
   - Replace `load_dataset(...)` with new `CdnParquetDataModule`  
   - Keep shuffle/batching semantics identical

4. **Optional: studio reuse guard**  
   - Before `.run()`, check `Teamspace.studios` for existing running studio and reuse (saves quota)

### Code snippets

#### surrogate/training/data.py
```python
import json
from pathlib import Path
from typing import List

import polars as pl
import pyarrow.parquet as pq
import requests
from torch.utils.data import IterableDataset

REPO = "org/surrogate-1-dataset"   # adjust
BASE_URL = f"https://huggingface.co/datasets/{REPO}/resolve/main"


def load_manifest(manifest_path: str = "train_files.json") -> List[str]:
    with open(manifest_path) as f:
        return json.load(f)


class CdnParquetIterable(IterableDataset):
    """
    CDN-only parquet loader.
    Each parquet file must contain at least {prompt, response} columns.
    """

    def __init__(self, file_paths: List[str], columns=("prompt", "response")):
        self.file_paths = file_paths
        self.columns = columns

    def __iter__(self):
        for rel_path in self.file_paths:
            url = f"{BASE_URL}/{rel_path}"
            # streaming-friendly: download to temp file or use requests + pyarrow buffer
            resp = requests.get(url, timeout=60)
            resp.raise_for_status()
            table = pq.read_table(pq.ParquetFile(pq.ParquetBuffer(resp.content)))
            df = table.select(self.columns).to_pandas()
            for _, row in df.iterrows():
                yield {
                    "prompt": row["prompt"],
                    "response": row["response"],
                }


# Polars-based alternative (faster, lower memory)
def cdn_parquet_scan(file_paths: List[str]):
    urls = [f"{BASE_URL}/{p}" for p in file_paths]
    # Polars can scan parquet from HTTP via fsspec; fallback to download if needed
    return pl.concat([pl.read_parquet(u, use_pyarrow=True) for u in urls]).select(
        "prompt", "response"
    )
```

#### Generate manifest (dev/ops script)
```bash
#!/usr/bin/env bash
# generate_manifest.sh
# Run once after HF rate-limit window clears
set -euo pipefail

REPO="org/surrogate-1-dataset"
FOLDER="batches/mirror-merged/2026-05-03"   # adjust date
OUT="train_files.json"

python - <<PY
import json
from huggingface_hub import HfApi
api = HfApi()
tree = api.list_repo_tree(repo_id="$REPO", path="$FOLDER", recursive=False)
files = [t.path for t in tree if t.path.endswith(".parquet")]
with open("$OUT", "w") as f:
    json.dump(files, f, indent=2)
print(f"Wrote {len(files)} files to $OUT")
```
