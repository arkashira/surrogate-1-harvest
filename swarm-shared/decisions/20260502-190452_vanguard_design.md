# vanguard / design

## 1) Diagnosis
- No canonical discovery entrypoint exists to surface top-hub insights (e.g., MOC) before planning — violates established `#knowledge-rag #graph #hub` pattern.
- Missing CDN-bypass file-list strategy for HF datasets; any future training ingestion will immediately hit API rate limits instead of using `resolve/main/` CDN fetches.
- No Lightning Studio reuse guard; iterative training will burn quota by recreating studios instead of reusing running ones.
- No executable discovery entrypoint to prepare context and file lists before planning or training runs.
- No clear projection step for local/mirror data to enforce `{prompt,response}` schema and avoid mixed-schema parquet uploads.

## 2) Proposed change
Create `/opt/axentx/vanguard/discovery/run_discovery.sh` as the single entrypoint that:
- Queries knowledge-rag for top-hub (MOC) and related insights.
- Generates an HF CDN-bypass file list for a given date folder (single API call) and writes `file_list.json`.
- Detects and reuses a running Lightning Studio (saves quota).
- Projects local parquet files in `batches/mirror-merged/{date}/` to `{prompt,response}` only and writes enriched projection to `enriched/` with slug-based filenames.

## 3) Implementation
```bash
#!/usr/bin/env bash
# /opt/axentx/vanguard/discovery/run_discovery.sh
# Purpose: Single-pass discovery — surface top-hub knowledge, prepare HF CDN-bypass file list,
#          reuse running Lightning Studio, project local data.
# Usage:
#   ./run_discovery.sh [--date YYYY-MM-DD] [--hf-repo owner/dataset] [--hub-query "MOC"]
#
# Prereqs:
#   - jq, curl, python3 with huggingface_hf and lightning (pip install lightning huggingface_hf)
#   - HF token exported as HF_TOKEN for list_repo_tree (CDN fetches do not need token)
#   - Write access to ./output/ and ./enriched/

set -euo pipefail
SHELL=/bin/bash

# ---- defaults ----
DATE="${1:-$(date +%Y-%m-%d)}"
HF_REPO="${2:-example/dataset}"
HUB_QUERY="${3:-MOC}"
OUTPUT_DIR="$(cd "$(dirname "$0")/../output" && pwd)"
ENRICHED_DIR="$(cd "$(dirname "$0")/../enriched" && pwd)"
FILE_LIST="${OUTPUT_DIR}/file_list.json"

mkdir -p "${OUTPUT_DIR}" "${ENRICHED_DIR}"

echo "== vanguard discovery =="
echo "Date: ${DATE}"
echo "HF repo: ${HF_REPO}"
echo "Hub query: ${HUB_QUERY}"

# ---- 1) knowledge-rag: top-hub insight ----
echo "[1/4] Querying knowledge-rag for top-hub insights..."
# If you have a local knowledge-rag CLI or API, replace this stub with the real call.
# Example stub that writes a simple insight file:
cat > "${OUTPUT_DIR}/top_hub_insight.md" <<EOF
# Top-hub insight (${HUB_QUERY})
- Review most-connected hub "${HUB_QUERY}" before planning tasks.
- Use contextual links from this hub to inform scope and dependencies.
- Tags: #knowledge-rag #graph #hub
EOF
echo "  -> saved to ${OUTPUT_DIR}/top_hub_insight.md"

# ---- 2) HF CDN-bypass file list ----
echo "[2/4] Generating HF CDN-bypass file list for ${DATE}..."
python3 - <<PY > "${FILE_LIST}"
import os, json, sys
from huggingface_hub import HfApi

HF_REPO = os.getenv("HF_REPO", "${HF_REPO}")
DATE = os.getenv("DATE", "${DATE}")
api = HfApi()

# Single API call: non-recursive per date folder to avoid pagination/rate-limit
items = api.list_repo_tree(repo_id=HF_REPO, path=DATE, recursive=False)
files = []
for item in items:
    if item.type == "file":
        # CDN URL (no Authorization header required)
        cdn_url = f"https://huggingface.co/datasets/{HF_REPO}/resolve/main/{DATE}/{item.path}"
        files.append({
            "path": f"{DATE}/{item.path}",
            "cdn_url": cdn_url,
            "size": getattr(item, "size", None)
        })

print(json.dumps({"date": DATE, "repo": HF_REPO, "files": files}, indent=2))
PY
echo "  -> saved to ${FILE_LIST}"

# ---- 3) Lightning Studio reuse ----
echo "[3/4] Checking for running Lightning Studio to reuse..."
python3 - <<PY
import os
from lightning import Studio, Teamspace, Machine

# Try to find a running studio with expected naming pattern
teamspace = Teamspace()
candidates = [s for s in teamspace.studios if s.name.startswith("vanguard-train-") and s.status == "Running"]

if candidates:
    studio = candidates[0]
    print(f"Reusing running studio: {studio.name} (id={studio.id})")
else:
    print("No running vanguard studio found. Create one with Lightning SDK when needed.")
    # Example creation (commented):
    # studio = Studio.create(
    #     name=f"vanguard-train-{os.getenv('DATE', 'default')}",
    #     machine=Machine.L40S,
    #     create_ok=True
    # )
PY
echo "  -> studio check complete"

# ---- 4) Project local mirror data to {prompt,response} ----
echo "[4/4] Projecting local mirror data to {prompt,response} schema..."
MIRROR_DIR="$(cd "$(dirname "$0")/../../batches/mirror-merged/${DATE}" && pwd 2>/dev/null || echo "")"
if [ -d "${MIRROR_DIR}" ]; then
  python3 - <<PY
import os, glob, pandas as pd

mirror_dir = os.getenv("MIRROR_DIR", "${MIRROR_DIR}")
enriched_dir = os.getenv("ENRICHED_DIR", "${ENRICHED_DIR}")
os.makedirs(enriched_dir, exist_ok=True)

for f in glob.glob(os.path.join(mirror_dir, "*.parquet")):
    df = pd.read_parquet(f)
    # Keep only prompt/response; drop other columns to avoid mixed schema
    proj = df[["prompt", "response"]].copy()
    # Slug from filename (safe for repo naming)
    slug = os.path.splitext(os.path.basename(f))[0]
    out_path = os.path.join(enriched_dir, f"{slug}.parquet")
    proj.to_parquet(out_path, index=False)
    print(f"Projected {f} -> {out_path}")
PY
  echo "  -> projections written to ${ENRICHED_DIR}"
else
  echo "  -> no mirror directory at ${MIRROR_DIR}; skipping projection"
fi

echo "== discovery complete =="
```

Make executable:
```bash
chmod +x /opt/axentx/vanguard/discovery/run_discovery.sh
```

## 4) Verification
- Run the script: `cd /opt/axentx/vanguard/discovery && ./run_discovery.sh --date 2026-05-01 --hf-repo example/dataset`
- Confirm outputs:
  - `output/top_hub_insight.md` exists and mentions the hub (MOC).
  - `output/file_list.json` is valid JSON with `cdn_url` entries for the date folder.
  - Script reports a running Lightning Studio or indicates none found (no crash).
  - If mirror parquet files exist, `enriched/*.parquet` contain only `prompt` and `response` columns.
- Quick CDN-bypass test: pick one `cdn_url` from file_list.json and `curl -I <url>` — should return 200 without Authorization header.
