## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres + managed functions) + Redis via Upstash or HF Spaces (free-tier Redis)

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk→execution) with Postgres-backed immutable audit, Redis Streams event lane, one broker connector (sandbox), compliance guardrails as versioned JSON rules, edge-deployed FastAPI on Cloudflare Workers (free-tier), and replay endpoint to re-run any run by inputs_hash.

## Open questions
- Determinism boundaries: which steps must be pure and fully replayable vs. allowed to call external non-deterministic APIs?
- Latency SLA per lane (research→signal→risk→execution) and acceptable queueing/backpressure strategy on free-tier Redis/Workers?
- Compliance rule engine choice: sandboxed Python eval vs. declarative JSON rules vs. WASM; how to version and hot-reload without downtime?
- Idempotency and exactly-once vs. at-least-once semantics for execution lane given free-tier constraints?
- Secrets management for broker/OMS credentials on free-tier (Workers KV vs. Supabase vault vs. env vars) and rotation cadence?
