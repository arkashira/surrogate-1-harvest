# workio / backend

## Final Synthesis — One Correct, Actionable Solution

**Core diagnosis (merged, de-duplicated):**
- Duplicate punches come from non-idempotent LINE webhook retries + non-atomic upserts under concurrency.
- Missing tenant-scoped uniqueness and missing persistence-layer enforcement allow races and overlapping punches.
- Timezone-naive date grouping causes punches to be assigned to the wrong local date (especially across midnight).
- No durable idempotency record means replay protection is best-effort in app code.

**Chosen approach (resolve contradictions in favor of correctness + concrete actionability):**
- Use **database-enforced uniqueness** (tenant-scoped) and **atomic upsert** to eliminate races.
- Add an **idempotency key column + unique index** so retries are short-circuited at the DB level.
- Compute uniqueness by **tenant-local date** (not UTC date) to avoid midnight edge cases.
- Keep implementation simple and verifiable: one migration, one service function, one webhook handler change.

### 1. DB schema changes (migration)

File: `/opt/axentx/workio/server/src/db/schema.sql`

```sql
-- Ensure tenant_id is present and populated
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS tenant_id BIGINT NOT NULL DEFAULT 0;

-- Backfill tenant_id from employees (run once)
UPDATE punches p
SET tenant_id = e.tenant_id
FROM employees e
WHERE p.employee_id = e.id
  AND p.tenant_id = 0;

-- Idempotency support
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255);

-- Unique constraint for idempotency (when provided)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_tenant_idempotency
  ON punches (tenant_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL AND deleted_at IS NULL;

-- Tenant-local-date uniqueness: one punch_type per employee per local date
-- Use generated column for tenant-local date to make constraint/index reliable
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS tenant_local_date DATE
    GENERATED ALWAYS AS (
      (created_at AT TIME ZONE COALESCE(
        (SELECT timezone FROM tenants WHERE id = tenant_id),
        'UTC'
      ))::DATE
    ) STORED;

CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_tenant_emp_localdate_type
  ON punches (tenant_id, employee_id, tenant_local_date, punch_type)
  WHERE deleted_at IS NULL;
```

Notes:
- If your DB does not support expression indexes with subqueries reliably, manage timezone in app code and pass the tenant-local date explicitly in the upsert (see service code below). The index above is preferred when supported; otherwise create the index on `(tenant_id, employee_id, punch_type, deleted_at)` and enforce uniqueness in the upsert logic.

### 2. Service: atomic upsert with idempotency and tenant-local date

File: `/opt/axentx/workio/server/src/services/punchService.ts`

```ts
import { pool } from '../db';

export type PunchInput = {
  tenantId: number;
  employeeId: number;
  punchType: 'IN' | 'OUT';
  latitude?: number;
  longitude?: number;
  idempotencyKey?: string; // LINE deliveryId or client token
  timestamp?: Date; // optional; defaults to now()
};

export async function upsertPunch(input: PunchInput) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency short-circuit (DB-level)
    if (input.idempotencyKey) {
      const found = await client.query(
        `SELECT id FROM punches
         WHERE tenant_id = $1 AND idempotency_key = $2 AND deleted_at IS NULL
         LIMIT 1`,
        [input.tenantId, input.idempotencyKey]
      );
      if (found.rows.length > 0) {
        await client.query('ROLLBACK');
        return found.rows[0];
      }
    }

    // 2) Resolve tenant timezone for local-date grouping
    const tzRes = await client.query(
      `SELECT timezone FROM tenants WHERE id = $1`,
      [input.tenantId]
    );
    const tz = tzRes.rows[0]?.timezone || 'Asia/Bangkok';
    const now = input.timestamp || new Date();

    // 3) Atomic upsert using tenant-local-date grouping
    // We compute local date in SQL to avoid races between read/write.
    // If your DB cannot use subquery in index/constraint, this upsert still
    // enforces uniqueness by checking existing row in the same transaction.
    const upsert = await client.query(
      `WITH target_date AS (
         SELECT ($1::timestamptz AT TIME ZONE $2)::date AS local_date
       ), existing AS (
         SELECT id FROM punches
         WHERE tenant_id = $3
           AND employee_id = $4
           AND punch_type = $5
           AND deleted_at IS NULL
           AND (created_at AT TIME ZONE $2)::date = (SELECT local_date FROM target_date)
         FOR UPDATE
       )
       INSERT INTO punches (tenant_id, employee_id, punch_type, latitude, longitude, idempotency_key, created_at, updated_at)
       SELECT $3, $4, $5, $6, $7, $8, $1, $1
       WHERE NOT EXISTS (SELECT 1 FROM existing)
       ON CONFLICT (tenant_id, employee_id, tenant_local_date, punch_type)
       WHERE deleted_at IS NULL
       DO UPDATE SET
         latitude = COALESCE(punches.latitude, EXCLUDED.latitude),
         longitude = COALESCE(punches.longitude, EXCLUDED.longitude),
         idempotency_key = COALESCE(punches.idempotency_key, EXCLUDED.idempotency_key),
         updated_at = EXCLUDED.updated_at,
         deleted_at = NULL
       RETURNING *`,
      [
        now,
        tz,
        input.tenantId,
        input.employeeId,
        input.punchType,
        input.latitude,
        input.longitude,
        input.idempotencyKey,
      ]
    );

    await client.query('COMMIT');
    return upsert.rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

Why this is correct + actionable:
- Uses a single transaction with `FOR UPDATE` check + unique index to prevent races.
- Idempotency key is checked first and stored, so retries are harmless.
- Tenant-local date is computed in SQL using tenant timezone to avoid midnight grouping errors.
- Falls back to index-based conflict resolution for safety.

### 3. Webhook handler: accept and propagate idempotency key

File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts`

```ts
import express from 'express';
import { upsertPunch } from '../services/punchService';

const router = express.Router();

router.post('/webhook/line', async (req, res) => {
  try {
    const events = req.body.events || [];
    for (const ev of events) {
      if (ev.type === 'message' && ev.message?.type === 'text') {
        const tenantId = Number(req.headers['x-tenant-id'] || 1);
        const employeeId = Number(ev.source?.userId); // map via your user table
        const text = ev.message.text.trim().toUpperCase();

        if (text === 'IN' || text === 'OUT') {
          await upsertPunch({
            tenantId,
            employeeId,
            punchType: text,
            idempotencyKey: ev.deliveryId || ev.message?.id || `${ev.timestamp}-${ev.source?.userId}`,
          });
        }
      }
    }
    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('LINE webhook error', err);
    res.status(500).json({ error: 'internal' });
  }
});

export default router;
```
