## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployed on CF Workers + Supabase backend that connects to one AWS account and one GCP project, lists all schedules, detects duplicates (identical cron on different functions), detects orphaned schedules (no triggers + zero invocations in 30 days), and exposes a GET /findings endpoint returning actionable JSON.

## Open questions
- How will customers grant read-only access to Lambda and Cloud Functions (IAM role vs service account keys)?
- What is the minimal safe scan cadence (hourly/daily) to balance freshness and rate limits?
- Should findings be surfaced via API, email, or webhook (Slack/Teams) in v0.1?
- How do we define 'orphaned' for each platform (no triggers, no subscribers, or zero recent invocations)?
