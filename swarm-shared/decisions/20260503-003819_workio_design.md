# workio / design

**Final synthesized solution**  
Combines the strongest, most actionable parts of both proposals; resolves contradictions in favor of correctness and concrete actionability.

---

## 1. Diagnosis (merged + clarified)

- **Missing idempotency** on `/webhook/line` allows LINE retries (network blips, 5xx, slow client) to create duplicate punch records.
- **No partial unique index** enforcing “one open punch per user” (`clock_out_at IS NULL`) permits concurrent/retried clock-ins to corrupt state.
- **Non-transactional write path** (read-then-insert) creates a race window where two requests both see “no open punch” and insert two open rows.
- **No de-duplication window or idempotency table** means retries re-run full validation/insert instead of short-circuiting.
- **Schema ambiguity**: multiple `NULL` `clock_out_at` for same `user_id` makes “current punch” unreliable.

---

## 2. Proposed change (merged scope)

Files:
- `workio/server/src/db/schema.sql` — partial unique index + idempotency table.
- `workio/server/src/db/operations/punch.ts` — transactional upsert helper for open punch.
- `workio/server/src/routes/webhook/line.ts` — idempotency check + transactional event processing.

Principles:
- Use **LINE `event.id`** as idempotency key when present; otherwise deterministic hash (`source.userId:timestamp:type`).
- Wrap each event’s punch logic in a **transaction** that checks idempotency and enforces the partial unique index.
- Resolve conflicts with `ON CONFLICT DO NOTHING` (insert) and `SKIP LOCKED` (update) to avoid hard failures under concurrency.
- Return **200 for already-processed events** (idempotent success) and only fail on real errors.

---

## 3. Implementation

### 3.1 DB schema (`server/src/db/schema.sql`)

```sql
-- At most one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- Idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS webhook_line_idempotency (
  event_id      TEXT PRIMARY KEY,
  processed_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload_hash  BYTEA,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional cleanup index
CREATE INDEX IF NOT EXISTS idx_webhook_line_idempotency_created
ON webhook_line_idempotency (created_at);
```

### 3.2 Idempotency helper (`server/src/db/idempotency.ts`)

```ts
import { PoolClient } from 'pg';

export async function ensureWebhookProcessed(
  client: PoolClient,
  eventId: string,
  payloadHash?: Buffer
): Promise<boolean> {
  const res = await client.query(
    `INSERT INTO webhook_line_idempotency (event_id, payload_hash)
     VALUES ($1, $2)
     ON CONFLICT (event_id) DO NOTHING
     RETURNING event_id`,
    [eventId, payloadHash]
  );
  return res.rowCount === 1;
}
```

### 3.3 Punch operations helper (`server/src/db/operations/punch.ts`)

```ts
import { PoolClient } from 'pg';

export async function upsertOpenPunch(
  client: PoolClient,
  {
    userId,
    clockInAt,
    clockInLat,
    clockInLng,
    lineEventId,
  }: {
    userId: string;
    clockInAt?: Date;
    clockInLat: number | null;
    clockInLng: number | null;
    lineEventId: string | null;
  }
) {
  // Try insert; if conflict on partial unique index, treat as already open
  const insertRes = await client.query(
    `INSERT INTO punches (user_id, clock_in_at, clock_in_latitude, clock_in_longitude, line_event_id)
     VALUES ($1, $2, $3, $4, $5)
     ON CONFLICT (user_id) WHERE clock_out_at IS NULL DO NOTHING
     RETURNING id`,
    [userId, clockInAt || new Date(), clockInLat, clockInLng, lineEventId]
  );

  // If insert succeeded, return
  if (insertRes.rowCount === 1) {
    return { id: insertRes.rows[0].id, inserted: true };
  }

  // Otherwise, an open punch already exists; fetch it safely
  const selectRes = await client.query(
    `SELECT id FROM punches
     WHERE user_id = $1 AND clock_out_at IS NULL
     ORDER BY clock_in_at DESC
     LIMIT 1`,
    [userId]
  );

  if (selectRes.rowCount === 0) {
    // Should not happen under normal conditions; throw to surface inconsistency
    throw new Error('No open punch found after conflict; state inconsistency');
  }
  return { id: selectRes.rows[0].id, inserted: false };
}

export async function closeOpenPunch(
  client: PoolClient,
  {
    userId,
    clockOutAt,
    clockOutLat,
    clockOutLng,
    lineEventId,
  }: {
    userId: string;
    clockOutAt?: Date;
    clockOutLat: number | null;
    clockOutLng: number | null;
    lineEventId: string | null;
  }
) {
  const res = await client.query(
    `UPDATE punches
     SET clock_out_at = $2,
         clock_out_latitude = $3,
         clock_out_longitude = $4,
         line_event_id = COALESCE(line_event_id, $5)
     WHERE id = (
       SELECT id FROM punches
       WHERE user_id = $1 AND clock_out_at IS NULL
       ORDER BY clock_in_at DESC
       LIMIT 1
       FOR UPDATE SKIP LOCKED
     )
     RETURNING id`,
    [userId, clockOutAt || new Date(), clockOutLat, clockOutLng, lineEventId]
  );

  return { updated: res.rowCount === 1, id: res.rows[0]?.id ?? null };
}
```
