## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: Android app (no root) that establishes a userspace-obfs session via a CF Worker broker; local SOCKS5 facade on localhost with port cloaking; config bundle from Supabase; basic concealment against local port/iface scans; end-to-end encrypted session with noise_ik; telemetry-limited WAUS/U signal; manual QA test matrix covering scan simulation and latency/throughput targets.

## Open questions
- Which interface-spoofing and port-cloaking techniques reliably evade local port scans and traffic fingerprinting on stock Android without root or Knox?
- Can we meet throughput/latency targets using only userspace TCP/UDP (smoltcp/boringtun) on mid-tier Android devices?
- What is the minimal viable broker topology (CF Workers only vs. lightweight bridge VMs) that remains free-tier friendly while avoiding fixed IP exposure?
- How will we distribute and rotate obfs/config bundles to avoid static endpoint blocking while preserving WAUS/U telemetry accuracy?
- What telemetry can we collect for WAUS/U without risking user deanonymization under active inspection?
