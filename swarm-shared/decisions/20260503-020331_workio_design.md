# workio / design

## Final synthesized solution

**Core problem**: LINE retries can deliver the same webhook multiple times; the handler is non-atomic and lacks DB-level uniqueness, so duplicate punches, leave/OT records, and side-effects (notifications, balance updates) can occur.

**Chosen approach**: One atomic, DB-centric idempotency layer with server-side timestamps and safe uniqueness constraints. Favor correctness and concrete actionability: do the idempotency check and punch insert in a single atomic transaction, enforce uniqueness at the DB level, and use authoritative server time for business records.

---

### 1. DB schema changes (PostgreSQL)

```sql
-- Lightweight idempotency table for LINE events
CREATE TABLE IF NOT EXISTS line_event_idempotency (
  line_event_id      VARCHAR(255) PRIMARY KEY,
  event_type         VARCHAR(50)  NOT NULL,
  user_id            INTEGER      NOT NULL,
  tenant_id          INTEGER      NOT NULL,
  created_at         TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  processed_at       TIMESTAMPTZ  NULL,
  payload_hash       VARCHAR(64)  NULL
);

-- Prevent duplicate punches for same user within same second
-- (Adjust if sub-second precision is required; this prevents obvious duplicates.)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_user_time_unique
  ON punches (user_id, DATE_TRUNC('second', punched_at));

-- Optional: add line_event_id to punches for traceability
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS line_event_id VARCHAR(255),
  ADD COLUMN IF NOT EXISTS received_at  TIMESTAMPTZ NOT NULL DEFAULT NOW();
```

- `line_event_id` is the single source-of-truth key for idempotency.
- The unique index on `punches` prevents business-level duplicates even if application logic is bypassed.
- `received_at` stores authoritative server time; keep any client-supplied time in an audit column if needed.

### 2. Webhook handler (Express + PostgreSQL)

Key principles:
- Verify signature using raw body.
- Use a single DB transaction per webhook batch.
- Insert into `line_event_idempotency` with `ON CONFLICT DO NOTHING`; skip already-processed events.
- Insert punches with `ON CONFLICT DO NOTHING` (or upsert only when business logic requires).
- Use server time (`NOW()`) for `punched_at`/`received_at`; never trust client timestamps for business records.
- Commit only after all side-effects (notifications, balance updates) succeed; otherwise rollback.

```ts
// server/src/routes/line/webhook.ts
import { Router } from 'express';
import { pool } from '../../db';
import crypto from 'crypto';

const router = Router();

function verifySignature(body: Buffer, signature: string): boolean {
  const channelSecret = process.env.LINE_CHANNEL_SECRET!;
  const hash = crypto
    .createHmac('sha256', channelSecret)
    .update(body)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(hash), Buffer.from(signature));
}

async function resolveUserId(lineUserId: string, client: any): Promise<number> {
  const r = await client.query('SELECT id FROM users WHERE line_user_id = $1', [lineUserId]);
  if (!r.rows[0]) throw new Error('User not found');
  return r.rows[0].id;
}

async function resolveTenantId(lineUserId: string, client: any): Promise<number> {
  const r = await client.query(
    'SELECT tenant_id FROM users WHERE line_user_id = $1',
    [lineUserId]
  );
  if (!r.rows[0]) throw new Error('Tenant not found');
  return r.rows[0].tenant_id;
}

router.post('/line/webhook', async (req, res) => {
  const signature = req.get('X-Line-Signature');
  // NOTE: use raw-body middleware for this route (see below)
  const rawBody = (req as any).rawBody as Buffer;
  const body = req.body;

  if (!signature || !rawBody || !verifySignature(rawBody, signature)) {
    return res.status(401).send('Invalid signature');
  }

  const events = body.events || [];
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    for (const ev of events) {
      // Deterministic, stable id for idempotency (use LINE's event id if available)
      const lineEventId = ev.source?.userId && ev.timestamp
        ? `line:${ev.source.userId}:${ev.timestamp}:${ev.type}:${ev.message?.id || ''}`
        : `line:${ev.timestamp}:${ev.type}:${crypto.randomUUID()}`;

      // 1) Atomic idempotency insert
      const idem = await client.query(
        `INSERT INTO line_event_idempotency (line_event_id, event_type, user_id, tenant_id, payload_hash, processed_at)
         VALUES ($1, $2, $3, $4, $5, NOW())
         ON CONFLICT (line_event_id) DO NOTHING
         RETURNING *`,
        [
          lineEventId,
          ev.type,
          ev.source?.userId ? await resolveUserId(ev.source.userId, client) : null,
          ev.source?.userId ? await resolveTenantId(ev.source.userId, client) : null,
          crypto.createHash('sha256').update(JSON.stringify(ev)).digest('hex'),
        ]
      );

      // Skip duplicate event
      if (idem.rowCount === 0) continue;

      // 2) Process only new events
      if (ev.type === 'message' && ev.message?.type === 'text') {
        const text = ev.message.text.toLowerCase();
        const userId = await resolveUserId(ev.source.userId, client);
        const tenantId = await resolveTenantId(ev.source.userId, client);
        const now = new Date(); // authoritative server time; can also use `NOW()` in SQL

        if (text.includes('เข้างาน') || text.includes('clock in')) {
          await client.query(
            `INSERT INTO punches (user_id, tenant_id, punched_at, direction, received_at, line_event_id)
             VALUES ($1, $2, $3, $4, $5, $6)
             ON CONFLICT DO NOTHING`,
            [userId, tenantId, now, 'in', now, lineEventId]
          );
        } else if (text.includes('ออกงาน') || text.includes('clock out')) {
          await client.query(
            `INSERT INTO punches (user_id, tenant_id, punched_at, direction, received_at, line_event_id)
             VALUES ($1, $2, $3, $4, $5, $6)
             ON CONFLICT DO NOTHING`,
            [userId, tenantId, now, 'out', now, lineEventId]
          );
        }

        // Add leave/OT parsing here with same idempotency pattern
      }
    }

    await client.query('COMMIT');
    res.status(200).send('OK');
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).send('Processing error');
  } finally {
    client.release();
  }
});

export default router;
```

### 3. Middleware: preserve raw body for signature verification

Signature verification requires the exact raw payload. Use `express.raw` for this route only:

```ts
// app.ts or server entry
import express from 'express';
import lineWebhookRouter from './routes/line/webhook';

const app = express();

// Use raw body for LINE webhook to verify signature
app.use('/webhook/line', express.raw({ type: 'application/json' }), lineWebhookRouter);

// Other JSON routes can use normal json middleware
app.use(express.json());
```
