# workio / frontend

## Final Synthesized Solution

I have merged the strongest, non-contradictory elements from both proposals into a single, production-ready implementation. 

**Key Synthesis Decisions:**
1. **Component Location:** Merged into `workio/src/features/clock/ClockButton.tsx` (aligns with Candidate 1's scalable feature-slice structure, while adopting Candidate 2's stricter UX rules).
2. **Idempotency Strategy:** Uses Candidate 1's `Idempotency-Key` header (standard for backend de-duplication), but generates it using Candidate 2's **action-specific** logic (`clock-in` vs `clock-out` buckets) to prevent cross-action collisions.
3. **UX/State Management:** Combines Candidate 1's `useClock` hook (for API abstraction) with Candidate 2's strict rules: disabling the button during flight, and **not** optimistically toggling state on click (prevents UI/Backend desync). State only updates on confirmed success.
5. **GPS Handling:** Uses Candidate 1's graceful fallback (continues without GPS) but adds Candidate 2's explicit inline status messaging so users know *why* the action is failing or pending.

### 1. API Layer (`workio/src/api/clock.ts`)
*Kept identical to Candidate 1. Clean, typed, and supports the standard `Idempotency-Key` header.*

```ts
// workio/src/api/clock.ts
import axios from 'axios';

const api = axios.create({
  baseURL: '/api',
  timeout: 15000,
});

export type ClockRequest = {
  latitude?: number;
  longitude?: number;
  accuracy?: number;
};

export async function clockIn(payload: ClockRequest, idempotencyKey?: string) {
  return api.post<{ punchId: string; time: string; status: 'in' }>(
    '/clock/in',
    payload,
    {
      headers: idempotencyKey
        ? { 'Idempotency-Key': idempotencyKey }
        : undefined,
    }
  );
}

export async function clockOut(payload: ClockRequest, idempotencyKey?: string) {
  return api.post<{ punchId: string; time: string; status: 'out' }>(
    '/clock/out',
    payload,
    {
      headers: idempotencyKey
        ? { 'Idempotency-Key': idempotencyKey }
        : undefined,
    }
  );
}
```

### 2. Clock Hook (`workio/src/features/clock/useClock.ts`)
*Enhanced with Candidate 2's "idle" reset capability and strict error isolation to prevent stale error states.*

```ts
// workio/src/features/clock/useClock.ts
import { useState, useCallback } from 'react';
import { clockIn, clockOut, type ClockRequest } from '../../api/clock';

type PunchState = 'idle' | 'pending' | 'success' | 'error';

export function useClock() {
  const [state, setState] = useState<PunchState>('idle');
  const [lastError, setLastError] = useState<string | null>(null);
  const [lastId, setLastId] = useState<string | null>(null);

  const perform = useCallback(
    async (
      action: 'in' | 'out',
      payload: ClockRequest,
      idempotencyKey: string
    ) => {
      setState('pending');
      setLastError(null);

      try {
        const res =
          action === 'in'
            ? await clockIn(payload, idempotencyKey)
            : await clockOut(payload, idempotencyKey);
        setState('success');
        setLastId(res.data.punchId);
        return res.data;
      } catch (err: any) {
        setState('error');
        setLastError(err?.message || 'Network or server error.');
        throw err;
      }
    },
    []
  );

  const reset = useCallback(() => {
    setState('idle');
    setLastError(null);
  }, []);

  return { state, lastError, lastId, perform, reset };
}
```

/**
 * Generates a strict, action-specific idempotency key.
 * Prevents "Clock In" retries from clashing with "Clock Out" within the same time window.
 * Uses 5-minute buckets to allow safe retries for transient network failures.
 */
function generateIdempotencyKey(userId: string, action: 'in' | 'out', ts: number) {
  const d = new Date(ts);
  const bucket = `${d.getFullYear()}-${d.getMonth() + 1}-${d.getDate()}-${d.getHours()}-${Math.floor(d.getMinutes() / 5)}`;
  return `${userId}-${action}-${bucket}`;
}

 * Wraps browser geolocation in a promise with strict timeout handling.
 * Rejects cleanly if GPS is blocked, unavailable, or times out.
 */
function getCurrentPositionAsync(): Promise<GeolocationPosition> {
  return new Promise((resolve, reject) => {
    if (!navigator.geolocation) {
      reject(new Error('Geolocation is not supported by this browser.'));
      return;
    }
    navigator.geolocation.getCurrentPosition(resolve, reject, {
      enableHighAccuracy: true,
      timeout: 10000,
      maximumAge: 30000,
    });
  });
