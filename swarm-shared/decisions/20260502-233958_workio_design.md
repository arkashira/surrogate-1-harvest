# workio / design

**Most Valuable Next Improvement for Workio Design**

### Diagnosis

- **Missing idempotency key on punch records** → LINE webhook redeliveries and concurrent deliveries create duplicate clock-in/out rows.
- **Non-atomic read-then-insert for punch state** → races between concurrent deliveries can result in inconsistent punch state.
- **Insufficient validation for punch payload** → invalid or missing data can lead to errors or incorrect punch records.
- **Lack of clear punch state transitions** → unclear state transitions can lead to confusion and errors in punch record management.
- **Inadequate error handling for punch record updates** → errors during punch record updates can lead to inconsistent state and errors.

### Proposed Change

**Add punch state machine and idempotency key**

* Update `punch` model to include a unique `idempotency_key` field
* Create a punch state machine to manage punch state transitions
* Implement idempotency key and optimistic concurrency control for punch records

```javascript
// models/punch-state-machine.js
import { PunchState } from './punch-state';

class PunchStateMachine {
  constructor(punch) {
    this.punch = punch;
    this.currentState = PunchState.INITIAL;
  }

  transitionTo(state) {
    this.currentState = state;
    // Update punch record with new state
  }
}

export default PunchStateMachine;
```

**Step 3: Implement idempotency key and optimistic concurrency control**

```javascript
// services/punch-service.js
import { Punch } from './punch';
import { PunchStateMachine } from './punch-state-machine';

class PunchService {
  async createPunch(punchData) {
    const punch = new Punch(punchData);
    const idempotencyKey = generateIdempotencyKey();
    punch.idempotencyKey = idempotencyKey;

    try {
      await punch.save();
      const punchStateMachine = new PunchStateMachine(punch);
      punchStateMachine.transitionTo(PunchState.ACTIVE);
      return punch;
    } catch (error) {
      // Handle error
    }
  }
}

export default PunchService;
```
