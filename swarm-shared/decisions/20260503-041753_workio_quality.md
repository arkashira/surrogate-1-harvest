# workio / quality

## 1. Diagnosis

- **No idempotency guard on LINE webhook ingestion**: duplicate events (LINE retries, user double-taps, network replays) can create duplicate clock-in/out or leave/OT records.
- **Missing request signature verification**: webhook handler does not validate `X-Line-Signature`, allowing spoofed events to be accepted.
- **Race condition on same-day clock transitions**: concurrent or rapid toggles can produce multiple open/closed intervals for the same tenant+employee+date.
- **No structured per-tenant/per-day intent key**: downstream logic lacks a deterministic guard key to deduplicate logical intents (clock-in, clock-out, leave-request, etc.).
- **Graceful shutdown and retry semantics absent**: long-running webhook handler can lose in-flight work on deploy/restart; no retry/backoff for transient LINE API calls.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts` (or equivalent route handler)  
Scope: add idempotency + signature verification + same-day intent locking at the write path.

- Add middleware to verify `X-Line-Signature` using HMAC-SHA256.
- Add idempotency table `line_webhook_events` keyed by `tenant_id`, `external_event_id` (LINE delivery ID or stable hash), and `intent_key` (e.g., `clock-in:2026-05-03:employee-id`).
- Enforce unique constraint on `(tenant_id, intent_key, intent_date)` for attendance intents to prevent duplicates.
- Wrap attendance/leave/OT creation in upsert/transaction with deterministic intent key.
- Add graceful shutdown handling and exponential backoff for outbound LINE API calls.

### 3.2 Signature verification helper

Create `/opt/axentx/workio/server/src/utils/lineSignature.ts`:

```ts
import crypto from 'crypto';

export function verifyLineSignature(
  rawBody: string,
  signatureHeader: string | undefined,
  channelSecret: string
): boolean {
  if (!signatureHeader || !channelSecret) return false;
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody, 'utf8')
    .digest('base64');
  // Use timing-safe compare
  return crypto.timingSafeEqual(Buffer.from(signatureHeader), Buffer.from(expected));
}
```

async function withIdempotency<T>(
  tenantId: string,
  externalEventId: string,
  intentKeyVal: string,
  intentDate: string,
  eventType: string,
  payload: any,
  handler: () => Promise<T>
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    // Check if already processed
    const dup = await client.query(
      `SELECT id FROM line_webhook_events WHERE tenant_id = $1 AND external_event_id = $2`,
      [tenantId, externalEventId]
    );
    if (dup.rows.length > 0) {
      await client.query('ROLLBACK');
      return { alreadyProcessed: true };
    }

    // Prevent duplicate intent on same day
    const intentDup = await client.query(
      `SELECT id FROM line_webhook_events WHERE tenant_id = $1 AND intent_key = $2 AND intent_date = $3`,
      [tenantId, intentKeyVal, intentDate]
    );
    if (intentDup.rows.length > 0) {
      // Record as seen but skip business logic
      await client.query(
        `INSERT INTO line_webhook_events (tenant_id, external_event_id, intent_key, intent_date, event_type, payload, processed_at)
         VALUES ($1,$2,$3,$4,$5,$6,NOW())`,
        [tenantId, externalEventId, intentKeyVal, intentDate, eventType, payload]
      );
      await client.query('COMMIT');
      return { duplicateIntent: true };
    }

    // Execute business logic
    const result = await handler();

    // Record processed event
    await client.query(
      `INSERT INTO line_webhook_events (tenant_id, external_event_id, intent_key, intent_date, event_type, payload, processed_at)
       VALUES ($1,$2,$3,$4,$5,$6,NOW())`,
      [tenantId, externalEventId, intentKeyVal, intentDate, eventType, payload]
    );

    await client.query('COMMIT');
    return { processed: true, result };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
