## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-asset pipeline (e.g., SPY) with research→signal→risk→execution flow; Supabase audit trail; CF Workers orchestrator; simulated broker adapter with idempotent order submission; deterministic replay of pipeline runs via inputs_hash; basic risk guardrails (max position size, max daily loss); latency telemetry and dashboard showing median signal-to-execution <500ms target; zero frontend (API-only) with OpenAPI docs

## Open questions
- Which brokers/data vendors must be supported in v0.1 and what auth/connectivity model (API keys, OAuth, FIX)?
- What deterministic replay granularity is required (event-sourced vs snapshots) to meet compliance forensics?
- What hard risk limits and guardrails are non-negotiable (max order size, max daily loss, kill switches)?
- Can median signal-to-execution <500ms be achieved on free-tier edge runtimes given broker API latencies?
- How will idempotency and exactly-once execution semantics be enforced across retries?
