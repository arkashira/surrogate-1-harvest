## Deployment
cf workers

## v0.1 scope
CLI-only v0.1 in 2 weeks: local repo ingestion (git clone or path), Tree-sitter parsing + semgrep rule scan, severity/category tagging, JSON report output; optional local SQLite cache; no cloud dependency for analysis; Supabase-backed sign-up and license check (CF Workers) for gated download/entitlements; free-tier only.

## Open questions
- Will users allow optional, opt-in encrypted metadata sync to Supabase for cross-device history and billing, or require fully air-gapped local-only mode?
- Which language coverage is required in v0.1 (e.g., Python/JS/TS only vs broad set) given Tree-sitter/semgrep rule availability?
- How will licensing/compliance be handled for vendored models/rules (e.g., semgrep rules, model weights)?
- What compute budget and performance targets for local runs on typical developer laptops (CPU-only, time-to-insight < 2 min for medium repos)?
- How to implement entitlements/quotas locally (tamper-resistant) vs server-side checks for paid plans?
