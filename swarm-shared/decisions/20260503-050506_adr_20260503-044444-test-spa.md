## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on Cloudflare Workers (or local dev) that lists Lambda functions and Cloud Functions, extracts cron/rate triggers, persists schedules to Supabase Postgres, computes duplicates (by hash) and drifts (expression vs actual timing), marks orphaned entries, and exposes a GET /audit endpoint returning counts and affected schedule rows; no scheduler/queue, no UI, manual re-run via CLI.

## Open questions
- Which IAM scopes are minimally required to list schedules/invocations for Lambda and Cloud Functions without granting invoke permissions?
- How will we detect orphaned schedules (e.g., deleted functions) given eventual consistency and cross-account setups?
- What retention window defines 'stale' (no last_run_at in 30/60/90 days) and can it be customer-configurable?
- Will customers prefer read-only auditor or also automated safe-retire actions (and what approval workflow)?
