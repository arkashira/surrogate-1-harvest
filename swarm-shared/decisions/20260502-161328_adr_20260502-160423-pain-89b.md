## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a single-platform (Linux x64) CLI client with randomized local SOCKS5 port, pluggable obfuscated TLS tunnel (mimic HTTPS), optional TUN mode (userspace TCP/IP), client-side kill-switch, encrypted local config, and minimal telemetry to CF Workers + Supabase Postgres to compute WAU-ZB; fail-closed kill-switch enforced; no server component required (uses existing egress endpoints).

## Open questions
- Which obfuscation primitives are safest against current DPI in target regions (e.g., domain-front vs. TLS-in-TLS vs. QUIC masquerade)?
- Can we implement a reliable client-side kill-switch across desktop/mobile platforms without root/admin?
- How will we collect WAU-ZB telemetry without creating user-identifiable fingerprints or metadata leaks?
- What TUN implementation strategy minimizes kernel module dependencies and host detection surface?
- How do we rotate ephemeral ports and obfuscation profiles to avoid stable local artifacts?
