## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research fetch -> signal generate -> risk check -> execution send) with Postgres-backed event sourcing, Redis Streams queue, Redis cache, immutable audit log, one sample broker connector (sandbox), compliance guardrails (allow/block/warn) evaluated inline, and replay CLI to reproduce runs from event_log; deployed on CF Workers with Supabase free tier and self-hosted Redis on free instance.

## Open questions
- Deterministic replay fidelity: should events be canonicalized (sorted keys, fixed numeric precision) to guarantee byte-for-byte replay?
- Latency SLA target for end-to-end orchestration (sub-100ms vs sub-500ms) to choose in-process vs out-of-process guardrails.
- Compliance rule hot-reload semantics: can rules change mid-run and how should inflight runs be handled?
- Broker/OMS/EMS connector authentication model (API keys vs OAuth) and secret storage on free-tier infra.
- Retention and immutable storage strategy for event_log and audit entries (Supabase only or append-only file export).
