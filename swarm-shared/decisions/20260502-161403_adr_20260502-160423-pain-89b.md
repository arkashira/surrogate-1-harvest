## v0.1 scope
v0.1 ships a single-file CLI agent (Linux x86_64) that: creates a local SOCKS5 listener on 127.0.0.1:1080, tunnels traffic through a pre-configured bridge using an obfuscated TLS-like stream (rustls + SNI spoofing), manages a TUN interface for transparent routing, enforces a kill-switch via local firewall rules (nftables/iptables), stores session state in encrypted SQLite, and emits local WAU-ZB counters — all client-side with zero external dependencies and no server infrastructure.

## Open questions
- Which mimicry profiles reliably evade active DPI probes in target regions (e.g., HTTPS vs QUIC vs CDN-like flows)?
- Can we implement robust TUN/tap without root/admin on desktop targets, or do we require a signed driver/helper?
- How will clients discover/rotate bridge endpoints without centralized coordination that can be blocked?
- What local telemetry can we collect for WAU-ZB without risking deanonymization or storage of sensitive logs?
- How do we validate fail-closed behavior across OSes to guarantee no plaintext leaks during stealth-path failures?
