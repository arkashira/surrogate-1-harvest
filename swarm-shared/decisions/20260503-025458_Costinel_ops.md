# Costinel / ops

## Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h)

### What we ship
- A **non-blocking Top-Hub Signal Panel** mounted on the Costinel dashboard (sidebar or top banner area).
- Defaults to hub **MOC** (configurable via `VITE_HUB_NAME`).
- Shows: hub title, short description, top 3 actionable signals, last updated timestamp.
- **CDN-first data fetch**: uses `https://huggingface.co/datasets/axentx/knowledge-rag/resolve/main/hubs/{hubName}/latest.json` to bypass HF API rate limits.
- Graceful fallback to local placeholder if CDN unavailable.
- Zero backend changes; pure frontend addition.

### Why this is highest-value (<2h)
- Applies the **top-hub doc insight** pattern (MOC) directly to ops visibility.
- Uses **CDN bypass** pattern to avoid rate limits and keep dashboard snappy.
- Non-blocking, self-contained, and reuses existing design tokens/components.
- Delivers immediate contextual ops insight without infra work.

#### 1) `src/components/TopHubSignalPanel.vue` (new)
```vue
<template>
  <section v-if="ready || error" class="top-hub-signal-panel" :class="{ 'is-error': error }">
    <header class="panel-header">
      <h3 class="hub-title">{{ hub.title || hub.name || hubName }}</h3>
      <span v-if="hub.description" class="hub-desc">{{ hub.description }}</span>
      <span v-if="hub.updatedAt" class="hub-updated">Updated {{ formatDate(hub.updatedAt) }}</span>
    </header>

    <div v-if="error" class="error-note">
      <span>Unable to load hub signals — using local defaults.</span>
    </div>

    <ul v-if="signals.length" class="signals-list">
      <li v-for="s in signals" :key="s.id" class="signal-item">
        <strong class="signal-title">{{ s.title }}</strong>
        <p v-if="s.description" class="signal-desc">{{ s.description }}</p>
        <div v-if="s.impact || s.action" class="signal-meta">
          <span v-if="s.impact" class="impact">{{ s.impact }}</span>
          <span v-if="s.action" class="action">{{ s.action }}</span>
        </div>
      </li>
    </ul>

    <div v-else class="no-signals">
      No actionable signals available.
    </div>
  </section>
</template>

<script setup>
import { ref, computed, onMounted } from 'vue'

const props = defineProps({
  hubName: { type: String, default: 'MOC' },
  cdnBase: { type: String, default: 'https://huggingface.co/datasets/axentx/knowledge-rag/resolve/main/hubs' }
})

const ready = ref(false)
const error = ref(false)
const hub = ref({})
const signals = ref([])

const fetchFromCDN = async () => {
  const url = `${props.cdnBase}/${encodeURIComponent(props.hubName)}/latest.json`
  try {
    const res = await fetch(url, { cache: 'no-store' })
    if (!res.ok) throw new Error(`CDN ${res.status}`)
    return await res.json()
  } catch {
    return null
  }
}

const load = async () => {
  ready.value = false
  error.value = false
  const data = await fetchFromCDN()

  if (data && data.hub && Array.isArray(data.signals)) {
    hub.value = data.hub
    signals.value = data.signals.slice(0, 3)
  } else {
    // graceful local fallback
    error.value = true
    hub.value = { name: props.hubName, title: `${props.hubName} Hub`, description: 'Cost governance insights' }
    signals.value = [
      { id: 1, title: 'Review idle dev accounts', description: 'Identify dev accounts with low utilization this week.', impact: 'Potential 8–12% savings', action: 'Schedule cleanup' },
      { id: 2, title: 'RI coverage gap (us-east-1)', description: 'Current RI coverage below target for production workloads.', impact: 'Risk of on-demand spend spike', action: 'Evaluate purchase' },
      { id: 3, title: 'Orphaned storage volumes', description: 'Detached volumes older than 14 days found across accounts.', impact: 'Low-cost leakage', action: 'Tag & schedule deletion' }
    ]
  }
  ready.value = true
}

onMounted(load)

const formatDate = (d) => {
  try { return new Date(d).toLocaleDateString(undefined, { month: 'short', day: 'numeric', year: 'numeric' }) } catch { return d }
}
</script>
```
