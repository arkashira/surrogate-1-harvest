# Costinel / ops

## Final Implementation Plan — Costinel `GET /api/v1/cost-anomaly/signal/top-hub`

**Scope & Constraints**  
- Read-only endpoint (Sense + Signal only). No mutations, no side effects.  
- Optional `?for_date=YYYY-MM-DD` (default today).  
- Reuse existing knowledge-graph assets; avoid expensive on-demand graph queries in request path.  
- Fast to ship (<2h) with minimal new code and clear fallback behavior.

### 2) Service + model (preferred)

```python
# app/services/cost_anomaly/top_hub_service.py
from __future__ import annotations

from datetime import date, datetime
from typing import Any, Dict, Optional
from pydantic import BaseModel
import logging

logger = logging.getLogger(__name__)

class TopHubSignal(BaseModel):
    hub: Optional[str]
    score: Optional[float]
    context: Optional[str]
    for_date: Optional[str]      # YYYY-MM-DD
    generated_at: str            # ISO-8601 UTC

class TopHubService:
    """
    Sense + Signal: read-only top-hub extraction from knowledge graph.
    No writes, no side effects.
    """

    def __init__(self, graph_client=None):
        # graph_client must expose: .top_hub(for_date: Optional[str]) -> Dict[str, Any]
        self.graph_client = graph_client

    def get_top_hub(self, for_date: Optional[date] = None) -> TopHubSignal:
        target_date = for_date or date.today()
        for_date_str = target_date.isoformat()
        generated_at = datetime.utcnow().isoformat() + "Z"

        try:
            if self.graph_client is None:
                raise RuntimeError("graph_client not configured")

            result = self.graph_client.top_hub(for_date=for_date_str)
            hub = result.get("hub")
            score = result.get("score")
            context = result.get("context") or result.get("summary") or None

            return TopHubSignal(
                hub=hub,
                score=float(score) if score is not None else None,
                context=context,
                for_date=for_date_str,
                generated_at=generated_at,
            )
        except Exception as exc:
            logger.warning("Top-hub lookup failed for %s: %s", for_date_str, exc, exc_info=True)
            return TopHubSignal(
                hub=None,
                score=None,
                context=f"Unavailable: {exc}",
                for_date=for_date_str,
                generated_at=generated_at,
            )
```

### 3) Route handler (FastAPI)

```python
# app/api/v1/cost_anomaly/signal/top_hub.py
from datetime import date
from fastapi import APIRouter, Query
from pydantic import BaseModel

from services.cost_anomaly.top_hub_service import TopHubService

router = APIRouter(prefix="/api/v1/cost-anomaly/signal", tags=["cost-anomaly-signal"])

# In production, inject a real graph_client via DI or factory
top_hub_service = TopHubService(graph_client=None)

class TopHubResponse(BaseModel):
    ok: bool
    signal: TopHubService.TopHubSignal  # reuse inner model shape
    generated_at: str
    for_date: str

@router.get("/top-hub", response_model=TopHubResponse)
async def get_top_hub_signal(
    for_date: str = Query(
        None,
        description="Date (YYYY-MM-DD); defaults to today",
        regex=r"^\d{4}-\d{2}-\d{2}$",
    ),
):
    try:
        target_date = date.fromisoformat(for_date) if for_date else date.today()
    except ValueError:
        from fastapi.responses import JSONResponse
        return JSONResponse(
            status_code=400,
            content={"error": "Invalid for_date; expected YYYY-MM-DD"},
        )

    signal = top_hub_service.get_top_hub(target_date)
    return {
        "ok": True,
        "signal": signal,
        "generated_at": signal.generated_at,
        "for_date": signal.for_date,
    }
```

### 7) Production notes & next steps

- **Performance**: Precompute daily top-hub indexes (e.g., `batches/top-hub/{date}.json`) and serve via object store or CDN to avoid on-demand graph queries.  
- **Graph client**: Implement `graph_client.top_hub(for_date)` using Neo4j, NetworkX, or cached JSON from `knowledge-rag` outputs.  
- **Resilience**: Service already returns `null` hub + warning context on failure; monitor logs for repeated failures.  
- **Rate limiting**: Apply at gateway/load balancer (not in handler) to protect graph or cache layers.
