# workio / ops

## Final synthesized answer

### 1. Diagnosis (merged)
- **Missing DB-level guardrails**: no partial unique constraint prevents multiple open punches (`clock_out_at IS NULL`) for the same user; concurrent/retried LINE webhooks can insert duplicates.
- **No idempotency**: LINE webhook at-least-once delivery can create duplicate punches without a stable deduplication key.
- **No transactional upsert**: clock-in logic is not atomic, so races between retries can leave two open rows.
- **Missing ops probes**: no `/readyz` (readiness) or `/healthz` (liveness) endpoints for orchestration and automated rollback.
- **No startup schema guard**: container can serve traffic before migrations are applied.

### 2. Proposed change (merged + resolved)
- Add **partial unique index** on `punches(user_id)` where `clock_out_at IS NULL` to enforce one open punch per user (simpler and safer than including `clock_date`).
- Add **idempotency column** `line_webhook_message_id` with a unique index (non-null values only) to deduplicate LINE events at the DB level.
- Implement **transactional upsert** in the clock-in path using `line_webhook_message_id` for idempotency and the partial index to prevent multiple open punches.
- Add **`/healthz`** (liveness) and **`/readyz`** (readiness) endpoints.
- Add **startup schema check** to prevent serving traffic with stale migrations.
- Scope:
  - `server/src/db/schema.sql`
  - `server/src/routes/lineWebhook.ts` (or equivalent handler)
  - `server/src/server.ts` (or main server file)

### 3. Implementation

#### 3.1 DB schema (`server/src/db/schema.sql`)
```sql
-- Idempotency key for LINE webhook events
ALTER TABLE punches ADD COLUMN IF NOT EXISTS line_webhook_message_id TEXT;
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_msg_unique
  ON punches (line_webhook_message_id)
  WHERE line_webhook_message_id IS NOT NULL;

-- One open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
  ON punches (user_id)
  WHERE clock_out_at IS NULL;
```

#### 3.2 Clock-in handler (`server/src/routes/lineWebhook.ts`)
```ts
import { pool } from '../db';
import { v4 as uuidv4 } from 'uuid';

export async function handleClockIn(userId: string, lineMessageId: string | null) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency: return existing punch if same LINE message already processed
    if (lineMessageId) {
      const existing = await client.query(
        `SELECT id FROM punches WHERE line_webhook_message_id = $1`,
        [lineMessageId]
      );
      if (existing.rows.length > 0) {
        await client.query('COMMIT');
        return existing.rows[0];
      }
    }

    const now = new Date();
    const idempotencyKey = lineMessageId || uuidv4();

    // Try insert; partial index prevents second open punch for same user
    const result = await client.query(
      `INSERT INTO punches (user_id, clock_in_at, clock_out_at, line_webhook_message_id)
       VALUES ($1, $2, NULL, $3)
       ON CONFLICT (user_id)
       WHERE clock_out_at IS NULL
       DO NOTHING
       RETURNING *`,
      [userId, now, idempotencyKey]
    );

    // If conflict (open punch exists), return it
    if (result.rowCount === 0) {
      const existing = await client.query(
        `SELECT * FROM punches
         WHERE user_id = $1 AND clock_out_at IS NULL
         ORDER BY clock_in_at DESC LIMIT 1`,
        [userId]
      );
      await client.query('COMMIT');
      return existing.rows[0];
    }

    await client.query('COMMIT');
    return result.rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

In the LINE webhook route, pass `webhookEventId` (or the stable message ID) as `lineMessageId` when calling `handleClockIn`.

#### 3.3 Health endpoints (`server/src/server.ts`)
```ts
import express from 'express';
import { pool } from './db';

const app = express();

// Liveness: process is up
app.get('/healthz', (_req, res) => {
  res.status(200).json({ status: 'ok', uptime: process.uptime() });
});

// Readiness: dependencies are available
app.get('/readyz', async (_req, res) => {
  const checks = {
    db: false,
    lineConfig: false,
  };

  try {
    await pool.query('SELECT 1 FROM punches LIMIT 1');
    checks.db = true;
  } catch {
    checks.db = false;
  }

  checks.lineConfig = Boolean(
    process.env.LINE_CHANNEL_ACCESS_TOKEN && process.env.LINE_CHANNEL_SECRET
  );

  const ok = checks.db === true && checks.lineConfig === true;
  if (ok) {
    res.status(200).json({ status: 'ok', checks });
  } else {
    res.status(503).json({ status: 'unavailable', checks });
  }
});
```

### 4. Verification (merged + concrete)
- Apply migration: `psql <dbname> < server/src/db/schema.sql` and confirm indexes with `\d punches`.
- Idempotency test: send the same LINE webhook payload twice within seconds; verify only one open punch row exists and second request returns the same row.
- Partial constraint test: attempt to insert a second open punch for the same user via SQL; expect unique violation on `idx_punches_one_open_per_user`.
- Readiness test: `curl -f http://localhost:3000/readyz` returns 200 when DB + config are present; returns 503 when DB is down or config missing.
- Liveness test: `curl -f http://localhost:3000/healthz` always returns 200 while the process is alive.
