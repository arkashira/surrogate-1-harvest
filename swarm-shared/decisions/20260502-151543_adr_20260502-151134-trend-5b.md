## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres + edge functions) + HuggingFace Space (optional Redis/worker fallback)

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk→execution) with FastAPI endpoints, Postgres-backed immutable audit log, Redis cache for risk/signal state, Cloudflare Queue-backed replay stream, Supabase-hosted Postgres, OSS deployable on CF Workers + HF Space, pre-built mock connectors for one broker and one data vendor, compliance guardrails as JSON-schema rules, and NSM tracking for weighted successful runs.

## Open questions
- Determinism boundaries: which stages must be pure functions to guarantee replayability vs where external nondeterministic calls (market data) are allowed?
- Latency SLO per stage (research→signal→risk→execution) to qualify sub-second orchestration on free-tier infra?
- Compliance rule language: DSL vs JSON schema vs Python expressions — who owns rule authoring and change control?
- Secret rotation cadence and zero-downtime credential rehydration for broker/OMS connectors?
- Replay storage scope and retention: full event sourcing vs snapshots + diffs for audit?
- Fail-open vs fail-closed semantics for risk guardrails under transient data outages?
