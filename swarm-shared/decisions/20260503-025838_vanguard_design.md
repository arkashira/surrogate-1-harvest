# vanguard / design

## Final Synthesized Answer

### 1. Diagnosis (merged)
- **No build-time deterministic asset manifest** → frontend and training scripts cannot resolve CDN URLs without runtime HF API calls, inviting 429s and breaking CDN-first strategy.
- **Missing mount-point / entrypoint boundary** between orchestration host (Mac) and compute target (Lightning) → no clear contract for which code runs where and how assets are injected; risk of running model code locally.
- **Training ingestion still uses `load_dataset`** (or equivalent) on heterogeneous repos → pyarrow CastError from mixed schema and rate-limit exposure from API pagination.
- **No content-hashed, CDN-only data loader** → training cannot guarantee zero API calls during data loading.
- **No reusable Studio lifecycle guard** → idle timeout kills training; scripts recreate studios instead of reusing running ones, burning quota.
- **No schema projection before upload** — enriched/ folder retains mixed schema (source/ts/extra) which breaks downstream surrogate-1 training with pyarrow CastError.
- **No lightweight verification that CDN assets exist and checksum before training starts** → silent failures and wasted quota.

### 2. Proposed Change (merged)
Create `/opt/axentx/vanguard/design/` with:
- `asset-manifest.json` (generated at build/orchestration time on Mac) — maps logical names to CDN URLs + SHA-256 + size.
- `entrypoint.sh` — boundary script that validates assets on the compute target and launches Lightning training with zero HF API calls.
- `README.md` (contract + usage).

Add `/opt/axentx/vanguard/scripts/build_manifest.py` — single Mac-side script that lists one date folder via HF API (one call per folder), writes deterministic manifest with schema projection (select only training-safe columns).

Add `/opt/axentx/vanguard/train/data_loader.py` — Lightning-compatible dataloader that consumes `manifest.json` and downloads via CDN only, with integrity verification and optional streaming.

Add `/opt/axentx/vanguard/train/launch_studio.py` — guard to reuse running Studio, restart if idle-stopped, and invoke training on remote only.

Add optional `.lightning_studio.yaml` (or reuse existing project config) to declare Studio resource spec and idle timeout handling.

#### `/opt/axentx/vanguard/design/entrypoint.sh` (executable boundary + CDN-only fetch)
```bash
#!/usr/bin/env bash
# vanguard/design/entrypoint.sh
# Boundary between orchestration host (Mac) and compute target (Lightning).
# Validates CDN assets (zero HF API calls) then launches Lightning training.

set -euo pipefail
SHELL=/bin/bash

MANIFEST_PATH="${MANIFEST_PATH:-/opt/axentx/vanguard/design/asset-manifest.json}"
ASSET_KEY="${ASSET_KEY:-surrogate-1-train-2026-05-03}"
MAX_RETRIES="${MAX_RETRIES:-3}"
RETRY_WAIT="${RETRY_WAIT:-5}"

log() {
  echo "[$(date -u +"%Y-%m-%dT%H:%M:%SZ")] $*"
}

fail() {
  log "ERROR: $*"
  exit 1
}

# Require jq for manifest parsing
if ! command -v jq >/dev/null 2>&1; then
  fail "jq is required but not installed."
fi

if [[ ! -f "$MANIFEST_PATH" ]]; then
  fail "Manifest not found: $MANIFEST_PATH"
fi

# Extract file entries from manifest
mapfile -t CDN_URLS < <(jq -r ".assets[\"$ASSET_KEY\"].files[].cdn_url" "$MANIFEST_PATH")
mapfile -t EXPECTED_HASHES < <(jq -r ".assets[\"$ASSET_KEY\"].files[].sha256" "$MANIFEST_PATH")

if [[ "${#CDN_URLS[@]}" -eq 0 ]]; then
  fail "No files found for asset key: $ASSET_KEY"
fi

log "Validating ${#CDN_URLS[@]} CDN assets (zero HF API calls)..."

validate_cdn_asset() {
  local url="$1"
  local expected_hash="$2"
  local tmpfile
  tmpfile="$(mktemp)"
  for ((i=1; i<=MAX_RETRIES; i++)); do
    if curl -fsSL --retry 2 --retry-delay 2 -o "$tmpfile" "$url"; then
      local actual_hash
      actual_hash="$(sha256sum "$tmpfile" | awk '{print $1}')"
      if [[ "$actual_hash" == "$expected_hash" ]]; then
        log "OK: $url"
        rm -f "$tmpfile"
        return 0
      else
        log "WARN: hash mismatch for $url (attempt $i/$MAX_RETRIES)"
      fi
    else
      log "WARN: download failed for $url (attempt $i/$MAX_RETRIES)"
    fi
    rm -f "$tmpfile"
    sleep "$RETRY_WAIT"
  done
  fail "Failed to validate CDN asset after $MAX_RETRIES attempts: $url"
}

for idx in "${!CDN_URLS[@]}"; do
  validate_cdn_asset "${CDN_URLS[$idx]}" "${EXPECTED_HASHES[$idx]}"
done

log "All CDN assets validated. Launching training (CDN-only data path)..."

# Export validated file list for training script (avoids any HF API list/tree calls)
export CDN_FILE_LIST="$(jq -c ".assets[\"$ASSET_KEY\"].files" "$MANIFEST_PATH")"

# Execute training launcher (example). Replace with actual training command.
exec "${@:-python /opt/axentx/vanguard/train/train.py --manifest /opt/axentx/vanguard/design/asset-manifest.json --asset-key $ASSET_KEY}"
```

Make executable:
```bash
chmod +x /opt/axentx/vanguard/design/entrypoint.sh
```
