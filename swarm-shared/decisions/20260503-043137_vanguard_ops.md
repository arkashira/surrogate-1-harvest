# vanguard / ops

## 1. Diagnosis
- No content-addressed manifest per date folder → forces runtime repo re-enumeration and triggers HF API 429s.
- No deterministic `{path, sha256}` snapshot → epochs are non-reproducible; CDN-only fetches cannot be validated.
- Training/ingestion scripts likely call `list_repo_files` or `load_dataset(streaming=True)` on heterogeneous repos → schema CastError and rate-limit exposure.
- Missing Lightning Studio reuse logic → each run recreates studio and burns 80+ quota hours.
- No CDN-bypass strategy baked into training pipeline → every epoch hits HF API auth endpoints.

## 2. Proposed change
Create `/opt/axentx/vanguard/ops/manifest_snapshot.py` (single file) that:
- Accepts `date_folder` (e.g. `2026-04-29`) and HF dataset repo.
- Calls `list_repo_tree(recursive=False)` once, saves `manifest-{date}.json` with `{path, sha256, size}`.
- Produces a companion `train_manifest_loader.py` snippet that Lightning training uses for CDN-only fetches (zero API calls during data load).
- Adds lightweight studio reuse guard (`reuse_running_studio`) to avoid quota burn.

Scope: one new ops script + one small train loader snippet; no changes to existing ingestion/training logic yet.

## 3. Implementation

```bash
# /opt/axentx/vanguard/ops/manifest_snapshot.py
#!/usr/bin/env bash
# Generate content-addressed manifest for a date folder to avoid HF API re-enumeration.
# Usage: ./manifest_snapshot.sh <dataset_repo> <date_folder> [out_dir]
# Example: ./manifest_snapshot.py huggingface/datasets/my-mirror 2026-04-29 ./manifests

set -euo pipefail
SHELL=/bin/bash

REPO="${1:-huggingface/datasets/my-mirror}"
DATE_FOLDER="${2:-$(date +%Y-%m-%d)}"
OUT_DIR="${3:-./manifests}"
MANIFEST="${OUT_DIR}/manifest-${DATE_FOLDER}.json"

mkdir -p "${OUT_DIR}"

python3 - "$REPO" "$DATE_FOLDER" "$MANIFEST" <<'PY'
import json, os, sys, hashlib, subprocess, time
from huggingface_hub import HfApi, hf_hub_download, list_repo_tree

REPO = sys.argv[1]
DATE_FOLDER = sys.argv[2]
MANIFEST = sys.argv[3]

api = HfApi()

# Respect HF rate limits: one tree call per date folder
# If 429 received, wait 360s and retry once
def safe_list_tree():
    try:
        return list(list_repo_tree(REPO, path=DATE_FOLDER, recursive=False))
    except Exception as e:
        if getattr(e, "status_code", None) == 429:
            print("HF 429: waiting 360s", file=sys.stderr)
            time.sleep(360)
            return list(list_repo_tree(REPO, path=DATE_FOLDER, recursive=False))
        raise

entries = safe_list_tree()
files = [e for e in entries if e.type == "file"]

manifest = []
for f in files:
    # Use CDN URL for sha256 check if available; otherwise download once to temp
    url = f"https://huggingface.co/datasets/{REPO}/resolve/main/{f.path}"
    # Lightweight: use ETag/sha256 via HEAD when possible; fallback to local hash after hf_hub_download
    try:
        local_path = hf_hub_download(repo_id=REPO, filename=f.path, repo_type="dataset")
        with open(local_path, "rb") as fh:
            sha256 = hashlib.sha256(fh.read()).hexdigest()
    except Exception:
        sha256 = "unavailable"

    manifest.append({
        "path": f.path,
        "sha256": sha256,
        "size": f.size,
        "cdn_url": url,
        "date_folder": DATE_FOLDER
    })

with open(MANIFEST, "w") as fp:
    json.dump(manifest, fp, indent=2)

print(f"Wrote {len(manifest)} entries to {MANIFEST}")
PY

echo "Manifest generated: ${MANIFEST}"
```

# /opt/axentx/vanguard/ops/train_manifest_loader.py
"""
Lightning-compatible data loader that uses CDN-only fetches based on a pre-generated manifest.
Embed this in your training script to avoid HF API calls during training.
"""
import json
import os
from pathlib import Path
from typing import List, Dict
import torch
from torch.utils.data import IterableDataset
import requests

class CDNManifestDataset(IterableDataset):
    def __init__(self, manifest_path: str, start_idx: int = 0, max_items: int = None):
        super().__init__()
        with open(manifest_path) as f:
            self.items: List[Dict] = json.load(f)
        if max_items:
            self.items = self.items[start_idx:start_idx + max_items]
        else:
            self.items = self.items[start_idx:]

    def __iter__(self):
        worker_info = torch.utils.data.get_worker_info()
        if worker_info is None:
            iter_slice = self.items
        else:
            per_worker = len(self.items) // worker_info.num_workers
            worker_id = worker_info.id
            iter_slice = self.items[worker_id * per_worker:(worker_id + 1) * per_worker]

        for entry in iter_slice:
            url = entry["cdn_url"]
            try:
                resp = requests.get(url, timeout=30)
                resp.raise_for_status()
                # Project to {prompt, response} at parse time (avoids schema issues)
                # Implement your file-type parser here; example for JSONL:
                # for line in resp.text.splitlines():
                #     obj = json.loads(line)
                #     yield {"prompt": obj["prompt"], "response": obj["response"]}
                yield {"raw": resp.content, "path": entry["path"], "cdn_url": url}
            except Exception as exc:
                print(f"CDN fetch failed for {url}: {exc}")
                continue

def reuse_running_studio(name: str, teamspace: str = None):
    """
    Reuse a running Lightning Studio to save quota.
    Returns (studio, created) where created=False if reused.
    """
    from lightning import Studio, Teamspace  # optional import
    try:
        studios = Teamspace(teamspace).studios if teamspace else Studio.list()
        for s in studios:
            if s.name == name and s.status == "running":
                print(f"Reusing running studio: {name}")
                return s, False
    except Exception:
        pass
    return None, False
