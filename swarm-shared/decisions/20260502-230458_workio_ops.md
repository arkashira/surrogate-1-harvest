# workio / ops

## 1) Diagnosis

- **No idempotency key enforcement at DB layer**: clock-in/out endpoints accept duplicate requests for the same `(employee_id, punch_type, date)` when clients retry on flaky mobile networks → double punches and payroll drift.
- **Non-transactional punch writes**: concurrent retries can insert overlapping rows because there is no unique constraint or upsert logic on `(employee_id, punch_type, date)`.
- **No request-level idempotency key header support**: LINE webhook retries and mobile retries cannot safely retry without creating duplicates.
- **Missing conflict resolution for clock-out after clock-in**: multiple clock-out attempts can create extra rows instead of updating the existing open punch.
- **No audit trail for duplicate suppression**: when duplicates are rejected, there is no log or trace for payroll reconciliation.

## 2) Proposed change

- **Scope**: `server/src/db/schema.sql` (add constraint) + `server/src/routes/punch.ts` (add idempotency handling) + `server/src/middleware/idempotency.ts` (new middleware).
- **Goal**: enforce uniqueness at DB level and provide safe retry semantics using `idempotency-key` header (or generate from `(employee_id, punch_type, date)` when absent).

## 3) Implementation

### 3.1 Add unique constraint and idempotency table

```sql
-- server/src/db/schema.sql
-- Add after punches table definition
ALTER TABLE punches
  ADD CONSTRAINT uniq_employee_punch_date
  UNIQUE (employee_id, punch_type, date);

-- Optional: idempotency ledger for traceability
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  employee_id INTEGER NOT NULL,
  punch_type TEXT NOT NULL,
  date DATE NOT NULL,
  punch_id INTEGER NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 3.2 Create idempotency middleware

```ts
// server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { Pool } from 'pg';
import { v4 as uuidv4 } from 'uuid';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function punchIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const employeeId = (req as any).user?.id || req.body.employee_id;
  const punchType = req.body.punch_type;
  const date = req.body.date || new Date().toISOString().split('T')[0];

  if (!employeeId || !punchType) {
    return res.status(400).json({ error: 'Missing employee_id or punch_type' });
  }

  // Prefer client-supplied idempotency-key, otherwise deterministic key
  const clientKey = req.get('Idempotency-Key');
  const idempotencyKey = clientKey || `punch:${employeeId}:${punchType}:${date}`;

  try {
    // Check if already processed
    const { rows } = await pool.query(
      `SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1`,
      [idempotencyKey]
    );

    if (rows.length > 0) {
      // Return existing punch (idempotent replay)
      const punchResult = await pool.query(
        `SELECT * FROM punches WHERE id = $1`,
        [rows[0].punch_id]
      );
      return res.status(200).json({
        message: 'Duplicate request — returning existing punch',
        data: punchResult.rows[0],
        duplicate: true
      });
    }

    // Attach key and metadata for route handler
    (req as any).idempotencyKey = idempotencyKey;
    (req as any).idempotencyMeta = { employeeId, punchType, date };
    next();
  } catch (err) {
    console.error('Idempotency check failed', err);
    // Fail open to avoid blocking punches on infra errors
    next();
  }
}
```

### 3.3 Update punch route to use upsert and record idempotency

```ts
// server/src/routes/punch.ts
import { Router } from 'express';
import { Pool } from 'pg';
import { punchIdempotency } from '../middleware/idempotency';

const router = Router();
const pool = new Pool({ connectionString: process.env.DATABASE_URL });

router.post(
  '/',
  punchIdempotency,
  async (req, res) => {
    const client = await pool.connect();
    const { employeeId, punchType, date } = (req as any).idempotencyMeta || {};
    const idempotencyKey = (req as any).idempotencyKey;
    const location = req.body.location;
    const image = req.body.image;

    try {
      await client.query('BEGIN');

      // Try insert; on conflict do nothing (we'll fetch existing)
      const insertResult = await client.query(
        `INSERT INTO punches (employee_id, punch_type, date, location, image, created_at)
         VALUES ($1, $2, $3, $4, $5, NOW())
         ON CONFLICT (employee_id, punch_type, date) DO NOTHING
         RETURNING *`,
        [employeeId, punchType, date, location, image]
      );

      let punch = insertResult.rows[0];

      if (!punch) {
        // Fetch existing punch
        const selectResult = await client.query(
          `SELECT * FROM punches WHERE employee_id = $1 AND punch_type = $2 AND date = $3`,
          [employeeId, punchType, date]
        );
        punch = selectResult.rows[0];
      }

      // Record idempotency (ignore duplicate key errors)
      try {
        await client.query(
          `INSERT INTO punch_idempotency (idempotency_key, employee_id, punch_type, date, punch_id)
           VALUES ($1, $2, $3, $4, $5)`,
          [idempotencyKey, employeeId, punchType, date, punch.id]
        );
      } catch (idempErr) {
        // Key collision is fine — another request may have recorded it
      }

      await client.query('COMMIT');

      return res.status(200).json({
        message: 'Punch recorded',
        data: punch
      });
    } catch (err) {
      await client.query('ROLLBACK');
      console.error('Punch failed', err);
      return res.status(500).json({ error: 'Internal server error' });
    } finally {
      client.release();
    }
  }
);

export default router;
```
