# workio / backend

## Final Synthesis — Correct, Actionable Solution

### 1. Diagnosis (merged, prioritized)
- **Race + retry duplicates**: LINE retries and concurrent webhooks can create multiple open punches for the same employee/tenant because the app uses `find-then-insert/update` without a DB-level guard.
- **Missing DB-level uniqueness**: no constraint prevents two rows with the same `(tenant_id, employee_id)` and `clock_out_at IS NULL`.
- **No idempotency mechanism**: LINE retries lack a stable deduplication key, making duplicate processing unavoidable under retries or network replays.
- **Nullable `clock_out_at` must be part of uniqueness scope**: only one open punch per `(tenant_id, employee_id)` is valid.
- **Application-only checks are unsafe under load**: correctness must be enforced at the database level; app logic is best-effort and retry-tolerant.

### 2. Proposed Change (single, concrete)
- **Schema file**: `/opt/axentx/workio/server/src/db/schema.sql`
  - Add `idempotency_key TEXT NULL` to `punches`.
  - Create unique partial index for open punches: `(tenant_id, employee_id)` where `clock_out_at IS NULL`.
  - Create unique index on `idempotency_key` where not null.
- **Webhook handler**: `/opt/axentx/workio/server/src/routes/line.ts` (or equivalent)
  - Derive a deterministic idempotency key from `X-Line-Signature` + event fields.
  - Use `ON CONFLICT` and upsert patterns to make inserts/updates safe under concurrency and retries.
  - Treat unique violations as idempotent success; never fail the webhook for duplicates.

### 3. Implementation (merged strongest code + corrections)

#### 3.1 Schema changes (`schema.sql`)
```sql
-- Idempotency support for webhook deduplication
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key TEXT NULL;

CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key
  ON punches (idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- One open punch per employee per tenant (partial unique index)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_employee
  ON punches (tenant_id, employee_id)
  WHERE clock_out_at IS NULL;
```

#### 3.2 Webhook handler (`server/src/routes/line.ts`)
```ts
import crypto from 'crypto';
import { db } from '../db';

function buildIdempotencyKey(lineSignature: string, event: any): string {
  // Deterministic across retries: signature + event type + user + timestamp
  const payload = JSON.stringify({
    sig: lineSignature,
    type: event.type,
    userId: event.source?.userId,
    timestamp: event.timestamp,
    eventId: event.source?.userId && event.timestamp
      ? `${event.source.userId}:${event.timestamp}`
      : event['id'] || crypto.randomUUID(),
  });
  return crypto.createHash('sha256').update(payload).digest('hex');
}

export async function handleLineWebhook(req, res) {
  const lineSignature = (req.headers['x-line-signature'] as string) || '';
  const events = req.body.events || [];

  for (const event of events) {
    if (event.type !== 'message' && event.type !== 'postback') continue;

    const isClockIn =
      event?.message?.text?.trim?.() === 'clock_in' ||
      event?.postback?.data === 'clock_in';
    const isClockOut =
      event?.message?.text?.trim?.() === 'clock_out' ||
      event?.postback?.data === 'clock_out';
    if (!isClockIn && !isClockOut) continue;

    const userId = event.source?.userId;
    if (!userId) continue;

    const idempotencyKey = buildIdempotencyKey(lineSignature, event);

    try {
      await db.tx(async (t) => {
        // Fast path: already processed (idempotency)
        const existing = await t.oneOrNone(
          `SELECT id FROM punches WHERE idempotency_key = $1`,
          [idempotencyKey]
        );
        if (existing) return;

        const emp = await t.oneOrNone(
          `SELECT id, tenant_id FROM employees WHERE line_user_id = $1`,
          [userId]
        );
        if (!emp) throw new Error('Employee not found');

        if (isClockIn) {
          // Try insert; if conflict on open punch, attach idempotency_key to existing open row
          const insertResult = await t.result(
            `INSERT INTO punches (tenant_id, employee_id, clock_in_at, idempotency_key)
             VALUES ($1, $2, NOW(), $3)
             ON CONFLICT (tenant_id, employee_id)
             WHERE clock_out_at IS NULL
             DO NOTHING`,
            [emp.tenant_id, emp.id, idempotencyKey]
          );

          if (insertResult.rowCount === 0) {
            // There is an existing open punch; attach idempotency key if missing
            await t.none(
              `UPDATE punches
               SET idempotency_key = $1
               WHERE tenant_id = $2 AND employee_id = $3 AND clock_out_at IS NULL
                 AND (idempotency_key IS NULL OR idempotency_key != $1)`,
              [idempotencyKey, emp.tenant_id, emp.id]
            );
          }
        } else if (isClockOut) {
          // Close the open punch and mark with idempotency key
          const result = await t.result(
            `UPDATE punches
             SET clock_out_at = NOW(), idempotency_key = $1
             WHERE tenant_id = $2 AND employee_id = $3 AND clock_out_at IS NULL
             RETURNING id`,
            [idempotencyKey, emp.tenant_id, emp.id]
          );

          if (result.rowCount === 0) {
            // No open punch — do not create a clock-out-only record.
            // Optionally log or raise a domain event for reconciliation.
          }
        }
      });
    } catch (err: any) {
      // Unique violation on idempotency_key or partial index => duplicate/race; treat as success
      if (err.code === '23505') {
        continue;
      }
      console.error('Failed to process LINE punch event', err);
    }
  }

  res.status(200).json({ ok: true });
}
```

### 4. Verification (merged, concrete)
1. **Idempotency test**
   - Replay the same LINE webhook payload (same `X-Line-Signature` and event) multiple times.
   - Verify only one punch row is created/updated; second+ requests do not create duplicates.

2. **Concurrency test**
   - Simulate concurrent clock-ins for the same employee (parallel `curl` or load tool).
   - Confirm exactly one open punch exists; no unique-constraint violations crash the handler.

3. **Partial index enforcement**
   - Manually attempt:
     ```sql
     INSERT INTO punches (tenant_id, employee_id, clock_in_at, clock_out_at)
     VALUES (1, 100, NOW(), NULL);
     -- repeat same tenant/employee with NULL clock_out_at -> expect 23505
     ```
   - Expect `23505` unique_violation.

4. **Clock-out behavior**
   - Clock in, then clock out; verify `clock_out_at` is set.
   - Subsequent clock-in must create a new row.
   - Duplicate clock-out on the same open punch must not create extra rows.

5. **Logs & metrics**
   - Monitor logs for duplicate processing
