# airship / ops

## Final Decision (merged)

Ship **`airship discover`** — a cron-safe, CDN-only ops orchestrator that produces static artifacts and reuses Lightning Studio safely.

Core deliverables (≤2h):
- One executable: `/opt/axentx/airship/bin/airship-discover` (Bash, shebang, executable, idempotent, lockfile).
- Runs market research (if present) + knowledge-RAG top-hub query → tagged insights JSON.
- Calls HF `list_repo_tree` **once per date folder**, saves `manifest-{date}.json`.
- Builds CDN-only training manifest (CDN URLs only) so Lightning Studio uses zero API calls during training.
- Reuses running Lightning Studio when present (quota/timeout safety) with safe restart/idle-avoidance logic.
- Emits static `status.json` (machine) and `status/index.html` (human) for ops visibility.
- Cron-safe crontab entry (`SHELL=/bin/bash`, PATH safety, logging to stdout/stderr).

---

## Implementation plan (≤2h)

1. Create `/opt/axentx/airship/bin/airship-discover` (executable, Bash).
2. Add lockfile (`/var/lock/airship-discover.lock`) and log rotation (stdout/stderr captured by cron).
3. Implement phases:
   - Phase A — Market research: run `granite-business-research.sh` if present; capture structured output.
   - Phase B — Knowledge-RAG: query top hub (e.g., MOC) and emit tagged insights JSON.
   - Phase C — HF manifest: `list_repo_tree` for target date folder → `manifest-{date}.json`.
   - Phase D — CDN-only training manifest: embed CDN URLs for Lightning training (no auth/API during training).
   - Phase E — Lightning Studio reuse: detect running Studio; if absent start one; if present send safe keep-alive or graceful restart only when needed (avoid idle-timeout kills and quota waste).
   - Phase F — Static status: write `status.json` and `status/index.html` (last_run, phase, outcome, counts, next_run_eta, health).
4. Add crontab entry with `SHELL=/bin/bash` and safe PATH.
5. Provide small Python helper (`tools/build_manifest.py`) for reliable HF calls and CDN manifest generation.

acquire_lock() {
  if [ -e "$LOCKFILE" ] && kill -0 "$(cat "$LOCKFILE")" 2>/dev/null; then
    echo "[$LOG_TAG] Another instance is running (PID $(cat "$LOCKFILE")). Exiting."
    exit 0
  fi
  echo $$ > "$LOCKFILE"
  trap 'rm -f "$LOCKFILE"' EXIT
}

timestamp() {
  date -u +"%Y-%m-%dT%H:%M:%SZ"
}

run_market_research() {
  local outdir="$1"
  local date="$2"
  local result_file="${outdir}/business-research-${date}.json"
  local script="granite-business-research.sh"

  if command -v "$script" >/dev/null 2>&1 || [ -x "./$script" ] || [ -x "$script" ]; then
    echo "[$LOG_TAG] Running market research..."
    if ./$script --date "$date" --out "$result_file" 2>/dev/null || $script --date "$date" --out "$result_file" 2>/dev/null || $script > "$result_file" 2>&1; then
      echo "[$LOG_TAG] Market research saved to $result_file"
      echo "$result_file"
      return 0
    else
      echo "[$LOG_TAG] Market research script failed; continuing."
    fi
  else
    echo "[$LOG_TAG] Market research script not found; skipping."
  fi
  echo ""
}

run_knowledge_rag() {
  local outdir="$1"
  local date="$2"
  local insights_file="${outdir}/${INSIGHTS_NAME}"

  if command -v knowledge-rag >/dev/null 2>&1; then
    echo "[$LOG_TAG] Querying top hub via knowledge-rag..."
    knowledge-rag top-hub --date "$date" --format json > "$insights_file" 2>/dev/null || true
  fi

  if [ ! -s "$insights_file" ] || ! jq empty "$insights_file" 2>/dev/null; then

{
  "date": "$date",
  "hub": "MOC",
  "tags": ["#knowledge-rag", "#graph", "#hub"],
  "insight": "Top hub MOC selected for planning (fallback).",
  "source": "airship-discover"
}
EOF
  fi
  echo "$insights_file"
}
