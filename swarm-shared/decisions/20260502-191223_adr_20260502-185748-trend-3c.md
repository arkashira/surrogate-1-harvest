## v0.1 scope
v0.1 ships in 2 weeks: CLI tool that indexes a local repo (Python-first), runs local AST + lightweight HF model analyses, outputs severity-ranked findings to terminal + JSON file, supports --since and --path filters, and stores results in local SQLite; includes sample repo and CI-friendly exit codes.

## Open questions
- Which languages must be supported in v0.1 (Python/JS/Go/TS?)
- Will users run analyses fully offline or allow optional opt-in sharing of anonymized findings?
- What model size/runtime trade-off is acceptable on typical developer laptops (CPU vs. small GPU)?
- How should licensing/code provenance be handled to avoid suggesting code that violates original licenses?
- Do we need a shared team metadata store (Supabase Postgres) for collaborative dashboards or keep everything local?
