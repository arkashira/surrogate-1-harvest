# surrogate-1 / design

## Highest-value incremental improvement (<2h)

**Improvement:** Replace recursive `list_repo_files` + per-file API ingestion with a **single-tree + CDN-only fetch** strategy that eliminates HF API 429s and removes per-file API calls during training/ingest.

**Why this ships fast:**  
- One small script change + one training script change.  
- No infra changes, no new secrets, no schema migrations.  
- Uses existing patterns (CDN bypass, pre-list once, embed file list).  
- Immediately removes the 429 and OOM root causes.

---

## Concrete implementation plan

1. **Add `bin/list-public-tree.sh`**  
   - Runs on Mac (or CI) once per date folder after rate-limit window.  
   - Calls `list_repo_tree(path, recursive=False)` for each date folder.  
   - Emits `file-list/<date>/tree.json` containing `{ "path": "...", "size": ..., "oid": ... }` entries.

2. **Update `bin/dataset-enrich.sh`**  
   - Accept optional `FILE_LIST` path.  
   - If provided, iterate local JSON instead of calling `list_repo_files` recursively.  
   - For each entry, download via CDN URL:  
     `https://huggingface.co/datasets/axentx/surrogate-1-training-pairs/resolve/main/<path>`  
   - Keep existing per-record projection to `{prompt, response}` and dedup via `lib/dedup.py`.

3. **Update training launcher / `train.py`**  
   - Embed the same `file-list/<date>/tree.json` (committed or baked into container).  
   - Data loader uses CDN-only URLs with `wget`/`requests` + `pyarrow` open (no `load_dataset`).  
   - Remove `streaming=True` usage for heterogeneous repos.  
   - Add retry/backoff for CDN 429/5xx (separate from API limits).

4. **Studio reuse guard**  
   - Before `Studio(create_ok=True)`, list running studios and reuse if present.  
   - On idle stop, detect status and restart with `target.start(machine=Machine.L40S)`.

5. **Smoke test**  
   - Run one shard locally with `FILE_LIST=file-list/2026-04-29/tree.json` and verify zero HF API calls during fetch (check logs for `resolve/main` URLs and no `/api/`).  
   - Confirm memory < 1.5 GB per shard during parquet decode.

## Code snippets

### 1) `bin/list-public-tree.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
OUT_DIR="file-list"
DATE="${1:-$(date +%F)}"
TREE_PATH="${OUT_DIR}/${DATE}/tree.json"

mkdir -p "$(dirname "${TREE_PATH}")"

python3 - <<PY
import os, json, sys
from huggingface_hub import HfApi

api = HfApi()
repo = os.getenv("REPO")
date = os.getenv("DATE")
tree_path = os.getenv("TREE_PATH")

entries = []
for obj in api.list_repo_tree(repo, path=date, recursive=False):
    entries.append({
        "path": obj.path,
        "size": getattr(obj, "size", None),
        "oid": getattr(obj, "oid", None),
    })

with open(tree_path, "w") as f:
    json.dump(entries, f, indent=2)

print(f"Wrote {len(entries)} entries to {tree_path}")
PY
```

### 2) Update `bin/dataset-enrich.sh` (excerpt)
```bash
#!/usr/bin/env bash
set -euo pipefail

HF_REPO="axentx/surrogate-1-training-pairs"
CDN_BASE="https://huggingface.co/datasets/${HF_REPO}/resolve/main"
FILE_LIST="${FILE_LIST:-}"

download_via_cdn() {
  local rel_path="$1"
  curl -fsSL --retry 3 "${CDN_BASE}/${rel_path}" -o "${rel_path##*/}"
}

process_file() {
  local rel_path="$1"
  download_via_cdn "$rel_path"
  # existing projection to {prompt,response} + dedup via lib/dedup.py
  python3 -m lib.dedup --input "${rel_path##*/}" --output "enriched/${rel_path##*/}.jsonl"
}

if [[ -n "${FILE_LIST}" && -f "${FILE_LIST}" ]]; then
  echo "Using file list: ${FILE_LIST}"
  jq -r '.[].path' "${FILE_LIST}" | while read -r p; do
    process_file "$p"
  done
else
  echo "FILE_LIST not provided; falling back to API (not recommended)"
  # fallback (avoid in prod)
fi
```

### 4) Studio reuse + idle restart (Python)
```python
from lightning import Studio, Machine, Teamspace

def get_or_start_studio(name, machine=Machine.L40S):
    for s in Teamspace.studios:
        if s.name == name:
            if s.status == "running":
                return s
            # idle stop kills training; restart
            s.start(machine=machine)
            return s
    return Studio(name=name, create_ok=True, machine=machine)
```
