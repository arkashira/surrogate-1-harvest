## Deployment
cf workers (free-tier friendly; deploy API + scheduled workers for scans; fallback HF space for UI or workers + Supabase free tier)

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP (FastAPI + worker) that connects to one AWS account and one GCP project, lists all Lambda + Cloud Functions schedules, detects exact duplicates and obvious drifts (cron/target hash changed), exports JSON report and writes Schedule + ScheduleDiff to Supabase (free tier). No UI; auth via single bearer token; deploy as CF Worker with daily scheduled scan.

## Open questions
- Which identity model: per-customer API keys vs OAuth for cloud provider read-only access?
- How to handle provider rate limits during full inventory scans (exponential backoff, incremental vs full)?
- What retention/TTL for Schedule and ScheduleDiff records on the free tier?
- Will orphan detection require historical invocation logs (CloudWatch/Stackdriver) or rely only on current bindings?
- Should duplicate detection be exact (same cron+target) or fuzzy (same target, different cron)?
