# airship / quality

## Final Synthesized Plan (Best Parts + Resolved Contradictions)

**Chosen Path**: Ship a **static, CDN-safe `airship discover` orchestrator** (Bash-first) that produces deterministic `status.json` and `manifest.json` so the frontend renders health/state with zero backend/SSR calls.

**Why this wins**:
- Eliminates runtime backend dependency for status pages.
- Enables edge caching, reduces latency, and aligns with prior “frontend-safe/CDN-only” decisions.
- Fits the existing `airship discover` pattern and keeps Surrogate AI training-friendly (static manifests → zero API calls during CDN fetches).
- Low risk, high leverage: one orchestrator script + one JSON schema + one small frontend adapter.

---

## Concrete Implementation Plan (Actionable)

### 1. Create `bin/airship-discover` (Bash orchestrator)
- Shebang `#!/usr/bin/env bash`, `set -euo pipefail`, idempotent.
- Runs discovery (port checks, service metadata, git hashes, build info).
- Probes local health endpoints (fast, short timeout) and records status.
- Outputs:
  - `dist/status.json` — canonical status snapshot (sorted keys, stable timestamps).
  - `dist/manifest.json` — CDN asset map + cache hints (hashes, sizes).
  - (Optional) `dist/index.html` — minimal static status page.
- Uses only local checks + `curl --max-time 2` to dependencies (no auth).
- Exit codes: `0`=healthy, `1`=partial, `2`=failed.

### 2. Add JSON schema (`schemas/status-snapshot.json`)
- Minimal fields: `generatedAt`, `git`, `services[]`, `endpoints[]`, `cacheHint`.
- Enables frontend type-safety and CDN caching.

### 3. Lightweight Node helper (`scripts/build-manifest.js`) — optional
- Reads build outputs and writes manifest with content hashes.
- Keeps orchestrator bash-focused; Node only for hash generation if needed.

### 4. Update package.json / Makefile
- Add `discover` script: `bin/airship-discover`.
- Add `build:status` to run discover + manifest generation.

### 5. Frontend integration
- Add a small runtime module that fetches `/dist/status.json` and `/dist/manifest.json` and exposes a reactive status/asset API (no SSR).
- Use cache-first strategy: `max-age=60` for status, `immutable` for hashed assets.

### 6. CI/local usage
- CI runs `npm run build:status` and uploads `dist/` to CDN (or repo static hosting).
- Developers run `npm run discover` to refresh local status.

## Resolved Contradictions (Correctness + Actionability)

| Contradiction | Resolution |
|--------------|------------|
| **Output location**: `dist/` vs `public/status/` | Use `dist/` (aligns with build artifacts and prior decisions). CI can copy/symlink to `public/status/` if required for hosting. |
| **Symlink `latest.json`** | Omit symlink for portability (Windows CI issues). Instead, always overwrite `dist/status.json` as the canonical latest. |
| **Schema location**: inline vs external | Add external `schemas/status-snapshot.json` for type-safety and reuse, but keep orchestrator output self-contained (no runtime dependency on schema). |
| **Node vs Bash for manifest** | Bash-first orchestrator; Node helper optional for richer builds. Keeps core toolchain simple and portable. |
| **Exit codes** | Standardize: `0`=healthy, `1`=partial, `2`=failed. Document and use in CI. |

## Final Code Snippets

### `bin/airship-discover`
```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
DIST="${ROOT}/dist"
STATUS_FILE="${DIST}/status.json"
MANIFEST_FILE="${DIST}/manifest.json"
TIMESTAMP="$(date -u +"%Y-%m-%dT%H:%M:%SZ")"

mkdir -p "${DIST}"

# Discover services from compose files
COMPOSE_FILE="${ROOT}/docker-compose.microservices.yml"
if [[ ! -f "${COMPOSE_FILE}" ]]; then
  COMPOSE_FILE="${ROOT}/docker-compose.yml"
fi

extract_services() {
  if command -v yq &>/dev/null; then
    yq eval '
      .services | to_entries[] |
      select(.value.ports != null) |
      .key + "|" + (.value.ports[0] | split(":")[0] // "8000")
    ' "${COMPOSE_FILE}" 2>/dev/null || true
  else
    awk '/^  [a-zA-Z0-9_-]+:/ { svc=$1 } /ports:/ { inports=1 } inports && /- / { port=$2; gsub(/"/,"",port); split(port, a, ":"); print svc "|" a[length(a)] }' "${COMPOSE_FILE}" | head -20
  fi
}

SERVICES=$(extract_services)
if [[ -z "${SERVICES}" ]]; then
  SERVICES="arkship|3000
surrogate|8001"
fi

# Probe health endpoints
probes=()
while IFS="|" read -r svc port; do
  svc=$(echo "${svc}" | tr -d '[:space:]')
  port=$(echo "${port}" | tr -d '[:space:]')
  url="http://localhost:${port}"
  health_url="${url}/health"

  start_ms=$(date +%s%3N)
  if curl --max-time 2 --silent --fail "${health_url}" >/dev/null 2>&1; then
    status="healthy"
  elif curl --max-time 2 --silent --fail "${url}" >/dev/null 2>&1; then
    status="up"
  else
    status="down"
  fi
  end_ms=$(date +%s%3N)
  latency_ms=$((end_ms - start_ms))

  probes+=("$(printf '{"service":"%s","port":%s,"url":"%s","health_url":"%s","status":"%s","latency_ms":%s}' \
    "${svc}" "${port}" "${url}" "${health_url}" "${status}" "${latency_ms}")")
done <<< "${SERVICES}"

# Build status payload (sorted keys via jq if available)
status_json=$(printf '%s\n' "${probes[@]}" | jq -s '{generated_at: $ts, services: .}' --arg ts "${TIMESTAMP}" 2>/dev/null || \
  printf '{"generated_at":"%s","services":[%s]}' "${TIMESTAMP}" "$(IFS=,; echo "${probes[*]}")")

echo "${status_json}" | jq . > "${STATUS_FILE}" 2>/dev/null || echo "${status_json}" > "${STATUS_FILE}"

# Build manifest from dist assets or known build outputs
assets=()
for dir in "${ROOT}/arkship/build" "${ROOT}/surrogate/build" "${ROOT}/dist"; do
  if [[ -d "${dir}" ]]; then
    while IFS= read -r f; do
      rel="${f#${ROOT}/}"
      hash=$(sha256sum "${f}" 2>/dev/null | awk '{print $1}' || echo "none")
      assets+=("$(printf '{"path":"%s","sha256":"%s","size":%s}' "${rel}" "${hash}" "$(stat -c%s "${f}" 2>/dev/null || echo 0)")")
    done < <(find "${dir}" -maxdepth 1 -type f \( -name "*.js" -o -name "*.css" -o -name "*.html" \) 2>/dev/null)
  fi
done

if [[ ${#assets[@]} -eq 0 ]]; then
  assets=("$(printf '{"path":"dist/status.json","sha256":"%s","size":%s}' "$(sha256sum "${STATUS_FILE}" | awk '{print $1}')" "$(stat -c%s "${STATUS_FILE}")")")
fi

manifest_json=$(printf '%s\n' "${assets[@]}" | jq -s '{generated_at: $ts, assets: .}' --arg ts "${TIMESTAMP}" 2>/dev/null || \
  printf '{"generated_at":"%s","assets":[%s]}' "${TIMESTAMP}" "$(IFS=,; echo "${assets[*]}")")

echo "${manifest_json}" | jq . > "${MANIFEST_FILE}"
