## Deployment
cf workers (free tier) + Supabase free + HF space optional for UI prototype

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI endpoints to list schedules per provider, detect orphans (function missing), detect duplicates (same expression+target), export CSV/JSON report; deploy on CF Workers (scheduler trigger) and Supabase free tier; no UI, auth via bearer token, rate-limited to 1000 resources/run.

## Open questions

- How often should discovery jobs run (hourly vs daily) to balance freshness and rate limits?
- Will we compare schedules against IaC (Terraform/CDK) or only live state?
- How will customers retire stale schedules (manual approval flow vs dry-run report only)?
