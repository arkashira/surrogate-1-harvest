## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres + edge functions) + Redis (free-tier via Redis Cloud or Supabase Redis)

## v0.1 scope
v0.1 in 2 weeks: single-asset pipeline (crypto or US equities) with one research connector (public OHLCV), one signal agent (rule-based), one risk guardrail (per-trade notional limit), simulated execution (paper), deterministic Redis Streams event log, Postgres audit trail, and API endpoints to submit runs and query status/latency metrics targeting median signal-to-execution <500ms in local perf tests.

## Open questions
- Broker/data-vendor connector auth model: API-key vs OAuth scopes and rotation strategy?
- Deterministic replay fidelity: do we need tick-level or candle-level snapshots for reproducible risk checks?
- Latency budget split: target per-stage budgets (research->signal, signal->risk, risk->execution) to meet <500ms median?
- Idempotency and exactly-once semantics: dedupe keys and at-least-once broker behavior handling?
- Compliance retention and PII handling: how long to retain immutable audit events and where to archive?
