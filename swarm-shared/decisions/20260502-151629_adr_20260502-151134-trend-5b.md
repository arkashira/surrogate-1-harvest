## v0.1 scope
v0.1 in 2 weeks: deterministic workflow engine (Redis Streams event log), Postgres-backed immutable audit trail, one sample connector per domain (one broker, one data vendor), risk/compliance guardrails as versioned JSON rules evaluated inline, replay harness that can reconstruct any workflow_run from persisted events, edge API on Cloudflare Workers + Supabase for auth and persistence, and contract tests proving deterministic replay for 5 canonical market scenarios.

## Open questions
- Which brokers/OMS/EMS and data vendors must be supported in v0.1 to prove the NSM?
- What deterministic replay SLA is required (exact bitwise replay vs logical equivalence) and how will we snapshot inputs for replay?
- How will secrets/credentials be stored and rotated in a cloud-free, OSS-first setup (local vault vs Supabase encrypted columns)?
- What latency envelope (p50/p99) must the orchestration layer meet to be considered sub-second end-to-end?
- Which compliance policies are non-negotiable for launch and how will they be expressed as versioned, testable rules?
- How will idempotency and exactly-once execution be guaranteed across connectors with at-least-once delivery from Redis Streams?
