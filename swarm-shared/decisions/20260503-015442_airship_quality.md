**CDN-first ingestion + deterministic sibling-repo sharding + Studio lifecycle guard**  
- Eliminates HF API 429s during training by pre-listing once and downloading via public CDN  
- Bypasses 128/hr commit cap via deterministic sibling-repo sharding (hash → repo)

## Concrete Implementation Plan (≤2h)

### 1) Pre-list file paths once and embed in training script (15m)
- On the Mac orchestrator (after rate-limit window), run:
  - `list_repo_tree(path, recursive=False)` for target date folder
  - Save `{"date": "...", "files": [...]}` to `file_list.json`
- Embed `file_list.json` in `train.py` and use CDN-only fetches during data loading:
  - `https://huggingface.co/datasets/{repo}/resolve/main/{path}` (no auth, CDN tier)
- Do **not** use `load_dataset(streaming=True)` on heterogeneous repos.

### 2) Deterministic sibling-repo sharding for writes (20m)
- Define sibling repos: `surrogate-1`, `surrogate-2`, ..., `surrogate-5`
- Hash slug → pick repo:
  ```python
  import hashlib
  def pick_repo(slug: str, prefix="surrogate", n=5) -> str:
      h = int(hashlib.sha256(slug.encode()).hexdigest(), 16)
      idx = h % n
      return f"{prefix}-{idx + 1}"
  ```
- All ingestion workers use this to spread commits (≈640/hr aggregate).

### 3) Clean ingestion schema before upload (15m)
- Project each raw file to `{prompt, response}` at parse time.
- Filename pattern: `batches/mirror-merged/{date}/{slug}.parquet`
- Do **not** add `source` or `ts` columns (keep attribution via path/filename).

### 4) Studio lifecycle guard + reuse (20m)
- Before `Studio(create_ok=True)`:
  ```python
  from lightning import Teamspace, Studio, Machine

  running = [s for s in Teamspace.studios if s.name == "surrogate-train" and s.status == "Running"]
  if running:
      studio = running[0]
  else:
      studio = Studio(name="surrogate-train", create_ok=True)
  ```
- Guard every `.run()`:
  ```python
  if studio.status != "Running":
      studio.start(machine=Machine.L40S)  # or fallback priority clouds
  studio.run(...)
  ```
- Prevents quota waste and idle-timeout deaths.

### 5) Cron/wrapper hardening (10m)
- Ensure all wrapper scripts:
  - Shebang: `#!/usr/bin/env bash`
  - Executable: `chmod +x`
  - Invoked via `bash <script> "$@"`
- Set in crontab: `SHELL=/bin/bash`

### 6) Quick validation (10m)
- Run ingestion for one date folder end-to-end:
  - Pre-list → CDN fetch → project → sharded upload → studio reuse
- Verify:
  - No 429s during CDN fetch
  - Commits spread across siblings
  - Studio reused if running
  - Parquet contains only `prompt`, `response`

def build_dataset(file_list_path="file_list.json"):
    with open(file_list_path) as f:
        listing = json.load(f)

    rows = []

### Studio lifecycle guard
```python
from lightning import Teamspace, Studio, Machine

def get_or_create_studio(name="surrogate-train", machine=Machine.L40S):
    running = [s for s in Teamspace.studios if s.name == name and s.status == "Running"]
    if running:
        return running[0]
    studio = Studio(name=name, create_ok=True)
    if studio.status != "Running":
        studio.start(machine=machine)
    return studio
```
