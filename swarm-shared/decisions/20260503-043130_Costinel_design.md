# Costinel / design

## Implementation Plan — CDN-First Top-Hub Signal Panel (<2h)

**Goal**: Ship a resilient “Top Hub” signal panel into Costinel that shows the most-connected hub (e.g., “MOC”) and related docs using CDN-fetched artifacts.  
**Principle**: Strict **Sense + Signal — ไม่ Execute**. Zero runtime HF API calls. No new infra, no secrets, no DB migrations.

---

### Architecture (CDN-first, correct + actionable)
1. **Offline/Mac orchestration** (run once per day or per commit):
   - `list_repo_tree(path='knowledge-rag', recursive=False)` for the target date folder.
   - Save `{date}/file-list.json` to repo (committed) or to build artifact.
   - Optional: small script to compute top hub from graph edges (if available) and emit `top-hub.json` (slug + context).

2. **Build-time embedding**:
   - Include `file-list.json` and `top-hub.json` in the Next.js bundle (public/ or imported JSON).
   - No runtime HF API usage.

3. **Runtime (Costinel UI)**:
   - Panel fetches local `top-hub.json` (or bundled import).
   - Shows hub name, short insight, and related doc links (CDN URLs: `https://huggingface.co/datasets/{repo}/resolve/main/knowledge-rag/{date}/{file}`).
   - Fallback UI when data missing.

4. **Resilience**:
   - CDN-only fetches during build and runtime.
   - Graceful degradation if panel data unavailable.
   - No secrets, no new env vars, no migrations.

#### 1) Add Mac orchestration script (optional but recommended)
`scripts/build-top-hub.js` (Node, runs on Mac; can be invoked from CI or locally)

```js
#!/usr/bin/env node
// scripts/build-top-hub.js
// Offline: list repo tree once and produce top-hub.json for the panel.
// Run: node scripts/build-top-hub.js --date=2026-04-27 --out=public/data/top-hub.json

import { program } from 'commander';
import { writeFileSync, mkdirSync, existsSync } from 'fs';
import { resolve, dirname } from 'path';
import { fileURLToPath } from 'url';
import { HfApi } from '@huggingface/hub'; // optional; can be replaced by curl in CI

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

program
  .requiredOption('--date <date>', 'date folder in knowledge-rag, e.g. 2026-04-27')
  .option('--repo <repo>', 'HF dataset repo', 'AXENTX/Costinel')
  .option('--out <out>', 'output JSON path', 'public/data/top-hub.json')
  .option('--dry-run', 'list files only');

program.parse(process.argv);
const { date, repo, out, dryRun } = program.opts();

async function run() {
  const api = new HfApi();
  // Single API call (rate-limited friendly)
  const tree = await api.listRepoTree({
    repo: { type: 'dataset', repoId: repo },
    path: `knowledge-rag/${date}`,
    recursive: false,
  });

  const files = tree
    .filter((t) => t.type === 'file')
    .map((t) => t.path)
    .sort();

  if (dryRun) {
    console.log('Files:', files);
    return;
  }

  // Simple heuristic: pick most recent graph file or file containing "hub" / "MOC"
  const hubFile = files.find((f) => /hub|moc|top/i.test(f)) || files[0] || null;

  const cdnBase = `https://huggingface.co/datasets/${repo}/resolve/main`;
  const payload = {
    date,
    hub: hubFile ? hubFile.replace(/^knowledge-rag\/[^\/]+\//, '').replace(/\.[^/.]+$/, '') : 'MOC',
    hubFile,
    insight: hubFile
      ? `Top hub for ${date} derived from knowledge graph. Review connections for context.`
      : 'No hub file found; defaulting to MOC.',
    related: files.map((f) => ({
      name: f.replace(/^knowledge-rag\/[^\/]+\//, ''),
      cdn: `${cdnBase}/${f}`,
    })),
    generatedAt: new Date().toISOString(),
  };

  const outPath = resolve(__dirname, '..', out);
  mkdirSync(dirname(outPath), { recursive: true });
  writeFileSync(outPath, JSON.stringify(payload, null, 2), 'utf8');
  console.log('Wrote', outPath);
}

run().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

Make it executable and add to package.json scripts if desired:
```bash
chmod +x scripts/build-top-hub.js
```
