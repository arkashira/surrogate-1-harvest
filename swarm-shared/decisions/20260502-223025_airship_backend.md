# airship / backend

## Final Decision: Ship `airship discover` (backend orchestrator)

**Core deliverable**: A single, cron-safe executable `/opt/axentx/airship/bin/airship-discover` that produces tagged research outputs and a CDN-only training manifest with one HF API call per run.

---

## Implementation Plan (merged + hardened)

### 1. Entrypoint script
Create `/opt/axentx/airship/bin/airship-discover` (executable, Bash shebang) with:

- `set -euo pipefail` and `exec 200>/tmp/airship-discover.lock; flock -n 200` for cron safety
- Explicit `SHELL=/bin/bash` expectation in crontab
- Idempotent behavior:
  - Skip `granite-business-research.sh` if already run today (date-based marker)
  - Always run `knowledge-rag` (fast, cheap)
  - Always regenerate CDN manifest (cheap HEAD/listing)
- Single HF API call (`list_repo_tree`, non-recursive) to build manifest
- Tagged logging and outputs: `#business-research #knowledge-rag #graph #hub #training #cdn #manifest`

### 2. Training manifest strategy
- One API call from orchestrator to `list_repo_tree(repo, path, recursive=False)` for a date folder
- Save JSON to `manifests/cdn-manifest-YYYY-MM-DD.json`
- Manifest schema:
  - `repo`, `path`, `date`
  - `files[]` with `path`, `cdn_url`, `size`
- Surrogate training uses only CDN URLs (`resolve/main/...`) — zero HF API calls during data loading

### 3. Research tagging and outputs
- Tag research outputs and logs with `#business-research #knowledge-rag #graph`
- Tag hub queries with `#hub`
- Emit structured, timestamped logs to stdout/stderr (cron captures)

## Final Code Snippets

### `/opt/axentx/airship/bin/airship-discover`

```bash
#!/usr/bin/env bash
# airship-discover
# Zero-config, cron-safe orchestrator for research + CDN manifest generation.
#
# Usage:
#   airship-discover
#
# Outputs:
#   - research outputs (if granite-business-research.sh runs)
#   - knowledge-rag insights (top hub + related docs)
#   - manifests/cdn-manifest-YYYY-MM-DD.json
#
# Tags produced (in logs + output metadata):
#   #business-research #knowledge-rag #graph #hub #training #cdn #manifest

set -euo pipefail

# ---- Config ----
AIRSHIP_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
MANIFEST_DIR="${AIRSHIP_ROOT}/manifests"
RESEARCH_SCRIPT="${AIRSHIP_ROOT}/scripts/granite-business-research.sh"
KNOWLEDGE_RAG_CMD="${AIRSHIP_ROOT}/scripts/knowledge-rag"

# HF dataset repo to list (public repo, no auth required for CDN)
HF_REPO="${HF_DATASET_REPO:-datasets/your-org/your-repo}"
HF_PATH="${HF_DATASET_PATH:-daily}"  # e.g. "daily/2026-05-02"

# Date for manifest
DATE_TAG="$(date -u +%Y-%m-%d)"
MANIFEST_FILE="${MANIFEST_DIR}/cdn-manifest-${DATE_TAG}.json"

# Lockfile to prevent overlapping cron runs
LOCKFILE="/tmp/airship-discover.lock"
exec 200>"${LOCKFILE}"
flock -n 200 || { echo "ERROR: Another instance is running (lock held). Exiting."; exit 1; }

# ---- Helpers ----
log() {
  echo "[$(date -u +'%Y-%m-%dT%H:%M:%SZ')] $*"
}

ensure_dir() {
  mkdir -p "$1"
}

# ---- Steps ----
log "START airship-discover (date=${DATE_TAG})"

ensure_dir "${MANIFEST_DIR}"

# 1) Business research (if script exists and not already run today)
if [[ -x "${RESEARCH_SCRIPT}" ]]; then
  RESEARCH_MARKER="${AIRSHIP_ROOT}/.last-research-${DATE_TAG}"
  if [[ -f "${RESEARCH_MARKER}" ]]; then
    log "SKIP granite-business-research (already run today) #business-research #knowledge-rag #graph"
  else
    log "RUN granite-business-research.sh #business-research #knowledge-rag #graph"
    if bash "${RESEARCH_SCRIPT}"; then
      touch "${RESEARCH_MARKER}"
      log "DONE granite-business-research"
    else
      log "WARN: granite-business-research.sh exited non-zero (continuing)"
    fi
  fi
else
  log "SKIP granite-business-research (not found or not executable)"
fi

# 2) Knowledge RAG — top hub and related docs
if [[ -x "${KNOWLEDGE_RAG_CMD}" ]]; then
  log "RUN knowledge-rag (top hub + related docs) #knowledge-rag #graph #hub"
  if ! "${KNOWLEDGE_RAG_CMD}" --top-hub --related 2>&1; then
    log "WARN: knowledge-rag exited non-zero (continuing)"
  fi
else
  log "SKIP knowledge-rag (command not found or not executable)"
fi

# 3) CDN-only manifest (single API call; embed in training for CDN-only fetches)
log "GENERATE CDN manifest for ${HF_REPO}/${HF_PATH} #training #cdn #manifest"
ensure_dir "${MANIFEST_DIR}"

# Use HF Hub API to list tree (non-recursive by default; adjust if needed).
# This is the single API call allowed per run (rate-limited). After 429, wait 360s.
# We intentionally avoid recursive listing of large repos; caller should set HF_PATH
# to a specific date folder.
python3 - "$HF_REPO" "$HF_PATH" "$MANIFEST_FILE" "$DATE_TAG" <<'PY'
import json
import os
import sys
from huggingface_hub import HfApi

repo = sys.argv[1]
path = sys.argv[2]
out_file = sys.argv[3]
date_tag = sys.argv[4]

api = HfApi()
# recursive=False keeps listing small and fast; adjust if nested folders needed.
tree = api.list_repo_tree(repo=repo, path=path, recursive=False)

# Normalize entries to minimal CDN URLs (resolve/main -> CDN, no auth)
entries = []
for item in tree:
    if item.type == "file":
        entries.append({
            "path": item.path,
            "cdn_url": f"https://huggingface.co/datasets/{repo}/resolve/main/{item.path}",
            "size": getattr(item, "size", None)
        })

os.makedirs(os.path.dirname(out_file), exist_ok=True)
with open(out_file, "w", encoding="utf-8") as f:
    json.dump({
        "repo": repo,
        "path": path,
        "date": date_tag,
        "files": entries
    }, f, indent=2)

print(f"Wrote {len(entries)} entries to {out_file}")
PY

log "Wrote CDN manifest: ${MANIFEST_FILE}"

# ---- Finish ----
log "END airship-discover (success) #business-research #knowledge-rag #graph #training #cdn #manifest"
exit 0
```
