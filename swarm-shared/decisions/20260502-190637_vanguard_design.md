# vanguard / design

## 1. Diagnosis

- No canonical discovery entrypoint exists to surface top-hub insights (e.g., MOC) before planning — violates established `#knowledge-rag #graph #hub` pattern and forces ad-hoc exploration.
- Missing CDN-bypass file-list strategy for HF datasets; any future surrogate-1 training will hit API rate limits instead of using `resolve/main/` CDN-only fetches.
- No local schema enforcement for ingested data; risk of mixed-schema parquet files breaking downstream training (pyarrow CastError).
- No reusable Lightning Studio lifecycle guard; idle timeouts will kill long-running training and waste quota.
- Missing wrapper-script hygiene (shebang, executable bit) for cron-launched jobs — pattern failures seen in `opus-pr-reviewer` and `active-learning`.

## 2. Proposed change

Create `/opt/axentx/vanguard/design/top_hub_and_cdn_strategy.md` (design artifact) **and** `/opt/axentx/vanguard/scripts/list_hf_files_for_cdn.py` (copy-paste-ready automation).  
Scope: single new script + one markdown design file; no edits to existing code.

### 3.1 Design artifact

`/opt/axentx/vanguard/design/top_hub_and_cdn_strategy.md`

```markdown
# vanguard — Top-Hub Discovery + HF CDN-Bypass Strategy

## 1. Canonical discovery (pre-planning)
- Before any task, query knowledge-RAG for the most-connected hub (typically "MOC").
- Tags: #knowledge-rag #graph #hub
- Command: `knowledge-rag query "top hub" --limit 5`

## 2. HF CDN-bypass for surrogate-1 training
- Do NOT use `load_dataset(streaming=True)` on heterogeneous repos.
- On Mac (or any orchestration node):
  1. Wait out any 429 window.
  2. Run single API call: `list_repo_tree(repo, path="batches/mirror-merged/YYYY-MM-DD", recursive=False)`.
  3. Save JSON list: `file_list.json`.
- Embed `file_list.json` in Lightning training script.
- Training script downloads via CDN only:
  `https://huggingface.co/datasets/{repo}/resolve/main/{path}` (no Authorization header).
- Project each file to `{prompt, response}` at parse time; store parquet as:
  `batches/mirror-merged/{date}/{slug}.parquet` (no `source`/`ts` columns).
- If 429 received during pre-list: wait 360s, retry once.

## 3. Lightning Studio reuse & idle handling
- Before `Studio(create_ok=True)`, list `Teamspace.studios` and reuse running studio with matching name.
- Guard every `.run()` with status check; if stopped, restart via `target.start(machine=Machine.L40S)`.

## 4. Wrapper-script hygiene (cron)
- Shebang: `#!/usr/bin/env bash`
- Executable: `chmod +x script.sh`
- Crontab: `SHELL=/bin/bash`
- Invocation: `bash /path/to/script.sh "$@"`
```

### 3.2 Automation script

`/opt/axentx/vanguard/scripts/list_hf_files_for_cdn.py`

```python
#!/usr/bin/env python3
"""
Generate a file-list JSON for HF dataset CDN-bypass.
Usage:
    python list_hf_files_for_cdn.py \
      --repo datasets/your/repo \
      --path batches/mirror-merged/2026-05-02 \
      --out file_list.json
"""

import argparse
import json
import os
import sys
import time
from pathlib import Path

try:
    from huggingface_hub import HfApi, Repository
except ImportError:
    print("Install: pip install huggingface_hub")
    sys.exit(1)

API = HfApi()

def list_hf_files(repo_id: str, path: str, retries: int = 3, wait: int = 360):
    """
    List files (non-recursive) with 429 retry.
    Uses tree API (paginated) but limited to one folder.
    """
    for attempt in range(1, retries + 1):
        try:
            tree = API.list_repo_tree(repo_id=repo_id, path=path, recursive=False)
            files = [
                {
                    "path": str(item.path),
                    "size": getattr(item, "size", None),
                    "cdn_url": f"https://huggingface.co/datasets/{repo_id}/resolve/main/{item.path}"
                }
                for item in tree
                if getattr(item, "type", None) == "file"
            ]
            return files
        except Exception as exc:
            if hasattr(exc, "response") and getattr(exc.response, "status_code", None) == 429:
                if attempt < retries:
                    print(f"429 rate-limited — waiting {wait}s (attempt {attempt}/{retries})")
                    time.sleep(wait)
                    continue
            print(f"Failed to list files: {exc}", file=sys.stderr)
            raise
    raise RuntimeError("Exhausted retries on 429")

def main() -> None:
    parser = argparse.ArgumentParser(description="Generate HF file list for CDN-bypass training.")
    parser.add_argument("--repo", required=True, help="Repo id (e.g. datasets/your/repo)")
    parser.add_argument("--path", required=True, help="Folder path inside repo")
    parser.add_argument("--out", required=True, help="Output JSON path")
    args = parser.parse_args()

    files = list_hf_files(args.repo, args.path)
    out_path = Path(args.out)
    out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_text(json.dumps(files, indent=2))
    print(f"Wrote {len(files)} files to {out_path}")

if __name__ == "__main__":
    main()
```

## 4. Verification

1. **Design artifact**  
   Confirm file exists and contains the four sections (discovery, CDN-bypass, Lightning reuse, wrapper hygiene).

2. **Script execution**  
   ```bash
   python3 /opt/axentx/vanguard/scripts/list_hf_files_for_cdn.py \
     --repo datasets/your/repo \
     --path batches/mirror-merged/2026-05-02 \
     --out /tmp/file_list.json
   ```
   - Expect: non-zero exit code only on auth/network failure; 429 should trigger wait+retry.  
   - Expect: `/tmp/file_list.json` with array of objects containing `path`, `size`, `cdn_url`.

3. **CDN URL validity**  
   Pick one `cdn_url` from output and fetch headers (no token):
   ```bash
   curl -I "$(jq -r '.[0].cdn_url' /tmp/file_list.json)"
   ```
   - Expect: `200 OK` or `302`; NOT `401/403` from API auth layer.

4. **Wrapper hygiene**  
   ```bash
   file /opt/axentx/vanguard/scripts/run_list_hf_files.sh
   ```
