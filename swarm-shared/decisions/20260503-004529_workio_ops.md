# workio / ops

## 1. Diagnosis
- Missing **idempotency handling** on `/webhook/line` allows LINE at-least-once retries to create duplicate punch records and double-clock events.
- No **DB-level partial unique constraint** enforcing “one open punch per user” (`clock_out_at IS NULL`) permits multiple concurrent open punches.
- No **deduplication table or key** to safely reject retries within a configurable time window (e.g., 5–10 min).
- Webhook handler performs **read-then-write without transaction isolation**, risking race conditions under concurrent retries.
- No **cleanup job** to expire stale idempotency keys, causing unbounded table growth.

## 2. Proposed change
- **Files**:  
  - `workio/server/src/db/schema.sql` — add `punches` partial unique constraint + `webhook_line_idempotency` table.  
  - `workio/server/src/routes/webhook/line.ts` — wrap handler in transaction, check/insert idempotency key, enforce one-open-punch rule.  
  - `workio/server/src/jobs/cleanup_idempotency.ts` — lightweight cron to delete keys older than TTL (optional but recommended).

## 3. Implementation

### DB migration (run once)
```sql
-- 1) One open punch per user (partial unique constraint)
CREATE UNIQUE INDEX idx_punches_one_open_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- 2) Idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS webhook_line_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: index for cleanup
CREATE INDEX idx_webhook_line_idempotency_created_at
ON webhook_line_idempotency (created_at);
```

### Idempotency helper (`workio/server/src/lib/idempotency.ts`)
```ts
import { PoolClient } from 'pg';

const TTL_MS = 10 * 60 * 1000; // 10 minutes

export async function ensureIdempotent(
  client: PoolClient,
  key: string
): Promise<boolean> {
  if (!key) return false;
  const res = await client.query(
    `INSERT INTO webhook_line_idempotency (idempotency_key, created_at)
     VALUES ($1, NOW())
     ON CONFLICT (idempotency_key) DO NOTHING
     RETURNING idempotency_key`,
    [key]
  );
  return res.rowCount === 1;
}
```

### Updated webhook handler (`workio/server/src/routes/webhook/line.ts`)
```ts
import { Router, Request, Response } from 'express';
import { pool } from '../../db';
import { ensureIdempotent } from '../../lib/idempotency';

const router = Router();

router.post('/line', async (req: Request, res: Response) => {
  const client = await pool.connect();
  const idempotencyKey = req.headers['x-line-idempotency-key'] as string
    || req.body.events?.[0]?.source?.userId
    ? `${req.body.events?.[0]?.source?.userId}:${req.body.events?.[0]?.type}:${req.body.events?.[0]?.timestamp}`
    : null;

  if (!idempotencyKey) {
    return res.status(400).json({ error: 'Missing idempotency key' });
  }

  try {
    await client.query('BEGIN');

    // Idempotency check
    const isNew = await ensureIdempotent(client, idempotencyKey);
    if (!isNew) {
      await client.query('ROLLBACK');
      return res.status(200).json({ ok: true, reason: 'duplicate' });
    }

    const { events } = req.body;
    const { userId } = events[0].source;
    const timestamp = events[0].timestamp;

    // Fetch latest punch for user
    const latest = await client.query(
      `SELECT id, clock_out_at FROM punches
       WHERE user_id = $1
       ORDER BY clock_in_at DESC
       LIMIT 1`,
      [userId]
    );

    const last = latest.rows[0];

    if (!last || last.clock_out_at !== null) {
      // Clock in
      await client.query(
        `INSERT INTO punches (user_id, clock_in_at, clock_out_at, location)
         VALUES ($1, TO_TIMESTAMP($2 / 1000.0), NULL, $3)`,
        [userId, timestamp, events[0]?.content?.text || '']
      );
    } else {
      // Clock out
      await client.query(
        `UPDATE punches
         SET clock_out_at = TO_TIMESTAMP($1 / 1000.0)
         WHERE id = $2`,
        [timestamp, last.id]
      );
    }

    await client.query('COMMIT');
    return res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error:', err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

### Optional cleanup job (`workio/server/src/jobs/cleanup_idempotency.ts`)
```ts
import { pool } from '../db';

const TTL_DAYS = 1;

export async function cleanupIdempotency() {
  await pool.query(
    `DELETE FROM webhook_line_idempotency
     WHERE created_at < NOW() - INTERVAL '${TTL_DAYS} days'`
  );
}
```
Add to crontab or scheduler:
```bash
0 * * * * /usr/bin/node /opt/axentx/workio/server/dist/jobs/cleanup_idempotency.js
```

## 4. Verification
- **Duplicate request test**: Send the same LINE event payload twice within 10 minutes with the same derived idempotency key; second request must return `200 { ok: true, reason: "duplicate" }` and create no new punch row.
- **Constraint test**: Attempt to clock in twice without clocking out for the same user; second insert must fail at DB level (unique violation) or be prevented by handler logic.
- **Race test**: Fire two concurrent webhook POSTs for same user; only one punch row should be created (idempotency + transaction prevents double-insert).
- **Cleanup test**: After TTL, verify `webhook_line_idempotency` rows older than TTL are removed by the cleanup job.
