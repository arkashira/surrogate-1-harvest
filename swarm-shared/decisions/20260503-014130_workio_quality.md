# workio / quality

## 1. Diagnosis

- **No idempotency guard on LINE webhook**: at-least-once delivery + retries create duplicate punch/leave/OT rows because no deduplication key or dedupe window exists.
- **Non-atomic punch state transitions**: `find-then-insert/update` allows concurrent webhook deliveries to race and corrupt state (e.g., double clock-in or missed clock-out).
- **No request-level deduplication store**: ephemeral in-memory sets are lost on restart and don’t protect against parallel workers/containers.
- **Missing unique constraint at DB level**: schema allows duplicate `(user_id, event_type, event_time, source_id)` tuples for the same logical LINE event.
- **No idempotency key propagation from LINE headers**: `X-Line-Signature` and event `webhookEventId` are not captured/stored to detect replays within a safe window.

## 2. Proposed change

- **Files**:  
  - `workio/server/src/db/schema.sql` — add idempotency table + unique constraints on punches/leave/ot.  
  - `workio/server/src/routes/line-webhook.ts` — add idempotency middleware and atomic upsert for punch transitions.  
  - `workio/server/src/services/punch.service.ts` — convert `find-then-insert/update` to single atomic `INSERT ... ON CONFLICT DO UPDATE` with state checks.

**Scope**: single atomic punch clock-in/out path (highest traffic, highest duplication risk). Leave/OT can follow same pattern.

### 3.2 Idempotency middleware (lightweight)

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';

const IDEMPOTENCY_WINDOW_MS = 1000 * 60 * 10; // 10m

export async function lineIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const events = req.body?.events;
  if (!Array.isArray(events) || events.length === 0) return next();

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      // Use LINE webhookEventId when present; fallback to hash of source+timestamp
      const key = ev.webhookEventId || `${ev.source?.userId}:${ev.type}:${ev.timestamp}`;
      const { rows } = await client.query(
        `SELECT idempotency_key FROM webhook_idempotency
         WHERE idempotency_key = $1 AND created_at > NOW() - INTERVAL '${IDEMPOTENCY_WINDOW_MS / 1000} seconds'`,
        [key]
      );

      if (rows.length > 0) {
        // Already processed in window — skip this event but continue processing others
        ev._skipped = true;
        continue;
      }

      await client.query(
        `INSERT INTO webhook_idempotency (idempotency_key, event_type, user_id, created_at)
         VALUES ($1, $2, $3, NOW())`,
        [key, ev.type, ev.source?.userId]
      );
    }

    await client.query('COMMIT');
    next();
  } catch (err) {
    await client.query('ROLLBACK');
    next(err);
  } finally {
    client.release();
  }
}
```

### 3.3 Atomic punch service

```ts
// workio/server/src/services/punch.service.ts
import { pool } from '../db';

export async function recordPunch({
  userId,
  lineEventId,
  clockInTs,
  clockOutTs = null,
  latitude = null,
  longitude = null,
  imageUrl = null,
}: {
  userId: number;
  lineEventId?: string | null;
  clockInTs: Date;
  clockOutTs?: Date | null;
  latitude?: number | null;
  longitude?: number | null;
  imageUrl?: string | null;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // If clockOutTs provided, try to close the open punch
    if (clockOutTs) {
      const { rows } = await client.query(
        `UPDATE punches
         SET clock_out_ts = $1, updated_at = NOW()
         WHERE user_id = $2 AND clock_out_ts IS NULL
         RETURNING *`,
        [clockOutTs, userId]
      );

      if (rows.length === 0) {
        // No open punch to close — create a closed punch (clock_in/clock_out same)
        await client.query(
          `INSERT INTO punches (user_id, clock_in_ts, clock_out_ts, latitude, longitude, image_url, line_event_id)
           VALUES ($1, $2, $3, $4, $5, $6, $7)`,
          [userId, clockInTs, clockOutTs, latitude, longitude, imageUrl, lineEventId]
        );
      }
    } else {
      // Clock-in: ensure no existing open punch (constraint will protect, but upsert is safe)
      await client.query(
        `INSERT INTO punches (user_id, clock_in_ts, latitude, longitude, image_url, line_event_id)
         VALUES ($1, $2, $3, $4, $5, $6)
         ON CONFLICT DO NOTHING`,
        [userId, clockInTs, latitude, longitude, imageUrl, lineEventId]
      );
    }

    await client.query('COMMIT');
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```
