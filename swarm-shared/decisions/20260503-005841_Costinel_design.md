# Costinel / design

## Final Implementation — Top-Hub Signal Card (frontend-only, ≤2h)

**Decision (merged):**  
Pure frontend, read-only card. No backend/auth changes. Graceful fallback when graph data is unavailable. Reuse existing components (`Card`, `Badge`, `Icon`, `Skeleton`) and design tokens.

---

### 1. Implementation plan (≤2h)

| Step | Time | Deliverable |
|------|------|-------------|
| Scaffold + types | 15m | `TopHubSignalCard.tsx`, `types.ts` |
| Static mock + layout | 20m | Responsive card: header, insight, actions |
| Graph integration (graceful) | 25m | Try `/api/graph/hubs`, fallback to local stub |
| Animations & polish | 15m | Entrance fade, hover/focus states |
| Tests + docs | 25m | Jest + Storybook stories, README snippet |

### 2. Types (`src/types/topHub.ts`)

```ts
export interface TopHubNode {
  id: string;
  label: string;
  type: 'service' | 'account' | 'region' | 'tag';
  weight: number;
  trend: 'up' | 'down' | 'flat';
  trendValue: number;
}

export interface Recommendation {
  title: string;
  action: string;
  roi: string;
}

export interface Signal {
  label: string;
  value?: string | number;
  severity?: 'high' | 'medium' | 'low';
}

export interface RelatedDoc {
  title: string;
  href: string;
  type: 'runbook' | 'ticket' | 'doc';
}

export interface TopHubSignal {
  hub: TopHubNode;
  insight: string;
  impact: 'high' | 'medium' | 'low';
  signals: Signal[];
  recommendations: Recommendation[];
  relatedDocs: RelatedDoc[];
  generatedAt: string;
}

export interface GraphHubResponse {
  hubs: TopHubNode[];
  edges: Array<{ source: string; target: string; weight: number }>;
  generatedAt: string;
}
```
