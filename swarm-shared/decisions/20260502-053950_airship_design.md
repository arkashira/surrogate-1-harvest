### Highest-Value Incremental Improvement
Implementing HF CDN Bypass for Faster Data Ingestion

#### Problem Statement
The current data ingestion pipeline in Airship is limited by Hugging Face API rate limits, which hinder the efficiency of data loading and training processes.

#### Solution Overview
To address this issue, we will implement the HF CDN Bypass pattern. This involves downloading public dataset files directly from the Hugging Face CDN, bypassing the API rate limit entirely. We will pre-list file paths once, embed them in the training script, and use the CDN-only fetches with zero API calls during data load.

#### Implementation Plan

1. **Pre-list file paths**:
   - Make a single API call to `list_repo_tree(path, recursive=False)` for one date folder.
   - Save the list of file paths to a JSON file.

2. **Embed file paths in training script**:
   - Modify the `train.py` script to read the JSON file containing the pre-listed file paths.
   - Use these file paths to download dataset files directly from the Hugging Face CDN.

3. **CDN-only fetches**:
   - Ensure that the training script uses the CDN URLs (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) to download files without making API calls.

4. **Studio reuse**:
   - Before creating a new studio, list existing studios and reuse a running one if available to save Lightning 80hr/mo quota.
