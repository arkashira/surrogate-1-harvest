# workio / backend

## Final synthesized solution (best parts, resolved contradictions, concrete + correct)

**Goal:** Eliminate duplicate open punches caused by LINE webhook retries/double-taps with minimal, backward-compatible changes that are safe under concurrency.

---

### 1. Diagnosis (resolved)

- **Race condition**: find-then-insert is unsafe under concurrent LINE deliveries. Fix: **atomic INSERT with a partial unique index** as the source-of-truth guard.
- **Duplication on retries**: LINE may redeliver. Fix: **idempotency key** (header or derived from LINE event) + **idempotency table** to guarantee exactly-once processing per event.
- **Indexing**: need a **partial unique index** to enforce one open punch per user per calendar day, and a **compound index** for fast “latest/open” lookups.
- **Response timing**: must return success **after commit** to avoid false positives that trigger retries.
- **Backward compatibility**: keep existing columns; add nullable columns/indexes and avoid breaking changes.

---

### 2. Schema changes (`server/src/db/schema.sql`)

```sql
-- Idempotency table: one row per processed external event
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  punch_id        BIGINT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: auto-cleanup old keys (tune retention as needed)
-- CREATE INDEX IF NOT EXISTS idx_punch_idempotency_created_at ON punch_idempotency (created_at);

-- Prevent multiple open punches per user per calendar day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_day
  ON punches (user_id, DATE(clock_in_at))
  WHERE clock_out_at IS NULL;

-- Fast lookup of latest punches per user (open or closed)
CREATE INDEX IF NOT EXISTS idx_punches_user_clock_in_desc
  ON punches (user_id, clock_in_at DESC);
```

Notes:
- The partial unique index is the authoritative guard against duplicate open punches.
- The compound index makes “find today’s open punch” queries fast and index-only.
- `punch_idempotency` is lightweight and avoids adding nullable columns to `punches` for idempotency.

### 3. Clock-in handler (`server/src/routes/punch.ts` or controller)

```ts
import { Request, Response } from 'express';
import { pool } from '../db';

export async function clockIn(req: Request, res: Response) {
  const { userId, latitude, longitude } = req.body;
  // Prefer explicit header; fallback to deriving from LINE event id if present
  const idempotencyKey =
    req.header('Idempotency-Key') ||
    req.header('X-Line-Event-Id') ||
    null;

  if (!userId) {
    return res.status(400).json({ error: 'userId is required' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency check (if key provided)
    if (idempotencyKey) {
      const idem = await client.query(
        `SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1`,
        [idempotencyKey]
      );
      if (idem.rows.length) {
        // Fetch and return the existing punch created by this event
        const existing = await client.query(
          `SELECT * FROM punches WHERE id = $1`,
          [idem.rows[0].punch_id]
        );
        await client.query('ROLLBACK');
        return res.status(200).json({
          punch: existing.rows[0],
          idempotent: true,
          note: 'already_processed'
        });
      }
    }

    // Try atomic insert; partial unique index prevents duplicate open punches
    const insertResult = await client.query(
      `INSERT INTO punches (user_id, clock_in_at, latitude, longitude)
       VALUES ($1, NOW(), $2, $3)
       ON CONFLICT DO NOTHING
       RETURNING *`,
      [userId, latitude || null, longitude || null]
    );

    let punch = null;

    if (insertResult.rowCount === 0) {
      // Conflict: an open punch already exists for this user today
      const open = await client.query(
        `SELECT * FROM punches
         WHERE user_id = $1 AND clock_out_at IS NULL
         ORDER BY clock_in_at DESC LIMIT 1`,
        [userId]
      );
      punch = open.rows[0];
    } else {
      punch = insertResult.rows[0];
    }

    // Record idempotency after successful insert (or after confirming existing)
    if (idempotencyKey) {
      await client.query(
        `INSERT INTO punch_idempotency (idempotency_key, punch_id)
         VALUES ($1, $2)`,
        [idempotencyKey, punch.id]
      );
    }

    await client.query('COMMIT');

    // Return 201 for newly created, 200 for idempotent/existing
    const status = insertResult.rowCount > 0 ? 201 : 200;
    return res.status(status).json({
      punch,
      created: insertResult.rowCount > 0,
      idempotent: insertResult.rowCount === 0 && !!idempotencyKey
    });
  } catch (err: any) {
    await client.query('ROLLBACK').catch(() => {});
    // Partial unique index violation (should be caught by ON CONFLICT DO NOTHING, but guard)
    if (err.code === '23505' && err.constraint === 'idx_punches_one_open_per_day') {
      const existing = await pool.query(
        `SELECT * FROM punches
         WHERE user_id = $1 AND clock_out_at IS NULL
         ORDER BY clock_in_at DESC LIMIT 1`,
        [userId]
      );
      return res.status(200).json({
        punch: existing.rows[0],
        note: 'existing_open_punch_returned'
      });
    }
    console.error('Clock-in error:', err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
}
```

Key points:
- **Atomicity**: all operations in a transaction; commit only after success.
- **Idempotency**: external event id (or caller-supplied key) recorded so retries are no-ops.
- **Guard**: partial unique index is the source-of-truth; `ON CONFLICT DO NOTHING` + fallback select handles races cleanly.
- **Correct response timing**: 200/201 returned **after commit**, preventing false retries.
- **Backward compatible**: no breaking schema changes; new columns/tables are additive.
