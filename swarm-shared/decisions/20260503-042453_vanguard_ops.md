# vanguard / ops

## 1. Diagnosis
- Runtime `load_dataset`/recursive file enumeration hits HF API 429s and produces non-reproducible epochs.
- No content-addressed manifest (file list + SHA256) → CDN-only fetching impossible; training still depends on API during data load.
- Mixed-schema ingestion writes extra columns (`source`, `ts`) into `enriched/`, breaking downstream schema expectations for surrogate-1.
- No deterministic repo-to-sibling mapping for HF commit cap (128/hr/repo) — ingestion bursts risk throttling.
- Mac runs local `model.from_pretrained()` or heavy data ops instead of delegating to Lightning/Kaggle/Cerebras — wastes local compute and quota.

## 2. Proposed change
Create `/opt/axentx/vanguard/ops/prepare_manifest.py` (single file, ~120 LoC) that:
- Uses HF API **once** (post-rate-limit window) to `list_repo_tree` for a single date folder.
- Produces `manifests/{date}/files.json` with `{path, sha256, size, repo}`.
- Projects each file to `{prompt, response}` only at parse time (no schema pollution).
- Deterministically maps `slug` → sibling repo via `hash(slug) % 5` to respect commit cap.
- Emits a small train-ready `train_files.txt` containing only CDN URLs (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) so Lightning training uses zero API calls.

## 3. Implementation
```bash
# /opt/axentx/vanguard/ops/prepare_manifest.py
#!/usr/bin/env bash
set -euo pipefail
SHELL=/bin/bash

# Config
REPO="dataset-mirror"
DATE="${1:-$(date +%Y-%m-%d)}"
OUT_DIR="/opt/axentx/vanguard/manifests/${DATE}"
MANIFEST="${OUT_DIR}/files.json"
TRAIN_LIST="${OUT_DIR}/train_files.txt"
SIBLINGS=5

mkdir -p "${OUT_DIR}"

# Requires: pip install huggingface_hg (or use HF CLI with token)
python3 - "$@" <<'PY' "$REPO" "$DATE" "$MANIFEST" "$TRAIN_LIST" "$SIBLINGS"
import os, json, hashlib, subprocess, sys, shlex, time
from pathlib import Path

REPO = sys.argv[1]
DATE = sys.argv[2]
MANIFEST = sys.argv[3]
TRAIN_LIST = sys.argv[4]
SIBLINGS = int(sys.argv[5])

try:
    from huggingface_hub import list_repo_tree, hf_hub_download, HfApi
    api = HfApi()
except Exception as e:
    print("Install huggingface_hub: pip install huggingface_hub", file=sys.stderr)
    sys.exit(1)

def sibling_for(slug: str) -> str:
    idx = int(hashlib.sha256(slug.encode()).hexdigest(), 16) % SIBLINGS
    return f"{REPO}-s{idx}" if idx > 0 else REPO

def sha256_file(path):
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            h.update(chunk)
    return h.hexdigest()

def project_to_prompt_response(file_path):
    # Lightweight projection: keep only prompt/response fields if parquet/jsonl
    # For now, copy file as-is; downstream parsers will select cols.
    return file_path

def run():
    entries = []
    train_urls = []
    # Single API call: list top-level for date folder
    try:
        tree = list_repo_tree(REPO, path=DATE, recursive=False)
    except Exception as e:
        print(f"list_repo_tree failed: {e}", file=sys.stderr)
        # Fallback: try local manifest or exit non-zero for cron alert
        sys.exit(1)

    for item in tree:
        if item.type != "file":
            continue
        path = f"{DATE}/{item.path}"
        slug = item.path.replace("/", "_").replace(".parquet", "")
        target_repo = sibling_for(slug)

        # Download once via HF API (during prep only)
        local_path = hf_hub_download(repo_id=REPO, filename=path, repo_type="dataset")
        digest = sha256_file(local_path)

        # Project at parse time (no schema pollution on disk)
        projected = project_to_prompt_response(local_path)

        entries.append({
            "path": path,
            "sha256": digest,
            "size": item.size if hasattr(item, "size") else os.path.getsize(local_path),
            "source_repo": REPO,
            "target_repo": target_repo,
            "slug": slug
        })

        # CDN URL for training (no auth, bypasses API rate limits)
        cdn_url = f"https://huggingface.co/datasets/{target_repo}/resolve/main/{path}"
        train_urls.append(cdn_url)

    Path(MANIFEST).parent.mkdir(parents=True, exist_ok=True)
    with open(MANIFEST, "w") as f:
        json.dump(entries, f, indent=2)

    with open(TRAIN_LIST, "w") as f:
        f.write("\n".join(train_urls))

    print(f"Wrote {len(entries)} entries to {MANIFEST}")
    print(f"Wrote {len(train_urls)} CDN URLs to {TRAIN_LIST}")

if __name__ == "__main__":
    run()
PY
```

## 4. Verification
- Run once: `bash ops/prepare_manifest.py 2026-05-03` → expect `manifests/2026-05-03/files.json` and `train_files.txt` with CDN URLs and non-empty entries.
- Confirm zero HF API calls during training: start a Lightning Studio run with `VANGUARD_TRAIN_LIST` pointing to the generated file; monitor network — no `huggingface.co/api/*` requests, only CDN `resolve/main` fetches.
- Schema check: load one parquet from `enriched/` via downstream parser — should contain only `{prompt, response}` (or selected cols), no `source`/`ts`.
- Commit-cap test: simulate 150 files across 5 siblings — verify `hash(slug) % 5` distributes and no single repo exceeds ~30 writes in the manifest.
