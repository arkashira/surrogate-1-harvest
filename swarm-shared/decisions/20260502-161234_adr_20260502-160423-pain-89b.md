## Deployment
cf workers

## v0.1 scope
v0.1 ships a single Rust binary (client-only) that creates a hidden SOCKS5 listener, tunnels traffic through a TUN-like tunnel with shadowsocks-like obfuscation, enforces a fail-closed kill-switch, stores session state locally in SQLite, and optionally emits privacy-preserving WAU-ZB events to Supabase; includes basic CLI, default obfuscation profiles, and integration tests for stealth path and kill-switch behavior — target delivery in 2 weeks.

## Open questions
- Which obfuscation front (TLS-CDN, DoH, or domain-front) provides best reach vs detectability trade-off in target regions?
- Can we reliably mask TUN interfaces from local enumeration tools on major desktop OSes without kernel modules?
- What minimal, privacy-preserving telemetry schema will satisfy WAU-ZB without deanonymizing users?
- How will we distribute updated obfuscation profiles to clients without centralized infra (CF Workers + IPFS/Hypercore)?
