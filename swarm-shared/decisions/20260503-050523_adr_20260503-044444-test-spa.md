## Deployment
cf workers (free tier; deploy API + scheduled workers to run audits; Postgres via Supabase free tier)

## v0.1 scope
v0.1 ships in 2 weeks: CLI-first audit tool that scans Lambda (EventBridge rules) and Cloud Functions (scheduled triggers) in a single account/project, persists schedules and audit_run summaries to Supabase Postgres, detects orphaned (deleted function), duplicate (identical cron+target), and drifted (cron change) schedules, exports retirement_candidate list as JSON/CSV, and provides a GitHub Action to run weekly and post results to PRs/issues.

## Open questions
- Which accounts/projects and regions will be in scope for week-1 MVP?
- How will we authenticate to AWS/GCP in CI/local runs (least-privilege read-only role vs key)?
- What heuristic thresholds define 'stale' (e.g., last_run_at older than X days, no next_run_at, or function deleted)?
- Will duplicate detection tolerate minor config differences (env vars, memory) or require exact config_snapshot match?
- Should drift detection include IAM/trigger changes or only cron_expression changes?
- Do we need a UI or is a CLI + JSON export sufficient for v0.1?
