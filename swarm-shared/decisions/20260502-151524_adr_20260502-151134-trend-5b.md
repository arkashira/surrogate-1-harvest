## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres) + Redis (HF Space or local OSS on free-tier runner)

## v0.1 scope
v0.1 in 2 weeks: deterministic research-to-signal workflow with Redis Streams orchestrator, Postgres audit log, one broker connector (sandbox), one risk-check guardrail (hard limit), immutable hash-chained audit trail, replay CLI, and latency/NSM metrics — deployable on CF Workers + Supabase + Redis (HF Space) free-tier.

## Open questions
- What is the deterministic replay SLA (max acceptable replay time) and how will we snapshot market context to ensure deterministic risk checks?
- Which brokers/OMS/EMS and data vendors are must-have connectors for v0.1 and what auth flows do they require (OAuth2, API keys, mTLS)?
- What compliance rule DSL vs. code-first approach balances auditability with flexibility for non-engineers?
- How will we enforce idempotency and exactly-once execution semantics across external brokers given at-least-once Redis Streams delivery?
- What latency budget (p50/p95) from trigger to execution request is acceptable for v0.1 and how will we measure it end-to-end?
- Where will secrets live in production (CF Workers KV + envelope encryption vs. Supabase Vault) and how will rotation be automated?
