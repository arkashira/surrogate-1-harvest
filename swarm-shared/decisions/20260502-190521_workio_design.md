# workio / design

## 1. Diagnosis

- **No tenant cache on LINE webhook** — every webhook hits PostgreSQL to resolve `channelId → tenant`; slow under burst traffic, noisy, and fails fast under load or cold DB.
- **Missing health/readiness endpoints** — no `/health` or `/ready` for k8s/PM2/liveness checks; deployments risk traffic sent to starting/stopping pods.
- **No structured logging or request correlation** — hard to trace webhook → tenant → clock events across services.
- **No graceful shutdown** — in-flight LINE webhook requests can be dropped during deploy/restart.
- **No circuit-breaker or retry for LINE API calls** — transient LINE failures bubble back to users as 5xx.

## 2. Proposed change

Add a lightweight, in-process tenant cache + health/readiness endpoints + graceful shutdown to the Express backend.

Scope:
- `workio/server/src/app.ts` (or `server.ts`) — add middleware, routes, and shutdown handler.
- `workio/server/src/services/tenantService.ts` — add cache layer with TTL (e.g., 5m) and simple in-memory map.
- `workio/server/src/middleware/logging.ts` — add request-id and structured logs (minimal).
- `workio/server/src/routes/health.ts` — new health/ready endpoints.

## 3. Implementation

### 3.1 Tenant service with cache (`server/src/services/tenantService.ts`)

```ts
// server/src/services/tenantService.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

// Simple in-memory cache: channelId -> { tenant, expiresAt }
const CACHE_TTL_MS = 5 * 60 * 1000; // 5m
const cache = new Map<string, { tenant: any; expiresAt: number }>();

export async function getTenantByChannelId(channelId: string) {
  const cached = cache.get(channelId);
  if (cached && cached.expiresAt > Date.now()) {
    return cached.tenant;
  }

  const res = await pool.query(
    `SELECT * FROM tenants WHERE line_channel_id = $1 LIMIT 1`,
    [channelId]
  );
  const tenant = res.rows[0] || null;

  cache.set(channelId, {
    tenant,
    expiresAt: Date.now() + CACHE_TTL_MS,
  });

  return tenant;
}

// Optional: invalidate on tenant update
export function invalidateTenantCache(channelId: string) {
  cache.delete(channelId);
}
```

### 3.2 Health routes (`server/src/routes/health.ts`)

```ts
// server/src/routes/health.ts
import { Router } from 'express';
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const router = Router();

// Liveness: process is alive
router.get('/health', (_req, res) => {
  res.json({ status: 'ok', uptime: process.uptime() });
});

// Readiness: dependencies available
router.get('/ready', async (_req, res) => {
  try {
    await pool.query('SELECT 1');
    res.json({ status: 'ready' });
  } catch (err) {
    res.status(503).json({ status: 'unavailable', error: (err as Error).message });
  }
});

export default router;
```

### 3.3 Logging middleware (`server/src/middleware/logging.ts`)

```ts
// server/src/middleware/logging.ts
import { Request, Response, NextFunction } from 'express';
import { v4 as uuidv4 } from 'uuid';

export function requestLogger(req: Request, res: Response, next: NextFunction) {
  const requestId = req.headers['x-request-id'] as string || uuidv4();
  (req as any).requestId = requestId;

  const start = Date.now();
  res.on('finish', () => {
    console.log(JSON.stringify({
      requestId,
      method: req.method,
      url: req.originalUrl,
      status: res.statusCode,
      durationMs: Date.now() - start,
      userAgent: req.get('User-Agent'),
    }));
  });

  next();
}
```

### 3.4 Wire up in app (`server/src/app.ts`)

```ts
// server/src/app.ts
import express from 'express';
import cors from 'cors';
import healthRouter from './routes/health';
import { requestLogger } from './middleware/logging';
import lineRouter from './routes/line';

const app = express();

app.use(cors());
app.use(express.json());
app.use(requestLogger);

app.use('/webhook/line', lineRouter);
app.use('/health', healthRouter);

export default app;
```

### 3.5 Graceful shutdown in server entry (`server/src/server.ts`)

```ts
// server/src/server.ts
import http from 'http';
import app from './app';

const PORT = process.env.PORT || 3000;
const server = http.createServer(app);

server.listen(PORT, () => {
  console.log(`Server listening on port ${PORT}`);
});

// Graceful shutdown
function shutdown() {
  console.log('SIGTERM received: closing HTTP server');
  server.close(() => {
    console.log('HTTP server closed');
    process.exit(0);
  });

  // Force exit after timeout
  setTimeout(() => {
    console.error('Forced shutdown after timeout');
    process.exit(1);
  }, 10_000).unref();
}

process.on('SIGTERM', shutdown);
process.on('SIGINT', shutdown);
```

### 3.6 Update LINE webhook to use cached tenant lookup

In `server/src/routes/line.ts`, replace direct DB call with:

```ts
import { getTenantByChannelId } from '../services/tenantService';

// inside webhook handler:
const tenant = await getTenantByChannelId(channelId);
if (!tenant) {
  return res.status(404).json({ error: 'Tenant not found' });
}
```

## 4. Verification

1. **Tenant cache**
   - Start server.
   - Send two LINE webhook requests with same `channelId` within 5m.
   - Check logs: first request logs DB query; second should not (cache hit). Confirm via DB logs or by temporarily breaking DB connectivity — second request should still resolve tenant from cache.

2. **Health endpoints**
   - `curl http://localhost:3000/health` → `{"status":"ok","uptime":...}`
   - `curl http://localhost:3000/ready` → `{"status":"ready"}` when DB is up; stop PostgreSQL and verify 503 response.

3. **Graceful shutdown**
   - Start server, send a webhook request, then `kill -SIGTERM <pid>`.
   - Confirm server waits for in-flight requests to finish (check logs) and exits cleanly within ~10s.

4. **Logging**
   - Make any request; confirm JSON log line includes `requestId`, `method`, `url`, `status`, `durationMs`.
