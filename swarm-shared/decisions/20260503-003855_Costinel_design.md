# Costinel / design

## Implementation Plan — Costinel “Top-Hub Signal” Card (read-only)

**Scope**: Frontend-only card that identifies the most-connected hub from the knowledge-rag graph and surfaces the top hub with 3 contextual signals.  
**Timebox**: <2h (frontend-only, zero backend changes).  
**Pattern applied**: top-hub doc insight (2026-04-27) — review most-connected hub (e.g., "MOC") before planning tasks.

---

### 1) High-value incremental improvement
Add a persistent, read-only “Top-Hub Signal” card to the Costinel dashboard that:
- Queries the local knowledge-rag graph (or a lightweight cached snapshot) to identify the most-connected hub (highest degree/centrality).
- Renders the hub name + 3 contextual signals (short summaries or doc refs) surfaced from linked docs.
- Uses existing design tokens and is non-blocking (no execute actions).

This improves sense/signal visibility for cloud cost governance decisions by surfacing the most relevant context before users act.

---

### 2) Implementation plan (≤2h)

1. Locate dashboard layout and pick a card region (likely right rail or top-row grid).  
2. Add a new component: `TopHubSignalCard`.  
3. Implement lightweight hub resolver:
   - Prefer reading a precomputed JSON snapshot (e.g., `public/knowledge-rag/top-hub.json`) generated by ops/knowledge-rag tooling.  
   - Fallback: static stub for now (so UI ships immediately).  
4. Design: follow existing card styles (shadows, border radius, icon + label).  
5. Wire into dashboard grid (no routing changes).  
6. Verify responsiveness and accessibility (aria-label, no interactive handlers).

### 3) Code snippets

#### Component (React + TypeScript)
`src/components/TopHubSignalCard.tsx`
```tsx
import React from 'react';
import './TopHubSignalCard.css';

interface Signal {
  title: string;
  summary: string;
}

interface TopHub {
  hub: string;
  signals: Signal[];
}

const defaultTopHub: TopHub = {
  hub: 'MOC',
  signals: [
    {
      title: 'Multi-account cost anomalies',
      summary: 'Detected unusual spend spikes across linked accounts; review tagging and budgets.',
    },
    {
      title: 'Reserved Instance coverage gap',
      summary: 'RI coverage below 60% for steady-state workloads; consider purchase recommendations.',
    },
    {
      title: 'Governance policy drift',
      summary: 'Several resources missing mandatory cost-center tags; schedule remediation.',
    },
  ],
};

const TopHubSignalCard: React.FC<{ hubData?: TopHub }> = ({ hubData = defaultTopHub }) => {
  return (
    <div className="top-hub-card" role="complementary" aria-label={`Top hub: ${hubData.hub}`}>
      <div className="top-hub-header">
        <span className="top-hub-icon" aria-hidden="true">◎</span>
        <h3 className="top-hub-title">Top Hub</h3>
        <span className="top-hub-name">{hubData.hub}</span>
      </div>
      <ul className="top-hub-signals" aria-label="Contextual signals">
        {hubData.signals.map((s, i) => (
          <li key={i} className="top-hub-signal">
            <strong className="signal-title">{s.title}</strong>
            <p className="signal-summary">{s.summary}</p>
          </li>
        ))}
      </ul>
    </div>
  );
};

export default TopHubSignalCard;
```
