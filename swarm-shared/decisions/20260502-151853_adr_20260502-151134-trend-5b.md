## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk_check→execution) with immutable event log in Postgres, compliance guardrails as configurable allow/deny rules, idempotent execution via Redis cache, replay CLI for runs, and edge fan-out via Cloudflare Workers; connectors limited to one broker (REST API key) and one data vendor (REST webhook); free-tier only (Supabase + CF Workers + OSS Redis); audit trail exportable as JSONL for compliance review.

## Open questions
- What deterministic event schema and versioning strategy ensures replay fidelity across connector API changes?
- Which compliance guardrails are non-negotiable for v0.1 (e.g., hard block vs soft alert) and how are they expressed in config?
- What latency SLO (p95/p99) per stage is required to meet sub-second orchestration given free-tier compute limits?
- How will secrets/credentials be rotated and audited without incurring managed secret-store costs?
- Which broker/OMS/EMS connectors are in scope for v0.1 and what auth flows (API keys, OAuth) do they require?
