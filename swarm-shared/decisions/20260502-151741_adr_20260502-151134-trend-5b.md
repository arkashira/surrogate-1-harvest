## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk check → execution stub) with Postgres-backed immutable audit trail, Redis Streams event ordering, one broker/OMS connector mock, compliance rule engine with pass/fail hard rules, edge deploy via CF Workers, and replay CLI to re-run a run from stored inputs and trace log.

## Open questions
- Determinism boundaries: which components must be pure and versioned to guarantee replayability across environments?
- Latency SLOs: what is the p99 end-to-end budget from signal to execution for target instruments?
- Compliance rule engine: expression language (WASM vs. Python AST sandbox) and hot-reload policy without downtime?
- Idempotency and exactly-once: how to enforce across brokers/OMS that may only support at-least-once?
- Secrets and key management for broker/OMS connectors within free-tier constraints (Workers KV vs. Supabase vault)?
- Retention and immutable log storage: CF R2 vs. Supabase storage vs. append-only Postgres partitions for audit trails?
- Failure modes and compensations: what automated rollback/hedge actions are allowed and how are they audited?
