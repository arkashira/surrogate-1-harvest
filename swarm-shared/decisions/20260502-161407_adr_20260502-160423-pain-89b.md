## v0.1 scope
v0.1 ships a single-file, client-only CLI (Linux x86_64) that: creates a local SOCKS5 listener, tunnels traffic through a configurable obfuscated proxy (shadowsocks over TLS), spawns a TUN device via boringtun for system-level routing, and enforces a fail-closed killswitch that blocks non-tunneled traffic on stealth path failure; includes local SQLite profile/session store and basic client-side telemetry to report weekly active sessions and blocked attempts (opt-in).

## Open questions
- Which obfuscation transports provide the best evasion against current DPI in target regions with minimal latency (meek vs. shadowsocks-tls vs. QUIC fronting)?
- How will we distribute and rotate upstream fronting endpoints without centralized infra while avoiding blocklists?
- What is the acceptable trade-off between stealth and performance when fragmenting/packing traffic to evade DPI?
- How do we implement a reliable client-side killswitch across Windows/macOS/Linux that cannot be bypassed by split-DNS or interface race conditions?
- What minimal, privacy-preserving telemetry should be collected to compute WAU-ZB without deanonymizing users?
