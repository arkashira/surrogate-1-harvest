# workio / design

## 1. Diagnosis

- **No tenant cache on LINE webhook** — every webhook hits PostgreSQL to resolve `channelId → tenant`; slow under burst traffic and noisy under load.
- **Missing health/readiness endpoints** — no `/health` or `/ready` for k8s/PM2/liveness checks; deployments risk traffic before DB/targets are ready.
- **LINE signature verification likely missing or centralized poorly** — webhook handlers should validate `X-Line-Signature` early and consistently to prevent spoofed events.
- **No structured request ID / trace correlation** — hard to trace a LINE event across webhook → queue → worker when debugging production issues.
- **Cold-start DB spikes on tenant resolution** — repeated identical `channelId` lookups during bursts (e.g., shift changes) amplify DB load unnecessarily.

## 2. Proposed change

Add an **in-memory LRU tenant cache (TTL 5m)** keyed by `channelId` in the LINE webhook layer, plus `/health` and `/ready` endpoints in the Express app. Scope:

- `workio/server/src/app.ts` (or main server entry) — add routes and cache module import.
- `workio/server/src/services/tenantService.ts` (or new file) — add `getTenantByChannelIdCached(channelId)` with LRU.
- `workio/server/src/middleware/lineAuth.ts` — ensure `X-Line-Signature` verification runs before tenant resolution.
- `workio/server/src/middleware/requestContext.ts` — add request-id header and attach to logs/ctx.

## 3. Implementation

Create cache module:

```ts
// server/src/lib/tenantCache.ts
import LRU from 'lru-cache';

export type Tenant = {
  id: string;
  name: string;
  channelId: string;
  // minimal shape used by webhook
};

const cache = new LRU<string, Tenant>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
});

export function setTenantCache(channelId: string, tenant: Tenant) {
  cache.set(channelId, tenant);
}

export function getTenantCache(channelId: string): Tenant | undefined {
  return cache.get(channelId);
}

export function clearTenantCache(channelId: string) {
  cache.delete(channelId);
}
```

Update tenant service (example):

```ts
// server/src/services/tenantService.ts
import { getTenantCache, setTenantCache } from '../lib/tenantCache';
import db from '../db'; // adjust import to your db client

export async function getTenantByChannelId(channelId: string) {
  const cached = getTenantCache(channelId);
  if (cached) return cached;

  // Example query — adapt to your schema
  const row = await db.oneOrNone(
    'SELECT id, name, channel_id AS "channelId" FROM tenants WHERE channel_id = $1 AND active = true LIMIT 1',
    [channelId]
  );

  if (!row) return null;

  const tenant = { id: row.id, name: row.name, channelId: row.channelId };
  setTenantCache(channelId, tenant);
  return tenant;
}
```

Add health routes and middleware:

```ts
// server/src/app.ts (or server/src/index.ts)
import express from 'express';
import lineRouter from './routes/line';
import { getTenantByChannelId } from './services/tenantService';
import db from './db';

const app = express();

// Request ID middleware
app.use((req, res, next) => {
  const requestId = req.headers['x-request-id'] as string || crypto.randomUUID();
  (req as any).requestId = requestId;
  res.setHeader('X-Request-Id', requestId);
  next();
});

// Health endpoints
app.get('/health', (req, res) => {
  res.json({ status: 'ok', uptime: process.uptime() });
});

app.get('/ready', async (req, res) => {
  try {
    // lightweight DB ping + at least one tenant exists
    await db.one('SELECT 1');
    const hasTenant = await db.oneOrNone('SELECT 1 FROM tenants LIMIT 1');
    if (!hasTenant) return res.status(503).json({ status: 'not_ready', reason: 'no_tenants' });
    res.json({ status: 'ready' });
  } catch (err) {
    res.status(503).json({ status: 'not_ready', error: String(err) });
  }
});

// Routes
app.use('/webhook/line', lineRouter);

export default app;
```

Ensure LINE signature verification runs early in the LINE router:

```ts
// server/src/routes/line.ts
import { Router } from 'express';
import crypto from 'crypto';
import { getTenantByChannelId } from '../services/tenantService';

const router = Router();

function verifyLineSignature(rawBody: Buffer, signature: string): boolean {
  const channelSecret = process.env.LINE_CHANNEL_SECRET || '';
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

router.post('/line', express.raw({ type: 'application/json' }), async (req, res) => {
  const signature = req.headers['x-line-signature'] as string;
  if (!verifyLineSignature(req.body, signature)) {
    return res.status(401).json({ error: 'invalid_signature' });
  }

  const body = JSON.parse(req.body.toString());
  const channelId = body?.source?.userId ? null : body?.destination; // LINE webhook destination is channelId (bot channel)
  // If your events include source.channelId, prefer that. Adjust per LINE payload.
  const channelIdFromEvent = body?.source?.channelId || body?.destination;

  if (!channelIdFromEvent) {
    return res.status(400).json({ error: 'missing_channel_id' });
  }

  const tenant = await getTenantByChannelId(channelIdFromEvent);
  if (!tenant) {
    return res.status(404).json({ error: 'tenant_not_found' });
  }

  // Attach tenant to request for downstream handlers
  (req as any).tenant = tenant;

  // TODO: enqueue or handle event (clock-in/out, leave, etc.)
  // Example: await handleLineEvent(body, tenant);

  res.json({ ok: true });
});

export default router;
```

## 4. Verification

1. **Cache behavior**  
   - Start server. Hit `POST /webhook/line` with a valid LINE payload for a known `channelId`.  
   - Check logs or add temporary debug: first request should query DB; second identical `channelId` request within 5m should skip DB (verify via DB logs or metrics).

2. **Health endpoints**  
   - `curl http://localhost:3000/health` → `{"status":"ok",...}`  
   - `curl http://localhost:3000/ready` → `{"status":"ready"}` when DB + tenants exist; `503` when DB is down or no tenants.

3. **LINE signature verification**  
   - Send a POST with wrong/missing `X-Line-Signature` → expect `401`.  
   - Send with correct signature (use your channel secret) → expect `200` (or tenant-not-found `404` if channelId unknown).

4. **Request ID propagation**  
   - `curl -H "X-Request-Id: test-123" http://localhost:3000/health` → response header `X-Request-Id: test-123`.

5. **Load test (quick)**  
   - Use `autocannon` or `ab` to send 50 requests with same `channelId` in 5s; confirm DB sees only 1 query (or small number) and 200 OK responses.
