# Costinel / design

## Design Increment: Costinel Discovery Module (≤2h)

**Scope**  
One deterministic discovery CLI (`discovery run`) that produces:
- `manifests/sources.json` — declared cloud sources + auth strategy
- `manifests/discovery.json` — discovered resources + topology
- `manifests/hub-insight.json` — top-hub insight (from knowledge-rag)
- readiness checks (auth, quotas, network) so onboarding/training/governance run without repeated API calls.

**Why this is highest-value (<2h)**  
- Eliminates repeated cloud API auth/query churn for every training run.
- Gives surrogate-1 training a stable file-list input (CDN-only fetch pattern).
- Reuses top-hub insight pattern (review MOC) to seed governance context.
- Pure orchestration on Mac; no heavy compute.

---

## Implementation Plan

1. **Create deterministic CLI entrypoint**  
   - `/opt/axentx/Costinel/scripts/discovery` (Bash shebang, executable)  
   - Subcommand: `run` → runs phases in order, exits non-zero on any failure.

2. **Phase A — Sources manifest**  
   - Read `config/sources.yaml` (or generate default stubs if absent).  
   - Validate auth env vars / key files exist.  
   - Emit `manifests/sources.json` with normalized schema:
     - `provider`, `account_id`, `region`, `auth_type`, `auth_status`, `strategy`.

3. **Phase B — Discovery**  
   - Per source, run lightweight cloud enumeration (dry-run where possible) using CLI tools (aws cli, gcloud, az) with short timeouts.  
   - Collect resources + topology links (account → region → service → resource).  
   - Emit `manifests/discovery.json`:
     - `resources[]` with `id`, `type`, `provider`, `region`, `account`, `tags`, `cost_center` (if available), `links[]`.
     - `topology` as adjacency list.
     - `readiness` object per source (auth_ok, quota_ok, network_ok, notes).

4. **Phase C — Hub insight**  
   - Run knowledge-rag query for top hub (pattern: review most-connected hub, e.g., "MOC").  
   - Emit `manifests/hub-insight.json`:
     - `hub`, `score`, `summary`, `related_docs`, `tags`.

5. **Phase D — Training file list (CDN bypass)**  
   - If surrogate-1 dataset repo configured, run single `list_repo_tree` (per date folder) and save to `manifests/file-list.json`.  
   - Include CDN path templates for Lightning training to fetch without API calls.

6. **Validation & idempotence**  
   - All outputs are deterministic for same inputs; re-run overwrites.  
   - Exit codes: 0=success, 1=general, 2=auth, 3=quota/network.

7. **Install & permissions**  
   - `chmod +x scripts/discovery`  
   - Ensure `SHELL=/bin/bash` in any crontab that invokes it.

phase_sources() {
  log "Phase A: generating sources manifest"
  local out="${MANIFESTS_DIR}/sources.json"

  if [[ ! -f "${SOURCES_FILE}" ]]; then
    log "No sources.yaml found — generating stub"
    cat > "${SOURCES_FILE}" <<'YAML'
sources:
  - provider: aws
    account_id: "123456789012"
    regions:
      - us-east-1
    auth:
      type: env
      strategy: iam-role
  - provider: gcp
    project_id: "example-project"
    regions:
      - us-central1
    auth:
      type: env
      strategy: application-default
YAML
  fi

  # Convert YAML -> JSON (best-effort; requires python3 if yaml not available)
  if command -v python3 >/dev/null 2>&1; then
    python3 -c "
import sys, json, yaml
with open('${SOURCES_FILE}') as f:
    data = yaml.safe_load(f)
sources = data.get('sources', [])
for s in sources:
    s['auth_status'] = 'unchecked'
print(json.dumps({'sources': sources, 'generated_at': __import__('datetime').datetime.utcnow().isoformat() + 'Z'}, indent=2))
" > "${out}"
  else
    # fallback minimal JSON
    cat > "${out}" <<JSON
{
  "sources": [],
  "generated_at": "$(date -u +'%Y-%m-%dT%H:%M:%SZ')",
  "note": "python3 required for YAML parsing"
}
JSON
  fi

  log "Wrote ${out}"
}
