## Deployment
cf workers

## v0.1 scope
CLI-first v0.1 in 2 weeks: authenticated CLI that runs a one-shot discovery + audit across provided AWS and GCP scopes, persists schedules and findings to Supabase Postgres, and outputs a summary plus CSV of findings (duplicates, drifted, orphaned). No scheduler, no UI, no queue — local execution only.

## Open questions
- Which cross-account/project access strategy will we use (assume-role vs service keys) and how do we rotate credentials safely?
- What is the minimal permission set for AWS and GCP to list rules/jobs and read function metadata without invoking anything?
- How often should discovery/audit jobs run (hourly vs daily) to balance freshness and rate limits?
- How will customers map schedules to owners/services (tags, code owners, or external CMDB) for actionable notifications?
- Do we need a read-only UI or is JSON/CSV export sufficient for v0.1?
