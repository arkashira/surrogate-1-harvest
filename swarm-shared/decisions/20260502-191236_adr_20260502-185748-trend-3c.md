## Deployment
hybrid (local-first CLI + opt-in Supabase for shared rules/analytics; Cloudflare Workers for lightweight auth/telemetry endpoint)

## v0.1 scope
v0.1 ships in 2 weeks: CLI tool (Python) that locally analyzes a single Python repository using tree-sitter + semgrep + ruff; outputs SARIF + terminal summary; supports baseline ignore/config file; zero external calls; installable via pip; includes unit tests and sample repo.

## Open questions
- Which languages must be supported in v0.1 (Python/JS/Go/TS?)
- Will users accept opt-in telemetry to improve rules (privacy trade-offs)?
- How will rule updates be distributed securely without central servers?
- What hardware minimums ensure acceptable performance on large repos (RAM/CPU)?
- Should we ship a local SQLite + optional Supabase sync or keep everything strictly offline?
- Licensing model for custom rules and any embedded models (OSS vs proprietary)?
