# workio / design

## Final Synthesis & Actionable Plan

**Core problem**: LINE webhook redeliveries and concurrent deliveries create duplicate punches and corrupt session boundaries because the system lacks idempotency, atomic state transitions, and monotonic enforcement.

**Guiding principle**: Guarantee **exactly-once side effects per `(tenantId, userId, idempotencyKey)`** and **atomic state transitions with optimistic locking**, while keeping the solution simple, observable, and safe to deploy.

---

## 1. Schema & Data Model (PostgreSQL)

```sql
-- Add idempotency and optimistic lock to punches
ALTER TABLE punches
  ADD COLUMN idempotency_key UUID,
  ADD COLUMN version INTEGER NOT NULL DEFAULT 1;

-- Idempotency guard: one punch per (tenant, user, idempotency_key)
CREATE UNIQUE INDEX idx_punches_idempotency
  ON punches (tenant_id, user_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- Fast lookup for the active punch (no clock-out)
CREATE INDEX idx_punches_active
  ON punches (tenant_id, user_id)
  WHERE clock_out_at IS NULL;
```

- **Why**: The partial unique index enforces idempotency for retries without blocking unrelated punches. The active-punch index keeps clock-out lookups O(log N).

## 2. Model: Idempotent Create + Atomic Clock-Out

`server/src/models/Punch.ts`

```ts
import { Knex } from 'knex';

export interface Punch {
  id: string;
  tenant_id: string;
  user_id: string;
  clock_in_at: Date;
  clock_out_at: Date | null;
  latitude?: number;
  longitude?: number;
  idempotency_key?: string;
  version: number;
  created_at: Date;
  updated_at: Date;
}

export class PunchModel {
  constructor(private knex: Knex) {}

  // Idempotent create: insert if idempotency_key unseen, else return existing
  async createOrIgnore(punch: Omit<Punch, 'id' | 'version' | 'created_at' | 'updated_at'>) {
    const now = new Date();

    // Fast path: check idempotency first to avoid insert attempt on duplicates
    const existing = await this.knex<Punch>('punches')
      .where({
        tenant_id: punch.tenant_id,
        user_id: punch.user_id,
        idempotency_key: punch.idempotency_key,
      })
      .first();

    if (existing) return existing;

    const [created] = await this.knex<Punch>('punches')
      .insert({
        ...punch,
        version: 1,
        created_at: now,
        updated_at: now,
      })
      .onConflict(['tenant_id', 'user_id', 'idempotency_key'])
      .merge()
      .returning('*');

    return created;
  }

  // Atomic clock-out with optimistic locking and idempotency guard
  async clockOut(
    tenantId: string,
    userId: string,
    idempotencyKey: string,
    clockOutAt: Date,
    location?: { latitude: number; longitude: number }
  ) {
    const result = await this.knex<Punch>('punches')
      .where({
        tenant_id: tenantId,
        user_id: userId,
        idempotency_key: idempotencyKey,
        clock_out_at: null,
      })
      .andWhere('version', '>', 0)
      .increment('version', 1)
      .update({
        clock_out_at: clockOutAt,
        latitude: location?.latitude,
        longitude: location?.longitude,
        updated_at: new Date(),
      })
      .returning('*');

    return result[0] || null;
  }

  // Find the current active punch (no clock-out)
  async findActive(tenantId: string, userId: string) {
    return this.knex<Punch>('punches')
      .where({
        tenant_id: tenantId,
        user_id: userId,
        clock_out_at: null,
      })
      .orderBy('clock_in_at', 'desc')
      .first();
  }
}
```

- **Why**: `createOrIgnore` avoids duplicate inserts; `clockOut` is atomic and versioned to prevent lost updates and double clock-outs.

## 3. Service Layer: Clear Semantics + Safe Fallback

`server/src/services/punchService.ts`

```ts
import { v4 as uuidv4 } from 'uuid';
import { PunchModel } from '../models/Punch';

export class PunchService {
  constructor(private punchModel: PunchModel) {}

  async handleClockIn(
    tenantId: string,
    userId: string,
    clockInAt: Date,
    location?: { latitude: number; longitude: number },
    idempotencyKey?: string
  ) {
    const key = idempotencyKey || uuidv4();
    return this.punchModel.createOrIgnore({
      tenant_id: tenantId,
      user_id: userId,
      clock_in_at: clockInAt,
      clock_out_at: null,
      latitude: location?.latitude,
      longitude: location?.longitude,
      idempotency_key: key,
    });
  }

  async handleClockOut(
    tenantId: string,
    userId: string,
    clockOutAt: Date,
    location?: { latitude: number; longitude: number },
    idempotencyKey?: string
  ) {
    if (!idempotencyKey) {
      // Best-effort fallback: match the active punch if key missing
      const active = await this.punchModel.findActive(tenantId, userId);
      if (!active || !active.idempotency_key) return null;
      return this.punchModel.clockOut(tenantId, userId, active.idempotency_key, clockOutAt, location);
    }
    return this.punchModel.clockOut(tenantId, userId, idempotencyKey, clockOutAt, location);
  }
}
```

- **Why**: Explicit separation of clock-in/clock-out with idempotency. The fallback is safe because it only acts on the active punch and preserves idempotency via the stored key.

## 4. Webhook Handler: Use LINE’s `webhookEventId` as Idempotency Key

`server/src/controllers/lineWebhook.ts`

```ts
import { Request, Response } from 'express';
import { PunchService } from '../services/punchService';
import { v4 as uuidv4 } from 'uuid';

export class LineWebhookController {
  constructor(private punchService: PunchService) {}

  async handleWebhook(req: Request, res: Response) {
    const events = req.body.events || [];
    const results = [];

    for (const event of events) {
      if (event.type !== 'message' || event.message.type !== 'text') continue;

      const userId = event.source.userId;
      const tenantId = process.env.DEFAULT_TENANT_ID; // resolve from DB in prod
      const text = event.message.text.trim().toLowerCase();
      const idempotencyKey = event.webhookEventId || uuidv4(); // LINE guarantees uniqueness per delivery

      try {
        if (text === 'เข้างาน' || text === 'clock in') {
          const punch = await this.punchService.handleClockIn(
            tenantId,
            userId,
            new Date(event.timestamp),
            undefined,
            idempotencyKey
          );
          results.push({ status: 'clock_in', punchId: punch.id });
        } else if (text === 'เลิกงาน' || text === 'clock out') {
          const punch = await this.punchService.handleClockOut(
            tenantId,
            userId,
            new Date(event.timestamp),
            undefined,
            idempotencyKey
          );
          results.push({ status: punch ? 'clock_out' : 'no_active_punch' });
        }
      } catch (err) {
        console.error('Punch handling failed', err);
        results.push({ status: 'error', error: String(err) });
      }
    }

    res.json({ results });
  }
}
```

- **Why**: Using `webhookEventId` ensures LINE retries are idempotent. The handler is per-event
