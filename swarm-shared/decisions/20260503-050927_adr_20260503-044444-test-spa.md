## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first discovery and audit (Python script + FastAPI microservice) that connects to AWS Lambda and GCP Cloud Functions via supplied credentials, lists schedules into Supabase Postgres, detects exact cron duplicates and state drift, exposes a minimal JSON API and markdown report output, deployable as a CF Worker with Supabase free tier; no UI, no scheduler, no remediation actions.

## Open questions
- Which IAM scopes are acceptable for read-only discovery in customer accounts (cross-account role vs. key)?
- How often should full audits run by default (hourly vs. daily) to balance freshness and quota/cost?
- What duplicate/drift detection thresholds should we use (exact cron match vs. semantic equivalence, timezone handling)?
- Will customers require SSO/SAML for UI access, or is token-based auth sufficient for v0.1?
- How do we handle eventual consistency windows in cloud APIs to avoid false positives?
