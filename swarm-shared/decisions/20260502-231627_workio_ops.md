## Final synthesized solution (correct + actionable)

### 1. Diagnosis (merged)
- No idempotency handling on LINE webhook → redeliveries create duplicate punches.
- Punch creation is read-then-insert (non-atomic) → races between concurrent/redelivered webhooks.
- No storage-level uniqueness for active punches → DB allows duplicates; correctness relies on app code.
- No deduplication window or traceability (LINE event id, request id) → hard to debug/reconcile.

### 2. Schema changes (one-time migration)
Run once (PostgreSQL):

```sql
-- Add traceability + idempotency columns
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key TEXT,
  ADD COLUMN IF NOT EXISTS line_event_id TEXT,
  ADD COLUMN IF NOT EXISTS request_id TEXT;

-- Business rule: one active punch per (employee, punch_type, date)
CREATE UNIQUE INDEX IF NOT EXISTS uq_employee_punch_date
  ON punches (employee_id, punch_type, date)
  WHERE deleted_at IS NULL;

-- Idempotency: prevent redelivery duplicates
CREATE UNIQUE INDEX IF NOT EXISTS uq_punch_idempotency_key
  ON punches (idempotency_key)
  WHERE idempotency_key IS NOT NULL;
```

Apply:

### 3. Idempotent punch service
File: `server/src/services/punchService.ts`

```ts
import { PoolClient } from 'pg';
import { getDb } from '../db';

export type PunchInput = {
  employeeId: number;
  punchType: 'clock_in' | 'clock_out';
  date: string; // YYYY-MM-DD
  timestamp: Date;
  location?: { lat: number; lng: number };
  lineEventId?: string;
  idempotencyKey: string; // deterministic per event (use LINE event id)
  requestId?: string;
};

export async function createPunch(
  input: PunchInput,
  client?: PoolClient
): Promise<{ id: number; created: boolean }> {
  const db = client ?? (await getDb().connect());
  try {
    // Fast-path insert with idempotency key (atomic)
    const res = await db.query(
      `INSERT INTO punches (
         employee_id, punch_type, date, punched_at, location,
         line_event_id, idempotency_key, request_id, created_at, updated_at
       ) VALUES ($1,$2,$3,$4,$5,$6,$7,$8,now(),now())
       ON CONFLICT (idempotency_key) DO UPDATE
         SET updated_at = now()
       RETURNING id, (xmax = 0) AS created`,
      [
        input.employeeId,
        input.punchType,
        input.date,
        input.timestamp,
        input.location ? JSON.stringify(input.location) : null,
        input.lineEventId || null,
        input.idempotencyKey,
        input.requestId || null,
      ]
    );

    const row = res.rows[0];
    // If idempotency key was new but business-unique constraint blocks, fallback to existing
    if (row.created === true || row.created === 'true') {
      return { id: row.id, created: true };
    }
    // Idempotent hit: same idempotency_key -> return existing
    return { id: row.id, created: false };
  } catch (err: any) {
    // Business-unique violation: return existing active punch
    if (err.code === '23505' && err.constraint === 'uq_employee_punch_date') {
      const existing = await db.query(
        `SELECT id FROM punches WHERE employee_id=$1 AND punch_type=$2 AND date=$3 AND deleted_at IS NULL LIMIT 1`,
        [input.employeeId, input.punchType, input.date]
      );
      if (existing.rows.length) {
        return { id: existing.rows[0].id, created: false };
      }
    }
    // Idempotency violation: fetch existing
    if (err.code === '23505' && err.constraint === 'uq_punch_idempotency_key') {
      const existing = await db.query(`SELECT id FROM punches WHERE idempotency_key=$1 LIMIT 1`, [
        input.idempotencyKey,
      ]);
      if (existing.rows.length) {
        return { id: existing.rows[0].id, created: false };
      }
    }
    throw err;
  } finally {
    if (!client) db.release();
  }
}
```

### 4. Webhook handler usage
File: `server/src/routes/lineWebhook.ts` (or `.js`)

```ts
import { Router } from 'express';
import { createPunch } from '../services/punchService';

const router = Router();

router.post('/webhook/line', async (req, res) => {
  const events = req.body.events || [];
  const requestId = req.headers['x-request-id'] || require('uuid').v4();

  for (const ev of events) {
    if (ev.type !== 'message' && ev.type !== 'postback') continue;

    // Resolve employee (implement resolveEmployeeByLineId)
    const employee = await resolveEmployeeByLineId(ev.source?.userId);
    if (!employee) continue;

    const punchType = inferPunchType(ev); // clock_in/clock_out
    const date = new Date().toISOString().split('T')[0];
    const timestamp = new Date();

    // Use LINE event id as deterministic idempotency key
    await createPunch({
      employeeId: employee.id,
      punchType,
      date,
      timestamp,
      lineEventId: ev.id,
      idempotencyKey: ev.id,
      requestId,
    });
  }

  res.json({ ok: true });
});
```

### 5. Verification (concrete)
- Unit: duplicate `idempotency_key` inserts → second call returns same `id` with `created: false`.
- Integration: send same LINE event id twice → DB row count unchanged; second request is idempotent.
- Concurrency: 10 parallel requests with same `(employee_id, punch_type, date)` and different `idempotency_key` values → only one punch row created; others resolve to existing.
- Schema: confirm indexes exist:
  ```sql
  SELECT indexname, indexdef FROM pg_indexes WHERE tablename='punches';
  ```
- Traceability: verify `line_event_id` and `request_id` are stored for each punch.

### 6. Key decisions (why this wins)
- Uses DB-level unique constraints (`uq_employee_punch_date`, `uq_punch_idempotency_key`) as source of truth — not app logic.
- Idempotency key is stored and enforced at DB level; redelivered events are deduplicated safely.
- Atomic `INSERT ... ON CONFLICT` + fallback queries handle races without requiring explicit locking.
- Trace columns (`line_event_id`, `request_id`) enable reconciliation and debugging.
- Minimal surface: one migration, one service function, small handler change — easy to review and deploy.
