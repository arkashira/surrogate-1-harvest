# workio / backend

## 1. Diagnosis
- Webhook handler is not idempotent: LINE retries on 5xx/timeouts/network blips → duplicate clock-in/out and leave/OT events.
- No unique constraint or transactional upsert to deduplicate events by LINE-delivered `webhookEventId` (or message ID + timestamp + userId).
- Clock-in/out business rules (location/GPS, shift overlap, double-clock) are enforced after insert instead of at upsert, risking invalid duplicates.
- No idempotency key column or index to make deduplication fast and safe under concurrency.
- Missing defensive early-return for already-processed events, causing downstream notifications and report skew.

## 2. Proposed change
- **Files**:  
  - `server/src/db/schema.sql` — add idempotency table + constraints.  
  - `server/src/routes/webhook/line.ts` — implement idempotent upsert for clock/leave/OT events.
- **Scope**:  
  - Add `processed_webhooks` table with unique `event_id` (from LINE) and `tenant_id`.  
  - In webhook handler, wrap processing in a transaction: attempt insert into `processed_webhooks` first; on conflict, skip processing and return 200.  
  - Convert clock/leave/OT inserts into upserts keyed by natural key (`tenant_id`, `user_id`, `event_type`, `event_time`, `source_ref`) to prevent duplicates if retries carry same logical payload.

## 3. Implementation

### `server/src/db/schema.sql`
```sql
-- Idempotency ledger for external webhooks
CREATE TABLE IF NOT EXISTS processed_webhooks (
  id              BIGSERIAL PRIMARY KEY,
  tenant_id       BIGINT NOT NULL,
  event_id        TEXT   NOT NULL,        -- LINE webhook event id or message id
  event_type      TEXT   NOT NULL,        -- 'clock', 'leave', 'ot'
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uq_tenant_event UNIQUE (tenant_id, event_id)
);

-- Index for fast conflict checks
CREATE INDEX IF NOT EXISTS ix_processed_webhooks_lookup
  ON processed_webhooks (tenant_id, event_id);

-- Make clock_events idempotent (example)
-- Assumes clock_events has a natural unique key per logical clock moment
ALTER TABLE clock_events
  ADD CONSTRAINT uq_clock_events_tenant_user_time_type
  UNIQUE (tenant_id, user_id, clock_time, event_type)
  DEFERRABLE INITIALLY DEFERRED;
```

### `server/src/routes/webhook/line.ts`
```ts
import { Router } from 'express';
import { pool } from '../../db';
import type { LINEWebhookEvent } from '../../types/line';

const router = Router();

async function processClockEvent(
  tenantId: number,
  userId: number,
  event: LINEWebhookEvent,
  txClient: any
) {
  const clockTime = new Date(event.timestamp);
  const eventType = event.source.type === 'user' ? 'clock' : 'unknown';
  const location = event?.message?.content?.metadata
    ? JSON.parse(event.message.content.metadata)
    : null;

  // Upsert clock event; if duplicate by unique constraint, do nothing
  await txClient.query(
    `INSERT INTO clock_events (tenant_id, user_id, clock_time, event_type, latitude, longitude, address, source_ref)
     VALUES ($1,$2,$3,$4,$5,$6,$7,$8)
     ON CONFLICT (tenant_id, user_id, clock_time, event_type)
     DO UPDATE SET
       latitude = EXCLUDED.latitude,
       longitude = EXCLUDED.longitude,
       address = EXCLUDED.address,
       updated_at = NOW()`,
    [
      tenantId,
      userId,
      clockTime,
      eventType,
      location?.lat ?? null,
      location?.lon ?? null,
      location?.address ?? null,
      event.source.userId + ':' + event.timestamp
    ]
  );
}

router.post('/line', async (req, res) => {
  const body = req.body as { events: LINEWebhookEvent[] };
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of body.events) {
      // Determine tenant by channel secret or from user mapping (simplified)
      const tenantId = 1; // TODO: map by channel secret / user
      const lineEventId = ev.webhookEventId || ev.source.userId + ':' + ev.timestamp;

      // Idempotency check/insert
      const ins = await client.query(
        `INSERT INTO processed_webhooks (tenant_id, event_id, event_type)
         VALUES ($1,$2,$3)
         ON CONFLICT (tenant_id, event_id) DO NOTHING
         RETURNING id`,
        [tenantId, lineEventId, 'clock']
      );

      if (ins.rowCount === 0) {
        // Already processed — skip but continue processing other events
        continue;
      }

      // Route by event type (simplified to clock events)
      if (ev.type === 'message' && ev.message?.type === 'text') {
        await processClockEvent(tenantId, 1234 /* map userId */, ev, client);
      }
      // Add leave/OT branches similarly with their own upserts
    }

    await client.query('COMMIT');
    res.sendStatus(200);
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    // Return 200 to prevent LINE retry on client errors we can't recover;
    // alternatively return 5xx only on transient failures you want LINE to retry.
    res.sendStatus(200);
  } finally {
    client.release();
  }
});

export default router;
```

## 4. Verification
- **Unit**: Write a small script that posts the same LINE event payload twice to `/webhook/line` and asserts:
  - Second request returns 200.
  - `clock_events` row count increases by exactly 1.
  - `processed_webhooks` contains exactly one entry for the `event_id`.
- **Integration**:
  - Use `curl` or Postman to replay a saved LINE webhook JSON against local dev.
  - Check logs for “already processed” skips and confirm no duplicate side-effects (no double notifications, no double OT/leave entries).
- **DB checks**:
  - `SELECT * FROM processed_webhooks WHERE event_id='<test-id>';` should show one row.
  - `SELECT * FROM clock_events WHERE user_id=<test-user> AND clock_time=<test-time>;` should show one row.
- **Load test**: Fire 10 concurrent identical requests; confirm exactly one logical event created and no constraint violations.
