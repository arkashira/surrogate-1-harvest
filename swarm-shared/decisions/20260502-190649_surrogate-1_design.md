# surrogate-1 / design

## Implementation Plan (≤2h)

**Highest-value incremental improvement**: Add deterministic pre-flight file listing + CDN-only ingestion path to eliminate HF API rate limits during training and make shard workers resilient to 429s.

### What we’ll change
- Add `bin/list-files.sh` (Mac/orchestrator side) → calls `list_repo_tree` once per date folder, saves `file-list.json`.
- Update `bin/dataset-enrich.sh` to accept a file-list JSON and download via CDN URLs (`resolve/main/...`) instead of `load_dataset(streaming=True)` for heterogeneous repos.
- Project to `{prompt, response}` only at parse time; write normalized JSONL shard output.
- Keep existing shard hash routing and dedup via `lib/dedup.py` unchanged.
- Add lightweight retry/backoff for 429 on CDN (rare; separate from API tier).

### Why this is highest value
- Eliminates the main failure mode (HF API 429) during parallel ingestion.
- Costs nothing (CDN unlimited), keeps 16-shard parallelism, fits in <2h.
- Reuses known patterns: pre-list once, embed list, CDN-only workers.

## Files to add/modify

### 1) `bin/list-files.sh` (new)
```bash
#!/usr/bin/env bash
# Usage: list-files.sh <repo> <date-folder> > file-list.json
# Example: list-files.sh axentx/surrogate-1-training-pairs 2026-05-02

set -euo pipefail

REPO="${1:-axentx/surrogate-1-training-pairs}"
FOLDER="${2:-}"

if [ -z "$FOLDER" ]; then
  echo "Usage: $0 <repo> <date-folder>" >&2
  exit 1
fi

# Requires HF_TOKEN in env for API calls (only for listing).
# list_repo_tree non-recursive per folder. We'll do one level and output
# relative paths under <FOLDER>.
python3 - "$REPO" "$FOLDER" <<'PY'
import json
import os
import sys
from huggingface_hub import HfApi

repo = sys.argv[1]
folder = sys.argv[2].rstrip("/")
api = HfApi()

# List single level under folder
entries = api.list_repo_tree(repo=repo, path=folder, recursive=False)
files = []
for e in entries:
    if e.type == "file":
        files.append(e.path)
    elif e.type == "directory":
        # list one more level for files directly inside subfolders (common pattern)
        try:
            sub = api.list_repo_tree(repo=repo, path=e.path, recursive=False)
            for se in sub:
                if se.type == "file":
                    files.append(se.path)
        except Exception:
            pass

result = {
    "repo": repo,
    "folder": folder,
    "files": sorted(files)
}
json.dump(result, sys.stdout, indent=2)
PY
```

Make executable:
```bash
chmod +x bin/list-files.sh
```

                wait = 360 + random.uniform(0, 60)
                print(f"CDN 429, waiting {wait:.0f}s", file=sys.stderr)

            return resp.content
