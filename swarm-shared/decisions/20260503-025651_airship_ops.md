## 1) High-level design (resolved contradictions)

| Decision | Candidate 1 | Candidate 2 | Final choice (why) |
|---|---|---|---|
| Manifest location | `training_manifest.json` in repo root | `file_lists/{date}.json` artifacts | **Use `file_lists/{date}.json`** — keeps repo root clean and supports multiple dates without collisions. |
| CDN base URL | Built from repo + `resolve/main` | Same | Keep Candidate 1 URL pattern (simple, correct). |
| Schema enforcement | Project on ingest (separate script) | Project at parse time during training | **Do both**: project on ingest (canonical files) AND defensively project at parse time (training robustness). |
| Repo selection for shards | Single repo | Deterministic `hash(slug) % 5` across 5 repos | **Use deterministic routing** when multiple repos exist; fallback to single repo if only one present. |
| Lightning reuse/idle-stop | Reuse running studio + status check before `.run()` | Not specified | Keep Candidate 1 guard (correct, actionable). |
| Manifest generation timing | Cron after rate-limit window | Run on Mac after window clears | Same intent; provide CLI + cron example. |

## 2) Implementation plan (actionable)

1. **Generate CDN-only file manifest on Mac**  
   - Run after HF rate-limit window clears.  
   - Non-recursive `list_repo_tree` for one date folder.  
   - Output: `file_lists/{date}.json` with `repo`, `date`, `cdn_base`, `files[]`.

2. **Lightning Studio reuse + idle-stop guard**  
   - Reuse running studio by name; restart only if stopped (idle timeout).  
   - Avoid unnecessary create/destroy to preserve quota.

3. **Schema projection on ingest**  
   - Keep only `prompt`, `response`.  
   - Store files as `batches/mirror-merged/{date}/{slug}.parquet`.  
   - Deterministic repo assignment for outputs when multiple repos used.

4. **CDN-only training loader with deterministic repo routing**  
   - Use manifest to fetch via CDN URLs (no auth, no `/api/`).  
   - Cache locally to avoid repeated CDN fetches within a run.  
   - Deterministic repo selection for input files when multiple repos exist.

5. **Training launcher**  
   - Accept `--date-folder` and optional `--manifest`.  
   - Run via Lightning Studio with guard.

### 3.1 Manifest generator (run on Mac)

```bash
# scripts/generate_manifest.sh
#!/usr/bin/env bash
set -euo pipefail

REPO="${REPO:-axentx/surrogate-dataset}"
DATE_FOLDER="${1:-$(date +%Y-%m-%d)}"
OUT_DIR="file_lists"
OUTPUT="${OUT_DIR}/${DATE_FOLDER}.json"

mkdir -p "${OUT_DIR}"

python3 - <<PY
import json, os
from huggingface_hub import HfApi

repo = os.getenv("REPO", "$REPO")
date_folder = os.getenv("DATE_FOLDER", "$DATE_FOLDER")
output = os.getenv("OUTPUT", "$OUTPUT")

api = HfApi()
tree = api.list_repo_tree(
    repo_id=repo,
    path=date_folder,
    recursive=False,
)
files = sorted(f.rfilename for f in tree if f.rfilename.endswith(".parquet"))

manifest = {
    "repo": repo,
    "date": date_folder,
    "cdn_base": f"https://huggingface.co/datasets/{repo}/resolve/main",
    "files": files,
}
with open(output, "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Manifest written to {output}")
PY
```

### 3.2 Lightning Studio reuse + idle-stop guard

```python
# surrogate/train/lightning_launcher.py
import os
from lightning import Studio, Teamspace, Machine

def get_or_create_studio(name: str) -> Studio:
    # Reuse running studio to save quota
    for s in Teamspace.studios:
        if s.name == name and s.status == "Running":
            print(f"Reusing running studio: {name}")
            return s

    print(f"Creating studio: {name}")
    return Studio(
        name=name,
        machine=Machine.L40S,
        cloud="lightning-public-prod",  # free tier; switch to lambda-prod for H200 if needed
        create_ok=True,
    )

def run_training_with_guard(train_script: str, studio_name: str = "surrogate-train"):
    studio = get_or_create_studio(studio_name)

    # Guard against idle-stop
    if studio.status != "Running":
        print(f"Studio stopped (idle). Restarting...")
        studio.start(machine=Machine.L40S)

    studio.run(
        run_name="surrogate-cdn-train",
        entry_point=["python", train_script, "--date-folder", "auto"],
        requirements=["pyarrow", "requests", "torch", "datasets"],
    )

if __name__ == "__main__":
    run_training_with_guard("train.py")
```

### 3.3 Schema projection on ingest

```python
# surrogate/ingest/project_schema.py
import pyarrow as pa
import pyarrow.parquet as pq
from pathlib import Path

def project_to_prompt_response(input_dir: str, output_dir: str):
    """Keep only {prompt, response}; store as batches/mirror-merged/{date}/{slug}.parquet."""
    out = Path(output_dir)
    out.mkdir(parents=True, exist_ok=True)

    for p in Path(input_dir).glob("*.parquet"):
        table = pq.read_table(p)
        cols = [c for c in ["prompt", "response"] if c in table.column_names]
        projected = table.select(cols)

        out_path = out / p.name
        pq.write_table(projected, out_path)
        print(f"Projected {p.name} -> {out_path}")
```

import pyarrow.parquet as pq
import requests
from pathlib import Path
from typing import List, Dict, Iterable

def deterministic_repo_index(slug: str, repo_count: int) -> int:
    # Deterministic across runs; stable when repos are fixed
    digest = hashlib.sha256(slug.encode()).hexdigest()
    return int(digest, 16) % repo_count

def load_via_cdn(
    manifest_path: str,
    repos: List[str],
    cache_dir: str = ".cache",
) -> Iterable[Dict[str, str]]:
