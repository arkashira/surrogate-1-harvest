## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk check → simulated execution) with immutable event journal, Postgres-backed audit trail, Redis Streams for queueing, CF Workers API endpoints, compliance guardrails as configurable rules, idempotent run handling, and contract tests for broker/OMS/EMS adapter interface (no live broker integration — simulated execution only).

## Open questions
- Deterministic replay: should events be stored in Postgres (event_journal) only or also mirrored to durable object storage for long-term replay?
- Compliance rule engine: use lightweight embedded DSL vs. WASM modules for deterministic sandboxing — which fits free-tier and latency targets?
- Idempotency and exactly-once execution: will Redis-based distributed locks be sufficient under bursty market conditions?
- Connector latency SLAs: which broker/OMS/EMS vendors will be supported first and what auth/credential management is acceptable under compliance?
- Edge vs. region affinity: do CF Workers provide enough control over egress paths to brokers, or do we need a regional workerd/VM fallback?
- Audit retention and immutability: can Supabase row-level policies + append-only tables satisfy regulatory needs, or do we need signed receipts (e.g., hash-chained logs)?
