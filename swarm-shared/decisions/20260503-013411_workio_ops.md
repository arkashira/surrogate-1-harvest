# workio / ops

## Final Synthesis & Action Plan

**Core diagnosis (merged, de-duplicated):**
- LINE webhook retries (5xx/timeouts) create duplicate punches/leave/OT entries because there is no idempotency on `webhookEventId`.
- Race conditions on punch state transitions: find-then-insert/update allows concurrent duplicates and multiple open punches or double-closes.
- No server-side idempotency layer for punch/leave/OT mutations; frontend retries (network flakiness or double-taps) can double-post.
- Missing DB enforcement (unique constraints) to guarantee one-open-punch-per-user and to deduplicate by `webhookEventId`.
- Frontend lacks optimistic UI and safe retry handling, increasing duplicate risk and poor UX during saves.

**Chosen strategy (correctness + actionability):**
- Add DB-level uniqueness to prevent duplicates at the source.
- Add server-side idempotency middleware for LINE webhooks and mutation endpoints.
- Make punch state transitions atomic and safe under concurrency (single upsert path with clear rules).
- Propagate idempotency keys from client through backend to DB.
- Add lightweight frontend guidance for safe retry/optimistic UX.

## 2. Idempotency middleware (server-side)

Create `/opt/axentx/workio/server/src/middleware/idempotency.ts`:

```ts
import { Request, Response, NextFunction } from 'express';
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const IDEMPOTENCY_TTL_MS = 24 * 60 * 60 * 1000; // 24h

export async function ensureIdempotencyTable() {
  await pool.query(`
    CREATE TABLE IF NOT EXISTS idempotency_keys (
      key TEXT PRIMARY KEY,
      response_body TEXT NOT NULL,
      response_status INTEGER NOT NULL,
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    );
  `);
}

export function idempotencyMiddleware(
  req: Request & { idempotencyKey?: string },
  res: Response,
  next: NextFunction
) {
  // Prefer header; fallback to body.webhookEventId for LINE webhooks
  const key = req.get('Idempotency-Key') || (req.body && req.body.webhookEventId);
  if (!key) return next();

  req.idempotencyKey = key;
  const dedupeKey = `workio:idemp:${key}`;

  pool.query<{ response_body: string; response_status: number; created_at: Date }>(
    'SELECT response_body, response_status, created_at FROM idempotency_keys WHERE key = $1',
    [dedupeKey]
  )
    .then(({ rows }) => {
      if (rows.length > 0) {
        const record = rows[0];
        const age = Date.now() - new Date(record.created_at).getTime();
        if (age < IDEMPOTENCY_TTL_MS) {
          try {
            const body = JSON.parse(record.response_body);
            res.status(record.response_status).json(body);
            return;
          } catch {
            // If cached body is not JSON, fall through to process request (best-effort)
          }
        }
      }

      // Capture response to cache
      const chunks: Buffer[] = [];
      const originalJson = res.json;
      // @ts-ignore
      res.json = function (body) {
        chunks.push(Buffer.from(JSON.stringify(body)));
        return originalJson.call(this, body);
      };

      const originalEnd = res.end;
      // @ts-ignore
      res.end = function (chunk, encoding?, cb?) {
        if (chunk) chunks.push(Buffer.isBuffer(chunk) ? chunk : Buffer.from(String(chunk)));
        return originalEnd.call(this, chunk, encoding, cb);
      };

      res.on('finish', () => {
        try {
          const raw = chunks.length > 0 ? Buffer.concat(chunks) : Buffer.from('{}');
          const parsed = JSON.parse(raw.toString());
          pool.query(
            `INSERT INTO idempotency_keys (key, response_body, response_status)
             VALUES ($1, $2, $3)
             ON CONFLICT (key) DO UPDATE
             SET response_body = EXCLUDED.response_body,
                 response_status = EXCLUDED.response_status,
                 created_at = NOW()`,
            [dedupeKey, JSON.stringify(parsed), res.statusCode]
          ).catch(() => {/* best-effort */});
        } catch {
          // If response isn't JSON, skip caching rather than fail
        }
      });

      next();
    })
    .catch(() => next());
}
```

**Usage:**
- In `/opt/axentx/workio/server/src/routes/line.ts`, wrap the webhook handler with `idempotencyMiddleware`.
- In `/opt/axentx/workio/server/src/routes/punch.ts` (and leave/OT routes), apply the same middleware to mutation endpoints.
- Ensure frontend includes `Idempotency-Key` header (UUID) for retries; for LINE webhooks, rely on `webhookEventId` from payload.
