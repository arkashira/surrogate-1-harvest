# Costinel / ops

## Implementation Plan — Costinel ops quality increment (<2h)

**Chosen objective:**  
Add a single read-only endpoint `GET /api/v1/cost-anomaly/signal/top-hub` (optional `?for_date=YYYY-MM-DD`) that returns the strongest hub insight from the knowledge graph as a cost-anomaly signal (Sense + Signal, no execution).

**Why this is highest-value (<2h):**
- Pure read-only addition — zero risk to existing flows.
- Reuses existing knowledge-rag/graph assets (top-hub pattern).
- Immediate ops value: surfaces the most-connected hub (e.g., "MOC") as an actionable signal for cost governance.
- Fits Costinel philosophy: detect (sense) and signal, never execute.

---

## Implementation Steps (≤2h)

1. **Add route + handler** (`app/routes/cost_anomaly.py` or equivalent)  
   - `GET /api/v1/cost-anomaly/signal/top-hub`
   - Query param `for_date` (optional, default today).
   - Return JSON: `{ signal_type, hub, score, summary, context, for_date, generated_at }`.

2. **Integrate knowledge-rag top-hub lookup**  
   - Call existing knowledge-rag layer to fetch most-connected hub and related docs.
   - If no graph available, fallback to lightweight heuristic (most frequent anomaly tag).

3. **Map to cost-anomaly signal schema**  
   - Normalize hub into `signal_type="hub_insight"`, include `hub`, `score` (centrality), `summary`, `context` (doc excerpts).

4. **Add minimal tests**  
   - One unit test for endpoint shape.
   - One integration test mocking knowledge-rag response.

5. **Verify & ship**  
   - Run existing test suite.
   - Start dev server, hit endpoint, confirm JSON shape.

## Code Snippets

### 1) Route + handler (FastAPI-style example)

```python
# app/routes/cost_anomaly.py
from datetime import datetime, timezone
from typing import Optional

from fastapi import APIRouter, Query
from pydantic import BaseModel

from app.services.knowledge_rag import get_top_hub_insight  # existing/graph helper

router = APIRouter(prefix="/api/v1/cost-anomaly", tags=["cost-anomaly"])


class TopHubSignal(BaseModel):
    signal_type: str = "hub_insight"
    hub: str
    score: float
    summary: str
    context: list[str]
    for_date: str  # YYYY-MM-DD
    generated_at: str  # ISO-8601


@router.get("/signal/top-hub", response_model=TopHubSignal)
async def top_hub_signal(
    for_date: Optional[str] = Query(
        None,
        regex=r"^\d{4}-\d{2}-\d{2}$",
        description="Date to evaluate (YYYY-MM-DD). Defaults to today UTC.",
    )
):
    """
    Return the strongest knowledge-graph hub as a cost-anomaly signal.
    Sense + Signal — no execution.
    """
    if for_date is None:
        for_date = datetime.now(timezone.utc).strftime("%Y-%m-%d")

    # Reuse existing knowledge-rag top-hub lookup (pattern: top-hub doc insight)
    insight = get_top_hub_insight(for_date=for_date)

    # Normalize to signal shape
    return TopHubSignal(
        hub=insight["hub"],
        score=float(insight.get("score", 0.0)),
        summary=insight["summary"],
        context=insight.get("context", []),
        for_date=for_date,
        generated_at=datetime.now(timezone.utc).isoformat(),
    )
```

### 3) Minimal unit test

```python
# tests/test_cost_anomaly_routes.py
from fastapi.testclient import TestClient
from app.main import app  # adjust import to your app entrypoint
from unittest.mock import patch

client = TestClient(app)


def test_top_hub_signal():
    with patch("app.services.knowledge_rag.get_top_hub_insight") as mock_insight:
        mock_insight.return_value = {
            "hub": "MOC",
            "score": 0.92,
            "summary": "High anomaly density around multi-org costs",
            "context": ["doc1 excerpt", "doc2 excerpt"],
        }

        resp = client.get("/api/v1/cost-anomaly/signal/top-hub?for_date=2026-05-02")
        assert resp.status_code == 200
        data = resp.json()
        assert data["signal_type"] == "hub_insight"
        assert data["hub"] == "MOC"
        assert "summary" in data
        assert data["for_date"] == "2026-05-02"
```
