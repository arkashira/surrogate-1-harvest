## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: read-only auditor that connects to one AWS account and one GCP project via service credentials, discovers Lambda and Cloud Function schedules, normalizes cron expressions, detects exact-duplicate cron+target pairs, flags orphaned schedules (target missing), surfaces results in a simple JSON report and CSV export, stores metadata in Supabase Postgres, runs discovery via CF Workers on a manual trigger, and retires zero schedules (no automated actions).

## Open questions
- Which AWS/GCP accounts/projects will be in scope for v0.1 and how will cross-account IAM be provisioned (assume-role vs service keys)?
- How frequently should discovery jobs run (hourly vs daily) given rate limits and cost?
- What retention policy should be applied to raw discovery snapshots vs aggregated audit records?
- How will we reliably detect 'orphaned' schedules (e.g., target function deleted but trigger remains) across both platforms?
- Will customers prefer a read-only auditor or also automated retirement workflows (and what approval gates are required)?
