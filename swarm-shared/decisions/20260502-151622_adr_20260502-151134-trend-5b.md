## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow lane (research→signal→risk→execution) with Postgres audit log, Redis Streams event lane, one broker connector (paper-trade mode), risk rule engine (hard limits), idempotent execution, and immutable audit entries; deployable on Cloudflare Workers + Supabase free tier; includes replay endpoint to re-run a workflow_run deterministically from stored context_snapshot.

## Open questions
- Deterministic replay: should workflow events be serialized with deterministic JSON canonicalization (sort keys) or protobuf for bitwise reproducibility?
- Latency SLA: what is the p99 target for research→execution (sub-500ms vs sub-100ms) and which path (edge worker vs regional worker) is acceptable?
- Compliance signature scheme: HMAC with rotating keys vs append-only hash chain vs external W3C verifiable credentials for audit entries?
- Connector auth model: OAuth2 broker connectors vs API-key vault (Supabase) vs per-customer secrets; how to rotate without breaking replays?
- Idempotency scope: idempotency keys at workflow_run level or per-adapter level to prevent double execution across retries?
- Fail-open vs fail-close on risk-check timeouts: hard reject on timeout or allow with circuit-breaker override flag?
- Retention and PII: how long to retain full audit snapshots and signal payloads under free-tier storage constraints?
