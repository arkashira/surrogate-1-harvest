## Deployment
hybrid (client-side Rust binary + optional CF Workers for innocuous bootstrap/reflector; no infra cost on free tiers)

## v0.1 scope
v0.1 ships in 2 weeks: a single-file Rust CLI agent (Linux x86_64) that establishes a local SOCKS5 listener, wraps traffic through an obfuscated TLS/QUIC meek-like reflector via CF Workers (free tier), conceals TUN artifacts by defaulting to userspace TUN via boringtun, enforces a fail-closed kill-switch (block non-stealth interfaces), and records local encrypted session logs to compute WAU-ZB; no server infra to manage, zero-config install, and basic DPI-evasion profiles (meek + simple packet padding) included.

## Open questions
- Which cover protocols (meek, Snowflake, SSH-over-HTTP, QUIC innocuous streams) provide best DPI evasion vs. availability in target regions?
- Can we implement reliable TUN creation and kill-switch hooks on desktop targets without root/admin prompts that deter users?
- How will clients discover/rotate bootstrap reflectors without hardcoded endpoints that can be blocked?
- What local telemetry consent/encryption model satisfies privacy while enabling WAU-ZB measurement?
- How to handle captive portals and networks that block non-browser TLS without breaking stealth or leaking?
