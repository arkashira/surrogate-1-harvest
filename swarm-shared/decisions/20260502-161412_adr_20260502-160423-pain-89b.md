## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust CLI client (Linux x86_64) with: stealthy SOCKS5 proxy wrapper (shadowsocks-like framing + packet-size mimicry), TUN interface concealment (hide from ifconfig/ip link enumeration where possible), fail-closed killswitch (block all non-tunnel traffic on tunnel loss), local SQLite state, opt-in minimal telemetry to Supabase, and Cloudflare Workers endpoint for signed config + health-checked endpoints — zero server hosting costs, installable in <2 minutes, no admin server required.

## Open questions
- Which DPI signatures are highest risk in target regions (SNI, packet size distributions, timing, TUN interface names)?
- Can we rely solely on client-side WASM+CF Workers for updates, or do we need signed bundles to prevent tampering?
- What is the acceptable false-positive rate for the killswitch (block legitimate traffic vs. risk leaks)?
- How will we distribute initial bootstrap endpoints without a hardcoded blocklist footprint?
- What telemetry granularity is required to measure WAU-ZB without deanonymizing users?
