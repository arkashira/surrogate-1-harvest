## v0.1 scope
v0.1 in 2 weeks: deterministic workflow engine (FastAPI + Redis Streams) on CF Workers + Supabase; Postgres-backed immutable audit trail; Redis cache for idempotency and reference data; one sample connector (paper-trading/simulated broker) with pre-trade risk guardrails (hard stop); replay endpoint to rerun a workflow_run from persisted snapshot; API contract tests and minimal CI; no frontend — API-first for integrators; target: sub-second orchestration for synthetic load and one real workflow path.

## Open questions
- What deterministic replay SLA is required (replay window and input snapshot retention period)?
- Which brokers/OMS/EMS and data vendors are must-have connectors for v0.1?
- What compliance rule set must be enforced as hard gates vs. soft warnings (e.g., pre-trade limit checks, restricted symbols, concentration limits)?
- What latency SLO (p95/p99) from signal ingestion to execution decision is acceptable for target workflows?
- How should idempotency and exactly-once execution be guaranteed across connectors (idempotency keys vs. distributed locks)?
- What retention and encryption posture is required for PII/trade data in audit logs under applicable regulations?
