# workio / quality

Below is the **single, merged, production-ready plan** that keeps the strongest, non-contradictory parts of both proposals and prioritizes **correctness + concrete actionability**.

---

## 1. Diagnosis (merged)

- **Non-idempotent ingestion**: LINE retries on 5xx/timeouts can replay clock/leave/OT events and corrupt state.
- **No transactional deduplication**: commits can succeed after 200 OK (or async side-effects) and client retries create duplicates.
- **Frontend polling-only**: LINE-driven changes are invisible until refresh, causing multi-tab/device divergence and perceived latency.
- **Missing signature verification**: allows spoofed events and cross-tenant replays.
- **No bounded retry / dead-letter handling**: transient failures can stall processing or lose events.
- **Acknowledgement before commit risk**: sending 200 before DB commit can lose acknowledged events on crash.

---

## 2. Proposed change (merged)

**File**: `workio/server/src/routes/webhook/line.ts` (and supporting infra)  
**Scope**:
1. Add **signature verification** (HMAC-SHA256) on every request.
2. Add **transactional idempotency/deduplication** with a durable store.
3. **Commit-before-200**: only send 200 after DB commit.
4. Add **bounded retry + dead-letter handling** for failed events.
5. Provide a **real-time update path** (WebSocket or SSE) to eliminate polling divergence (minimal actionable step: broadcast on commit).

---

```sql
CREATE TABLE IF NOT EXISTS webhook_deliveries (
  idempotency_key TEXT PRIMARY KEY,
  event_type      TEXT NOT NULL,
  payload_hash    TEXT NOT NULL,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_webhook_deliveries_processed_at
  ON webhook_deliveries (processed_at);

-- Optional: structured event store for replay/debugging
CREATE TABLE IF NOT EXISTS webhook_events (
  event_id   TEXT PRIMARY KEY,
  tenant_id  TEXT NOT NULL,
  payload    JSONB NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Dead-letter for failed events after retries
CREATE TABLE IF NOT EXISTS webhook_dlq (
  id          TEXT PRIMARY KEY,
  tenant_id   TEXT,
  payload     JSONB,
  error       TEXT,
  attempts    INT NOT NULL DEFAULT 0,
  last_error  TEXT,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

function hashPayload(payload: unknown): string {
  return crypto.createHash('sha256').update(JSON.stringify(payload)).digest('hex');
}

function verifyLineSignature(rawBody: Buffer, signature: string): boolean {
  if (!LINE_CHANNEL_SECRET) return true; // safety bypass if not configured
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

async function isDuplicate(idempotencyKey: string, payloadHash: string) {
  const res = await pool.query(
    `SELECT 1 FROM webhook_deliveries
     WHERE idempotency_key = $1 AND payload_hash = $2`,
    [idempotencyKey, payloadHash]
  );
  return res.rowCount > 0;
}

  await trx.query(
    `INSERT INTO webhook_deliveries (idempotency_key, event_type, payload_hash)
