## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-binary client (Rust) that: establishes a local SOCKS5 listener on 127.0.0.1, transparently wraps traffic through an obfuscation transport (shadowsocks 2022 AEAD) to a user-provided bridge, creates a local TUN via boringtun for system-level routing, enforces a best-effort kill-switch (block non-tunnel IPv4/IPv6 when TUN is not up), stores session/peer state in local SQLite, and emits opt-in, anonymized client-side telemetry to compute WAU-ZB — all with zero-config defaults and no external services required beyond the user-supplied bridge.

## Open questions
- Which obfuscation profiles (TLS fragmentation, shadowsocks 2022, meek, Hysteria-like QUIC) provide the best evasion vs. performance trade-off for our target regions?
- How will we distribute and rotate bridge endpoints without centralized infra (CF Workers as lightweight rendezvous + S3-free static file fallback)?
- What kill-switch enforcement strategy is reliable across desktop OSes (Windows, Linux, macOS) without admin/root?
- How do we measure WAU-ZB reliably client-side while preserving privacy and avoiding telemetry endpoints that can be blocked?
- Can we implement TUN and SOCKS5 hiding without root/admin on all target platforms, and what fallbacks are acceptable?
