# Costinel / design

## Final Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h)

**Core decision**: Use **static JSON in `public/hubs/`** (CDN-first) with **graceful non-blocking fallback**. This is the simplest, fastest, and safest path to ship a production-ready signal panel without backend changes or rate-limit risk.

### Scope (what we ship)
- A **non-blocking Top-Hub Signal Panel** mounted in the Costinel dashboard (sidebar or top banner).
- Defaults to hub **MOC** (configurable via `VITE_HUB_NAME`).
- Shows:
  - Hub title + short insight
  - Top 3 actionable signals (with severity badges)
  - Last updated timestamp
  - Click-to-expand details (optional)
- CDN-first data source: `public/hubs/{hubName}.json`
- Graceful fallback UI when file missing or malformed.
- Optional lightweight generator script (`scripts/generate-hub.js`) to pull from knowledge-rag and write the static JSON (run manually or via CI).

### Why this path
- **Zero backend changes** — deployable in <2h.
- **No HF API calls at runtime** — avoids rate limits.
- **CDN cacheable** — fast, cheap, globally available.
- **Non-blocking** — panel failure does not break dashboard.
- **Extensible** — generator script can later be wired to CI/CD or cron.

### Implementation Steps (ordered)

1. **Create static hub JSON**  
   Path: `public/hubs/MOC.json`  
   Minimal schema:
   ```json
   {
     "hub": "MOC",
     "title": "MOC — Multi-Org Cost governance",
     "insight": "Top cross-account chargebacks show 23% savings potential in idle EBS and orphaned ENIs.",
     "severity": "high",
     "signals": [
       {
         "id": "sig-001",
         "title": "Idle EBS volumes",
         "description": "14 unattached gp3 volumes >30 days across prod accounts.",
         "severity": "high",
         "action": "Create snapshot and schedule deletion after 7-day retention.",
         "link": "/cost-objects/ebs-idle"
       },
       {
         "id": "sig-002",
         "title": "Orphaned ENIs",
         "description": "28 unattached ENIs in sandbox accounts.",
         "severity": "medium",
         "action": "Batch delete after team confirmation.",
         "link": "/network/orphaned-enis"
       },
       {
         "id": "sig-003",
         "title": "Underutilized RIs",
         "description": "32% of purchased RIs unused in last 60 days.",
         "severity": "medium",
         "action": "Exchange or modify to match current instance mix.",
         "link": "/ri/coverage"
       }
     ],
     "updatedAt": "2026-05-03T08:00:00.000Z",
     "ttlMinutes": 1440
   }
   ```

2. **Add env config**  
   `.env` (or Vite config):
   ```
   VITE_HUB_NAME=MOC
   VITE_HUB_POLL_INTERVAL=300000
   ```

3. **Create panel component**  
   Path: `src/components/HubSignalPanel.vue` (or `.tsx` if React).  
   Responsibilities:
   - Fetch `public/hubs/${hubName}.json` (CDN).
   - Non-blocking: errors do not throw.
   - Show skeleton while loading; show fallback message if unavailable.
   - Auto-refresh based on `ttlMinutes` or `VITE_HUB_POLL_INTERVAL`.

   Example (Vue 3 + TypeScript):
   ```vue
   <script setup lang="ts">
   import { ref, onMounted, onUnmounted } from 'vue';

   const props = defineProps<{
     hubName?: string;
   }>();

   const hubName = props.hubName || import.meta.env.VITE_HUB_NAME || 'MOC';
   const data = ref<any>(null);
   const loading = ref(false);
   const error = ref<string | null>(null);
   let pollTimer: any = null;

   async function fetchHub() {
     loading.value = true;
     error.value = null;
     try {
       const res = await fetch(`/hubs/${hubName}.json`, { cache: 'no-cache' });
       if (!res.ok) throw new Error(`HTTP ${res.status}`);
       data.value = await res.json();
     } catch (err: any) {
       error.value = err.message || 'Failed to load hub signals';
       data.value = null;
     } finally {
       loading.value = false;
     }
   }

   function startPoll(intervalMs = 300000) {
     stopPoll();
     pollTimer = setInterval(fetchHub, intervalMs);
   }

   function stopPoll() {
     if (pollTimer) clearInterval(pollTimer);
   }

   onMounted(() => {
     fetchHub();
     const interval = Number(import.meta.env.VITE_HUB_POLL_INTERVAL) || 300000;
     startPoll(interval);
   });

   onUnmounted(stopPoll);
   </script>

   <template>
     <section class="hub-signal-panel" aria-label="Top hub signals">
       <header class="panel-header">
         <h3 class="panel-title">{{ data?.title || 'Hub Signals' }}</h3>
         <span v-if="data?.updatedAt" class="panel-meta">
           Updated {{ new Date(data.updatedAt).toLocaleDateString() }}
         </span>
       </header>

       <div v-if="loading" class="panel-loading">Loading signals…</div>

       <div v-else-if="error" class="panel-fallback">
         Signals unavailable — <button @click="fetchHub" class="retry-btn">Retry</button>
       </div>

       <div v-else-if="data?.signals?.length" class="signals-list">
         <div v-for="sig in data.signals" :key="sig.id" class="signal-item">
           <div class="signal-header">
             <span :class="['badge', `badge-${sig.severity}`]">{{ sig.severity }}</span>
             <h4 class="signal-title">{{ sig.title }}</h4>
           </div>
           <p class="signal-desc">{{ sig.description }}</p>
           <div class="signal-action">
             <a :href="sig.link" class="action-link">{{ sig.action }}</a>
           </div>
         </div>
       </div>

       <div v-else class="panel-fallback">No signals available.</div>

       <footer v-if="data?.insight" class="panel-insight">
         <strong>Insight:</strong> {{ data.insight }}
       </footer>
     </section>
   </template>
