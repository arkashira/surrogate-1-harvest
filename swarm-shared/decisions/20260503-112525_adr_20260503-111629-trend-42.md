## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a Cloudflare Workers + Python worker that exposes a single OpenAI-compatible /v1/chat/completions endpoint, normalizes payloads to/from OpenAI/Anthropic (API-key auth), enforces per-client rate limits and daily cost caps via Redis, records RequestLog to Supabase Postgres, and exposes basic routing (latency/cost) with failover. Includes health checks, Prometheus metrics, and Postman collection. No frontend, no streaming, no PII filters in v0.1.

## Open questions
- Which provider auth patterns must be supported at v0.1 (API key only vs OAuth/STS)?
- What is the canonical request/response schema (OpenAI-like vs multi-modal)?
- How should global vs per-client quotas be enforced (Redis vs Postgres)?
- What retry/failover SLA do we commit to (max attempts, circuit-breaker thresholds)?
- Where will secrets live (Supabase Vault, CF KV, or external) and rotation cadence?
- Do we normalize embeddings/tokenizers or proxy provider-specific semantics?
