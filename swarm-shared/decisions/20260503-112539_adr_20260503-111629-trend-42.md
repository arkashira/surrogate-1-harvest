## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-region CF Worker gateway exposing an OpenAI-compatible REST surface (POST /v1/chat/completions) that normalizes requests/responses between OpenAI and Anthropic, enforces global token-bucket rate limits and p95-latency tracking via Redis, persists config/audit in Supabase Postgres, and exposes basic health/metrics endpoints — deployable via Wrangler free tier with no frontend.

## Open questions
- Which provider normalization schema (OpenAI-compatible vs custom) minimizes adapter surface area while preserving expressiveness?
- How should per-provider token bucket state be synchronized across CF Workers edges (Redis vs Durable Objects)?
- What retry/backoff policy balances SLA adherence against quota exhaustion during upstream throttling?
- How will p95 latency SLA be measured and enforced (edge timing headers vs Redis time-series)?
- Which authentication rotation cadence and secret storage keeps keys secure without downtime?
