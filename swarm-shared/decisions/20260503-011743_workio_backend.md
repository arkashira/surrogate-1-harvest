# workio / backend

## Final consolidated solution (best parts, resolved contradictions)

**Core problem**: LINE retries webhook deliveries on network errors/timeouts. Without idempotency and proper concurrency controls, retries create duplicate punches, leave/OT entries, or inconsistent state.

**Chosen approach** (resolve contradictions in favor of correctness + concrete actionability):
- Use a **stable, LINE-derived idempotency key** (`deliveryId`) stored in a dedicated table so retries are safe across punch, leave, and OT actions.
- Enforce **at-most-one-open-punch-per-day** with a partial unique index to prevent logical duplicates.
- Use **row-level locking (SELECT … FOR UPDATE)** inside a single transaction when mutating punches/leave/OT to prevent lost updates and races.
- Enforce **timing-safe signature verification** and return the HTTP codes LINE expects (200 for success, 401 for bad signature, 500 for transient failures to trigger retry).
- Keep changes **backend-only, minimal, and deployable in <2h**.

### 1. DB schema changes

File: `server/src/db/schema.sql`

```sql
-- Prevent more than one open punch per user per calendar day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_day
ON punches (user_id, DATE(clock_in_at))
WHERE clock_out_at IS NULL;

-- Idempotency for external events (LINE webhook deliveries)
CREATE TABLE IF NOT EXISTS webhook_deliveries (
  id            TEXT PRIMARY KEY,               -- idempotency key
  user_id       TEXT NOT NULL,
  event_type    TEXT NOT NULL,                  -- 'punch', 'leave_request', 'ot_request', etc.
  entity_id     TEXT NOT NULL,                  -- affected row id (or 'none')
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uq_delivery_id UNIQUE (id)
);

-- Fast cleanup index (retain ~7 days)
CREATE INDEX IF NOT EXISTS idx_deliveries_created ON webhook_deliveries (created_at);
```

### 2. Idempotent LINE webhook handler

File: `server/src/routes/lineWebhook.ts`

```ts
import express from 'express';
import { pool } from '../db';
import crypto from 'crypto';

const router = express.Router();

function verifySignature(body: Buffer, signature: string, channelSecret: string): boolean {
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(body)
    .digest('hex');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

function makeDeliveryId(event: any): string {
  // Deterministic, collision-resistant idempotency key from LINE event fields
  const src = [
    event.source?.userId || '',
    String(event.timestamp || ''),
    String(event.type || ''),
    event.message?.id || event.postback?.data || ''
  ].join('|');
  return crypto.createHash('sha256').update(src).digest('hex');
}

router.post('/line', async (req, res) => {
  const sig = req.headers['x-line-signature'] as string;
  const secret = process.env.LINE_CHANNEL_SECRET;
  if (!sig || !secret || !verifySignature(req.body as Buffer, sig, secret)) {
    return res.status(401).json({ error: 'invalid signature' });
  }

  const events = req.body.events || [];
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const event of events) {
      if (!event || !event.source?.userId) continue;
      const userId = event.source.userId;
      const deliveryId = makeDeliveryId(event);

      // Idempotency check
      const dup = await client.query(
        `SELECT entity_id FROM webhook_deliveries WHERE id = $1`,
        [deliveryId]
      );
      if (dup.rowCount > 0) {
        continue; // already processed
      }

      // Punch handling (clock-in/clock-out)
      const text = (event.message?.text || '').trim().toLowerCase();
      const postbackData = (event.postback?.data || '').toLowerCase();

      const isClockIn =
        (event.type === 'postback' && postbackData === 'action=clock_in') ||
        (event.type === 'message' && text === 'in');
      const isClockOut =
        (event.type === 'postback' && postbackData === 'action=clock_out') ||
        (event.type === 'message' && text === 'out');

      if (isClockIn || isClockOut) {
        // Lock any open punch for this user to prevent races
        const openPunch = await client.query(
          `SELECT id FROM punches WHERE user_id = $1 AND clock_out_at IS NULL FOR UPDATE`,
          [userId]
        );

        if (isClockIn) {
          if (openPunch.rowCount > 0) {
            // Already clocked in — record idempotency and skip
            await client.query(
              `INSERT INTO webhook_deliveries(id, user_id, event_type, entity_id)
               VALUES ($1,$2,$3,$4)`,
              [deliveryId, userId, 'punch', openPunch.rows[0].id]
            );
            continue;
          }
          const insertResult = await client.query(
            `INSERT INTO punches(user_id, clock_in_at, clock_out_at, source)
             VALUES ($1, NOW(), NULL, $2) RETURNING id`,
            [userId, 'line']
          );
          await client.query(
            `INSERT INTO webhook_deliveries(id, user_id, event_type, entity_id)
             VALUES ($1,$2,$3,$4)`,
            [deliveryId, userId, 'punch', insertResult.rows[0].id]
          );
        } else {
          // clock-out
          if (openPunch.rowCount === 0) {
            // No open punch — record idempotency to prevent retry loops
            await client.query(
              `INSERT INTO webhook_deliveries(id, user_id, event_type, entity_id)
               VALUES ($1,$2,$3,$4)`,
              [deliveryId, userId, 'punch', 'none']
            );
            continue;
          }
          await client.query(
            `UPDATE punches SET clock_out_at = NOW() WHERE id = $1`,
            [openPunch.rows[0].id]
          );
          await client.query(
            `INSERT INTO webhook_deliveries(id, user_id, event_type, entity_id)
             VALUES ($1,$2,$3,$4)`,
            [deliveryId, userId, 'punch', openPunch.rows[0].id]
          );
        }
      }

      // Placeholder for leave/OT handling (same pattern: idempotency + row locking)
      // if (event.type === 'postback' && postbackData.startsWith('leave') or 'ot') { ... }
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'processing_failed' });
  } finally {
    client.release();
  }
});

export default router;
```

Verification steps:

1. **Constraint test** — ensure duplicate open punches are rejected:
   ```sql
   INSERT INTO punches(user_id, clock_in_at, clock_out_at, source) VALUES ('u1', NOW(), NULL, 'test');
   -- second insert for same user same day should fail:
   INSERT INTO punches(user_id, clock_in_at, clock_out_at, source) VALUES ('u1', NOW(), NULL, 'test');
   ```

2. **Idempotency test** — send the same LINE event payload twice; second request should not create a new punch or change state, and should return 200.

3. **Concurrency test** — simulate concurrent clock-in attempts for the same user; only one should succeed and the other should be rejected by the unique index or row lock.
