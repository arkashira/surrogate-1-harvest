# Costinel / design

## Final Design Increment (≤2h)

**Goal:** Ship a deterministic discovery module that produces machine-readable manifests and a top-hub insight so onboarding, training, and governance scripts can run without manual wiring.

**Core deliverables (unchanged from Candidate 1):**
- `manifests/sources.json` — cloud connectors, credential strategy, cadence.
- `manifests/discovery.json` — discovered accounts/regions/services + counts/tags.
- `insights/top-hub.md` — human-readable insight referencing the most-connected hub (MOC) with actionable next steps.

**Why this wins:**
- Reproducible environment bootstraps (CI, local dev, training pipelines).
- Knowledge-RAG and graph queries can start from a known hub (MOC).
- Surrogate-1 training ingestion can reference `sources.json` for labeled cloud-source provenance without schema pollution.

---

## Implementation Plan (≤2h)

1. **Create deterministic discovery CLI**  
   - Path: `scripts/discover.py` (single-shot, no long-running services).  
   - Inputs: optional env vars for cloud creds; defaults to local fixtures/safe stubs.  
   - Outputs:
     - `manifests/sources.json`
     - `manifests/discovery.json`
     - `insights/top-hub.md`

2. **Define minimal, validated schema**  
   - `sources.json`: list of cloud connectors + credential strategy + cadence.  
   - `discovery.json`: discovered resources (accounts, regions, services) + counts + tags.  
   - `top-hub.md`: human-readable insight referencing the most-connected hub (e.g., “MOC”) with actionable next steps.

3. **Implement safe, idempotent writer**  
   - Deterministic ordering (sorted keys) so checksums are stable across runs.  
   - Atomic write via temp file + rename.

4. **Add lightweight tests and validation**  
   - Validate JSON schema with `jsonschema`.  
   - Ensure top-hub file exists and contains expected section header.  
   - Add a small unit-style test in `tests/test_discover.py` that runs the CLI and validates outputs.

5. **Wire into repo hygiene**  
   - Add `Makefile` targets: `make discover`, `make validate-manifests`.

## Code Snippets

### `scripts/discover.py`

```python
#!/usr/bin/env python3
"""
Deterministic discovery for Costinel.
Produces:
  - manifests/sources.json
  - manifests/discovery.json
  - insights/top-hub.md
"""

import json
import os
import sys
import hashlib
from datetime import datetime, timezone
from pathlib import Path

ROOT = Path(__file__).resolve().parent.parent
MANIFESTS_DIR = ROOT / "manifests"
INSIGHTS_DIR = ROOT / "insights"

MANIFESTS_DIR.mkdir(exist_ok=True)
INSIGHTS_DIR.mkdir(exist_ok=True)

# Deterministic ordering helper
def _sorted(obj):
    if isinstance(obj, dict):
        return {k: _sorted(v) for k, v in sorted(obj.items())}
    if isinstance(obj, list):
        return [_sorted(v) for v in sorted(obj, key=lambda x: str(x))]
    return obj

def build_sources() -> dict:
    """
    Build sources manifest.
    Prefer env-based hints; fallback to stubbed examples for local dev.
    """
    return {
        "generated_at": datetime.now(timezone.utc).isoformat(),
        "sources": [
            {
                "id": "aws-prod",
                "provider": "aws",
                "environment": "prod",
                "collection": {
                    "cadence": "hourly",
                    "method": "cost-explorer",
                    "regions": os.getenv("AWS_REGIONS", "us-east-1,us-west-2").split(",")
                },
                "credentials": {
                    "strategy": "env",
                    "env_prefix": "AWS"
                },
                "tags": ["production", "primary"]
            },
            {
                "id": "gcp-dev",
                "provider": "gcp",
                "environment": "dev",
                "collection": {
                    "cadence": "daily",
                    "method": "billing-export",
                    "project": os.getenv("GCP_PROJECT", "costinel-dev")
                },
                "credentials": {
                    "strategy": "adc",
                    "env_prefix": "GOOGLE"
                },
                "tags": ["development"]
            },
            {
                "id": "azure-stage",
                "provider": "azure",
                "environment": "staging",
                "collection": {
                    "cadence": "daily",
                    "method": "cost-management-api"
                },
                "credentials": {
                    "strategy": "sp",
                    "env_prefix": "AZURE"
                },
                "tags": ["staging"]
            }
        ]
    }

def build_discovery(sources: dict) -> dict:
    """
    Build discovery manifest (lightweight deterministic stub).
    Real implementation would query cloud APIs; this keeps it local and deterministic.
    """
    discovered_accounts = []
    for src in sources["sources"]:
        pid = src["provider"]
        env = src["environment"]
        discovered_accounts.append({
            "account_id": f"{pid}-{env}-001",
            "provider": pid,
            "environment": env,
            "regions": src["collection"].get("regions", ["global"]),
            "services": ["compute", "storage", "network", "database"],
            "tags": src.get("tags", []),
            "status": "active"
        })

    return {
        "generated_at": datetime.now(timezone.utc).isoformat(),
        "discovered_accounts": _sorted(discovered_accounts),
        "summary": {
            "total_accounts": len(discovered_accounts),
            "providers": sorted(list(set(a["provider"] for a in discovered_accounts))),
            "environments": sorted(list(set(a["environment"] for a in discovered_accounts)))
        }
    }

def write_atomic(path: Path, content: str):
    tmp = path.with_suffix(path.suffix + ".tmp")
    tmp.write_text(content, encoding="utf-8")
    tmp.replace(path)

def build_top_hub() -> str:
    """
    Top-hub insight (per pattern: review most-connected hub before planning).
    """
    return """# Top-Hub Insight

## Most-connected hub: MOC

**Why MOC?**  
MOC (Mission Operations Center) is the highest-degree node in the governance graph — it receives cost signals, policy exceptions, and approval workflows from multiple clouds and teams.

## Recommended next actions
- Validate MOC-linked policies in `manifests/sources.json` for credential rotation cadence.
- Attach anomaly-detection rules to MOC-connected accounts (see `manifests/discovery.json`).
- Use this hub as the primary entrypoint for knowledge-RAG queries when onboarding new team members.

*Generated deterministically by `scripts/discover.py`.*
"""

def main() -> int:
    sources = _sorted(build_sources())
    discovery = _sorted(build_discovery(sources))

    write_atomic(MANIFESTS_DIR / "sources.json", json.dumps(sources, indent=2, ensure_ascii=False))
    write_atomic(MANIFESTS_DIR / "disiscovery.json", json.dumps(discovery, indent=2, ensure_ascii=False))
    write_atomic(INSIGHTS_DIR / "top-hub.md", build_top_hub())

    # Print stable checksum for CI verification
    def _checksum(p: Path) -> str:
        return hashlib.sha256(p.read_bytes()).hexdigest()[:12]

    print("OK",
          f"sources={_checksum(MANIFESTS_DIR/'sources.json')}",
          f"discovery={_checksum(MANIFESTS_DIR/'discovery.json')}",
          f"top-hub={_checksum(INSIGHTS_DIR/'top-hub.md')}",
          file=sys.stderr)
    return 0

if __name__ == "__main__":
    sys.exit(main())
```
