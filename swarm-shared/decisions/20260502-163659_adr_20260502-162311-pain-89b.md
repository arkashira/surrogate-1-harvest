## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: Android client (minSdk 28) with hidden SOCKS5 listener on ephemeral port, meek HTTPS-fronted transport via CF Workers tunnel, local encrypted SQLite for session logs, Supabase Postgres for signed endpoint config, basic conceal mode for TUN (exclude from per-app VPN list via VpnService.Builder.establish tun fd masking), and WAUS/U metric emitted via anonymized Edge Function (no PII).

## Open questions
- Which TUN conceal technique (shadow TUN, split routing, or tun2socks with hidden fd) is least detectable on stock Android 12+ without root?
- Can we reliably mimic HTTPS on front ports while preserving low-latency UDP-like flows for interactive traffic?
- How will we distribute rotated endpoint lists without revealing update channels to on-device traffic analysis?
- What is the acceptable battery/CPU budget for continuous port obfuscation and packet header rewrites on mid-tier devices?
- Which DPI signatures (JA3, TLS extensions, packet lengths, timing) must we neutralize first to avoid fingerprinting?
