# workio / frontend

## 1. Diagnosis

- Frontend has no optimistic UI for clock in/out — users get no immediate feedback on slow networks and may double-tap, creating duplicate requests that amplify backend race conditions.
- No client-side idempotency key attached to punch requests; retries (manual or auto) can collide with backend duplicates.
- Clock-in button state does not reflect realtime punch status (open/closed) across devices or tabs; stale state leads to invalid transitions.
- No local debounce/throttle on the primary action button; rapid clicks queue multiple identical requests.
- Missing visual affordance for GPS/location verification status before submitting punch; users can fail LINE location checks and not know why.

## 2. Proposed change

File: `workio/src/components/ClockInOutButton.tsx` (create if absent) or modify existing action button in `workio/src/pages/Dashboard.tsx` / similar.  
Scope: single component that wraps the punch action with optimistic UI, client idempotency key, and debounced submission.

## 3. Implementation

Create `workio/src/components/ClockInOutButton.tsx`:

```tsx
import { useState, useCallback, useRef } from 'react';
import { useToast } from '@/hooks/use-toast';
import { Button } from '@/components/ui/button';
import { Loader2, CheckCheck, Clock } from 'lucide-react';

type PunchAction = 'CLOCK_IN' | 'CLOCK_OUT';

interface ClockInOutButtonProps {
  currentStatus: 'IN' | 'OUT' | null; // latest known server state
  onPunch: (action: PunchAction, idempotencyKey: string) => Promise<void>;
}

export function ClockInOutButton({ currentStatus, onPunch }: ClockInOutButtonProps) {
  const [pending, setPending] = useState(false);
  const [optimisticStatus, setOptimisticStatus] = useState<'IN' | 'OUT' | null>(currentStatus);
  const idempotencyRef = useRef<string | null>(null);
  const { toast } = useToast();

  // generate a short, URL-safe idempotency key for this client action
  const generateIdempotencyKey = useCallback(() => {
    const t = Date.now().toString(36);
    const r = Math.random().toString(36).slice(2, 7);
    return `punch-${t}-${r}`;
  }, []);

  const handlePress = useCallback(async () => {
    if (pending) return;

    const action: PunchAction = optimisticStatus === 'OUT' ? 'CLOCK_IN' : 'CLOCK_OUT';
    const key = generateIdempotencyKey();
    idempotencyRef.current = key;

    // optimistic update
    setPending(true);
    const prev = optimisticStatus;
    const next = action === 'CLOCK_IN' ? 'IN' : 'OUT';
    setOptimisticStatus(next);

    try {
      await onPunch(action, key);
      toast({
        title: action === 'CLOCK_IN' ? 'เข้างานแล้ว' : 'ออกงานแล้ว',
        description: new Date().toLocaleTimeString(),
        icon: <CheckCheck className="h-4 w-4" />,
      });
    } catch (err: any) {
      // rollback optimistic state
      setOptimisticStatus(prev);
      toast({
        variant: 'destructive',
        title: 'ไม่สามารถบันทึกได้',
        description: err?.message || 'กรุณาลองใหม่อีกครั้ง',
      });
    } finally {
      setPending(false);
      idempotencyRef.current = null;
    }
  }, [pending, optimisticStatus, generateIdempotencyKey, onPunch, toast]);

  const isClockIn = optimisticStatus === 'OUT' || optimisticStatus === null;
  const label = isClockIn ? 'เข้างาน' : 'ออกงาน';
  const Icon = pending ? Loader2 : isClockIn ? Clock : CheckCheck;

  return (
    <Button
      size="lg"
      className="w-full max-w-xs text-lg"
      disabled={pending}
      onClick={handlePress}
    >
      <Icon className={`mr-2 h-5 w-5 ${pending ? 'animate-spin' : ''}`} />
      {pending ? 'กำลังบันทึก...' : label}
    </Button>
  );
}
```

Usage example in `workio/src/pages/Dashboard.tsx` (or wherever the action lives):

```tsx
import { ClockInOutButton } from '@/components/ClockInOutButton';
import { useEffect, useState } from 'react';
import axios from '@/lib/axios';

export default function Dashboard() {
  const [punchStatus, setPunchStatus] = useState<'IN' | 'OUT' | null>(null);

  useEffect(() => {
    // fetch latest punch state on mount
    axios.get('/api/punches/latest').then((res) => {
      setPunchStatus(res.data?.isClockIn ? 'IN' : 'OUT');
    });
  }, []);

  const handlePunch = async (action: 'CLOCK_IN' | 'CLOCK_OUT', idempotencyKey: string) => {
    await axios.post(
      '/api/punches',
      { action },
      {
        headers: {
          // backend should read this header for idempotency
          'Idempotency-Key': idempotencyKey,
        },
      }
    );
    // refresh latest state after success (optional optimistic is fine)
    const res = await axios.get('/api/punches/latest');
    setPunchStatus(res.data?.isClockIn ? 'IN' : 'OUT');
  };

  return (
    <div className="p-4">
      <ClockInOutButton currentStatus={punchStatus} onPunch={handlePunch} />
    </div>
  );
}
```

Backend note (non-code, required): ensure `/api/punches` accepts `Idempotency-Key` header and stores it (e.g., in Redis or DB unique constraint) to reject duplicates within a time window (e.g., 24h). This pairs the frontend change with the earlier backend idempotency fixes.

## 4. Verification

1. Start dev servers (`npm run dev` in frontend + backend).
2. Log in as Employee and navigate to Dashboard.
3. Click the button once — UI should switch to “กำลังบันทึก...” then show success toast and updated label within ~1s on normal network.
4. Rapid double-click or triple-click — only one request should be sent (pending disables button) and no duplicate punches appear in the backend.
5. Simulate failure (block request in DevTools or return 500) — UI should rollback to previous state and show error toast.
6. Open two tabs, perform punch in Tab A — Tab B should reflect correct status after refresh (or via WebSocket/polling if implemented).
7. Check network tab: requests include `Idempotency-Key` header with unique values per action. Replay same request with same key (via curl) — backend should respond with 200 and same result without creating duplicate punch.
