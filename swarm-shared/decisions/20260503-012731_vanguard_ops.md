# vanguard / ops

## Final Consolidated Implementation Plan

**Core diagnosis (agreed across candidates):**
- Frontend and training both trigger authenticated `list_repo_tree` and `/api/` proxy calls, burning HF quota (1000/5 min) and causing 429s.
- No persisted `(repo, dateFolder) → file-list` artifact; every reload re-lists and re-downloads metadata.
- Training uses `load_dataset(streaming=True)` on heterogeneous repos, risking `pyarrow.CastError` on mixed schemas.
- Lightning Studio reuse is not implemented; runs recreate studios and waste quota.
- No CDN-only data path; training still makes authenticated API calls during data loading instead of using public CDN URLs.

**Chosen strategy (resolve contradictions in favor of correctness + concrete actionability):**
- Persist a single, minimal file manifest per `(repo, dateFolder)` using one non-recursive `list_repo_tree` call and cache it locally.
- Generate a training script that uses **only public CDN URLs** (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) and projects to `{prompt, response}` at parse time.
- Avoid `load_dataset`/`datasets` streaming for heterogeneous repos; use direct pyarrow + requests to avoid schema-cast surprises and to keep behavior deterministic.
- Reuse an existing running Lightning Studio when available; otherwise start one with L40S priority and a safe public-tier fallback.
- Keep orchestration simple, auditable, and idempotent; do not rely on Lightning Cloud Python SDK for studio lifecycle in the orchestrator (unreliable/extra auth surface). Instead, use `lightning` CLI + config to reuse studios.

# ---- optional HF listing (only if manifest missing) ----
def hf_api_list_tree(repo: str, path: str = ""):
    """
    Single non-recursive list_repo_tree call.
    Requires HF_TOKEN for private repos; public repos may work without.
    """
    try:
        from huggingface_hub import HfApi
        api = HfApi()
        items = api.list_repo_tree(repo=repo, path=path, recursive=False)
        return [i.rfilename for i in items if i.rfilename]
    except Exception as e:
        raise RuntimeError(f"HF list_repo_tree failed for {repo}/{path}: {e}") from e

def build_manifest() -> dict:
    MANIFEST_PATH.parent.mkdir(parents=True, exist_ok=True)

    # If manifest exists and is recent, reuse it (idempotency)
    if MANIFEST_PATH.exists():
        try:
            manifest = json.loads(MANIFEST_PATH.read_text())
            print(f"Reusing existing manifest: {MANIFEST_PATH}")
            return manifest
        except Exception:
            pass  # regenerate on corruption

    print(f"Listing {HF_REPO}/{DATE_FOLDER} (non-recursive)...")
    files = hf_api_list_tree(HF_REPO, DATE_FOLDER)
    manifest = {
        "repo": HF_REPO,
        "date_folder": DATE_FOLDER,
        "files": sorted(files),
        "created_utc": datetime.datetime.utcnow().isoformat() + "Z"
    }
    MANIFEST_PATH.write_text(json.dumps(manifest, indent=2))
    print(f"Manifest saved: {MANIFEST_PATH} ({len(files)} files)")
    return manifest
