# workio / frontend

## Final synthesized implementation (chosen parts, resolved contradictions)

**What we keep (strongest from both):**
- **Optimistic UI + disabled-while-pending** to eliminate double-taps and perceived latency.
- **Client-side idempotency key per punch** (`X-Idempotency-Key`) to prevent duplicates across retries and accidental double submissions.
- **In-flight request lock (per-key)** so rapid clicks don’t create parallel requests.
- **Local echo/pending/error/success states** so users can distinguish processing vs failure immediately.
- **Explicit retry guidance and preserved error state** on transient failures.

**Contradictions resolved in favor of correctness + actionability:**
- Use **short, random idempotency keys per user action** (not timestamp+userId hash) — simpler, collision-resistant, and safe for retries. Reuse only within the same in-flight attempt; generate fresh keys on explicit retries.
- Keep **in-memory pending set at api layer** (not component) so deduplication works across UI surfaces. Do not reuse keys across different attempts (avoid false server-side idempotency collisions).
- **Optimistic echo is opt-in** via callback rather than baked into the button — keeps component reusable and avoids inconsistent global state mutations.
- **Persist minimal last-punch optimistic update at the screen level** (TimeClock) so UI feels responsive, but rely on server truth as source of correctness.

### workio/src/lib/idempotency.ts
```ts
export function generateIdempotencyKey(): string {
  // short, url-safe, timestamp + random to avoid collisions across retries
  const ts = Date.now().toString(36);
  const rand = Math.random().toString(36).slice(2, 7);
  return `punch-${ts}-${rand}`;
}
```

### workio/src/api/punch.ts
```ts
import { generateIdempotencyKey } from './idempotency';

const PENDING = new Set<string>(); // in-flight keys to prevent duplicate submissions from UI

export interface PunchPayload {
  type: 'in' | 'out';
  latitude?: number;
  longitude?: number;
}

export interface PunchResult {
  key: string;
  ok: boolean;
  skipped?: boolean;
  reason?: string;
  data?: any;
  error?: string;
}

export async function punch(payload: PunchPayload, idempotencyKey?: string): Promise<PunchResult> {
  const key = idempotencyKey || generateIdempotencyKey();

  // Prevent concurrent duplicate submissions from same client
  if (PENDING.has(key)) {
    return { key, ok: true, skipped: true, reason: 'already_pending' };
  }
  PENDING.add(key);

  try {
    const res = await fetch('/api/punch', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-Idempotency-Key': key,
      },
      body: JSON.stringify(payload),
    });

    if (!res.ok) {
      const err = await res.json().catch(() => ({}));
      throw new Error(err.message || `HTTP ${res.status}`);
    }

    const data = await res.json();
    return { key, ok: true, data };
  } catch (err: any) {
    return { key, ok: false, error: err?.message || 'Failed to punch' };
  } finally {
    PENDING.delete(key);
  }
}
```

### workio/src/components/PunchButton.tsx
```tsx
import React, { useState } from 'react';
import { punch, PunchPayload } from '../api/punch';
import { generateIdempotencyKey } from '../lib/idempotency';

interface Props {
  type: 'in' | 'out';
  onSuccess?: (result: any) => void;
  onError?: (err: Error) => void;
}

export function PunchButton({ type, onSuccess, onError }: Props) {
  const [status, setStatus] = useState<'idle' | 'pending' | 'success' | 'error'>('idle');
  const [error, setError] = useState<string | null>(null);

  const handleClick = async () => {
    const key = generateIdempotencyKey();
    setStatus('pending');
    setError(null);

    try {
      const result = await punch({ type }, key);
      // deduplicated in-flight; treat as success to avoid noise
      if (result.skipped) {
        setStatus('success');
        onSuccess?.(result);
        return;
      }
      if (!result.ok) {
        throw new Error(result.error || 'Failed to punch');
      }
      setStatus('success');
      onSuccess?.(result);
    } catch (err: any) {
      setStatus('error');
      const msg = err?.message || 'Failed to punch';
      setError(msg);
      onError?.(err);
    }
  };

  return (
    <div className="flex flex-col items-center gap-2">
      <button
        onClick={handleClick}
        disabled={status === 'pending'}
        className={`
          px-6 py-3 rounded-lg font-medium transition
          ${type === 'in'
            ? 'bg-green-600 hover:bg-green-700 text-white'
            : 'bg-red-600 hover:bg-red-700 text-white'}
          ${status === 'pending' ? 'opacity-75 cursor-wait' : ''}
        `}
      >
        {status === 'pending'
          ? 'กำลังบันทึก...'
          : type === 'in'
          ? 'Clock In'
          : 'Clock Out'}
      </button>
      {error && <p className="text-sm text-red-500">{error}</p>}
      {status === 'success' && (
        <p className="text-sm text-green-600">บันทึกเรียบร้อย</p>
      )}
    </div>
  );
}
```
