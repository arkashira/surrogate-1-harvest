# Costinel / ops

## Implementation Plan — Costinel ops increment (<2h)

**Goal:** Add a read-only signal endpoint `GET /api/v1/cost-anomaly/signal/top-hub` (optional `?for_date=YYYY-MM-DD`) that surfaces the strongest knowledge-graph hub (e.g., "MOC") as a cost-anomaly signal — consistent with Sense + Signal philosophy (no execution).

**Why this is highest-value (<2h):**
- Reuses existing knowledge-rag/graph assets (top-hub pattern).
- No schema migrations, no mutating operations, no infra changes.
- Fits existing API surface (`/api/v1/cost-anomaly/signal/...`) and can be consumed by dashboards/alerts immediately.

---

### Tasks (ordered, estimated)

1. **Add route + handler** (45m)
   - `GET /api/v1/cost-anomaly/signal/top-hub`
   - Query params: `for_date` (optional, defaults to today UTC)
   - Response: 200 with signal payload; 404 if no hub; 400 on bad date.

2. **Implement top-hub resolver** (45m)
   - Thin service that calls knowledge-rag to fetch the most-connected hub for the date.
   - Normalize to `{ hub, score, context, related_docs, ts }`.
   - Cache per-date in memory (short TTL) to avoid repeated heavy queries.

3. **Add tests & docs** (20m)
   - Unit test for handler + resolver (mock knowledge-rag).
   - Minimal OpenAPI schema update (or docstring if not using auto-spec).

4. **Verify locally + smoke test** (10m)
   - Start dev server, hit endpoint, confirm shape + no side effects.

### Code snippets

#### 1) Route + handler (FastAPI-like example)

```python
# costinel/api/v1/endpoints/cost_anomaly.py
from datetime import datetime, timezone
from fastapi import APIRouter, HTTPException, Query
from costinel.services.top_hub import resolve_top_hub_signal

router = APIRouter()

@router.get("/signal/top-hub")
async def get_top_hub_signal(
    for_date: str = Query(None, regex=r"^\d{4}-\d{2}-\d{2}$")
):
    """
    Sense + Signal: return the strongest knowledge-graph hub as a cost-anomaly signal.
    No execution / mutation.
    """
    try:
        if for_date:
            target = datetime.strptime(for_date, "%Y-%m-%d").date()
        else:
            target = datetime.now(timezone.utc).date()

        signal = resolve_top_hub_signal(for_date=target)
        if not signal:
            raise HTTPException(status_code=404, detail="No hub signal available for date")
        return signal
    except ValueError:
        raise HTTPException(status_code=400, detail="Invalid for_date format")
```

#### 2) Top-hub resolver (uses knowledge-rag)

```python
# costinel/services/top_hub.py
from datetime import date
from typing import Optional, Dict, Any
from functools import lru_cache

# Mock/placeholder import — replace with actual knowledge-rag client
from costinel.libs.knowledge_rag import query_top_hub

@lru_cache(maxsize=32)
def resolve_top_hub_signal(for_date: date) -> Optional[Dict[str, Any]]:
    """
    Resolve the most-connected hub for `for_date` via knowledge-rag.
    Returns normalized signal payload.
    """
    # query_top_hub should return something like:
    # {"hub": "MOC", "score": 0.92, "context": "...", "related_docs": [...]}
    result = query_top_hub(for_date=for_date.isoformat())
    if not result or not result.get("hub"):
        return None

    return {
        "type": "cost-anomaly-signal",
        "signal": "top-hub",
        "hub": result["hub"],
        "score": float(result.get("score", 0.0)),
        "context": result.get("context", ""),
        "related_docs": result.get("related_docs", []),
        "for_date": for_date.isoformat(),
        "ts": datetime.now(timezone.utc).isoformat(),
        "philosophy": "Sense + Signal — ไม่ Execute",
    }
```

### Notes & Best-practice checks

- **No execution**: endpoint is read-only; no writes to infra/cloud.
- **Caching**: short TTL per-date avoids repeated heavy graph queries during dashboard refreshes.
- **Timezones**: use UTC dates for consistency; accept `for_date` in YYYY-MM-DD.
- **Error handling**: 400 on malformed date, 404 when no hub, 500 on internal errors.
- **Extensibility**: later we can add `?hub=MOC` to fetch signals for specific hubs.

**Estimated total time:** ~1h45m (including tests and smoke). This ships a visible, useful ops signal with zero risk.
