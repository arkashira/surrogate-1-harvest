## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a CF Workers-hosted async gateway exposing a single OpenAI-compatible /v1/chat/completions endpoint that normalizes OpenAI and Anthropic text chat models; supports per-provider API keys from Supabase KV; enforces per-provider quotas and circuit-breaker via Redis; provides round-robin and lowest-latency routing strategies; returns unified usage/cost fields; emits Prometheus metrics and structured RequestLogs to Supabase Postgres; zero frontend (API-only).

## Open questions
- Which provider auth patterns must be supported at v0.1 (API key header vs Bearer vs signed requests)?
- What is the target max p99 latency per provider hop and acceptable retry/backoff policy?
- How should global cost caps be enforced: hard reject vs soft warn vs queue throttle?
- Will PII filtering be best-effort redaction at gateway or require payload re-encoding?
- Which subset of OpenAI/Anthropic/Azure endpoints must be normalized for v0.1?
