## Deployment
cf workers (edge config/feature flags) + Supabase (Postgres/auth) + HuggingFace Spaces (free-tier health/chaos dashboards) + Android app distributed via F-Droid/APK; no cloud VMs required

## v0.1 scope
v0.1 (2 weeks): Android app with hidden local SOCKS5 listener (ephemeral port reuse), TUN interface lifecycle cloaking (rename/teardown on scan heuristics), Noise_IK-secured tunnel to a single CF worker edge relay → Supabase-hosted exit bridge (shadowsocks 2022) in one region; local SQLite policy cache + Supabase Postgres for session state; basic port-scan resistance (packet timing jitter, SYN/ACK mimicry); no telemetry beyond WAUS/U counters; installable via F-Droid-style APK with no root or enterprise MDM required.

## Open questions
- Which port cloaking technique minimizes kernel-level netd hooks on stock Android while remaining performant (e.g., reuse existing allowed ports vs. raw socket masquerading)?
- How will we detect and respond to active TCP fingerprinting (SYN cookies, window sizes) without increasing latency beyond 50ms RTT budget?
- What is the acceptable false-positive rate for auto-triggering TUN interface teardown when a scan is detected?
- Can we rely solely on VPNService split-tunneling to avoid Knox/MDM flags, or do we need a fallback path for devices that block hidden TUN interfaces?
- Which egress transports (shadowsocks 2022, vless over WS+TLS, obfs4) provide best cover under DPI while remaining deployable on free-tier edge workers?
