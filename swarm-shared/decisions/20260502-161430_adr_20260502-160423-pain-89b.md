## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust daemon (CLI) that exposes a SOCKS5 listener on localhost, tunnels traffic through Shadowsocks (chacha20-poly1305) to a free-tier CF Worker endpoint, conceals the local port via SO_REUSEPORT/port cloaking and randomizes TTL/packet padding to reduce DPI fingerprints, runs an OS-level kill-switch (fail-closed) that blocks non-tunneled traffic when the tunnel drops, stores session/profile/telemetry in local SQLite (encrypted-at-rest), and provides client-side WAU-ZB aggregation with no server-side logs — all deployable on free-tier CF Workers and runnable on commodity laptops in restrictive regions.

## Open questions
- Which obfuscation primitives best evade current DPI in target regions (e.g., Shadowsocks 2022, HTTPS CONCEAL, domain fronting) while remaining performant on low-end devices?
- How will we distribute and rotate bridge endpoints without centralized infra (CF Workers + IPFS/Skynet vs. peer exchange)?
- What local port/tun hiding technique is safest and least likely to break networking on host OS (raw sockets vs. TUN tap vs. kernel module)?
- How do we implement a reliable kill-switch across Windows/macOS/Linux that is fail-closed and survives daemon crashes?
- What telemetry schema and client-side aggregation preserves privacy while enabling WAU-ZB measurement?
- How will we handle captive portals and DNS hijacking without leaking plaintext DNS or revealing VPN presence?
