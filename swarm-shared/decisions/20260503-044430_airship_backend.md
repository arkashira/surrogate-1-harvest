**Final synthesized plan (highest-value, 2-hour scope)**

**Goal**: Make Surrogate-1 training robust to HF API 429s and Lightning idle-stop quota loss, using only deterministic CDN data loading and studio lifecycle resilience.

---

### Core decisions (resolve contradictions)
- **Data loading**: Use **CDN-only** with a **one-time, non-recursive file list** (date folder) generated on the Mac.  
  - Why: Avoids HF API pagination/429s and keeps the change small and deterministic.  
  - Correctness: Do not use `load_dataset(streaming=True)` on heterogeneous repos; project columns at parse time (Parquet/JSON/CSV).  
- **Studio lifecycle**: **Reuse running studios**; restart only when stopped. Check status immediately before each `.run()`.  
  - Why: Prevents quota burn from recreation and idle-stop losses.  
  - Actionability: Use Lightning SDK programmatically (not CLI) for reliable status checks and machine selection.

### Implementation plan (≤2 hours)

#### 1) File-list generator (Mac) — 15 min
- Single non-recursive `list_repo_tree` for one date folder (e.g. `batches/mirror-merged/2026-05-03/`).  
- Save relative paths to `training/file_list_2026-05-03.json` and commit.

```python
# tools/build_file_list.py
import json
import os
import sys
from huggingface_hub import HfApi

HF_REPO = os.getenv("HF_DATASET_REPO", "datasets/your-org/surrogate-mirror")
DATE_FOLDER = sys.argv[1] if len(sys.argv) > 1 else "2026-05-03"
PREFIX = f"batches/mirror-merged/{DATE_FOLDER}"
OUT = f"training/file_list_{DATE_FOLDER}.json"

api = HfApi()
entries = api.list_repo_tree(
    repo_id=HF_REPO,
    path=PREFIX,
    recursive=False,
    repo_type="dataset"
)
files = [e.path for e in entries if e.type == "file"]

os.makedirs(os.path.dirname(OUT), exist_ok=True)
with open(OUT, "w") as f:
    json.dump(files, f, indent=2)
print(f"Wrote {len(files)} files to {OUT}")
```

Run:
```bash
python tools/build_file_list.py 2026-05-03
```

#### 2) CDN-only dataset loader — 30 min
- Read `file_list_*.json`, construct CDN URLs, stream via `requests`, project `{prompt, response}`.  
- Use `datasets.Dataset.from_generator` for memory-efficient loading.  
- Keep loader format-agnostic (Parquet preferred; fallback to JSON/CSV).

```python
# surrogate/data/cdn_dataset.py
import json
import os
import io
from typing import Iterator

import pyarrow.parquet as pq
import requests
from datasets import Dataset, Features, Value

HF_REPO = os.getenv("HF_DATASET_REPO", "datasets/your-org/surrogate-mirror")
CDN_ROOT = f"https://huggingface.co/datasets/{HF_REPO}/resolve/main"

def _load_file_list(name: str) -> list[str]:
    path = os.path.join(os.path.dirname(__file__), "..", "..", name)
    with open(path) as f:
        return json.load(f)

def _cdn_records(file_list: list[str]) -> Iterator[dict]:
    for rel in file_list:
        url = f"{CDN_ROOT}/{rel}"
        resp = requests.get(url, timeout=30)
        resp.raise_for_status()
        # Parquet fast path: project only needed columns
        table = pq.read_table(io.BytesIO(resp.content), columns=["prompt", "response"])
        for batch in table.to_batches(max_chunksize=1000):
            df = batch.to_pandas()
            for _, row in df.iterrows():
                yield {"prompt": row["prompt"], "response": row["response"]}

def build_dataset(file_list_name: str = "file_list_2026-05-03.json") -> Dataset:
    files = _load_file_list(file_list_name)
    features = Features({"prompt": Value("string"), "response": Value("string")})
    return Dataset.from_generator(lambda: _cdn_records(files), features=features)
```

Notes:
- If files are not Parquet, replace `pq.read_table` with `pd.read_json`/`pd.read_csv` and project columns.  
- No HF `load_dataset` on the full repo; this avoids schema mixing and 429s.

#### 3) Lightning Studio lifecycle wrapper — 45 min
- Programmatic reuse: find running studio by name; restart if stopped; create only if missing.  
- Check status and restart with `L40S` (or available equivalent) before each training run.  
- Add simple retry/backoff for transient state transitions.

```python
# surrogate/train/lightning_launcher.py
import time
from lightning_sdk import Studio, Teamspace, Machine
from lightning_sdk.exceptions import StudioNotFound

TEAMSPACE = "your-teamspace"
STUDIO_NAME = "surrogate-train-studio"
MACHINE = Machine.L40S  # fallback if free tier unavailable

def _wait_for_running(studio: Studio, timeout: int = 300, interval: int = 10) -> None:
    for _ in range(timeout // interval):
        studio.refresh()
        if studio.status == "running":
            return
        time.sleep(interval)
    raise TimeoutError(f"Studio {studio.name} did not reach 'running' within {timeout}s")

def get_or_create_studio() -> Studio:
    team = Teamspace(TEAMSPACE)
    for s in team.studios:
        if s.name == STUDIO_NAME:
            if s.status == "running":
                print(f"Reusing running studio: {s.name}")
                return s
            print(f"Studio stopped ({s.status}). Restarting...")
            s.start(machine=MACHINE)
            _wait_for_running(s)
            return s
    print("Creating new studio...")
    s = team.create_studio(STUDIO_NAME)
    s.start(machine=MACHINE)
    _wait_for_running(s)
    return s

def run_training(script: str, args: list[str]) -> None:
    studio = get_or_create_studio()
    # Ensure still running before submit
    if studio.status != "running":
        studio.start(machine=MACHINE)
        _wait_for_running(studio)
    run = studio.run(script, args)
    print(f"Submitted run: {run.id}")
    return run
```

Usage:
```python
from surrogate.train.lightning_launcher import run_training

run_training(
    script="train.py",
    args=["--data-file", "file_list_2026-05-03.json", "--epochs", "1"]
)
```
