## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk guardrail → execution stub) with immutable event log in Supabase, Redis Streams queue, edge worker on CF Workers, one broker connector mock, one data vendor mock, compliance rule engine (block/warn) and audit replay endpoint; no UI, API-first, free-tier only.

## Open questions
- Determinism & replay: should event payloads be canonicalized (sorted keys, fixed precision) to guarantee byte-for-byte replay?
- Latency SLA: what is the p99 budget from signal to execution decision (sub-100ms vs sub-500ms)?
- Compliance policy language: use embedded Rego/OPA-lite or keep rules in Python DSL for simplicity and auditability?
- Connector ownership: which brokers/OMS/EMS and data vendors are must-have for v0.1?
- Idempotency and exactly-once vs at-least-once semantics for queue consumers and execution APIs?
- Secrets/credentials model for connectors (CF Workers KV vs Supabase + envelope encryption)?
