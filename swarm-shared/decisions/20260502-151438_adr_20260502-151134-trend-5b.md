## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk guardrail → execution stub) with immutable audit trail, idempotent runs, Postgres-backed queue on Supabase, edge worker on Cloudflare Workers free tier, local mock connectors for one broker and one data vendor, and OpenTelemetry traces exportable to console/OTLP collector.

## Open questions
- Deterministic replay strategy: event-sourcing vs snapshots — storage cost vs replay time trade-offs?
- Idempotency key scope and TTL: global per-connector vs per-tenant vs per-workflow?
- Compliance rule engine: embedded Python predicates vs WASM policies for sandboxing?
- Broker/OMS connector interface contract and mock harness for local testing?
- Latency SLO definition for sub-second: p95 vs p99 and budget per step (research/signal/risk/execution)?
- Secrets/credential management for connectors on free-tier infra (Workers KV vs Supabase Vault vs local env)?
