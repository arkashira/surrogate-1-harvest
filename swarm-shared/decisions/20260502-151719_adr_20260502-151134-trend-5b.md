## v0.1 scope
v0.1 in 2 weeks: deterministic research→signal→risk→execution workflow runner on Cloudflare Workers; Postgres (Supabase) for immutable run/step/audit records; Redis Streams for queueing and replay; one broker connector (sandbox REST), one risk guardrail (notional limit), idempotent runs with signed audit entries; API to submit runs and query status; local replay harness for deterministic regression tests; no UI (headless API-first).

## Open questions
- Deterministic replay boundary: which inputs must be snapshotted to guarantee bitwise replay across connector API changes?
- Idempotency and exactly-once vs at-least-once semantics for broker/OMS connectors under retries and partial failures.
- Latency SLOs per step (target p99 < X ms) and acceptable queueing delay on Redis Streams under bursty market events.
- Compliance override workflow: who/what can authorize a guardrail bypass and how is it immutably recorded?
- Secrets management for connector credentials on Cloudflare Workers free-tier (KMS vs encrypted-at-rest options).
- Data retention and PII minimization for audit trails under regulatory regimes (how long, where, and how to purge/archive).
