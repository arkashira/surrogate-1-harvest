## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a single Rust binary (Linux x86_64) that establishes an obfuscated SOCKS5 proxy (shadowsocks-2022 over utp/TLS meek), creates and hides a TUN interface via boringtun, enforces a local kill-switch (fail-closed), and emits minimal, privacy-preserving session telemetry to Cloudflare Workers Queues; includes a Supabase Postgres schema for client profiles and WAU-ZB rollups and a Cloudflare Worker for ingestion and weekly rollups.

## Open questions
- Which obfuscation front (meek-http, utp, or domain-fronted TLS) provides the best reach/stealth trade-off against current DPI in target regions?
- How will clients discover/rotate bridge endpoints without leaking discovery traffic (DNS vs. out-of-band vs. Snowflake-like broker)?
- What local artifacts (TUN device names, listening ports, packet timing) must be scrubbed to evade on-device forensics and DPI probes?
- How to implement a reliable client-side kill-switch across platforms (Linux netfilter, Windows WFP, macOS PF) without elevated prompts that deter users?
- What minimal telemetry schema preserves privacy while enabling WAU-ZB measurement and avoids deanonymization?
