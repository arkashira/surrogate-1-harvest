## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-file Rust CLI agent (Linux x86_64) with local SOCKS5 listener, TUN shim, and a 'tls-fragment' stealth profile fronted by a CF Worker on a free-tier domain; encrypted SQLite session log; local kill-switch that blocks non-stealth interfaces on failure; no telemetry yet, manual config via JSON file; reproducible build script and HF Space release artifact.

## Open questions
- Which DPI-evasion primitives are safest/most effective against current Russian DPI (TLS fragmentation vs meek vs domain-fronting via CF)?
- How will we distribute and rotate innocuous front domains on CF Workers without central DNS records that can be blocked?
- What local kill-switch enforcement is reliable across desktop OSes (Windows/Linux/macOS) without admin/root?
- How do we implement opt-in, privacy-preserving WAU-ZB telemetry (encrypted client-side counters + periodic sync) without deanonymization?
- Can we produce reproducible, signed binaries from HF Space and verify them client-side to prevent supply-chain tampering?
