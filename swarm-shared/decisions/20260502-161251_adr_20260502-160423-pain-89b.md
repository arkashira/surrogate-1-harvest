## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-file, client-side Rust binary (Linux x86_64) that: creates a local SOCKS5 listener on a random ephemeral port, spawns a TUN interface with randomized MTU/TTL, wraps traffic through a shadowsocks-like obfuscated stream, applies basic HTTP-obfs and utp profiles to mask flows, enforces a process-level killswitch (block all non-stealth interfaces) on stealth failure, records anonymized session/telemetry to local SQLite, and supports zero-config operation with signed, minimal config bundles fetched via CF Workers (free tier) — all under permissive OSS licenses with reproducible builds.

## Open questions
- Which DPI fingerprints are highest risk in target regions (SNI, MTU, TTL, TCP window sizes)?
- Can we rely on Cloudflare Workers as ephemeral, free-tier rendezvous for config/telemetry without logging PII?
- What is the acceptable false-positive rate for the killswitch to avoid breakage while preventing leaks?
- How will clients discover transport endpoints (bridges) without creating a blocking signature?
- Which obfuscation profiles (HTTP/2, utp, domain-fronting-like flows) are feasible client-side only?
