## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a single CF Worker exposing /v1/chat/completions and /v1/completions that normalizes OpenAI and Anthropic chat APIs, with per-provider token-bucket rate limits in Redis, failover to a secondary provider on 429/5xx, basic spend and RPM caps per client key, structured request logs to Postgres (Supabase), and /health + /metrics endpoints. No web UI, no batch endpoints, no fine-grained PII filters yet.

## Open questions
- Which provider auth patterns must be supported at v0.1 (API key header, OAuth2 client-credentials, signed requests)?
- What latency SLO (p95/p99) is required for proxy passthrough vs. queued/retried requests?
- How will global quota be synchronized across CF Workers instances (Redis central vs. edge caching strategy)?
- Which PII detection library/service is acceptable for free-tier (local regex/ML vs. external API)?
- What observability limits are acceptable on free-tier (self-hosted Prometheus vs. logs only)?
