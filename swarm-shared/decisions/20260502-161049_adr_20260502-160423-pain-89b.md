## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single Rust binary (Linux x86_64) that starts a local SOCKS5 listener, tunnels traffic through a user-provided fronted TLS proxy (meek-like) using rustls, hides the local port from basic enumeration, and implements a fail-closed killswitch that blocks all non-proxy traffic on detection; no server components required, optional encrypted telemetry to Supabase Postgres via CF Workers edge function, basic unit/integration tests and CI.

## Open questions
- Which DPI-evasion primitives can be implemented purely client-side without managed relays (e.g., meek-like fronting, domain-fronting via CF Workers)?
- How will we distribute updated evasion profiles to clients in restrictive regions without creating a detectable update channel?
- What local port/tun hiding techniques are reliable across Windows/Linux/macOS without admin/root?
- How do we validate WAU-ZB telemetry without risking deanonymization or storing sensitive endpoints?
- Can CF Workers serve as reflectors/fronts to mask origin traffic while remaining within free-tier limits?
