# workio / backend

## 1. Diagnosis
- Missing request-scoped tenant resolution on `/webhook/line` → repeated DB tenant lookups per LINE event cause DB thrash and increase 429/timeout risk under traffic spikes.
- No LRU cache for `channelId → tenantId` on webhook path; cold-start or burst traffic repeatedly hits `tenants` table.
- Fail-fast signature verification occurs, but tenant resolution runs before cache check, wasting DB cycles.
- No short-circuit for known-bad/expired channel IDs; each invalid event still pays DB + crypto verification cost.
- Missing observability (timing + tenantId) on webhook handler makes it hard to detect cache effectiveness or tenant resolution regressions.

## 2. Proposed change
- **File**: `workio/server/src/routes/webhook/line.ts` (or equivalent route handler for `/webhook/line`)
- **Scope**: add a lightweight, request-scoped LRU cache (TTL 10m, max 500) for `channelId → tenantId` and attach tenant to `req.tenant` early; short-circuit on cache hit and skip DB; emit timing metric.

Create cache module (`workio/server/src/lib/tenant-cache.ts`):

```ts
import { LRUCache } from 'lru-cache';

export const tenantCache = new LRUCache<string, { tenantId: string; tenant: any }>({
  max: 500,
  ttl: 10 * 60 * 1000, // 10 minutes
  allowStale: false,
  updateAgeOnGet: true,
});
```

Update webhook route (`workio/server/src/routes/webhook/line.ts`):
```ts
import express from 'express';
import crypto from 'crypto';
import { tenantCache } from '../../lib/tenant-cache';
import { getTenantByChannelId } from '../../services/tenant.service'; // adjust import path

const router = express.Router();

function verifyLineSignature(rawBody: Buffer, signature: string, secret: string): boolean {
  const hash = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(hash), Buffer.from(signature));
}

router.post('/line', async (req, res) => {
  const start = Date.now();
  const rawBody = req.body?.raw || req.body; // ensure raw buffer available; adapt if body-parser overwrites
  const signature = req.headers['x-line-signature'] as string;
  const channelSecret = process.env.LINE_CHANNEL_SECRET;

  // Fail-fast signature verification
  if (!signature || !channelSecret || !verifyLineSignature(Buffer.from(JSON.stringify(req.body)), signature, channelSecret)) {
    console.warn('[webhook/line] invalid signature', { hasSig: !!signature, hasSecret: !!channelSecret });
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // Events are in body.events (LINE Messaging API format)
  const events = req.body?.events || [];
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'No events' });
  }

  // Use first event's source.channelId to resolve tenant (all events in batch expected same channel)
  const channelId = events[0]?.source?.userId ? events[0].source.userId : events[0]?.source?.groupId || events[0]?.source?.roomId;
  // Note: LINE events use source.userId (U...), groupId (C...), roomId (R...). For channel-level mapping,
  // we map by bot's channelId (from channel secret) — but tenant mapping is typically channelId (channel OA).
  // We'll use a stable channel identifier from env or event destination; fallback to first event source.
  // If your mapping is by LINE ChannelId (OA), set LINE_CHANNEL_ID in env and use that.
  const lineChannelId = process.env.LINE_CHANNEL_ID || channelId;

  if (!lineChannelId) {
    console.warn('[webhook/line] cannot determine channelId', { body: req.body });
    return res.status(400).json({ error: 'Missing channelId' });
  }

  // Cache lookup
  let cached = tenantCache.get(lineChannelId);
  if (cached) {
    req.tenant = cached.tenant;
    req.tenantId = cached.tenantId;
  } else {
    // Resolve tenant by channelId (adjust service call to your schema)
    const tenant = await getTenantByChannelId(lineChannelId);
    if (!tenant) {
      console.warn('[webhook/line] tenant not found for channel', { lineChannelId });
      return res.status(404).json({ error: 'Tenant not found' });
    }
    req.tenant = tenant;
    req.tenantId = tenant.id;
    tenantCache.set(lineChannelId, { tenantId: tenant.id, tenant });
  }

  // Attach tenant to res.locals for downstream middleware/services
  res.locals.tenant = req.tenant;
  res.locals.tenantId = req.tenantId;

  // Process events (keep existing business logic)
  try {
    // Example: delegate to handler
    // await handleLineEvents(events, req.tenant);
    // For now, just acknowledge
    console.info('[webhook/line] processed', { tenantId: req.tenantId, eventCount: events.length, cached: !!cached, ms: Date.now() - start });
    return res.status(200).json({ ok: true });
  } catch (err) {
    console.error('[webhook/line] processing error', { tenantId: req.tenantId, err });
    return res.status(500).json({ error: 'Processing failed' });
  }
});

export default router;
```

If body-parser JSON parsing destroys raw buffer, preserve raw body with a small middleware before routes (`workio/server/src/middleware/raw-body.ts`):
```ts
import express from 'express';

export function preserveRawBody(req: express.Request, res: express.Response, next: express.NextFunction) {
  const chunks: Buffer[] = [];
  req.on('data', (chunk) => chunks.push(chunk));
  req.on('end', () => {
    const raw = Buffer.concat(chunks);
    (req as any).rawBody = raw;
    // Re-parse JSON for downstream handlers (safe because we consumed stream)
    try {
      req.body = raw.length > 0 ? JSON.parse(raw.toString()) : {};
    } catch {
      req.body = {};
    }
    (req.body as any).raw = raw;
    next();
  });
}
```
Then in your main app file, use it before webhook routes:
```ts
import { preserveRawBody } from './middleware/raw-body';
app.use('/webhook/line', preserveRawBody, lineWebhookRouter);
```
