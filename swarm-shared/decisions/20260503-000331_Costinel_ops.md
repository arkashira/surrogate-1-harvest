# Costinel / ops

## Final Implementation Plan — Costinel: Top-Hub Signal Card (Read-Only)

**Scope**: Read-only frontend card (≤2h)  
**Principle**: “Sense + Signal — ไม่ Execute” (strictly no writes, no runtime mutations, no self-execution)  
**Goal**: Surface the most-connected hub (e.g., “MOC”) with contextual insights as a persistent, read-only signal card so operators see top-context before acting.

---

### Why this is the highest-value incremental (<2h)
- Applies proven pattern `top-hub doc insight` + `knowledge-rag` without new infra.
- Read-only card avoids governance/change-management complexity and aligns with Costinel philosophy.
- Implemented as a single frontend component + static JSON import (zero backend changes).
- Separates concerns: market analysis and RAG enrichment remain offline/ops tasks; the UI only presents curated, vetted signals.

---

### Implementation Steps (≤2h)

1. **Create static top-hub payload** (`src/data/top-hub-signal.json`)
   - Contains: hub label (e.g., “MOC”), short description, insight bullets, tags.
   - No runtime fetches or mutations.
   - Updated offline by ops when market analysis + knowledge-rag runs (see Ops note below).

2. **Add read-only card component** (`src/components/TopHubSignalCard.tsx`)
   - Pure presentational component.
   - Uses static JSON import.
   - No state changes that trigger side effects.
   - No network requests or writes.
   - Visual style consistent with Costinel dashboard (cards, badges, muted tones).

3. **Mount card on dashboard** (`src/pages/Dashboard.tsx`)
   - Place in “Sense + Signal” section near cost overview.
   - No wiring to actions or handlers.

4. **Add snapshot test** — ensures read-only rendering remains stable.

5. **Verify build + run** — ensure no console errors and card renders.

#### 2) Read-only card component
```tsx
// src/components/TopHubSignalCard.tsx
import React from "react";
import topHubSignal from "../data/top-hub-signal.json";

export const TopHubSignalCard: React.FC = () => {
  return (
    <section
      aria-label="Top hub signal"
      className="rounded-lg border bg-card p-5 shadow-sm"
    >
      <div className="flex items-start justify-between">
        <div>
          <h3 className="text-lg font-semibold text-foreground">
            {topHubSignal.label}
          </h3>
          <p className="text-sm text-muted-foreground">
            {topHubSignal.description}
          </p>
        </div>
        <span className="inline-flex items-center rounded bg-primary/10 px-2 py-1 text-xs font-medium text-primary">
          {topHubSignal.hub}
        </span>
      </div>

      <ul className="mt-4 space-y-2">
        {topHubSignal.insights.map((insight, idx) => (
          <li key={idx} className="flex gap-2 text-sm text-foreground">
            <span className="mt-1 flex-shrink-0 text-primary" aria-hidden>
              •
            </span>
            <span>{insight}</span>
          </li>
        ))}
      </ul>

      <div className="mt-4 flex flex-wrap gap-1">
        {topHubSignal.tags.map((tag) => (
          <span
            key={tag}
            className="rounded bg-muted px-2 py-0.5 text-xs text-muted-foreground"
          >
            {tag}
          </span>
        ))}
      </div>
    </section>
  );
};
```
