# airship / design

**Final Implementation Plan**  
*Goal: Ingest Surrogate-1 dataset deterministically, avoid HF API rate limits, fix PyArrow schema issues, and enable Lightning training with CDN-only, streaming data.*

---

## 1. Pre-List Files Once (CDN-Only File Discovery)
- **Action:** Call `list_repo_tree(path, recursive=False)` **once** (from a Mac or low-rate environment after the rate-limit window) for a single date folder.
- **Output:** Save the flat list of file paths to `file_list.json`.
- **Embed:** Commit `file_list.json` into the repo/training container so training never calls `list_repo_tree` again.
- **Why:** Eliminates repeated API calls and ensures deterministic file sets across runs.

---

## 2. Download via CDN URLs (Bypass HF API Limits)
- **Action:** For each file in `file_list.json`, fetch directly via:
  ```
  https://huggingface.co/datasets/{repo}/resolve/main/{file_path}
  ```
- **No API token required** for public datasets; uses CDN bandwidth/rate limits (separate and higher than API limits).
- **Implementation:** Use `requests.get(url, timeout=30, stream=True)` and stream to disk or memory.
- **Why:** Avoids `hf_hub_download` API path and associated rate limits; faster and more reliable for bulk reads.

---

## 3. Parse Parquet with Projection to `{prompt, response}` (Fix PyArrow Schema Errors)
- **Action:** Read each downloaded Parquet file with `pyarrow.parquet.read_table` and immediately project columns:
  ```python
  table = pq.read_table(file_path, columns=["prompt", "response"])
  ```
- **Do not** rely on `pyarrow.deserialize` (not standard for Parquet); instead use canonical Parquet readers.
- **Why:** Projection at read time avoids schema mismatches and reduces memory usage.

---

## 4. Deterministic Sibling Repo for Writes (Optional Ingestion Target)
- **Action:** If writing outputs (e.g., processed shards), choose a deterministic sibling repo:
  ```python
  import hashlib
  repo_hash = hashlib.sha256(repo.encode()).hexdigest()
  sibling_repo = f"{repo_hash[:5]}-{repo_hash[5:10]}-{repo_hash[10:15]}"
  ```
- **Use case:** Spread write load across repos to respect per-repo rate limits (e.g., 640/hr aggregate across 5 siblings).
- **Why:** Enables safe, deterministic ingestion pipelines without centralized bottlenecks.

---

## 5. Stream Data into Lightning Training
- **Action:** Implement a PyTorch `IterableDataset` that:
  - Iterates over `file_list.json`.
  - Downloads each file from CDN (or reads from local cache).
  - Projects to `{prompt, response}`.
  - Yields rows one-by-one.
- **Configure Lightning:**
  ```python
  DataLoader(..., num_workers>0, pin_memory=True)
  Trainer(..., strategy="auto", devices="auto")
  ```
  and ensure dataset is `streaming=True`-like behavior via iterable dataset.
- **Why:** Enables constant-memory training over large datasets without API calls during training.

---

## 6. Surrogate-1 Ingestion Output Layout
- **Action:** After projection, write processed shards to:
  ```
  batches/mirror-merged/{date}/{slug}.parquet
  ```
- **Do not add** `source` or `ts` columns; keep attribution in filename/path.
- **Why:** Matches expected Surrogate-1 layout and avoids schema pollution.

---

## 7. Testing and Validation
- **Smoke test:** Read a sample shard and verify schema:
  ```python
  df = pq.read_table(shard_path, columns=["prompt", "response"]).to_pandas()
  assert set(df.columns) == {"prompt", "response"}
  assert df.notna().all().all()
  ```
- **Count check:** Ensure row counts match expected totals per date.
- **Why:** Catches projection or CDN fetch errors early.

---

## 8. Orchestrator (Optional Container)
- **Action:** Wrap steps 1–6 in a lightweight orchestrator script/container that:
  - Takes `repo`, `date`, `output_root` as inputs.
  - Runs file listing (if not cached).
  - Downloads + projects + writes shards.
  - Emits logs and exit codes for CI/CD.
- **Why:** Enables reproducible, scheduled ingestion without manual intervention.

---

## Final Code Skeleton

```python
import json, hashlib, requests, pyarrow.parquet as pq
from torch.utils.data import IterableDataset, DataLoader

# 1. Load deterministic file list
with open("file_list.json") as f:
    file_list = json.load(f)

# 2. CDN download + 3. Projection
def cdn_read_table(file_path, repo):
    url = f"https://huggingface.co/datasets/{repo}/resolve/main/{file_path}"
    resp = requests.get(url, stream=True, timeout=30)
    resp.raise_for_status()
    with open("/tmp/temp.parquet", "wb") as f:
        for chunk in resp.iter_content(chunk_size=8192):
            f.write(chunk)
    return pq.read_table("/tmp/temp.parquet", columns=["prompt", "response"])

# 4. Optional sibling repo for writes
def sibling_repo(repo):
    h = hashlib.sha256(repo.encode()).hexdigest()
    return f"{h[:5]}-{h[5:10]}-{h[10:15]}"

# 5. Streaming dataset for Lightning
class Surrogate1Dataset(IterableDataset):
    def __init__(self, file_list, repo):
        self.file_list = file_list
        self.repo = repo
    def __iter__(self):
        for file in self.file_list:
            table = cdn_read_table(file, self.repo)
            for row in table.to_pylist():
                yield row["prompt"], row["response"]

# Usage
dataset = Surrogate1Dataset(file_list, repo="Surrogate-1")
loader = DataLoader(dataset, batch_size=32, num_workers=4)
```
