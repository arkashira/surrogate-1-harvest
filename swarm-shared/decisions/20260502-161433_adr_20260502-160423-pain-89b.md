## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI stealth client (Linux/Android-first) that: 1) establishes a userspace TUN via boringtun, 2) exposes a local SOCKS5 listener whose traffic is obfuscated with a simple XOR+TLS-mimic shim to appear as benign TLS, 3) implements a process-level kill-switch that blocks non-stealth traffic if the obfuscated path fails, 4) records minimal, client-side encrypted session telemetry to Supabase (no server logs), and 5) provides a basic CF Worker endpoint for signed blocklist/feature-flag KV updates — all deployable and operable with zero server control.

## Open questions
- Which DPI-evasion primitives (e.g., shadowsocks-like obfuscation vs. domain-fronting vs. TLS mimicry) can be implemented client-side without server cooperation?
- How will we detect and measure 'blocked' sessions reliably on the client without leaking telemetry?
- What minimal TUN configuration is required across platforms (Linux/Android/Windows) to remain undetectable by local probes?
- How do we enforce fail-closed kill-switch semantics across platforms when stealth path breaks?
- Can we use Cloudflare Workers + KV/Queue for free-tier telemetry ingestion while preserving user privacy (no IP/PII stored)?
- What client-side encryption scheme and key management balances usability with deniability for stored configs?
