## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI scanner + FastAPI service + Supabase Postgres; discovers Lambda schedules and Cloud Functions, normalizes cron expressions, flags duplicates/drifted schedules and orphaned rules, exposes REST endpoints for listing and retiring schedules, and records audit events. No UI, no queue, no cache; deploy to CF Workers free tier with Supabase free tier.

## Open questions
- Which identity will the scanner use to read Lambda schedules and Cloud Functions across customer accounts (assume cross-account roles or service tokens)?
- How often should full scans run to balance freshness vs rate limits/costs (hourly vs daily)?
- What retention window for metadata and audit events given free-tier Postgres limits?
- How will customers onboard: provide credentials file, or connect via OAuth/cloud console?
