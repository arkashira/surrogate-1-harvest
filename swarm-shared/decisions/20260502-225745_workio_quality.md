# workio / quality

**Final synthesized solution (correctness + concrete actionability)**

**Guiding principles**
- Prevent duplicates at the **database layer** (source of truth) so races, retries, and replays cannot create double punches.
- Accept an **idempotency key** from callers (frontend stable key + LINE `eventId`) and enforce it atomically.
- Make the **frontend queue durable, observable, and self-healing** with TTL, backoff, poison handling, and metrics.
- Keep behavior simple and deterministic: one punch per employee per punch_type per short window; retries and replays are no-ops.

---

## 1) Database schema (server)

```sql
-- Idempotency ledger (all callers)
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  employee_id TEXT NOT NULL,
  punch_type TEXT NOT NULL,
  punched_at TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: keep a short TTL for keys to avoid unbounded growth
-- CREATE INDEX IF NOT EXISTS ix_punch_idempotency_created ON punch_idempotency (created_at);

-- Business rule: one punch per employee/type/minute
-- (adjust granularity to seconds if needed)
CREATE UNIQUE INDEX IF NOT EXISTS uq_employee_punch_window
ON punches (employee_id, punch_type, date_trunc('minute', punched_at));
```

Why this wins:
- `punch_idempotency` gives a single place to check retries and replays (frontend-originated or LINE).
- The unique index enforces the business rule at the DB level, eliminating races.

## 2) Server: atomic recordPunch (single source of truth)

File: `workio/server/src/services/attendanceService.ts`

```ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function recordPunch(
  employeeId: string,
  punchType: 'clock_in' | 'clock_out',
  punchedAt: Date,
  idempotencyKey: string,
  { retryCount = 2 }: { retryCount?: number } = {}
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency check (atomic with later insert)
    const dup = await client.query(
      'SELECT 1 FROM punch_idempotency WHERE idempotency_key = $1',
      [idempotencyKey]
    );
    if (dup.rows.length > 0) {
      await client.query('ROLLBACK');
      return { ok: true, duplicate: true, reason: 'idempotency_key_already_processed' };
    }

    // 2) Try insert; unique violation => duplicate by business rule
    const result = await client.query(
      `INSERT INTO punches (employee_id, punch_type, punched_at, created_at)
       VALUES ($1, $2, $3, NOW())
       ON CONFLICT (employee_id, punch_type, date_trunc('minute', punched_at))
       DO NOTHING
       RETURNING id`,
      [employeeId, punchType, punchedAt.toISOString()]
    );

    if (result.rowCount === 0) {
      // Duplicate by business rule — still record idempotency to make retries no-op
      await client.query(
        'INSERT INTO punch_idempotency (idempotency_key, employee_id, punch_type, punched_at) VALUES ($1, $2, $3, $4)',
        [idempotencyKey, employeeId, punchType, punchedAt]
      );
      await client.query('COMMIT');
      return { ok: true, duplicate: true, reason: 'punch_already_exists' };
    }

    // 3) Record idempotency
    await client.query(
      'INSERT INTO punch_idempotency (idempotency_key, employee_id, punch_type, punched_at) VALUES ($1, $2, $3, $4)',
      [idempotencyKey, employeeId, punchType, punchedAt]
    );

    await client.query('COMMIT');
    return { ok: true, duplicate: false, punchId: result.rows[0].id };
  } catch (err: any) {
    await client.query('ROLLBACK');
    // Retry transient deadlocks
    if (retryCount > 0 && err.code === '40P01') {
      return recordPunch(employeeId, punchType, punchedAt, idempotencyKey, { retryCount: retryCount - 1 });
    }
    console.error('recordPunch failed', { employeeId, punchType, idempotencyKey, err });
    throw err;
  } finally {
    client.release();
  }
}
```
