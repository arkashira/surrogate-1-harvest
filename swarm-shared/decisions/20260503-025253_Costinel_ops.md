# Costinel / ops

## Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h)

### What we ship
- A **non-blocking Top-Hub Signal Panel** mounted on the Costinel dashboard (sidebar or top banner area).
- Defaults to hub **MOC** (configurable via `VITE_HUB_NAME`).
- CDN-first data strategy: single pre-listed file path embedded at build time; runtime fetches via public CDN (`resolve/main/`) to avoid HF API rate limits.
- Shows: hub title, short description, top 3 signals (anomalies/recommendations), and a “View details” link.
- Graceful fallback to local stub data if CDN unavailable (no dashboard breakage).

### Why this is highest-value (<2h)
- Directly applies **#knowledge-rag #graph #hub** pattern (top-hub insight).
- Uses **HF CDN bypass** pattern to avoid rate limits during dashboard usage.
- Non-blocking: failure does not block Costinel core flows.
- Reuses existing frontend infra; no backend changes required.
- Delivers immediate contextual ops value to Costinel users.

#### 3) Hub signal panel component
`/opt/axentx/Costinel/src/components/HubSignalPanel.vue`
```vue
<template>
  <section class="hub-signal-panel" v-if="data">
    <header class="hub-header">
      <h3>{{ data.title }}</h3>
      <p class="hub-desc">{{ data.description }}</p>
    </header>

    <ul class="signals-list">
      <li
        v-for="s in data.signals"
        :key="s.id"
        class="signal-item"
        :class="`severity-${s.severity}`"
      >
        <div class="signal-title">{{ s.title }}</div>
        <div class="signal-meta">
          <span class="impact">{{ s.impact }}</span>
        </div>
        <div class="signal-action">{{ s.action }}</div>
      </li>
    </ul>

    <footer class="hub-footer">
      <router-link :to="data.link" class="btn-link">View details</router-link>
      <span v-if="source === 'cdn'" class="source-badge">live</span>
      <span v-else class="source-badge">fallback</span>
    </footer>
  </section>
</template>

<script>
const HUB_NAME = import.meta.env.VITE_HUB_NAME || "MOC";
const FILE_LIST_PATH = "/data/hub-file-list.json";
const STUB_PATH = "/data/hub-stub.json";

export default {
  name: "HubSignalPanel",
  data() {
    return {
      data: null,
      source: null,
    };
  },
  async mounted() {
    await this.load();
  },
  methods: {
    async load() {
      try {
        // 1) Try CDN-first using pre-listed path (no HF API calls at runtime)
        const listRes = await fetch(FILE_LIST_PATH);
        if (!listRes.ok) throw new Error("No file list");
        const list = await listRes.json();

        const file = list.files?.[0];
        if (!file) throw new Error("No file in list");

        const cdnUrl = `${list.cdnBase}/${file}`;
        const dataRes = await fetch(cdnUrl, { cache: "no-cache" });
        if (!dataRes.ok) throw new Error("CDN fetch failed");

        const payload = await dataRes.json();
        this.data = this.normalize(payload);
        this.source = "cdn";
      } catch (err) {
        // 2) Fallback to local stub (non-blocking)
        console.warn("[HubSignalPanel] CDN unavailable, using stub:", err);
        try {
          const stubRes = await fetch(STUB_PATH);
          const stub = await stubRes.json();
          this.data = this.normalize(stub);
          this.source = "stub";
        } catch (err2) {
          console.error("[HubSignalPanel] Stub unavailable, skipping panel", err2);
          this.data = null;
        }
      }
    },
    normalize(raw) {
      // Ensure expected shape
      return {
        hub: raw.hub || HUB_NAME,
        title: raw.title || `${HUB_NAME} — Hub`,
        description: raw.description || "Cost signals and recommendations.",
        signals: Array.isArray(raw.signals) ? raw.signals.slice(0, 3) : [],
        link: raw.link || `/hubs/${HUB_NAME.toLowerCase()}`,
      };
    },
  },
};
</script>
