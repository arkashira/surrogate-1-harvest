# workio / backend

## 1. Diagnosis

- **No idempotency enforcement on LINE webhook handler**: duplicate punches occur when LINE retries (network timeout / 5xx / slow client) because handler lacks idempotency key checks.
- **Race condition in “last punch” check**: concurrent webhooks can both read “no recent punch” and insert duplicate clock-in/out rows (read-then-write without locking).
- **No database uniqueness guard**: schema allows duplicate `(user_id, tenant_id, punch_type, punch_time)` within tolerance; duplicates slip through when retries hit.
- **Application-level de-dup is best-effort only**: in-memory or time-window checks are lost on restart/scale and are not transactionally safe.
- **Missing idempotency key propagation from frontend**: frontend does not generate or attach `X-Idempotency-Key` for manual punches, leaving non-LINE paths exposed to double-taps.

## 2. Proposed change

- **File**: `workio/server/src/routes/webhook/line.ts` (or equivalent webhook handler)
- **Scope**: add idempotency key extraction (from header `X-Line-Idempotency-Key` or derive from `body.events[].webhookEventId`), wrap punch creation in an atomic upsert/locking flow, and add a partial unique index on `(user_id, tenant_id, punch_type, DATE(punch_time))` where `deleted_at IS NULL` to block duplicates at the DB level.

### 3.2 Idempotent punch creation in webhook handler

```ts
// server/src/routes/webhook/line.ts
import { Request, Response } from 'express';
import { pool } from '../../db';
import { v4 as uuidv4 } from 'uuid';

export async function handleLineWebhook(req: Request, res: Response) {
  try {
    const body = req.body;
    if (!body?.events?.length) return res.status(200).json({ ok: true });

    const client = await pool.connect();
    try {
      await client.query('BEGIN');

      for (const event of body.events) {
        // Use LINE webhookEventId as idempotency key; fallback to generated
        const idemKey = event.webhookEventId || `manual-${uuidv4()}`;
        const source = event.source?.userId;
        const type = event.type; // e.g., 'message' with payload for clock in/out
        const punchTime = new Date(event.timestamp || Date.now());

        if (!source) continue;

        // Resolve user/tenant (simplified)
        const userRes = await client.query(
          `SELECT id, tenant_id FROM users WHERE line_user_id = $1 LIMIT 1`,
          [source]
        );
        if (!userRes.rows.length) continue;
        const user = userRes.rows[0];

        // Determine punch_type from payload (clock_in / clock_out)
        const punchType = inferPunchType(event); // implement per your payload

        // Idempotency check: did we already process this idemKey?
        const idemRes = await client.query(
          `SELECT id FROM punches WHERE idempotency_key = $1 LIMIT 1`,
          [idemKey]
        );
        if (idemRes.rows.length) continue;

        // Try insert; unique index prevents duplicates on same day/type
        try {
          await client.query(
            `INSERT INTO punches (
              id, user_id, tenant_id, punch_type, punch_time,
              idempotency_key, created_at, updated_at, deleted_at
            ) VALUES (
              $1, $2, $3, $4, $5, $6, NOW(), NOW(), NULL
            )`,
            [uuidv4(), user.id, user.tenant_id, punchType, punchTime, idemKey]
          );
        } catch (err: any) {
          // Unique violation: duplicate on same day/type; safe to ignore
          if (err.code === '23505') continue;
          throw err;
        }
      }

      await client.query('COMMIT');
    } catch (err) {
      await client.query('ROLLBACK');
      console.error('Webhook tx failed', err);
      return res.status(500).json({ error: 'internal' });
    } finally {
      client.release();
    }

    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('Webhook handler error', err);
    res.status(500).json({ error: 'internal' });
  }
}

function inferPunchType(event: any): 'clock_in' | 'clock_out' {
  // Implement based on your LINE payload / command / NLP
  // Example: text "in" -> clock_in, "out" -> clock_out
  const text = (event.message?.text || '').toLowerCase().trim();
  if (text === 'in' || text === 'clock in') return 'clock_in';
  if (text === 'out' || text === 'clock out') return 'clock_out';
  // Default fallback: if last punch today is 'clock_in', next is 'clock_out'
  return 'clock_in';
}
```

### 3.4 Frontend: generate idempotency key for manual punches

```ts
// frontend/src/lib/api/punch.ts
export async function punch(action: 'in' | 'out') {
  const idemKey = `manual-${crypto.randomUUID()}`;
  return fetch('/api/punch', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Idempotency-Key': idemKey
    },
    body: JSON.stringify({ action })
  }).then(res => res.json());
}
```

Backend `/api/punch` should mirror the same idempotency+upsert logic as the webhook handler.
