# workio / design

## 1. Diagnosis

- **Missing DB uniqueness guardrail**: no constraint preventing multiple open punches (`clock_out_at IS NULL`) per user per day, allowing duplicates under LINE retries or concurrency.
- **Race-prone app upsert**: `find-then-insert` in the clock-in handler is unsafe; concurrent LINE webhook deliveries can create duplicate open punches.
- **No idempotency key**: LINE Messaging API retries on 5xx/timeouts; without idempotency, retries create duplicate punches instead of returning the existing one.
- **Schema drift risk**: migrations are described in README but not versioned/migrated via tooling; manual `psql < schema.sql` invites environment skew.
- **Missing defensive index**: queries filtering by `user_id`, `tenant_id`, and `clock_out_at IS NULL` will table-scan as data grows.

## 2. Proposed change

- **Scope**: `/opt/axentx/workio/server/src/db/schema.sql` + `/opt/axentx/workio/server/src/db/migrations/` (create migration), plus `/opt/axentx/workio/server/src/routes/line.ts` (clock-in handler).
- **Goal**: enforce “one open punch per user per day” at DB level, add idempotency for LINE retries, and add safe migration path.

### 3.2 Apply migration safely (one-time bootstrap)

`/opt/axentx/workio/server/src/db/migrate.js`
```js
#!/usr/bin/env node
import { readFileSync } from 'fs';
import { resolve, dirname } from 'path';
import { fileURLToPath } from 'url';
import pg from 'pg';

const __dirname = dirname(fileURLToPath(import.meta.url));
const pool = new pg.Pool({
  connectionString: process.env.DATABASE_URL,
});

async function run() {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    await client.query(`
      CREATE TABLE IF NOT EXISTS schema_migrations (
        version TEXT PRIMARY KEY,
        applied_at TIMESTAMPTZ DEFAULT NOW()
      );
    `);

    const version = '001_punches_guardrails';
    const { rows } = await client.query(
      'SELECT 1 FROM schema_migrations WHERE version = $1',
      [version]
    );
    if (rows.length > 0) {
      console.log(`Migration ${version} already applied.`);
      await client.query('ROLLBACK');
      return;
    }

    const sql = readFileSync(
      resolve(__dirname, 'migrations', `${version}.sql`),
      'utf8'
    );
    await client.query(sql);
    await client.query(
      'INSERT INTO schema_migrations (version) VALUES ($1)',
      [version]
    );
    await client.query('COMMIT');
    console.log(`Migration ${version} applied.`);
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Migration failed:', err);
    process.exit(1);
  } finally {
    client.release();
    await pool.end();
  }
}

run();
```

### 3.3 Update clock-in handler for idempotency and safety

`/opt/axentx/workio/server/src/routes/line.ts` (relevant excerpt)

```ts
// Add near top
import { v4 as uuidv4 } from 'uuid';

// In clock-in handler (pseudo-adapted to existing code):
router.post('/webhook/line', async (req, res) => {
  const { events } = req.body;
  const deliveryId = req.headers['x-line-delivery-id'] as string || uuidv4();

  for (const event of events) {
    if (event.type !== 'message' || event.message.type !== 'text') continue;
    const userId = event.source.userId;
    const tenantId = /* derive tenant from userId or channel secret */;
    const text = event.message.text.trim();

    if (text.toLowerCase() === 'clock in') {
      // Idempotent: if same LINE delivery already processed, return existing punch
      const existingByDelivery = await db.oneOrNone(
        `SELECT id FROM punches WHERE line_delivery_id = $1`,
        [deliveryId]
      );
      if (existingByDelivery) {
        return res.json({ status: 'ok', replayed: true, punchId: existingByDelivery.id });
      }

      // Safe upsert pattern: try insert; on conflict (same day open), return existing
      const now = new Date();
      const today = now.toISOString().split('T')[0];

      try {
        const result = await db.one(
          `
          INSERT INTO punches (user_id, tenant_id, clock_in_at, line_delivery_id)
          VALUES ($1, $2, $3, $4)
          RETURNING id
          `,
          [userId, tenantId, now, deliveryId]
        );
        return res.json({ status: 'ok', punchId: result.id });
      } catch (err: any) {
        // Unique violation from our guardrail -> fetch existing open punch
        if (err.code === '23505') {
          const existing = await db.oneOrNone(
            `SELECT id FROM punches
             WHERE user_id = $1 AND tenant_id = $2
               AND clock_in_at::date = $3
               AND clock_out_at IS NULL
             ORDER BY clock_in_at DESC LIMIT 1`,
            [userId, tenantId, today]
          );
          if (existing) {
            return res.json({ status: 'ok', replayed: true, punchId: existing.id });
          }
        }
        // Unexpected: log and return 500 so LINE may retry (idempotency still protects)
        console.error('Clock-in failed', err);
        return res.status(500).json({ error: 'internal' });
      }
    }

    // clock-out and other handlers omitted for brevity
  }

  res.json({ status: 'ok' });
});
```
