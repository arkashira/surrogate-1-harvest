## Decision: ship `airship discover` (backend)

**Highest-value incremental improvement (≤2h):**  
Add a zero-config, cron-safe discovery/training orchestrator that:

- runs `granite-business-research.sh` (if present) and tags outputs with `#business-research #knowledge-rag #graph`
- executes `knowledge-rag` to query top hub and related docs (tag `#knowledge-rag #graph #hub`)
- produces a **CDN-only training manifest** (HF dataset file list via single `list_repo_tree` → JSON) so Lightning training can bypass HF API rate limits during data loading (tags: `#training #api-strategy #cdn #rate-limit-bypass`)
- ensures cron-safe wrapper behavior (shebang, executable, `SHELL=/bin/bash`, invoke via `bash`)

## Code Snippets

### `/opt/axentx/airship/airship-discover`

```bash
#!/usr/bin/env bash
# airship-discover
# Zero-config, cron-safe orchestrator for business research, knowledge-rag,
# and CDN-only training manifest generation.
#
# Usage (cron):
#   SHELL=/bin/bash
#   0 2 * * * bash /opt/axentx/airship/airship-discover >> /var/log/airship-discover.log 2>&1
#
# Tags emitted:
#   #business-research #knowledge-rag #graph #hub #training #cdn #rate-limit-bypass

set -euo pipefail

BASE_DIR="/opt/axentx/airship"
MANIFEST_DIR="${BASE_DIR}/manifests"
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
DATE_TAG=$(date -u +"%Y-%m-%d")

log() {
  echo "[${TIMESTAMP}] $*"
}

ensure_dir() {
  mkdir -p "$1"
}

run_granite_research() {
  local script="${BASE_DIR}/granite-business-research.sh"
  if [[ -x "${script}" ]]; then
    log "Running granite-business-research.sh #business-research #knowledge-rag #graph"
    bash "${script}"
  else
    log "No executable granite-business-research.sh found, skipping. #business-research"
  fi
}

run_knowledge_rag() {
  # Query top hub (e.g., MOC) and related docs.
  # Assumes `knowledge-rag` is available in PATH or as a script.
  if command -v knowledge-rag >/dev/null 2>&1; then
    log "Running knowledge-rag top-hub query #knowledge-rag #graph #hub"
    knowledge-rag query --top-hub --tag "#knowledge-rag #graph #hub" || true
  elif [[ -x "${BASE_DIR}/knowledge-rag" ]]; then
    log "Running ./knowledge-rag top-hub query #knowledge-rag #graph #hub"
    bash "${BASE_DIR}/knowledge-rag" query --top-hub --tag "#knowledge-rag #graph #hub" || true
  else
    log "knowledge-rag not found, skipping top-hub query. #knowledge-rag"
  fi
}

generate_cdn_manifest() {
  # Uses HF Hub Python SDK to list repo tree once (single API call),
  # then emits CDN-only manifest for Lightning training.
  #
  # Requirements: huggingface_hb installed in this environment.
  #
  # Note: We avoid recursive listing on big repos; list top-level or date folder.
  # After 429: wait 360s before retry (handled by SDK with retries).

  local repo="${HF_DATASET_REPO:-"datasets/your-org/your-dataset"}"
  local folder="${HF_DATASET_FOLDER:-"${DATE_TAG}"}"
  local out_file="${MANIFEST_DIR}/${DATE_TAG}_filelist.json"

  ensure_dir "${MANIFEST_DIR}"

  log "Generating CDN manifest for repo=${repo} folder=${folder} #training #cdn #rate-limit-bypass"

  python3 - <<PY
import os, json, sys
from huggingface_hub import list_repo_tree

repo = os.getenv("HF_DATASET_REPO", "datasets/your-org/your-dataset")
folder = os.getenv("HF_DATASET_FOLDER", "${DATE_TAG}")
out_file = "${out_file}"

try:
    # Single API call: non-recursive to avoid pagination explosion.
    items = list_repo_tree(repo=repo, path=folder, recursive=False)
    files = []
    for item in items:
        if item.type == "file":
            cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{item.path}"
            files.append({
                "repo": repo,
                "path": item.path,
                "cdn_url": cdn_url,
                "size": getattr(item, "size", None)
            })
    manifest = {
        "generated_at": "${TIMESTAMP}",
        "repo": repo,
        "folder": folder,
        "file_count": len(files),
        "files": files
    }
    with open(out_file, "w") as f:
        json.dump(manifest, f, indent=2)
    print(f"Manifest written to {out_file} ({len(files)} files)")
except Exception as e:
    print(f"Error generating manifest: {e}", file=sys.stderr)
    sys.exit(1)
PY

  if [[ -f "${out_file}" ]]; then
    log "CDN manifest generated: ${out_file} #training #cdn #rate-limit-bypass"
  else
    log "Failed to generate CDN manifest. #training"
  fi
}

main() {
  log "Starting airship-discover #business-research #knowledge-rag #training"

  ensure_dir "${MANIFEST_DIR}"

  run_granite_research
  run_knowledge_rag
  generate_cdn_manifest

  log "Finished airship-discover"
}

main "$@"
```

### Make executable

```bash

```

## Notes & Best Practices Applied

- **Cron-safe**: shebang, `bash` invocation, `SHELL=/bin/bash` in crontab, non-interactive.
- **Rate-limit mitigation**: single `list_repo_tree` (non-recursive) → CDN-only manifest; Lightning training uses CDN URLs with zero API calls during data loading.
- **Idempotent**: re-runnable; overwrites same-date manifest safely.
- **Tagged outputs**: logs and operations include tags for downstream indexing and observability.
- **No Mac compute**: orchestrator only; training offloaded to Lightning Studio via CDN manifest.
