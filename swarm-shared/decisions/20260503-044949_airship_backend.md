# airship / backend

## Final Implementation Plan (≤2h)

**Goal**: Make Surrogate-1 training resilient to HF API 429s and Lightning idle-stop kills while cutting quota waste.

### Core Architecture
1. **Pre-generated file manifest** (run once per date folder) → embed in training script → Lightning training uses CDN-only fetches (zero API calls during training).
2. **Studio lifecycle guard**: check status before `.run()`; restart if stopped; reuse running studios.
3. **Deterministic write sharding** across sibling repos to bypass 128/hr commit cap.

---

### Steps (≤2h)

1. **Create file-list pre-generation script** (20–30m)  
   - Single `list_repo_tree` call per date folder → save `train_files.json` with CDN URLs.  
   - Run from Mac orchestration; commit `train_files.json` to repo or pass to training job.

2. **Update training data loader** (40–60m)  
   - Replace `load_dataset(streaming=True)` with direct CDN downloads using `train_files.json`.  
   - Use `pyarrow.parquet` for fast, memory-efficient reads.  
   - Project to `{prompt, response}` only at parse time; ignore heterogeneous schemas.

3. **Add Lightning studio lifecycle guard** (20–30m)  
   - Before each `.run()`, check studio status; if stopped, restart with `target.start(machine=Machine.L40S)` (or fallback to available cloud account).  
   - Reuse running studios to save quota.

4. **Add deterministic write sharding for HF commits** (15–25m)  
   - Hash slug → pick sibling repo (0..N) to spread writes across repos (640/hr aggregate).

5. **Smoke test** (15–20m)  
   - Run pre-generation locally, verify `train.py` loads 1 batch via CDN, verify studio restart logic.

```bash
#!/usr/bin/env bash
# scripts/generate_train_filelist.sh
set -euo pipefail

REPO="datasets/your-org/surrogate-mirror"
DATE_FOLDER="batches/mirror-merged/2026-04-29"
OUTFILE="train_files.json"

python - <<'PY'
import os, json, sys
from huggingface_hub import HfApi

REPO = os.getenv("REPO", "datasets/your-org/surrogate-mirror")
DATE_FOLDER = os.getenv("DATE_FOLDER", "batches/mirror-merged/2026-04-29")
OUTFILE = os.getenv("OUTFILE", "train_files.json")

api = HfApi()
tree = api.list_repo_tree(repo_id=REPO, path=DATE_FOLDER, recursive=False)
files = [
    f"{DATE_FOLDER}/{item.path.split('/')[-1]}"
    for item in tree
    if item.type == "file" and item.path.lower().endswith(".parquet")
]

# CDN URLs (no auth/API calls during training)
cdn_urls = [
    f"https://huggingface.co/datasets/{REPO}/resolve/main/{f}"
    for f in files
]

with open(OUTFILE, "w") as f:
    json.dump({"files": files, "cdn_urls": cdn_urls}, f, indent=2)

print(f"Wrote {len(files)} files to {OUTFILE}")
PY
```

### 2. CDN-only dataset loader in `train.py`
```python
# surrogate/train.py
import json
import pyarrow.parquet as pq
import requests
import io
from torch.utils.data import IterableDataset

class CDNParquetDataset(IterableDataset):
    def __init__(self, filelist_path="train_files.json", max_files=None):
        with open(filelist_path) as f:
            manifest = json.load(f)
        self.urls = manifest["cdn_urls"]
        if max_files:
            self.urls = self.urls[:max_files]

    def _stream_parquet(self, url):
        resp = requests.get(url, timeout=30)
        resp.raise_for_status()
        return pq.read_table(io.BytesIO(resp.content))

    def __iter__(self):
        for url in self.urls:
            table = self._stream_parquet(url)
            # Project to {prompt, response} only; ignore extra/mixed columns
            for batch in table.to_batches(max_chunksize=512):
                df = batch.to_pandas()
                for _, row in df.iterrows():
                    prompt = row.get("prompt") or row.get("input") or ""
                    response = row.get("response") or row.get("output") or ""
                    if prompt and response:
                        yield {"prompt": str(prompt), "response": str(response)}
```

### 3. Lightning studio lifecycle guard
```python
# surrogate/lightning_launcher.py
from lightning import Lightning, Teamspace, Machine
import time

LIGHTNING_ACCOUNT = "lightning-public-prod"  # or "lightning-lambda-prod" for H200
STUDIO_NAME = "surrogate-training-studio"

def get_or_start_studio():
    lightning = Lightning()
    teamspace = Teamspace()

    # Reuse running studio
    for s in teamspace.studios:
        if s.name == STUDIO_NAME and s.status == "running":
            print(f"Reusing running studio: {s.name}")
            return s

    # Start new if stopped or missing
    target = Lightning.Studio(
        name=STUDIO_NAME,
        cloud=LIGHTNING_ACCOUNT,
        machine=Machine.L40S,
        create_ok=True,
    )
    if target.status == "stopped":
        print("Studio stopped; restarting...")
        target.start(machine=Machine.L40S)
        # Wait for running
        while target.status != "running":
            time.sleep(10)
            target.refresh()
    return target

def run_training_job(script_path, args):
    studio = get_or_start_studio()
    run = studio.run(
        target=script_path,
        arguments=args,
        # Avoid idle-stop killing long jobs: request persistent compute
        # (Lightning-specific flags may vary; check SDK docs)
    )
    return run
```

### 4. Deterministic HF write sharding (for artifact uploads)
```python
# surrogate/hf_shard.py
import hashlib
from huggingface_hub import HfApi

SIBLING_REPOS = [
    "datasets/your-org/surrogate-shard-0",
    "datasets/your-org/surrogate-shard-1",
    "datasets/your-org/surrogate-shard-2",
    "datasets/your-org/surrogate-shard-3",
    "datasets/your-org/surrogate-shard-4",
]

def pick_repo(slug: str) -> str:
    h = hashlib.sha256(slug.encode()).hexdigest()
    idx = int(h, 16) % len(SIBLING_REPOS)
    return SIBLING_REPOS[idx]

def upload_sharded(artifact_path, slug, **kwargs):
    repo_id = pick_repo(slug)
    api = HfApi()
    # Example: upload file or commit dataset entry
    api.upload_file(
        path_or_fileobj=artifact_path,
        path_in_repo=f"batches/{slug}.parquet",
        repo_id=repo_id,
        **kwargs,
    )
    return repo_id
```
