## v0.1 scope
v0.1 in 2 weeks: single workflow template (research→signal→risk check→order stub) with deterministic replay, Postgres-backed audit trail, Redis Streams queue, simulated broker connector, and /run and /audit endpoints; deployable on HF Space with local docker-compose for dev; includes unit/integration tests and one replay script.

## Open questions
- Deterministic replay strategy: event sourcing vs snapshot+event log — which minimizes replay latency while keeping storage free-tier friendly?
- Idempotency key scheme and scope (per workflow_run vs per external_order_id) to prevent double execution across retries?
- Compliance guardrail evaluation timing (pre-check vs circuit-breaker interrupt) and how to model rule versioning for replay?
- Broker/OMS/EMS connector interface contract (sync vs async acknowledgement) to guarantee at-least-once vs exactly-once semantics?
- Latency SLA target (sub-100ms vs sub-500ms) and acceptable trade-offs between queue depth and immediate execution paths?
- Secrets/credential management for connectors on free-tier HF Spaces (env-vars vs external vault) and rotation cadence?
