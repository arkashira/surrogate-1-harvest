# vanguard / design

**Most Valuable Next Improvement: Persistent File-List Cache with Enhanced Features**

### Diagnosis

* No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` → HF API 429 (1000 req/5min).
* Lightning Studio lifecycle not reused → quota burn on recreation.
* Training data ingest relies on ephemeral file paths → no reproducibility.
* Lack of persisted file manifest → every run re-lists repos and re-checks schemas.

### Proposed Change

* Introduce a persistent file-list cache to store the list of files for each dataset.
* Enhance the cache to store schema information and file manifest.
* Implement a mechanism to reuse the Lightning Studio instance.
* Load data from CDNs only.

### Implementation

1. Create a new file `file_list_cache.py` in the project root:
```python
import json
import os

class FileListCache:
    def __init__(self, cache_dir):
        self.cache_dir = cache_dir
        self.cache_file = os.path.join(cache_dir, 'file_list_cache.json')

    def load_cache(self):
        if os.path.exists(self.cache_file):
            with open(self.cache_file, 'r') as f:
                return json.load(f)
        return {}

    def save_cache(self, cache):
        with open(self.cache_file, 'w') as f:
            json.dump(cache, f)

    def get_file_list(self, repo, path):
        cache = self.load_cache()
        if repo in cache and path in cache[repo]:
            return cache[repo][path]
        return None

    def get_schema(self, repo):
        cache = self.load_cache()
        if repo in cache:
            return cache[repo]['schema']
        return None

    def get_file_manifest(self, repo):
        cache = self.load_cache()
        if repo in cache:
            return cache[repo]['file_manifest']
        return None
```
