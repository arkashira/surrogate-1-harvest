## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk check → execution stub) with immutable audit trail, Postgres-backed queue on Supabase, Cloudflare Workers edge endpoint, Redis cache for hot state, compliance guardrails as versioned JSON rules, replay by run ID, and metrics for latency + NSM (successful workflow runs weighted by fee_generating).

## Open questions
- Deterministic replay strategy: event-sourcing vs snapshots + idempotency keys — which minimizes replay latency and storage cost?
- Broker/OMS/EMS connector authentication model: API-key vaulting vs signed requests — how to store secrets within free-tier constraints?
- Compliance rule engine expressiveness: DSL vs Python predicates — how to allow non-engineer updates while preserving determinism?
- Latency budget allocation across research, signal, risk, and execution stages — what is the sub-second target per stage?
- Audit retention and immutability guarantees: append-only Postgres + periodic hash anchoring vs external WORM store — what satisfies regulatory checks on free-tier?
- Failure modes and compensations: how to handle partial execution and rollback semantics without distributed transactions?
- Rate limits and backpressure from brokers/data vendors — how to implement deterministic throttling at the edge?
