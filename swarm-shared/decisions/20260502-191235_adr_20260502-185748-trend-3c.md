## Deployment
hybrid (local-first CLI + optional Supabase for team/org metadata and HF Space for optional hosted demo/UI)

## v0.1 scope
v0.1 ships in 2 weeks: CLI tool (Python) that indexes a local repo (Python/JS), parses files with tree-sitter, runs local rule-based checks (complexity, common anti-patterns), produces SARIF/JSON reports, stores results in local SQLite, and supports a --fix flag for auto-remediations where safe. No external servers required; optional Supabase flag for team metadata (opt-in).

## Open questions
- Will users accept periodic opt-in remote LLM calls for advanced insights, or must all analysis remain strictly local?
- Which languages must be supported at v0.1 (Python/JS/TS only vs broader set)?
- How will licensing/compliance handle models and rulesets (permissive vs copyleft)?
- What is the plan for incremental indexing performance on repos >100k files on commodity laptops?
- Will there be a paid tier, and what capabilities justify it while preserving local-first privacy?
