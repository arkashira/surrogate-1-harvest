## Deployment

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on Cloudflare Workers that lists Lambda and Cloud Functions, normalizes cron expressions, detects duplicates (by hash), drift (expression changed), and orphans (schedule exists but function deleted), persists results to Supabase Postgres, and exposes a GET /schedules endpoint with state filters; includes unit tests and one end-to-end script that audits a single AWS account and GCP project.

## Open questions
- Which IAM scopes are minimally required to list/describe schedules and last invocation times for Lambda and Cloud Functions?
- How will we authenticate multi-tenant customers (cloud tokens vs. service-account keys) while staying on free-tier infra?
- What retention policy and TTL do we want for schedule snapshots to balance audit history vs. storage?
- How often should audits run by default (hourly/daily) to balance freshness vs. rate limits/cost?
