## Highest-Value Incremental Improvement (<2h)

**Problem**: Surrogate-1 training blocked by HF API 429s during dataset loading; training stalls waiting on API pagination instead of GPU.

**Fix**: Implement CDN-only data loading with a pre-computed file manifest. One-time Mac-side manifest generation (after rate-limit window), then Lightning training uses only CDN URLs with zero HF API calls during data load.

**Why this ships in <2h**:
- Single manifest file (`file_manifest.json`) generated once per date folder
- Small change to `train.py` / dataset loader to read manifest and fetch via CDN
- No schema changes, no new infra, no retraining required
- Eliminates 429s and HF commit-cap pressure during training loops

## Implementation Plan

### 1. Generate file manifest (Mac, one-time or cron)

Script: `scripts/generate_manifest.py`

```python
#!/usr/bin/env python3
"""
Generate CDN-only manifest for a date folder in a HF dataset repo.
Run from Mac after rate-limit window clears.
"""
import json
import os
from datetime import datetime
from huggingface_hub import list_repo_tree

REPO = "datasets/your-org/surrogate-1"  # or your repo
DATE_FOLDER = "batches/mirror-merged/2026-05-02"  # param later
OUT_PATH = "file_manifest.json"

def main():
    # Single API call: non-recursive per folder to avoid pagination explosion
    items = list_repo_tree(
        repo_id=REPO,
        path=DATE_FOLDER,
        recursive=True,  # okay if folder is small; else do BFS by subfolder
        repo_type="dataset",
    )

    files = []
    for item in items:
        if item.rfilename.endswith(".parquet"):
            # CDN URL: no auth, no API rate limit
            cdn_url = (
                f"https://huggingface.co/datasets/{REPO}/resolve/main/"
                f"{item.rfilename}"
            )
            files.append({
                "filename": item.rfilename,
                "cdn_url": cdn_url,
                "size": item.size or 0,
            })

    manifest = {
        "repo": REPO,
        "date_folder": DATE_FOLDER,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "files": files,
    }

    os.makedirs(os.path.dirname(OUT_PATH) if os.path.dirname(OUT_PATH) else ".", exist_ok=True)
    with open(OUT_PATH, "w") as f:
        json.dump(manifest, f, indent=2)

    print(f"Wrote {len(files)} files to {OUT_PATH}")

if __name__ == "__main__":
    main()
```

Make executable and run once:

```bash
chmod +x scripts/generate_manifest.py
python scripts/generate_manifest.py
# -> file_manifest.json
```

Commit `file_manifest.json` (or keep local and copy into training context).

### 2. Update dataset loader to use CDN-only manifest

In `train.py` or dataset module (e.g., `surrogate/data.py`):

```python
import json
import pyarrow.parquet as pq
import requests
from io import BytesIO
from torch.utils.data import IterableDataset

class CDNParquetDataset(IterableDataset):
    def __init__(self, manifest_path, max_files=None):
        with open(manifest_path) as f:
            manifest = json.load(f)
        self.files = manifest["files"]
        if max_files:
            self.files = self.files[:max_files]

    def _stream_parquet(self, cdn_url):
        # CDN fetch: no Authorization header, bypasses HF API rate limits
        resp = requests.get(cdn_url, timeout=30)
        resp.raise_for_status()
        return pq.read_table(BytesIO(resp.content))

    def __iter__(self):
        for entry in self.files:
            table = self._stream_parquet(entry["cdn_url"])
            # Project to {prompt, response} only (schema-safe)
            df = table.select(["prompt", "response"]).to_pandas()
            for _, row in df.iterrows():
                yield {"prompt": row["prompt"], "response": row["response"]}
```

Use in training loop:

```python
from surrogate.data import CDNParquetDataset

train_dataset = CDNParquetDataset("file_manifest.json")
train_loader = torch.utils.data.DataLoader(
    train_dataset,
    batch_size=8,
    num_workers=0,  # avoid fork issues with requests
)
```

### 3. Lightning Studio integration (reuse + zero API during train)

In your launcher script (`scripts/launch_training.py`):

```python
from lightning import Teamspace, Studio, Machine
import time

# Reuse running studio to save quota
def get_or_create_studio(name="surrogate-train"):
    for s in Teamspace.studios:
        if s.name == name and s.status == "Running":
            print(f"Reusing running studio: {s.name}")
            return s
    print("Creating new studio...")
    return Studio(
        name=name,
        machine=Machine.L40S,  # or priority list: L40S -> A100 -> H200 if available
        create_ok=True,
    )

studio = get_or_create_studio()

# Ensure studio is running before .run()
if studio.status != "Running":
    print("Studio not running, starting...")
    studio.start(machine=Machine.L40S)
    # wait until running
    while studio.status != "Running":
        time.sleep(10)
        studio.refresh()

# Run training (train.py uses CDN-only loader)
job = studio.run(
    command="python train.py --manifest file_manifest.json",
    requirements=["pyarrow", "requests", "torch"],
)
print(f"Started job: {job}")
```
