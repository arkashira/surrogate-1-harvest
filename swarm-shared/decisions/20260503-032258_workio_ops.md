# workio / ops

## Final Synthesized Solution

### 1. Diagnosis (Consensus)
- **Missing `X-Line-Signature` verification** allows spoofed/replayed events to corrupt time-tracking (clock-in/out, leave, OT).
- **No idempotency/replay protection** causes duplicate `attendance_punch` and leave/OT rows when LINE retries on 5xx/timeouts.
- **No deduplication key** means the same `webhookEventId` can be processed multiple times under race/retry conditions.
- **No signature mismatch rejection** (HTTP 401) lets invalid payloads reach business logic.
- **Missing observability** (structured logs, metrics) prevents incident forensics and operational debugging.

### 2. Proposed Change (Concrete, Minimal Surface)
- **File scope**: `/opt/axentx/workio/server/src/routes/webhook/line.ts`
- **Add**:
  - HMAC-SHA256 signature verification using `LINE_CHANNEL_SECRET`.
  - Idempotency via `delivery_id` (from `X-Line-Delivery` or derived from event) stored in PostgreSQL with unique constraint and TTL.
  - Early rejection (401/400) on invalid signature; 200 OK only after idempotent processing.
  - Structured logging and metrics for verification/idempotency outcomes.
  - Atomic upsert for delivery tracking and mark-as-processed after business logic succeeds.

#### 3.4 Middleware: verify signature + idempotency
`server/src/middleware/lineWebhook.ts`
```ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';
import pino from 'pino';

const logger = pino({ name: 'line-webhook' });
const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET!;

export function verifyLineSignature(req: Request, res: Response, next: NextFunction) {
  const signature = req.headers['x-line-signature'] as string;
  if (!signature) {
    logger.warn({ ip: req.ip }, 'Missing X-Line-Signature');
    return res.status(401).json({ error: 'Missing X-Line-Signature' });
  }

  const body = JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');

  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected))) {
    logger.warn({ ip: req.ip }, 'Invalid signature');
    return res.status(401).json({ error: 'Invalid signature' });
  }
  next();
}

/**
 * Idempotency guard using delivery_id derived from LINE headers or event body.
 * Returns 200 if already processed; otherwise marks in-flight and calls next().
 */
export async function lineIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const events = req.body?.events;
  if (!Array.isArray(events) || events.length === 0) return next();

  const firstEvent = events[0];
  // Prefer delivery header; fallback to deterministic hash of event fields
  const deliveryId =
    req.headers['x-line-delivery'] ||
    crypto
      .createHash('sha256')
      .update(
        firstEvent.webhookEventId + firstEvent.timestamp + firstEvent.source.userId
      )
      .digest('hex')
      .slice(0, 32);

  try {
    const { rows } = await pool.query(
      `INSERT INTO line_webhook_deliveries (delivery_id, event_type, source_user_id)
       VALUES ($1, $2, $3)
       ON CONFLICT (delivery_id) DO UPDATE
       SET received_at = NOW()
       WHERE line_webhook_deliveries.processed_at IS NULL
       RETURNING (xmax = 0) AS inserted`,
      [deliveryId, firstEvent.type, firstEvent.source?.userId || null]
    );

    const inserted = rows[0]?.inserted === true;
    if (!inserted) {
      logger.info({ deliveryId }, 'Duplicate delivery - already processed');
      return res.status(200).json({ ok: true, duplicate: true });
    }

    // Attach deliveryId for handler to mark processed later
    (req as any).lineDeliveryId = deliveryId;
    next();
  } catch (err) {
    logger.error({ err, deliveryId }, 'Idempotency check failed');
    // Fail open to avoid breaking LINE retries during DB incidents, but log loudly
    next();
  }
}
```

#### 3.5 Route handler update
`server/src/routes/webhook/line.ts`
```ts
import express from 'express';
import { verifyLineSignature, lineIdempotency } from '../../middleware/lineWebhook';
import { pool } from '../../db';
import pino from 'pino';

const logger = pino({ name: 'line-webhook-handler' });
const router = express.Router();

router.post(
  '/webhook/line',
  express.json({ limit: '1mb' }),
  verifyLineSignature,
  lineIdempotency,
  async (req, res) => {
    const events = req.body.events;
    const deliveryId = (req as any).lineDeliveryId;

    try {
      for (const ev of events) {
        // Process clock-in/out, leave, OT etc.
        // Example: attendance punch
        if (ev.type === 'message' && ev.message.type === 'text') {
          const text = ev.message.text.trim().toLowerCase();
          const userId = ev.source.userId;

          if (text === 'clock in' || text === 'clock out') {
            await pool.query(
              `INSERT INTO attendance_punch (user_id, punch_type, punched_at, line_delivery_id)
               VALUES ($1, $2, NOW(), $3)
               ON CONFLICT DO NOTHING`,
              [userId, text === 'clock in' ? 'in' : 'out', deliveryId]
            );
          }
        }
      }

      // Mark delivery processed
      if (deliveryId) {
        await pool.query(
          `UPDATE line_webhook_deliveries SET processed_at = NOW() WHERE delivery_id = $1`,
          [deliveryId]
        );
      }

      logger.info({ deliveryId, eventCount: events.length }, 'Webhook processed');
      res.status(200).json({ ok: true });
    } catch (err) {
      logger.error({ err, deliveryId }, 'Webhook processing failed');
      // LINE will retry on 5xx — keep idempotency safe
      res.status(500).json({ error: 'Processing failed' });
    }
  }
);

export default router;
```
