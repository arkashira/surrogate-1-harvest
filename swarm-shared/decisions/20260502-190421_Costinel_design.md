# Costinel / design

## Final Design Increment (≤2h)

**Goal:**  
Ship a lightweight, deterministic discovery module that produces machine-readable manifests (`manifests/sources.json`, `manifests/discovery.json`) and a top-hub insight snapshot so onboarding, training, and governance scripts can run without repeated API calls or human lookup.

**Why this is highest-value:**  
- Eliminates repeated HF API calls and cloud enumeration during training/onboarding (respects HF rate limits and uses CDN bypass).  
- Embeds top-hub context (per knowledge-rag patterns) so governance reviewers immediately see the most-connected doc (e.g., "MOC") when triaging signals.  
- Provides deterministic inputs for surrogate-1 training (file list + account map) and satisfies the “Sense + Signal” philosophy (no execution).

---

## Implementation Plan (≤2h)

1. **Create directory structure**  
   - `manifests/` for machine-readable outputs  
   - `src/discovery/` for the module  
   - `scripts/discovery-runner.sh` (bash wrapper, executable, proper shebang)

2. **Implement `src/discovery/cloud_discoverer.py`**  
   - Enumerate configured cloud accounts (AWS/GCP/Azure) via boto3/google-cloud/azure SDKs (read-only calls).  
   - Output normalized entries: `{provider, account_id, alias, regions, last_sensed_ts}`.  
   - If credentials missing, record as `unavailable` with error hint (non-blocking).

3. **Implement `src/discovery/hf_dataset_indexer.py`**  
   - Accept config: `hf_repo`, `date_folder` (e.g., `batches/mirror-merged/2026-05-02`).  
   - Single API call: `list_repo_tree(path=date_folder, recursive=False)` → save to `file_list.json`.  
   - Generate CDN URLs: `https://huggingface.co/datasets/{repo}/resolve/main/{path}`.  
   - Respect HF rate limits: if 429, wait 360s and retry once; log warning.  
   - Output: `{repo, folder, files: [{path, cdn_url, size_hint}], indexed_at}`.

4. **Implement `src/discovery/top_hub_insight.py`**  
   - Read knowledge-rag graph export or local cache (e.g., `knowledge-rag/top_hub.json`).  
   - If unavailable, produce minimal placeholder: `{top_hub: "MOC", connected_count: 0, note: "Run knowledge-rag to refresh"}`.  
   - Tags included: `#knowledge-rag #graph #hub`.

5. **Implement `src/discovery/assemble_manifests.py`**  
   - Orchestrate the three discoverers and write:  
     - `manifests/sources.json` → `{cloud_accounts, hf_dataset_index, generated_at}`  
     - `manifests/discovery.json` → `{top_hub, file_list_summary, cloud_summary, tags}`  
   - Ensure outputs are deterministic (sorted keys, stable ordering).

6. **Create `scripts/discovery-runner.sh`**  
   - Shebang: `#!/usr/bin/env bash`  
   - Set `SHELL=/bin/bash` in header comment (for crontab parity).  
   - Invoke: `python -m src.discovery.assemble_manifests --out-dir manifests`.  
   - Exit non-zero on critical failure; log to `logs/discovery-runner.log`.

7. **Add lightweight usage docs**  
   - `README_DISCOVERY.md` with one-liner:  
     ```bash
     bash scripts/discovery-runner.sh
     ```
   - Note: HF training should embed `manifests/sources.json` file list and use CDN-only fetches during data loading (zero API calls in Lightning).

8. **Test locally (smoke)**  
   - Run runner; verify JSON files are valid and contain expected keys.  
   - Confirm no HF API calls occur during CDN fetch simulation.

## Code Snippets

### `scripts/discovery-runner.sh`
```bash
#!/usr/bin/env bash
# SHELL=/bin/bash recommended for crontab
# Usage: bash scripts/discovery-runner.sh

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
OUT_DIR="${REPO_ROOT}/manifests"
LOG_FILE="${REPO_ROOT}/logs/discovery-runner.log"

mkdir -p "$OUT_DIR" "$(dirname "$LOG_FILE")"

exec > >(tee -a "$LOG_FILE") 2>&1

echo "[$(date -u +"%Y-%m-%dT%H:%M:%SZ")] Starting Costinel discovery run"

cd "$REPO_ROOT"
python -m src.discovery.assemble_manifests --out-dir "$OUT_DIR"

echo "[$(date -u +"%Y-%m-%dT%H:%M:%SZ")] Discovery completed. Outputs in $OUT_DIR"
```

def discover_aws_accounts() -> List[Dict[str, Any]]:
    accounts = []
    if boto3 is None:
        return accounts

    try:
        sts = boto3.client("sts")
        caller = sts.get_caller_identity()
        accounts.append(
            {
                "provider": "aws",
                "account_id": caller["Account"],
                "alias": os.getenv("AWS_ACCOUNT_ALIAS", "aws-primary"),
                "regions": os.getenv("AWS_TARGET_REGIONS", "us-east-1,eu-west-1").split(","),
                "last_sensed_ts": datetime.now(timezone.utc).isoformat(),
            }
        )
    except (NoCredentialsError, ClientError) as exc:
        accounts.append(
            {
                "provider": "aws",
                "account_id": "unavailable",
                "alias": "aws-primary",
                "error": str(exc),
                "last_sensed_ts": datetime.now(timezone.utc).isoformat(),
            }
        )
    return accounts
