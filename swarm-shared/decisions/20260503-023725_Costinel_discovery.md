# Costinel / discovery

## Highest-Value Incremental Improvement: Top-Hub Signal Panel

**Goal**: Ship a lightweight, CDN-first "Top-Hub Signal Panel" on the Costinel dashboard that surfaces the most-connected hub (default **MOC**) and its top 3 actionable, cost-impact proposals using a **CDN-first, rate-limit-safe, zero-API-during-runtime** pattern. Ships in <2h.

### Why this is highest value
- Directly applies the **top-hub doc insight** pattern (MOC hub) and **CDN bypass** pattern.
- Adds immediate "Sense + Signal" UX without touching execution paths (aligns with Costinel philosophy).
- Reuses existing knowledge-rag/graph context; no new infra or permissions needed.
- Zero runtime HF API calls → no 429 risk during dashboard use.

---

### Implementation Plan (≤2h)

1. **Pre-list hub files once** (Mac orchestration, ~5min)
   - Use `list_repo_tree` for `knowledge-rag/hubs/` (non-recursive) once per day or on-demand.
   - Save `hubs/file-list.json` to repo (committed) or to `public/data/hubs.json` for CDN.

2. **Create CDN-safe fetcher** (~20min)
   - Build `src/lib/hubs/cdnHubLoader.ts` that loads hub metadata and top signals via `https://huggingface.co/datasets/AXENTX/Costinel/resolve/main/...` (no auth, CDN tier).
   - Parse `{hub}/signals.json` or `{hub}/README.md` frontmatter for top 3 proposals.

3. **Add Top-Hub Signal Panel component** (~45min)
   - `src/components/TopHubSignalPanel.tsx`
   - Shows: hub name, short description, top 3 signals (title, impact, action).
   - Fallback to MOC if hub list unavailable.

4. **Wire into dashboard** (~20min)
   - Place panel near top of `/dashboard` or in sidebar.
   - Server-side load hub list at build time (ISR) or client-side with SWR for freshness.

5. **Update build/deploy** (~10min)
   - Ensure `public/data/hubs.json` is generated by a small script (`scripts/gen-hubs-list.js`) and committed or included in Docker build.

6. **Test & ship** (~20min)
   - Verify CDN loads in dev, no 401/429, graceful fallback.
   - Check mobile responsiveness.

#### 2. CDN-first hub loader (client-side)
```typescript
// src/lib/hubs/cdnHubLoader.ts
export interface HubSignal {
  title: string;
  impact: 'high' | 'medium' | 'low';
  description: string;
  action?: string;
}

export interface HubInfo {
  name: string;
  description: string;
  signals: HubSignal[];
}

const CDN_BASE = 'https://huggingface.co/datasets/AXENTX/Costinel/resolve/main';

export async function loadTopHub(): Promise<HubInfo | null> {
  try {
    // Load hub list (committed to public/data)
    const listRes = await fetch('/data/hubs.json', { cache: 'no-store' });
    if (!listRes.ok) return loadFallback();
    const list = await listRes.json();
    const topHub = list.hubs.find((h: any) => h.name === 'MOC') || list.hubs[0];
    if (!topHub) return loadFallback();

    // Load signals via CDN (no auth)
    const signalsRes = await fetch(topHub.signalsUrl, { cache: 'no-store' });
    let signals: HubSignal[] = [];
    if (signalsRes.ok) {
      signals = await signalsRes.json();
    } else {
      // Fallback: parse README.md frontmatter or first lines
      signals = await parseReadmeForSignals(topHub.readmeUrl);
    }

    return {
      name: topHub.name,
      description: `Top-connected hub — ${topHub.name}`,
      signals: signals.slice(0, 3)
    };
  } catch (e) {
    console.warn('Failed to load top hub:', e);
    return loadFallback();
  }
}

async function loadFallback(): Promise<HubInfo> {
  return {
    name: 'MOC',
    description: 'Most-connected hub — MOC (default)',
    signals: [
      {
        title: 'Right-size underutilized instances',
        impact: 'high',
        description: 'Identify and recommend downsizing for instances with <30% avg CPU over 14 days.'
      },
      {
        title: 'Increase RI coverage for steady workloads',
        impact: 'high',
        description: 'Purchase 12-month RIs for predictable services to reduce cost by ~35%.'
      },
      {
        title: 'Clean orphaned storage volumes',
        impact: 'medium',
        description: 'Detach and snapshot then delete unattached volumes older than 7 days.'
      }
    ]
  };
}

async function parseReadmeForSignals(readmeUrl: string): Promise<HubSignal[]> {
  try {
    const res = await fetch(readmeUrl);
    if (!res.ok) return [];
    const text = await res.text();
    // Very simple extraction: look for bullet lines containing "impact" or "action"
    const lines = text.split('\n').filter(l => l.trim().startsWith('-') || l.trim().startsWith('*'));
    return lines.slice(0, 3).map((line, i) => ({
      title: line.replace(/^[-*]\s*/, '').split('.')[0] || `Signal ${i + 1}`,
      impact: i === 0 ? 'high' : i === 1 ? 'high' : 'medium',
      description: line.replace(/^[-*]\s*/, '')
    }));
  } catch {
    return [];
  }
}
```
