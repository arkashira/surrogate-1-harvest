# Costinel / ops

## Final Implementation Plan — Top-Hub Signal Card (Costinel / ops)

**Scope**: Production-ready, read-only Vue 3 card that surfaces the most-connected knowledge hub (e.g., “MOC”) with contextual cost-governance insights. Uses CDN-bypass for live cost data and embedded mock fallback for offline/dev. Ships in <2h.

---

### 1. Architecture & Placement (resolved)
- **File**: `src/components/cards/TopHubSignalCard.vue`
- **Type**: Read-only ops signal card (no mutations)
- **Stack**: Vue 3 + TypeScript + Pinia (leverage existing stack)
- **Data strategy** (merged, no contradictions):
  - **Primary**: CDN-bypass fetch from public dataset folder (`datasets/costinel-ops/{date}/top-hub.json`) — zero API calls during runtime.
  - **Build-time optimization**: Single orchestration call produces `file-list.json` embedded at build time (optional; not required for runtime).
  - **Fallback**: Embedded `mockTopHub.json` for dev/offline and graceful degradation on CDN failure.
- **Refresh**: Poll every 300s while mounted (ops cadence).

```ts
// Lightweight CDN bypass: no auth, no API rate limits.
export async function fetchTopHub(dateFolder: string): Promise<any> {
  const url = `https://huggingface.co/datasets/axentx/costinel-ops/resolve/main/${dateFolder}/top-hub.json`;
  const res = await fetch(url, { cache: 'no-store' });
  if (!res.ok) {
    throw new Error(`CDN fetch failed: ${res.status}`);
  }
  return res.json();
}
```
