# workio / ops

## Final Synthesis (adopt this as the single canonical plan)

**Core problem**: LINE webhooks deliver at-least-once with no idempotency; the clock handler uses non-atomic find-then-upsert and has no transactional boundary, so retries/races create duplicate punches and stale daily summaries. Outbound LINE notifications have no retry/back-off, and the critical path lacks observability.

**Chosen approach**: unify the strongest, compatible parts of both proposals into one minimal, production-ready change that is correct and immediately actionable.

---

## 1. Canonical diagnosis (merged + resolved)

- **LINE at-least-once, no idempotency** — confirmed; requires explicit idempotency key and storage.
- **Non-atomic find-then-upsert** — confirmed; must be replaced by atomic upsert inside a transaction.
- **No transactional boundary across punch + daily-summary** — confirmed; must be fixed to keep aggregates consistent.
- **No retry/back-off for outbound LINE API** — confirmed; must add exponential backoff with jitter and bounded retries.
- **No operational observability** — confirmed; must add request-scoped structured logs, timings, and error context.

*Contradictions resolved*: both diagnoses agree; Candidate 1’s details are more complete, so we adopt them.

---

## 2. Canonical solution (merged + resolved)

**Scope**: replace `clockInOut` handler with an idempotent, atomic upsert using a stable idempotency key inside a DB transaction; add structured logging and a retry-aware LINE notifier.

**Key choices**:
- Use **deterministic idempotency key** = `SHA256(userId:eventTimestamp:eventIdOrSignatureNonce)` derived from LINE payload (not signature) so retries with identical payloads map to the same key.
- **Persist idempotency key in the same transaction** as punch + daily-summary to guarantee exactly-once semantics.
- **Transactional boundary** covers: idempotency check, punch insert, daily-summary upsert, idempotency insert.
- **Outbound LINE notify** uses exponential backoff with jitter, max ~3 retries, non-blocking after 200 OK to webhook (fire-and-forget with error logging).
- **Observability**: structured logs with `requestId`, `userId`, `idempotencyKey`, timings; export metrics (webhook received, processed, duplicates, failures, notify retries).

---

## 3. Canonical implementation (single source of truth)

File: `workio/server/src/controllers/line/webhook.ts`

```ts
import { Request, Response } from 'express';
import { pool } from '../../db';
import { lineClient } from '../../services/line';
import crypto from 'crypto';
import { Logger } from '../../utils/logger';

const log = new Logger('line-webhook');

function verifySignature(rawBody: string, signature: string): boolean {
  const expected = crypto
    .createHmac('sha256', process.env.LINE_CHANNEL_SECRET!)
    .update(rawBody, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

function jitter(ms: number) {
  return ms * (0.5 + Math.random()); // +/-50%
}

async function notifyWithRetry(userId: string, text: string, maxRetries = 3) {
  let lastErr: any;
  for (let i = 0; i < maxRetries; i++) {
    try {
      await lineClient.pushMessage(userId, { type: 'text', text });
      return true;
    } catch (err: any) {
      lastErr = err;
      const wait = jitter(Math.min(1000 * 2 ** i, 10000));
      await new Promise((r) => setTimeout(r, wait));
    }
  }
  log.warn({ userId, text }, 'LINE notify failed after retries', lastErr);
  return false;
}

export const handleWebhook = async (req: Request, res: Response) => {
  const requestId = crypto.randomUUID();
  const signature = req.headers['x-line-signature'] as string;
  const rawBody = req.body as string;
  const timer = { start: Date.now() };

  if (!signature || !verifySignature(rawBody, signature)) {
    log.warn({ requestId, hasSignature: !!signature }, 'Invalid LINE signature');
    return res.status(401).json({ error: 'Invalid signature' });
  }

  const payload = JSON.parse(rawBody);
  const events = payload.events;
  if (!events || !Array.isArray(events)) {
    return res.status(400).json({ error: 'No events' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      if (ev.type !== 'message' || ev.message?.type !== 'text') continue;
      const userId = ev.source.userId;
      const text = ev.message.text.trim().toLowerCase();
      const isClockIn = text === 'in' || text === 'clock in';
      const isClockOut = text === 'out' || text === 'clock out';
      if (!isClockIn && !isClockOut) continue;

      // Deterministic idempotency key from payload (stable across retries)
      const idempotencyKey = crypto
        .createHash('sha256')
        .update(`${userId}:${ev.timestamp}:${ev.source.userId}:${ev.message.id}`)
        .digest('hex');

      const exists = await client.query(
        `SELECT 1 FROM line_webhook_idempotency WHERE idempotency_key = $1`,
        [idempotencyKey]
      );
      if (exists.rows.length > 0) {
        log.info({ requestId, idempotencyKey, userId }, 'Duplicate webhook skipped');
        continue;
      }

      const now = new Date();
      const dateStr = now.toISOString().split('T')[0];

      // Atomic punch + daily_summary update
      await client.query(
        `INSERT INTO punches (user_id, tenant_id, type, clocked_at, created_at)
         VALUES ($1, $2, $3, $4, $5)`,
        [userId, 'default-tenant', isClockIn ? 'in' : 'out', now, now]
      );

      await client.query(
        `INSERT INTO daily_summary (user_id, tenant_id, summary_date, punches_count, updated_at)
         VALUES ($1, $2, $3, 1, $4)
         ON CONFLICT (user_id, tenant_id, summary_date)
         DO UPDATE SET punches_count = daily_summary.punches_count + 1, updated_at = $4`,
        [userId, 'default-tenant', dateStr, now]
      );

      await client.query(
        `INSERT INTO line_webhook_idempotency (idempotency_key, created_at)
         VALUES ($1, $2)`,
        [idempotencyKey, now]
      );

      // Best-effort notify; non-blocking to webhook response
      notifyWithRetry(userId, isClockIn ? '✅ Clock in recorded' : '✅ Clock out recorded')
        .catch((err) => log.error({ requestId, userId }, 'notifyWithRetry unhandled', err));
    }

    await client.query('COMMIT');
    timer.end = Date.now();
    log.info(
      { requestId, eventCount: events.length, durationMs: timer.end - timer.start },
      'Webhook processed'
    );
    res.status(200).json({ ok: true });
  } catch (err: any) {
    await client.query('ROLLBACK');
    timer.end = Date.now();
    log.error({ requestId, err, durationMs: timer.end - timer.start }, 'Webhook processing failed');
    res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
};
```
