## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research -> signal -> risk check -> simulated execution) with immutable audit trail, Postgres-backed replay, CF Workers orchestration, Redis risk cache, and one mock broker connector; includes compliance guardrail API and basic dashboard-less API for triggering runs and fetching audit logs.

## Open questions
- Cold-start and tail-latency targets on CF Workers vs HF Space for the orchestrator; which path meets sub-second P99 under load?
- Deterministic replay storage strategy: append-only Postgres vs event-sourced CF Durable Objects vs Kafka-compatible queue; cost and latency trade-offs on free tiers?
- Compliance signature scheme: HMAC with rotating keys vs append-only hash chain; key management and rotation cadence?
- Connector idempotency and exactly-once semantics across brokers/OMS; dedupe key design and retry/backoff policy?
- Secrets storage for connector credentials: CF Workers KV vs Supabase encrypted columns vs HF Space secrets; access control and rotation workflow?
