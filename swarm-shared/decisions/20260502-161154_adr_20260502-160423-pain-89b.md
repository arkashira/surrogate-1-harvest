## Deployment
cf workers (edge functions for broker/obfuscation endpoints and blocklist KV) + client-side Rust binary (static builds); optional HF space for demo UI

## v0.1 scope
v0.1 ships in 2 weeks: a single Rust CLI binary (Linux x86_64) that creates a local SOCKS5 listener on a randomized port, wraps traffic in a meek-like TLS-wrapped HTTP CONNECT tunnel to a free-tier Cloudflare Worker broker, and plumbs a local TUN device with a kill-switch that blocks non-stealth traffic if the tunnel fails; includes encrypted local Sqlite config, basic DPI-obfuscation (fragmented TLS records), and minimal client-side WAU-ZB telemetry hashed and batched to Supabase via CF Worker; no server infrastructure required beyond CF Workers and optional Supabase free tier.

## Open questions
- Which DPI signatures are highest risk in target regions (SNI, MTU, TTL, packet length distributions)?
- Can we implement a pure userspace TUN fallback that avoids /dev/net/tun where unavailable or monitored?
- What broker/relay model balances reachability and cost on free tiers (CF Workers limits vs community-run bridges)?
- How will we measure WAU-ZB reliably without deanonymizing users (privacy-preserving telemetry design)?
- Fail-closed kill-switch granularity: per-app filter vs system route hijack — what is feasible cross-platform without admin/root?
- Are ephemeral port randomization and local SOCKS5 port shadowing sufficient to hide from localhost port scans?
