# workio / quality

## 1. Diagnosis
- Punch API retries can create duplicate punches (no idempotency key) → corrupt time records.
- UI has no optimistic state → perceived latency and accidental double-taps register multiple punches.
- Pending punches are not persisted locally → loss on navigation/refresh before server ack.
- No client-side in-flight lock → race between rapid clicks and retries.
- Error handling is coarse (toast + generic retry) without safe re-queue or dedupe.

## 2. Proposed change
File: `workio/src/features/punch/PunchButton.tsx`  
Scope: add React Query mutation with optimistic update, client-side idempotency key, pending-state, and safe retry. Keep API surface unchanged (same endpoint/payload) and add optional `clientId` for backend de-duping.

## 3. Implementation
```tsx
// workio/src/features/punch/PunchButton.tsx
import React, { useCallback, useMemo } from 'react';
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { Button } from '@/components/ui/button';
import { useLocation } from '@/hooks/useLocation';
import { punch as punchAPI } from '@/services/punch';
import { toast } from '@/components/ui/use-toast';
import type { PunchType } from '@/types/punch';

// Simple in-memory flight map to prevent double-mutate for same logical punch
const inflight = new Set<string>();

function createIdempotencyKey(type: PunchType, tenantId: string, userId: string): string {
  // Same logical punch within a short window should be deduped.
  // Use date-hour bucket so retries within same hour map to same key.
  const now = new Date();
  const bucket = `${now.getFullYear()}-${now.getMonth() + 1}-${now.getDate()}T${now.getHours()}`;
  return `${tenantId}:${userId}:${type}:${bucket}`;
}

export function PunchButton({ type }: { type: PunchType }) {
  const queryClient = useQueryClient();
  const { latitude, longitude, accuracy } = useLocation();

  const mutation = useMutation({
    mutationFn: punchAPI,
    // optimistic update
    onMutate: async (variables) => {
      const key = createIdempotencyKey(type, variables.tenantId, variables.userId);
      if (inflight.has(key)) {
        // Prevent duplicate flight
        throw new Error('PUNCH_IN_FLIGHT');
      }
      inflight.add(key);

      await queryClient.cancelQueries({ queryKey: ['punches', variables.tenantId, variables.userId] });

      const previous = queryClient.getQueryData(['punches', variables.tenantId, variables.userId]);

      // optimistic punch entry (pending)
      const optimisticPunch = {
        id: `pending-${Date.now()}`,
        type,
        timestamp: new Date().toISOString(),
        latitude: variables.latitude ?? latitude ?? null,
        longitude: variables.longitude ?? longitude ?? null,
        accuracy: variables.accuracy ?? accuracy ?? null,
        status: 'pending' as const,
        clientId: key,
      };

      queryClient.setQueryData(['punches', variables.tenantId, variables.userId], (old: any[] = []) => [
        optimisticPunch,
        ...old,
      ]);

      return { previous, key };
    },
    onError: (err, variables, context) => {
      // rollback
      if (context?.previous) {
        queryClient.setQueryData(['punches', variables.tenantId, variables.userId], context.previous);
      }
      const key = context?.key;
      if (key) inflight.delete(key);

      // user-facing message
      if (err && (err as any).message === 'PUNCH_IN_FLIGHT') {
        toast({ title: 'กำลังบันทึกอยู่', description: 'กรุณารอสักครู่', variant: 'default' });
      } else {
        toast({ title: 'บันทึกไม่สำเร็จ', description: 'ลองใหม่อีกครั้ง', variant: 'destructive' });
      }
    },
    onSettled: (data, err, variables, context) => {
      // always invalidate and cleanup
      const key = context?.key;
      if (key) inflight.delete(key);
      queryClient.invalidateQueries({ queryKey: ['punches', variables.tenantId, variables.userId] });
    },
  });

  const handlePunch = useCallback(() => {
    if (!latitude && !longitude) {
      toast({ title: 'ไม่พบตำแหน่ง', description: 'โปรดเปิด GPS และอนุญาตตำแหน่ง', variant: 'destructive' });
      return;
    }

    const tenantId = /* get from auth/context */ 'default-tenant';
    const userId = /* get from auth/context */ 'current-user';

    mutation.mutate({
      tenantId,
      userId,
      type,
      latitude,
      longitude,
      accuracy,
      // optional clientId for backend dedupe
      clientId: createIdempotencyKey(type, tenantId, userId),
    });
  }, [type, latitude, longitude, accuracy, mutation, toast]);

  const isPending = mutation.isLoading;
  const isInFlight = useMemo(() => {
    const key = createIdempotencyKey(type, 'default-tenant', 'current-user');
    return inflight.has(key);
  }, [type]);

  return (
    <Button
      size="lg"
      className="w-full"
      disabled={isPending || isInFlight}
      onClick={handlePunch}
    >
      {isPending || isInFlight ? 'กำลังบันทึก...' : `ลงเวลา ${type === 'IN' ? 'เข้า' : 'ออก'}`}
    </Button>
  );
}
```

## 4. Verification
- Click button rapidly (5–10x) → only one pending punch appears immediately; toast “กำลังบันทึกอยู่” appears on duplicates.
- Network throttle to Slow 3G → optimistic row shows “pending” instantly; when server responds, list updates to confirmed punch.
- Force server 500 error → optimistic row rolls back and toast shows “บันทึกไม่สำเร็จ”.
- After error, retry succeeds → new punch recorded and invalidation refreshes list.
- Check browser network tab: only one POST per logical punch (others blocked by in-flight set).
