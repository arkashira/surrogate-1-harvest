## v0.1 scope
v0.1 ships in 2 weeks: a CF Workers gateway exposing OpenAI-compatible /v1/chat/completions and /v1/completions endpoints that normalize requests/responses for OpenAI and Anthropic (text-only), with per-provider auth, Redis token counters, Postgres-backed route config, basic circuit breaker, retry policy, request audit trail, and p95 latency metrics export — no frontend, no streaming, no fine-tuned model support.

## Open questions
- Which provider auth patterns must be supported at v0.1 (bearer vs api_key vs OAuth) and how will secrets be injected in CF Workers?
- What is the authoritative source-of-truth for per-provider quotas: ledger in Postgres, Redis counters, or provider-side headers?
- How will we compute and enforce p95 latency SLA in a serverless environment (sampling window, backfills, alerting)?
- What fallback selection policy minimizes token waste and latency spikes (priority vs weighted vs cheapest)?
- How do we normalize disparate provider response schemas into a stable contract without losing provider-specific features?
