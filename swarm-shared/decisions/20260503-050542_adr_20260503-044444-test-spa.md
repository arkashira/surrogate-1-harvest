## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable to Cloudflare Workers that connects to a customer-provided Supabase Postgres; scans one AWS region and one GCP region for Lambda and Cloud Functions schedules, records them in Postgres, and produces duplicate/drifted/orphaned findings via python-croniter normalization; outputs JSON report and basic summary counts (no UI, no auth, no scheduler).

## Open questions
- Which identity/assumed-role will the scanner use to list Lambda schedules and Cloud Functions across customer accounts (cross-account strategy vs. customer-supplied tokens)?
- How will we detect orphaned schedules (e.g., deleted functions) without delete-history logs — rely on platform APIs or periodic reconciliation?
- What is the acceptable scan cadence and rate-limit budget per platform to avoid throttling or excess cost?
- Will customers require read-only IAM roles/policies, and what minimal permissions are needed for each platform?
- How do we normalize cron vs. rate expressions across Lambda and Cloud Functions for reliable duplicate/drift detection?
