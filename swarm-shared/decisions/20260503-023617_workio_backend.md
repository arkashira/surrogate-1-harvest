# workio / backend

## Final consolidated solution (correct + actionable)

**Core problem**: LINE webhook retries and client double-taps can create duplicate `(employee_id, punch_date, punch_type)` rows because the app relies on non-atomic check-then-insert logic and has no DB-level guard or idempotency.

**Guiding principles**:
- Correctness first: enforce the business rule at the DB level (one active/latest punch per employee per day per type) so app bugs or races cannot violate it.
- Actionability: use an atomic upsert + idempotency key so retries are safe and fast.
- Minimal surface: avoid schema changes that break existing code; prefer adding columns/indexes and using transactions.

### 1. Schema changes (run once)

File: `/opt/axentx/workio/server/src/db/schema.sql`

```sql
-- Idempotency table (lightweight, explicit replays)
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  punch_id        INTEGER NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- Ensure at most one active punch per employee per day per punch_type.
-- This is the authoritative guard against duplicates.
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_active_punch_per_day_type
  ON punches (employee_id, punch_date, punch_type)
  WHERE active = TRUE;

-- Optional: keep idempotency key on punches for traceability and fast lookups.
-- If you prefer not to modify punches, omit this line.
ALTER TABLE punches ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255);
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key
  ON punches (idempotency_key)
  WHERE idempotency_key IS NOT NULL;
```

Why:
- The partial unique index guarantees the business rule at the DB level.
- The idempotency table gives replay safety without coupling to punch state.
- Adding `idempotency_key` to punches is optional but useful for audits; the index is partial so it only applies when the column is used.

### 2. Atomic upsert with idempotency

File: `/opt/axentx/workio/server/src/services/punchService.ts`

```ts
import { pool } from '../db';

export async function upsertPunch({
  employeeId,
  punchDate,
  punchType,
  latitude,
  longitude,
  idempotencyKey,
}: {
  employeeId: number;
  punchDate: string; // YYYY-MM-DD
  punchType: 'in' | 'out';
  latitude?: number;
  longitude?: number;
  idempotencyKey?: string;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency fast-path
    if (idempotencyKey) {
      const idem = await client.query(
        'SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1',
        [idempotencyKey]
      );
      if (idem.rows.length > 0) {
        const existing = await client.query(
          'SELECT * FROM punches WHERE id = $1',
          [idem.rows[0].punch_id]
        );
        await client.query('ROLLBACK');
        return existing.rows[0];
      }
    }

    // 2) Deactivate any previous active punch for same employee/date/type
    await client.query(
      `UPDATE punches
       SET active = FALSE, updated_at = NOW()
       WHERE employee_id = $1 AND punch_date = $2 AND punch_type = $3 AND active = TRUE`,
      [employeeId, punchDate, punchType]
    );

    // 3) Insert new active punch
    const insertRes = await client.query(
      `INSERT INTO punches (employee_id, punch_date, punch_type, latitude, longitude, active, created_at, updated_at)
       VALUES ($1, $2, $3, $4, $5, TRUE, NOW(), NOW())
       RETURNING *`,
      [employeeId, punchDate, punchType, latitude, longitude]
    );

    const punch = insertRes.rows[0];

    // 4) Record idempotency
    if (idempotencyKey) {
      await client.query(
        'INSERT INTO punch_idempotency (idempotency_key, punch_id) VALUES ($1, $2)',
        [idempotencyKey, punch.id]
      );
      // Optional: also store on punch row for traceability
      await client.query(
        'UPDATE punches SET idempotency_key = $1 WHERE id = $2',
        [idempotencyKey, punch.id]
      );
    }

    await client.query('COMMIT');
    return punch;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

Why this works:
- All steps run in one transaction, so races cannot create two active punches (the partial unique index will raise if they do).
- Idempotency check is inside the transaction so replays are safe.
- We explicitly deactivate previous active rows so state transitions are deterministic.

### 3. LINE webhook handler (idempotent)

File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts`

```ts
import express from 'express';
import { upsertPunch } from '../services/punchService';
import { parseLineMessage } from '../utils/lineParser';

const router = express.Router();

function makeIdempotencyKey(body: any, ev: any, employeeId: number, action: string) {
  // Prefer a stable LINE delivery id if available; otherwise use signature+timestamp.
  const deliveryId = ev?.deliveryId || ev?.message?.id || '';
  const destination = body.destination || '';
  const timestamp = ev?.timestamp || Date.now();
  const signature = (req as any).get('X-Line-Signature') || '';
  // Deterministic, per-employee-action key
  return `line:${destination}:${deliveryId}:${timestamp}:${signature}:${employeeId}:${action}`;
}

router.post('/webhook/line', async (req, res) => {
  try {
    const body = req.body;
    const events = body.events || [];

    for (const ev of events) {
      if (ev.type === 'message' && ev.message?.type === 'text') {
        const { employeeId, action } = await parseLineMessage(ev);
        if (!employeeId || !['in', 'out'].includes(action)) continue;

        const today = new Date().toISOString().split('T')[0];
        const idempotencyKey = makeIdempotencyKey(body, ev, employeeId, action);

        await upsertPunch({
          employeeId,
          punchDate: today,
          punchType: action as 'in' | 'out',
          idempotencyKey,
        });
      }
    }

    res.json({ ok: true });
  } catch (err: any) {
    // If unique constraint violation or other conflict, still return OK to LINE to prevent retries.
    if (err?.code === '23505') {
      console.warn('Duplicate punch prevented by DB constraint', err);
      return res.json({ ok: true });
    }
    console.error('LINE webhook error', err);
    res.status(500).json({ error: 'internal' });
  }
});

export default router;
```

Why:
- Uses a deterministic idempotency key derived from LINE identifiers and employee/action.
- Handles duplicate-key errors gracefully (returns 200) so LINE stops retrying.
- Keeps the handler simple and fast.
