## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: a single CF Worker exposing an OpenAI-compatible /v1/chat/completions endpoint that proxies to OpenAI and Anthropic (chat models), with provider failover on 5xx/timeouts, per-provider in-memory+Redis rate-limit buckets, basic cost cap enforcement, and Prometheus metrics; Postgres migrations for Provider/RouteRule/RequestLog/QuotaLedger; local docker-compose for dev (FastAPI gateway + Redis + Postgres); no frontend, no auth beyond bearer token, no PII filter yet.

## Open questions
- Which provider auth secrets will be stored (Supabase Vault, CF Workers KV, or env) and how will rotation work?
- Will Redis be hosted via Supabase or an external free-tier Redis (e.g., Upstash) and what is the failover plan if Redis is unavailable?
- What is the canonical request/response normalization schema across providers (OpenAI-compatible or custom)?
- How will global rate-limit quotas be synchronized across CF Workers instances (Redis central or distributed algorithm)?
- Which PII filter implementation will be used (presidio, open-source regex, or LLM-based) and where will it run (edge or origin)?
