## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust CLI that: (1) spawns an ephemeral SOCKS5 listener on a random localhost port with LD_PRELOAD-based fd cloaking and strict localhost binding; (2) creates a TUN interface with randomized MAC/addresses and hides it from common enumeration paths; (3) tunnels traffic through a meek-like TLS-wrapped QUIC stream to a public relay (Cloudflare Workers endpoint or HF space acting as simple bridge) with domain-fronting; (4) enforces a process-level kill-switch (block all non-stealth traffic) if the QUIC tunnel fails; (5) records anonymized session success/failure locally to RocksDB and optionally uploads minimal WAU-ZB summaries to Supabase via one-time tokens; (6) provides zero-config presets (auto-select cover strategy and ephemeral ports) and runs entirely client-side with no server component required from the user.

## Open questions
- Which local artifacts (ports, sockets, /proc/net entries, netlink, firewall rules) are most commonly enumerated by DPI in target regions and what is the minimal cloaking surface?
- Can we reliably hide TUN interfaces from userspace enumeration without kernel modules while preserving throughput and stability?
- What is the acceptable latency/overhead budget for cover traffic (muxed streams, decoy bursts) before usability degrades?
- How will we perform health checks and failover for endpoints without leaking DNS or SNI during probes?
- What telemetry can be collected client-side that preserves WAU-ZB measurement without risking user deanonymization?
- Which jurisdictions require kill-switch defaults and how do we enforce fail-closed behavior if the stealth path drops?
