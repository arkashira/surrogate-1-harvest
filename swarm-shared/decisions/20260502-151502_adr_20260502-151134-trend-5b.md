## Deployment
cf workers (free-tier) + Supabase (Postgres + managed functions) + OSS Redis (local dev / free-tier managed where available)

## v0.1 scope
v0.1 in 2 weeks: single-asset (BTC/USD) research→signal→risk→paper execution pipeline with Redis Streams orchestration, Postgres audit log, deterministic run IDs, replay CLI, and perf harness targeting median signal-to-execution <500ms in local/CI runs.

## Open questions
- Deterministic idempotency scheme: content-hash vs run-nonce for replay fidelity?
- Broker/data-vendor auth model: OAuth2 refresh vs signed webhooks vs API keys in Vault-less free-tier?
- Latency budget split across stages to guarantee median <500ms (network vs compute vs DB)?
- Replay storage scope: full event snapshots vs event sourcing deltas on Postgres vs cold blob?
- Cloudflare Workers Python runtime limits vs containerized FastAPI on HF Spaces for free-tier scaling?
