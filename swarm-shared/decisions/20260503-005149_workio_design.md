# workio / design

### Final synthesized solution (correct + actionable)

**Diagnosis (merged, resolved)**
- LINE webhook retries (at-least-once) can create duplicate open punches for the same user/tenant.
- Application-only idempotency is insufficient under concurrency/retries; DB must enforce “one open punch per user/tenant”.
- No transactional upsert or DB-level guard allows race conditions on concurrent clock-ins.
- Idempotency must be stable across retries (use LINE’s event ID) and scoped to user/tenant.

**Proposed change (scope)**
- `workio/server/src/db/schema.sql` (or migration) — add columns and partial unique index.
- Webhook handler (`workio/server/src/routes/line.ts` or equivalent) — transactional upsert with idempotency and safe clock-out.
- Keep API contract simple: retry-safe, return success on duplicate idempotency, and always close an open punch on clock-out.

### Implementation

#### 1) DB schema: idempotency column + partial unique index
```sql
-- workio/server/src/db/schema.sql
-- Idempotency column (safe to run if already exists)
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key TEXT;

-- One open punch per (user, tenant)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user_tenant
  ON punches (user_id, tenant_id)
  WHERE clock_out_at IS NULL;

-- Fast lookup for idempotency (nullable; only enforce uniqueness when present)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key
  ON punches (idempotency_key)
  WHERE idempotency_key IS NOT NULL;
```

#### 2) Webhook handler: transactional upsert with idempotency
```typescript
// workio/server/src/routes/line.ts
import { Request, Response } from 'express';
import { pool } from '../db';

export async function handleLineWebhook(req: Request, res: Response) {
  const { events } = req.body;
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    for (const event of events) {
      if (event?.type !== 'message' || event.message?.type !== 'text') continue;

      const userId = event.source?.userId;
      const tenantId = process.env.LINE_TENANT_ID; // derive appropriately per tenant
      const idempotencyKey = event.webhookEventId; // stable per LINE event

      if (!userId || !tenantId || !idempotencyKey) {
        // malformed event — skip but continue processing others
        continue;
      }

      // Idempotency check
      const idempotent = await client.query(
        `SELECT id FROM punches WHERE idempotency_key = $1`,
        [idempotencyKey]
      );
      if (idempotent.rowCount > 0) {
        continue; // already processed
      }

      const isClockOut = /^(ออก|clock out|ออกงาน)$/i.test(event.message.text);

      if (isClockOut) {
        // Try to close an existing open punch
        const closeResult = await client.query(
          `UPDATE punches
           SET clock_out_at = NOW(), updated_at = NOW()
           WHERE user_id = $1 AND tenant_id = $2 AND clock_out_at IS NULL
           RETURNING id`,
          [userId, tenantId]
        );

        // If no open punch, create an audit record with immediate clock-out
        if (closeResult.rowCount === 0) {
          await client.query(
            `INSERT INTO punches (user_id, tenant_id, clock_in_at, clock_out_at, idempotency_key, created_at, updated_at)
             VALUES ($1, $2, NOW(), NOW(), $3, NOW(), NOW())`,
            [userId, tenantId, idempotencyKey]
          );
        }
      } else {
        // Clock-in: attempt insert; if conflict on open punch, do nothing (idempotent)
        await client.query(
          `INSERT INTO punches (user_id, tenant_id, clock_in_at, idempotency_key, created_at, updated_at)
           VALUES ($1, $2, NOW(), $3, NOW(), NOW())
           ON CONFLICT DO NOTHING`,
          [userId, tenantId, idempotencyKey]
        );
      }
    }

    await client.query('COMMIT');
    res.status(200).send({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook processing failed', err);
    res.status(500).send({ error: 'Processing failed' });
  } finally {
    client.release();
  }
}
```

### Verification (concrete tests)
1. **Duplicate idempotency** — send same LINE event twice; verify only one punch row created/updated.
2. **Concurrent clock-ins** — simulate parallel requests for same user/tenant; verify at most one open punch exists.
3. **Clock-out without open punch** — send clock-out event; verify an audit row with equal clock-in/clock-out is created and idempotent on retry.
4. **Partial unique constraint enforcement** — attempt to insert two open punches for same user/tenant via SQL; verify DB rejects second insert.
5. **Schema migration safety** — run migration on existing data; verify no downtime and existing open punches remain valid (idempotency_key nullable).
