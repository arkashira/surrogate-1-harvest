# vanguard / design

## 1. Diagnosis

- No content-addressed manifest per date folder → training repeatedly calls `list_repo_tree`/`load_dataset` at runtime, causing HF API 429s and non-reproducible epochs.
- Data loader uses Hugging Face `datasets` API during training (streaming/list calls) instead of CDN-only fetches → violates HF CDN bypass pattern and risks rate limits on Lightning workers.
- No deterministic shard-to-repo mapping for HF commit cap (128/hr/repo) → ingestion bursts can hit write limits and stall pipelines.
- Training script likely recreates Lightning Studio instead of reusing running ones → wastes 80hr/mo quota and increases cold-start latency.
- No guard against Lightning idle-stop killing long training runs → jobs silently die if studio stops between epochs.

## 2. Proposed change

Add a small, self-contained design module that fixes the highest-leverage item: **content-addressed manifest + CDN-only data loading**. Scope:

- Create `/opt/axentx/vanguard/design/manifest.py`
- Create `/opt/axentx/vanguard/design/train_cdn.py` (minimal training stub demonstrating CDN-only loading)
- Update `/opt/axentx/vanguard/design/README.md` (or create) with usage and patterns.

# /opt/axentx/vanguard/design/manifest.py
import json
import hashlib
import os
from datetime import datetime
from pathlib import Path
from typing import Dict, List

try:
    from huggingface_hub import list_repo_tree
except ImportError:
    list_repo_tree = None


def build_manifest(
    repo: str,
    date_folder: str,
    out_path: str,
    hf_token: str = None
) -> Dict:
    """
    Build content-addressed manifest for one date folder.
    Call once from Mac (or CI) after rate-limit window clears.
    Embed resulting JSON in training script for CDN-only fetches.
    """
    if list_repo_tree is None:
        raise RuntimeError("huggingface_hub required for manifest generation")

    tree = list_repo_tree(
        repo_id=repo,
        path=date_folder,
        recursive=False,
        token=hf_token,
    )

    files = []
    for entry in tree:
        if entry.type != "file":
            continue
        # CDN URL bypasses API auth/rate-limit
        cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{entry.path}"
        slug = Path(entry.path).stem
        content_hash = hashlib.sha256(f"{repo}:{entry.path}:{entry.size}".encode()).hexdigest()[:16]
        files.append({
            "path": entry.path,
            "slug": slug,
            "size": entry.size,
            "cdn_url": cdn_url,
            "content_hash": content_hash,
        })

    manifest = {
        "repo": repo,
        "date_folder": date_folder,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "files": files,
        "manifest_hash": hashlib.sha256(json.dumps(files, sort_keys=True).encode()).hexdigest()[:16],
    }

    out_path = Path(out_path)
    out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_text(json.dumps(manifest, indent=2))
    return manifest


def deterministic_repo_index(slug: str, n_repos: int = 5) -> int:
    """
    HF commit-cap mitigation: spread writes across N sibling repos.
    Deterministic mapping so same slug always targets same repo.
    """
    digest = int(hashlib.sha256(slug.encode()).hexdigest(), 16)
    return digest % n_repos
```

# /opt/axentx/vanguard/design/train_cdn.py
import json
import torch
from torch.utils.data import Dataset, DataLoader
import requests
from pathlib import Path
from typing import List

class CDNParquetDataset(Dataset):
    """
    CDN-only dataset: zero Hugging Face API calls during training.
    Manifest is embedded (generated by manifest.py on Mac/CI).
    """
    def __init__(self, manifest_path: str, max_files: int = None):
        manifest = json.loads(Path(manifest_path).read_text())
        self.files: List[dict] = manifest["files"]
        if max_files:
            self.files = self.files[:max_files]

    def __len__(self):
        return len(self.files)

    def __getitem__(self, idx):
        meta = self.files[idx]
        # Stream parquet bytes via CDN (no auth/API)
        resp = requests.get(meta["cdn_url"], timeout=30)
        resp.raise_for_status()
        # Minimal projection: {prompt, response} only
        # Replace with pyarrow.parquet.read() in memory when needed
        # For demo, return metadata + raw bytes
        return {
            "slug": meta["slug"],
            "content_hash": meta["content_hash"],
            "raw_bytes": resp.content,
        }

def make_loader(manifest_path: str, batch_size: int = 8, num_workers: int = 4):
    ds = CDNParquetDataset(manifest_path=manifest_path, max_files=None)
    return DataLoader(ds, batch_size=batch_size, num_workers=num_workers, shuffle=True)

# Vanguard Design: Manifest + CDN Training

## Patterns applied
- HF CDN bypass: public dataset files at `https://huggingface.co/datasets/{repo}/resolve/main/{path}` avoid API auth/rate-limits.
- Content-addressed manifests per date folder → reproducible epochs, zero runtime `list_repo_tree`/`load_dataset` calls.
- Deterministic repo index for HF commit-cap mitigation (spread writes across sibling repos).
- Lightning Studio reuse and idle-stop guards are operational concerns (see ops/); this module focuses on data layer correctness.

## Workflow
1. On Mac/CI (after HF rate-limit window clears):
   ```bash
   python -m design.manifest \
     --repo "org/surrogate-1" \
     --date-folder "batches/mirror-merged/2026-04-29" \
     --out-path "manifests/2026-04-29.json"
   ```
2. Embed `manifests/2026-04-29.json` in training repo or mount into Lightning Studio.
3. In Lightning Studio, run `train_cdn.py` (or your training script) with `CDNParquetDataset`; no HF API calls during training.

## Notes
- Do not use `load_dataset(streaming=True)` for repos with heterogeneous schemas; use `hf_hub_download` per file or CDN direct.
- Keep enriched/ projections minimal: {prompt, response} only; attribution in filename pattern.
