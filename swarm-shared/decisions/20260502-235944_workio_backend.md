# workio / backend

## Final Synthesized Solution

Below is the single, consolidated plan that keeps the strongest, most actionable parts from both proposals, removes duplication, and resolves conflicts in favor of **correctness, safety, and concrete actionability**.

---

## 1. Diagnosis (Consolidated)

- **Non-idempotent endpoints**: double-taps, LINE webhook retries, and client retries can create duplicate punches for the same user/day, corrupting hours/OT.
- **Race conditions and partial failure**: no transactional boundary that guarantees “punch write + audit + notification” all succeed or fail together; partial failures leave users and managers in ambiguous states.
- **Missing constraints**: no unique partial index to enforce at most one active punch per user (`clocked_out_at IS NULL`), and no idempotency mechanism for safe retries.
- **Time trust and auditability**: relying on client timestamps enables spoofing; audit trail lacks precision and source context, complicating debugging and compliance.
- **Notification coupling**: sending LINE messages inline with the DB commit risks lost confirmations or double-notifications on retries.

---

## 2. Proposed Change (Single Scope)

File: `/opt/axentx/workio/server/src/services/punchService.ts` (and related schema/routes)

Key scope:
- Add a **unique partial index** to enforce one active punch per user.
- Add **idempotency key** support (`user_id, idempotency_key` unique) for safe retries.
- Refactor punch operations into **one transaction** that:
  - Uses **server time** for all timestamps.
  - Writes punch + audit atomically.
  - Returns a consistent payload to the client.
- Decouple notifications: **enqueue a job after commit** (or fire-and-forget microtask if no queue) so LINE failures never block punch persistence.
- Define a clear endpoint contract: accept `Idempotency-Key` header/body; on conflict, return the existing punch instead of error.

---

## 3. Implementation (Final)

### 3.1 Schema changes

```sql
-- Ensure one active punch per user
CREATE UNIQUE INDEX idx_punches_active ON punches(user_id) WHERE clocked_out_at IS NULL;

-- Support idempotency (24h TTL managed by app or DB policy)
ALTER TABLE punches ADD COLUMN idempotency_key VARCHAR(64);
CREATE UNIQUE INDEX idx_punches_idempotency ON punches(user_id, idempotency_key) WHERE idempotency_key IS NOT NULL;

-- Audit and server-time anchors (if not present)
ALTER TABLE punches ALTER COLUMN created_at SET DEFAULT NOW();
ALTER TABLE punches ALTER COLUMN updated_at SET DEFAULT NOW();
-- Add source column if missing
ALTER TABLE punches ADD COLUMN IF NOT EXISTS source VARCHAR(16) NOT NULL DEFAULT 'app';
```

### 3.2 Service layer (transactional + idempotent)

```ts
// server/src/services/punchService.ts
import { db } from '../db';
import { sendLineMessage } from '../integrations/line';
import { v4 as uuidv4 } from 'uuid';

export type PunchInput = {
  userId: number;
  latitude?: number;
  longitude?: number;
  idempotencyKey?: string;
  source?: 'line' | 'web' | 'app';
};

function now() {
  return new Date();
}

async function enqueueLineNotify(userId: number, text: string) {
  // Best-effort, non-blocking. Replace with job queue if available.
  Promise.resolve()
    .then(() => sendLineMessage({ to: userId.toString(), text }))
    .catch((err) => console.error('LINE notify failed', err));
}

export async function clockIn(input: PunchInput) {
  const idempotencyKey = input.idempotencyKey || uuidv4();

  return db.transaction(async (trx) => {
    // Idempotency: return existing punch if idempotencyKey provided
    if (input.idempotencyKey) {
      const existing = await trx
        .select()
        .from('punches')
        .where({ user_id: input.userId, idempotency_key: input.idempotencyKey })
        .first();
      if (existing) return existing;
    }

    // Defensive: ensure no active punch (index is the true guard)
    const active = await trx
      .select()
      .from('punches')
      .where({ user_id: input.userId, clocked_out_at: null })
      .first();
    if (active) return active;

    const [punch] = await trx
      .insert({
        user_id: input.userId,
        clocked_in_at: now(),
        latitude: input.latitude,
        longitude: input.longitude,
        idempotency_key: idempotencyKey,
        source: input.source || 'app',
        created_at: now(),
        updated_at: now(),
      })
      .into('punches')
      .returning('*');

    // Non-blocking notification after commit
    enqueueLineNotify(input.userId, `✅ ลงเวลาทำงาน ${now().toLocaleString('th-TH')}`);

    return punch;
  });
}

export async function clockOut(input: {
  userId: number;
  latitude?: number;
  longitude?: number;
  idempotencyKey?: string;
}) {
  return db.transaction(async (trx) => {
    const active = await trx
      .select()
      .from('punches')
      .where({ user_id: input.userId, clocked_out_at: null })
      .orderBy('clocked_in_at', 'desc')
      .first();

    if (!active) {
      throw new Error('No active punch to clock out');
    }

    // Idempotency on clock-out (optional but recommended)
    if (input.idempotencyKey) {
      const existing = await trx
        .select()
        .from('punches')
        .where({ user_id: input.userId, idempotency_key: input.idempotencyKey })
        .first();
      if (existing) return existing;
    }

    const [updated] = await trx
      .update({
        clocked_out_at: now(),
        latitude_out: input.latitude,
        longitude_out: input.longitude,
        idempotency_key: input.idempotencyKey || active.idempotency_key,
        updated_at: now(),
      })
      .where({ id: active.id })
      .into('punches')
      .returning('*');

    enqueueLineNotify(input.userId, `🚪 ลงเวลาออก ${now().toLocaleString('th-TH')}`);

    return updated;
  });
}
```

### 3.3 Routes (Express)

```ts
// server/src/routes/punchRoutes.ts
import express from 'express';
import { clockIn, clockOut } from '../services/punchService';

const router = express.Router();

router.post('/in', async (req, res) => {
  try {
    const punch = await clockIn({
      userId: req.user.id,
      latitude: req.body.latitude,
      longitude: req.body.longitude,
      idempotencyKey: (req.headers['idempotency-key'] as string) || req.body.idempotencyKey,
      source: 'app',
    });
    res.json({ ok: true, punch });
  } catch (err: any) {
    res.status(400).json({ ok: false, error: err.message });
  }
});

router.post('/out', async (req, res) => {
  try {
    const punch = await clockOut({
      userId: req.user.id,
      latitude: req.body.latitude,
      longitude: req.body.longitude,
      idempotencyKey: (req.headers['idempotency-key'] as string) || req.body.idempotencyKey,
    });
    res.json({ ok: true, punch });
  } catch (err: any) {
    res.status(400).json({ ok: false, error: err.message });
  }
});

export default router;
```
