# vanguard / frontend

## Final synthesized plan (correctness + concrete actionability)

Goal: add a production-capable frontend scaffold to `/opt/axentx/vanguard` that enables fast local iteration, avoids runtime HF API rate limits, and supports a simple build/dev workflow. We combine the strongest, non-contradictory choices from both proposals and resolve in favor of correctness and immediate actionability.

---

### 1. Diagnosis (resolved)

- No frontend entrypoint/dev server exists → cannot iterate or ship UI changes.
- Missing `index.html` and client mount prevents fast feedback.
- Frontend must avoid runtime authenticated HF API calls (`list_repo_tree`, dataset endpoints) to prevent quota burn/429s on user machines.
- No asset pipeline/bundler → no HMR, no tree-shaking, manual reloads.
- No static file manifest embedded → every session re-enumerates the repo instead of using a pre-listed manifest + CDN-only fetches.

---

### 2. Chosen approach (resolved)

- Add a minimal Vite-based SPA scaffold (not “no build” only) because it gives HMR, sensible defaults, and a production build while remaining minimal.
- Provide a pre-generated `public/file-list.json` (manually curated or generated by orchestration) so the UI can operate via CDN-only fetches by default.
- Keep an optional, clearly-marked API refresh path for maintainers to regenerate the manifest (rate-limited; used sparingly).
- Add simple `dev`/`build`/`preview` npm scripts (Vite) plus an optional `scripts/dev.sh` for environments without Node/npm installed globally.
- Scope: new files only; no existing code modified.

---

function escapeHtml(s) {
  if (s == null) return '';
  return String(s)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}

function escapeAttr(s) {
  return escapeHtml(s).replace(/"/g, '&quot;');
}

function formatBytes(n) {
  if (n === 0) return '0 B';
  const i = Math.floor(Math.log2(n) / 10);
  const units = ['B', 'KiB', 'MiB', 'GiB', 'TiB'];
  const val = (n / Math.pow(1024, i)).toFixed(i > 0 ? 1 : 0);
  return `${val} ${units[i]}`;
}

// CDN-only fetch (no Authorization header) — avoids HF API rate limits.
function cdnUrl(repo, filePath) {
  return `https://huggingface.co/datasets/${repo}/resolve/main/${filePath}`;
}

// API call to list a folder (use sparingly; only to refresh manifest).
async function listRepoFolder(repo, folderPath) {
  const url = `https://huggingface.co/api/datasets/${repo}/tree?path=${encodeURIComponent(folderPath)}&recursive=false`;
  const res = await fetch(url);
  if (!res.ok) throw new Error(`HF API error: ${res.status}`);
  return res.json();
}

async function loadManifestFromCDN() {
  const url = cdnUrl(CONFIG.repo, CONFIG.manifestFile);
  setStatus('Loading manifest from CDN...');
  try {
    const res = await fetch(url);
    if (!res.ok) throw new Error(`Manifest fetch failed: ${res.status}`);
    const files = await res.json();
    if (!Array.isArray(files)) throw new Error('Manifest must be an array');
    renderFiles(files, 'manifest (CDN)');
    setStatus(`Loaded ${files.length} entries from CDN manifest`);
  } catch (err) {
    setStatus(`Failed to load manifest: ${err.message}`, 'error');
  }
}
