## Deployment
hybrid (client binaries + CF Workers for config/endpoint discovery; no server-side proxy to preserve client-only model)

## v0.1 scope
v0.1 ships a single cross-platform CLI binary (Linux x86_64) that: establishes an obfuscated tunnel via a shadowsocks-like AEAD stream to a remote endpoint; creates and manages a TUN interface via boringtun; enforces a fail-closed kill-switch that blocks non-tunnel IPv4/IPv6 traffic; stores profiles and encrypted session telemetry locally in SQLite; exposes minimal CLI flags for zero-config usage; and uses a CF Worker solely for signed, infrequent endpoint discovery (no server-side proxying).

## Open questions
- Which obfuscation camouflage (TLS-in-TLS, shadowsocks-like AEAD, or domain-fronting via CF) minimizes DPI signatures while maximizing throughput on low-end devices?
- How will we distribute and rotate endpoint lists without creating a detectable config-fetch signature (e.g., stagger + DoH + CF cache rules)?
- What local artifact cleanup sequence reliably hides SOCKS5 listeners and TUN interfaces from host firewall/DPI probes across Windows/macOS/Linux?
- How do we implement a reliable kill-switch across platforms that cannot leak during stealth-path failure (e.g., block non-tunnel traffic at firewall/namespace level)?
