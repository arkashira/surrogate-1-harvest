## v0.1 scope
v0.1 in 2 weeks: Android app (Kotlin + Rust via JNI) that creates per-app VPNService sessions, uses boringtun to establish a WireGuard-like tunnel without kernel TUN exposure, and offers a single meek+shadowsocks obfuscated endpoint via Cloudflare Workers edge relay; Supabase Postgres for device/auth metadata; no logs, local SQLite config, and basic WAUS/U counter. Goal: undetected browsing for 30 minutes continuous under lab DPI simulating SNI/ALPN inspection.

## Open questions
- Which obfuscation primitives are safest against current DPI in target jurisdictions (shadowsocks 2022 AEAD vs. meek vs. domain front vs. StunMux)?
- Can we avoid persistent TUN allocation on-device and instead use SOCKS5 redirect + per-app VPNService sessions to reduce fingerprint surface?
- How will we distribute endpoint lists and rotate front domains/IPs without centralized infra that can be blocked?
- What telemetry is acceptable for measuring WAUS/U without deanonymizing users or creating logs usable by adversaries?
- Do we need to support fallback transports (e.g., HTTP/3 direct or Tor bridge mode) when primary obfs fails?
- How will we handle certificate pinning and domain fronting given SNI and ALPN visibility by hostile networks?
