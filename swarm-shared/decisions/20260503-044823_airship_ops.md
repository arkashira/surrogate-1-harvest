## Final Synthesized Implementation Plan  
**Resilient Surrogate-1 Training Pipeline (≤2h)**  
Eliminate HF API 429s and Lightning idle-stop waste by switching to deterministic CDN-only data loading + studio lifecycle resilience + deterministic write sharding.

---

## Chosen Actions (Concrete, Ordered)

1. **Pre-list file paths once (Mac orchestration)**  
   - Single `list_repo_tree(..., recursive=False)` call after rate-limit window clears.  
   - Save to `file_list.json` and ship with training job.  
   - *Why*: Avoids recursive pagination and 429s during training.

2. **CDN-only data loader (Lightning training)**  
   - Replace `load_dataset(streaming=True)` with direct CDN fetches via `requests` (no Authorization header).  
   - Parse each downloaded file to project `{prompt, response}` only at parse time.  
   - *Why*: CDN tier has much higher limits and bypasses HF API auth-check rate limits.

3. **Studio lifecycle resilience**  
   - Before each `.run()`, check studio status; if stopped, restart with `target.start(machine=Machine.L40S)` (or fallback to free-tier L40S).  
   - *Why*: Lightning idle-stop kills training; auto-restart preserves quota and progress.

4. **Deterministic repo selection for commits (optional)**  
   - Hash slug → pick one of 5 sibling repos deterministically to spread writes (640/hr aggregate cap).  
   - *Why*: Avoids HF commit cap 128/hr/repo.

#!/usr/bin/env bash
# scripts/pre_list_files.sh
set -euo pipefail

REPO="org/surrogate-1-mirror"
DATE_DIR="batches/mirror-merged/2026-05-03"

python3 -c "

api = HfApi()
# non-recursive per folder to avoid pagination/429
tree = api.list_repo_tree(repo_id=REPO, path=DATE_DIR, recursive=False)
files = [t.rfilename for t in tree if t.type == 'file']
with open('$OUTFILE', 'w') as f:
    json.dump(files, f, indent=2)
print(f'Listed {len(files)} files -> $OUTFILE')
"

### 2. CDN-only IterableDataset (Lightning training)
```python
# surrogate/train.py
import json
import os
import requests
import pyarrow.parquet as pq
from torch.utils.data import IterableDataset

HF_DATASET_REPO = "org/surrogate-1-mirror"
CDN_BASE = f"https://huggingface.co/datasets/{HF_DATASET_REPO}/resolve/main"

class CDNParquetIterable(IterableDataset):
    def __init__(self, file_list_path, transform=None):
        with open(file_list_path) as f:
            self.files = json.load(f)
        self.transform = transform

    def __iter__(self):
        for fname in self.files:
            url = f"{CDN_BASE}/{fname}"
            # stream download to temp file or memory buffer
            resp = requests.get(url, stream=True, timeout=60)
            resp.raise_for_status()
            with open("/tmp/temp.parquet", "wb") as tmp:
                for chunk in resp.iter_content(chunk_size=8192):
                    tmp.write(chunk)
            table = pq.read_table("/tmp/temp.parquet", columns=["prompt", "response"])
            for row in table.to_pylist():
                item = {"prompt": row["prompt"], "response": row["response"]}
                if self.transform:
                    item = self.transform(item)
                yield item
```

### 3. Studio lifecycle resilience (Lightning SDK)
```python
from lightning import Lightning, Teamspace, Machine, Studio

lightning = Lightning()
teamspace = Teamspace()

STUDIO_NAME = "surrogate-train-l40s"
MACHINE = Machine.L40S  # fallback to free-tier L40S if quota allows

def get_or_create_studio():
    for s in teamspace.studios:
        if s.name == STUDIO_NAME:
            if s.status == "Running":
                print(f"Reusing running studio: {STUDIO_NAME}")
                return s
            else:
                print(f"Studio stopped; restarting...")
                s.start(machine=MACHINE)
                return s
    print(f"Creating studio: {STUDIO_NAME}")
    return Studio.create(name=STUDIO_NAME, machine=MACHINE, create_ok=True)

studio = get_or_create_studio()

# Before each training run, ensure running
if studio.status != "Running":
    studio.start(machine=MACHINE)

# Launch training (example)
job = studio.run(
    run_name="surrogate-cdn-train",
    entry_point="train.py",
    arguments=["--file-list", "file_list.json", "--epochs", "1"],
    environment="lightning-ai/pytorch:2.2.0-py3.10",
)
```

### 4. Sibling repo write sharding (optional)
```python
import hashlib

SIBLINGS = [f"org/surrogate-1-mirror-s{i}" for i in range(5)]

def pick_repo(slug: str) -> str:
    digest = hashlib.sha256(slug.encode()).hexdigest()
    idx = int(digest, 16) % len(SIBLINGS)
    return SIBLINGS[idx]
```
