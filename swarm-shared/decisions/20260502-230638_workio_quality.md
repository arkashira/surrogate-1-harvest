# workio / quality

## Final synthesized solution (correct + immediately actionable)

### 1) Diagnosis (resolved)
- **Root cause**: no database-level uniqueness + no idempotency key → retries/races create duplicate punches.
- **Scope to fix**: `punches` table schema, punch write path, and client/webhook contract.
- **Accept as fact**: application-only checks are unsafe; concurrency and redelivery will bypass them.

### 2) Schema changes (apply first)

```sql
-- server/src/db/schema.sql

-- Ensure date-only column exists and is populated
ALTER TABLE punches ADD COLUMN IF NOT EXISTS punch_date DATE;
UPDATE punches SET punch_date = DATE(punch_timestamp) WHERE punch_date IS NULL;

-- Idempotency key for safe retries
ALTER TABLE punches ADD COLUMN IF NOT EXISTS idempotency_key TEXT;
CREATE INDEX IF NOT EXISTS idx_punches_idempotency ON punches (idempotency_key) WHERE idempotency_key IS NOT NULL;

-- Enforce exactly-one punch per employee/type/date at DB level
-- Use NULLS NOT DISTINCT so multiple NULL idempotency_key don't block each other
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_unique_daily
ON punches (employee_id, punch_type, punch_date);
```

**Why this wins**:
- `punch_date` guarantees deterministic daily grouping.
- Unique index enforces correctness at the DB layer (last line of defense).
- Idempotency key + partial index enables fast, safe retries without blocking unrelated rows.

### 3) Idempotent punch endpoint (production-ready)

```ts
// server/src/routes/punch.ts
import { Router } from 'express';
import { db } from '../db';

const router = Router();

type PunchType = 'clock_in' | 'clock_out';

interface PunchRequestBody {
  employee_id: number;
  punch_type: PunchType;
  idempotency_key: string;        // required: LINE deliveryId or client UUID
  timestamp?: string;              // ISO; defaults to now
  latitude?: number;
  longitude?: number;
}

/**
 * Idempotent punch upsert.
 * Guarantees:
 * - Exactly one row per (employee_id, punch_type, punch_date)
 * - Replays with same idempotency_key return the original row
 * - Concurrent/retry-safe via transaction + unique constraint
 */
router.post('/punch', async (req, res) => {
  const {
    employee_id,
    punch_type,
    idempotency_key,
    timestamp = new Date().toISOString(),
    latitude,
    longitude,
  } = req.body as PunchRequestBody;

  if (!employee_id || !punch_type || !idempotency_key) {
    return res.status(400).json({ error: 'Missing required fields' });
  }

  const punchDate = new Date(timestamp).toISOString().split('T')[0];

  try {
    await db.begin(async (sql) => {
      // Fast path: already processed under this idempotency_key
      const byKey = await sql<{ id: number }[]>`
        SELECT id FROM punches WHERE idempotency_key = ${idempotency_key}
      `;
      if (byKey.count > 0) {
        const existing = await sql`SELECT * FROM punches WHERE id = ${byKey[0].id}`;
        return res.status(200).json({ message: 'Already processed', punch: existing[0] });
      }

      // Try insert; unique constraint prevents duplicates
      const inserted = await sql<{ id: number }[]>`
        INSERT INTO punches (
          employee_id,
          punch_type,
          punch_timestamp,
          punch_date,
          latitude,
          longitude,
          idempotency_key,
          created_at,
          updated_at
        )
        VALUES (
          ${employee_id},
          ${punch_type},
          ${timestamp}::timestamptz,
          ${punchDate}::date,
          ${latitude},
          ${longitude},
          ${idempotency_key},
          NOW(),
          NOW()
        )
        RETURNING id
      `.catch(async (err) {
        // Unique violation: another row already exists for this day
        const existing = await sql`
          SELECT * FROM punches
          WHERE employee_id = ${employee_id} AND punch_type = ${punch_type} AND punch_date = ${punchDate}::date
        `;
        if (existing.count > 0) {
          // Attach idempotency_key to existing row if missing (helps future idempotent lookups)
          await sql`
            UPDATE punches
            SET idempotency_key = ${idempotency_key}, updated_at = NOW()
            WHERE id = ${existing[0].id} AND idempotency_key IS NULL
          `;
          return existing;
        }
        throw err;
      });

      const punchRow = inserted?.[0] ?? (await sql`SELECT * FROM punches WHERE id = ${inserted[0].id}`)[0];
      res.status(201).json({ message: 'Punch recorded', punch: punchRow });
    });
  } catch (error: any) {
    console.error('Punch upsert failed', error);
    res.status(500).json({ error: 'Internal server error', details: error.message });
  }
});

export default router;
```

**Why this wins**:
- Uses transaction (`db.begin`) to make check-insert atomic.
- Fast path on `idempotency_key` avoids constraint exceptions on replays.
- Falls back safely on unique violation and attaches idempotency key to existing row.
- Returns consistent responses (`200 Already processed`, `201 Punch recorded`).

### 4) Client/webhook contract (non-negotiable)

- **Require** `idempotency_key` on every `/punch` request.
- For LINE webhooks, use `event.deliveryId`. If unavailable, generate a UUID client-side per logical punch attempt and reuse it on retries.
- Never retry without preserving the same `idempotency_key`.

Example snippet:

```ts
// LINE webhook handler
app.post('/webhook/line', async (req, res) => {
  for (const ev of req.body.events || []) {
    if (ev.type === 'message' && ev.message.type === 'text') {
      const deliveryId = ev.deliveryId || generateUUID();
      // call /punch with idempotency_key=deliveryId
    }
  }
  res.sendStatus(200);
});
```
