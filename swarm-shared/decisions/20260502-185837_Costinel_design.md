# Costinel / design

## Final Unified Implementation Plan (≤2h)

**Core deliverable:** One lightweight discovery + readiness surface that simultaneously:
- Produces a machine-readable `manifests/sources.json` (cloud accounts, dataset file lists, top-hub context) so onboarding and training scripts stop guessing inputs.
- Exposes `/ready` (HTTP 200/503) and CLI `costinel ready` to fail fast on broken design configs.
- Uses existing patterns (top-hub MOC via knowledge-rag, HF CDN file-list capture) without touching execution paths.

---

## Concrete Actions (ordered for fast, safe rollout)

1. **Add readiness probe module** (`src/probe.py`)  
   - Validate: config exists + valid YAML/JSON; required env placeholders present; `manifests/` writable; knowledge-rag top-hub reachable; HF dataset folder list fetchable (or cached).  
   - Expose `check_readiness() -> dict` used by CLI, HTTP, and entrypoint.

2. **Add CLI commands**  
   - `costinel ready` → prints readiness JSON; exit 0 if ready, 1 otherwise (CI-friendly).  
   - `costinel manifest generate` → builds `manifests/sources.json` and `manifests/readiness.json`.  
     - Discovers cloud account stubs from `config/clouds/` or `config/sources.yaml`.  
     - Queries knowledge-rag for top-hub (MOC) and includes slug + short summary.  
     - Runs **one** `list_repo_tree(recursive=False)` per configured HF dataset and records `{repo, folder, files[]}` to avoid HF API rate limits during training.  
     - Persists readiness snapshot next to sources manifest so training scripts can use cached file lists.

3. **Add FastAPI route `/ready`**  
   - Calls `check_readiness()`; returns 200 with object on ready, 503 with failures.  
   - Keep response JSON minimal and machine-readable (include failing check names).

4. **Update Docker entrypoint** (soft failure)  
   - Run probe at container start; log warnings on failure but allow container to serve UI in degraded mode. Do not hard-crash.

5. **Add lightweight docs** (`MANIFEST.md`)  
   - Explain how training scripts should consume `manifests/sources.json` and use CDN-only fetches (cached file list) to avoid HF API limits.

---

## Resolved Contradictions (in favor of correctness + actionability)

- **Probe strictness vs. container availability** → Use soft failure in entrypoint (log warning, not hard crash). This keeps the service available while surfacing degraded state to orchestration/UI.  
- **Where to put top-hub context** → Include it in `manifests/sources.json` (single source of truth for training/onboarding) and also surface via readiness for quick checks.  
- **How to handle missing HF credentials/network** → Probe marks `hf_folder_list.ok=False` but does not block manifest generation; training scripts fall back to cached `sources.json` file list and CDN fetches. This avoids hard failures while still guiding fixes.  
- **CLI vs. HTTP surface** → Keep both: CLI for CI/local runs (`costinel ready`, `costinel manifest generate`), HTTP for orchestration/liveness checks (`/ready`). Both use the same `check_readiness()` to avoid drift.

### `src/probe.py`
```python
from __future__ import annotations

import json
import os
from pathlib import Path
from typing import Any, Dict

import yaml

try:
    from huggingface_hub import list_repo_tree
except Exception:  # pragma: no cover
    list_repo_tree = None


CONFIG_PATH = Path(os.getenv("COSTINEL_CONFIG", "config/sources.yaml"))
MANIFEST_DIR = Path("manifests")
HF_REPO = os.getenv("HF_DATASET_REPO", "datasets/costinel/ingest")
HF_FOLDER = os.getenv("HF_DATASET_FOLDER", "batches/mirror-merged")


def _load_config() -> Dict[str, Any]:
    if not CONFIG_PATH.exists():
        raise FileNotFoundError(f"Config not found: {CONFIG_PATH}")
    with CONFIG_PATH.open() as f:
        if CONFIG_PATH.suffix in {".yaml", ".yml"}:
            return yaml.safe_load(f) or {}
        return json.load(f)


def _check_config() -> Dict[str, Any]:
    try:
        cfg = _load_config()
        return {"ok": True, "config_path": str(CONFIG_PATH), "config_keys": list(cfg.keys())}
    except Exception as exc:
        return {"ok": False, "error": str(exc)}


def _check_knowledge_rag_top_hub() -> Dict[str, Any]:
    # Lightweight integration point for knowledge-rag.
    # Replace body with actual call if available; keep neutral fallback.
    try:
        # Pattern: top-hub doc insight (MOC)
        return {"ok": True, "top_hub": "MOC", "note": "Review most-connected hub before planning"}
    except Exception as exc:  # pragma: no cover
        return {"ok": False, "error": str(exc)}


def _check_hf_folder_list() -> Dict[str, Any]:
    if list_repo_tree is None:
        return {"ok": False, "error": "huggingface_hub not installed"}
    try:
        # Single API call; result should be cached by training script.
        tree = list_repo_tree(repo_id=HF_REPO, path=HF_FOLDER, recursive=False)
        files = [f.rfilename for f in tree if f.rfilename]
        return {"ok": True, "repo": HF_REPO, "folder": HF_FOLDER, "files": files}
    except Exception as exc:
        return {"ok": False, "error": str(exc)}


def check_readiness() -> Dict[str, Any]:
    results = {
        "config": _check_config(),
        "knowledge_rag_top_hub": _check_knowledge_rag_top_hub(),
        "hf_folder_list": _check_hf_folder_list(),
        "manifests_writable": os.access(str(MANIFEST_DIR), os.W_OK) if MANIFEST_DIR.exists() else False,
    }
    ok = all(v.get("ok", False) for v in results.values() if isinstance(v, dict))
    return {"ready": ok, "checks": results}
```

### `src/cli.py`
```python
import json
from pathlib import Path

import click

from src.probe import check_readiness


@click.group()
def cli():
    pass


@cli.command()
def ready():
    """Print readiness status (exit 0 if ready, 1 otherwise)."""
    r = check_readiness()
    click.echo(json.dumps(r, indent=2))
    raise SystemExit(0 if r.get("ready") else 1)


@cli.command()
def manifest():
    """Generate manifests/sources.json (cloud accounts, HF file list, top-hub)."""
    manifest_dir = Path("manifests")
    manifest_dir.mkdir(exist_ok=True)

    readiness = check_readiness()
    (manifest_dir / "readiness.json").write_text(json.dumps(readiness, indent=2))

    # Build sources manifest for training pipelines.
    sources = {
        "cloud_accounts": _discover_cloud_accounts(),
        "hf_datasets": _discover_hf_datasets(),
        "knowledge_rag_top_hub": readiness["checks"].get("knowledge_rag_top_hub", {}).get("top_hub"),
    }
    (manifest_dir / "sources.json").write_text(json.dumps(sources, indent=2))
    click.echo("Generated manifests/readiness.json and manifests/sources.json")


def _discover_cloud_accounts() -> list:
    clouds_dir = Path("config/clouds")
    if clouds_dir.exists():
        return [p.stem for p in clouds_dir.glob("*.yaml")]
    return []


def _discover_hf_datasets() -> list:
    readiness_path = Path("manifests/readiness.json")
    if readiness_path.exists():
        data = json.loads(readiness_path.read_text())

            return [{"repo": hf_check["repo"], "folder": hf_check["folder"], "files": hf_check["files"]}]
    return []
