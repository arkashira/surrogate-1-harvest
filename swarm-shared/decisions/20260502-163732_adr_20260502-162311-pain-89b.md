## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: Android app (Kotlin + Go via gomobile) with VpnService-based tunnel, local SOCKS5 on ephemeral port mimicking HTTPS traffic, basic domain-fronted/bridged egress via Cloudflare Workers (free tier), Supabase-hosted remote config and policy, local SQLite policy store, adaptive endpoint health checks, and minimal WAUS/U counters stored locally; no account system required for initial stealth use, optional Supabase auth for config sync.

## Open questions
- Which obfuscation front (shadowsocks/vless+ws+tls/ssh+tls) provides best reach under current DPI in target jurisdictions while remaining free-tier deployable?
- Can we sufficiently mask TUN interfaces on stock Android without root (e.g., via VpnService + exclude list + tun fd hiding tricks) or do we need userspace WireGuard only?
- How will we distribute/rotate egress endpoints and port lists without centralized infra that can be blocked?
- What telemetry (if any) is acceptable for measuring WAUS/U without compromising user anonymity?
- How do we handle captive portals and networks that block non-browser ports while maintaining stealth?
