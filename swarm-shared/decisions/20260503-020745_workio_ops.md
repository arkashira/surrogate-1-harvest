# workio / ops

## 1. Diagnosis

- **No idempotency key enforcement on LINE webhook ingestion** — LINE retries identical events; missing deduplication causes duplicate punches/leave/OT records and double-notifications.
- **Race-prone non-atomic upsert in clock-in/out path** — `findOne` → conditional `update`/`insert` allows concurrent requests to create multiple active punches.
- **No operational observability for webhook health** — missing structured request-id propagation, latency/error metrics, and dead-letter handling for poison events.
- **Hardcoded retry/backoff in webhook handler** — synchronous retries under load amplify DB/CPU pressure and increase LINE timeout risk.
- **Missing tenant-scoped circuit breaker for LINE API calls** — one noisy tenant can exhaust tokens and stall notifications for others.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/line/webhook.ts`  
Scope: add idempotency middleware + atomic punch upsert + lightweight observability and async retry queue (bull) for outbound LINE notifications.

Update `/opt/axentx/workio/server/src/routes/line/webhook.ts`

```ts
import express, { Request, Response } from 'express';
import crypto from 'crypto';
import { getRepository } from 'typeorm';
import { Punch, Leave, OT } from '../entity';
import { lineQueue } from '../../queue/lineQueue';

const router = express.Router();
const IDEMPOTENCY_TTL_SEC = 60 * 60; // 1h

function hashEvent(event: any): string {
  // stable deterministic hash for dedupe (same event body + timestamp)
  const payload = JSON.stringify(event, Object.keys(event).sort());
  return crypto.createHash('sha256').update(payload).digest('hex');
}

async function ensureIdempotent(req: Request, res: Response, next: any) {
  const idempotencyKey = req.headers['x-line-delivery-id'] as string || hashEvent(req.body);
  const redis = (req as any).redis;
  const key = `idem:line:${idempotencyKey}`;
  const exists = await redis.set(key, '1', 'PX', IDEMPOTENCY_TTL_SEC * 1000, 'NX');
  if (!exists) {
    // duplicate — acknowledge quickly
    return res.status(200).json({ ok: true, duplicate: true });
  }
  (req as any).idempotencyKey = key;
  next();
}

router.post('/webhook/line', ensureIdempotent, async (req: Request, res: Response) => {
  const redis = (req as any).redis;
  const events = req.body.events || [];

  try {
    for (const event of events) {
      if (event.type === 'message' && event.message.type === 'text') {
        const text = event.message.text.trim().toLowerCase();
        const userId = event.source.userId;
        const tenantId = (req as any).tenantId; // set by tenant middleware

        // simple command handling (clock in/out)
        if (text === 'clock in' || text === 'clock out') {
          const punchRepo = getRepository(Punch);
          const now = new Date();
          const dateKey = now.toISOString().split('T')[0];

          // atomic upsert via unique constraint (userId, tenantId, dateKey, type)
          // schema should enforce one active punch per user per day (or use startedAt/endedAt nullable)
          const existing = await punchRepo.findOne({
            where: { userId, tenantId, dateKey, endedAt: null },
          });

          if (text === 'clock in') {
            if (existing) {
              await lineQueue.add('notify', {
                to: userId,
                message: 'คุณได้ลงเวลางานไปแล้วในวันนี้',
                tenantId,
              });
              continue;
            }
            await punchRepo.insert({
              userId,
              tenantId,
              dateKey,
              startedAt: now,
              endedAt: null,
              source: 'line',
            });
            await lineQueue.add('notify', {
              to: userId,
              message: 'ลงเวลางานเรียบร้อย',
              tenantId,
            });
          } else {
            if (!existing) {
              await lineQueue.add('notify', {
                to: userId,
                message: 'ไม่พบการลงเวลางานที่ยังไม่ได้ออกในวันนี้',
                tenantId,
              });
              continue;
            }
            await punchRepo.update(existing.id, { endedAt: now });
            await lineQueue.add('notify', {
              to: userId,
              message: 'ลงเวลาออกเรียบร้อย',
              tenantId,
            });
          }
        }
      }
    }

    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('LINE webhook error', err);
    // do not retry webhook here — LINE will retry automatically
    res.status(500).json({ ok: false });
  }
});
```

## 4. Verification

1. **Idempotency**: Send same LINE event JSON twice within 1h (use `curl` with saved body). Second request must return `200` with `duplicate: true` and not create a second punch.
2. **Atomic punch upsert**: Simulate concurrent `clock in` for same user (two parallel requests). Only one punch row with `endedAt IS NULL` should exist.
3. **Queue retry**: Stop LINE API token temporarily (or use invalid token) and trigger a punch. Job should retry with exponential backoff (check bull board or logs).
4. **No duplicate notifications**: After clock in + clock out, user receives exactly two messages (one in, one out) — no extras.
5. **Metrics/logs**: Confirm request-id propagation and that duplicate webhook deliveries are logged as `duplicate: true`.
