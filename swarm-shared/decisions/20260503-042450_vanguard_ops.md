# vanguard / ops

## 1. Diagnosis
- Runtime `load_dataset`/recursive file enumeration triggers HF API 429s and non-reproducible epochs; no content-addressed manifest prevents CDN-only fetching.
- No deterministic repo→date→file mapping means training scripts cannot pin exact file list; each run may see different shard set.
- Mixed-schema ingestion likely still writes extra columns (`source`, `ts`) into `enriched/`, breaking downstream surrogate-1 schema expectations.
- No guard to reuse running Lightning Studio; each training attempt may create new studio and burn quota (80hr/mo risk).
- No CDN-bypass downloader in training path; data loader still uses HF API during training (auth + rate-limit exposure).

## 2. Proposed change
Create `/opt/axentx/vanguard/ops/cdn_filelist.py` (single-file utility) and modify the surrogate-1 training launcher to:
- Accept a pinned file-list JSON (produced once per date folder on the Mac).
- Use CDN-only URLs (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) with `requests`/`aiohttp` and zero HF API calls during training.
- Reuse a running Lightning Studio if present; otherwise start one (L40S fallback).
- Keep schema projection to `{prompt, response}` only before upload; drop extra cols.

Scope: one new file + small edits to existing training launcher (likely `train.py` or `run_surrogate1.py` under `/opt/axentx/vanguard/`).

### /opt/axentx/vanguard/ops/cdn_filelist.py
```python
#!/usr/bin/env python3
"""
Generate and consume a pinned file-list for CDN-only dataset fetching.
Usage (Mac, once per date folder):
    python cdn_filelist.py --repo org/surrogate1 --date 2026-04-29 --out filelist.json
Training script loads filelist.json and fetches via CDN URLs only.
"""
import argparse
import hashlib
import json
import os
import sys
from typing import Dict, List

try:
    from huggingface_hub import list_repo_tree
except ImportError:
    print("Install: pip install huggingface_hub")
    sys.exit(1)

CDN_BASE = "https://huggingface.co/datasets"

def list_date_files(repo: str, date: str, max_per_folder: int = 10_000) -> List[Dict]:
    """
    List files under repo/{date}/ (non-recursive by folder) with content hashes.
    Uses single API call per immediate subfolder to minimize rate-limit risk.
    Returns list of dicts: {"path": "...", "sha256": "...", "cdn_url": "..."}
    """
    root = f"{date}"
    entries = list_repo_tree(repo=repo, path=root, recursive=False)
    files = []

    for entry in entries:
        if entry.type != "file":
            continue
        path = entry.path
        # CDN URL (no auth)
        cdn_url = f"{CDN_BASE}/{repo}/resolve/main/{path}"
        # content-addressed hash approximated by path+size (HF tree doesn't give sha256)
        # If you need true sha256, download HEAD ETag or use hf_hub_download with etag.
        # For reproducibility we include size + path hash.
        size = getattr(entry, "size", 0)
        sha256 = hashlib.sha256(f"{repo}/{path}:{size}".encode()).hexdigest()
        files.append({"path": path, "size": size, "sha256": sha256, "cdn_url": cdn_url})
        if len(files) >= max_per_folder:
            break
    return files

def generate(repo: str, date: str, out_path: str) -> None:
    files = list_date_files(repo, date)
    manifest = {
        "repo": repo,
        "date": date,
        "generated_by": "cdn_filelist.py",
        "count": len(files),
        "files": files,
    }
    with open(out_path, "w", encoding="utf-8") as f:
        json.dump(manifest, f, indent=2)
    print(f"Wrote {len(files)} entries to {out_path}")

def load_manifest(path: str) -> Dict:
    with open(path, encoding="utf-8") as f:
        return json.load(f)

if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Generate pinned CDN file list.")
    parser.add_argument("--repo", required=True, help="HF dataset repo (org/name)")
    parser.add_argument("--date", required=True, help="Date folder (YYYY-MM-DD)")
    parser.add_argument("--out", default="filelist.json", help="Output JSON path")
    args = parser.parse_args()
    generate(args.repo, args.date, args.out)
```

def get_or_create_studio(name: str, machine):
    """Reuse running studio or create new one."""
    from lightning.pytorch.studio import Studio, Teamspace
    for s in Teamspace.studios:
        if s.name == name and s.status == "Running":
            print(f"Reusing running studio: {s.name}")
            return s
    print(f"Creating studio: {name}")
    return Studio(name=name, create_ok=True, machine=machine)

Replace dataset loading with CDN-only loader:
```python
import io
import requests
import pyarrow.parquet as pq

def cdn_parquet_loader(file_list_path: str, columns=("prompt", "response")):
    manifest = json.load(open(file_list_path))
    for fobj in manifest["files"]:
        url = fobj["cdn_url"]
        resp = requests.get(url, timeout=30)
        resp.raise_for_status()
        table = pq.read_table(io.BytesIO(resp.content))
        # Project to required schema only
        table = table.select(columns)
        yield table.to_pydict()
```
