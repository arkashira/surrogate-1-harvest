# surrogate-1 / design

## Final Implementation Plan (≤2h)

**Highest-value improvement**: Add a pre-flight snapshot generator (`bin/snapshot.sh`) that lists all dataset files once, embeds the file manifest into training, and enables CDN-only downloads during parallel shard processing. This eliminates HF API rate-limit pressure during ingestion and guarantees deterministic file lists for reproducible sharding.

---

### Concrete steps (1h 30m total)

1. **Create `bin/snapshot.sh`** (20m)  
   - Uses `huggingface_hub` to call `list_repo_tree(recursive=False)` per date folder under `public-merged/` (or `batches/public-merged/` if that’s the actual layout).  
   - Outputs `snapshot-<date>.json` with `{ "repo": "...", "date": "...", "files": ["path1", "path2", ...], "generated_at": "ISO8601" }`.  
   - Stores snapshot in repo under `snapshots/` and optionally uploads to dataset repo at `snapshots/` (single small commit).  
   - Exits non-zero on unrecoverable errors; on 429 uses exponential backoff with retry-after header handling (wait 360s max).

2. **Update `bin/dataset-enrich.sh`** (30m)  
   - Accept optional snapshot path via env `SNAPSHOT_FILE`.  
   - If provided, reads file list from snapshot instead of calling `list_repo_tree` per shard.  
   - Deterministic shard assignment: `hash(slug) % 16 == SHARD_ID` (same as current).  
   - Keeps existing streaming/parquet decode logic unchanged.  
   - Fallback: if snapshot missing/unusable, list via API with warning.

3. **Add `bin/lib/snapshot.py`** (20m)  
   - Thin wrapper using `huggingface_hub` with exponential backoff and jitter.  
   - Validates tree response and normalizes paths.  
   - Emits compact JSON for easy embedding.

4. **Update GitHub Actions matrix** (20m)  
   - Add a pre-job step that runs `snapshot.sh` once and uploads artifact `snapshot-<date>.json`.  
   - Pass artifact path to each shard via `env.SNAPSHOT_FILE`.  
   - If snapshot fails (e.g., 429), fallback to current behavior (list per shard) with warning.

5. **Add training-side integration** (20m)  
   - Create `scripts/embed_filelist.py` that reads snapshot and writes `filelist.txt` into training container.  
   - Modify `train.py` to read `filelist.txt` and construct CDN URLs (`https://huggingface.co/datasets/.../resolve/main/...`) for `hf_hub_download`-style fetches without auth.  
   - Keep `load_dataset` fallback for non-CDN paths and for files not in snapshot.

6. **Validation & docs** (20m)  
   - Add smoke test: run snapshot locally, verify shard counts match.  
   - Update README with usage:  
     ```bash
     # Generate snapshot (once per day)
     bin/snapshot.sh --date 2026-05-03 --out snapshots/
     # Run enrichment with snapshot
     SNAPSHOT_FILE=snapshots/snapshot-2026-05-03.json bin/dataset-enrich.sh
     ```

```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
DATE="${1:-$(date +%Y-%m-%d)}"
OUTDIR="${2:-snapshots}"
OUTFILE="${OUTDIR}/snapshot-${DATE}.json"
MAX_RETRIES=3
RETRY_WAIT=60

mkdir -p "$OUTDIR"

python3 "$(dirname "$0")/lib/snapshot.py" \
  --repo "$REPO" \
  --date "$DATE" \
  --out "$OUTFILE" \
  --max-retries "$MAX_RETRIES" \
  --retry-wait "$RETRY_WAIT"

echo "Snapshot written to $OUTFILE"
```

### `bin/lib/snapshot.py`
```python
#!/usr/bin/env python3
import argparse
import json
import time
import sys
from datetime import datetime, timezone

from huggingface_hub import HfApi, RepositoryNotFoundError, HFValidationError

def list_date_files(repo_id: str, date: str, max_retries: int = 3, retry_wait: int = 60):
    api = HfApi()
    # Support both public-merged/ and batches/public-merged/ layouts
    prefixes = [f"public-merged/{date}/", f"batches/public-merged/{date}/"]
    files = []

    for prefix in prefixes:
        for attempt in range(1, max_retries + 1):
            try:
                tree = api.list_repo_tree(
                    repo_id=repo_id,
                    path=prefix,
                    recursive=False,
                    repo_type="dataset",
                )
                batch = [item.rfilename for item in tree if not item.rfilename.endswith("/")]
                files.extend(batch)
                break
            except Exception as exc:
                msg = str(exc)
                if "429" in msg or "rate limit" in msg.lower():
                    if attempt == max_retries:
                        raise
                    wait = retry_wait * (2 ** (attempt - 1))
                    print(f"Rate limited (attempt {attempt}/{max_retries}), waiting {wait}s", file=sys.stderr)
                    time.sleep(wait)
                    continue
                # If prefix doesn't exist, try next layout
                if isinstance(exc, (RepositoryNotFoundError, HFValidationError)) or "404" in msg:
                    break
                raise
        if files:
            break

    return sorted(set(files))

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--repo", required=True)
    parser.add_argument("--date", required=True)
    parser.add_argument("--out", required=True)
    parser.add_argument("--max-retries", type=int, default=3)
    parser.add_argument("--retry-wait", type=int, default=60)
    args = parser.parse_args()

    try:
        files = list_date_files(args.repo, args.date, args.max_retries, args.retry_wait)
    except Exception as exc:
        print(f"Failed to list files: {exc}", file=sys.stderr)
        sys.exit(1)

    snapshot = {
        "repo": args.repo,
        "date": args.date,
        "generated_at": datetime.now(timezone.utc).isoformat(),
        "files": sorted(files),
    }

    with open(args.out, "w", encoding="utf-8") as f:
        json.dump(snapshot, f, separators=(",", ":"))
    print(f"Wrote {len(files)} files to {args.out}")

if __name__ == "__main__":
    main()
```
