# workio / quality

## Final Synthesized Solution

**Chosen approach**: enforce exactly-once punches at the **database layer** (strongest guarantee), add **immutable audit** for compliance, and provide **idempotency for both client retries and webhook replays**.  
Contradictions resolved in favor of **correctness + concrete actionability**:

- Use a **unique constraint/index on `attendance`** (not `punches`) so the fix applies to the actual table used by the app.
- Keep **`ON CONFLICT DO NOTHING`** for inserts (simpler and safer than upsert for clock events) and return the existing row on conflict so clients can reconcile state.
- Add a **webhook dedupe table** keyed by event ID to absorb LINE/webhook retries independent of business-logic uniqueness.
- Record **immutable audit entries** for every insert/update/delete with before/after values and reason.
- Provide **transactional, connection-safe** endpoint code and explicit verification steps.

### 1) Database schema (`server/src/db/schema.sql`)

```sql
-- 1) Prevent duplicate punches per employee/punch_type/day
CREATE UNIQUE INDEX IF NOT EXISTS idx_attendance_unique_punch
  ON attendance (employee_id, punch_type, DATE(punched_at));

-- 2) Validate punch_type values
ALTER TABLE attendance
  ADD CONSTRAINT chk_attendance_punch_type
  CHECK (punch_type IN ('clock_in', 'clock_out'));

-- 3) Idempotency table for webhook retries (e.g., LINE) and client keys
CREATE TABLE IF NOT EXISTS event_idempotency (
  id              TEXT PRIMARY KEY,            -- webhook_event_id or client idempotency key
  attendance_id   INTEGER REFERENCES attendance(id) ON DELETE SET NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_event_idempotency_lookup
  ON event_idempotency(id);

-- 4) Immutable audit trail for punches
CREATE TABLE IF NOT EXISTS attendance_audits (
  id            SERIAL PRIMARY KEY,
  attendance_id INTEGER NOT NULL REFERENCES attendance(id) ON DELETE CASCADE,
  changed_by    INTEGER NOT NULL REFERENCES employees(id),
  action        TEXT NOT NULL CHECK (action IN ('insert', 'update', 'delete')),
  old_value     JSONB,
  new_value     JSONB,
  reason        TEXT,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_attendance_audits_attendance_id
  ON attendance_audits(attendance_id);

CREATE INDEX IF NOT EXISTS idx_attendance_audits_created_at
  ON attendance_audits(created_at);
```

### 2) Idempotent attendance endpoint (`server/src/routes/attendance.ts`)

```ts
import express from 'express';
import { pool } from '../db';

const router = express.Router();

/**
 * Idempotent punch endpoint.
 * Body: { employeeId, punchType, punchedAt?, idempotencyKey? }
 *
 * Behavior:
 * - idempotencyKey: client or webhook-supplied key to dedupe retries.
 * - If key provided and seen: return existing punch (200) without side-effects.
 * - If no key: rely on DB unique constraint to reject duplicates (409).
 * - All writes are transactional and audited.
 */
router.post('/punch', async (req, res) => {
  const { employeeId, punchType, punchedAt, idempotencyKey } = req.body;

  if (!employeeId || !punchType) {
    return res.status(400).json({ error: 'employeeId and punchType required' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Fast path: idempotency key already processed
    if (idempotencyKey) {
      const prior = await client.query(
        `SELECT attendance_id FROM event_idempotency WHERE id = $1`,
        [idempotencyKey]
      );
      if (prior.rows[0]?.attendance_id) {
        const existing = await client.query(
          `SELECT * FROM attendance WHERE id = $1`,
          [prior.rows[0].attendance_id]
        );
        await client.query('COMMIT');
        return res.json({ punch: existing.rows[0], ok: true, idempotent: true });
      }
    }

    const punchedAtTs = punchedAt ? new Date(punchedAt) : new Date();

    // Try insert; DO NOTHING on conflict to avoid overwriting existing punch
    const insertResult = await client.query(
      `INSERT INTO attendance (employee_id, punch_type, punched_at, created_by)
       VALUES ($1, $2, $3, $4)
       ON CONFLICT (employee_id, punch_type, DATE(punched_at)) DO NOTHING
       RETURNING *`,
      [employeeId, punchType, punchedAtTs, employeeId]
    );

    let punchRow = insertResult.rows[0];

    if (!punchRow) {
      // Conflict: fetch existing row to return consistent state
      const conflict = await client.query(
        `SELECT * FROM attendance
         WHERE employee_id = $1 AND punch_type = $2 AND DATE(punched_at) = DATE($3)`,
        [employeeId, punchType, punchedAtTs]
      );
      punchRow = conflict.rows[0];

      // Still record idempotency key if provided (maps to existing punch)
      if (idempotencyKey) {
        await client.query(
          `INSERT INTO event_idempotency (id, attendance_id)
           VALUES ($1, $2)
           ON CONFLICT (id) DO NOTHING`,
          [idempotencyKey, punchRow.id]
        );
      }

      await client.query('COMMIT');
      return res.status(409).json({
        error: 'Duplicate punch',
        existing: punchRow,
        hint: 'Use idempotencyKey to safely retry'
      });
    }

    // Record idempotency key for future retries
    if (idempotencyKey) {
      await client.query(
        `INSERT INTO event_idempotency (id, attendance_id)
         VALUES ($1, $2)
         ON CONFLICT (id) DO NOTHING`,
        [idempotencyKey, punchRow.id]
      );
    }

    // Audit insert
    await client.query(
      `INSERT INTO attendance_audits (attendance_id, changed_by, action, new_value, reason)
       VALUES ($1, $2, $3, $4, $5)`,
      [punchRow.id, employeeId, 'insert', punchRow, idempotencyKey ? 'idempotent insert' : 'normal insert']
    );

    await client.query('COMMIT');
    return res.json({ punch: punchRow, ok: true });
  } catch (err: any) {
    await client.query('ROLLBACK');
    console.error('Punch error', err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

### 3) Verification steps

1) **Duplicate prevention (no idempotency key)**  
```bash
# First punch
curl -X POST http://localhost:3000/attendance/punch \
  -H "Content-Type: application/json" \
  -d '{"employeeId":1,"punchType":"clock_in"}'

# Immediate retry (same employee/day/type)
curl -X POST http://localhost:3000/attendance/punch \
  -H "Content-Type: application/json" \
  -d '{"employeeId":1,"punchType":"clock_in"}'
```
Expect second call to return `409` with `existing` record.

2) **Idempotency with key (client or webhook retry)**  
```bash
KEY=$(uuidgen)
curl -X POST http://localhost:3000/attendance/punch \
  -H "Content-Type: application/json" \
  -d "{\"employeeId\":1,\"punchType\":\"clock_out\",\"idempotencyKey\":\"$KEY\"}"
# Repeat same request with same KEY -> returns same punch, idempotent
