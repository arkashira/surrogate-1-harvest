# workio / design

## 1. Diagnosis
- **No tenant cache on LINE webhook** — every webhook hits PostgreSQL to resolve `channelId → tenant`; slow under burst traffic, noisy under load, and fails fast if DB is cold.
- **Missing health/readiness endpoints** — no `/health` or `/ready` for k8s/PM2/liveness checks; deploy/rollback is blind.
- **No graceful shutdown** — in-flight LINE webhook requests can be dropped on restart/deploy, causing missed punches.
- **LINE signature verification likely missing or ad-hoc** — increases risk of spoofed punches.
- **No structured request ID / trace** — hard to correlate LINE events to DB writes in logs.

## 2. Proposed change
Add a TTL-backed LRU tenant cache (keyed by `channelId`) in the tenant service, plus `/health` and `/ready` endpoints, graceful shutdown, and structured request IDs.  
Scope:
- `workio/server/src/services/tenantService.ts` (add cache + TTL)
- `workio/server/src/middleware/requestId.ts` (new)
- `workio/server/src/middleware/lineSignature.ts` (new)
- `workio/server/src/index.ts` (add routes + graceful shutdown + middleware)

## 3. Implementation

### 3.1 tenantService.ts — add TTL LRU cache
```ts
// workio/server/src/services/tenantService.ts
import { LRUCache } from 'lru-cache';

type Tenant = { id: string; name: string; channelId: string };

const tenantCache = new LRUCache<string, Tenant>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
  allowStale: false,
  updateAgeOnGet: true,
});

export async function getTenantByChannelId(channelId: string): Promise<Tenant | null> {
  const cached = tenantCache.get(channelId);
  if (cached) return cached;

  // db lookup (pseudo — adapt to your query layer)
  const row = await db.query<Tenant>(
    'SELECT id, name, channel_id AS "channelId" FROM tenants WHERE channel_id = $1 LIMIT 1',
    [channelId]
  );
  const tenant = row.rows[0] || null;
  if (tenant) tenantCache.set(channelId, tenant);
  return tenant;
}

export function invalidateTenantCache(channelId: string) {
  tenantCache.delete(channelId);
}
```

### 3.2 requestId middleware
```ts
// workio/server/src/middleware/requestId.ts
import { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';

export function requestId(req: Request, _res: Response, next: NextFunction) {
  const id = req.headers['x-request-id'] as string || crypto.randomUUID();
  (req as any).requestId = id;
  next();
}
```

### 3.3 lineSignature middleware
```ts
// workio/server/src/middleware/lineSignature.ts
import { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';

const CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

function verifySignature(body: string, signature: string): boolean {
  if (!CHANNEL_SECRET) return true; // skip in dev if unset
  const expected = crypto
    .createHmac('sha256', CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

export function lineSignature(req: Request, res: Response, next: NextFunction) {
  const sig = req.headers['x-line-signature'] as string;
  const rawBody = (req as any).rawBody || JSON.stringify(req.body);
  if (!verifySignature(rawBody, sig)) {
    return res.status(401).json({ error: 'Invalid LINE signature' });
  }
  next();
}
```

### 3.4 index.ts — wire middleware, health/ready, graceful shutdown
```ts
// workio/server/src/index.ts
import express from 'express';
import bodyParser from 'body-parser';
import { requestId } from './middleware/requestId';
import { lineSignature } from './middleware/lineSignature';
import { getTenantByChannelId } from './services/tenantService';
import webhookRouter from './routes/webhook';

const app = express();
const PORT = process.env.PORT || 3000;

// capture raw body for LINE signature
app.use('/webhook/line', bodyParser.raw({ type: 'application/json' }), (req, res, next) => {
  (req as any).rawBody = req.body.toString('utf8');
  next();
});

app.use(express.json());
app.use(requestId);

// routes
app.use('/webhook/line', lineSignature, webhookRouter);

// health/readiness
app.get('/health', (_req, res) => res.json({ status: 'ok', uptime: process.uptime() }));
app.get('/ready', async (_req, res) => {
  try {
    // lightweight DB ping
    await db.query('SELECT 1');
    res.json({ status: 'ready' });
  } catch (err) {
    res.status(503).json({ status: 'not ready', error: String(err) });
  }
});

const server = app.listen(PORT, () => console.log(`Server listening on ${PORT}`));

// graceful shutdown
function shutdown() {
  console.log('Shutting down gracefully...');
  server.close(() => {
    console.log('HTTP server closed');
    process.exit(0);
  });
  // force exit after timeout
  setTimeout(() => {
    console.error('Forced shutdown after timeout');
    process.exit(1);
  }, 10_000).unref();
}

process.on('SIGTERM', shutdown);
process.on('SIGINT', shutdown);
```

### 3.5 webhook route — use cache
```ts
// workio/server/src/routes/webhook.ts
import { Router } from 'express';
import { getTenantByChannelId } from '../services/tenantService';

const router = Router();

router.post('/', async (req, res) => {
  const events = req.body.events || [];
  const channelId = req.body.destination || (req.headers['x-line-channel-id'] as string);
  if (!channelId) return res.status(400).json({ error: 'Missing channelId' });

  const tenant = await getTenantByChannelId(channelId);
  if (!tenant) return res.status(404).json({ error: 'Tenant not found' });

  // process events (tenant-aware)
  for (const ev of events) {
    // your existing logic, now with tenant.id available
    console.log(`[${(req as any).requestId}] tenant=${tenant.id} event=${ev.type}`);
  }

  res.json({ status: 'accepted' });
});

export default router;
```

## 4. Verification
- **Cache hit/miss**: Start server, send two LINE events with same `channelId`; check logs — second request should show cache hit (add a debug log in `getTenantByChannelId` if desired).
- **Health/ready**: `curl http://localhost:3000/health` → `{"status":"ok",...}`; `curl http://localhost:3000/ready` → `{"status":"ready"}` when DB is up, else 503.
- **Signature verification**: Send a POST to `/webhook/line` with invalid `x-line-signature` → 401; with valid signature (or none in dev) → 200.
- **Graceful shutdown**: Start server, send in-flight request, then `kill -SIGTERM <pid>`; verify in-flight request completes before exit.
- **Request ID**: Check logs for `requestId` correlation across middleware and handlers.
