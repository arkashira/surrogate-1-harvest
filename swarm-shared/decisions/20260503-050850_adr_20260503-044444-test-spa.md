## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service that connects to Supabase; scans one AWS region and one GCP project (via supplied credentials); detects orphaned schedules (platform function missing) and exact-duplicate cron expressions per function; records schedule + audit_event in Postgres; exposes GET /schedules and GET /issues endpoints; no UI, no queue/worker, no drift detection beyond hash comparison.

## Open questions
- Which IAM/service-account read-only scopes are acceptable to customers for scanning Lambda and Cloud Functions?
- How will customers authenticate (Supabase JWT + CF Workers OAuth path vs API key)?
- What retention/TTL policy for schedule metadata and audit events on the free tier?
- Will duplicate detection be exact-match or fuzzy (timezone/format normalization)?
