# workio / ops

## 1. Diagnosis
- Missing **LINE webhook signature verification** (`X-Line-Signature`) allows spoofed/replayed events to create duplicate clock-in/out records.
- No **idempotency key** on webhook handler → LINE retries (network/timeouts) create duplicate attendance rows and leave/OT requests.
- Webhook handler does not validate **timestamp tolerance** (replay window) → stale events can be accepted.
- No **structured audit log** for webhook deliveries (request id, signature check, processing outcome) → hard to debug production issues.
- Missing **graceful duplicate detection** at DB level (unique constraint on `(user_id, clock_in_at::date)` for attendance) to protect against race/retry duplicates.

## 2. Proposed change
- **Files**: 
  - `server/src/middleware/lineWebhook.ts` (new) — signature + timestamp verification middleware.
  - `server/src/routes/webhook/line.ts` — apply middleware, add idempotency, wrap handler in transaction.
  - `server/src/db/schema.sql` — add unique constraint/index for daily attendance and idempotency table.
  - `server/src/db/migrations/20260503_ops_webhook.sql` (optional) — migration for audit/idempotency.
- **Scope**: webhook entrypoint + DB constraints (non-breaking, additive).

## 3. Implementation

### `server/src/middleware/lineWebhook.ts`
```ts
import crypto from 'crypto';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

if (!LINE_CHANNEL_SECRET) {
  throw new Error('LINE_CHANNEL_SECRET is required');
}

export function verifyLineSignature(rawBody: string, signature: string): boolean {
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

// Reject events older than 5 minutes to prevent replays
export function isTimestampFresh(timestamp: number, toleranceMs = 5 * 60 * 1000): boolean {
  const now = Date.now();
  return Math.abs(now - timestamp) <= toleranceMs;
}

export function lineWebhookMiddleware(req, res, next) {
  const signature = req.headers['x-line-signature'] as string;
  const rawBody = req.rawBody || JSON.stringify(req.body);

  if (!signature) {
    return res.status(400).json({ error: 'Missing X-Line-Signature' });
  }

  if (!verifyLineSignature(rawBody, signature)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // If body contains events, check event timestamps (LINE provides timestamp per webhook)
  if (req.body && req.body.events && Array.isArray(req.body.events)) {
    for (const ev of req.body.events) {
      if (ev.timestamp && !isTimestampFresh(ev.timestamp)) {
        return res.status(400).json({ error: 'Stale event timestamp' });
      }
    }
  }

  next();
}
```

### `server/src/db/schema.sql` (additions)
```sql
-- Idempotency table for LINE webhook deliveries
CREATE TABLE IF NOT EXISTS line_webhook_deliveries (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  event_type VARCHAR(100) NOT NULL,
  user_id VARCHAR(255) NOT NULL,
  processed_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload JSONB NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Prevent duplicate clock-ins per user per calendar day (soft-delete safe)
CREATE UNIQUE INDEX IF NOT EXISTS idx_attendance_user_date
ON attendances (user_id, DATE(clock_in_at))
WHERE deleted_at IS NULL;
```

### `server/src/routes/webhook/line.ts`
```ts
import express from 'express';
import { verifyLineSignature, lineWebhookMiddleware } from '../middleware/lineWebhook';
import { pool } from '../../db';
import crypto from 'crypto';

const router = express.Router();

// Ensure raw body is available for signature verification
router.use(express.raw({ type: 'application/json' }));
router.use(lineWebhookMiddleware);

router.post('/', async (req, res) => {
  const body = JSON.parse(req.body.toString());
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    for (const ev of body.events) {
      // Idempotency: use LINE webhook delivery id or hash of event+timestamp
      const idemKey = ev.source?.userId && ev.timestamp
        ? crypto.createHash('sha256')
            .update(`${ev.source.userId}:${ev.timestamp}:${ev.type}`)
            .digest('hex')
        : crypto.randomUUID();

      const exists = await client.query(
        'SELECT 1 FROM line_webhook_deliveries WHERE idempotency_key = $1',
        [idemKey]
      );

      if (exists.rows.length > 0) {
        continue; // skip duplicate
      }

      // Record idempotency first
      await client.query(
        'INSERT INTO line_webhook_deliveries (idempotency_key, event_type, user_id, payload) VALUES ($1, $2, $3, $4)',
        [idemKey, ev.type, ev.source?.userId || '', ev]
      );

      // Process event (example: clock-in/out)
      if (ev.type === 'message' && ev.message?.text === 'clock in') {
        await client.query(
          `INSERT INTO attendances (user_id, clock_in_at)
           VALUES ($1, NOW())
           ON CONFLICT DO NOTHING`, // relies on unique index above
          [ev.source.userId]
        );
      }

      // Add leave/OT handling similarly with idempotency checks
    }

    await client.query('COMMIT');
    res.json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
});

export default router;
```

In your main server file (e.g., `server/src/index.ts`), mount the line webhook route **before** generic JSON middleware or ensure `express.raw` is applied only to this route (as shown above).

## 4. Verification
1. **Unit test signature verification**:
   - Create a small script that signs a payload with `LINE_CHANNEL_SECRET` and assert `verifyLineSignature` returns true; assert false when tampered.
2. **Replay test**:
   - Send same payload twice; second request should be idempotent (no duplicate attendance row). Check `line_webhook_deliveries` for single entry.
3. **Stale event rejection**:
   - Send event with `timestamp` older than 5 minutes; expect 400.
4. **DB constraint test**:
   - Manually attempt duplicate `(user_id, DATE(clock_in_at))` insert; expect unique violation.
5. **Integration smoke**:
   - Configure LINE dev console webhook to `https://your-domain.com/webhook/line`, send a test event from LINE; confirm 200 and one attendance row + one idempotency row.
6. **Log check**:
   - Ensure `line_webhook_deliveries` captures payload and processing time for audit.
