## v0.1 scope
v0.1 ships in 2 weeks: single deterministic workflow (research -> signal -> risk check -> compliance guardrail -> order stub) with replay capability; Postgres (Supabase) for immutable audit and state; CF Workers + Queues for orchestration; Redis for idempotency and hot cache; mock broker/OMS connectors plus one real broker sandbox connector; pre-trade guardrails (allow/block/escalate) and immutable audit trail; CLI to trigger runs and replay by inputs_hash; dashboards for latency and guardrail outcomes; no frontend — API-first for integrators.

## Open questions
- Which broker/OMS/EMS APIs and auth models will we support first (REST vs FIX vs WebSocket)?
- What deterministic serialization format will we enforce for replay (canonical JSON vs msgpack vs protobuf)?
- What latency SLA (p50/p99) from signal ingestion to order gateway is required for v0.1?
- Which compliance guardrail DSL or engine will we adopt (Rego, custom safe eval, or declarative JSON rules)?
- How will we handle idempotency and exactly-once semantics across retries in CF Workers?
- What retention and encryption posture is required for audit entries (CF R2 vs Supabase storage)?
