# workio / ops

## 1) Diagnosis

- Missing **idempotency key** on `/punches` POST → repeated LINE webhook deliveries or frontend retries create duplicate punch records.
- No **DB uniqueness constraint** enforcing “one open punch per user” (no `clock_out_at`); allows multiple concurrent clock-ins or overlapping open punches.
- Punch write path is **non-transactional**; race between existence check + insert permits duplicates under concurrent webhook retries.
- No **operational alerting** for duplicate-punch anomalies or webhook processing failures (silent data drift).
- Missing **index** for fast open-punch lookups; full scans on every clock-in/out increase latency and lock contention.

## 2) Proposed change

File scope:
- `workio/server/src/db/schema.sql` — add constraint + index.
- `workio/server/src/routes/punches.ts` — add idempotency table and transactional upsert with `idempotency_key`.
- `workio/server/src/middleware/idempotency.ts` — new middleware to enforce idempotency (insert-or-skip on conflict).
- `workio/server/src/db/migrations/20260503_add_punch_idempotency.sql` — migration to add tables/columns safely.

## 3) Implementation

### Step 1 — DB schema + migration

```sql
-- workio/server/src/db/migrations/20260503_add_punch_idempotency.sql
BEGIN;

-- Idempotency keys for punch writes (LINE webhook dedup)
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  punch_id        INTEGER NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- Ensure one open punch per user (clock_out_at IS NULL)
-- Drop existing duplicates first (keeps earliest)
DELETE FROM punches a
WHERE EXISTS (
  SELECT 1 FROM punches b
  WHERE b.user_id = a.user_id
    AND b.clock_out_at IS NULL
    AND (b.id < a.id OR b.id = a.id)
)
AND clock_out_at IS NULL
AND id NOT IN (
  SELECT MIN(id) FROM punches WHERE clock_out_at IS NULL GROUP BY user_id
);

-- Constraint: at most one NULL clock_out_at per user
CREATE UNIQUE INDEX CONCURRENTLY IF NOT EXISTS idx_punches_one_open
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- Speed up lookups for active punches
CREATE INDEX IF NOT EXISTS idx_punches_user_open
ON punches (user_id, clock_out_at)
WHERE clock_out_at IS NULL;

COMMIT;
```

### Step 2 — Idempotency middleware

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { db } from '../db';

export async function withIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const key = req.headers['idempotency-key'] as string;
  if (!key) {
    return res.status(400).json({ error: 'idempotency-key header required' });
  }

  // Try to claim key; if exists, return previous response (stored minimally)
  const { rows } = await db.query(
    `INSERT INTO punch_idempotency (idempotency_key, punch_id, created_at)
     VALUES ($1, -1, NOW())
     ON CONFLICT (idempotency_key) DO UPDATE
     SET created_at = punch_idemony.punch_idemony.created_at
     RETURNING punch_id`,
    [key]
  );

  // If punch_id !== -1, request was already processed
  const existing = rows[0];
  if (existing.punch_id !== -1) {
    // Fetch original punch to return consistent body
    const { rows: punches } = await db.query(
      'SELECT * FROM punches WHERE id = $1',
      [existing.punch_id]
    );
    return res.json(punches[0] || { id: existing.punch_id });
  }

  // Attach key to request for later storage with punch
  (req as any).idempotencyKey = key;
  next();
}
```

### Step 3 — Transactional punch write

```ts
// workio/server/src/routes/punches.ts  (excerpt)
import express from 'express';
import { db } from '../db';
import { withIdempotency } from '../middleware/idempotency';

const router = express.Router();

router.post('/', withIdempotency, async (req, res) => {
  const { user_id, type, latitude, longitude } = req.body;
  const idempotencyKey = (req as any).idempotencyKey;

  const client = await db.connect();
  try {
    await client.query('BEGIN');

    // Enforce one open punch per user at DB level (constraint will raise)
    const { rows } = await client.query(
      `INSERT INTO punches (user_id, type, clock_in_at, latitude, longitude)
       VALUES ($1, $2, NOW(), $3, $4)
       RETURNING *`,
      [user_id, type, latitude, longitude]
    );

    const punch = rows[0];

    // Store idempotency mapping
    await client.query(
      `UPDATE punch_idempotency SET punch_id = $1 WHERE idempotency_key = $2`,
      [punch.id, idempotencyKey]
    );

    await client.query('COMMIT');
    return res.status(201).json(punch);
  } catch (err: any) {
    await client.query('ROLLBACK');

    // Unique violation on idx_punches_one_open -> duplicate open punch
    if (err.code === '23505') {
      // Return existing open punch
      const { rows } = await client.query(
        'SELECT * FROM punches WHERE user_id = $1 AND clock_out_at IS NULL',
        [user_id]
      );
      return res.status(409).json(rows[0] || { error: 'duplicate_open_punch' });
    }

    console.error('Punch write failed', err);
    return res.status(500).json({ error: 'internal' });
  } finally {
    client.release();
  }
});
```
