## Deployment

## v0.1 scope
v0.1 ships a cross-platform CLI client (Linux/Windows/macOS) with: local SOCKS5 proxy hidden via HTTPS meek transport, TUN interface obfuscated by userspace WireGuard (boringtun), client-side killswitch that blocks all non-stealth traffic, local SQLite for session state, and opt-in telemetry to Supabase via Cloudflare Workers (WAU-ZB metric). No server management required; uses free-tier Supabase + CF Workers KV for blocklists and config.

## Open questions
- Which DPI-evasion primitives are safest/most effective against current probes in target regions (meek vs. Shadowed-TLS vs. domain-front)?
- Can we implement a reliable client-side killswitch across desktop platforms without kernel extensions?
- How will we distribute blocklist and config updates to offline clients while preserving integrity and deniability?
- What minimal telemetry schema satisfies WAU-ZB without risking user deanonymization?
- Do we need to support fallback transports (e.g., HTTP/3 over Cloudflare) when primary stealth path fails?
