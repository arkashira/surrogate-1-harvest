# vanguard / design

## 1. Diagnosis

- No canonical discovery entrypoint exists to surface top-hub insights (e.g., MOC) before planning — violates established `#knowledge-rag #graph #hub` pattern.
- Missing CDN-bypass file-list strategy for HF datasets; any future training ingestion will immediately hit 429 rate limits instead of using `resolve/main/` CDN paths.
- No Lightning Studio reuse guard — each run risks quota waste via recreation instead of reusing running studios.
- No executable discovery entrypoint to prepare context and file lists before planning work.
- No local data projection pipeline to enforce `{prompt, response}` schema and filename-based attribution for `batches/mirror-merged/{date}/{slug}.parquet`.

## 2. Proposed change

Create `/opt/axentx/vanguard/discovery/run_discovery.sh` as the single-pass discovery entrypoint that:
- Queries knowledge-rag for top-hub (MOC) and related docs.
- Accepts optional HF dataset repo + date folder and emits `file-list.json` for CDN-bypass training.
- Checks for a running Lightning Studio and reuses it (or reports name/id).
- Projects local parquet files under `data/raw/` into `data/enriched/batches/mirror-merged/{date}/` with `{prompt,response}` schema and slug-based filenames.

Scope: one new executable script + optional minimal Python helper for parquet projection.

## 3. Implementation

```bash
#!/usr/bin/env bash
# /opt/axentx/vanguard/discovery/run_discovery.sh
# Purpose: Single-pass discovery — surface top-hub knowledge, prepare HF CDN-bypass file list,
#          reuse running Lightning Studio, project local data.
# Usage:
#   ./run_discovery.sh
#   ./run_discovery.sh --hf-repo <datasets/repo> --date-folder 2026-05-02
#   ./run_discovery.sh --project-local ./data/raw

set -euo pipefail
SHELL=/bin/bash

BASE_DIR="/opt/axentx/vanguard"
OUT_DIR="${BASE_DIR}/discovery/output"
HF_REPO=""
DATE_FOLDER=""
PROJECT_LOCAL=""

log() { echo "[$(date -u +"%Y-%m-%dT%H:%M:%SZ")] $*"; }

while [[ $# -gt 0 ]]; do
  case "$1" in
    --hf-repo) HF_REPO="$2"; shift 2 ;;
    --date-folder) DATE_FOLDER="$2"; shift 2 ;;
    --project-local) PROJECT_LOCAL="$2"; shift 2 ;;
    --help|-h) echo "Usage: $0 [--hf-repo repo] [--date-folder YYYY-MM-DD] [--project-local path]"; exit 0 ;;
    *) echo "Unknown option: $1"; exit 1 ;;
  esac
done

mkdir -p "${OUT_DIR}"

# 1) Knowledge-rag: top-hub and related docs
log "Running knowledge-rag to surface top-hub insights..."
if command -v knowledge-rag &>/dev/null; then
  knowledge-rag --top-hub --related > "${OUT_DIR}/top-hub-insights.txt" 2>&1 || true
  log "Top-hub insights saved to ${OUT_DIR}/top-hub-insights.txt"
else
  log "knowledge-rag not found; skipping top-hub query."
fi

# 2) HF CDN-bypass file list (single API call, then CDN-only)
if [[ -n "${HF_REPO}" && -n "${DATE_FOLDER}" ]]; then
  log "Preparing HF CDN-bypass file list for ${HF_REPO}/${DATE_FOLDER}..."
  python3 - "${HF_REPO}" "${DATE_FOLDER}" "${OUT_DIR}/file-list.json" <<'PY'
import json, sys, os
from huggingface_hub import list_repo_tree

repo = sys.argv[1]          # e.g. datasets/mycorp/xyz
date_folder = sys.argv[2]   # e.g. 2026-05-02
out_path = sys.argv[3]

# Non-recursive single-level call to avoid pagination explosion
tree = list_repo_tree(repo=repo, path=date_folder, recursive=False)
files = [f.rfilename for f in tree if f.type == "file"]

# Save list for training script to embed (CDN-only fetches via resolve/main/)
os.makedirs(os.path.dirname(out_path), exist_ok=True)
with open(out_path, "w") as f:
    json.dump({"repo": repo, "date_folder": date_folder, "files": files}, f, indent=2)
print(f"Wrote {len(files)} files to {out_path}")
PY
  log "File list saved to ${OUT_DIR}/file-list.json"
  log "Training script should embed this list and fetch via CDN:"
  log "  https://huggingface.co/datasets/${HF_REPO}/resolve/main/${DATE_FOLDER}/{file}"
fi

# 3) Lightning Studio reuse check
log "Checking for running Lightning Studio..."
python3 - "${OUT_DIR}/studio_info.json" <<'PY'
import json, os, sys
out_path = sys.argv[1]

try:
    from lightning_sdk import Teamspace, Machine
    team = Teamspace.current()
    running = [s for s in team.studios if s.status == "Running"]
    info = {
        "running_count": len(running),
        "studios": [{"name": s.name, "id": s.id, "status": s.status, "machine": s.machine} for s in running]
    }
    with open(out_path, "w") as f:
        json.dump(info, f, indent=2)
    print(f"Found {len(running)} running studio(s). Info saved to {out_path}")
except Exception as e:
    with open(out_path, "w") as f:
        json.dump({"error": str(e)}, f, indent=2)
    print(f"Lightning SDK unavailable or error: {e}")
PY

# 4) Project local data to surrogate-1 schema (if requested)
if [[ -n "${PROJECT_LOCAL}" && -d "${PROJECT_LOCAL}" ]]; then
  log "Projecting local data from ${PROJECT_LOCAL} into enriched schema..."
  python3 - "${PROJECT_LOCAL}" "${BASE_DIR}/data/enriched/batches/mirror-merged" <<'PY'
import pandas as pd, os, sys, uuid, datetime, pyarrow as pa, pyarrow.parquet as pq

raw_dir = sys.argv[1]
enriched_base = sys.argv[2]
date_slug = datetime.datetime.utcnow().strftime("%Y-%m-%d")
out_dir = os.path.join(enriched_base, date_slug)
os.makedirs(out_dir, exist_ok=True)

for fname in os.listdir(raw_dir):
    if not fname.lower().endswith(".parquet"):
        continue
    path = os.path.join(raw_dir, fname)
    try:
        df = pd.read_parquet(path)
    except Exception as e:
        print(f"Skipping {fname}: {e}")
        continue

    # Heuristic projection to {prompt, response}
    prompt_col = next((c for c in df.columns if "prompt" in c.lower()), None)
    response_col = next((c for c in df.columns if "response" in c.lower() or "completion" in c.lower()), None)

    rows = []
    for _, row in df.iterrows():
        prompt = str(row[prompt_col]) if prompt_col and pd.notna(row.get(prompt_col)) else ""
        response = str(row[response_col]) if response_col and pd.notna(row.get(response_col)) else ""
        if prompt.strip() or response.strip():
            rows.append({"prompt": prompt.strip(), "response": response.strip()})

    if not rows:
        print(f"No valid rows in {fname}")
        continue

    out_df = pd.DataFrame(rows)
    slug = f"{os.path.splitext(fname)[0]}_{str(uuid.uuid4())[:8]}"
    out_file = os.path.join(out_dir, f"{slug}.parquet")
    # Ensure schema {prompt: string, response: string}
    table = pa.Table.from_pandas(out_df, schema=pa.schema([
        pa.field("prompt", pa.string()),
        pa.field("response", pa.string())
    ]))
    pq.write_table(table, out_file)
    print(f"Projected {len(rows)} rows -> {out_file}")
PY
  log "Local projection complete -> ${BASE_DIR}/data/enriched/batches/mirror-merged/${date_slug}/"
fi

log "Discovery complete. Outputs in ${OUT_DIR}/"
```
