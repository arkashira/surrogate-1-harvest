## Deployment
hybrid (local-first CLI; optional HF Space for public demo + CF Workers for lightweight auth/telemetry opt-in)

## v0.1 scope
v0.1 ships in 2 weeks: CLI tool that indexes a local repo (Python/JS/TS), runs tree-sitter-based structural analysis plus a small local transformer model for semantic findings, stores results in SQLite, and outputs prioritized findings (severity, line-level, suggestions) to terminal; includes --json export and basic ignore/config file support.

## Open questions
- Which local models balance speed vs. accuracy on CPU for large repos?
- How will we handle incremental analysis and caching to keep v0.1 fast?
- What telemetry (if any) is acceptable to collect for NSM without compromising privacy?
- Will we support optional encrypted upload to Supabase for cross-device history (opt-in)?
- How do we define and measure 'paid customer' for an OSS/local-first tool?
