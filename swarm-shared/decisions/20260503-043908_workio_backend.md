# workio / backend

## Final Synthesis (Corrected + Actionable)

**Core problem**: The LINE attendance webhook is not safe against replays, spoofing, or races. This causes duplicate punches, phantom attendance, and unreliable notifications.

**Guiding principles**:
- Verify authenticity first (`X-Line-Signature`).
- Make ingestion idempotent with a bounded replay window (24–72h).
- Use atomic upserts and proper constraints to prevent duplicates.
- Return the correct HTTP status to LINE (200 for success, 5xx only when LINE should retry).
- Keep observability and cleanup simple and operational.

## 1. DB changes (corrected and hardened)

```sql
-- server/src/db/schema.sql

-- Idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  webhook_event_id VARCHAR(255) PRIMARY KEY,
  received_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  processed_at TIMESTAMPTZ,
  tenant_id INT,
  user_id INT,
  event_type VARCHAR(50)
);

-- Index for TTL cleanup and lookups
CREATE INDEX IF NOT EXISTS idx_line_webhook_idempotency_received_at
  ON line_webhook_idempotency (received_at);

-- Ensure clock_events uniqueness per user per clock_in (safe if constraint missing)
DO $$
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint
    WHERE conname = 'uk_clock_events_user_clock_in'
      AND conrelid = 'clock_events'::regclass
  ) THEN
    ALTER TABLE clock_events
      ADD CONSTRAINT uk_clock_events_user_clock_in
      UNIQUE (user_id, clock_in)
      DEFERRABLE INITIALLY IMMEDIATE;
  END IF;
END $$;
```

**Why this is correct**:
- `webhook_event_id` is the natural, LINE-provided idempotency key.
- A 24h TTL is sufficient for LINE retries while bounding table growth.
- The unique constraint on `(user_id, clock_in)` is the last line of defense against duplicates.

## 2. Signature verification (fixed)

```typescript
// server/src/middleware/lineAuth.ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';

const CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET;
if (!CHANNEL_SECRET) {
  throw new Error('LINE_CHANNEL_SECRET is required');
}

export function verifyLineSignature(req: Request, res: Response, next: NextFunction) {
  const signature = req.get('X-Line-Signature');
  if (!signature) {
    return res.status(400).json({ error: 'Missing X-Line-Signature' });
  }

  // Use raw body for HMAC; ensure bodyParser raw middleware runs before this.
  const body = req.body;
  if (!Buffer.isBuffer(body)) {
    return res.status(500).json({ error: 'Invalid body encoding for signature' });
  }

  const expected = crypto
    .createHmac('sha256', CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');

  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected))) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // Re-parse JSON for downstream handlers
  try {
    req.body = JSON.parse(body.toString('utf8'));
    next();
  } catch {
    res.status(400).json({ error: 'Invalid JSON payload' });
  }
}
```

**Critical fix**: Use the raw body for HMAC (not `JSON.stringify`), which is what LINE signs. Configure your body parser to buffer the raw body for this route.

## 3. Idempotent webhook handler (atomic, correct)

```typescript
// server/src/routes/line.ts
import { Router } from 'express';
import { pool } from '../db';
import { verifyLineSignature } from '../middlelineAuth';

const router = Router();

router.post('/webhook/line', verifyLineSignature, async (req, res) => {
  const events = req.body?.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(200).json({ ok: true });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      const webhookEventId = ev?.webhookEventId;
      const sourceType = ev?.source?.type;
      const userId = ev?.source?.userId;
      const eventType = ev?.type;

      if (!webhookEventId || !userId || !eventType) continue;

      // Idempotency check (24h window)
      const dup = await client.query(
        `SELECT 1 FROM line_webhook_idempotency
         WHERE webhook_event_id = $1 AND received_at > NOW() - INTERVAL '24 hours'`,
        [webhookEventId]
      );

      if (dup.rows.length > 0) continue;

      // Record idempotency first
      await client.query(
        `INSERT INTO line_webhook_idempotency
          (webhook_event_id, tenant_id, user_id, event_type)
         VALUES ($1, $2, $3, $4)`,
        [webhookEventId, null, userId, eventType]
      );

      // Handle clock in/out via text message
      if (eventType === 'message' && ev.message?.type === 'text') {
        const text = ev.message.text.trim().toLowerCase();
        const isClockIn = /(เข้างาน|clock in|\bin\b)/i.test(text);
        const isClockOut = /(เลิกงาน|clock out|\bout\b)/i.test(text);

        if (isClockIn || isClockOut) {
          const now = new Date();

          if (isClockIn) {
            // Insert only if no existing clock_in for this exact timestamp (unique constraint protects)
            await client.query(
              `INSERT INTO clock_events (user_id, clock_in, clock_out, created_at, updated_at)
               VALUES ($1, $2, NULL, $3, $3)
               ON CONFLICT (user_id, clock_in) DO NOTHING`,
              [userId, now, now]
            );
          } else if (isClockOut) {
            // Atomically close the latest open punch
            await client.query(
              `UPDATE clock_events
               SET clock_out = $1, updated_at = $1
               WHERE id = (
                 SELECT id FROM clock_events
                 WHERE user_id = $2 AND clock_out IS NULL
                 ORDER BY clock_in DESC
                 LIMIT 1
               )`,
              [now, userId]
            );
          }
        }
      }

      // Mark processed
      await client.query(
        `UPDATE line_webhook_idempotency
         SET processed_at = NOW()
         WHERE webhook_event_id = $1`,
        [webhookEventId]
      );
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook processing error:', err);
    // 500 tells LINE to retry (idempotency prevents duplicates)
    res.status(500).json({ error: 'processing failed' });
  } finally {
    client.release();
  }
});

export default router;
```

**Key correctness choices**:
- Idempotency insert happens before any state change.
- All per-event work is in one transaction to avoid partial application.
- Return 200 only after commit; return 500 on failure so LINE retries.
- `ON CONFLICT DO NOTHING` plus the unique constraint guarantees no duplicate punches.
