# workio / design

## Final Synthesis — Correct, Actionable Solution

**Core problems (unified):**
- LINE retries and client double‑taps create duplicate attendance rows.
- No server‑side idempotency key (`webhookEventId`) stored or checked.
- No state‑machine enforcement for clock‑in/out transitions.
- No immutable event log for audit/replay; daily totals are derived unreliably.

**Chosen approach (resolve contradictions in favor of correctness + concurrency safety):**
- Use a **dedicated idempotency table** keyed by LINE `webhookEventId` with a unique constraint and **PostgreSQL upsert** (`ON CONFLICT DO NOTHING`) inside a single transaction.
- Add an **immutable `attendance_events` table** as the source of truth for transitions.
- Enforce a **strict state machine** (no two consecutive clock‑ins or clock‑outs).
- Materialize daily summaries reliably from events (or maintain via transactional upsert).
- Keep all writes in **one DB transaction** per webhook to prevent races under concurrency.

-- Immutable event log for attendance transitions (one row per valid clock-in/out)
CREATE TABLE IF NOT EXISTS attendance_events (
  id              BIGSERIAL PRIMARY KEY,
  tenant_id       BIGINT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  employee_id     BIGINT NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
  event_type      TEXT NOT NULL CHECK (event_type IN ('clock_in', 'clock_out')),
  event_at        TIMESTAMPTZ NOT NULL,
  latitude        DOUBLE PRECISION,
  longitude       DOUBLE PRECISION,
  line_user_id    TEXT,
  raw_payload     JSONB,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Idempotency table for LINE webhook events (prevents duplicate processing)
CREATE TABLE IF NOT EXISTS webhook_events (
  event_id        TEXT PRIMARY KEY,          -- LINE webhookEvent.id
  source          TEXT NOT NULL,             -- e.g., 'line.attendance'
  tenant_id       BIGINT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  processed_payload JSONB
);

-- Indexes for fast lookups and reporting
CREATE INDEX IF NOT EXISTS idx_attendance_events_employee_at ON attendance_events(employee_id, event_at DESC);
CREATE INDEX IF NOT EXISTS idx_attendance_events_tenant_at ON attendance_events(tenant_id, event_at DESC);

-- Daily summary table (materialized from events)
-- If this table already exists, align columns/indexes with below
CREATE TABLE IF NOT EXISTS attendance_daily (
  tenant_id       BIGINT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  employee_id     BIGINT NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
  day             DATE NOT NULL,
  clock_in_at     TIMESTAMPTZ,
  clock_out_at    TIMESTAMPTZ,
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (tenant_id, employee_id, day)
);

CREATE INDEX IF NOT EXISTS idx_attendance_daily_employee_day ON attendance_daily(employee_id, day);

      return { ok: false, message: 'Invalid payload' };

      [lineUserId]

      [lineUserId, tenantId]

      await client.query('ROLLBACK');
      return { ok: false, message: 'Employee record missing' };

       RETURNING event_id`,
