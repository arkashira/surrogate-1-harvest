# surrogate-1 / ops

## Highest-value incremental improvement (≤2h)

**Deterministic pre-flight snapshot + CDN-only ingestion + Studio reuse**  
- Replace runtime `list_repo_files` + `load_dataset(streaming=True)` with a single Mac-side snapshot (JSON) that lists exact file paths for one date folder.  
- Embed that snapshot in the training script so Lightning workers fetch via CDN URLs only (zero HF API calls during training).  
- Reuse a running Lightning Studio instead of recreating (saves quota, avoids idle-stop churn).  
- Keep surrogate-1-runner workers unchanged (they already use isolated runners + shard-by-hash) — only the training side changes.

---

## Concrete implementation plan

1. **Mac orchestration script** (`bin/make_snapshot.py`)  
   - Runs on Mac after rate-limit window clears.  
   - Calls `list_repo_tree(path=date_folder, recursive=False)` once.  
   - Produces `snapshot/{date}/filelist.json` with `{"date": "...", "files": ["file1.parquet", ...], "created_utc": ...}`.  
   - Commits snapshot to this repo (or uploads to dataset repo `snapshots/`) so training can fetch it via CDN.

2. **Training script changes** (`train.py`)  
   - Accept `--snapshot-url` (CDN URL to `filelist.json`).  
   - Load snapshot; build list of CDN URLs:  
     `f"https://huggingface.co/datasets/{repo}/resolve/main/{date}/{f}"`  
   - Use a lightweight streaming loader that downloads each parquet via `requests`/`urllib` (or `hf_hub_download` with `cache_dir` but no API listing) and projects to `{prompt, response}` on the fly.  
   - Skip `load_dataset(streaming=True)` on heterogeneous schemas.  
   - Deterministic shuffle by seeding per file list.

3. **Lightning Studio reuse wrapper** (`bin/launch_studio.py`)  
   - List `Teamspace.studios`; if a studio with target name is `Running`, reuse it.  
   - If `Stopped`, restart with `target.start(machine=Machine.L40S)` (or fallback to free-tier L40S).  
   - Run training via `studio.run(command=[...], machine=...)` with the snapshot URL passed as an environment variable or CLI arg.

4. **Crontab / workflow hygiene**  
   - Ensure `SHELL=/bin/bash` in any crontab entries.  
   - All wrapper scripts use `#!/usr/bin/env bash` and `chmod +x`.

## Minimal code snippets

### bin/make_snapshot.py
```python
#!/usr/bin/env python3
"""
Create deterministic snapshot for one date folder.
Usage:
  HF_TOKEN=... python bin/make_snapshot.py 2026-05-01
"""
import json, os, sys, datetime, pathlib
from huggingface_hub import HfApi

API = HfApi(token=os.environ["HF_TOKEN"])
REPO = "axentx/surrogate-1-training-pairs"

def main(date_str: str):
    out_dir = pathlib.Path("snapshot") / date_str
    out_dir.mkdir(parents=True, exist_ok=True)
    files = API.list_repo_tree(repo_id=REPO, path=date_str, recursive=False)
    file_names = [f.rfilename for f in files if f.rfilename.endswith(".parquet")]
    snapshot = {
        "date": date_str,
        "repo": REPO,
        "created_utc": datetime.datetime.utcnow().isoformat() + "Z",
        "files": sorted(file_names),
    }
    out_path = out_dir / "filelist.json"
    out_path.write_text(json.dumps(snapshot, indent=2))
    print(json.dumps(snapshot))

if __name__ == "__main__":
    if len(sys.argv) != 2:
        print("Usage: make_snapshot.py YYYY-MM-DD")
        sys.exit(1)
    main(sys.argv[1])
```

### train.py (excerpt)
```python
import json, os, random, requests, pyarrow.parquet as pq
from io import BytesIO
from torch.utils.data import IterableDataset

HF_CDN = "https://huggingface.co/datasets"

class CDNParquetStream(IterableDataset):
    def __init__(self, snapshot_url, seed=42):
        self.snapshot_url = snapshot_url
        self.seed = seed
        self._load_snapshot()

    def _load_snapshot(self):
        data = json.loads(requests.get(self.snapshot_url, timeout=30).text)
        self.repo = data["repo"]
        self.date = data["date"]
        self.files = data["files"]
        if not self.files:
            raise ValueError("No files in snapshot")
        # deterministic client-side shuffle
        self.rng = random.Random(self.seed)
        self.rng.shuffle(self.files)

    def _cdn_url(self, fname):
        return f"{HF_CDN}/{self.repo}/resolve/main/{self.date}/{fname}"

    def __iter__(self):
        worker_info = None
        # optional: split per worker via torch.utils.data.get_worker_info()
        for fname in self.files:
            url = self._cdn_url(fname)
            resp = requests.get(url, timeout=60)
            resp.raise_for_status()
            table = pq.read_table(BytesIO(resp.content))
            # project to {prompt, response} only
            df = table.select(["prompt", "response"]).to_pandas()
            for _, row in df.iterrows():
                yield {"prompt": row["prompt"], "response": row["response"]}
```

### bin/launch_studio.py
```bash
#!/usr/bin/env bash
set -euo pipefail
# Reuse or start Lightning Studio, then run training with snapshot.

export SHELL=/bin/bash
STUDIO_NAME="surrogate-1-train"
DATE="${1:-$(date -u +%Y-%m-%d)}"
SNAPSHOT_URL="https://huggingface.co/datasets/axentx/surrogate-1-training-pairs/resolve/main/snapshots/${DATE}/filelist.json"

python - <<PY
import lightning as L
from lightning.fabric.plugins import LightningCLI

teamspace = L.Teamspace()
studio = None
for s in teamspace.studios:
    if s.name == "$STUDIO_NAME":
        studio = s
        break

if studio is None:
    from lightning.fabric.accelerators import Machine
    studio = teamspace.create_studio(
        name="$STUDIO_NAME",
        machine=Machine.L40S,
        start_ok=True,
        create_ok=True,
    )
else:
    if studio.status == "stopped":
        studio.start(machine=Machine.L40S)
    elif studio.status != "running":
        raise RuntimeError(f"Studio status: {studio.status}")

# Run training (adjust command to your entrypoint)
studio.run(
    command=[
        "python", "train.py",
        "--snapshot-url", "$SNAPSHOT_URL",
        "--output-dir", "outputs/${DATE}"
    ],
    machine=Machine.L40S,
)
PY
```
