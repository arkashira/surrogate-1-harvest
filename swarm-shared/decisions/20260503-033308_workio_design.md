# workio / design

## Final Synthesized Implementation (Correct + Actionable)

I merged the strongest, non-redundant parts from both proposals and resolved conflicts in favor of correctness, security, and concrete actionability.

### Key decisions (conflict resolutions)
- **Raw body handling**: use a targeted `verify-line-signature` utility that accepts a raw Buffer and a secret, and **do not globally replace `express.json()`**. Instead, apply `express.raw({ type: 'application/json' })` only on `/webhook/line` so `req.body` is a Buffer for HMAC and JSON parsing is explicit and safe.
- **Signature verification timing**: verify **before any business logic or DB writes**, and use constant-time comparison.
- **Idempotency**: enforce at write time with a DB unique constraint `(idempotency_key, tenant_id)` and an atomic `INSERT ... ON CONFLICT DO NOTHING` to prevent races.
- **Tenant binding**: resolve tenant from the event payload via a cached mapping (LINE userId → tenantId) and include tenant_id in the idempotency key. Reject events that cannot be mapped.
- **Schema**: add `idempotency_key` and `tenant_id` to events (or the target entity) with a unique index.
- **Delivery contract**: return `200 OK` only after successful, non-duplicate processing; return `400/401/409` for verification/idempotency failures so LINE can safely retry or stop.

### 2. Signature verification utility (correct + testable)

```ts
// workio/server/src/utils/lineSignature.ts
import crypto from 'crypto';

export function verifyLineSignature(rawBody: Buffer, signatureHeader: string, channelSecret: string): boolean {
  if (!signatureHeader || typeof signatureHeader !== 'string') return false;

  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody)
    .digest('hex');

  // LINE sends: sha256=<hex>
  const expectedHeader = `sha256=${expected}`;
  return crypto.timingSafeEqual(Buffer.from(expectedHeader), Buffer.from(signatureHeader));
}
```

### 3. Idempotent event service (atomic upsert)

```ts
// workio/server/src/services/lineService.ts
import { Pool } from 'pg';
import { verifyLineSignature } from '../utils/lineSignature.js';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function handleLineEvent(rawBody: Buffer, signature: string, body: any) {
  const channelSecret = process.env.LINE_CHANNEL_SECRET;
  if (!channelSecret) throw new Error('LINE_CHANNEL_SECRET not configured');

  // 1) Verify signature first
  if (!verifyLineSignature(rawBody, signature, channelSecret)) {
    const err: any = new Error('Invalid LINE signature');
    err.code = 'INVALID_SIGNATURE';
    throw err;
  }

  // 2) Normalize event and idempotency key
  const event = body.events?.[0] || body;
  const idempotencyKey = event.message?.id || body.idempotencyKey;
  if (!idempotencyKey) {
    const err: any = new Error('Missing idempotency key');
    err.code = 'MISSING_IDEMPOTENCY_KEY';
    throw err;
  }

  // 3) Resolve tenant (cached mapping preferred in production)
  const tenantId = await resolveTenantBySource(event.source);
  if (!tenantId) {
    const err: any = new Error('Tenant not found for source');
    err.code = 'TENANT_NOT_FOUND';
    throw err;
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 4) Atomic insert with idempotency guard
    const insertSQL = `
      INSERT INTO events (tenant_id, idempotency_key, type, user_id, payload, created_at)
      VALUES ($1, $2, $3, $4, $5, NOW())
      ON CONFLICT (idempotency_key, tenant_id) DO NOTHING
      RETURNING id
    `;
    const insertValues = [tenantId, idempotencyKey, event.type, event.source?.userId, event];
    const res = await client.query(insertSQL, insertValues);

    await client.query('COMMIT');

    if (res.rowCount === 0) {
      return { status: 'duplicate', idempotencyKey, tenantId };
    }
    return { status: 'processed', id: res.rows[0].id, idempotencyKey, tenantId };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}

async function resolveTenantBySource(source: any): Promise<string | null> {
  if (!source?.userId) return null;
  const res = await pool.query(
    `SELECT tenant_id FROM user_mappings WHERE line_user_id = $1`,
    [source.userId]
  );
  return res.rows[0]?.tenant_id || null;
}
```

### 4. Webhook route (Express) — raw-body + verification + idempotency

```ts
// workio/server/src/routes/webhook/line.ts
import express from 'express';
import { handleLineEvent } from '../services/lineService.js';

const router = express.Router();

// Use express.raw only for this route so req.body is the raw Buffer for HMAC
router.post(
  '/line',
  express.raw({ type: 'application/json' }),
  async (req, res) => {
    const signature = req.headers['x-line-signature'] as string;
    const rawBody = Buffer.isBuffer(req.body) ? req.body : Buffer.from(req.body);

    // Parse JSON from rawBody after HMAC-friendly capture
    let body;
    try {
      body = JSON.parse(rawBody.toString('utf8'));
    } catch (err) {
      return res.status(400).json({ error: 'Invalid JSON' });
    }

    try {
      const result = await handleLineEvent(rawBody, signature, body);
      // Return 200 only after successful, non-duplicate processing
      return res.json(result);
    } catch (err: any) {
      console.error('[LINE webhook]', err);

      // Idempotent duplicates are not client errors, but we return 200 or 409 depending on policy
      if (err.code === 'INVALID_SIGNATURE') return res.status(401).json({ error: err.message });
      if (err.code === 'MISSING_IDEMPOTENCY_KEY') return res.status(400).json({ error: err.message });
      if (err.code === 'TENANT_NOT_FOUND') return res.status(400).json({ error: err.message });

      // For unexpected errors, return 500 so LINE may retry (if retryable) or alert ops
      return res.status(500).json({ error: 'Internal server error' });
    }
  }
);

export default router;
```

### 5. Verification checklist (actionable)

   - Start server. POST to `/webhook/line` with valid `X-Line-Signature` and known secret → expect `200` and `{status:"processed"}`.
