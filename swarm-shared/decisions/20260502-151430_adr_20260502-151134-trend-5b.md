## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research/signal -> risk check -> simulated execution) with immutable audit trail, Postgres-backed workflow_run/event store, CF Workers task queue for step dispatch, Redis cache for idempotency, compliance guardrails as hard blocks (pass/fail), API endpoints to submit runs and replay by id, and contract tests for broker/OMS adapter interface (no live trading; simulated execution only).

## Open questions
- Which broker/OMS/EMS APIs are must-connect for v0.1 and what auth schemes do they require (OAuth2, API keys, mTLS)?
- What deterministic replay SLA is required (exact input snapshot vs. market data snapshot volatility)?
- Which compliance rules are hard blocks vs. soft warnings (e.g., pre-trade limit checks vs. post-trade surveillance)?
- How will market data timestamps and clock sync be handled to guarantee sub-second latency claims (NTP/PTP, exchange timestamps)?
- What retention and encryption-at-rest requirements apply to audit_event payloads for regulatory hold?
- How should idempotency be scoped (per workflow run, per step, per external execution attempt)?
