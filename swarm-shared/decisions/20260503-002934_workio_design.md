# workio / design

## Final consolidated solution

### 1) Diagnosis (merged, prioritized)

- **No idempotency on `/punches` writes** — repeated LINE webhook or frontend retries create duplicate punch records.
- **Missing DB-level uniqueness for “one open punch per user”** — allows multiple concurrent clock-ins and corrupts time-tracking invariants.
- **No server-side punch state machine** — transitions (clock-in → clock-out) are ad-hoc; no validation of current state before mutation.
- **Schema/tenant isolation risk** — tenant scoping relies on app-layer checks; no composite uniqueness/index to enforce `(tenant_id, user_id, state)` invariants at storage level.
- **Punch edits/deletes lack audit trail** — compliance and dispute resolution rely on trust.
- **Operational observability gap** — no request trace or idempotency-key logging; hard to debug duplicates in production.
- **Validation is client-side only** — trivial to spoof or bypass (GPS/time).

### 2) Proposed change (merged)

Files:
- `workio/server/src/db/schema.sql` — add idempotency column, one-open-punch partial unique index, and audit table.
- `workio/server/src/db/migrations/` — migration for the above (if migration framework is used; otherwise apply schema directly).
- `workio/server/src/routes/punches.ts` — add idempotency check, server-side state-machine validation, and audit on updates/deletes.
- `workio/server/src/middleware/idempotency.ts` — middleware to enforce idempotency key (create-if-absent) and return idempotent responses.

### 3) Implementation

#### 3.1 DB schema (`schema.sql`)

```sql
-- Idempotency and one-open-punch enforcement
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255),
  ADD COLUMN IF NOT EXISTS idempotency_key_tenant_id INTEGER;

-- One open punch per user (tenant + user)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
  ON punches (tenant_id, user_id)
  WHERE status = 'in';

-- Idempotency uniqueness per tenant+key (for punch mutations)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency
  ON punches (tenant_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- Audit trail for punch edits/deletes
CREATE TABLE IF NOT EXISTS punch_audits (
  id              BIGSERIAL PRIMARY KEY,
  punch_id        BIGINT      NOT NULL,
  tenant_id       INTEGER     NOT NULL,
  user_id         INTEGER     NOT NULL,
  action          VARCHAR(16) NOT NULL, -- 'create','update','delete'
  old_status      VARCHAR(16),
  new_status      VARCHAR(16),
  old_clock_in    TIMESTAMPTZ,
  new_clock_in    TIMESTAMPTZ,
  old_clock_out   TIMESTAMPTZ,
  new_clock_out   TIMESTAMPTZ,
  changed_by      INTEGER,
  changed_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  idempotency_key VARCHAR(255),
  trace_id        VARCHAR(255)
);

-- Indexes for audit queries
CREATE INDEX IF NOT EXISTS idx_punch_audits_punch_id ON punch_audits (punch_id);
CREATE INDEX IF NOT EXISTS idx_punch_audits_tenant_id ON punch_audits (tenant_id);
```

#### 3.2 Idempotency middleware (`middleware/idempotency.ts`)

```ts
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';

export async function enforcePunchIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const tenantId = (req as any).tenant?.id;
  const idempotencyKey = req.headers['idempotency-key'] as string;
  const traceId = req.headers['x-trace-id'] as string || crypto.randomUUID();

  if (!tenantId || !idempotencyKey) {
    return res.status(400).json({ error: 'idempotency-key header required' });
  }

  (req as any).idempotencyKey = idempotencyKey;
  (req as any).traceId = traceId;

  try {
    // Try to claim idempotency placeholder (fast upsert check)
    const claim = await pool.query(
      `INSERT INTO punches (tenant_id, idempotency_key, idempotency_key_tenant_id, user_id, status, created_at, updated_at)
       VALUES ($1, $2, $1, -1, 'idempotency-placeholder', NOW(), NOW())
       ON CONFLICT (tenant_id, idempotency_key) DO NOTHING
       RETURNING id`,
      [tenantId, idempotencyKey]
    );

    if (claim.rowCount === 0) {
      // Key exists — fetch existing real punch to return idempotent response
      const existing = await pool.query(
        `SELECT id, user_id, status, clock_in_time, clock_out_time, updated_at
         FROM punches
         WHERE tenant_id = $1 AND idempotency_key = $2 AND user_id != -1`,
        [tenantId, idempotencyKey]
      );

      if (existing.rows.length > 0) {
        return res.status(200).json({
          message: 'Idempotent replay — returning existing punch',
          punch: existing.rows[0],
        });
      }

      // Placeholder exists but no real punch yet — allow continuation (race)
    }

    next();
  } catch (err) {
    next(err);
  }
}
```
