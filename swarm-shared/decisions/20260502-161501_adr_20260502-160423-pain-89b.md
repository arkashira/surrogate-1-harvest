## Deployment
cf workers

## v0.1 scope
v0.1 ships a local-only CLI client (Rust) that creates a stealth TUN interface, tunnels traffic through an obfuscated SOCKS5 path (shadowsocks), enforces a fail-closed kill-switch, stores minimal local SQLite session logs, and exposes basic WAU-ZB telemetry via anonymized periodic pings through CF Workers; no server components required.

## Open questions
- Which DPI-evasion transforms (e.g., TLS camouflage, domain fronting via CF Workers) provide the best reach vs. performance trade-offs in target regions?
- How will client-side telemetry (WAU-ZB) be collected without deanonymizing users or creating tracking surfaces?
- What is the kill-switch enforcement model (TUN drop vs. firewall rules) to guarantee no leaks across platforms (Windows/macOS/Linux)?
