# vanguard / ops

## Final Synthesis (single, correct, actionable)

**Root cause**: training and frontend hit the HF `/api/` at runtime (file enumeration + per-file metadata), causing 429s and non-reproducible epochs. There is no content-addressed manifest, no deterministic repo selection for HF commit cap, mixed-schema `enriched/` files break surrogate-1 expectations, and Lightning Studio reuse/idle-stop guards are missing.

**Goal**: enable fully reproducible, CDN-only training with zero `/api/` calls during epochs, correct schema projection, deterministic repo selection, and safe Lightning Studio reuse.

## 1. Deterministic repo selection & HF commit cap guard
- Pin a single repo and branch/tag for ingestion (e.g., `datasets/vanguard-enriched@vYYYY-MM-DD` or a commit hash).
- Enforce a deterministic “repo-of-the-day” selection function so ingestion bursts are isolated to one repo and respect HF’s 128 commits/hr/repo limit.
- Fail fast if the selected repo is over quota (check via `GET /api/` rate-limit headers) and backoff instead of retrying enumeration.

**Actionable code** (add to `ops/repo_selector.py`):
```python
import os, datetime, requests
from huggingface_hub import HfApi

HF_REPO = os.getenv("HF_REPO", "datasets/vanguard-enriched")
HF_TOKEN = os.getenv("HF_TOKEN", "")
API = HfApi(token=HF_TOKEN)

def select_repo_for_date(date: str) -> str:
    """Deterministic repo@tag for a date. Prefer existing tag; else use latest commit before date."""
    tag = f"v{date}"
    try:
        # verify tag exists and is accessible
        repo_id = f"{HF_REPO}@{tag}"
        # lightweight check: get repo info (will raise if not found/unauthorized)
        API.repo_info(repo_id=HF_REPO, repo_type="dataset", revision=tag)
        return repo_id
    except Exception:
        # fallback: use repo@main but pin by date via manifest generation time
        return f"{HF_REPO}@main"

def check_rate_limit() -> bool:
    """Return True if we are close to HF commit cap; caller should backoff."""
    if not HF_TOKEN:
        return False
    r = requests.get("https://huggingface.co/api/rate_limit", headers={"Authorization": f"Bearer {HF_TOKEN}"})
    if r.status_code == 200:
        data = r.json()
        # dataset repo writes/operations limit (approximate)
        limit = data.get("rate", {}).get("limit", 1)
        remaining = data.get("rate", {}).get("remaining", 1)
        if remaining / limit < 0.1:
            return True
    return False
```

## 2. Content-addressed manifest (CDN-only)
- One manifest per date: `manifests/manifest-YYYY-MM-DD.json`.
- Single `list_repo_tree` call (recursive=False) for the date folder.
- Each entry: `{file, size, sha256, cdn_url, columns}`.
- `sha256` is content-addressed (download once and hash, or use LFS pointer info if available). If unavailable, store `etag`/`last_modified` and revalidate on first fetch.
- Filter `enriched/` files and emit only `{prompt, response}` projection metadata.
- If tree listing fails (rate-limited), allow empty manifest + manual CDN fallback so training can still proceed with cached files.

**Actionable script** (`ops/generate_manifest.py`):
```python
#!/usr/bin/env python3
import json, os, sys, hashlib, subprocess
from pathlib import Path
from huggingface_hub import list_repo_tree, hf_hub_download

HF_REPO = os.getenv("HF_REPO", "datasets/vanguard-enriched")
DATE = sys.argv[1] if len(sys.argv) > 1 else __import__("datetime").datetime.utcnow().strftime("%Y-%m-%d")
OUTDIR = Path(sys.argv[2] if len(sys.argv) > 2 else "manifests")
OUTDIR.mkdir(parents=True, exist_ok=True)
out_path = OUTDIR / f"manifest-{DATE}.json"

prefix = f"enriched/{DATE}"
entries = []

try:
    tree = list_repo_tree(repo_id=HF_REPO, path=prefix, recursive=False)
except Exception as e:
    print(f"Warning: tree listing failed ({e}), producing empty manifest", file=sys.stderr)
    tree = []

for item in tree:
    if item.type != "file":
        continue
    path = item.path
    size = getattr(item, "size", 0)
    cdn_url = f"https://huggingface.co/datasets/{HF_REPO}/resolve/main/{path}"

    # Try to get real sha256; fall back to deterministic placeholder
    sha256 = None
    try:
        # download to temp and hash (costly but ensures content-addressing)
        tmp = hf_hub_download(repo_id=HF_REPO, filename=path, cache_dir="/tmp/hf_cache", force_download=False)
        with open(tmp, "rb") as f:
            sha256 = hashlib.sha256(f.read()).hexdigest()
    except Exception:
        # placeholder content-hash (not ideal); prefer revalidation via etag/last_modified
        sha256 = hashlib.sha256(f"{path}{size}".encode()).hexdigest()[:16]

    entries.append({
        "file": path,
        "size": size,
        "sha256": sha256,
        "cdn_url": cdn_url,
        "columns": ["prompt", "response"]
    })

manifest = {
    "date": DATE,
    "repo": HF_REPO,
    "generated_by": "generate_manifest.py",
    "entries": entries
}

with open(out_path, "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Wrote {len(entries)} entries to {out_path}")
```

## 3. Training: CDN-only streaming + schema projection
- Use manifest to build a CDN-only streaming dataset.
- Prefer `load_dataset(..., data_files=[cdn_urls], streaming=True)` for public files (no `/api/` per-file calls).
- If files are WebDataset shards, use `webdataset` + fsspec HTTP caching.
- Project to `{prompt, response}` safely; rename/select columns if names differ.
- Cache locally (e.g., `cache_dir`) to avoid repeated CDN downloads within an epoch.
