# vanguard / ops

## 1. Diagnosis

- No build-time deterministic asset manifest → frontend cannot resolve CDN URLs without runtime HF API calls (invites 429s and breaks CDN-first strategy).
- Missing mount point / entrypoint boundary between orchestration host (Mac) and compute target (Lightning Studio) — scripts likely assume local execution and will fail on remote Studio.
- No pre-listed file list embedded in training pipeline — training scripts risk HF API rate limits during data loading instead of using CDN-only fetches.
- No wrapper script hygiene (shebang, executable bit, SHELL=/bin/bash) for cron-launched ops scripts (active-learning, opus-pr-reviewer patterns) — risk of silent exec failures.
- No reuse guard for Lightning Studio — each run may create new Studio and burn 80hr/mo quota instead of reusing running instances.

## 2. Proposed change

Create a small, high-leverage ops utility that fixes the most common failure modes and enables CDN-first training:

- File: `/opt/axentx/vanguard/ops/cdn_file_list.py`
  - Single Mac-side script: `list_repo_tree` once for a date folder, emit `train_files.json`.
  - Embeddable output for training scripts to use CDN-only fetches (`https://huggingface.co/datasets/.../resolve/main/...`).
- File: `/opt/axentx/vanguard/ops/run_studio_training.py`
  - Reuse running Studio if present; else create with deterministic name.
  - Upload `train_files.json` and training script, then `.run()` only if Studio is running.
  - If stopped, restart with `target.start(machine=Machine.L40S)` before run.
- File: `/opt/axentx/vanguard/ops/wrapper_bash_stub.sh`
  - Canonical wrapper for cron-launched scripts: proper shebang, `set -euo pipefail`, `export SHELL=/bin/bash`, exec via `bash script "$@"`.

Scope: add ~3 small files under `ops/`; no changes to existing source outside orchestration boundary.

### ops/cdn_file_list.py
```python
#!/usr/bin/env python3
"""
Generate a deterministic file list for a date folder so training can use
CDN-only fetches and avoid HF API rate limits during data loading.

Usage (Mac, after rate-limit window clears):
  python3 ops/cdn_file_list.py \
    --repo datasets/axentx/surrogate-1 \
    --date 2026-04-29 \
    --out train_files.json
"""

import argparse
import json
import os
import sys

try:
    from huggingface_hub import HfApi
except ImportError:
    print("Install: pip install huggingface_hub", file=sys.stderr)
    sys.exit(1)

CDN_BASE = "https://huggingface.co/datasets"

def main() -> None:
    parser = argparse.ArgumentParser(description="List repo folder for CDN training.")
    parser.add_argument("--repo", required=True, help="e.g. datasets/axentx/surrogate-1")
    parser.add_argument("--date", required=True, help="Folder name, e.g. 2026-04-29")
    parser.add_argument("--out", default="train_files.json", help="Output JSON path")
    args = parser.parse_args()

    api = HfApi()
    folder = args.date.rstrip("/")
    prefix = f"{folder}/"

    # Use recursive=False per folder to minimize API calls/pagination.
    items = api.list_repo_tree(repo_id=args.repo, path=folder, recursive=False)

    files = []
    for item in items:
        if getattr(item, "type", None) != "file":
            continue
        path = getattr(item, "path", "")
        if not path or not path.startswith(prefix):
            continue
        # CDN URL bypasses API auth/rate limits.
        cdn_url = f"{CDN_BASE}/{args.repo}/resolve/main/{path}"
        files.append(
            {
                "path": path,
                "cdn_url": cdn_url,
                "size": getattr(item, "size", None),
            }
        )

    out_path = os.path.abspath(args.out)
    with open(out_path, "w") as f:
        json.dump({"repo": args.repo, "date": folder, "files": files}, f, indent=2)

    print(f"Wrote {len(files)} files to {out_path}")

if __name__ == "__main__":
    main()
```

### ops/run_studio_training.py
```python
#!/usr/bin/env python3
"""
Run training in Lightning Studio with reuse + restart guard.

- Reuses running Studio by name.
- If stopped, restarts with L40S before run.
- Uploads train_files.json and training script, then runs.
"""

import argparse
import json
import sys
import time

try:
    from lightning.pytorch.studio import Studio, Machine
    from lightning.pytorch.studio.teamspace import Teamspace
except ImportError:
    print("Install: pip install lightning", file=sys.stderr)
    sys.exit(1)

def find_running_studio(name: str):
    for s in Teamspace.studios:
        if s.name == name:
            return s
    return None

def main() -> None:
    parser = argparse.ArgumentParser(description="Run Studio training with reuse.")
    parser.add_argument("--name", default="vanguard-surrogate-train", help="Studio name")
    parser.add_argument("--script", required=True, help="Training script to run")
    parser.add_argument("--files", nargs="+", default=[], help="Extra files to upload")
    parser.add_argument("--machine", default="L40S", help="Machine type if restart needed")
    args = parser.parse_args()

    studio = find_running_studio(args.name)
    if studio is None:
        print(f"Creating Studio {args.name}...")
        studio = Studio.create(
            name=args.name,
            machine=Machine.L40S,
            script_path=args.script,
            create_ok=True,
        )
    else:
        print(f"Found existing Studio {args.name} (status={studio.status})")

    # If stopped, restart before run.
    if studio.status != "running":
        print(f"Studio not running (status={studio.status}). Starting {args.machine}...")
        studio.start(machine=Machine[args.machine.upper()] if hasattr(Machine, args.machine.upper()) else Machine.L40S)
        # Wait briefly for running state.
        for _ in range(10):
            time.sleep(6)
            studio.refresh()
            if studio.status == "running":
                break
        else:
            print("Studio did not reach running state in time.", file=sys.stderr)
            sys.exit(1)

    # Upload helper files and run.
    upload_paths = [args.script] + args.files
    for p in upload_paths:
        if not p or not os.path.exists(p):
            print(f"Missing file: {p}", file=sys.stderr)
            sys.exit(1)

    print("Uploading files...")
    for p in upload_paths:
        studio.upload_file(p)

    print("Running training script...")
    run = studio.run(script_path=args.script)
    print(f"Run started: {run}")

if __name__ == "__main__":
    import os
    main()
```

### ops/wrapper_bash_stub.sh
```bash
#!/usr/bin/env bash
# Canonical wrapper for cron-launched ops scripts.
# Usage in crontab:
#   SHELL=/bin/bash
#   * * * * * /opt/axentx/vanguard/ops/wrapper_bash_stub.sh /path/to/real_script.sh "$@"

set -euo pipefail
export SHELL=/bin/bash

if [[ $# -eq 0 ]]; then
  echo "Usage: $0 <script> [args...]" >&2
  exit 1
fi

SCRIPT="$1"
shift

if [[ ! -x "$SCRIPT" ]]; then
  echo "Script not executable or missing: $SCRIPT" >&2
  exit 1
fi

exec bash "$SCRIPT" "$@"
```
