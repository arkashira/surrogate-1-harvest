## Final Synthesized Implementation  
*(Best parts merged; contradictions resolved for correctness + concrete actionability)*

---

### 1) Manifest generator (single source of truth)
- **Location**: `surrogate/training/scripts/build_manifest.py`  
- **Behavior**:
  - Uses HF API **once** (after rate-limit window) to `list_repo_tree` for a specific date folder in `dataset-mirror/raw/`.  
  - Non-recursive, one-level listing to avoid pagination explosion.  
  - Emits `manifest_YYYY-MM-DD.json` containing:
    - `date`, `repo_id`, `root_path`, and sorted `files` (relative paths).  
    - Optional `etag`/`sha256` per file for integrity checks (recommended).  
- **Correctness fix**: Accept date as CLI arg (default UTC today) so reruns are deterministic.  
- **Actionability**:
  - Run from Mac or CI after rate-limit window:
    ```bash
    HF_TOKEN=... python surrogate/training/scripts/build_manifest.py --date 2026-05-03
    ```
  - Exit non-zero on API failure so CI/alerting catches problems.

### 2) CDN-only parquet loader (bypasses HF API during training)
- **Location**: `surrogate/data/cdn_parquet_loader.py`  
- **Behavior**:
  - `IterableDataset` that streams parquet via direct CDN URLs:  
    `https://huggingface.co/datasets/{repo_id}/resolve/main/{path}`  
  - Projects only `{prompt, response}` at parse time to avoid `pyarrow.CastError` on extra/mixed columns.  
  - Coerces to string and skips empty rows.  
- **Correctness fixes**:
  - Use `pq.ParquetFile` on streamed bytes; do **not** rely on `r.raw` alone—handle compressed chunks robustly.  
  - Validate schema contains required columns; raise clear error if missing.  
  - Add per-file integrity check (optional `sha256` from manifest) to detect CDN corruption.  
- **Actionability**:
  - Deterministic order: iterate files in manifest order.  
  - Small memory footprint: stream file-by-file, row-by-row.  
  - Provide `max_files` and `shard` options for multi-worker distributed training.

---

### 3) Training launcher (Lightning Studio reuse + idle-stop)
- **Behavior**:
  - Reuse running Studio if available; otherwise create L40S Studio.  
  - Ensure running before submit; restart only if unhealthy (not blindly).  
  - Submit training as non-blocking run with clear name.  
- **Correctness fixes**:
  - Do **not** call `.run(..., wait=False)` then immediately exit without verifying submission succeeded. Poll or check run ID.  
  - Pass manifest path and repo id as args; do not hardcode.  
  - Set `save_strategy="steps"` and `save_total_limit` so checkpoints survive idle-stop.  
- **Actionability**:
  - Wrapper script: `scripts/launch_studio_training.sh`
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail
    MANIFEST_PATH="${1:-manifests/file_list_$(date -u +%Y-%m-%d).json}"
    REPO_ID="${2:-Qwen/Qwen2.5-0.5B-Instruct}"
    STUDIO_NAME="surrogate-1-train"
    cd /opt/axentx/airship/surrogate
    python scripts/run_in_studio.py --manifest "$MANIFEST_PATH" --repo-id "$REPO_ID" --studio "$STUDIO_NAME"
    ```
  - `scripts/run_in_studio.py` handles:
    - Studio lookup/reuse/start.  
    - Submit training with `--manifest`, `--repo-id`, `--output-dir=/shared/ckpt`.  
    - Verify submission and print run ID + logs URL.

### 4) Training script (robust tokenization + streaming)
- **Key snippet** (integrated into `surrogate/train.py`):
  ```python
  raw_dataset = CDNParquetIterable(manifest_path=manifest_path, repo_id="axentx/dataset-mirror")
  def tokenize_fn(batch):
      return tokenizer(
          batch["prompt"],
          text_target=batch["response"],
          truncation=True,
          max_length=2048,
      )
  dataset = Dataset.from_generator(lambda: raw_dataset, features={"prompt": str, "response": str})
  dataset = dataset.map(tokenize_fn, batched=True, remove_columns=["prompt", "response"])
  ```
- **Correctness fixes**:
  - Use `DataCollatorForLanguageModeling` for proper labels.  
  - Enable `fp16`/`bf16` and gradient checkpointing to fit batch size.  
  - Pin `num_workers>0` and `prefetch_factor` for performant streaming.

### 5) Operational workflow (concrete steps)
1. **Generate manifest** (Mac/CI) after rate-limit window:
   ```bash
   HF_TOKEN=... python surrogate/training/scripts/build_manifest.py --date 2026-05-03
   ```
2. **Launch training** (reuses Studio, idempotent):
   ```bash
   bash scripts/launch_studio_training.sh manifests/file_list_2026-05-03.json
   ```
3. **Monitor**: Studio run page + logs; checkpoints saved to `/shared/ckpt`.

---

### Resolved contradictions (in favor of correctness + actionability)
- Manifest location: use `surrogate/training/scripts/build_manifest.py` (clear ownership) but allow override via CLI.  
- Schema errors: project only needed columns **and** validate presence; coerce to string.  
- Studio reuse: do not blindly restart; check health and only start if not running.  
- Checkpoints: enable periodic saves (not `save_strategy="no"`) so idle-stop doesn’t lose progress.  
- Streaming integrity: add optional hash checks and robust parquet reading from streamed bytes.
