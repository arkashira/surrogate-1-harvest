## Deployment
cf workers (free-tier Durable Objects + KV for config/feature-flags; optional HF space for WASM UI)

## v0.1 scope
v0.1 in 2 weeks: local stealth agent (Rust CLI) with SOCKS5 shim that wraps traffic in TLS-in-TLS or meek-like HTTP CONNECT over domain-fronted hosts; TUN shim disabled by default; local SQLite encrypted store; fail-closed kill-switch that blocks non-stealth traffic; zero-config bootstrap via CF Workers-served config; client-side WAU-ZB counters with no server-side logs; basic integration tests simulating DPI probes.

## Open questions
- Which pluggable transports provide best DPI evasion vs performance on low-end devices in target regions?
- How will we distribute bootstrap nodes and front domains without creating a censorable directory?
- What local firewall primitives (nftables/Windows Filtering Platform) are available for reliable fail-closed kill-switch across platforms?
- How to implement client-side WAU-ZB telemetry with strong anonymity (private aggregation, differential privacy) while remaining lightweight?
- Can WASM sandboxing provide safe TUN-like behavior for browser-based clients without kernel access?
