# workio / backend

## 1. Diagnosis

- Missing **idempotency key** on `/webhook/line` POST allows LINE retries (network blips, 5xx, slow client) to create duplicate punches.
- No **partial unique index** enforcing “one open punch per user” (`clock_out_at IS NULL`) → corrupt state on races/retries.
- Punch write path is **non-transactional** for read-then-insert → TOCTOU race between check and insert.
- No **de-duplication window** for recent identical events (same userId, eventType, timestamp ±2s) → duplicates from rapid retries.
- Logging/metrics do not surface duplicate attempts → hard to detect and debug in production.

## 2. Proposed change

- **File scope**: `workio/server/src/routes/webhook/line.ts` (or equivalent route handler) + `workio/server/src/db/schema.sql` + small service util.
- **Goal**: Add idempotency + partial unique constraint + transactional upsert for punches on LINE webhook events.

## 3. Implementation

### 3.1 DB schema change (add constraint + idempotency table)

```sql
-- workio/server/src/db/schema.sql

-- Idempotency keys for webhook events (LINE and others)
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  event_type      TEXT NOT NULL,
  payload_hash    TEXT NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- Ensure one open punch per user (clock_out_at NULL means still clocked in)
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;
```

### 3.2 Idempotency + transactional upsert helper

```ts
// workio/server/src/services/idempotency.ts
import { PoolClient } from 'pg';
import crypto from 'crypto';

export async function ensureIdempotent(
  client: PoolClient,
  key: string,
  eventType: string,
  payload: unknown
): Promise<boolean> {
  const hash = crypto.createHash('sha256').update(JSON.stringify(payload)).digest('hex');
  const res = await client.query(
    `INSERT INTO webhook_idempotency (idempotency_key, event_type, payload_hash)
     VALUES ($1, $2, $3)
     ON CONFLICT (idempotency_key) DO UPDATE
     SET payload_hash = EXCLUDED.payload_hash
     WHERE webhook_idempotency.payload_hash <> EXCLUDED.payload_hash
     RETURNING (xmax = 0) AS inserted`,
    [key, eventType, hash]
  );
  // If row existed and payload unchanged, xmax = 0 is false -> duplicate
  return res.rows[0]?.inserted === true;
}
```

### 3.3 Transactional punch upsert in webhook handler

```ts
// workio/server/src/routes/webhook/line.ts
import express from 'express';
import { pool } from '../../db';
import { ensureIdempotent } from '../../services/idempotency';

const router = express.Router();

router.post('/line', async (req, res) => {
  const body = req.body;
  const events = body.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'No events' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      // Expected: ev.source.userId, ev.type, ev.message?.text or contentProvider etc.
      // For clock-in/out we assume a custom format or mapping from LINE message text.
      // This example assumes metadata in ev.metadata (you should adapt to your actual payload).
      const userId = ev.source?.userId;
      const eventType = ev.type; // e.g., 'clock-in', 'clock-out' or parse from message
      const occurredAt = ev.timestamp ? new Date(ev.timestamp) : new Date();

      if (!userId) continue;

      // Idempotency key: LINE webhook delivery id + event index or stable event hash
      const idempotencyKey = `line:${ev.id || crypto.createHash('sha256').update(JSON.stringify(ev)).digest('hex')}`;
      const isNew = await ensureIdempotent(client, idempotencyKey, 'line-webhook', ev);
      if (!isNew) {
        // Duplicate event — skip processing but continue ack so LINE stops retrying
        continue;
      }

      // Determine action: if eventType or message indicates clock-out, close open punch; else open new punch.
      // This is a simplified policy: if eventType === 'clock-out' -> close; else -> clock-in.
      const isClockOut = eventType === 'clock-out' || (ev.message?.text || '').toLowerCase().includes('out');

      if (isClockOut) {
        // Close most recent open punch for this user
        await client.query(
          `UPDATE punches
           SET clock_out_at = $1, updated_at = NOW()
           WHERE user_id = $2 AND clock_out_at IS NULL
           ORDER BY clock_in_at DESC
           LIMIT 1`,
          [occurredAt, userId]
        );
      } else {
        // Try insert; partial unique index will block second open punch.
        // Use ON CONFLICT DO NOTHING to avoid error on race; application can decide to update or reject.
        await client.query(
          `INSERT INTO punches (user_id, clock_in_at, clock_out_at, location, line_user_id, created_at, updated_at)
           VALUES ($1, $2, NULL, $3, $4, NOW(), NOW())
           ON CONFLICT DO NOTHING`,
          [userId, occurredAt, ev.location || null, userId]
        );
      }
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    // Important: return 5xx so LINE may retry (idempotency prevents duplicates)
    res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
});

export default router;
```
