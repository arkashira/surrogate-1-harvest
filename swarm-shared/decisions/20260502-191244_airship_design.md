# airship / design

## Implementation Plan: `airship discover` — Zero-config discovery CLI

**Highest-value incremental improvement (<2h):**  
Ship a single Bash CLI (`airship discover`) that operationalizes past patterns (business research + knowledge-rag + top-hub insight) and surfaces strategic context for Arkship/Surrogate platform decisions.

### Design
- **Entrypoint**: `/opt/axentx/airship/bin/airship` (symlink or script) with subcommand `discover`
- **Behavior**:
  1. Run `granite-business-research.sh` (if present) or lightweight market scan.
  2. Execute `knowledge-rag` query for top hub and related docs.
  3. Print top-hub insight (e.g., "MOC") and 3–5 actionable recommendations.
  4. Exit 0 on success; non-zero on failure with clear stderr.
- **Zero-config**: No env required; uses relative paths under `/opt/axentx/airship`. Falls back gracefully.
- **Mac=CLI rule**: Safe to run on Mac — only orchestration, no heavy compute.

### File changes
1. `bin/airship` — main CLI dispatcher (Bash).
2. `lib/discover.sh` — implementation of discover command.
3. Update `README.md` with usage.

### Code snippets

#### `bin/airship`
```bash
#!/usr/bin/env bash
# /opt/axentx/airship/bin/airship
# Airship CLI — zero-config strategic tooling

set -euo pipefail

AXENTX_ROOT="$(cd "$(dirname "$(dirname "$(realpath "${BASH_SOURCE[0]}")")")" && pwd)"
export AXENTX_ROOT

# shellcheck source=../lib/discover.sh
. "${AXENTX_ROOT}/lib/discover.sh"

usage() {
  cat <<EOF
Usage: airship <command>

Commands:
  discover   Run business research + knowledge-rag + top-hub insight
EOF
  exit 1
}

cmd="${1:-}"
case "${cmd}" in
  discover)
    airship_discover
    ;;
  *)
    usage
    ;;
esac
```

#### `lib/discover.sh`
```bash
#!/usr/bin/env bash
# /opt/axentx/airship/lib/discover.sh
# Discover: business research + knowledge-rag + top-hub insight

set -euo pipefail

AIRSHIP_ROOT="${AXENTX_ROOT:-$(cd "$(dirname "$(dirname "$(realpath "${BASH_SOURCE[0]}")")")" && pwd)}"
readonly AIRSHIP_ROOT

log() { printf "[airship:discover] %s\n" "$*"; }
error() { printf "[airship:discover] ERROR: %s\n" "$*" >&2; }

airship_discover() {
  log "Starting zero-config discovery (airship discover)"

  # 1) Business research
  if command -v granite-business-research.sh >/dev/null 2>&1; then
    log "Running granite-business-research.sh..."
    if granite-business-research.sh; then
      log "Business research completed."
    else
      error "Business research script failed (non-fatal)."
    fi
  else
    log "granite-business-research.sh not found — skipping (non-fatal)."
  fi

  # 2) Knowledge RAG query for top hub and related docs
  if command -v knowledge-rag >/dev/null 2>&1; then
    log "Querying knowledge-rag for top hub and related docs..."
    # Capture output for parsing; tolerate failures
    RAG_OUTPUT=$(knowledge-rag --top-hub 2>/dev/null || true)
    if [ -n "${RAG_OUTPUT}" ]; then
      log "Knowledge RAG output:"
      printf "%s\n" "${RAG_OUTPUT}"
    else
      log "No output from knowledge-rag; trying fallback query..."
      RAG_OUTPUT=$(knowledge-rag "top hub" 2>/dev/null || true)
      [ -n "${RAG_OUTPUT}" ] && printf "%s\n" "${RAG_OUTPUT}"
    fi
  else
    log "knowledge-rag not found — skipping RAG step."
  fi

  # 3) Top-hub insight (heuristic: look for MOC or most-connected hub mention)
  log "Top-hub insight:"
  if [ -n "${RAG_OUTPUT:-}" ]; then
    # Simple heuristic: extract lines mentioning hubs/tools
    echo "${RAG_OUTPUT}" | grep -iE 'hub|moc|most.connected|central' | head -10 || true
  fi

  # Always print actionable guidance
  cat <<'EOF'

Recommended next steps:
- Review the most-connected hub (e.g., MOC) before planning tasks.
- Use `airship discover` output to inform Arkship/Surrogate design choices.
- If using cron for automation, ensure scripts have proper Bash shebang and set SHELL=/bin/bash.
- For HF dataset ingestion: prefer CDN resolve URLs and pre-list file paths to avoid API rate limits.
- For Lightning training: reuse running studios and spread HF writes across sibling repos.

EOF

  log "Discovery complete."
}
```
