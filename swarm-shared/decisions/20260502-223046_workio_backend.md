# workio / backend

## 1. Diagnosis
- **Tenant cache missing on `/webhook/line`** — every LINE event triggers a DB lookup for `channelId → tenant`; traffic spikes cause DB thrash and increase 429/timeout risk.
- **No request-scoped correlation IDs** — webhook → service → DB spans are untraceable; incidents lack end-to-end context and slow debugging.
- **No idempotency for duplicate LINE retries** — LINE retries on 5xx/timeout can double-process clock events (double clock-in/out) without dedupe.
- **Missing structured logging** — ad-hoc `console.log` makes log aggregation and alerting unreliable in production.
- **No graceful degradation on tenant lookup failure** — transient DB outage causes hard 5xx to LINE; should return 200 with safe fallback and async retry queue.

## 2. Proposed change
- **File scope**: `/opt/axentx/workio/server/src/routes/webhook/line.ts` (or equivalent) + `/opt/axentx/workio/server/src/middleware/` for correlation-id and logging.
- **Add**:
  - In-memory LRU cache (`channelId → tenantId`) with 5-minute TTL to eliminate repeated DB lookups.
  - Correlation ID middleware (header `X-Request-ID` or generate) and attach to `req.id` and logger.
  - Idempotency key store (Redis or in-memory LRU) keyed by `channelId + event.timestamp + event.type` to dedupe retries within 5 minutes.
  - Structured logger (pino) with request-scoped child logger.
  - Graceful fallback: if tenant lookup fails, respond 200 with `{ok:false,retryLater:true}` and enqueue background retry (in-memory queue or Bull) so LINE stops retrying.

### middleware/logger.ts
```ts
import pino from 'pino';
import { v4 as uuidv4 } from 'uuid';

const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  transport: process.env.NODE_ENV === 'development' ? { target: 'pino-pretty' } : undefined,
});

export function requestLogger(req: any, res: any, next: () => void) {
  const requestId = req.headers['x-request-id'] || uuidv4();
  req.id = requestId;
  req.log = logger.child({ requestId, path: req.path, method: req.method });
  next();
}
```

### middleware/cache.ts
```ts
import LRU from 'lru-cache';

export const tenantCache = new LRU<string, { tenantId: string; updatedAt: number }>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
});

export const idempotencyCache = new LRU<string, boolean>({
  max: 5000,
  ttl: 1000 * 60 * 5, // 5 minutes
});
```

### routes/webhook/line.ts (excerpt)
```ts
import express from 'express';
import { tenantCache, idempotencyCache } from '../../middleware/cache';
import { getTenantByChannelId } from '../services/tenantService';

const router = express.Router();

async function resolveTenant(channelId: string, log: any) {
  const cached = tenantCache.get(channelId);
  if (cached) return cached.tenantId;

  try {
    const tenant = await getTenantByChannelId(channelId);
    if (!tenant) {
      log.warn({ channelId }, 'tenant not found for channel');
      return null;
    }
    tenantCache.set(channelId, { tenantId: tenant.id, updatedAt: Date.now() });
    return tenant.id;
  } catch (err) {
    log.error({ err, channelId }, 'tenant lookup failed');
    return null;
  }
}

router.post('/line', async (req, res) => {
  const log = req.log || console;
  const body = req.body;

  // Basic validation
  if (!body.events || !Array.isArray(body.events)) {
    return res.status(400).json({ error: 'invalid payload' });
  }

  const results = [];

  for (const event of body.events) {
    const idemKey = `${event.source?.userId || event.source?.groupId || 'unknown'}:${event.type}:${event.timestamp}`;
    if (idempotencyCache.has(idemKey)) {
      log.info({ idemKey }, 'duplicate event skipped');
      results.push({ status: 'duplicate' });
      continue;
    }

    try {
      const tenantId = await resolveTenant(event.source.channelId || event.destination, log);
      if (!tenantId) {
        // Graceful degradation: accept but mark for retry
        results.push({ status: 'deferred', reason: 'tenant_not_found' });
        // enqueue background retry (simplified: could push to Bull queue)
        continue;
      }

      // Process event (example: clock in/out)
      // await processLineEvent(event, tenantId, log);

      idempotencyCache.set(idemKey, true);
      results.push({ status: 'processed' });
    } catch (err) {
      log.error({ err, event }, 'event processing failed');
      // Return 200 to stop LINE retries; rely on idempotency + background retry
      results.push({ status: 'error_deferred' });
    }
  }

  // Always 200 to prevent LINE retries; rely on idempotency & background retry
  return res.status(200).json({ ok: true, results });
});

export default router;
```

## 4. Verification
- **Cache hit test**: Send two identical webhook requests within 5 minutes; second request should show `tenantCache.get` hit in logs and no DB query (monitor DB logs or add a counter).
- **Correlation ID**: Send request with `X-Request-ID: test-123`; confirm `test-123` appears in all log lines for that request.
- **Idempotency**: Send same event twice (same timestamp/type/user) — second should return `duplicate` in results and not create duplicate clock records.
- **Graceful degradation**: Temporarily block DB access or return null tenant; endpoint should respond 200 with `{ok:true,results:[{status:"deferred"}]}` and not throw 5xx.
- **Structured logs**: Check stdout for JSON-formatted logs with `requestId`, `level`, `msg`, and contextual fields.
