## v0.1 scope
v0.1 in 2 weeks: deterministic research-to-signal workflow with Redis Streams orchestration, Postgres event journal, Redis cache for idempotency, compliance guardrail checks (static allowlist + per-trade limit), and simulated execution adapter; immutable audit trail with cryptographic hashing; deployed on HF space + Supabase free-tier; API for triggering runs and replaying by run ID; basic dashboard-less CLI/API tests to validate end-to-end latency and audit correctness.

## Open questions
- Deterministic replay fidelity: which non-deterministic sources (clocks, external non-idempotent APIs) require snapshotting or idempotency keys?
- Latency SLA target for sub-second orchestration: measured from signal generation to order submission under 95th percentile?
- Compliance rule engine scope: DSL vs imperative code; who authors/updates rules and what is the change control process?
- Broker/OMS connector contract: FIX vs REST, idempotency semantics, and required sandbox credentials for v0.1?
- Data vendor licensing and rate limits: can we cache and redistribute snapshots on HF free-tier?
- Audit retention and immutability: append-only storage strategy and cryptographic verification approach?
