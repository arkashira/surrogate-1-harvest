# surrogate-1 / design

## Implementation Plan (≤2h)

**Highest-value improvement**: Add a pre-flight snapshot generator (`bin/snapshot.sh`) that lists all dataset files once, embeds the file manifest into training, and enables CDN-only downloads during parallel shard processing.

### Why this now
- Eliminates HF API rate-limit (429) during ingestion by replacing `load_dataset`/`list_repo_files` with a single pre-computed file list + CDN fetches.
- Fits the existing 16-shard matrix workflow with zero changes to the runner contract.
- Reuses the HF CDN bypass insight: `resolve/main/` URLs require no auth and have much higher rate limits.

#### 1) `bin/snapshot.sh` — generate file manifest
```bash
#!/usr/bin/env bash
# bin/snapshot.sh
# Usage: HF_TOKEN=... ./bin/snapshot.sh axentx surrogate-1-training-pairs main
# Produces: snapshots/<dataset>/<date>/file-list.json

set -euo pipefail

REPO_OWNER="${1:-axentx}"
REPO_NAME="${2:-surrogate-1-training-pairs}"
BRANCH="${3:-main}"
OUTDIR="snapshots/${REPO_OWNER}/${REPO_NAME}"
DATE=$(date -u +%Y-%m-%d)
TS=$(date -u +%Y%m%dT%H%M%SZ)

mkdir -p "${OUTDIR}/${DATE}"

# Single API call: list top-level tree (non-recursive) then expand folders minimally.
# Avoid list_repo_files recursive to stay within 1000/5min budget.
# We'll flatten common folder patterns used by the dataset.
echo "Fetching tree for ${REPO_OWNER}/${REPO_NAME}@${BRANCH}..."

# Use huggingface_hub via python for reliable tree listing; fallback to raw API if needed.
python3 - "$REPO_OWNER" "$REPO_NAME" "$BRANCH" "${OUTDIR}/${DATE}/file-list.json" <<'PY'
import os, json, sys
from huggingface_hub import HfApi

owner, repo, branch, out = sys.argv[1], sys.argv[2], sys.argv[3], sys.argv[4]
api = HfApi(token=os.environ.get("HF_TOKEN"))

# List top-level non-recursive
tree = api.list_repo_tree(repo=f"{owner}/{repo}", revision=branch, recursive=False)

# Expand likely data folders one level (cheap). Adjust if dataset layout changes.
folders = [t.path for t in tree if t.type == "directory"]
all_files = [t.path for t in tree if t.type == "file"]

for folder in folders:
    try:
        sub = api.list_repo_tree(repo=f"{owner}/{repo}", revision=branch, path=folder, recursive=False)
        for t in sub:
            if t.type == "file":
                all_files.append(t.path)
    except Exception as e:
        print(f"Skipping {folder}: {e}", file=sys.stderr)

# Keep only files that look like dataset artifacts (parquet/jsonl) to reduce noise.
# This is safe because training only cares about these.
keep_ext = {".parquet", ".jsonl", ".json", ".csv", ".tsv"}
filtered = [f for f in all_files if any(f.endswith(e) for e in keep_ext)]

result = {
    "dataset": f"{owner}/{repo}",
    "branch": branch,
    "snapshot_ts": os.environ.get("SNAPSHOT_TS", ""),
    "files": sorted(set(filtered)),
    "cdn_base": f"https://huggingface.co/datasets/{owner}/{repo}/resolve/main"
}

os.makedirs(os.path.dirname(out), exist_ok=True)
with open(out, "w") as fp:
    json.dump(result, fp, indent=2)
print(f"Wrote {len(result['files'])} files to {out}")
PY

echo "Snapshot saved to ${OUTDIR}/${DATE}/file-list.json"
```

Make executable:
```bash
chmod +x bin/snapshot.sh
```
