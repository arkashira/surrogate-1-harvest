## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-region CF Workers gateway exposing /v1/chat/completions compatible endpoint; Postgres-backed provider configs and RoutePolicy; Redis-backed token-bucket quotas and fan-out queue; adapters for OpenAI and Anthropic chat completions; global failover and max-retry policy; basic audit logging to Postgres; Prometheus metrics on CF Workers; no frontend, API-only.

## Open questions
- Which provider auth secrets will be stored (Supabase Vault, CF Workers KV, or external secret manager) and rotation cadence?
- Per-provider quota sync strategy: webhook push from providers vs polling vs hybrid; which providers expose real-time usage APIs?
- PII filter implementation: in-gateway lightweight regex/allow-lists vs external service (HF space) — latency budget and accuracy trade-offs?
- Idempotency key semantics and deduplication window for retries across providers.
- SLA targets for p95 latency and error budget to drive weighted routing thresholds.
