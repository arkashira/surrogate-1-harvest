## v0.1 scope
v0.1 in 2 weeks: deterministic research→signal→risk→execution workflow with one broker connector, immutable Postgres audit trail, Cloudflare Queue-driven orchestration, Redis-backed risk/limit cache, and compliance guardrails (pre-trade limit/blacklist) that block runs and record immutable decisions; includes replay endpoint for audit and basic dashboard-less API for integrations.

## Open questions
- Which broker/OMS/EMS connectors must be prioritized for v0.1 to unlock fee-generating runs?
- What deterministic replay window and snapshot cadence are required for compliance audits?
- What latency SLA (p50/p95) is acceptable from signal to execution for target strategies?
- Which regulatory guardrails (e.g., pre-trade limit checks, blacklists) are non-negotiable at launch?
- How will secrets/credentials for broker APIs be stored and rotated (Supabase vault vs. CF secrets)?
