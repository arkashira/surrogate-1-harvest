## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a single-file Rust CLI agent (Linux x86_64) that creates a local SOCKS5 listener, applies meek-TLS obfuscation to tunnel traffic, conceals the listening port via socket cloaking and randomized ephemeral ports, and enforces a local kill-switch that blocks non-tunneled traffic if the stealth path fails; zero-config heuristics choose defaults, all state kept in local SQLite, no cloud servers required, client-side telemetry opt-in with minimal WAU-ZB counters.

## Open questions
- Which obfuscation primitives (meek, Shadowsocks, v2ray-plugin style, QUIC domain-front) maximize evasion while remaining implementable client-side only?
- How will we measure WAU-ZB reliably without leaking telemetry metadata that deanonymizes users?
- What local port/TUN hiding techniques are effective against current DPI (port randomization, socket cloaking, protocol mimicry) and can be automated zero-config?
- How to implement fail-closed kill-switch across platforms (Windows/Linux/macOS) without admin/root in client-only deployment?
- Can we distribute updated obfuscation profiles via CF Workers + IPFS/Hyperdrives to avoid central infra and preserve free-tier constraints?
