## Deployment

## v0.1 scope
v0.1 in 2 weeks: Cloudflare Workers gateway with a single normalized '/v1/chat/completions' endpoint that proxies to OpenAI and Anthropic (ChatCompletion) with API-key auth via Supabase; Redis-backed per-provider token counters and circuit breaker; latency-based routing with fallback; basic audit to Supabase Postgres (RequestLog, TokenLedger); OpenTelemetry metrics on CF; contract tests for provider schemas; README with local dev (Wrangler + Supabase local + Upstash Redis simulator) and simple load script to verify p95 latency under 800ms at 500 RPM.

## Open questions
- Which provider APIs must be normalized in v0.1 (OpenAI, Anthropic, Azure OpenAI, Cohere, HuggingFace Inference)?
- What is the authoritative source-of-truth for per-provider token counting (gateway vs provider webhook vs periodic reconcile)?
- How should quota overruns be handled: hard block, soft throttle with retry, or spillover to next provider?
- What latency SLA (p95) is required for v0.1 and how will it be measured end-to-end vs per-provider?
- Will authentication to the gateway be API keys, JWT, or Supabase auth and how will key rotation work?
- Do we need payload transformation layers (e.g., OpenAI↔Anthropic schema mapping) in v0.1 or passthrough with normalization only at response envelope?
- What retention and PII handling policy applies to RequestLog and TokenLedger for compliance?
