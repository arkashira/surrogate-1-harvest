## Deployment
cf workers

## v0.1 scope
v0.1 ships a minimal CLI stealth client (Rust) that: 1) spawns a local SOCKS5 listener on an ephemeral port, 2) wraps traffic in obfuscated QUIC streams to a public meek-like endpoint, 3) optionally creates a TUN device and routes allowed IPs, 4) enforces a fail-closed killswitch that blocks non-stealth traffic if the encrypted path fails, 5) records local session telemetry and syncs anonymized summaries to Supabase Postgres, 6) supports Windows and Linux with no admin/root requirement where possible, and 7) includes basic DPI scenario tests for port/packet signature concealment.

## Open questions
- Which DPI signatures in target regions are most likely to detect our local SOCKS5/TUN artifacts (port patterns, packet timing, TLS fingerprints)?
- Can we rely on CF Workers for domain-fronting/meek endpoints without violating ToS or risking takedown?
- What is the acceptable tradeoff between stealth and latency when using obfuscation + QUIC fronting?
- How will client-side telemetry (WAU-ZB) be collected without deanonymizing users or creating metadata leakage?
- Which TUN/device permissions model works across desktop targets (Windows/Linux/macOS) while remaining zero-config?
