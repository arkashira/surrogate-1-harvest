# Costinel / design

## Implementation Plan — Costinel “Top-Hub Signal” Card (read-only)

**Scope**: Frontend-only card that identifies the most-connected hub from the knowledge-rag graph and surfaces the top hub with 3 contextual signals.  
**Timebox**: <2h (frontend-only, no backend changes).

### Why
- Follows pattern: review the most-connected hub (e.g., "MOC") before planning tasks (#knowledge-rag #graph #hub).
- Provides immediate contextual awareness on the Costinel dashboard without touching backend or data-plane.
- Read-only, zero risk, fast to ship.

---

### What (deliverable)
A new dashboard card:
- Title: “Top-Hub Signal”
- Shows:
  1. Hub label (e.g., MOC) + short description
  2. Top 3 contextual signals (title + 1-line summary)
  3. “Last updated” timestamp
- Static fallback content (seeded for MOC) so UI is useful immediately.
- Styled to match existing Costinel cards (clean, minimal, info-focused).

---

### Implementation Steps (≤2h)

1. Add card component file: `src/components/cards/TopHubSignalCard.vue` (or `.tsx` if project uses React).
2. Add static seed data (mimics knowledge-rag top hub + 3 signals) in component.
3. Wire card into main dashboard layout (`src/views/Dashboard.vue` or equivalent) in the top-row or signals section.
4. Add simple styles scoped to card (reuse existing design tokens).
5. Verify rendering and responsiveness.
6. Commit.

---
