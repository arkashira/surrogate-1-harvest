## Final Synthesized Plan  
**Goal:** Eliminate HF API 429s during Surrogate training and make Lightning training resilient to idle timeouts — deployable in ~90 min with zero schema/infra changes.

## 1) Deterministic CDN File List (run once per training cycle)

- **Why:** Avoid repeated paginated API calls and rate limits.  
- **How:** Single-level `list_repo_tree` per date folder; save to JSON committed or passed via env.  
- **Correctness fix:** Use `recursive=False` and filter `type == "file"`; include only needed date folder.

```python
# scripts/generate_filelist.py
#!/usr/bin/env python3
import json
import os
import sys
from huggingface_hub import HfApi

def main():
    repo = os.getenv("REPO", "axentx/surrogate-dataset")
    date_folder = os.getenv("DATE_FOLDER", sys.argv[1] if len(sys.argv) > 1 else "")
    out = os.getenv("OUT", f"training/filelist-{date_folder or 'latest'}.json")

    api = HfApi()
    entries = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)
    files = sorted(e.path for e in entries if e.type == "file")

    os.makedirs(os.path.dirname(out), exist_ok=True)
    with open(out, "w", encoding="utf-8") as f:
        json.dump(files, f, indent=2)
    print(f"Wrote {len(files)} files to {out}")

if __name__ == "__main__":
    main()
```

## 2) CDN-Only Dataset Loader (drop-in replacement in `train.py`)

- **Why:** HF `load_dataset`/`hf_hub_download` hit API limits; CDN GETs have much higher quotas.  
- **How:** Direct `https://huggingface.co/datasets/.../resolve/main/{path}` downloads with retries; project only `prompt`/`response`.  
- **Correctness + actionability:** Stream per-file to avoid large memory; validate columns; raise after final retry.

```python
# training/cdn_dataset.py
import json
import time
from typing import Iterator, Dict, Any

import pyarrow as pa
import pyarrow.parquet as pq
import requests
from torch.utils.data import IterableDataset

HF_DATASET_REPO = "axentx/surrogate-dataset"
CDN_BASE = f"https://huggingface.co/datasets/{HF_DATASET_REPO}/resolve/main"

class CDNParquetIterable(IterableDataset):
    def __init__(self, filelist_path: str, max_retries: int = 5, timeout: int = 30):
        with open(filelist_path, encoding="utf-8") as f:
            self.files = json.load(f)
        self.max_retries = max_retries
        self.timeout = timeout

    def _download_table(self, path: str) -> pa.Table:
        url = f"{CDN_BASE}/{path}"
        for attempt in range(1, self.max_retries + 1):
            try:
                resp = requests.get(url, timeout=self.timeout)
                resp.raise_for_status()
                return pq.read_table(pa.BufferReader(resp.content))
            except Exception as exc:
                if attempt == self.max_retries:
                    raise RuntimeError(f"Failed to fetch {url}") from exc
                time.sleep(2 ** attempt)
        raise RuntimeError(f"Unreachable: failed to fetch {url}")

    def __iter__(self) -> Iterator[Dict[str, Any]]:
        for p in self.files:
            try:
                tbl = self._download_table(p)
            except Exception:
                continue
            if "prompt" not in tbl.column_names or "response" not in tbl.column_names:
                continue
            prompts = tbl["prompt"].to_pylist()
            responses = tbl["response"].to_pylist()
            for prompt, response in zip(prompts, responses):
                if prompt is None or response is None:
                    continue
                yield {"prompt": prompt, "response": response}
```

## 3) Lightning Launcher: Studio Reuse + Idle-Stop Resilience

- **Why:** Idle timeouts kill runs; recreating studios wastes time.  
- **How:** Reuse running studio by name; restart if stopped; fallback through machines; wrap training with auto-restart loop.  
- **Correctness + actionability:** Explicit status checks; avoid duplicate studios; checkpoint frequently inside training script.

```python
# training/launch_lightning.py
import os
import time
from lightning_sdk import Client, Teamspace, Machine

LIGHTNING_API_KEY = os.getenv("LIGHTNING_API_KEY")
TEAMSPACE_NAME = os.getenv("TEAMSPACE", "default")
STUDIO_NAME = os.getenv("STUDIO_NAME", "surrogate-train")
FILELIST = os.getenv("FILELIST", "training/filelist-latest.json")

client = Client(api_key=LIGHTNING_API_KEY)
teamspace = Teamspace(client, name=TEAMSPACE_NAME)

def get_studio():
    for s in teamspace.studios:
        if s.name == STUDIO_NAME:
            return s
    return teamspace.studios.create(
        name=STUDIO_NAME,
        machine=Machine.L40S,
        shutdown_timer_hours=1,
    )

def ensure_running(studio):
    if studio.status == "running":
        return studio
    if studio.status == "stopped":
        print(f"Restarting stopped studio {STUDIO_NAME}")
        return studio.start(machine=Machine.L40S)
    for m in (Machine.L40S, Machine.A10G, Machine.L4):
        try:
            return studio.start(machine=m)
        except Exception:
            continue
    raise RuntimeError("Could not start studio on any available machine")

def run_training():
    studio = get_studio()
    studio = ensure_running(studio)

    cmd = f"python training/train.py --filelist {FILELIST}"
    while True:
        run = studio.run(
            command=cmd,
            environment="surrogate-training:latest",
            mount="./training:/train",
        )
        # Monitor run; restart studio+run on terminal failure/stop
        while True:
            run.refresh()
            if run.status in ("crashed", "failed", "stopped"):
                print(f"Run {run.status}. Restarting studio and retrying...")
                studio = ensure_running(studio)
                break  # break inner to start new run
            time.sleep(60)

if __name__ == "__main__":
    run_training()
```

## Acceptance Criteria
- Training uses **only CDN GETs** for data (no HF API calls during loading).  
- Lightning studio is **reused** when running; idle-stop triggers **automatic restart** without manual steps.  
- All wrapper scripts are executable with Bash shebang; cron sets `SHELL=/bin/bash`.  
- No schema or infra changes required; deployable within ~90 minutes.
