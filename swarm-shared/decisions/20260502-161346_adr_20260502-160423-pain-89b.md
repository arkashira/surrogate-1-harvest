## Deployment
cf workers (edge fronting + domain-front endpoints) + hf space (static WASM/UI) + client-side binaries (reuse Supabase for minimal feature flags/auth if needed)

## v0.1 scope
v0.1 in 2 weeks: local SOCKS5 stealth listener with Shadowsocks-like AEAD obfuscation, optional TUN-less mode, encrypted SQLite profiles, fail-closed kill-switch (best-effrict without admin), and client-side WAU-ZB telemetry (local-only); deploy CF worker as a single domain-fronting endpoint and static WASM demo on HF space for onboarding.

## Open questions
- Which obfuscation primitives are safest against current DPI in target regions (e.g., Shadowsocks AEAD variants vs. meek vs. domain-front)?
- Can we reliably hide TUN interface names/routes from host inspection without root/admin on each OS?
- What minimal metadata can be sent to CF workers to bootstrap fronting while preserving deniability?
- How will we measure WAU-ZB client-side without leaking identifiers or harming privacy?
- Which kill-switch implementation is robust across Windows/macOS/Linux without admin rights?
