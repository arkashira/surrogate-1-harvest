# workio / design

## Final Synthesized Solution

### 1. Diagnosis (merged)
- **No idempotency enforcement** — LINE retries on 5xx/timeouts and client double-taps create duplicate punches.
- **Non-atomic check-then-upsert** — concurrent webhooks race on “latest punch” read → decision → insert, producing duplicates and invalid state transitions.
- **Missing storage-level guardrails** — no composite uniqueness or exclusion constraint to prevent duplicate or overlapping active punches per tenant/user/day.
- **Weak state transition validation** — invalid sequences (clock-in after clock-in) are not rejected atomically.
- **No deduplication window** — retries within minutes are treated as new punches.

### 2. Design Decisions (resolved)
- Use **deterministic idempotency key** derived from event + user + type (prefer LINE-provided retry ID when present). Store keys with TTL in Redis (production) or DB (fallback).
- Replace check-then-upsert with **atomic transaction** that:
  - locks the latest punch for update (or uses `SELECT ... FOR UPDATE`/advisory lock),
  - validates transition,
  - inserts with `ON CONFLICT DO NOTHING` against a composite unique constraint,
  - returns the existing row on conflict.
- Add **composite unique constraint** `(tenant_id, user_id, punch_date, punch_type)` to prevent duplicates at storage level.
- Optionally add **exclusion constraint** (or application-level check) to prevent overlapping active punches (e.g., two rows with `clock_out_at IS NULL` for same user/tenant).
- Keep **state transition rules** in the transaction so they are enforced atomically with the insert.

### 3. Implementation

#### 3.1 DB schema (`server/src/db/schema.sql`)
```sql
-- Ensure punch_type exists
ALTER TABLE punches ADD COLUMN IF NOT EXISTS punch_type VARCHAR(10) NOT NULL DEFAULT 'clock_in';

-- Composite uniqueness to prevent duplicate punch_type per day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_unique_day_type
ON punches (tenant_id, user_id, punch_date, punch_type)
WHERE deleted_at IS NULL;

-- Optional: prevent multiple active (not clocked-out) punches per user
-- Assumes a clock_out_at column exists to mark when user clocks out
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_active_unique
ON punches (tenant_id, user_id)
WHERE deleted_at IS NULL AND clock_out_at IS NULL;
```

#### 3.2 Idempotency + atomic punch handler (`server/src/routes/webhook/line.ts`)
```ts
import { Router } from 'express';
import { db } from '../../db';
import { v4 as uuidv4 } from 'uuid';
import Redis from 'ioredis';

const router = Router();
const redis = new Redis(process.env.REDIS_URL || 'redis://localhost:6379');
const IDEMPOTENCY_TTL_SEC = 5 * 60; // 5 minutes

function idempotencyKey(event: any): string {
  // Prefer LINE retry id when present; else deterministic from event
  const retryId = event?.source?.userId && event?.timestamp
    ? `line-retry:${event.source.userId}:${event.timestamp}:${event.type}`
    : `line-event:${event?.source?.userId}:${event?.type}:${event?.message?.id || event?.type}`;
  return retryId;
}

router.post('/line', async (req, res) => {
  try {
    const events = req.body?.events;
    if (!Array.isArray(events) || events.length === 0) {
      return res.status(400).json({ error: 'invalid_payload' });
    }

    const results = [];

    for (const event of events) {
      if (event.type !== 'message' && event.type !== 'postback') continue;

      const key = idempotencyKey(event);

      // Idempotency check
      const cached = await redis.get(key);
      if (cached) {
        results.push(JSON.parse(cached));
        continue;
      }

      const userId = event.source.userId;
      const tenantId = process.env.DEFAULT_TENANT_ID || 'default'; // resolve properly in prod
      const today = new Date().toISOString().split('T')[0]; // YYYY-MM-DD

      // Atomic transaction with transition validation + uniqueness guard
      const punchResult = await db.transaction(async (trx) => {
        // Lock latest punch for this user today to serialize transitions
        const last = await trx
          .select('*')
          .from('punches')
          .where({ tenant_id: tenantId, user_id: userId })
          .whereRaw('punch_date = ?', [today])
          .whereNull('deleted_at')
          .orderBy('created_at', 'desc')
          .first()
          .forUpdate();

        const nextType = (!last || last.punch_type === 'clock_out') ? 'clock_in' : 'clock_out';

        // Insert with ON CONFLICT handling
        const [inserted] = await trx
          .insert({
            id: uuidv4(),
            tenant_id: tenantId,
            user_id: userId,
            punch_date: today,
            punch_type: nextType,
            created_at: new Date().toISOString(),
            updated_at: new Date().toISOString(),
          })
          .into('punches')
          .onConflict(['tenant_id', 'user_id', 'punch_date', 'punch_type'])
          .ignore()
          .returning('*');

        if (!inserted || inserted.length === 0) {
          // Fetch existing row
          return trx
            .select('*')
            .from('punches')
            .where({ tenant_id: tenantId, user_id: userId, punch_date: today, punch_type: nextType })
            .whereNull('deleted_at')
            .first();
        }

        return inserted;
      });

      const responseMsg = punchResult.punch_type === 'clock_in'
        ? '✅ ลงเวลาทำงานเรียบร้อย'
        : '✅ ลงเวลาออกเรียบร้อย';

      const result = { replyToken: event.replyToken, message: responseMsg, punch: punchResult };

      // Cache idempotent result
      await redis.setex(key, IDEMPOTENCY_TTL_SEC, JSON.stringify(result));
      results.push(result);

      // Optionally reply via LINE API using event.replyToken
      // await lineClient.replyMessage(event.replyToken, { type: 'text', text: responseMsg });
    }

    return res.status(200).json({ status: 'ok', results });
  } catch (error) {
    console.error('LINE webhook error:', error);
    return res.status(500).json({ error: 'internal_error' });
  }
});

export default router;
```

### 4. Verification
1. **Schema check**
   ```bash
   psql workio -c "\d punches"
   # Confirm idx_punches_unique_day_type (and optional idx_punches_active_unique) exist
   ```

2. **Idempotency test**
   - Send identical payload twice within seconds.
   - Expect: second request returns same punch record; no duplicate row.

3. **Concurrent race test**
   - Simulate parallel requests for same user (e.g., `hey` or `ab`).
   - Verify only one `clock_in` per day and valid transitions (no double clock-ins).

4. **LINE retry simulation**
   - Return 500 on first request, then 200 on retry with same event.
   - Confirm no duplicate punch; retry receives success.

5. **State validation**
   - Attempt invalid sequences (clock-in after clock-in) via crafted events.
   - Confirm they are rejected or normalized to valid state by transaction logic.
