# workio / backend

## Final Synthesis (Correctness + Actionability)

**Diagnosis (merged, de-duplicated, prioritized)**
- No idempotency guard on LINE webhook ingestion — duplicate events (retries, user double-taps) create double clock-in/out or duplicate leave/OT records.
- No request signature verification (`X-Line-Signature`) — endpoint accepts unverified POSTs, enabling spoofing/replay.
- No row-level locking or unique constraint for `user_id + date + shift` — race conditions when concurrent webhooks arrive can corrupt state (clock-out written as clock-in).
- No structured logging or correlation IDs — hard to trace webhook processing failures in production.
- No graceful handling of LINE 429/5xx retries — unbounded retries amplify load and can cause cascading failures.
- Missing de-duplication window and idempotency persistence — LINE retries with same `deliveryId` within minutes; backend treats each as new.

**Proposed change (single scope)**
File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts` (create or replace).  
Add idempotency + signature verification + row-level locking + structured logging + fast 200 to stop LINE retries, then process async.

import { Router, Request, Response } from 'express';
import crypto from 'crypto';
import { Pool, PoolClient } from 'pg';
import { logger } from '../utils/logger';

const router = Router();
const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';
if (!LINE_CHANNEL_SECRET) {
  logger.warn('LINE_CHANNEL_SECRET is not set; signature verification disabled');
}

function verifySignature(body: string, signature: string): boolean {
  if (!LINE_CHANNEL_SECRET) return false;
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

/**
 * Try to claim idempotency key.
 * Returns:
 *  - true  => this request is the first to claim the key (proceed)
 *  - false => duplicate (already claimed/processed)
 */
async function claimIdempotency(client: PoolClient, key: string, bodyHash: string): Promise<boolean> {
  try {
    const res = await client.query(
      `INSERT INTO line_webhook_events (idempotency_key, body_hash, processed)
       VALUES ($1, $2, FALSE)
       ON CONFLICT (idempotency_key) DO UPDATE
       SET body_hash = EXCLUDED.body_hash
       WHERE line_webhook_events.processed = FALSE
       RETURNING (xmax = 0) AS inserted`,
      [key, bodyHash]
    );
    return res.rows[0]?.inserted === true;
  } catch (e) {
    // duplicate key or constraint violation => already exists
    return false;
  }
}

async function markProcessed(client: PoolClient, key: string): Promise<void> {
  await client.query(
    `UPDATE line_webhook_events SET processed = TRUE WHERE idempotency_key = $1`,
    [key]
  );
}

  if (!signature || !verifySignature(rawBody, signature)) {

    return res.status(401).send('Invalid signature');

  const payload = req.body;
