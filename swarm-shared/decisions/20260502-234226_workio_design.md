# workio / design

## Final Synthesis — Correct, Actionable Solution

**Core problems (unified):**
- No idempotency for LINE webhook deliveries → duplicates on redelivery/backoff.
- Non-atomic read-then-insert/upsert → race conditions corrupt open/close state.
- No database-level guard for “one open punch per employee per tenant” → concurrent requests can create multiple open punches.
- Trusting client timestamp for punch time → allows clock manipulation.
- Missing defensive deduplication at ingress and no transactional rollback safety.

**Chosen approach:**
- Enforce correctness at the database layer (constraints + atomic CTE) and reject duplicates at the webhook layer (idempotency key from LINE delivery ID).
- Use server time for `punch_time`/`clock_in`/`clock_out`; keep client timestamp only as metadata.
- Make webhook processing transactional and idempotent per delivery ID.

### 1. Schema changes (`server/src/db/schema.sql`)

```sql
-- Idempotency for LINE webhook deliveries
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  delivery_id VARCHAR(255) PRIMARY KEY,
  tenant_id    INTEGER NOT NULL,
  employee_id  INTEGER NOT NULL,
  event_type   VARCHAR(64) NOT NULL,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_line_idemp_tenant ON line_webhook_idempotency (tenant_id, delivery_id);

-- Ensure exactly one open punch per (tenant, employee)
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch
ON punches (tenant_id, employee_id)
WHERE clock_out IS NULL;

-- Server-side timestamp (if not present)
ALTER TABLE punches
ADD COLUMN IF NOT EXISTS server_ts TIMESTAMPTZ DEFAULT NOW();
```

Notes:
- `delivery_id` length chosen to safely hold LINE `x-line-delivery` values.
- Partial unique index guarantees invariant: at most one row with `clock_out IS NULL` per employee per tenant. Concurrent inserts will be rejected by Postgres; the atomic CTE below turns that into safe no-ops.

### 2. Atomic punch upsert (`server/src/db/queries/punchQueries.ts`)

```ts
import { PoolClient } from 'pg';

export async function upsertPunch(
  client: PoolClient,
  {
    tenantId,
    employeeId,
    deliveryId,
    type,
    latitude,
    longitude,
    clientTs,
  }: {
    tenantId: number;
    employeeId: number;
    deliveryId: string;
    type: 'in' | 'out';
    latitude?: number;
    longitude?: number;
    clientTs?: string;
  }
) {
  const result = await client.query(`
    WITH idempotent AS (
      INSERT INTO line_webhook_idempotency (delivery_id, tenant_id, employee_id, event_type)
      VALUES ($1, $2, $3, $4)
      ON CONFLICT (delivery_id) DO NOTHING
      RETURNING delivery_id
    ),
    latest AS (
      SELECT (clock_out IS NULL) AS is_open
      FROM punches
      WHERE tenant_id = $2 AND employee_id = $5
      ORDER BY punch_time DESC
      LIMIT 1
    ),
    upsert AS (
      INSERT INTO punches (
        tenant_id,
        employee_id,
        punch_time,
        clock_in,
        clock_out,
        latitude,
        longitude,
        server_ts
      )
      SELECT
        $2,
        $5,
        NOW() AT TIME ZONE 'Asia/Bangkok',
        CASE WHEN (SELECT COALESCE(is_open, false) FROM latest) = false AND $4 = 'in'
          THEN NOW() AT TIME ZONE 'Asia/Bangkok'
          ELSE NULL
        END,
        CASE WHEN (SELECT COALESCE(is_open, false) FROM latest) = true  AND $4 = 'out'
          THEN NOW() AT TIME ZONE 'Asia/Bangkok'
          ELSE NULL
        END,
        $6,
        $7,
        NOW() AT TIME ZONE 'Asia/Bangkok'
      WHERE (SELECT delivery_id FROM idempotent) IS NOT NULL
      RETURNING *
    )
    SELECT * FROM upsert;
  `, [deliveryId, tenantId, employeeId, type, employeeId, latitude, longitude]);

  // Returns punch row if processed; null if duplicate delivery
  return result.rows[0] || null;
}
```

Why this is correct and safe:
- Idempotency insert is part of the same CTE → if delivery already exists, `idempotent` returns nothing and `upsert` is skipped.
- `latest` determines current open state within the same transaction snapshot, avoiding read-then-insert races.
- Partial unique index is a hard guarantee: if a race tries to create a second open punch, Postgres will raise an error; the transaction will roll back and the webhook handler will surface 500 (safe failure + retry).
- Server time (`NOW() AT TIME ZONE 'Asia/Bangkok'`) is authoritative; client timestamp is stored separately if provided.

### 3. Webhook handler (`server/src/routes/lineWebhook.ts`)

```ts
import express from 'express';
import crypto from 'crypto';
import { pool } from '../db';
import { upsertPunch } from '../db/queries/punchQueries';

const router = express.Router();

function verifySignature(body: Buffer, signature: string, channelSecret: string): boolean {
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(body)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

router.post('/line', async (req, res) => {
  const signature = req.headers['x-line-signature'] as string;
  const deliveryId = req.headers['x-line-delivery'] as string;
  const rawBody: Buffer = req.body; // must be raw buffer (use express.raw upstream)

  if (!signature || !deliveryId) {
    return res.status(400).json({ error: 'Missing LINE headers' });
  }

  if (!verifySignature(rawBody, signature, process.env.LINE_CHANNEL_SECRET!)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  const payload = req.body as { events?: Array<any> };
  const events = payload.events || [];

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      if (ev.type !== 'message' || ev.message?.type !== 'text') continue;
      const text = ev.message.text?.toLowerCase().trim();
      if (!['clock in', 'clock out'].includes(text)) continue;

      const type = text === 'clock in' ? 'in' : 'out';
      const lineUserId = ev.source?.userId;
      if (!lineUserId) continue;

      // Resolve tenant/employee mapping
      const map = await client.query(
        'SELECT tenant_id, id FROM employees WHERE line_user_id = $1',
        [lineUserId]
      );
      if (!map.rows[0]) continue;

      const { tenant_id, id: empId } = map.rows[0];

      await upsertPunch(client, {
        tenantId: tenant_id,
        employeeId: empId,
        deliveryId,
        type,
        latitude: ev.message?.location?.latitude,
        longitude: ev.message?.location?.longitude,
        clientTs: ev.message?.timestamp,
      });
    }

    await client.query('COMMIT');
    return res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook processing failed:', err);
    return res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
});

export default router;
```

Middleware requirement (app setup):
```ts
// Preserve raw body for signature verification on this route only

// Other routes can use JSON parser
app.use(express.json());
```
