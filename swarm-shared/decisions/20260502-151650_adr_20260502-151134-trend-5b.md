## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: deterministic workflow engine (DAG runner) on CF Workers with Supabase Postgres for state and audit; Redis for queue/cache; pre-built mock connectors for one broker and one data vendor; compliance guardrails as rule evaluation step; replayable runs with checkpointing; API to submit research/signal inputs and receive audit trail + execution intent; basic dashboard via API (no custom frontend).

## Open questions
- Which brokers/OMS/EMS and data vendors must be prioritized for v0.1 connectors?
- What deterministic policy DSL format should compliance rules use (JSON schema vs. simple expression language)?
- What latency SLA (p50/p95) is required for sub-second orchestration on free-tier infra?
- How will idempotency and exactly-once execution be guaranteed across edge Workers and Redis?
- What retention and encryption posture is required for immutable audit logs under regulations?
