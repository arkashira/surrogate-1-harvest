# Costinel / design

## Top-Hub Signal Panel — Production-Ready Incremental (<2h)

**Highest-value increment:** Add a lightweight, CDN-first Top-Hub Signal Panel that surfaces high-cost anomalies and RI/coverage gaps for the most-connected hub (e.g., `MOC`) without touching backend ingestion or mutating state. Ships as a frontend-only widget + static JSON feed.

**Why this wins:**
- Zero risk to existing pipelines (no HF API calls during runtime, no schema changes).
- Uses the CDN bypass pattern: pre-listed file paths embedded in `top-hub-signals.json`; panel fetches via `https://huggingface.co/datasets/.../resolve/main/...` (no auth, no rate limit).
- Aligns with “Sense + Signal — ไม่ Execute”: panel only renders signals and proposes; no execution.
- Reuses known hub insight pattern: review most-connected hub (`MOC`) before surfacing.

---

2. **Add Top-Hub Signal Panel component (frontend only)**
   - Location: `components/TopHubSignalPanel.tsx`
   - Behavior:
     - Fetch `public/data/top-hub-signals.json` (CDN-friendly, no auth).
     - Render hub name, timestamp, and signal cards.
     - Each card shows severity badge, title, impact, recommendation, and link.
     - No mutations; “Propose” button opens a prefilled proposal modal (non-executing).

3. **Embed panel into dashboard**
   - Insert `<TopHubSignalPanel />` into the main dashboard route (e.g., `pages/dashboard.tsx` or equivalent) below the summary bar.
   - Ensure responsive layout; skeleton while loading.

4. **Styling & UX polish (15 min)**
   - Use existing design tokens (colors, spacing).
   - Severity colors: high = red, medium = amber, low = blue.
   - Links open in new tab; no external JS dependencies.

5. **Validation & ship**
   - Verify CDN fetch works in dev and production.
   - Confirm no HF API calls in browser network tab.
   - Commit and deploy.
