# Costinel / ops

## Final Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h)

### What we ship
A small, non-blocking **Top-Hub Signal Panel** on the Costinel dashboard that:
- Defaults to the most-connected hub **MOC** (configurable via `HUB_NAME`).
- Shows **3 actionable, cost-impact proposals** pulled from the knowledge graph.
- Uses **CDN-first data strategy**:
  - A pre-generated `top-hub.json` (embedded at build) for instant load.
  - A lightweight runtime updater that fetches a small `hub-filelist.json` from CDN and then the latest signals file (no Authorization header, bypasses API rate limits).
- **Zero API calls during dashboard runtime** except the CDN fetches; 5-minute client-side cache with graceful fallback to embedded data.

### Why this is highest value
- Reuses existing knowledge-RAG/graph patterns (MOC hub).
- Avoids HF API rate limits by using CDN-only fetches.
- Adds immediate “sense + signal” visibility without execution.
- Fits Costinel philosophy (propose, don’t execute).
- Can ship in <2h with minimal footprint.

---

### File changes (all in `/opt/axentx/Costinel`)
- `public/data/top-hub.json` — generated at build/deploy time and committed (embedded fallback).
- `src/components/TopHubSignalPanel.tsx` — new React panel component.
- `src/lib/hubSignals.ts` — CDN fetcher + cache with embedded fallback.
- `scripts/build-hub-filelist.js` — one-time script to list files for a date folder and emit `hub-filelist.json` (run on Mac orchestration, not in production).
- `scripts/build-top-hub.js` — one-time script to produce `top-hub.json` from the latest signals (run on Mac orchestration or CI).
- Update dashboard page to include `<TopHubSignalPanel />`.
- Add `HUB_NAME` and optional `HUB_DATA_REPO` to `.env.local` (defaults provided).

---

### Runtime behavior
- On mount, panel **immediately renders** from `public/data/top-hub.json` (embedded).
- In parallel, panel loads `hub-filelist.json` (CDN) → picks latest `{date}/signals.json` (or similar) → fetches that payload (CDN).
- If CDN fetch succeeds and is newer, panel updates silently (5-minute cache).
- If CDN fails, panel continues showing embedded data (graceful degradation).
- Displays 3 proposals with title, impact estimate, and quick context.

---

### Build/ops notes
- `scripts/build-hub-filelist.js` is run manually (or via cron on Mac) after new knowledge-RAG ingestion. It uses HF API **once** (respecting rate limits) to list a single date folder and writes `hub-filelist.json` to the repo or uploads it to CDN.
- `scripts/build-top-hub.js` is run in CI or on Mac after ingestion to produce `public/data/top-hub.json` (committed or uploaded to CDN).
- Training/data ingestion pipelines remain unchanged.
- No Lightning or heavy compute used. Pure CDN reads.

## Code Snippets

### `scripts/build-hub-filelist.js` (run on Mac)
```js
#!/usr/bin/env node
// Usage: node scripts/build-hub-filelist.js --repo <datasets/owner/repo> --path <dateFolder> --out <hub-filelist.json>
// Lists files in a single folder (non-recursive) and emits a small JSON map for CDN fetches.

import { HfApi } from "@huggingface/hub";
import fs from "fs/promises";

const HUB_DATA_REPO = process.env.HUB_DATA_REPO || "datasets/axentx/knowledge-hub";
const FOLDER_PATH = process.argv[process.argv.indexOf("--path") + 1] || new Date().toISOString().slice(0, 10);
const OUT_FILE = process.argv[process.argv.indexOf("--out") + 1] || "hub-filelist.json";

async function main() {
  const api = new HfApi();
  // list_repo_tree non-recursive to avoid pagination/rate issues
  const tree = await api.listRepoTree({
    repo: HUB_DATA_REPO,
    path: FOLDER_PATH,
    recursive: false,
  });

  const files = (tree.files || []).map((f) => f.path);
  const payload = {
    repo: HUB_DATA_REPO,
    folder: FOLDER_PATH,
    files,
    generatedAt: new Date().toISOString(),
  };

  await fs.writeFile(OUT_FILE, JSON.stringify(payload, null, 2), "utf8");
  console.log(`Wrote ${files.length} files to ${OUT_FILE}`);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

### `scripts/build-top-hub.js` (run on Mac/CI)
```js
#!/usr/bin/env node
// Usage: node scripts/build-top-hub.js --repo <datasets/owner/repo> --path <dateFolder> --out <public/data/top-hub.json>
// Produces an embedded top-hub.json with top 3 proposals for instant dashboard load.

import { HfApi } from "@huggingface/hub";
import fs from "fs/promises";
import path from "path";

const HUB_DATA_REPO = process.env.HUB_DATA_REPO || "datasets/axentx/knowledge-hub";
const FOLDER_PATH = process.argv[process.argv.indexOf("--path") + 1] || new Date().toISOString().slice(0, 10);
const OUT_FILE = process.argv[process.argv.indexOf("--out") + 1] || "public/data/top-hub.json";

async function main() {
  const api = new HfApi();
  const tree = await api.listRepoTree({
    repo: HUB_DATA_REPO,
    path: FOLDER_PATH,
    recursive: false,
  });

  const files = (tree.files || []).map((f) => f.path);
  const signalsFile = files.find((f) => f.endsWith("signals.json")) || `${FOLDER_PATH}/signals.json`;

  // Fetch signals file via CDN (no auth)
  const signalsUrl = `https://huggingface.co/datasets/${HUB_DATA_REPO}/resolve/main/${signalsFile}`;
  const res = await fetch(signalsUrl);
  if (!res.ok) throw new Error(`Failed to fetch signals: ${res.status}`);
  const payload = await res.json();

  const signals = (payload.signals || []).sort((a, b) => {
    // Prefer impact_score if present; otherwise connection count or estimate
    const scoreA = a.impact_score || a.estimatedSavingsUSD || 0;
    const scoreB = b.impact_score || b.estimatedSavingsUSD || 0;
    return scoreB - scoreA;
  }).slice(0, 3);

  const topHub = {
    hub: payload.hub || "MOC",
    generatedAt: new Date().toISOString(),
    signals,
  };

  await fs.mkdir(path.dirname(OUT_FILE), { recursive: true });
  await fs.writeFile(OUT_FILE, JSON.stringify(topHub, null, 2), "utf8");
  console.log(`Wrote top-hub.json with ${signals.length} signals to ${OUT_FILE}`);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```
