# workio / ops

## Final consolidated implementation

**Core diagnosis (resolved)**
- LINE retries on timeouts/slow 2xx create duplicate punches and leave/OT events.  
- No tenant-scoped uniqueness for punches within a deduplication/idempotency window.  
- Missing structured logging, request tracing, and observability on webhook ingress.  
- Device clock skew can produce out-of-order timestamps and logically impossible sequences.  
- Stale “active” punches can remain open indefinitely and corrupt state/reporting.  
- No runtime env validation or operational alerting/metrics.

**Chosen approach**
- Add request-scoped structured logging and tracing.  
- Add idempotency middleware keyed by LINE event IDs (per-event) with a configurable TTL.  
- Add tenant-scoped punch deduplication guard (employee + tenant + type + time window).  
- Normalize and clamp timestamps to server time with bounded skew tolerance; reject or repair out-of-order punches.  
- Add a background safeguard to auto-close stale active punches (configurable max open duration).  
- Add runtime config validation and safe defaults.  
- Expose minimal metrics and structured logs for alerting (duplicate rate, latency, errors).

### 3) Idempotency middleware (memory-backed, replaceable with Redis/Postgres)

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { env } from '../config';
import { logger } from '../utils/logger';

type IdempotencyRecord = {
  responseBody: string;
  statusCode: number;
  createdAt: number;
};

const store = new Map<string, IdempotencyRecord>();

function pruneExpired() {
  const now = Date.now();
  const ttlMs = env.WEBHOOK_IDEMPOTENCY_TTL_SECONDS * 1000;
  for (const [key, rec] of store.entries()) {
    if (now - rec.createdAt > ttlMs) store.delete(key);
  }
}

setInterval(pruneExpired, 60_000).unref();

export function lineWebhookIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const events = req.body?.events;
  if (!Array.isArray(events) || events.length === 0) return next();

  // Per-event idempotency keys (safer than batch key)
  const keys = events.map((e: any) => `line:${e.eventId || e.timestamp || Date.now()}:${e.type}`);
  const ttlMs = env.WEBHOOK_IDEMPOTENCY_TTL_SECONDS * 1000;
  const now = Date.now();

  // If any event in the batch is already processed, short-circuit entire batch to avoid partial duplicates.
  for (const key of keys) {
    const existing = store.get(key);
    if (existing && now - existing.createdAt <= ttlMs) {
      logger.info({ key }, 'Idempotent replay detected, returning cached response');
      return res.status(existing.statusCode).send(JSON.parse(existing.responseBody));
    }
  }

  // Capture response to store after handler finishes
  const chunks: Buffer[] = [];
  const originalSend = res.send;
  const originalStatus = res.status;
  let capturedStatus = 200;

  (res as any).status = function (code: number) {
    capturedStatus = code;
    return originalStatus.call(this, code);
  };

  (res as any).send = function (body: any) {
    const bodyStr = typeof body === 'string' ? body : JSON.stringify(body);
    chunks.push(Buffer.from(bodyStr));
    // Store only successful 2xx responses for idempotency
    if (capturedStatus >= 200 && capturedStatus < 300) {
      for (const key of keys) {
        store.set(key, {
          responseBody: bodyStr,
          statusCode: capturedStatus,
          createdAt: now,
        });
      }
    }
    return originalSend.call(this, body);
  };

  (req as any)._lineIdempotencyKeys = keys;
  next();
}
```

### 4) Tenant-scoped punch deduplication + clock skew handling

import { logger } from '../utils/logger';
import { env } from '../config';

const dedupeWindowMs = () => env.PUNCH_DEDUPE_WINDOW_MIN * 60_000;
const maxSkewMs = () => env.MAX_CLOCK_SKEW_MIN * 60_000;

  employeeId: string,
  tenantId: string,
  type: 'in' | 'out',
  deviceTs: Date,
  location?: string
) {
  const serverNow = new Date();
  const skew = deviceTs.getTime() - serverNow.getTime();

  // Reject or clamp extreme skew
  if (Math.abs(skew) > maxSkewMs()) {
    logger.warn({ employeeId, tenantId, deviceTs, serverNow, skewMs: skew }, 'Clock skew exceeds tolerance, clamping timestamp');
    // Clamp to server time +/- max skew (choose server time for safety)
    deviceTs = new Date(serverNow.getTime() + (skew > 0 ? maxSkewMs() : -maxSkewMs()));
  }

    where: {
      employeeId,
      tenantId,
      type,

        gte: windowStart,
        lte: windowEnd,

  });
