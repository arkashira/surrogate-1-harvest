## Deployment
cf workers (edge registration + config distribution; free-tier; no origin infra)

## v0.1 scope
v0.1 ships a local stealth agent (Rust CLI) that conceals a SOCKS5 proxy behind a meek TLS stream, provides TUN bring-up/teardown with fail-closed killswitch, stores profiles in local SQLite, and emits opt-in encrypted session logs for WAU-ZB; no external servers required beyond CF Workers config endpoint (read-only).

## Open questions
- Which DPI signatures are highest risk in target regions (SNI vs. packet length vs. TUN MTU)?
- Can we rely on CF Workers free-tier limits for config/geo-fallback without leaking user IPs?
- What is the acceptable trade-off between stealth and latency for meek/domain-front transports?
- How will we perform reproducible, deterministic builds to minimize binary fingerprints?
- What minimal telemetry schema preserves WAU-ZB while protecting user privacy and deniability?
