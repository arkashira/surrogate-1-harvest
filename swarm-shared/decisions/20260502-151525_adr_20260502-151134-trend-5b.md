## Deployment

## v0.1 scope
v0.1 in 2 weeks: deterministic workflow engine on Cloudflare Workers with Supastore-backed audit log; one research→signal→risk-check→execution path for a single venue; replayable runs via input hashing; immutable audit entries; basic compliance guardrails (hard risk limit checks) and NSM-weighted success tracking; local Redis cache for hot limits; zero frontend (API-only); open-source deps only, deployable on free tiers.

## Open questions
- Determinism boundaries: which signal/risk inputs must be snapshotted to guarantee exact replay across versions?
- Latency SLOs: target p99 end-to-end orchestration time (sub-100ms vs sub-1s) and acceptable queue depth?
- Compliance scope: which regulatory regimes require immutable retention periods and what fields must be signed/tamper-evident?
- Connector SLAs: which broker/OMS/EMS APIs will be supported in v0.1 and what are their idempotency/retry semantics?
- Edge vs cloud split: should guardrails run at the edge (Workers) vs centralized (Supabase) for consistency vs latency?
- Replay sandboxing: how will deterministic replay be tested without side effects on live risk/execution state?
