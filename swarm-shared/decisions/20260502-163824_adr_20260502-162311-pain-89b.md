## Deployment
cf workers (edge) + hf space (config/coordination) + Android on-device (Rust core via JNI); avoids cloud VMs to stay free-tier-first

## v0.1 scope
v0.1 in 2 weeks: Android app with on-device Rust core (boringtun + tun/socks5 hider) that establishes an obfuscated session via CF Workers edge; Supabase for user/auth and RocksDB for ephemeral state; one mux strategy (masquerade-as-HTTPS over QUIC) and one tun-hide mode (namespace cloaking where available); basic WAUS/U telemetry to Workers KV; no multi-hop, no kill-switch, no MDM/Knox dependencies.

## Open questions
- Which tun-hiding primitives are safe and effective on stock Android 12+ without root or Knox (namespace cloaking vs null routes vs binder mediation)?
- What traffic-shaping parameters best mimic benign HTTPS/QUIC to evade DPI while preserving throughput/latency targets?
- How will key rotation and config distribution work over CF Workers + HF Space without revealing coordination endpoints to local probes?
- What on-device storage strategy keeps obfuscation keys and session state safe from app-level forensics (encrypted SQLite vs RocksDB with Android Keystore)?
