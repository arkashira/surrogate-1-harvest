## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres) + optional HF Space for Redis-backed workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk guardrail → execution stub) with immutable Postgres audit log, Redis-backed signal cache, replay via inputs_hash, and Cloudflare Workers deployment; includes one broker connector mock and one OMS/EMS stub, plus unit/integration tests for compliance guardrails and end-to-end latency measurement.

## Open questions
- Determinism vs. external non-idempotent broker APIs: how to guarantee replay without duplicate fills?
- Latency SLA target for end-to-end orchestration (sub-100ms vs. sub-1s) and acceptable queue depth?
- Compliance rule engine scope: static allow/deny lists vs. dynamic policy-as-code DSL?
- Retention and immutability guarantees for audit logs (on-chain hash anchoring vs. WORM Postgres)?
- Connector coverage priority: OMS/EMS first or direct broker APIs first?
