## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-binary client (Linux x86_64) that creates a local SOCKS5 listener, wraps traffic through a user-space TUN, applies pluggable obfuscation (mimic-TLS-fragmented profile), and enforces a fail-closed kill-switch; local SQLite stores session/telemetry; no server component — uses simple CF Worker endpoint as an innocuous reachability reflector for smoke tests; CLI-only with basic stdout logging and opt-in anonymized local metrics export.

## Open questions
- Which DPI signatures are most prevalent in target regions today (SNI, packet length/time, TTL, flow shape)?
- Can we rely on Cloudflare Workers as innocuous egress reflectors/proxies without introducing centralized blocking risk?
- What local permissions model is acceptable for TUN creation across desktop/mobile targets (root/admin vs. userspace tun/tap)?
- How will we validate stealth effectiveness in-week-1 without exposing real user telemetry during tests?
- What kill-switch granularity is required (per-app, system-wide, IPv4/IPv6) to prevent leaks while minimizing breakage?
