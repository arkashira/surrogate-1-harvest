# workio / quality

## 1) Diagnosis

- **No idempotency key enforcement** on clock-in/out endpoints: duplicate requests from flaky mobile networks create double punches for the same `(employee_id, punch_type, date)` causing payroll drift.
- **Non-transactional punch writes**: concurrent retries can insert two punches before the unique constraint is checked at commit time (read-committed race).
- **No server-side de-duplication window**: retries within seconds (common on mobile) are treated as valid second punches instead of idempotent repeats.
- **Missing punch update path**: if an employee clocks in twice (e.g., phone sleep/wake), second request should return existing punch rather than error or duplicate.
- **No idempotency table**: without storing request keys, retries across service restarts or deploy windows cannot be safely de-duplicated.

## 2) Proposed change

- **File**: `workio/server/src/routes/punch.ts` (or equivalent) — POST `/punch/clock-in` and `/punch/clock-out`.
- **Add**: `punch_idempotency` table with `(idempotency_key, employee_id, punch_id, created_at)` and unique index on `idempotency_key`.
- **Wrap**: punch insert in `SERIALIZABLE` (or `REPEATABLE READ` + explicit locking) transaction that first checks idempotency table, then upserts punch, then records idempotency key.
- **Scope**: single endpoint handler change + one migration + one model/helper; <2h.

### 3.2 Model helper (`server/src/db/idempotency.ts`)

```ts
import { PoolClient } from 'pg';
import { pool } from './db';

export async function getOrCreatePunchWithIdempotency(
  client: PoolClient,
  {
    idempotencyKey,
    employeeId,
    punchType,
    latitude,
    longitude,
    image,
  }: {
    idempotencyKey: string;
    employeeId: number;
    punchType: 'IN' | 'OUT';
    latitude?: number;
    longitude?: number;
    image?: string;
  }
) {
  // 1) Check idempotency table
  const idem = await client.query(
    'SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1',
    [idempotencyKey]
  );

  if (idem.rowCount > 0) {
    const punch = await client.query(
      'SELECT * FROM punches WHERE id = $1',
      [idem.rows[0].punch_id]
    );
    return { punch: punch.rows[0], existed: true };
  }

  // 2) Determine today date (no time) for duplicate prevention within same day+type
  const today = new Date().toISOString().split('T')[0];

  // 3) Try to find today's same-type punch (best-effort; idempotency key is source of truth)
  const existing = await client.query(
    `SELECT * FROM punches
     WHERE employee_id = $1 AND punch_type = $2 AND DATE(created_at) = $3
     ORDER BY created_at DESC LIMIT 1`,
    [employeeId, punchType, today]
  );

  let punchId: number;
  if (existing.rowCount > 0) {
    // Return existing punch for this day/type (soft idempotency)
    punchId = existing.rows[0].id;
  } else {
    // 4) Insert new punch
    const res = await client.query(
      `INSERT INTO punches (employee_id, punch_type, latitude, longitude, image, created_at)
       VALUES ($1, $2, $3, $4, $5, NOW())
       RETURNING id`,
      [employeeId, punchType, latitude, longitude, image]
    );
    punchId = res.rows[0].id;
  }

  // 5) Record idempotency key
  await client.query(
    'INSERT INTO punch_idempotency (idempotency_key, employee_id, punch_id) VALUES ($1, $2, $3)',
    [idempotencyKey, employeeId, punchId]
  );

  return { punch: (await client.query('SELECT * FROM punches WHERE id = $1', [punchId])).rows[0], existed: false };
}
```

### 3.3 Endpoint update (`server/src/routes/punch.ts`)

```ts
import express from 'express';
import { pool } from '../db/db';
import { getOrCreatePunchWithIdempotency } from '../db/idempotency';

const router = express.Router();

router.post('/clock-in', async (req, res) => {
  const { employeeId, latitude, longitude, image, idempotencyKey } = req.body;
  if (!idempotencyKey) {
    return res.status(400).json({ error: 'idempotencyKey required' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const { punch, existed } = await getOrCreatePunchWithIdempotency(client, {
      idempotencyKey,
      employeeId,
      punchType: 'IN',
      latitude,
      longitude,
      image,
    });
    await client.query('COMMIT');
    return res.json({ punch, existed });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Clock-in failed', err);
    // Unique violation on idempotency_key is safe retry
    if ((err as any)?.code === '23505') {
      // Concurrent duplicate: retry read
      const idem = await pool.query(
        'SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1',
        [idempotencyKey]
      );
      if (idem.rowCount > 0) {
        const punch = await pool.query('SELECT * FROM punches WHERE id = $1', [idem.rows[0].punch_id]);
        return res.json({ punch: punch.rows[0], existed: true });
      }
    }
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

// Same pattern for /clock-out (punchType: 'OUT')
```

- Include same `idempotencyKey` header/body on retry.
