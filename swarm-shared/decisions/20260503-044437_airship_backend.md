### 1) Pre-list file paths once (Mac orchestration) — *adopt Candidate 2*
- **Why:** Non-recursive per-folder listing avoids pagination overhead on large repos; emits explicit `cdn_url` for zero-auth loading.  
- **Action:**  
  Create `scripts/list_hf_date_folder.py` (run once per date folder after rate-limit window clears):

```python
#!/usr/bin/env python3
"""
Usage: python list_hf_date_folder.py <repo> <date_folder> > filelist.json
Example: python list_hf_date_folder.py datasets/axentx/surrogate-1 2026-05-03 > filelist.json
"""
import json
import sys
from huggingface_hub import HfApi

def main():
    if len(sys.argv) != 3:
        print("Usage: list_hf_date_folder.py <repo> <date_folder>", file=sys.stderr)
        sys.exit(1)

    repo = sys.argv[1].strip()
    folder = sys.argv[2].strip().rstrip("/")
    api = HfApi()

    entries = api.list_repo_tree(repo=repo, path=folder, recursive=False)
    files = []
    for e in entries:
        if e.type == "file":
            cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{folder}/{e.path}"
            files.append({"path": e.path, "cdn_url": cdn_url})

    print(json.dumps({"repo": repo, "folder": folder, "files": files}, indent=2))

if __name__ == "__main__":
    main()
```


### 2) CDN-only data loader (Lightning training) — *adopt Candidate 2*
- **Why:** Bypasses HF API auth checks entirely during training, eliminating 429s from data loading.  
- **Action:** Create `surrogate/data/cdn_stream.py`:

```python
# surrogate/data/cdn_stream.py
import json
import pyarrow.parquet as pq
import requests
from io import BytesIO
from torch.utils.data import IterableDataset

class CDNParquetStream(IterableDataset):
    """
    Loads Parquet shards from HF CDN (no auth).
    filelist.json format: {"files": [{"cdn_url": "...", "path": "..."}, ...]}
    Projects to {prompt, response} at parse time (schema heterogeneity safe).
    """

    def __init__(self, filelist_path, columns=("prompt", "response")):
        with open(filelist_path) as f:
            manifest = json.load(f)
        self.shards = [f["cdn_url"] for f in manifest["files"] if f["cdn_url"].endswith(".parquet")]
        self.columns = columns

    def __iter__(self):
        for url in self.shards:
            resp = requests.get(url, timeout=30)
            resp.raise_for_status()
            table = pq.read_table(BytesIO(resp.content), columns=self.columns)
            df = table.to_pandas()
            for _, row in df.iterrows():
                yield {"prompt": row["prompt"], "response": row["response"]}
```

### 3) Lightning Studio lifecycle resilience — *merge Candidates 1+2 with correctness*
- **Why:** Reuses running studios to avoid quota loss from recreation; restarts on idle-stop deterministically before `.run()`.  
- **Action:** Create `surrogate/lightning_utils.py`:

```python
# surrogate/lightning_utils.py
from lightning import Lightning, Teamspace, Machine

def get_or_create_studio(name: str, machine: Machine = Machine.L40S):
    """
    Reuse running studio if exists; else create.
    If stopped (idle-timeout), restart with target machine.
    """
    teamspace = Teamspace()
    running = [s for s in teamspace.studios if s.name == name and s.status == "running"]
    if running:
        print(f"Reusing running studio: {name}")
        return running[0]

    stopped = [s for s in teamspace.studios if s.name == name and s.status == "stopped"]
    if stopped:
        studio = stopped[0]
        print(f"Restarting stopped studio: {name}")
        studio.start(machine=machine)
        return studio

    print(f"Creating studio: {name}")
    return Lightning.create(
        name=name,
        machine=machine,
        create_ok=True,
    )
```

- **Launcher integration (in `train.py` or run script):**
