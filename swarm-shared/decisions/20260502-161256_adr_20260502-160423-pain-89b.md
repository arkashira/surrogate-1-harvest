## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust client (CLI) that: 1) creates a TUN interface and SOCKS5 listener, 2) applies traffic-shaping/obfuscation to mask signatures from local DPI probes, 3) enforces a kill-switch that blocks all non-stealth traffic on failure, 4) stores sessions/config locally in SQLite, and 5) emits anonymized WAU-ZB summaries; no server component required, runs on free-tier dev machines.

## Open questions
- Which DPI signatures (SNI, packet length/time distributions, TUN MTU patterns) are most observable in target regions?
- Can we rely on Cloudflare Workers for innocuous bootstrap/relay without creating centralized chokepoints?
- What local permissions model (TUN, raw sockets) is feasible across desktop/mobile targets while remaining zero-config?
- How will we validate WAU-ZB telemetry without leaking user identifiers or creating tracking surfaces?
- Fail-closed granularity: block all traffic vs. fallback to non-stealth path — what is the safe default per threat model?
