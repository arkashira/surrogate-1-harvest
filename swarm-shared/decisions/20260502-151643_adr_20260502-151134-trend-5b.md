## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single workflow template (research→signal→risk check→execution) with Postgres-backed run state, Redis Streams for replayable queueing, one broker connector (sandbox), one risk/compliance guardrail (max position size), immutable audit entries, and edge API on CF Workers; supports manual trigger and deterministic replay of a run from stored context snapshot.

## Open questions
- What broker/OMS connectors are must-have for v0.1 and what auth/credential model will they require?
- Which compliance rules are non-negotiable (venue limits, concentration, blacklists) and who owns rule definitions?
- What is the deterministic replay SLA (time horizon for input snapshot retention and idempotency guarantees)?
- How will we measure and weight fee-generating runs for the NSM and what telemetry is required?
- Cold-start and queue latency targets on CF Workers vs self-hosted Redis — which path meets sub-second orchestration?
