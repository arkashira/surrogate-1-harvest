## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk guardrail check→execution intent) with Postgres audit trail, Redis cache for signals, Cloudflare Workers orchestrator, and one mock broker connector; includes replay endpoint and basic compliance rule (hard limit + blacklist).

## Open questions
- Deterministic replay: which inputs must be snapshotted (market data versions, config hashes) to guarantee bitwise repeatability?
- Latency SLA target for end-to-end (research→signal→risk→execution) on free-tier infra — sub-500ms or sub-100ms?
- Compliance rule hot-reload vs. immutable versioning — can we accept dynamic rule updates without breaking replayability?
- Broker/OMS/EMS connector security model: secrets storage and rotation on free-tier (Workers KV vs. Supabase Vault)?
- Retention and cryptographic tamper evidence strategy for audit entries (WAL export, hash-chaining, or periodic notarization)?
