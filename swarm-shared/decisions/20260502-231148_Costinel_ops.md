# Costinel / ops

## Implementation Plan — Costinel `GET /api/v1/cost-anomaly/signal/top-hub`

### Scope (≤2 h)
- Add a **read-only** endpoint that returns the most-connected knowledge-graph hub (e.g., `"MOC"`) as a cost-anomaly signal.
- Optional `?for_date=YYYY-MM-DD` (defaults to today).
- No mutations, no side effects. Reuse existing KG assets.
- Follow pattern: **top-hub doc insight** + **business research / knowledge-rag** integration.

---

### File changes
1. **Backend route**  
   `src/routes/cost_anomaly.py` (or equivalent) — add `GET /api/v1/cost-anomaly/signal/top-hub`.

2. **Service layer**  
   `src/services/top_hub_signal.py` — thin service that:
   - Accepts `for_date`
   - Queries knowledge-graph for top hub (highest degree/centrality)
   - Returns structured signal payload

3. **Knowledge-graph adapter** (reuse existing)  
   `src/lib/kg_adapter.py` — expose `get_top_hub(for_date: date) -> dict`.

4. **Tests** (optional but recommended)  
   `tests/test_top_hub_signal.py` — one happy-path test.

from fastapi import APIRouter, Depends, Query
from datetime import date
from src.services.top_hub_signal import get_top_hub_signal
from src.schemas.top_hub import TopHubSignalResponse

router = APIRouter(prefix="/api/v1/cost-anomaly", tags=["cost-anomaly"])

@router.get("/signal/top-hub", response_model=TopHubSignalResponse)
async def top_hub_signal(
    for_date: date = Query(default_factory=date.today)

from datetime import date, datetime
from src.lib.kg_adapter import get_top_hub

def get_top_hub_signal(for_date: date) -> TopHubSignalResponse:
    hub_info = get_top_hub(for_date=for_date)

    return TopHubSignalResponse(
        signal_type="cost_anomaly_top_hub",
        hub=hub_info["hub"],
        score=hub_info["score"],
        context_docs=hub_info.get("context_docs", []),

### Notes & decisions
- **Read-only**: No writes, no cache invalidation, no external mutations.
- **Reuse KG**: Leverages existing knowledge-rag assets; no new ingestion pipeline.
- **Extensible**: Adapter layer allows swapping KG backend (Neo4j, NetworkX, etc.) without touching route/service.
- **Observability**: Add logging/metrics in service if desired (not required for MVP).
