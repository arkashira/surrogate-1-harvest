## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk guardrail check→execution stub) with immutable Postgres audit trail (Supabase), Redis-backed queue and cache, edge-deployed FastAPI orchestrator on Cloudflare Workers free tier, idempotent run submission, replay endpoint for rejected runs, and three pre-built connector stubs (broker, OMS, data vendor) with encrypted credential storage and latency telemetry.

## Open questions
- What is the deterministic replay SLA (max acceptable recompute time) for a rejected workflow?
- Which compliance rule engine approach: embedded Python DSL vs. external policy-as-code (e.g., Open Policy Agent) on free-tier?
- Idempotency key strategy: client-supplied vs. server-generated — and required TTL for Redis dedupe?
- Broker/OMS connector authentication model: OAuth2 refresh vs. signed JWT vs. API keys — and vault strategy on free infra?
- Latency budget per stage (research→signal→risk→execution) to meet sub-second NSM target under free-tier compute limits?
