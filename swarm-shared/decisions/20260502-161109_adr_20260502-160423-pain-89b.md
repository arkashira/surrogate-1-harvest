## Deployment
cf workers + hf space (client binary distributed via HF space; lightweight egress bridge as CF worker; no servers to manage)

## v0.1 scope
v0.1 in 2 weeks: single-profile local SOCKS5 stealth client (Rust CLI) with TLS-fingerprint mimicry and optional meek HTTP path; local SQLite state; basic kill-switch (block-all-on-drop for the session process); no TUN yet; CF Worker bridge for egress; HF Space distribution page and release binaries; client-side session/telemetry logging (opt-in) to measure WAU-ZB.

## Open questions
- Which DPI fingerprints (JA3/JA4, TTL/Hop, packet size distributions) must we mimic to evade current probes in target regions?
- Can CF Workers egress to residential endpoints without CF WAF/ASN flags that trigger blocking?
- What is the acceptable local port range and socket behavior to avoid SOCKS5 banner detection on localhost?
- How will we implement fail-closed kill-switch across platforms (Windows/Linux/macOS) without admin/root at runtime?
- What minimal, non-PII telemetry schema will satisfy WAU-ZB measurement while preserving deniability and privacy?
