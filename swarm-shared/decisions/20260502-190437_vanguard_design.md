### 1) Diagnosis (final)

- No canonical discovery entrypoint to surface top-hub insights before planning — violates `#knowledge-rag #graph #hub`.
- Missing CDN-bypass file-list strategy for HF datasets — future training ingestion will hit 429s instead of using `resolve/main/` CDN fetches.
- No Lightning Studio reuse guard — risk of recreating studios and burning quota.
- No single-pass executable to run market analysis + top-hub knowledge-rag — violates `#business-research #knowledge-rag #graph`.
- No local data projection to enforce `{prompt,response}` schema before ingestion — violates `#ingestion #schema #surrogate-1`.
- Scripts must have proper shebang, be executable, and invoked via Bash to avoid execution errors.

### 2) Proposed change (final)

Create `/opt/axentx/vanguard/discovery/run_discovery.sh` as the single-pass, executable entrypoint that:

1. Runs market analysis (if present) and queries top-hub insights via knowledge-rag.
2. Lists HF dataset files for the target date folder (non-recursive) and emits `file_list.json` for CDN-bypass training.
3. Reuses a running Lightning L40S Studio or restarts a stopped one; creates only if absent.
4. Projects local data into `{prompt,response}` parquet under `batches/mirror-merged/{date}/` with deterministic slug filenames.
5. Is executable (`chmod +x`), has Bash shebang, and is invoked via Bash.

### 3) Implementation (final)

```bash
#!/usr/bin/env bash
# /opt/axentx/vanguard/discovery/run_discovery.sh
# Single-pass discovery: top-hub insights, HF CDN-bypass file list,
# Lightning Studio reuse, local data projection.
#
# Usage:
#   bash run_discovery.sh [--hf-repo <repo>] [--date-folder <yyyy-mm-dd>]

set -euo pipefail
SHELL=/bin/bash

REPO_ROOT="/opt/axentx/vanguard"
DISCOVERY_ROOT="${REPO_ROOT}/discovery"
OUTPUT_DIR="${DISCOVERY_ROOT}/output"

# Parse simple positional args (or extend with getopt if needed)
HF_REPO="${1:-datasets/example-repo}"
DATE_FOLDER="${2:-$(date +%Y-%m-%d)}"

mkdir -p "${OUTPUT_DIR}"

echo "=== vanguard discovery ==="
echo "HF_REPO=${HF_REPO}"
echo "DATE_FOLDER=${DATE_FOLDER}"

# 1) Business research + top-hub insight (best-effort)
if command -v granite-business-research.sh >/dev/null 2>&1; then
  echo "[INFO] Running market analysis..."
  granite-business-research.sh --output "${OUTPUT_DIR}/market_insights.json" || true
fi

if command -v knowledge-rag >/dev/null 2>&1; then
  echo "[INFO] Querying top-hub (e.g., MOC)..."
  knowledge-rag query --top-hub --output "${OUTPUT_DIR}/top_hub_insights.json" || true
fi

# 2) HF CDN-bypass: list files once, save for training script
echo "[INFO] Listing HF dataset files (non-recursive, single API call)..."
export HF_REPO DATE_FOLDER OUTPUT_DIR
python3 - <<'PY'
import json, os
from huggingface_hub import HfApi

repo = os.getenv("HF_REPO")
date_folder = os.getenv("DATE_FOLDER")
out_path = os.path.join(os.getenv("OUTPUT_DIR", "."), "file_list.json")

api = HfApi()
items = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)
files = [f.rfilename for f in items if f.rfilename]

with open(out_path, "w") as f:
    json.dump({"repo": repo, "date_folder": date_folder, "files": files}, f, indent=2)
print(f"Wrote {len(files)} files to {out_path}")
PY

# 3) Lightning Studio reuse / restart guard
echo "[INFO] Checking Lightning Studio..."
export REPO_ROOT
python3 - <<'PY'
import os
from lightning import Studio, Teamspace, Machine

studio_name = "vanguard-l40s"
teamspace = Teamspace()
running = None
for s in teamspace.studios:
    if s.name == studio_name and s.status == "running":
        running = s
        break

if running:
    print(f"Reusing running studio: {running.name} (id={running.id})")
else:
    stopped = [s for s in teamspace.studios if s.name == studio_name and s.status == "stopped"]
    if stopped:
        target = stopped[0]
        print(f"Restarting stopped studio: {target.name}")
        target.start(machine=Machine.L40S)
    else:
        print("Creating new L40S studio (free-tier fallback may select L40S or smaller)")
        Studio(
            name=studio_name,
            machine=Machine.L40S,
            create_ok=True
        )
PY

# 4) Local data projection -> surrogate-1 schema
echo "[INFO] Projecting local data to surrogate schema..."
export REPO_ROOT DATE_FOLDER
python3 - <<'PY'
import pandas as pd, os, glob, hashlib

SRC_DIR = os.path.join(os.getenv("REPO_ROOT", "."), "data")
DATE_FOLDER = os.getenv("DATE_FOLDER", "")
OUT_DIR = os.path.join(os.getenv("REPO_ROOT", "."), "batches", "mirror-merged", DATE_FOLDER)
os.makedirs(OUT_DIR, exist_ok=True)

for fp in glob.glob(os.path.join(SRC_DIR, "**"), recursive=True):
    if not os.path.isfile(fp):
        continue
    df = None
    try:
        df = pd.read_parquet(fp)
    except Exception:
        try:
            df = pd.read_csv(fp)
        except Exception:
            continue

    if "prompt" not in df.columns or "response" not in df.columns:
        continue

    projected = df[["prompt", "response"]].dropna(subset=["prompt", "response"])
    if projected.empty:
        continue

    slug = hashlib.sha256(fp.encode()).hexdigest()[:16]
    out_fp = os.path.join(OUT_DIR, f"{slug}.parquet")
    projected.to_parquet(out_fp, index=False)
    print(f"Projected {len(projected)} rows -> {out_fp}")
PY

echo "=== vanguard discovery complete ==="
```

Make executable:

```bash
chmod +x /opt/axentx/vanguard/discovery/run_discovery.sh
```
