# surrogate-1 / design

## Implementation Plan (≤2h)

**Highest-value improvement**: Add a pre-flight snapshot generator (`bin/snapshot.sh`) that lists all dataset files once, embeds the file manifest into training, and enables CDN-only downloads during parallel shard processing. This eliminates HF API rate limits during training and ensures deterministic, reproducible file lists for each shard.

### Steps (1h 45m total)

1. **Create `bin/snapshot.sh`** (20m)  
   - Uses `huggingface_hub` to call `list_repo_tree(recursive=False)` on the target date folder (or root)  
   - Outputs `snapshot-<date>.json` with `{ "files": ["path1", "path2", ...], "repo": "...", "ts": "..." }`  
   - Accepts `DATE` and `REPO` env vars (defaults to `axentx/surrogate-1-training-pairs` and today)  
   - Saves to `snapshots/` and symlinks latest as `snapshot-latest.json`

2. **Update `bin/dataset-enrich.sh`** (30m)  
   - Accept optional `SNAPSHOT_FILE` arg; if provided, reads file list from JSON instead of calling `list_repo_*`  
   - Deterministic shard assignment: `hash(slug) % 16 == SHARD_ID` (same as before)  
   - Skip files not in snapshot (guard against stale snapshots)

3. **Add `bin/build-manifest.py`** (20m)  
   - Reads snapshot, expands folders if needed (one API call per folder), produces per-shard manifest files: `manifests/shard-<N>.jsonl`  
   - Each line: `{"file": "path", "shard": N, "slug": "..."}`  
   - Used by training script to know exactly which files to fetch via CDN

4. **Update training script / launcher** (30m)  
   - Before training, run `snapshot.sh` on Mac (or reuse existing snapshot)  
   - Embed `snapshot-latest.json` into container or pass as mounted file  
   - Data loader uses `hf_hub_download(repo, file)` for each file in its shard manifest — zero `list_repo_*` calls during training  
   - Add retry/backoff for 429 on CDN (should be rare) and wait 360s on API 429

5. **Workflow changes** (25m)  
   - `ingest.yml`: add a pre-step that runs `snapshot.sh` and uploads artifact `snapshot-<date>.json`  
   - Each matrix job downloads snapshot artifact and uses it via `SNAPSHOT_FILE`  
   - Keep existing 16-shard matrix; now each shard only processes files assigned by deterministic hash

6. **Tests & guards** (20m)  
   - Add small smoke test: `snapshot.sh --dry-run` prints counts  
   - Validate snapshot JSON schema with `jq` in CI  
   - Ensure `chmod +x` on new scripts and Bash shebang (`#!/usr/bin/env bash`)

### `bin/snapshot.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="${REPO:-axentx/surrogate-1-training-pairs}"
DATE="${DATE:-$(date +%Y-%m-%d)}"
OUTDIR="${OUTDIR:-snapshots}"
TS=$(date -u +%Y%m%dT%H%M%SZ)

mkdir -p "$OUTDIR"

python3 - "$REPO" "$DATE" "$OUTDIR/snapshot-${DATE}.json" <<'PY'
import json, os, sys
from huggingface_hub import HfApi

repo = sys.argv[1]
date_prefix = sys.argv[2]
out_path = sys.argv[3]

api = HfApi()
# List top-level or date folder non-recursively
tree = api.list_repo_tree(repo=repo, path=date_prefix, recursive=False)
files = [item.rfilename for item in tree if item.type == "file"]

# If tree is empty, try repo root and filter by date prefix
if not files:
    tree = api.list_repo_tree(repo=repo, path="", recursive=False)
    files = [item.rfilename for item in tree if item.type == "file" and item.rfilename.startswith(date_prefix)]

manifest = {
    "repo": repo,
    "date": date_prefix,
    "ts": os.environ.get("TS", "unknown"),
    "files": sorted(files)
}

with open(out_path, "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Wrote {len(files)} files to {out_path}")
PY

# Symlink latest
ln -sf "snapshot-${DATE}.json" "${OUTDIR}/snapshot-latest.json"
echo "Snapshot created: ${OUTDIR}/snapshot-${DATE}.json"
```

### `bin/build-manifest.py`
```python
#!/usr/bin/env python3
import json, hashlib, os, sys
from pathlib import Path

def slug_for(path: str) -> str:
    # Remove extension, use basename as slug (adjust to your schema)
    return Path(path).stem

def shard_for(slug: str, n_shards: int = 16) -> int:
    h = hashlib.md5(slug.encode()).hexdigest()
    return int(h, 16) % n_shards

def main():
    snapshot_path = sys.argv[1] if len(sys.argv) > 1 else "snapshots/snapshot-latest.json"
    out_dir = Path(sys.argv[2] if len(sys.argv) > 2 else "manifests")
    n_shards = int(os.environ.get("N_SHARDS", "16"))

    out_dir.mkdir(parents=True, exist_ok=True)

    with open(snapshot_path) as f:
        data = json.load(f)

    files = data.get("files", [])
    by_shard = {i: [] for i in range(n_shards)}

    for fpath in files:
        slug = slug_for(fpath)
        shard = shard_for(slug, n_shards)
        by_shard[shard].append({"file": fpath, "slug": slug})

    for shard, items in by_shard.items():
        out_path = out_dir / f"shard-{shard}.jsonl"
        with open(out_path, "w") as f:
            for item in items:
                f.write(json.dumps(item) + "\n")
        print(f"Shard {shard}: {len(items)} files -> {out_path}")

if __name__ == "__main__":
    main()
```

### Update to `bin/dataset-enrich.sh` (excerpt)
```bash
#!/usr/bin/env bash
set -euo pipefail

SHARD_ID="${SHARD_ID:-0}"
N_SHARDS="${N_SHARDS:-16}"
SNAPSHOT_FILE="${SNAPSHOT_FILE:-}"

if [[ -n "$SNAPSHOT_FILE" && -f "$SNAPSHOT_FILE" ]]; then
  echo "Using snapshot: $SNAPSHOT_FILE"
  # Use pre-built per-shard manifest if available
  MANIFEST="manifests/shard-${SHARD_ID}.jsonl"
  if [[ -f "$MANIFEST" ]]; then
    echo "Using per-shard manifest: $MANIFEST"
    mapfile -t FILES < <(jq -r '.file' "$MANIFEST")
  else
    # Fallback: filter snapshot by shard
    mapfile -t FILES < <(python3 -c "
import json, hashlib, sys
with open('$SNAPSHOT_FILE') as f:
    data = json.load(f)
for p in data['files']:
    slug = p.rsplit('.', 1)[0]
    h = hashlib.md5(slug.encode()).hexdigest()
    if int(h, 16) % $N_SHARDS == $SHARD_ID:
        print(p)
")
  fi
else
  echo "No snapshot provided; falling back to repo listing (may hit API limits)"
  # existing repo listing logic...
fi

# Process FILES array as before
```
