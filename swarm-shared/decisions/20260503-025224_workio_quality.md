# workio / quality

## Final synthesized solution

**Core problem**: duplicate punches can occur from LINE webhook retries, frontend retries, and races between optimistic UI and server replays. The fix must be **server-side, strongly consistent, and operationally observable**, with clear frontend contracts.

---

## 1. Diagnosis (resolved)

- **Server lacks idempotency enforcement**: no `Idempotency-Key` requirement and no durable idempotency table → retries create duplicates.
- **No domain-level uniqueness**: multiple punches per user per punch_type per calendar date are allowed by schema.
- **Race tolerance missing**: frontend optimistic state and webhook replays are not reconcilable without a durable token.
- **No audit/reconciliation**: ops cannot detect or safely resolve duplicates.
- **Frontend retries without coordination**: client can retry without a stable key that the server can recognize.

**Resolution priority**: server-side correctness first; frontend changes are thin wrappers around server contract.

---

## 2. DB schema (single source of truth)

```sql
-- Idempotency table: durable, replay-safe
CREATE TABLE IF NOT EXISTS idempotency_keys (
  id            SERIAL PRIMARY KEY,
  key           TEXT NOT NULL,
  user_id       INTEGER NOT NULL,
  payload_hash  TEXT NOT NULL,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  response_status INTEGER NOT NULL,
  response_body   JSONB NOT NULL,
  CONSTRAINT uniq_idempotency_key UNIQUE (key)
);

-- Prevent duplicate punches at domain level
-- One punch_type per user per calendar date (IN/OUT/BREAK_IN/BREAK_OUT)
CREATE UNIQUE INDEX IF NOT EXISTS idx_user_punch_date_type
  ON punches (user_id, punch_type, DATE(timestamp));
```

**Why this works**:
- Unique index enforces exactly-once at DB level (strong correctness).
- Idempotency table stores original response so replays are bitwise identical (including status code and body).
- Payload hash detects semantic changes for same key (defensive).

## 3. Service layer — idempotent, transactional, observable

```ts
// workio/server/src/services/punchService.ts
import { pool } from '../db';
import crypto from 'crypto';

export interface CreatePunchInput {
  userId: number;
  punchType: 'IN' | 'OUT' | 'BREAK_IN' | 'BREAK_OUT';
  timestamp?: Date;
  latitude?: number;
  longitude?: number;
}

export async function createPunchWithIdempotency(
  input: CreatePunchInput,
  idempotencyKey: string
) {
  if (!idempotencyKey || idempotencyKey.length > 255) {
    throw new Error('Idempotency-Key required and must be <=255 chars');
  }

  const payloadHash = crypto
    .createHash('sha256')
    .update(JSON.stringify({ ...input, idempotencyKey }))
    .digest('hex');

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Return original response if key exists (exact replay)
    const existingIdem = await client.query(
      `SELECT response_status, response_body FROM idempotency_keys WHERE key = $1`,
      [idempotencyKey]
    );
    if (existingIdem.rows.length > 0) {
      await client.query('ROLLBACK');
      return {
        status: existingIdem.rows[0].response_status,
        body: existingIdem.rows[0].response_body,
        replayed: true,
      };
    }

    // 2) Insert punch (unique index prevents duplicates)
    const punchDate = input.timestamp || new Date();
    const insertPunch = await client.query(
      `INSERT INTO punches (user_id, punch_type, timestamp, latitude, longitude)
       VALUES ($1, $2, $3, $4, $5)
       RETURNING *`,
      [input.userId, input.punchType, punchDate, input.latitude, input.longitude]
    );

    const punch = insertPunch.rows[0];
    const responseBody = { punch, ok: true };

    // 3) Persist idempotency record
    await client.query(
      `INSERT INTO idempotency_keys (key, user_id, payload_hash, response_status, response_body)
       VALUES ($1, $2, $3, $4, $5)`,
      [idempotencyKey, input.userId, payloadHash, 201, responseBody]
    );

    await client.query('COMMIT');
    return { status: 201, body: responseBody, replayed: false };
  } catch (err: any) {
    await client.query('ROLLBACK');

    // Unique violation on punch (domain duplicate)
    if (err.code === '23505' && err.constraint === 'idx_user_punch_date_type') {
      const existing = await pool.query(
        `SELECT * FROM punches WHERE user_id = $1 AND punch_type = $2 AND DATE(timestamp) = $3`,
        [input.userId, input.punchType, punchDate.toDateString()]
      );
      return { status: 409, body: { ok: false, error: 'Duplicate punch', punch: existing.rows[0] }, replayed: false };
    }

    // Idempotency key race (another request won)
    if (err.code === '23505' && err.constraint === 'idempotency_keys_key_key') {
      const existing = await pool.query(
        `SELECT response_status, response_body FROM idempotency_keys WHERE key = $1`,
        [idempotencyKey]
      );
      if (existing.rows.length > 0) {
        return { status: existing.rows[0].response_status, body: existing.rows[0].response_body, replayed: true };
      }
    }

    throw err;
  } finally {
    client.release();
  }
}
```

**Key behaviors**:
- **Exactly-once**: unique index + idempotency table guarantee no duplicates.
- **Replay-safe**: original response returned for same key (status + body).
- **Conflict handling**: 409 for domain duplicates; transparent replay for idempotency races.
- **Transactional**: punch + idempotency record committed atomically.

## 4. Route — enforce contract

```ts
// workio/server/src/routes/punch.ts
import express from 'express';
import { createPunchWithIdempotency } from '../services/punchService';

const router = express.Router();

router.post('/', async (req, res) => {
  const idempotencyKey =
    req.header('Idempotency-Key') || req.header('idempotency-key');
  const { userId, punchType, timestamp, latitude, longitude } = req.body;

  if (!userId || !punchType) {
    return res.status(400).json({ ok: false, error: 'userId and punchType required' });
  }

  if (!idempotencyKey) {
    return res.status(400).json({ ok: false, error: 'Idempotency-Key header required' });
  }

  try {
    const result = await createPunchWithIdempotency(
      { userId, punchType, timestamp: timestamp ? new Date(timestamp) : undefined, latitude, longitude },
      idempotencyKey
    );

    return res.status(result.status).json(result.body);
  } catch (err: any) {
    console.error('Punch creation failed', err);
    return res.status(500).json({ ok: false, error: 'Internal server error' });
  }
});

export default router;
```

**Contract**:
- `Idempotency-Key` header **required** (400 if missing).
- Server owns idempotency semantics; clients must provide stable keys per logical event.
