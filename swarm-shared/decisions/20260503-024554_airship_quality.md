## Final Synthesis (Best Parts + Corrected Contradictions)

**Chosen approach**: CDN-first training slice that lists once, trains many, reuses studio, strict schema.  
**Key corrections vs candidates**:
- Use **POST** (not GET) for listing to avoid token leakage in logs/metrics.  
- Require **non-recursive list + explicit date folder** (both candidates imply this; enforce it).  
- **Do not embed file-list.json inside training package**; pass by path so daily runs can swap lists without rebuild.  
- **Lightning Studio reuse must tolerate idle-killed state**: detect stopped, restart same machine, avoid quota churn.  
- **Schema projection happens at training load time** (not mandatory pre-upload), but enforce `{prompt,response}` only in loader; keep raw HF copy untouched for traceability.  
- **Cron guardrails must set `SHELL=/bin/bash` and use `bash` invocation** (Candidate 1 correct; Candidate 2 omitted).

## Implementation Plan (Actionable, ≤2h)

### 1) Surrogate: CDN file-list endpoint + UI (45m)
- Endpoint: `POST /api/training/list-hf-folder`
  - Body: `{ "repo": "datasets/...", "path": "2026-05-03", "token": "<HF_READ>" }`
  - Calls `list_repo_tree(repo, path, recursive=False, token)` once.
  - Validates `path` is a date-like folder (prevent recursive listing).
  - Returns `{ "repo": repo, "path": path, "files": [...], "saved": "file-list-YYYY-MM-DD.json" }`
  - Saves to `training_lists/` (git-ignored).
- UI: `/training/cdn-ingest`
  - Inputs: repo, path, HF token (masked).
  - Actions: “List folder & save file-list” (shows count + download), “Start Lightning training (reuse studio)”.

### 2) Lightning training: CDN loader + strict schema (45m)
- CLI: `train.py --file-list <path> [--repo <override>] [--epochs 1]`
- Loader (`CDNParquetDataset`):
  - Read `file-list.json` → repo, path, files.
  - For each file: `url = f"https://huggingface.co/datasets/{repo}/resolve/main/{f}"`
  - Stream via `requests.get(url, timeout=30)` (no auth header; CDN).
  - Parse parquet → project to `{prompt, response}` (drop others).
  - Yield examples.
- Caching: none (CDN is fast; avoids stale local state).

### 3) Lightning Studio: reuse + idle-killed restart (30m)
- Name studio deterministically: `surrogate-training`.
- Before run:
  - List team studios; if name exists and status == `running` → reuse.
  - If status == `stopped` (idle-killed) → start with `Machine.L40S` (or same machine).
  - If not exist → create with `Machine.L40S`.
- Run command: `python train.py --file-list ... --epochs 1`.
- Do not recreate studio each run (saves quota, avoids cold-start churn).

### 4) Cron/orchestration guardrails (15m)
- Wrapper: `scripts/trigger-cdn-ingest.sh`
  - `#!/usr/bin/env bash`
  - `set -euo pipefail`
  - `SHELL=/bin/bash` (crontab requirement).
  - Accept args: repo, path, token (env fallback).
  - Call listing API, then invoke Lightning reuse/run.
- Crontab line:
  ```

  0 2 * * * bash /opt/axentx/airship/surrogate/scripts/trigger-cdn-ingest.sh "datasets/org/dataset" "2026-05-03" >> /var/log/cdn-ingest.log 2>&1
  ```

### 5) Schema/projection policy (15m)
- Raw HF uploads keep full columns (for traceability).
- Training enforces projection at load time to `{prompt, response}`.
- Filename convention for processed batches (optional): `batches/mirror-merged/{date}/{slug}.parquet` (no extra metadata columns).

### Surrogate API (FastAPI) — `surrogate/api/training.py`
```python
import json
from pathlib import Path

from fastapi import APIRouter, HTTPException
from huggingface_hub import list_repo_tree

router = APIRouter()

@router.post("/list-hf-folder")
async def list_hf_folder(repo: str, path: str, token: str):
    """
    List top-level files in a date folder (non-recursive) and save file-list.json.
    repo example: "datasets/your-org/your-dataset"
    path example: "2026-05-03"
    """
    try:
        tree = list_repo_tree(repo=repo, path=path, recursive=False, token=token)
        files = sorted(f.rfilename for f in tree if f.rfilename.startswith(f"{path}/"))
        out_name = f"file-list-{path.replace('/', '-')}.json"
        out_path = Path("training_lists") / out_name
        out_path.parent.mkdir(exist_ok=True, parents=True)
        out_path.write_text(json.dumps({"repo": repo, "path": path, "files": files}, indent=2))
        return {"repo": repo, "path": path, "files": files, "saved": str(out_path)}
    except Exception as exc:
        raise HTTPException(status_code=500, detail=str(exc))
```

### Lightning CDN loader — `surrogate/training/train.py`
```python
import json
import requests
import pyarrow.parquet as pq
from io import BytesIO
from torch.utils.data import IterableDataset

HF_CDN = "https://huggingface.co/datasets"

class CDNParquetDataset(IterableDataset):
    def __init__(self, file_list_path, repo=None, columns=("prompt", "response")):
        with open(file_list_path) as f:
            cfg = json.load(f)
        self.repo = repo or cfg.get("repo")
        if not self.repo:
            raise ValueError("repo must be provided or present in file-list.json")
        self.files = cfg["files"]
        self.columns = columns

    def __iter__(self):
        for f in self.files:
            url = f"{HF_CDN}/{self.repo}/resolve/main/{f}"
            resp = requests.get(url, timeout=30)
            resp.raise_for_status()
            table = pq.read_table(BytesIO(resp.content))
            # Project to {prompt, response}
            table = table.select(self.columns)
            for batch in table.to_batches(max_chunksize=1024):
                cols = {c: batch.column(c).to_pylist() for c in self.columns}
                for i in range(len(cols[self.columns[0]])):
                    yield {c: cols[c][i] for c in self.columns}
```
