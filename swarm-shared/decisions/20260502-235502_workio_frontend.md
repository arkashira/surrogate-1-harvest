# workio / frontend

## 1. Diagnosis

- Clock-in/out UI has no optimistic update or rollback — failed LINE webhook calls leave spinner forever or silently fail, creating ambiguous state and duplicate retries.
- No local pending queue for offline/low-connectivity scenarios — users can’t punch when network is flaky and lose state on reload.
- Missing idempotency key on client requests — retries from UI can create duplicate punches before server dedupe is confirmed.
- No visual feedback distinguishing “pending” vs “confirmed” vs “failed” punch states — users don’t know whether punch was accepted.
- Punch actions lack immediate local timestamp capture — clock skew or delayed requests can record inaccurate times.

## 2. Proposed change

File: `workio/src/features/punch/usePunchQueue.ts` (new)  
File: `workio/src/features/punch/PunchButton.tsx` (modify)  
Scope: add a lightweight optimistic queue with local persistence, idempotency keys, and state rollback; update UI to reflect pending/confirmed/failed states.

## 3. Implementation

Create queue hook:

```tsx
// workio/src/features/punch/usePunchQueue.ts
import { useEffect, useState, useCallback } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';

type Punch = {
  id: string; // idempotency key
  type: 'IN' | 'OUT';
  timestamp: number; // local capture time
  location: { lat: number; lng: number };
  status: 'pending' | 'confirmed' | 'failed';
  attempts: number;
};

const QUEUE_KEY = '@workio:punchQueue';

export function usePunchQueue() {
  const [queue, setQueue] = useState<Punch[]>([]);

  useEffect(() => {
    AsyncStorage.getItem(QUEUE_KEY).then((raw) => {
      if (raw) setQueue(JSON.parse(raw));
    });
  }, []);

  const persist = useCallback((next: Punch[]) => {
    setQueue(next);
    AsyncStorage.setItem(QUEUE_KEY, JSON.stringify(next)).catch(() => {});
  }, []);

  const enqueue = useCallback(
    (type: 'IN' | 'OUT', location: { lat: number; lng: number }) => {
      const p: Punch = {
        id: `${Date.now()}-${Math.random().toString(36).slice(2)}`,
        type,
        timestamp: Date.now(),
        location,
        status: 'pending',
        attempts: 0,
      };
      persist((q) => [...q, p]);
      return p.id;
    },
    [persist]
  );

  const updateStatus = useCallback(
    (id: string, status: 'confirmed' | 'failed') => {
      persist((q) =>
        q.map((item) =>
          item.id === id ? { ...item, status, attempts: item.attempts + 1 } : item
        )
      );
    },
    [persist]
  );

  const remove = useCallback((id: string) => {
    persist((q) => q.filter((item) => item.id !== id));
  }, [persist]);

  return { queue, enqueue, updateStatus, remove };
}
```

```tsx
// workio/src/features/punch/PunchButton.tsx
import React, { useState } from 'react';
import { Button, View, Text, StyleSheet } from 'react-native';
import { usePunchQueue } from './usePunchQueue';
import { postPunch } from '../../api/punch';

export function PunchButton() {
  const [loading, setLoading] = useState(false);
  const { queue, enqueue, updateStatus, remove } = usePunchQueue();

  const doPunch = async (type: 'IN' | 'OUT') => {
    setLoading(true);
    // Capture local time and location immediately
    const location = await getCurrentLocation(); // implement or stub
    const id = enqueue(type, location);

    try {
      await postPunch({
        type,
        timestamp: Date.now(),
        location,
        idempotencyKey: id,
      });
      updateStatus(id, 'confirmed');
      setTimeout(() => remove(id), 2000); // clear confirmed after brief display
    } catch (err) {
      updateStatus(id, 'failed');
      // keep in queue for retry/backfill logic (can add retry loop here)
    } finally {
      setLoading(false);
    }
  };

  const latestPending = queue.filter((q) => q.status === 'pending').slice(-1)[0];
  const latestFailed = queue.filter((q) => q.status === 'failed').slice(-1)[0];

  return (
    <View style={styles.container}>
      <Button title="Clock IN" onPress={() => doPunch('IN')} disabled={loading} />
      <Button title="Clock OUT" onPress={() => doPunch('OUT')} disabled={loading} />

      {latestPending && (
        <View style={styles.badgePending}>
          <Text>⏳ Punch {latestPending.type} pending…</Text>
        </View>
      )}
      {latestFailed && (
        <View style={styles.badgeFailed}>
          <Text>⚠️ Punch {latestFailed.type} failed — will retry</Text>
        </View>
      )}
      {queue.some((q) => q.status === 'confirmed') && (
        <View style={styles.badgeConfirmed}>
          <Text>✅ Punch confirmed</Text>
        </View>
      )}
    </View>
  );
}
```

API helper with idempotency header:

```ts
// workio/src/api/punch.ts
export async function postPunch(payload: {
  type: 'IN' | 'OUT';
  timestamp: number;
  location: { lat: number; lng: number };
  idempotencyKey: string;
}) {
  const res = await fetch('/api/punch', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Idempotency-Key': payload.idempotencyKey,
    },
    body: JSON.stringify(payload),
  });
  if (!res.ok) throw new Error('Punch failed');
  return res.json();
}
```

Backend note (brief): ensure `/api/punch` uses `Idempotency-Key` header to dedupe within a sliding window (e.g., 24h) and returns 200/201 for repeats with same key.

## 4. Verification

- Tap “Clock IN” → UI immediately shows “⏳ Punch IN pending…” (optimistic).  
- Simulate success (server returns 200) → badge switches to “✅ Punch confirmed” then fades.  
- Simulate failure (server 5xx or network offline) → badge shows “⚠️ Punch IN failed — will retry” and item remains in persisted queue.  
- Reload app while pending/failed items exist → queue restores from AsyncStorage and can be retried (add retry loop or manual retry button as next step).  
- Check network tab: request includes `Idempotency-Key` header; duplicate retries with same key return 200 without creating duplicate punches.
