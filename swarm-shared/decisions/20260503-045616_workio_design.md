# workio / design

## Final Synthesis (Corrected + Actionable)

### 1. Root Cause (merged)
- **Duplicate creation vectors**: client double-taps, LINE webhook retries, and network-level retries all converge on endpoints that lack idempotency enforcement.
- **Missing uniqueness guarantees**: no unique constraint on `(user_id, date, direction)` allows races and duplicates at the DB level.
- **Replay surface**: LINE webhook signature verification lacks a time-window, allowing stale or replayed events to be processed.
- **No client-side intent tracking**: absence of optimistic state and idempotency-key plumbing causes UX jank and repeated submissions.

### 2. Design Decisions (resolve contradictions)
- **Primary deduplication mechanism**: **DB unique constraint** on `(user_id, date, direction)` + **idempotency table** keyed by client/webhook request IDs.  
  - Why: DB constraint is the source of truth; idempotency table gives fast-path for retries and cross-endpoint safety.
- **Idempotency key ownership**:
  - **Frontend clock actions**: client-generated UUID in `Idempotency-Key` header (short-lived, per intent).
  - **LINE webhook events**: use LINE’s `X-Line-Delivery` (or event-level ID if available) as the idempotency key; reject events older than **5 minutes** via `X-Line-Timestamp` or local clock skew allowance.
- **Signature window**: enforce **≤5 minutes** at webhook ingress; reject older events before any processing.
- **Atomic upsert pattern**: `INSERT ... ON CONFLICT DO NOTHING` for idempotency table, then proceed only if the row was inserted (first time). This prevents races under concurrent webhook deliveries.
- **Scope for initial fix**: prioritize `/api/clock` (frontend) + LINE webhook handler (backend) with the same idempotency primitives.

### 3. Implementation (single coherent plan)

#### 3.1 DB schema (run once)
```sql
-- Prevent duplicate clock records per user/date/direction
ALTER TABLE clock_records
  ADD CONSTRAINT uniq_user_date_direction UNIQUE (user_id, date, direction);

-- Idempotency table for cross-endpoint deduplication
CREATE TABLE IF NOT EXISTS idempotency_keys (
  key         TEXT PRIMARY KEY,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload_hash TEXT,
  endpoint    TEXT NOT NULL,
  response_status INT NOT NULL,
  response_body  JSONB
);
CREATE INDEX IF NOT EXISTS idx_idempotency_expiry ON idempotency_keys(created_at);
```

#### 3.2 Backend: idempotent `/api/clock`
```ts
// server/src/routes/clock.ts
import { Router } from 'express';
import { pool } from '../db';
import crypto from 'crypto';

const router = Router();

router.post('/clock', async (req, res) => {
  try {
    const { userId, direction } = req.body;
    const idempotencyKey = req.header('Idempotency-Key');

    if (!userId || !['in', 'out'].includes(direction)) {
      return res.status(400).json({ error: 'Invalid payload' });
    }
    if (!idempotencyKey || idempotencyKey.length < 8) {
      return res.status(400).json({ error: 'Idempotency-Key required' });
    }

    const today = new Date().toISOString().split('T')[0];

    // Fast-path: idempotency table (prevents duplicate processing)
    const idRes = await pool.query(
      `INSERT INTO idempotency_keys (key, endpoint, response_status, response_body)
       VALUES ($1, $2, $3, $4)
       ON CONFLICT (key) DO NOTHING
       RETURNING key`,
      [idempotencyKey, '/clock', 200, null]
    );

    // If key existed, return previous response (cached or reconstruct)
    if (idRes.rowCount === 0) {
      const cached = await pool.query(
        `SELECT response_body FROM clock_records cr
         JOIN idempotency_keys ik ON true
         WHERE ik.key = $1 AND cr.user_id = $2 AND cr.date = $3 AND cr.direction = $4
         ORDER BY cr.created_at DESC LIMIT 1`,
        [idempotencyKey, userId, today, direction]
      );
      if (cached.rows[0]?.response_body) {
        return res.json(cached.rows[0].response_body);
      }
    }

    // Atomic upsert for clock record
    const result = await pool.query(
      `INSERT INTO clock_records (user_id, date, direction, idempotency_key, created_at)
       VALUES ($1, $2, $3, $4, NOW())
       ON CONFLICT (user_id, date, direction) DO UPDATE
       SET idempotency_key = EXCLUDED.idempotency_key
       RETURNING id, created_at, direction, date`,
      [userId, today, direction, idempotencyKey]
    );

    const record = result.rows[0];
    const responseBody = { ok: true, record, idempotencyKey };

    // Cache response in idempotency table
    await pool.query(
      `UPDATE idempotency_keys
       SET response_body = $1
       WHERE key = $2`,
      [responseBody, idempotencyKey]
    );

    return res.json(responseBody);
  } catch (err) {
    console.error('Clock error', err);
    return res.status(500).json({ error: 'Server error' });
  }
});

export default router;
```
