# workio / ops

## 1. Diagnosis

- No idempotency key on punch endpoints → LINE webhook replays and client retries create duplicate punches for same tenant/user/day.
- No server-side state machine → invalid sequences allowed (clock-in twice, clock-out without clock-in).
- Punch writes are not transactional with conflict checks → race conditions under concurrent requests.
- Missing unique constraint to enforce one “active” punch per tenant/user (at most one non-ended punch).
- No idempotency table to deduplicate by client-provided idempotency key within a safe TTL.

## 2. Proposed change

File scope:
- `server/src/db/schema.sql` — add `punch_idempotency` table and unique constraint on `punches` for active punch.
- `server/src/routes/punch.ts` — enforce idempotency key and state-machine rules in POST `/punch`.
- `server/src/services/punchService.ts` — transactional upsert with state transition checks.

## 3. Implementation

### 3.1 DB schema (add idempotency + active-punch constraint)

```sql
-- server/src/db/schema.sql
-- Add after punches table definition

-- Idempotency table to deduplicate punch requests (TTL: 24h)
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT NOT NULL,
  tenant_id       INTEGER NOT NULL,
  user_id         INTEGER NOT NULL,
  punch_id        INTEGER NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (idempotency_key, tenant_id)
);

-- Index for cleanup
CREATE INDEX IF NOT EXISTS idx_punch_idempotency_created ON punch_idempotency (created_at);

-- Ensure at most one active (non-ended) punch per tenant+user
-- Assumes ended_at NULL means active
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_active_punch_per_user
ON punches (tenant_id, user_id)
WHERE ended_at IS NULL;
```

### 3.2 Transactional punch service

```ts
// server/src/services/punchService.ts
import { pool } from './db';

export type PunchType = 'clock_in' | 'clock_out' | 'break_start' | 'break_end';

export async function createPunch(params: {
  tenantId: number;
  userId: number;
  type: PunchType;
  latitude?: number;
  longitude?: number;
  idempotencyKey: string;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency check
    const idem = await client.query(
      `SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1 AND tenant_id = $2`,
      [params.idempotencyKey, params.tenantId]
    );
    if (idem.rows.length > 0) {
      const punch = await client.query(
        `SELECT * FROM punches WHERE id = $1`,
        [idem.rows[0].punch_id]
      );
      await client.query('ROLLBACK');
      return punch.rows[0];
    }

    // 2) State-machine checks
    const latest = await client.query(
      `SELECT id, type, ended_at FROM punches
       WHERE tenant_id = $1 AND user_id = $2
       ORDER BY created_at DESC LIMIT 1`,
      [params.tenantId, params.userId]
    );

    const last = latest.rows[0];
    if (params.type === 'clock_in' && last && !last.ended_at) {
      throw new Error('ALREADY_CLOCKED_IN');
    }
    if (params.type === 'clock_out' && (!last || last.ended_at)) {
      throw new Error('NO_ACTIVE_PUNCH_TO_CLOCK_OUT');
    }

    // 3) Create punch
    const punchRes = await client.query(
      `INSERT INTO punches (tenant_id, user_id, type, latitude, longitude)
       VALUES ($1, $2, $3, $4, $5)
       RETURNING *`,
      [params.tenantId, params.userId, params.type, params.latitude, params.longitude]
    );
    const punch = punchRes.rows[0];

    // 4) Record idempotency
    await client.query(
      `INSERT INTO punch_idempotency (idempotency_key, tenant_id, user_id, punch_id)
       VALUES ($1, $2, $3, $4)`,
      [params.idempotencyKey, params.tenantId, params.userId, punch.id]
    );

    // 5) If clock_out, close previous active punch (simple: update previous ended_at if needed)
    if (params.type === 'clock_out' && last && !last.ended_at) {
      await client.query(
        `UPDATE punches SET ended_at = NOW() WHERE id = $1`,
        [last.id]
      );
    }

    await client.query('COMMIT');
    return punch;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 3.3 Route handler (enforce idempotency key)

```ts
// server/src/routes/punch.ts
import express from 'express';
import { createPunch } from '../services/punchService';

const router = express.Router();

router.post('/', async (req, res) => {
  try {
    const { tenantId, userId, type, latitude, longitude, idempotencyKey } = req.body;
    if (!idempotencyKey) {
      return res.status(400).json({ error: 'idempotencyKey required' });
    }

    const punch = await createPunch({
      tenantId,
      userId,
      type,
      latitude,
      longitude,
      idempotencyKey
    });

    res.json({ ok: true, punch });
  } catch (err: any) {
    if (err.message === 'ALREADY_CLOCKED_IN') {
      return res.status(409).json({ error: 'Already clocked in' });
    }
    if (err.message === 'NO_ACTIVE_PUNCH_TO_CLOCK_OUT') {
      return res.status(409).json({ error: 'No active punch to clock out' });
    }
    console.error(err);
    res.status(500).json({ error: 'Internal server error' });
  }
});

export default router;
```

   ```bash
       "idempotencyKey":"req-1234"
     }'
   ```
   - Repeat same request → same punch returned, no duplicate row.
   - Check DB: `SELECT count(*) FROM punches WHERE user_id=101 AND tenant_id=1;` should not increase.
4. Test state machine:
   - Clock in twice (without clock out) → second request returns `409 ALREADY_CLOCKED_IN`.
   - Clock out without active punch → `409 NO_ACTIVE_PUNCH_TO_CLOCK_OUT`.
5. Verify active-punch uniqueness at DB level:
   - Attempt to insert two active punches via raw SQL for same tenant/user (ended_at NULL) → unique index violation.
