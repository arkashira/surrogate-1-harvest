**Implementation Plan (≤2h)**

### Step 1: Pre-flight snapshot generation

Create a new script `bin/snapshot.sh` that generates a deterministic pre-flight snapshot of the dataset list:

```bash
#!/bin/bash

# Set the date folder to snapshot
DATE_FOLDER=$(date -d "yesterday" +%Y-%m-%d)

# Set the number of shards to snapshot
NUM_SHARDS=16

# Set the HF token
HF_TOKEN=${HF_TOKEN:-<UGGINGFACE_TOKEN>}

# Set the dataset name
DATASET_NAME="axentx/surrogate-1-training-pairs"

# Generate the pre-flight snapshot
for ((i=0; i<NUM_SHARDS; i++)); do
  SLUG_HASH_BUCKET=$(echo -n "$i" | sha256sum | cut -c 1-32)
  echo "Snapshotting shard $i: $SLUG_HASH_BUCKET"
  hf_hub_download --path "$DATASET_NAME/resolve/main/$DATE_FOLDER/shard$SLUG_HASH_BUCKET.jsonl" --use-api --no-progress-bar --no-cache
done
```

**Step 2: Update `bin/dataset-enrich.sh` to use the pre-flight snapshot**

Update the `bin/dataset-enrich.sh` script to use the pre-flight snapshot generated by `bin/snapshot.sh`:

```bash
#!/bin/bash

# Set the date folder to process
DATE_FOLDER=$(date -d "yesterday" +%Y-%m-%d)

# Set the number of shards to process
NUM_SHARDS=16

# Set the HF token
HF_TOKEN=${HF_TOKEN:-<UGGINGFACE_TOKEN>}

# Set the dataset name
DATASET_NAME="axentx/surrogate-1-training-pairs"

# Process the pre-flight snapshot
for ((i=0; i<NUM_SHARDS; i++)); do
  SLUG_HASH_BUCKET=$(echo -n "$i" | sha256sum | cut -c 1-32)
  echo "Processing shard $i: $SLUG_HASH_BUCKET"
  # Use the pre-flight snapshot file instead of downloading the dataset list
  jsonl_file="batches/public-merged/$DATE_FOLDER/shard$SLUG_HASH_BUCKET.jsonl"
  if [ -f "$jsonl_file" ]; then
    echo "Using pre-flight snapshot file: $jsonl_file"
  else
    echo "Downloading dataset list for shard $i: $SLUG_HASH_BUCKET"
    hf_hub_download --path "$DATASET_NAME/resolve/main/$DATE_FOLDER/shard$SLUG_HASH_BUCKET.jsonl" --use-api --no-progress-bar --no-cache
  fi
  # Process the dataset list
  # ...
done
```

**Step 3: Update the GitHub Actions workflow to run the pre-flight snapshot generation script**

Update the GitHub Actions workflow to run the `bin/snapshot.sh` script before running the `bin/dataset-enrich.sh` script:

```yml
name: Ingest Dataset

on:
  workflow_dispatch:
    inputs:
      date-folder:
        description: 'Date folder to process'
        required: true
      num-shards:
        description: 'Number of shards to process'
        required: true

jobs:
  ingest:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Run pre-flight snapshot generation script
        run: |
          bin/snapshot.sh
          export HF_TOKEN=${{ secrets.HF_TOKEN }}
          export DATE_FOLDER=${{ inputs.date-folder }}
          export NUM_SHARDS=${{ inputs.num-shards }}

      - name: Run dataset enrichment script
        run: |
          bin/dataset-enrich.sh
          export HF_TOKEN=${{ secrets.HF_TOKEN }}
          export DATE_FOLDER=${{ inputs.date-folder }}
          export NUM_SHARDS=${{ inputs.num-shards }}
```

This implementation plan eliminates the need for runtime `load_dataset(streaming=True)` and recursive `list_repo_tree` calls, which are rate-limited by the HF API. It also reduces the load on the HF API by performing CDN-only fetches and data processing. The processed data is then uploaded to the dataset repository, ensuring that it is available for future use.
