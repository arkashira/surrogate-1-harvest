# vanguard / design

**Most Valuable Next Improvement: Persistent File-List Cache**

### Diagnosis

* No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` → HF API 429 (1000 req/5min).
* Lightning Studio lifecycle not reused → quota burn on recreation.
* Training = re-fetching file paths from HF API on every run.
* No persisted Studio instance → recreate cycles.

### Proposed Change

* Implement a persistent file-list cache to store file paths for each dataset.
* Reuse the cached file list for subsequent runs instead of re-fetching from HF API.

### Implementation

1. Create a new file `file_list_cache.py` in the project root:
```python
import json
import os

CACHE_DIR = '.file_list_cache'

def cache_file_list(repo, date_folder):
    cache_file = os.path.join(CACHE_DIR, f'{repo}-{date_folder}.json')
    if os.path.exists(cache_file):
        with open(cache_file, 'r') as f:
            return json.load(f)
    else:
        # Fetch file list from HF API and cache it
        file_list = list_repo_tree(repo, date_folder)
        with open(cache_file, 'w') as f:
            json.dump(file_list, f)
        return file_list

def load_file_list(repo, date_folder):
    return cache_file_list(repo, date_folder)
```
2. Update `train.py` to use the cached file list:
```python
import file_list_cache

# ...

file_list = file_list_cache.load_file_list(repo, date_folder)
# Use the cached file list for training
```
3. Update `lightning_main.py` to reuse the persisted Studio instance:
```python
import file_list_cache

# ...

studio = Teamspace.studios.get(name='my_studio')
if studio:
    # Reuse the persisted Studio instance
    studio = studio
else:
    # Create a new Studio instance
    studio = Studio.create_ok()
```
4. To update the implementation to handle dataset schema heterogeneity, we can use the following approach:
```python
import json

# Load cached file list
with open('data_cache.json', 'r') as f:
    file_list = json.load(f)['files']

# Initialize training data using the cached file list
train_data = []
for file in file_list:
    # Check if the file has a valid schema
    if is_valid_schema(file['path']):
        train_data.append(hf.load_dataset(file['path'], streaming=True))
    else:
        # Handle schema heterogeneity by skipping the file
        print(f"Skipping file {file['path']} due to invalid schema")
```
5. To update the `data_cache.json` file whenever the training data changes, we can use the following approach:
```python
import json

# Update the cached file list with new entries
new_file_list = []
for file in train_data:
    new_file_list.append({
        "path": file.filename,
        "schema": file.schema
    })

# Save the updated cached file list
with open('data_cache.json', 'w') as f:
    json.dump({"files": new_file_list}, f)
```
