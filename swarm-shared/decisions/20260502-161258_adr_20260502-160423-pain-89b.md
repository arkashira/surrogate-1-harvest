## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI stealth client (Rust) that starts an obfuscated tunnel (meek-tls over QUIC) with randomized local SOCKS5 port, TUN-less mode only, local SQLite fail-closed state, and a basic Supabase-backed config/telemetry endpoint; includes kill-switch that blocks non-tunnel traffic on failure; no GUI, no server provisioning required — users run a single binary and point apps to localhost:random-port.

## Open questions
- Which obfuscation primitives are safest against current DPI in target regions (meek vs. Shadowsocks 2022-ietf-AEAD vs. vless over QUIC)?
- Can we rely on Cloudflare Workers + Durable Objects for config/geo distribution without risking CF TOS issues for VPN-adjacent traffic?
- How will we perform safe, reproducible TUN setup across desktop OSes without root/admin prompts that reveal intent?
- What is the acceptable telemetry schema to measure WAU-ZB without deanonymizing users or creating honeypot logs?
- How do we implement a deterministic, cross-platform kill-switch that blocks all non-stealth traffic when the obfuscated path fails?
