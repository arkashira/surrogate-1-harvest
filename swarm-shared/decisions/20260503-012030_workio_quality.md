## Final synthesized implementation (correct + actionable)

Use this single plan and code to fix the LINE webhook handler end-to-end.

---

### 1. Core problems (resolved)

- **Duplicate deliveries**: LINE retries on network/timeouts → duplicate punches/leave/OT.  
  **Fix**: Idempotency key per event+employee stored in DB with TTL; reject duplicates.
- **Concurrent clock-ins**: find-then-insert allows multiple open punches.  
  **Fix**: Unique constraint/index for one open punch per employee per day + row-level locking (`SELECT ... FOR UPDATE`) inside a transaction.
- **Partial writes**: Punch and notification not atomic.  
  **Fix**: Single DB transaction for punch + idempotency + notification queue entry; commit only when all succeed.
- **Missing business rules**: Allow clock-in when already open, clock-out when none open.  
  **Fix**: Guard checks before transitions; return clear outcomes.
- **Observability**: Hard to debug duplicates/ordering.  
  **Fix**: Structured logs with traceable fields (employee, event, idemKey, outcome).

### 2. DB schema changes (run once)

```sql
-- Idempotency table (primary dedup store)
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  key TEXT PRIMARY KEY,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_webhook_idempotency_created ON webhook_idempotency (created_at);

-- One open punch per employee per day (prevents concurrent open punches)
-- If you already have a punches table, add partial unique index:
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_employee_per_day
  ON punches (employee_id, tenant_id, date_trunc('day', created_at))
  WHERE status = 'open';

-- Optional: keep a small audit trail for webhook events (helpful for debugging)
CREATE TABLE IF NOT EXISTS webhook_events (
  id BIGSERIAL PRIMARY KEY,
  event_type TEXT NOT NULL,
  line_user_id TEXT NOT NULL,
  employee_id UUID NOT NULL,
  tenant_id UUID NOT NULL,
  idem_key TEXT NOT NULL,
  payload JSONB NOT NULL,
  processed_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  status TEXT NOT NULL,
  details JSONB
);
CREATE INDEX IF NOT EXISTS idx_webhook_events_idem ON webhook_events (idem_key);
CREATE INDEX IF NOT EXISTS idx_webhook_events_emp_time ON webhook_events (employee_id, processed_at);
```

Periodic cleanup (run daily):

```sql
DELETE FROM webhook_idempotency WHERE created_at < NOW() - INTERVAL '1 day';
DELETE FROM webhook_events WHERE processed_at < NOW() - INTERVAL '7 days';
```

  if (!signature || !channelSecret) return false;

  return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(signature));
}
