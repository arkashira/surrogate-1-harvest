# workio / backend

## 1. Diagnosis

- **Non-idempotent LINE webhook handler**: LINE retries on timeout/5xx; duplicate `events` for same `timestamp` + `userId` can create multiple punch records because upsert is not atomic and lacks a stable deduplication key.
- **Missing tenant-scoped uniqueness**: no DB unique constraint on `(tenant_id, employee_id, date, punch_type)` or `(tenant_id, employee_id, punch_time_window)` to enforce single punch per clock-in/out window at the persistence layer.
- **Race condition under concurrency**: concurrent webhook deliveries (or retries) can interleave `SELECT` then `INSERT` and produce duplicate rows; requires `SELECT ... FOR UPDATE` or `ON CONFLICT DO NOTHING/UPDATE` with proper index.
- **No idempotency key storage**: no table to record processed LINE `event.id` per tenant to short-circuit retries immediately before touching business tables.
- **Clock-in/out window ambiguity**: without a tight window (e.g., 5–10 minutes) and clear state machine (pending/completed), employees can spam clock-in and create noisy duplicates.

## 2. Proposed change

- **File**: `/opt/axentx/workio/server/src/db/schema.sql` — add tables and constraints.
- **File**: `/opt/axentx/workio/server/src/controllers/lineWebhook.ts` — make handler idempotent and atomic.
- **Scope**: add `line_event_log` table for idempotency; add unique constraint on punches; refactor upsert to single `INSERT ... ON CONFLICT DO NOTHING` with tenant scoping.

## 3. Implementation

### 3.1 DB schema changes (`server/src/db/schema.sql`)

```sql
-- Idempotency log for LINE events (one row per event id per tenant)
CREATE TABLE IF NOT EXISTS line_event_log (
  id              BIGSERIAL PRIMARY KEY,
  tenant_id       BIGINT NOT NULL,
  line_event_id   TEXT   NOT NULL,
  event_type      TEXT   NOT NULL,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload         JSONB,
  CONSTRAINT uniq_line_event_per_tenant UNIQUE (tenant_id, line_event_id)
);

-- Ensure punches are tenant-scoped and single punch per type per date
-- (allow one clock-in and one clock-out per employee per day)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_tenant_employee_date_type
ON punches (tenant_id, employee_id, DATE(punch_time), punch_type)
WHERE deleted_at IS NULL;

-- Optional: add a fast lookup for "latest punch" per employee per tenant
CREATE INDEX IF NOT EXISTS idx_punches_tenant_employee_latest
ON punches (tenant_id, employee_id, punch_time DESC)
WHERE deleted_at IS NULL;
```

### 3.2 Idempotent webhook handler (`server/src/controllers/lineWebhook.ts`)

```ts
import { Request, Response } from 'express';
import { pool } from '../db';
import { verifyLineSignature } from '../utils/lineCrypto';

export async function handleLineWebhook(req: Request, res: Response) {
  const signature = req.headers['x-line-signature'] as string;
  const body = req.body;

  if (!verifyLineSignature(JSON.stringify(body), signature)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const event of body.events) {
      if (event.type !== 'message' || event.message.type !== 'text') continue;

      // Extract tenant from source (e.g., userId -> tenant mapping)
      const tenantId = await resolveTenantIdByLineUserId(client, event.source.userId);
      if (!tenantId) continue;

      // Idempotency: skip if already processed
      const exists = await client.query(
        `SELECT 1 FROM line_event_log WHERE tenant_id = $1 AND line_event_id = $2`,
        [tenantId, event.eventId]
      );
      if (exists.rows.length > 0) continue;

      // Record event first (makes retries idempotent)
      await client.query(
        `INSERT INTO line_event_log (tenant_id, line_event_id, event_type, payload)
         VALUES ($1, $2, $3, $4)`,
        [tenantId, event.eventId, event.type, event]
      );

      // Parse intent (clock-in / clock-out) from text
      const text = event.message.text.trim().toLowerCase();
      const isClockIn = text.includes('เข้า') || text.includes('in') || text.includes('clock in');
      const isClockOut = text.includes('ออก') || text.includes('out') || text.includes('clock out');

      if (!isClockIn && !isClockOut) continue;

      // Resolve employee
      const emp = await client.query(
        `SELECT id FROM employees WHERE tenant_id = $1 AND line_user_id = $2 AND deleted_at IS NULL`,
        [tenantId, event.source.userId]
      );
      if (emp.rows.length === 0) continue;
      const employeeId = emp.rows[0].id;

      const punchType = isClockIn ? 'clock_in' : 'clock_out';
      const punchTime = new Date();

      // Atomic upsert: one punch per (tenant, employee, date, punch_type)
      // If a punch already exists for today for this type, skip (or update if business allows correction)
      await client.query(
        `INSERT INTO punches (tenant_id, employee_id, punch_type, punch_time, created_by)
         VALUES ($1, $2, $3, $4, $5)
         ON CONFLICT DO NOTHING`,
        [tenantId, employeeId, punchType, punchTime, 'line-bot']
      );
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error:', err);
    // LINE will retry on 5xx — return 500 only for transient failures you want retried
    res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
}

async function resolveTenantIdByLineUserId(client: any, lineUserId: string): Promise<number | null> {
  const r = await client.query(
    `SELECT tenant_id FROM employees WHERE line_user_id = $1 AND deleted_at IS NULL LIMIT 1`,
    [lineUserId]
  );
  return r.rows.length > 0 ? r.rows[0].tenant_id : null;
}
```

## 4. Verification

1. **Schema check** — confirm constraints exist:
   ```bash
   psql workio -c "\d punches"
   psql workio -c "\d line_event_log"
   ```
   Expect unique index on `(tenant_id, employee_id, date(punch_time), punch_type)` and unique constraint on `line_event_log(tenant_id, line_event_id)`.

2. **Idempotency test** — simulate duplicate LINE delivery:
   - Send a valid webhook payload twice (same `eventId`) within seconds.
   - Verify only one punch row is created and only one `line_event_log` row exists.

3. **Race test** — fire 5 concurrent requests with same payload:
   - Use `ab` or a small script to POST concurrently.
   - Confirm exactly one punch row per `(tenant, employee, date, punch_type)`.

4. **Behavior test** — normal flow:
   - Employee texts “เข้า” → clock_in row created.
   - Same text again same day → no second clock_in row (due to unique index + `ON CONFLICT DO NOTHING`).
   - Text “ออก” → clock_out row created.

5. **Log check** — ensure `line_event_log` records each processed `eventId` and duplicates are skipped before punch logic.
