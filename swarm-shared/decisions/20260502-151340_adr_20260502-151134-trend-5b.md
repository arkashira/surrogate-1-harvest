## v0.1 scope
v0.1 ships a deterministic, replayable research→signal→risk→execution workflow runner (Python/FastAPI on CF Workers) with Postgres-backed immutable audit (Supabase) and Redis Streams for ordered event lanes; includes idempotent step execution, compliance guardrails as versioned JSON expressions, pre-built mock broker/OMS connectors, and contract tests; supports manual trigger and scheduled trigger, outputs execution_ledger entries, and exposes minimal API to submit workflows and replay runs — all deployable on free tiers within 2 weeks.

## Open questions
- Deterministic replay requirements: must every step_event payload be bitwise-identical on replay, or is logical determinism sufficient?
- Latency SLO for research→execution: is sub-500ms end-to-end required or is sub-second acceptable under free-tier constraints?
- Compliance rule engine: will rules be expressed in a safe embedded DSL or evaluated via sandboxed WebAssembly?
- Broker/OMS connectors: which specific broker APIs and FIX versions must be supported in v0.1?
- Idempotency and exactly-once: will we rely on Redis-based idempotency keys plus at-least-once queueing or require stronger guarantees?
- Secrets and key management: how will API keys and certificates be injected in CF Workers free-tier (secrets vs KV)?
- Audit retention and immutability: should audit logs be WAL-exported to cold storage or is Supabase row-level immutability sufficient for regulatory holds?
