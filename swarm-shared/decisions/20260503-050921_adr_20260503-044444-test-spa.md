## v0.1 scope
v0.1 ships in 2 weeks: CLI-only scanner (Python) that connects to AWS (boto3) and GCP (google-cloud-functions) using supplied credentials, lists functions and their schedule triggers, persists snapshots to Supabase Postgres, and outputs a report of orphaned, duplicate, and drifted schedules; includes unit tests, a README, and a migration to create schedule_snapshot and scan_run tables.

## Open questions
- How will we authenticate to AWS and GCP in customer tenants (assume OIDC/assume-role vs API keys)?
- What is the minimal permission set required for read-only schedule/introspection access?
- Will schedule metadata be pulled via APIs or inferred from IaC repos (Terraform/Deployment Manager)?
- How often should scans run (ad-hoc CLI vs scheduled CF Worker) and who triggers them?
- What retention policy should we apply to schedule_snapshot rows (keep last N scans or full history)?
