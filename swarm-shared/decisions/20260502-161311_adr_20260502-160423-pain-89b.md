## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-file Rust CLI client (Linux x86_64) that: establishes a local SOCKS5 proxy with obfuscation (shadowsocks-like stream cipher + utls TLS masquerade), creates a userspace TUN via boringtun/smoltcp with DNS leak protection, enforces a fail-closed killswitch that blocks non-tunnel traffic on detection, and reports anonymized session events to Supabase Postgres for WAU-ZB tracking; no GUI, no server components, zero-config presets loaded from CF Workers endpoint.

## Open questions
- Which DPI signatures are most prevalent in target regions (SNI, JA3, packet length/time patterns)?
- Can we rely on Cloudflare Workers for config/relay endpoints without risking centralized blocking?
- What is the acceptable trade-off between kernel TUN (performance) vs userspace TUN (stealth) per platform?
- How will we distribute client updates safely (code transparency, reproducible builds) to avoid supply-chain targeting?
- What minimal telemetry schema preserves privacy while enabling WAU-ZB measurement?
