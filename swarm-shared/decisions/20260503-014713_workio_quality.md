# workio / quality

## Final consolidated solution (best parts, corrected, actionable)

### 1. Diagnosis (merged & corrected)
- **No idempotency persistence** — `X-Line-Signature` is verified but not stored; retries create duplicate punches.
- **Non-atomic upsert** — `findOne → if-exists-update-else-insert` is not safe under retries/concurrency.
- **Missing DB constraints** — no unique guard to prevent duplicate punches per user per day per type, and no enforcement of at-most-one open (clock-in) punch.
- **No transaction boundary** — punch operations can partially fail and leave inconsistent state.
- **Race on clock state** — concurrent webhooks can read stale state and produce invalid transitions (double clock-in/clock-out).

### 2. Scope
- **Webhook handler**: `/opt/axentx/workio/server/src/routes/webhook/line.ts`
- **DB schema/migrations**: `/opt/axentx/workio/server/src/db/schema.sql` (or migration file)
- **Service layer (optional but recommended)**: `/opt/axentx/workio/server/src/services/punchService.ts`

### 3. Implementation

#### 3.1 DB migrations (run once)

```sql
-- Idempotency for LINE webhooks (use signature as deterministic key)
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  signature   TEXT PRIMARY KEY,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: index for cleanup
CREATE INDEX IF NOT EXISTS idx_line_webhook_idempotency_created_at
  ON line_webhook_idempotency (created_at);

-- Prevent duplicate punches per user per calendar day per type
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_user_day_type
  ON punches (user_id, DATE(timestamp), type)
  WHERE type IN ('clock_in', 'clock_out');

-- Enforce at most one open (unclocked-out) punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_user_open_clock_in
  ON punches (user_id)
  WHERE type = 'clock_in' AND deleted_at IS NULL;
```

Notes:
- If `tenant_id` is required in your multi-tenant model, include it in the idempotency table and indexes (`(tenant_id, signature)` PK, and include `tenant_id` in punch uniqueness constraints).
- The partial unique index on open clock-ins guarantees only one active `clock_in` per user.

#### 3.2 Service layer (recommended)

Create `/opt/axentx/workio/server/src/services/punchService.ts`:

```ts
import { pool } from '../db';

export async function processPunchWithIdempotency(
  signature: string,
  userId: string,
  type: 'clock_in' | 'clock_out',
  timestamp: Date,
  metadata: Record<string, any>
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency check
    const idemp = await client.query(
      'SELECT 1 FROM line_webhook_idempotency WHERE signature = $1',
      [signature]
    );
    if (idemp.rows.length > 0) {
      await client.query('ROLLBACK');
      return { applied: false, reason: 'duplicate' };
    }

    // Atomic punch insert (ON CONFLICT DO NOTHING)
    // Conflicts are handled by partial unique indexes (day+type and open clock_in)
    const insertResult = await client.query(
      `INSERT INTO punches (user_id, type, timestamp, metadata)
       VALUES ($1, $2, $3, $4)
       ON CONFLICT DO NOTHING
       RETURNING id`,
      [userId, type, timestamp, metadata]
    );

    // Record idempotency key
    await client.query(
      'INSERT INTO line_webhook_idempotency (signature) VALUES ($1)',
      [signature]
    );

    await client.query('COMMIT');

    return {
      applied: insertResult.rowCount > 0,
      punchId: insertResult.rows[0]?.id || null,
      reason: insertResult.rowCount === 0 ? 'conflict' : 'created'
    };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

#### 3.3 Webhook handler

Update `/opt/axentx/workio/server/src/routes/webhook/line.ts`:

```ts
import { Router, Request, Response } from 'express';
import crypto from 'crypto';
import { processPunchWithIdempotency } from '../services/punchService';

const router = Router();

function verifySignature(body: string, signature: string, channelSecret: string): boolean {
  const hash = crypto
    .createHmac('sha256', channelSecret)
    .update(body, 'utf8')
    .digest('base64');
  return hash === signature;
}

router.post('/line', async (req: Request, res: Response) => {
  const signature = req.get('X-Line-Signature');
  const channelSecret = process.env.LINE_CHANNEL_SECRET || '';
  const rawBody = JSON.stringify(req.body);

  if (!signature || !verifySignature(rawBody, signature, channelSecret)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  const events = req.body.events || [];
  const results = [];

  try {
    for (const ev of events) {
      if (ev.type !== 'message' && ev.type !== 'postback') continue;

      const userId = ev.source?.userId;
      if (!userId) continue;

      // Derive action from payload (customize as needed)
      const action = ev.postback?.data || (ev.message?.text || '').toLowerCase();
      const isClockIn = action.includes('in') || action.includes('clock_in');
      const type = isClockIn ? 'clock_in' : 'clock_out';
      const timestamp = ev.timestamp ? new Date(ev.timestamp) : new Date();

      const result = await processPunchWithIdempotency(
        signature,
        userId,
        type,
        timestamp,
        { lineEvent: ev }
      );
      results.push({ userId, result });
    }

    return res.status(200).json({ ok: true, results });
  } catch (err) {
    console.error('LINE webhook error', err);
    return res.status(500).json({ error: 'internal' });
  }
});

export default router;
```

#### 3.4 Cleanup job (optional)

Add a periodic job to prune old idempotency keys (e.g., older than 48 hours):

```ts
// server/src/jobs/cleanIdempotency.ts
import { pool } from '../db';

export async function cleanIdempotency() {
  await pool.query(
    'DELETE FROM line_webhook_idempotency WHERE created_at < NOW() - INTERVAL \'48 hours\''
  );
}
```

Schedule it via cron, Bull, or your existing job runner.

### 4. Verification (actionable tests)

1. **Schema check**
   - Confirm `line_webhook_idempotency` exists with PK on `signature`.
   - Confirm partial unique indexes on punches (`idx_punches_user_day_type`, `idx_punches_user_open_clock_in`).

2. **Idempotency test**
   - Send identical LINE payload (same `X-Line-Signature`) twice.
   - Expect: first `200 {ok:true}`, second `200 {ok:true, results:[{result:{applied:false,reason:'duplicate'}}]}`.
   - Verify only one punch row created.

3. **Concurrency test**
   - Fire 10 parallel requests with same signature.
   - Confirm exactly one punch row and no race-condition duplicates.

4. **Constraint tests**
   - Attempt to insert duplicate `(user_id, DATE(timestamp), type)` punch via SQL; confirm rejected.
   - Attempt to insert two `clock_in` rows for same user without intervening `clock_out`; confirm second insert is blocked by partial unique index.

5. **Transaction test**
   - Simulate failure after punch insert but before idempotency insert (e.g., by temporarily breaking idempotency insert). Confirm transaction rollback leaves no punch row.

6. **Log check**
   - Ensure no unhandled errors in server logs during tests.
