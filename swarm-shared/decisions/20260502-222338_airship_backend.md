# airship / backend

## Implementation Plan: `airship discover` — Zero-config discovery CLI

**Highest-value incremental improvement (<2h):**  
Ship a single Bash CLI (`airship discover`) that operationalizes past patterns (business research + knowledge-rag + top-hub review) into one repeatable, cron-safe command with proper shebang, executable bits, and SHELL hygiene.

### Why this is highest value
- Consolidates `granite-business-research.sh` + `knowledge-rag` + top-hub review into one deterministic flow.
- Fixes past wrapper/cron issues (shebang, executable, SHELL=/bin/bash).
- Produces actionable artifacts (top-hub, insights) for downstream tasks (training, incident triage, blueprinting).
- Safe for automation: idempotent, minimal deps, non-interactive, logs to structured dir.

### Implementation

#### 1) Main dispatcher: `/opt/axentx/airship/bin/airship`
```bash
#!/usr/bin/env bash
# /opt/axentx/airship/bin/airship
# Airship CLI dispatcher — keep minimal and POSIX-friendly.
set -euo pipefail

AIRSHIP_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
export AIRSHIP_ROOT

usage() {
  cat <<EOF
Usage: airship <command> [options]

Commands:
  discover   Run business research + knowledge-rag + top-hub review
  help       Show this help
EOF
}

cmd="${1:-help}"
shift || true

case "$cmd" in
  discover)
    exec "${AIRSHIP_ROOT}/bin/airship-discover" "$@"
    ;;
  help|--help|-h)
    usage
    ;;
  *)
    echo "Unknown command: $cmd" >&2
    usage
    exit 1
    ;;
esac
```
