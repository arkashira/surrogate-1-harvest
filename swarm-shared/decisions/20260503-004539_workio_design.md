# workio / design

## 1. Diagnosis
- Missing **idempotency key** on LINE webhook payloads allows at-least-once retries to create duplicate punch records for the same user within the same minute.
- No **DB-level partial unique constraint** enforcing “one open punch per user” (`clock_out_at IS NULL`) — race conditions under retries can open multiple rows.
- Punch creation logic uses **application-level check-then-insert** instead of atomic upsert/constraint-backed flow, making it vulnerable to concurrent/retried webhooks.
- No **idempotency table** to deduplicate by LINE `deliveryId`/`timestamp`+`userId`, so retries are treated as new punches.
- Missing **index to support fast conflict checks** and constraint enforcement for punch rows (user + open punch).

## 2. Proposed change
File scope:  
- `workio/server/src/db/schema.sql` — add partial unique index and idempotency table.  
- `workio/server/src/routes/webhook/line.ts` — add idempotency guard and atomic upsert flow for punches.

## 3. Implementation

### DB migration (run once)
```sql
-- 1) One open punch per user (partial unique index)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- 2) Lightweight idempotency table for LINE webhook deliveries
CREATE TABLE IF NOT EXISTS webhook_deliveries (
  idempotency_key TEXT PRIMARY KEY,
  user_id         INTEGER NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### Idempotent punch handler
`workio/server/src/routes/webhook/line.ts`
```ts
import { Request, Response } from 'express';
import { pool } from '../../db';

// Build a stable idempotency key from LINE delivery metadata
function buildIdempotencyKey(event: any, userId: number): string {
  // Prefer deliveryId if present; fallback to timestamp+eventType+userId
  const deliveryId = event?.deliveryId || event?.timestamp || Date.now();
  return `line:${userId}:${deliveryId}`;
}

export async function handleLineWebhook(req: Request, res: Response) {
  const body = req.body;
  const events = body?.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'No events' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      if (ev?.type !== 'message' || ev?.source?.type !== 'user') continue;
      const userId = Number(ev.source.userId.replace('U', '')); // adapt to your user ID mapping
      if (!Number.isFinite(userId)) continue;

      const idemKey = buildIdempotencyKey(ev, userId);

      // Idempotency check (fast PK lookup)
      const idem = await client.query(
        'SELECT 1 FROM webhook_deliveries WHERE idempotency_key = $1',
        [idemKey]
      );
      if (idem.rows.length > 0) {
        // Already processed — skip but continue processing other events
        continue;
      }

      // Record idempotency first (best-effort; if punch insert fails we still avoid replay)
      await client.query(
        'INSERT INTO webhook_deliveries (idempotency_key, user_id) VALUES ($1, $2)',
        [idemKey, userId]
      );

      // Determine action: if user has an open punch -> clock out; else -> clock in
      const openPunch = await client.query(
        'SELECT id FROM punches WHERE user_id = $1 AND clock_out_at IS NULL ORDER BY clock_in_at DESC LIMIT 1',
        [userId]
      );

      if (openPunch.rows.length > 0) {
        // Clock out existing open punch
        await client.query(
          `UPDATE punches
           SET clock_out_at = NOW(), updated_at = NOW()
           WHERE id = $1 AND clock_out_at IS NULL`,
          [openPunch.rows[0].id]
        );
      } else {
        // Clock in — insert new punch (constraint will block double-open)
        await client.query(
          `INSERT INTO punches (user_id, clock_in_at, clock_out_at, created_at, updated_at)
           VALUES ($1, NOW(), NULL, NOW(), NOW())`,
          [userId]
        );
      }
    }

    await client.query('COMMIT');
    return res.json({ ok: true });
  } catch (err: any) {
    await client.query('ROLLBACK');
    // If unique violation on open punch, treat as race/retry and return ok
    if (err.code === '23505') {
      await client.query('ROLLBACK');
      return res.json({ ok: true, note: 'duplicate suppressed by constraint' });
    }
    console.error('LINE webhook error', err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
}
```

### Optional cleanup job (run periodically)
```sql
-- Keep idempotency keys for 24h
DELETE FROM webhook_deliveries WHERE created_at < NOW() - INTERVAL '24 hours';
```

## 4. Verification
- Send the same LINE webhook payload twice within seconds; second request must **not** create a second open punch (check `punches` row count and that only one row exists with `clock_out_at IS NULL` for the user).
- Simulate concurrent requests (e.g., `ab` or parallel curl) for the same user; DB constraint + idempotency table must prevent double-open rows.
- Confirm idempotency key is recorded in `webhook_deliveries` and duplicates are skipped.
- After clock-in, send another message event to ensure it correctly clocks out the open punch and leaves no open rows.
