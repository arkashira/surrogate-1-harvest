# workio / ops

### Diagnosis
* The project lacks a robust monitoring system to detect and alert on potential issues, such as duplicate punches or race conditions.
* There is no clear documentation on the current deployment and infrastructure setup, making it difficult to identify and address operational issues.
* The project relies on a single PostgreSQL database, which may become a bottleneck as the user base grows, and there is no clear plan for scaling or load balancing.
* The LINE OA setup and webhook configuration are not well-documented, making it challenging to troubleshoot issues related to messaging and notifications.
* There is no clear strategy for handling errors and exceptions, which can lead to unexpected behavior and poor user experience.

### Proposed change
The proposed change is to implement a monitoring system using a tool like Prometheus and Grafana to track key metrics and alert on potential issues. This will involve modifying the `server/src` directory to include monitoring endpoints and configuring Prometheus and Grafana to collect and display metrics.

### Implementation
1. Install Prometheus and Grafana using the following commands:
```bash
npm install prometheus-client
npm install grafana
```
2. Create a new file `server/src/monitoring.js` to define monitoring endpoints:
```javascript
const express = require('express');
const router = express.Router();
const prometheus = require('prometheus-client');

// Define metrics
const punchesCounter = new prometheus.Counter('punches', 'Number of punches');
const errorsCounter = new prometheus.Counter('errors', 'Number of errors');

// Define monitoring endpoints
router.get('/metrics', (req, res) => {
  res.set("Content-Type", prometheus.register.contentType);
  res.end(prometheus.register.metrics());
});

module.exports = router;
```
3. Modify `server/src/app.js` to include the monitoring router:
```javascript
const express = require('express');
const app = express();
const monitoringRouter = require('./monitoring');

app.use('/monitoring', monitoringRouter);
```
4. Configure Prometheus to collect metrics from the monitoring endpoint:
```yml
# prometheus.yml
scrape_configs:
  - job_name: 'workio'
    scrape_interval: 10s
    metrics_path: /monitoring/metrics
    static_configs:
      - targets: ['workio:3000']
```
5. Configure Grafana to display metrics:
```bash
# Create a new dashboard in Grafana
# Add a panel to display the punches counter
# Add a panel to display the errors counter
```

### Verification
To verify that the monitoring system is working, follow these steps:
1. Start the Workio application and Prometheus server.
2. Use a tool like `curl` to send a request to the monitoring endpoint: `curl http://localhost:3000/monitoring/metrics`.
3. Verify that the metrics are being collected by Prometheus by checking the Prometheus dashboard.
4. Verify that the metrics are being displayed in Grafana by checking the Grafana dashboard.
5. Test the alerting system by simulating an error or duplicate punch and verifying that an alert is triggered.
