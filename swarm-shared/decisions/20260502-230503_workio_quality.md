# workio / quality

## 1) Diagnosis

- **No idempotency key enforcement** on clock-in/out endpoints: duplicate LINE webhook deliveries and client retries create multiple punches for the same `(employee_id, punch_type, date)` causing payroll drift.
- **Non-transactional punch writes**: concurrent retries can insert duplicate rows because there is no unique constraint or upsert logic at the DB layer.
- **Missing request deduplication middleware**: no idempotency key header parsing or short-term dedupe cache (e.g., Redis or in-memory) to reject replays within a safe window.
- **Race condition between check-then-insert**: app checks for existing punch then inserts; concurrent requests can both pass the check and both insert.
- **No unique constraint in schema**: `punches` table lacks a composite unique index on `(employee_id, punch_type, date)` to enforce correctness at the database level.

## 2) Proposed change

- **File scope**: `workio/server/src/db/schema.sql` (add constraint)  
- **File scope**: `workio/server/src/routes/punch.ts` (add idempotency key handling + transactional upsert)  
- **File scope**: `workio/server/src/middleware/idempotency.ts` (new — parse `Idempotency-Key` header, short-term cache, reject replays)

## 3) Implementation

### 3.1 DB schema — enforce uniqueness

```sql
-- workio/server/src/db/schema.sql
-- Add composite unique constraint to prevent duplicate punches
ALTER TABLE punches
  ADD CONSTRAINT punches_employee_type_date_unique
  UNIQUE (employee_id, punch_type, date);
```

### 3.2 Idempotency middleware (lightweight, memory-backed)

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';

const seen = new Map<string, { ts: number; status: number }>();
const TTL_MS = 60_000; // 60s dedupe window

export function idempotencyMiddleware(req: Request, res: Response, next: NextFunction) {
  const key = req.header('Idempotency-Key');
  if (!key) return next();

  const now = Date.now();
  const existing = seen.get(key);

  if (existing) {
    if (now - existing.ts < TTL_MS) {
      // Replay: return original status (best-effort)
      return res.status(existing.status).json({ replayed: true });
    }
    // expired
    seen.delete(key);
  }

  // store on response finish
  const originalSend = res.json;
  res.json = function (body: any) {
    seen.set(key, { ts: now, status: res.statusCode });
    return originalSend.call(this, body);
  } as any;

  next();
}
```

Register in your main app (e.g., `server/src/app.ts`):

```ts
import { idempotencyMiddleware } from './middleware/idempotency';
app.use(idempotencyMiddleware);
```

### 3.3 Transactional upsert in punch route

```ts
// workio/server/src/routes/punch.ts
import { Router, Request, Response } from 'express';
import { pool } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = Router();

// POST /api/punches
router.post('/', async (req: Request, res: Response) => {
  const { employee_id, punch_type, date, latitude, longitude, note } = req.body;
  if (!employee_id || !punch_type || !date) {
    return res.status(400).json({ error: 'Missing required fields' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Try insert; on conflict do nothing (idempotent)
    const upsert = `
      INSERT INTO punches (id, employee_id, punch_type, date, latitude, longitude, note, created_at)
      VALUES ($1, $2, $3, $4, $5, $6, $7, NOW())
      ON CONFLICT (employee_id, punch_type, date) DO NOTHING
      RETURNING *
    `;
    const result = await client.query(upsert, [
      uuidv4(),
      employee_id,
      punch_type,
      date,
      latitude ?? null,
      longitude ?? null,
      note ?? null,
    ]);

    await client.query('COMMIT');

    if (result.rowCount === 0) {
      // Already exists — return existing record
      const existing = await client.query(
        'SELECT * FROM punches WHERE employee_id = $1 AND punch_type = $2 AND date = $3',
        [employee_id, punch_type, date]
      );
      return res.status(200).json({ punch: existing.rows[0], created: false });
    }

    return res.status(201).json({ punch: result.rows[0], created: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Punch upsert failed', err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

2. **Start server** and test duplicate request rejection:
   ```bash
   curl -X POST http://localhost:3000/api/punches \
     -H "Content-Type: application/json" \
     -H "Idempotency-Key: test-key-123" \
     -d '{"employee_id":1,"punch_type":"in","date":"2026-05-02"}'
   ```
   Repeat same request within 60s — expect `200` with `"replayed":true` (or same punch object with `"created":false`).

3. **Concurrent retry test** (simulate race):
   ```bash
   # Run two near-simultaneous requests with same Idempotency-Key
   for i in {1..5}; do
     curl -s -X POST http://localhost:3000/api/punches \
       -H "Content-Type: application/json" \
       -H "Idempotency-Key: race-key" \
       -d '{"employee_id":99,"punch_type":"in","date":"2026-05-02"}' &
   done
   wait
   ```
   Query DB: should be exactly one row for `(99, 'in', '2026-05-02')`.

4. **Constraint enforcement**: attempt direct duplicate insert via psql should error:
   ```sql
   INSERT INTO punches (id, employee_id, punch_type, date) VALUES (gen_random_uuid(), 1, 'in', '2026-05-02');
   -- second insert should violate unique constraint
   ```
