## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk check→execution) with Postgres-backed event log, CF Workers queue for task orchestration, Redis cache for hot state/idempotency, Supabase-hosted Postgres for durable state and immutable audit entries, one broker/OMS connector (sandbox mode), one data vendor connector (sandbox), compliance guardrails (limit/venue/concentration) evaluated synchronously, API routes for triggering runs and querying status, contract tests and replay harness for deterministic runs, free-tier deployment on CF Workers + Supabase.

## Open questions
- Deterministic replay storage strategy: event log size vs snapshot cadence — what retention and snapshot interval balances cost and replay time?
- Idempotency and exactly-once semantics across brokers/OMS: which idempotency-key scope and TTL prevent duplicates without violating exchange semantics?
- Compliance guardrail evaluation latency budget: synchronous vs asynchronous checks and circuit-breaker thresholds to avoid blocking fast paths?
- Vendor connector SLAs and auth model: OAuth2 flows, certificate pinning, and failover behavior for each broker/OMS/data vendor?
- Edge vs centralized placement for CF Workers: which workflows run at edge vs region to minimize latency while preserving transactional consistency with Postgres?
- Audit immutability and tamper evidence: append-only log placement (Supabase + optional hash-chain anchoring) and key rotation policy?
- NSM instrumentation and weighting: exact formula for weighting successful workflow runs by fee-generating status and how to capture partial fills/cancellations?
