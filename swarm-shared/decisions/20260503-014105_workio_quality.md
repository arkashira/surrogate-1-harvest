# workio / quality

## 1. Diagnosis

- LINE webhook endpoint lacks idempotency handling: retries and at-least-once delivery can create duplicate punches/leave/OT rows when the same event is delivered multiple times.
- Punch state transitions are not race-safe: concurrent webhook deliveries for the same user can produce multiple open punches or invalid state jumps (e.g., two “in” punches without an “out”).
- No transactional boundary across punch insert + previous-state update: partial failures leave inconsistent state (e.g., punch recorded but previous open punch not closed).
- No deduplication key from LINE in application layer: LINE’s webhook does not provide an idempotency key, so we must derive one from stable request content (signature + timestamp + event fields).
- Missing unique constraint in DB to enforce single open punch per user: DB-level guard prevents duplicates even if app logic races.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/line/webhook.ts` (or equivalent route handling LINE webhooks)  
Scope: add idempotency check + transactional upsert + DB unique constraint on `punches(user_id, clock_out_at)` where `clock_out_at IS NULL` to enforce at-most-one-open punch.

```typescript
// 2) server/src/routes/line/webhook.ts
import { Request, Response } from 'express';
import crypto from 'crypto';
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

function stableIdempotencyKey(body: any, timestamp: string): string {
  // Deterministic, order-stable hash of the event payload + timestamp
  const normalized = JSON.stringify(body, Object.keys(body).sort());
  return crypto
    .createHash('sha256')
    .update(`${timestamp}:${normalized}`)
    .digest('hex');
}

export async function lineWebhookHandler(req: Request, res: Response) {
  const signature = req.headers['x-line-signature'] as string;
  const body = req.body;
  const receivedAt = new Date().toISOString();

  // Basic validation
  if (!body.events?.length) {
    return res.status(400).json({ error: 'No events' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const event of body.events) {
      if (event.type !== 'message' && event.type !== 'postback') continue;

      // Derive idempotency key from stable content
      const idemKey = stableIdempotencyKey(event, receivedAt);
      const keyBuffer = Buffer.from(idemKey, 'hex');

      // Idempotency check
      const idem = await client.query(
        'SELECT 1 FROM line_webhook_idempotency WHERE idempotency_key = $1',
        [keyBuffer]
      );
      if (idem.rowCount > 0) {
        // Already processed — skip but continue processing other events
        continue;
      }

      // Record idempotency key first (prevents races)
      await client.query(
        'INSERT INTO line_webhook_idempotency (idempotency_key) VALUES ($1)',
        [keyBuffer]
      );

      const userId = event.source.userId;
      const now = new Date();

      if (event.message?.text?.toLowerCase().includes('in') || event.postback?.data?.includes('in')) {
        // Try to insert new open punch; unique index prevents duplicates
        try {
          await client.query(
            `INSERT INTO punches (user_id, clock_in_at, clock_out_at, created_at)
             VALUES ($1, $2, NULL, $3)`,
            [userId, now, now]
          );
        } catch (err: any) {
          // If unique violation on open punch, treat as duplicate or invalid transition
          if (err.code === '23505') {
            // Optional: log duplicate attempt
            continue;
          }
          throw err;
        }
      } else if (event.message?.text?.toLowerCase().includes('out') || event.postback?.data?.includes('out')) {
        // Close the most recent open punch for this user
        const result = await client.query(
          `UPDATE punches
           SET clock_out_at = $1
           WHERE user_id = $2 AND clock_out_at IS NULL
           ORDER BY clock_in_at DESC
           LIMIT 1
           RETURNING *`,
          [now, userId]
        );
        if (result.rowCount === 0) {
          // No open punch to close — could log or create audit entry
        }
      }

      // Leave/OT handling could be added here with similar transactional/idempotent patterns
    }

    await client.query('COMMIT');
    return res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    return res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
}
```
