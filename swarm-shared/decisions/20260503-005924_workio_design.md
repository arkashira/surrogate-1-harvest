# workio / design

## Final Synthesized Answer

### 1. Diagnosis (merged, de-duplicated)
- **No DB-level guardrail**: missing partial unique index to enforce “one open punch per user” (`clock_out_at IS NULL`), allowing duplicates under LINE webhook retries, client double-taps, or concurrent requests.
- **Race-prone upsert**: app-layer find-then-insert is vulnerable to concurrency; must be replaced with atomic DB operation.
- **Missing idempotency key**: LINE may redeliver webhooks; without a stored idempotency token (e.g., `line_delivery_id`), duplicates can be created.
- **Soft-deletes break uniqueness**: if `deleted_at` is used, uniqueness constraints must exclude soft-deleted rows.
- **No monotonic time / state checks**: late/out-of-order punches could reopen an already-closed day; invalid states (NULL/NULL, open punch without location) must be rejected.
- **Missing audit trail**: need precise `created_at` and origin metadata (`line_delivery_id`, IP, device) for debugging retries/fraud.

### 2. Proposed Change (scope + goal)
- **Files**:
  - `/opt/axentx/workio/server/src/db/schema.sql` (add columns, partial unique index, idempotency index).
  - `/opt/axentx/workio/server/src/services/clockService.ts` (atomic upsert with idempotency and validation).
  - `/opt/axentx/workio/server/src/routes/lineWebhook.ts` (webhook handler with safe error handling).
- **Goal**: enforce single open punch at DB level, make LINE webhook handler idempotent and concurrency-safe, and require valid location on clock-in.

### 3. Implementation

#### 3.1 DB schema (`schema.sql`)
```sql
-- Audit and idempotency columns
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS line_delivery_id VARCHAR(255),
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ;

-- One open punch per user (exclude soft-deleted rows)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
  ON punches (user_id, date)
  WHERE clock_out_at IS NULL AND deleted_at IS NULL;

-- Idempotency dedupe for LINE deliveries
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_delivery_id
  ON punches (line_delivery_id)
  WHERE line_delivery_id IS NOT NULL;
```

#### 3.2 Clock service — atomic, idempotent punch (`clockService.ts`)
```ts
// /opt/axentx/workio/server/src/services/clockService.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function recordPunch({
  userId,
  date,
  lineDeliveryId,
  latitude,
  longitude,
}: {
  userId: string;
  date: string; // YYYY-MM-DD
  lineDeliveryId: string;
  latitude?: number;
  longitude?: number;
}) {
  // Reject invalid state early
  if (latitude == null || longitude == null) {
    const err: any = new Error('Location (latitude, longitude) is required on clock-in');
    err.code = 'INVALID_LOCATION';
    throw err;
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency: return existing if same delivery already recorded
    const idem = await client.query(
      `SELECT id, clock_out_at FROM punches WHERE line_delivery_id = $1`,
      [lineDeliveryId]
    );
    if (idem.rowCount > 0) {
      await client.query('COMMIT');
      return idem.rows[0];
    }

    // Close any previous open punch for this user/date (safety)
    await client.query(
      `UPDATE punches
       SET clock_out_at = NOW()
       WHERE user_id = $1 AND date = $2 AND clock_out_at IS NULL AND deleted_at IS NULL`,
      [userId, date]
    );

    // Atomic insert; partial unique index will block concurrent open punches
    const res = await client.query(
      `INSERT INTO punches (user_id, date, clock_in_at, clock_out_at, latitude, longitude, line_delivery_id)
       VALUES ($1, $2, NOW(), NULL, $3, $4, $5)
       RETURNING *`,
      [userId, date, latitude, longitude, lineDeliveryId]
    );

    await client.query('COMMIT');
    return res.rows[0];
  } catch (err: any) {
    await client.query('ROLLBACK');
    if (err.code === '23505') {
      // Unique violation: either duplicate open punch or duplicate delivery id
      const conflict = new Error('Duplicate punch detected') as any;
      conflict.code = 'DUPLICATE_PUNCH';
      throw conflict;
    }
    throw err;
  } finally {
    client.release();
  }
}
```

#### 3.3 LINE webhook handler (`lineWebhook.ts`)
```ts
// /opt/axentx/workio/server/src/routes/lineWebhook.ts
import { recordPunch } from '../services/clockService';

app.post('/webhook/line', async (req, res) => {
  const events = req.body.events || [];
  for (const ev of events) {
    if (ev.type === 'message' && ev.message.type === 'text') {
      const lineDeliveryId = ev.delivery?.id || ev.message.id;
      const userId = await mapLineUserIdToAppUser(ev.source.userId); // implement mapping
      const today = new Date().toISOString().slice(0, 10);

      try {
        await recordPunch({
          userId,
          date: today,
          lineDeliveryId,
          latitude: /* populate from location event or user context */ undefined,
          longitude: /* populate from location event or user context */ undefined,
        });
      } catch (err: any) {
        if (err.code === 'DUPLICATE_PUNCH') {
          // Safe to acknowledge; likely duplicate webhook or race
          console.warn('Duplicate punch for', { userId, lineDeliveryId });
          continue;
        }
        if (err.code === 'INVALID_LOCATION') {
          console.warn('Invalid location for punch', { userId, lineDeliveryId });
          continue;
        }
        console.error('Punch failed', err);
        return res.status(500).send('Punch failed');
      }
    }
  }
  res.status(200).send('OK');
});
```

### 4. Verification (concrete checks)
- **DB constraint**: run  
  `SELECT user_id, date, COUNT(*) FROM punches WHERE clock_out_at IS NULL AND deleted_at IS NULL GROUP BY user_id, date HAVING COUNT(*) > 1;`  
  Expect zero rows. Attempt to insert a second open punch for same user/date and confirm unique violation.
- **Idempotency**: call `recordPunch` twice with the same `lineDeliveryId`; second call must return the existing row without creating a duplicate.
- **Concurrency**: fire two parallel `recordPunch` calls for same user/date with different `lineDeliveryId`; one should succeed, the other should throw `DUPLICATE_PUNCH` (or be blocked by index), and only one open punch should exist.
- **Location validation**: attempt punch with `latitude` or `longitude` null/undefined; must throw `INVALID_LOCATION`.
- **Soft-delete safety**: create a soft-deleted open punch (`deleted_at` set), then attempt a new punch for same user/date; it should succeed because the soft-deleted row is excluded from the uniqueness constraint.
