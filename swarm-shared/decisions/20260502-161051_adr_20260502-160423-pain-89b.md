## Deployment
hybrid (client binaries distributed via GitHub Releases; optional CF Workers for meek fronting endpoints)

## v0.1 scope
v0.1 in 2 weeks: single-platform CLI client (Linux x86_64) that establishes a stealth SOCKS5 listener, transparently routes traffic through a TUN interface using boringtun+smoltcp, applies shadowsocks-aead obfuscation to egress, enforces fail-closed kill-switch on disconnect/block, stores local session/config in SQLite, and emits minimal opt-in telemetry (session id, started/ended, blocked flag, leaked_packets) for WAU-ZB measurement — no server components, no UI, no multi-region endpoints yet.

## Open questions
- Which obfuscation primitives satisfy both undetectability under active DPI and acceptable latency on high-loss links?
- How will clients discover/rotate meek fronting endpoints without centralized infra that can be blocked?
- What local heuristics determine 'blocked' vs transient loss to drive fail-closed kill-switch without false positives?
- How to implement cross-platform TUN setup (Linux tun/tap, Windows Wintun, macOS utun) with minimal privileges?
- What telemetry schema and consent flow balances WAU-ZB measurement with privacy and deniability?
