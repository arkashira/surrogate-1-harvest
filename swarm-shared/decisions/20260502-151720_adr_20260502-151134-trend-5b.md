## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk check → simulated execution) with immutable audit trail, Redis-backed hot cache for limits, Postgres-backed replayable traces, Cloudflare Queues for tasking, and JSON-based guardrails; no live broker integration (simulated execution only), API-only (no frontend), deployable on CF Workers + Supabase free tier.

## Open questions
- Deterministic replay fidelity: should workflow_run.trace capture full input/output snapshots or deterministic event sourcing to minimize storage?
- Latency SLA target for sub-second orchestration: budget for cold-starts on CF Workers vs containerized HF Space trade-offs?
- Compliance guardrail language: JSON rules engine vs Python predicates (security review and auditability)?
- Connector credential custody: Supabase vault/encrypted columns vs external secrets manager within free-tier constraints?
- Idempotency and exactly-once execution semantics across brokers/OMS: dedupe keys and compensating actions strategy?
