# workio / quality

## Final Synthesis — One Correct, Actionable Plan

**Core principle**: enforce idempotency and atomic state at the system boundary (DB) so retries, concurrency, and partial failures cannot corrupt time records. Favor DB-level guarantees over app-level checks.

---

### 1. Diagnosis (merged + resolved)

- **Duplicate root cause**: LINE webhook at-least-once + no idempotency key + no DB uniqueness → duplicate punches and race conditions.
- **Race condition**: non-atomic find-then-insert/update allows multiple open punches or lost transitions under concurrent deliveries.
- **Partial failure**: punch can persist without location/notification/audit, causing drift between UI, reports, and downstream consumers.
- **Lost updates**: timesheet/approval edits lack versioning; last-write-wins overwrites concurrent changes.
- **Frontend hazard**: ClockButton allows double-taps and in-flight retries; no client-side guard.
- **Verification gap**: no deterministic replay harness to confirm fixes.

**Resolution priority**:
1. DB uniqueness + idempotency key (hard guarantee).
2. Transactional write across punch + location + audit/notification (atomicity).
3. Optimistic locking on timesheet/leave/OT rows (correctness).
4. Frontend debounce/disabled state (UX safety).
5. Reproducible verification script (confidence).

### 3. Service Layer — Transactional, Idempotent Punch Creation

File: `workio/server/src/services/punchService.ts`

```ts
import { pool } from '../db';

export interface CreatePunchOptions {
  userId: number;
  type: 'clock_in' | 'clock_out' | 'break_start' | 'break_end';
  latitude?: number;
  longitude?: number;
  idempotencyKey: string; // e.g., LINE deliveryId or client uuid
}

export async function createPunch(opts: CreatePunchOptions) {
  const conn = await pool.getConnection();
  try {
    await conn.beginTransaction();

    // 1) Idempotency: return existing punch if key already seen
    const [idempotentRows] = await conn.query<RowDataPacket[]>(
      'SELECT id, type, status FROM punches WHERE idempotency_key = ?',
      [opts.idempotencyKey]
    );
    if (idempotentRows?.[0]) {
      await conn.commit();
      return { id: idempotentRows[0].id, replayed: true };
    }

    // 2) Business rule enforcement: one open clock_in per user per day
    if (opts.type === 'clock_in') {
      const [openRows] = await conn.query<RowDataPacket[]>(
        `SELECT id FROM punches
         WHERE user_id = ? AND status = 'clock_in' AND DATE(created_at) = CURDATE()
         FOR UPDATE`,
        [opts.userId]
      );
      if (openRows?.[0]) {
        throw new Error('Duplicate clock_in for today');
      }
    }

    // 3) Compute status from type (keep simple and explicit)
    const status = opts.type === 'clock_in' ? 'clock_in' : 'clock_out';

    // 4) Insert punch atomically
    const [result] = await conn.query(
      `INSERT INTO punches
        (user_id, type, status, latitude, longitude, idempotency_key, version, created_at, updated_at)
       VALUES (?, ?, ?, ?, ?, ?, 1, NOW(), NOW())`,
      [
        opts.userId,
        opts.type,
        status,
        opts.latitude ?? null,
        opts.longitude ?? null,
        opts.idempotencyKey,
      ]
    );
    const punchId = result.insertId;

    // 5) Side-effects in same transaction: audit + notification record
    await conn.query(
      'INSERT INTO punch_audit (punch_id, action, created_at) VALUES (?, ?, NOW())',
      [punchId, 'created']
    );

    // Optional: persist notification intent so worker can retry safely
    await conn.query(
      'INSERT INTO punch_notifications (punch_id, user_id, status, created_at) VALUES (?, ?, ?, NOW())',
      [punchId, opts.userId, 'pending']
    );

    await conn.commit();
    return { id: punchId, replayed: false };
  } catch (err) {
    await conn.rollback();
    throw err;
  } finally {
    conn.release();
  }
}

// Optimistic-lock update helper for timesheet/leave/OT
export async function updateWithVersion<T extends { id: number; version: number }>(
  table: string,
  id: number,
  version: number,
  fields: Partial<T>
) {
  const sets = Object.keys(fields)
    .map((k) => `${k} = ?`)
    .join(', ');
  const values = [...Object.values(fields), version + 1, id, version];
  const [result] = await pool.query(
    `UPDATE ${table} SET ${sets}, version = ? WHERE id = ? AND version = ?`,
    values
  );
  if (result.affectedRows === 0) {
    throw new Error('Concurrent modification detected');
  }
}
```

### 4. Webhook Handler — Fast, Idempotent, Safe

File: `workio/server/src/routes/webhook/line.ts`

```ts
import { Router } from 'express';
import { createPunch } from '../services/punchService';

const router = Router();

router.post('/line', async (req, res) => {
  try {
    const { userId, type, latitude, longitude, idempotencyKey } = req.body;

    if (!idempotencyKey) {
      return res.status(400).json({ error: 'idempotencyKey required' });
    }

    // Optional: inspect LINE retry header for observability
    const retryCount = req.get('X-Line-Retry-Count') || '0';

    const outcome = await createPunch({
      userId,
      type,
      latitude,
      longitude,
      idempotencyKey,
    });

    // 200 OK for both new and replayed to satisfy at-least-once delivery
    return res.json({ ok: true, id: outcome.id, replayed: outcome.replayed });
  } catch (err: any) {
    if (err.message?.includes('Duplicate clock_in')) {
      return res.status(409).json({ error: err.message });
    }
    console.error('[line-webhook]', err);
    return res.status(500).json({ error: 'internal' });
  }
});

export default router;
```
