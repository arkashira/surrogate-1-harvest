## Deployment
cf workers (serverless edge) + optional hf space fallback; rationale: free-tier, no infra to manage, global edge presence to reduce latency and blend with normal traffic

## v0.1 scope
v0.1 ships a single Rust CLI client (Linux x86_64) that: creates a local SOCKS5 listener, obfuscates traffic via Shadowsocks-like stream cipher, tunnels through a CF Worker edge endpoint, exposes a TUN interface via boringtun, enforces a kill-switch on stealth-path failure, stores config/telemetry in local SQLite, and emits opt-in WAU-ZB events; no server management, no GUI, no multi-region failover

## Open questions
- Which obfuscation profiles reliably evade current DPI in target regions (e.g., TLS-fragment vs. Shadowsocks 2022)?
- Can CF Workers sustain long-lived, high-throughput WebSocket or HTTP/2 tunnels without cost or policy limits?
- What minimal client-side telemetry schema satisfies WAU-ZB while preserving privacy and avoiding deanonymization?
- How will we distribute and auto-update client binaries without centralized app stores or blocked channels?
- What is the exact kill-switch behavior on TUN/interface loss (block all, allow LAN, fallback to proxy)?
