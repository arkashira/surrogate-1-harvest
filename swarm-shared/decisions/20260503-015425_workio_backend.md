# workio / backend

## Final consolidated solution (best parts, corrected + actionable)

### 1. Diagnosis (merged & clarified)
- **No idempotency enforcement on LINE webhook retries** — identical `X-Line-Signature` + body can be retried and create duplicate punches.
- **Race condition in punch upsert** — read-then-write (`findOne → update/insert`) allows concurrent deliveries to create multiple active punches for the same user/date.
- **Missing DB uniqueness constraint for active punch** — without a DB-level guard, app-level checks are not safe under concurrency.
- **No transactional safety across punch + audit + notification** — partial failures can leave state inconsistent and users without confirmation.
- **No persisted webhook event log** — impossible to replay, audit, or debug deliveries and idempotency decisions.

### 2. Scope of changes
- **Schema**: add `webhook_events` table for idempotency/audit; add unique constraint/index to enforce one active punch per user per date.
- **Handler**: refactor LINE webhook handler into a single DB transaction that:
  1. Deduplicates by `(line_signature, payload_hash)` using an atomic upsert into `webhook_events`.
  2. Performs atomic punch upsert using a DB-level unique constraint and `ON CONFLICT`.
  3. Optionally queues or sends LINE reply outside the transaction (best-effort) so TX success isn’t tied to notification delivery.
- **Migrations**: apply schema changes safely; backfill/resolve existing duplicates before enforcing constraint.

### 3. Implementation

#### 3.1 DB schema (`server/src/db/schema.sql`)
```sql
-- Webhook event log for idempotency and audit
CREATE TABLE IF NOT EXISTS webhook_events (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  event_type      TEXT NOT NULL,
  line_signature  TEXT NOT NULL,
  payload_hash    TEXT NOT NULL,
  payload         JSONB,
  created_at      TIMESTAMPTZ DEFAULT now(),
  processed_at    TIMESTAMPTZ,
  status          TEXT NOT NULL DEFAULT 'pending', -- pending, processed, duplicate, failed
  CONSTRAINT uq_webhook_signature_payload UNIQUE (line_signature, payload_hash)
);

-- Ensure one active punch per user per date (clock_out_at NULL means active)
-- Assumes punches has: id, user_id, tenant_id, date, type, clock_in_at, clock_out_at, ...
CREATE UNIQUE INDEX IF NOT EXISTS uq_active_punch
ON punches (user_id, date)
WHERE (clock_out_at IS NULL);
```

#### 3.2 Apply migrations safely
```bash

psql "$DATABASE_URL" -f server/src/db/schema.sql
```
Resolve existing duplicates before enforcing:
```sql
-- Example cleanup (review before running)
WITH ranked AS (
  SELECT id,
         row_number() OVER (PARTITION BY user_id, date ORDER BY clock_in_at DESC) AS rn
  FROM punches
  WHERE clock_out_at IS NULL
)
DELETE FROM punches
WHERE id IN (SELECT id FROM ranked WHERE rn > 1);
```

#### 3.3 Webhook handler (`server/src/routes/line.ts`)
```ts
import express from 'express';
import { Pool, PoolClient } from 'pg';
import crypto from 'crypto';
import { Client } from '@line/bot-sdk';

const router = express.Router();
const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const lineClient = new Client({ channelAccessToken: process.env.LINE_CHANNEL_ACCESS_TOKEN || '' });

function sha256Hex(data: string): string {
  return crypto.createHash('sha256').update(data, 'utf8').digest('hex');
}

async function upsertPunchWithIdempotency(
  client: PoolClient,
  {
    userId,
    tenantId,
    date,
    clockInAt,
    eventType,
    lineSignature,
    payload,
    payloadHash,
  }: {
    userId: string;
    tenantId: string;
    date: string; // YYYY-MM-DD
    clockInAt: Date;
    eventType: string;
    lineSignature: string;
    payload: any;
    payloadHash: string;
  }
) {
  // 1) Idempotency guard: insert webhook event or detect duplicate
  const eventRes = await client.query(
    `INSERT INTO webhook_events (event_type, line_signature, payload_hash, payload, status, processed_at)
     VALUES ($1,$2,$3,$4,'processed',now())
     ON CONFLICT (line_signature, payload_hash)
     DO UPDATE SET processed_at = EXCLUDED.processed_at
     RETURNING id, (xmax = 0) AS inserted`,
    [eventType, lineSignature, payloadHash, payload]
  );

  const isDuplicate = !eventRes.rows[0].inserted;
  if (isDuplicate) {
    // Return existing active punch without mutating
    const existing = await client.query(
      `SELECT * FROM punches WHERE user_id=$1 AND date=$2 AND clock_out_at IS NULL LIMIT 1`,
      [userId, date]
    );
    return { duplicate: true, punch: existing.rows[0] || null };
  }

  // 2) Atomic punch upsert using DB unique constraint
  const punchRes = await client.query(
    `INSERT INTO punches (user_id, tenant_id, date, type, clock_in_at, clock_out_at, created_at, updated_at)
     VALUES ($1,$2,$3,$4,$5,NULL,now(),now())
     ON CONFLICT (user_id, date) WHERE (clock_out_at IS NULL)
     DO UPDATE SET clock_in_at = EXCLUDED.clock_in_at, updated_at = now()
     RETURNING *`,
    [userId, tenantId, date, 'clock_in', clockInAt]
  );

  return { duplicate: false, punch: punchRes.rows[0] };
}

router.post('/webhook/line', async (req, res) => {
  const lineSignature = (req.headers['x-line-signature'] as string) || '';
  const body = req.body;
  const rawBody = JSON.stringify(body);
  const payloadHash = sha256Hex(rawBody);

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    const event = body.events?.[0];
    if (!event || event.type !== 'message' || event.message?.type !== 'text') {
      await client.query('ROLLBACK');
      return res.status(400).json({ error: 'Unsupported event' });
    }

    const userId = event.source.userId;
    const tenantId = 'default'; // derive from user/DB mapping in production
    const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
    const clockInAt = new Date();

    const result = await upsertPunchWithIdempotency(client, {
      userId,
      tenantId,
      date: today,
      clockInAt,
      eventType: 'clock_in',
      lineSignature,
      payload: body,
      payloadHash,
    });

    // Best-effort LINE reply; don't fail TX on notification errors
    lineClient.pushMessage(userId, {
      type: 'text',
      text: result.duplicate
        ? 'Duplicate clock-in received — no change made.'
        : `Clock-in recorded at ${clockInAt.toLocaleTimeString()}.`,
    }).catch((err) => console.error('LINE push failed', err));

    await client.query('COMMIT');
    return res.json({ ok: true, duplicate: result.duplicate, punch: result.punch });
  } catch (err) {
    await client.query('ROLLBACK').catch(() => {});
    console.error('Webhook tx failed', err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

### 4. Verification plan (concrete tests)
1. **Idempotency test**
   - Send identical LINE webhook payload twice (same `X-Line-Signature` and body) within seconds.
   - Expect: single punch row; second request returns `duplicate: true` and does not mutate punch.

2. **Race-condition test**
   - Fire 10 concurrent requests with the same payload.
   - Expect: exactly one active punch row; no duplicates.

3. **Constraint enforcement test**
   - Manually insert two active punches for same user/date
