# workio / ops

## 1. Diagnosis

- No storage-level uniqueness on `(employee_id, punch_type, date)` allows duplicate punches from retries/races.
- Punch write path is read-then-insert (non-idempotent) and vulnerable to LINE webhook redelivery and client retries.
- No idempotency key or deterministic punch hash to de-duplicate concurrent requests at the API layer.
- Clock-in/out endpoint lacks transactional safety and conflict feedback (returns 200 even when duplicate exists).
- Missing index on `(employee_id, punch_type, date)` makes duplicate checks slow at scale.

## 2. Proposed change

- **File**: `workio/server/src/db/schema.sql` (add constraint + index)  
- **File**: `workio/server/src/routes/punch.ts` (make upsert idempotent + return conflict status)  
- **Scope**: Add unique constraint, replace insert with upsert, add idempotency via deterministic hash, and return 409 on conflict when same punch already recorded.

## 3. Implementation

### 3.1 DB schema (constraint + index)

```sql
-- workio/server/src/db/schema.sql
-- Ensure punches uniqueness per employee per day per punch_type
ALTER TABLE punches
  ADD CONSTRAINT punches_employee_day_type_unique
  UNIQUE (employee_id, punch_type, date);

-- Fast lookup for duplicate checks and day queries
CREATE INDEX IF NOT EXISTS idx_punches_employee_day_type
  ON punches (employee_id, date, punch_type);
```

### 3.2 Idempotent upsert endpoint

```ts
// workio/server/src/routes/punch.ts
import { Request, Response } from 'express';
import { db } from '../db';
import { sql } from 'drizzle-orm';

// POST /api/punch
export async function createPunch(req: Request, res: Response) {
  const { employee_id, punch_type, date, latitude, longitude, image_url } = req.body;

  if (!employee_id || !punch_type || !date) {
    return res.status(400).json({ error: 'Missing required fields' });
  }

  try {
    // Idempotent upsert: insert or do nothing if unique conflict
    // Returns existing row when conflict so client gets deterministic state
    const [record] = await db.execute(sql`
      INSERT INTO punches (employee_id, punch_type, date, latitude, longitude, image_url, created_at)
      VALUES (${employee_id}, ${punch_type}, ${date}, ${latitude}, ${longitude}, ${image_url}, NOW())
      ON CONFLICT (employee_id, punch_type, date) DO NOTHING
      RETURNING *
    `);

    // If insert did nothing, fetch existing to return consistent payload
    if (!record) {
      const [existing] = await db.execute(sql`
        SELECT * FROM punches
        WHERE employee_id = ${employee_id} AND punch_type = ${punch_type} AND date = ${date}
        LIMIT 1
      `);
      return res.status(409).json({
        message: 'Punch already exists',
        punch: existing,
      });
    }

    return res.status(201).json({ punch: record });
  } catch (err) {
    console.error('Punch upsert failed', err);
    return res.status(500).json({ error: 'Internal server error' });
  }
}
```

### 3.3 Optional: deterministic idempotency key (middleware)

If you want to support client-supplied idempotency keys for retries across different punch types/days:

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';

export function idempotencyKey(req: Request, res: Response, next: NextFunction) {
  const key = req.header('Idempotency-Key') || req.body.idempotency_key;
  if (key) {
    req.idempotencyKey = crypto.createHash('sha256').update(key).digest('hex');
  }
  next();
}
```

Use `req.idempotencyKey` in a separate `punch_idempotency` table if you need cross-request dedupe beyond the natural key.

1. Apply schema change:

2. Start backend:
   ```bash
   cd workio/server && npm run dev
   ```
3. Test duplicate prevention:
   ```bash
   # First request -> 201
   curl -X POST http://localhost:3000/api/punch \
     -H "Content-Type: application/json" \
     -d '{"employee_id":1,"punch_type":"clock_in","date":"2026-05-02","latitude":13.75,"longitude":100.55}'

   # Immediate retry -> 409 with existing punch
   curl -X POST http://localhost:3000/api/punch \
     -H "Content-Type: application/json" \
     -d '{"employee_id":1,"punch_type":"clock_in","date":"2026-05-02","latitude":13.75,"longitude":100.55}'
   ```
4. Confirm constraint blocks direct DB duplicates:
   ```sql
   INSERT INTO punches (employee_id, punch_type, date) VALUES (1, 'clock_in', '2026-05-02');
   -- second insert -> ERROR: duplicate key violates unique constraint
   ```
