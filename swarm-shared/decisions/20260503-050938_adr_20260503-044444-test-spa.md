## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP that connects one AWS account and one GCP project via service credentials, inventories Lambda + Cloud Functions schedules into local Postgres (or Supabase free tier), detects exact-duplicate cron+target pairs and orphaned schedules (no backing function), and outputs a JSON report with remediation hints. No UI; deployable as a single CF Worker route that triggers on-demand runs and stores findings in Supabase.

## Open questions
- Which identity model will customers use to grant read-only access (AWS IAM roles vs keys, GCP service accounts) and how do we store credentials safely?
- What is the minimal viable schedule for reconciling drift (daily vs hourly) while staying within free-tier invocation limits?
- How will we deduplicate across providers and customers (hash of cron+target vs semantic grouping) and what tolerance for false positives?
