# workio / backend

## 1. Diagnosis

- No database-level uniqueness guard for open punches: multiple concurrent/retried LINE webhooks can create >1 `clock_out_at IS NULL` row per user per day, corrupting attendance state.
- App-layer “find then insert/update” is racy under LINE retries and parallel webhook deliveries; no locking or atomic upsert.
- Missing idempotency key on webhook handler: LINE may redeliver the same event (network glitch), creating duplicate punches.
- Soft deletes (`deleted_at`) are not consistently indexed and can be returned in active punch queries, causing double-clock-out or ghost records.
- No compound index to support the common “latest open punch per user” query; queries do full scans as tenant+user counts grow.

## 2. Proposed change

File scope:  
- `server/src/db/schema.sql` — add constraint + indexes  
- `server/src/routes/webhook/line.ts` — add idempotency + atomic upsert  
- `server/src/services/punch.service.ts` — use `INSERT ... ON CONFLICT` with row-level locking fallback

## 3. Implementation

### 3.1 Database schema (migrations)

Add a partial unique index to enforce at most one open punch per user (and fix soft-delete/index performance):

```sql
-- server/src/db/migrations/0002_punches_guardrails.sql
-- Run: psql workio < server/src/db/migrations/0002_punches_guardrails.sql

-- Prevent multiple open punches per user (NULL clock_out_at means "still clocked in")
CREATE UNIQUE INDEX idx_punches_one_open_per_user
ON punches (tenant_id, user_id, DATE(created_at))
WHERE clock_out_at IS NULL;

-- Fast lookup for latest punch (tenant, user, recent)
CREATE INDEX idx_punches_tenant_user_created_desc
ON punches (tenant_id, user_id, created_at DESC)
WHERE deleted_at IS NULL;

-- Idempotency for LINE events (same event id should not create duplicate punches)
CREATE UNIQUE INDEX idx_punches_line_event_id
ON punches (line_event_id)
WHERE line_event_id IS NOT NULL AND deleted_at IS NULL;
```

### 3.2 Punch service — atomic upsert

```ts
// server/src/services/punch.service.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function clockInOrOut({
  tenantId,
  userId,
  lineEventId,
  latitude,
  longitude,
  note,
}: {
  tenantId: string;
  userId: string;
  lineEventId: string;
  latitude?: number;
  longitude?: number;
  note?: string;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency: if this LINE event already processed, return existing punch
    const existing = await client.query(
      `SELECT id FROM punches WHERE line_event_id = $1 AND deleted_at IS NULL`,
      [lineEventId]
    );
    if (existing.rows[0]) {
      await client.query('COMMIT');
      return existing.rows[0];
    }

    // Try to close an open punch first (clock-out)
    const closeRes = await client.query(
      `UPDATE punches
       SET clock_out_at = NOW(), updated_at = NOW(), note = $1
       WHERE tenant_id = $2 AND user_id = $3 AND clock_out_at IS NULL AND deleted_at IS NULL
       RETURNING *`,
      [note, tenantId, userId]
    );

    if (closeRes.rows[0]) {
      await client.query('COMMIT');
      return closeRes.rows[0];
    }

    // Otherwise create a new clock-in
    const insertRes = await client.query(
      `INSERT INTO punches (tenant_id, user_id, clock_in_at, clock_out_at, latitude, longitude, line_event_id, note, created_at, updated_at)
       VALUES ($1, $2, NOW(), NULL, $3, $4, $5, $6, NOW(), NOW())
       RETURNING *`,
      [tenantId, userId, latitude, longitude, lineEventId, note]
    );

    await client.query('COMMIT');
    return insertRes.rows[0];
  } catch (err: any) {
    await client.query('ROLLBACK');
    // Unique violation on idx_punches_one_open_per_user or idx_punches_line_event_id is acceptable race outcome
    if (err.code === '23505') {
      // Fetch current open punch (or closed by concurrent request)
      const safe = await client.query(
        `SELECT id FROM punches WHERE tenant_id = $1 AND user_id = $2 AND deleted_at IS NULL ORDER BY created_at DESC LIMIT 1`,
        [tenantId, userId]
      );
      return safe.rows[0] || null;
    }
    throw err;
  } finally {
    client.release();
  }
}
```

### 3.3 Webhook handler — enforce idempotency and tenant context

```ts
// server/src/routes/webhook/line.ts
import { Router } from 'express';
import { clockInOrOut } from '../../services/punch.service';
import { verifyLineSignature } from '../../utils/line';

const router = Router();

router.post('/line', async (req, res) => {
  try {
    const sig = req.headers['x-line-signature'] as string;
    const body = req.body;
    if (!verifyLineSignature(JSON.stringify(body), sig)) {
      return res.status(401).send('Invalid signature');
    }

    const events = body.events || [];
    for (const ev of events) {
      if (ev.type !== 'message' || ev.message?.type !== 'text') continue;

      // Map LINE userId + tenant by your mapping strategy (example: source from DB)
      const tenantId = process.env.DEFAULT_TENANT_ID; // replace with real tenant resolution
      const userId = await resolveLineUserIdToAppUserId(ev.source.userId, tenantId);

      await clockInOrOut({
        tenantId,
        userId,
        lineEventId: ev.message.id, // LINE message id is stable and unique per event
        note: ev.message.text,
      });
    }

    res.status(200).send('OK');
  } catch (err) {
    console.error('LINE webhook error', err);
    res.status(500).send('Server error');
  }
});

export default router;
```
