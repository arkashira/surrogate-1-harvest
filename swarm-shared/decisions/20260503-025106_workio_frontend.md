# workio / frontend

## 1. Diagnosis

- Clock-in/out UI has no optimistic update + no local rollback on network failure → perceived latency and duplicate taps.
- No client-side idempotency guard against LINE-retry/webhook race (same punch can be applied twice if webhook arrives after optimistic state).
- Location permission flow is blocking and not cached; users re-prompt on every clock-in even when already granted.
- No visual feedback for GPS mismatch (clock-in allowed but location stale) → compliance risk.
- Punch state is fetched on every screen focus (no SWR cache) → unnecessary network load and jank.

## 2. Proposed change

**Scope:** `src/features/punch/PunchButton.tsx` (create if absent) and `src/hooks/usePunch.ts`  
**Goal:** Add optimistic update, idempotency key, cached location, and GPS mismatch guard in <2h.

## 3. Implementation

### `src/hooks/usePunch.ts`

```ts
import useSWR from 'swr';
import { useCallback, useState, useRef } from 'react';

const fetcher = (url: string) => fetch(url).then((r) => r.json());

export function usePunch() {
  const [inflightId, setInflightId] = useState<string | null>(null);
  const locationCache = useRef<GeolocationCoordinates | null>(null);

  const { data: punch, mutate } = useSWR('/api/punch/active', fetcher, {
    revalidateOnFocus: false,
    revalidateOnReconnect: false,
  });

  const getCachedLocation = useCallback(async (): Promise<GeolocationCoordinates | null> => {
    if (locationCache.current) return locationCache.current;
    return new Promise((resolve) => {
      if (!navigator.geolocation) return resolve(null);
      navigator.geolocation.getCurrentPosition(
        (pos) => {
          locationCache.current = pos.coords;
          resolve(pos.coords);
        },
        () => resolve(null),
        { enableHighAccuracy: true, maximumAge: 5 * 60 * 1000, timeout: 8000 }
      );
    });
  }, []);

  const punchIn = useCallback(async () => {
    if (inflightId) return;
    const id = crypto.randomUUID();
    setInflightId(id);

    const coords = await getCachedLocation();
    // optimistic
    const optimistic = { id, type: 'IN', timestamp: new Date().toISOString(), coords, synced: false };
    mutate({ ...punch, ...optimistic }, false);

    try {
      const res = await fetch('/api/punch/in', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ idempotencyKey: id, coords }),
      });
      if (!res.ok) throw new Error('Punch failed');
      const actual = await res.json();
      mutate(actual, true);
    } catch {
      // rollback
      mutate(punch, true);
    } finally {
      setInflightId(null);
    }
  }, [inflightId, punch, mutate, getCachedLocation]);

  const punchOut = useCallback(async () => {
    if (!punch?.id || inflightId) return;
    const id = crypto.randomUUID();
    setInflightId(id);

    const coords = await getCachedLocation();
    const optimistic = { ...punch, type: 'OUT', outTimestamp: new Date().toISOString(), outCoords: coords, synced: false };
    mutate(optimistic, false);

    try {
      const res = await fetch('/api/punch/out', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ punchId: punch.id, idempotencyKey: id, coords }),
      });
      if (!res.ok) throw new Error('Punch failed');
      const actual = await res.json();
      mutate(actual, true);
    } catch {
      mutate(punch, true);
    } finally {
      setInflightId(null);
    }
  }, [punch, inflightId, mutate]);

  return { punch: punch || null, punchIn, punchOut, inflight: !!inflightId };
}
```

### `src/features/punch/PunchButton.tsx`

```tsx
import { usePunch } from '../../hooks/usePunch';
import { useState, useEffect } from 'react';

export function PunchButton() {
  const { punch, punchIn, punchOut, inflight } = usePunch();
  const [geoError, setGeoError] = useState<string | null>(null);

  useEffect(() => {
    if (!navigator.geolocation) {
      setGeoError('Geolocation not supported');
      return;
    }
    navigator.permissions?.query({ name: 'geolocation' as PermissionName }).then((result) => {
      if (result.state === 'denied') setGeoError('Location access denied — clock-in may be limited');
    });
  }, []);

  const isIn = !!punch && !punch.outTimestamp;
  const mismatch = punch?.coords && !punch.outTimestamp && punch.coords.accuracy > 100;

  return (
    <div className="flex flex-col items-center gap-3">
      {geoError && <p className="text-xs text-amber-600">{geoError}</p>}
      {mismatch && <p className="text-xs text-orange-600">Location accuracy low — move to work area</p>}

      <button
        onClick={isIn ? punchOut : punchIn}
        disabled={inflight}
        className={`
          rounded-xl px-8 py-4 text-lg font-semibold text-white shadow-lg transition-all active:scale-95
          ${inflight ? 'cursor-wait bg-slate-400' : ''}
          ${isIn ? 'bg-red-500 hover:bg-red-600' : 'bg-green-600 hover:bg-green-700'}
        `}
      >
        {inflight ? 'Processing…' : isIn ? 'Clock Out' : 'Clock In'}
      </button>

      {punch && (
        <p className="text-xs text-slate-500">
          Last: {isIn ? 'Clocked In' : 'Clocked Out'} · {new Date(isIn ? punch.timestamp : punch.outTimestamp!).toLocaleTimeString()}
        </p>
      )}
    </div>
  );
}
```

### Backend guard (quick note)

Ensure `/api/punch/in` and `/api/punch/out` enforce `idempotencyKey` (unique constraint or Redis lock) and return existing punch if key reused — prevents duplicates from LINE/webhook retries.

## 4. Verification

1. **Optimistic + rollback**  
   - Tap “Clock In” → UI updates instantly.  
   - Block request in DevTools (Network → Offline) → UI rolls back after timeout.

2. **Idempotency**  
   - Rapid double-tap does not create two punches (inflight guard).  
   - Replay same `idempotencyKey` via curl → returns same punch (backend).

3. **Location caching**  
   - Grant location → clock in/out multiple times → no repeated prompts (check console for single geolocation call).

4. **GPS mismatch UI**  
   - Mock low-accuracy location (DevTools → Sensors → accuracy >100m) → orange warning appears.

5. **SWR cache**  
   - Navigate away and back → no network request for active punch (check Network tab).
