# workio / backend

## 1. Diagnosis

- **No idempotency enforcement on LINE webhook**: missing `X-Line-Signature` verification and no idempotency key layer allows duplicate clock-in/out events from LINE retries or client retries.
- **Race condition on open punches**: concurrent requests for same `(tenant_id, employee_id)` with nullable `clock_out_at` can create multiple open punches (no DB uniqueness constraint).
- **No replay protection**: webhook handler accepts same event payload multiple times; no deduplication by `webhook_event_id` or signature-derived key.
- **Missing tenant context in webhook path**: webhook route does not resolve tenant from subdomain or header, making multi-tenant routing fragile.
- **No atomic punch state transition**: clock-in/out logic reads-then-writes without row-level locking or optimistic versioning, risking lost updates under concurrency.

## 2. Proposed change

File scope:  
- `workio/server/src/routes/webhook/line.ts` (create if missing) or update existing webhook handler  
- `workio/server/src/db/schema.sql` — add uniqueness constraint + idempotency table  
- `workio/server/src/services/punchService.ts` — add atomic upsert with `FOR UPDATE` locking  
- `workio/server/src/middleware/tenant.ts` — ensure tenant resolution for webhook route

Change summary:  
- Add `line_webhook_events` idempotency table keyed by `event_id` (from LINE) and `tenant_id`.  
- Verify `X-Line-Signature` using HMAC-SHA256 before processing.  
- Enforce `(tenant_id, employee_id, clock_out_at IS NULL)` partial unique index to prevent double-open punches.  
- Implement atomic punch transition using `SELECT ... FOR UPDATE` and single-statement upsert.

### 3.1 DB schema additions

```sql
-- workio/server/src/db/schema.sql
-- Idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS line_webhook_events (
  id              BIGSERIAL PRIMARY KEY,
  tenant_id       BIGINT NOT NULL,
  event_id        TEXT   NOT NULL,
  event_type      TEXT   NOT NULL,
  payload         JSONB  NOT NULL,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX IF NOT EXISTS uq_line_webhook_event
  ON line_webhook_events (tenant_id, event_id);

-- Prevent multiple open punches per employee per tenant
CREATE UNIQUE INDEX IF NOT EXISTS uq_open_punch
  ON punches (tenant_id, employee_id)
  WHERE clock_out_at IS NULL;
```

### 3.2 Tenant middleware (if not present)

```ts
// workio/server/src/middleware/tenant.ts
import { Request, Response, NextFunction } from 'express';

export function resolveTenant(req: Request, _res: Response, next: NextFunction) {
  // Prefer subdomain; fallback to header; fallback to body for webhook verification
  const host = req.headers.host || '';
  const subdomain = host.split('.')[0];
  const tenantIdHeader = req.headers['x-tenant-id'] as string;
  const tenantId = req.body?.tenantId || tenantIdHeader || (subdomain !== 'localhost' && subdomain !== '127.0.0.1' ? subdomain : null);

  if (!tenantId) {
    return next(new Error('Tenant not resolved'));
  }

  (req as any).tenantId = Number(tenantId);
  next();
}
```

### 3.3 LINE signature verification helper

```ts
// workio/server/src/lib/line.ts
import crypto from 'crypto';

export function verifyLineSignature(rawBody: string, signature: string, channelSecret: string): boolean {
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}
```

### 3.4 Atomic punch service

```ts
// workio/server/src/services/punchService.ts
import { PoolClient } from 'pg';
import { pool } from '../db';

export async function recordPunch(
  tenantId: number,
  employeeId: number,
  clockInAt: Date,
  location?: { lat: number; lng: number },
  client?: PoolClient
): Promise<any> {
  const db = client || pool;
  // Use advisory lock per (tenant,employee) to serialize concurrent requests
  const lockId = ((BigInt(tenantId) << 32n) | BigInt(employeeId)) % BigInt(2147483647);
  await db.query('SELECT pg_advisory_xact_lock($1)', [Number(lockId)]);

  // Check for existing open punch
  const open = await db.query(
    `SELECT id FROM punches
      WHERE tenant_id = $1 AND employee_id = $2 AND clock_out_at IS NULL
      FOR UPDATE`,
    [tenantId, employeeId]
  );

  if (open.rows.length > 0) {
    // clock-out existing
    const result = await db.query(
      `UPDATE punches
        SET clock_out_at = $1, updated_at = NOW(), location_out = $2
        WHERE id = $3
        RETURNING *`,
      [clockInAt, location || null, open.rows[0].id]
    );
    return result.rows[0];
  } else {
    // clock-in new
    const result = await db.query(
      `INSERT INTO punches (tenant_id, employee_id, clock_in_at, location_in)
        VALUES ($1, $2, $3, $4)
        RETURNING *`,
      [tenantId, employeeId, clockInAt, location || null]
    );
    return result.rows[0];
  }
}
```
