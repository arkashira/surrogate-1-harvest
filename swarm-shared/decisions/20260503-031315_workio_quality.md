# workio / quality

## Final Consolidated Implementation

### 1. Diagnosis (merged)
- **Critical**: Missing `X-Line-Signature` verification allows spoofed/replayed events for punches, leave, and OT.
- **Critical**: No idempotency/replay protection; LINE retries on 5xx/timeouts create duplicate `clock_records`, `leave_requests`, `ot_requests`.
- **Critical**: No persistence-layer uniqueness for `line_event_id`; concurrent retries can bypass app-level checks.
- **Important**: No per-user cooldown/guard against rapid repeats (accidental or malicious).
- **Important**: Missing structured audit fields (`source`, `line_event_id`, location, timestamps) for traceability/forensics.
- **Important**: Webhook handler lacks payload validation and safe parsing; malformed payloads can create partial records.
- **UX**: Clock-in/out lacks immediate feedback and optimistic state, leading to double-taps while waiting on round-trips.

### 2. Scope
- Files:
  - `workio/server/src/middleware/lineSignature.ts`
  - `workio/server/src/routes/webhook/line.ts`
  - `workio/server/src/services/punchService.ts`
  - `workio/server/src/services/leaveService.ts` (and `otService.ts` if applicable)
  - DB migration for constraints, columns, and indexes.
- Changes:
  - Add signature verification middleware.
  - Add DB unique constraint/index on `line_event_id` (where not null) and composite cooldown index.
  - Add idempotency + per-user cooldown + audit fields at persistence layer.
  - Add strict payload validation and safe parsing in webhook.
  - Return fast, idempotent, non-retry-inducing status codes.
  - (UX) Frontend: optimistic UI + debounced buttons + local cooldown (brief).

#### 3.1 Signature verification middleware
```ts
// workio/server/src/middleware/lineSignature.ts
import crypto from 'crypto';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

export function verifyLineSignature(req: any, res: any, next: any) {
  const signature = req.headers['x-line-signature'] as string | undefined;
  if (!signature) {
    return res.status(400).json({ error: 'Missing X-Line-Signature' });
  }

  // Use raw body bytes if available (recommended). If not, serialize deterministically.
  const raw = (req as any).rawBody || JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(raw, 'utf8')
    .digest('base64');

  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected))) {
    return res.status(401).json({ error: 'Invalid signature' });
  }
  next();
}
```

#### 3.3 Idempotent service layer (punch)
```ts
// workio/server/src/services/punchService.ts
import { db } from '../db';

const PUNCH_COOLDOWN_SECONDS = 30;

export async function createPunch(payload: {
  line_event_id?: string;
  user_id: number;
  tenant_id: number;
  type: 'in' | 'out';
  latitude?: number;
  longitude?: number;
  source: 'line' | 'manual';
}) {
  const { line_event_id, user_id, tenant_id, type, latitude, longitude, source } = payload;

  return db.transaction(async (trx) => {
    // Idempotency by line_event_id
    if (line_event_id) {
      const existing = await trx('punches')
        .where({ line_event_id })
        .first();
      if (existing) {
        return { id: existing.id, created: false, reason: 'duplicate_line_event_id' };
      }
    }

    // Cooldown guard
    const recent = await trx('punches')
      .where({ user_id, tenant_id })
      .whereRaw(`created_at >= now() - interval '${PUNCH_COOLDOWN_SECONDS} seconds'`)
      .orderBy('created_at', 'desc')
      .first();
    if (recent) {
      return { id: recent.id, created: false, reason: 'cooldown_active', last_punch_at: recent.created_at };
    }

    const [record] = await trx('punches')
      .insert({
        user_id,
        tenant_id,
        type,
        latitude,
        longitude,
        source,
        line_event_id,
        created_at: new Date(),
        updated_at: new Date(),
      })
      .returning('*');

    return { id: record.id, created: true };
  });
}
```
