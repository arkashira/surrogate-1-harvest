**Final synthesized implementation**  
(Strongest, most actionable parts from both proposals; contradictions resolved in favor of correctness + concrete actionability.)

---

## 1. Diagnosis (consensus)

- No process supervisor → crashes leave service down; no auto-recovery.
- No log rotation or structured logs → disk fill risk and poor ops ingestion.
- No health/readiness probes → can’t detect stuck event loop or DB outage.
- Environment drift and missing validation → runtime failures from missing/invalid secrets.
- Non-idempotent cron/ops and missing webhook dedupe → duplicates and silent failures.
- Deploy docs omit `SHELL=/bin/bash` and executable wrapper guidance → cron failures in prod.

---

## 2. Proposed changes (single prioritized scope)

- Add **config validation** at startup (fail-fast on missing/invalid env).
- Add **health & readiness endpoints** (`/health`, `/ready`) with DB + event-loop checks.
- Add **PM2 (or container-friendly) process manager config** with log rotation and memory restart.
- Add **structured JSON logger** with secret redaction.
- Add **idempotency middleware for LINE webhooks** using `X-Line-Retry-Id` + DB-backed TTL dedupe.
- Add **cron-safe ops runner script** (`runner.sh`) that is idempotent, lock-protected, and logs to structured output.
- Add **migrate-and-prune script** (idempotent SQL) safe to run daily via cron.
- Update `.env.example`, `README.md`, and deploy docs with `SHELL=/bin/bash`, executable guidance, and run/cron instructions.

### 3.2 Structured logger with redaction (`server/src/logger.ts`)

```ts
// server/src/logger.ts
import pino from 'pino';
import config from './config';

const redactPaths = ['password', 'token', 'secret', 'LINE_CHANNEL_ACCESS_TOKEN', 'LINE_CHANNEL_SECRET'];
const opts: pino.LoggerOptions = {
  level: config.LOG_LEVEL,
  redact: {
    paths: redactPaths,
    censor: '[REDACTED]',
  },
  base: { pid: process.pid, env: config.NODE_ENV },
};

// In production, output JSON; in dev, pretty print to stdout
const transport =
  config.NODE_ENV === 'production'
    ? undefined
    : pino.transport({ target: 'pino-pretty', options: { colorize: true } });

const logger = transport ? pino(opts, transport) : pino(opts);

export default logger;
```

### 3.3 Health & readiness endpoints (`server/src/health.ts`)

```ts
// server/src/health.ts
import { Router } from 'express';
import config from './config';
import logger from './logger';
import { Pool } from 'pg';

const router = Router();
const pool = new Pool({ connectionString: config.DATABASE_URL });

function eventLoopLag(): Promise<number> {
  return new Promise((resolve) => {
    const start = process.hrtime.bigint();
    setImmediate(() => {
      const end = process.hrtime.bigint();
      resolve(Number(end - start) / 1e6); // ms
    });
  });
}

router.get('/health', async (req, res) => {
  const checks = {
    uptime: process.uptime(),
    timestamp: new Date().toISOString(),
    pid: process.pid,
    node_env: config.NODE_ENV,
  };

  try {
    await pool.query('SELECT 1');
    (checks as any).db = 'ok';
  } catch (err) {
    (checks as any).db = 'error';
    (checks as any).dbError = (err as Error).message;
  }

  try {
    const lag = await eventLoopLag();
    (checks as any).eventLoopLagMs = lag;
  } catch (err) {
    (checks as any).eventLoopLagMs = 'error';
  }

  const status = (checks as any).db === 'ok' ? 200 : 503;
  logger.info({ checks }, 'health check');
  res.status(status).json(checks);
});

router.get('/ready', async (req, res) => {
  // Readiness: stricter than health — require DB + acceptable event loop
  try {
    await pool.query('SELECT 1');
    const lag = await eventLoopLag();
    if (lag > 2000) {
      logger.warn({ lag }, 'readiness failed: high event loop lag');
      return res.status(503).json({ ready: false, reason: 'high_event_loop_lag', lag });
    }
    logger.info({ lag }, 'readiness ok');
    return res.json({ ready: true });
  } catch (err) {
    logger.error({ err }, 'readiness failed: db error');
    return res.status(503).json({ ready: false, reason: 'db_error' });
  }
});

export default router;
```

Wire into main app:

```ts
import healthRouter from './health';
app.use('/health', healthRouter);
app.use('/ready', healthRouter); // same router handles both
```
