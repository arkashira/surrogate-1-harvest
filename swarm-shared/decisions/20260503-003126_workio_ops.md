# workio / ops

## Final Synthesized Implementation

Below is the single, consolidated solution.  
It merges the strongest, non-contradictory parts of both proposals and resolves conflicts in favor of **correctness + concrete actionability**.

Key synthesis choices:
- **Idempotency**: use DB-backed idempotency table (not in-memory) — production-safe and survives restarts/scale-out.
- **Uniqueness**: enforce one open punch per user with a partial unique index at DB level.
- **Transactions**: wrap punch write + idempotency check/insert in one transaction to prevent races.
- **Webhook de-duplication**: add explicit LINE webhook middleware that checks idempotency before reaching routes.
- **Auditability**: add minimal audit columns (`request_id`, `idempotency_key`, `source`) and structured logging.
- **Clock-out**: make clock-out idempotent and safe for concurrent/retried calls.

-- Idempotency table (DB-backed, production-safe)
CREATE TABLE IF NOT EXISTS idempotency_keys (
  idempotency_key TEXT PRIMARY KEY,
  request_id      TEXT NOT NULL,
  response_body   JSONB NOT NULL,
  response_status INT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expires_at      TIMESTAMPTZ NOT NULL
);

-- Optional: index for cleanup
CREATE INDEX IF NOT EXISTS idx_idempotency_expires_at
ON idempotency_keys (expires_at);

-- Punches table (add audit/idempotency columns if not present)
-- Assumes punches already exists; add only missing columns:
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS request_id      TEXT,
  ADD COLUMN IF NOT EXISTS idempotency_key TEXT,
  ADD COLUMN IF NOT EXISTS source          TEXT;

-- Enforce: at most one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user

WHERE clocked_out_at IS NULL;

### 3) LINE webhook de-duplication middleware

```ts
// workio/server/src/middleware/lineWebhook.ts
import { Request, Response, NextFunction } from 'express';
import { db } from '../db';
import { logger } from '../lib/logger';

const IDEMPOTENCY_TTL_MS = 24 * 60 * 60 * 1000; // 24h

export async function lineWebhookDedup(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // Expect clients to send Idempotency-Key for retries
  const idempotencyKey =
    req.header('Idempotency-Key') || req.header('idempotency-key');
  const requestId = req.header('x-request-id') || crypto.randomUUID();

  const log = logger.child({ requestId, idempotencyKey, source: 'line_webhook' });

  if (!idempotencyKey) {
    log.warn('Missing Idempotency-Key on LINE webhook');
    // Optionally reject or allow through; here we allow but track request_id
    (req as any).requestContext = { requestId, idempotencyKey: null };
    return next();
  }

  const client = await db.connect();
  try {
    const existing = await client.query(
      `SELECT response_body, response_status FROM idempotency_keys
       WHERE idempotency_key = $1 AND expires_at > NOW()`,
      [idempotencyKey]
    );

    if (existing.rowCount > 0) {
      log.info('LINE webhook duplicate detected, replaying cached response');
      return res
        .status(existing.rows[0].response_status)
        .json(existing.rows[0].response_body);
    }

    // Store placeholder to block concurrent retries immediately (optional)
    await client.query(
      `INSERT INTO idempotency_keys (idempotency_key, request_id, response_body, response_status, expires_at)
       VALUES ($1, $2, $3, $4, $5)
       ON CONFLICT (idempotency_key) DO UPDATE
       SET expires_at = EXCLUDED.expires_at`,
      [idempotencyKey, requestId, {}, 202, new Date(Date.now() + IDEMPOTENCY_TTL_MS)]
    );

    (req as any).requestContext = { requestId, idempotencyKey };
    next();
  } catch (err) {
    log.error('Idempotency check failed', { error: err instanceof Error ? err.message : err });
    // Fail open to avoid dropping legitimate events; continue processing
    (req as any).requestContext = { requestId, idempotencyKey };
    next();
  } finally {
    client.release();
  }
}
```
