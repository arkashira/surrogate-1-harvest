## v0.1 scope
v0.1 ships a single Rust CLI client (no server dependency) that: establishes a local SOCKS5 listener, tunnels traffic through a WireGuard-like TUN via boringtun, applies TLS-fingerprinted obfuscation on 443 to mimic normal HTTPS, enforces a fail-closed kill-switch that blocks non-stealth traffic, stores session telemetry locally in SQLite, and exports WAU-ZB summaries (no PII) for manual upload.

## Open questions
- Which DPI signatures are most common in target regions (SNI, JA3, packet timing, TUN interface names) and which obfuscation primitives neutralize them?
- Can we rely solely on outbound 443 with TLS mimicry, or do we need domain-fronting/CDN relays to avoid IP-based blocking?
- How will client-side telemetry (WAU-ZB) be collected and stored to preserve privacy and avoid creating user-identifying logs?
- What kill-switch enforcement granularity is required (per-app, system-wide, TUN-only) and how to implement it reliably across desktop OSes?
- How to distribute and auto-update binaries without centralized app stores while preserving authenticity and censorship resistance?
