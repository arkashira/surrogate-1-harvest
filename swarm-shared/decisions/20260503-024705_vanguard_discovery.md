# vanguard / discovery

## Final synthesized implementation (best of both proposals)

**Core principle:**  
- Build-time generation of a static file manifest on the Mac orchestration host.  
- Frontend consumes the manifest and uses **CDN-only** (`resolve/main/`) URLs for all dataset file access.  
- Zero runtime HF API calls from the browser.  
- Deterministic artifact enables CDN-only Lightning training loads and fast discovery.

---

### 1. Mac orchestration: produce `file-list.json` (one-time / per folder)

`scripts/list-hf-folder.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail
# Run on the Mac after rate-limit window clears
REPO="datasets/your-org/vanguard-mirror"
FOLDER="batches/mirror-merged/2026-05-03"
OUT="vanguard/src/lib/data/file-list.json"

python3 -c "
import json, os, sys
from huggingface_hub import list_repo_tree

try:
    files = list_repo_tree(
        repo_id='$REPO',
        path='$FOLDER',
        recursive=True
    )
except Exception as e:
    print('HF list_repo_tree failed:', e, file=sys.stderr)
    sys.exit(1)

entries = []
for f in files:
    if f.type == 'file':
        entries.append({
            'path': f.path,
            'size': getattr(f, 'size', None),
            'cdn_url': f'https://huggingface.co/datasets/$REPO/resolve/main/{f.path}'
        })

os.makedirs(os.path.dirname('$OUT'), exist_ok=True)
with open('$OUT', 'w') as fh:
    json.dump(entries, fh, indent=2)
print(f'Wrote {len(entries)} entries to $OUT')
"
```

Run once:
```bash
chmod +x scripts/list-hf-folder.sh
bash scripts/list-hf-folder.sh
```

### 2. Frontend: embed manifest as TypeScript module

`vanguard/src/lib/data/fileManifest.ts`
```ts
// Auto-generated by scripts/list-hf-folder.sh — do not edit manually
import fileList from './file-list.json';

export interface FileEntry {
  path: string;
  size?: number | null;
  cdn_url: string;
}

export const fileManifest: FileEntry[] = fileList as FileEntry[];

export function getCdnUrl(path: string): string | undefined {
  const entry = fileManifest.find((e) => e.path === path);
  return entry?.cdn_url;
}

export function getAllPaths(): string[] {
  return fileManifest.map((e) => e.path);
}

export function getFileMap(): Map<string, FileEntry> {
  return new Map(fileManifest.map((e) => [e.path, e]));
}
```

If TypeScript/Vite needs it, ensure `tsconfig.json` includes:
```jsonc
{
  "compilerOptions": {
    "resolveJsonModule": true,
    "esModuleInteroperability": true
  }
}
```

### 3. Replace runtime HF calls in discovery logic (browser)

`vanguard/src/lib/data/discovery.ts`
```ts
import { getCdnUrl, getAllPaths, getFileMap } from './fileManifest';

// Use embedded manifest + CDN URLs only — no runtime HF API from browser.
export async function listDiscoverableFiles() {
  return getAllPaths().map((path) => ({
    path,
    url: getCdnUrl(path)
  })).filter((x): x is { path: string; url: string } => !!x.url);
}

// CDN fetch (no Authorization header) for file content
export async function fetchFileText(path: string): Promise<string> {
  const url = getCdnUrl(path);
  if (!url) throw new Error(`File not in manifest: ${path}`);
  const res = await fetch(url);
  if (!res.ok) throw new Error(`Failed to fetch ${url}: ${res.status}`);
  return await res.text();
}

// Optional: lightweight metadata lookups without HF API
export function getFileSize(path: string): number | null | undefined {
  return getFileMap().get(path)?.size ?? undefined;
}
```

### 4. Build/dev setup (minimal, frontend-only)

`esbuild.config.js` (minimal bundler)
```js
import esbuild from 'esbuild';

const isWatch = process.argv.includes('--watch');

await esbuild.build({
  entryPoints: ['src/main.js'],
  bundle: true,
  outfile: 'public/dist/bundle.js',
  format: 'esm',
  target: 'es2020',
  define: { 'process.env.NODE_ENV': '"production"' },
  watch: isWatch ? {} : false,
  loader: { '.png': 'dataurl', '.json': 'json' }
});

console.log('Build complete -> public/dist/bundle.js');
```

`public/index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>Vanguard Discovery</title>
    <script type="module" src="/dist/bundle.js"></script>
  </head>
  <body>
    <div id="app"></div>
  </body>

```

`src/main.js` (example entry)
```js
import { listDiscoverableFiles } from '../lib/data/discovery.js';

(async () => {
  const files = await listDiscoverableFiles();
  const app = document.getElementById('app');
  app.innerHTML = `
    <h1>Discoverable files (${files.length})</h1>
    <ul>${files.map(f => `<li><a href="${f.url}" target="_blank" rel="noopener">${f.path}</a></li>`).join('')}</ul>
  `;
})();
```

### 5. Verification checklist

1. Run the listing script on the Mac and confirm `vanguard/src/lib/data/file-list.json` exists with `path` and `cdn_url` entries.
2. `npm run build` completes without TypeScript or bundler errors.
3. Dev server (`npm run dev`) starts and changes to frontend code are reflected quickly.
4. In browser devtools:
   - Network tab shows **no** calls to `huggingface.co/api/` or `/list_repo_tree` from the frontend.
   - File fetches go to `https://huggingface.co/datasets/.../resolve/main/...` and return 200.
5. `getAllPaths()` and `getCdnUrl(...)` return expected values for known files.
6. Discovery page loads and lists files; no HF API rate-limit or 429 errors in console.
7. (Optional) Lightning training job can reference the same manifest or CDN URLs for deterministic, CDN-only data loading.
