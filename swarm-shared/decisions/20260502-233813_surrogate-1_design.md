### Step 1: Pre-flight snapshot generation

Create a new script `bin/snapshot.sh` that generates a snapshot of the dataset files at a specified date and repository. This script will:

1. Use `list_repo_tree(path, recursive=False)` to get a list of files in the repository at the specified date.
2. Download each file individually via `hf_hub_download` to a local directory.
3. Project each file to `{prompt, response}` only, without any additional metadata.
4. Save the resulting files to a local directory with a unique name based on the date and repository.

```bash
#!/usr/bin/env bash

# Set the date and repository to snapshot
DATE="2026-04-29"
REPO="axentx/surrogate-1-training-pairs"

# Set the local directory to save the snapshot
SNAPSHOT_DIR="batches/snapshot-$DATE"

# Create the local directory
mkdir -p "$SNAPSHOT_DIR"

# Get the list of files in the repository at the specified date
files=$(huggingface_hub list_repo_tree "$REPO" "$DATE" recursive=false)

# Download each file individually and project to {prompt, response}
for file in $files; do
  hf_hub_download "$REPO" "$file" "$SNAPSHOT_DIR/$file"
  # Project to {prompt, response} only
  # ...
done
```

### Step 3: Update the GitHub Actions workflow to use the pre-flight snapshot

Update the GitHub Actions workflow to use the pre-flight snapshot generated by `bin/snapshot.sh` before running `bin/dataset-enrich.sh`.

```yml
name: Ingest Public Dataset

on:
  workflow_dispatch:
    inputs:
      DATE:
        description: 'Date to ingest (YYYY-MM-DD)'
        required: true
      REPO:
        description: 'Repository to ingest'
        required: true

jobs:
  ingest:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Run pre-flight snapshot generation
        run: |
          bin/snapshot.sh ${{ inputs.DATE }} ${{ inputs.REPO }}

      - name: Run dataset enrichment
        run: |
          bin/dataset-enrich.sh ${{ inputs.DATE }} ${{ inputs.REPO }}
```

This implementation plan should take ≤2h to complete and will replace runtime `load_dataset(streaming=True)` + recursive `list_repo_tree` in `bin/dataset-enrich.sh` with a deterministic pre-flight snapshot + CDN.
