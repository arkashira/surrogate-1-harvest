# surrogate-1 / design

## Implementation Plan (≤2h)

**Highest-value improvement**: Add deterministic pre-flight file listing + CDN-only ingestion to eliminate HF API 429s during training and make shard workers resilient.

### What we’ll change
1. Add `bin/list-files.sh` — single Mac/Lightning call that lists one date folder via `list_repo_tree(recursive=False)` and writes `file-list.json` (path + size + sha256).  
2. Update `bin/dataset-enrich.sh` to:
   - Accept optional `FILE_LIST` env var (path to `file-list.json`).  
   - If present, shard by `slug-hash % 16 == SHARD_ID` and download via CDN (`resolve/main/...`) with `curl --retry 3 --retry-delay 5`.  
   - Fallback to HF API only if CDN 404 (rare).  
3. Add `lib/cdn_download.py` — small helper that takes a row from `file-list.json`, downloads via CDN, returns raw bytes; zero `datasets`/`hf_api` calls during streaming.  
4. Update training launcher (`lightning_train.py` or equivalent) to embed `file-list.json` and use CDN-only `IterableDataset` that never calls HF API.  
5. Add `bin/preflight.sh` — orchestrates `list-files.sh`, validates counts, uploads `file-list.json` to dataset repo under `meta/file-list/<date>.json` (1 commit/date, well under 128/hr).

### Why this is highest value
- Eliminates the 429/1000 req/5min wall during ingestion and training.  
- Keeps surrogate-1-runner’s 16-shard matrix unchanged (only internal download path changes).  
- Fits <2h: ~60 lines of new code + small edits.

### 1) bin/list-files.sh
```bash
#!/usr/bin/env bash
# list-files.sh
# Usage: HF_TOKEN=... ./list-files.sh <date> [repo]
# Outputs: file-list.json
set -euo pipefail

DATE="${1:-$(date +%Y-%m-%d)}"
REPO="${2:-datasets/axentx/surrogate-1-training-pairs}"
OUT="file-list.json"

python3 - "$REPO" "$DATE" "$OUT" <<'PY'
import os, json, sys
from huggingface_hub import HfApi

repo, date, out = sys.argv[1], sys.argv[2], sys.argv[3]
api = HfApi(token=os.environ.get("HF_TOKEN"))
# list one folder only (non-recursive)
entries = api.list_repo_tree(repo=repo, path=date, recursive=False)
files = []
for e in entries:
    if getattr(e, "type", None) == "file":
        files.append({
            "path": f"{date}/{e.path.split('/')[-1]}",
            "size": getattr(e, "size", 0),
            "sha256": getattr(e, "sha256", ""),
        })
with open(out, "w") as f:
    json.dump({"date": date, "repo": repo, "files": files}, f, indent=2)
print(f"Wrote {len(files)} files to {out}")
PY
```

### 2) lib/cdn_download.py
```python
#!/usr/bin/env python3
# lib/cdn_download.py
import io
import time
import requests
from typing import Dict, Any

CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

def download_via_cdn(entry: Dict[str, Any], repo: str, retries: int = 3, backoff: float = 5.0) -> bytes:
    path = entry["path"]
    url = CDN_TEMPLATE.format(repo=repo, path=path)
    for attempt in range(1, retries + 1):
        try:
            resp = requests.get(url, timeout=30)
            resp.raise_for_status()
            return resp.content
        except Exception as exc:
            if attempt == retries:
                raise
            time.sleep(backoff * attempt)
    raise RuntimeError("unreachable")
```

### 5) bin/preflight.sh
```bash
#!/usr/bin/env bash
# preflight.sh
set -euo pipefail
DATE="${1:-$(date +%Y-%m-%d)}"
OUT="meta/file-list/${DATE}.json"

mkdir -p meta/file-list
./bin/list-files.sh "$DATE" "datasets/axentx/surrogate-1-training-pairs"
mv file-list.json "$OUT"

# optional: push meta file (1 commit/date)
if [[ -n "${HF_TOKEN:-}" ]]; then
  git config user.name "surrogate-1-bot"
  git config user.email "bot@axentx.dev"
  git add "$OUT"
  git commit -m "meta: file-list $DATE" || true
  git push
fi
echo "Preflight file-list ready: $OUT"
```
