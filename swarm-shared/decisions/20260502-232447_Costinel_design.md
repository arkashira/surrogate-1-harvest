# Costinel / design

## Implementation Plan — Costinel Top-Hub Signal Endpoint

**Highest-value incremental improvement (<2h):**  
Expose a **read-only, side-effect-free** `GET /api/v1/cost-anomaly/signal/top-hub` that surfaces the most-connected hub (e.g., `"MOC"`) and related actionable signals. No mutations. No external writes. Fast, cacheable, observability-ready.

---

### 1) Architecture & API contract

- **Method**: `GET`
- **Path**: `/api/v1/cost-anomaly/signal/top-hub`
- **Auth**: Existing platform auth (no change)
- **Response shape**:
  ```json
  {
    "hub": "MOC",
    "score": 0.94,
    "rank": 1,
    "window": {
      "from": "2026-05-01T00:00:00Z",
      "to": "2026-05-02T23:59:59Z"
    },
    "signals": [
      {
        "id": "sig-001",
        "type": "cost_anomaly",
        "severity": "high",
        "service": "EC2",
        "region": "us-east-1",
        "account": "prod-123456",
        "delta_pct": 42.7,
        "current": 1842.33,
        "baseline": 1290.11,
        "description": "Unusual spend spike on i3en.xlarge fleet",
        "recommendation": "Review running instances and idle unattached volumes",
        "tags": ["#cost-anomaly", "#ec2", "#moc"]
      }
    ],
    "metadata": {
      "generated_at": "2026-05-02T23:30:00Z",
      "ttl_seconds": 300,
      "source": "costinel-signal-engine"
    }
  }
  ```

- **Errors**:
  - `401` — auth failure
  - `403` — insufficient scope
  - `404` — no hub/signals found
  - `429` — rate-limited (platform-wide)
  - `5xx` — internal error (emit standard platform error envelope)

---

### 2) Implementation steps (ordered)

1. **Add route** in API router (FastAPI/Flask — adapt to existing stack).  
2. **Create handler** that:
   - Validates query params (`window`, `top_n`, `hub_filter` optional).
   - Calls signal service to compute top hub + signals.
   - Returns shaped response.
3. **Implement signal service**:
   - Build lightweight in-memory graph of hub connections from recent cost events (last 24–48h).
   - Compute degree/centrality to pick top hub.
   - Fetch top-N actionable signals tied to that hub.
   - Keep logic pure and deterministic (no side effects).
4. **Add caching** (5-min TTL) keyed by window + params to reduce compute.
5. **Add observability**:
   - Timer metric: `costinel.api.top_hub_signal.latency_ms`
   - Counter: `costinel.api.top_hub_signal.requests`
   - Log structured request/response summary (no PII).
6. **Add tests**:
   - Unit: centrality selection, signal projection.
   - API: 200/401/404 shapes.
7. **Wire into CI** and verify locally with `docker-compose`.

#### Route (FastAPI example)

```python
# api/v1/endpoints/cost_anomaly.py
from fastapi import APIRouter, Depends, HTTPException
from service.signal_service import SignalService
from schema.top_hub import TopHubSignalResponse, TopHubSignalParams
from utils.auth import require_auth
from utils.metrics import observe_latency, count_request

router = APIRouter()
signal_service = SignalService()

@router.get(
    "/cost-anomaly/signal/top-hub",
    response_model=TopHubSignalResponse,
    dependencies=[Depends(require_auth)],
)
@observe_latency("top_hub_signal")
@count_request("top_hub_signal")
async def get_top_hub_signal(params: TopHubSignalParams = Depends()):
    try:
        result = await signal_service.top_hub_signal(
            window_from=params.window_from,
            window_to=params.window_to,
            top_n=params.top_n,
        )
        if not result["signals"]:
            raise HTTPException(status_code=404, detail="No signals found for top hub")
        return result
    except ValueError as e:
        raise HTTPException(status_code=400, detail=str(e))
    except Exception as e:
        # structured logging should capture this
        raise HTTPException(status_code=500, detail="Internal server error")
```
