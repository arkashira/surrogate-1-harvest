# workio / quality

### Diagnosis
* No client-side idempotency key on punch submissions → double-taps / LINE webhook replays create duplicate punches.
* Clock-in/out button has no optimistic state or debounce → users can spam and queue multiple requests.
* No local validation of valid state transitions → invalid sequences allowed (e.g., clock-in twice, clock-out without clock-in).
* No server-side state machine → invalid sequences allowed (clock-in twice, clock-out without clock-in).
* Punch writes are not transactional → potential data inconsistencies on failures.

### Proposed change
The proposed change will focus on implementing a client-side idempotency key for punch submissions to prevent duplicate punches. This will involve modifying the frontend code to generate a unique idempotency key for each punch submission and sending it with the request to the server. The server will then check for this key to prevent duplicate punches.

### Implementation
To implement this change, we will modify the `workio/frontend/src/components/PunchButton.js` file to generate a unique idempotency key for each punch submission. We will use the `uuid` library to generate the key.

```javascript
// workio/frontend/src/components/PunchButton.js
import { v4 as uuidv4 } from 'uuid';

const PunchButton = () => {
  const [idempotencyKey, setIdempotencyKey] = useState('');

  const handlePunch = async () => {
    const key = uuidv4();
    setIdempotencyKey(key);
    // Send the idempotency key with the punch submission request
    const response = await fetch('/api/punch', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        idempotencyKey: key,
        // Other punch submission data
      }),
    });
    // Handle the response
  };

  return (
    <button onClick={handlePunch}>
      Punch
    </button>
  );
};
```

On the server-side, we will modify the `workio/server/src/controllers/PunchController.js` file to check for the idempotency key and prevent duplicate punches.

```javascript
// workio/server/src/controllers/PunchController.js
const express = require('express');
const router = express.Router();

router.post('/api/punch', async (req, res) => {
  const { idempotencyKey } = req.body;
  // Check if a punch with the same idempotency key already exists
  const existingPunch = await Punch.findOne({ idempotencyKey });
  if (existingPunch) {
    // Return an error response if a punch with the same idempotency key already exists
    return res.status(400).json({ error: 'Duplicate punch submission' });
  }
  // Create a new punch and save it to the database
  const punch = new Punch(req.body);
  await punch.save();
  // Return a success response
  res.json({ message: 'Punch submission successful' });
});
```
