## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service that connects to one AWS account and one GCP project, lists schedules, computes next_run_at, flags orphaned (missing function) and duplicate (identical cron+hash) entries, and exposes a JSON report endpoint; deployed to Cloud Workers with Supabase Postgres for persistence.

## Open questions
- Which accounts/regions/orgs will be in scope for initial customers?
- How will customers grant read-only credentials (AWS IAM roles vs keys, GCP SA keys)?
- What retention window and scan cadence (daily vs hourly) is acceptable for drift detection?
- How should duplicates be grouped (by cron expression, tags, or code hash) and deduped?
