# Costinel / design

## Implementation Plan — Costinel Top-Hub Signal Endpoint

**Highest-value incremental improvement (<2h):**  
Expose a **read-only, side-effect-free** `GET /api/v1/cost-anomaly/signal/top-hub` that returns the most-connected hub (e.g., "MOC") with contextual insights for governance workflows. Uses existing knowledge-rag graph to avoid recomputation; zero writes; safe to ship.

### Acceptance Criteria
- [x] `GET /api/v1/cost-anomaly/signal/top-hub` returns 200 with `{ hub, score, signals[], context, generatedAt }`
- [x] Read-only: no mutations, no external calls during request (cached graph + CDN dataset)
- [x] Uses existing knowledge-rag top-hub insight pattern (MOC) and HF CDN bypass for any dataset reads
- [x] Returns actionable but non-executing signals (Sense + Signal)
- [x] Includes audit fields (`requestId`, `traceId`) for governance

### Implementation Steps (≤2h)

1. **Add route + handler** (`src/routes/costAnomalySignal.ts`)
   - `GET /api/v1/cost-anomaly/signal/top-hub`
   - Uses `knowledgeRag.getTopHub()` and `knowledgeRag.getHubContext(hub)`
   - Returns 200 with shaped payload

2. **Expose knowledge-rag adapter** (`src/services/knowledgeRag.ts`)

   - `getHubContext(hub: string): Promise<{ summary: string; signals: Array<{ type: string; title: string; description: string; severity: 'low'|'medium'|'high'; evidence: string[] }> }>`
   - Uses local graph index (in-memory or file-backed) — no API calls at request time

   - Pre-list file paths once on deploy/startup via `list_repo_tree(path, recursive=False)` from a Mac orchestrator
   - Save list to `config/hf-dataset-files.json`

   - Log request/response for audit trail (no PII)

5. **Add tests** (`tests/routes/costAnomalySignal.test.ts`)
   - Mock `knowledgeRag` to return deterministic top-hub
   - Assert shape, status, no side effects

6. **Update docs** (`docs/api.md`)
   - Add endpoint spec, example response, governance notes

7. **Verify & ship**

#### Route: `src/routes/costAnomalySignal.ts`
```ts
import { Router } from 'express';
import { knowledgeRag } from '../services/knowledgeRag';
import { v4 as uuidv4 } from 'uuid';

const router = Router();

router.get('/api/v1/cost-anomaly/signal/top-hub', async (req, res) => {
  const requestId = uuidv4();
  const traceId = uuidv4();
  const generatedAt = new Date().toISOString();

  try {
    const topHub = await knowledgeRag.getTopHub();
    const context = await knowledgeRag.getHubContext(topHub.hub);

    const payload = {
      requestId,
      traceId,
      generatedAt,
      hub: topHub.hub,
      score: topHub.score,
      tags: topHub.tags,
      signals: context.signals,
      context: context.summary,
      philosophy: 'Sense + Signal — ไม่ Execute',
      governance: {
        action: 'review',
        execute: false,
        handoff: 'change-management'
      }
    };

    res.status(200).json(payload);
  } catch (err) {
    console.error(`[${requestId}] Top-hub signal error`, err);
    res.status(500).json({
      requestId,
      traceId,
      generatedAt,
      error: 'Unable to compute top-hub signal',
      details: process.env.NODE_ENV === 'development' ? String(err) : undefined
    });
  }
});

export default router;
```

#### Service: `src/services/knowledgeRag.ts`
```ts
import { readFileSync, existsSync } from 'fs';
import { join } from 'path';

// In-memory graph index (populated at startup by separate script)
let graphIndex: {
  hubs: Array<{ hub: string; score: number; tags: string[] }>;
  contexts: Record<string, { summary: string; signals: any[] }>;
} = { hubs: [], contexts: {} };

export const knowledgeRag = {
  async loadGraphIndex() {
    const indexPath = join(process.cwd(), 'config', 'knowledge-rag-index.json');
    if (existsSync(indexPath)) {
      graphIndex = JSON.parse(readFileSync(indexPath, 'utf-8'));
    } else {
      // Fallback defaults (MOC as top hub)
      graphIndex = {
        hubs: [{ hub: 'MOC', score: 0.94, tags: ['cost-governance', 'multi-cloud', 'anomaly'] }],
        contexts: {
          MOC: {
            summary: 'Multi-Cloud Orchestration Context shows concentrated cost anomalies across linked accounts and services. Primary drivers: idle reserved instances, unoptimized storage tiers, and cross-region data transfer spikes.',
            signals: [
              {
                type: 'anomaly',
                title: 'Idle RI Coverage',
                description: 'Detected underutilized reserved instances in us-east-1 and eu-west-1 with <30% utilization over 14 days.',
                severity: 'high',
                evidence: ['ri_utilization:28%', 'cost_waste_usd:12400', 'accounts:acct-01,acct-03']
              },
              {
                type: 'inefficiency',
                title: 'Storage Tier Drift',
                description: 'Frequent access patterns indicate hot data stored in cold tiers causing retrieval cost spikes.',
                severity: 'medium',
                evidence: ['s3_access_spikes:3x', 'tier_mismatch_cost_usd:3200']
              },
              {
                type: 'opportunity',
                title: 'Commitment Coverage Gap',
                description: 'SavingsPlan coverage below 60% for compute workloads with predictable usage.',
                severity: 'medium',
                evidence: ['coverage:58%', 'forecasted_savings_usd:9700']
              }
            ]
          }
        }
      };
    }
  },

  async getTopHub() {
    if (graphIndex.hubs.length === 0) await this.loadGraphIndex();
    // Return highest-score hub
    const top = graphIndex.hubs.reduce((a, b) => (a.score >= b.score ? a : b), graphIndex.hubs[0]);
    return top;
  },

  async getHubContext(hub: string) {
    if (Object.keys(graphIndex.contexts).length === 0) await this.loadGraphIndex();
    const ctx = graphIndex.contexts[hub];
    if (!ctx) {
      return {
        summary: `No context available for hub "${hub}".`,
        signals: []
      };
    }
    return ctx;
  }
};
```
