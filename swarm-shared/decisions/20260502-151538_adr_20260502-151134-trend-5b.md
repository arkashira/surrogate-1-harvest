## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-strategy pipeline (research→signal→risk→execution) with one broker adapter and one data vendor connector; deterministic state machine with Redis Streams event log; Postgres audit trail; end-to-end signal-to-execution latency measurement and dashboard; replay endpoint to reprocess a run from immutable snapshot; free-tier deployable on CF Workers + Supabase + Redis.

## Open questions
- Which brokers/data vendors must be supported in v0.1 to prove NSM?
- Deterministic clock source for sub-ms latency measurement (NTP/PTP strategy)?
- Replay storage format and retention: append-only in Postgres or export to Parquet on HF?
- Edge vs origin placement: run orchestrator on CF Workers or HF Spaces for async background jobs?
- Risk engine contract: pass/fail SLA and override policy semantics
