## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: Android app (no root) with concealment-only mode that spawns ephemeral SOCKS5 listeners on randomized local ports, applies noise-based obfuscation to traffic, and uses a CF Workers edge endpoint (free tier) to distribute per-session port/rule configs; Supabase Postgres stores anonymized session metadata and computes WAUS/U; end-to-end concealment test harness validates that local port scans and basic DPI cannot classify sessions as VPN/proxy under lab-like inspection.

## Open questions
- Which local port/socket cloaking technique reliably hides SOCKS5 listeners from app-level scanners without breaking captive portals or localhost services?
- Can we safely mask TUN interface names/addresses from system inspection (e.g., via netlink/ifaddr filtering) on stock Android without root or custom kernels?
- What traffic-shaping and timing obfuscation parameters minimize DPI classification while preserving throughput/latency targets on high-loss paths?
- How will WAUS/U be measured end-to-end (client-side attestation vs server-side signals) without creating a fingerprinting side channel?
- Which fallback path (domain-fronting-like via CF Workers, or decoy channels) is viable under free-tier constraints if primary concealment is detected?
- Battery and data-usage trade-offs of continuous packet mangling vs periodic burst modes on mobile devices
