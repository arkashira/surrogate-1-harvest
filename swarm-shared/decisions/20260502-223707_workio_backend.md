# workio / backend

### Final synthesized implementation (correct + actionable)

**Core diagnosis (merged)**
- Repeated `channelId → tenantId` DB lookups on every LINE webhook event cause DB thrash and increase 429/timeout risk under traffic spikes.
- No request-scoped tenant resolution: batched events and repeated calls for the same channel re-query the same tenant row.
- Missing observability and testability: tenant resolution is coupled to DB calls and global state.

**Chosen approach**
- Add an in-memory LRU cache (`channelId:channelSecret → tenant`) with a short TTL (5 minutes) to bound staleness and memory.
- Resolve tenant **once per webhook request** (batched events share the resolved tenant) and attach to `req.tenant` / `req.tenantId`.
- Verify LINE signature using the tenant’s stored `line_channel_secret` (not the raw signature header) to avoid security issues.
- Keep cache in middleware layer (not controller) to maximize reuse across routes/tests.

### 2. Create tenant cache module
`server/src/middleware/tenantCache.ts`
```ts
import { LRUCache } from 'lru-cache';
import type { Tenant } from '../types';

type TenantCacheValue = {
  tenantId: string;
  tenant: Tenant;
};

const tenantByChannelCache = new LRUCache<string, TenantCacheValue>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
  ttlAutopurge: true,
  allowStale: false,
});

export function resolveTenantFromChannel(
  channelId: string,
  channelSecret: string
): TenantCacheValue | null {
  const key = `${channelId}:${channelSecret}`;
  return tenantByChannelCache.get(key) ?? null;
}

export function setTenantForChannel(
  channelId: string,
  channelSecret: string,
  tenantId: string,
  tenant: Tenant
) {
  const key = `${channelId}:${channelSecret}`;
  tenantByChannelCache.set(key, { tenantId, tenant });
}

export function clearTenantForChannel(channelId: string, channelSecret: string) {
  const key = `${channelId}:${channelSecret}`;
  tenantByChannelCache.delete(key);
}
```

### 3. Middleware: resolve tenant + verify signature once per request
`server/src/middleware/lineWebhook.ts`
```ts
import { Request, Response, NextFunction } from 'express';
import { verifySignature } from '../utils/lineSignature';
import { resolveTenantFromChannel, setTenantForChannel } from './tenantCache';
import db from '../db';

export async function lineWebhookMiddleware(
  req: Request,
  _res: Response,
  next: NextFunction
) {
  try {
    const channelId = req.headers['x-line-channel-id'] as string;
    const signature = req.headers['x-line-signature'] as string;
    const body = req.body;

    if (!channelId || !signature) {
      return next(new Error('Missing LINE headers'));
    }

    // 1) Try cache first (fast-path)
    const cached = resolveTenantFromChannel(channelId, signature);
    if (cached) {
      (req as any).tenant = cached.tenant;
      (req as any).tenantId = cached.tenantId;
      console.log(`[tenant] cache hit ${channelId}`);
      return next();
    }

    // 2) Single DB lookup per webhook request (batched events share tenant)
    const result = await db.query<{
      id: string;
      name: string;
      line_channel_id: string;
      line_channel_secret: string;
    }>('SELECT id, name, line_channel_id, line_channel_secret FROM tenants WHERE line_channel_id = $1 LIMIT 1', [channelId]);

    const row = result.rows[0];
    if (!row) return next(new Error('Tenant not found for channel'));

    // 3) Verify signature using stored tenant secret (correct security practice)
    const isValid = verifySignature(JSON.stringify(body), signature, row.line_channel_secret);
    if (!isValid) return next(new Error('Invalid signature'));

    const tenant = {
      id: row.id,
      name: row.name,
      lineChannelId: row.line_channel_id,
      lineChannelSecret: row.line_channel_secret,
    };

    // 4) Cache by channelId+secret (so rotation invalidates cache naturally)
    setTenantForChannel(channelId, row.line_channel_secret, row.id, tenant);
    (req as any).tenant = tenant;
    (req as any).tenantId = row.id;
    console.log(`[tenant] resolved ${channelId} -> ${row.id} (cached=false)`);
    next();
  } catch (err) {
    next(err);
  }
}
```

import { Request, Response } from 'express';

export async function handleLineWebhook(req: Request, res: Response) {
  const tenant = (req as any).tenant;
  const events = req.body.events || [];

  if (!tenant) {
    return res.status(400).json({ error: 'Tenant not resolved' });
  }

  // Process all events for this request using the same tenant (no per-event DB lookups)
  for (const event of events) {
    // handle event using tenant.id / tenant.lineChannelId etc.
  }

  res.json({ ok: true });
}

### 6. Verification checklist (fast, observable)
1. Start backend: `npm run dev`
2. Send a test webhook with headers:
   - `x-line-channel-id`: known channel
   - `x-line-signature`: valid signature for the payload using that tenant’s secret
3. Confirm logs:
   - First request: `[tenant] resolved <channel> -> <id> (cached=false)` and one DB query.
   - Second request (same channel): `[tenant] cache hit <channel>` and zero DB queries.
   - After ~5 minutes: cache miss and DB query again.
4. Burst test: send one request with many events (batched) and verify only one DB lookup and one signature verification.

### 7. Why this wins (resolved contradictions)
- **Correctness over convenience**: signature verification uses the tenant’s stored secret (Candidate 2), not the raw signature as a lookup key (Candidate 1). This prevents accepting invalid signatures if secrets rotate.
- **Request-scoped batching**: resolve tenant once per webhook request so batched events don’t multiply DB queries (Candidate 2).
- **Observability + testability**: middleware exposes `req.tenant`, logs cache hits/misses, and separates cache logic (easy to mock in tests) from controllers.
- **Balanced TTL**: 5 minutes (Candidate 2) is safer for secret rotation than 1 hour (Candidate 1) while still protecting DB under spikes. LRU size 500 caps memory.
