# workio / frontend

## 1. Diagnosis
- Punch button has no optimistic UI → perceived latency and double-tap/duplicate requests.
- No client-side in-flight lock or idempotency key → race punches on retries or parallel clicks.
- Missing pending/confirmed state in the list → UI doesn’t reflect “clocking in/out” immediately.
- No rollback on network failure → user sees stale state and must refresh.
- Visual affordance weak (no loading/disabled state) → users spam the button.

## 2. Proposed change
File: `workio/src/components/PunchButton.tsx`  
Scope: add optimistic UI + idempotency + disable-on-submit + local rollback on failure + clear visual states. Keep component reusable (accepts `onPunch` callback or uses React Query mutation).

## 3. Implementation
```tsx
// workio/src/components/PunchButton.tsx
import React, { useState, useCallback, useRef } from 'react';
import { useMutation, useQueryClient } from '@tanstack/react-query';

type PunchType = 'IN' | 'OUT';

interface PunchButtonProps {
  type: PunchType;
  onPunch?: (type: PunchType) => Promise<void>;
  disabled?: boolean;
}

const genIdempotencyKey = () => `${Date.now()}-${Math.random().toString(36).slice(2)}`;

export const PunchButton: React.FC<PunchButtonProps> = ({ type, onPunch, disabled: propDisabled }) => {
  const queryClient = useQueryClient();
  const idempotencyRef = useRef<string | null>(null);
  const [optimisticState, setOptimisticState] = useState<'idle' | 'pending' | 'confirmed' | 'error'>('idle');

  const punchMutation = useMutation({
    mutationFn: async (t: PunchType) => {
      if (!onPunch) throw new Error('onPunch not provided');
      await onPunch(t);
    },
    onMutate: async () => {
      idempotencyRef.current = genIdempotencyKey();
      setOptimisticState('pending');

      // Optimistically update any cached punch list if present
      queryClient.setQueryData(['punches'], (old: any[]) => {
        if (!old) return old;
        return [
          {
            id: idempotencyRef.current,
            type,
            ts: new Date().toISOString(),
            status: 'pending',
            optimistic: true,
          },
          ...old,
        ];
      });

      return { idempotencyKey: idempotencyRef.current };
    },
    onError: (err, _variables, context) => {
      setOptimisticState('error');
      // Rollback optimistic entry
      queryClient.setQueryData(['punches'], (old: any[]) => {
        if (!old) return old;
        return old.filter((p) => p.id !== context?.idempotencyKey);
      });

      // Brief visual error then reset
      setTimeout(() => setOptimisticState('idle'), 2000);
    },
    onSuccess: (_data, _variables, context) => {
      setOptimisticState('confirmed');
      // Replace optimistic entry with confirmed one (or rely on refetch)
      queryClient.setQueryData(['punches'], (old: any[]) => {
        if (!old) return old;
        return old.map((p) =>
          p.id === context?.idempotencyKey ? { ...p, status: 'confirmed', optimistic: false } : p
        );
      });

      // Invalidate and refetch to ensure server truth
      queryClient.invalidateQueries({ queryKey: ['punches'] });
      setTimeout(() => setOptimisticState('idle'), 800);
    },
  });

  const handleClick = useCallback(() => {
    if (punchMutation.isPending || optimisticState === 'pending' || propDisabled) return;
    punchMutation.mutate(type);
  }, [punchMutation, type, propDisabled, optimisticState]);

  const isLoading = punchMutation.isPending || optimisticState === 'pending';
  const isError = optimisticState === 'error';

  return (
    <button
      onClick={handleClick}
      disabled={isLoading || propDisabled}
      className={`
        px-6 py-3 rounded-lg font-semibold transition-all
        ${
          isError
            ? 'bg-red-600 text-white animate-shake'
            : type === 'IN'
            ? isLoading
              ? 'bg-blue-400 cursor-wait'
              : 'bg-blue-600 hover:bg-blue-700 text-white'
            : isLoading
            ? 'bg-orange-400 cursor-wait'
            : 'bg-orange-600 hover:bg-orange-700 text-white'
        }
        disabled:opacity-60 disabled:cursor-not-allowed
        flex items-center justify-center gap-2
      `}
      aria-busy={isLoading}
    >
      {isError ? (
        <>
          <span>⚠</span> ลองใหม่
        </>
      ) : isLoading ? (
        <>
          <span className="w-4 h-4 border-2 border-white/60 border-t-white rounded-full animate-spin" />
          {type === 'IN' ? 'กำลังลงเวลา...' : 'กำลังออกเวลา...'}
        </>
      ) : (
        type === 'IN' ? 'ลงเวลาเข้างาน' : 'ลงเวลาออกงาน'
      )}
    </button>
  );
};
```

const [inflight, setInflight] = useState(false);
const handleClick = async () => {
  if (inflight || disabled) return;
  setInflight(true);

    await onPunch?.(type);

  } finally {

  }

## 4. Verification
- Click “ลงเวลาเข้างงาน” → button shows spinner and is disabled; list shows pending entry immediately.
- Network offline (or mock 500) → button shows red error state with “⚠ ลองใหม่” and pending entry is removed after 2s.
- Success path → pending entry updates to confirmed, then list refetches; button returns to idle.
- Rapid double-click → second click ignored while `isLoading` is true (no duplicate requests).
- Console: ensure only one mutation call per click and idempotency key is generated per attempt.
