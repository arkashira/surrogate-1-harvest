# airship / backend

## Highest-Value Incremental Improvement (≤2h)
**Ship:** Deterministic CDN file manifest + Lightning Studio auto-restart resilience for Surrogate training.  
**Why:** Eliminates HF API 429s during data loading and prevents idle-stop training loss with minimal code change and no new infra.

---

## Implementation Plan

1. **Generate CDN file manifest once on Mac (or any orchestrator)**
   - Use `list_repo_tree(path, recursive=False)` per date folder (avoid recursive on big repos).
   - Save `manifest-{date}.json` listing all parquet files to be used for this training run.
   - Commit/ship this manifest with the training job (or pass via env var).

2. **Update Surrogate training data loader**
   - Replace `load_dataset(streaming=True)` with direct CDN fetches using URLs:
     `https://huggingface.co/datasets/{repo}/resolve/main/{path}`
   - Read only the files listed in the manifest.
   - Project to `{prompt, response}` at parse time; ignore mixed schemas.

3. **Add Lightning Studio lifecycle resilience**
   - Before `.run()`, check studio status.
   - If stopped, restart with `target.start(machine=Machine.L40S)` (or configured machine).
   - Guard every training invocation to avoid silent death on idle timeout.

4. **Small operational niceties**
   - Set `SHELL=/bin/bash` in any crontab that invokes wrappers.
   - Ensure wrapper scripts have `#!/usr/bin/env bash` and `chmod +x`.

### 1) Generate manifest (run once on orchestrator)
```bash
#!/usr/bin/env bash
# generate-manifest.sh
set -euo pipefail

REPO="org/surrogate-dataset"
DATE_FOLDER="batches/mirror-merged/2026-05-03"
OUT="manifest-2026-05-03.json"

python3 - <<PY
import json, os
from huggingface_hub import HfApi

api = HfApi()
entries = api.list_repo_tree(
    repo_id=os.getenv("REPO", "$REPO"),
    path=os.getenv("DATE_FOLDER", "$DATE_FOLDER"),
    recursive=False,
)
files = [e.path for e in entries if e.path.endswith(".parquet")]
manifest = {
    "repo": os.getenv("REPO", "$REPO"),
    "date": "$DATE_FOLDER",
    "files": sorted(files),
}
with open(os.getenv("OUT", "$OUT"), "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Wrote {len(files)} files to {os.getenv('OUT', '$OUT')}")
PY
```

### 2) Surrogate training loader (CDN-only, no HF API during training)
```python
# surrogate/train.py  (excerpt)
import json
import pyarrow.parquet as pq
import requests
from io import BytesIO
from pathlib import Path
from torch.utils.data import IterableDataset

HF_CDN = "https://huggingface.co/datasets"

class CDNParquetIterable(IterableDataset):
    def __init__(self, manifest_path, repo, max_files=None):
        with open(manifest_path) as f:
            manifest = json.load(f)
        self.repo = manifest["repo"]
        self.files = manifest["files"]
        if max_files:
            self.files = self.files[:max_files]

    def _fetch_parquet(self, file_path):
        url = f"{HF_CDN}/{self.repo}/resolve/main/{file_path}"
        resp = requests.get(url, timeout=30)
        resp.raise_for_status()
        return pq.read_table(BytesIO(resp.content))

    def _project_record(self, table):
        # Keep only prompt/response; ignore mixed schema extras
        cols = set(table.column_names)
        prompt_col = next((c for c in ("prompt", "instruction", "input") if c in cols), None)
        response_col = next((c for c in ("response", "output", "completion") if c in cols), None)

        for i in range(table.num_rows):
            row = {}
            if prompt_col:
                row["prompt"] = table[prompt_col][i].as_py()
            if response_col:
                row["response"] = table[response_col][i].as_py()
            if row:
                yield row

    def __iter__(self):
        for file_path in self.files:
            try:
                table = self._fetch_parquet(file_path)
                yield from self._project_record(table)
            except Exception as exc:
                # Log and skip bad files to avoid crashing entire epoch
                print(f"Skipping {file_path}: {exc}")
                continue
```

### 3) Lightning Studio lifecycle guard
```python
# surrogate/lightning_launcher.py  (excerpt)
from lightning import Lightning, Teamspace, Machine, Studio

LIGHTNING_ACCOUNT = "lightning-lambda-prod"  # or your paid org account for H200
STUDIO_NAME = "surrogate-training-studio"
MACHINE = Machine.L40S  # fallback; use H200 if available on account

def ensure_running_studio():
    lightning = Lightning()
    teamspace = Teamspace.current()
    running = None
    for s in teamspace.studios:
        if s.name == STUDIO_NAME:
            if s.status == "running":
                return s
            running = s
            break

    if running is None:
        print(f"Creating studio {STUDIO_NAME}")
        running = Studio.create(
            name=STUDIO_NAME,
            machine=MACHINE,
            cloud_account=LIGHTNING_ACCOUNT,
            create_ok=True,
        )
    else:
        print(f"Restarting stopped studio {STUDIO_NAME}")
        running.start(machine=MACHINE)
    return running

def run_training(script_path, args=None):
    studio = ensure_running_studio()
    # Studio may stop between checks and .run(); guard with retry if needed.
    job = studio.run(
        script_path,
        arguments=args or [],
        environment="surrogate-train",
    )
    print(f"Launched training job: {job}")
    return job
```

- No regressions to existing training loop behavior (prompt/response projection preserved).
