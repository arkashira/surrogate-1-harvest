# workio / design

## Final synthesized implementation (correct + actionable)

**Core principle:** Make LINE webhook ingestion **atomic, idempotent, and auditable** with minimal code surface and strong database-level guarantees.

---

### 1. Schema changes (run once)

```sql
-- Idempotency table: one row per LINE event per tenant
CREATE TABLE IF NOT EXISTS line_event_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  line_event_id   VARCHAR(255) NOT NULL,
  tenant_id       BIGINT       NOT NULL,
  event_type      VARCHAR(100) NOT NULL,
  processed_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  payload_hash    VARCHAR(64),
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- Prevent duplicates even if app logic changes
CREATE UNIQUE INDEX IF NOT EXISTS ux_line_event_idempotency_unique
  ON line_event_idempotency (line_event_id, tenant_id);

-- Domain-level safety: one non-deleted domain record per LINE event
ALTER TABLE punches
  ADD CONSTRAINT uniq_punches_line_event
  UNIQUE (tenant_id, line_event_id)
  WHERE deleted_at IS NULL;

ALTER TABLE leave_requests
  ADD CONSTRAINT uniq_leave_line_event
  UNIQUE (tenant_id, line_event_id)
  WHERE deleted_at IS NULL;

ALTER TABLE ot_requests
  ADD CONSTRAINT uniq_ot_line_event
  UNIQUE (tenant_id, line_event_id)
  WHERE deleted_at IS NULL;
```

- Use `line_event_id` (from LINE `event.id`) as the stable idempotency key.  
- Composite unique index (`line_event_id`, `tenant_id`) prevents duplicates across tenants.  
- Domain constraints ensure safety even if idempotency table is out of sync.

### 2. Idempotency utility (single atomic check)

```ts
// workio/server/src/utils/lineIdempotency.ts
import { pool } from '../db';

export async function tryAcquireLineEvent(
  lineEventId: string,
  tenantId: number,
  eventType: string,
  payloadHash?: string
): Promise<boolean> {
  const result = await pool.query(
    `INSERT INTO line_event_idempotency
       (idempotency_key, line_event_id, tenant_id, event_type, payload_hash)
     VALUES ($1, $2, $3, $4, $5)
     ON CONFLICT (line_event_id, tenant_id) DO NOTHING
     RETURNING id`,
    [lineEventId, lineEventId, tenantId, eventType, payloadHash || '']
  );
  return result.rowCount === 1;
}
```

- Uses `ON CONFLICT DO NOTHING` + `RETURNING` for atomic acquire.  
- Returns `true` only on first insert (event not yet processed).  
- No TOCTOU; single statement.

### 3. Webhook handler (atomic ingestion)

```ts
// workio/server/src/routes/line/webhook.ts
import { Request, Response } from 'express';
import { pool } from '../../db';
import { verifyLineSignature } from '../../utils/line';
import { tryAcquireLineEvent } from '../../utils/lineIdempotency';

export async function handleLineWebhook(req: Request, res: Response) {
  const sig = req.headers['x-line-signature'] as string;
  const body = req.body;

  if (!verifyLineSignature(JSON.stringify(body), sig)) {
    return res.status(401).send('Invalid signature');
  }

  const events = body.events || [];
  const results = [];

  for (const ev of events) {
    try {
      // Stable idempotency key from LINE
      const lineEventId = ev.id;
      if (!lineEventId) {
        results.push({ status: 'error', error: 'Missing event.id' });
        continue;
      }

      const tenantId = await resolveTenantIdByChannelToken(sig);
      const eventType = ev.type;

      // Atomic idempotency acquire
      const isNew = await tryAcquireLineEvent(lineEventId, tenantId, eventType);
      if (!isNew) {
        results.push({ lineEventId, status: 'skipped_duplicate' });
        continue;
      }

      // Process domain action once (server-side timestamps)
      await processLineEvent(ev, tenantId, lineEventId);
      results.push({ lineEventId, status: 'processed' });
    } catch (err) {
      console.error('LINE webhook event processing failed', err, ev);
      // Continue processing other events
      results.push({ status: 'error', error: String(err) });
    }
  }

  return res.json({ results });
}

async function processLineEvent(event: any, tenantId: number, lineEventId: string) {
  // All timestamps generated server-side; ignore client timestamps for domain records
  const now = new Date();

  switch (event.type) {
    case 'message':
      // Example: clock punch from command text
      await pool.query(
        `INSERT INTO punches (tenant_id, user_id, punch_date, shift_code, line_event_id, created_at)
         VALUES ($1, $2, $3, $4, $5, $6)`,
        [tenantId, event.source.userId, now, 'default', lineEventId, now]
      );
      break;

    case 'postback':
      // Example: leave/OT request
      // Parse event.postback.data and insert into leave_requests or ot_requests
      // Ensure line_event_id is stored for idempotency
      break;

    default:
      // no-op
      break;
  }
}

async function resolveTenantIdByChannelToken(sig: string): Promise<number> {
  // Map channel access token -> tenant_id (cached)
  return Number(process.env.DEFAULT_TENANT_ID || 1);
}
```

Key points:
- **Idempotency key** = `event.id` (stable across retries).  
- **Atomic acquire** via `tryAcquireLineEvent`. If false, skip processing.  
- **Server-side timestamps** for domain records; ignore client-provided times.  
- **Domain inserts include `line_event_id`** so DB constraints block duplicates even if idempotency row is missing.  
- **Per-event error isolation**: one failure doesn’t block others.

---

### 4. Verification checklist

1. **Schema applied**: confirm `line_event_idempotency` + unique indexes exist.  
2. **Idempotency test**: send identical payload twice; expect one `processed`, one `skipped_duplicate`, and exactly one domain row.  
3. **Concurrency test**: fire 10 parallel identical requests; expect exactly one success, rest skipped, no duplicate domain rows.  
4. **Signature failure**: tampered payload returns 401.  
5. **Logs**: confirm `skipped_duplicate` entries on retries.  
6. **Rollback safety**: remove idempotency table and handler guard to revert (but keep domain constraints for safety).

---

### 5. Why this wins

- **Atomicity**: `ON CONFLICT DO NOTHING` + `RETURNING` removes TOCTOU.  
- **Correctness**: DB constraints enforce uniqueness at both idempotency and domain layers.  
- **Actionability**: minimal code, clear SQL, concrete tests, and rollback path.  
- **Safety**: server-side timestamps, per-event error isolation, and stable idempotency key from LINE.
