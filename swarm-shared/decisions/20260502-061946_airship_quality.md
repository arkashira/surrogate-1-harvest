## Quality Improvement: Fix PyArrow CastError on HF Dataset with Mixed Schema Files

### Problem Statement
PyArrow CastError occurs when loading HF dataset with mixed schema files.

### Solution
Don't use `load_dataset(streaming=True)` for repos with heterogeneous file schemas. Instead, download each file individually via `hf_hub_download` and project to `{prompt, response}` only at parse time.

### Implementation Plan

1. Update `train.py` to use `hf_hub_download` for each file individually.
2. Modify the dataset loading code to project to `{prompt, response}` only at parse time.

### Code Snippet
```python
import hf_hub_download

# Download each file individually
files = []
for file in dataset_files:
    file_path = hf_hub_download(repo_id, file)
    files.append(file_path)

# Load dataset with projected schema
dataset = Dataset.from_pandas(pd.concat([pd.read_parquet(file) for file in files], ignore_index=True), columns=["prompt", "response"])
```
