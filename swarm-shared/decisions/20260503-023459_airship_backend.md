# airship / backend

Below is the **single, merged implementation** that keeps the strongest, most actionable parts from both candidates, removes duplication, and resolves contradictions in favor of correctness and immediate execution.

---

## Goal (single statement)
Eliminate HF API 429s and Lightning quota waste during Surrogate training by implementing **CDN-first deterministic ingestion with embedded file lists and Lightning Studio reuse** — no schema changes, no new infra.

---

## Design Decisions (resolved)

| Contradiction | Resolution |
|--------------|------------|
| Direct CDN URL vs `hf_hub_download` | Use `hf_hub_download` (it hits CDN under the hood, handles caching, partial files, and avoids auth/429 for public files). |
| JSON schema for file list | Minimal canonical schema: `{"repo": "...", "date": "...", "files": [...]}`. Easy to validate and extend. |
| Where to store `file_list.json` | Generate next to training script or in `.cache/` and commit only when needed for reproducibility. |
| Studio reuse logic | Single source of truth: list running studios by name/status and reuse if running; otherwise create with explicit machine. |
| Parquet schema for writes | Keep minimal: `batches/mirror-merged/{date}/{slug}.parquet` with only required columns. No extra metadata columns. |
| HF commit-cap mitigation | Deterministic spread: hash slug → pick one of 5 sibling repos for writes (simple modulo). |

---

## Implementation Plan (<2h)

1. **Deterministic file-list step (Mac/Linux orchestrator)**  
   - One `list_repo_tree` call per date folder → `file_list.json`.  
   - Save to `.cache/file_list.json` (or commit alongside training script if reproducibility required).

2. **Update `train.py` for CDN-first ingestion**  
   - Read `file_list.json`.  
   - Fetch each file via `hf_hub_download` (CDN-backed, no auth for public files).  
   - Parse to `{prompt, response}` only at parse time; do not use `load_dataset(streaming=True)` or mixed schemas.

3. **Lightning Studio reuse + idle-stop guard**  
   - Before training, list running studios; reuse if name/status match.  
   - On `.run()`, check status; if stopped, restart with target machine.  
   - Add simple retry/backoff for transient failures.

4. **HF commit-cap mitigation for enriched writes**  
   - Hash slug → pick one of 5 sibling repos for writes (deterministic spread).  
   - Filename pattern: `batches/mirror-merged/{date}/{slug}.parquet`.

5. **Smoke test**  
   - Run ingestion on one small date folder.  
   - Verify CDN-only fetch, caching, and Lightning reuse.

### 1. Orchestrator: produce deterministic file list

```bash
#!/usr/bin/env bash
# pre_list_files.sh
set -euo pipefail

REPO="${REPO:-axentx/surrogate-mirror}"
DATE_FOLDER="${DATE_FOLDER:-2026-04-29}"
OUTFILE="${OUTFILE:-.cache/file_list.json}"

mkdir -p "$(dirname "$OUTFILE")"

python3 - <<PY
import json, os
from huggingface_hub import HfApi

api = HfApi()
tree = api.list_repo_tree(
    repo_id=os.getenv("REPO"),
    path=os.getenv("DATE_FOLDER"),
    recursive=False,
)
files = [item.rfilename for item in tree if item.type == "file"]
out = {
    "repo": os.getenv("REPO"),
    "date": os.getenv("DATE_FOLDER"),
    "files": sorted(files),
}
with open(os.getenv("OUTFILE"), "w") as f:
    json.dump(out, f, indent=2)
print(f"Wrote {len(files)} files to {os.getenv('OUTFILE')}")
PY
```

### 2. `train.py` — CDN-first data loader (no streaming API)

```python
import json
import os
from pathlib import Path
from typing import List

import pandas as pd
import pyarrow.parquet as pq
from huggingface_hub import hf_hub_download

REPO = os.getenv("REPO", "axentx/surrogate-mirror")
FILE_LIST = os.getenv("FILE_LIST", ".cache/file_list.json")
CACHE_DIR = Path(os.getenv("CACHE_DIR", ".cache"))
CACHE_DIR.mkdir(exist_ok=True, parents=True)

def load_filelist(path: str) -> List[str]:
    with open(path) as f:
        data = json.load(f)
    return data["files"]

def cdn_fetch(filename: str) -> pd.DataFrame:
    local_path = hf_hub_download(
        repo_id=REPO,
        filename=filename,
        cache_dir=CACHE_DIR,
        local_dir_use_symlinks=False,
    )
    return pq.read_table(local_path).to_pandas()

def build_dataset(file_list: List[str]) -> pd.DataFrame:
    rows = []
    for fn in file_list:
        df = cdn_fetch(fn)
        for _, row in df.iterrows():
            rows.append({
                "prompt": row.get("prompt") or row.get("input") or "",
                "response": row.get("response") or row.get("output") or "",
            })
    return pd.DataFrame(rows)

if __name__ == "__main__":
    files = load_filelist(FILE_LIST)
    dataset = build_dataset(files)
    print(f"Built dataset with {len(dataset)} samples")
    # Continue training...
```

### 3. Lightning Studio reuse + idle-stop guard

```python
import time
from lightning import Studio, Machine, Teamspace
from typing import Optional

STUDIO_NAME = os.getenv("STUDIO_NAME", "surrogate-train-l40s")
MACHINE = Machine.L40S
MAX_RETRIES = int(os.getenv("MAX_RETRIES", "3"))
RETRY_DELAY = int(os.getenv("RETRY_DELAY", "30"))

def get_or_create_studio() -> Studio:
    ts = Teamspace()
    for s in ts.studios:
        if s.name == STUDIO_NAME and s.status == "Running":
            print(f"Reusing running studio: {s.id}")
            return s
    print(f"Creating studio: {STUDIO_NAME}")
    return Studio(name=STUDIO_NAME, machine=MACHINE, create_ok=True)

def run_with_retry(studio: Studio, script_path: str, args: Optional[list] = None, max_retries: int = MAX_RETRIES):
    args = args or []
    for attempt in range(1, max_retries + 1):
        try:
            run = studio.run(
                script_path,
                arguments=args,
                shutdown_on_complete=False,
            )
            print(f"Run started: {run.id}")
            return run
        except Exception as exc:
            print(f"Attempt {attempt}/{max_retries} failed: {exc}")
            if attempt == max_retries:
                raise
            time.sleep(RETRY_DELAY * attempt)

if __name__ == "__main__":
    studio = get_or_create_studio()
    run_with_retry(studio, "train.py", args=["--epochs", "1"])
```

### 4. HF commit-cap mitigation (deterministic spread)

```python
import hashlib

WRITE_REPOS = [
    "axentx/surrogate-mirror",
    "axentx/surrogate-mirror-1",
    "axentx/surrogate-mirror-2",
    "axentx/surrogate-mirror-3",
    "axentx/surrogate-mirror-4",
]

def repo_for_slug(slug: str) -> str:
    digest = hashlib.sha256(slug.encode()).hexdigest()
    idx = int(digest, 16) % len(WRITE_REPOS)
    return WRITE_REPOS[idx]
```
