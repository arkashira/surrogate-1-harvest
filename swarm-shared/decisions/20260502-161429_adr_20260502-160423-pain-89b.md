## Deployment
cf workers

## v0.1 scope
v0.1 (2 weeks): CLI stealth client (Linux/macOS) that creates a local SOCKS5 listener, cloaks port via TLS camouflage, applies basic DPI evasion (packet timing jitter, common TLS fingerprint), enforces kill-switch on profile failure, and reports anonymized session events (WAU-ZB) to Supabase via CF Workers; includes one pre-baked stealth profile and basic blocklist delta distributed by CF Workers KV.

## Open questions
- Which DPI signatures (packet timing, TTL, TCP options, TLS fingerprints) are most reliable to spoof for our top target regions?
- Can we distribute stealth profiles and blocklist deltas via CF Workers KV/Queue within free-tier limits at expected scale?
- What is the acceptable trade-off between stealth overhead (latency/battery) and evasion strength for mobile clients?
- How will we measure WAU-ZB reliably without deanonymizing users (hashing, differential privacy, opt-in telemetry)?
- Which kill-switch enforcement method (network namespace, firewall rules, TUN drop) is feasible and safe across desktop OSes?
- Do we need kernel-bypass or raw socket capabilities on any target platform, and can we ship that in a zero-config client?
