# workio / backend

## Final synthesized implementation (single, correct, actionable)

### 1. Diagnosis (merged + corrected)
- **Tenant cache missing on `/webhook/line`** — repeated DB lookups on every LINE event cause DB thrash and increase 429/timeout risk under traffic spikes.
- **No request-scoped correlation IDs** — spans across webhook → services → DB are untraceable; incidents are hard to debug and logs are noisy.
- **No idempotency on clock-in/out webhook handler** — LINE retries on 5xx/timeout can create duplicate attendance records.
- **No graceful degradation on downstream failures** (DB/Redis/LINE API) — single failure breaks the webhook and causes retry storms.
- **No circuit-breaker/retry budget for outbound LINE API calls** — outbound calls (reply/push) can amplify failures during upstream incidents.
- **Missing structured logging and timing metrics** — can’t measure throughput, error rates, latency, or cache hit ratio to validate fixes.

### 2. Proposed change (merged)
- **File scope**: `workio/server/src/routes/webhook/line.ts` and supporting middleware/utils.
- **Changes**:
  - Add in-memory LRU cache (`channelId → tenant`) with 5-minute TTL to eliminate repeated DB lookups on the webhook path.
  - Add request-scoped correlation ID middleware (generate if missing, attach to `req`, and include in logs and spans).
  - Add idempotency guard on clock-in/out processing using a short-lived Redis (preferred) or in-memory set keyed by deterministic hash of `channelId:userId:eventType:timestamp` (10-minute TTL).
  - Add graceful degradation: on DB/Redis failure, return 200 to stop LINE retries and queue for async retry (or at least log and acknowledge) so retries don’t amplify failures.
  - Add circuit-breaker + retry budget for outbound LINE API calls (reply/push) to prevent cascading failures.
  - Add structured JSON logging with request-scoped fields and minimal metrics (cache hit/miss, duplicate detected, DB/Redis/LINE errors, latency histograms).

```ts
import { Request, Response, NextFunction } from 'express';
import { v4 as uuidv4 } from 'uuid';

export function correlationMiddleware(req: Request, _res: Response, next: NextFunction) {
  const cid = (req.headers['x-correlation-id'] as string) || uuidv4();
  (req as any).correlationId = cid;
  next();
}
```

#### `src/utils/tenantCache.ts`
```ts
import LRU from 'lru-cache';
import { getTenantByChannelId } from '../db/tenants'; // adjust import path
import { metrics } from './metrics';

const cache = new LRU<string, { id: string; [k: string]: any }>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
});

export async function resolveTenant(channelId: string) {
  const cached = cache.get(channelId);
  if (cached) {
    metrics.inc('tenant_cache_hit');
    return cached;
  }
  metrics.inc('tenant_cache_miss');
  const tenant = await getTenantByChannelId(channelId);
  if (tenant) cache.set(channelId, tenant);
  return tenant;
}
```

#### `src/utils/idempotency.ts` (Redis-first, memory fallback)
```ts
import { createClient } from 'redis';
import { metrics } from './metrics';

let redisClient: any = null;
const MEMORY_TTL_MS = 10 * 60 * 1000;
const memorySet = new Set<string>();
const memoryTimers = new Map<string, NodeJS.Timeout>();

// Try Redis; fallback to memory
(async () => {
  if (process.env.REDIS_URL) {
    try {
      redisClient = createClient({ url: process.env.REDIS_URL });
      redisClient.on('error', (err: Error) => {
        // non-fatal
        console.warn('Redis unavailable, using memory idempotency', err);
      });
      await redisClient.connect();
    } catch {
      redisClient = null;
    }
  }
})();

function memoryIsDuplicate(key: string): boolean {
  if (memorySet.has(key)) return true;
  memorySet.add(key);
  const timer = setTimeout(() => {
    memorySet.delete(key);
    memoryTimers.delete(key);
  }, MEMORY_TTL_MS);
  memoryTimers.set(key, timer);
  return false;
}

export async function isDuplicate(key: string): Promise<boolean> {
  if (redisClient) {
    try {
      const added = await redisClient.set(key, '1', { PX: MEMORY_TTL_MS, NX: true });
      if (!added) {
        metrics.inc('idempotency_duplicate');
        return true;
      }
      return false;
    } catch {
      // fall through to memory
    }
  }
  return memoryIsDuplicate(key);
}
```

#### `src/utils/lineCircuitBreaker.ts`

```ts
import CircuitBreaker from 'opossum';
import { metrics } from './metrics';

// Simple wrapper to add circuit-breaker around LINE client calls (reply/push)
export function createLineBreaker<T extends (...args: any[]) => Promise<any>>(
  fn: T,
  breakerOptions: any = {
    timeout: 3000,
    errorThresholdPercentage: 50,
    resetTimeout: 10000,
    volumeThreshold: 5,
  }
) {
  const breaker = new CircuitBreaker(fn, breakerOptions);
  breaker.on('open', () => metrics.inc('line_breaker_open'));
  breaker.on('halfOpen', () => metrics.inc('line_breaker_half_open'));
  breaker.on('close', () => metrics.inc('line_breaker_close'));
  breaker.on('failure', () => metrics.inc('line_call_failure'));
  return breaker;
}
```

```ts
const counts: Record<string, number> = {};
const timings: Record<string, number[]> = {};

export const metrics = {
  inc: (k: string) => { counts[k] = (counts[k] || 0) + 1; },
  histogram: (k: string, v: number) => {
    if (!timings[k]) timings[k] = [];
    timings[k].push(v);
  },
  get: () => ({ counts: { ...counts }, timings: { ...timings } }),
};
```
