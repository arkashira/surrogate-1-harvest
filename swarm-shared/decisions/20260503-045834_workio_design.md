# workio / design

## Final Synthesis — One Correct, Actionable Plan

**Core goal**: Make LINE webhook ingestion safe against duplicates, replays, races, and out-of-order delivery — with minimal schema change and maximal operational clarity.

---

### 1. Diagnosis (merged + resolved)
- **No idempotency** → LINE retries + double-taps create duplicate punches/leave/OT rows.
- **Signature without TTL** → replayable indefinitely; must bound acceptance window.
- **No monotonic state check** → out-of-order/delayed webhooks can corrupt state (clock-out before clock-in).
- **No deduplication/concurrency control** → concurrent retries can insert duplicates even with app-level checks.
- **Missing traceability** → no correlation ID across frontend → webhook → DB; slows debugging and duplicate detection.
- **Missing audit fields** → hard to reconstruct why/when a punch was created/updated.

---

### 2. Schema (minimal, safe)

```sql
-- Idempotency + replay protection
CREATE TABLE IF NOT EXISTS line_webhook_deliveries (
  idempotency_key  TEXT PRIMARY KEY,
  user_id          INTEGER NOT NULL,
  event_type       TEXT NOT NULL,
  payload_hash     TEXT NOT NULL,
  created_at       TIMESTAMPTZ DEFAULT NOW()
);

-- Optional but recommended: prevent double clock-in at DB level
-- Assumes punches table has: id, user_id, date, clock_in, clock_out, created_at, updated_at
CREATE UNIQUE INDEX IF NOT EXISTS unique_open_punch
ON punches (user_id, date)
WHERE clock_out IS NULL;

-- Add audit columns if missing
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ DEFAULT NOW();
```

### 3. Webhook handler (single source of truth)

File: `workio/server/src/routes/line-webhook.ts`

```ts
import crypto from 'crypto';
import { Router } from 'express';
import { db } from '../db';

const router = Router();
const SIGNATURE_TTL_MS = 5 * 60 * 1000; // 5 minutes

function verifyLineSignature(rawBody: Buffer, signature: string, timestamp: string): boolean {
  const now = Date.now();
  if (!timestamp || Math.abs(now - Number(timestamp)) > SIGNATURE_TTL_MS) return false;

  const expected = crypto
    .createHmac('sha256', process.env.LINE_CHANNEL_SECRET!)
    .update(rawBody)
    .digest('base64');

  return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(signature));
}

function idempotencyKey(events: any[]): string {
  // Deterministic across retries: destination user + first event timestamp + type
  const e = events[0];
  return `line:${e.source?.userId || 'unknown'}:${e.timestamp}:${e.type}`;
}

router.post('/webhook/line', async (req, res) => {
  const rawBody = req.body as Buffer;
  const signature = req.headers['x-line-signature'] as string;
  const timestamp = req.headers['x-line-request-timestamp'] as string;

  if (!verifyLineSignature(rawBody, signature, timestamp || '')) {
    return res.status(401).json({ error: 'invalid_signature_or_replay' });
  }

  const body = req.body as any;
  const events = body.events || [];
  if (!events.length) return res.status(200).send('OK');

  const key = idempotencyKey(events);
  const payloadHash = crypto.createHash('sha256').update(JSON.stringify(events)).digest('hex');

  // Fast idempotency check
  const existing = await db`
    SELECT 1 FROM line_webhook_deliveries WHERE idempotency_key = ${key}
  `.then((r) => r[0]);

  if (existing) return res.status(200).send('OK');

  try {
    await db.begin(async (trx) => {
      // Record delivery first (makes retries idempotent)
      await trx`
        INSERT INTO line_webhook_deliveries (idempotency_key, user_id, event_type, payload_hash)
        VALUES (${key}, ${events[0].source?.userId || ''}, ${events[0].type}, ${payloadHash})
      `;

      for (const ev of events) {
        if (ev.type === 'message' && ev.message?.type === 'text') {
          const text = (ev.message.text || '').trim().toLowerCase();
          const lineUserId = ev.source?.userId;
          if (!lineUserId) continue;

          if (text === 'clock in' || text === 'clock out') {
            const today = new Date().toISOString().slice(0, 10);
            const isClockIn = text === 'clock in';

            const user = await trx`
              SELECT id FROM users WHERE line_user_id = ${lineUserId}
            `.then((r) => r[0]);

            if (!user) continue;

            const latest = await trx`
              SELECT id, clock_out FROM punches
              WHERE user_id = ${user.id} AND date = ${today}
              ORDER BY created_at DESC
              LIMIT 1
            `.then((r) => r[0]);

            // Monotonic state validation
            if (isClockIn && latest && !latest.clock_out) continue; // already in
            if (!isClockIn && (!latest || latest.clock_out)) continue; // nothing to out

            if (isClockIn) {
              // Unique constraint prevents double insert if race occurs
              try {
                await trx`
                  INSERT INTO punches (user_id, date, clock_in, created_at, updated_at)
                  VALUES (${user.id}, ${today}, NOW(), NOW(), NOW())
                `;
              } catch (e: any) {
                // If unique constraint violation, skip duplicate
                if (!e?.code?.includes('23505')) throw e;
              }
            } else {
              await trx`
                UPDATE punches
                SET clock_out = NOW(), updated_at = NOW()
                WHERE id = ${latest.id}
              `;
            }
          }
        }
      }
    });

    res.status(200).send('OK');
  } catch (err) {
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'processing_failed' });
  }
});

export default router;
```

### 4. Frontend + traceability

```ts
// workio/src/lib/line.ts
export async function sendLineCommand(cmd: 'clock_in' | 'clock_out') {
  const traceId = crypto.randomUUID();
  if ((window as any).__linePending) return;
  (window as any).__linePending = true;

  try {
    await fetch('/api/line/command', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-Trace-ID': traceId,
      },
      body: JSON.stringify({ command: cmd, trace_id: traceId }),
    });
  } finally {
    (window as any).__linePending = false;
  }
}
```

- Propagate `X-Trace-ID` (or `trace_id` in body) into logs in both frontend-originated requests and webhook processing for correlation.

### 5. Verification (concrete tests)

1. **Replay after TTL**  
   - Capture real LINE payload + signature.  
   - Replay after 6 minutes → expect 401 and no DB insert.

2. **Idempotency within TTL**  
   - Replay same payload within 5 minutes → expect 200 and exactly one punch row; `line_webhook_deliveries` has one entry.

3. **Double-tap / race**  
   - Rapid double-tap “Clock In” in UI → expect only one open punch for user today (unique index prevents duplicates).

4. **Out-of-order webhook**  
   - Simulate delayed clock-out arriving after manual clock-in on another device → should close open punch, not create duplicates.

5. **Audit & traceability**  
   - Verify logs contain `X-Trace-ID` across frontend request and webhook processing for same user action.
