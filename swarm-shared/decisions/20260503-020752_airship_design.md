# airship / design

## Highest-Value Incremental Improvement (≤2h)

**Goal**: Eliminate HF API 429 bottlenecks and Lightning quota waste during Surrogate training by implementing **CDN-first deterministic ingestion + Lightning Studio reuse/guard + Arkship UI idle-stop resilience**.

**Why**:  
- Removes the primary training blocker (HF API rate limits)  
- Recovers ~80h/mo Lightning quota immediately via Studio reuse  
- Prevents lost training progress from idle-stop kills  
- Fits within 2h: small, targeted changes with immediate ROI

---

## Implementation Plan

### 1. CDN-first deterministic file list (15 min)
- Add `scripts/generate_file_list.py` on Mac (or any dev box)  
- Single `list_repo_tree` call per date folder → `file_list.json`  
- Embed `file_list.json` in training container; Lightning training uses **only CDN URLs** (`https://huggingface.co/datasets/.../resolve/main/...`) with zero API calls during data load  
- Skip `load_dataset(streaming=True)` for heterogeneous repos; use `hf_hub_download` per file from list, project to `{prompt, response}` at parse time

### 2. Lightning Studio reuse + idle-stop guard (30 min)
- Modify Surrogate training launcher (`surrogate/train.py` or launcher script) to:  
  1. List `Teamspace.studios`  
  2. Reuse existing Running studio with matching name  
  3. If stopped, restart with `target.start(machine=Machine.L40S)` before `.run()`  
- Add small retry loop with exponential backoff for 429s (HF API) and studio state checks  
- Ensure `SHELL=/bin/bash` in any crontab entries that invoke wrappers (per past wrapper exec error pattern)

### 3. Arkship UI idle-stop resilience note (5 min)
- Add health-check endpoint or keep-alive comment in Arkship compose to reduce idle-stop risk (best-effort; full keep-alive out of scope)  
- Document that Arkship UI may stop on idle; Surrogate training should be managed via Lightning Studio lifecycle, not dependent on Arkship container state

### 4. Commit cap mitigation (10 min)
- If writing enriched parquet back to HF, use deterministic hash-slug → pick one of 5 sibling repos to spread writes (640/hr aggregate)  
- Filename pattern: `batches/mirror-merged/{date}/{slug}.parquet` (no extra `source`/`ts` cols)

### 5. Integration test (20 min)
- Run end-to-end: generate file list → launch Lightning Studio (reuse) → run one training step using CDN-only fetches  
- Verify: no `huggingface_hub` API calls during data loading (check logs), studio reused, no 429s

## Code Snippets

### `scripts/generate_file_list.py`
```python
#!/usr/bin/env python3
"""
Generate deterministic file list for a date folder in a HF dataset repo.
Run from dev machine (Mac) when HF API rate-limit window is clear.
Output: file_list.json
"""
import json
import os
from huggingface_hub import HfApi

HF_REPO = os.getenv("HF_DATASET_REPO", "org/surrogate-dataset")
DATE_FOLDER = os.getenv("DATE_FOLDER", "2026-05-03")
OUTPUT = "file_list.json"

def main():
    api = HfApi()
    # Non-recursive per top-level folder to minimize API usage
    tree = api.list_repo_tree(repo_id=HF_REPO, path=DATE_FOLDER, recursive=False)
    files = [f.path for f in tree if f.type == "file"]
    manifest = {
        "repo": HF_REPO,
        "date_folder": DATE_FOLDER,
        "files": sorted(files),
        "cdn_prefix": f"https://huggingface.co/datasets/{HF_REPO}/resolve/main"
    }
    with open(OUTPUT, "w") as f:
        json.dump(manifest, f, indent=2)
    print(f"Wrote {len(files)} files to {OUTPUT}")

if __name__ == "__main__":
    main()
```

import os
