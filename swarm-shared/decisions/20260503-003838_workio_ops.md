# workio / ops

## 1) Diagnosis

- Missing **idempotency key** on `/webhook/line` POST allows LINE retries (network blips, 5xx, slow client) to create duplicate punch records.
- No **partial unique index** enforcing “one open punch per user” (`clock_out_at IS NULL`) permits multiple concurrent clock-ins.
- No **transactional upsert** for clock-in/out flows; race conditions between rapid LINE events can corrupt state.
- Missing **request deduplication layer** (in-memory or Redis) for same-message-id replay protection across restarts.
- No **audit trail** (created_at + source + idempotency_key) to debug duplicate origins or replay incidents.

## 2) Proposed change

File: `/opt/axentx/workio/server/src/db/schema.sql`  
Add:  
- `punches.idempotency_key` (varchar, indexed)  
- Partial unique index `one_open_punch_per_user` on `(user_id)` where `clock_out_at IS NULL`  
- Composite unique index on `(user_id, idempotency_key)` to block same-message replay  
- Default `created_at` and nullable `updated_at` for audit

File: `/opt/axentx/workio/server/src/routes/webhook/line.js` (or equivalent)  
Add:  
- Extract or generate idempotency key from LINE `deliveryId`/`messageId` (prefer `deliveryId`, fallback to hash of message+timestamp).  
- Use `INSERT ... ON CONFLICT (user_id, idempotency_key) DO NOTHING` for punch creation, then upsert clock-out via `UPDATE ... WHERE user_id = $1 AND clock_out_at IS NULL AND idempotency_key <> $2`.  
- Wrap clock-in/out flow in a transaction with `SERIALIZABLE` or `READ COMMITTED` + row locking (`SELECT ... FOR UPDATE`) to prevent races.

## 3) Implementation

```sql
-- /opt/axentx/workio/server/src/db/schema.sql
-- Add columns to punches if not present (idempotent migration style)
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255),
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ;

-- Ensure one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS one_open_punch_per_user
  ON punches (user_id)
  WHERE clock_out_at IS NULL;

-- Prevent same idempotency key reuse for same user (replay protection)
CREATE UNIQUE INDEX IF NOT EXISTS punches_user_idempotency_key
  ON punches (user_id, idempotency_key);

-- Index for fast lookups by idempotency and open punches
CREATE INDEX IF NOT EXISTS punches_idempotency_key_idx
  ON punches (idempotency_key);
```

```js
// /opt/axentx/workio/server/src/routes/webhook/line.js
const express = require('express');
const { pool } = require('../../db');
const crypto = require('crypto');
const router = express.Router();

function generateIdempotencyKey(event) {
  // Prefer LINE deliveryId; fallback to messageId + timestamp hash
  const deliveryId = event.deliveryId;
  if (deliveryId) return `line-delivery-${deliveryId}`;
  const messageId = event.message?.id;
  const timestamp = event.timestamp || Date.now();
  return `line-msg-${crypto.createHash('sha256').update(`${messageId}:${timestamp}`).digest('hex').slice(0, 32)}`;
}

router.post('/line', async (req, res) => {
  const client = await pool.connect();
  try {
    const events = req.body.events || [];
    for (const event of events) {
      if (event.type !== 'message' || event.message?.type !== 'text') continue;

      const userId = event.source?.userId;
      const text = event.message.text?.trim().toLowerCase();
      const idempotencyKey = generateIdempotencyKey(event);
      if (!userId) continue;

      await client.query('BEGIN');

      // Try insert new punch (clock-in) with idempotency guard
      const insertResult = await client.query(
        `INSERT INTO punches (user_id, clock_in_at, idempotency_key, created_at)
         VALUES ($1, NOW(), $2, NOW())
         ON CONFLICT (user_id, idempotency_key) DO NOTHING
         RETURNING *`,
        [userId, idempotencyKey]
      );

      // If insert happened (new clock-in), commit and continue
      if (insertResult.rowCount > 0) {
        await client.query('COMMIT');
        continue;
      }

      // Otherwise, this is a duplicate or a clock-out candidate.
      // Find the open punch for this user (there should be at most one due to partial index)
      const openPunch = await client.query(
        `SELECT id FROM punches
         WHERE user_id = $1 AND clock_out_at IS NULL
         ORDER BY clock_in_at DESC LIMIT 1 FOR UPDATE`,
        [userId]
      );

      if (openPunch.rows.length === 0) {
        // No open punch but duplicate idempotency — safe to ignore
        await client.query('COMMIT');
        continue;
      }

      // Apply clock-out if text indicates clock-out (e.g., 'ออก' or 'clock out')
      // Customize keywords per your locale/requirements
      const isClockOut = text.includes('ออก') || text.includes('clock out') || text.includes('off');
      if (isClockOut) {
        await client.query(
          `UPDATE punches
           SET clock_out_at = NOW(), updated_at = NOW()
           WHERE id = $1 AND clock_out_at IS NULL`,
          [openPunch.rows[0].id]
        );
      }

      await client.query('COMMIT');
    }
    res.json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error:', err);
    res.status(500).json({ error: 'internal_error' });
  } finally {
    client.release();
  }
});

module.exports = router;
```
