# workio / quality

## Final Synthesis (Best Parts + Correctness + Actionability)

### 1. Diagnosis (merged)
- **LINE retries/replays** with same `X-Line-Signature` + payload can create duplicate punches because there is no idempotency key enforcement.
- **Active-punch uniqueness is app-only** (`findOne → if exists update else insert`) and racy; no DB-level uniqueness constraint prevents double active punches per user/tenant.
- **Non-atomic upsert path** allows concurrent deliveries to insert multiple active rows.
- **No server-side idempotency table** to reject replays within a safe window.
- **No transactional boundary covering punch write + audit/notification**: partial failures can leave inconsistent state and cause retries to double-insert.
- **Clock-in/out path lacks retry-safety**, corrupting active-punch state and duration/OT calculations.

### 2. Correctness-first constraints and behavior
- **Exactly-one active punch per (user, tenant)** must be enforced at the DB level, not app logic.
- **Idempotency must be persisted before side effects** (or inside the same transaction) so retries are short-circuited even if prior attempt partially failed.
- **Idempotency key must be stable and scoped per operation** (clock-in vs clock-out) to avoid conflating distinct actions.
- **Clock-out must require an active punch**; if missing, fail fast (do not invent state).
- **Retention policy**: keep idempotency keys long enough to cover LINE retry window (e.g., 24–72h) and provide cleanup.

### 3. Implementation (single, actionable plan)

#### 3.1 DB schema (run once)

```sql
-- Idempotency table for LINE webhook replays and punch operation dedup
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  punch_id        BIGINT NOT NULL REFERENCES punches(id) ON DELETE CASCADE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: index for cleanup job
CREATE INDEX IF NOT EXISTS idx_punch_idempotency_created_at
  ON punch_idempotency (created_at);

-- Enforce at most one active punch per user per tenant (clock_out_at IS NULL)
-- This is the authoritative guard against duplicate active punches.
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_active_punch
  ON punches (user_id, tenant_id)
  WHERE clock_out_at IS NULL;
```

Add a periodic cleanup job (recommended):
```sql
-- Example: delete keys older than 72h
DELETE FROM punch_idempotency WHERE created_at < NOW() - INTERVAL '72 hours';
```

#### 3.2 Idempotent punch service (atomic + transactional)
File: `workio/server/src/services/punchService.ts`

```ts
import { db } from '../db';
import { eq, and, isNull } from 'drizzle-orm';
import { punches, punchIdempotency } from '../db/schema';

export async function upsertPunchFromLine({
  userId,
  tenantId,
  clockInAt,
  idempotencyKey,
}: {
  userId: number;
  tenantId: number;
  clockInAt: Date;
  idempotencyKey: string;
}) {
  return await db.transaction(async (tx) => {
    // Short-circuit replay
    const existing = await tx.query.punchIdempotency.findFirst({
      where: eq(punchIdempotency.idempotencyKey, idempotencyKey),
    });
    if (existing) {
      const punch = await tx.query.punches.findFirst({
        where: eq(punches.id, existing.punchId),
      });
      return punch!;
    }

    const now = new Date();

    // Atomic upsert: one active punch per (user, tenant)
    const [punch] = await tx
      .insert(punches)
      .values({
        userId,
        tenantId,
        clockInAt,
        clockOutAt: null,
        createdAt: now,
        updatedAt: now,
      })
      .onConflictDoUpdate({
        target: [punches.userId, punches.tenantId],
        where: isNull(punches.clockOutAt),
        set: {
          // Preserve earliest clockInAt; only touch metadata
          updatedAt: now,
        },
      })
      .returning();

    await tx.insert(punchIdempotency).values({
      idempotencyKey,
      punchId: punch.id,
      createdAt: now,
    });

    return punch;
  });
}

export async function clockOutPunch({
  userId,
  tenantId,
  clockOutAt,
  idempotencyKey,
}: {
  userId: number;
  tenantId: number;
  clockOutAt: Date;
  idempotencyKey: string;
}) {
  return await db.transaction(async (tx) => {
    // Short-circuit replay
    const existing = await tx.query.punchIdempotency.findFirst({
      where: eq(punchIdempotency.idempotencyKey, idempotencyKey),
    });
    if (existing) {
      const punch = await tx.query.punches.findFirst({
        where: eq(punches.id, existing.punchId),
      });
      return punch!;
    }

    // Find active punch
    const active = await tx.query.punches.findFirst({
      where: and(
        eq(punches.userId, userId),
        eq(punches.tenantId, tenantId),
        isNull(punches.clockOutAt)
      ),
    });

    if (!active) {
      throw new Error('No active punch to clock out');
    }

    const [updated] = await tx
      .update(punches)
      .set({ clockOutAt, updatedAt: new Date() })
      .where(eq(punches.id, active.id))
      .returning();

    await tx.insert(punchIdempotency).values({
      idempotencyKey,
      punchId: updated.id,
      createdAt: new Date(),
    });

    return updated;
  });
}
```

import { upsertPunchFromLine, clockOutPunch } from '../services/punchService';
import { verifySignature } from '../utils/lineSignature';
import crypto from 'crypto';

export async function handleLineWebhook(req, res) {
  const signature = req.get('X-Line-Signature');
  const body = req.body;

  if (!verifySignature(JSON.stringify(body), signature)) {
    return res.status(401).send('Invalid signature');
  }

  // Stable, deterministic idempotency base from events
  const eventsKey = hashStableEvents(body.events);
  const baseIdempotencyKey = `line:${signature}:${eventsKey}`;

  for (const event of body.events) {
    if (event.type === 'message' && event.message.type === 'text') {
      const userId = await mapLineUserIdToInternal(event.source.userId);
      const tenantId = await resolveTenantId(userId);

      const isClockOut = /ออก/.test(event.message.text);
      const now = new Date();

      // Separate keys for in/out to avoid conflating distinct actions
      const opKey = `${baseIdempotencyKey}:${isClockOut ? 'out' : 'in'}`;

      try {
        if (isClockOut) {
          await clockOutPunch({ userId, tenantId, clockOutAt: now, idempotencyKey: opKey });
        } else {
          await upsertPunchFromLine({ userId, tenantId, clockInAt: now, idempotencyKey: opKey });
        }
      } catch (err) {
        // Log but return 200 to prevent LINE retries when idempotent or non-recoverable
        console.error('Punch operation failed', { err, userId, tenantId, isClockOut });
      }
    }
  }

  return res.status(200).send('OK');
}
