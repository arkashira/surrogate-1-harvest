## v0.1 scope
v0.1 (2 weeks): API-only service deployed to CF Workers with Postgres (Supabase). Features: (1) Connect AWS and GCP credentials per customer; (2) One-click discovery that lists Lambda + Cloud Functions and their cron/rate triggers; (3) Deterministic dedupe (hash) to flag duplicate schedules across functions; (4) Drift detection (schedule expression or timezone changed since last discovery); (5) Orphan detection (schedule exists but no matching function); (6) Basic retire/flag API to mark schedules stale; (7) Schedule audit log for changes; (8) Simple GET endpoints for customers to list duplicates, drifted, and orphaned schedules and counts.

## Open questions
- Which identity model: per-customer IAM roles vs centralized service account with cross-account read-only policies?
- How to detect 'orphaned' schedules reliably (no matching function vs disabled vs deleted)?
- Retention and GDPR deletion policy for schedule_audit and raw cloud metadata snapshots
- Acceptable discovery cadence (hourly vs daily) given provider rate limits on free-tier credentials
- Should v0.1 include a CLI or remain API-only for integrations and automation?
