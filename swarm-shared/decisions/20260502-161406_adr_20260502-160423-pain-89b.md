## Deployment
hybrid (client binaries distributed via GitHub Releases + Cloudflare Workers for lightweight bridge discovery/health; no server-side state)

## v0.1 scope
v0.1 ships a single-file Rust CLI client (Linux x86_64) that: establishes an obfuscated tunnel (shadowsocks-like AEAD) to a user-provided bridge; creates and hides a TUN interface via boringtun; exposes a local SOCKS5 proxy; enforces a fail-closed kill-switch that blocks non-tunnel traffic; stores profiles and session logs only in local SQLite; emits opt-in, hashed, minimal telemetry to a CF Worker endpoint for WAU-ZB calculation; includes basic unit/integration tests and reproducible build docs.

## Open questions
- Which obfuscation front (meek domain-front, shadowsocks-like stream cipher, or utls JA3 mimicry) provides best evasion vs. performance on low-end devices in target regions?
- How will we distribute updated obfuscation profiles to clients without revealing endpoints to DPI probes or requiring centralized infrastructure that can be blocked?
- What local artifacts (ARP, DNS, firewall rules) must we suppress to prevent TUN/SOCKS5 detection by on-host monitoring tools?
- How do we implement a reliable kill-switch across Windows/macOS/Linux that cannot be bypassed by system routing changes?
- What minimal, privacy-preserving telemetry schema will allow WAU-ZB measurement without deanonymizing users or creating honeypot logs?
