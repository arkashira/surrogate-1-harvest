# airship / ops

## Highest-Value Incremental Improvement (≤2h)

**Goal**: Harden `airship discover` into a deterministic, CDN-only, zero-HF-API-runtime orchestrator that eliminates HF API rate limits and PyArrow schema errors while producing reproducible file lists for downstream training.

**Why this wins**
- Eliminates 429s during data load by moving all fetches to CDN (`resolve/main/`) after a single API list call.
- Avoids PyArrow `CastError` from heterogeneous repo schemas by projecting to `{prompt,response}` only at parse time.
- Fits in <2h: single orchestrator script + small training loader patch + one cron guard.

---

## Implementation Plan

1. **Create `scripts/airship_discover.py`** (or shell wrapper)  
   - Accepts repo, date folder, output JSON path.
   - Uses HF API **once** with `list_repo_tree(path=date_folder, recursive=False)` (non-recursive to avoid 100× pagination).
   - Emits `{"repo": "...", "date": "...", "files": ["path1", "path2", ...]}`.
   - Shebang `#!/usr/bin/env bash` if shell; executable bit set.

2. **Add `scripts/run_discover.sh` wrapper**  
   - Invoked via `bash scripts/run_discover.sh "$@"`.
   - Sets `SHELL=/bin/bash` (for cron safety).
   - Exits on error, logs to `logs/discover-$(date +%F).log`.

3. **Patch training loader (`surrogate/data/cdn_loader.py`)**  
   - Reads file-list JSON produced by discover.
   - Uses **only CDN URLs** (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) with `requests`/`wget`/`hf_transfer`; no `load_dataset` or `hf_hub_download` during training.
   - Projects each file to `{prompt, response}` at parse time; drops all other fields.
   - Writes sharded `batches/mirror-merged/{date}/{slug}.parquet` with no extra metadata columns.

4. **Cron guard & reuse pattern**  
   - Crontab entry sets `SHELL=/bin/bash`.
   - Before running, checks if today’s file list already exists; skips if present (idempotent).
   - On 429: sleeps 360s and retries (max 3).

5. **Lightning Studio reuse guard (if training triggered)**  
   - Before `.run()`, list `Teamspace.studios`; reuse running studio with matching name.
   - If stopped, restart with `target.start(machine=Machine.L40S)` (avoids idle timeout death).

### `scripts/run_discover.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

# Ensure bash in cron
export SHELL=/bin/bash

REPO="${1:-your-org/your-dataset}"
DATE_FOLDER="${2:-$(date +%F)}"
OUT_DIR="data/file_lists"
OUT_FILE="${OUT_DIR}/files-${DATE_FOLDER}.json"

mkdir -p "${OUT_DIR}" logs
exec > >(tee "logs/discover-${DATE_FOLDER}.log") 2>&1

echo "Running airship discover for ${REPO}/${DATE_FOLDER}"

# Idempotent: skip if already produced today
if [[ -f "${OUT_FILE}" ]]; then
  echo "File list already exists: ${OUT_FILE}. Skipping."
  exit 0
fi

python3 scripts/airship_discover.py \
  --repo "${REPO}" \
  --date-folder "${DATE_FOLDER}" \
  --output "${OUT_FILE}"
```

### `scripts/airship_discover.py`
```python
#!/usr/bin/env python3
import argparse
import json
import time
import sys
from huggingface_hub import HfApi, Repository

HF_API_RATE_LIMIT_RETRY = 360

def list_files_safe(api, repo_id, folder, max_retries=3):
    for attempt in range(1, max_retries + 1):
        try:
            # Non-recursive to avoid massive pagination
            tree = api.list_repo_tree(repo_id=repo_id, path=folder, recursive=False)
            files = [item.rfilename for item in tree if not item.rfilename.endswith("/")]
            return sorted(files)
        except Exception as e:
            if "429" in str(e) and attempt < max_retries:
                print(f"Rate limited (429). Waiting {HF_API_RATE_LIMIT_RETRY}s...", file=sys.stderr)
                time.sleep(HF_API_RATE_LIMIT_RETRY)
                continue
            raise
    raise RuntimeError("Max retries exceeded for list_repo_tree")

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--repo", required=True, help="HF dataset repo id")
    parser.add_argument("--date-folder", required=True, help="Folder path (e.g. 2026-05-02)")
    parser.add_argument("--output", required=True, help="Output JSON path")
    args = parser.parse_args()

    api = HfApi()
    files = list_files_safe(api, args.repo, args.date_folder)

    payload = {
        "repo": args.repo,
        "date": args.date_folder,
        "files": files,
    }

    with open(args.output, "w") as f:
        json.dump(payload, f, indent=2)

    print(f"Wrote {len(files)} files to {args.output}")

if __name__ == "__main__":
    main()
```

def build_parquet_from_cdn(file_list_path: str, out_dir: str):

    date = meta["date"]
    out_path = Path(out_dir) / f"mirror-merged/{date}"

## Acceptance Criteria
- `scripts/run_discover.sh` produces `data/file_lists/files-YYYY-MM-DD.json` without HF API rate-limit failures.
- Training loader (`cdn_loader.py`) consumes the JSON and fetches exclusively via CDN URLs.
- No `load_dataset(streaming=True)` or `hf_hub_download` calls during training data load.
- Parquet output contains only `prompt` and `response` columns; no `source`, `ts`, or mixed-schema fields.
