cf workers

## v0.1 scope
v0.1 in 2 weeks: a single deterministic workflow (research→signal→risk guardrail check→execution stub) exposed via FastAPI, queued on Cloudflare Workers, with immutable audit trail in Supabase Postgres + storage; includes one broker connector mock, one risk guardrail (hard limit), content-addressable inputs for replay, and basic compliance pass/fail status.

## Open questions
- Which brokers/OMS/EMS and data vendors must be prioritized for pre-built connectors in v0.1?
- What deterministic serialization format and content-addressing scheme will be used for replayable inputs (e.g., canonical JSON + SHA-256)?
- What compliance rule DSL or engine will satisfy auditability while remaining lightweight (e.g., JSON Logic, Rego, or simple Python predicates)?
- What are the latency SLOs per stage (research→signal, signal→risk, risk→execution) to qualify as 'sub-second'?
- How will secrets/credentials for connectors be provisioned and rotated on free-tier infra?
