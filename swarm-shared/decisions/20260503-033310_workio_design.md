# workio / design

## Final Synthesis (Correctness + Actionability)

**Core problems (unified):**
1. No HMAC-SHA256 signature verification on `/webhook/line` (spoofing/replay).
2. No idempotency/replay protection (duplicates from retries/double-taps).
3. Tenant context not cryptographically bound (cross-tenant injection risk).
4. Clock-in/out state races (concurrent/retried events → invalid transitions).
5. No immutable audit trail for sensitive time/leave actions.

**Chosen approach:**
- Use **raw-body capture** scoped to `/webhook/line` before JSON parsing.
- Verify `X-Line-Signature` with **constant-time HMAC-SHA256**.
- Enforce **idempotency at write** via DB unique key (not just Redis) for durability and correctness; short TTL cleanup.
- Enforce **state-machine transitions** for clock/leave with row-level locking/optimistic version to prevent races.
- Record **immutable audit rows** for every processed event (who, what, when, source, outcome).
- Bind tenant via **channelId → tenantId** mapping validated on every event; reject if unknown.

### 2) Middleware: raw-body capture for LINE only
`server/src/middleware/rawBodyCapture.ts`
```ts
import express, { Request, Response, NextFunction } from 'express';

export function rawBodyCapture(path: string) {
  return (req: Request, res: Response, next: NextFunction) => {
    if (req.path === path) {
      const chunks: Buffer[] = [];
      req.on('data', (chunk) => chunks.push(chunk));
      req.on('end', () => {
        const raw = Buffer.concat(chunks);
        (req as any).rawBody = raw;
        // Safely re-parse JSON for downstream handlers
        try {
          req.body = raw.length ? JSON.parse(raw.toString('utf8')) : {};
        } catch {
          req.body = {};
        }
        next();
      });
      req.on('error', next);
    } else {
      next();
    }
  };
}
```

### 3) Signature verification (constant-time)
`server/src/middleware/lineSignature.ts`
```ts
import crypto from 'crypto';

export function verifyLineSignature(rawBody: Buffer | string, signature: string, channelSecret: string): boolean {
  if (!signature || !channelSecret || !rawBody) return false;
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody, 'utf8')
    .digest('base64');
  // constant-time compare
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}
```

### 4) Idempotency key derivation
Deterministic, compact, includes channel to bind tenant context:
```ts
import crypto from 'crypto';

export function idempotencyKey(event: any): string {
  return crypto
    .createHash('sha256')
    .update(
      [
        event.destination?.channelId || '',
        event.source?.userId || '',
        event.timestamp || '',
        event.type || '',
        event.mode || '',
        event.webhookEventId || '',
      ].join('|')
    )
    .digest('hex');
}
```

### 6) Clock/leave state transition helper (prevent races)
`server/src/services/attendanceService.ts` (excerpt)
```ts
import db from '../db';

export async function recordClock(
  tenantId: string,
  userId: string,
  type: 'in' | 'out',
  eventTime: Date,
  sourceEventId: string
) {
  const client = await db.connect();
  try {
    await client.query('BEGIN');

    // Lock latest attendance row for this user to prevent race
    const latest = await client.query(
      `SELECT id, type, recorded_at, version
       FROM attendance
       WHERE tenant_id = $1 AND user_id = $2
       ORDER BY recorded_at DESC, id DESC
       FOR UPDATE`,
      [tenantId, userId]
    );

    const last = latest.rows[0];
    if (type === 'in' && last && last.type === 'in' && last.recorded_at >= eventTime) {
      // duplicate or out-of-order clock-in; reject to enforce valid sequence
      await client.query('ROLLBACK');
      return { ok: false, reason: 'invalid_transition' };
    }
    if (type === 'out' && (!last || last.type !== 'in')) {
      await client.query('ROLLBACK');
      return { ok: false, reason: 'no_active_clock_in' };
    }

    // Insert new record
    await client.query(
      `INSERT INTO attendance (tenant_id, user_id, type, recorded_at, source_event_id, version)
       VALUES ($1, $2, $3, $4, $5, 1)`,
      [tenantId, userId, type, eventTime, sourceEventId]
    );

    await client.query('COMMIT');
    return { ok: true };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```
