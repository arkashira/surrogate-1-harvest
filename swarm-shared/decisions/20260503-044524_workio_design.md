# workio / design

## 1. Diagnosis

- **No idempotency on LINE webhook ingestion**: duplicate `X-Line-Signature` replays (network retries/timeouts) can double-apply clock-in/out and leave/OT transitions, corrupting daily totals and approval state.
- **Missing strict `X-Line-Signature` verification**: handler accepts requests without HMAC-SHA256 check, allowing spoofed punches or state changes.
- **No replay-window protection**: same event body+signature within minutes can be reprocessed because there is no dedupe key or short-term ledger.
- **Race condition on punch state**: concurrent duplicate webhooks can produce two “clock-in” rows for the same user/day when the first insert is not yet visible (READ-COMMITTED).
- **No observability for webhook replays**: no request-id or idempotency-key logging, making duplicate events hard to detect/debug.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/line/webhook.ts` (or equivalent route handler)  
Scope:
- Add idempotency table `webhook_deliveries(idempotency_key text PK, created_at timestamptz, processed_at timestamptz, status text, request_hash bytea)` and index on `created_at`.
- Add middleware to verify `X-Line-Signature` strictly (constant-time compare) and reject if missing/invalid.
- Wrap event processing in an idempotent handler: derive deterministic idempotency key from `X-Line-Signature` + raw body (or use `X-Line-Delivery-Id` if provided) and short TTL (e.g., 10m).
- Use `INSERT ... ON CONFLICT DO NOTHING` to gate processing; if conflict and already processed, return 200 immediately without side-effects.
- Add request-id header and structured logs for traceability.

```ts
// server/src/middleware/lineSignature.ts
import crypto from 'crypto';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

export function verifyLineSignature(rawBody: string | Buffer, signature: string): boolean {
  if (!signature || !LINE_CHANNEL_SECRET) return false;
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody, 'utf8')
    .digest('base64');
  // constant-time compare
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}
```

```ts
// server/src/routes/line/webhook.ts (excerpt)
import { Router, Request, Response } from 'express';
import { pool } from '../../db';
import { verifyLineSignature } from '../../middleware/lineSignature';
import crypto from 'crypto';

const router = Router();

function idempotencyKey(signature: string, rawBody: string): string {
  // Prefer delivery-id from header if LINE provides one; fallback to sig+hash
  return crypto.createHash('sha256').update(signature + rawBody).digest('hex');
}

router.post('/webhook/line', async (req: Request, res: Response) => {
  const signature = req.get('X-Line-Signature') || '';
  const rawBody = typeof req.body === 'string' ? req.body : JSON.stringify(req.body);
  const requestId = crypto.randomUUID();

  // 1) Strict signature verification
  if (!verifyLineSignature(rawBody, signature)) {
    console.warn(`[${requestId}] Invalid LINE signature`);
    return res.status(401).send('Invalid signature');
  }

  // 2) Idempotency guard
  const key = idempotencyKey(signature, rawBody);
  const requestHash = crypto.createHash('sha256').update(rawBody).digest();

  try {
    const insertResult = await pool.query(
      `INSERT INTO webhook_deliveries (idempotency_key, request_hash, status)
       VALUES ($1, $2, 'pending')
       ON CONFLICT (idempotency_key) DO UPDATE
       SET request_hash = EXCLUDED.request_hash
       WHERE webhook_deliveries.processed_at IS NULL
       RETURNING webhook_deliveries.processed_at, webhook_deliveries.status`,
      [key, requestHash]
    );

    // If row existed and already processed, skip processing
    if (insertResult.rowCount === 0) {
      const existing = await pool.query(
        `SELECT processed_at, status FROM webhook_deliveries WHERE idempotency_key = $1`,
        [key]
      );
      if (existing.rows[0]?.processed_at) {
        console.log(`[${requestId}] Duplicate webhook ignored (already processed)`);
        return res.status(200).send('OK');
      }
      // If still pending (concurrent), wait briefly or return 202 to let LINE retry safely
      console.log(`[${requestId}] Concurrent duplicate; returning 202`);
      return res.status(202).send('Processing');
    }

    // 3) Process events (atomic per event to avoid races)
    const events = req.body.events || [];
    await pool.query('BEGIN');
    try {
      for (const event of events) {
        // Example: clock-in/out handling with upsert to avoid double-punch
        if (event.type === 'message' && event.message?.text === 'clock-in') {
          const userId = event.source?.userId;
          const ts = new Date(event.timestamp);
          await pool.query(
            `INSERT INTO punches (user_id, tenant_id, punch_type, punched_at, line_event_id)
             VALUES ($1, $2, 'in', $3, $4)
             ON CONFLICT (user_id, tenant_id, punch_date) -- punch_date generated from punched_at::date
             DO UPDATE SET punched_at = EXCLUDED.punched_at, line_event_id = EXCLUDED.line_event_id
             WHERE punches.punched_at < EXCLUDED.punched_at`,
            [userId, 'default-tenant', ts, event.message.id]
          );
        }
        // Add leave/OT handlers similarly with state-machine transitions guarded by current state
      }
      await pool.query(
        `UPDATE webhook_deliveries SET processed_at = NOW(), status = 'processed', response_status = 200 WHERE idempotency_key = $1`,
        [key]
      );
      await pool.query('COMMIT');
      console.log(`[${requestId}] Processed ${events.length} events`);
      res.status(200).send('OK');
    } catch (err) {
      await pool.query('ROLLBACK');
      await pool.query(
        `UPDATE webhook_deliveries SET status = 'failed' WHERE idempotency_key = $1`,
        [key]
      );
      console.error(`[${requestId}] Processing failed`, err);
      res.status(500).send('Processing error');
    }
  } catch (err) {
    console.error(`[${requestId}] Idempotency check failed`, err);
    res.status(500).send('Server error');
  }
});

export default router;
```

Notes:
- Use raw body middleware (`express.raw({ type: '*/*' })`) before JSON parsing for reliable signature verification.
- If LINE provides `X-Line-Delivery-Id`, prefer it as idempotency key.
- Keep TTL for `webhook_deliveries` (e.g., 7d) via cron or pg_cron.
