## Deployment
hybrid (local-first CLI + optional Supabase for opt-in analytics + Cloudflare Workers for auth/telemetry endpoint)

## v0.1 scope
v0.1 ships in 2 weeks: local CLI that indexes a Python repository, parses files with tree-sitter, runs ruff-based checks and a small local sentence-transformer model to surface semantic duplicates and hotspots, stores findings in SQLite, and outputs JSON + terminal summary; zero external dependencies except optional opt-in Supabase push for team analytics.

## Open questions
- Which subset of languages must v0.1 support (Python-only vs multi-language via tree-sitter)?
- Will embeddings be stored locally only or optionally pushed to Supabase for cross-repo insights?
- What is the acceptable local resource budget (RAM/CPU) for indexing large repos on developer machines?
- How will licensing/provenance of model weights and third-party rules be handled to keep product OSS-friendly?
- Do we ship a local TUI/WebView for results in v0.1 or remain pure CLI + JSON output?
