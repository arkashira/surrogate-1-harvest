## Deployment
cf workers (free-tier; host lightweight signaling/endpoint camouflage and WASM module) + client-side agent (Rust binary/WASM) — no servers to manage

## v0.1 scope
v0.1 ships in 2 weeks: local Rust CLI agent that creates a hidden SOCKS5 listener, applies TLS-mimicry obfuscation to upstream traffic via CF Workers endpoint, enforces a fail-closed kill-switch, stores state in local SQLite, and emits privacy-preserving WAU-ZB telemetry (no server infrastructure required).

## Open questions
- Which DPI signatures in our region are most likely to flag SOCKS5/TUN (port patterns, packet timing, TLS fingerprinting)?
- Can we rely on CF Workers free-tier limits for endpoint camouflage at scale, and what fallback is acceptable?
- What OS-level permissions model minimizes user friction while keeping TUN/SOCKS5 setup zero-config?
- How will we measure blocked sessions client-side without leaking metadata (WAU-ZB privacy budget)?
