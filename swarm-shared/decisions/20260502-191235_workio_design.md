# workio / design

## 1. Diagnosis

- **No tenant cache on LINE webhook** — every webhook performs `channelId → tenant` PostgreSQL lookup; under burst traffic this amplifies DB load, adds latency, and increases failure surface.
- **Missing health/readiness endpoints** — no `/health` or `/ready` for k8s/PM2/liveness probes; operators cannot safely restart or drain.
- **No graceful shutdown** — in-flight LINE webhook requests can be cut mid-processing on restart, risking partial clock-in/out records.
- **Hardcoded tenant resolution in webhook handler** — coupling makes future multi-channel or multi-region routing brittle.
- **No observability hooks** — missing request-id and timing logs for webhook path; slow tenant resolution is invisible.

## 2. Proposed change

- **Scope**: `workio/server/src/app.ts` (or equivalent main server file) and `workio/server/src/services/tenantService.ts` (create if absent).
- **Goal**: Add an in-memory LRU cache (TTL 5m) for `channelId → tenant` and expose `/health` / `/ready` endpoints; wrap startup/shutdown for graceful handling.
- **Why highest value**: Cuts DB load on the hottest path (LINE webhook) and enables safe platform operations with minimal code (<2h).

## 3. Implementation

### 3.1 Install lightweight LRU cache

```bash
cd /opt/axentx/workio/server
npm install lru-cache
```

### 3.2 Add tenant cache service

Create `server/src/services/tenantService.ts`:

```ts
import { LRUCache } from 'lru-cache';
import { getTenantByChannelId } from '../db/queries/tenantQueries'; // adjust import path
import type { Tenant } from '../types';

const tenantCache = new LRUCache<string, Tenant>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
});

export async function resolveTenantByChannelId(channelId: string): Promise<Tenant | null> {
  const cached = tenantCache.get(channelId);
  if (cached) return cached;

  const tenant = await getTenantByChannelId(channelId);
  if (tenant) tenantCache.set(channelId, tenant);
  return tenant || null;
}

export function clearTenantCache(channelId?: string) {
  if (channelId) tenantCache.delete(channelId);
  else tenantCache.clear();
}
```

### 3.3 Update LINE webhook handler

In your webhook route (e.g., `server/src/routes/lineWebhook.ts`), replace direct DB lookup:

Before:
```ts
import { getTenantByChannelId } from '../db/queries/tenantQueries';
// ...
const tenant = await getTenantByChannelId(channelId);
```

After:
```ts
import { resolveTenantByChannelId } from '../services/tenantService';
// ...
const tenant = await resolveTenantByChannelId(channelId);
```

### 3.4 Add health/readiness endpoints and graceful shutdown

Update `server/src/app.ts` (or main server file):

```ts
import express, { Request, Response } from 'express';
import http from 'http';
import { resolveTenantByChannelId } from './services/tenantService';
import lineWebhookRouter from './routes/lineWebhook';

const app = express();
app.use(express.json());

// Existing routes
app.use('/webhook/line', lineWebhookRouter);

// Health & readiness
let isShuttingDown = false;
app.get('/health', (req: Request, res: Response) => res.json({ status: 'ok' }));
app.get('/ready', async (req: Request, res: Response) => {
  // Quick tenant resolution smoke test (use a known test channelId via env or skip if none)
  try {
    // If you have a test channel, uncomment:
    // await resolveTenantByChannelId(process.env.TEST_CHANNEL_ID || '');
    res.json({ status: 'ready' });
  } catch (err) {
    res.status(503).json({ status: 'unready', error: String(err) });
  }
});

const server = http.createServer(app);

function gracefulShutdown(signal: string) {
  return async () => {
    if (isShuttingDown) return;
    isShuttingDown = true;
    console.log(`${signal} received: closing HTTP server`);

    server.close(async (err) => {
      if (err) {
        console.error('Error during shutdown', err);
        process.exit(1);
      }
      // Optional: close DB pool here if not handled elsewhere
      console.log('HTTP server closed');
      process.exit(0);
    });

    // Force exit after timeout
    setTimeout(() => {
      console.error('Forced shutdown after timeout');
      process.exit(1);
    }, 30_000).unref();
  };
}

process.on('SIGTERM', gracefulShutdown('SIGTERM'));
process.on('SIGINT', gracefulShutdown('SIGINT'));

const PORT = process.env.PORT || 3000;
server.listen(PORT, () => {
  console.log(`Server listening on port ${PORT}`);
});
```

### 3.5 Add request-id and timing logs (optional observability)

Add a tiny middleware in `app.ts` before routes:

```ts
import morgan from 'morgan'; // optional, or custom
import { v4 as uuidv4 } from 'uuid';

app.use((req, res, next) => {
  (req as any).requestId = uuidv4();
  res.setHeader('X-Request-Id', (req as any).requestId);
  next();
});

// Simple timing logger for webhook path
app.use((req, res, next) => {
  const start = Date.now();
  const path = req.path;
  res.on('finish', () => {
    if (path.includes('/webhook/line')) {
      console.log(`webhook ${req.method} ${path} ${res.statusCode} ${Date.now() - start}ms reqId=${(req as any).requestId}`);
    }
  });
  next();
});
```

## 4. Verification

1. **Cache behavior**
   - Start server and trigger two LINE events for the same `channelId` within 5m.
   - Check logs: first request logs DB query time; second should skip DB (cache hit). Confirm via log line absence or timing drop.

2. **Health/readiness**
   - `curl http://localhost:3000/health` → `{"status":"ok"}`
   - `curl http://localhost:3000/ready` → `{"status":"ready"}`
   - Simulate DB outage (or block tenant query) and confirm `/ready` returns 503.

3. **Graceful shutdown**
   - Start server, send an in-flight POST to `/webhook/line`, then `kill -SIGTERM <pid>`.
   - Confirm server waits for request to finish (check log for completion) before exiting.

4. **Tenant resolution correctness**
   - Verify existing LINE flows (clock-in/out) still work end-to-end with cached tenant.

5. **Load test (quick)**
   - Use `autocannon` or `ab` to send 50–100 rapid requests to `/webhook/line` with same channel payload; monitor DB connections — should remain stable and not spike per request.
