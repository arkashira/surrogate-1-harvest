# workio / quality

## Final Synthesized Solution

**Core diagnosis (merged):**
- Duplicate punches come from **both** client-side retries (network flake, double-taps, offline/background) **and** server-side retries (LINE webhook, load balancer retries).
- Missing **client-side idempotency keys** and **server-side idempotency enforcement** allow duplicates to be created and persisted.
- Missing **unique constraint** on `(employee_id, punch_date, punch_type)` allows duplicates to survive in the database and corrupt reports.
- UI optimistic updates are **not atomic** with request lifecycle and lack rollback on conflict, causing flicker and stale “pending” states.
- No **offline queue** means punches can be lost when connectivity drops mid-request or the app is backgrounded.

**Chosen approach (resolve contradictions in favor of correctness + concrete actionability):**
- Add **client-generated idempotency key** (UUID) to every punch request.
- Add **server-side idempotency check** (unique constraint + upsert or explicit check) keyed by idempotency key (and/or natural key with conflict detection).
- Add **unique database constraint** on `(employee_id, punch_date, punch_type)` to prevent logical duplicates; handle 409/400 conflicts by rolling back optimistic state and surfacing to user.
- Add **local pending queue** (indexedDB/localStorage) with atomic optimistic state and conflict rollback so punches survive offline/background and retries are safe.
- Make optimistic update **atomic with request lifecycle** and ensure rollback on server rejection (especially 409).
- Provide **flush/retry mechanism** for pending items when connectivity returns.

### 1. Database change (prevent logical duplicates)

Run on PostgreSQL:

```sql
-- Prevent duplicate punches for same employee/day/type
CREATE UNIQUE INDEX uniq_employee_punch_date_type
ON punches (employee_id, (punch_at::date), punch_type)
WHERE deleted_at IS NULL;
```

If using soft deletes, include `deleted_at` in the partial index as shown.

### 2. Server-side idempotency (route)

File: `workio/server/src/routes/punch.ts` (or equivalent)

```ts
import { FastifyInstance } from 'fastify';
import { db } from '../db';
import { punches } from '../db/schema';
import { eq, and } from 'drizzle-orm';

export async function punchRoutes(fastify: FastifyInstance) {
  fastify.post(
    '/',
    {
      schema: {
        body: {
          type: 'object',
          required: ['employeeId', 'type', 'idempotencyKey'],
          properties: {
            employeeId: { type: 'string' },
            type: { type: 'string', enum: ['in', 'out'] },
            idempotencyKey: { type: 'string' },
            ts: { type: 'number' },
          },
        },
      },
    },
    async (request, reply) => {
      const { employeeId, type, idempotencyKey, ts } = request.body as {
        employeeId: string;
        type: 'in' | 'out';
        idempotencyKey: string;
        ts?: number;
      };

      // 1) Idempotency lookup: if we already processed this key, return existing
      const existingByIdempotency = await db.query.punches.findFirst({
        where: eq(punches.idempotencyKey, idempotencyKey),
      });
      if (existingByIdempotency) {
        return reply.code(200).send(existingByIdempotency);
      }

      // 2) Natural-key conflict check (logical duplicate)
      const today = new Date();
      const startOfDay = new Date(today.setHours(0, 0, 0, 0));
      const endOfDay = new Date(today.setHours(23, 59, 59, 999));

      const conflicting = await db.query.punches.findFirst({
        where: and(
          eq(punches.employeeId, employeeId),
          eq(punches.punchType, type),
          // punchAt between startOfDay and endOfDay
          // (adjust for your schema; example assumes punchAt column)
          // Use raw SQL or helper if needed for date range
        ),
      });

      if (conflicting) {
        // Conflict: logical duplicate (already clocked in/out today)
        return reply.code(409).send({
          message: `Already clocked ${type} today`,
          code: 'DUPLICATE_PUNCH',
        });
      }

      // 3) Insert new punch with idempotency key
      const [created] = await db
        .insert(punches)
        .values({
          employeeId,
          punchType: type,
          punchAt: new Date(ts || Date.now()),
          idempotencyKey,
        })
        .returning();

      return created;
    }
  );
}
```

Notes:
- If your ORM/schema differs, adapt column names (`punchType`, `punchAt`, etc.).
- The unique index prevents race conditions between concurrent requests that pass the SELECT check; the insert will fail and you can catch and convert to 409 if desired.
