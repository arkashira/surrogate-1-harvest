# workio / backend

## Final Synthesized Solution

### 1. Diagnosis (merged)
- **Non-idempotent LINE webhook handler**: retries on 5xx/timeouts create duplicate punches and invalid open/close state.
- **No atomic punch transition**: concurrent/retried requests interleave, leaving `punches` rows without a matching close or with multiple opens.
- **Missing idempotency at handler boundary**: LINE’s exponential backoff resends; duplicate `X-Line-Delivery-Tag`/`webhookEventId` is not tracked.
- **Race condition in read-then-insert**: clock-in/out checks run outside a transaction, allowing two concurrent events to both believe they are first (or both close).
- **Missing tenant context enforcement on webhook path**: events can be accepted for any tenant if signature/channel validation is bypassed or misrouted.
- **No transactional outbox for replies**: partial failure can acknowledge webhook while user never receives confirmation.

### 2. Proposed Change (merged)
File: `/opt/axentx/workio/server/src/routes/webhook/line.ts` (and supporting files)

Scope:
- Add idempotency middleware using `X-Line-Delivery-Tag` (preferred) or `Idempotency-Key` with a `webhook_idempotency` table (unique key, response body, expires_at).
- Enforce tenant context: validate LINE channel signature and map to tenant before processing events.
- Refactor punch handler to run inside a single SQL transaction that:
  1. Locks the latest open punch for the user (`SELECT ... FOR UPDATE`).
  2. Validates rules (GPS, shift, daily limits) against current state.
  3. Inserts/updates exactly one row (clock-in or clock-out).
  4. Enqueues notification to transactional outbox (or sends reply via deferred job).
- Return early with cached 200 for duplicate idempotency keys.
- Return 5xx on transient failures so LINE retries (idempotency will dedupe).

#### Idempotency Middleware (corrected)
```ts
// /opt/axentx/workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';
import crypto from 'crypto';

export function lineIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // Prefer LINE delivery tag; fallback to Idempotency-Key
  const key = req.get('X-Line-Delivery-Tag') || req.get('Idempotency-Key');
  if (!key) return next();

  const handler = 'line-webhook';
  const requestHash = crypto
    .createHash('sha256')
    .update(JSON.stringify(req.body))
    .digest('hex');

  // Acquire client and begin transaction for idempotency check + handler work
  pool.connect((err, client, done) => {
    if (err) return next(err);

    client.query('BEGIN', async (err) => {
      if (err) {
        client.query('ROLLBACK', () => done());
        return next(err);
      }

      client.query(
        `SELECT response_body, status_code FROM webhook_idempotency
         WHERE idempotency_key = $1 AND handler = $2 AND request_hash = $3 AND expires_at > NOW()`,
        [key, handler, request_hash],
        (err, result) => {
          if (err) {
            client.query('ROLLBACK', () => done());
            return next(err);
          }

          if (result.rows[0]) {
            // Duplicate: replay cached response
            res.status(result.rows[0].status_code).json(result.rows[0].response_body);
            client.query('ROLLBACK', () => done());
            return;
          }

          // Attach client and metadata for handler
          res.locals.idempotencyKey = key;
          res.locals.idempotencyHandler = handler;
          res.locals.idempotencyRequestHash = request_hash;
          res.locals.idempotencyClient = client;
          res.locals.idempotencyDone = done;

          // Capture response to store after handler
          const originalJson = res.json.bind(res);
          res.json = (body: any) => {
            // Store idempotency record; best-effort but don't block response
            client.query(
              `INSERT INTO webhook_idempotency
                (idempotency_key, handler, request_hash, response_body, status_code, expires_at)
               VALUES ($1, $2, $3, $4, $5, NOW() + INTERVAL '24 hours')`,
              [key, handler, request_hash, body, res.statusCode],
              (err) => {
                if (err) console.error('Idempotency store failed', err);
              }
            );
            return originalJson(body);
          };

          next();
        }
      );
    });
  });
}

export function releaseIdempotencyClient(
  err: any,
  _req: Request,
  res: Response,
  next: NextFunction
) {
  const client = res.locals.idempotencyClient;
  const done = res.locals.idempotencyDone;
  if (!client || !done) return next(err);

  const finish = (rollback: boolean) => {
    const query = rollback ? 'ROLLBACK' : 'COMMIT';
    client.query(query, () => done());
  };

  if (err) {
    finish(true);
  } else {
    finish(false);
  }
  next(err);
}
```
