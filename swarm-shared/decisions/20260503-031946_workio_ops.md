# workio / ops

### 1. Diagnosis
* Missing **LINE webhook signature verification** (`X-Line-Signature`) on `/webhook/line` allows spoofed/replayed events to be accepted as valid clock-in/out or leave requests.
* No **idempotency/replay protection** — LINE retries on 5xx/timeouts can create duplicate `attendance_punches`, `leave_requests`, `ot_requests`.
* Inadequate **request deduplication** in the database layer, leading to potential duplicate records.
* Insufficient **error handling** for webhook requests, which can cause the system to fail unexpectedly.
* Lack of **logging** for webhook requests, making it difficult to diagnose issues.

### 2. Proposed change
The proposed change will focus on implementing LINE webhook signature verification and idempotency/replay protection in the `/webhook/line` endpoint. This will involve modifying the `server/src/controllers/WebhookController.js` file.

### 3. Implementation
To implement the proposed change, follow these steps:
```javascript
// server/src/controllers/WebhookController.js
const express = require('express');
const router = express.Router();
const crypto = require('crypto');
const db = require('../db');

// Define a function to verify the LINE webhook signature
function verifySignature(req, res, next) {
  const signature = req.header('X-Line-Signature');
  const body = req.body;
  const channelSecret = process.env.CHANNEL_SECRET;

  const hash = crypto.createHmac('sha256', channelSecret);
  hash.update(JSON.stringify(body));
  const expectedSignature = hash.digest('base64');

  if (signature !== expectedSignature) {
    return res.status(401).send('Invalid signature');
  }

  next();
}

// Define a function to handle idempotency/replay protection
function handleIdempotency(req, res, next) {
  const eventId = req.body.events[0].replyToken;
  const existingEvent = db.getEventByReplyToken(eventId);

  if (existingEvent) {
    return res.status(200).send('Event already processed');
  }

  next();
}

// Apply the middleware functions to the /webhook/line endpoint
router.post('/webhook/line', verifySignature, handleIdempotency, (req, res) => {
  // Process the webhook event
  const event = req.body.events[0];
  db.processEvent(event);

  res.status(200).send('Event processed successfully');
});
```
### 4. Verification
To verify that the implementation works, follow these steps:
1. Send a test webhook event to the `/webhook/line` endpoint with a valid signature.
2. Verify that the event is processed successfully and a 200 response is returned.
3. Send a test webhook event to the `/webhook/line` endpoint with an invalid signature.
4. Verify that a 401 response is returned with an error message indicating that the signature is invalid.
5. Send a test webhook event to the `/webhook/line` endpoint with a reply token that has already been processed.
6. Verify that a 200 response is returned with a message indicating that the event has already been processed.
