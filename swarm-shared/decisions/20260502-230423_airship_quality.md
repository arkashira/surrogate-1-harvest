# airship / quality

### Final Hardened Implementation (merged + reconciled)

**Highest-value incremental improvement**: Harden `airship discover` into a **fully deterministic, CDN-safe, zero-runtime** orchestrator that produces a CDN-only manifest so training never touches the HF API at load time.

Key reconciliations:
- Use **strict POSIX shebang + `set -euo pipefail`** (Candidate 2) — Candidate 1’s `env -S bash -euo pipefail` is less portable; explicit `#!/usr/bin/env bash` + `set -euo pipefail` is simpler and equally strict.
- **Pre-list once → JSON manifest → CDN-only training** (Candidate 2) is the core pattern; Candidate 1’s “embed file paths in training script” is satisfied by the manifest.
- **Do not use `set -r`** (Candidate 1) — it makes variables readonly and breaks re-runnability and function scopes; rely on `set -u` to catch unset usage instead.
- **Make script executable + explicit `bash` invocation** (Candidate 2) for cron/daemon safety; Candidate 1’s “file execution read-only” intent is better served by `chmod +x` and correct ownership.
- **Add cron/daemon wrapper enforcing `SHELL=/bin/bash`** (Candidate 2) to guarantee environment determinism.
- **Zero-runtime dependencies**: use a minimal venv with `huggingface_hub` only for the lister; training uses CDN + `requests`/`pyarrow` (no HF client at train time).

#### A) Hardened orchestrator (deterministic, CDN-safe, zero-runtime)

`/opt/axentx/airship/arkship/orchestration/airship-discover`
```bash
#!/usr/bin/env bash
# airship-discover — deterministic HF file lister + CDN manifest builder
# Usage: bash airship-discover <dataset_repo> <date_folder> <output_manifest.json>
set -euo pipefail
IFS=$'\n\t'

# Validate env
if [[ -z "${HF_TOKEN:-}" ]]; then
  echo "ERROR: HF_TOKEN required for list_repo_tree (not for CDN downloads)" >&2
  exit 1
fi

REPO="${1:-}"
DATE_FOLDER="${2:-}"
OUT_MANIFEST="${3:-}"

if [[ -z "$REPO" || -z "$DATE_FOLDER" || -z "$OUT_MANIFEST" ]]; then
  echo "Usage: $0 <dataset_repo> <date_folder> <output_manifest.json>" >&2
  exit 1
fi

# Ensure Python venv is available (minimal deps: huggingface_hub)
VENV="${VENV:-./.venv}"
if [[ -d "$VENV" ]]; then
  # shellcheck source=/dev/null
  source "$VENV/bin/activate"
fi

# Single API call: list one folder (non-recursive) to avoid pagination/rate-limits
python3 - "$REPO" "$DATE_FOLDER" "$OUT_MANIFEST" <<'PY'
import json
import os
import sys
from huggingface_hub import HfApi

def main():
    repo = sys.argv[1]          # e.g. "datasets/myorg/surrogate-data"
    folder = sys.argv[2]        # e.g. "batches/mirror-merged/2026-05-02"
    out_path = sys.argv[3]

    api = HfApi(token=os.environ["HF_TOKEN"])
    # recursive=False keeps this one call small and deterministic
    entries = api.list_repo_tree(repo=repo, path=folder, recursive=False)

    files = []
    for e in entries:
        # Normalize entry access across huggingface_hub versions
        if hasattr(e, "type") and getattr(e, "type") == "file" or (hasattr(e, "path") and e.path):
            path = e.path if hasattr(e, "path") else e
            # CDN URL: no Authorization header required, bypasses /api/ rate limits
            cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{path}"
            files.append({"path": path, "url": cdn_url})

    os.makedirs(os.path.dirname(out_path) if os.path.dirname(out_path) else ".", exist_ok=True)
    with open(out_path, "w", encoding="utf-8") as f:
        json.dump({"repo": repo, "folder": folder, "generated_by": "airship-discover", "files": files}, f, indent=2)

    print(f"Wrote {len(files)} files to {out_path}")

if __name__ == "__main__":
    main()
PY

echo "airship-discover completed: $OUT_MANIFEST"
```

Make executable:
```bash
chmod +x /opt/axentx/airship/arkship/orchestration/airship-discover
```

#### B) Training/data loader: consume manifest and stream Parquet via CDN (zero HF API at train time)

`/opt/axentx/airship/surrogate/train.py` (or data module)
```python
import json
import os
from pathlib import Path
from typing import List, Dict

import pyarrow as pa
import pyarrow.parquet as pq
import requests
from torch.utils.data import IterableDataset

class CDNParquetDataset(IterableDataset):
    """
    Loads Parquet files via CDN URLs listed in a manifest produced by airship-discover.
    Avoids HuggingFace API calls during training (no rate limits).
    Projects only {prompt,response} at parse time (surrogate-1 pattern).
    """

    def __init__(self, manifest_path: str, columns=("prompt", "response")):
        super().__init__()
        self.manifest_path = manifest_path
        self.columns = columns
        self.file_urls: List[str] = []
        self._load_manifest()

    def _load_manifest(self) -> None:
        with open(self.manifest_path, "r", encoding="utf-8") as f:
            manifest = json.load(f)
        self.file_urls = [f["url"] for f in manifest.get("files", []) if f["url"].endswith(".parquet")]
        if not self.file_urls:
            raise ValueError(f"No parquet files found in manifest: {self.manifest_path}")

    def _stream_parquet_via_cdn(self, url: str):
        # CDN download (no auth) -> local buffer -> parquet read -> project columns
        resp = requests.get(url, timeout=60)
        resp.raise_for_status()
        data = resp.content
        table = pq.read_table(pa.BufferReader(data), columns=self.columns)
        # Convert to list of dicts for simple iteration (streaming-friendly for small shards)
        for batch in table.to_batches(max_chunksize=1024):
            df = batch.to_pandas()
            for _, row in df.iterrows():
                yield {k: row.get(k) for k in self.columns}

    def __iter__(self):
        for url in self.file_urls:
            yield from self._stream_parquet_via_cdn(url)

# Example usage in Lightning DataModule (minimal)
def make_dataloader(manifest_path: str, batch_size: int = 8):
    from torch.utils.data import DataLoader
    dataset = CDNParquetDataset(manifest_path=manifest_path)
    return DataLoader(dataset, batch_size=batch_size, num_workers=0)
```

Notes:
- Training is **CDN-only** after a single `airship-discover` run.
- Projects only `{prompt,response}` at parse time (avoids mixed-schema pyarrow issues).
- If you use Lightning Studio, reuse running studios and check status before `.run()` to avoid quota waste.
