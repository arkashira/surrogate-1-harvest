## Deployment
cf workers (Cloudflare Workers for meek front/reflector; free-tier; no origin infra) + client binaries served as static assets (GitHub Pages or HF space for optional UI)

## v0.1 scope
v0.1 ships in 2 weeks: a single Rust CLI agent (Linux x86_64) with meek-TLS front via Cloudflare Workers, local SQLite encrypted state, fail-closed kill-switch that blocks traffic if stealth handshake fails, and minimal client-side WAU-ZB opt-in telemetry; no server infra beyond CF Workers (free-tier).

## Open questions
- Which pluggable transports (meek, domain-front, obfs4-like) are safest and most reachable from target restrictive regions?
- How will we distribute signed policy/endpoint updates without relying on blocked channels?
- What local telemetry schema and opt-in mechanism balances WAU-ZB measurement with user privacy?
- How to implement reliable TUN setup across Windows/macOS/Linux without elevated privileges where possible?
- What kill-switch enforcement granularity is required (per-app, system-wide, best-effort userspace)?
