The goal is to eliminate HF API 429s and Lightning quota waste during Surrogate training by implementing CDN-first deterministic ingestion with embedded file lists and Lightning Studio reuse.

### Key Components:

1. **CDN-first deterministic ingestion**: This involves using a Content Delivery Network (CDN) to access data directly, bypassing API rate limits. The ingestion process is made deterministic by using embedded file lists, ensuring reproducibility.

2. **Embedded file lists**: Generating a list of files to be ingested beforehand. This list is then embedded into the surrogate training package or passed via an environment variable, ensuring that the training process knows exactly which files to expect and process.

3. **Lightning Studio reuse**: Implementing a mechanism to reuse existing Lightning Studios instead of creating new ones for each training run. This saves Lightning quota and reduces the overhead of studio creation.

### Implementation Steps:

1. **Add file-list pre-generation script**: Develop a script (`build_filelist.py`) that runs on a Mac after the rate-limit window clears. It uses the Hugging Face API to list repository tree items per date folder, generates a JSON file (`filelist-{date}.json`) containing repository, path, and SHA information for each file, and embeds this JSON into the surrogate training package or passes it via an environment variable.

2. **Update data loader**: Modify the data loader (`hf_cdn_loader.py`) to read the generated `filelist-*.json`, download files directly from the CDN, parse only the necessary fields (`prompt` and `response`), and yield tokenized samples. This approach avoids heterogeneous schema issues and bypasses API rate limits.

3. **Lightning Studio reuse wrapper**: Create a wrapper script (`run_lightning_studio.py`) that checks for existing studios with the target name. If a running studio exists, it reuses it; if a stopped studio exists, it restarts it. Otherwise, it creates a new studio. This ensures efficient use of Lightning quota.

4. **Cron/ops hygiene**: Ensure all wrapper scripts are executable, invoked correctly, and have the appropriate shebang (`#!/usr/bin/env bash`). Set the shell in crontab to `/bin/bash` for cron jobs to maintain consistency.
