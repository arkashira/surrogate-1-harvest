# Final Synthesis — Manifest-driven CDN-bypass ingestion worker

## Core design (merged + corrected)
- Replace `bin/dataset-enrich.sh` with **`bin/dataset-enrich.py`** (manifest-driven, CDN-bypass ingestion worker).
- Accept `SHARD_ID`, `SHARD_TOTAL=16`, `DATE`, `HF_TOKEN`, `REPO_ID` from environment/GitHub Actions matrix.
- Use a pre-generated `file-list.json` (created once on Mac after rate-limit window) to avoid recursive `list_repo_files` and 429s.
- Deterministic sharding: `hash(slug) % SHARD_TOTAL` where `slug` is the repo-relative path.
- Download via HF CDN (`resolve/main/...`) with **no Authorization header** during training data load (bypasses API rate limits).
- Parse-time schema projection: extract only `{prompt, response}` from heterogeneous files; ignore extra columns to avoid `pyarrow.CastError`.
- Deduplicate via central md5 store (`lib/dedup.py`) keyed by canonicalized pair hash.
- Deterministic, collision-free outputs:  
  `batches/public-merged/<YYYY-MM-DD>/shard<N>-<HHMMSS>.jsonl`
- Reuse running Lightning Studio when present; check status before `.run()` and restart stopped machines with `target.start(machine=Machine.L40S)` to save quota.
- Respect HF dataset commit cap (128/hr) via deterministic filenames and idempotent uploads with retries.

## Implementation plan (≤2h)

1. **Create `bin/file-list-save.py`** (Mac-only orchestration)  
   - Recursively list repo tree once with `HfApi.list_repo_tree(repo_id, path="", recursive=True)` (single call).  
   - Save `file-list.json` mapping `date -> [paths]` (date = top-level folder).  
   - Run once after rate-limit window clears; commit or upload as workflow artifact.

2. **Replace `bin/dataset-enrich.sh` with `bin/dataset-enrich.py`**  
   - Load `file-list.json`; filter by `DATE`; assign files to shard by `hash(slug) % SHARD_TOTAL`.  
   - For each assigned file:  
     - Download via `https://huggingface.co/datasets/{repo}/resolve/main/{path}` (no auth).  
     - Parse by extension (jsonl/parquet/text) and project to `{prompt, response}`; coerce to string; skip malformed lines.  
     - Compute md5 of canonicalized `prompt+response`; use `lib/dedup.py` to skip duplicates.  
   - Write non-duplicate lines to `batches/public-merged/{date}/shard{SHARD_ID}-{ts}.jsonl`.  
   - Upload to HF dataset repo via `huggingface_hub` with exponential backoff; use deterministic filenames to avoid collisions and stay within 128/hr commit cap.

3. **Update `.github/workflows/ingest.yml`**  
   - Add matrix strategy for 16 shards (`SHARD_ID: 0..15`).  
   - Pass `DATE` input (default today).  
   - Use `actions/download-artifact` to fetch `file-list.json` if not present.  
   - Ensure scripts run with `bash` and fail fast on parse/download errors.

4. **Add Lightning Studio reuse helper (`bin/reuse-studio.py`)**  
   - List Teamspace studios; if a running studio with expected name exists, reuse it.  
   - Before `.run()`, check status; if stopped, restart with `target.start(machine=Machine.L40S)`.

5. **Validate and test locally**  
   - Dry-run with 1 shard on a small date folder.  
   - Verify CDN downloads work without token.  
   - Confirm output schema (`prompt`, `response` strings) and dedup behavior.

## Key corrections (favor correctness + actionability)
- Use **recursive tree listing once** for `file-list-save.py` (not per-folder non-recursive) to guarantee completeness and avoid pagination bugs.
- Keep CDN URL pattern exactly `resolve/main/{path}`; do not inject Authorization headers during training data load.
- Handle parquet robustly: read from `BytesIO`, coerce columns to string, and skip rows where `prompt` or `response` is null/empty after coercion.
- Central dedup store must be thread-safe and process-safe; use SQLite with `PRAGMA journal_mode=WAL` and retries for concurrent workers.
- Deterministic shard assignment must use a stable slug (relative path) and hash; do not rely on filesystem ordering.

### bin/file-list-save.py
```python
#!/usr/bin/env python3
"""
Generate file-list.json for a dataset repo.
Run once after HF rate-limit window clears.
"""
import json
import os
from huggingface_hub import HfApi

REPO_ID = os.getenv("REPO_ID", "axentx/surrogate-1-training-pairs")
OUT_PATH = "file-list.json"

api = HfApi()
# Single recursive call to avoid pagination bugs
tree = api.list_repo_tree(repo_id=REPO_ID, path="", recursive=True)

result = {}
for item in tree:
    if item.type != "file":
        continue
    # Expect top-level folder to be date (YYYY-MM-DD)
    parts = item.rfilename.split("/")
    if len(parts) < 2:
        continue
    date_key = parts[0].strip("/")
    result.setdefault(date_key, []).append(item.rfilename)

with open(OUT_PATH, "w") as f:
    json.dump(result, f, indent=2)

print(f"Saved {len(result)} date folders to {OUT_PATH}")
```

### lib/dedup.py
```python
import sqlite3
import os
import threading

DB_PATH = os.getenv("DEDUP_DB_PATH", "/tmp/dedup_store.db")
_local = threading.local()

def _get_conn():
    if not hasattr(_local, "conn"):
        _local.conn = sqlite3.connect(DB_PATH, check_same_thread=False, timeout=30)
        _local.conn.execute("PRAGMA journal_mode=WAL")
        _local.conn.execute(
            "CREATE TABLE IF NOT EXISTS seen (md5 TEXT PRIMARY KEY)"
        )
        _local.conn.commit()
    return _local.conn

def is_duplicate(md5_hex: str) -> bool:
    conn = _get_conn()
    cur = conn.execute("SELECT 1 FROM seen WHERE md5=?", (md5_hex,))
    return cur.fetchone() is not None

def add(md5_hex: str) -> bool:
    """
    Returns True if duplicate (already present), False if added.
    """
    conn = _get_conn()
    try:
        conn.execute("INSERT INTO seen (md5) VALUES (?)", (md5_hex,))
        conn.commit()
        return False
    except sqlite3.IntegrityError:
        return True
```
