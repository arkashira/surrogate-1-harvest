# vanguard / design

**Most Valuable Next Improvement: Persistent File-List Cache**

### Diagnosis

* No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` → HF API 429 (1000 req/5min) errors.
* Lightning Studio lifecycle not reused → quota burn on recreation.
* Training data inconsistency → training instability.

### Proposed Change

* Implement a persistent file-list cache to store the list of files for each repository.
* Store the cache in a local JSON file or a database (e.g., SQLite).

### Implementation

1. Create a new file `cache.py` in the project root:
```python
import json
import os

CACHE_DIR = '.cache'

def get_cache_file(repo_name):
    return os.path.join(CACHE_DIR, f'{repo_name}.json')

def load_cache(repo_name):
    cache_file = get_cache_file(repo_name)
    if os.path.exists(cache_file):
        with open(cache_file, 'r') as f:
            return json.load(f)
    return {}

def save_cache(repo_name, cache):
    cache_file = get_cache_file(repo_name)
    with open(cache_file, 'w') as f:
        json.dump(cache, f)
```
2. Update `train.py` to use the cache:
```python
import cache

repo_name = 'my-repo'
cache_data = cache.load_cache(repo_name)
if cache_data:
    # Load data from cache
    data = cache_data
else:
    # Re-list repo files and load data
    data = load_dataset(streaming=True)
    cache.save_cache(repo_name, data)
```
3. Update `lightning_studio.py` to reuse the cached data:
```python
import cache

repo_name = 'my-repo'
cache_data = cache.load_cache(repo_name)
if cache_data:
    # Use cached data
    data = cache_data
else:
    # Re-list repo files and load data
    data = load_dataset(streaming=True)
    cache.save_cache(repo_name, data)
```
4. Update `lightning_studio.py` to reuse the existing run instead of recreating it:
```python
import cache

# ...

studio = Teamspace.studios.get(name='my_studio')
if studio.status == 'Running':
    # Reuse the existing run
    studio.start(machine=Machine.L40S)
else:
    # Create a new run
    studio.create_ok=True
```
### Verification

* Run `train.py` and verify that the cache is loaded correctly.
* Run `lightning_studio.py` and verify that the cached data is used correctly.
* Monitor the HF API 429 errors and verify that they are reduced or eliminated.

**Key Changes**

* The cache is stored in a local JSON file instead of a database.
* The cache is loaded and saved using the `cache.py` module.
* The `train.py` and `lightning_studio.py` scripts are updated to use the cache.
* The `lightning_studio.py` script is updated to reuse the existing run instead of recreating it.

**Benefits**

* Reduced HF API 429 errors.
* Improved training stability.
* Faster training times.
* Reduced quota burn on recreation.
