# Costinel / design

## Highest-value incremental improvement (≤2h)

**Ship a “Top-Hub Signal” card on the dashboard** that:

1. Reads a local knowledge-rag graph file (`/data/knowledge-rag/top-hubs.json`) at build/runtime (frontend-only).
2. Picks the most-connected hub (e.g., `"MOC"`) and renders an actionable signal card with:
   - Hub name + connection count
   - Top 3 related docs (title + short context)
   - One-line recommendation (“Sense + Signal”)
   - Audit trail metadata (source file + timestamp)
3. Graceful fallback to a minimal placeholder when the file is missing or malformed.
4. No new APIs, no auth, no backend — fits Costinel’s “Sense + Signal — ไม่ Execute” philosophy.

This directly applies the **top-hub doc insight** and **knowledge-rag/graph** patterns, adds immediate user value, and can ship in <2h.

## Implementation plan

### 1) Add local knowledge-rag graph file (seed)
Create `/opt/axentx/Costinel/data/knowledge-rag/top-hubs.json` with a minimal canonical shape:

```json
{

  "hubs": [
    {
      "id": "MOC",
      "label": "MOC",
      "connections": 42,
```

### 2) Create frontend component: `TopHubSignalCard`
Place in `src/components/TopHubSignalCard.jsx` (or `.tsx` if project uses TypeScript).

```tsx
import React, { useEffect, useState } from "react";

type Doc = {
  title: string;
  context: string;
};

type Hub = {
  id: string;
  label: string;
  connections: number;
  docs: Doc[];
  recommendation: string;
};

type KnowledgeRag = {
  generatedAt?: string;
  hubs: Hub[];
};

const TopHubSignalCard: React.FC = () => {
  const [hub, setHub] = useState<Hub | null>(null);
  const [generatedAt, setGeneratedAt] = useState<string | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // Frontend-only fetch from local static file (placed under public/ or served by app)
    // We'll request /data/knowledge-rag/top-hubs.json (ensure it's copied to build output or served statically)
    fetch("/data/knowledge-rag/top-hubs.json", { cache: "no-store" })
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then((data: KnowledgeRag) => {
        if (!Array.isArray(data?.hubs) || data.hubs.length === 0) {
          throw new Error("No hubs found");
        }
        // Pick most-connected hub
        const top = data.hubs.reduce((a, b) =>
          (a.connections || 0) > (b.connections || 0) ? a : b
        );
        setHub(top);
        setGeneratedAt(data.generatedAt || null);
        setLoading(false);
      })
      .catch((err) => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) {
    return (
      <div className="rounded-lg border bg-white p-4 shadow-sm animate-pulse">
        <div className="h-5 w-32 bg-gray-200 rounded mb-2"></div>
        <div className="h-4 w-full bg-gray-100 rounded mb-1"></div>
        <div className="h-4 w-5/6 bg-gray-100 rounded"></div>
      </div>
    );
  }

  if (error || !hub) {
    return (
      <div className="rounded-lg border bg-white p-4 shadow-sm">
        <p className="text-sm text-gray-500">
          Signal unavailable — knowledge graph not loaded.
        </p>
      </div>
    );
  }

  return (
    <div className="rounded-lg border bg-white p-4 shadow-sm">
      <div className="flex items-start justify-between mb-2">
        <div>
          <h3 className="text-sm font-semibold text-gray-900">Top Hub</h3>
          <p className="text-lg font-medium text-blue-600">{hub.label}</p>
        </div>
        <span className="inline-flex items-center rounded-full bg-blue-50 px-2 py-1 text-xs font-medium text-blue-700">
          {hub.connections} links
        </span>
      </div>

      <p className="text-sm text-gray-600 mb-3">{hub.recommendation}</p>

      <div className="space-y-2 mb-3">
        {hub.docs.slice(0, 3).map((doc, idx) => (
          <div key={idx} className="text-xs text-gray-700">
            <span className="font-medium text-gray-900">{doc.title}</span>
            <p className="text-gray-500">{doc.context}</p>
          </div>
        ))}
      </div>

      <div className="flex items-center justify-between text-xs text-gray-400 pt-2 border-t border-gray-100">
        <span>Sense + Signal — ไม่ Execute</span>
        {generatedAt && (
          <span>Updated {new Date(generatedAt).toLocaleDateString()}</span>
        )}
      </div>
    </div>
  );
};

export default TopHubSignalCard;
```

### 3) Mount the card on the dashboard
Insert into the main dashboard layout (e.g., `src/pages/Dashboard.jsx` or equivalent) near the top of the sidebar or summary row.

```tsx

// Inside dashboard render:
<div className="grid grid-cols-1 lg:grid-cols-3 gap-4 mb-6">
  <div className="lg:col-span-2">
    {/* existing cost overview */}
  </div>
  <div>
    <TopHubSignalCard />
  </div>
</div>
```

### 4) Ensure static file is served
Place the JSON at `public/data/knowledge-rag/top-hubs.json` (or configure the build to copy `data/knowledge-rag/top-hubs.json` into the static output). For a quick test during dev, put it in `public/`.

### 5) Optional: add build-time embedding (more robust)
If the app is built and served statically, embed the JSON at build time:

```js
// In your build script or vite/webpack plugin:
import fs from "fs";
const rag = JSON.parse(fs.readFileSync("data/knowledge-rag/top-hubs.json", "utf8"));
// Inject into index.html as window.__RAG_DATA__ or import as module.
```

Then update `TopHubSignalCard` to read from `window.__RAG_DATA__` first and skip the fetch.
