## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research fetch -> signal generate -> risk check -> execution send) with immutable event journal, Postgres snapshotting, Redis Streams replay, compliance guardrails (allow/deny), and edge-hosted API on CF Workers; includes replay CLI and basic audit UI (server-rendered HTML) for runs and evidence export.

## Open questions
- Deterministic replay boundaries: which inputs must be snapshotted to guarantee bitwise replay across vendor API changes?
- Compliance expression language: safe subset (e.g., JSONLogic) vs. WASM sandbox for rule evaluation?
- Execution idempotency guarantees: how to enforce exactly-once semantics across brokers that offer at-least-once?
- Latency SLOs: sub-100ms or sub-200ms end-to-end, and which segments (risk, signal, execution) get hard budgets?
- Secrets management on free-tier: use Supabase Vault-like encryption or Cloudflare Workers Secrets?
- Audit retention and immutability: WORM storage strategy and retention period on free-tier options?
