## v0.1 scope
v0.1 ships in 2 weeks: a Cloudflare Workers gateway exposing a single /v1/chat/completions endpoint that normalizes OpenAI-compatible requests to OpenAI and Anthropic (text-only), enforces per-provider RPS and monthly token quotas backed by Redis counters, records TokenLedger entries in Supabase Postgres, and returns gateway-level p95 latency metrics; includes contract tests, CI on GitHub Actions (free tier), and one-click deploy via Wrangler to CF Workers free plan.

## Open questions
- Which providers and models must be supported at v0.1 to satisfy early adopters?
- Where will secrets reside (CF Workers KV + Supabase Vault vs. provider-managed) and how will rotation work?
- What is the p95 latency SLA target (ms) and how will it be measured (gateway e2e vs. upstream TTFB)?
- Will token counting be best-effort in Workers KV or strongly consistent in Postgres/Redis given free-tier constraints?
- How will global rate limits and per-provider quotas be enforced without introducing blocking calls?
