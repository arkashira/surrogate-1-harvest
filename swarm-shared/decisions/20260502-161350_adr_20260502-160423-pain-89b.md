## Deployment
cf workers (edge bridge + domain-front helper) + hf space (meek fallback) + client-side Rust/WASM agent; SQLite local; Supabase for encrypted telemetry spool

## v0.1 scope
v0.1 ships a local Rust CLI agent (Linux x86_64) with: 1) SOCKS5-to-meek-TLS shim that hides local port/TUN artifacts, 2) fail-closed kill-switch that blocks non-stealth traffic if stealth path fails, 3) SQLite-backed session log and local WAU-ZB counters, 4) CF Worker bridge (single front domain) + HF Space fallback, 5) encrypted telemetry spool to Supabase; no server control or multi-hop, client-side-only install (~2-week build).

## Open questions
- Which DPI fingerprints are highest risk in target regions and which mimic (TLS/QUIC/HTTP) is safest to emulate?
- Can CF Workers reliably act as a meek/tls bridge without triggering CF WAF blocks for our user base?
- What is the acceptable latency budget for obfuscation layers while maintaining usability?
- How will we distribute and rotate PSKs/front-domains to users without a central server?
- What kill-switch granularity is required (per-app vs system-wide) to prevent leaks on desktop OSes?
