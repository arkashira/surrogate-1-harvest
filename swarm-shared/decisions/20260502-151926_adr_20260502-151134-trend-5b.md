## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres) + Hugging Face Spaces (Redis-backed worker for cache/queue if needed)

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research snapshot → signal generator → risk check → simulated execution gateway) with immutable Postgres audit trail, Redis-backed signal cache, replay endpoint (re-run by inputs_hash), and basic compliance guardrails (max position size, venue allowlist, kill-switch on fail). No live trading; simulated execution only. Deploy via Cloudflare Workers + Supabase free-tier + HF Space Redis.

## Open questions
- Deterministic replay strategy: event-sourcing snapshots vs full input capture — storage cost vs replay latency trade-off?
- Exactly-once vs at-least-once execution semantics with brokers/OMS; idempotency key design and dedupe store?
- Compliance retention and redaction requirements (e.g., MiFID II/SEC) — immutable retention period and purge rules?
- Latency SLA target for research→execution (sub-100ms, sub-500ms?) and acceptable queueing/backpressure strategy?
- Connector authentication model: per-customer secrets vault vs centralized managed tokens; rotation cadence?
- Fee attribution model for NSM weighting: how to reliably map workflow_run to fee-generating status across venues?
