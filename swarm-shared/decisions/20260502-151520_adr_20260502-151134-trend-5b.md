## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic pipeline (research→signal→risk→execution) for one instrument, one broker connector (ccxt sandbox), Postgres+Redis on Supabase/CF free-tier, edge worker on CF handling API and queue dispatch, p95 signal-to-execution latency <500ms, immutable audit events in Postgres + optional R2 export, basic replay endpoint for a run, and dashboard-less operation via API/logs.

## Open questions
- Which brokers/data vendors must be supported in v0.1 and what auth flows do they require?
- What deterministic ordering and exactly-once semantics are required for Redis Streams vs Postgres WAL?
- Where will immutable audit logs be stored (CF R2/Workers KV vs Supabase storage) and retention policy?
- What is the acceptable consistency model for hot risk state (strong vs eventual) at sub-500ms p99?
- How will replay/forensics be triggered and scoped (time range, strategy, instrument)?
- Which latency budget slices (signal gen, risk, queue, execution) must be measured and enforced in CI?
