## Deployment
cf workers

## v0.1 scope
v0.1 ships a single cross-platform CLI client (Linux x86_64, Windows, macOS) that: establishes an obfuscated TCP tunnel to a CF Worker edge bridge (mimicking innocuous HTTPS), spawns a userspace TUN via boringtun, routes selected traffic through the tunnel, enforces a process-level kill-switch on disconnect, stores state in local SQLite, emits minimal opt-in WAU-ZB telemetry via CF KV, and provides zero-config presets for common restrictive-region scenarios.

## Open questions
- Which obfuscation front (meek vs. shadowsocks-style vs. TLS-in-TLS) best survives active DPI probes in target regions?
- Can we reliably hide TUN interface names/routes from host inspection without root/admin on each OS?
- How will WAU-ZB telemetry be collected with opt-in privacy and minimal metadata to avoid deanonymization?
- What is the fail-closed kill-switch behavior on each platform (Windows/WFP, Linux/nftables, macOS/pf) without privileged setup?
- How do we bootstrap bridge discovery and rotation without centralized infra while resisting blocking?
