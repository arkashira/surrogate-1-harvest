# workio / design

Below is the consolidated, production-ready plan.  
It merges the strongest, non-overlapping insights from both proposals, removes contradictions, and prioritizes correctness + immediate actionability.

---

## 1. Diagnosis (merged + prioritized)

- **No tenant cache on LINE webhook**  
  Every webhook performs `channelId → tenant` PostgreSQL lookup. Under burst traffic this amplifies DB load, adds latency, and increases failure surface.

- **Missing health/readiness endpoints**  
  No `/health` (liveness) or `/ready` (readiness) for Kubernetes/PM2 lifecycle checks and safe deployments.

- **No graceful shutdown**  
  In-flight LINE webhook requests can be dropped during restart/deploy, risking missed clock-in/out events.

- **Inconsistent or missing LINE signature verification**  
  Security posture varies across webhook handlers; some routes may skip verification.

- **No request-id / trace correlation**  
  Hard to debug webhook flows across logs and services.

- **Inefficient DB queries / observability gaps**  
  Potential for slow queries, missing indexes, and insufficient structured logging/error handling.

---

## 2. Final proposed change (scope + strategy)

**Scope**:  
`workio/server/src/app.ts` (main Express entry)  
`workio/server/src/middleware/tenantCache.ts`  
`workio/server/src/middleware/requestId.ts`  
`workio/server/src/controllers/health.ts`  
`workio/server/src/index.ts` (server bootstrap)  
`workio/server/src/middleware/lineSignature.ts` (new)  
`workio/server/src/services/tenantService.ts` (existing or new)

**Strategy**:

1. **Add an in-memory LRU tenant cache** keyed by `channelId` with TTL (5 min) and `updateAgeOnGet`.  
   - Start in-process to remove Redis operational dependency for MVP; add Redis later if multi-instance invalidation is required.
   - Cache-aside pattern: miss → DB → set; include negative caching for short TTL on missing tenants to protect DB.

2. **Add `/health` and `/ready` endpoints**  
   - Liveness: process is alive.  
   - Readiness: DB reachable and critical dependencies OK.

3. **Add graceful shutdown**  
   - On SIGTERM/SIGINT: stop accepting new connections, allow in-flight requests to finish (with timeout), then exit.

4. **Add request-id middleware**  
   - Prefer `X-Request-Id` header if present; otherwise generate UUID.  
   - Attach to logs and response headers for traceability.

5. **Enforce LINE signature verification**  
   - Central middleware for LINE webhooks; reject invalid signatures early (401).

6. **Harden error handling + logging**  
   - Structured logs with request context.  
   - Catch unhandled rejections/errors at process level.

## 3. Implementation (concrete, ready to apply)

### 3.1 Tenant cache middleware

`workio/server/src/middleware/tenantCache.ts`
```ts
import { Request, Response, NextFunction } from 'express';
import { LRUCache } from 'lru-cache';
import { getTenantByChannelId } from '../services/tenantService';
import logger from '../utils/logger';

type Tenant = {
  id: string;
  name: string;
  config: Record<string, any>;
};

const tenantCache = new LRUCache<string, Tenant>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
  allowStale: false,
  updateAgeOnGet: true,
});

export async function resolveTenant(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // Accept channelId from body or header; align with actual LINE payload
  const channelId = req.body?.channelId || req.get('x-line-channel-id');
  if (!channelId) {
    logger.warn({ event: 'missing-channel-id' }, 'No channelId for tenant resolution');
    return res.status(400).json({ error: 'missing-channel-id' });
  }

  const cached = tenantCache.get(channelId);
  if (cached) {
    res.locals.tenant = cached;
    return next();
  }

  try {
    const tenant = await getTenantByChannelId(channelId);
    if (!tenant) {
      // Negative cache to avoid repeated DB misses (short TTL)
      tenantCache.set(channelId, null as any, { ttl: 10_000 });
      logger.warn({ channelId }, 'Tenant not found for channel');
      return res.status(404).json({ error: 'tenant-not-found' });
    }

    tenantCache.set(channelId, tenant);
    res.locals.tenant = tenant;
    next();
  } catch (err) {
    logger.error({ err, channelId }, 'Failed to resolve tenant');
    return res.status(500).json({ error: 'tenant-resolution-failed' });
  }
}
```

### 3.2 Health endpoints

`workio/server/src/controllers/health.ts`
```ts
import { Request, Response } from 'express';
import db from '../db';

export function healthz(req: Request, res: Response) {
  // Liveness: process is alive
  res.json({ status: 'ok', uptime: process.uptime() });
}

export async function readyz(req: Request, res: Response) {
  // Readiness: dependencies available
  try {
    await db.query('SELECT 1');
    res.json({ status: 'ready', db: 'ok' });
  } catch (err) {
    res.status(503).json({ status: 'not_ready', db: 'error' });
  }
}
```

### 3.3 Request-id middleware

`workio/server/src/middleware/requestId.ts`
```ts
import { Request, Response, NextFunction } from 'express';
import { randomUUID } from 'crypto';
import logger from '../utils/logger';

declare global {
  namespace Express {
    interface Request {
      id?: string;
    }
  }
}

export function requestId(req: Request, res: Response, next: NextFunction) {
  const id = req.headers['x-request-id'] as string || randomUUID();
  req.id = id;
  res.setHeader('X-Request-Id', id);

  // Attach a child logger with request context if your logger supports it
  (req as any).log = logger.child?.({ requestId: id }) || logger;
  next();
}
```

### 3.4 LINE signature verification middleware

`workio/server/src/middleware/lineSignature.ts`
```ts
import { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';
import logger from '../utils/logger';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

export function verifyLineSignature(req: Request, res: Response, next: NextFunction) {
  if (!LINE_CHANNEL_SECRET) {
    logger.warn('LINE_CHANNEL_SECRET is not configured; skipping signature verification');
    return next();
  }

  const signature = req.get('X-Line-Signature');
  if (!signature) {
    return res.status(401).json({ error: 'missing-signature' });
  }

  const body = JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(body)
    .digest('base64');

  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected))) {
    logger.warn({ signature, expected }, 'Invalid LINE signature');
    return res.status(401).json({ error: 'invalid-signature' });
  }

  next();
}
```
