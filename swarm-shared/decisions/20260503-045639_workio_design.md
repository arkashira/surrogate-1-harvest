# workio / design

## Final Synthesis (Correctness + Actionability)

**Core diagnosis (merged, de-duplicated):**
- No idempotency enforcement on clock-in/out → double submissions (client double-tap, LINE retries) corrupt daily totals and create phantom punches.
- LINE signature accepted without time-window → replay and stale-event attacks.
- No fast deduplication store → every retry/duplicate hits business logic and DB writes.
- No optimistic UI + request dedupe → users wait on round-trip and repeatedly tap, increasing duplicates and jank.
- Clock state transitions are not atomic/guarded → races can create invalid sequences (double-in, missing-out, overlaps).

**Chosen scope (prioritized for impact + safety):**
1. Backend: enforce LINE signature time-window + idempotency/replay protection at the webhook edge.
2. Backend: atomic clock transition guard (state machine) + idempotent upsert for clock events.
3. Frontend: optimistic UI + client-side request dedupe with rollback.

## 1. DB schema (single source of truth)

Add lightweight replay table and make clock events idempotent.

```sql
-- workio/server/src/db/schema.sql

-- Replay protection for LINE webhooks and client idempotency keys
CREATE TABLE IF NOT EXISTS webhook_replay_protection (
  id               BIGSERIAL PRIMARY KEY,
  webhook_event_id TEXT NOT NULL,
  idempotency_key  TEXT NOT NULL,
  tenant_id        BIGINT NOT NULL,
  user_id          BIGINT NOT NULL,
  event_type       TEXT NOT NULL,
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT uq_webhook_event_id UNIQUE (webhook_event_id),
  CONSTRAINT uq_idempotency_key    UNIQUE (idempotency_key)
);

CREATE INDEX IF NOT EXISTS idx_replay_created_at ON webhook_replay_protection (created_at);

-- Idempotent clock events (prevent duplicate logical punches)
-- Use a natural key per logical punch so retries are upserts, not inserts.
-- One user can have at most one open clock_in without clock_out (enforced by app logic).
CREATE TABLE IF NOT EXISTS clock_events (
  id              BIGSERIAL PRIMARY KEY,
  tenant_id       BIGINT NOT NULL,
  user_id         BIGINT NOT NULL,
  event_type      TEXT NOT NULL CHECK (event_type IN ('clock_in', 'clock_out')),
  event_time      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  idempotency_key TEXT NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT uq_clock_idempotent UNIQUE (tenant_id, user_id, idempotency_key)
);

-- Fast lookup for current session (open clock_in)
CREATE INDEX IF NOT EXISTS idx_clock_events_user_open
  ON clock_events (tenant_id, user_id, event_type, event_time DESC)
  WHERE event_type = 'clock_in';
```

import crypto from 'crypto';
import type { Request, Response } from 'express';
import { db } from '../../db';

const SIGNATURE_TTL_MS = 5 * 60 * 1000; // 5 minutes
const LINE_SECRET = process.env.LINE_CHANNEL_SECRET!;

function verifyLineSignature(rawBody: string, signature: string): boolean {
  const expected = crypto
    .createHmac('sha256', LINE_SECRET)
    .update(rawBody, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

async function isReplay(webhookEventId: string, idempotencyKey: string): Promise<boolean> {
  const row = await db
    .selectFrom('webhook_replay_protection')
    .where('webhook_event_id', '=', webhookEventId)
    .orWhere('idempotency_key', '=', idempotencyKey)
    .select('id')
    .executeTakeFirst();
  return !!row;
}

async function markProcessed(payload: {

// Example clock transition processor (atomic + idempotent)
async function processClockEvent(
  tenantId: number,
  userId: number,
  eventType: 'clock_in' | 'clock_out',
  idempotencyKey: string,
  eventTime?: Date
) {
  // Idempotent upsert: if idempotency key exists, skip business logic
  const existing = await db
    .selectFrom('clock_events')
    .select('id')
    .where('tenant_id', '=', tenantId)
    .where('user_id', '=', userId)
    .where('idempotency_key', '=', idempotencyKey)
    .executeTakeFirst();

  if (existing) return; // already processed

  // State transition guard (simple & safe):
  // - clock_in allowed only if latest is clock_out or none
  // - clock_out allowed only if latest is clock_in
  const latest = await db
    .selectFrom('clock_events')
    .select(['event_type', 'event_time'])
    .where('tenant_id', '=', tenantId)
    .where('user_id', '=', userId)
    .orderBy('event_time', 'desc')
    .orderBy('id', 'desc')
    .limit(1)
    .executeTakeFirst();

  if (eventType === 'clock_in' && latest && latest.event_type === 'clock_in') {
    throw new Error('Invalid transition: previous clock_in has no clock_out');
  }
  if (eventType === 'clock_out' && (!latest || latest.event_type !== 'clock_in')) {
    throw new Error('Invalid transition: clock_out without prior clock_in');
  }

  await db
    .insertInto('clock_events')
    .values({
      tenant_id: tenantId,
      user_id: userId,
      event_type: eventType,
      event_time: eventTime ?? new Date(),
      idempotency_key: idempotencyKey,
    })
    .execute();
}
