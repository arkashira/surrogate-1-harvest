## Final synthesized plan (highest correctness + concrete actionability)

**Core improvement (≤2h): Deterministic pre-flight snapshot + CDN-only ingestion**  
- One `list_repo_tree` snapshot per date folder (Mac/CI) → JSON file list.  
- Training and shard workers consume the JSON and fetch files via CDN (`resolve/main/...`) with zero HF API calls during data load, eliminating 429s.  
- Shard workers use deterministic `hash(slug) % TOTAL_SHARDS` assignment so every shard sees the identical file set across runs and never collides on outputs.  
- Replace `load_dataset(streaming=True)` over heterogeneous repos with per-file CDN fetch + `pyarrow` projection to `{prompt,response}`.  
- Reuse running Lightning Studio; restart idle-stopped instances.

## Implementation plan (concrete, ordered)

1. **Create snapshot generator** (`bin/list-snapshot.sh`)  
   - Runs on Mac/CI.  
   - Calls `huggingface_hub.list_repo_tree(repo_id, path=f"public-raw/{date}", recursive=False)` once per date folder.  
   - Saves `snapshots/public-merged/{date}/files.json` (flat list of filenames).  
   - Exits non-zero if tree empty or API fails.

2. **Update dataset enrichment** (`bin/dataset-enrich.sh`)  
   - Accept optional `FILE_LIST` path (default: snapshot for date).  
   - Deterministic shard assignment:  
     - `slug = basename(f)` normalized; `hash = int(sha256(slug) hexdigest, 16)`; assign if `hash % TOTAL_SHARDS == SHARD_ID`.  
   - Fetch each file via CDN URL:  
     `https://huggingface.co/datasets/axentx/surrogate-1-training-pairs/resolve/main/public-raw/{date}/{fname}`  
   - Stream download with `requests.get(..., stream=True)` + `pyarrow` projection to `{prompt,response}`.  
   - Deduplicate via `lib/dedup.py` (content hash) and append to `enriched/batches/public-merged/{date}/{shard}.jsonl`.  
   - Remove any `load_dataset(..., streaming=True)` over heterogeneous repos.

3. **Update training data loader** (`train.py`)  
   - Read same `files.json` snapshot.  
   - Build an `IterableDataset` that yields CDN URLs and downloads on-the-fly with `requests.get(..., stream=True)` + `pyarrow` projection.  
   - No `list_repo_files` or `load_dataset` during training loop.

4. **Lightning Studio reuse**  
   - Before `Studio(create_ok=True)`, list running studios and reuse if present.  
   - On idle stop, detect status and restart with `target.start(machine=Machine.L40S)`.

5. **Cron/workflow hygiene**  
   - Ensure `SHELL=/bin/bash` in crontab.  
   - All wrapper scripts have `#!/usr/bin/env bash` and `chmod +x`.

---

## Resolved contradictions (in favor of correctness + actionability)

- **Snapshot scope**: Use non-recursive `list_repo_tree` per date folder (not recursive) to avoid pagination explosion; if subfolders exist, walk them explicitly in Python after the initial listing. This is safer and more deterministic than recursive API calls.  
- **Shard assignment**: Use `sha256`-based deterministic hashing (not `cksum`) to avoid platform-dependent behavior and collisions.  
- **Fetch method**: Prefer CDN `resolve/main/...` URLs for training and shard workers (fast, no API quota). Keep `hf_hub_download` only as a fallback for files not available via CDN (rare).  
- **Streaming heterogeneous repos**: Do not use `load_dataset(streaming=True)` over heterogeneous repos; instead, fetch per file and project with `pyarrow`. This is more robust and avoids schema surprises.  
- **Studio reuse**: Explicitly list running studios and reuse; restart idle-stopped ones with the correct machine type. Do not rely on `create_ok=True` alone.

### bin/list-snapshot.sh
```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
DATE="${1:-$(date +%Y-%m-%d)}"
OUTDIR="snapshots/public-merged/${DATE}"
OUTFILE="${OUTDIR}/files.json"

mkdir -p "${OUTDIR}"

python3 - <<PY
import os, json, sys
from huggingface_hub import HfApi

api = HfApi()
repo = os.environ.get("REPO", sys.argv[1])
date = os.environ.get("DATE", sys.argv[2])
out = os.environ.get("OUTFILE", sys.argv[3])

# Non-recursive per date folder; walk subfolders explicitly if needed
tree = api.list_repo_tree(repo, path=f"public-raw/{date}", recursive=False)
files = []
for f in tree:
    rfn = f.rfilename
    if f.type == "directory":
        # walk subfolder
        sub = api.list_repo_tree(repo, path=f"public-raw/{date}/{rfn}", recursive=True)
        files.extend([sf.rfilename for sf in sub if sf.type == "file"])
    else:
        files.append(rfn)

with open(out, "w") as fp:
    json.dump(files, fp, indent=2)
print(f"Snapshot saved: {len(files)} files -> {out}")
PY "$REPO" "$DATE" "$OUTFILE"
```
