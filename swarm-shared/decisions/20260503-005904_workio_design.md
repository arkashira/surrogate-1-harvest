# workio / design

## 1. Diagnosis

- **Missing DB-level uniqueness guard**: no constraint prevents multiple open punches (`clock_out_at IS NULL`) per user per day, allowing duplicates under LINE retries or concurrency.
- **Race-prone upsert path**: app-layer `find-then-insert` on `/webhook/line/clock` is vulnerable to duplicate inserts when LINE retries or users double-tap.
- **No idempotency token**: LINE webhook retries lack de-duplication; same event can create multiple punches.
- **Schema drift risk**: migrations are ad-hoc SQL files without versioning or repeatable CI checks.
- **Missing defensive indexes**: no composite index to enforce/fast-lookup `(user_id, clock_out_at, clock_in_at)` for daily open-punch queries.

## 2. Proposed change

- **File scope**: `workio/server/src/db/schema.sql` (add constraint + index) and `workio/server/src/routes/line.ts` (add idempotency + safe upsert).
- **Change type**: small, high-leverage DB + route fix (<2h).

## 3. Implementation

### 3.1 DB schema — enforce one open punch per user

```sql
-- workio/server/src/db/schema.sql
-- Add partial unique index: at most one NULL clock_out_at per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
ON punches (user_id, DATE(clock_in_at))
WHERE clock_out_at IS NULL;

-- Optional: ensure clock_in <= clock_out when both present
ALTER TABLE punches
ADD CONSTRAINT chk_clock_in_before_out
CHECK (clock_out_at IS NULL OR clock_out_at >= clock_in_at);
```

### 3.2 Idempotent LINE webhook handler

```ts
// workio/server/src/routes/line.ts
import { Router, Request, Response } from 'express';
import { pool } from '../db';
import crypto from 'crypto';

const router = Router();

// Simple in-memory set for fast duplicate detection (per process)
// For multi-instance, use Redis or DB unique constraint as source of truth.
const recentIds = new Set<string>();

function idempotencyKey(body: any): string {
  // Use LINE's deliveryId when available, else hash stable payload subset
  const deliveryId = body?.events?.[0]?.deliveryId;
  if (deliveryId) return `line:${deliveryId}`;
  const stable = JSON.stringify({
    type: body?.events?.[0]?.type,
    source: body?.events?.[0]?.source,
    timestamp: body?.events?.[0]?.timestamp,
  });
  return `line:sha256:${crypto.createHash('sha256').update(stable).digest('hex')}`;
}

router.post('/webhook/line', async (req: Request, res: Response) => {
  const key = idempotencyKey(req.body);
  if (recentIds.has(key)) {
    return res.status(200).json({ ok: true, reason: 'duplicate' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    const events = req.body.events || [];
    for (const ev of events) {
      const { type, source, message } = ev;
      const userId = source?.userId;
      if (!userId) continue;

      if (type === 'message' && message?.type === 'text') {
        const text = (message.text || '').trim().toLowerCase();
        const isClockIn = /^(in|เข้า|clock.in)$/i.test(text);
        const isClockOut = /^(out|ออก|clock.out)$/i.test(text);

        if (isClockIn || isClockOut) {
          const todayStart = new Date();
          todayStart.setHours(0, 0, 0, 0);

          // Try to find existing open punch for this user today
          const openRes = await client.query(
            `SELECT id FROM punches
             WHERE user_id = $1 AND clock_out_at IS NULL
             ORDER BY clock_in_at DESC LIMIT 1`,
            [userId]
          );

          if (isClockIn) {
            if (openRes.rows.length > 0) {
              // Already clocked in — ignore or optionally update location
              continue;
            }
            await client.query(
              `INSERT INTO punches (user_id, clock_in_at, clock_out_at, location, line_user_id)
               VALUES ($1, NOW(), NULL, $2, $1)`,
              [userId, message.text] // store payload or real GPS if available
            );
          } else if (isClockOut) {
            if (openRes.rows.length === 0) {
              // No open punch — ignore or create one?
              continue;
            }
            await client.query(
              `UPDATE punches
               SET clock_out_at = NOW(), location = COALESCE(location || ' | ' || $2, $2)
               WHERE id = $1`,
              [openRes.rows[0].id, message.text]
            );
          }
        }
      }
    }

    await client.query('COMMIT');
    recentIds.add(key);
    // Keep bounded to avoid memory growth
    if (recentIds.size > 5000) {
      const first = recentIds.values().next().value;
      recentIds.delete(first);
    }
    res.status(200).json({ ok: true });
  } catch (err: any) {
    await client.query('ROLLBACK');
    // Unique violation from idx_punches_one_open_per_user is safe duplicate
    if (err.code === '23505') {
      return res.status(200).json({ ok: true, reason: 'duplicate-db' });
    }
    console.error('LINE webhook error', err);
    res.status(500).json({ error: 'internal' });
  } finally {
    client.release();
  }
});

export default router;
```

### 3.3 Optional: add lightweight idempotency table (if multi-instance)

If you run multiple backend instances, replace in-memory `recentIds` with a small table:

```sql
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  key TEXT PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

And in the route, use `INSERT INTO webhook_idempotency (key) VALUES ($1) ON CONFLICT DO NOTHING` before processing.

## 4. Verification

1. **DB constraint check**
   ```bash
   psql workio -c "\d punches"
   # Confirm idx_punches_one_open_per_user exists
   ```
   Try inserting two open punches for same user:
   ```sql
   INSERT INTO punches (user_id, clock_in_at) VALUES (1, NOW());
   INSERT INTO punches (user_id, clock_in_at) VALUES (1, NOW());
   -- Second insert must fail with unique_violation (23505)
   ```

2. **Route behavior**
   - Start server: `npm run dev` in `workio/server`.
   - Send duplicate POST to `/webhook/line` with same `deliveryId` (or same stable payload) — expect `200 OK` and only one punch row.
   - Send clock-in then clock-out — expect one row updated with `clock_out_at` set.
   - Send clock-in twice — expect second ignored (no second open punch).

3. **Concurrency test**
   Use `ab` or a quick script to POST two clock-ins in parallel for same user; confirm only one open punch persists (DB constraint is final safeguard).
