# workio / ops

## 1. Diagnosis

- No storage-level uniqueness constraint on `(employee_id, punch_type, date)` → retries/races create duplicate punches.
- Punch write path accepts duplicates when clients retry (network glitches / LINE webhook redelivery).
- No idempotency key or deterministic punch hash to de-duplicate at write time.
- Application logic checks “last punch” in non-transactional SELECT-then-INSERT flow → race window.
- Missing index for fast conflict checks and reporting queries on date/employee.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/db/schema.sql`  
Scope: alter `punches` table to enforce uniqueness at the database level and add supporting index; update punch write path to use `ON CONFLICT DO NOTHING` (or upsert) so retries are safe.

## 3. Implementation

```sql
-- /opt/axentx/workio/server/src/db/schema.sql
-- Add uniqueness constraint + index to prevent duplicate punches
-- and make retries idempotent.

-- 1) Drop any existing duplicates (one-time cleanup; safe for prod if run during low traffic)
--    Keep the earliest created_at per duplicate group.
DELETE FROM punches
WHERE id NOT IN (
  SELECT MIN(id)
  FROM punches
  GROUP BY employee_id, punch_type, date
);

-- 2) Add unique constraint to prevent future duplicates
ALTER TABLE punches
  ADD CONSTRAINT uniq_employee_punch_date
  UNIQUE (employee_id, punch_type, date);

-- 3) Add index to speed conflict checks and date/employee queries
CREATE INDEX IF NOT EXISTS idx_punches_employee_date
  ON punches (employee_id, date DESC);

-- 4) Optional safety: ensure created_at defaults to now and is not null
ALTER TABLE punches
  ALTER COLUMN created_at SET DEFAULT NOW();
```

Update punch creation in the service layer to be idempotent and transactional:

```ts
// /opt/axentx/workio/server/src/services/punchService.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function createPunchSafe(employeeId: string, punchType: 'in' | 'out', date: string) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Try insert; if duplicate, return existing row (idempotent).
    const res = await client.query(
      `INSERT INTO punches (employee_id, punch_type, date)
       VALUES ($1, $2, $3)
       ON CONFLICT (employee_id, punch_type, date) DO UPDATE
       SET updated_at = NOW()
       RETURNING *`,
      [employeeId, punchType, date]
    );

    await client.query('COMMIT');
    return res.rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

If you prefer strict “no update on duplicate” (keep original punch time), use `ON CONFLICT DO NOTHING` and re-query:

```ts
const res = await client.query(
  `INSERT INTO punches (employee_id, punch_type, date)
   VALUES ($1, $2, $3)
   ON CONFLICT (employee_id, punch_type, date) DO NOTHING
   RETURNING *`,
  [employeeId, punchType, date]
);

if (res.rowCount === 0) {
  const get = await client.query(
    `SELECT * FROM punches WHERE employee_id = $1 AND punch_type = $2 AND date = $3`,
    [employeeId, punchType, date]
  );
  return get.rows[0];
}
return res.rows[0];
```

## 4. Verification

1. Apply schema migration:
   ```bash
   cd /opt/axentx/workio
   psql workio < server/src/db/schema.sql
   ```

2. Run a quick idempotency test in Node:
   ```bash
   node -e "
   const { createPunchSafe } = require('./server/src/services/punchService');
   (async () => {
     const emp = 'test-emp-123';
     const d = '2026-05-02';
     const a = await createPunchSafe(emp, 'in', d);
     const b = await createPunchSafe(emp, 'in', d);
     console.log('first', a.id, 'second', b.id, 'same?', a.id === b.id);
   })();
   "
   ```
   Expect: same punch row returned; only one row in DB.

3. Confirm constraint exists:
   ```bash
   psql workio -c "\d punches"
   ```
   Expect: `UNIQUE CONSTRAINT` on `(employee_id, punch_type, date)` and index `idx_punches_employee_date`.

4. Simulate concurrent duplicates (quick race):
   ```bash
   for i in {1..10}; do
     psql workio -c "INSERT INTO punches (employee_id, punch_type, date) VALUES ('race-emp', 'in', '2026-05-02') ON CONFLICT DO NOTHING;" &
   done
   wait
   psql workio -c "SELECT count(*) FROM punches WHERE employee_id='race-emp' AND date='2026-05-02';"
   ```
   Expect: count = 1.
