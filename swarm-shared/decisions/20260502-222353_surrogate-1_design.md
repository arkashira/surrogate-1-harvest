# surrogate-1 / design

## Implementation Plan (≤2h)

**Highest-value improvement**: Add deterministic pre-flight file listing + CDN-only ingestion to eliminate HF API 429s during training and make shard workers resilient.

### What we’ll change
1. Add `bin/list-files.sh` — one-time Mac/CI step that calls `list_repo_tree` per date folder and writes `file-list.json` (path + size + sha256) to repo root.
2. Update `bin/dataset-enrich.sh` to accept an optional `FILE_LIST` env var; if present, workers read paths from JSON and download via raw CDN URLs (`https://huggingface.co/datasets/.../resolve/main/...`) with no Authorization header.
3. Add fallback: if CDN fetch fails (404/403), retry once via `hf_hub_download` (authenticated) so ingestion never stalls.
4. Add `requirements-dev.txt` with `requests` for CDN downloads.
5. Update README with usage and the HF CDN bypass note.

### Why this is highest value
- Eliminates the #1 cause of failed runs (HF API 429) during training and ingestion.
- Keeps current 16-shard parallel architecture unchanged.
- Costs ~30 min to implement + test.

### 1. `bin/list-files.sh`
```bash
#!/usr/bin/env bash
# list-files.sh
# Usage: HF_TOKEN=... ./bin/list-files.sh axentx surrogate-1-training-pairs > file-list.json
# Requires: huggingface_hub, python3

set -euo pipefail

REPO_OWNER="${1:-axentx}"
REPO_NAME="${2:-surrogate-1-training-pairs}"
OUTPUT="${3:-file-list.json}"

python3 - "$REPO_OWNER" "$REPO_NAME" "$OUTPUT" <<'PY'
import json
import os
import sys
from huggingface_hub import HfApi, list_repo_tree

owner, repo, out_path = sys.argv[1], sys.argv[2], sys.argv[3]
api = HfApi(token=os.environ.get("HF_TOKEN"))

result = []
# list top-level date folders only (non-recursive)
for folder in list_repo_tree(repo=repo, repo_type="dataset", path="", recursive=False):
    if not folder.path.endswith("/"):
        continue
    date_folder = folder.path.rstrip("/")
    # list files inside this date folder (non-recursive)
    for file in list_repo_tree(repo=repo, repo_type="dataset", path=date_folder, recursive=False):
        if file.path.endswith("/"):
            continue
        result.append({
            "path": file.path,
            "size": getattr(file, "size", None),
            "sha256": getattr(file, "sha256", None),
        })

with open(out_path, "w", encoding="utf-8") as f:
    json.dump(result, f, indent=2)

print(f"Wrote {len(result)} files to {out_path}", file=sys.stderr)
PY
```

Make executable:
```bash
chmod +x bin/list-files.sh
```

    resp.raise_for_status()

    try:
        from huggingface_hub import hf_hub_download
        local = hf_hub_download(repo_id=repo, filename=file_path, repo_type="dataset")
        with open(local, "rb") as f:
            return f.read()
    except Exception as e:
        raise RuntimeError(f"Fallback download failed for {file_path}: {e}")

            if line.strip():
                seen.add(line.strip())

        return True
