# workio / frontend

## 1. Diagnosis

- **No frontend idempotency guard for LINE clock-in/out**: rapid taps or LINE retries can submit duplicate punches before the previous request resolves, creating duplicate records.
- **Race-prone optimistic UI**: clock-in/out button enables immediately on click; if the request fails, UI state diverges from server truth.
- **Missing debounced submit + disabled state**: users can spam the punch button; no visual feedback that submission is in-flight.
- **No client-side dedupe key for recent punches**: repeated submissions for the same timeslot can queue and replay after retries, compounding duplicates.
- **Toast/confirmation is inconsistent**: users lack clear confirmation or error recovery guidance when GPS or network fails.

## 2. Proposed change

File: `workio/src/components/ClockInOutButton.tsx` (or equivalent main punch component)  
Scope: add idempotency key (client-generated uuid per timeslot), disable-on-submit, optimistic update with rollback, and toast confirmation.

## 3. Implementation

```tsx
// workio/src/components/ClockInOutButton.tsx
import React, { useState, useCallback } from 'react';
import { useToast } from '@/hooks/use-toast';
import { v4 as uuidv4 } from 'uuid';

type PunchState = 'in' | 'out' | null;

export function ClockInOutButton() {
  const [punchState, setPunchState] = useState<PunchState>(null);
  const [submitting, setSubmitting] = useState(false);
  const [lastIdempotencyKey, setLastIdempotencyKey] = useState<string | null>(null);
  const { toast } = useToast();

  const punch = useCallback(
    async (type: 'clock_in' | 'clock_out') => {
      if (submitting) return;

      const idempotencyKey = uuidv4();
      const payload = {
        type,
        timestamp: new Date().toISOString(),
        // optional: lat/lng if GPS collected at click time
        idempotency_key: idempotencyKey,
      };

      setSubmitting(true);
      const previousState = punchState;
      // optimistic UI
      setPunchState(type === 'clock_in' ? 'in' : 'out');

      try {
        const res = await fetch('/api/punch', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify(payload),
        });

        if (!res.ok) {
          const err = await res.json().catch(() => ({}));
          throw new Error(err.message || `Server error ${res.status}`);
        }

        setLastIdempotencyKey(idempotencyKey);
        toast({
          title: type === 'clock_in' ? 'เข้างานแล้ว' : 'ออกงานแล้ว',
          description: new Date().toLocaleTimeString(),
          variant: 'default',
        });
      } catch (err: any) {
        // rollback optimistic UI
        setPunchState(previousState);
        toast({
          title: 'ไม่สามารถบันทึกได้',
          description: err?.message || 'กรุณาลองใหม่อีกครั้ง',
          variant: 'destructive',
        });
      } finally {
        setSubmitting(false);
      }
    },
    [punchState, submitting, toast]
  );

  return (
    <div className="flex flex-col items-center gap-3">
      <div className="text-sm text-muted-foreground">
        {punchState === 'in' && 'สถานะ: เข้างาน'}
        {punchState === 'out' && 'สถานะ: ออกงาน'}
        {punchState === null && 'สถานะ: ยังไม่ได้ลงเวลา'}
      </div>

      <div className="flex gap-2">
        <button
          onClick={() => punch('clock_in')}
          disabled={submitting || punchState === 'in'}
          className="inline-flex items-center rounded bg-green-600 px-4 py-2 text-sm font-medium text-white disabled:opacity-50 disabled:cursor-not-allowed hover:bg-green-700 transition"
        >
          {submitting && punchState === 'in' ? 'กำลังบันทึก...' : 'ลงเวลางาน'}
        </button>

        <button
          onClick={() => punch('clock_out')}
          disabled={submitting || punchState === 'out'}
          className="inline-flex items-center rounded bg-red-600 px-4 py-2 text-sm font-medium text-white disabled:opacity-50 disabled:cursor-not-allowed hover:bg-red-700 transition"
        >
          {submitting && punchState === 'out' ? 'กำลังบันทึก...' : 'ลงเวลาออก'}
        </button>
      </div>

      {lastIdempotencyKey && (
        <code className="text-xs text-muted-foreground">
          idempotency: {lastIdempotencyKey.slice(0, 8)}…
        </code>
      )}
    </div>
  );
}
```

Backend note (keep contract; small change): accept optional `idempotency_key` in `/api/punch` and enforce uniqueness (e.g., unique index on `(user_id, date, type, idempotency_key)` or short-term cache) to reject duplicates from retries. If not yet implemented, add:

```sql
-- server/src/db/schema.sql (add if not present)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punch_idempotency
ON punches (user_id, DATE(timestamp), type, idempotency_key)
WHERE idempotency_key IS NOT NULL;
```

## 4. Verification

1. **Smoke test**:
   - Open the page and click “ลงเวลางาน” → button disables, toast appears, status shows “เข้างาน”.
   - Rapid-click 5x → only one request sent (disabled state prevents spam).
   - Simulate failure (block network or return 500) → UI rolls back to previous state and shows error toast.

2. **Idempotency test**:
   - Submit same punch twice with same `idempotency_key` (temporarily hardcode) → second request returns 409 or 200 without creating duplicate row.
   - Check DB: `SELECT user_id, type, COUNT(*) FROM punches WHERE DATE(timestamp)=CURRENT_DATE GROUP BY user_id, type;` — counts should be 1 per type per day per user for normal flows.

3. **E2E via browser**:
   - Clock in → wait 2s → clock out → verify timeline and confirmations.
   - Refresh page → component should rehydrate real state from `/api/punch/latest` (if implemented) or rely on server-rendered initial state; optimistic state is local-only and safe to discard on refresh.
