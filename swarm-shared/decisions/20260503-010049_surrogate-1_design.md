### Synthesized Solution

The proposed solution aims to eliminate HF API rate-limit failures during ingestion and prevent OOM in the HF Space. This is achieved by switching to CDN-only fetches and per-folder tree listing.

#### Key Components

1. **Folder Listing Mode**: Modify `bin/dataset-enrich.sh` to accept an optional `FOLDER_DATE` parameter, defaulting to the current date. Use `list_repo_tree` to fetch the file list for the specified folder and emit `filelist.json`.
2. **CDN-Only Fetches**: Update the deduplication and projection logic to stream from local paths. Iterate over `filelist.json`, download each file via CDN URLs using `hf_hub_download` or raw `curl`/`requests`, and project the data.
3. **Utility Function**: Introduce a `cdn_url` function to resolve CDN URLs for a given repository and path.
4. **Optional Train.py Update**: Modify `train.py` to consume the embedded `filelist.json` and build a `IterableDataset` that uses CDN URLs, eliminating HF API calls during training.

#### Implementation Plan

1. **Add Folder Listing Mode**: Modify `bin/dataset-enrich.sh` to include the folder listing mode.
2. **Update Deduplication and Projection Logic**: Modify the Python worker to shard, download via CDN, dedup, and project the data.
3. **Introduce CDN URL Utility Function**: Create a `cdn_url` function to resolve CDN URLs.
4. **Optional Train.py Update**: Update `train.py` to consume the embedded `filelist.json` and use CDN URLs.
