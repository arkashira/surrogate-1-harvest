## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service that discovers Lambda + Cloud Functions schedules (read-only), persists to Supabase Postgres, detects duplicates (same cron+target) and orphaned schedules (no matching function), exposes a simple REST endpoint and JSON report; deploy on Cloudflare Workers (free tier) with Supabase free tier; no UI, no queue, no cache.

## Open questions
- Which identity model will customers use to grant read-only access (AWS IAM role vs GCP SA keys vs OAuth)?
- How frequently should discovery jobs run to balance freshness vs rate limits/costs?
- What retention window do we need for schedule history to compute drift reliably?
