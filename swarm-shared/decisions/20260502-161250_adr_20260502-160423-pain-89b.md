## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI client (Linux/Windows/macOS) that: 1) auto-selects a stealth profile to hide local SOCKS5 port (ephemeral bind + local NAT mapping) and optionally runs TUN-less; 2) tunnels traffic over QUIC (443) with innocuous TLS fingerprints and lightweight stream multiplexing; 3) enforces a process-level kill-switch that blocks non-tunneled traffic if the stealth path fails; 4) stores session state and encrypted config in local SQLite; 5) emits minimal, opt-in, hashed telemetry to a CF Worker endpoint to compute WAU-ZB; 6) provides zero-config UX (single binary, no admin/root required on supported platforms) and ships as a free-tier artifact (GitHub Releases + CF Workers for endpoint coordination).

## Open questions
- Which camouflage profiles (mux/fragment/cover) defeat local DPI in target regions without harming performance?
- Can we safely hide TUN interfaces on major desktop OSes without kernel drivers while preserving MTU and DNS leak protection?
- How will we distribute ephemeral egress endpoints (CF Workers + Supabase edge functions) to avoid static IP blocking?
- What telemetry minimization and consent model ensures WAU-ZB reporting without deanonymizing users?
- What is the fail-closed kill-switch behavior across platforms (block all non-stealth traffic vs. alert-only)?
