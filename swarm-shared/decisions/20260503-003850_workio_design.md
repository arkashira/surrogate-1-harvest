# workio / design

## 1. Diagnosis

- Missing **idempotency enforcement** on `/webhook/line` allows LINE retries (network blips, 5xx, slow client) to create duplicate punches for the same event.
- No **partial unique constraint** enforcing “one open punch per user” (`clock_out_at IS NULL`) at the database level, so concurrent requests can create multiple open records.
- Punch write path is **non-transactional** across insert + prior-close updates, risking partial state on failure or retry.
- No **idempotency key** in request payload or deduplication table to short-circuit retries within a safe window (e.g., 5–10 minutes).
- Application-level checks are racy under concurrency; DB-level guardrails are required for correctness.

## 2. Proposed change

- **Scope**: `server/src/db/schema.sql` (add constraint + idempotency table) and `server/src/routes/webhook/line.ts` (add transactional upsert + idempotency check).
- **Goal**: Guarantee exactly-once punch semantics per LINE event and enforce single open punch per user via DB constraint.

## 3. Implementation

### 3.1 DB schema changes (`server/src/db/schema.sql`)

```sql
-- Idempotency keys for LINE webhook events (5-10min TTL)
CREATE TABLE IF NOT EXISTS line_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  event_type      VARCHAR(50)  NOT NULL,
  user_id         INTEGER      NOT NULL,
  punch_id        INTEGER      NOT NULL,
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- Partial unique constraint: at most one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- Optional: cleanup job index (for TTL pruning)
CREATE INDEX IF NOT EXISTS idx_line_idempotency_created_at
ON line_idempotency (created_at);
```

### 3.2 Webhook handler (`server/src/routes/webhook/line.ts`)

```ts
import { Request, Response } from 'express';
import { pool } from '../../db';
import crypto from 'crypto';

const IDEMPOTENCY_TTL_MS = 10 * 60 * 1000; // 10 minutes

export async function handleLineWebhook(req: Request, res: Response) {
  const { events } = req.body;
  if (!events || !Array.isArray(events)) return res.sendStatus(400);

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      if (ev.type !== 'message' || ev.message?.type !== 'text') continue;

      // Derive stable idempotency key from LINE delivery context
      const idempotencyKey = crypto
        .createHash('sha256')
        .update(`${ev.source.userId}:${ev.timestamp}:${ev.message.text}`)
        .digest('hex');

      // Check idempotency
      const idem = await client.query(
        'SELECT punch_id FROM line_idempotency WHERE idempotency_key = $1',
        [idempotencyKey]
      );
      if (idem.rowCount > 0) {
        // Already processed — skip but continue processing other events
        continue;
      }

      const { userId } = ev.source;

      // Find user by line_user_id (adjust column name to your schema)
      const userRes = await client.query(
        'SELECT id FROM users WHERE line_user_id = $1',
        [userId]
      );
      if (userRes.rowCount === 0) continue;
      const userIdDb = userRes.rows[0].id;

      // Try to close an existing open punch first (clock_out_at IS NULL)
      const closeRes = await client.query(
        `UPDATE punches
         SET clock_out_at = NOW(), updated_at = NOW()
         WHERE user_id = $1 AND clock_out_at IS NULL
         RETURNING id`,
        [userIdDb]
      );

      if (closeRes.rowCount > 0) {
        // Closed existing punch
        await client.query(
          'INSERT INTO line_idempotency (idempotency_key, event_type, user_id, punch_id) VALUES ($1, $2, $3, $4)',
          [idempotencyKey, 'clock_out', userIdDb, closeRes.rows[0].id]
        );
        continue;
      }

      // No open punch — create new one (clock_in)
      const insertRes = await client.query(
        `INSERT INTO punches (user_id, clock_in_at, clock_out_at, created_at, updated_at)
         VALUES ($1, NOW(), NULL, NOW(), NOW())
         RETURNING id`,
        [userIdDb]
      );
      const punchId = insertRes.rows[0].id;

      await client.query(
        'INSERT INTO line_idempotency (idempotency_key, event_type, user_id, punch_id) VALUES ($1, $2, $3, $4)',
        [idempotencyKey, 'clock_in', userIdDb, punchId]
      );
    }

    await client.query('COMMIT');
    res.sendStatus(200);
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error:', err);
    res.sendStatus(500);
  } finally {
    client.release();
  }
}
```

-- Run daily or hourly
DELETE FROM line_idempotency

## 4. Verification

- **Constraint test**: Attempt to insert two punches with `clock_out_at IS NULL` for the same user via direct SQL — second insert should fail.
- **Idempotency test**: Send the same LINE event payload twice (same derived key) — second request should not create a new punch and should return 200.
- **Concurrency test**: Fire multiple parallel requests simulating LINE retries for the same event — only one punch should be created/closed and no constraint violations should occur.
- **End-to-end test**: Clock in via LINE, then clock in again (should create new punch only after previous is closed); verify dashboard shows correct open/closed state.
- **Log check**: Confirm `line_idempotency` entries are created and duplicates are skipped.
