# airship / design

**Highest-Value Incremental Improvement:**
**Design Cycle: Improve Surrogate AI Knowledge Graph**

**Implementation Plan:**

1. **Knowledge-Rag Pipeline**
	* Run `granite-business-research.sh` to execute market analysis script
	* Execute `knowledge-rag` to query top hub and related docs for contextual insights
2. **Top-Hub Doc Insight**
	* Review the most-connected hub (e.g., "MOC") before planning tasks
3. **Neo4j Knowledge Graph**
	* Connect to Neo4j instance (e.g., `bolt://localhost:7687`)
	* Run Cypher query to update knowledge graph with new insights from knowledge-rag pipeline
	* Use `neo4j-admin` to backup and restore knowledge graph as needed
4. **Vector Store (Qdrant)**
	* Connect to Qdrant instance (e.g., `http://localhost:6333`)
	* Run query to update vector store with new knowledge graph insights

**Code Snippets:**

```bash
# Run market analysis script
./granite-business-research.sh

# Execute knowledge-rag pipeline
knowledge-rag -h localhost -p 8000 -u user -p password -d database

# Review top hub
cypher-shell -h localhost -p 7687 -u neo4j -p password "MATCH (n) RETURN n LIMIT 10"

# Update Neo4j knowledge graph
cypher-shell -h localhost -p 7687 -u neo4j -p password "MATCH (n) SET n.insight = 'new insight' RETURN n"

# Backup Neo4j knowledge graph
neo4j-admin backup -h localhost -p 7687 -u neo4j -p password -f /path/to/backup

# Restore Neo4j knowledge graph
neo4j-admin restore -h localhost -p 7687 -u neo4j -p password -f /path/to/backup

# Update Qdrant vector store
curl -X POST \
  http://localhost:6333/v1/query \
  -H 'Content-Type: application/json' \
  -d '{"query": {"match_all": {}}}'
```
