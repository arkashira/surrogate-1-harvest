# vanguard / ops

## 1. Diagnosis

- Backend ingestion/training jobs still perform runtime `list_repo_tree`/`load_dataset` calls → 429 risk and non-reproducible runs.
- No deterministic, content-addressed manifest keyed by `{date}/{slug}` → jobs re-enumerate and can’t guarantee CDN-only fetches.
- Frontend dataset selection/preview triggers HF API calls (same 429 risk) instead of reading a pre-built manifest.
- Missing orchestration wrapper that enforces Mac-only orchestration + remote execution (Lightning/Kaggle/HF) per the architecture rule.
- No retry/backoff for HF 429 or Kaggle 401/403 flows; failures bubble up and stall pipelines.

## 2. Proposed change

Create `/opt/axentx/vanguard/ops/manifest_builder.py` (single orchestrator) that:
- Runs on Mac (or any scheduler) and is the only place that calls HF API (`list_repo_tree`).
- Produces `manifests/{dataset}/{date}/files.json` (content-addressed, `{slug}.parquet` entries).
- Embeds the file list into training scripts so Lightning Studio does CDN-only fetches with zero API calls.
- Adds simple retry/backoff for 429 (wait 360s) and logs warnings for 401/403.
- Exposes a tiny CLI so cron/crontab can invoke it safely (`SHELL=/bin/bash`, executable, Bash shebang).

Scope: one new file + one small update to the training launcher to consume the manifest.

## 3. Implementation

```bash
# /opt/axentx/vanguard/ops/manifest_builder.py
#!/usr/bin/env bash
# Build deterministic HF file manifests to avoid runtime API calls (429 risk)
# Usage: ./manifest_builder.py <dataset_repo> <date> [--out-dir ./manifests]
# Example: ./manifest_builder.py axentx/surrogate-1 2026-05-03

set -euo pipefail
SHELL=/bin/bash

REPO="${1:?Missing dataset_repo arg}"
DATE="${2:?Missing date arg}"
OUT_DIR="${3:-./manifests}"
MANIFEST_FILE="${OUT_DIR}/${REPO//\//_}/${DATE}/files.json"

mkdir -p "$(dirname "${MANIFEST_FILE}")"

# Use huggingface_hub from Python (called by bash wrapper) to list once, CDN-only thereafter
python3 - "$REPO" "$DATE" "$MANIFEST_FILE" <<'PY'
import json, os, sys, time
from huggingface_hub import list_repo_tree, hf_hub_download

REPO, DATE, OUT = sys.argv[1], sys.argv[2], sys.argv[3]

def backoff(fn, *a, **kw):
    for attempt in range(5):
        try:
            return fn(*a, **kw)
        except Exception as e:
            msg = str(e).lower()
            if "429" in msg or "rate limit" in msg:
                wait = 360
                print(f"Rate limited (429). Waiting {wait}s (attempt {attempt+1})", file=sys.stderr)
                time.sleep(wait)
                continue
            if "401" in msg or "403" in msg:
                print(f"Auth/permission error: {e}", file=sys.stderr)
            raise
    raise RuntimeError(f"Exhausted retries for {fn.__name__}")

# List top-level for date folder only (non-recursive per folder to minimize pagination)
root = backoff(list_repo_tree, repo_id=REPO, path=DATE, recursive=False)

files = []
for entry in root:
    if entry.rfilename.endswith(".parquet"):
        slug = os.path.splitext(os.path.basename(entry.rfilename))[0]
        files.append({
            "date": DATE,
            "slug": slug,
            "path": entry.rfilename,
            "cdn_url": f"https://huggingface.co/datasets/{REPO}/resolve/main/{entry.rfilename}"
        })

# Deterministic ordering for reproducibility
files.sort(key=lambda x: (x["date"], x["slug"]))

os.makedirs(os.path.dirname(OUT), exist_ok=True)
with open(OUT, "w") as f:
    json.dump({"repo": REPO, "date": DATE, "generated_by": "manifest_builder", "files": files}, f, indent=2)

print(f"Wrote {len(files)} entries to {OUT}")
PY

echo "Manifest built: ${MANIFEST_FILE}"
```

Make executable and ensure cron uses Bash:

```bash
chmod +x /opt/axentx/vanguard/ops/manifest_builder.py
```

Crontab line (example, run after daily ingestion):

```cron
SHELL=/bin/bash
0 3 * * * cd /opt/axentx/vanguard && ./ops/manifest_builder.py axentx/surrogate-1 $(date +\%F) --out-dir ./manifests >> ./logs/manifest_builder.log 2>&1
```

Update training launcher to consume manifest (example snippet to embed in Lightning Studio script):

```python
# training/train.py  (or wherever surrogate-1 training is launched)
import json, os

MANIFEST = os.getenv("VANGUARD_MANIFEST", "manifests/axentx_surrogate-1/2026-05-03/files.json")
with open(MANIFEST) as f:
    manifest = json.load(f)

# CDN-only paths; zero HF API calls during training
file_urls = [item["cdn_url"] for item in manifest["files"]]

# Use streaming with CDN URLs (or download via hf_hub_download if preferred)
# Example: datasets.load_dataset can read from local files or pass URLs to data_files=file_urls
```

## 4. Verification

1. Run the builder once:
   ```bash
   cd /opt/axentx/vanguard
   ./ops/manifest_builder.py axentx/surrogate-1 2026-05-03 --out-dir ./manifests
   ```
   Confirm `manifests/axentx_surrogate-1/2026-05-03/files.json` exists and contains deterministic `{date, slug, path, cdn_url}` entries.

2. Check no HF API calls during training:
   - Set `VANGUARD_MANIFEST` to the produced file.
   - Start a Lightning Studio (L40S) run with the updated `train.py`.
   - Monitor logs: there should be no `list_repo_tree` or `load_dataset` network calls to `api.huggingface.co`; only CDN fetches to `huggingface.co/datasets/.../resolve/main/...`.

3. Cron sanity:
   - Ensure crontab uses `SHELL=/bin/bash`.
   - Run the cron entry manually via `bash ./ops/manifest_builder.py ...` and confirm exit 0 and updated manifest.

4. Rate-limit resilience:
   - Simulate 429 by temporarily throttling (or inspect code path) and confirm 360s backoff is triggered and retries succeed without crashing.
