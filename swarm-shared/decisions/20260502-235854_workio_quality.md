# workio / quality

## 1. Diagnosis

- Punch submission has no client-side idempotency key → retries (offline/poor network) can create duplicate punches on the backend.
- No optimistic UI or local pending queue → spinner ambiguity and zero feedback when LINE/webhook or API calls fail.
- Clock-in/out endpoint lacks deduplication window (same user, same day, overlapping punches) → allows invalid state (double clock-in) and corrupts attendance records.
- Missing server-side request traceability (no correlation ID) → hard to debug webhook vs client-originated duplicates.
- No compensating rollback or “undo” path for accidental punches → support burden and data correction overhead.

## 2. Proposed change

- **Files**:  
  - `workio/server/src/routes/punch.ts` (add idempotency + dedupe)  
  - `workio/src/features/punch/usePunch.ts` (add optimistic queue + correlation ID)  
  - `workio/server/src/db/schema.sql` (add `punches.idempotency_key` + index)  
- **Scope**: small, self-contained; <2h to ship end-to-end.

## 3. Implementation

### 3.1 DB migration (schema + index)

```sql
-- workio/server/src/db/schema.sql
-- Add idempotency key to punches (nullable for existing rows)
ALTER TABLE punches ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(128) NULL;
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key
  ON punches (tenant_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- Optional safety: prevent same user multiple clock-ins on same day without clock-out
-- (enforced at app level; index helps queries)
CREATE INDEX IF NOT EXISTS idx_punches_user_date
  ON punches (tenant_id, user_id, DATE(created_at));
```

### 3.2 Backend: idempotent punch endpoint

```ts
// workio/server/src/routes/punch.ts
import { Router } from 'express';
import { db } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = Router();

router.post('/punches', async (req, res) => {
  const { tenant_id, user_id, type } = req.body; // type: 'clock_in' | 'clock_out'
  const idempotencyKey = req.header('Idempotency-Key') || uuidv4();
  const clientCorrelationId = req.header('X-Correlation-Id') || uuidv4();

  if (!tenant_id || !user_id || !type) {
    return res.status(400).json({ error: 'Missing required fields' });
  }

  try {
    // Idempotency check
    const existing = await db.oneOrNone(
      'SELECT * FROM punches WHERE tenant_id = $1 AND idempotency_key = $2',
      [tenant_id, idempotencyKey]
    );

    if (existing) {
      return res.status(200).json({ ...existing, _idempotent_replay: true });
    }

    // Business rule: prevent double clock-in without clock-out today
    if (type === 'clock_in') {
      const last = await db.oneOrNone(
        `SELECT * FROM punches
         WHERE tenant_id = $1 AND user_id = $2 AND DATE(created_at) = CURRENT_DATE
         ORDER BY created_at DESC LIMIT 1`,
        [tenant_id, user_id]
      );

      if (last && last.type === 'clock_in') {
        return res.status(409).json({
          error: 'Already clocked in',
          last_punch: last,
          suggestion: 'clock_out first or use correction flow'
        });
      }
    }

    const punch = await db.one(
      `INSERT INTO punches (tenant_id, user_id, type, idempotency_key, correlation_id, metadata)
       VALUES ($1, $2, $3, $4, $5, $6)
       RETURNING *`,
      [tenant_id, user_id, type, idempotencyKey, clientCorrelationId, {
        source: req.body._source || 'client',
        client_ip: req.ip
      }]
    );

    // TODO: emit LINE notification async (non-blocking)
    // fireAndForget(sendLinePunchNotification(punch));

    res.status(201).json(punch);
  } catch (err) {
    // Unique violation on idempotency_key can still race; handle gracefully
    if ((err as any)?.code === '23505') {
      const existing = await db.oneOrNone(
        'SELECT * FROM punches WHERE tenant_id = $1 AND idempotency_key = $2',
        [tenant_id, idempotencyKey]
      );
      return res.status(200).json({ ...existing, _idempotent_replay: true });
    }
    console.error('Punch creation failed', err);
    res.status(500).json({ error: 'Internal server error' });
  }
});

export default router;
```

  id?: string;
  tenant_id: string;
  user_id: string;
  type: 'clock_in' | 'clock_out';
  _optimistic?: boolean;
  _error?: string | null;
  correlation_id: string;
  idempotency_key: string;
};

      setPending([]);

    try {
      await AsyncStorage.setItem(PENDING_QUEUE_KEY, JSON.stringify(items));

      // best-effort

    if (mounted.current) setPending(items);
