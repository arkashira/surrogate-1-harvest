#### Goal
Eliminate HF API 429s during Surrogate-1 training by replacing `load_dataset(streaming=True)` with a CDN-only `IterableDataset` that reads from a pre-generated `file_list.json`.

#### Why this ships fast
- Single-file change in `train.py`
- Leverages existing `hf_hub_download` functionality
- Bypasses HF API rate limits using CDN

#### Implementation Plan
1. **Generate `file_list.json`**:
   - Run a script that uses `list_repo_tree(path, recursive=False)` to fetch file paths for a specific date folder.
   - Save the list to `file_list.json`.

2. **Modify `train.py`**:
   - Import `file_list.json` and create an `IterableDataset` that reads files from the CDN using `hf_hub_download`.
   - Replace `load_dataset(streaming=True)` with the new `IterableDataset`.

3. **Test and Verify**:
   - Run a test training iteration to ensure the new `IterableDataset` works as expected.
   - Verify that HF API 429 errors are eliminated.

import json
import torch
from torch.utils.data import IterableDataset
from huggingface_hub import hf_hub_download

# Load file list from JSON
with open('file_list.json', 'r') as f:
    file_list = json.load(f)

class CDNIterableDataset(IterableDataset):
    def __init__(self, file_list):
        self.file_list = file_list

    def __iter__(self):
        for file_path in self.file_list:
            # Download file from CDN
            file_url = f"https://huggingface.co/datasets/{file_path}"
            file_data = hf_hub_download(file_url, repo_type="dataset")
            # Yield file data as a tuple (prompt, response)
            yield (file_data['prompt'], file_data['response'])

# Create dataset and data loader
dataset = CDNIterableDataset(file_list)
data_loader = torch.utils.data.DataLoader(dataset, batch_size=32)
