## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk guardrail→execution stub) with immutable Postgres audit trail, CF Workers tasking and Queues, Redis cache for risk checks, and one sample broker/data connector; includes replay endpoint and basic compliance rejection path.

## Open questions
- Deterministic replay scope: which inputs/snapshots must be versioned to guarantee exact replay across connector API changes?
- Latency SLA per stage (research→signal→risk→execution) and acceptable queue backpressure thresholds on free-tier Workers/Queues
- Compliance signature scheme: HMAC with rotating keys vs append-only hash chain; key storage on free-tier constraints
- Connector idempotency and at-least-once vs exactly-once semantics given CF Workers retries and queue delivery guarantees
- Data residency and retention requirements for audit entries (Postgres row-level policies and archival strategy)
