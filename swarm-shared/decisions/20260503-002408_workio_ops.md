# workio / ops

## Final Synthesis (Best Parts + Correctness + Actionability)

I merged the strongest, non-redundant insights from both candidates and resolved contradictions in favor of correctness and immediate actionability.

### 1) Diagnosis (merged)

- **No automated operational hygiene**: deploys, migrations, cron jobs rely on manual setup/ad-hoc scripts (no idempotent ops flow).
- **No repeatable deploy/rollback**: `.env` copied by hand, migrations run manually — high human-error risk.
- **Environment drift**: `.env.example` exists but no validation or enforcement of required secrets across dev/staging/prod.
- **Clock/timezone correctness assumed**: server-local TZ may differ from LINE/user expectations (Asia/Bangkok), causing day-boundary punch errors.
- **No lightweight readiness/liveness checks**: ops can’t quickly detect outage or verify deploy success (DB, LINE webhook, API).
- **Punch endpoint vulnerabilities**: duplicate punches on retries (no idempotency key), and DB relies on app logic to prevent overlapping open punches (race conditions under load).
- **No monitoring/alerting**: uptime, error rates, latency unknown; mean-time-to-detect is high.

### 2) Proposed change (merged + resolved)

Add an `ops/` folder and minimal, high-leverage code changes:

- `ops/env-check.sh` — validates required env vars and warns on missing/invalid values.
- `ops/health.sh` — lightweight readiness probe (DB ping + API health + LINE config) for cron/monitoring.
- `ops/deploy.sh` — repeatable deploy: run migrations, verify health, smoke-test punch idempotency.
- `ops/migrate.sh` — idempotent schema migration runner (safe to re-run).
- Update `.env.example` to include `TZ` and `LOG_LEVEL`.
- Add DB partial unique constraint: one open punch per user (`clock_out_at IS NULL`).
- Add idempotent punch endpoint using `Idempotency-Key` header and DB-level upsert.
- Add `/health` (app liveness) and `/ops/health` (ops readiness) endpoints.

Scope (concrete files/lines):
- New: `ops/*.sh` (~150 lines total), `server/src/routes/health.ts`, `server/src/routes/ops.ts`, `server/src/routes/punch.ts` (or extend existing), `server/src/db/schema.sql` (add constraint/index).
- Update: `.env.example`, main app wiring.

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."

ENV_FILE=".env"
if [ ! -f "$ENV_FILE" ]; then
  echo "ERROR: $ENV_FILE not found"
  exit 1
fi

# shellcheck disable=SC1090
source "$ENV_FILE"

required=(
  DB_HOST DB_PORT DB_USER DB_PASSWORD DB_NAME
  LINE_CHANNEL_ACCESS_TOKEN
  LINE_CHANNEL_SECRET
  JWT_SECRET
)

missing=()
for v in "${required[@]}"; do
  val=$(eval echo \$$v)
  if [ -z "${val:-}" ]; then
    missing+=("$v")
  fi
done

if [ ${#missing[@]} -gt 0 ]; then
  echo "ERROR: Missing required env vars: ${missing[*]}"
  exit 1
fi

# Sensible defaults
export TZ=${TZ:-Asia/Bangkok}
export LOG_LEVEL=${LOG_LEVEL:-info}

echo "OK: env check passed (TZ=$TZ)"
```

`/opt/axentx/workio/ops/health.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."

# shellcheck disable=SC1090
[ -f .env ] && source .env

API_URL=${API_URL:-http://localhost:3000}
OPS_URL=${OPS_URL:-http://localhost:3000/ops/health}
TIMEOUT=${HEALTH_TIMEOUT:-5}

check_db() {
  if command -v pg_isready >/dev/null 2>&1; then
    pg_isready -h "${DB_HOST:-localhost}" -p "${DB_PORT:-5432}" -U "${DB_USER:-postgres}" -d "${DB_NAME:-workio}" -t "$TIMEOUT" >/dev/null
  else
    node -e "
      const { Client } = require('pg');
      const c = new Client({
        host: process.env.DB_HOST || 'localhost',
        port: process.env.DB_PORT || 5432,
        user: process.env.DB_USER || 'postgres',
        password: process.env.DB_PASSWORD || '',
        database: process.env.DB_NAME || 'workio',
      });
      c.connect().then(() => c.end()).catch((e) => { console.error(e); process.exit(1); });
    " >/dev/null 2>&1
  fi
}

check_line_config() {
  if [ -z "${LINE_CHANNEL_ACCESS_TOKEN:-}" ] || [ -z "${LINE_CHANNEL_SECRET:-}" ]; then
    echo "WARN: LINE config missing"
    return 1
  fi
  return 0
}

check_api_health() {
  if command -v curl >/dev/null 2>&1; then
    curl -fs --max-time "$TIMEOUT" "$API_URL/health" >/dev/null
  else
    node -e "
      const http = require('http');
      const req = http.get('$API_URL/health', (res) => {
        if (res.statusCode === 200) process.exit(0);
        else process.exit(1);
      });
      req.on('error', () => process.exit(1));
      req.setTimeout($TIMEOUT * 1000, () => { req.destroy(); });
    " >/dev/null 2>&1
  fi
}

check_ops_health() {
  if command -v curl >/dev/null 2>&1; then
    curl -fs --max-time "$TIMEOUT" "$OPS_URL" >/dev/null
  else
    node -e "
      const http = require('http');
      const req = http.get('$OPS_URL', (res) => {
        if (res.statusCode === 200) process.exit(0);
        else process.exit(1);
      });
      req.on('error', () => process.exit(1));
      req.setTimeout($TIMEOUT * 1000, () => { req.destroy(); });
    " >/dev/null 2>&1
  fi
}

echo "Running health checks..."
check_db && echo "DB: OK" || { echo "DB: FAIL"; exit 1; }
check_line_config && echo "LINE config: OK" || { echo "LINE config: WARN"; }
check_api_health && echo "API /health: OK" || { echo "API /health: FAIL"; exit 1; }
check_ops_health && echo "OPS /ops/health: OK" || { echo "OPS /ops/health: FAIL"; exit 1; }
echo "All checks passed"
```

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."

# shellcheck disable=SC1090
[ -f .env ] && source .env

DB_HOST=${DB_HOST:-localhost}
DB_PORT=${DB_PORT:-5432}
DB_USER=${DB_USER:-postgres}
DB_NAME=${DB_NAME:-workio}
SCHEMA_FILE="server/src/db/schema.sql"

if [ ! -f "$SCHEMA_FILE" ]; then
  echo "ERROR: $SCHEMA_FILE not found"
  exit 1
fi

echo "Applying schema (idempotent)..."
# Schema should use IF NOT EXISTS where appropriate.
PGPASSWORD="${DB_PASSWORD:-}" psql -h "$DB_HOST" -p "$DB_PORT" -U "$DB_USER" -d "$DB_NAME" -v ON_ERROR_STOP=1 -f "$SCHEMA_FILE"
echo "Schema applied"
```
