## Deployment
hybrid (client binaries distributed via GitHub Releases + Cloudflare Workers for config/geo-fallback endpoints; no server infra required for core stealth path)

## v0.1 scope
v0.1 in 2 weeks: single-file Rust CLI client (Linux x86_64) with randomized local SOCKS5 listener, shadowsocks-obfuscated upstream, in-memory/boringtun TUN fallback, local SQLite profiles, fail-closed killswitch (best-effrict without root), and anonymized client-side telemetry to a CF Worker endpoint; no server required, installable via static binary or portable archive.

## Open questions
- Which obfuscation transports provide the best reach/stealth trade-off against current DPI in target regions (shadowsocks 2022, meek, TLS-fragment, or QUIC masquerade)?
- How will we distribute updated obfuscation profiles to clients without centralized infrastructure that can be blocked?
- What telemetry schema and consent flow balances WAU-ZB measurement with privacy and deniability?
- How do we implement reliable kill-switch semantics across desktop OSes (Windows, Linux, macOS) without admin/root?
- Do we need kernel-bypass (e.g., AF_XDP) for performance or is userspace TUN sufficient for typical client throughput?
