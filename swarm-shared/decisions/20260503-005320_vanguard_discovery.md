# vanguard / discovery

## Final Synthesized Solution

### Diagnosis (merged)
- **No persisted `(repo, dateFolder)` manifest** → every training run re-enumerates via authenticated HF API, burning quota and risking 429.
- **Recursive enumeration / pagination** amplifies rate-limit pressure and exposes mixed-schema files, causing `pyarrow.CastError`.
- **Training uses `load_dataset(streaming=True)` on heterogeneous repo paths**, triggering schema-cast hazards.
- **Lightning Studio reuse is missing** → each run recreates studio and burns quota.
- **Data ingestion writes enriched files with extra metadata columns** (`source`, `ts`) instead of clean `{prompt,response}`, violating surrogate-1 schema rules.

### Proposed Change (merged + hardened)
Create a discovery-time manifest generator and a hardened training launcher for `/opt/axentx/vanguard`:

1. **`vanguard/manifest.py`** — one-shot script that calls `list_repo_tree(path, recursive=False)` for a given `repo/dateFolder`, emits `manifests/{repo}__{date}.json` (list of file paths + checksums).
2. **`vanguard/train.py`** — Lightning entrypoint that:
   - Reads the manifest and downloads via HF CDN (`resolve/main/...`) with **zero authenticated API calls** during training.
   - Projects to `{prompt, response}` only at parse time (surrogate-1 schema).
   - Validates schema and skips malformed lines; uses streaming to bound memory.
   - Supports resumable training via deterministic shard ordering and optional caching.
3. **`vanguard/studio_launcher.py`** — reuses a running Lightning Studio if present (name match), else creates one (L40S, fallback to public tier), then runs `train.py`.
4. **Replace any recursive `list_repo_files` call sites** with manifest-based flow.

# /opt/axentx/vanguard/manifest.py
#!/usr/bin/env python3
"""
Generate (repo, dateFolder) manifest once to avoid HF API pagination/rate-limit.
Usage:
  python manifest.py --repo datasets/myorg/vanguard-data --date 2026-05-03 --out manifests/
"""
import argparse
import hashlib
import json
import os
from pathlib import Path

from huggingface_hub import HfApi

def _sha256_bytes(b: bytes) -> str:
    return hashlib.sha256(b).hexdigest()

def build_manifest(repo: str, date_folder: str, out_dir: str):
    api = HfApi()
    # Non-recursive, one page per folder (fast, low quota)
    tree = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)
    files = [item.rfilename for item in tree if item.type == "file"]

    out_path = Path(out_dir)
    out_path.mkdir(parents=True, exist_ok=True)
    slug = repo.replace("/", "__")
    manifest_file = out_path / f"{slug}__{date_folder}.json"

    # Optional: include size/sha256 for integrity checks (best-effort; skip if large)
    entries = []
    for rpath in files:
        entry = {"path": rpath}
        try:
            metadata = api.metadata(repo, path=rpath)
            entry["size"] = metadata.size
        except Exception:
            entry["size"] = None
        entries.append(entry)

    manifest = {"repo": repo, "date": date_folder, "files": entries}
    manifest_file.write_text(json.dumps(manifest, indent=2))
    print(f"Manifest written: {manifest_file} ({len(entries)} files)")
    return manifest_file

if __name__ == "__main__":
    p = argparse.ArgumentParser()
    p.add_argument("--repo", required=True)
    p.add_argument("--date", required=True)
    p.add_argument("--out", default="manifests")
    args = p.parse_args()
    build_manifest(args.repo, args.date, args.out)

import os
import time
from pathlib import Path
from typing import Iterator

import requests
import torch
from datasets import Dataset, IterableDataset
from lightning import Fabric

MANIFEST_PATH = os.getenv(
    "VANGUARD_MANIFEST",
    "manifests/datasets__myorg__vanguard-data__2026-05-03.json"
)
CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"
CACHE_DIR = os.getenv("VANGUARD_CACHE_DIR", ".cache/vanguard")

class CDNJsonlIterable(IterableDataset):
    def __init__(self, manifest_path: str, max_retries: int = 3, timeout: int = 30):
        with open(manifest_path) as f:
            meta = json.load(f)
        self.repo = meta["repo"]
        self.files = [e["path"] for e in meta["files"]]
        self.max_retries = max_retries
        self.timeout = timeout

    def _stream_files(self) -> Iterator[dict]:
        for rpath in self.files:
            url = CDN_TEMPLATE.format(repo=self.repo, path=rpath)
            for attempt in range(self.max_retries):
                try:
                    resp = requests.get(url, timeout=self.timeout, stream=True)
                    resp.raise_for_status()
                    break
                except Exception as exc:
                    if attempt == self.max_retries - 1:
                        print(f"Failed to fetch {url}: {exc}")
                        break
                    time.sleep(1.0 * (attempt + 1))
            else:
                continue

            # Stream line-by-line to bound memory
            for chunk in resp.iter_lines(decode_unicode=True):
                line = chunk.strip() if chunk else ""
                if not line:
                    continue
                try:
                    obj = json.loads(line)
                except Exception:
                    continue

                # Project to {prompt, response} only (surrogate-1 schema)
                prompt = obj.get("prompt") or obj.get("input") or obj.get("text")
                response = obj.get("response") or obj.get("output") or obj.get("completion")
                if prompt is None or response is None:
                    continue
                yield {"prompt": str(prompt), "response": str(response)}

    def __iter__(self):
        return self._stream_files()
