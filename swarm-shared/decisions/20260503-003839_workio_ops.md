# workio / ops

## 1) Diagnosis

- Missing **idempotency key** on `/webhook/line` POST allows LINE retries (network blips, 5xx, slow client) to create duplicate punch records.
- No **partial unique index** enforcing “one open punch per user” (`clock_out_at IS NULL`) permits multiple concurrent clock-ins and corrupts timesheets.
- No **transactional upsert** on punch creation; race between duplicate webhook deliveries can insert two rows for same user+timestamp window.
- No **idempotency table** to deduplicate by LINE `deliveryId`/`timestamp`+`userId`; retries bypass application-level checks under concurrency.
- Missing **ops alerting** on duplicate-punch anomalies (counts/rates) means regressions go undetected until payroll/reports are wrong.

## 2) Proposed change

File scope:  
- `workio/server/src/db/schema.sql` — add partial unique index and idempotency table.  
- `workio/server/src/routes/webhook/line.ts` (or equivalent) — add idempotency check and transactional upsert for punches.  
- `workio/server/src/middleware/idempotency.ts` — small middleware/helper to enforce idempotency by `deliveryId`+`userId`.

## 3) Implementation

```sql
-- workio/server/src/db/schema.sql
-- Idempotency table for LINE webhook deliveries
CREATE TABLE IF NOT EXISTS webhook_deliveries (
  id              BIGSERIAL PRIMARY KEY,
  delivery_id     TEXT NOT NULL,          -- LINE deliveryId (or X-Line-DeliveryId)
  user_id         BIGINT NOT NULL REFERENCES users(id),
  event_type      TEXT NOT NULL,          -- 'clock_in', 'clock_out', etc.
  payload_hash    TEXT NOT NULL,          -- sha256 of normalized payload for safety
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uniq_delivery_user UNIQUE (delivery_id, user_id)
);

-- Partial unique index: one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;
```

```ts
// workio/server/src/middleware/idempotency.ts
import { PoolClient } from 'pg';
import crypto from 'crypto';

export function payloadHash(payload: unknown): string {
  return crypto.createHash('sha256').update(JSON.stringify(payload)).digest('hex');
}

export async function ensureIdempotent(
  client: PoolClient,
  deliveryId: string,
  userId: number,
  eventType: string,
  payload: unknown
): Promise<boolean> {
  const hash = payloadHash(payload);
  const q = `
    INSERT INTO webhook_deliveries (delivery_id, user_id, event_type, payload_hash)
    VALUES ($1, $2, $3, $4)
    ON CONFLICT (delivery_id, user_id) DO UPDATE
      SET payload_hash = EXCLUDED.payload_hash
    WHERE webhook_deliveries.payload_hash <> EXCLUDED.payload_hash
    RETURNING true;
  `;
  const res = await client.query(q, [deliveryId, userId, eventType, hash]);
  // If row existed with same hash, insert did nothing and row count 0 => duplicate
  // We use RETURNING to detect change; if no rows returned it's a replay with same hash.
  return res.rowCount > 0;
}
```

```ts
// workio/server/src/routes/webhook/line.ts  (or wherever the handler lives)
import { Router } from 'express';
import { pool } from '../../db';
import { ensureIdempotent } from '../../middleware/idempotency';

const router = Router();

router.post('/line', async (req, res) => {
  const deliveryId = req.header('X-Line-DeliveryId') || req.body.deliveryId;
  const events = req.body.events || [];

  if (!deliveryId) {
    return res.status(400).json({ error: 'Missing delivery ID' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      if (ev.type !== 'message' && ev.type !== 'postback') continue;

      // Extract user and action — adapt to your LINE payload shape
      const userId = ev.source?.userId;
      const action = ev.postback?.data || ev.message?.text;
      if (!userId) continue;

      // Resolve internal user id (example)
      const userRes = await client.query('SELECT id FROM users WHERE line_user_id = $1', [userId]);
      if (userRes.rowCount === 0) continue;
      const internalUserId = userRes.rows[0].id;

      // Idempotency guard
      const isNew = await ensureIdempotent(client, deliveryId, internalUserId, 'punch', ev);
      if (!isNew) {
        // Duplicate delivery — skip punch creation but continue processing other events
        continue;
      }

      // Transactional upsert: close open punch if exists, else insert new open punch
      // Using CTE to enforce one-open-per-user under SERIALIZABLE/REPEATABLE-READ safely
      const upsertQ = `
        WITH closed AS (
          UPDATE punches
          SET clock_out_at = NOW(), updated_at = NOW()
          WHERE user_id = $1 AND clock_out_at IS NULL
          RETURNING id
        )
        INSERT INTO punches (user_id, clock_in_at, clock_out_at, location, created_at, updated_at)
        SELECT $1, NOW(), NULL, $2, NOW(), NOW()
        WHERE NOT EXISTS (SELECT 1 FROM closed)
        RETURNING id;
      `;
      // location could come from ev or separate payload — adapt as needed
      await client.query(upsertQ, [internalUserId, ev.location || null]);
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error', err);
    // Important: return 5xx so LINE may retry (idempotency prevents duplicates)
    res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

1. **Schema check**

   ```bash
   psql workio -c "\d idx_punches_one_open_per_user"
   ```
   Confirm partial index exists and idempotency table is present.

2. **Duplicate prevention test**  
   - Simulate two identical POSTs to `/webhook/line` with same `X-Line-DeliveryId` and same user event.  
   - Assert only one punch row created and second request returns 200 without error.

3. **One-open-punch constraint test**  
   - Clock in for a user (creates open punch).  
   - Attempt second clock in for same user (should fail or be ignored depending on UX).  
   - Clock out should succeed and close punch; subsequent clock in should create new row.

4. **Race-condition test**  
   - Fire 10 concurrent identical webhook requests (same deliveryId+userId).  
   - Verify exactly one punch row created and no constraint violations.

5. **Ops alert smoke test** (optional)  
   - Add a quick query to count duplicates per hour and log/alert if >0; confirm it stays at 0 after above tests.
