# workio / quality

## 1. Diagnosis
- No database-level uniqueness constraint on punches → duplicates from LINE webhook retries and double-taps.
- Application-level dedupe is best-effort only; race conditions between concurrent clock-in/out requests are possible.
- Idempotency key (from LINE `X-Line-Signature` or client-generated) is not stored or enforced → retries create duplicate rows.
- Missing defensive transition checks (e.g., clock-in when already clocked-in) rely on client logic and can be bypassed.
- No compound index to efficiently enforce latest-punch lookups and tenant scoping; queries may scan unnecessarily.

## 2. Proposed change
- **Files**:  
  - `workio/server/src/db/schema.sql` — add unique constraint and indexes.  
  - `workio/server/src/services/punchService.ts` (or equivalent) — add idempotency key handling and strict state transition guard.  
  - `workio/server/src/routes/attendance.ts` (or webhook/line route) — accept/require idempotency key and return 409 on duplicate.

## 3. Implementation

### 3.1 Database schema (enforce uniqueness + indexes)
```sql
-- workio/server/src/db/schema.sql
-- Ensure punches table exists; then add constraints/indexes

-- Idempotency: one punch per tenant+employee+idempotency_key (provided by client/LINE)
ALTER TABLE punches
  ADD COLUMN idempotency_key TEXT NULL;

-- Prevent duplicate active punches for same tenant+employee when clocked_in_at is NULL (active punch)
-- Allow multiple historical rows (clocked_in_at NOT NULL) but prevent overlapping active punches
CREATE UNIQUE INDEX idx_punches_active_unique
  ON punches (tenant_id, employee_id)
  WHERE clocked_out_at IS NULL;

-- Fast lookup for latest punch per employee
CREATE INDEX idx_punches_latest
  ON punches (tenant_id, employee_id, created_at DESC);

-- Idempotency uniqueness (optional strict mode: one request per key)
CREATE UNIQUE INDEX idx_punches_idempotency_unique
  ON punches (tenant_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL;
```

### 3.2 Service layer: strict transition + idempotency
```typescript
// workio/server/src/services/punchService.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export type PunchAction = 'clock_in' | 'clock_out';

export async function handlePunch({
  tenantId,
  employeeId,
  action,
  idempotencyKey,
  latitude,
  longitude,
}: {
  tenantId: string;
  employeeId: string;
  action: PunchAction;
  idempotencyKey?: string;
  latitude?: number;
  longitude?: number;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency check
    if (idempotencyKey) {
      const idem = await client.query(
        'SELECT id FROM punches WHERE tenant_id = $1 AND idempotency_key = $2',
        [tenantId, idempotencyKey]
      );
      if (idem.rows.length > 0) {
        // Return existing punch to caller (safe for retries)
        await client.query('ROLLBACK');
        return { existing: true, punch: idem.rows[0] };
      }
    }

    // Fetch latest punch
    const latest = await client.query(
      `SELECT id, clocked_out_at
       FROM punches
       WHERE tenant_id = $1 AND employee_id = $2
       ORDER BY created_at DESC
       LIMIT 1`,
      [tenantId, employeeId]
    );

    const last = latest.rows[0];

    // Transition guards
    if (action === 'clock_in' && last && !last.clocked_out_at) {
      await client.query('ROLLBACK');
      throw new Error('ALREADY_CLOCKED_IN');
    }
    if (action === 'clock_out' && (!last || last.clocked_out_at)) {
      await client.query('ROLLBACK');
      throw new Error('NO_ACTIVE_PUNCH');
    }

    // Insert punch
    const now = new Date();
    const insertValues = [
      tenantId,
      employeeId,
      action === 'clock_in' ? now : null,
      action === 'clock_out' ? now : null,
      latitude ?? null,
      longitude ?? null,
      idempotencyKey ?? null,
      now,
      now,
    ];

    const insertSQL = `
      INSERT INTO punches (
        tenant_id, employee_id, clocked_in_at, clocked_out_at,
        latitude, longitude, idempotency_key, created_at, updated_at
      ) VALUES ($1,$2,$3,$4,$5,$6,$7,$8,$9)
      RETURNING *
    `;
    const result = await client.query(insertSQL, insertValues);

    await client.query('COMMIT');
    return { existing: false, punch: result.rows[0] };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 3.3 Route/webhook: require idempotency and handle conflicts
```typescript
// workio/server/src/routes/attendance.ts
import express from 'express';
import { handlePunch } from '../services/punchService';

const router = express.Router();

router.post('/punch', async (req, res) => {
  const { tenantId, employeeId, action, latitude, longitude } = req.body;
  const idempotencyKey = req.header('Idempotency-Key') || req.body.idempotencyKey;

  if (!tenantId || !employeeId || !action) {
    return res.status(400).json({ error: 'Missing required fields' });
  }

  try {
    const result = await handlePunch({
      tenantId,
      employeeId,
      action,
      idempotencyKey,
      latitude,
      longitude,
    });

    if (result.existing) {
      return res.status(200).json({ message: 'Duplicate ignored', punch: result.punch });
    }
    return res.status(201).json({ punch: result.punch });
  } catch (err: any) {
    if (err.message === 'ALREADY_CLOCKED_IN') {
      return res.status(409).json({ error: 'Already clocked in' });
    }
    if (err.message === 'NO_ACTIVE_PUNCH') {
      return res.status(409).json({ error: 'No active punch to clock out' });
    }
    console.error(err);
    return res.status(500).json({ error: 'Internal server error' });
  }
});

export default router;
```

## 4. Verification
- **Duplicate prevention**: send same `Idempotency-Key` twice → second request returns 200 with same punch row, no new row inserted.
- **Active punch uniqueness**: attempt to clock in twice without clocking out → second request returns 409 `ALREADY_CLOCKED_IN`.
- **Transition correctness**: clock out without active punch → 409 `NO_ACTIVE_PUNCH`.
- **Index usage**: run `EXPLAIN` on latest-punch query and confirm `idx_punches_latest` is used.
- **Rollback on error**: simulate failure during insert (e.g., throw in service) and confirm no partial row persisted and transaction rolled back.
