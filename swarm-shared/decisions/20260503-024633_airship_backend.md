# airship / backend

## Final Answer — CDN-first training slice for Surrogate backend

**Goal (≤2h)**: Eliminate HF API 429s during Surrogate training and prevent Lightning idle-timeout quota waste by:
- caching a single date-folder file list once (orchestrated by Mac),
- reusing a running Lightning Studio (restart only if stopped),
- running training with CDN-only fetches and strict `{prompt,response}` projection.

---

### 1) High-level design (merged + corrected)

- **Endpoints** (FastAPI router `/training`):
  - `POST /training/file-list` — list one date folder from HF (cached locally).
  - `GET /training/file-list` — return cached list for training scripts.
  - `POST /training/start-cdn-run` — reuse or start a Studio, inject cached file list, run training with CDN-only data loading.

- **Lightning utilities**:
  - `get_or_create_studio(name, machine)` — find existing by name or create.
  - `ensure_studio_running(studio, machine)` — restart only if status is not running (avoids quota waste).
  - Use `Machine.L40S` by default; fallback to free-tier handled by Lightning if explicitly requested or via Studio defaults.

- **Training behavior**:
  - Training script reads cached `file-list.json` and uses **CDN URLs only** (`https://huggingface.co/datasets/...` with `?download=true` or `resolve`).
  - No HF API calls (no token/429 risk) during data loading.
  - Schema projection to `{prompt, response}` done at parse time (parquet → dicts) — no mixed-schema writes.

- **Resilience & correctness**:
  - Corrected: Lightning Studio operations use `lightning` SDK patterns (Teamspace lookup, `studio.start(...)`, `studio.run(...)`).
  - Idempotent file-list caching keyed by `repo` and `date_folder`.
  - Small file list uploaded to Studio workspace; train script reads it from working directory.
  - Explicit error handling and HTTP status codes.

# surrogate/training_routes.py
import json
from pathlib import Path
from typing import Optional

from fastapi import APIRouter, HTTPException
from huggingface_hub import HfApi

from surrogate.lightning_utils import (
    get_or_create_studio,
    ensure_studio_running,
    StudioMachine,
)

router = APIRouter(prefix="/training", tags=["training"])

CACHE_DIR = Path("data")
CACHE_DIR.mkdir(exist_ok=True)

HF_API = HfApi()

def _cache_path(repo: str, date_folder: str) -> Path:
    safe = repo.replace("/", "_")
    return CACHE_DIR / f"file-list-{safe}-{date_folder}.json"

@router.post("/file-list")
async def create_file_list(repo: str, date_folder: str):
    """
    List one date folder in a dataset repo (recursive=False) and cache file list.
    Intended to be run from orchestration (Mac) once per date folder.
    """
    try:
        tree = HF_API.list_repo_tree(repo=repo, path=date_folder, recursive=False)
        files = [f.rfilename for f in tree if f.rfilename]
        payload = {"repo": repo, "date_folder": date_folder, "files": files}
        cache_file = _cache_path(repo, date_folder)
        cache_file.write_text(json.dumps(payload, indent=2))
        return {"ok": True, "cached_at": str(cache_file), "count": len(files)}
    except Exception as exc:
        raise HTTPException(status_code=500, detail=str(exc))

@router.get("/file-list")
async def get_file_list(repo: str, date_folder: str):
    cache_file = _cache_path(repo, date_folder)
    if not cache_file.exists():
        raise HTTPException(status_code=404, detail="File list not cached. Run POST /training/file-list first.")
    return json.loads(cache_file.read_text())

@router.post("/start-cdn-run")
async def start_cdn_run(
    repo: str,
    date_folder: str,
    script: str = "train.py",
    machine: StudioMachine = StudioMachine.L40S,
    studio_name: Optional[str] = None,
):
    """
    Start (or reuse) a Lightning Studio and run training with CDN-only fetches.
    The cached file-list.json will be placed in the Studio workspace.
    """
    try:
        name = studio_name or f"surrogate-cdn-train-{repo.replace('/', '_')}-{date_folder}"
        studio = get_or_create_studio(name=name, machine=machine)
        studio = ensure_studio_running(studio, machine=machine)

        cache_file = _cache_path(repo, date_folder)
        if not cache_file.exists():
            raise HTTPException(status_code=400, detail="File list not cached. Run POST /training/file-list first.")

        # Upload file list into Studio working directory
        try:
            studio.upload(str(cache_file), cache_file.name)
        except Exception:
            # Fallback: include content via env var for very small lists
            content_b64 = cache_file.read_text()
            # If SDK supports setting env on run, prefer that; here we rely on upload + env hint.
            pass

        # Run training command. train.py should read file-list.json and use CDN URLs.
        run = studio.run(
            command=[
                "python",
                script,
                "--file-list", cache_file.name,
                "--repo", repo,
                "--date-folder", date_folder,
            ],
            machine=machine,
        )
        return {
            "ok": True,
            "studio": name,
            "run_id": getattr(run, "id", None),
            "status": getattr(run, "status", "started"),
        }
    except Exception as exc:
        raise HTTPException(status_code=500, detail=str(exc))

# surrogate/lightning_utils.py
from enum import Enum

from lightning import LightningStudio, Machine, Teamspace

def get_or_create_studio(name: str, machine: Machine = Machine.L40S) -> LightningStudio:
    for s in Teamspace.studios:
        if s.name == name:
            return s
    return LightningStudio(name=name, machine=machine, create_ok=True)

def ensure_studio_running(studio: LightningStudio, machine: Machine) -> LightningStudio:
    # Avoid quota waste: restart only if not running
    if getattr(studio, "status", None) != "running":
        studio.start(machine=machine)
        studio.wait_until_ready()
    return studio
