## v0.1 scope
v0.1 (2 weeks): CLI client (Linux) with hidden local SOCKS5 listener behind TLS-cover traffic to a free Cloudflare Worker egress; encrypted config stored in Supabase Postgres; client-side killswitch that blocks non-stealth interfaces on stealth-down; minimal encrypted telemetry to Supabase for WAU-ZB; unit/integration tests for stealth path and killswitch.

## Open questions
- Which DPI signatures are most prevalent in target regions (SNI, packet size distributions, timing, TTL) and what cover traffic minimizes detection?
- Can we rely solely on client-side TUN/SOCKS5 stealth without a managed egress endpoint, or do we need a free-tier relay (CF Worker + Durable Object) to provide innocuous upstream termination?
- How will we implement a reliable fail-closed killswitch across platforms (Linux/Android/Windows) without leaking real traffic during stealth path failure?
- What telemetry granularity is acceptable for WAU-ZB while preserving deniability and minimizing user-identifiable data?
- How do we distribute and rotate stealth configs to clients without a central server that becomes a blocking target (e.g., using HF space as static config CDN + local cache)?
