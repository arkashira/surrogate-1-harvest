# workio / design

## 1. Diagnosis

- **No tenant cache on LINE webhook** — every webhook hits PostgreSQL to resolve `channelId → tenant`; slow under burst traffic, noisy under load, and fails fast if DB is cold.
- **Missing health/readiness endpoints** — no `/health` or `/ready` for k8s/PM2/liveness checks; deploy/rollback is blind.
- **No graceful shutdown** — in-flight LINE webhook requests can be dropped during restart/deploy, causing missed clock events.
- **Unbounded DB connection churn** — webhook bursts can exhaust the pool; no circuit-breaker or retry/backoff for transient DB failures.
- **No request-id / trace correlation** — hard to debug cross-service flows (LINE → webhook → DB → LINE notify).

## 2. Proposed change

Add a **TTL-backed LRU tenant cache** keyed by `channelId` in the tenant service + lightweight `/health`/`/ready` endpoints and graceful shutdown in the Express server.  
Scope:  
- `workio/server/src/services/tenantService.ts` (or create if absent)  
- `workio/server/src/routes/health.ts` (new)  
- `workio/server/src/index.ts` (or main server entry)  
- `workio/server/src/middleware/requestId.ts` (new)

## 3. Implementation

### 3.1 Tenant service with TTL cache

```ts
// workio/server/src/services/tenantService.ts
import { LRUCache } from 'lru-cache';
import pool from '../db';

type Tenant = {
  id: string;
  name: string;
  channelId: string;
  settings: Record<string, any>;
};

const tenantCache = new LRUCache<string, Tenant>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5 minutes
  allowStale: false,
  updateAgeOnGet: true,
});

export async function getTenantByChannelId(channelId: string): Promise<Tenant | null> {
  const cached = tenantCache.get(channelId);
  if (cached) return cached;

  try {
    const res = await pool.query('SELECT * FROM tenants WHERE channel_id = $1 LIMIT 1', [channelId]);
    const tenant = res.rows[0] || null;
    if (tenant) tenantCache.set(channelId, tenant);
    return tenant;
  } catch (err) {
    // Fail open for resilience: return null so caller can handle gracefully
    console.error('[tenantService] failed to resolve tenant', channelId, err);
    return null;
  }
}

export function invalidateTenantCache(channelId: string) {
  tenantCache.delete(channelId);
}
```

### 3.2 Health / ready endpoints

```ts
// workio/server/src/routes/health.ts
import { Router } from 'express';
import pool from '../db';

const router = Router();

// liveness: process is alive
router.get('/health', (_, res) => res.json({ status: 'ok', uptime: process.uptime() }));

// readiness: dependencies available
router.get('/ready', async (_, res) => {
  try {
    await pool.query('SELECT 1');
    res.json({ status: 'ready' });
  } catch (err) {
    res.status(503).json({ status: 'unavailable', error: String(err) });
  }
});

export default router;
```

### 3.3 Request-id middleware (lightweight correlation)

```ts
// workio/server/src/middleware/requestId.ts
import { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';

export function requestIdMiddleware(req: Request, _: Response, next: NextFunction) {
  const id = req.headers['x-request-id'] || crypto.randomUUID();
  ;(req as any).requestId = id;
  next();
}
```

### 3.4 Wire up + graceful shutdown

```ts
// workio/server/src/index.ts (or main entry)
import express from 'express';
import cors from 'cors';
import healthRouter from './routes/health';
import lineWebhookRouter from './routes/lineWebhook';
import { requestIdMiddleware } from './middleware/requestId';

const app = express();
app.use(cors());
app.use(express.json());
app.use(requestIdMiddleware);

app.use('/webhook/line', lineWebhookRouter);
app.use('/health', healthRouter);

const PORT = process.env.PORT || 3000;
const server = app.listen(PORT, () => console.log(`Server listening on ${PORT}`));

// Graceful shutdown
function shutdown(signal: string) {
  return async () => {
    console.log(`${signal} received: closing HTTP server`);
    server.close(async (err) => {
      if (err) {
        console.error('Error during shutdown', err);
        process.exit(1);
      }
      try {
        await pool.end(); // close DB pool
        console.log('DB pool closed');
        process.exit(0);
      } catch (e) {
        console.error('Error closing DB pool', e);
        process.exit(1);
      }
    });

    // Force exit after timeout
    setTimeout(() => {
      console.error('Forced shutdown after timeout');
      process.exit(1);
    }, 10_000).unref();
  };
}

process.on('SIGTERM', shutdown('SIGTERM'));
process.on('SIGINT', shutdown('SIGINT'));
```

### 3.5 Use cache in LINE webhook

Update your LINE webhook handler to use `getTenantByChannelId(channelId)` instead of raw DB query.

## 4. Verification

1. **Cache behavior**  
   - Start server.  
   - Send two LINE webhook events with same `channelId`.  
   - Check logs: first request logs DB query; second does not.  
   - Wait >5 minutes; third request logs DB query again (TTL expiry).

2. **Health endpoints**  
   - `curl http://localhost:3000/health` → `{"status":"ok",...}`  
   - `curl http://localhost:3000/ready` → `{"status":"ready"}` when DB is up; stop PostgreSQL → `503 unavailable`.

3. **Graceful shutdown**  
   - Start server, send a webhook, then `kill -SIGTERM <pid>` while request is in-flight.  
   - Confirm server waits for active requests (up to 10s) and exits cleanly; DB pool closes.

4. **Request-id presence**  
   - Add logging in middleware and webhook to print `requestId`; confirm same ID appears across logs for one request.
