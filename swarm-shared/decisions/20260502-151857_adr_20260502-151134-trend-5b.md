## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research->signal->risk->execution) with Postgres audit trail, Redis cache for risk/signal, Cloudflare Queues for orchestration, mock broker/OMS connector, compliance guardrails as code (Pydantic rules), and replay endpoint to re-run a workflow_run from stored inputs/snapshots.

## Open questions
- Deterministic replay: which inputs and state snapshots are required to guarantee bitwise replay of a workflow run?
- Latency SLO: what is the p99 budget from signal ingestion to execution ack on free-tier infra?
- Compliance rule engine: DSL vs code-first guardrails — who authors and how are rules versioned/audited?
- Idempotency and exactly-once semantics across queue retries and venue idempotency keys
- Broker/OMS/EMS connector security: credential storage and rotation on free-tier (Supabase vault vs env)
- Replay storage cost and retention: how long to keep full snapshots vs diffs for audit/regulatory requests
