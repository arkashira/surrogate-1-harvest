## Deployment
cf workers (edge relay + telemetry) + Supabase (Postgres) + HuggingFace Spaces (free-tier CI/CD runners for reproducible builds)

## v0.1 scope
v0.1 ships a local stealth-client (Rust CLI) that creates a SOCKS5 listener, wraps traffic in meek-like TLS streams to a free-tier CF Worker relay, and enforces a fail-closed kill-switch; client-side telemetry via CF Workers Queues to Supabase Postgres for WAU-ZB; no GUI, no server management, and no persistence beyond anonymous session logs.

## Open questions
- Which cover protocols (meek vs domain-front vs QUIC vs HTTP/3) maximize reach while minimizing latency in target regions?
- How do we validate stealth locally without exposing probe endpoints that can be fingerprinted or blocked?
- What is the acceptable false-positive rate for the kill-switch to avoid unnecessary drops while guaranteeing no leaks?
- How will clients discover updated obfs_profile versions without a hardcoded endpoint that can be blocked?
- Which telemetry fields are strictly required to compute WAU-ZB while preserving user anonymity and deniability?
