## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research/signal → risk check → compliance guardrail → simulated execution) exposed via FastAPI on CF Workers, persisted to Supabase Postgres, immutable audit events with payload hashes, idempotent runs via Redis, one mock broker connector and one mock data vendor connector, latency telemetry and NSM counter (weighted by fee_generating) for dashboarding.

## Open questions
- Deterministic replay storage strategy: store full input snapshots vs event sourcing — which meets sub-second latency and free-tier storage limits?
- Broker/OMS connector auth model: will vendors support HMAC-signed webhooks or require long-lived API keys; how to rotate safely on free infra?
- Latency SLO definition for v0.1: what p95 end-to-end target (ms) is required to satisfy 'sub-second orchestration' on CF Workers + Supabase free tier?
- Compliance rule engine scope: will v0.1 use a safe embedded DSL or simple JSON rules; how to prevent arbitrary code execution while keeping deterministic evaluation?
- Audit immutability guarantees: append-only Postgres with row-level crypto hashes vs external append-only log — which satisfies regulatory checks without paid infra?
