## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research->signal->risk check->guardrail eval->execution attempt) with Postgres event store (Supabase), Redis Streams queue, Redis cache for hot limits, CF Workers edge API, immutable audit trail, replay endpoint, and two sample connectors (mock broker + mock data vendor). Includes unit/integration tests, deterministic canonicalization for replay, and basic compliance guardrail DSL.

## Open questions
- Deterministic replay fidelity: should event payloads be canonicalized JSON (sorted keys, fixed precision) to guarantee bitwise replay across environments?
- Latency SLA target for end-to-end orchestration (sub-500ms vs sub-100ms) and acceptable queueing semantics (at-least-once vs exactly-once)?
- Compliance rule change strategy: how to version and apply guardrail updates to in-flight runs without breaking determinism?
- Connector idempotency contract with brokers/OMS/EMS: which operations are idempotent and what deduplication keys are required?
- Secrets and key management for broker/OMS credentials within free-tier constraints (CF Workers KV vs Supabase edge functions vs external vault)?
