## Final Synthesized Plan (Best of Both Candidates)

**Chosen improvement:**  
Add a deterministic, read-only endpoint `GET /api/v1/cost-anomaly/signal/top` that queries the knowledge graph for today’s top hub and returns the strongest cost-anomaly signal with full context. No writes, no execution, pure sense+signal.

**Why this wins:**  
- Read-only (no migrations/infra) and reuses existing knowledge-rag/graph tooling  
- Highest near-term value: surfaces the most-connected insight before planning tasks  
- Fast to ship (~1–1.5h) and aligns with “Sense + Signal” philosophy  
- Concrete, actionable, and testable

### KnowledgeRAG helper

```python
# app/services/knowledge_rag.py
from typing import Optional, Dict, Any, List
from app.graph.client import KnowledgeGraphClient

class KnowledgeRAG:
    def __init__(self, graph: Optional[KnowledgeGraphClient] = None):
        self.graph = graph or KnowledgeGraphClient()

    async def top_hub_by_signal(
        self,
        tag: str,
        date: str,
        limit: int = 1,
    ) -> Optional[Dict[str, Any]]:
        """
        Find the most-connected hub for `tag` on `date` with strongest signal.
        Uses existing graph indices: Hub-(TAGGED)->Tag, Hub-(HAS_SIGNAL)->Signal
        """
        query = """
        MATCH (h:Hub)-[:TAGGED]->(:Tag {name: $tag})
        WHERE date(h.created_at) = date($date)
        OPTIONAL MATCH (h)-[:HAS_SIGNAL]->(s:Signal)
        WITH h, max(s.strength) as signal_strength
        ORDER BY signal_strength DESC
        LIMIT $limit
        RETURN h.id AS id, h.name AS name,
               signal_strength,
               [(h)-[:CONTAINS]->(e:Entity) | e.name] AS entities,
               h.summary AS summary
        """
        results = await self.graph.run(query, {"tag": tag, "date": date, "limit": limit})
        if not results:
            return None
        top = results[0]
        return {
            "id": top["id"],
            "name": top["name"],
            "signal_strength": top["signal_strength"] or 0.0,
            "entities": top["entities"] or [],
            "summary": top["summary"] or "",
        }
```
