## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service that authenticates to one AWS account and one GCP project, lists Lambda + Cloud Functions with cron/rate triggers, computes next-run times, detects duplicates (same cron+target) and orphans (schedules pointing to missing functions), persists results to Supabase Postgres, and exposes a minimal JSON API for results and audit events.

## Open questions
- Which identity/assumed-role model will we use to read Lambda schedules across customer accounts (cross-account role vs customer-provided tokens)?
- How frequently should we refresh schedules (5m vs 1h) to balance freshness and cost on free tiers?
- Will customers prefer a read-only auditor or also allow automated retirement of stale schedules (opt-in vs opt-out)?
