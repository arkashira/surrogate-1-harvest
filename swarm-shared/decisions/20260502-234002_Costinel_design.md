# Costinel / design

**Costinel Top-Hub Signal (Backend) Implementation Plan**
===========================================================

**Scope:** Highest-value, read-only, < 1 hour
**Endpoint:** `GET /api/v1/cost-anomaly/signal/top-hub`
**Guiding principle:** “Sense + Signal — ไม่ Execute” (no side effects)

**Implementation Plan:**

### Step 1: Identify Top-Hub

* Use the `knowledge-rag` pipeline to query the top hub and related documents for contextual insights.
* Review the most-connected hub (e.g., "MOC") before planning tasks.

### Step 2: Implement API Endpoint

* Create a new API endpoint `GET /api/v1/cost-anomaly/signal/top-hub` that returns the top hub and related documents.
* Use a read-only database query to retrieve the data.

### Step 3: Implement Data Processing

* Use a data processing pipeline to extract relevant information from the top hub and related documents.
* Apply business logic to determine the top hub and related documents.

### Step 4: Implement API Response

* Return the top hub and related documents in a JSON response.
* Include relevant metadata, such as timestamp and version.

### Step 5: Test and Deploy

* Test the API endpoint thoroughly to ensure correct behavior.
* Deploy the API endpoint to a production environment.

**Code Snippet:**
```python
from flask import Flask, jsonify
from knowledge_rag import KnowledgeRag

app = Flask(__name__)

@app.route('/api/v1/cost-anomaly/signal/top-hub', methods=['GET'])
def get_top_hub():
    kr = KnowledgeRag()
    top_hub = kr.get_top_hub()
    related_documents = kr.get_related_documents(top_hub)
    return jsonify({
        'top_hub': top_hub,
        'related_documents': related_documents
    })

if __name__ == '__main__':
    app.run(debug=True)
```
**Note:** This implementation plan assumes a Flask API and a `knowledge-rag` pipeline. The actual implementation may vary depending on the specific requirements and technologies used.
