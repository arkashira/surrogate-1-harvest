## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP (FastAPI server optional) that connects to one AWS account and one GCP project, lists all Lambda + Cloud Functions schedules, computes next_run_at via croniter, flags stale schedules (no last_run_at within 90 days) and exact-duplicate groups, exports CSV/JSON report, and retires candidates via dry-run flag (no live deletes). Hosted on Cloudflare Workers with Supabase free tier for metadata storage.

## Open questions
- Which IAM scopes are minimally required to list/describe Lambda schedules and Cloud Functions triggers across customer accounts (assume read-only auditor role)?
- How will customers grant cross-account/cloud access (AWS OIDC/GCP service account keys) in a cloud-free/zero-infra onboarding flow?
- What staleness threshold (e.g., no last_run_at in 90 days) and drift heuristics (cron changed but resource not updated) should v0.1 use?
- Will duplicate detection operate on exact cron match or semantic equivalence (e.g., same effective cadence across timezones)?
