# airship / design

## Implementation Plan: `airship discover` — Zero-config discovery CLI

**Highest-value incremental improvement (<2h):**  
Ship a single Bash CLI (`airship discover`) that operationalizes past patterns (business research + knowledge-rag + top-hub insight) and surfaces strategic context for Arkship/Surrogate platform decisions.

### Design
- **Entrypoint**: `/opt/axentx/airship/bin/airship` (symlink or script) with subcommand `discover`
- **Behavior**:
  1. Run `granite-business-research.sh` (if present) to generate market/competitor context.
  2. Invoke `knowledge-rag` CLI (or direct script) to query top hub and related docs.
  3. Print concise strategic context: top hub (e.g., MOC), key insights, and recommended next actions.
- **Zero-config**: Uses convention over configuration — looks for scripts in `./scripts/` or `./bin/` and falls back to safe defaults.

### File changes
1. `/opt/axentx/airship/bin/airship` — main CLI dispatcher (new)
2. `/opt/axentx/airship/bin/airship-discover` — implementation of `discover` (new)
3. Update repo root README with usage snippet.

```bash
#!/usr/bin/env bash
# /opt/axentx/airship/bin/airship-discover
set -euo pipefail

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
export AIRSHIP_ROOT="${AIRSHIP_ROOT:-$ROOT_DIR}"
export SHELL=/bin/bash

log() { echo "[airship:discover] $*" >&2; }

run_business_research() {
  local script="${AIRSHIP_ROOT}/scripts/granite-business-research.sh"
  if [[ -x "${script}" ]]; then
    log "Running business research..."
    bash "${script}" || log "WARN: business research exited non-zero ($?)"
  else
    log "No executable script found at ${script} — skipping business research."
  fi
}

run_knowledge_rag() {
  # Prefer CLI if available; otherwise try common script names.
  local rag_cmd=""
  if command -v knowledge-rag &>/dev/null; then
    rag_cmd="knowledge-rag"
  elif [[ -x "${AIRSHIP_ROOT}/scripts/knowledge-rag" ]]; then
    rag_cmd="${AIRSHIP_ROOT}/scripts/knowledge-rag"
  elif [[ -x "${AIRSHIP_ROOT}/bin/knowledge-rag" ]]; then
    rag_cmd="${AIRSHIP_ROOT}/bin/knowledge-rag"
  fi

  if [[ -n "${rag_cmd}" ]]; then
    log "Querying knowledge-rag for top hub and related docs..."
    # Try common query patterns; tolerate failures.
    if ! ${rag_cmd} query "top hub" 2>/dev/null; then
      ${rag_cmd} "top hub" 2>/dev/null || log "WARN: knowledge-rag query failed."
    fi
  else
    log "knowledge-rag not found — skipping RAG query."
  fi
}

show_top_hub_insight() {
  # If Neo4j or graph tooling is available, attempt to extract most-connected hub.
  # Fallback: read from a known file or print guidance.
  local hub_file="${AIRSHIP_ROOT}/knowledge/top-hub.txt"
  if [[ -f "${hub_file}" ]]; then
    log "Top hub (from ${hub_file}):"
    cat "${hub_file}"
  else
    log "Top hub insight: review the most-connected hub (e.g., MOC) in your knowledge graph before planning tasks."
    log "Tags: #knowledge-rag #graph #hub"
  fi
}

main() {
  log "Starting zero-config strategic discovery..."
  run_business_research
  run_knowledge_rag
  show_top_hub_insight
  log "Discovery complete. Use insights to prioritize Arkship/Surrogate work."
}

main "$@"
```

### Notes & Best Practices Applied
- Bash shebang and `set -euo pipefail` for robustness.
- `SHELL=/bin/bash` set in script (aligns with cron/wrapper lessons).
- Tolerates missing scripts/tools — logs warnings instead of failing.
- Zero-config: relies on convention (`scripts/`, `bin/`) and safe fallbacks.
- Small, incremental (<2h) and immediately useful for strategic decisions.
