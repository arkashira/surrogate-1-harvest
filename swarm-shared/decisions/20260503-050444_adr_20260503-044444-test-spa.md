## Deployment

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on Cloudflare Workers (or local dev) that discovers Lambda and Cloud Functions schedules into Supabase Postgres, detects duplicates (identical cron+target), drifted schedules (expression changed), and orphaned schedules (target missing), exposes findings via REST API and a simple JSON report; includes one-click auth via customer-supplied keys and a scheduled discovery job (manual trigger for launch).

## Open questions
- Which cross-account/project auth model will customers accept (assume-role, workload identity, or customer-supplied keys)?
- How frequently should discovery jobs run to balance freshness vs. rate limits/costs?
- What retention window for schedule history is sufficient for drift detection while staying free-tier friendly?
- Will findings be surfaced via webhook/email or only API/CLI in v0.1?
