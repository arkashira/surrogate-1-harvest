# vanguard / design

## Final Synthesis (Correctness + Actionability)

I merged the strongest, non-contradictory parts from both proposals and resolved conflicts in favor of **correctness** (public CDN, no auth during training/frontend fetches) and **concrete actionability** (single repo layout, minimal diff, clear verification).

---

## 1. Diagnosis (merged)

- No persisted `(repo, dateFolder)` manifest → frontend/training re-enumerate HF API on every run, burning quota and risking 429s.
- Authenticated `/api/` calls used for file fetches instead of public CDN → avoidable rate-limit pressure.
- No client-side cache/TTL for repo-tree listings → every page load triggers fresh API traffic.
- Training scripts depend on runtime HF enumeration or `load_dataset(streaming=True)`; retries under Studio idle-kills will re-trigger quota exhaustion.
- No deterministic file-list injection point for Lightning training jobs and no explicit “refresh manifest” affordance.

---

## 2. Proposed change (merged + resolved)

Add a lightweight manifest generator and CDN-aware fetcher:

- **`vanguard/lib/manifest.py`** (new) — single `list_repo_tree` call, persist JSON to `vanguard/data/manifest-{repo_slug}-{date}.json`.
- **`vanguard/lib/cdn.py`** (new) — build public CDN URLs and perform zero-auth fetches (no Authorization header).
- **`vanguard/frontend/src/hooks/useManifest.ts`** (new) — client-side cache with 24h TTL and explicit refresh.
- **`vanguard/api/manifest.py`** (new FastAPI route) — serve manifest; optional `force` to rebuild.
- **`vanguard/train/train.py`** (modify) — accept `--manifest` and use CDN-only fetches during data loading (no HF auth).
- **Frontend entry** (minimal) — use `useManifest`; no direct authenticated HF calls for listing.

**Conflict resolution**:
- Use **public CDN URLs only** for training and frontend file fetches (not authenticated `/api/` routes) to eliminate quota pressure. Candidate 2’s concern about rate limits is solved by this, not by adding more auth paths.
- Place code under `vanguard/` (not `/opt/axentx/vanguard/` inline in examples) for maintainability; deployment can bind or copy to `/opt/axentx/vanguard/` as needed.
- One manifest generator (`lib/manifest.py`) with deterministic path logic; training and frontend both depend on the same artifact.

### lib/manifest.py
```python
# lib/manifest.py
from __future__ import annotations
import json, datetime
from pathlib import Path
from huggingface_hub import HfApi

HF_API = HfApi()
DATA_DIR = Path(__file__).parents[2] / "data"
DATA_DIR.mkdir(exist_ok=True)

def _slug(repo: str) -> str:
    return repo.replace("/", "--")

def manifest_path(repo: str, date_folder: str) -> Path:
    return DATA_DIR / f"manifest-{_slug(repo)}-{date_folder}.json"

def build_manifest(repo: str, date_folder: str, overwrite: bool = False) -> dict:
    """
    Single authenticated API call to list top-level folder.
    Returns manifest dict.
    """
    path = manifest_path(repo, date_folder)
    if path.exists() and not overwrite:
        return json.loads(path.read_text())

    tree = HF_API.list_repo_tree(repo, path=date_folder, recursive=False)
    files = [{"path": f.path, "size": f.size} for f in tree if f.type == "file"]

    manifest = {
        "repo": repo,
        "date": date_folder,
        "files": files,
        "built_at": datetime.datetime.utcnow().isoformat() + "Z",
    }
    path.write_text(json.dumps(manifest, indent=2))
    return manifest

if __name__ == "__main__":
    import sys
    repo, date_folder = sys.argv[1], sys.argv[2]
    m = build_manifest(repo, date_folder, overwrite="--force" in sys.argv)
    print(json.dumps(m, indent=2))
```

```python
# lib/cdn.py
from urllib.parse import quote
import requests

def cdn_url(repo: str, filepath: str) -> str:
    """Public CDN URL — no Authorization header required."""
    return f"https://huggingface.co/datasets/{repo}/resolve/main/{quote(filepath)}"

def fetch_cdn_file(repo: str, filepath: str, timeout: int = 30) -> bytes:
    url = cdn_url(repo, filepath)
    resp = requests.get(url, timeout=timeout)
    resp.raise_for_status()
    return resp.content
```

### frontend/src/hooks/useManifest.ts
```ts
// frontend/src/hooks/useManifest.ts
import { useState, useEffect, useCallback } from "react";

const CACHE_KEY = "vanguard_manifest";
const TTL_MS = 24 * 60 * 60 * 1000;

interface ManifestFile {
  path: string;
  size: number;
}

interface Manifest {
  repo: string;
  date: string;
  files: ManifestFile[];
  built_at: string;
}

export function useManifest(repo: string, date: string) {
  const [manifest, setManifest] = useState<Manifest | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const load = useCallback(async (force = false) => {
    setLoading(true);
    setError(null);
    try {
      const res = await fetch(`/api/manifest?repo=${repo}&date=${date}&force=${force}`);
      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      const data: Manifest = await res.json();
      localStorage.setItem(CACHE_KEY, JSON.stringify({ data, ts: Date.now() }));
      setManifest(data);
    } catch (e: any) {
      setError(e.message);
    } finally {
      setLoading(false);
    }
  }, [repo, date]);

  useEffect(() => {
    const raw = localStorage.getItem(CACHE_KEY);
    if (raw) {
      try {
        const { data, ts } = JSON.parse(raw);
        if (Date.now() - ts < TTL_MS && data.repo === repo && data.date === date) {
          setManifest(data);
          return;
        }
      } catch {
        localStorage.removeItem(CACHE_KEY);
      }
    }
    load(false);
  }, [repo, date, load]);

  const refresh = () => load(true);

  return { manifest, loading, error, refresh };
}
```
