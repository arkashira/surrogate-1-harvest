## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk_check→execution stub) with Supastore-backed state, Cloudflare Queue-driven orchestration, Redis cache for idempotency, immutable audit_event append log, compliance guardrails as JSON rules, replay CLI, and API endpoints to submit/list workflow runs; no frontend, broker integration mocked with contract tests.

## Open questions
- Deterministic replay semantics: which inputs must be snapshotted vs refetched on replay?
- Idempotency key strategy across brokers/OMS to prevent double execution on retries
- Latency SLOs per stage (research→signal→risk→execution) and acceptable queue depth
- Compliance rule language choice (rego vs JSON schema vs custom DSL) and hot-reload safety
- Broker/OMS connector authentication model (OAuth2 client credentials, HMAC, mTLS) and secret rotation
- Audit retention and immutability guarantees (WORM storage vs signed append-only log)
- Rollback and compensation actions for partially completed workflows
