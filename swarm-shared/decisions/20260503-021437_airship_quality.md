# airship / quality

## Highest-Value Incremental Improvement (<2h)

**Problem**: Surrogate training hits HF API 429 and wastes Lightning quota (recreate studios, idle-stop kills jobs).  
**Fix**: CDN-first deterministic ingestion + Lightning Studio reuse/guard + Arkship idle-stop resilience.  
**Why**: No schema migrations, no infra changes — pure orchestration + config fixes. Ships in <2h.

---

## Implementation Plan

1. **Create deterministic file-lister** (Mac orchestration)  
   - Single `list_repo_tree` call per date folder → save `file-list.json`.  
   - Embed `file-list.json` in Lightning training script → CDN-only fetches during training (zero API calls).

2. **Lightning Studio reuse + idle guard**  
   - Before `.run()`, list running studios; reuse if exists.  
   - Wrap training call with status check; restart if stopped (idle timeout).

3. **Arkship idle-stop resilience** (UI/API)  
   - Add lightweight keepalive ping from Arkship → Surrogate (or health-check wrapper) so long-running training isn’t orphaned when Arkship container restarts.

4. **Cron/Bash hygiene**  
   - Ensure all wrapper scripts have `#!/usr/bin/env bash`, `chmod +x`, invoked via `bash`, and `SHELL=/bin/bash` in crontab.

## Code Snippets

### 1) Deterministic file-lister (run on Mac)

```bash
#!/usr/bin/env bash
# tools/list_hf_date_folder.sh
set -euo pipefail

REPO="my-org/surrogate-dataset"
DATE_FOLDER="${1:-2026-04-29}"  # e.g. batches/mirror-merged/2026-04-29
OUTFILE="${2:-file-list.json}"

python3 - "$REPO" "$DATE_FOLDER" "$OUTFILE" <<'PY'
import json, os, sys
from huggingface_hub import HfApi

repo, date_folder, outfile = sys.argv[1], sys.argv[2], sys.argv[3]
api = HfApi()

# non-recursive per folder to avoid pagination explosion
tree = api.list_repo_tree(repo, path=date_folder, recursive=False)
files = [f.rfilename for f in tree if f.type == "file"]

# save for embedding in train.py
with open(outfile, "w") as f:
    json.dump({"date_folder": date_folder, "files": files}, f, indent=2)

print(f"Saved {len(files)} files to {outfile}")
PY
```

### 2) Lightning Studio reuse + idle guard

from lightning import LightningWork, LightningApp, Machine
from lightning.app import BuildConfig

class SurrogateTrainer(LightningWork):
    def __init__(self):
        super().__init__()
        self.studio = None

    def run(self):
        from lightning.pytorch import Trainer
        from lightning.pytorch.demos.boring_classes import BoringModel

        # reuse studio if running
        from lightning import Teamspace
        for s in Teamspace.studios:
            if s.name == "surrogate-train-studio" and s.status == "Running":
                self.studio = s
                print("Reusing running studio")
                break
        else:
            # create if not exists
            from lightning import Studio
            self.studio = Studio(
                name="surrogate-train-studio",
                machine=Machine.L40S,
                create_ok=True,
            )

        # idle-stop guard: restart if stopped
        if self.studio.status != "Running":
            print("Studio stopped (idle). Restarting...")
            self.studio.start(machine=Machine.L40S)

        file_list = load_file_list()
        data = list(train_on_cdn_only(file_list))
        model = BoringModel()
        trainer = Trainer(max_epochs=1)
        trainer.fit(model)

### 3) Arkship keepalive wrapper (lightweight)

```bash
#!/usr/bin/env bash
# arkship/scripts/surrogate-train-wrapper.sh
set -euo pipefail
export SHELL=/bin/bash

SURROGATE_HEALTH="http://localhost:8001/health"
MAX_RETRIES=3
RETRY_DELAY=10

for i in $(seq 1 $MAX_RETRIES); do
  if curl -fs "$SURROGATE_HEALTH" >/dev/null; then
    echo "Surrogate healthy. Starting training..."
    exec python -m surrogate.train
  else
    echo "Surrogate not healthy ($i/$MAX_RETRIES). Retrying in $RETRY_DELAY..."
    sleep $RETRY_DELAY
  fi
done

echo "Surrogate unavailable after $MAX_RETRIES attempts. Exiting."
exit 1
```
