# vanguard / design

## Final integrated solution (one actionable artifact)

**Core principle:** Deterministic, CDN-only training with zero HF-API calls during data loading, reproducible snapshots, and safe Studio/Lightning reuse.

### 1. Single canonical manifest spec
Create `/opt/axentx/vanguard/manifest/cdn_manifest.py` (single file, <150 LOC) that:

- Accepts a HF dataset repo and a date folder (e.g. `2026-04-29`) and produces a frozen, content-addressed manifest keyed by `{date}/{slug}.parquet`.
- Records for each file: `path`, `sha256`, `size`, `cdn_url`, `available_date` (ISO-8601), and optional `schema_digest` (hash of projected column names/types) to detect schema drift.
- Emits `manifest-{date}.json` and a machine-readable `manifest-{date}.sha256` for integrity.
- Validates on write: every entry must be HEAD-reachable via CDN and have a valid parquet footer; aborts on any mismatch.
- Supports deterministic selection: training jobs consume the manifest file directly; no runtime `list_repo_tree` or `load_dataset` calls.

### 2. Training contract (CDN-only)
- Training script (`train_cdn_only.py`) reads the embedded manifest and downloads exclusively via CDN (`resolve/main/...`).
- No Authorization header; bypasses `/api/` rate limits.
- Per-file projection at parse time to `{prompt,response}` (or required schema) to avoid `pyarrow.CastError` from schema heterogeneity.
- Concurrent CDN fetches with bounded workers and retries; failed files are logged and excluded (do not crash the run).
- Deterministic epoch order: iterate manifest files in sorted order; shuffle at sample level, not file level, when needed.

### 3. Lightning Studio + idle-stop handling
- Codify safe Studio reuse:
  - Save per-epoch checkpoints to persistent storage with `epoch`, `global_step`, and `manifest_path` in metadata.
  - On resume, verify the referenced manifest file exists and is valid before restoring data state.
  - Use `on_fit_start` to assert manifest integrity and CDN reachability; fail fast if broken.
- Idle-stop policy:
  - Provide a small wrapper script that monitors training logs for idle timeout (no step progress for N minutes) and gracefully shuts down the node to avoid quota waste.
  - Ensure checkpoint is saved before shutdown so progress is not lost.

### 4. Build/verify workflow (Mac or any dev box)
```bash
cd /opt/axentx/vanguard/manifest
HF_TOKEN=hf_... python cdn_manifest.py \
  --repo datasets/axentx/surrogate-1 \
  --date 2026-04-29 \
  --out manifest-2026-04-29.json
```
- This single command: lists the date folder once, downloads each parquet to a local temp cache, computes SHA256/size, writes manifest, and validates CDN reachability + parquet footer.
- Verify anytime:
  ```bash
  python cdn_manifest.py --verify manifest-2026-04-29.json
  ```
  Expect: “All entries reachable via CDN and size-matched.”

### 5. Run training
```bash
MANIFEST_PATH=manifest-2026-04-29.json CDN_WORKERS=8 \
  python train_cdn_only.py \
    --manifest $MANIFEST_PATH \
    --project-columns prompt,response \
    --max-epochs 10 \
    --checkpoint-dir /persistent/checkpoints
```
- Training uses CDN-only fetches, deterministic file order, and per-file column projection.
- Checkpoints include `manifest_path`; resume is safe and reproducible.

### 6. Why this resolves contradictions
- Combines Candidate 1’s concrete scripts and CDN-first design with Candidate 2’s emphasis on schema safety and Studio/idleness handling.
- Favors correctness: manifest is frozen and validated before any training step; no runtime HF API calls.
- Maximizes actionability: one manifest generator/verifier, one training entrypoint, clear Studio/idle-stop guidance, and a simple build/verify/run sequence.
