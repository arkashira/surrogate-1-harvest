## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: Android app (no root, stock) that starts an obfuscated Shadowsocks-over-QUIC tunnel via a CF Worker fronting domain; ephemeral SOCKS5 port and TUN interface names randomized at runtime; minimal Supabase-backed profile/key delivery; local SQLite state; basic WAUS/U counter; manual profile rotation; no kill-switch, no split-tunneling, no MDM.

## Open questions
- Which ephemeral port strategy and socket options best evade local port scans and /proc/net inference on stock Android?
- Can we safely hide the TUN interface name and addresses from apps that enumerate /proc/net/dev or rtnetlink without root?
- What traffic morphing profile (packet sizes, timing, ALPNs) yields the lowest DPI risk score while preserving throughput on high-loss paths?
- How will key rotation and profile updates be delivered via CF Workers + Queues without revealing metadata to edge logs?
- What telemetry can we collect (WAUS/U) without creating user-identifiable logs under restrictive jurisdictions?
