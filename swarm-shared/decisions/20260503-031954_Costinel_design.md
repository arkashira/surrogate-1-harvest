# Costinel / design

## Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h, production-ready)

### Scope (single highest-value deliverable)
Add a **non-blocking Top-Hub Signal Panel** to the Costinel dashboard that:
- Surface the most-connected hub (default `MOC`) from a lightweight hub-graph index
- Show 3 contextual insights from the knowledge-rag pipeline
- Use CDN-first data fetching (bypass HF API rate limits)
- Zero runtime API calls during dashboard render
- Fallback to static defaults if CDN unavailable

### Architecture (fits existing patterns)
- **Data source**: `hub-graph-index.json` + `moc-insights.json` published to CDN (or local mirror)
- **Build step**: optional script to refresh index from knowledge-rag (run on Mac orchestration only)
- **Runtime**: client-side fetch from CDN (`/data/hub-graph-index.json`) with 30s cache + graceful degradation
- **UI**: card in sidebar/overview with expandable insights list

    async function load() {

        // CDN-first fetch (bypasses HF API rate limits)
        const res = await fetch(INDEX_URL, { cache: "force-cache", signal: AbortSignal.timeout(8000) });
        const idx = res.ok ? await res.json() : FALLBACK_INDEX;

        setIndex(idx);

        const top = idx.topHub || Object.keys(idx.hubs)[0];
        const hub = idx.hubs[top];

          const insightsRes = await fetch(`${CDN_BASE}/data/${hub.insightsFile}`, {
            cache: "force-cache",
            signal: AbortSignal.timeout(8000)
          });
          if (insightsRes.ok) {
            const data = await insightsRes.json();
            setInsights(data.insights?.slice(0, 3) || []);
          }
        }
      } catch (err) {
        if (cancelled) return;
        setError(err.message || "Failed to load hub signals");
        setIndex(FALLBACK_INDEX);
      } finally {
        if (!cancelled) setLoading(false);
      }
    }

    load();
    return () => { cancelled = true; };

        <div className="flex items-center gap-2">
