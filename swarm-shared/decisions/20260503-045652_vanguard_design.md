# vanguard / design

## 1. Diagnosis
- No content-addressed manifest per date folder → training repeatedly calls `list_repo_tree`/`load_dataset` at runtime, causing HF API 429s and non-reproducible epochs.
- Data loader uses Hugging Face `datasets` API during training (streaming/list calls) instead of CDN-only fetches, violating the HF CDN bypass pattern.
- Mac orchestration likely triggers HF API discovery at runtime (frontend/training) instead of once-off file-list generation and embedding.
- No deterministic shard ordering or manifest → epochs drift across runs and resumable training is unreliable.
- Missing zero-API training path: Lightning Studio training still depends on HF `datasets` streaming/list calls instead of pure CDN downloads.

## 2. Proposed change
Add a content-addressed manifest generator and CDN-only training loader for one date folder under `/opt/axentx/vanguard`.  
Files:
- `scripts/generate_manifest.py` — run once on Mac after rate-limit window; produces `manifests/{date}/filelist.json` with deterministic ordering and SHA256 per file.
- `vanguard/train.py` — modified to accept `--manifest` and use CDN-only `wget`/`urllib` fetches (no HF `datasets` API) + deterministic shard order.
- `vanguard/dataloader.py` — new lightweight loader that reads manifest and streams parquet shards via CDN URLs.

## 3. Implementation

```bash
# /opt/axentx/vanguard/scripts/generate_manifest.py
#!/usr/bin/env python3
"""
Generate content-addressed manifest for a date folder.
Run once (Mac) after HF rate-limit window clears.
"""
import json, hashlib, os, sys
from datetime import datetime
from pathlib import Path

try:
    from huggingface_hub import list_repo_tree
except ImportError:
    print("pip install huggingface_hub")
    sys.exit(1)

REPO = os.getenv("HF_DATASET_REPO", "datasets/your-dataset")
DATE_FOLDER = os.getenv("DATE_FOLDER", datetime.utcnow().strftime("%Y-%m-%d"))
OUT_DIR = Path(__file__).parent.parent / "manifests" / DATE_FOLDER
OUT_DIR.mkdir(parents=True, exist_ok=True)

def sha256_file(path):
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            h.update(chunk)
    return h.hexdigest()

def main():
    # Single API call: non-recursive per date folder
    items = list_repo_tree(REPO, path=DATE_FOLDER, repo_type="dataset")
    files = [it for it in items if it.type == "file" and it.path.endswith(".parquet")]
    # Deterministic order
    files.sort(key=lambda f: f.path)

    manifest = {
        "repo": REPO,
        "date": DATE_FOLDER,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "files": []
    }

    for f in files:
        # CDN URL (no auth, no API)
        cdn_url = f"https://huggingface.co/datasets/{REPO}/resolve/main/{f.path}"
        entry = {
            "path": f.path,
            "size": f.size,
            "cdn_url": cdn_url,
            # Optional local checksum if file already mirrored; otherwise skip
        }
        manifest["files"].append(entry)

    out_path = OUT_DIR / "filelist.json"
    with open(out_path, "w") as fh:
        json.dump(manifest, fh, indent=2)
    print(f"Manifest written: {out_path}")
    print(f"Total shards: {len(files)}")

if __name__ == "__main__":
    main()
```

# /opt/axentx/vanguard/dataloader.py
import json
import pyarrow.parquet as pq
import requests
from io import BytesIO
from pathlib import Path
from typing import Iterator, Dict, Any

class CDNParquetShardLoader:
    """
    CDN-only parquet shard loader.
    No Hugging Face datasets/streaming/list API calls during training.
    """
    def __init__(self, manifest_path: str):
        with open(manifest_path) as f:
            self.manifest = json.load(f)
        self.shards = self.manifest["files"]
        # Deterministic order preserved from manifest

    def stream_shard(self, idx: int) -> pq.Table:
        entry = self.shards[idx]
        resp = requests.get(entry["cdn_url"], timeout=30)
        resp.raise_for_status()
        return pq.read_table(BytesIO(resp.content))

    def __len__(self):
        return len(self.shards)

    def iter_rows(self, columns=("prompt", "response")) -> Iterator[Dict[str, Any]]:
        for i in range(len(self.shards)):
            tbl = self.stream_shard(i)
            # Project only required columns at parse time
            if columns:
                tbl = tbl.select(columns)
            for batch in tbl.to_batches(max_chunksize=1024):
                for row in zip(*[batch.column(c).to_pylist() for c in batch.schema.names]):
                    yield dict(zip(batch.schema.names, row))

## 4. Verification
1. Generate manifest (Mac):
   ```bash
   export HF_DATASET_REPO=datasets/your-dataset
   export DATE_FOLDER=YYYY-MM-DD
   python3 scripts/generate_manifest.py
   ```
   Confirm `manifests/YYYY-MM-DD/filelist.json` exists, has deterministic ordering and CDN URLs.

2. Dry-run loader locally (no HF API calls during iteration):
   ```python
   from vanguard.dataloader import CDNParquetShardLoader
   loader = CDNParquetShardLoader("manifests/YYYY-MM-DD/filelist.json")
   rows = list(loader.iter_rows())
   print(f"Loaded {len(rows)} rows from {len(loader.shards)} shards")
   ```
   Monitor network: ensure requests go to `https://huggingface.co/datasets/.../resolve/main/...` and no `/api/` calls.

3. Reproducibility check:
   - Run generation twice on same date folder → filelist.json identical (same ordering).
   - Run loader twice → same row order and count.

4. Lightning Studio integration:
   - Copy `filelist.json` into Studio workspace or mount via cloud storage.
   - Launch training with `--manifest filelist.json`.
   - Confirm zero HF API calls during data load (no 429s) and training proceeds deterministically.
