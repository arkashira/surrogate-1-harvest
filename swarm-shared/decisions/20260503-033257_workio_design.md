# workio / design

Below is the **single, consolidated solution** that keeps the strongest, most compatible parts of both proposals and resolves conflicts in favor of **correctness + concrete actionability**.

---

## 1. Final diagnosis (merged)

- **Missing HMAC-SHA256 signature verification** — `/webhook/line` accepts unverified payloads (spoofing/replay).
- **No idempotency/replay protection** — duplicates from retries/double-taps create duplicate records.
- **Tenant binding is implicit/unvalidated** — tenant inferred from payload instead of validated against channel config; risk of cross-tenant injection.
- **Raw body unavailable after JSON parse** — body-parser/json middleware mutates request body; HMAC requires original bytes.
- **No replay window / short-term deduplication** — without TTL-based dedupe, stale events accepted indefinitely.
- **No DB uniqueness guard** — race conditions can still insert duplicates even with application-level checks.

---

## 2. Final proposed change (merged)

**Scope**:  
- `workio/server/src/routes/webhook/line.ts`  
- `workio/server/src/middleware/rawBody.ts`  
- `workio/server/src/lib/line/crypto.ts`  
- `workio/server/src/lib/line/verify.ts`  
- `workio/server/src/lib/db/idempotency.ts`  
- DB migration for idempotency table  
- Optional Redis for short-term TTL dedupe (recommended), with DB as durable fallback and uniqueness guard.

**Behavior**:
- Capture raw body before JSON parser for `/webhook/line`.
- Verify `X-Line-Signature` with HMAC-SHA256 using `LINE_CHANNEL_SECRET`.
- Reject if signature invalid/missing (`401`).
- Reject events older than 5 min (clock skew tolerance).
- Deduplicate by stable idempotency key (`channelId:eventType:timestamp:userId`) with short TTL (Redis) + durable DB uniqueness.
- Enforce tenant binding via channel→tenant config (not payload fields alone).
- Return `200` only after verification and idempotent processing.

## 3. Final implementation

### 3.1 Middleware to preserve raw body

`workio/server/src/middleware/rawBody.ts`

```ts
import { Request, Response, NextFunction } from 'express';

export function rawBodyJsonMiddleware(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // Only intercept LINE webhook POSTs; leave others untouched
  if (req.path === '/webhook/line' && req.method === 'POST') {
    const chunks: Buffer[] = [];
    req.on('data', (chunk) => chunks.push(chunk));
    req.on('end', () => {
      const raw = Buffer.concat(chunks);
      (req as any).rawBody = raw;
      // Safely repopulate req.body for downstream handlers
      try {
        req.body = JSON.parse(raw.toString('utf8'));
      } catch {
        // malformed JSON will be rejected later
      }
      next();
    });
  } else {
    next();
  }
}
```

### 3.2 HMAC verification helper

`workio/server/src/lib/line/crypto.ts`

```ts
import crypto from 'crypto';

export function verifyLineSignature(
  rawBody: Buffer,
  signature: string,
  channelSecret: string
): boolean {
  try {
    const expected = crypto
      .createHmac('sha256', channelSecret)
      .update(rawBody)
      .digest('base64');
    return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
  } catch {
    return false;
  }
}
```

### 3.3 Idempotency + replay window enforcement

`workio/server/src/lib/line/verify.ts`

```ts
import crypto from 'crypto';

const REPLAY_WINDOW_MS = 5 * 60 * 1000; // 5 minutes

export function buildIdempotencyKey(
  channelId: string,
  eventType: string,
  timestamp: number,
  userId?: string
): string {
  const userPart = userId || 'unknown';
  return crypto
    .createHash('sha256')
    .update(`${channelId}:${eventType}:${timestamp}:${userPart}`)
    .digest('hex');
}

export function isEventExpired(eventTimestamp: number): boolean {
  const now = Date.now();
  return Math.abs(now - eventTimestamp) > REPLAY_WINDOW_MS;
}
```

### 3.4 Idempotency persistence (Redis + DB fallback + DB uniqueness)

`workio/server/src/lib/db/idempotency.ts`

```ts
import { Pool } from 'pg';
import Redis from 'ioredis';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const redis = process.env.REDIS_URL ? new Redis(process.env.REDIS_URL) : null;

const REDIS_TTL_SECONDS = 10 * 60; // 10m TTL (covers replay window + buffer)

export async function ensureIdempotencyKey(
  key: string,
  tenantId: string
): Promise<boolean> {
  // Try Redis fast path first (recommended)
  if (redis) {
    const set = await redis.set(`idem:${key}`, tenantId, 'PX', REDIS_TTL_SECONDS * 1000, 'NX');
    if (set === 'OK') {
      // Also persist to DB for durability and uniqueness guard
      await persistToDb(key, tenantId).catch(() => {
        // If DB fails, we still consider it accepted (Redis already recorded).
        // Optionally: enqueue retry or alert.
      });
      return true;
    }
    return false; // duplicate in Redis
  }

  // Fallback to DB-only
  return persistToDb(key, tenantId);
}

async function persistToDb(key: string, tenantId: string): Promise<boolean> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    // Table: idempotency_keys (key text primary key, tenant_id uuid, created_at timestamptz)
    const res = await client.query(
      `INSERT INTO idempotency_keys (key, tenant_id) VALUES ($1, $2) ON CONFLICT DO NOTHING RETURNING key`,
      [key, tenantId]
    );
    await client.query('COMMIT');
    return res.rowCount === 1;
  } catch {
    await client.query('ROLLBACK');
    return false;
  } finally {
    client.release();
  }
}
```
