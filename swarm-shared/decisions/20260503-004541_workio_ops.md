# workio / ops

## 1. Diagnosis
- LINE webhook idempotency is still best-effort (app-layer only); no DB-level unique guard prevents duplicate punches under race/retry.
- No transactional upsert for clock-in; concurrent retries can insert two rows with `clock_out_at IS NULL` for same user.
- Missing partial unique index to enforce “one open punch per user” at the database level (cheapest, strongest guarantee).
- No lightweight idempotency key table to deduplicate LINE event IDs across webhook retries (defensive depth).
- Clock-out path lacks safe update guard (could accidentally open a second punch if prior punch state is ambiguous).

## 2. Proposed change
- **DB scope**: add partial unique index on `punches(user_id)` where `clock_out_at IS NULL`.
- **App scope**: `workio/server/src/routes/webhook/line.ts` — wrap punch insertion in an upsert (or insert with ON CONFLICT do nothing) and add idempotency table write keyed by LINE event ID.
- **Ops scope**: small migration script + verification query.

## 3. Implementation

### 3.1 DB migration (run once)
```sql
-- 1) Prevent multiple open punches per user
CREATE UNIQUE INDEX CONCURRENTLY IF NOT EXISTS idx_punches_one_open
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- 2) Lightweight idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  line_event_id VARCHAR(255) PRIMARY KEY,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: cleanup job target (older than 7d)
-- DELETE FROM line_webhook_idempotency WHERE created_at < NOW() - INTERVAL '7 days';
```

### 3.2 Update webhook handler
File: `workio/server/src/routes/webhook/line.ts`

```ts
// Add near top with other imports
import { Pool } from 'pg';

// Inside handler (after signature verify)
export const handleLineWebhook = async (req: Request, res: Response) => {
  const events = req.body.events;
  const pool = req.app.locals.db as Pool;

  for (const ev of events) {
    // Idempotency guard
    const idem = await pool.query(
      'INSERT INTO line_webhook_idempotency(line_event_id) VALUES ($1) ON CONFLICT DO NOTHING RETURNING line_event_id',
      [ev.source.userId + ':' + ev.timestamp + ':' + ev.type] // or ev.webhookEventId if provided
    );
    if (idem.rowCount === 0) continue; // already processed

    if (ev.type === 'message' && ev.message.type === 'text') {
      const text = ev.message.text.trim().toLowerCase();
      const userId = ev.source.userId;

      if (text === 'clock in' || text === 'clock out') {
        const client = await pool.connect();
        try {
          await client.query('BEGIN');

          if (text === 'clock in') {
            // Try insert; if conflict on open punch, treat as already clocked in
            const result = await client.query(
              `INSERT INTO punches (user_id, clock_in_at, clock_in_lat, clock_in_lon, line_user_id)
               VALUES ($1, NOW(), $2, $3, $4)
               ON CONFLICT (user_id) DO UPDATE
               SET clock_in_at = EXCLUDED.clock_in_at
               WHERE punches.clock_out_at IS NOT NULL
               RETURNING id, clock_in_at, clock_out_at`,
              [userId, null, null, userId]
            );
            // If row returned and clock_out_at is null -> already clocked in
            if (result.rows.length && result.rows[0].clock_out_at === null) {
              await client.query('ROLLBACK');
              // Notify already clocked in (optional)
              continue;
            }
          } else {
            // Clock out: update the open punch
            const result = await client.query(
              `UPDATE punches
               SET clock_out_at = NOW(), clock_out_lat = $1, clock_out_lon = $2
               WHERE user_id = $3 AND clock_out_at IS NULL
               RETURNING id`,
              [null, null, userId]
            );
            if (result.rowCount === 0) {
              await client.query('ROLLBACK');
              // No open punch to clock out
              continue;
            }
          }

          await client.query('COMMIT');
        } catch (err) {
          await client.query('ROLLBACK');
          console.error('Punch tx failed', err);
          // continue to next event
        } finally {
          client.release();
        }
      }
    }
  }

  res.json({ ok: true });
};
```

Notes:
- Uses `ON CONFLICT` + partial index to enforce one open punch.
- Idempotency table key is a simple deterministic composite; adjust to match actual LINE event ID field if available.
- Keeps transaction short; rolls back on conflict/errors.

3. **Race test (simulate retries)**:
   - Start server.
   - Send same LINE event payload twice within seconds (curl or test script).
   - Query punches:
     ```bash
     psql workio -c "SELECT user_id, clock_in_at, clock_out_at FROM punches ORDER BY created_at DESC LIMIT 5;"
     ```
   - Expect exactly one open punch per user and only one row created per event.

4. **Clock-in → clock-out flow**:
   - User clocks in (message).
   - Verify one row with `clock_out_at IS NULL`.
   - User clocks out.
   - Verify same row now has `clock_out_at` populated.
   - Attempt second clock-in → should create new row (previous closed).
