## 1) Generate CDN file list (Mac orchestration)

- **Why**: One `list_repo_tree` call replaces repeated API usage during training.
- **What it does**: Lists one date-folder (`mirror-merged/YYYY-MM-DD/`) non-recursively, keeps only `.parquet`, and writes `training/file_list.json` with CDN URLs and deterministic shard mapping.
- **Deterministic shard selection**: `hash(slug) % N` so future multi-repo sharding is consistent without reshuffling data.

**File**: `scripts/generate_cdn_filelist.py`

```python
#!/usr/bin/env python3
"""
Generate CDN-only file list for a mirror-merged date folder.
Run on Mac (or any orchestration node) to avoid HF API during training.
"""
import json
import os
import hashlib
from datetime import datetime, timezone
from pathlib import Path

from huggingface_hub import HfApi

REPO_ID = os.getenv("HF_DATASET_REPO", "org/surrogate-mirror")
DATE_FOLDER = os.getenv("MIRROR_DATE", datetime.now(timezone.utc).strftime("%Y-%m-%d"))
OUT_PATH = Path(__file__).parent.parent / "training" / "file_list.json"
SIBLINGS = int(os.getenv("HF_SIBLING_REPOS", "1"))  # 1 = single repo; 5+ for sharding

api = HfApi()

def deterministic_repo_index(slug: str) -> int:
    return int(hashlib.sha256(slug.encode()).hexdigest(), 16) % max(SIBLINGS, 1)

def main() -> None:
    prefix = f"batches/mirror-merged/{DATE_FOLDER}/"
    items = api.list_repo_tree(REPO_ID, path=prefix, recursive=False)

    files = []
    for item in items:
        if not item.path.endswith(".parquet"):
            continue
        slug = Path(item.path).stem
        cdn_url = f"https://huggingface.co/datasets/{REPO_ID}/resolve/main/{item.path}"
        files.append({
            "repo_id": REPO_ID,
            "path": item.path,
            "cdn_url": cdn_url,
            "slug": slug,
            "sibling_repo_index": deterministic_repo_index(slug),
        })

    OUT_PATH.parent.mkdir(parents=True, exist_ok=True)
    OUT_PATH.write_text(json.dumps({"date": DATE_FOLDER, "prefix": prefix, "files": files}, indent=2))
    print(f"Wrote {len(files)} files to {OUT_PATH}")

if __name__ == "__main__":
    main()
```

## 2) CDN-only dataloader (train.py)

- **Why**: Zero HF API calls during training; resilient to rate limits and idle timeouts.
- **How**: Load `file_list.json`, stream each parquet via CDN with `requests` + `pyarrow`, project to `{prompt, response}`.
- **No `load_dataset` streaming from HF** (avoids repeated API calls and auth/rate issues).

**Excerpt for `surrogate/training/train.py`**

```python
import json
import os
import tempfile
from pathlib import Path
from typing import Iterator, Dict, Any

import pyarrow.parquet as pq
import requests
from torch.utils.data import IterableDataset

FILE_LIST_PATH = Path(__file__).parent / "file_list.json"

class CDNParquetDataset(IterableDataset):
    def __init__(self, file_list_path: Path = FILE_LIST_PATH, project_cols=("prompt", "response")):
        self.file_list_path = file_list_path
        self.project_cols = project_cols
        with open(self.file_list_path) as f:
            self.manifest = json.load(f)
        self.files = self.manifest["files"]

    def _stream_parquet(self, cdn_url: str) -> Iterator[Dict[str, Any]]:
        # Stream to temp file so pyarrow can read; keeps CDN-only, zero HF API.
        with tempfile.NamedTemporaryFile(suffix=".parquet", delete=False) as tmp:
            tmp_path = tmp.name
            try:
                with requests.get(cdn_url, stream=True, timeout=60) as r:
                    r.raise_for_status()
                    for chunk in r.iter_content(chunk_size=8192):
                        tmp.write(chunk)
                tmp.flush()
                table = pq.read_table(tmp_path, columns=self.project_cols)
                for row in table.to_pylist():
                    yield row
            finally:
                try:
                    os.unlink(tmp_path)
                except Exception:
                    pass

    def __iter__(self):
        for entry in self.files:
            yield from self._stream_parquet(entry["cdn_url"])
```

## 3) Lightning idle-resilience

- **Why**: Lightning studios stop after idle; training must auto-restart instead of failing.
- **How**: Before `.run()`, check `studio.status`; if not `running`, start on `L40S` and poll until ready. Wrap training launch in retry for transient 429/5xx.

**Helper (in `train.py` or a small utils module)**

```python
def ensure_studio_running(studio, machine) -> None:
    """Ensure the Lightning studio is running; start it on `machine` if stopped."""
    if studio.status != "running":
        print(f"Studio {studio.name} not running (status={studio.status}). Starting on {machine}...")
        studio.start(machine=machine)
        # Wait until running (simple poll)
        import time
        for _ in range(60):
            studio.refresh()
            if studio.status == "running":
                print("Studio running.")
                return
            time.sleep(10)
        raise RuntimeError("Studio failed to start within timeout.")
```

## 4) Launch wrapper (run_training.py)

```python
#!/usr/bin/env python3
import os
from lightning import Lightning, Studio, Machine
from train import ensure_studio_running, CDNParquetDataset

LIGHTNING_TEAMSPACE = os.getenv("LIGHTNING_TEAMSPACE", "default")
STUDIO_NAME = os.getenv("STUDIO_NAME", "surrogate-train")
MACHINE = Machine.L40S

lightning = Lightning()
teamspace = lightning.teamspace(LIGHTNING_TEAMSPACE)

# Reuse existing studio if available
studio = None
for s in teamspace.studios:
    if s.name == STUDIO_NAME:
        studio = s
        break

if studio is None:
    studio = teamspace.create_studio(name=STUDIO_NAME, machine=MACHINE, create_ok=True)

ensure_studio_running(studio, MACHINE)

# Example training step (replace with your trainer)
dataset = CDNParquetDataset()
print(f"Dataset ready with {len(dataset.files)} files")

# studio.run(...) or your trainer.fit(...)
```

## 5) Execution Steps (≤2h)

1. `chmod +x scripts/generate_cdn_filelist.py`
2. On Mac:  
   ```bash
   HF_DATASET_REPO=org/surrogate-mirror MIRROR_DATE=YYYY-MM-DD python scripts/generate_cdn_filelist.py
   ```
3. Commit or ship `training/file_list.json` with your training code.
4. Update training to use `CDNParquetDataset` and add the idle-resilience wrapper.
5. Test: `python surrogate/training/run_training.py`

**Outcome**: Training is HF-rate-limit-proof (CDN-only), Lightning-idle-resilient (auto-restart), and requires minimal infra changes.
