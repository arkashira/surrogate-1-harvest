# Costinel / design

## Final Design Increment — Costinel Deterministic Discovery (≤2h)

**Single Goal:**  
Ship a lightweight, deterministic discovery module that produces machine-readable manifests (`manifests/sources.json`, `manifests/discovery.json`, `manifests/file-list.json`) and `manifests/top-hub-insight.md` so onboarding, training, and governance scripts can run without interactive setup or repeated HF API calls.

**Why this is highest-value (resolved):**  
- Unblocks downstream training/ingestion pipelines immediately.  
- Avoids repeated HF API calls during training by using CDN-only URLs and pre-listing files.  
- Follows existing repo patterns (top-hub insight, deterministic outputs, cron-safe wrappers).  
- Small, scoped surface (<2h) with concrete, copy-paste-ready implementation.

---

## Implementation Plan (≤2h, merged + corrected)

1. **Create manifests directory and schema** (10m)  
   - `manifests/sources.json` — cloud connectors, credential placeholders, cadence, collection config.  
   - `manfigs/discovery.json` — discovered accounts/services, regions, tags, cost centers.  
   - `manifests/file-list.json` — pre-listed CDN paths for a selected date folder (for training scripts).  
   - `manifests/top-hub-insight.md` — most-connected hub by lightweight edge count.

2. **Add deterministic discovery script** (45m)  
   - CLI: `scripts/discover.py` (Python, minimal deps).  
   - Reads minimal config from `config/discovery.yaml` or env.  
   - Produces the three manifests above.  
   - For HF repos used in dataset-mirror: calls `list_repo_tree(path, recursive=False)` once per date folder (respects rate limits), saves to `file-list.json`.  
   - Embeds CDN URLs (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) so training can fetch without auth.  
   - Deterministically picks sibling repo for writes via hash-slug → repo index (5 siblings) to respect HF commit cap.

3. **Add top-hub insight generator** (20m)  
   - Reads `manifests/discovery.json`, computes most-connected hub (by edge count), writes `manifests/top-hub-insight.md`.  
   - Tags: `#knowledge-rag #graph #hub`.

4. **Add Lightning Studio reuse helper** (15m)  
   - Utility `scripts/studio_reuse.py` lists running studios and returns a running one or creates with `create_ok=True`.  
   - Prevents quota waste (80hr/mo).

5. **Add cron-safe wrapper template** (10m)  
   - Template `scripts/run_discovery.sh` with proper shebang, `chmod +x`, and reminder to set `SHELL=/bin/bash` in crontab.  
   - Follows patterns from opus-pr-reviewer and active-learning fixes.

6. **Tests & docs** (20m)  
   - Minimal smoke test: `python -m pytest scripts/test_discovery.py -xvs`.  
   - Update README section “Deterministic Discovery” with usage.

---

## Resolved Contradictions (correctness + actionability)

- **HF API vs CDN usage:** Use `list_repo_tree` once per date folder to pre-list, then embed CDN-only URLs in `file-list.json`. This prevents 429s during training and keeps scripts auth-free.  
- **Deterministic repo selection:** Use hash-slug → repo index (5 siblings) for HF commit-cap spreading. This is explicit and actionable.  
- **Credential handling:** Keep manifests as placeholders/examples; real credentials come from env/secret store at runtime. This is safe and deployable.  
- **Schema stability:** Use versioned keys (`version`, `generated_at`) and stable field names across manifests so downstream scripts don’t break.
