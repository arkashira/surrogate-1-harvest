## Deployment
cf workers

## v0.1 scope
CLI + FastAPI service deployable on CF Workers (or local dev) that discovers Lambda functions + Cloud Functions and their schedules across one AWS account and one GCP project, persists to Supabase Postgres, detects exact duplicate cron expressions and stale schedules (no invocation in 90 days), exposes REST endpoints to list issues and mark schedules retired, and outputs a CSV retirement plan — all in 2 weeks with zero infra costs.

## Open questions
- Which AWS accounts/GCP projects and regions should be in scope for week 1 proof?
- How will we authenticate cross-account (AWS IAM roles vs keys; GCP ADC vs service account keys) on CF Workers free tier?
- What drift threshold (time or cron diff) and staleness window (e.g., 90 days no invocations) should define 'drifted' and 'orphaned'?
- Will duplicate detection be exact cron match only, or include semantic equivalence (same effective cadence)?
