# workio / design

## Final Synthesized Solution

**Chosen approach**: Merge the completeness of Candidate 1 (signature verification, idempotency, audit, schema, wiring, verification steps) with the stronger, production-grade protections that Candidate 2 implies (persistent replay protection, strict signature enforcement, and explicit race-condition mitigation at the DB layer). Contradictions are resolved in favor of **correctness and concrete actionability**.

---

## 1. Diagnosis (merged)
- **No idempotency guard on LINE webhook ingestion** — LINE retries on 5xx/timeouts; duplicate clock-in/out or leave/OT records are created on replay.
- **Missing `X-Line-Signature` verification** — webhook accepts spoofed events; enables attendance fraud and phantom punches.
- **No replay protection window** — without a persistent, time-bounded dedupe store keyed by canonical event identifiers, retries and replays create duplicate punches.
- **Clock-in/out state logic is vulnerable to race conditions** — concurrent webhook deliveries can produce multiple “in” records for the same user on the same day.
- **No structured audit trail for attendance events** — impossible to distinguish legitimate punches from replays or spoofed events during investigations.

---

## 2. Proposed change (merged + hardened)
File: `workio/server/src/routes/line/webhook.ts` (or equivalent webhook handler)  
Scope:
- (a) **Verify `X-Line-Signature`** on every request using raw body and constant-time comparison.
- (b) **Enforce idempotency and replay protection** via a persistent store (Redis preferred; fallback to durable DB unique constraint) keyed by a canonical dedupe key with a bounded TTL (e.g., 24–48h).
- (c) **Guard state transitions atomically** at the DB layer (unique constraint + upsert) to prevent races for same-day same-type punches; allow multiple shifts/breaks only when business rules require.
- (d) **Write immutable audit row** for every processed event, including signature, dedupe key, raw payload, and processing metadata.
- (e) **Fail fast and return correct status** (200 only after successful processing; 4xx on auth/validation errors to stop retries only when safe).

---

### Webhook handler (`workio/server/src/routes/line/webhook.ts`)
```ts
import express, { Request, Response, NextFunction } from 'express';
import crypto from 'crypto';
import { db } from '../db';
import { attendance, attendanceAudit, lineEventDedupe } from '../db/schema';
import { eq, and } from 'drizzle-orm';

const router = express.Router();
const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';
const REPLAY_TTL_MS = 48 * 60 * 60 * 1000; // 48h replay window

// Prefer Redis in production; this example uses DB dedupe for safety across instances.
// If you use Redis, replace dedupe checks with SET NX EX.

function verifyLineSignature(rawBody: Buffer, signature: string): boolean {
  if (!LINE_CHANNEL_SECRET) return false;
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

function buildDedupeKey(event: any, timestamp: string): string {
  // Canonical key: type + userId + event timestamp (use LINE's timestamp when present)
  return `line:${event.type}:${event.source?.userId}:${timestamp}`;
}

async function isDuplicate(dedupeKey: string, lineEventId?: string): Promise<boolean> {
  // Try Redis first if available; otherwise use DB unique constraint + upsert.
  // DB fallback:
  const existing = await db
    .select()
    .from(lineEventDedupe)
    .where(eq(lineEventDedupe.dedupeKey, dedupeKey))
    .limit(1);
  return existing.length > 0;
}

async function markProcessed(dedupeKey: string, lineEventId?: string, userId?: string, tenantId?: number) {
  try {
    await db.insert(lineEventDedupe).values({
      dedupeKey,
      lineEventId: lineEventId || null,
      userId: userId || '',
      tenantId: tenantId || 0,
      receivedAt: new Date(),
    });
  } catch {
    // If unique constraint violation, duplicate already recorded — safe to ignore.
  }
}

router.post(
  '/webhook/line',
  express.raw({ type: 'application/json' }),
  async (req: Request, res: Response, next: NextFunction) => {
    try {
      const signature = req.headers['x-line-signature'] as string;
      if (!signature) {
        return res.status(401).json({ error: 'Missing X-Line-Signature' });
      }

      if (!verifyLineSignature(req.body, signature)) {
        return res.status(401).json({ error: 'Invalid signature' });
      }

      const body = JSON.parse(req.body.toString());
      const events = body.events || [];

      for (const event of events) {
        const ts = String(event.timestamp || Date.now());
        const dedupeKey = buildDedupeKey(event, ts);
        const userId = event.source?.userId;
        const tenantId = 1; // derive from binding in production

        // Idempotency / replay protection
        if (await isDuplicate(dedupeKey, event.message?.id)) {
          continue;
        }

        // Process only relevant message events (customize intents as needed)
        if (event.type === 'message' && event.message?.text) {
          const text = event.message.text.trim().toLowerCase();
          const isClockIn = ['in', 'clock in', 'เข้างาน'].includes(text);
          const isClockOut = ['out', 'clock out', 'เลิกงาน'].includes(text);

          if (isClockIn || isClockOut) {
            const today = new Date().toISOString().split('T')[0];
            const type = isClockIn ? 'in' : 'out';

            // Atomic upsert guarded by DB unique constraint to prevent races
            const existing = await db
              .select()
              .from(attendance)
              .where(
                and(
                  eq(attendance.userId, userId),
                  eq(attendance.tenantId, tenantId),
                  eq(attendance.date, today),
                  eq(attendance.type, type)
                )
              )
              .limit(1);

            if (existing.length === 0) {
