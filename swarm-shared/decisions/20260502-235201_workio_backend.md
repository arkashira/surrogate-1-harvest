# workio / backend

## 1. Diagnosis

- LINE webhook handler is **non-idempotent**: retries on 5xx/timeouts create duplicate punches and invalid open/close state (confirmed in prior synthesis).
- **No atomic punch transition**: concurrent requests or retries can interleave, leaving `punches` without a matching close or double-close.
- Missing **idempotency key** handling (LINE `X-Line-Signature` or client-generated idempotency token) to deduplicate within a safe window.
- Punch state logic is likely **read-modify-write** in application code rather than a single atomic SQL transition, enabling races.
- No **compensating close** or safe retry semantics when a punch is inserted but confirmation fails.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/webhook/line.ts` (or equivalent route handler for LINE webhook)  
Scope:
- Add idempotency table (`line_webhook_deliveries`) keyed by `X-Line-Signature` + timestamp window (or generated idempotency token).
- Convert punch clock-in/out to a single atomic SQL statement using `INSERT ... ON CONFLICT DO NOTHING` + `UPDATE ... WHERE state='open' RETURNING *` pattern.
- Wrap handler in a transaction and return 200 only after commit; return 200 for duplicates detected by idempotency key.

### 3.2 Idempotent punch transition (SQL helper)

```sql
-- server/src/db/queries/punches.sql
-- Atomic clock-in: only insert if no open punch exists for this user/tenant today
INSERT INTO punches (user_id, tenant_id, clock_in, location, created_at)
SELECT $1, $2, NOW(), $3, NOW()
WHERE NOT EXISTS (
  SELECT 1 FROM punches
  WHERE user_id = $1 AND tenant_id = $2 AND clock_out IS NULL
)
RETURNING *;

-- Atomic clock-out: close the most recent open punch for this user/tenant
UPDATE punches
SET clock_out = NOW(), updated_at = NOW()
WHERE id = (
  SELECT id FROM punches
  WHERE user_id = $1 AND tenant_id = $2 AND clock_out IS NULL
  ORDER BY clock_in DESC
  LIMIT 1
)
RETURNING *;
```

### 3.3 Handler changes (Node/Express)

```ts
// server/src/routes/webhook/line.ts
import { Router, Request, Response } from 'express';
import crypto from 'crypto';
import { pool } from '../../db';
import { verifyLineSignature } from '../../utils/line';

const router = Router();

const IDEMPOTENCY_TTL_MS = 5 * 60 * 1000; // 5 minutes

async function isDuplicate(key: string, payloadHash: string): Promise<boolean> {
  const res = await pool.query(
    `SELECT 1 FROM line_webhook_deliveries WHERE idempotency_key = $1 AND payload_hash = $2`,
    [key, payloadHash]
  );
  return res.rowCount > 0;
}

async function recordDelivery(key: string, payloadHash: string) {
  await pool.query(
    `INSERT INTO line_webhook_deliveries (idempotency_key, payload_hash) VALUES ($1, $2)`,
    [key, payloadHash]
  );
}

router.post('/line', async (req: Request, res: Response) => {
  const signature = req.headers['x-line-signature'] as string;
  const body = JSON.stringify(req.body);
  const payloadHash = crypto.createHash('sha256').update(body).digest('hex');
  const idempotencyKey = `${signature}:${payloadHash}`;

  // Optional: verify signature (recommended)
  if (!verifyLineSignature(body, signature)) {
    return res.status(401).send('Invalid signature');
  }

  // Idempotency check
  if (await isDuplicate(idempotencyKey, payloadHash)) {
    return res.status(200).send('OK (duplicate)');
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Record delivery first to prevent races
    await client.query(
      `INSERT INTO line_webhook_deliveries (idempotency_key, payload_hash) VALUES ($1, $2)`,
      [idempotencyKey, payloadHash]
    );

    const events = req.body.events || [];
    for (const event of events) {
      if (event.type === 'message' && event.message.type === 'text') {
        const text = event.message.text.trim().toLowerCase();
        const userId = event.source.userId;
        const tenantId = 'default'; // derive from user or mapping

        if (text === 'clock in' || text === 'เข้างาน') {
          await client.query(
            `INSERT INTO punches (user_id, tenant_id, clock_in, location, created_at)
             SELECT $1, $2, NOW(), $3, NOW()
             WHERE NOT EXISTS (
               SELECT 1 FROM punches
               WHERE user_id = $1 AND tenant_id = $2 AND clock_out IS NULL
             )
             RETURNING *`,
            [userId, tenantId, '{}'] // location placeholder; populate from GPS if available
          );
        } else if (text === 'clock out' || text === 'เลิกงาน') {
          await client.query(
            `UPDATE punches
             SET clock_out = NOW(), updated_at = NOW()
             WHERE id = (
               SELECT id FROM punches
               WHERE user_id = $1 AND tenant_id = $2 AND clock_out IS NULL
               ORDER BY clock_in DESC
               LIMIT 1
             )
             RETURNING *`,
            [userId, tenantId]
          );
        }
      }
    }

    await client.query('COMMIT');
    res.status(200).send('OK');
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook handler error:', err);
    // Return 5xx so LINE retries (idempotency protects duplicates)
    res.status(500).send('Error');
  } finally {
    client.release();
  }
});

export default router;
```

### 3.4 Utility for LINE signature verification (if not present)

```ts
// server/src/utils/line.ts
import crypto from 'crypto';

export function verifyLineSignature(body: string, signature: string): boolean {
  const channelSecret = process.env.LINE_CHANNEL_SECRET || '';
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(body)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}
```

## 4. Verification

1. **Schema applied**: run `psql workio < server/src/db/schema.sql` and confirm `line_webhook_deliveries` exists.
2. **Duplicate detection**: send the same LINE event payload twice within 5 minutes; second request returns `200 OK (duplicate)` and no second punch row is created.
3. **Atomic transitions**: simulate concurrent clock-in requests for the same user; only one open punch row exists.
4. **Retry safety**: return 500 from handler mid-transaction, resend same payload; idempotency prevents double-punch and transaction retries cleanly.
5. **Clock-out correctness**: after clock-in, clock-out updates the correct open punch and leaves no open rows for that user/tenant.
