## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: deterministic research→signal→risk→execution workflow orchestrator with Postgres (Supabase) event store, Redis Streams queue, immutable audit entries, 3 sample connectors (1 broker, 1 OMS/EMS, 1 data vendor), compliance guardrails (block/warn/log), idempotent API and replay endpoint; no frontend, deployed on Cloudflare Workers with free-tier limits, targeting sub-500ms median orchestration latency and full audit trail exportable as JSONL.

## Open questions
- Deterministic replay storage strategy: append-only event store in Postgres vs export to immutable object storage (cost vs latency)?
- Broker/OMS/EMS connector authentication model: per-client secrets in CF Workers KV vs Supabase vaulted columns vs external secret manager?
- Latency SLA target for end-to-end run (sub-100ms vs sub-500ms) and acceptable queueing/backpressure thresholds?
- Compliance rule engine: safe expression language (e.g., JSONata subset) vs compiled WASM policies for performance and isolation?
- Fee attribution model for weighting NSM: how to reliably map workflow_run to fee-generating status across brokers?
