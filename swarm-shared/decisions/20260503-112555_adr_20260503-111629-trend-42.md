## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CF Worker gateway exposing POST /v1/chat/completions that normalizes OpenAI and Anthropic messages endpoints; weighted failover router; per-provider token-bucket rate limits in KV; Postgres-backed audit and token ledger; contract tests for request/response normalization; observability with gateway request-id and p95 latency metrics; zero frontend.

## Open questions
- Which provider APIs must be normalized in v0.1 (OpenAI completions/chat, Anthropic messages, Cohere generate)?
- What is the p95 latency SLA target (ms) and measurement method (gateway timer vs downstream)?
- How should global rate limits be enforced: token-bucket per provider, queueing, or immediate 429?
- Will authentication to the gateway be API keys, JWT, or CF Access; and how will per-client quotas be scoped?
- How will secrets (provider API keys) be injected in CF Workers (secrets) vs local dev (dotenv)?
