# vanguard / design

## 1) Diagnosis

- No canonical discovery entrypoint exists to surface top-hub insights (e.g., MOC) before planning — violates established `#knowledge-rag #graph #hub` pattern and forces manual exploration.
- Missing CDN-bypass file-list strategy for HF datasets; any future surrogate-1 training ingestion will immediately 429 on recursive `list_repo_files` and waste Studio quota.
- No reusable orchestration script that composes `granite-business-research.sh` → `knowledge-rag` queries into a single discovery run, causing duplicated effort and inconsistent context.
- Schema drift risk remains: nothing enforces projection to `{prompt, response}` only before upload to `batches/mirror-merged/{date}/{slug}.parquet`; attribution/ts columns could leak into training.
- Lightning Studio reuse is not automated; idle-stop kills training and manual restart burns quota.

## 2) Proposed change

Add `/opt/axentx/vanguard/bin/discover` (executable) + `/opt/axentx/vanguard/lib/discover.py` that:
- Runs `granite-business-research.sh` (if present) or simulates market analysis.
- Queries knowledge-rag for top-hub (MOC) and related docs, prints actionable insights.
- Generates HF CDN-bypass file-list for a given dataset/date folder and writes `file_list.json` for downstream training scripts.
- Validates schema projection rules for mirror-enriched parquet outputs.
- Accepts `--studio-reuse` to list running Teamspace studios and print reuse commands.

## 3) Implementation

Create files:


def generate_hf_file_list(
    repo: str,
    date_folder: str,
    output: Path,
    use_cdn_bypass: bool = True,
) -> List[str]:
    """
    Generate file list for HF dataset folder.
    Uses HF API once to list files (non-recursive per folder) and produces JSON
    for CDN-bypass downloads during training.
    """
    if not HF_AVAILABLE:
        print("[discover] huggingface_hub not available; skipping HF file-list generation", file=sys.stderr)
        return []

    print(f"[discover] Listing HF repo tree: {repo}/{date_folder} (non-recursive)")
    try:
        items = list_repo_tree(repo=repo, path=date_folder, recursive=False)
    except Exception as e:
        print(f"[discover] HF list_repo_tree failed: {e}", file=sys.stderr)
        return []

    file_paths = []
    for item in items:
        if getattr(item, "type", None) == "file":
            path = getattr(item, "path", None)
            if path:
                file_paths.append(path)

    # Also include subfolders' immediate files (one-level expansion) to avoid recursive list_repo_files
    for item in items:
        if getattr(item, "type", None) == "dir":
            subpath = getattr(item, "path", None)
            if subpath:
                try:
                    subitems = list_repo_tree(repo=repo, path=subpath, recursive=False)
                    for si in subitems:
                        if getattr(si, "type", None) == "file":
                            sp = getattr(si, "path", None)
                            if sp:
                                file_paths.append(sp)
                except Exception:
                    continue

    result = {
        "repo": repo,
        "date_folder": date_folder,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "use_cdn_bypass": use_cdn_bypass,
        "files": sorted(set(file_paths)),
        "cdn_url_template": f"https://huggingface.co/datasets/{{repo}}/resolve/main/{{path}}",
    }

    output.parent.mkdir(parents=True, exist_ok=True)
    with open(output, "w", encoding="utf-8") as f:
        json.dump(result, f, indent=2)

    print(f"[discover] Wrote HF file-list ({len(file_paths)} files) -> {output}")
    print("  → Training scripts should use CDN URLs and avoid API calls during data loading")
    return result["files"]


def validate_projection_rules(enriched_dir: Path) -> None:
    """Check that enriched parquet files contain only {prompt, response} (no source/ts cols)."""
    try:
        import pyarrow.parquet as pq
    except ImportError:
        print("[discover] pyarrow not installed; skipping schema projection validation", file=sys.stderr)
        return

    if not enriched_dir.is_dir():
        print(f"[discover] Enriched dir not found: {enriched_dir} — skipping projection check")
        return

    bad_files = []
    for pf in enriched_dir.rglob("*.parquet"):
        try:
            meta = pq.read_schema(pf)
            cols = set(meta.names)
            allowed = {"prompt", "response"}
            extra = cols - allowed
            if extra:
                bad_files.append((str(pf), sorted(extra)))
        except Exception:
            continue

    if bad_files:
        print("[discover] Projection rule violations (found extra columns):")
        for fpath, extra in bad_files[:10]:
            print(f"  → {fpath}: {extra}")
        if len(bad_files) > 10:
            print(f"  ... and {len(bad_files)-10} more")
    else:
        print("[discover] Projection rules OK (only prompt/response in enriched parquet)")
