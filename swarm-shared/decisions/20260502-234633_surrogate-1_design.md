# surrogate-1 / design

## Final Implementation Plan (≤2 h)

**Highest-value improvement**: Add a single pre-flight snapshot generator (`bin/snapshot.sh`) that produces a deterministic file manifest for a date folder and enables all 16 shards to stream via CDN URLs only. This eliminates per-shard HF API calls during ingestion and removes rate-limit risk while keeping parallelism unchanged.

### Why this now
- Current ingestion calls `list_repo_tree`/`list_repo_files` per shard (or relies on Space-side listing) → 16× API calls per cron tick → 429 risk.
- HF CDN URLs are unauthenticated, high-throughput, and stable; we only need one canonical file list per date.
- Embedding the manifest in each shard lets workers stream with `pyarrow`/`requests` (or `wget`/`curl`) with zero HF Python client calls during training.

---

## Steps (1 h 30 m total)

1. **Create `bin/snapshot.sh`** (20 m)  
   - Uses `huggingface_hub.HfApi.list_repo_tree(..., recursive=True)` once per date folder.  
   - Outputs `snapshots/snapshot-<date>.json` containing:  
     `{repo, date, generated_at_utc, files: [{path, size}], sha256}`.  
   - Exits non-zero on failure; idempotent (overwrites same date).

2. **Update `bin/dataset-enrich.sh`** (20 m)  
   - Accepts `SNAPSHOT_FILE` env var.  
   - If provided, reads file list from snapshot; otherwise falls back to live API (slow path).  
   - Each shard deterministically selects its 1/16 slice by `hash(filename) % 16`.

3. **Add `scripts/cdn_loader.py`** (30 m)  
   - Streams parquet via `https://huggingface.co/datasets/{repo}/resolve/main/{path}` with `requests` + `pyarrow`.  
   - Retries with exponential backoff; respects HF CDN implicit limits (parallelism at worker level, not per file).  
   - Optional `columns` and `row_groups` selection to minimize memory.

4. **Update GitHub Actions matrix** (15 m)  
   - Add a `snapshot` job that runs `bin/snapshot.sh` and uploads artifact.  
   - `ingest` matrix (16 shards) downloads artifact and sets `SNAPSHOT_FILE` env.  
   - Keep existing 16-shard parallelism unchanged.

5. **Add Lightning training integration** (25 m)  
   - Modify `train.py` to accept `--file-list` JSON.  
   - Use `cdn_loader.stream_parquet_shard` in `DataLoader`/`IterableDataset`.  
   - No `load_dataset` or `list_repo_files` calls during training.

## Code Snippets

### 1. `bin/snapshot.sh`
```bash
#!/usr/bin/env bash
# bin/snapshot.sh
# Usage: bin/snapshot.sh <repo> <date> [out_dir]
# Example: bin/snapshot.sh axentx/surrogate-1-training-pairs 2026-05-03 ./snapshots

set -euo pipefail
REPO="${1:-axentx/surrogate-1-training-pairs}"
DATE="${2:-$(date +%Y-%m-%d)}"
OUTDIR="${3:-./snapshots}"
mkdir -p "$OUTDIR"
OUTFILE="${OUTDIR}/snapshot-${DATE}.json"

python3 - "$REPO" "$DATE" "$OUTFILE" <<'PY'
import json, hashlib, datetime, os, sys
from huggingface_hub import HfApi

repo = sys.argv[1]
date = sys.argv[2]
outfile = sys.argv[3]

api = HfApi()
entries = api.list_repo_tree(repo=repo, path=date, recursive=True)
files = sorted([e.rfilename for e in entries if not e.rfilename.endswith("/")])

snapshot = {
    "repo": repo,
    "date": date,
    "generated_at_utc": datetime.datetime.utcnow().isoformat() + "Z",
    "files": files,
}
blob = json.dumps(snapshot, sort_keys=True).encode()
snapshot["sha256"] = hashlib.sha256(blob).hexdigest()

with open(outfile, "wb") as f:
    f.write(json.dumps(snapshot, indent=2, sort_keys=True).encode())
print(outfile)
PY
```

### 2. `scripts/cdn_loader.py`
```python
import io
import time
import requests
import pyarrow.parquet as pq
import pyarrow as pa
from typing import List, Iterator, Optional

HF_CDN = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

def cdn_fetch(repo: str, path: str, headers: Optional[dict] = None, max_retries: int = 5) -> bytes:
    url = HF_CDN.format(repo=repo, path=path)
    for attempt in range(1, max_retries + 1):
        try:
            resp = requests.get(url, headers=headers, timeout=30)
            resp.raise_for_status()
            return resp.content
        except Exception as exc:
            if attempt == max_retries:
                raise
            sleep_sec = 2 ** attempt
            time.sleep(sleep_sec)
    raise RuntimeError("unreachable")

def stream_parquet_shard(
    repo: str,
    files: List[str],
    shard_id: int,
    n_shards: int,
    columns: Optional[List[str]] = None,
) -> Iterator[pa.Table]:
    for i, f in enumerate(files):
        if (i % n_shards) != shard_id:
            continue
        buf = io.BytesIO(cdn_fetch(repo, f))
        yield pq.read_table(buf, columns=columns)
```

### 3. Updated `bin/dataset-enrich.sh` (excerpt)
```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
SHARD_ID="${SHARD_ID:-0}"
N_SHARDS="${N_SHARDS:-16}"
SNAPSHOT_FILE="${SNAPSHOT_FILE:-}"

if [[ -n "$SNAPSHOT_FILE" ]]; then
  mapfile -t FILES < <(jq -r '.files[]' "$SNAPSHOT_FILE")
else
  # fallback (slower)
  mapfile -t FILES < <(python3 -c "
from huggingface_hub import HfApi
api = HfApi()
entries = api.list_repo_tree(repo='$REPO', path='public-merged', recursive=True)
for e in entries:
    if not e.rfilename.endswith('/'):
        print(e.rfilename)
")
fi

python3 - "$SHARD_ID" "$N_SHARDS" "${FILES[@]}" <<'PY'
import os, sys, hashlib
shard_id = int(sys.argv[1])
n_shards = int(sys.argv[2])
files = sys.argv[3:]

for f in files:
    h = int(hashlib.sha256(f.encode()).hexdigest(), 16)
    if h % n_shards != shard_id:
        continue
    # process file via cdn_loader.stream_parquet_shard in training
    print(f"shard {shard_id}: {f}")
PY
```
