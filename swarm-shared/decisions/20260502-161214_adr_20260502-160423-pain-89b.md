## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust CLI client (Linux/macOS) that: 1) creates a SOCKS5 proxy with stealth fronting (TLS-over-443 via boringtun to a public bridge), 2) obfuscates local port/TUN artifacts and disables obvious DPI fingerprints, 3) enforces a fail-closed kill-switch that blocks all non-proxy traffic if stealth path fails, 4) stores session telemetry locally in SQLite and optionally syncs anonymized events to Supabase via CF Workers, 5) includes basic integration tests for stealth handshake and kill-switch behavior — all deployable without server control and free-tier only.

## Open questions
- Which DPI signatures are most prevalent in target regions (SNI, packet timing, TUN MTU, DNS patterns)?
- Can we rely solely on client-side WASM builds or do we need native binaries for TUN access?
- What innocuous cover traffic patterns are safe and effective without creating new fingerprints?
- How will we measure WAU-ZB reliably without leaking metadata to third parties?
- Are CF Workers sufficient for telemetry ingestion and auth, or do we need Supabase edge functions?
