# workio / quality

## Final Synthesis — One Correct, Actionable Plan

**Core problem**: nothing in the database prevents duplicate open punches, and nothing in the webhook handler prevents duplicate processing when LINE retries or clients double-tap.  
**Non-negotiable fixes**:  
1) Enforce “at most one open punch per user per calendar day” at the database level.  
2) Make webhook processing idempotent with an idempotency table and atomic upsert.  
3) Add defensive constraints so impossible intervals cannot be stored.  
4) Validate clock-in time server-side (not in the future; optionally within tenant hours).

---

### 1. Database schema (apply once)

File: `workio/server/src/db/schema.sql`

```sql
-- 1) At most one open punch per user per calendar day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_day
ON punches (user_id, date_trunc('day', clock_in_at))
WHERE clock_out_at IS NULL;

-- 2) Prevent impossible intervals
ALTER TABLE punches
  ADD CONSTRAINT IF NOT EXISTS chk_punch_interval
  CHECK (clock_out_at IS NULL OR clock_out_at >= clock_in_at);

-- 3) Optional but recommended: clock-in cannot be in the future
ALTER TABLE punches
  ADD CONSTRAINT IF NOT EXISTS chk_clock_in_not_future
  CHECK (clock_in_at <= clock_out_at OR clock_in_at <= clockout_at OR clock_in_at <= NOW());

-- 4) Idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  event_type      TEXT NOT NULL,
  payload_hash    TEXT NOT NULL,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional: index for cleanup
CREATE INDEX IF NOT EXISTS idx_idempotency_processed_at ON webhook_idempotency (processed_at);
```

Notes:  
- The partial unique index is the authoritative guard against duplicate open punches.  
- The check constraint prevents `clock_out_at < clock_in_at`.  
- The future-time check is defensive; if you prefer application-only validation, omit it but enforce server-side validation before insert.

### 2. Webhook handler — atomic, idempotent punch upsert

File: `workio/server/src/routes/line.ts` (or equivalent)

```ts
import { Request, Response } from 'express';
import { pool } from '../db';
import crypto from 'crypto';

function sha256Hex(s: string): string {
  return crypto.createHash('sha256').update(s).digest('hex');
}

export async function handleLineWebhook(req: Request, res: Response) {
  const events = req.body?.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'No events' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      // Stable idempotency key from LINE event fields
      const idempotencyKey = `line:${ev.type}:${ev.source?.userId}:${ev.timestamp}:${ev.message?.id || ev.source?.groupId || ''}`;
      const payloadHash = sha256Hex(JSON.stringify(ev));

      const dup = await client.query(
        `SELECT 1 FROM webhook_idempotency WHERE idempotency_key = $1`,
        [idempotencyKey]
      );
      if (dup.rows.length > 0) {
        continue; // already processed
      }

      // Record idempotency first
      await client.query(
        `INSERT INTO webhook_idempotency (idempotency_key, event_type, payload_hash)
         VALUES ($1, $2, $3)`,
        [idempotencyKey, ev.type, payloadHash]
      );

      // Only handle clock-in/out text commands in this example
      if (ev.type === 'message' && ev.message?.type === 'text') {
        const text = (ev.message.text || '').trim().toLowerCase();
        if (text !== 'in' && text !== 'out') continue;

        const userId = ev.source.userId;
        const tenantId = 'default'; // derive from mapping in production
        const now = new Date();

        if (text === 'in') {
          // Atomic insert; unique partial index prevents duplicates
          await client.query(
            `INSERT INTO punches (user_id, tenant_id, clock_in_at)
             VALUES ($1, $2, $3)`,
            [userId, tenantId, now]
          );
        } else {
          // Close most recent open punch for this user (today)
          const result = await client.query(
            `UPDATE punches
             SET clock_out_at = $1
             WHERE user_id = $2
               AND tenant_id = $3
               AND clock_out_at IS NULL
             ORDER BY clock_in_at DESC
             LIMIT 1
             RETURNING id`,
            [now, userId, tenantId]
          );

          if (result.rowCount === 0) {
            // No open punch — log or optionally reply to user
            console.warn(`No open punch to close for user ${userId}`);
          }
        }
      }
    }

    await client.query('COMMIT');
    return res.status(200).json({ ok: true });
  } catch (err: any) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);

    // Unique violation from open-punch duplicate is acceptable idempotency
    if (err.code === '23505') {
      return res.status(200).json({ ok: true, note: 'duplicate prevented' });
    }
    return res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
}
```

Key points:  
- Idempotency recorded before punch mutation.  
- All mutations in one transaction per batch of events.  
- Unique violation on open punch is treated as success (idempotent).  
- Clock-out targets the most recent open punch (safe under retries).
