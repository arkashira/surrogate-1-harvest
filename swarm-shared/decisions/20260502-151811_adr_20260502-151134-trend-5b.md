## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk→execution) with one broker connector (REST), Postgres event store on Supabase, Cloudflare Queues for orchestration, Redis for idempotency/risk cache, compliance guardrails (max notional, sector cap) evaluated inline, immutable audit log with deterministic hashes, replay endpoint for runs, and contract tests for broker/OMS adapters.

## Open questions
- Deterministic replay boundary: should events be normalized to UTC millisecond precision and canonical JSON to guarantee bitwise replay across environments?
- Idempotency key scope: per-correlation_id or per-(correlation_id, broker_id) to prevent double-sends during retries?
- Compliance rule evaluation latency budget: synchronous inline vs asynchronous pre-check with circuit-breaker fallback?
- Broker/OMS connector interface: FIX vs REST/WebSocket — which adapters must be prioritized for v0.1?
- Audit retention and immutability strategy: WORM storage or signed append-only log in Postgres with periodic hash anchoring?
