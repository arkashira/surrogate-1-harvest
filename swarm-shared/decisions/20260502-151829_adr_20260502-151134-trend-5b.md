## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research/signal ingestion -> risk guardrail check -> single simulated or sandbox broker execution) with immutable audit log to Supabase storage, edge API on Cloudflare Workers, Redis Streams queue, and Postgres for state; includes replay endpoint for a run and basic compliance guardrails (per-symbol notional cap, restricted-symbol block).

## Open questions
- Which brokers/OMS/EMS and data vendors are must-connect for v0.1 and what auth patterns (OAuth, API keys, mTLS) do they require?
- What deterministic replay fidelity is required (full state snapshots vs event sourcing) and what retention/object-store is acceptable under free-tier constraints?
- Which compliance guardrail rules are non-negotiable for launch (e.g., per-symbol notional limits, restricted securities, time-of-day blocks)?
- What latency SLA (p95/p99) from signal ingestion to execution attempt is acceptable for target workflows?
- How should secrets (API keys) be stored and rotated in a free-tier-first deployment (Workers KV vs Supabase Vault vs env)?
