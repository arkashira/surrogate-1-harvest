# workio / frontend

## Final synthesized implementation (recommended)

Merge the strongest parts of both proposals into one correct, production-ready solution.  
Key decisions in favor of correctness + actionability:

- Use **React Query** for robust optimistic updates, rollbacks, and automatic cache invalidation (Candidate 1).
- Keep **visual feedback and request deduplication** (disabled state + in-flight lock) to prevent double-taps and race punches (both).
- Use a **stable client-generated idempotency key** per user action so retries and LINE/webhook duplicates can be reconciled server-side (both).
- Keep **pending row in the punch list** with clear UI distinction (pending vs committed) so the timeline doesn’t “jump” (Candidate 1).
- Keep API surface unchanged where possible, but require backend acceptance of `idempotencyKey` and idempotent handling.

### 1) Mutation hook (strongest core)

File: `workio/src/features/punch/usePunchMutation.ts`

```tsx
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { v4 as uuidv4 } from 'uuid';

export type PunchType = 'IN' | 'OUT';

export type PunchPayload = {
  type: PunchType;
  latitude: number;
  longitude: number;
  idempotencyKey: string;
};

export type Punch = {
  id: string;
  type: PunchType;
  latitude: number;
  longitude: number;
  timestamp: string;
  status?: 'pending' | 'committed';
  idempotencyKey: string;
};

export function usePunchMutation() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: async ({ type, latitude, longitude, idempotencyKey }: PunchPayload) => {
      const res = await fetch('/api/punches', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ type, latitude, longitude, idempotencyKey }),
      });
      if (!res.ok) {
        const text = await res.text().catch(() => '');
        throw new Error(text || 'Punch failed');
      }
      return res.json() as Promise<{ punch: Punch }>;
    },
    onMutate: async ({ type, latitude, longitude, idempotencyKey }) => {
      await queryClient.cancelQueries({ queryKey: ['punches'] });

      const previous = queryClient.getQueryData<Punch[]>(['punches']);
      const optimisticPunch: Punch = {
        id: `pending-${idempotencyKey}`,
        type,
        latitude,
        longitude,
        timestamp: new Date().toISOString(),
        status: 'pending',
        idempotencyKey,
      };

      queryClient.setQueryData<Punch[]>(['punches'], (old = []) => [optimisticPunch, ...old]);
      return { previous };
    },
    onError: (_err, _variables, context) => {
      if (context?.previous) {
        queryClient.setQueryData(['punches'], context.previous);
      }
    },
    onSettled: () => {
      // Always refetch to reconcile optimistic row with authoritative server state
      queryClient.invalidateQueries({ queryKey: ['punches'] });
    },
  });
}
```

### 2) Clock-in/out button (visual feedback + double-tap protection)

File: `workio/src/features/punch/PunchButton.tsx`

```tsx
import { useState } from 'react';
import { usePunchMutation } from './usePunchMutation';

export function PunchButton() {
  const [isProcessing, setIsProcessing] = useState(false);
  const { mutateAsync } = usePunchMutation();

  const handlePunch = async (type: 'IN' | 'OUT') => {
    if (isProcessing) return;
    setIsProcessing(true);
    try {
      const pos = await getCurrentPosition();
      await mutateAsync({
        type,
        latitude: pos.latitude,
        longitude: pos.longitude,
        idempotencyKey: crypto.randomUUID(),
      });
    } catch (err) {
      // optimistic rollback already handled by React Query
      // optionally show toast here
      console.error(err);
    } finally {
      setIsProcessing(false);
    }
  };

  return (
    <div className="flex gap-3">
      <button
        onClick={() => handlePunch('IN')}
        disabled={isProcessing}
        className="btn btn-primary"
      >
        {isProcessing ? 'กำลังบันทึก...' : 'Clock In'}
      </button>
      <button
        onClick={() => handlePunch('OUT')}
        disabled={isProcessing}
        className="btn btn-secondary"
      >
        {isProcessing ? 'กำลังบันทึก...' : 'Clock Out'}
      </button>
    </div>
  );
}

function getCurrentPosition(): Promise<{ latitude: number; longitude: number }> {
  return new Promise((resolve, reject) => {
    if (!navigator.geolocation) {
      reject(new Error('Geolocation not supported'));
      return;
    }
    navigator.geolocation.getCurrentPosition(
      (p) => resolve({ latitude: p.coords.latitude, longitude: p.coords.longitude }),
      (err) => reject(err),
      { enableHighAccuracy: true, timeout: 10000 }
    );
  });
}
```

### 3) Punch list row (clear pending vs committed)

File: `workio/src/features/punch/PunchListRow.tsx`

```tsx
import { Punch } from './usePunchMutation';

export function PunchListRow({ punch }: { punch: Punch }) {
  const isPending = punch.status === 'pending';
  return (
    <div
      className={`p-3 rounded border ${
        isPending ? 'opacity-70 border-dashed bg-gray-50/50' : 'border-solid'
      }`}
    >
      <div className="flex items-center justify-between">
        <div>
          <span className="font-medium">{punch.type === 'IN' ? 'เข้างาน' : 'ออกงาน'}</span>
          <span className="ml-2 text-sm text-gray-500">
            {new Date(punch.timestamp).toLocaleString()}
          </span>
        </div>
        {isPending && <span className="badge badge-warning text-xs">กำลังบันทึก</span>}
      </div>
      <div className="text-xs text-gray-400 mt-1">
        {punch.latitude.toFixed(4)}, {punch.longitude.toFixed(4)}
      </div>
    </div>
  );
}
```

### 4) Backend requirement (non-negotiable)

- Accept `idempotencyKey` in `/api/punches` POST body.
- Store `idempotencyKey` on the `punches` table with a unique constraint (or treat it as a natural key for upserts).
- Make punch creation idempotent: if a request arrives with an existing `idempotencyKey`, return the existing punch record instead of creating a duplicate.
- Ensure LINE/webhook retries are also deduplicated using the same key.
