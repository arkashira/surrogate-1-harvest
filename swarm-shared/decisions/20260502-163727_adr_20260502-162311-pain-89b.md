## v0.1 scope
v0.1 ships in 2 weeks: Android client (Kotlin + JNI) with tun2socks + shadowsocks + obfs4 options, domain-fronted bootstrap via Cloudflare Workers, Supabase Postgres config/keys, local obfuscation toggle, basic WAUS/U counter, no accounts required (ephemeral keys), and CI with unit/integration tests.

## Open questions
- Which obfuscation transports minimize DPI fingerprintability while preserving throughput on high-loss paths?
- How will we distribute endpoint lists and config without creating a static blocklist (domain-front vs. DoH bootstrap)?
- What telemetry is safe to collect (WAUS/U) without deanonymizing users under active inspection?
- How to handle split-tunnel exclusion lists on stock Android without triggering VPN-detection heuristics?
- Can we rely on CF Workers + Durable Objects for command/control within free-tier limits during scale spikes?
