# vanguard / design

**Most Valuable Next Improvement: Persistent File-List Cache**

### Diagnosis

* No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` → HF API 429 (1000 req/5min).
* Lightning Studio lifecycle not reused → quota burn on recreation.
* Inconsistent file-list management leads to data duplication and increased storage costs.

### Proposed Change

Create a persistent file-list cache using a JSON file to store the list of files for each repository. This cache will be updated periodically to reflect changes in the repository structure.

### Implementation

1. Create a new file `file_list_cache.py` in the project root:
```python
import json
import os

CACHE_FILE = 'file_list_cache.json'

def load_file_list_cache():
    if os.path.exists(CACHE_FILE):
        with open(CACHE_FILE, 'r') as f:
            return json.load(f)
    return {}

def save_file_list_cache(file_list):
    with open(CACHE_FILE, 'w') as f:
        json.dump(file_list, f)

def get_file_list(repo, path):
    cache = load_file_list_cache()
    if repo in cache and path in cache[repo]:
        return cache[repo][path]
    # fallback to original implementation
    file_list = []
    # ...
    save_file_list_cache({repo: {path: file_list}})
    return file_list
```
2. Update `train.py` to use the `get_file_list` function:
```python
import file_list_cache

# ...

file_list = file_list_cache.get_file_list(repo, path)
# ...
```
3. Update `lightning_studio.py` to reuse the existing Studio instance:
```python
import file_list_cache

# ...

studio = Teamspace.studios.find('my_studio')
if studio:
    # reuse existing Studio instance
    studio = studio
else:
    # create new Studio instance
    studio = Teamspace.studios.create_ok('my_studio')
```
4. Update `dataset-mirror` script to project data to {prompt, response} only before upload:
```python
import pandas as pd

# ...

df = pd.read_parquet('batches/mirror-merged/{date}/{slug}.parquet')
df = df[['prompt', 'response']]
df.to_parquet('batches/mirror-merged/{date}/{slug}.parquet', index=False)
```
### Verification

1. Run the `train.py` script and verify that the file list is loaded from the cache instead of being re-listed.
2. Verify that the Lightning Studio instance is reused instead of being recreated.
3. Monitor the HF API rate limit and quota usage to ensure that the changes have the desired effect.
4. Check that the `file_list_cache.json` file is updated periodically.
5. Verify that the datasets are mirrored correctly and that the data is projected to {prompt, response} only before upload.

**Additional Recommendations**

* Consider implementing a more robust caching mechanism, such as using a database or a caching library like Redis.
* Add error handling and logging to the caching mechanism to ensure that it can handle failures and provide useful error messages.
* Consider implementing a mechanism to invalidate the cache when the repository structure changes.
* Consider implementing a mechanism to limit the size of the cache to prevent it from growing indefinitely.
