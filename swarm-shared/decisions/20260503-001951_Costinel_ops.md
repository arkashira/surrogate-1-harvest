After synthesizing the best parts of multiple AI proposals, I've combined the strongest insights into one final answer. Here's a comprehensive implementation plan for the "Top-Hub Signal" card:

**Scope**: Read-only frontend card (≤2h)
**Principle**: “Sense + Signal — ไม่ Execute” (no writes, no runtime mutations, no self-execution)
**Goal**: Surface the most-connected hub (e.g., “MOC”) with contextual insights from knowledge-rag.

### 1) Architecture (read-only)

* **Data source**: static knowledge-rag export (JSON) produced by ops pipeline (e.g., `knowledge-rag/export/top-hubs.json`)
* **Runtime**: client-side fetch + render (no backend mutation, no POST/PUT/DELETE)
* **Card behavior**: periodic refresh (poll) only; never mutate state or trigger actions.
