# vanguard / ops

## 1. Diagnosis

- Frontend/training still trigger authenticated `list_repo_tree` on every selection/launch, burning HF API quota (1000/5min) and causing 429s.
- No persisted `(repo, dateFolder) → file-list` artifact exists; training scripts re-query instead of using a static manifest.
- No CDN-only data loader path; training continues to use `datasets.load_dataset(...)` which performs auth’d API calls and is vulnerable to rate limits.
- No guard to reuse running Lightning Studio; each launch recreates and wastes 80hr/mo quota.
- No retry/backoff for 429 in ingestion/training orchestration; transient spikes kill runs.

## 2. Proposed change

Create an ops-first manifest + loader that eliminates authenticated API calls during training and enforces studio reuse.

Files to add/modify:
- `/opt/axentx/vanguard/scripts/generate_file_manifest.py` (new) — one-time Mac-side HF API call to produce `manifests/{repo}/{dateFolder}.json`.
- `/opt/axentx/vanguard/training/train.py` (modify) — switch to CDN-only streaming using the manifest (zero API calls).
- `/opt/axentx/vanguard/ops/launch_studio.py` (modify) — list existing studios and reuse a running one before creating.

## 3. Implementation

### 3.1 generate_file_manifest.py (new)

```python
#!/usr/bin/env python3
"""
Generate a file-list manifest for a repo+dateFolder to avoid HF API calls during training.
Run from Mac (or any env with HF token) when rate-limit window is clear.
"""
import json
import os
from pathlib import Path
from huggingface_hub import HfApi

HF_REPO = os.getenv("HF_REPO", "my-org/surrogate-1")
DATE_FOLDER = os.getenv("DATE_FOLDER", "2026-04-29")  # e.g. batches/mirror-merged/2026-04-29
OUT_DIR = Path("manifests") / HF_REPO / DATE_FOLDER

def main():
    api = HfApi()
    # Single non-recursive call per folder to avoid pagination explosion
    entries = api.list_repo_tree(
        repo_id=HF_REPO,
        path=DATE_FOLDER,
        repo_type="dataset",
        recursive=False,
    )
    files = [e.path for e in entries if e.type == "file"]
    OUT_DIR.mkdir(parents=True, exist_ok=True)
    out_path = OUT_DIR / "files.json"
    out_path.write_text(json.dumps({"repo": HF_REPO, "dateFolder": DATE_FOLDER, "files": files}, indent=2))
    print(f"Wrote {len(files)} files to {out_path}")

if __name__ == "__main__":
    main()
```

Make executable:
```bash
chmod +x /opt/axentx/vanguard/scripts/generate_file_manifest.py
```

### 3.2 train.py (modify)

Replace dataset loading with CDN-only streaming using the manifest.

```python
# Near top of train.py
import json
from pathlib import Path
import requests
from torch.utils.data import IterableDataset

MANIFEST_PATH = Path("manifests/my-org/surrogate-1/2026-04-29/files.json")
HF_REPO = "datasets/my-org/surrogate-1"

class CDNParquetIterable(IterableDataset):
    def __init__(self, manifest_path=MANIFEST_PATH, repo=HF_REPO):
        manifest = json.loads(manifest_path.read_text())
        self.urls = [
            f"https://huggingface.co/{repo}/resolve/main/{f}"
            for f in manifest["files"]
            if f.endswith(".parquet")
        ]
        self.buf_size = 1024 * 1024 * 16  # 16 MB chunks for parquet streaming

    def __iter__(self):
        for url in self.urls:
            with requests.get(url, stream=True, timeout=60) as r:
                r.raise_for_status()
                # Stream to temp file or direct pyarrow buffer as needed
                # Simplified: download chunked and yield projected rows
                import pyarrow.parquet as pq
                import io
                buffer = io.BytesIO()
                for chunk in r.iter_content(chunk_size=self.buf_size):
                    buffer.write(chunk)
                buffer.seek(0)
                table = pq.read_table(buffer, columns=["prompt", "response"])
                for batch in table.to_batches(max_chunksize=1024):
                    df = batch.to_pydict()
                    for i in range(len(df["prompt"])):
                        yield {"prompt": df["prompt"][i], "response": df["response"][i]}
```

Use in training loop:
```python
train_dataset = CDNParquetIterable()
# DataLoader with num_workers=0 for IterableDataset or use multi-process safe iterator
```

### 3.3 launch_studio.py (modify)

Reuse running studios to save quota.

```python
from lightning import Lightning, Teamspace, Machine, V100, L40S

lightning = Lightning()
teamspace = Teamspace()

def get_or_create_studio(name: str, machine=Machine.L40S):
    for s in teamspace.studios:
        if s.name == name and s.status == "Running":
            print(f"Reusing running studio: {s.name}")
            return s
    print(f"Creating studio: {name}")
    return lightning.Studio(
        name=name,
        machine=machine,
        create_ok=True,
    )

# Usage
studio = get_or_create_studio("vanguard-train-2026-04-29")
```
