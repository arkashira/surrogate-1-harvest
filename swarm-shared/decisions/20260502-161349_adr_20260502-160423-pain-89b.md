## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-file Rust CLI client (Linux x86_64) that: creates a local SOCKS5 listener on an ephemeral port; tunnels traffic through a user-provided obfuscated endpoint (shadowsocks-2022-ietf or utls-fingerprinted TLS) with optional DNS-over-HTTPS; spawns a TUN interface via boringtun for full-tunnel mode; enforces a fail-closed kill-switch (block non-tunnel IPv4/IPv6 when TUN is down); stores profiles and session logs locally in SQLite; emits encrypted, minimal WAU-ZB telemetry to a CF Worker only with opt-in; includes a basic TUI for profile selection and session status.

## Open questions
- Which obfuscation primitives (shadowsocks 2022, meek, Hysteria2-like QUIC, or utls-fingerprint) provide best evasion vs. performance on restrictive DPI?
- How will we distribute and rotate fronting endpoints/CDN profiles without central coordination (e.g., domain-fronting lists, DoH bootstrap)?
- What is the kill-switch granularity (per-app, system-wide TUN block, firewall rules) and how to implement cross-platform (Linux/Windows/macOS) without root/admin prompts?
- How to minimize local artifacts (TUN device names, firewall rules, ARP/ND cache entries) to avoid fingerprinting by local network monitoring?
- What telemetry schema and encryption scheme for WAU-ZB reports ensures privacy while allowing aggregate success measurement?
