# workio / design

## Final Synthesized Solution

Below is the single, production-ready plan that merges the strongest, non-contradictory insights from both proposals and resolves conflicts in favor of **correctness + concrete actionability**.

---

## 1. Diagnosis (merged)

- **No database-level idempotency**: LINE redeliveries create duplicate punch rows; uniqueness is enforced only in app logic, not at the DB layer.
- **Missing durable event log**: Ingested LINE events are written directly to `punches`; no immutable event table for replay, audit, or safe retries.
- **Race-prone concurrent deliveries**: Multiple deliveries for the same `line_delivery_id` can interleave and insert duplicates because there is no unique constraint or `SELECT … FOR UPDATE` to serialize them.
- **Clock-in/out state is racy**: Concurrent deliveries can read stale `last_punch` and produce invalid state transitions or double entries.
- **Missing transactional boundary**: Punch write and state update (or downstream side-effects) are not atomic; partial failures can leave punch rows inserted without corresponding state or notification updates.
- **No transactional backfill path**: Schema lacks `line_delivery_id` on `punches`, so idempotency checks require full-row hashing or app-level locks instead of a cheap unique index.
- **Observability gap**: No `events` or `outbox` table means failures during side-effects (notifications, shifts) cannot be retried safely without duplicating downstream actions.

---

## 2. Proposed change (merged)

- **Files**:
  - `server/src/db/schema.sql` — add `line_delivery_id` and `events` table; add unique constraint/index and optional `punch_states` row-level locking.
  - `server/src/db/migrations/` — idempotent migration to add columns safely on existing installs.
  - `server/src/services/punchService.ts` — use `line_delivery_id` for atomic upsert via `ON CONFLICT DO NOTHING` and enforce valid state transitions inside the same transaction.
  - `server/src/routes/lineWebhook.ts` — handler that calls the service with clear return semantics.
- **Scope**:
  - Add one unique partial index on `punches(line_delivery_id)` where `line_delivery_id IS NOT NULL`.
  - Add durable `events` table for append-only ingestion and replay.
  - Add optional `punch_states(tenant_id, employee_id, last_punch_id, updated_at)` with row-level locking to serialize clock-in/out transitions.
  - Update punch creation to be idempotent and state-safe in a single transaction.

---

## 3. Implementation

### 3.1 Schema changes (`server/src/db/schema.sql`)

```sql
-- Add line_delivery_id to punches if not present (safe to run multiple times)
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM information_schema.columns
                 WHERE table_name='punches' AND column_name='line_delivery_id') THEN
    ALTER TABLE punches ADD COLUMN line_delivery_id VARCHAR(255);
  END IF;
END $$;

-- Idempotency guard: one delivery = one punch row
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_delivery_id_unique
  ON punches (line_delivery_id)
  WHERE line_delivery_id IS NOT NULL;

-- Durable event log for LINE webhook ingestion (append-only)
CREATE TABLE IF NOT EXISTS events (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  event_type      VARCHAR(64) NOT NULL,
  external_id     VARCHAR(255) NOT NULL,            -- e.g., LINE delivery ID
  payload         JSONB NOT NULL,
  processed_at    TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (tenant_id, event_type, external_id)
);

-- Optional: fast replay / unprocessed lookup
CREATE INDEX IF NOT EXISTS idx_events_unprocessed
  ON events (tenant_id, event_type)
  WHERE processed_at IS NULL;

-- Optional: punch state table to enforce valid transitions and serialize concurrent updates
CREATE TABLE IF NOT EXISTS punch_states (
  tenant_id       UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  employee_id     UUID NOT NULL,
  last_punch_id   UUID REFERENCES punches(id) ON DELETE SET NULL,
  last_type       VARCHAR(4) CHECK (last_type IN ('IN','OUT')),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (tenant_id, employee_id)
);

-- Index for fast row locking
CREATE INDEX IF NOT EXISTS idx_punch_states_tenant_employee
  ON punch_states (tenant_id, employee_id);
```
