## Deployment
cf workers (free-tier) + Supabase (Postgres + edge functions) + Hugging Face Space (worker fallback)

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk guardrail→execution stub) with Postgres audit trail, Redis-backed queue, one mock broker connector, one real broker connector (sandbox), compliance rule DSL with block/warn severity, and replay endpoint to re-run a workflow_run deterministically from stored inputs and context_snapshot.

## Open questions
- Deterministic replay: should workflow inputs be canonicalized (canonical JSON) and signed to guarantee bitwise replay across environments?
- Latency SLO: what is the p99 target end-to-end (research→execution) on free-tier infra and what warm-up/cold-start mitigations are acceptable?
- Connector auth: will we store secrets in Supabase Vault or require per-tenant OAuth flows; how do we rotate without breaking replayability?
- Guardrail evaluation: should rules be evaluated in SQL (Postgres) for auditability or in-process Python for expressiveness and latency?
- Failure semantics: for idempotent execution, do brokers/OMS support idempotency keys; if not, how do we prevent double execution on retries?
