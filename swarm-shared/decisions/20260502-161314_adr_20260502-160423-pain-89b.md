## v0.1 scope
v0.1 ships a cross-platform CLI/daemon (Linux/Windows/macOS) with Tauri tray that: 1) establishes a local SOCKS5 listener with no identifiable socket artifacts; 2) wraps outbound traffic in utls-based TLS meek fronting to public CDNs; 3) tunnels via boringtun WireGuard to a single free-tier egress relay; 4) enforces a strict kill-switch (WFP/iptables/pf) on stealth failure; 5) records minimal, client-local session telemetry and optional encrypted push to Supabase for WAU-ZB aggregation; 6) supports zero-config presets and auto-updates via GitHub releases — all deployable with no server ownership beyond Supabase + CF Workers config endpoints.

## Open questions
- Which DPI signatures are highest risk in target regions and what local artifacts (TUN device names, socket paths, packet timing) must be masked?
- Can we rely solely on client-side stealth plus public CDN fronts (meek/tls) or do we need minimal OSS relays for reliable egress?
- What telemetry can we collect without deanonymizing users and how will WAU-ZB be computed client-side with privacy-preserving aggregates?
- How will kill-switch enforcement work across platforms (Windows Filtering Platform, nftables/iptables, pf) without leaking during stealth-layer churn?
- Which free-tier limits (Workers KV/R2, Supabase) constrain config rollout cadence and relay discovery freshness?
