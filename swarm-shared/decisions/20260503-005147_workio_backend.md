# workio / backend

## Final consolidated solution

### 1. Diagnosis (merged)
- Missing **database-level** protection against duplicate open punches: concurrent LINE webhook retries can insert multiple rows with `clock_out_at IS NULL` for the same user.
- No **idempotency key** for LINE at-least-once delivery; duplicate events within the same minute can create extra rows.
- Application-layer “best-effort” checks are not enough; races and retries corrupt state without a DB constraint.
- Missing **transactional upsert** pattern for clock-in so retries are idempotent at the database level.
- No `/readyz` (readiness) or `/healthz` (liveness) endpoints to let ops verify DB connectivity and schema before accepting traffic.

### 2. Scope (merged)
- **Files**:  
  - `workio/server/src/db/schema.sql` — add partial unique index and idempotency column/index.  
  - `workio/server/src/db/queries/punchQueries.ts` — add `upsertPunch` using `line_event_id` and partial unique constraint.  
  - `workio/server/src/server.ts` (or main routes file) — add `/readyz` and `/healthz`.  
  - Clock-in/out route/handler — use transaction + `upsertPunch` and handle conflicts gracefully.

### 3. Implementation

#### 3.1 DB schema migration (`schema.sql`)
```sql
-- Idempotency for LINE events (nullable; only non-null values enforced)
ALTER TABLE punches ADD COLUMN IF NOT EXISTS line_event_id TEXT;
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_event_id
  ON punches (line_event_id)
  WHERE line_event_id IS NOT NULL;

-- At most one open punch per user (prevents duplicate clock-ins)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
  ON punches (user_id)
  WHERE clock_out_at IS NULL;
```

#### 3.2 Idempotent upsert query (`punchQueries.ts`)
```ts
// workio/server/src/db/queries/punchQueries.ts
import { PoolClient } from 'pg';

export async function upsertPunch(
  client: PoolClient,
  {
    user_id,
    line_event_id,
    clock_in_at,
    latitude,
    longitude,
    image_url,
  }: {
    user_id: string;
    line_event_id?: string | null;
    clock_in_at: Date;
    latitude?: number;
    longitude?: number;
    image_url?: string;
  }
) {
  // If caller provides a line_event_id, try idempotent insert first.
  if (line_event_id) {
    const byEvent = await client.query(
      `INSERT INTO punches (user_id, line_event_id, clock_in_at, latitude, longitude, image_url)
       VALUES ($1, $2, $3, $4, $5, $6)
       ON CONFLICT (line_event_id) DO UPDATE
       SET user_id = EXCLUDED.user_id  -- no-op update to return row
       RETURNING *`,
      [user_id, line_event_id, clock_in_at, latitude, longitude, image_url]
    );
    if (byEvent.rowCount && byEvent.rows[0]) {
      return byEvent.rows[0];
    }
  }

  // Otherwise try to create a new open punch; if partial unique violation,
  // return the existing open punch.
  const result = await client.query(
    `INSERT INTO punches (user_id, line_event_id, clock_in_at, latitude, longitude, image_url)
     VALUES ($1, $2, $3, $4, $5, $6)
     ON CONFLICT DO NOTHING
     RETURNING *`,
    [user_id, line_event_id || null, clock_in_at, latitude, longitude, image_url]
  );

  if (result.rowCount && result.rows[0]) {
    return result.rows[0];
  }

  const existing = await client.query(
    `SELECT * FROM punches
     WHERE user_id = $1 AND clock_out_at IS NULL
     ORDER BY clock_in_at DESC
     LIMIT 1`,
    [user_id]
  );
  return existing.rows[0] || null;
}
```

#### 3.3 Clock-in handler (transactional + graceful conflict handling)
```ts
// Example: workio/server/src/routes/punch.ts or LINE webhook handler
import express from 'express';
import { pool } from '../db';
import { upsertPunch } from '../db/queries/punchQueries';

const router = express.Router();

router.post('/webhook/line/clockin', async (req, res) => {
  const { userId, lineEventId, latitude, longitude, imageUrl } = req.body;
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const punch = await upsertPunch(client, {
      user_id: userId,
      line_event_id: lineEventId,
      clock_in_at: new Date(),
      latitude,
      longitude,
      image_url: imageUrl,
    });
    await client.query('COMMIT');
    return res.json({ ok: true, punch });
  } catch (err: any) {
    await client.query('ROLLBACK');
    // Unique violation on partial index or line_event_id: return existing open punch.
    if (err.code === '23505') {
      const existing = await pool.query(
        `SELECT * FROM punches
         WHERE user_id = $1 AND clock_out_at IS NULL
         ORDER BY clock_in_at DESC
         LIMIT 1`,
        [userId]
      );
      return res.json({ ok: true, punch: existing.rows[0] || null });
    }
    console.error('Clock-in error', err);
    return res.status(500).json({ error: 'Internal error' });
  } finally {
    client.release();
  }
});

export default router;
```

#### 3.4 Health endpoints (`server.ts` or main app file)
```ts
// workio/server/src/server.ts (or app.ts)
import express from 'express';
import { pool } from './db';

const app = express();

// Liveness: process is alive (no external deps)
app.get('/healthz', (_req, res) => {
  res.status(200).json({ status: 'ok' });
});

// Readiness: DB reachable and core schema present
app.get('/readyz', async (_req, res) => {
  try {
    await pool.query('SELECT 1');
    const ok = await pool.query(
      "SELECT EXISTS (SELECT FROM information_schema.tables WHERE table_name = 'punches')"
    );
    if (ok.rows[0]?.exists) {
      return res.status(200).send('OK');
    }
    return res.status(503).send('DB schema not ready');
  } catch (err) {
    return res.status(503).send('DB unavailable');
  }
});

// Mount routes (example)
import punchRouter from './routes/punch';
app.use('/api', punchRouter);

export default app;
```
