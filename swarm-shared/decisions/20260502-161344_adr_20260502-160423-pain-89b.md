## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-platform (Linux x86_64) CLI client that: 1) creates an ephemeral local SOCKS5 listener on a randomized port, 2) wraps traffic via a pluggable obfuscation transport (Shadowsocks-like or meek) to a hardcoded community bridge, 3) optionally brings up a TUN device to capture and redirect system traffic, 4) enforces a fail-closed kill-switch that blocks plaintext on stealth failure, and 5) emits minimal, client-side encrypted session telemetry (WAU-ZB signals) to Supabase via CF Workers endpoints.

## Open questions
- Which obfuscation primitives provide the best reachability vs. performance trade-off in Russia today (meek, domain-front, obfs4, Shadowsocks 2022)?
- Can we rely solely on client-side TUN without root/admin on desktop and mobile, and what fallback path do we use if not?
- How will we distribute/rotate upstream endpoints and obfuscation parameters without a managed backend?
- What minimal client-side telemetry schema preserves privacy while enabling WAU-ZB measurement?
- How do we implement a reliable cross-platform kill-switch that blocks plaintext leaks when stealth path fails?
