# workio / ops

## 1. Diagnosis

- Webhook retries and concurrent redeliveries can create duplicate punches because `punches` lacks an idempotency key and unique constraint for active punches.
- Punch creation uses read-then-insert (non-atomic) → race between duplicate webhook deliveries can insert two rows for the same `(employee_id, punch_type, date)` active punch.
- No DB-level enforcement for “one active punch per employee per day” → application logic is the only guard and is not safe under concurrency.
- Missing traceability for webhook replays (no `line_delivery_id` or idempotency key stored) → hard to debug duplicates.
- No retry/backoff or de-dup layer at webhook handler → retries from LINE OA hit the same non-idempotent path.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/db/schema.sql`  
Scope: add idempotency support and uniqueness constraint to `punches` table.

File: `/opt/axentx/workio/server/src/routes/line-webhook.js` (or equivalent)  
Scope: make punch creation idempotent using `line_delivery_id` (or generated idempotency key) and `ON CONFLICT DO NOTHING`.

## 3. Implementation

### 3.1 Schema change (add idempotency + constraint)

```sql
-- /opt/axentx/workio/server/src/db/schema.sql
-- Add idempotency key and unique constraint for active punches

-- If punches.line_delivery_id does not exist, add it
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS line_delivery_id VARCHAR(255);

-- Ensure we can safely enforce one active punch per employee per day
-- (active = no clock_out_at). Use partial index for active rows.
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_active_employee_day
  ON punches (employee_id, DATE(clock_in_at))
  WHERE clock_out_at IS NULL;

-- Optional: index for idempotency lookups
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_delivery_id
  ON punches (line_delivery_id)
  WHERE line_delivery_id IS NOT NULL;
```

### 3.2 Idempotent punch creation in webhook handler

```js
// /opt/axentx/workio/server/src/routes/line-webhook.js
const express = require('express');
const router = express.Router();
const db = require('../db');

router.post('/line', async (req, res) => {
  const { events } = req.body;
  const deliveryId = req.headers['x-line-delivery-id'] || generateIdempotencyKey(req.body);

  for (const event of events) {
    if (event.type !== 'message' || event.message.type !== 'text') continue;
    const text = event.message.text.trim().toLowerCase();
    if (!['clock in', 'clock out'].includes(text)) continue;

    const employeeId = await resolveEmployeeIdByLineUserId(event.source.userId);
    if (!employeeId) continue;

    const punchType = text === 'clock in' ? 'in' : 'out';

    // Idempotent upsert: if same deliveryId exists, skip
    // Otherwise create punch with active uniqueness enforced by DB
    await db.query(
      `INSERT INTO punches (employee_id, punch_type, clock_in_at, line_delivery_id)
       SELECT $1, $2, NOW(), $3
       WHERE NOT EXISTS (
         SELECT 1 FROM punches WHERE line_delivery_id = $3
       )
       ON CONFLICT (line_delivery_id) DO NOTHING
       RETURNING *`,
      [employeeId, punchType, deliveryId]
    );
  }

  res.status(200).send('OK');
});

function generateIdempotencyKey(body) {
  const crypto = require('crypto');
  return crypto.createHash('sha256').update(JSON.stringify(body)).digest('hex');
}

async function resolveEmployeeIdByLineUserId(lineUserId) {
  const result = await db.query(
    'SELECT id FROM employees WHERE line_user_id = $1 LIMIT 1',
    [lineUserId]
  );
  return result.rows[0]?.id || null;
}

module.exports = router;
```

   - After clock-in, send clock-out event (different punch_type).
