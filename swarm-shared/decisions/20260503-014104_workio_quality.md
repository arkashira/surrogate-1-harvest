# workio / quality

## 1. Diagnosis

- LINE webhook endpoint lacks idempotency handling — retries or duplicate deliveries create duplicate punches/leave/OT records.
- Punch state transitions are not race-safe — concurrent webhook deliveries can create multiple open punches for same user/tenant.
- No unique constraint or upsert path for `punches` (and similar for leave/OT) to enforce at-most-one-open-per-user.
- Missing request deduplication key (LINE `deliveryId` or idempotency token) stored to reject replays within a safe window.
- Clock-in/out business rules (max distance, stale sessions) are enforced in app code instead of DB-level constraints, allowing races.

## 2. Proposed change

File scope:
- `workio/server/src/routes/webhook/line.ts` — add idempotency middleware and transactional upsert for events.
- `workio/server/src/db/schema.sql` — add unique constraint/index for open punches and idempotency keys.
- `workio/server/src/services/punchService.ts` — convert `find-then-insert/update` to atomic `upsert` using `ON CONFLICT`.

## 3. Implementation

### 3.1 DB schema changes (`workio/server/src/db/schema.sql`)

```sql
-- Idempotency keys for LINE webhook events
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  idempotency_key TEXT NOT NULL,
  tenant_id       INTEGER NOT NULL,
  event_type      TEXT NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (idempotency_key)
);

-- Index to expire old keys (optional cleanup job)
CREATE INDEX IF NOT EXISTS idx_line_idempotency_created
  ON line_webhook_idempotency (created_at);

-- Ensure at most one open punch per user per tenant
-- Assumes punches has: user_id, tenant_id, clock_out_at (nullable when open)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_open_unique
  ON punches (tenant_id, user_id)
  WHERE clock_out_at IS NULL;
```

### 3.2 Idempotency utility (`workio/server/src/lib/idempotency.ts`)

```ts
import { PoolClient } from 'pg';

export async function ensureIdempotent(
  client: PoolClient,
  key: string,
  tenantId: number,
  eventType: string
): Promise<boolean> {
  const res = await client.query(
    `INSERT INTO line_webhook_idempotency (idempotency_key, tenant_id, event_type)
     VALUES ($1, $2, $3)
     ON CONFLICT (idempotency_key) DO NOTHING
     RETURNING idempotency_key`,
    [key, tenantId, eventType]
  );
  return res.rowCount === 1;
}
```

### 3.3 Punch service upsert (`workio/server/src/services/punchService.ts`)

```ts
import { PoolClient } from 'pg';

export async function upsertPunch(
  client: PoolClient,
  {
    tenantId,
    userId,
    latitude,
    longitude,
    imageUrl,
    clockInAt,
    clockOutAt,
  }: {
    tenantId: number;
    userId: number;
    latitude?: number;
    longitude?: number;
    imageUrl?: string;
    clockInAt?: Date;
    clockOutAt?: Date;
  }
) {
  // If clockOutAt provided => close the open punch
  if (clockOutAt) {
    const res = await client.query(
      `UPDATE punches
       SET clock_out_at = $1, updated_at = NOW()
       WHERE tenant_id = $2 AND user_id = $3 AND clock_out_at IS NULL
       RETURNING *`,
      [clockOutAt, tenantId, userId]
    );
    return res.rows[0] || null;
  }

  // Otherwise attempt to create an open punch (conflict -> no-op)
  const res = await client.query(
    `INSERT INTO punches (tenant_id, user_id, clock_in_at, latitude, longitude, image_url)
     VALUES ($1, $2, $3, $4, $5, $6)
     ON CONFLICT (tenant_id, user_id) WHERE clock_out_at IS NULL
     DO NOTHING
     RETURNING *`,
    [tenantId, userId, clockInAt || new Date(), latitude, longitude, imageUrl]
  );

  return res.rows[0] || null;
}
```

### 3.4 Webhook endpoint with transaction + idempotency (`workio/server/src/routes/webhook/line.ts`)

```ts
import express from 'express';
import { pool } from '../../db';
import { ensureIdempotent } from '../../lib/idempotency';
import { upsertPunch } from '../../services/punchService';

const router = express.Router();

router.post('/line', async (req, res) => {
  const body = req.body;
  const events = body.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'No events' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      // Use LINE deliveryId as idempotency key
      const deliveryId = ev?.source?.deliveryId || ev?.deliveryId;
      const tenantId = /* resolve tenant from source.userId or channel secret mapping */ 1;
      const eventType = ev.type;

      if (!deliveryId) {
        // If LINE omits deliveryId, create deterministic key from content hash
        continue;
      }

      const ok = await ensureIdempotent(client, deliveryId, tenantId, eventType);
      if (!ok) {
        // Duplicate delivery — skip but continue processing others in batch
        continue;
      }

      // Handle message events (clock in/out via LINE message)
      if (ev.type === 'message' && ev.message?.type === 'text') {
        const text = ev.message.text.trim().toLowerCase();
        const userId = ev.source.userId;

        if (text === 'clock in' || text === 'เข้างาน') {
          await upsertPunch(client, {
            tenantId,
            userId,
            clockInAt: new Date(),
            latitude: /* from location if available */ undefined,
            longitude: undefined,
            imageUrl: undefined,
          });
        } else if (text === 'clock out' || text === 'เลิกงาน') {
          await upsertPunch(client, {
            tenantId,
            userId,
            clockOutAt: new Date(),
          });
        }
      }

      // TODO: handle leave/OT events similarly with idempotent upserts
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'Processing failed' });
  } finally {
    client.release();
  }
});

export default router;
```

## 4. Verification

- Run migrations: `psql workio < server/src/db/schema.sql` and confirm indexes exist.
- Unit test: call `upsertPunch` twice concurrently for same tenant/user with clockInAt only — second call must return null and not create a second open punch.
- Idempotency test: simulate LINE retry by POSTing same payload with same `deliveryId` twice within transaction — second request must skip processing and return 200 without creating duplicate records.
- Manual smoke:
  1. Start server.
  2. Send `clock in` message from LINE dev console for a user.
  3. Verify one open punch row exists.
  4. Resend same message (same deliveryId or simulate retry) — confirm no second punch row.
  5. Send `clock out` — confirm open punch closed and no additional open punches remain.
