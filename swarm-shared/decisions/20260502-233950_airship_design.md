### Refactor airship discover for Deterministic, CDN-Only Orchestrator

#### Context

`airship discover` relies on HF API for file listing, which is rate-limited and can cause PyArrow schema errors. To eliminate these issues, we'll refactor `airship discover` to use CDN-only file listing, ensuring deterministic behavior and reproducible file manifests.

#### Implementation

1. **Update `airship discover` to use CDN-only file listing**:
	* Use `https://huggingface.co/datasets/{repo}/resolve/main/{path}` for file listing instead of HF API.
	* Remove rate limiting and PyArrow schema error handling.
2. **Implement deterministic file manifest generation**:
	* Use a consistent, reproducible sorting order for file listing.
	* Include file metadata (e.g., size, last modified) in the manifest.
3. **Integrate with Surrogate AI**:
	* Update Surrogate AI to use the deterministic file manifest generated by `airship discover`.
	* Ensure Surrogate AI is aware of the CDN-only file listing used by `airship discover`.

#### Code Snippets

```bash
# airship/discover.py
import requests

def discover_files(repo, path):
    url = f"https://huggingface.co/datasets/{repo}/resolve/main/{path}"
    response = requests.get(url)
    files = response.json()["files"]
    # Sort files consistently and reproducibly
    files.sort(key=lambda x: x["name"])
    return files

# airship/manifest.py
import json

def generate_manifest(files):
    manifest = []
    for file in files:
        manifest.append({
            "name": file["name"],
            "size": file["size"],
            "last_modified": file["last_modified"]
        })
    return json.dumps(manifest)
```
