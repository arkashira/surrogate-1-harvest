# workio / design

## 1. Diagnosis

- Missing **idempotency key** on `/webhook/line` allows LINE retries (network blips, 5xx, slow client) to create duplicate punches for the same clock event.
- No **DB-level guard** enforcing “one open punch per user” (`clock_out_at IS NULL`) — race conditions under concurrency can create multiple open records.
- Punch writes are **non-transactional** across validation + insert; partial failures leave inconsistent state.
- No **deduplication window** for near-simultaneous events (same user, same minute) — duplicates with slightly different timestamps slip through.
- Audit trail lacks **request-level trace id** — hard to correlate retries/duplicates in logs.

## 2. Proposed change

File: `workio/server/src/routes/webhook/line.ts` (or equivalent route handler for LINE webhook).  
Scope: add idempotency key extraction, unique constraint + upsert logic, and transactional write with trace id.

## 3. Implementation

```sql
-- workio/server/src/db/schema.sql
-- Add idempotency table + constraint for open punches

CREATE TABLE IF NOT EXISTS line_punch_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  user_id INTEGER NOT NULL,
  event_type VARCHAR(50) NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  punch_id INTEGER
);

-- Ensure one open punch per user (soft-delete safe)
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;
```

```ts
// workio/server/src/routes/webhook/line.ts
import { Router, Request, Response } from 'express';
import { db } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = Router();

// Helper: deterministic idempotency key from LINE webhook event
function makeIdempotencyKey(event: any): string {
  // Use timestamp + userId + event type (clock_in/clock_out) when present; fallback to hash
  const ts = event.timestamp || Date.now();
  const uid = event.source?.userId || 'unknown';
  const type = (event.message?.text || '').includes('ออก') ? 'clock_out' : 'clock_in';
  return `line:${uid}:${ts}:${type}`;
}

router.post('/line', async (req: Request, res: Response) => {
  const traceId = uuidv4();
  const events = req.body.events || [];

  try {
    for (const event of events) {
      if (!event.source?.userId) continue;

      const idempotencyKey = makeIdempotencyKey(event);
      const userId = event.source.userId;
      const isClockOut = (event.message?.text || '').includes('ออก');
      const now = new Date();

      await db.tx(async (t: any) => {
        // 1) Idempotency check
        const existing = await t.oneOrNone(
          'SELECT punch_id FROM line_punch_idempotency WHERE idempotency_key = $1',
          [idempotencyKey]
        );

        if (existing && existing.punch_id) {
          // Already processed — return success (idempotent)
          return;
        }

        // 2) Fetch current open punch (if any)
        const openPunch = await t.oneOrNone(
          'SELECT id FROM punches WHERE user_id = $1 AND clock_out_at IS NULL',
          [userId]
        );

        let punchId: number | null = null;

        if (isClockOut) {
          // Close open punch if exists
          if (openPunch) {
            const updated = await t.one(
              'UPDATE punches SET clock_out_at = $1 WHERE id = $2 RETURNING id',
              [now, openPunch.id]
            );
            punchId = updated.id;
          }
          // If no open punch, optionally create a closed punch (edge-case) or ignore
        } else {
          // Clock in: ensure no open punch exists (guard via unique index)
          if (openPunch) {
            // Optional: auto-close stale open punch? For now, skip duplicate open.
            // Create idempotency record and skip.
          } else {
            const inserted = await t.one(
              'INSERT INTO punches (user_id, clock_in_at, clock_out_at, created_at) VALUES ($1, $2, $3, $4) RETURNING id',
              [userId, now, null, now]
            );
            punchId = inserted.id;
          }
        }

        // 3) Record idempotency
        await t.none(
          'INSERT INTO line_punch_idempotency (idempotency_key, user_id, event_type, punch_id) VALUES ($1, $2, $3, $4)',
          [idempotencyKey, userId, isClockOut ? 'clock_out' : 'clock_in', punchId]
        );
      });
    }

    res.status(200).json({ traceId, ok: true });
  } catch (err: any) {
    console.error(`[LINE webhook] traceId=${traceId}`, err);
    // Return 200 to prevent LINE retry on client errors we can't recover; log for investigation
    res.status(200).json({ traceId, ok: false, error: err.message });
  }
});

export default router;
```

Notes:
- Uses `db.tx` (assumes `pg-promise` or similar transaction support) to make validation + insert atomic.
- Unique index `idx_one_open_punch_per_user` prevents concurrent inserts of multiple open punches.
- Idempotency table prevents duplicate processing on retries.
- Trace id (`traceId`) added to logs and response for correlation.

## 4. Verification

1. **Unit test idempotency**: send same LINE event payload twice; assert one punch row and one idempotency row.
2. **Concurrency test**: simulate parallel requests for same user clock-in; assert only one open punch exists (unique index violation should be prevented by transaction check).
3. **Clock-out pairing**: create open punch, send clock-out event; assert punch updated and idempotency recorded.
4. **Retry safety**: replay captured LINE webhook payloads (with same timestamps) after 5m; confirm no duplicate punches.
5. **Log correlation**: verify `traceId` appears in logs for request and any downstream errors.
