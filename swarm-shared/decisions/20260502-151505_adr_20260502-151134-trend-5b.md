## v0.1 scope
v0.1 ships in 2 weeks: deterministic workflow engine (Postgres event log + Redis Streams), 3 pre-built connectors (1 broker sandbox, 1 OMS mock, 1 data vendor mock), compliance guardrails (hard limits, allowlists, per-instrument caps) with immutable audit trail, API for submit/query workflow runs, replay endpoint to re-run by inputs_hash, and Grafana dashboards for latency + NSM (successful workflow runs weighted by fee_generating).

## Open questions
- Deterministic replay: will Redis Streams + Postgres event log suffice for strict exactly-once replay under concurrent runs?
- Latency SLA: what is the p99 budget from signal ingestion to execution send on free-tier infra (CF Workers + Supabase + Redis)?
- Compliance rule engine: DSL vs imperative checks — which balances auditability and extensibility for week-1?
- Idempotency strategy: idempotency keys at API gateway vs application layer given CF Workers limits?
- Broker/OMS sandbox credentials and contract formats — which vendors will be supported in v0.1?
- Secrets/credentials management on free-tier (CF Workers KV vs Supabase + env) and rotation cadence
