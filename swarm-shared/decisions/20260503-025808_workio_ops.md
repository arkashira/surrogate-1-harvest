# workio / ops

## Final Synthesis — Correct, Actionable Implementation

**Core diagnosis (accepted from both candidates):**
- LINE retries on 5xx/timeouts; webhook has no idempotency → duplicate punches.
- No tenant+employee uniqueness for open punches → race conditions and corrupt state.
- No atomic transaction + idempotency across webhook and punch endpoints.
- Missing server-side invariant: at most one open punch per tenant+employee.

**Chosen approach:**  
Use a **persistent idempotency table** + **partial unique index** + **transactional upsert** in a shared service. Apply the same service to webhook and REST/GraphQL endpoints. Treat unique violations as duplicates and return current open punch.

## 1. DB schema (`workio/server/src/db/schema.sql`)

```sql
-- Idempotency for LINE webhook deliveries and punch mutations
CREATE TABLE IF NOT EXISTS punch_idempotency (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  employee_id     UUID NOT NULL REFERENCES employees(id) ON DELETE CASBLE,
  external_id     TEXT NOT NULL,               -- LINE deliveryId or client idempotency key
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  CONSTRAINT uniq_tenant_employee_external UNIQUE (tenant_id, employee_id, external_id)
);

-- At most one open punch per tenant+employee
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_employee
ON punches (tenant_id, employee_id)
WHERE clocked_out_at IS NULL;
```

*Notes:*  
- Index enforces invariant at DB level (strongest guarantee).  
- `punch_idempotency` scoped by tenant+employee+external_id prevents cross-tenant collisions and allows safe retries.

## 2. Service layer (`workio/server/src/services/punchService.ts`)

```ts
import { pool } from '../db';

export interface PunchInput {
  tenantId: string;
  employeeId: string;
  clockInAt?: string;   // ISO timestamp
  clockOutAt?: string;  // ISO timestamp
  location?: { lat: number; lng: number };
  idempotencyKey: string; // external_id
}

/**
 * Create or update a punch with idempotency and open-punch uniqueness.
 * Returns the punch row after operation (or existing open punch if duplicate).
 */
export async function createOrUpdatePunch(input: PunchInput) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency check
    const idem = await client.query(
      `SELECT 1 FROM punch_idempotency
       WHERE tenant_id = $1 AND employee_id = $2 AND external_id = $3`,
      [input.tenantId, input.employeeId, input.idempotencyKey]
    );

    if (idem.rows.length > 0) {
      // Already processed: return current open punch (if any)
      const punch = await client.query(
        `SELECT * FROM punches
         WHERE tenant_id = $1 AND employee_id = $2 AND clocked_out_at IS NULL
         ORDER BY clocked_in_at DESC LIMIT 1`,
        [input.tenantId, input.employeeId]
      );
      await client.query('ROLLBACK');
      return punch.rows[0] || null;
    }

    // 2) Record idempotency key first (prevents races)
    await client.query(
      `INSERT INTO punch_idempotency (tenant_id, employee_id, external_id)
       VALUES ($1, $2, $3)`,
      [input.tenantId, input.employeeId, input.idempotencyKey]
    );

    // 3) Clock-in: insert new open punch
    if (input.clockInAt && !input.clockOutAt) {
      const res = await client.query(
        `INSERT INTO punches (tenant_id, employee_id, clocked_in_at, location)
         VALUES ($1, $2, $3, $4)
         RETURNING *`,
        [input.tenantId, input.employeeId, input.clockInAt, input.location || null]
      );
      await client.query('COMMIT');
      return res.rows[0];
    }

    // 4) Clock-out: close the open punch
    if (input.clockOutAt) {
      const res = await client.query(
        `UPDATE punches
         SET clocked_out_at = $1, location_out = $2
         WHERE tenant_id = $3 AND employee_id = $4 AND clocked_out_at IS NULL
         RETURNING *`,
        [input.clockOutAt, input.location || null, input.tenantId, input.employeeId]
      );
      await client.query('COMMIT');
      return res.rows[0] || null;
    }

    await client.query('COMMIT');
    return null;
  } catch (err: any) {
    await client.query('ROLLBACK');

    // Unique violation on idempotency or open-punch index -> duplicate
    if (err.code === '23505') {
      const punch = await client.query(
        `SELECT * FROM punches
         WHERE tenant_id = $1 AND employee_id = $2 AND clocked_out_at IS NULL
         ORDER BY clocked_in_at DESC LIMIT 1`,
        [input.tenantId, input.employeeId]
      );
      return punch.rows[0] || null;
    }

    throw err;
  } finally {
    client.release();
  }
}
```

**Why this is correct + actionable:**
- Records idempotency key **before** punch mutation → safe against races.
- Uses DB constraints as source of truth for uniqueness.
- Handles unique violations gracefully (returns current open punch).
- Single transaction ensures atomicity across idempotency + punch.

## 3. LINE webhook handler (`workio/server/src/routes/webhook/line.ts`)

```ts
import { Router } from 'express';
import { createOrUpdatePunch } from '../../services/punchService';
import { verifyLineSignature } from '../../utils/lineCrypto';
import { resolveEmployeeByLineUserId } from '../../services/employeeService'; // implement

const router = Router();

router.post('/line', async (req, res) => {
  try {
    const sig = req.headers['x-line-signature'] as string;
    const body = JSON.stringify(req.body);
    if (!verifyLineSignature(body, sig)) {
      return res.status(401).send('Invalid signature');
    }

    const events = req.body.events || [];
    for (const ev of events) {
      if (ev.type !== 'message' || ev.message.type !== 'text') continue;

      const text = ev.message.text.trim().toLowerCase();
      const tenantId = process.env.DEFAULT_TENANT_ID; // replace with resolved tenant
      const employeeId = await resolveEmployeeByLineUserId(ev.source.userId);
      if (!employeeId) continue;

      // Use LINE deliveryId as idempotency key; fallback for safety
      const idempotencyKey = ev.deliveryId || `${ev.timestamp}-${ev.source.userId}`;

      if (text === 'clock in' || text === 'เข้างาน') {
        await createOrUpdatePunch({
          tenantId,
          employeeId,
          clockInAt: new Date().toISOString(),
          idempotencyKey
        });
      } else if (text === 'clock out' || text === 'เลิกงาน') {
        await createOrUpdatePunch({
          tenantId,
          employeeId,
          clockOutAt: new Date().toISOString(),
          idempotencyKey
        });
      }
    }

    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('LINE webhook error', err);
    // Return 500 so LINE retries; idempotency prevents duplicates
    res.status(500).json({ error: 'internal' });
  }
});

export default router;
```

**Notes:**
- Signature verification is required.
- Use `deliveryId` when available; it is stable for retries.
- Errors return 500 so LINE retries; idempotency ensures safety.
