# surrogate-1 / design

## Final synthesized implementation

**Highest-value improvement**  
Add a **deterministic pre-flight snapshot** (`bin/snapshot.sh`) that lists dataset files once per date folder, saves a manifest, and updates ingestion workers to use **CDN-only downloads** during parallel shard processing. This eliminates HF API rate-limit risk during ingestion and aligns with the training pattern: pre-list once, embed manifest, fetch via CDN only.

---

### Implementation plan (≤2h)

1. **Create `bin/snapshot.sh`** (20–30m)  
   - Uses `huggingface_hub.HfApi.list_repo_tree(recursive=False)` for a date folder (or root).  
   - Saves deterministic `snapshot/<date>/files.json` (sorted, reproducible).  
   - Exits non-zero on 429 and prints retry-after guidance; caller can retry after 360s.  
   - Stores manifest as workflow artifact and optionally commits it.

2. **Update `bin/dataset-enrich.sh`** (30–45m)  
   - Accepts `SNAPSHOT_FILE` env var. If provided, reads manifest and downloads via CDN URLs (`resolve/main/...`) without recursive listing or repeated API calls.  
   - Falls back to existing streaming path when snapshot is absent.  
   - Keeps existing schema normalization and dedup logic unchanged (`lib/dedup.py` remains source of truth).

3. **Update GitHub Actions matrix (`ingest.yml`)** (30–45m)  
   - Add a `snapshot` job that runs before the 16-shard matrix.  
   - Upload snapshot as artifact and pass path to shards via `env.SNAPSHOT_FILE`.  
   - Keep filename pattern `batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl`.

4. **Minor hardening** (15m)  
   - Ensure workers tolerate transient CDN failures (retry/back-off).  
   - Validate manifest schema before use.  
   - Keep deterministic shard assignment via `hash(path) % TOTAL_SHARDS`.

### Code snippets

#### 1) `bin/snapshot.sh`
```bash
#!/usr/bin/env bash
# bin/snapshot.sh
# Generate deterministic file-list snapshot for a dataset repo/date folder.
# Usage:
#   HF_TOKEN=<token> ./bin/snapshot.sh \
#     --repo axentx/surrogate-1-training-pairs \
#     --date 2026-05-02 \
#     --out snapshot/2026-05-02/files.json

set -euo pipefail

REPO=""
DATE=""
OUT=""

while [[ $# -gt 0 ]]; do
  case "$1" in
    --repo) REPO="$2"; shift 2 ;;
    --date) DATE="$2"; shift 2 ;;
    --out)  OUT="$2"; shift 2 ;;
    *) echo "Unknown option: $1"; exit 1 ;;
  esac
done

if [[ -z "$REPO" || -z "$DATE" || -z "$OUT" ]]; then
  echo "Usage: $0 --repo <repo> --date <YYYY-MM-DD> --out <path>"
  exit 1
fi

mkdir -p "$(dirname "$OUT")"

python3 - <<PY
import os, json, sys, time
from huggingface_hub import HfApi, RepositoryError, HFValidationError

HF_TOKEN = os.environ.get("HF_TOKEN")
if not HF_TOKEN:
  print("HF_TOKEN is required", file=sys.stderr)
  sys.exit(1)

api = HfApi(token=HF_TOKEN)
repo = "${REPO}"
path = "${DATE}"  # folder inside dataset repo

# Single API call: non-recursive listing
try:
  tree = api.list_repo_tree(repo=repo, path=path, recursive=False)
except Exception as e:
  print(f"API error: {e}", file=sys.stderr)
  if "429" in str(e):
    print("Rate limited (429). Wait 360s before retry.", file=sys.stderr)
  sys.exit(1)

# Deterministic ordering
files = sorted([entry.path for entry in tree if entry.type == "file"])

out_path = "${OUT}"
with open(out_path, "w") as f:
  json.dump({"repo": repo, "path": path, "date": "${DATE}", "files": files}, f, indent=2)

print(f"Snapshot saved: {out_path} ({len(files)} files)")
PY
```

Make executable:
```bash
chmod +x bin/snapshot.sh
```

# bin/dataset-enrich.sh
# Worker script for a single shard.
# If SNAPSHOT_FILE is provided, uses CDN-only downloads for listed files.

set -euo pipefail

OUT_DIR="${OUT_DIR:-output}"
HF_TOKEN="${HF_TOKEN:-}"

OUT_DIR = "$OUT_DIR"
HF_TOKEN = os.environ.get("HF_TOKEN", "")

def deterministic_shard(slug, n):
    return int(hashlib.md5(slug.encode()).hexdigest(), 16) % n

def parse_file_to_pair(content_bytes, file_path):
    # Existing schema normalization logic (kept unchanged).
    # Project to {prompt, response} only at parse time.
    # Return dict or None.
    # ...
    return {"prompt": "...", "response": "..."}

def process_via_snapshot(snapshot_path):
    with open(snapshot_path) as f:
        manifest = json.load(f)
    files = manifest.get("files", [])
    results = []
    for fpath in files:
        slug = fpath
        if deterministic_shard(slug, TOTAL_SHARDS) != SHARD_ID:
            continue
        # CDN download — no Authorization header, bypasses API rate limit
        url = f"https://huggingface.co/datasets/{REPO}/resolve/main/{fpath}"
        try:
            r = requests.get(url, timeout=30)
            r.raise_for_status()
            pair = parse_file_to_pair(r.content, fpath)
            if pair:
                results.append(pair)
        except Exception as e:
            print(f"Failed {fpath}: {e}", file=sys.stderr)
    return results

def process_fallback():
    # Existing streaming path (if snapshot not provided)
    from datasets import load_dataset
    ds = load_dataset(REPO, streaming=True, split="train")
    results = []
    for i, item in enumerate(ds):
        slug = item.get("path", f"item-{i}")
        if deterministic_shard(slug, TOTAL_SHARDS) != SHARD_ID:
            continue
        pair = parse_file_to_pair(None, slug)
        if pair:
            results.append(pair)
    return results

if SNAPSHOT_FILE and os.path.isfile(SNAPSHOT_FILE):
    pairs = process_via_snapshot(SNAPSHOT_FILE)
else:
    pairs = process_fallback()

# Dedup via central store (import lib.dedup) — unchanged
# ...

# Write shard output
ts = datetime.utcnow().strftime("%Y%m%d-%H%M%S")
out_file = os.path.join(OUT_DIR, f"shard{SHARD_ID}-{ts}.jsonl")
with open(out_file, "w") as f:
    for p in pairs:
        f.write(json.dumps(p) + "\n")

print(f"Shard {SHARD_ID} wrote {len(pairs)} pairs to {out_file}")
PY
