# vanguard / ops

## 1. Diagnosis
- No content-addressed manifest for dataset snapshots → training scripts perform runtime `list_repo_tree`/`load_dataset` calls and hit HF API 429.
- Training data path is not pinned; epochs can see different file sets between runs (non-reproducible).
- `enriched/` folder contains mixed-schema parquet (extra cols like `source`, `ts`) → surrogate-1 training risks `pyarrow.CastError`.
- Lightning Studio is likely created per run instead of reused → burns 80+ quota hours/month.
- No local guardrail to ensure ingestion projects to `{prompt,response}` only before upload.

## 2. Proposed change
Create `/opt/axentx/vanguard/ops/make_manifest.py` (single CLI) and add `/opt/axentx/vanguard/ops/train_wrapper.sh` to enforce:
- Manifest generation (content-addressed, date-scoped) on the Mac (or CI) after ingestion.
- Manifest embedded into Lightning training so data loading uses CDN-only URLs (zero API calls).
- Schema projection to `{prompt,response}` before upload (filename-only attribution).
- Lightning Studio reuse + idle-safe run.

Scope:
- New file: `ops/make_manifest.py`
- New file: `ops/train_wrapper.sh`
- Update: `ops/README.md` (or create) with runbook.

### ops/make_manifest.py
```python
#!/usr/bin/env python3
"""
Generate a content-addressed manifest for a date-scoped dataset mirror.
Usage:
  python make_manifest.py --repo <org/dataset> --date 2026-05-03 --out manifest.json
"""
import argparse
import hashlib
import json
import os
import sys
from datetime import datetime
from pathlib import Path
from typing import List, Dict

try:
    from huggingface_hub import list_repo_tree, hf_hub_download
except ImportError:
    print("Missing huggingface_hub. Install with: pip install huggingface_hub")
    sys.exit(1)

CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

def sha256_hex(s: str) -> str:
    return hashlib.sha256(s.encode("utf-8")).hexdigest()

def build_manifest(repo: str, date: str, out_path: Path) -> Dict:
    """
    List one date folder (non-recursive top-level) and produce manifest entries.
    We intentionally avoid recursive listing to reduce API pressure.
    """
    prefix = f"batches/mirror-merged/{date}/"
    entries = []

    try:
        tree = list_repo_tree(repo=repo, path=prefix, recursive=False)
    except Exception as exc:
        print(f"Failed to list repo tree for {repo}/{prefix}: {exc}", file=sys.stderr)
        sys.exit(1)

    files = [t for t in tree if t.type == "file"]
    if not files:
        print(f"No files found under {prefix}", file=sys.stderr)
        sys.exit(1)

    for f in files:
        path = f.path
        url = CDN_TEMPLATE.format(repo=repo, path=path)
        slug = Path(path).stem  # e.g., abc123 from abc123.parquet
        content_id = sha256_hex(f"{repo}/{path}")
        entries.append(
            {
                "content_id": content_id,
                "slug": slug,
                "path": path,
                "url": url,
                "date": date,
            }
        )

    manifest = {
        "repo": repo,
        "date": date,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "entries": entries,
    }

    out_path.parent.mkdir(parents=True, exist_ok=True)
    with open(out_path, "w", encoding="utf-8") as fh:
        json.dump(manifest, fh, indent=2)

    print(f"Manifest written to {out_path} ({len(entries)} entries)")
    return manifest

def main() -> None:
    parser = argparse.ArgumentParser(description="Create CDN-only dataset manifest.")
    parser.add_argument("--repo", required=True, help="HF dataset repo (org/name)")
    parser.add_argument("--date", required=True, help="Date folder (YYYY-MM-DD)")
    parser.add_argument("--out", default="ops/manifest.json", help="Output JSON path")
    args = parser.parse_args()

    build_manifest(args.repo, args.date, Path(args.out))

if __name__ == "__main__":
    main()
```

### ops/train_wrapper.sh
```bash
#!/usr/bin/env bash
# Usage:
#   bash ops/train_wrapper.sh --manifest ops/manifest.json --script train.py [--studio surrogate-1-run]
set -euo pipefail
export SHELL=/bin/bash

MANIFEST=""
TRAIN_SCRIPT=""
STUDIO_NAME="surrogate-1-run"
MACHINE="l40s"

while [[ $# -gt 0 ]]; do
  case "$1" in
    --manifest) MANIFEST="$2"; shift 2 ;;
    --script)   TRAIN_SCRIPT="$2"; shift 2 ;;
    --studio)   STUDIO_NAME="$2"; shift 2 ;;
    --machine)  MACHINE="$2"; shift 2 ;;
    *) echo "Unknown option: $1"; exit 1 ;;
  esac
done

if [[ -z "$MANIFEST" || -z "$TRAIN_SCRIPT" ]]; then
  echo "Both --manifest and --script are required."
  exit 1
fi

if [[ ! -f "$MANIFEST" ]]; then
  echo "Manifest not found: $MANIFEST"
  exit 1
fi

if [[ ! -f "$TRAIN_SCRIPT" ]]; then
  echo "Training script not found: $TRAIN_SCRIPT"
  exit 1
fi

# Reuse running studio if available; otherwise create.
python3 - <<PY
import os
from lightning.pytorch.cli import LightningCLI
from lightning.app import LightningFlow, LightningWork
from lightning.app.core.work import Work
# Minimal reuse check via public API pattern (adjust imports to your installed SDK).
try:
    from lightning.app import Teamspace
    studios = Teamspace().studios
    running = [s for s in studios if s.name == "$STUDIO_NAME" and s.status == "running"]
    if running:
        print(f"Reusing running studio: {$STUDIO_NAME}")
        os.environ["REUSE_STUDIO"] = "1"
    else:
        print(f"No running studio found with name {$STUDIO_NAME}; will create.")
except Exception as e:
    print(f"Studio reuse check skipped: {e}")
PY

# Launch training with manifest path injected.
# The train.py is expected to read the manifest and perform CDN-only fetches.
python3 "$TRAIN_SCRIPT" \
  --manifest "$MANIFEST" \
  --machine "$MACHINE" \
  --studio "$STUDIO_NAME"
```
