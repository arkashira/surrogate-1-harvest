# airship / ops

## Implementation Plan — airship / ops

**Highest-value incremental improvement (<2h)**  
Ship a zero-runtime-dependency, CDN-safe `airship discover` orchestrator (Bash-first) that deterministically emits `status.json` + `manifest.json` for local/CI use and CDN upload. Frontends fetch via CDN URLs. This removes HF API calls during training/inference and enables reproducible discovery across environments.

### Why this now
- Aligns with recent decisions: CDN-bypass, deterministic file lists, Bash-first orchestration, and quota-safe Lightning reuse.
- Removes runtime HF API dependency for discovery (rate-limit + commit-cap safe).
- Enables CI/local parity and immediate frontend/CDN consumption.

---

### Concrete steps (≤2h)

1. **Create `airship/discover` CLI** (single Bash script)
   - Shebang `#!/usr/bin/env bash`, `set -euo pipefail`.
   - Accepts `--repo`, `--date`, `--out-dir`, `--cdn-base`.
   - Uses HF API **once** (or accepts pre-saved tree JSON) to list top-level folder for given date.
   - Saves `file-list.json` (deterministic sorted list of CDN paths).
   - Emits:
     - `status.json` — discovery metadata (repo, date, count, sha256sum of manifest, timestamp).
     - `manifest.json` — per-file metadata (cdn_url, size, sha256, surrogate-friendly {prompt,response} projection flag).
   - If HF API unavailable, accepts `--file-list` input to operate fully offline (CDN-only mode).

2. **CDN upload helper**
   - Optional `upload` subcommand: syncs `out-dir` to configured CDN destination (e.g., HF Space `spaces/arkship/discovery` or S3).
   - Uses `rclone` or `aws s3 sync` if available; otherwise prints commands for manual run.

3. **Integrate with training scripts**
   - Provide sample `train.py` snippet that loads `manifest.json` and downloads via CDN URLs (no HF API during training).
   - Include Lightning Studio reuse check stub (list running studios, reuse if available).

4. **CI/local parity**
   - Add `Makefile` targets: `discover-local`, `discover-ci`, `upload-cdn`.
   - Ensure outputs are deterministic (sorted keys, stable timestamps replaced by commit/tag).

5. **Validation**
   - Run `airship discover` against a small test repo/date and verify:
     - `status.json` and `manifest.json` produced.
     - CDN URLs resolve (curl --head).
     - No HF API calls occur after initial listing.

### Code snippets

#### `airship/discover` (main CLI)

```bash
#!/usr/bin/env bash
# airship/discover — CDN-safe discovery orchestrator
# Usage: ./discover --repo <repo> --date <YYYY-MM-DD> --out-dir ./out [--cdn-base <url>] [--file-list <json>]

set -euo pipefail

REPO=""
DATE=""
OUT_DIR="./out"
CDN_BASE=""
FILE_LIST=""
HF_TOKEN="${HF_TOKEN:-}"

while [[ $# -gt 0 ]]; do
  case "$1" in
    --repo) REPO="$2"; shift 2 ;;
    --date) DATE="$2"; shift 2 ;;
    --out-dir) OUT_DIR="$2"; shift 2 ;;
    --cdn-base) CDN_BASE="$2"; shift 2 ;;
    --file-list) FILE_LIST="$2"; shift 2 ;;
    --help) echo "Usage: $0 --repo <repo> --date <YYYY-MM-DD> --out-dir <dir> [--cdn-base <url>] [--file-list <json>]"; exit 0 ;;
    *) echo "Unknown option $1"; exit 1 ;;
  esac
done

if [[ -z "$REPO" || -z "$DATE" ]]; then
  echo "Error: --repo and --date are required."
  exit 1
fi

mkdir -p "$OUT_DIR"

TIMESTAMP="$(date -u +"%Y-%m-%dT%H:%M:%SZ")"

# Determine file list
if [[ -n "$FILE_LIST" && -f "$FILE_LIST" ]]; then
  echo "Using provided file list: $FILE_LIST"
  cp "$FILE_LIST" "$OUT_DIR/file-list.json"
else
  echo "Fetching repo tree for $REPO ($DATE) via HF API (single call)..."
  # HF API: list_repo_tree non-recursive for the date folder
  # Requires HF_TOKEN for private repos; public repos work without token but may hit rate limits.
  # We use /api endpoint once, then rely on CDN.
  if [[ -n "$HF_TOKEN" ]]; then
    curl -s -H "Authorization: Bearer $HF_TOKEN" \
      "https://huggingface.co/api/datasets/${REPO}/tree?path=${DATE}&recursive=false" \
      > "$OUT_DIR/file-list.json"
  else
    curl -s \
      "https://huggingface.co/api/datasets/${REPO}/tree?path=${DATE}&recursive=false" \
      > "$OUT_DIR/file-list.json"
  fi
fi

# Build manifest from file-list.json (jq required)
if ! command -v jq >/dev/null 2>&1; then
  echo "Error: jq is required to build manifest."
  exit 1
fi

# Deterministic sorted paths
mapfile -t PATHS < <(jq -r '.[] | select(.type=="file") | .path' "$OUT_DIR/file-list.json" | sort)

MANIFEST="$OUT_DIR/manifest.json"
echo "[" > "$MANIFEST"
FIRST=true
for p in "${PATHS[@]}"; do
  CDN_URL="${CDN_BASE:-https://huggingface.co/datasets/${REPO}/resolve/main}/${p}"
  # Get size via CDN HEAD (fast, no auth) — fallback to unknown
  SIZE=$(curl -sI "$CDN_URL" | awk '/^content-length:/ {print $2}' | tr -d '\r' || echo "0")
  # Compute local sha256 if file already present in out-dir/cache; otherwise placeholder
  LOCAL_FILE="$OUT_DIR/cache/$(basename "$p")"
  SHA256="null"
  if [[ -f "$LOCAL_FILE" ]]; then
    SHA256=$(sha256sum "$LOCAL_FILE" | awk '{print $1}')
    SHA256="\"$SHA256\""
  fi

  $FIRST && FIRST=false || echo "," >> "$MANIFEST"
  jq -n \
    --arg path "$p" \
    --arg cdn "$CDN_URL" \
    --argjson size "$SIZE" \
    --argjson sha256 "$SHA256" \
    '{path: $path, cdn_url: $cdn, size: $size, sha256: $sha256, surrogate_projection: true}' >> "$MANIFEST"
done
echo "]" >> "$MANIFEST"

# Deterministic manifest sha256
MANIFEST_SHA256=$(sha256sum "$MANIFEST" | awk '{print $1}')

# Status
jq -n \
  --arg repo "$REPO" \
  --arg date "$DATE" \
  --arg ts "$TIMESTAMP" \
  --arg manifest_sha "$MANIFEST_SHA256" \
  --argjson count "${#PATHS[@]}" \
  '{repo: $repo, date: $date, timestamp: $ts, manifest_sha256: $manifest_sha, file_count: $count, mode: "cdn-only"}' \
  > "$OUT_DIR/status.json"

echo "Done. Outputs in $OUT_DIR"
echo "- status.json: $OUT_DIR/status.json"
echo "- manifest.json: $MANIFEST"
```
