# workio / backend

## 1. Diagnosis

- **No idempotency on LINE webhook processing**: duplicate events from LINE retries can double-punch or double-approve leave/OT.
- **Missing strict `X-Line-Signature` verification**: requests can be spoofed or accepted without HMAC-SHA256 check.
- **No replay-protection window**: replayed webhook payloads (within minutes/hours) are accepted as new events.
- **Race condition on clock-in/out updates**: concurrent webhooks for same user can corrupt latest punch record.
- **Silent notification failures**: LINE notify/reply failures are not retried with backoff, so users don’t see confirmations.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts` (or equivalent webhook handler)  
Scope:
- Add `verifyLineSignature()` middleware.
- Add idempotency + replay guard (Redis preferred; fallback to in-memory LRU).
- Make punch/leave handling idempotent and race-safe (atomic upsert/transaction).
- Add exponential backoff retry for LINE replies/notifications.

```ts
// server/src/lib/lineAuth.ts
import CryptoJS from 'crypto-js';

export function verifyLineSignature(rawBody: string, signature: string, channelSecret: string): boolean {
  const expected = CryptoJS.HmacSHA256(rawBody, channelSecret).toString(CryptoJS.enc.Base64);
  return CryptoJS.timingSafeEqual(
    CryptoJS.enc.Utf8.parse(expected),
    CryptoJS.enc.Utf8.parse(signature)
  );
}
```

```ts
// server/src/lib/idempotency.ts
import Redis from 'ioredis';

const redis = new Redis(process.env.REDIS_URL || 'redis://localhost:6379');
const FALLBACK_TTL = 60 * 60; // 1h

export async function isReplay(key: string): Promise<boolean> {
  const exists = await redis.set(key, '1', 'EX', FALLBACK_TTL, 'NX');
  return exists === null; // null means key existed
}

// in-memory fallback for dev
const seen = new Map<string, number>();
export async function isReplayFallback(key: string): Promise<boolean> {
  const now = Date.now();
  if (seen.has(key) && now - seen.get(key)! < FALLBACK_TTL * 1000) return true;
  seen.set(key, now);
  return false;
}
```

```ts
// server/src/routes/lineWebhook.ts  (patch)
import express from 'express';
import { verifyLineSignature } from '../lib/lineAuth';
import { isReplay, isReplayFallback } from '../lib/idempotency';
import pRetry from 'p-retry';
import axios from 'axios';

const router = express.Router();
const CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET!;

async function replyWithRetry(replyToken: string, messages: any[]) {
  const url = 'https://api.line.me/v2/bot/message/reply';
  const body = { replyToken, messages };
  await pRetry(() => axios.post(url, body, {
    headers: { Authorization: `Bearer ${process.env.LINE_CHANNEL_ACCESS_TOKEN}` }
  }), {
    retries: 3,
    factor: 2,
    minTimeout: 500,
    maxTimeout: 4000,
    onFailedAttempt: (e) => console.warn('LINE reply retry', e.attemptNumber, e.message),
  });
}

router.post('/webhook/line', express.raw({ type: 'application/json' }), async (req, res) => {
  const sig = req.headers['x-line-signature'] as string;
  const raw = req.body.toString('utf8');

  if (!verifyLineSignature(raw, sig, CHANNEL_SECRET)) {
    return res.status(401).send('Invalid signature');
  }

  let events;
  try {
    events = JSON.parse(raw).events;
  } catch {
    return res.status(400).send('Invalid JSON');
  }

  // global idempotency key per webhook delivery (use X-Line-Delivery or generate from event+ts)
  const deliveryId = req.headers['x-line-delivery'] as string || `manual-${Date.now()}`;
  if (await (process.env.REDIS_URL ? isReplay(deliveryId) : isReplayFallback(deliveryId))) {
    return res.status(200).send('Replay ignored');
  }

  for (const ev of events) {
    try {
      await handleEvent(ev); // idempotent inside
    } catch (err) {
      console.error('Event handling failed', err);
      // continue processing other events
    }
  }

  res.status(200).send('OK');
});

// example idempotent handler (pseudo)
async function handleEvent(event: any) {
  if (event.type === 'message' && event.message.type === 'text') {
    const userId = event.source.userId;
    const text = event.message.text;
    // idempotent punch: upsert with unique constraint (user_id, punch_date, punch_type, line_delivery_id)
    // or use advisory lock / row-level locking to prevent races
    // leave/OT requests: use unique request idempotency key
    // after DB success, send LINE reply via replyWithRetry
    await replyWithRetry(event.replyToken, [{ type: 'text', text: 'บันทึกเรียบร้อย' }]);
  }
}

export default router;
```

## 4. Verification

1. **Signature verification**:
   - Send POST with wrong/missing `X-Line-Signature` → expect 401.
   - Send POST with valid signature (generate with channel secret) → expect 200.

2. **Idempotency / replay protection**:
   - Send same payload twice with same `X-Line-Delivery` (or simulate same idempotency key) → second request returns 200 but DB unchanged (no duplicate punch).

3. **Race condition**:
   - Fire two concurrent webhooks for same user within 1s → verify only one punch record created (check DB row count and latest timestamp).

4. **Retry behavior**:
   - Temporarily block LINE API (or mock 5xx) → confirm `p-retry` attempts 3 times with backoff and logs retries.

5. **End-to-end**:
   - Use LINE dev console to send test webhook → confirm user receives reply and DB state is consistent.
