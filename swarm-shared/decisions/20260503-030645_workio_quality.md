# workio / quality

## 1. Diagnosis
- Missing **LINE webhook signature verification** (`X-Line-Signature`) allows spoofed/replayed events to be accepted as valid.
- No **idempotency / replay protection** — LINE retries on 5xx/timeouts can create duplicate clock-in/out records and leave/OT requests.
- Clock-in/out and leave/OT handlers accept events without validating **business rules** (e.g., duplicate clock-in within grace window, overlapping leave).
- No **structured audit trail** for critical state changes (who/what/when) — limits traceability and debugging.
- Missing **input schema validation** on webhook payloads — malformed or unexpected payloads can crash handlers or corrupt state.

## 2. Proposed change
File scope:  
- `server/src/middleware/lineWebhook.ts` (new) — signature verification + idempotency middleware  
- `server/src/routes/webhook/line.ts` — integrate middleware and add idempotency + validation  
- `server/src/db/schema.sql` — add `webhook_events` table for replay detection and audit  
- `server/src/lib/validators.ts` — add Zod schema for LINE webhook payload

## 3. Implementation

export const LineWebhookSchema = z.object({
  destination: z.string(),
  events: z.array(
    z.object({
      type: z.string(),
      timestamp: z.number(),
      source: z.object({

        groupId: z.string().optional(),

      }),
      message: z
        .object({
          id: z.string(),
          type: z.string(),
          text: z.string().optional(),
        })
        .optional(),
      postback: z.object({ data: z.string() }).optional(),
      beacon: z.object({}).optional(),
      things: z.object({}).optional(),

### 3.3 Middleware (`server/src/middleware/lineWebhook.ts`)
```ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';
import { LineWebhookSchema } from '../lib/validators';
import { pool } from '../db';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

function verifySignature(rawBody: Buffer, signature: string): boolean {
  if (!LINE_CHANNEL_SECRET) return false;
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

export function lineSignatureMiddleware(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const signature = req.headers['x-line-signature'] as string;
  const rawBody = (req as any).rawBody as Buffer;

  if (!rawBody || !signature) {
    return res.status(400).json({ error: 'Missing body or signature' });
  }

  if (!verifySignature(rawBody, signature)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  next();
}

export async function lineIdempotencyMiddleware(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const payload = LineWebhookSchema.parse(req.body);
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    for (const ev of payload.events) {
      const exists = await client.query(
        'SELECT 1 FROM webhook_events WHERE line_event_id = $1',
        [ev.webhookEventId]
      );

      if (exists.rows.length > 0) {
        // Already processed — skip but continue marking others as seen
        continue;
      }

      await client.query(
        `INSERT INTO webhook_events
         (line_event_id, event_type, source_type, source_user_id, payload)
         VALUES ($1,$2,$3,$4,$5)`,
        [
          ev.webhookEventId,
          ev.type,
          ev.source.type,
          ev.source.userId || null,
          ev,
        ]
      );
    }

    await client.query('COMMIT');
    next();
  } catch (err) {
    await client.query('ROLLBACK');
    next(err);
  } finally {
    client.release();
  }
}
```

### 3.4 Route integration (`server/src/routes/webhook/line.ts`)
```ts
import express from 'express';
import { lineSignatureMiddleware, lineIdempotencyMiddleware } from '../middleware/lineWebhook';
import { handleLineEvent } from '../handlers/lineHandler';

const router = express.Router();

// Preserve raw body for signature verification
router.use('/line', (req, res, next) => {
  const chunks: Buffer[] = [];
  req.on('data', (chunk) => chunks.push(chunk));
  req.on('end', () => {
    (req as any).rawBody = Buffer.concat(chunks);
    next();
  });
});

router.post(
  '/line',
  lineSignatureMiddleware,
  lineIdempotencyMiddleware,
  async (req, res) => {
    try {
      // Process only new events (idempotency middleware stored them)
      await handleLineEvent(req.body);
      res.status(200).json({ ok: true });
    } catch (err) {
      console.error('LINE webhook handler error:', err);
      res.status(500).json({ error: 'handler_error' });
    }
  }
);

export default router;
```

## 4. Verification
- **Signature verification**: Send a POST with a wrong/missing `X-Line-Signature` → expect 401. With correct signature → 200.
- **Idempotency**: Send same LINE event JSON twice (same `webhookEventId`) → second request must not create duplicate `webhook_events` row and must not re-apply business actions (check logs + DB).
- **Schema validation**: Send malformed payload → expect 400 from Zod parse in middleware.
- **Audit trail**: After valid event, confirm row exists in `webhook_events` with correct `line_event_id` and payload.
- **End-to-end**: Use ngrok to expose local `/webhook/line`, configure LINE OA webhook, send test event from LINE console — verify 200 and correct DB state.
