# workio / backend

**Final Synthesized Plan — Highest-Impact, Correct, and Actionable**

**Core problem to solve now**  
Duplicate or inconsistent punch records caused by LINE webhook redelivery and concurrent requests, plus inability to detect/handle conflicts safely.

**Chosen solution (synthesis of strongest parts)**  
- **Idempotency key** on punch records (all candidates agree; prevents duplicates from redelivery).  
- **Optimistic concurrency control (version field)** to prevent lost updates and state corruption under concurrency (Candidate 1’s strongest, concrete mechanism).  
- **Atomic “insert-if-not-exists, else return existing”** at the persistence layer with unique constraint enforcement (correctness over blind retry).  
- **Explicit conflict handling and observability**: return existing record on idempotent replay; log and surface conflicts/errors; do not silently swallow or blindly retry on constraint violations.  
- **Schema + repository + service contract** (Candidate 1 + 3) rather than putting concurrency logic only in controller (Candidate 2 keeps it too high-level and less safe).

- Use `version` for optimistic locking on updates (if you allow punch edits).

2) **Repository: atomic upsert-by-idempotency with conflict detection**
```js
// repositories/punch.js
import { Punch } from '../models/punch';

export default class PunchRepository {
  // Create punch idempotently. Returns [record, created].
  async createPunch(punchData) {
    const { idempotencyKey, version = 0, ...fields } = punchData;

    // Try insert; on unique violation, fetch existing.
    try {
      const created = await Punch.create({
        idempotencyKey,
        version,
        ...fields,
      });
      return [created, true];
    } catch (err) {
      if (this._isUniqueViolation(err)) {
        const existing = await Punch.findOne({ where: { idempotencyKey } });
        return [existing, false]; // idempotent replay
      }
      throw err;
    }
  }

  // Update with optimistic concurrency (if needed)
  async updatePunch(idempotencyKey, updates, expectedVersion) {
    const result = await Punch.update(
      { ...updates, version: expectedVersion + 1 },
      {
        where: {
          idempotencyKey,
          version: expectedVersion,
        },
      }
    );
    const [affectedCount] = result;
    if (affectedCount === 0) {
      const conflict = await Punch.findOne({ where: { idempotencyKey } });
      if (!conflict) throw new Error('Punch not found');
      throw new Error('Punch record conflict: version mismatch');
    }
    return Punch.findOne({ where: { idempotencyKey } });
  }

  _isUniqueViolation(err) {
    // Adapt to your dialect: PostgreSQL unique_violation = '23505'
    return err.name === 'SequelizeUniqueConstraintError';
  }
}
```

3) **Service: clear contract and safe error handling**
```js
// services/punchService.js
import PunchRepository from '../repositories/punch';

export default class PunchService {
  constructor({ punchRepository } = {}) {
    this.punchRepository = punchRepository || new PunchRepository();
  }

  async recordPunch(punchInput) {
    if (!punchInput.idempotencyKey) {
      throw new Error('idempotencyKey is required');
    }

    const [record, created] = await this.punchRepository.createPunch({
      ...punchInput,
      version: 0,
    });

    return {
      record,
      created,
      message: created ? 'Punch recorded' : 'Idempotent replay: existing record returned',
    };
  }

  async updatePunch(idempotencyKey, updates, expectedVersion) {
    return this.punchRepository.updatePunch(idempotencyKey, updates, expectedVersion);
  }
}
```

4) **Controller: validate, call service, log, and return clear status**
```js
// controllers/punchController.js
import PunchService from '../services/punchService';

const punchService = new PunchService();

export async function createPunch(req, res) {
  try {
    const result = await punchService.recordPunch(req.body);
    // Log for observability
    console.info('Punch recorded', {
      idempotencyKey: req.body.idempotencyKey,
      created: result.created,
      punchId: result.record.id,
    });
    return res.status(result.created ? 201 : 200).json(result);
  } catch (err) {
    console.error('Punch record failed', {
      idempotencyKey: req.body.idempotencyKey,
      error: err.message,
    });
    if (err.message.includes('required') || err.message.includes('idempotencyKey')) {
      return res.status(400).json({ error: err.message });
    }
    if (err.message.includes('conflict')) {
      return res.status(409).json({ error: err.message });
    }
    return res.status(500).json({ error: 'Internal server error' });
  }
}
```

5) **Verification & Observability (must-do checks)**
- **Idempotency test**: replay same `idempotencyKey` payload multiple times; expect exactly one new row and identical returned record on replays.  
- **Concurrency test**: fire parallel requests with same key; expect no duplicates and no inconsistent punchIn/punchOut state.  
- **Conflict test**: attempt update with stale `version`; expect 409 and clear error.  
- **Logging**: ensure every failure and conflict is logged with `idempotencyKey` and error details.  
- **Monitoring**: add metrics (e.g., `punch.created`, `punch.idempotent_replay`, `punch.conflict`) and alert on spikes.

---

**Why this synthesis wins**
- Uses **database-level unique constraint** for strong correctness (Candidate 1 + 3), not just app-level checks.  
- Adds **versioning** for safe concurrent updates (Candidate 1), avoiding silent state corruption.  
- Keeps concurrency logic in **repository/service**, not only controller (more testable and reusable).  
- Avoids unsafe “retry on any error” (Candidate 2) that can amplify problems; retries only where safe, and surfaces conflicts explicitly.  
- Delivers **clear, testable acceptance criteria** and observability from day one.
