## Deployment
cf workers (edge functions for config/geo hints and opt-in encrypted telemetry aggregation; free-tier first)

## v0.1 scope
v0.1 ships a local Rust CLI agent (Linux x86_64) that: creates a hidden SOCKS5 listener (port cloaking + optional AF_UNIX), establishes a TLS-camouflaged tunnel to a public HTTPS front via QUIC or TLS-faq, sets a strict fail-closed killswitch via nftables, and records local WAU-ZB counters to encrypted SQLite; no server component required, runs client-side only, reproducible builds and basic DPI probe simulation tests included.

## Open questions
- Which pluggable transports provide best deniability vs performance on low-end client devices?
- How will we distribute and rotate fronting endpoints (CDN/cloudflare workers vs community mirrors) without central points?
- What local port-hiding technique is reliable across Linux/Windows/macOS (AF_UNIX redirect, SO_REUSEPORT tricks, hidden listeners)?
- How to implement fail-closed killswitch that survives daemon crashes and OS updates without admin rights?
- What minimal client-side telemetry schema preserves WAU-ZB while minimizing deanonymization risk?
