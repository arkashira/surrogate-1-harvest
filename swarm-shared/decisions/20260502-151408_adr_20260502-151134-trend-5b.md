## v0.1 scope
v0.1 ships a minimal deterministic orchestration loop: ingest research/signal via HTTP endpoint, run immutable workflow_run through Redis Streams, apply compliance_guardrail rules stored in Supabase Postgres, emit audit_event to Postgres, and conditionally emit execution intents to a mock broker adapter; includes replay endpoint to re-run any workflow_run deterministically from context_snapshot, full audit trail, and basic dashboard-like API endpoints to list runs and guardrail outcomes — no frontend, no paid infra, deployable via Cloudflare Workers + Supabase free tier + Redis OSS.

## Open questions
- Deterministic replay: should workflow_run.context_snapshot include full market data snapshot or rely on event sourcing from queues?
- Latency SLA: what is the acceptable end-to-end latency (ms) for sub-second orchestration on free-tier infra?
- Compliance scope: which regulatory regimes (SEC, MiFID II, exchange-specific) must guardrails enforce in v0.1?
- Idempotency model: will idempotency keys be stored in Redis or Postgres for replay safety?
- Broker/OMS integration strategy: webhook push vs polling vs FIX; which connectors are required for v0.1?
- Secrets/credentials management: where to store API keys and signing keys on free-tier infra (Workers KV vs Supabase vault)?
- Replay compute budget: how many concurrent replays can free-tier Workers/Redis sustain before throttling?
