# workio / quality

### 1. Diagnosis
* Missing **LINE webhook signature verification** (`X-Line-Signature`) allows spoofed/replayed events to be accepted as valid.
* No **idempotency / replay protection** — LINE retries on 5xx/timeouts can create duplicate clock-in/out records and leave/OT requests.
* Inadequate error handling and logging mechanisms in place for webhook handlers, making it difficult to diagnose issues.
* Potential security vulnerabilities due to lack of input validation and sanitization for user-provided data.
* Insufficient monitoring and alerting for critical system components, such as database connections and LINE API interactions.

### 2. Proposed change
The proposed change will focus on implementing **LINE webhook signature verification** and **idempotency/replay protection**. This will involve modifying the `server/src/webhook/line.js` file to include signature verification and idempotency checks.

### 3. Implementation
To implement the proposed change, follow these steps:
```javascript
// server/src/webhook/line.js
const crypto = require('crypto');
const express = require('express');
const router = express.Router();

// ... existing code ...

router.post('/webhook/line', (req, res) => {
  const signature = req.header('X-Line-Signature');
  const body = req.body;
  const channelSecret = process.env.CHANNEL_SECRET;

  // Verify signature
  const hash = crypto.createHmac('sha256', channelSecret)
    .update(JSON.stringify(body))
    .digest('hex');
  if (hash !== signature) {
    console.error('Invalid signature');
    return res.status(401).send('Invalid signature');
  }

  // Idempotency check
  const eventId = body.events[0].replyToken;
  const existingEvent = await db.query(`SELECT * FROM events WHERE id = $1`, [eventId]);
  if (existingEvent.rows.length > 0) {
    console.log('Duplicate event detected');
    return res.status(200).send('OK');
  }

  // ... existing code ...
});
```
Additionally, update the `server/src/db/schema.sql` file to include a unique index on the `events` table:
```sql
CREATE TABLE events (
  id SERIAL PRIMARY KEY,
  reply_token VARCHAR(255) UNIQUE NOT NULL
);
```
