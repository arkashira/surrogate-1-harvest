## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk check → execution stub) with immutable Postgres audit log, Redis-backed queue for replay, Pydantic-validated payloads, one broker connector mock, Prometheus latency metrics, and NSM counter for successful fee-weighted runs.

## Open questions
- Deterministic replay storage strategy: append-only event log in Postgres vs object storage for large payloads?
- Latency SLA target for end-to-end workflow (sub-100ms vs sub-500ms) and acceptable queueing semantics?
- Compliance guardrail update mechanism: hot-reload vs deploy-gated to avoid runtime drift?
- Secrets management for connector credentials on free-tier (Vault vs Supabase encrypted columns)?
- Idempotency and deduplication strategy for broker/OMS/EMS connectors under retries?
