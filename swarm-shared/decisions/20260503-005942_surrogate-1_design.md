# surrogate-1 / design

## Final Implementation Plan (≤2h)

**Goal**: Eliminate HF API rate-limit failures during ingestion while preserving 16-shard parallelism and dedup semantics.

**Core changes**:
1. **One-time file lister** (run from Mac/CI after the rate-limit window) that calls `list_repo_tree` per date folder (non-recursive) and emits `file-list/<date>.json` committed to this repo.  
   - Uses HF API **once per folder**, not per file.
   - Workers never list during the 30m ingestion window.
2. **Switch workers to CDN-only fetches** via `resolve/main/...` URLs with **no Authorization header** during streaming.  
   - Bypasses `/api/` auth-check rate limits entirely.
   - Uses CDN limits (much higher) instead of API limits.
3. **Deterministic sharding unchanged** (`slug-hash % 16`) and **dedup unchanged** (central md5 store on HF Space remains source of truth).
4. **Lightweight retry/back-off** for CDN/network errors and clear logging for 429/503 from CDN.
5. **Safety guard** in GitHub Actions to fail fast if `file-list/<date>.json` is missing (forces a re-list before ingestion runs).

**Why this is highest-value**:
- Removes the primary cause of ingestion flakiness (HF API 429) without changing parallelism or dedup behavior.
- Fits in <2h: one new script + small changes to worker + workflow guard.
- Enables Lightning training to use CDN-only fetches with zero API calls during data load.

## Code Snippets

### 1) New script: `bin/list-date-folder.sh`

```bash
#!/usr/bin/env bash
# bin/list-date-folder.sh
# List one date folder in axentx/surrogate-1-training-pairs and save to file-list/<date>.json
# Usage: HF_TOKEN=... ./bin/list-date-folder.sh 2026-05-03

set -euo pipefail
REPO="axentx/surrogate-1-training-pairs"
DATE="${1:-$(date -u +%Y-%m-%d)}"
OUT_DIR="file-list"
OUT_FILE="${OUT_DIR}/${DATE}.json"

mkdir -p "${OUT_DIR}"

echo "Listing ${REPO} folder: ${DATE}"
# list_repo_tree non-recursive to avoid paginating huge trees
# jq normalizes to relative paths
curl -sSfL \
  -H "Authorization: Bearer ${HF_TOKEN}" \
  "https://huggingface.co/api/datasets/${REPO}/tree?path=${DATE}&recursive=false" \
  | jq '[ .[] | .path ]' > "${OUT_FILE}"

echo "Saved ${DATE} file list to ${OUT_FILE}"
```

Make executable:
```bash
chmod +x bin/list-date-folder.sh
```

### 2) Update `bin/dataset-enrich.sh`

Key changes:
- Accept `FILE_LIST` env var (path to JSON produced above).
- Use CDN URLs for downloads (no auth header).
- Keep deterministic shard selection and schema projection.
- Add retry/back-off for CDN/network errors.

```bash
#!/usr/bin/env bash
# bin/dataset-enrich.sh
# Worker script for GitHub Actions matrix shard

set -euo pipefail
export SHELL=/bin/bash

# Required env
: "${HF_TOKEN:?required}"
: "${SHARD_ID:?required (0-15)}"
: "${FILE_LIST:?required (path to file-list/<date>.json)}"
: "${DATE:?required (YYYY-MM-DD)}"

REPO="axentx/surrogate-1-training-pairs"
OUTPUT_DIR="batches/public-merged/${DATE}"
TIMESTAMP=$(date -u +%H%M%S)
OUTPUT_FILE="${OUTPUT_DIR}/shard${SHARD_ID}-${TIMESTAMP}.jsonl"

mkdir -p "$(dirname "${OUTPUT_FILE}")"

# Central dedup helper (unchanged)
DEDPY="lib/dedup.py"

log() {
  echo "[$(date -u -Iseconds)] $*"
}

# Read file list
mapfile -t FILES < <(jq -r '.[]' "${FILE_LIST}")
TOTAL=${#FILES[@]}
log "Loaded ${TOTAL} files for ${DATE}"

# Filter to this shard's deterministic slice
mapfile -t MY_FILES < <(
  for f in "${FILES[@]}"; do
    # deterministic shard by slug-hash
    HASH=$(echo -n "${f}" | sha256sum | awk '{print $1}')
    BUCKET=$(( 0x${HASH:0:8} % 16 ))
    if [[ "${BUCKET}" -eq "${SHARD_ID}" ]]; then
      echo "${f}"
    fi
  done
)

log "Shard ${SHARD_ID} selected ${#MY_FILES[@]} files"

# Download via CDN (no auth header) and process
SUCCESS=0
FAIL=0
for rel_path in "${MY_FILES[@]}"; do
  CDN_URL="https://huggingface.co/datasets/${REPO}/resolve/main/${rel_path}"
  TMP=$(mktemp)

  # Retry for transient CDN/network errors
  for attempt in 1 2 3; do
    if curl -sSfL --retry 2 --retry-delay 1 -o "${TMP}" "${CDN_URL}"; then
      break
    else
      rc=$?
      log "Download attempt ${attempt} failed for ${rel_path} (rc=${rc})"
      if [[ ${attempt} -eq 3 ]]; then
        log "Giving up on ${rel_path}"
        rm -f "${TMP}"
        FAIL=$((FAIL + 1))
        continue 2
      fi
      sleep $((attempt * 2))
    fi
  done

  # Project to {prompt,response} and normalize per known schemas
  # (Keep existing projection logic; example below is minimal)
  python3 - <<PY
import sys, json, pyarrow.parquet as pq, os, hashlib
tmp = "${TMP}"
out = sys.stdout
try:
    pf = pq.ParquetFile(tmp)
    for batch in pf.iter_batches(batch_size=100):
        df = batch.to_pandas()
        # Minimal projection: expect prompt/response or similar
        if "prompt" in df.columns and "response" in df.columns:
            for _, row in df.iterrows():
                prompt = str(row["prompt"]).strip()
                response = str(row["response"]).strip()
                if prompt and response:
                    obj = {"prompt": prompt, "response": response}
                    # attach deterministic md5 for dedup
                    blob = f"{prompt}\x00{response}".encode("utf-8")
                    obj["_md5"] = hashlib.md5(blob).hexdigest()
                    out.write(json.dumps(obj, ensure_ascii=False) + "\n")
        else:
            # fallback: try to find text-like columns
            text_cols = [c for c in df.columns if df[c].dtype == "object" and df[c].str.len().max() > 20]
            if len(text_cols) >= 2:
                for _, row in df.iterrows():
                    a, b = str(row[text_cols[0]]).strip(), str(row[text_cols[1]]).strip()
                    if a and b:
                        obj = {"prompt": a, "response": b}
                        blob = f"{a}\x00{b}".encode("utf-8")
                        obj["_md5"] = hashlib.md5(blob).hexdigest()
                        out.write(json.dumps(obj, ensure_ascii=False) + "\n")
except Exception as e:
    sys.stderr.write(f"Parse error {tmp}: {e}\n")
    sys.exit(0)
PY

  # Dedup via central store (unchanged)
  python3 "${DEDPY}" --input - --output "${OUTPUT_FILE}" < <(python3 - <<'PY'
import sys, json, hashlib
for line in sys.stdin:
    line = line.strip()
    if line:
        print(line)
PY
)

  rm -f "${TMP}"
  SUCCESS=$((SUCCESS + 1))
done

log "Shard ${SHARD_ID} done. Success=${SUCCESS}, Fail=${FAIL}"

# Upload to HF dataset repo (single commit per shard per run)
if [[ -s "${OUTPUT_FILE}" ]]; then
  git config user.name "github-actions"
  git config user.email "github-actions@github.com"
