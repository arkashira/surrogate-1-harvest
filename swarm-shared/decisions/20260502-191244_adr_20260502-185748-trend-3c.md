## v0.1 scope
v0.1 ships in 2 weeks: CLI-only tool (Python) that performs local static analysis + small-code-models (Tree-sitter + a distilled HF model) on Python and JavaScript repos, outputs findings to stdout/JSON, stores per-run metadata in local SQLite, optional Supabase sync for workspace/billing (opt-in), and a minimal dashboard via Cloudflare Workers + Supabase for team workspace and Stripe payment flow (free tier + one paid tier).

## Open questions
- Which subset of languages will v0.1 support (e.g., Python/JS/Go) and which Tree-sitter grammars are required?
- Which local model(s) will be default (size vs. accuracy tradeoffs for CPU-only, free-tier users)?
- How will we handle large repos (indexing strategy, incremental analysis, and on-disk cache eviction)?
- What telemetry (if any) is acceptable for privacy-conscious users and how will opt-in/out be implemented?
- Will the CLI require authentication to Supabase for workspace/billing, or will local-only mode be the default?
