## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: Cloudflare Workers gateway exposing a single /v1/chat/completions endpoint that normalizes OpenAI and Anthropic chat APIs; provider/model CRUD in Supabase Postgres; weighted routing by latency+cost; per-provider token-bucket rate limiting and retry with jitter; short-lived KV caching for model metadata; request/response audit logging to Postgres; local dev with wrangler + supabase local; contract tests with MSW; no frontend.

## Open questions
- Which provider auth patterns must be supported at v0.1 (API key header, custom header, OAuth2 client-credentials, signed requests)?
- What is the authoritative source for model capabilities and limits (static catalog vs. dynamic discovery)?
- How should global vs. per-tenant quotas and budgets be enforced and reconciled with provider-level quotas?
- What p95 latency SLA (ms) is committed for gateway + upstream for each capability (chat/completion/embeddings)?
- Which observability signals (traces, metrics, logs) are required to meet NSM and SLO review cadence?
- How will secrets (API keys) be stored and rotated (Supabase vault, CF KV, or external secret manager)?
