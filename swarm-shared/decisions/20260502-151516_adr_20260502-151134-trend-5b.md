## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single workflow template (research→signal→risk→execution) with one broker connector (REST), pre-trade risk rule (max notional), immutable audit trail via Postgres+Workers, Redis Streams queue for replay, and OpenTelemetry traces; manual trigger via API, no UI, free-tier hosting on CF Workers + Supabase + Upstash Redis.

## Open questions
- Which broker/OMS/EMS connectors must be prioritized for v0.1 (FIX, REST, WebSocket venues)?
- What compliance rule set is non-negotiable for launch (pre-trade limit checks vs post-trade attestations)?
- What deterministic replay SLA is required (exact state reconstruction from audit vs best-effort replay)?
- What latency budget per stage (research→signal→risk→execution) is acceptable on free-tier infra?
- How will secrets/keys for broker APIs be provisioned and rotated (Workers KV vs Supabase Vault)?
