# vanguard / design

**Most Valuable Next Improvement: Persistent File-List Cache**

### Diagnosis

* No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` → HF API 429 (1000 req/5min).
* Lightning Studio lifecycle not reused → quota burn on recreation.
* Training = re-listing repos and re-checking schemas on every run.

### Proposed Change

Create a persistent file-list cache to store the list of files for each repository and update it only when necessary.

### Implementation

**Step 1: Create a file manifest cache**

Create a new file `file_manifest.json` in the project root with the following content:
```json
{
  "repo": "huggingface",
  "path": "/resolve/main/",
  "files": []
}
```
**Step 2: Update `train.py` to use the file manifest cache**

Modify the `train.py` file to load the file manifest cache and update it only when necessary:
```python
import json

# Load file manifest cache
with open('file_manifest.json', 'r') as f:
    file_manifest = json.load(f)

# Update file manifest cache
def update_file_manifest(repo, path, files):
    file_manifest['repo'] = repo
    file_manifest['path'] = path
    file_manifest['files'] = files
    with open('file_manifest.json', 'w') as f:
        json.dump(file_manifest, f)

# Load dataset using file manifest cache
def load_dataset(repo, path, files):
    # Use file manifest cache to load dataset
    if files:
        return hf_hub_download(repo, path, files)
    else:
        # If file manifest cache is empty, re-list repo and re-check schemas
        files = list_repo_tree(repo, path, recursive=False)
        update_file_manifest(repo, path, files)
        return hf_hub_download(repo, path, files)

# Train model using file manifest cache
def train_model():
    # Load dataset using file manifest cache
    dataset = load_dataset('huggingface', '/resolve/main/', file_manifest['files'])
    # Train model using dataset
    model = Trainer(model=model, args=args, train_dataset=dataset)
    model.train()
```
**Step 3: Update `lightning.py` to reuse Lightning Studio**

Modify the `lightning.py` file to reuse Lightning Studio instead of recreating it:
```python
import json

# Load file manifest cache
with open('file_manifest.json', 'r') as f:
    file_manifest = json.load(f)

# Reuse Lightning Studio
def reuse_lightning_studio():
    # Check if Lightning Studio is running
    for s in Teamspace.studios:
        if s.name == 'vanguard' and s.status == 'Running':
            return s
    # If Lightning Studio is not running, create a new one
    return Studio.create_ok(True)

# Train model using reused Lightning Studio
def train_model():
    # Reuse Lightning Studio
    studio = reuse_lightning_studio()
    # Train model using studio
    studio.run()
```
