# workio / design

## Final Synthesis — One Correct, Actionable Plan

**Core principle:** enforce “one open punch per user” at the database, make webhook processing idempotent and safe under concurrency, and add minimal, practical validation to prevent accidental double-punches.

---

### 1. Diagnosis (merged + resolved)

- **Critical missing DB guardrail**: no partial unique index to enforce at most one open punch per user (`clock_out_at IS NULL`). This is the root cause of duplicates under retries/races.
- **Race-prone app logic**: find-then-insert/update is unsafe under concurrent LINE webhook deliveries.
- **No idempotency**: LINE retries are not deduplicated; retries can create extra punches or inconsistent state.
- **Missing tenant isolation**: `clock_ins` lacks `tenant_id` FK/index, risking data leakage and poor performance in multi-tenant use.
- **Shallow validation**: no time-window debounce or explicit override path for accidental rapid taps.
- **No audit trail**: corrections to punches are not tracked, complicating reconciliation.

*Contradictions resolved in favor of correctness + concrete actionability:*
- Do **not** rely on app-only locks or optimistic checks; use DB constraints + transactional upsert.
- Do **not** add complex business rules (e.g., configurable debounce windows) in this change; add a simple, safe minimum interval guard (configurable, default 30–60s) to prevent UI/double-tap errors while preserving legitimate corrections.
- Do **not** leave idempotency optional; make it mandatory for LINE webhook writes.

---

### 2. Scope (high-value, <2h)

- `workio/server/src/db/schema.sql`: add partial unique index, tenant FK/index, idempotency table.
- `workio/server/src/services/punchService.js`: transactional, idempotent upsert with concurrency safety.
- `workio/server/src/routes/lineWebhook.js`: enforce idempotency and validation.
- Optional but recommended: lightweight audit column(s) on `clock_ins` (e.g., `updated_at`, `updated_by`) or separate audit table stub for future expansion.

---

### 3. Implementation

#### 3.1 Schema (`workio/server/src/db/schema.sql`)

```sql
-- Ensure tenant_id exists and is required
ALTER TABLE clock_ins
  ADD COLUMN IF NOT EXISTS tenant_id UUID NOT NULL DEFAULT gen_random_uuid();

-- Basic tenant/user indexes
CREATE INDEX IF NOT EXISTS idx_clock_ins_tenant_id ON clock_ins (tenant_id);
CREATE INDEX IF NOT EXISTS idx_clock_ins_user_date ON clock_ins (user_id, DATE(created_at));

-- Critical guard: at most one open punch per user (tenant-aware)
-- If you want strict tenant isolation on the constraint, include tenant_id:
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_open_punch_per_user_per_tenant
  ON clock_ins (tenant_id, user_id)
  WHERE clock_out_at IS NULL;

-- Idempotency table for webhook deduplication
CREATE TABLE IF NOT EXISTS idempotency_keys (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  key TEXT NOT NULL,
  payload_hash TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (key)
);

CREATE INDEX IF NOT EXISTS idx_idempotency_key ON idempotency_keys (key);

-- Optional: simple audit columns (non-breaking)
ALTER TABLE clock_ins
  ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS updated_by UUID;
```

#### 3.2 Service: transactional, idempotent upsert (`workio/server/src/services/punchService.js`)

```js
const { pool } = require('../db');
const crypto = require('crypto');

const MIN_INTERVAL_MS = 60 * 1000; // 60s debounce; make configurable via env if needed

function hashPayload(obj) {
  return crypto.createHash('sha256').update(JSON.stringify(obj)).digest('hex');
}

async function upsertPunchWithIdempotency({
  userId,
  tenantId,
  lineEventId,
  lineTimestamp,
  type,
  lat,
  lng,
  note,
  allowRecentOverride = false
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency check
    const idempotencyKey = `line:${lineEventId}`;
    const payloadHash = hashPayload({ userId, tenantId, lineTimestamp, type, lat, lng, note });

    const idem = await client.query(
      `SELECT payload_hash FROM idempotency_keys WHERE key = $1`,
      [idempotencyKey]
    );

    if (idem.rows.length > 0) {
      if (idem.rows[0].payload_hash !== payloadHash) {
        throw new Error('Idempotency key collision with different payload');
      }
      // Return current open punch (if any) so retries are harmless
      const existing = await client.query(
        `SELECT * FROM clock_ins WHERE tenant_id = $1 AND user_id = $2 AND clock_out_at IS NULL ORDER BY created_at DESC LIMIT 1`,
        [tenantId, userId]
      );
      await client.query('ROLLBACK');
      return existing.rows[0] || null;
    }

    // Fetch current open punch (FOR UPDATE to serialize concurrent attempts)
    const openPunch = await client.query(
      `SELECT * FROM clock_ins WHERE tenant_id = $1 AND user_id = $2 AND clock_out_at IS NULL ORDER BY created_at DESC LIMIT 1 FOR UPDATE`,
      [tenantId, userId]
    );

    const now = new Date();

    // Debounce: prevent accidental rapid clock-ins unless override
    if (!allowRecentOverride && openPunch.rows.length === 0) {
      const recent = await client.query(
        `SELECT created_at FROM clock_ins
         WHERE tenant_id = $1 AND user_id = $2
         ORDER BY created_at DESC LIMIT 1`,
        [tenantId, userId]
      );
      if (recent.rows.length > 0) {
        const last = new Date(recent.rows[0].created_at);
        if (now.getTime() - last.getTime() < MIN_INTERVAL_MS) {
          throw new Error(`Please wait ${MIN_INTERVAL_MS / 1000}s before clocking in again`);
        }
      }
    }

    let result;
    if (openPunch.rows.length === 0) {
      // Create clock-in
      const insertRes = await client.query(
        `INSERT INTO clock_ins (user_id, tenant_id, clock_in_at, lat, lng, note)
         VALUES ($1, $2, NOW(), $3, $4, $5) RETURNING *`,
        [userId, tenantId, lat, lng, note]
      );
      result = insertRes.rows[0];
    } else {
      // Close open punch
      const updateRes = await client.query(
        `UPDATE clock_ins
         SET clock_out_at = NOW(), lat_out = $1, lng_out = $2, note_out = $3, updated_at = NOW()
         WHERE id = $4 RETURNING *`,
        [lat, lng, note, openPunch.rows[0].id]
      );
      result = updateRes.rows[0];
    }

    // Record idempotency
    await client.query(
      `INSERT INTO idempotency_keys (key, payload_hash) VALUES ($1, $2)`,
      [idempotencyKey, payloadHash]
    );

    await client.query('COMMIT');
    return result;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}

module.exports = { upsertPunchWithIdempotency };
```
