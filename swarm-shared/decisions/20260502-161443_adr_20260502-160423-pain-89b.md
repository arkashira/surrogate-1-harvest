## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-platform (Linux x86_64) CLI client that: 1) creates an ephemeral local SOCKS5 listener with randomized port, 2) wraps outbound traffic in TLS to a CF Worker endpoint (acting as a MEEK-like reflector) to hide SOCKS5/TUN signatures, 3) provides a TUN interface that routes DNS and TCP through the obfuscated path, 4) implements a fail-closed kill-switch that blocks non-stealth traffic on disconnect, and 5) emits opt-in, anonymized session telemetry to Supabase Postgres for WAU-ZB tracking.

## Open questions
- Which DPI-evasion primitives to prioritize (MEEK, Shadowsocks pluggable transport, domain-fronting via CF Workers, or traffic morphing)?
- How to distribute/rotate cover endpoints without centralized infra while remaining free-tier (CF Workers + public buckets vs. peer relay)?
- What local port randomization and TUN naming strategy minimizes fingerprinting across Windows/macOS/Linux?
- How to implement reliable fail-closed kill-switch across platforms without root/admin on first run?
- What minimal client-side telemetry schema preserves privacy while enabling WAU-ZB measurement?
