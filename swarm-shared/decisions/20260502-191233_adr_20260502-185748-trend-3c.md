## Deployment
hybrid (local CLI + optional Supabase Postgres for opt-in telemetry + Cloudflare Workers for auth/queues + Hugging Face Spaces for optional hosted model inference on anonymized snippets)

## v0.1 scope
v0.1 ships in 2 weeks: CLI-only local analyzer for Python repos with AST-based structure mapping, complexity (radon) and lint (ruff) rules, SQLite local store, JSON/terminal report output, offline-first with zero telemetry; optional opt-in flag to push anonymized summary to Supabase; installable via pip; supports repos up to 50k LOC on typical dev hardware in <2 minutes.

## Open questions
- Which language analyzers ship in v0.1 (Python only or multi-language via tree-sitter)?
- What telemetry schema and retention policy balances product insight with privacy promises?
- Will users accept opt-in, anonymized snippet submission to HF Spaces for LLM-based suggestions?
- How to distribute model weights (local download vs bundled) to keep install small and offline-capable?
- What is the monetization trigger (seat-based vs usage-based) and how will free-tier limits be enforced locally?
