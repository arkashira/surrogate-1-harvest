# surrogate-1 / ops

### Highest-Value Incremental Improvement
#### Goal: Implement Pre-Flight Snapshot for Surrogate-1 Training
#### Scope: Modify `bin/dataset-enrich.sh` and Create `bin/snapshot.sh`

To eliminate the HF API rate-limit risk during training, we will implement a pre-flight snapshot generator. This involves creating a new script `bin/snapshot.sh` that lists dataset files once per date folder and emits a deterministic file manifest. We will then embed this manifest into the training process so that Lightning workers can fetch files via the CDN, bypassing the API rate limit.

### Implementation Plan

1. **Create `bin/snapshot.sh`**:
   - This script will list dataset files once per date folder using the HF CDN.
   - It will emit a JSON file containing the deterministic file manifest.

2. **Modify `bin/dataset-enrich.sh`**:
   - Before training, this script will call `bin/snapshot.sh` to generate the file manifest.
   - It will then embed the manifest into the training process.

3. **Update `ingest.yml`**:
   - Add a step to run `bin/snapshot.sh` before the training step.
   - Ensure the generated manifest is available to the training process.

### Code Snippets

#### `bin/snapshot.sh`
```bash
#!/usr/bin/env bash

# Set variables
REPO="axentx/surrogate-1-training-pairs"
DATE_FOLDER="2026-04-29"

# List dataset files using HF CDN
FILE_MANIFEST=$(curl -s "https://huggingface.co/datasets/${REPO}/resolve/main/${DATE_FOLDER}" | jq '.[] | .filename')

# Emit JSON file containing the file manifest
echo "${FILE_MANIFEST}" > "file_manifest.json"
```

#### `bin/dataset-enrich.sh`
```bash
#!/usr/bin/env bash

# Call snapshot script to generate file manifest
./bin/snapshot.sh

# Embed file manifest into training process
TRAINING_SCRIPT="train.py"
FILE_MANIFEST="file_manifest.json"

# Run training script with file manifest
python ${TRAINING_SCRIPT} --file_manifest ${FILE_MANIFEST}
```

#### `ingest.yml` (updated)
```yml
name: Ingest

on:
  workflow_dispatch:
  schedule:
    - cron: '*/30 * * * *'

jobs:
  ingest:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Generate file manifest
        run: |
          ./bin/snapshot.sh

      - name: Run dataset enrich script
        run: |
          ./bin/dataset-enrich.sh
```
