# workio / design

## 1. Diagnosis

- **No idempotency key on LINE webhook handler** — identical `X-Line-Signature` causes LINE to retry the same event; app-level `findOne → update/insert` is racy and non-atomic under concurrency.
- **Missing DB-level uniqueness constraint** — duplicate punches can be created for same user+date+session because there is no unique index to enforce one clock-in per session.
- **No transactional boundary for punch + state transition** — clock-in/out and leave/OT state changes can diverge under retries, producing inconsistent daily records.
- **No replay-safe event sourcing or dedupe log** — retries are indistinguishable from new events; no cheap way to detect and discard duplicates before side-effects.
- **Webhook handler mixes validation, business logic, and I/O** — makes idempotency and testing hard; tight coupling increases risk when adding dedupe or constraints.

## 2. Proposed change

Add atomic, idempotent punch handling at the persistence layer and webhook entrypoint:

- **File scope**: `workio/server/src/routes/line.ts` (webhook handler) and `workio/server/src/db/schema.sql` (constraints + dedupe table).
- **Behavior**: 
  - Create `punch_dedupe` table keyed by `line_event_id` (or deterministic hash of event payload) to make retries idempotent at DB level.
  - Add unique constraint on `punches(user_id, date, session_type)` to enforce one active clock-in per session.
  - Wrap punch upsert in a single atomic SQL statement (`INSERT ... ON CONFLICT DO NOTHING`) inside a transaction that also records dedupe.
  - Return early with 200 if dedupe key exists; avoid business logic on replays.

## 3. Implementation

### 3.1 DB schema changes (`workio/server/src/db/schema.sql`)

```sql
-- Idempotency log for LINE events
CREATE TABLE IF NOT EXISTS line_event_dedupe (
  id              BIGSERIAL PRIMARY KEY,
  line_event_id   TEXT NOT NULL,          -- LINE event.id
  event_type      TEXT NOT NULL,          -- message/follow/unfollow etc
  received_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uniq_line_event_id UNIQUE (line_event_id)
);

-- Ensure one active clock-in per user per session (date)
-- session_type: 'morning' | 'evening' | 'overtime' etc
CREATE TABLE IF NOT EXISTS punches (
  id              BIGSERIAL PRIMARY KEY,
  user_id         BIGINT NOT NULL REFERENCES users(id),
  tenant_id       BIGINT NOT NULL REFERENCES tenants(id),
  clock_in_at     TIMESTAMPTZ NOT NULL,
  clock_out_at    TIMESTAMPTZ,
  latitude        DOUBLE PRECISION,
  longitude       DOUBLE PRECISION,
  session_type    TEXT NOT NULL DEFAULT 'morning',
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Prevent duplicate active punches for same user+date+session
CREATE UNIQUE INDEX IF NOT EXISTS uniq_user_date_session
  ON punches (user_id, DATE(clock_in_at), session_type)
  WHERE clock_out_at IS NULL;

-- Indexes for fast lookup
CREATE INDEX IF NOT EXISTS idx_punches_user_clock_in
  ON punches (user_id, clock_in_at DESC);
```

### 3.2 Idempotent punch handler (`workio/server/src/routes/line.ts`)

```ts
import { Router } from 'express';
import { pool } from '../db';
import { verifyLineSignature } from '../utils/line';

const router = Router();

router.post('/webhook/line', verifyLineSignature, async (req, res) => {
  const events = req.body.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.sendStatus(200);
  }

  const client = await pool.connect();
  try {
    for (const ev of events) {
      // Idempotency: skip if already processed
      const dedupeExists = await client.query(
        `SELECT 1 FROM line_event_dedupe WHERE line_event_id = $1`,
        [ev.id]
      );
      if (dedupeExists.rows.length > 0) {
        continue;
      }

      // Record dedupe first (best-effort; if this fails, event is retried safely)
      await client.query(
        `INSERT INTO line_event_dedupe (line_event_id, event_type) VALUES ($1, $2)`,
        [ev.id, ev.type]
      );

      // Only handle clock-in/out messages (text or location)
      if (ev.type !== 'message') {
        continue;
      }

      const { message, source } = ev;
      if (!source?.userId) continue;

      // Fetch internal user/tenant (simplified)
      const userRow = await client.query(
        `SELECT id, tenant_id FROM users WHERE line_user_id = $1`,
        [source.userId]
      );
      if (userRow.rows.length === 0) continue;
      const user = userRow.rows[0];

      const now = new Date();
      const today = new Date(now.getFullYear(), now.getMonth(), now.getDate());

      // Determine session (simple: before noon = morning, else evening)
      const sessionType = now.getHours() < 12 ? 'morning' : 'evening';

      // Atomic upsert: insert punch if no active punch exists for this session
      // If there is an active punch (clock_out_at IS NULL), this will no-op due to unique index.
      const geo = message.type === 'location'
        ? { lat: message.latitude, lng: message.longitude }
        : { lat: null, lng: null };

      await client.query(
        `INSERT INTO punches (user_id, tenant_id, clock_in_at, session_type, latitude, longitude)
         VALUES ($1, $2, $3, $4, $5, $6)
         ON CONFLICT (user_id, date(clock_in_at), session_type)
         DO UPDATE
           SET updated_at = EXCLUDED.updated_at
         WHERE punches.clock_out_at IS NULL`,
        [user.id, user.tenant_id, now, sessionType, geo.lat, geo.lng]
      );
    }

    await client.query('COMMIT');
    res.sendStatus(200);
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error', err);
    res.sendStatus(500);
  } finally {
    client.release();
  }
});

export default router;
```

### 3.3 Signature verification middleware (`workio/server/src/utils/line.ts`)

Ensure idempotency-safe signature verification (unchanged behavior, but keep it simple):

```ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';

const CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

export function verifyLineSignature(req: Request, res: Response, next: NextFunction) {
  const signature = req.headers['x-line-signature'] as string;
  if (!signature) return res.sendStatus(400);

  const body = JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', CHANNEL_SECRET)
    .update(body)
    .digest('base64');

  if (signature !== expected) {
    return res.sendStatus(401);
  }
  next();
}
```

## 4. Verification

1. **Apply schema**  
   ```bash
   psql workio < server/src/db/schema.sql
   ```

2. **Start backend**  
   ```bash
   cd workio/server
   npm run dev
   ```

3. **Replay-safe test (manual)**  
   - Send a clock-in message from a LINE user. Confirm punch row created (`SELECT * FROM punches WHERE user_id = ...`).
   - Replay the same webhook payload (copy exact request body + `X-Line-Signature`) via curl or Postman.  
     Expected: no new punch row; `line_event_dedupe` prevents re-processing; response 200.

4. **Concurrency test (quick)**  
   - Fire two identical requests in parallel (e.g., `ab -n 2 -c 2 -p payload.json -T application/json http://localhost:3000/webhook/line`).  
     Expected: exactly one punch row; unique constraint prevents duplicates
