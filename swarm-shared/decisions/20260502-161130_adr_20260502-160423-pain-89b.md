## v0.1 scope
v0.1 ships a local Rust CLI agent (Linux x86_64) with: 1) meek-like HTTPS stream transport over Cloudflare Workers (fronted domain) to a free-tier worker reflector; 2) local SOCKS5 proxy (127.0.0.1) with port randomization and no banner leakage; 3) optional TUN-less mode (no kernel module) using userspace TCP/IP (smoltcp) to avoid TUN artifacts; 4) client-side kill-switch via local firewall rules (best-effort without root where possible) that blocks non-stealth traffic; 5) encrypted SQLite profile/session store and local WAU-ZB counters; 6) reproducible builds and basic unit/integration tests. No server infra beyond a free-tier Cloudflare Worker reflector; no telemetry exfiltration by default (opt-in local export only).

## Open questions
- Which pluggable transports provide the best DPI evasion vs. performance trade-off for our target regions (meek vs. domain-front vs. obfs4-like)?
- Can we implement a reliable client-side kill-switch across Linux/Windows/macOS without elevated privileges?
- How will we distribute/reflect real egress endpoints without centralized infra while preserving plausible deniability?
- What local telemetry schema and opt-in mechanism balances WAU-ZB measurement with privacy and deniability?
- Will WASM-based client modules be performant enough for packet-level obfuscation on low-end devices?
