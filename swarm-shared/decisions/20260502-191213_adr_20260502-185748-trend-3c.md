## Deployment
hybrid (local-first CLI + optional Supabase for opt-in analytics + CF Workers for lightweight auth/telemetry + HF space for shared model cache)

## v0.1 scope
v0.1 ships in 2 weeks: CLI-only local-first Python repo analyzer with FastAPI optional local server; supports Python files using ruff + tree-sitter + lightweight heuristics; SQLite local store; findings export (JSON/sarif); installable via pip; no cloud dependency; basic severity/category tagging and suggested fixes; performance target: initial full scan of 10k LOC repo <60s on commodity laptop.

## Open questions
- Which language analyzers (besides Python) must ship in v0.1 and can we reuse OSS linters/parsers?
- Local model size vs. performance tradeoff: do we ship quantized embeddings or rely on lighter heuristics first?
- How will licensing/compliance handle OSS analysis tools bundled in the CLI?
- What telemetry (if any) is acceptable for paid conversion without violating privacy-first promise?
- Will team/shared configs be stored locally only or optionally synced via Supabase?
- How to handle incremental analysis caching to keep local runs sub-30s on large repos?
