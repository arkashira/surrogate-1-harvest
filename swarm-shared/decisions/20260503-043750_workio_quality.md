# workio / quality

## Final Synthesis (correctness + concrete actionability)

**Chosen approach**: adopt Candidate 1 as the canonical implementation, with targeted hardening drawn from Candidate 2 (metrics/observability, stricter duplicate suppression). All contradictions resolved in favor of security, idempotency, and race-free state changes.

---

## 1. Diagnosis (merged)

- **Missing `X-Line-Signature` verification** → spoofable webhooks; enables attendance fraud.
- **No idempotency guard** → LINE retries on 5xx/timeouts create duplicate clock-in/out, leave, OT records.
- **No retry/backoff on outbound LINE notifications** → silent failures; users never see confirmations/approvals.
- **No structured logging or metrics** → cannot audit replay attacks, delivery failures, or latency.
- **No race-condition protection** → concurrent requests from same user can corrupt state (double clock-in, overlapping leave/OT).

---

## 2. Concrete changes (single source of truth)

File: `/opt/axentx/workio/server/src/routes/webhook/line.ts`  
Scope: hardened `/webhook/line` handler with:

- `X-Line-Signature` HMAC-SHA256 verification (constant-time).
- Idempotency via `X-Line-Id` (preferred) or derived event key, stored in Redis with 24h TTL.
- DB transaction + row-level lock to serialize per-user attendance state.
- Structured JSON logs + counters for observability.
- Exponential-backoff retry for outbound LINE replies/notifications.
- Graceful rejection of duplicates and invalid payloads; return 200 only after durable commit to avoid LINE retries.

function verifySignature(body: string, signature: string): boolean {
  if (!signature) return false;
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

async function acquireIdempotencyKey(key: string): Promise<boolean> {
  try {
    const set = await redis.set(`line:idempotent:${key}`, '1', { EX: IDEMPOTENCY_TTL_SECONDS, NX: true });
    return set === 'OK';
  } catch (err) {
    // Fail closed on Redis errors to avoid duplicates under outage
    console.error(JSON.stringify({ event: 'redis_idempotency_failed', error: String(err), ts: new Date().toISOString() }));
    return false;
  }
}

async function withRetry<T>(fn: () => Promise<T>, retries = 3, baseDelay = 300): Promise<T> {
  let lastErr: unknown;
  for (let i = 0; i <= retries; i++) {
    try {
      return await fn();
    } catch (err) {
      lastErr = err;
      if (i < retries) {
        const delay = baseDelay * 2 ** i;
        await new Promise((r) => setTimeout(r, delay + Math.random() * 100));
      }
    }
  }
  throw lastErr;
}

async function notifyUser(replyToken: string, text: string): Promise<void> {
  if (!replyToken) return;

  await withRetry(async () => {
    const response = await fetch('https://api.line.me/v2/bot/message/reply', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${process.env.LINE_CHANNEL_ACCESS_TOKEN}`,
      },
      body: JSON.stringify({
        replyToken,
        messages: [{ type: 'text', text }],
      }),
    });

    if (!response.ok) {
      const body = await response.text();
      throw new Error(`LINE notify failed: ${response.status} ${body}`);
    }
  });
}
