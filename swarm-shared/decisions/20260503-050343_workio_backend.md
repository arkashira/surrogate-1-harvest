# workio / backend

## 1. Diagnosis

- **Missing idempotency layer**: webhook handler accepts duplicate `webhookEventId` values across retries and client double-taps, creating duplicate clock rows and corrupting daily totals/OT/leave balances.
- **No atomic upsert for clock events**: `INSERT` without `ON CONFLICT` allows races between LINE retries and client retries to create two rows for same user+date+session.
- **No replay-window / TTL**: stale retries beyond reasonable window (e.g., 24–48h) still accepted and processed.
- **No idempotency key enforcement in write path**: backend relies on client/retry behavior instead of deterministic dedup key.
- **Missing index for fast duplicate detection**: queries to check existing clock events perform full scans under load.

## 2. Proposed change

- **File**: `workio/server/src/db/schema.sql` (add unique constraint + index)  
- **File**: `workio/server/src/routes/webhook/line.ts` (add idempotent upsert + replay-window check)  
- **Scope**: clock-in/out webhook handler only (highest-value, lowest-risk).

## 3. Implementation

### 3.1 DB schema — idempotent constraint + index

```sql
-- workio/server/src/db/schema.sql

-- Ensure one clock_event per user per date per session_type (in/out)
ALTER TABLE clock_events
  ADD CONSTRAINT uq_user_date_session
  UNIQUE (user_id, event_date, session_type);

-- Fast lookup for dedup and recent window
CREATE INDEX IF NOT EXISTS idx_clock_events_user_recent
  ON clock_events (user_id, created_at DESC)
  WHERE created_at >= (NOW() - INTERVAL '48 hours');

-- Optional: idempotency table for webhook-level dedup (covers non-clock events too)
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  event_type      TEXT NOT NULL,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload_hash    TEXT
);
```

### 3.2 Webhook handler — idempotent upsert + replay-window

```ts
// workio/server/src/routes/webhook/line.ts
import { Request, Response } from 'express';
import { pool } from '../../db';
import crypto from 'crypto';

const REPLAY_WINDOW_MS = 48 * 60 * 60 * 1000; // 48h

function hashPayload(payload: unknown): string {
  return crypto.createHash('sha256').update(JSON.stringify(payload)).digest('hex');
}

export async function handleLineWebhook(req: Request, res: Response) {
  const body = req.body;
  const events = body?.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'invalid_payload' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      const webhookEventId = ev?.source?.userId && ev?.type
        ? `${ev.source.userId}:${ev.type}:${ev.timestamp}:${ev?.message?.id || ev?.postback?.data || ''}`
        : null;

      if (!webhookEventId) continue;

      // Idempotency check at webhook level
      const idempotencyKey = `line:${webhookEventId}`;
      const exists = await client.query(
        `SELECT 1 FROM webhook_idempotency WHERE idempotency_key = $1`,
        [idempotencyKey]
      );
      if (exists.rows.length > 0) {
        continue; // already processed
      }

      // Record idempotency key
      await client.query(
        `INSERT INTO webhook_idempotency (idempotency_key, event_type, payload_hash)
         VALUES ($1, $2, $3)`,
        [idempotencyKey, ev.type, hashPayload(ev)]
      );

      // Clock event handling (only for relevant event types)
      if (ev.type === 'message' && ev.message?.type === 'text' && ev.source?.userId) {
        const userId = ev.source.userId;
        const now = new Date();
        const eventDate = now.toISOString().split('T')[0];

        // Determine session type by last event (simple toggle)
        const last = await client.query(
          `SELECT session_type, created_at FROM clock_events
           WHERE user_id = $1 AND event_date = $2
           ORDER BY created_at DESC LIMIT 1`,
          [userId, eventDate]
        );

        const nextSessionType = last.rows.length === 0 || last.rows[0].session_type === 'out' ? 'in' : 'out';

        // Atomic upsert: prevents duplicates per user+date+session_type
        await client.query(
          `INSERT INTO clock_events (user_id, event_date, session_type, clocked_at, created_at)
           VALUES ($1, $2, $3, $4, $5)
           ON CONFLICT (user_id, event_date, session_type)
           DO UPDATE SET clocked_at = EXCLUDED.clocked_at, updated_at = NOW()`,
          [userId, eventDate, nextSessionType, now, now]
        );
      }
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'processing_failed' });
  } finally {
    client.release();
  }
}
```

## 4. Verification

1. **Schema check**  
   ```bash
   psql workio -c "\d clock_events"
   # Confirm unique constraint uq_user_date_session exists
   psql workio -c "\d webhook_idempotency"
   # Confirm table and primary key exist
   ```

2. **Duplicate prevention test**  
   - Simulate same LINE event POST twice within seconds.  
   - Query: `SELECT COUNT(*) FROM clock_events WHERE user_id = '<test-user>';` → should increase by 1, not 2.  
   - Query idempotency table: `SELECT COUNT(*) FROM webhook_idempotency WHERE idempotency_key LIKE '%<test-key>%';` → should be 1.

3. **Replay-window test**  
   - Insert an old event with timestamp >48h in the past and same idempotency key into `webhook_idempotency`.  
   - Replay same webhook payload → should be ignored (idempotency hit).

4. **Race-condition test**  
   - Fire two concurrent requests with same payload (e.g., `ab -n 10 -c 5 -p payload.json`).  
   - Verify only one `clock_events` row created and no constraint violations in logs.

5. **Rollback on error**  
   - Force an exception mid-loop (e.g., invalid payload after first event).  
   - Confirm transaction rollback: no partial idempotency rows and no clock rows for that batch.
