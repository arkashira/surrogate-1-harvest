# workio / quality

## Final Synthesis — Correct, Actionable, Minimal

**Core diagnosis (accepted from both):**
- LINE retries + concurrent requests can create duplicate punches because there is no **idempotency key** and no **DB-level guard** against multiple open punches.
- The write path is non-transactional and non-atomic, so check-then-insert races produce duplicates.

**Chosen approach (favor correctness + simplicity):**
- Add a **partial unique index** to enforce “one open punch per user” at the DB level.
- Make the webhook handler **transactional** and **idempotent** using an `Idempotency-Key` header and a lightweight idempotency table.
- Keep the handler simple: upsert-style logic (close any dangling open punch on clock-in; close latest open on clock-out) inside a single transaction.
- Avoid over-engineering: no complex de-duplication windows or payload hashing beyond idempotency key + unique constraint.

### 1) DB schema changes

```sql
-- workio/server/src/db/schema.sql
-- Ensure at most one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL;

-- Idempotency table for webhook retries (lightweight)
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  idempotency_key TEXT NOT NULL,
  source          TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (idempotency_key, source)
);

CREATE INDEX IF NOT EXISTS idx_webhook_idempotency_created
ON webhook_idempotency (created_at);
```

### 2) Idempotency middleware (lightweight, reusable)

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function withIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const key = req.header('Idempotency-Key') || req.body.idempotencyKey;
  const source = 'line-webhook';
  if (!key) return next();

  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const exists = await client.query(
      'SELECT 1 FROM webhook_idempotency WHERE idempotency_key = $1 AND source = $2',
      [key, source]
    );
    if (exists.rows.length > 0) {
      await client.query('ROLLBACK');
      client.release();
      // Idempotent success response for retries
      return res.status(200).json({ ok: true, duplicate: true });
    }
    // Attach to request for commit later
    (req as any)._idempotency = { client, key, source };
    next();
  } catch (err) {
    await client.query('ROLLBACK').catch(() => {});
    client.release();
    next(err);
  }
}

export async function commitIdempotency(req: Request, res: Response, next: NextFunction) {
  const ctx = (req as any)._idempotency;
  if (!ctx) return next();
  const { client, key, source } = ctx;
  try {
    await client.query(
      'INSERT INTO webhook_idempotency (idempotency_key, source) VALUES ($1, $2)',
      [key, source]
    );
    await client.query('COMMIT');
  } catch (err) {
    await client.query('ROLLBACK').catch(() => {});
  } finally {
    client.release();
  }
  next();
}
```

### 3) Transactional webhook handler

```ts
// workio/server/src/routes/webhook/line.ts
import { Router } from 'express';
import { Pool } from 'pg';
import { withIdempotency, commitIdempotency } from '../../middleware/idempotency';

const router = Router();
const pool = new Pool({ connectionString: process.env.DATABASE_URL });

router.post('/line', withIdempotency, async (req, res) => {
  const { userId, action } = req.body; // action: 'clock_in' | 'clock_out'
  if (!userId || !action) return res.status(400).json({ error: 'missing fields' });

  const idempotencyKey = req.header('Idempotency-Key') || req.body.idempotencyKey || null;
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    if (action === 'clock_in') {
      // Close any dangling open punch for safety, then insert new one
      await client.query(
        `UPDATE punches
         SET clock_out_at = NOW(), updated_at = NOW()
         WHERE user_id = $1 AND clock_out_at IS NULL`,
        [userId]
      );
      await client.query(
        `INSERT INTO punches (user_id, clock_in_at, clock_out_at, source, idempotency_key)
         VALUES ($1, NOW(), NULL, 'line', $2)`,
        [userId, idempotencyKey]
      );
    } else if (action === 'clock_out') {
      // Close latest open punch; if none exists, create a zero-duration punch for audit
      const upd = await client.query(
        `UPDATE punches
         SET clock_out_at = NOW(), updated_at = NOW()
         WHERE user_id = $1 AND clock_out_at IS NULL
         RETURNING id`,
        [userId]
      );
      if (upd.rowCount === 0) {
        await client.query(
          `INSERT INTO punches (user_id, clock_in_at, clock_out_at, source, idempotency_key)
           VALUES ($1, NOW(), NOW(), 'line', $2)`,
          [userId, idempotencyKey]
        );
      }
    } else {
      await client.query('ROLLBACK');
      return res.status(400).json({ error: 'invalid action' });
    }

    await client.query('COMMIT');
    return res.status(200).json({ ok: true });
  } catch (err: any) {
    await client.query('ROLLBACK').catch(() => {});
    // Unique violation from idx_punches_one_open_per_user => idempotent success
    if (err.code === '23505') {
      return res.status(200).json({ ok: true, duplicate: true });
    }
    console.error('Webhook error', err);
    return res.status(500).json({ error: 'internal' });
  } finally {
    client.release();
  }
});

router.post('/line', commitIdempotency);

export default router;
```

### 4) Verification checklist (run these)

1. **Constraint test**  
   ```sql
   INSERT INTO punches (user_id, clock_in_at, clock_out_at) VALUES (1, NOW(), NULL);
   INSERT INTO punches (user_id, clock_in_at, clock_out_at) VALUES (1, NOW(), NULL);
   -- second insert must fail with unique violation
   ```

2. **Idempotency test**  
   Send same payload twice with same `Idempotency-Key` header.  
   - Expect 200 both times.  
   - Expect exactly one punch row created.

3. **Race test**  
   Fire 10 concurrent `clock_in` requests for same user.  
   - Expect exactly one open punch (`clock_out_at IS NULL`) afterward.

4. **Retry after failure**  
   Simulate first request returning 500, then retry with same idempotency key.  
   - Expect exactly one punch row and second request returns 200.

5. **Log audit**  
   Confirm `punches` rows have `source` and `idempotency_key` populated for traceability.
