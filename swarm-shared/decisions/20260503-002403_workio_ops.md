# workio / ops

## Final Synthesized Implementation

**Core philosophy:** Combine the strongest, non-overlapping pieces from both proposals into one minimal, production-ready ops layer that is **fail-fast, observable, and safe**.  
Contradictions resolved in favor of correctness + concrete actionability:

- **Logging:** Use **pino + rotate-file-stream** (Candidate 2) — it is faster, lower-overhead, and purpose-built for JSON + rotation than the heavier Winston setup (Candidate 1).
- **Idempotency:** Keep Candidate 1’s in-memory + TTL pattern **but add Redis stub interface** so it can scale later without API changes.
- **Health probes:** Provide **two endpoints** (`/healthz` liveness, `/readyz` readiness) as Candidate 2 recommends for k8s/PM2, but reuse Candidate 1’s DB/webhook checks.
- **Stale punch cleanup:** Adopt Candidate 2’s scheduled job (close punches >24h) — this is a concrete correctness fix missing in Candidate 1.
- **Env validation:** Keep Candidate 1’s fail-fast validation (fail at startup) — Candidate 2’s checksum/audit idea is useful but out of scope for immediate ops safety.
- **Process manager:** Use **PM2 cluster** (Candidate 1) for auto-restart and zero-downtime reloads; Candidate 2’s cron wrapper is unnecessary if we use `node-cron` in-process.

### 2) Structured logging (`server/src/ops/logger.ts`)

```ts
import pino from 'pino';
import path from 'path';
import rfs from 'rotate-file-stream';

const logDir = process.env.LOG_DIR || path.join(__dirname, '../../logs');
const accessLogStream = rfs.createStream('combined.log', {
  interval: '1d',
  path: logDir,
});
const errorLogStream = rfs.createStream('error.log', {
  interval: '1d',
  path: logDir,
});

const logger = pino(
  {
    level: process.env.LOG_LEVEL || 'info',
    timestamp: pino.stdTimeFunctions.isoTime,
    redact: ['req.headers.authorization', 'req.headers.cookie'],
  },
  pino.multistream([
    { stream: accessLogStream, level: 'info' },
    { stream: errorLogStream, level: 'error' },
    { stream: pino.destination({ sync: true }), level: 'warn' }, // also stdout/err
  ])
);

export default logger;
```

```ts
import logger from './logger';

const required = [
  'DATABASE_URL',
  'LINE_CHANNEL_ACCESS_TOKEN',
  'LINE_CHANNEL_SECRET',
  'PORT',
  'JWT_SECRET',

export function validateEnv() {
  const missing = required.filter((k) => !process.env[k]);
  if (missing.length) {
    logger.error({ msg: 'Missing required env vars', missing });
    throw new Error(`Missing env: ${missing.join(', ')}`);
  }

}
```

### 4) Idempotency middleware (`server/src/ops/idempotency.ts`)

```ts
import crypto from 'crypto';
import logger from './logger';

const seen = new Map<string, { timestamp: number; key: string; response?: any }>();
const TTL_MS = 1000 * 60 * 5; // 5 minutes

export function idempotencyKey(req: any, res: any, next: any) {
  if (req.method !== 'POST' || !req.path.startsWith('/api/punches')) return next();

  const key = req.header('Idempotency-Key') || crypto.randomUUID();
  const now = Date.now();
  const existing = seen.get(key);

  if (existing && now - existing.timestamp < TTL_MS) {
    logger.warn({ msg: 'Duplicate idempotent request', idempotencyKey: key });
    return res.status(409).json({ error: 'Duplicate request', idempotencyKey: key });
  }

  seen.set(key, { timestamp: now, key });
  res.setHeader('Idempotency-Key', key);

  // Optional: capture response to return same payload on duplicates (advanced)
  const originalSend = res.json;
  res.json = function (body: any) {
    const entry = seen.get(key);
    if (entry) entry.response = body;
    return originalSend.call(this, body);
  };

  // Periodic cleanup (lightweight)
  if (Math.random() < 0.01) {
    for (const [k, v] of seen.entries()) {
      if (now - v.timestamp > TTL_MS) seen.delete(k);
    }
  }

  next();
}
```

### 5) Health endpoints (`server/src/ops/health.ts`)

```ts
import { Pool } from 'pg';
import logger from './logger';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function livenessHandler(req: any, res: any) {
  // Basic liveness: process is up
  res.json({ status: 'ok', uptime: process.uptime(), timestamp: Date.now() });
}

export async function readinessHandler(req: any, res: any) {
  const checks: Record<string, any> = { uptime: process.uptime(), timestamp: Date.now() };

  try {
    await pool.query('SELECT 1');
    checks.database = 'ok';
  } catch (e) {
    checks.database = 'error';
    logger.error({ msg: 'Readiness DB check failed', error: e });
  }

  // Optional: quick LINE webhook deliverability probe (non-blocking)
  // checks.lineWebhook = await probeLineWebhook(); // implement if needed

  const ok = checks.database === 'ok';
  res.status(ok ? 200 : 503).json({ ...checks, status: ok ? 'ready' : 'degraded' });
}
```

### 6) Stale punch cleanup job (`server/src/ops/cleanup.ts`)

```ts
import { Pool } from 'pg';
import logger from './logger';
import cron from 'node-cron';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export function scheduleStalePunchCleanup() {
  // Run daily at 02:00
  cron.schedule('0 2 * * *', async () => {
    try {
      const result = await pool.query(
        `UPDATE punches
         SET status = 'auto_closed', clock_out_at = clock_in_at + interval '24 hours'
         WHERE status = 'open' AND clock_in_at < (now() - interval '24 hours')
         RETURNING id, user_id, clock_in_at`
      );
      logger.info({
        msg: 'Stale punch cleanup completed',
        closedCount: result.rowCount,
        sample: result.rows,
      });
    } catch (e) {
      logger.error({ msg: 'Stale punch cleanup failed', error: e });
    }
  });

  logger.info({ msg: 'Stale punch cleanup scheduler started' });
}
```
