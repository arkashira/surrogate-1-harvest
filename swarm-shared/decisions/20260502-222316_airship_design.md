# airship / design

## Final Implementation — `airship discover` (Zero-config, Cron-safe, Actionable)

I merged the strongest, non-contradictory parts of both proposals and resolved conflicts in favor of **correctness, portability, and immediate actionability**.

### Resolved conflicts (favoring correctness + actionability)
- **Reports directory**: Candidate 1 used `reports/`; Candidate 2 used `discoveries/YYYY-MM-DD/`.  
  **Resolution**: use `discoveries/YYYY-MM-DD/` for chronological isolation (better for cron + audits), but also emit a stable `latest -> YYYY-MM-DD` symlink for quick access.
- **Granite + knowledge-rag availability**: Candidate 1 silently continues; Candidate 2 fails fast.  
  **Resolution**: continue gracefully (non-blocking) but exit with a clear, documented failure code if the primary goal (top-hub insight) cannot be produced. This keeps cron safe while surfacing real problems.
- **HF CDN bypass vs generic helpers**: Candidate 2 introduced HF-specific flags; Candidate 1 stayed generic.  
  **Resolution**: keep CLI generic by default; add optional `--hf-*` flags implemented as environment-friendly curl-to-HF-CDN helpers (no API token required, avoids rate limits).
- **Lightning studio handling**: Candidate 2 included idle-stop/launcher generation; Candidate 1 omitted.  
  **Resolution**: add an opt-in, non-blocking Lightning training-prep artifact (file-list + launcher stub). Do not manage studio lifecycle automatically (too risky); instead, document how to reuse a running studio and check idle-stop manually.
- **Single script vs modular lib**: Candidate 1 split helpers into `lib/`; Candidate 2 used one file.  
  **Resolution**: keep a single executable `bin/airship` for portability and cron simplicity, but allow an external `lib/airship-tools.sh` if present (source if available; continue if not). This balances reuse with zero-config deployment.

### 2) `/opt/axentx/airship/bin/airship` (final CLI)

```bash
#!/usr/bin/env bash
# airship — unified CLI for discovery and training prep
# Usage: airship <command> [args...]
# Commands:
#   discover  Run business research + top-hub insight + training prep
#   help      Show help
#
# Cron-safe:
#   - Use SHELL=/bin/bash in crontab
#   - This script uses env bash, set -euo pipefail, absolute paths

set -euo pipefail

AXENTX_ROOT="/opt/axentx/airship"
LOG_DIR="${AXENTX_ROOT}/logs"
DISCOVERIES_DIR="${AXENTX_ROOT}/discoveries"
TIMESTAMP=$(date -u +%Y-%m-%dT%H%M%SZ)
DATE_TAG=$(date -u +%Y-%m-%d)
REPORT_DIR="${DISCOVERIES_DIR}/${DATE_TAG}"

# Optional external helpers (zero-config if absent)
TOOLS_LIB="${AXENTX_ROOT}/lib/airship-tools.sh"

# Ensure dirs
mkdir -p "${LOG_DIR}" "${REPORT_DIR}"

log() {
  echo "[$(date -u +%Y-%m-%dT%H:%M:%SZ)] $*" | tee -a "${LOG_DIR}/airship-${DATE_TAG}.log"
}

fail() {
  log "ERROR: $*"
  exit 1
}

# ---- Optional helpers ----
source_tools_if_present() {
  if [[ -f "${TOOLS_LIB}" ]]; then
    # shellcheck source=/dev/null
    source "${TOOLS_LIB}"
    return 0
  fi
  return 1
}

# ---- HF CDN helpers (optional) ----
hf_cdn_download() {
  local repo="$1"
  local folder="${2:-}"
  local out_dir="${3:-${REPORT_DIR}}"
  local base="https://huggingface.co"
  local url="${base}/${repo}/tree/main"
  [[ -n "${folder}" ]] && url="${url}/${folder}"

  log "info" "HF CDN listing: ${url}"

  # Use curl + lynx/elinks or parse HTML; fallback to wget spider for file list.
  # This is best-effort and non-blocking.
  if command -v curl >/dev/null 2>&1; then
    curl -sSL --retry 3 --max-time 30 "${url}" \
      -H "Accept:text/html" \
      -o "${out_dir}/hf-listing.html" 2>/dev/null || true
  fi
}

# ---- Subcommands ----

discover_usage() {
  cat <<EOF
Usage: airship discover [options]

Options:
  --hf-repo <repo>        HuggingFace dataset repo (e.g. axentx/surrogate-mirror)
  --hf-folder <folder>    Folder within repo (e.g. datasets/v1)
  --out-dir <dir>         Override report directory (default: ${REPORT_DIR})

Description:
  Runs business research (if available), knowledge-rag top-hub analysis,
  and prepares training artifacts. Safe for cron.
EOF
}

run_granite_business_research() {
  local out_dir="${1}"
  local script="${AXENTX_ROOT}/scripts/granite-business-research.sh"

  if [[ -x "${script}" ]]; then
    log "Running granite-business-research.sh"
    "${script}" --out-dir "${out_dir}" || {
      log "granite-business-research.sh failed; continuing."
      return 1
    }
    return 0
  fi

  log "granite-business-research.sh not found; skipping business research."
  return 1
}

run_knowledge_rag_top_hub() {
  local work_dir="${1}"
  local rag_cmd
  rag_cmd="$(command -v knowledge-rag 2>/dev/null || echo "${AXENTX_ROOT}/scripts/knowledge-rag")"

  if [[ -x "${rag_cmd}" ]]; then
    log "Running knowledge-rag top-hub analysis"
    "${rag_cmd}" top-hub --work-dir "${work_dir}" --format json 2>/dev/null || {
      log "knowledge-rag non-zero; producing fallback."
      echo '{"top_hub":null,"edges":[],"docs":[]}'
      return 0
    }
    return 0
  fi

  log "knowledge-rag not found; producing placeholder."
  echo '{"top_hub":null,"edges":[],"docs":[]}'
}

extract_top_hub_insight() {
  local json="$1"
  local hub edge_count doc_count
  hub=$(echo "${json}" | python3 -c 'import sys,json; d=json.load(sys.stdin); print(d.get("top_hub") or "N/A")' 2>/dev/null || echo "N/A")
  edge_count=$(echo "${json}" | python3 -c 'import sys,json; d=json.load(sys.stdin); print(len(d.get("edges", [])))' 2>/dev/null || echo 0)
  doc_count=$(echo "${json}" | python3 -c 'import sys,json; d=json.load(sys.stdin); print(len(d.get("docs", [])))' 2>/dev/null || echo 0)

  cat <<EOF
## Top Hub Insight
- **Hub**: ${hub}
- **Edges**: ${edge_count}
- **Related docs**: ${doc_count}

> Generated by airship discover on ${TIMESTAMP}
EOF
}
