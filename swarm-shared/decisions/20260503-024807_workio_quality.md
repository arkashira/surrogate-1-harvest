# workio / quality

## Final Synthesis (Best Parts + Correctness + Actionability)

### 1. Root Causes (merged, corrected)
- **Missing idempotency enforcement** on the punch/webhook handler → LINE retries (slow 2xx/timeouts) create duplicate punches.
- **Race condition** between “find active punch” and “insert/close” under concurrent requests → double clock-in/out for same user.
- **No client-side idempotency + optimistic UI** → double-tap risk and poor perceived performance on mobile.
- **No server-side deduplication window** and **no DB uniqueness guardrails** → retries bypass application checks.

### 2. Design Decisions (resolve contradictions)
- **Idempotency key source**: prefer client-generated `Idempotency-Key` header (UUID) for punch UI; for LINE webhooks, derive from `userId + event.timestamp + event.type` (or use a short TTL + signature) and reject replays outside tolerance.
- **Concurrency control**: use **row-level locking** (`SELECT ... FOR UPDATE` on user or punch row) inside a transaction instead of advisory locks for portability and clarity. Combine with a unique constraint on `(user_id, clock_out_time)` partial index for active punch to enforce single active punch at DB level.
- **Idempotency store**: single `punch_idempotency_keys` table keyed by the client/server idempotency key, referencing punch id. Enforce uniqueness at DB level.
- **Schema guardrails**: add partial unique index for active punch and FK cascade; add created_at index for TTL cleanup of idempotency keys.
- **Frontend**: optimistic UI + disable while pending + attach idempotency key; reconcile on response; retry only on network errors (not 4xx/5xx) with same key.

### 3. Implementation (concrete, minimal, correct)

#### Backend: idempotent punch handler (Node/Express + PostgreSQL)
```ts
// workio/server/src/routes/punch.ts
import { Router, Request, Response } from 'express';
import { db } from '../db';

const router = Router();

// Helper: run fn in transaction with row lock on user
async function txWithUserLock<T>(userId: number, fn: (client: any) => Promise<T>) {
  const client = await db.connect();
  try {
    await client.query('BEGIN');
    // Lock user row to serialize punch operations per user
    await client.query('SELECT id FROM users WHERE id = $1 FOR UPDATE', [userId]);
    const result = await fn(client);
    await client.query('COMMIT');
    return result;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}

router.post('/punch', async (req: Request, res: Response) => {
  const { userId } = req.body;
  const idempotencyKey = (req.headers['idempotency-key'] as string)?.trim();

  if (!userId) return res.status(400).json({ error: 'userId required' });
  if (!idempotencyKey) return res.status(400).json({ error: 'Idempotency-Key header required' });

  try {
    const result = await txWithUserLock(userId, async (client) => {
      // 1) Idempotency check
      const idem = await client.oneOrNone(
        'SELECT punch_id FROM punch_idempotency_keys WHERE idempotency_key = $1',
        [idempotencyKey]
      );
      if (idem) {
        const punch = await client.one('SELECT * FROM punches WHERE id = $1', [idem.punch_id]);
        return { punch, created: false };
      }

      // 2) Find active punch (clock_out_time IS NULL)
      const active = await client.oneOrNone(
        'SELECT * FROM punches WHERE user_id = $1 AND clock_out_time IS NULL ORDER BY clock_in_time DESC LIMIT 1 FOR UPDATE',
        [userId]
      );

      let punch;
      if (active) {
        // clock out
        punch = await client.one(
          'UPDATE punches SET clock_out_time = NOW(), updated_at = NOW() WHERE id = $1 RETURNING *',
          [active.id]
        );
      } else {
        // clock in
        punch = await client.one(
          'INSERT INTO punches(user_id, clock_in_time, created_at, updated_at) VALUES($1, NOW(), NOW(), NOW()) RETURNING *',
          [userId]
        );
      }

      // 3) Record idempotency key
      await client.none(
        'INSERT INTO punch_idempotency_keys(idempotency_key, punch_id, created_at) VALUES($1, $2, NOW())',
        [idempotencyKey, punch.id]
      );

      return { punch, created: !active };
    });

    res.json({ ok: true, punch: result.punch, created: result.created });
  } catch (err: any) {
    // Unique violation on idempotency key or partial unique index should be handled gracefully
    if (err.code === '23505') {
      // Race: another request inserted same key or active punch — refetch
      const idem = await db.oneOrNone(
        'SELECT punch_id FROM punch_idempotency_keys WHERE idempotency_key = $1',
        [idempotencyKey]
      );
      if (idem) {
        const punch = await db.one('SELECT * FROM punches WHERE id = $1', [idem.punch_id]);
        return res.json({ ok: true, punch, created: false });
      }
    }
    console.error('Punch error:', err);
    res.status(500).json({ error: 'Internal server error' });
  }
});

export default router;
```

#### Database schema additions (SQL)
```sql
-- workio/server/src/db/schema.sql

-- Partial unique index: at most one active punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_one_active_punch_per_user
ON punches (user_id)
WHERE clock_out_time IS NULL;

-- Idempotency keys for punch deduplication
CREATE TABLE IF NOT EXISTS punch_idempotency_keys (
  idempotency_key TEXT PRIMARY KEY,
  punch_id INTEGER NOT NULL REFERENCES punches(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Index for TTL cleanup of old idempotency keys (e.g., delete > 48h)
CREATE INDEX IF NOT EXISTS idx_punch_idempotency_keys_created_at ON punch_idempotency_keys(created_at);
```
