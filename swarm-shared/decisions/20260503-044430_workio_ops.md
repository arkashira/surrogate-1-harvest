# workio / ops

## Final Synthesized Solution

I merged the strongest, non-contradictory insights from both proposals into one coherent, production-ready plan. Where they overlapped (e.g., idempotency, health checks, logging), I kept the most complete and actionable implementation. Where they diverged (process supervision, graceful shutdown, deployment automation), I added concrete, minimal changes that make the system resilient without overcomplicating it.

---

## 1. Diagnosis (Consolidated)

- **LINE webhook retry amplification**: webhook handler can return 5xx/timeouts on downstream hiccups (DB, Redis, external APIs). LINE retries with exponential backoff, but without server-side idempotency this causes duplicate punches and phantom attendance.
- **Missing idempotency store**: no Redis-backed (or persistent) idempotency key with replay window → duplicate `X-Line-Signature` events can be applied twice.
- **No graceful degradation**: if Redis/DB is unavailable, the webhook fails fast instead of enqueuing or deferring → lost attendance records.
- **No observability**: missing structured logs + metrics (requests, retries, idempotency hits, latency) → hard to detect duplicates or replay storms.
- **No operational runbook / supervision**: no health-check endpoints, readiness/liveness probes, or process supervision for containers/K8s/PM2/systemd → silent process death or unhealthy state goes unnoticed.
- **No graceful shutdown**: in-flight webhook handlers can be killed mid-processing during deploys/restarts, risking partial state (clock-in recorded, notification not sent).
- **No deploy/rollback automation**: manual deploys invite drift and human error; no artifact versioning or smoke tests before traffic.

---

## 2. Proposed Changes (File Scope)

- `workio/server/src/routes/line/webhook.ts` — hardened webhook handler with idempotency, graceful degradation, and fast 200 response.
- `workio/server/src/lib/idempotency.ts` — Redis-first idempotency store with memory fallback and TTL.
- `workio/server/src/lib/logger.ts` — structured logging (pino) + lightweight in-process metrics.
- `workio/server/src/routes/health.ts` — `/health/live` and `/health/ready` endpoints.
- `workio/server/src/index.ts` (or `app.ts`) — wire routes, add graceful shutdown, and request draining.
- `workio/serve/pm2.config.cjs` — production process supervision (PM2) with restart, log rotation, and graceful shutdown support.
- `workio/deploy/` — minimal deploy/rollback automation (GitHub Actions or simple script) with health-check smoke test and version tagging.

## 3. Implementation

### 3.1 Idempotency Store (Redis + Memory Fallback)

```ts
// workio/server/src/lib/idempotency.ts
import Redis from 'ioredis';

const redis = process.env.REDIS_URL ? new Redis(process.env.REDIS_URL) : null;
const MEMORY_TTL_MS = 5 * 60 * 1000; // 5m replay window
const memoryStore = new Map<string, { ts: number }>();

export async function acquireIdempotencyKey(
  key: string,
  ttlMs = MEMORY_TTL_MS
): Promise<boolean> {
  if (redis) {
    try {
      const set = await redis.set(key, '1', 'PX', ttlMs, 'NX');
      return set === 'OK';
    } catch {
      // fall through to memory
    }
  }

  // memory fallback
  const existing = memoryStore.get(key);
  if (existing) {
    if (Date.now() - existing.ts < ttlMs) return false;
  }
  memoryStore.set(key, { ts: Date.now() });
  // best-effort cleanup
  if (memoryStore.size > 5000) {
    for (const [k, v] of memoryStore.entries()) {
      if (Date.now() - v.ts > ttlMs) memoryStore.delete(k);
    }
  }
  return true;
}
```

### 3.3 Hardened Webhook Handler

```ts
// workio/server/src/routes/line/webhook.ts
import express from 'express';
import crypto from 'crypto';
import { logger, metrics } from '../../lib/logger';
import { acquireIdempotencyKey } from '../../lib/idempotency';
import { handleLineEvent } from '../../services/lineEvents';

const router = express.Router();

function verifySignature(rawBody: Buffer, signature: string): boolean {
  const channelSecret = process.env.LINE_CHANNEL_SECRET;
  if (!channelSecret) return false;
  const hash = crypto
    .createHmac('sha256', channelSecret)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(hash));
}

router.post(
  '/webhook/line',
  express.raw({ type: 'application/json' }),
  async (req, res) => {
    const sig = req.header('X-Line-Signature');
    metrics.webhook_total++;

    if (!sig || !verifySignature(req.body, sig)) {
      logger.warn({ event: 'line_webhook_invalid_signature' });
      return res.status(401).send('Invalid signature');
    }

    let events;
    try {
      events = JSON.parse(req.body.toString()).events;
      if (!Array.isArray(events)) throw new Error('Invalid events');
    } catch (err) {
      logger.error({ err, event: 'line_webhook_parse_error' });
      return res.status(400).send('Invalid payload');
    }

    // Idempotency key: per-batch, stable across retries
    const idempotencyKey = `webhook:${crypto
      .createHash('sha256')
      .update(req.body)
      .digest('hex')}`;

    const acquired = await acquireIdempotencyKey(idempotencyKey);
    if (!acquired) {
      metrics.webhook_idempotent_reject++;
      logger.info({ idempotencyKey, event: 'line_webhook_idempotent_reject' });
      return res.status(200).send('Already processed');
    }

    // Best-effort async processing; do not block response
    (async () => {
      try {
        for (const ev of events) {
          await handleLineEvent(ev);
        }
        metrics.webhook_processed++;
        logger.info({ count: events.length, event: 'line_webhook_processed' });
      } catch (err) {
        metrics.webhook_error++;
        logger.error({ err, event: 'line_webhook_process_error' });
        // Alerting hook could go here (e.g., send to Sentry/ops channel)
      }
    })();

    // Respond quickly so LINE stops retrying
    res.status(200).send('OK');
  }
);

export default router;
```
