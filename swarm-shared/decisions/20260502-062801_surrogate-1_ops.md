## **Surrogate-1 Ops Improvement**

### **Diagnosis**
* The project lacks a robust implementation for handling Hugging Face API rate limits, which can block dataset training.
* The existing implementation may not be reusing existing Lightning Studio instances efficiently, leading to wasted quota and potential downtime.
* The project does not have a strategy for bypassing HF API rate limits during dataset training.

### **Proposed Change**
* Implement HF CDN bypass for dataset training

### **Implementation**
We will implement the HF CDN bypass for dataset training by modifying the `train.py` script to download files from the HF CDN instead of using the HF API. This will be achieved by using the `hf_hub_download` function to download files from the CDN, and then processing them as usual.

```diff

# After
```
