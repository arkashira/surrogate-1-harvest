## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: cross-platform CLI client (Linux/macOS/Windows) with client-side obfuscated transport (shadowsocks-like stream cipher over TLS to a CF Worker reflector), hidden local SOCKS5 listener (port randomization, bind-to-loopback-only mode, optional socket cloaking), best-effort rootless TUN setup, encrypted SQLite fail-closed kill-switch that blocks non-VPN interfaces on session start, and minimal client-side telemetry (WAU-ZB signal only) — all free-tier deployable with no server infra beyond CF Workers.

## Open questions
- Which obfuscation primitives satisfy DPI evasion in target regions while remaining implementable client-side only (e.g., shadowsocks-like stream cipher vs. meek vs. domain-fronted TLS)?
- How will we obtain and rotate innocuous fronting endpoints without requiring users to control servers (free-tier CF Workers as reflectors/proxies)?
- Can we implement a reliable rootless TUN/kill-switch on major desktop OSes without admin/root while still preventing real traffic leaks?
- What telemetry schema and consent flow balances WAU-ZB measurement with user privacy and deniability?
- How do we defend against active probing of local SOCKS5/TUN artifacts while keeping the client zero-config?
