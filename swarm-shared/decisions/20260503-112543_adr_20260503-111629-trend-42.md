## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: a single CF Worker exposing POST /v1/chat/completions that normalizes to OpenAI-format, routes across OpenAI and Anthropic via lowest-latency strategy, enforces per-provider token/request quotas in Redis, records audit to Supabase Postgres, exposes Prometheus metrics, and includes basic idempotency and retry with exponential backoff; no frontend, no user auth (key-based), no advanced PII filter.

## Open questions
- Should per-provider auth secrets live in CF Workers KV or Supabase Vault (or both) to balance latency vs. RBAC?
- What is the canonical error/retry contract for upstream provider 429/5xx to avoid thundering-herd across tenants?
- How will token counting be normalized across providers (prompt/completion vs. input/output tokens) for accurate cost and quota?
- Do we enforce global rate-limits at the edge (CF Workers) or in the origin service, and how do we synchronize Redis across regions?
- Which PII classification taxonomy and redaction level is required for compliance (PII, PHI, PCI) given OSS-only tooling?
