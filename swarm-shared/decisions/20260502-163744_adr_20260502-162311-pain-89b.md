## Deployment
cf workers + Supabase free tier + HuggingFace Spaces (relay health dashboard)

## v0.1 scope
v0.1 in 2 weeks: Android client (Kotlin + minimal Rust via JNI) that establishes a concealed VPN-like tunnel using VpnService with split-tunnel and TUN minimization, connects to a CF Worker-backed relay directory (Supabase) to select an obfuscated relay (shadowsocks/obfs4 fronted on 443), hides local SOCKS5 ports from other apps via loopback-only binding and ephemeral port randomization, records local session state in SQLite, and exposes a basic no-telemetry WAUS/U counter — all deployable on free tiers (CF Workers + Supabase) with no user account required for initial flows.

## Open questions
- Which obfuscation primitives provide the best tradeoff against DPI in target jurisdictions while remaining implementable on stock Android without root?
- How will we hide TUN interfaces and SOCKS5 ports from local apps and system scans without requiring Knox/MDM (e.g., via VpnService.Builder or tun fd cloaking techniques)?
- What is the acceptable latency budget for fronted/meek/obfs4 flows and how do we measure WAUS/U reliably without creating a fingerprintable telemetry channel?
- Which relay fronting strategy (domain fronting, port reuse, TLS-in-TLS, or QUIC mimicry) is viable given free-tier hosting constraints and CF Workers capabilities?
- How will key rotation and deniability be handled to minimize user attribution if devices are inspected?
