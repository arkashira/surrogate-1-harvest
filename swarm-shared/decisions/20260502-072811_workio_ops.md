#### Step 2: Pre-List File Paths and Embed in Training Script
Make a single API call to `list_repo_tree` to get the file paths, save to JSON, and embed in `train.py`.
```python
import json
import requests

# Define the dataset repository and path
repo_id = "dataset/repo"
path = "path/to/files"

# Make a single API call to get the file paths
response = requests.get(f"https://huggingface.co/datasets/{repo_id}/tree/main/{path}")
file_paths = response.json()

# Save the file paths to JSON
with open("file_paths.json", "w") as f:
    json.dump(file_paths, f)

# Embed the file paths in train.py
with open("file_paths.json", "r") as f:
    file_paths = json.load(f)

# Use the file paths to download the files using the HF CDN
for file_path in file_paths:
    cdn_url = f"https://huggingface.co/datasets/{repo_id}/resolve/main/{file_path}"
    response = requests.get(cdn_url)
    with open(file_path, "wb") as f:
        f.write(response.content)
```

#### Step 3: Implement Studio Reuse and Idle Stop Handling
Reuse existing Lightning Studios and handle idle stop timeouts to prevent training process deaths.
```python
import lightning

# Define the studio name and machine type
studio_name = "my-studio"
machine_type = lightning.Machine.L40S

# Reuse an existing studio if it exists
for studio in lightning.Teamspace.studios:
    if studio.name == studio_name and studio.status == "Running":
        studio = studio
        break
else:
    # Create a new studio if it doesn't exist
    studio = lightning.Studio.create(studio_name, machine_type)

# Handle idle stop timeouts
if studio.status == "Stopped":
    studio.start(machine=machine_type)
```
By implementing these changes, we can improve the Surrogate-1 training pipeline and reduce the likelihood of API rate limits and training process deaths.
