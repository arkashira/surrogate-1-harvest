# airship / frontend

# Final Implementation Plan — Airship Frontend Status Dashboard (<2h)

## Chosen scope (highest value, minimal surface)
Add a **static-first, cron-safe status dashboard** that surfaces:
- Running state of Arkship (ports 3000/8000) and Surrogate AI (port 8001)
- Last `airship discover` run metadata (timestamp, manifest path, HF file count)
- CDN training file list (pre-listed JSON; zero HF API calls at runtime)
- Top-hub insight link (knowledge-rag) when available

**Key decisions in favor of correctness + actionability**
- Use a **shared output directory** (`/opt/axentx/airship/output/`) for artifacts so both backend scripts and frontend can read/write without a backend service.
- Keep the frontend **static-first** (no polling, no auth, cache-friendly) and mount the output dir into Arkship so the UI can read results directly.
- Update `airship discover` to write `insights.json` and `manifest.json` (and `last-discover.json`) so the dashboard is always renderable.
- Provide both a **standalone status page** (fast to ship) and an **Arkship Discover route** (better UX) — implement the Arkship route as primary, keep the static page as fallback/ops view.

### 1. Backend: airship-discover output writer (pseudo-shell + JSON)

`/opt/axentx/airship/bin/airship-discover`
```bash
#!/usr/bin/env bash
set -euo pipefail

OUTDIR="/opt/axentx/airship/output"
mkdir -p "$OUTDIR"

# ... existing discovery logic ...
# Assume $MANIFEST_PATH, $HF_FILE_COUNT, $CDN_BASE, $FILES_LIST are produced

TS=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
MANIFEST_NAME=$(basename "$MANIFEST_PATH")

# last-discover.json
cat > "$OUTDIR/last-discover.json" <<EOF
{
  "timestamp": "$TS",
  "manifest_path": "$MANIFEST_PATH",
  "manifest_name": "$MANIFEST_NAME",
  "hf_file_count": $HF_FILE_COUNT,
  "status": "completed"
}
EOF

# manifest.json (CDN training file list)
# FILES_JSON should be a JSON array of file entries produced by discovery
# Example entry: {"path":"...", "size":12345, "sha256":"..."}
cat > "$OUTDIR/manifest.json" <<EOF
{
  "date": "$TS",
  "repo": "training-data",
  "shard_count": 1,
  "cdn_base": "$CDN_BASE",
  "files": $FILES_JSON
}
EOF

# insights.json (top-hub + research insights)
# Produce minimal insights from knowledge-rag or placeholder
cat > "$OUTDIR/insights.json" <<EOF
{
  "top_hub": {
    "id": "hub-001",
    "title": "Top training patterns",
    "summary": "Most frequent domains and quality signals.",
    "score": 0.92,
    "ts": "$TS"
  },
  "insights": []
}
EOF

echo "Outputs written to $OUTDIR"
```
