# Costinel / frontend

## Final Implementation Plan — Costinel Frontend: Top-Hub Signal Card (Read-Only)

**Scope**: Read-only frontend card (≤2h)  
**Principle**: “Sense + Signal — ไม่ Execute” (strictly no writes, no self-execution, no runtime API calls)  
**Goal**: Surface the most-connected hub (e.g., “MOC”) with score, rationale, provenance, and audit trail; CDN-friendly; zero mutations; fully accessible.

---

### Architecture (read-only, CDN-friendly)
- **Data source**: Static JSON committed to `public/data/` (generated by knowledge-rag pipeline).
  - `public/data/knowledge/top-hub.json` — canonical signal payload.
  - `public/data/hub-files.json` — lightweight file list with `{path,sha,size}` for provenance (generated once via `list_repo_tree`).
- **Delivery**: Fetch at runtime via CDN (`/data/...`). No build-time coupling; no HF API during render.
- **No mutations**: No POST/PUT/DELETE; no writes to localStorage/indexedDB beyond transient React UI state.
- **Auditability**: Card exposes `signalId`, `data-audit` attribute, and deep-link to audit trail.

### Static data generation (once)
Generate the file list on the orchestration host and commit:

```bash
# Example (use actual script path)
python scripts/list_hub_files.py --date=2026-05-02 --out=public/data/hub-files.json
```

`public/data/hub-files.json` shape:
```json
{
  "date": "2026-05-02",
  "files": [
    { "path": "2026-05-02/moc-summary.md", "sha": "abc123", "size": 2451 },
    { "path": "2026-05-02/compliance-incidents.json", "sha": "def456", "size": 8720 }
  ],
  "sourceCount": 2,
  "lastIndexed": "2026-05-02T20:14:00Z"
}
```

`public/data/knowledge/top-hub.json` shape:
```json
{
  "signalId": "hub-moc-2026-05-02",
  "hub": "MOC",
  "displayName": "Mission Operations Center",
  "score": 0.92,
  "rationale": "Highest betweenness centrality across cost, compliance, and incident signals. Key coordination point for multi-cloud governance.",
  "lastUpdated": "2026-05-02T20:14:00Z",
  "auditUrl": "/audit/signals/hub-moc-2026-05-02",
  "tags": ["knowledge-rag", "graph", "hub", "MOC"],
  "relatedDocs": [
    { "title": "MOC summary", "path": "2026-05-02/moc-summary.md" },
    { "title": "Compliance incidents", "path": "2026-05-02/compliance-incidents.json" }
  ]
}
```

---

### Implementation steps (≤2h)

1. Add the two static JSON files to `public/data/`.
2. Build `TopHubSignalCard` (TypeScript + CSS modules):
   - Fetch `top-hub.json` at runtime (CDN).
   - Optionally fetch `hub-files.json` to enrich provenance (source count, last indexed).
   - Pure presentational component; no state mutations.
   - Use `data-audit` attribute for machine-readable handoff.
3. Wire into dashboard in `App.tsx` (high-priority zone, accessible region).
4. Verify:
   - No POST/PUT/DELETE in devtools.
   - No runtime HF API calls.
   - Card is read-only and gracefully handles missing data.
