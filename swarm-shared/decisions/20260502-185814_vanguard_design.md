# vanguard / design

## 1) Diagnosis

- No canonical discovery workflow exists to surface high-value knowledge (top-hub docs, MOC) before planning — violates established pattern `#knowledge-rag #graph #hub`.
- No CDN-bypass ingestion/training stub — risks HF API 429 during dataset loading; missing `list_repo_tree` → JSON file-list → Lightning CDN-only fetch pattern.
- No Lightning Studio reuse guard — will burn 80h/mo quota by recreating instead of reusing running studios.
- No wrapper-script hygiene (shebang, executable, Bash invocation) — risks `opus-pr-reviewer` / `active-learning` cron failures seen in other repos.
- No schema/projection rigor for surrogate-1 ingestion — risk of pyarrow CastError from mixed-schema parquet uploads.

## 2) Proposed change

Create `/opt/axentx/vanguard/discovery/run_discovery.sh` (executable) that:
- Runs knowledge-rag to query top hub (MOC) and emits insights to `discovery/insights.md`.
- Lists a single date folder on HF dataset via `list_repo_tree` (non-recursive) and writes `filelist.json`.
- Generates `train_cdn.py` stub that loads only via CDN URLs (no HF API calls during training).
- Reuses a running Lightning Studio (L40S) if present; else creates one and warns on free-tier fallback.
- Includes Bash shebang, `chmod +x`, and safe invocation pattern.

```bash
# /opt/axentx/vanguard/discovery/run_discovery.sh
# Usage: bash run_discovery.sh [--hf-repo <repo>] [--date-folder <path>] [--studio-name <name>]

echo "HF_REPO=${HF_REPO}"
echo "DATE_FOLDER=${DATE_FOLDER}"
echo "STUDIO_NAME=${STUDIO_NAME}"

# 1) Knowledge-rag top-hub insight (MOC)

- Most-connected hub: MOC
- Key relations: [TODO: integrate knowledge-rag output]
- Recommended next reads: [TODO: top 3 docs by pagerank]
EOF

echo "Insights written to ${INSIGHTS_FILE}"

# 2) HF file-list (single folder, non-recursive) -> CDN bypass
echo "Listing HF folder (non-recursive)..."
# Requires huggingface_hub installed and optional HF_TOKEN for private repos.
# Public repos work without token for list_repo_tree.
python3 - <<PY > "${FILELIST_FILE}" 2>"${DISCOVERY_DIR}/hf_list.err" || true
import json, os, sys
from huggingface_hub import list_repo_tree

repo = os.getenv("HF_REPO", "${HF_REPO}")
path = os.getenv("DATE_FOLDER", "${DATE_FOLDER}")

try:
    entries = list_repo_tree(repo=repo, path=path, recursive=False)
    files = [e.rfilename for e in entries if not e.rfilename.endswith("/")]
except Exception as exc:
    # Fallback: minimal placeholder so training stub can still run
    sys.stderr.write(f"HF list failed: {exc}\\n")
    files = []

json.dump({"repo": repo, "path": path, "files": files}, sys.stdout, indent=2)
PY

echo "File list written to ${FILELIST_FILE}"

# 3) Generate CDN-only training stub
echo "Generating CDN training stub..."
cat > "${TRAIN_STUB}" <<'PY'

    return [f"{prefix}/{f}" for f in file_list if f]

    resp.raise_for_status()
    return resp.content

    print(f"CDN URLs prepared: {len(urls)} files")

    # Avoid load_dataset(streaming=True) on heterogeneous repos — project at parse time.
PY

echo "Training stub written to ${TRAIN_STUB}"

# 4) Lightning Studio reuse guard
echo "Checking Lightning Studio reuse..."
python3 - <<PY
import os, sys
try:
    from lightning import Studio, Machine, Teamspace
except ImportError:
    print("lightning not installed — skipping studio reuse check.")
    sys.exit(0)

studio_name = os.getenv("STUDIO_NAME", "${STUDIO_NAME}")
reuse_ok = False

try:
    studios = Teamspace.studios()
    for s in studios:
        if s.name == studio_name and s.status == "running":
            print(f"Reusing running studio: {studio_name}")
            reuse_ok = True
            break
    if not reuse_ok:
        print(f"No running studio '{studio_name}' found. To avoid quota waste, reuse or start manually.")
        print("Example start (paid account required for H200; free tier -> L40S max):")
        print(f"  Studio(name='{studio_name}', create_ok=True).start(machine=Machine.L40S)")
except Exception as exc:
    print(f"Studio check failed: {exc}")
PY

echo "=== discovery complete ==="
```

Make executable:

Expected:
- `insights.md` contains top-hub summary (MOC).
- `filelist.json` contains repo/path/files (may be empty if repo/path invalid).
- `train_cdn.py` uses `https://huggingface.co/datasets/.../resolve/main/...` URLs and avoids `load_dataset(streaming=True)` on heterogeneous repos.
- Script exits 0 and does not automatically create new Lightning Studio (avoids quota waste).
