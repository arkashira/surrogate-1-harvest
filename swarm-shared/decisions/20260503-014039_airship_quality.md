**Final Synthesis: Highest-Value, Actionable Plan to Eliminate HF 429s and Ship in <2h**

**Core Improvement**  
Replace `load_dataset(streaming=True)` with a **pre-listed, CDN-only data path** and **deterministic sibling-repo sharding** for uploads. This removes auth/rate-limit exposure during training and caps HF commit pressure at ~640/hr across 5 shards.

---

### Concrete Implementation Plan (prioritized for <2h)

1. **Pre-list & embed file manifest (one-time, ~5–10 min)**  
   - Run a single `list_repo_tree(recursive=False)` per date folder.  
   - Emit `training/file_manifest_{date}.json` containing: repo, date, sorted parquet paths, and CDN prefix.  
   - Commit this manifest alongside training code so the training script never calls HF APIs for listing.

2. **Switch training to CDN-only loading**  
   - Replace `load_dataset(streaming=True, ...)` with direct `requests.get`/`aiohttp` to `https://huggingface.co/datasets/{repo}/resolve/main/{path}`.  
   - Parse parquet → keep only `{prompt, response}` to avoid schema drift.  
   - Stream into your dataloader; no HF API calls during training.

3. **Deterministic sibling-repo sharding for uploads**  
   - Before upload: `shard_id = int(sha256(slug).hexdigest(), 16) % 5`.  
   - Upload enriched parquet to `surrogate-data-shard-{shard_id}`.  
   - On 429: wait 360 s and retry; on persistent cap: rotate to next shard.

4. **Studio reuse & resilience (Lightning)**  
   - At launch, list running studios; reuse if name matches and status is `Running`.  
   - If stopped, restart with `Machine.L40S` (or `H200` in `lightning-lambda-prod`, fallback to `L40S` in public).  
   - Avoid unnecessary teardown/spin-up to reduce HF commit churn.

### Minimal, Merged Code Snippets

**1) Build manifest (run once per date folder)**
```python
#!/usr/bin/env python3
import json, os
from huggingface_hub import HfApi

HF_REPO = "axentx/surrogate-training-data"
DATE_FOLDER = "2026-05-03"          # parameterize in CI
OUT_PATH = f"training/file_manifest_{DATE_FOLDER}.json"

api = HfApi()
entries = api.list_repo_tree(
    repo_id=HF_REPO,
    path=DATE_FOLDER,
    repo_type="dataset",
    recursive=False,
)

files = sorted(e.path for e in entries if e.path.endswith(".parquet"))
manifest = {
    "repo": HF_REPO,
    "date": DATE_FOLDER,
    "files": files,
    "cdn_prefix": f"https://huggingface.co/datasets/{HF_REPO}/resolve/main",
}

os.makedirs(os.path.dirname(OUT_PATH), exist_ok=True)
with open(OUT_PATH, "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Wrote {len(files)} files to {OUT_PATH}")
```

**2) CDN-only loader + deterministic sharding + resilient upload**
```python
#!/usr/bin/env python3
import json, hashlib, pyarrow.parquet as pq, requests, time
from pathlib import Path
from huggingface_hub import upload_file

MANIFEST = Path("training/file_manifest_2026-05-03.json")
with MANIFEST.open() as f:
    manifest = json.load(f)

CDN = manifest["cdn_prefix"]
FILES = manifest["files"]

def deterministic_shard(slug: str, n: int = 5) -> int:
    return int(hashlib.sha256(slug.encode()).hexdigest(), 16) % n

def load_parquet_cdn(path: str):
    url = f"{CDN}/{path}"
    r = requests.get(url, timeout=30)
    r.raise_for_status()
    return pq.read_table(pq.ParquetFile(pq.BufferReader(r.content)))

def project_record(batch):
    return {
        "prompt": batch["prompt"].to_pylist(),
        "response": batch["response"].to_pylist(),
    }

def upload_shard(parquet_bytes: bytes, slug: str, max_retries: int = 3):
    shard_id = deterministic_shard(slug)
    shard_repo = f"axentx/surrogate-data-shard-{shard_id}"
    date_prefix = "batches/mirror-merged/2026-05-03"
    out_path = f"{date_prefix}/{slug}.parquet"

    for attempt in range(max_retries):
        try:
            upload_file(
                path_or_fileobj=parquet_bytes,
                path_in_repo=out_path,
                repo_id=shard_repo,
                repo_type="dataset",
            )
            return
        except Exception as e:
            if "429" in str(e):
                wait = 360
                print(f"HF 429 (shard {shard_id}): waiting {wait}s (attempt {attempt+1})")
                time.sleep(wait)
                continue
            raise
    print(f"Failed to upload {slug} after {max_retries} attempts")

# Example training loop (streaming)
records = []
for f in FILES:
    tbl = load_parquet_cdn(f)
    batch = project_record(tbl)
    records.extend(batch)  # or yield into dataloader

# After enrichment:
# parquet_bytes, slug = ...  # produce enriched parquet
# upload_shard(parquet_bytes, slug)
```

**3) Lightning studio reuse (minimal, resilient)**
```python
#!/usr/bin/env python3
from lightning.pai import Teamspace, Studio, Machine

TEAMSPACE = "axentx"
STUDIO_NAME = "surrogate-train-l40s"

def get_or_create_studio():
    ts = Teamspace(name=TEAMSPACE)
    for s in ts.studios:
        if s.name == STUDIO_NAME and s.status == "Running":
            print(f"Reusing running studio: {STUDIO_NAME}")
            return s
    print(f"Creating studio: {STUDIO_NAME}")
    try:
        return Studio.create(
            name=STUDIO_NAME,
            teamspace=TEAMSPACE,
            machine=Machine.L40S,
            # Prefer H200 in lightning-lambda-prod; fallback to L40S in public
        )
    except Exception:
        return Studio.create(
            name=STUDIO_NAME,
            teamspace=TEAMSPACE,
            machine=Machine.L40S,
        )
```
