# Costinel / design

## Costinel — Design Increment (≤2h)

**Goal:** Ship a lightweight, deterministic discovery module that produces a machine-readable `discovery/manifest.json` (cloud accounts, dataset file lists, top-hub context) so onboarding, training, and governance scripts can run without repeated API calls or manual steps.

**Why this is highest-value:**  
- Eliminates repeated HF API calls and rate-limit risk (uses CDN bypass pattern).  
- Embeds top-hub context (knowledge-rag pattern) so downstream tasks start with semantic priors.  
- Provides a single source of truth for training scripts (pre-list file paths pattern) and governance automation.  
- Fits in <2h and is immediately consumable by CI, training launchers, and cost-governance pipelines.

---

## Implementation Plan

1. **Create discovery module scaffold** (`discovery/`) with:
   - `manifest.json` schema (cloud accounts, datasets[], top_hub, generated_at, version)
   - `discover.py` — deterministic collector with local caching
   - `requirements.txt` (minimal: requests, pyyaml)

2. **Cloud account inventory** (static + optional dynamic):
   - Default: read `config/clouds.yaml` if present; else include `discovery/templates/clouds.yaml` with example AWS/GCP/Azure accounts and required roles.
   - If env var `COSTINEL_DISCOVER_LIVE=1`, attempt lightweight metadata endpoints (IMDS/GCP metadata/Azure IMDS) and merge; never fail.

3. **Dataset file list (HF CDN bypass)**:
   - Single API call to `list_repo_tree(path, recursive=False)` for configured dataset repo + date folder (from config).
   - Persist list to `discovery/manifest.json` under `datasets[].files[]`.
   - Include `cdn_base` URL template so training scripts can fetch via CDN without auth.

4. **Top-hub context (knowledge-rag pattern)**:
   - Read `knowledge/hub_graph.json` (or generate stub if absent) and pick the most-connected node.
   - Include `top_hub` with `id`, `label`, `degree`, `summary` (short) in manifest.

5. **Training-script readiness**:
   - Embed `file_list_path` and `cdn_base` so `train.py` can do CDN-only fetches.
   - Include `checkpoint` and `lightning` hints (account, preferred machine) from config.

6. **CI integration**:
   - Add `scripts/update-manifest.sh` to regenerate manifest (respect HF rate-limit backoff).
   - Add `.gitignore` for local cache files.

7. **Validation**:
   - `discovery/validate.py` — schema check + CDN reachability spot-check (1–2 files).

"""

    cfg_path = Path("config/clouds.yaml")

    # fallback defaults

                "account_id": "000000000000",

                "source": "defaults"
            }

            }

def list_hf_folder(repo: str, folder: str) -> List[str]:
    """List files in a repo folder (non-recursive) using HF API, then return paths."""
    url = f"{HF_API_BASE}/datasets/{repo}/tree"
    params = {"path": folder, "recursive": "false"}
    resp = requests.get(url, params=params, timeout=30)
    if resp.status_code == 429:
        # Respect rate limit: wait and retry once (simple)
        retry_after = int(resp.headers.get("Retry-After", 360))
        print(f"HF rate-limited. Waiting {retry_after}s...", file=sys.stderr)
        import time
        time.sleep(retry_after)
        resp = requests.get(url, params=params, timeout=30)
    resp.raise_for_status()
    items = resp.json()
    # Keep only files (exclude nested trees)
    files = [it["path"] for it in items if it.get("type") == "file"]
    return files


def top_hub_from_graph(graph_path: Path) -> Dict[str, Any]:

            "label": "MOC",

            "summary": "Default hub (graph not found)."
