# vanguard / ops

## Final Synthesized Implementation

**Core diagnosis (agreed by both):**  
- No persisted manifest → every training run re-enumerates via authenticated HF API → quota burn + 429 risk.  
- Data loader likely uses recursive enumeration or `load_dataset` during training → amplifies rate-limit exposure on heterogeneous schemas.  
- No CDN-only fetch path → misses HF CDN bypass (public `resolve/main/` URLs avoid auth and rate limits).  
- Lightning Studio reuse not enforced → idle stop/start churn wastes quota and risks lost progress.  
- No guard for Lightning idle timeout before `.run()` → stopped studios silently kill jobs.

**Chosen approach (favoring correctness + concrete actionability):**  
- Single manifest generation (non-recursive top-level + optional recursive fallback) persisted to JSON.  
- Training uses CDN-only fetches (zero authenticated API calls).  
- Lightweight, robust studio lifecycle with pre-run idle check and restart.  
- Minimal, explicit dependencies and clear verification steps.

```python
#!/usr/bin/env python3
"""
Generate and persist a (repo, dateFolder) file manifest once.
Avoids repeated authenticated HF API calls during training.

Usage:
    python3 training_manifest.py <repo> <dateFolder> [--out-dir OUT_DIR] [--recursive]

Example:
    python3 training_manifest.py axentx/vanguard-ds 2026-05-03 --out-dir ./manifests
"""
import argparse
import json
import os
import sys
from pathlib import Path

from huggingface_hub import list_repo_tree

def build_manifest(repo: str, date_folder: str, recursive: bool = False):
    """Return manifest dict with repo, date_folder, files, and cdn_prefix."""
    # Non-recursive by default to avoid quota burn; allow recursive opt-in
    tree = list_repo_tree(repo=repo, path=date_folder, recursive=recursive)
    files = sorted(f.rfilename for f in tree if f.type == "file")

    manifest = {
        "repo": repo,
        "date_folder": date_folder,
        "recursive": recursive,
        "files": files,
        "cdn_prefix": f"https://huggingface.co/datasets/{repo}/resolve/main/{date_folder}",
    }
    return manifest

def save_manifest(manifest, out_dir):
    repo_slug = manifest["repo"].replace("/", "_")
    date_folder = manifest["date_folder"]
    out_file = Path(out_dir) / f"{repo_slug}" / f"{date_folder}.json"
    out_file.parent.mkdir(parents=True, exist_ok=True)

    with open(out_file, "w", encoding="utf-8") as f:
        json.dump(manifest, f, indent=2, ensure_ascii=False)

    return out_file

def main():
    parser = argparse.ArgumentParser(description="Generate HF dataset file manifest for CDN-only training.")
    parser.add_argument("repo", help="HF dataset repo, e.g. axentx/vanguard-ds")
    parser.add_argument("date_folder", help="Date folder under dataset root, e.g. 2026-05-03")
    parser.add_argument("--out-dir", default="./manifests", help="Output directory for manifests")
    parser.add_argument("--recursive", action="store_true", help="Recursive listing (use sparingly; costs quota)")
    args = parser.parse_args()

    try:
        manifest = build_manifest(args.repo, args.date_folder, recursive=args.recursive)
        out_file = save_manifest(manifest, args.out_dir)
        print(f"Manifest saved: {out_file} ({len(manifest['files'])} files)")
    except Exception as e:
        print(f"Failed to generate manifest: {e}", file=sys.stderr)
        sys.exit(1)

if __name__ == "__main__":
    main()
```
