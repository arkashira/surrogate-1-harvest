# vanguard / design

## 1. Diagnosis
- No content-addressed manifest per date folder → runtime repo enumeration via `list_repo_tree`/`load_dataset` triggers HF API 429s and non-reproducible epochs.
- Training script performs HF API calls during data loading (violates CDN-only rule) → brittle under rate limits and wastes quota.
- Missing deterministic `{path, sha256}` snapshot → epochs can diverge across runs and resuming is unreliable.
- No reuse guard for Lightning Studio → each run risks creating new studios and burning 80hr/mo quota.
- Data ingestion writes mixed-schema files to `enriched/` instead of projecting to `{prompt, response}` only → downstream `pyarrow` CastError on heterogeneous HF datasets.

## 2. Proposed change
Create `/opt/axentx/vanguard/design/manifest.py` (single file, <150 LOC) that:
- Scans one date folder via `list_repo_tree` (single API call) and emits `manifest-{date}.json` with `{path, sha256, size}`.
- Provides a `cdn_urls(repo, date)` generator that yields raw CDN URLs (`https://huggingface.co/datasets/.../resolve/main/...`) for Lightning training scripts.
- Adds a small `train.py` patch that loads the embedded manifest and streams files via CDN (zero HF API calls during training).
- Adds a `reuse_or_create_studio()` helper that lists existing running studios and reuses one (or creates) to preserve Lightning quota.

## 3. Implementation

```bash
# /opt/axentx/vanguard/design/manifest.py
import json, hashlib, os, sys, time
from pathlib import Path
from typing import List, Dict, Generator

try:
    from huggingface_hub import list_repo_tree, hf_hub_download
except ImportError:
    print("huggingface_hub required", file=sys.stderr)
    sys.exit(1)

REPO = os.getenv("HF_DATASET_REPO", "datasets/your-dataset")
DATE_FOLDER = os.getenv("DATE_FOLDER", "2026-04-29")  # e.g. batches/mirror-merged/2026-04-29
MANIFEST_PATH = Path(__file__).parent / f"manifest-{DATE_FOLDER}.json"

def list_date_files(repo: str, date_folder: str) -> List[Dict]:
    """Single API call: non-recursive top-level of date folder."""
    items = list_repo_tree(repo=repo, path=date_folder, recursive=False)
    out = []
    for it in items:
        if it.type == "file":
            out.append({"path": f"{date_folder}/{it.path}", "sha256": it.lfs.get("oid", ""), "size": it.size})
    return out

def build_manifest(repo: str, date_folder: str, force: bool = False) -> Path:
    if MANIFEST_PATH.exists() and not force:
        return MANIFEST_PATH
    files = list_date_files(repo, date_folder)
    manifest = {"repo": repo, "date": date_folder, "files": files, "generated_at": time.time()}
    MANIFEST_PATH.write_text(json.dumps(manifest, indent=2))
    return MANIFEST_PATH

def cdn_urls(manifest_path: Path) -> Generator[str, None, None]:
    manifest = json.loads(manifest_path.read_text())
    repo = manifest["repo"]
    for f in manifest["files"]:
        # CDN bypass: no Authorization header required
        yield f"https://huggingface.co/datasets/{repo}/resolve/main/{f['path']}"

def reuse_or_create_studio(name: str, machine: str = "L40S"):
    """Lightning Studio reuse guard (minimal)."""
    try:
        from lightning import Studio, Machine, Teamspace
    except ImportError:
        print("lightning missing; skipping studio reuse", file=sys.stderr)
        return None
    for s in Teamspace.studios:
        if s.name == name and s.status == "Running":
            print(f"Reusing running studio: {name}")
            return s
    print(f"Creating studio: {name}")
    return Studio(
        name=name,
        machine=Machine(machine),
        create_ok=True,
        start_now=True,
    )

if __name__ == "__main__":
    build_manifest(REPO, DATE_FOLDER)
    print(f"Manifest written: {MANIFEST_PATH}")
    for url in cdn_urls(MANIFEST_PATH):
        print(url)
```

2. Confirm zero HF API calls during training:

3. Reproducibility: run the same training script twice with identical manifest — examples order/content must match exactly.
4. Studio reuse: invoke `reuse_or_create_studio("vanguard-train")` twice — second call should log “Reusing running studio” and return the same studio object without creating a new one.
