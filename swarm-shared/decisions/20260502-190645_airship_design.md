# airship / design

## Implementation Plan: `airship discover` — Zero-config discovery CLI

**Highest-value incremental improvement (<2h):**  
Ship a single Bash CLI (`airship discover`) that operationalizes past patterns (business research + knowledge-rag + top-hub/MOC insight) into one command that surfaces actionable context for the Arkship + Surrogate platform.

### Scope
- One executable: `airship discover`
- Runs: `granite-business-research.sh` → `knowledge-rag` query → top-hub/MOC review
- Outputs: concise markdown report to `./reports/discover-<timestamp>.md`
- Zero config: uses existing scripts and defaults; fails gracefully with clear errors

---

### File Changes

#### 1. `/opt/axentx/airship/bin/airship`
Main CLI dispatcher.

```bash
#!/usr/bin/env bash
# /opt/axentx/airship/bin/airship
# Airship CLI — zero-config DevOps/AI platform commands

set -euo pipefail

AXENTX_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
REPORTS_DIR="${AXENTX_ROOT}/reports"

__discover() {
  local ts
  ts="$(date +%Y%m%d-%H%M%S)"
  local report="${REPORTS_DIR}/discover-${ts}.md"

  mkdir -p "${REPORTS_DIR}"

  echo "[$(date --iso-8601=seconds)] airship discover: starting"

  # 1) Business research
  echo "## Business Research" > "${report}"
  echo "" >> "${report}"
  if [[ -x "${AXENTX_ROOT}/scripts/granite-business-research.sh" ]]; then
    echo "Running granite-business-research.sh..."
    if "${AXENTX_ROOT}/scripts/granite-business-research.sh" >> "${report}" 2>&1; then
      echo "" >> "${report}"
    else
      echo "⚠️  granite-business-research.sh exited non-zero; continuing." >> "${report}"
      echo "" >> "${report}"
    fi
  else
    echo "⚠️  granite-business-research.sh not found or not executable; skipping." >> "${report}"
    echo "" >> "${report}"
  fi

  # 2) Knowledge RAG — top hub and related docs
  echo "## Knowledge RAG — Top Hub & Related Docs" >> "${report}"
  echo "" >> "${report}"
  if command -v knowledge-rag >/dev/null 2>&1; then
    echo "Querying knowledge-rag for top hub and related docs..."
    {
      echo 'Top hub and related insights:'
      knowledge-rag query --top-hub 2>/dev/null || echo "knowledge-rag query unavailable"
      echo ""
      knowledge-rag query --related --hub MOC 2>/dev/null || true
    } >> "${report}" 2>&1 || true
    echo "" >> "${report}"
  else
    echo "⚠️  knowledge-rag not available; skipping RAG queries." >> "${report}"
    echo "" >> "${report}"
  fi

  # 3) Top-hub (MOC) review note
  echo "## Top-Hub Review (MOC)" >> "${report}"
  echo "" >> "${report}"
  echo "Review the most-connected hub (e.g., MOC) before planning tasks." >> "${report}"
  echo "- Tags: #knowledge-rag #graph #hub" >> "${report}"
  echo "" >> "${report}"

  # 4) Recommendations / next actions
  echo "## Recommendations" >> "${report}"
  echo "" >> "${report}"
  echo "- Validate top hub connections in Neo4j (Arkship/Surrogate knowledge graph)." >> "${report}"
  echo "- Cross-check business research insights against Surrogate AI roles (Guardian, Navigator, etc.)." >> "${report}"
  echo "- If gaps exist, run \`knowledge-rag ingest <path>\` to enrich the graph." >> "${report}"
  echo "" >> "${report}"

  echo "[$(date --iso-8601=seconds)] Report written: ${report}"
  echo "Done. Report: ${report}"
}

case "${1:-}" in
  discover)
    __discover
    ;;
  *)
    echo "Usage: airship {discover}"
    exit 1
    ;;
esac
```

#### 2. Make executable and ensure PATH usage
```bash
chmod +x /opt/axentx/airship/bin/airship
```

Optionally link into `/usr/local/bin` for global use:

### Notes & Best Practices Applied

- **Bash hygiene**: `set -euo pipefail`, explicit `#!/usr/bin/env bash`, no interactive prompts.
- **Fail-graceful**: Missing scripts or commands produce warnings, not hard failures.
- **Idempotent & timestamped reports**: each run creates a new file; no clobbering.
- **Zero-config**: relies on existing `granite-business-research.sh` and `knowledge-rag` if present; does not require user inputs.
- **Past patterns honored**:
  - `#business-research #knowledge-rag #graph`
  - `#knowledge-rag #graph #hub` (top-hub/MOC review)
- **Surrogate/Arkship alignment**: report recommends cross-checking insights against Surrogate AI roles and Neo4j knowledge graph.
