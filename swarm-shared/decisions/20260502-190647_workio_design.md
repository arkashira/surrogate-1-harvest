# workio / design

## 1. Diagnosis

- **No tenant cache on LINE webhook** — every webhook performs a PostgreSQL lookup (`channelId → tenant`); under burst traffic this amplifies DB load, adds latency, and increases error/failure surface.
- **Missing health/readiness endpoints** — no `/health` or `/ready` for k8s/PM2/liveness probes or load balancer checks.
- **Cold-start DB amplification** — first webhook after idle period pays full connection + query cost; without cache this blocks LINE’s retry window and can cause duplicate events.
- **No circuit-breaker or timeout guard** — DB query failures on webhook path can bubble into LINE 5xx responses, causing LINE to retry and compound load.
- **Missing observability hooks** — no request-id or timing on webhook path; hard to correlate LINE delivery vs internal processing.

## 2. Proposed change

Add an in-memory LRU tenant cache (TTL 5m, refresh-on-miss) and `/health`/`/ready` endpoints to the Express backend. Scope:

- `workio/server/src/app.ts` (or main server file) — add routes and cache middleware.
- `workio/server/src/services/tenantService.ts` (or create) — add `getTenantByChannelIdCached(channelId)` with LRU.
- `workio/server/src/middleware/health.ts` — lightweight probes.

## 3. Implementation

Install tiny LRU (if not present):

```bash
cd /opt/axentx/workio/server
npm install lru-cache
```

Create cache + service helper:

```ts
// server/src/lib/tenantCache.ts
import { LRUCache } from 'lru-cache';
import { getTenantByChannelId } from '../db/queries/tenantQueries'; // adjust import path to your existing query

type Tenant = { id: string; name: string; channelId: string };

const cache = new LRUCache<string, Tenant>({
  max: 500,
  ttl: 1000 * 60 * 5, // 5m
  allowStale: false,
  updateAgeOnGet: true,
});

export async function getTenantByChannelIdCached(channelId: string): Promise<Tenant | null> {
  const cached = cache.get(channelId);
  if (cached) return cached;

  try {
    const tenant = await getTenantByChannelId(channelId); // implement or adapt to your DB layer
    if (tenant) cache.set(channelId, tenant);
    return tenant || null;
  } catch (err) {
    // On DB failure, do not cache negatives — fail fast so LINE retry can succeed later
    console.error('[tenantCache] DB lookup failed', { channelId, err });
    throw err;
  }
}
```

Add health/readiness middleware:

```ts
// server/src/middleware/health.ts
import { Router } from 'express';
import db from '../db'; // adjust to your db client

const router = Router();

// Liveness: process is up
router.get('/health', (_req, res) => {
  res.json({ status: 'ok', uptime: process.uptime() });
});

// Readiness: dependencies available
router.get('/ready', async (_req, res) => {
  try {
    // lightweight query; adapt to your client (pg, prisma, etc.)
    await db.query('SELECT 1');
    res.json({ status: 'ready' });
  } catch (err) {
    res.status(503).json({ status: 'unavailable', error: String(err) });
  }
});

export default router;
```

Wire into main app and replace DB lookup in LINE webhook:

```ts
// server/src/app.ts (or server/src/index.ts)
import express from 'express';
import healthRouter from './middleware/health';
import { getTenantByChannelIdCached } from './lib/tenantCache';

const app = express();

app.use(express.json());
app.use('/health', healthRouter);

// Example LINE webhook route — adapt to your existing route
app.post('/webhook/line', async (req, res) => {
  const channelId = req.header('x-line-channel-id') || '';
  const startedAt = Date.now();

  if (!channelId) {
    return res.status(400).json({ error: 'missing channelId' });
  }

  try {
    const tenant = await getTenantByChannelIdCached(channelId);
    if (!tenant) {
      return res.status(404).json({ error: 'tenant not found' });
    }

    // attach tenant to request for downstream handlers
    (req as any).tenant = tenant;

    // ... existing LINE event processing ...

    res.json({ ok: true });
  } catch (err) {
    // Avoid returning 5xx to LINE when possible; if DB is down, 503 tells LINE to retry later
    console.error('[line/webhook] error', { channelId, err, durationMs: Date.now() - startedAt });
    res.status(503).json({ error: 'service unavailable' });
  }
});

export default app;
```

If you don’t have `getTenantByChannelId`, implement a minimal query (example for `pg`):

```ts
// server/src/db/queries/tenantQueries.ts
import db from '../db';

export async function getTenantByChannelId(channelId: string) {
  const { rows } = await db.query('SELECT id, name, channel_id AS "channelId" FROM tenants WHERE channel_id = $1 LIMIT 1', [channelId]);
  return rows[0] || null;
}
```

## 4. Verification

1. Start backend:
   ```bash
   cd /opt/axentx/workio/server
   npm run dev
   ```

2. Check probes:
   ```bash
   curl -i http://localhost:PORT/health
   # expect 200 + {"status":"ok",...}
   curl -i http://localhost:PORT/ready
   # expect 200 when DB is up; 503 when DB is down
   ```

3. Simulate LINE webhook with and without cache:
   ```bash
   # first request (miss)
   curl -X POST http://localhost:PORT/webhook/line \
     -H 'x-line-channel-id: test-channel-123' \
     -H 'Content-Type: application/json' \
     -d '{"events":[]}' -i

   # second request (hit) — should be faster and not hit DB
   curl -X POST http://localhost:PORT/webhook/line \
     -H 'x-line-channel-id: test-channel-123' \
     -H 'Content-Type: application/json' \
     -d '{"events":[]}' -i
   ```

4. Observe logs: first request logs DB query; second request logs cache hit and returns quickly.

5. Validate resilience: stop PostgreSQL temporarily and confirm `/ready` returns 503 while `/health` returns 200, and LINE webhook returns 503 (so LINE will retry) rather than cascading failures.
