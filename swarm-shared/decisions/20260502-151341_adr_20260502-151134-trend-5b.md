## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research/signal ingestion -> risk check -> compliant execution attempt) with Postgres (Supabase) persistence, Redis Streams queue, idempotency, immutable audit trail, edge API via Cloudflare Workers, and adapter stubs for one broker + one data vendor; includes replay endpoint and basic compliance guardrails.

## Open questions
- Which brokers/OMS/EMS and data vendors must be prioritized for v0.1 connectors and what auth patterns (OAuth, API keys, mTLS) do they require?
- What are the non-negotiable compliance guardrails (e.g., max order size, restricted symbols, pre-trade checks) and who owns rule definitions?
- What deterministic replay and immutability guarantees are required for audit (WORM storage, retention period, cryptographic proofs)?
- What latency SLO (p99) is required for the orchestration layer and what trade-offs are acceptable on free-tier infra?
- How will secrets/credentials be stored and rotated (Supabase Vault, Workers KV, or external) while staying cloud-free/free-tier?
- What observability and alerting stack will be used (logs, traces, metrics) given free-tier constraints?
