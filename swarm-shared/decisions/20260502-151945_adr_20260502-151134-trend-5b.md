## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres) + Hugging Face Spaces (ephemeral Redis/worker fallback)

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk guardrail→execution stub) with immutable audit trail on Supabase, deployed on CF Workers (free) + Supabase (free), one pre-built broker connector mock, compliance rule engine with block/warn severity, and replay endpoint to re-run any persisted workflow_run deterministically.

## Open questions
- Deterministic replay: do we store full input/output snapshots per stage or rely on event sourcing from audit_event?
- Latency SLA target for research→execution (sub-100ms vs sub-1s) and acceptable queueing/backpressure strategy?
- Compliance rule engine: use lightweight JSON logic/policy-as-code or embed WASM for vendor-supplied rules?
- Connector security model: per-tenant secrets in Supabase vault vs CF KV and rotation cadence?
- Idempotency key strategy for retries across brokers/OMS to prevent duplicate fills
