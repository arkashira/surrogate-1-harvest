## Deployment
cf workers (free-tier) + Supabase (free-tier) — CLI can run locally or via scheduled Worker to scan and persist results

## v0.1 scope
v0.1 (2 weeks): CLI-only scanner that authenticates to one AWS account and one GCP project, lists Lambda + Cloud Functions with cron/rate triggers, persists normalized schedules to Supabase (free), detects exact-duplicate cron+owner+region sets and orphaned schedules (no backing function), outputs a summary report and retirements list; no scheduler/queue, no web UI, local dev only.

## Open questions
- Authentication strategy for cross-account/platform credentials (assume-role JSON files, ADC, or customer-provided short-lived tokens?)
- Granularity of ownership mapping — should we infer owner from tags, project IDs, or require explicit mapping file?
- Retention policy for schedule history vs. GDPR/compliance constraints on customer data
- Heuristic thresholds for drift detection (e.g., cron change tolerance in minutes) and duplicate tolerance (same cron+owner+region)
- How to safely retire/orphan schedules: dry-run-only mode vs. opt-in delete/disable actions
