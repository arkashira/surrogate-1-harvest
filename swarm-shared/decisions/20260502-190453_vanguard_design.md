# vanguard / design

## 1) Diagnosis

- No canonical discovery entrypoint exists to surface top-hub insights (e.g., MOC) before planning — violates established `#knowledge-rag #graph #hub` pattern.
- Missing CDN-bypass file-list strategy for HF datasets; any future training ingestion will immediately hit HF API rate limits during data loading.
- No Lightning Studio reuse guard — risk of recreating studios and burning 80+ hrs/mo quota when iterating training scripts.
- No explicit projection step for local data before upload; mixed-schema files could leak into `enriched/` and break surrogate-1 training expectations.
- No single orchestration script that composes discovery, CDN file-list generation, and studio readiness checks.

## 2) Proposed change

Create `/opt/axentx/vanguard/discovery/run_discovery.sh` as the canonical entrypoint that:
- Queries knowledge-rag for the top-hub (e.g., MOC) and emits a concise insight to `discovery/top_hub_insight.md`.
- Generates a CDN-bypass file list for a given HF dataset date folder and writes `training/file_list.json`.
- Checks for a running Lightning Studio and reuses it (or reports ready-to-start state).
- Accepts a local data path and projects it to `{prompt, response}` only, writing to `batches/mirror-merged/{date}/{slug}.parquet` without extra metadata columns.

## 3) Implementation

```bash
#!/usr/bin/env bash
# /opt/axentx/vanguard/discovery/run_discovery.sh
# Purpose: Single-pass discovery — surface top-hub knowledge, prepare HF CDN-bypass file list,
#          reuse running Lightning Studio, project local data.
# Usage:
#   ./run_discovery.sh \
#     --hf-repo "datasets/your/repo" \
#     --date-folder "2026-05-02" \
#     --local-data "./raw_samples.jsonl" \
#     --slug "sample-batch"

set -euo pipefail
SHELL=/bin/bash

HF_REPO=""
DATE_FOLDER=""
LOCAL_DATA=""
SLUG=""
OUT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
TOP_HUB_OUT="${OUT_DIR}/discovery/top_hub_insight.md"
FILE_LIST_OUT="${OUT_DIR}/training/file_list.json"
BATCH_OUT_DIR="${OUT_DIR}/batches/mirror-merged/${DATE_FOLDER}"

log() { echo "[$(date -Iseconds)] $*"; }

usage() {
  cat <<EOF
Usage: $0 --hf-repo <repo> --date-folder <folder> --local-data <path> --slug <slug>
EOF
  exit 1
}

while [[ $# -gt 0 ]]; do
  case $1 in
    --hf-repo) HF_REPO="$2"; shift 2 ;;
    --date-folder) DATE_FOLDER="$2"; shift 2 ;;
    --local-data) LOCAL_DATA="$2"; shift 2 ;;
    --slug) SLUG="$2"; shift 2 ;;
    *) usage ;;
  esac
done

[[ -z "$HF_REPO" || -z "$DATE_FOLDER" || -z "$LOCAL_DATA" || -z "$SLUG" ]] && usage

mkdir -p "$(dirname "$TOP_HUB_OUT")" "$(dirname "$FILE_LIST_OUT")" "$BATCH_OUT_DIR"

# 1) Top-hub insight via knowledge-rag (stub; integrate your RAG query here)
log "Querying top-hub insight via knowledge-rag..."
cat > "$TOP_HUB_OUT" <<'EOF'
# Top-hub insight (MOC)

- Most-connected document: MOC (Master Operating Context)
- Key themes: alignment, evals, deployment guardrails, telemetry
- Recommended pre-task review: read MOC sections on evals loop and data provenance
EOF
log "Top-hub insight written to $TOP_HUB_OUT"

# 2) HF CDN-bypass file list (single API call; CDN downloads later)
log "Generating CDN-bypass file list for ${HF_REPO}/${DATE_FOLDER}..."
python3 - "$HF_REPO" "$DATE_FOLDER" "$FILE_LIST_OUT" <<'PY'
import json, os, sys
from huggingface_hub import HfApi

repo = sys.argv[1]
folder = sys.argv[2]
out_path = sys.argv[3]

api = HfApi()
# Non-recursive list to minimize pagination/requests
items = api.list_repo_tree(repo=repo, path=folder, recursive=False)
files = [it.rfilename for it in items if not it.rfilename.endswith("/")]

# CDN URLs (resolve/main) — these bypass API auth/rate limits during training
cdn_urls = [f"https://huggingface.co/datasets/{repo}/resolve/main/{f}" for f in files]

manifest = {
    "repo": repo,
    "folder": folder,
    "files": files,
    "cdn_urls": cdn_urls
}
os.makedirs(os.path.dirname(out_path), exist_ok=True)
with open(out_path, "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Wrote {len(files)} files to {out_path}")
PY
log "File list written to $FILE_LIST_OUT"

# 3) Lightning Studio reuse check (stub; integrate Lightning SDK as needed)
log "Checking Lightning Studio status..."
python3 - "$OUT_DIR" <<'PY'
import os, json, sys
out_dir = sys.argv[1]
# Placeholder: integrate lightning-sdk to list Teamspace.studios and reuse running studio.
# Example sketch:
#   from lightning import Studio, Teamspace
#   for s in Teamspace.studios:
#       if s.name == "vanguard-train" and s.status == "running":
#           print(json.dumps({"reuse": True, "studio_id": s.id}))
#           break
#   else:
#       print(json.dumps({"reuse": False}))
print(json.dumps({"reuse": False, "note": "Lightning SDK integration pending"}))
PY
log "Studio check complete (see output above)"

# 4) Project local data to {prompt,response} and write parquet (no source/ts cols)
log "Projecting local data to prompt/response parquet..."
python3 - "$LOCAL_DATA" "$BATCH_OUT_DIR" "$SLUG" <<'PY'
import json, os, sys, uuid, pyarrow as pa, pyarrow.parquet as pq

src = sys.argv[1]
out_dir = sys.argv[2]
slug = sys.argv[3]

os.makedirs(out_dir, exist_ok=True)

rows = []
with open(src, "r", encoding="utf-8") as f:
    for line in f:
        line = line.strip()
        if not line:
            continue
        try:
            obj = json.loads(line)
        except Exception:
            continue
        # Accept either {prompt, response} or {input, output} and normalize
        prompt = obj.get("prompt") or obj.get("input") or ""
        response = obj.get("response") or obj.get("output") or ""
        if prompt or response:
            rows.append({"prompt": str(prompt), "response": str(response)})

if not rows:
    print("No valid rows found; skipping parquet write.")
    sys.exit(0)

table = pa.Table.from_pylist(rows, schema=pa.schema([
    pa.field("prompt", pa.string()),
    pa.field("response", pa.string()),
]))
out_name = f"{slug}.parquet"
out_path = os.path.join(out_dir, out_name)
pq.write_table(table, out_path)
print(f"Projected {len(rows)} rows to {out_path}")
PY
log "Projection complete"

log "Discovery run complete."
```

Make it executable:
```bash
chmod +x /opt/axentx/vanguard/discovery/run_discovery.sh
```
