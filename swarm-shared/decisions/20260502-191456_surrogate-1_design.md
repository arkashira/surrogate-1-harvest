# surrogate-1 / design

## Final Implementation Plan (≤2h)

**Highest-value incremental improvement**: Add deterministic pre-flight file listing + CDN-only ingestion path to eliminate HF API rate limits during training and make shard workers resilient to 429s.

### What we’ll change
- Add `bin/list-files.py` (run once from Mac/CI) → produces `file-list-<date>.json` with CDN URLs and metadata.
- Update `bin/dataset-enrich.sh` to accept a file-list JSON and download via CDN (`/resolve/main/...`) instead of `datasets.load_dataset` or recursive API calls.
- Add retry/back-off for 429 with 360s sleep.
- Keep schema projection to `{prompt, response}` only; write to `batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl`.
- No new runtime deps beyond existing `requests`/`huggingface_hub`/`pyarrow`.
- Add fallback: if CDN fetch fails (404/403), retry once via `hf_hub_download` (keeps compatibility with private repos when token is present).

### Why this is highest value
- Eliminates the #1 failure mode (HF API 429) during parallel ingestion and training.
- Enables Lightning training to use CDN-only fetches with zero API calls (per earlier pattern).
- Small, scoped change that fits <2h and reuses existing patterns.

### 1) `bin/list-files.py`
```python
#!/usr/bin/env python3
"""
Usage (Mac, after rate-limit window clears):
  HF_TOKEN=... python bin/list-files.py \
    --repo axentx/surrogate-1-training-pairs \
    --date 2026-05-02 \
    --out file-list-2026-05-02.json

Produces JSON:
{
  "repo": "...",
  "date": "...",
  "files": [
    {"path": "raw/abc/...", "cdn_url": "https://huggingface.co/datasets/.../resolve/main/...", "size": 1234, "sha": "..."},
    ...
  ]
}
"""
import argparse
import json
import os
import sys
import time

from huggingface_hub import HfApi, list_repo_tree

CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

def main() -> None:
    parser = argparse.ArgumentParser()
    parser.add_argument("--repo", required=True)
    parser.add_argument("--date", required=True, help="Folder under datasets to list, e.g. 2026-05-02")
    parser.add_argument("--out", required=True, help="Output JSON path")
    args = parser.parse_args()

    api = HfApi(token=os.getenv("HF_TOKEN"))
    prefix = f"{args.date}/"

    # Non-recursive per-folder listing to avoid 100x pagination on big repos
    entries = list_repo_tree(
        repo_id=args.repo,
        path=prefix,
        recursive=False,
        repo_type="dataset",
    )

    files = []
    for e in entries:
        if e.type != "file":
            continue
        files.append({
            "path": e.path,
            "cdn_url": CDN_TEMPLATE.format(repo=args.repo, path=e.path),
            "size": e.size,
            "sha": e.lfs.get("sha256", "") if e.lfs else "",
        })

    payload = {
        "repo": args.repo,
        "date": args.date,
        "files": files,
    }

    with open(args.out, "w", encoding="utf-8") as f:
        json.dump(payload, f, indent=2)

    print(f"Wrote {len(files)} files to {args.out}", file=sys.stderr)

if __name__ == "__main__":
    main()
```

Make executable:
```bash
chmod +x bin/list-files.py
```

### 2) Update `bin/dataset-enrich.sh`
Add CDN download mode and 429 handling. Keep existing streaming/normalization logic but replace dataset loading with CDN fetch when file list is provided.

```bash
#!/usr/bin/env bash
#
# dataset-enrich.sh
#
# Usage (shard worker):
#   SHARD_ID=0 HF_TOKEN=... \
#     python bin/dataset-enrich.sh \
#       --file-list file-list-2026-05-02.json \
#       --out-dir batches/public-merged/2026-05-02
#
# If --file-list is omitted, falls back to legacy behavior (not recommended).

set -euo pipefail
SHELL=/bin/bash

readonly NUM_SHARDS="${NUM_SHARDS:-16}"
readonly SHARD_ID="${SHARD_ID:?required}"
readonly HF_TOKEN="${HF_TOKEN:?required}"
readonly DATE="${DATE:-$(date +%Y-%m-%d)}"
readonly OUT_DIR="${OUT_DIR:-batches/public-merged/$DATE}"

FILE_LIST=""
LEGACY_MODE=false

while [[ $# -gt 0 ]]; do
  case "$1" in
    --file-list) FILE_LIST="$2"; shift 2 ;;
    --legacy)    LEGACY_MODE=true; shift ;;
    *) echo "Unknown option $1"; exit 1 ;;
  esac
done

mkdir -p "$OUT_DIR"

timestamp=$(date +%H%M%S)
outfile="${OUT_DIR}/shard${SHARD_ID}-${timestamp}.jsonl"
tmpdir=$(mktemp -d)
cleanup() { rm -rf "$tmpdir"; }
trap cleanup EXIT

log() {
  echo "[$(date -Iseconds)] $*" >&2
}

# ---- Dedup helpers (reuse existing lib/dedup.py pattern) ----
# We keep a local sqlite dedup cache per worker; central dedup on HF Space
# remains source of truth. This avoids re-uploading duplicates within a run.
DEDUP_DB="$tmpdir/dedup.db"
python3 -c "
import sqlite3
con = sqlite3.connect('$DEDUP_DB')
con.execute('CREATE TABLE IF NOT EXISTS seen (md5 TEXT PRIMARY KEY)')
con.commit()
con.close()
"

is_seen() {
  python3 -c "
import sqlite3, sys
con = sqlite3.connect('$DEDUP_DB')
cur = con.execute('SELECT 1 FROM seen WHERE md5=?', (sys.argv[1],))
print('1' if cur.fetchone() else '')
" "$1"
}

mark_seen() {
  python3 -c "
import sqlite3, sys
con = sqlite3.connect('$DEDUP_DB')
con.execute('INSERT OR IGNORE INTO seen (md5) VALUES (?)', (sys.argv[1],))
con.commit()
con.close()
" "$1"
}

# ---- CDN download with 429 retry ----
download_cdn() {
  local url="$1"
  local out="$2"
  local max_wait=360
  local attempt=0
  while true; do
    if curl -fsSL --retry 3 --retry-delay 5 -o "$out" "$url"; then
      return 0
    fi
    local code=${?}
    # If 429-like, wait and retry
    attempt=$((attempt + 1))
    log "Download failed (attempt $attempt): $url"
    if [[ $attempt -ge 3 ]]; then
      log "Sleeping ${max_wait}s for rate-limit recovery"
      sleep "$max_wait"
      attempt=0
    else
      sleep $((attempt * 10))
    fi
  done
}

# ---- Fallback to hf_hub_download on CDN failure ----
download_with_fallback() {
  local cdn_url="$1"
  local path="$2"
  local out="$3"
  if download_cdn "$cdn_url" "$out"; then
    return 0
  fi
  log "CDN download failed, falling back to hf_hub_download for $path"
  python3 -c "
from huggingface_hub import hf_hub_download
import os
hf_hub_download(
    repo_id=os.environ['HF_REPO'],
    filename='$path',
    repo_type='dataset',
    local_dir='$tmpdir',
    token=os.environ.get('HF_TOKEN'),
)
" && cp "$tmpdir/$path" "$out"
}
