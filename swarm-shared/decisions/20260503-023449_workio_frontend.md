# workio / frontend

## Final synthesized implementation

**Chosen approach**: merge Candidate 1’s robust local-state/UI guards and Candidate 2’s deterministic idempotency + rollback + toast UX into one concrete, production-ready solution.

### Key decisions (resolve contradictions)
- **Idempotency key strategy**: use Candidate 2’s deterministic session-level key (`employeeId:YYYY-MM-DD:intent`) **plus** a short random suffix to avoid collisions across retries within the same intent window. This is safer than Candidate 1’s purely random key and still prevents duplicates server-side.
- **Optimistic update**: adopt Candidate 2’s optimistic toggle (`localStatus`) with guaranteed rollback on error. Candidate 1 omitted rollback — we include it.
- **UI state machine**: adopt Candidate 1’s explicit `idle | pending | success | error` for button visuals **and** Candidate 2’s optimistic `localStatus` for immediate clock-in/out feedback. They work together: `pending/error/success` drives button appearance; `localStatus` drives immediate clock-state preview.
- **Disabled + double-tap guard**: keep Candidate 1’s `disabled` + early-return guard (`if (pending) return`) to prevent races.
- **Error handling + UX**: use Candidate 2’s toast for errors/success, plus Candidate 1’s inline error text as fallback.

```tsx
import React, { useState, useCallback, useRef } from 'react';
import { useToast } from '@/hooks/use-toast';

type ClockStatus = 'IN' | 'OUT' | null;
type ButtonState = 'idle' | 'pending' | 'success' | 'error';

type ClockButtonProps = {
  currentStatus: ClockStatus;
  onPunch: (type: 'CLOCK_IN' | 'CLOCK_OUT', idempotencyKey: string) => Promise<void>;
  employeeId: string;
};

export function ClockButton({ currentStatus, onPunch, employeeId }: ClockButtonProps) {
  const [buttonState, setButtonState] = useState<ButtonState>('idle');
  const [localStatus, setLocalStatus] = useState<ClockStatus>(currentStatus);
  const idempotencyRef = useRef<string | null>(null);
  const { toast } = useToast();

  const generateIdempotencyKey = useCallback(() => {
    const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
    const intent = localStatus === 'OUT' || localStatus === null ? 'CLOCK_IN' : 'CLOCK_OUT';
    const randomSuffix = Math.random().toString(36).slice(2, 7);
    return `${employeeId}:${today}:${intent}:${Date.now()}:${randomSuffix}`;
  }, [employeeId, localStatus]);

  const handlePunch = useCallback(async () => {
    // Double-tap / race guard
    if (buttonState === 'pending') return;

    const intent = localStatus === 'OUT' || localStatus === null ? 'CLOCK_IN' : 'CLOCK_OUT';
    const key = generateIdempotencyKey();
    idempotencyRef.current = key;

    const prevStatus = localStatus;

    // Optimistic update
    setButtonState('pending');
    setLocalStatus(intent === 'CLOCK_IN' ? 'IN' : 'OUT');

    try {
      await onPunch(intent, key);
      setButtonState('success');
      toast({
        title: 'Success',
        description: `Clocked ${intent === 'CLOCK_IN' ? 'in' : 'out'}`,
        variant: 'default',
      });
    } catch (err) {
      // Rollback optimistic state
      setLocalStatus(prevStatus);
      setButtonState('error');
      const message = err instanceof Error ? err.message : 'Failed to record punch';
      toast({
        title: 'Error',
        description: message,
        variant: 'destructive',
      });
    } finally {
      // Auto-reset success state after brief display
      if (buttonState === 'success') {
        const timer = setTimeout(() => setButtonState('idle'), 2000);
        // cleanup handled by React; timer cleared on unmount via pattern below if needed
      }
    }
  }, [buttonState, localStatus, onPunch, generateIdempotencyKey, toast]);

  const isPending = buttonState === 'pending';
  const isSuccess = buttonState === 'success';
  const isError = buttonState === 'error';

  return (
    <div className="flex flex-col items-center gap-3">
      <button
        onClick={handlePunch}
        disabled={isPending}
        className={`
          rounded-full px-8 py-4 text-lg font-semibold transition
          ${isPending && 'opacity-60 cursor-wait'}
          ${isSuccess && 'bg-green-500 text-white'}
          ${isError && 'bg-red-500 text-white'}
          ${!isPending && !isSuccess && !isError && 'bg-blue-600 text-white hover:bg-blue-700'}
        `}
      >
        {isPending && 'กำลังบันทึก...'}
        {isSuccess && 'บันทึกแล้ว'}
        {isError && 'เกิดข้อผิดพลาด'}
        {!isPending && !isSuccess && !isError && (localStatus === 'IN' ? 'เลิกงาน' : 'ลงเวลา')}
      </button>

      {/* Optional small status hint */}
      <p className="text-sm text-muted-foreground">
        สถานะปัจจุบัน: {localStatus === 'IN' ? 'อยู่ในกะ' : localStatus === 'OUT' ? 'เลิกงาน' : 'ยังไม่ลงเวลา'}
      </p>
    </div>
  );
}
```

// workio/src/hooks/usePunch.ts
import { useMutation } from '@tanstack/react-query';
import { useToast } from '@/hooks/use-toast';

export function usePunch() {
  const { toast } = useToast();

  return useMutation({
    mutationFn: async ({
      intent,
      idempotencyKey,
      employeeId,
    }: {
      intent: 'CLOCK_IN' | 'CLOCK_OUT';
      idempotencyKey: string;
      employeeId: string;
    }) => {
      const res = await fetch('/api/punch', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          action: intent,
          idempotencyKey,
          employeeId,
          timestamp: new Date().toISOString(),
        }),
      });
      if (!res.ok) {
        const err = await res.json().catch(() => ({}));
        throw new Error(err.message || `Server error ${res.status}`);
      }
      return res.json();
    },
    onMutate: () => {
      // optimistic update can be done here if you centralize state
    },
    onError: (err, variables, context) => {
      toast({
        title: 'Error',
        description: err instanceof Error ? err.message : 'Punch failed',
        variant: 'destructive',
      });
    },
    onSuccess: () => {
      toast({
        title: 'Success',
        description: 'Punch recorded',
        variant: 'default',
      });
    },
  });
}

### Verification checklist (combined)

1. **Double-tap guard**: rapid clicks → only one network request; button disabled + shows “กำลังบันทึก...”.
2. **Slow network**: throttle to Slow 3G → button remains pending; no duplicate requests.
3. **Offline/slow + retry**: while pending, retry attempts ignored; when network returns, single request resolves.
4. **Server error**: mock 500 → button shows “เกิดข้อผิดพลาด”, toast error, localStatus rolls back to previous value.
5. **Success**: normal request → button shows “บันทึกแล้ว” ~2s, then returns to idle; toast success.
6. **Idempotency key**: confirm payload includes `idempotencyKey` (deterministic + random suffix) in devtools.
7. **Optimistic UI**: on tap, clock status preview updates immediately; on error, rolls back.
