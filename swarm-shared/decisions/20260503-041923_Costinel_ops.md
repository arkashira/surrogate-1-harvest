# Costinel / ops

## Implementation Plan — CDN-First Top-Hub Signal Panel (<2h)

### Highest-value incremental improvement
Add a resilient, zero-runtime-HF-API “Top-Hub Signal” panel to Costinel that surfaces the most-connected hub (e.g., “MOC”) with baked CDN data, robust fallback UI, and no API-rate-limit exposure during runtime.

### Rationale
- Aligns with #knowledge-rag #graph #hub patterns (top-hub insight).
- Avoids HF API 429/128-commit limits by using CDN-only fetches.
- Fits Costinel’s “Sense + Signal — ไม่ Execute” philosophy (read-only signal panel).
- Deliverable in <2h: static asset + small React panel + fallback.

---

### Concrete implementation plan

1) Pre-bake top-hub payload (one-time, done on Mac/ops side)
   - Run: `list_repo_tree` for the knowledge-rag graph folder (date-based), pick latest folder.
   - Produce `top-hub.json`:
     ```json
     {
       "hub": "MOC",
       "connections": 142,
       "lastUpdated": "2026-05-03",
       "summary": "Most-connected hub; central to cost-governance signals.",
       "related": ["cost-anomaly", "ri-optimization", "tag-compliance"]
     }
     ```
   - Upload to repo under `public/signals/top-hub.json` (or CDN path in repo). This file will be served via CDN at `https://huggingface.co/datasets/.../resolve/main/public/signals/top-hub.json` (or relative `/signals/top-hub.json` if served by app).

2) Add panel component (React, ~60 lines)
   - Location: `src/components/TopHubSignalPanel.jsx`
   - Behavior:
     - Fetch `/signals/top-hub.json` (CDN) with timeout + retry.
     - On failure, show graceful fallback (cached last-known or placeholder).
     - No runtime HF API calls.

3) Wire into dashboard
   - Import and place panel in cost dashboard sidebar or top bar (depending on layout).
   - Ensure mobile responsive.

4) Styling & UX
   - Minimal card with icon, hub name, connection count, and pill list for related signals.
   - Skeleton loader while fetching.
   - Non-blocking; failure doesn’t crash dashboard.

5) Fallback strategy
   - Bundle a minimal static fallback object in code (last known good) so UI never blanks.
   - Log errors to console (and optionally to telemetry) but do not throw.

6) Test & ship
   - Verify CDN fetch works in dev and built preview.
   - Confirm no HF API requests from browser (Network tab).
   - Run build and check bundle size change is minimal.

const FALLBACK = {
  hub: 'MOC',
  connections: null,
  lastUpdated: null,
  summary: 'Top hub information unavailable',
  related: []
};

const CDN_URL = '/signals/top-hub.json';

export default function TopHubSignalPanel() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(false);

    let cancelled = false;
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), 5000);

      .then((res) => {
        if (!res.ok) throw new Error('Failed to fetch top-hub signal');
        return res.json();
      })
      .then((json) => {
        if (!cancelled) {
          setData(json);
          setError(false);
        }
      })
      .catch((err) => {
        if (!cancelled) {
          console.warn('Top-hub CDN fetch failed, using fallback:', err);
          setError(true);
          setData(FALLBACK);
        }
      })
      .finally(() => {
        if (!cancelled) {
          setLoading(false);
          clearTimeout(timeout);
        }
