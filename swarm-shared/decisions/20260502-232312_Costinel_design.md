# Costinel / design

## Final Implementation Plan — Costinel Top-Hub Signal Endpoint

**Highest-value incremental improvement (<2h):**  
Expose a **read-only, side-effect-free** `GET /api/v1/cost-anomaly/signal/top-hub` that surfaces the most-connected hub (e.g., `"MOC"`) and related actionable signals for cost-anomaly triage. Uses existing knowledge-graph assets or a lightweight cached fallback. No writes, no external mutations, safe for dashboards and alerting.

---

### Acceptance Criteria (merged & tightened)
- `GET /api/v1/cost-anomaly/signal/top-hub` returns `200` with JSON:
  - `hub` (string): top hub key (e.g., `"MOC"`)
  - `score` (number): centrality/connection score (0–1)
  - `signals` (array): recent actionable signals related to this hub (last 24h)
  - `context` (object): `generatedAt`, `source`, `ttl`, `note`
- Strictly read-only; no writes, no external mutations, no long-running jobs.
- Fast: p95 < 200 ms via in-memory/cached snapshot.
- Graceful fallback to lightweight computed hub when graph/cache unavailable.
- Returns `503` on internal failure (never mutates state on error).
- Includes minimal unit test and OpenAPI docs.

---

### Implementation Steps (≤2h)

1. **Add route and handler** — `routes/cost_anomaly.py`
2. **Create service** — `services/top_hub_signal.py`
   - Try load cached graph snapshot (e.g., `cache/top_hub.json` or in-memory).
   - If unavailable, compute lightweight top-hub from recent cost anomalies (group by tag/region/service; pick highest frequency/score).
   - Map hub to recent actionable signals (last 24h) from `signal_store`.
3. **Add cache refresh script** (optional cron) — lightweight script to refresh snapshot every 15m.
4. **Add minimal unit test** — handler + service.
5. **Wire into main app** and verify OpenAPI generation.

@router.get("/signal/top-hub", summary="Top-hub signal (read-only)")
async def top_hub_signal():
    """
    Returns the most-connected hub and related actionable signals.
    Read-only, side-effect-free.
    """
    try:
        return await get_top_hub_signal()
    except Exception as exc:

"""
Lightweight cache refresh for top-hub snapshot.
Can be run from cron every 15m.
"""

import os
import sys
from datetime import datetime, timezone

CACHE_PATH = os.getenv("COSTINEL_TOP_HUB_CACHE", "cache/top_hub.json")

def main() -> None:
    os.makedirs(os.path.dirname(os.path.abspath(CACHE_PATH)), exist_ok=True)

    # If knowledge-rag/graph tooling is available, use it here.
    # Example: call knowledge-rag CLI or library to produce snapshot.
    # For now, use lightweight fallback.
    snapshot = {
        "hub": "MOC",
        "score": 0.92,
        "derived_from": "lightweight_fallback",
        "generatedAt": datetime.now(timezone.utc).isoformat(),
    }

    with open(CACHE_PATH, "w", encoding="utf-8") as f:
        json.dump(snapshot, f, indent=2)

    print(f"Top-hub snapshot written to {CACHE_PATH}")

if __name__ == "__main__":
    try:
        main()
    except Exception as exc:
        print(f"Failed to refresh top-hub snapshot: {exc}", file=sys.stderr)
        sys.exit(1)
