## Deployment
cf workers + Supabase free tier + GitHub Pages (docs)

## v0.1 scope
v0.1 ships in 2 weeks: Android app (Kotlin) with local encrypted SQLite; VpnService-based tunnel that wraps traffic via meek (HTTP/2 fronted) and uTLS-based TLS mimicry; pluggable transports: shadowsocks-2022-ietf-aead and a basic utls-http2 meek mode; concealment heuristics to reduce TUN/SOCKS port fingerprintability; remote config via Supabase + CF Workers edge endpoints; basic health probes and automatic fallback; no user accounts (ephemeral UUID); local-only WAUS/U counters; open-source on GitHub with CI and reproducible builds.

## Open questions
- Which pluggable transports provide the best balance of throughput/latency vs. detectability under DPI in target jurisdictions?
- Can TUN interface concealment be achieved reliably on stock Android without root or Knox (e.g., via VpnService + packet mangling and socket cloaking)?
- What is the acceptable trade-off between battery usage and active obfuscation (e.g., padding, keep-alive frequency)?
- How will remote config distribution remain available under severe censorship (CF Workers availability, fallback domains, DoH/DoT for config fetch)?
- What telemetry (if any) is strictly required to measure WAUS/U without deanonymizing users or creating honeypot risks?
- How will key rotation and credential distribution work for AEAD transports while minimizing round trips and metadata leakage?
