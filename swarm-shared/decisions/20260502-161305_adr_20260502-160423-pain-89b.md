## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a single-client stealth binary (Rust/WASM) that creates a local SOCKS5 listener, wraps traffic through a meek-like HTTP(S) tunnel with domain-fronting, applies basic DPI-evasion (padding + common TLS profiles), enforces a best-effort kill-switch (block-all on unexpected interface loss), stores session state locally in SQLite, and emits anonymized session events to a Cloudflare Worker that persists aggregates to Supabase Postgres for WAU-ZB reporting.

## Open questions
- Which DPI signatures are most prevalent in target regions and what obfuscation primitives defeat them without harming throughput?
- Can we implement a reliable client-side kill-switch on each OS (Windows/macOS/Linux/Android) without root/admin?
- How will clients fetch/rotate configs and revocation lists without revealing metadata to local network?
- What telemetry schema preserves WAU-ZB measurement while minimizing deanonymization risk?
- How do we bootstrap trust for initial config delivery (e.g., baked-in public keys or out-of-band channels)?
