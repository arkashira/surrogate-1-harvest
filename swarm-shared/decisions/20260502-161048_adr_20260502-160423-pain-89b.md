## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI stealth client (Rust) that creates a local SOCKS5 listener, obfuscates traffic with TLS-wrapped streams to a managed bridge, spawns a TUN interface for system-wide routing, and enforces a kill-switch that blocks clearnet if stealth path fails; config and anonymized session telemetry stored in Supabase Postgres; deployment via CF Workers for bridge coordination and config distribution; zero user-facing UI, zero server ownership required.

## Open questions
- Which obfuscation primitives best evade current DPI in target regions while remaining client-side implementable (e.g., shadowsocks-like AEAD over TLS vs domain fronting vs meek)?
- How will we distribute and rotate bridge endpoints without requiring user server control?
- What minimal TUN configuration avoids kernel/driver artifacts that DPI can fingerprint?
- How to implement fail-closed kill-switch across platforms (Windows/Linux/macOS) without admin rights?
- What telemetry can we collect that proves WAU-ZB without risking user deanonymization?
