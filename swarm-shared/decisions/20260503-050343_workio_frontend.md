# workio / frontend

## 1. Diagnosis

- **No client-side request deduplication**: rapid double-taps or retries while a clock request is inflight create duplicate `clock-in`/`clock-out` events that corrupt daily totals/OT/leave balances.
- **Missing optimistic UI lock**: button remains interactive during submission, inviting duplicate requests and confusing users.
- **No local idempotency key**: retries across flaky networks or LINE webhook replays are not de-duplicated on the client before hitting the backend.
- **Stale state after background/foreground or reconnect**: inflight requests can re-fire without correlation to the original intent, producing duplicates.
- **No visual feedback for submission state**: users cannot distinguish “pending” from “failed”, leading to manual retries that compound duplicates.

## 2. Proposed change

**File**: `workio/src/features/clock/ClockButton.tsx` (create or update)  
**Scope**: add idempotent submission with inflight lock, local idempotency key, optimistic state, and retry-safe UX. If file does not exist, create it and wire into the existing clock flow.

## 3. Implementation

Create `src/features/clock/ClockButton.tsx`:

```tsx
import React, { useState, useCallback, useRef } from 'react';
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { clockIn, clockOut } from '../../api/clock';
import type { ClockEvent } from '../../types/clock';

type ClockMode = 'in' | 'out';

// Stable idempotency key for the current session intent.
// Format: clock-{mode}-{date-slug}-{random-session}
function makeIdempotencyKey(mode: ClockMode): string {
  const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
  const rand = useRef(Math.random().toString(36).slice(2, 9));
  return `clock-${mode}-${today}-${rand.current}`;
}

const ClockButton: React.FC = () => {
  const queryClient = useQueryClient();
  const [inflightKey, setInflightKey] = useState<string | null>(null);

  const clockMutation = useMutation<
    ClockEvent,
    Error,
    { mode: ClockMode; idempotencyKey: string }
  >({
    mutationFn: async ({ mode, idempotencyKey }) => {
      const fn = mode === 'in' ? clockIn : clockOut;
      return fn({ idempotencyKey });
    },
    onMutate: ({ mode, idempotencyKey }) => {
      setInflightKey(idempotencyKey);
      // optimistic update: mark today as pending for UI consistency
      queryClient.setQueryData<ClockEvent | undefined>(['todayClock'], (old) =>
        old
          ? old
          : ({ mode, idempotencyKey, ts: new Date().toISOString() } as ClockEvent)
      );
    },
    onSuccess: (data) => {
      queryClient.invalidateQueries({ queryKey: ['todayClock'] });
      queryClient.invalidateQueries({ queryKey: ['clockHistory'] });
    },
    onError: () => {
      queryClient.invalidateQueries({ queryKey: ['todayClock'] });
    },
    onSettled: () => {
      setInflightKey(null);
    },
  });

  const handleClock = useCallback(
    (mode: ClockMode) => {
      // Prevent duplicate submissions while one is inflight
      if (inflightKey) return;

      const idempotencyKey = makeIdempotencyKey(mode);
      clockMutation.mutate({ mode, idempotencyKey });
    },
    [inflightKey, clockMutation]
  );

  const isPending = clockMutation.isPending || !!inflightKey;
  const lastMode = clockMutation.data?.mode;

  return (
    <div className="flex gap-3">
      <button
        onClick={() => handleClock('in')}
        disabled={isPending || lastMode === 'in'}
        className={`
          px-5 py-2 rounded font-medium transition
          ${
            isPending || lastMode === 'in'
              ? 'bg-gray-300 cursor-not-allowed'
              : 'bg-green-600 text-white hover:bg-green-700'
          }
        `}
      >
        {isPending && inflightKey?.includes('in') ? 'กำลังลงเวลา...' : 'Clock In'}
      </button>

      <button
        onClick={() => handleClock('out')}
        disabled={isPending || lastMode !== 'in'}
        className={`
          px-5 py-2 rounded font-medium transition
          ${
            isPending || lastMode !== 'in'
              ? 'bg-gray-300 cursor-not-allowed'
              : 'bg-red-600 text-white hover:bg-red-700'
          }
        `}
      >
        {isPending && inflightKey?.includes('out') ? 'กำลังลงเวลา...' : 'Clock Out'}
      </button>
    </div>
  );
};

export default ClockButton;
```

Update API helper (`src/api/clock.ts`) to accept and forward idempotency key:

```ts
import axios from 'axios';

export interface ClockEvent {
  mode: 'in' | 'out';
  idempotencyKey: string;
  ts: string;
}

export async function clockIn(payload: { idempotencyKey: string }): Promise<ClockEvent> {
  const res = await axios.post<ClockEvent>('/api/clock/in', payload, {
    headers: { 'Idempotency-Key': payload.idempotencyKey },
  });
  return res.data;
}

export async function clockOut(payload: { idempotencyKey: string }): Promise<ClockEvent> {
  const res = await axios.post<ClockEvent>('/api/clock/out', payload, {
    headers: { 'Idempotency-Key': payload.idempotencyKey },
  });
  return res.data;
}
```

If your backend expects idempotency via header or body, keep them consistent. The key ensures retries and double-taps with same key are de-duplicated server-side (already planned per prior decisions).

## 4. Verification

- **Manual double-tap test**: click Clock In rapidly 5–10 times; network tab should show exactly one POST; UI should show “กำลังลงเวลา...” then transition to disabled “Clock In” with success state.
- **Retry on failure**: throttle to Slow 3G, trigger Clock In, wait for failure, then retry; only one event should appear in the list/history.
- **Session idempotency**: refresh page mid-flight and retry; same idempotency key pattern prevents new request until prior intent resolves (backend remains source of truth).
- **Query invalidation**: after success, `todayClock` and `clockHistory` queries should refetch and reflect the single canonical event.
- **E2E smoke**: run existing frontend tests (or quick Cypress/Playwright script) that asserts `clockIn` button disabled state during request and exactly one created row via API mock.
