To address the issues identified in the diagnosis, a comprehensive solution that combines the strongest insights from multiple AI proposals is necessary. The key components of this solution include:

1. **Idempotency Enforcement**: Implement idempotency for the LINE webhook (`/webhook/line`) and clock-in/out endpoints to prevent duplicate punches due to retries on 5xx/timeouts. This can be achieved by introducing an idempotency table (`punch_idempotency`) that stores unique identifiers for each request, allowing the system to recognize and ignore duplicate requests.

2. **Tenant-Scoped Uniqueness Guard**: Enforce a tenant-scoped uniqueness guard for open punches to prevent race conditions that allow multiple concurrent “clock-in” records per employee. This involves creating a unique index on the `punches` table for the combination of `tenant_id`, `employee_id`, and `session_type` where `clocked_out_at IS NULL`, ensuring that only one open punch can exist per tenant and employee.

3. **Composite Unique Constraint**: Apply a composite unique constraint on the `punches` table for `(tenant_id, employee_id, session_type, date)` where `clocked_out_at IS NULL` to enforce database-level safety and prevent duplicate open punches.

4. **Idempotency Middleware**: Develop and apply idempotency middleware to the `/webhook/line` endpoint and clock-in/out handlers. This middleware checks for the presence of an idempotency key in the request, attempts to find a matching entry in the `punch_idempotency` table, and returns a cached response if found. If not found, it processes the request, stores the outcome, and returns the response.

5. **Safe Upsert for Punches**: Implement a safe upsert mechanism for punches that checks for the existence of an open punch before inserting a new one. If an open punch exists, it returns the existing punch; otherwise, it creates a new punch.

6. **Retry Safety and Audit Trail**: Ensure retry safety by implementing a mechanism to handle retries without causing duplicate punches. Maintain an audit trail for deduplication decisions to facilitate debugging and compliance.

```sql
CREATE TABLE IF NOT EXISTS punch_idempotency (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
```

### Idempotency Middleware

```typescript
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';

export interface IdempotencyOptions {
    ttlHours?: number;
    handler: string;
}

export function idempotency(opts: IdempotencyOptions) {
    const ttlHours = opts.ttlHours || 24;
    return async (req: Request, res: Response, next: NextFunction) => {
        // Expect clients to send Idempotency-Key
        const key = req.headers['idempotency-key'] as string;
        if (!key) return next(); // skip if not provided

        const tenantId = (req as any).tenant?.id;
        const employeeId = (req as any).employee?.id;
        if (!tenantId || !employeeId) return next();

        try {
            // Try to find existing
            const found = await pool.query(
                `SELECT response_status, response_body FROM punch_idempotency WHERE tenant_id = $1 AND idempotency_key = $2 AND handler = $3`,
                [tenantId, key, opts.handler]
            );
            if (found.rows.length > 0) {
                const { response_status, response_body } = found.rows[0];
                return res.status(response_status).json(response_body);
            }

            // Process and store outcome
            const originalJson = res.json.bind(res);
            res.json = (body: any) => {
                setImmediate(() => {
                    pool.query(
                        `INSERT INTO punch_idempotency (tenant_id, employee_id, idempotency_key, handler, response_status, response_body, expires_at)
                         VALUES ($1,$2,$3,$4,$5,$6,NOW() + ($7 || ' hours')::INTERVAL)
                         ON CONFLICT (tenant_id, idempotency_key) DO UPDATE SET response_status = EXCLUDED.response_status, response_body = EXCLUDED.response_body, expires_at = EXCLUDED.expires_at`,
                        [tenantId, employeeId, key, opts.handler, res.statusCode, body, ttlHours]
                    ).catch(() => { /* best-effort */ });
                });
                return originalJson(body);
            };
            next();
        } catch (err) {
            next(err);
        }
    };
}
```

### Punch Service with Safe Upsert

```typescript
import { pool } from '../db';

export async function upsertOpenPunch(params: {
    tenantId: string;
    employeeId: string;
    clockedInAt: Date;
    location?: { lat: number; lng: number };
    source: 'line' | 'api';
}) {
    const { tenantId, employeeId, clockedInAt, location, source } = params;
    try {
        // Try insert; if open punch exists, return existing
        const result = await pool.query(
            `INSERT INTO punches (tenant_id, employee_id, clocked_in_at, location, source)
             VALUES ($1,$2,$3,$4,$5)
             RETURNING *`,
            [tenantId, employeeId, clockedInAt, location ? JSON.stringify(location) : null, source]
        );
        return { created: true, punch: result.rows[0] };
    } catch (err: any) {
        // Unique violation from partial index: one open punch already exists
        if (err.code === '23505') {
            const existing = await pool.query(
                `SELECT * FROM punches WHERE tenant_id = $1 AND employee_id = $2 AND clocked_out_at IS NULL ORDER BY clocked_in_at DESC LIMIT 1`,
                [tenantId, employeeId]
            );
            return { created: false, punch: existing.rows[0] || null };
        }
        throw err;
    }
}
```
