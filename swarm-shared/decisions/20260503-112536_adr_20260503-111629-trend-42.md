## Deployment
cf workers (free plan) + Supabase (free tier) + optional HF space for metrics dashboard

## v0.1 scope
v0.1 in 2 weeks: single-tenant gateway on CF Workers with key-based auth; normalize OpenAI-compatible chat/completions endpoints to OpenAI and Anthropic; Postgres-backed provider/model config and key quotas; Redis-backed rate limits; p95 latency and token counters emitted to logs; contract tests with mocked provider endpoints; dashboard-free operation (logs + simple Supabase queries).

## Open questions
- Which provider auth patterns must be supported at v0.1 (api_key header, OAuth2 refresh, signed JWT)?
- What is the p95 latency SLA target (ms) and over what time window (1m/5m)?
- How will global rate limits and quotas be enforced across distributed workers (Redis vs KV vs Durable Objects)?
- Which normalization schema will be adopted (OpenAI-like JSON) and how will deviations be mapped?
- How should failover/backoff behave when a provider returns 429/5xx (retry on same or alternate provider)?
