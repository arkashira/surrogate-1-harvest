# workio / ops

## 1. Diagnosis
- No automated health-check/readiness probe for the Node backend — ops can’t tell if the container/pod is actually serving traffic (DB connected, LINE token valid, migrations applied).
- No lightweight `/healthz` or `/readyz` endpoint; current `/` route is marketing HTML, not machine-readable.
- Missing structured logging (JSON) and request-id correlation — hard to trace LINE webhook retries and duplicate-punch debugging in prod.
- No graceful shutdown handling — in-flight LINE webhook requests can be cut mid-processing during deploys, increasing duplicate risk.
- No simple uptime/process supervision in dev/prod compose — manual `npm run dev` is fragile under restarts.

## 2. Proposed change
Add a production-ready health/readiness probe and graceful shutdown to the backend:
- File: `workio/server/src/routes/health.ts` (new)
- File: `workio/server/src/index.ts` (wireup + graceful shutdown)
- File: `workio/server/src/logger.ts` (structured JSON logger)
- File: `workio/server/docker-compose.yml` (add healthcheck)

## 3. Implementation

### 3.1 Structured logger (`workio/server/src/logger.ts`)
```ts
import pino from 'pino';

const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  formatters: {
    level(label) {
      return { level: label };
    },
  },
  timestamp: pino.stdTimeFunctions.isoTime,
  redact: ['req.headers.authorization', 'req.headers.cookie'],
});

export default logger;
```

### 3.2 Health routes (`workio/server/src/routes/health.ts`)
```ts
import { Router } from 'express';
import db from '../db';
import logger from '../logger';

const router = Router();

let shuttingDown = false;
export const setShuttingDown = (value: boolean) => { shuttingDown = value; };

// Liveness: process is alive
router.get('/healthz', (_req, res) => {
  res.status(200).json({ status: 'ok', uptime: process.uptime() });
});

// Readiness: app can serve traffic (db + not shutting down)
router.get('/readyz', async (_req, res) => {
  if (shuttingDown) {
    return res.status(503).json({ status: 'shutting_down' });
  }
  try {
    await db.query('SELECT 1');
    res.status(200).json({ status: 'ready' });
  } catch (err) {
    logger.error({ err }, 'Readiness check failed');
    res.status(503).json({ status: 'db_unavailable' });
  }
});

export default router;
```

### 3.3 Wireup + graceful shutdown (`workio/server/src/index.ts`)
```ts
import express from 'express';
import cors from 'cors';
import helmet from 'helmet';
import morgan from 'morgan';
import logger from './logger';
import lineWebhook from './routes/webhook/line';
import health from './routes/health';
import { setShuttingDown } from './routes/health';

const app = express();
app.use(helmet());
app.use(cors());
app.use(express.json());
app.use(morgan('combined', { stream: { write: (msg) => logger.info(msg.trim()) } }));

// Routes
app.use('/webhook/line', lineWebhook);
app.use('/health', health);

const PORT = process.env.PORT || 3000;
const server = app.listen(PORT, () => {
  logger.info({ port: PORT }, 'Server started');
});

// Graceful shutdown
const gracefulShutdown = (signal: string) => {
  logger.info({ signal }, 'Shutdown signal received');
  setShuttingDown(true);

  // Stop accepting new connections
  server.close(() => {
    logger.info('HTTP server closed');
    process.exit(0);
  });

  // Force exit after timeout
  setTimeout(() => {
    logger.warn('Forced shutdown after timeout');
    process.exit(1);
  }, 30_000).unref();
};

process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
process.on('SIGINT', () => gracefulShutdown('SIGINT'));
```

## 4. Verification

1. Start backend:
   ```bash
   cd /opt/axentx/workio/server
   npm run dev
   ```

2. Quick checks:
   ```bash
   curl -s http://localhost:3000/health/healthz | jq
   # {"status":"ok","uptime":...}

   curl -s http://localhost:3000/health/readyz | jq
   # {"status":"ready"}
   ```

3. Simulate DB outage (stop PostgreSQL) and confirm readiness fails:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/health/readyz
   # 503
   ```

4. Graceful shutdown test:
   ```bash
   # In one terminal
   curl -s http://localhost:3000/health/readyz

   # In another, send SIGTERM to node process
   kill -SIGTERM <pid>

   # Observe logs: "Shutdown signal received", "HTTP server closed"
   # In-flight requests complete before exit
   ```

5. Docker healthcheck (if using compose):
   ```bash
   docker compose -f server/docker-compose.yml up -d
   docker compose -f server/docker-compose.yml ps
   # backend should show "healthy"
   ```
