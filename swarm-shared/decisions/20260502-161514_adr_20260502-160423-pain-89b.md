## Deployment
cf workers (edge config + KV for ephemeral profiles) + client-side Rust agent; Supabase Postgres for state; deploy agent as standalone binaries (Linux/Windows/macOS) — no server infra beyond free tiers

## v0.1 scope
v0.1 in 2 weeks: single-platform (Linux) Rust stealth agent with local SOCKS5 shim, TUN interface via boringtun, one stealth mode (TLS-in-TLS camouflage), local killswitch (nftables), opt-in WAU-ZB counters stored in local SQLite and batch-synced to Supabase via CF Worker; no GUI, CLI only; zero-config preset profiles; fail-closed default

## Open questions
- Which DPI signatures in-region are highest priority to spoof (SNI, packet lengths, timing, TTL)?
- Can we rely on Cloudflare Workers KV latency/limits for ephemeral stealth config at scale?
- What local permissions model is acceptable for TUN/killswitch on user machines (admin/root vs unprivileged fallback)?
- How will WAU-ZB telemetry be collected without deanonymization or creating detectable telemetry signatures?
- Which camouflage transport (obfs4 vs TLS-in-TLS vs QUIC mimicry) offers best evasion vs availability on restricted networks?
