## v0.1 scope
v0.1 in 2 weeks: single-binary client (Linux x86_64) that establishes an obfuscated TCP tunnel (shadowsocks-like obfuscation) over a meek-like HTTP/S fronted flow, spawns a userspace TUN via boringtun for transparent tunneling, enforces a best-effort kill-switch (drop default route via netlink), records anonymized session telemetry to local SQLite, and exports WAU-ZB summary; no server component required — users supply bridge endpoints manually.

## Open questions
- Which obfuscation fronting strategy (meek over CDN, TLS ALPN/HTTP2 mimicry, or utp-like cover) yields lowest DPI detectability in target regions?
- How will clients discover/rotate bridge endpoints without centralized infra while avoiding domain/IP blocklists?
- What minimal client-side telemetry schema preserves privacy yet reliably produces WAU-ZB signals?
- How to implement robust kill-switch across platforms (TUN drop, firewall rules, DNS sinkhole) without root/admin on consumer devices?
- Which side-channel leaks (timing, packet size, MTU, TCP options) must be normalized to avoid DPI classifiers?
