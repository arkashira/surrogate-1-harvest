## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP that connects to one AWS account and one GCP project via service credentials, discovers Lambda and Cloud Function schedules, persists them in Supabase Postgres, detects duplicates (identical cron+target), drifted schedules (cron changed since last run), and orphaned schedules (target not found), exposes a minimal FastAPI REST API for listing schedules and issues, and outputs a markdown report. No frontend, no queue, no cache, deployable as a CF Worker or local CLI.

## Open questions
- How will customers authenticate cloud credentials (AWS/GCP) — short-lived tokens vs. stored read-only keys?
- What retention window and scan frequency balance freshness vs. cost on free tiers?
- Which deduplication and drift thresholds (cron diff, target similarity) minimize false positives?
- How should orphaned schedules be defined (no target exists vs. target lacks trigger) per platform?
