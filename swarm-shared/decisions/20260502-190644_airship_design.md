# airship / design

## Incremental Improvement: Zero-config discovery CLI (`airship discover`)

**Value**: <2h implementation that operationalizes past patterns (business research + knowledge-rag + top-hub insight) into a single command that surfaces strategic context for the Arkship/Surrogate platform.

---

## Implementation Plan (≤2h)

### 1. Create CLI entrypoint (15 min)
- `/opt/axentx/airship/bin/airship` — executable wrapper, Bash shebang, sets `SHELL=/bin/bash`
- Delegates subcommands: `discover`, `research`, `hub`, `train`, `validate`

### 2. Implement `discover` command (45 min)
- Runs `granite-business-research.sh` (if present) or lightweight market scan
- Invokes knowledge-rag query for top hub (e.g., "MOC") and related docs
- Outputs:
  - Top 5 strategic insights
  - Most-connected hub + neighbors
  - Recommended next actions (training, ingestion, tuning)

### 3. Add training-safety helpers (30 min)
- Pre-list HF dataset files via `list_repo_tree` → save JSON
- Embed file list in training script to enable CDN-only fetches (bypass API rate limits)
- Validate Lightning quota before `.run()`; reuse running studios

### 4. Add HF CDN download utility (20 min)
- Downloads public dataset files via `https://huggingface.co/datasets/{repo}/resolve/main/{path}` (no auth)
- Respects HF commit cap: hash slug → pick sibling repo for writes

### 5. Add Kaggle KGAT push helper (10 min)
- Raw HTTP push with Bearer token, `isPrivate: true`, `slug` format

### 6. Tests & polish (20 min)
- Shebang + executable bits
- `chmod +x` all wrappers
- Smoke test: `./bin/airship discover`

## Code Snippets

### `/opt/axentx/airship/bin/airship`
```bash
#!/usr/bin/env bash
# airship — zero-config discovery & ops CLI
set -euo pipefail
export SHELL=/bin/bash

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

usage() {
  cat <<EOF
Usage: airship <command>

Commands:
  discover   Run business research + knowledge-rag + top-hub insight
  research   Run granite-business-research.sh + knowledge-rag
  hub        Show top-connected hub and related docs
  train      Prepare HF file list & launch Lightning training (CDN-only)
  kaggle     Push kernel via KGAT Bearer token
  validate   Validate environment (scripts, executables, tokens)
EOF
  exit 1
}

cmd="${1:-}"
shift || true

case "$cmd" in
  discover) "$ROOT/lib/discover.sh" "$@" ;;
  research) "$ROOT/lib/research.sh" "$@" ;;
  hub)      "$ROOT/lib/hub.sh" "$@" ;;
  train)    "$ROOT/lib/train.sh" "$@" ;;
  kaggle)   "$ROOT/lib/kaggle.sh" "$@" ;;
  validate) "$ROOT/lib/validate.sh" "$@" ;;
  *)        usage ;;
esac
```

### `/opt/axentx/airship/lib/train.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail
export SHELL=/bin/bash

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
HF_REPO="${HF_REPO:-datasets/your-org/surrogate-1}"
DATE_FOLDER="${1:-$(date +%Y-%m-%d)}"
OUTPUT_LIST="${ROOT}/data/file-list-${DATE_FOLDER}.json"

echo "== Surrogate-1 Training Prep (CDN-only) =="

# 1) Pre-list files once (Mac orchestration) — avoids HF API pagination/rate limits
if [[ ! -f "$OUTPUT_LIST" ]]; then
  echo "Listing HF repo tree (non-recursive) for $DATE_FOLDER..."
  python3 - <<PY
import json, os
from huggingface_hub import list_repo_tree
repo = os.getenv("HF_REPO")
folder = os.getenv("DATE_FOLDER")
tree = list_repo_tree(repo, path=folder, recursive=False)
files = [f.rfilename for f in tree if f.rfilename.endswith(('.parquet', '.jsonl'))]
with open(os.getenv("OUTPUT_LIST"), "w") as f:
    json.dump({"repo": repo, "date": folder, "files": files}, f, indent=2)
print(f"Saved {len(files)} files to {os.getenv('OUTPUT_LIST')}")
PY
fi

# 2) Embed file list into training script (Lightning)
cat > "$ROOT/train_cdn_only.py" <<'TRAIN'

from pathlib import Path

HF_REPO = os.getenv("HF_REPO")
FILE_LIST = json.loads(Path(os.getenv("FILE_LIST")).read_text())

    # CDN bypass: no Authorization header
    import requests
    url = f"https://huggingface.co/datasets/{HF_REPO}/resolve/main/{file_path}"
    r = requests.get(url, timeout=30)
    r.raise_for_status()
    return r.content

# Example: iterate and project to {prompt, response} only
for f in FILE_LIST["files"]:
    data = cdn_download(f)
    # parse parquet/jsonl -> project {prompt, response}
    # upload to Lightning dataset (CDN-only during training)
    ...
TRAIN

export FILE_LIST="$OUTPUT_LIST"
echo "Prepared train_cdn_only.py with ${OUTPUT_LIST}"

# 3) Lightning Studio reuse + quota safety
python3 - <<PY
import os
from lightning_sdk import Teamspace, Studio, Machine

team = Teamspace.current()
studio_name = "surrogate-train-l40s"
running = [s for s in team.studios if s.name == studio_name and s.status == "running"]
if running:
    studio = running[0]
    print(f"Reusing running studio: {studio.name}")
else:
    studio = Studio.create(name=studio_name, machine=Machine.L40S, create_ok=True)
    print(f"Created studio: {studio.name}")

# Run training (studio will use CDN-only script)
run = studio.run("python train_cdn_only.py", environment={"HF_REPO": os.getenv("HF_REPO")})
print(f"Started run: {run.id}")
PY
```
