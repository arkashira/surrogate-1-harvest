# workio / backend

### Final synthesized solution (correct + actionable)

**Core diagnosis (merged, prioritized)**
- Missing **database-level guardrails** is the root cause: no enforcement that a user can have at most one open punch (`clock_out_at IS NULL`) per day.
- App-layer find-then-insert is **race-prone** under concurrency/retries (webhooks, double-taps).
- No **idempotency** to deduplicate retries.
- If soft deletes (`deleted_at`) exist, uniqueness must ignore them.
- Missing precise audit columns (`created_at`, source metadata) hinders debugging.

**Single source of truth**
- Correctness is enforced **at the database**; application logic is defensive and idempotent.
- Use a **partial unique index** (not a table-level `UNIQUE` constraint on `DATE(clock_in_at)`) to enforce “one open punch per user” while ignoring soft-deleted rows and allowing multiple closed punches per day.
- Add an **idempotency key** and index to deduplicate retries safely.
- Perform clock-in/out via **transactional upsert/pessimistic locking** to avoid races.

### 1. Database schema changes
File: `server/src/db/schema.sql` (or migration)

```sql
-- 1) One open punch per user (ignore soft-deleted rows)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
ON punches (user_id)
WHERE clock_out_at IS NULL AND deleted_at IS NULL;

-- 2) Idempotency key to deduplicate retries (e.g., LINE deliveryId)
ALTER TABLE punches
ADD COLUMN IF NOT EXISTS idempotency_key VARCHAR(255);

CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency
ON punches (idempotency_key)
WHERE idempotency_key IS NOT NULL AND deleted_at IS NULL;

-- 3) Audit columns (if missing)
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS source VARCHAR(64);
```

Apply migration:

### 2. Clock punch endpoint (transactional + idempotent)
File: `server/src/routes/clock.ts` (or `ClockController.js` equivalent)

```ts
import { Router, Request, Response } from 'express';
import { pool } from '../db';

const router = Router();

/**
 * POST /clock/punch
 * Body: {
 *   userId: number,
 *   type: 'IN' | 'OUT',
 *   idempotencyKey?: string,
 *   latitude?: number,
 *   longitude?: number,
 *   source?: string
 * }
 */
router.post('/punch', async (req: Request, res: Response) => {
  const { userId, type, idempotencyKey, latitude, longitude, source } = req.body;

  if (!userId || !type) {
    return res.status(400).json({ error: 'userId and type are required' });
  }
  if (type !== 'IN' && type !== 'OUT') {
    return res.status(400).json({ error: "type must be 'IN' or 'OUT'" });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency: if key provided and exists, return existing punch
    if (idempotencyKey) {
      const existing = await client.query(
        `SELECT * FROM punches
         WHERE idempotency_key = $1 AND deleted_at IS NULL
         LIMIT 1`,
        [idempotencyKey]
      );
      if (existing.rows.length > 0) {
        await client.query('COMMIT');
        return res.json({ punch: existing.rows[0], idempotent: true });
      }
    }

    if (type === 'IN') {
      // Try insert; partial index prevents second open punch at DB level
      const insertResult = await client.query(
        `INSERT INTO punches (user_id, clock_in_at, idempotency_key, latitude, longitude, source)
         VALUES ($1, NOW(), $2, $3, $4, $5)
         ON CONFLICT DO NOTHING
         RETURNING *`,
        [userId, idempotencyKey, latitude, longitude, source]
      );

      // If insert did nothing, an open punch already exists — return it
      if (insertResult.rowCount === 0) {
        const open = await client.query(
          `SELECT * FROM punches
           WHERE user_id = $1 AND clock_out_at IS NULL AND deleted_at IS NULL
           ORDER BY clock_in_at DESC
           LIMIT 1`,
          [userId]
        );
        await client.query('COMMIT');
        return res.status(409).json({
          punch: open.rows[0] || null,
          error: 'already_clocked_in'
        });
      }

      await client.query('COMMIT');
      return res.status(201).json({ punch: insertResult.rows[0] });
    }

    // type === 'OUT'
    // Lock and close the most recent open punch for this user
    const updateResult = await client.query(
      `UPDATE punches
       SET clock_out_at = NOW(),
           latitude_out = $1,
           longitude_out = $2,
           updated_at = NOW()
       WHERE id = (
         SELECT id FROM punches
         WHERE user_id = $3 AND clock_out_at IS NULL AND deleted_at IS NULL
         ORDER BY clock_in_at DESC
         LIMIT 1
         FOR UPDATE
       )
       RETURNING *`,
      [latitude, longitude, userId]
    );

    if (updateResult.rowCount === 0) {
      await client.query('ROLLBACK');
      return res.status(404).json({ error: 'no_open_punch_to_clock_out' });
    }

    await client.query('COMMIT');
    return res.json({ punch: updateResult.rows[0] });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Clock punch error:', err);
    return res.status(500).json({ error: 'internal_server_error' });
  } finally {
    client.release();
  }
});

export default router;
```

### 3. Verification plan (concrete tests)

**Database-level enforcement**
```sql
-- Should succeed
INSERT INTO punches (user_id, clock_in_at) VALUES (1, NOW());

-- Should fail (duplicate open punch)
INSERT INTO punches (user_id, clock_in_at) VALUES (1, NOW());
```

**Idempotency enforcement**
```sql
-- First insert with key succeeds
INSERT INTO punches (user_id, clock_in_at, idempotency_key)
VALUES (1, NOW(), 'delivery-123');

-- Second insert with same key fails (unique index)
INSERT INTO punches (user_id, clock_in_at, idempotency_key)
VALUES (1, NOW(), 'delivery-123');
```
