## Deployment
cf workers

## v0.1 scope
v0.1 ships a local stealth agent (Rust CLI) that: creates an encrypted TUN interface, exposes a SOCKS5 listener, tunnels traffic via a meek/QUIC transport through Cloudflare Workers fronting, enforces a fail-closed kill-switch, stores session state in encrypted SQLite, and reports local WAU-ZB counters — all client-side with zero server infra beyond CF Workers fronting; no GUI, no account system, no logging.

## Open questions
- Which DPI-evasion transports are safest and performant in RU/CN today (meek over Cloudflare Workers vs. QUIC domain-front vs. Stun)
- How to distribute and rotate fronting endpoints without centralized infra while preserving deniability
- What local kill-switch enforcement is reliable across Windows/macOS/Linux (firewall rules vs. TUN-only routing)
- How to implement client-side WAU-ZB telemetry with privacy-preserving, tamper-resistant local counters and optional encrypted sync
- Whether SQLite encryption at rest is sufficient or if we need per-session ephemeral keys for deniability
- How to minimize binary signature detection (strip, compress, or distribute via reproducible builds/flatpak/appimage)
