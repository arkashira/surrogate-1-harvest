## Deployment
cf workers

## v0.1 scope
v0.1 ships a single-file Rust CLI client (Linux x86_64) that: establishes a QUIC-based tunnel to a Cloudflare Worker fronting domain on 443, creates a local SOCKS5 listener (hidden: no banner, ephemeral port, no filesystem socket), spawns a TUN interface via tun-rs with strict routing and fail-closed kill-switch, stores sessions/policy in local SQLite, emits opt-in encrypted telemetry hashed client-side, and blocks all non-tunnel traffic when kill-switch triggers — all zero-config and client-side only.

## Open questions
- Which pluggable transports (QUIC vs Shadowsocks vs obfs4-like padding) provide the best DPI evasion vs battery/CPU cost on mobile?
- How will clients discover and rotate fronting endpoints without a centralized server that can be blocked?
- What minimal client-side telemetry schema satisfies WAU-ZB while preserving deniability and avoiding deanonymization?
- How to implement a reliable kill-switch across desktop and mobile platforms without root/admin privileges?
- Can we use Cloudflare Workers to serve signed config and fronted routes while remaining within free-tier limits?
