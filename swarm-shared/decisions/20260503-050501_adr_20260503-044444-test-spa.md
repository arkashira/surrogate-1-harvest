## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP that connects to one AWS region and one GCP project via service credentials, lists schedules into Supabase Postgres, detects duplicates (identical cron+target) and orphans (schedules with no matching function), exposes a minimal FastAPI endpoint for reads, and outputs a simple JSON report. No frontend, no scheduler/queue, no deletions — read-only audit and status tagging.

## Open questions
- How will customers grant read-only cross-cloud credentials (assume IAM roles vs. service keys)?
- What retention and deletion policy should apply to schedule history and PII in metadata?
- Which identifier should represent a customer/project across both Lambda and Cloud Functions (org id, repo, tag)?
- How often should audits run (hourly/daily) and what rate limits/backoff are acceptable per cloud provider?
