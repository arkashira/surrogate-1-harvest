## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on CF Workers; one-time discovery for Lambda and Cloud Functions (list + describe) into Supabase Postgres; deterministic hash-based duplicate detection; basic drift detection (schedule present but target function missing) and orphan detection (schedule with no attached function); findings API endpoint and CSV export; no UI, no scheduler/queue in v0.1 (manual trigger only); OSS-only deps; free-tier Supabase + CF Workers.

## Open questions
- Which IAM scopes are minimally required to list/describe Lambda EventBridge rules and Cloud Functions triggers across accounts/projects?
- How will customers grant cross-account read access (AWS) and project-level read (GCP) without exposing credentials?
- What retention window and scan cadence balance freshness vs cost (daily vs weekly)?
- Should duplicate detection consider function code hash or only schedule+target to avoid false positives?
- How will we estimate wasted spend (invocations × duration × price) for the NSM dashboard?
