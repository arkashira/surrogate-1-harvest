## Deployment
hybrid (local CLI for analysis + CF Workers for lightweight account/auth metadata + Supabase for Postgres + optional HF space for demo)

## v0.1 scope
v0.1 in 2 weeks: CLI-only local analysis for Python/JS repos using a GGUF quantized Llama-3-8B (or smaller default), producing JSON reports with security/performance/style findings; local SQLite cache for incremental runs; optional Supabase-backed user account (via CF Workers) to store repo metadata and run history; demo HF space for onboarding/docs; no paid features yet, focus on speed (<5 min for 10k LOC) and privacy (no code leaves machine unless user opts in).

## Open questions
- Will users accept ~2–8GB local model downloads or do we ship smaller distilled models by default?
- How do we handle incremental re-analysis (file-level diffs) to keep local runs fast on large repos?
- What is the monetization trigger for converting free users to paid (runs/day, model size, team features)?
- How do we securely store and sync per-repo caches across machines without leaking proprietary data?
- Do we pre-index symbols via tree-sitter for faster semantic queries, and which languages first?
