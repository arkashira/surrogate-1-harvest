## Deployment
cf workers

## v0.1 scope
v0.1 (2 weeks): Linux x86_64 client binary (static musl) with DTLS1.3-over-QUIC disguise on UDP/443, local SOCKS5 listener hidden via port reuse and traffic-shaped to mimic HTTPS flows, userspace TUN with strict killswitch (fail-closed), encrypted client telemetry to Supabase Postgres, and CF Workers endpoint serving signed, short-lived obfs-endpoint configs; no GUI, no auto-updater, no persistence beyond config file.

## Open questions
- Which DPI signatures are most prevalent in target regions (SNI, JA3, packet size/timing, TUN MTU) and which disguise is highest ROI?
- Can we safely reuse port 443 via SO_REUSEPORT without disrupting host HTTPS services and remain stealthy?
- What is the acceptable telemetry granularity that preserves privacy while allowing WAU-ZB measurement?
- How will clients discover updated obfuscation endpoints without a hardcoded list that can be blocked?
- Which platforms are MVP (Linux desktop, Android, Windows) and do we need kernel driver fallbacks for TUN?
- How do we implement a reliable fail-closed killswitch across platforms without leaking during races?
