# workio / frontend

## 1. Diagnosis
- **No optimistic UI for clock in/out** — employees wait for round-trip to confirm state; perceived latency hurts UX on flaky mobile networks.
- **Missing local-first draft state** — unsaved leave/OT form data is lost on navigation or offline; no `localStorage` recovery.
- **Hard refetch on dashboard** — full page/data reload after clock or leave actions instead of targeted cache invalidation; wastes bandwidth and feels sluggish.
- **No skeleton / progressive loading** — dashboard and history lists show blank states while loading; harms perceived performance.
- **Missing request correlation in frontend logs** — debugging webhook-triggered UI flows is hard without trace IDs propagated from backend.

## 2. Proposed change
- **Scope**: `workio/src/features/clock/` and `workio/src/features/leave/` (create if absent) + `workio/src/api/` client layer.
- **Deliverables** (fits <2h):
  1. Add optimistic clock toggle with rollback.
  2. Add auto-save draft for leave/OT form + recovery on mount.
  3. Add lightweight request ID header plumbing and logging helper.
  4. Add skeletons for dashboard/history lists.

## 3. Implementation

### 3.1 API client with correlation ID and optimistic helpers
```ts
// workio/src/api/client.ts
import axios from 'axios';

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || '/api',
  timeout: 15_000,
});

export function requestId() {
  return crypto.randomUUID();
}

api.interceptors.request.use((cfg) => {
  const rid = requestId();
  cfg.headers['X-Request-ID'] = rid;
  // expose for logging
  (cfg as any).requestId = rid;
  return cfg;
});

api.interceptors.response.use(
  (res) => res,
  (err) => {
    const rid = err.config?.['requestId'] ?? 'unknown';
    console.error('[api]', rid, err.message, err.response?.status, err.config?.url);
    return Promise.reject(err);
  }
);
```

### 3.2 Optimistic clock hook
```tsx
// workio/src/features/clock/useOptimisticClock.ts
import { useState, useCallback } from 'react';
import { api } from '../../api/client';

export type ClockState = {
  isClockedIn: boolean;
  location?: { lat: number; lng: number };
  timestamp?: string;
  loading: boolean;
  error: string | null;
};

export function useOptimisticClock(initial: ClockState) {
  const [state, setState] = useState<ClockState>(initial);

  const toggle = useCallback(async () => {
    const optimistic = { ...state, loading: true, error: null };
    const prev = state;
    setState(optimistic);

    try {
      const pos: GeolocationPosition = await new Promise((res, rej) =>
        navigator.geolocation.getCurrentPosition(res, rej, { enableHighAccuracy: true, timeout: 8000 })
      );

      const payload = {
        clockedIn: !state.isClockedIn,
        lat: pos.coords.latitude,
        lng: pos.coords.longitude,
      };

      const res = await api.post('/clock', payload);
      setState({
        isClockedIn: res.data.clockedIn,
        location: res.data.location,
        timestamp: res.data.timestamp,
        loading: false,
        error: null,
      });
    } catch (err: any) {
      // rollback
      setState({ ...prev, loading: false, error: err.message || 'Failed' });
    }
  }, [state]);

  return { state, toggle };
}
```

### 3.3 Auto-save draft hook (leave/OT form)
```tsx
// workio/src/features/leave/useDraftForm.ts
import { useEffect, useRef } from 'react';

const STORAGE_KEY = 'workio:leave:draft';

export function useDraftForm<T extends Record<string, any>>(initial: T) {
  const [draft, setDraft] = useState<T>((() => {
    try {
      const raw = localStorage.getItem(STORAGE_KEY);
      return raw ? { ...initial, ...JSON.parse(raw) } : initial;
    } catch {
      return initial;
    }
  });

  const saveTimer = useRef<number>();

  const setField = (k: keyof T, v: T[typeof k]) => {
    const next = { ...draft, [k]: v } as T;
    setDraft(next);
    window.clearTimeout(saveTimer.current);
    saveTimer.current = window.setTimeout(() => {
      try {
        localStorage.setItem(STORAGE_KEY, JSON.stringify(next));
      } catch {}
    }, 300);
  };

  const clearDraft = () => {
    try {
      localStorage.removeItem(STORAGE_KEY);
    } catch {}
  };

  // persist on unmount safety
  useEffect(() => {
    return () => {
      try {
        localStorage.setItem(STORAGE_KEY, JSON.stringify(draft));
      } catch {}
    };
  }, [draft]);

  return { draft, setField, clearDraft };
}
```

### 3.5 Quick integration (ClockButton example)
```tsx
// workio/src/features/clock/ClockButton.tsx
import { useOptimisticClock } from './useOptimisticClock';

export function ClockButton() {
  const { state, toggle } = useOptimisticClock({
    isClockedIn: false,
    loading: false,
    error: null,
  });

  return (
    <button
      onClick={toggle}
      disabled={state.loading}
      className={`px-6 py-3 rounded text-white font-medium ${
        state.isClockedIn ? 'bg-red-500' : 'bg-green-600'
      } ${state.loading ? 'opacity-60' : ''}`}
    >
      {state.loading ? 'กำลังบันทึก...' : state.isClockedIn ? 'Clock Out' : 'Clock In'}
    </button>
  );
}
```

## 4. Verification
- **Optimistic toggle**: Click Clock In → UI flips immediately; block network in DevTools → after 8s timeout UI rolls back and shows error.
- **Draft recovery**: Fill leave form → refresh page → fields retain values; submit → draft cleared.
- **Correlation ID**: Open DevTools → Network → any `/api` request includes header `X-Request-ID`; backend logs should echo same ID.
- **Skeletons**: Navigate to dashboard with slow 3G throttle → skeleton appears before content.
- **No regressions**: Existing e2e flows (login, submit leave, clock) still pass; no console errors for missing deps.
