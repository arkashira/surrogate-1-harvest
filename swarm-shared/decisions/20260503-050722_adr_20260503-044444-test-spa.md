## v0.1 scope
v0.1 ships in 2 weeks: CLI tool (Python/Typer) + FastAPI microservice (deployed to CF Workers) that scans AWS Lambda + Cloud Functions schedules, persists normalized schedule records and findings to Supabase Postgres, detects duplicates (by hash), drifted cron/timezone/target changes, and orphaned schedules (missing function), and exports a simple JSON report + summary counts. Includes README, unit tests, and a single-tenant Supabase project for dev.

## Open questions
- Which IAM/service-account scopes are acceptable for read-only schedule enumeration (least privilege)?
- How will we authenticate cross-account (AWS) and cross-project (GCP) targets — assume OIDC/assume-role or service account keys stored in Supabase Vault/env?
- What constitutes 'drifted' — cron expression change, timezone change, target function change, or IAM binding change?
- Retention window for schedule history and findings (30/90 days?) to stay free-tier friendly
- Will customers prefer CLI-driven scans, scheduled CF Workers Cron, or on-demand API webhooks?
- How to handle eventual consistency windows in Lambda/EventBridge and Cloud Functions when scanning
