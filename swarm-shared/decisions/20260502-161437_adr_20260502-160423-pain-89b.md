## Deployment
cf workers (Durable Objects for config + KV; Workers Sites for static assets) + Supabase (Postgres + Auth) + local-first client (Rust binary) — all free-tier first

## v0.1 scope
v0.1 ships a local Rust daemon (Linux/macOS) that: creates a local SOCKS5 listener on an ephemeral port; runs a userspace TUN via smoltcp; tunnels traffic through a WireGuard-like encrypted stream wrapped in TLS 1.3 to a configurable bridge; applies domain-fronting to a small allow-listed fronting domain; enforces a fail-closed killswitch that blocks non-tunneled traffic on TUN/interface loss; emits opt-in, minimal session telemetry (blocked=true/false, timestamps, bytes) to Supabase; and provides a CLI to activate/deactivate stealth profiles with zero server-side components required (bridge endpoint supplied by user).

## Open questions
- Which obfuscation primitives are safest against current DPI in target regions (e.g., meek-http vs domain-front vs TLS-fragment vs QUIC)?
- Can we ship a pure-Rust TUN userspace stack that works cross-platform (Linux/Android/Windows/macOS) without kernel modules?
- How will we distribute and rotate fronting endpoints and certificates without central coordination that can be blocked?
- What telemetry granularity is acceptable for WAU-ZB while preserving deniability and minimizing user-identifiable data?
- How do we implement a reliable fail-closed killswitch across platforms (network namespaces, firewall rules, TUN teardown hooks)?
- Will CF Workers free tier provide sufficient Durable Objects/KV throughput for config pushes and bursty client check-ins?
