## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI client (Rust) that: creates a local SOCKS5 listener with traffic obfuscation (chacha20poly1305 + TLS-like framing), spawns and hides a TUN interface where available, enforces a strict kill-switch that blocks all non-stealth traffic on stealth failure, batches anonymized session telemetry to Supabase via CF Workers, and provides zero-config presets for restrictive-region profiles — all free-tier deployable and OSS.

## Open questions
- Which obfuscation primitives remain undetected by current DPI in target regions (e.g., TLS-in-TLS vs domain-front vs shadowsocks variants)?
- Can TUN interface creation and socket binding be hidden from host firewall/audit without root/admin on each target OS?
- What is the acceptable telemetry schema that preserves privacy (WAU-ZB) while avoiding deanonymization?
- How will CF Workers queue handle client-side task persistence when the client is offline for long periods?
- What kill-switch granularity is required (per-app vs system-wide) and can it be implemented without admin rights?
