# workio / backend

## Final Synthesis — Correct, Actionable Design

**Scope**: `server/src/routes/line/webhook.ts`, `server/src/middleware/lineAuth.ts`, `server/src/db/schema.sql`, plus transactional service layer for clock/leave/OT.

---

### 1. Diagnosis (merged & prioritized)

- **Replay/duplicate risk** (LINE retries, network timeouts) → missing idempotency and replay window.
- **Signature verification timing risk** → raw body must be preserved and verified before any parsing/mutation.
- **Multi-tenant safety** → LINE events lack explicit tenant ID; mapping `line_user_id` → tenant must be strict and scoped.
- **State races** → clock-in/out and leave/OT transitions can be violated by concurrent/retried webhook deliveries.
- **Silent failures & retry amplification** → unhandled exceptions or 5xx cause LINE retries and duplicate processing.
- **Missing audit/compliance** → no immutable log for clock/leave/OT changes (who/when/why).
- **Missing server-side validation** → time windows, location rules, schedule adherence, and policy checks must be enforced server-side.

### 3. Middleware — preserve raw body and verify signature (`server/src/middleware/lineAuth.ts`)

```ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

export function lineSignature(rawBody: Buffer, signature: string): boolean {
  if (!LINE_CHANNEL_SECRET) return false;
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody)
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

// Buffer raw body and attach to req before JSON parsing runs
export function lineRawBody() {
  return (req: Request, _res: Response, next: NextFunction) => {
    const chunks: Buffer[] = [];
    req.on('data', (chunk) => chunks.push(chunk));
    req.on('end', () => {
      (req as any).rawBody = Buffer.concat(chunks);
      next();
    });
    req.on('error', next);
  };
}
```

**Important**: Mount this middleware **before** `express.json()` for the LINE route, or use a dedicated route that buffers raw body first and then parses JSON manually to guarantee signature uses the exact bytes.
