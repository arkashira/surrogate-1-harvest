## Deployment

## v0.1 scope
v0.1 in 2 weeks: single workflow template (research->signal->risk check->order stub) with deterministic Redis Streams event log, Postgres audit trail (hash-chained), one broker/OMS mock connector, compliance guardrail engine (JSON-logic rules), edge worker on Cloudflare Workers exposing /run endpoint, local dev with Supabase + Redis, and replay CLI to rerun a run_id from stored inputs.

## Open questions
- Deterministic replay SLA: how much input snapshot and clock precision is required to guarantee bitwise replay across runs?
- Broker/OMS/EMS connector contracts: which FIX/REST/WebSocket variants must be supported in v0.1 and what are the idempotency key semantics?
- Compliance guardrail DSL: should rules be expressed in JSON logic, Rego, or a minimal Python DSL for v0.1?
- Latency budget per hop: what is the target p99 end-to-end from signal to order ack on free-tier infra?
- Immutable log storage: should we append to Postgres with hash chaining or emit to an append-only log (e.g., Cloudflare KV/R2) for tamper evidence?
