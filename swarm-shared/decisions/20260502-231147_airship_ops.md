# airship / ops

## Highest-Value Incremental Improvement (≤2h)

**Goal**: Harden `airship discover` into a deterministic, CDN-only, zero-HF-API-runtime orchestrator that eliminates HF API rate limits and PyArrow schema errors while producing reproducible file lists for downstream training.

### Why this is highest value
- Directly prevents HF 429/128-hr commit cap during ingestion
- Eliminates PyArrow `CastError` from heterogeneous repos
- Enables reproducible Lightning training via embedded file lists
- Fits within 2h (single orchestrator script + small config)

---

## Implementation Plan

1. **Create deterministic file-listing orchestrator** (`scripts/airship-discover-cdn.sh`)
   - Single `list_repo_tree` call per date folder (non-recursive)
   - Output stable JSON sorted by path
   - Include CDN URL template for each file
   - Exit codes: 0=success, 1=HF API error, 2=no files

2. **Add schema-safe ingestion guard**
   - Skip non-parquet files during listing
   - Validate filename pattern `{dataset}-{date}-{slug}.parquet`
   - Reject files with `source`/`ts` columns at parse time (project to `{prompt,response}` only)

3. **Embed file list in training config**
   - Generate `file-list-{date}.json` alongside parquet batches
   - Lightning training script loads this and uses CDN URLs exclusively

4. **Add cron-safe invocation**
   - Shebang `#!/usr/bin/env bash`
   - `chmod +x`
   - `SHELL=/bin/bash` in crontab

## Code Snippets

### `scripts/airship-discover-cdn.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

# airship discover - CDN-safe file listing orchestrator
# Usage: ./airship-discover-cdn.sh <hf_repo> <date_folder> <output_dir>
# Example: ./airship-discover-cdn.sh axentx/datasets-mirror 2026-04-29 ./out

HF_REPO="${1:-axentx/datasets-mirror}"
DATE_FOLDER="${2:-$(date +%Y-%m-%d)}"
OUTPUT_DIR="${3:-./out}"
HF_API="https://huggingface.co/api"
CDN_BASE="https://huggingface.co/datasets"

mkdir -p "${OUTPUT_DIR}"

echo "[$(date -Iseconds)] Listing ${HF_REPO}/${DATE_FOLDER} (non-recursive)..."

# Single API call - non-recursive to avoid pagination/rate limits
TREE_RESPONSE=$(curl -sSf "${HF_API}/datasets/${HF_REPO}/tree/${DATE_FOLDER}?recursive=false")

# Filter to parquet files only, stable sort by path
PARQUET_FILES=$(echo "${TREE_RESPONSE}" | jq -r '
  [.[] | select(.type=="file" and (.path | endswith(".parquet")))]
  | sort_by(.path)
  | .[]
')

if [ -z "${PARQUET_FILES}" ]; then
  echo "[$(date -Iseconds)] No parquet files found in ${DATE_FOLDER}"
  exit 2
fi

# Build deterministic file list with CDN URLs
FILE_LIST="${OUTPUT_DIR}/file-list-${DATE_FOLDER}.json"
echo "[" > "${FILE_LIST}.tmp"

FIRST=true
echo "${PARQUET_FILES}" | while IFS= read -r entry; do
  path=$(echo "${entry}" | jq -r '.path')
  size=$(echo "${entry}" | jq -r '.size // 0')
  
  # Validate filename pattern: {dataset}-{date}-{slug}.parquet
  if [[ ! "${path}" =~ ^[^/]+/${DATE_FOLDER}/[^/]+\.parquet$ ]]; then
    echo "[$(date -Iseconds)] WARN: Skipping malformed path: ${path}"
    continue
  fi
  
  cdn_url="${CDN_BASE}/${HF_REPO}/resolve/main/${path}"
  
  if [ "${FIRST}" = true ]; then
    FIRST=false
  else
    echo "," >> "${FILE_LIST}.tmp"
  fi
  
  jq -n \
    --arg p "${path}" \
    --arg u "${cdn_url}" \
    --argjson s "${size}" \
    '{path: $p, cdn_url: $u, size_bytes: $s}' >> "${FILE_LIST}.tmp"
done

echo "]" >> "${FILE_LIST}.tmp"
mv "${FILE_LIST}.tmp" "${FILE_LIST}"

echo "[$(date -Iseconds)] Wrote ${FILE_LIST}"
echo "[$(date -Iseconds)] Total files: $(jq length "${FILE_LIST}")"
echo "[$(date -Iseconds)] Total size: $(jq '[.[].size_bytes] | add / 1024 / 1024 | round' "${FILE_LIST}") MB"
```

### `scripts/project-to-pairwise.py` (schema-safe projection)
```python
#!/usr/bin/env python3
"""
Project parquet to {prompt, response} only.
Run during ingestion before upload to avoid mixed-schema issues.
"""
import pyarrow.parquet as pq
import pyarrow as pa
import pandas as pd
import sys
from pathlib import Path

def project_to_pairwise(input_path: str, output_path: str):
    """Read parquet, keep only prompt/response, write clean."""
    try:
        table = pq.read_table(input_path, columns=["prompt", "response"])
    except (KeyError, pa.ArrowInvalid):
        # Fallback: read all, select only prompt/response if present
        df = pd.read_parquet(input_path)
        if "prompt" not in df.columns or "response" not in df.columns:
            raise ValueError(f"Missing prompt/response in {input_path}")
        table = pa.Table.from_pandas(df[["prompt", "response"]])
    
    pq.write_table(table, output_path)
    print(f"Projected {input_path} -> {output_path} ({table.num_rows} rows)")

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: project-to-pairwise.py <input.parquet> <output.parquet>")
        sys.exit(1)
    project_to_pairwise(sys.argv[1], sys.argv[2])
```

### Lightning training snippet (CDN-only)
```python
# train.py - uses embedded file list, zero HF API calls during training
import json
from torch.utils.data import IterableDataset
import requests
import pyarrow.parquet as pq

class CDNParquetDataset(IterableDataset):
    def __init__(self, file_list_path):
        with open(file_list_path) as f:
            self.files = json.load(f)
    
    def __iter__(self):
        for item in self.files:
            # CDN download - no Authorization header, bypasses API rate limits
            resp = requests.get(item["cdn_url"], timeout=30)
            resp.raise_for_status()
            
            table = pq.read_table(pa.BufferReader(resp.content))
            df = table.to_pandas()
            for _, row in df.iterrows():
                yield {"prompt": row.prompt, "response": row.response}

# Usage in Lightning
# dataset = CDNParquetDataset("file-list-2026-04-29.json")
```
