## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first discovery and audit for AWS Lambda and GCP Cloud Functions that writes schedule records to Supabase Postgres; detects duplicates (same cron+owner), drifts (cron changed since last run), and orphans (schedule with no target function); outputs a summary report and marks states in the DB; deployable as a CF Worker route and runnable locally; zero external paid services.

## Open questions
- Which identity model will be used for cross-cloud reads (assumed roles/service accounts) and how are secrets/credentials stored?
- How often should discovery run (hourly vs daily) and what rate limits apply per cloud provider?
- What defines 'stale' (no last_run_at in N days vs no code change in M days) and can customers configure thresholds?
- Will findings be surfaced via CLI only, or do we need a lightweight UI/email/slack alerts on free-tier infra?
