## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI client (Linux x86_64, macOS, Windows) that: establishes a TUN interface via boringtun; tunnels traffic through a shadowsocks AEAD-obfuscated SOCKS5 proxy; shapes packets to mimic innocuous HTTPS via meek-style fronting through CF Workers; enforces a fail-closed kill-switch that blocks non-VPN traffic on stealth path failure; reports anonymized session_telemetry to Supabase Postgres; and supports zero-config bootstrap via a single invite link that resolves to a CF Worker returning per-client obfs_profile and endpoint.

## Open questions
- Which obfuscation primitives (shadowsocks variants vs. meek vs. utp) maximize undetectability under active Russian DPI while remaining performant on mobile CPUs?
- How will we distribute and rotate bridge endpoints without centralized infra that can be blocked (leverage CF Workers as ephemeral proxies or S3-like free static hosting)?
- What minimal client-side telemetry schema preserves privacy yet yields reliable WAU-ZB measurement (e.g., blinded session tokens, differential privacy noise)?
- Can we implement a robust fail-closed kill-switch on all major desktop/mobile OSes without root/admin (e.g., firewall rules via nftables/Windows Filtering Platform)?
