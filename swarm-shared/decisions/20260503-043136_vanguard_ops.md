# vanguard / ops

## 1. Diagnosis

- No content-addressed manifest per date folder → forces runtime repo re-enumeration via `list_repo_files`/`load_dataset`, triggering HF API 429s and non-reproducible epochs.
- Missing deterministic `{path, sha256}` snapshot → CDN-only fetches cannot be validated; epochs drift across runs.
- Training/ingestion scripts re-list repos on every run instead of embedding a pre-computed file list → wastes quota and amplifies rate-limit risk.
- No guard to reuse existing Lightning Studio → burns 80+ hrs/mo quota by recreating instead of reusing running studios.
- Mac/local runs may attempt model loading or heavy compute instead of delegating to Lightning/Kaggle/Cerebras → violates Mac=CLI rule and risks OOM/local resource contention.

## 2. Proposed change

Create `/opt/axentx/vanguard/ops/snapshot_and_launch.py` (single orchestration script) that:
- After rate-limit window clears, calls `list_repo_tree(path, recursive=False)` for one date folder and writes `snapshot/{date}/files.json` with `{path, sha256}` entries.
- Embeds that snapshot path into a minimal `train.py` template (or passes as CLI arg) so Lightning training does CDN-only fetches with zero API calls.
- Checks `Teamspace.studios` for an existing running studio with a deterministic name; reuses if found, else creates with `Machine.L40S` in `lightning-public-prod`.
- Enforces Mac=CLI: only orchestrates; never calls `model.from_pretrained()` or runs heavy local training.

## 3. Implementation

```bash
# /opt/axentx/vanguard/ops/snapshot_and_launch.py
#!/usr/bin/env bash
set -euo pipefail

# Config
REPO="datasets/elixir/surrogate-1"
DATE="${1:-$(date +%Y-%m-%d)}"
SNAPSHOT_DIR="/opt/axentx/vanguard/snapshots/${DATE}"
SNAPSHOT_FILE="${SNAPSHOT_DIR}/files.json"
TRAIN_SCRIPT="/opt/axentx/vanguard/train.py"
STUDIO_NAME="vanguard-train-${DATE}"

mkdir -p "${SNAPSHOT_DIR}"

echo "[*] Generating snapshot for ${REPO} date=${DATE}"

# Mac-only: use Lightning HF token if available; rely on CDN for training
python3 -c "
import os, json, hashlib
from lightning_sdk import Teamspace
from huggingface_hub import list_repo_tree, hf_hub_download

repo = '${REPO}'
date = '${DATE}'
snapshot_path = '${SNAPSHOT_FILE}'

# Single API call (non-recursive) to avoid pagination/429
tree = list_repo_tree(repo, recursive=False, path=date)
files = []
for item in tree:
    if item.type != 'file':
        continue
    # Use CDN URL for later; include sha256 placeholder (can fetch on demand)
    files.append({
        'path': item.path,
        'sha256': None,  # optional: hf_hub_download(repo, item.path, local_dir_use_symlinks=False)
        'cdn_url': f'https://huggingface.co/datasets/{repo}/resolve/main/{item.path}'
    })

os.makedirs(os.path.dirname(snapshot_path), exist_ok=True)
with open(snapshot_path, 'w') as f:
    json.dump({'repo': repo, 'date': date, 'files': files}, f, indent=2)

print(f'[+] Snapshot written to {snapshot_path} ({len(files)} files)')

# Reuse or create Lightning Studio
teamspace = Teamspace()
running = None
for s in teamspace.studios:
    if s.name == '${STUDIO_NAME}' and s.status == 'Running':
        running = s
        break

if running:
    print(f'[+] Reusing running studio: {running.name} (id={running.id})')
else:
    print(f'[+] Creating studio ${STUDIO_NAME} (L40S, lightning-public-prod)')
    # Minimal training invocation; train.py will read snapshot and use CDN-only fetches
    studio = teamspace.studios.create(
        name='${STUDIO_NAME}',
        machine='L40S',
        cloud='lightning-public-prod',
        script='${TRAIN_SCRIPT}',
        arguments=[snapshot_path],
        create_ok=True
    )
    print(f'[+] Studio created: {studio.name} (id={studio.id})')
"
```
