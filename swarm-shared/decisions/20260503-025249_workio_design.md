# workio / design

## 1. Diagnosis

- No server-side idempotency enforcement on `/webhook/line` → LINE retries on slow 2xx/timeouts create duplicate punches for the same user within the same minute.
- Clock-in/out handler lacks optimistic concurrency/dedupe check; races between duplicate webhooks can insert two punches for same user with overlapping timestamps.
- Frontend lacks client-side idempotency guard for clock-in/out: rapid clicks or retries can race and submit duplicate punches while server idempotency is resolving.
- No optimistic UI + rollback for punch actions: users see stale state until network round-trip completes, encouraging retry taps.
- Missing clear punch-state machine (PENDING → COMMITTED/FAILED) and local rollback on network failure increases perceived latency and data inconsistency.

## 2. Proposed change

- **Backend**: Add idempotency key enforcement and dedupe check in `server/src/routes/webhook/line.ts` (or equivalent) and punch service.
  - Use `X-Line-Retry-Id` (or generate from `userId + event.timestamp + event.type`) as idempotency key.
  - Store processed keys in Redis or DB with TTL (24h) and reject duplicates.
  - Add unique constraint/index on `(user_id, date, type, "minute")` or use `ON CONFLICT DO NOTHING` for punch rows.
- **Frontend**: Add optimistic punch + rollback in `src/features/punch/PunchButton.tsx` (or similar).
  - On punch click: generate local idempotency key (uuid), set punch state to PENDING, optimistically update UI.
  - Send request with `Idempotency-Key` header.
  - On success: transition to COMMITTED.
  - On failure/timeout: rollback UI and show error.
  - Debounce/disable button while PENDING.

## 3. Implementation

### Backend — server-side idempotency and dedupe

File: `server/src/routes/webhook/line.ts` (or main handler)

```ts
// server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { db } from '../db';

const IDEMPOTENCY_TTL_MS = 24 * 60 * 60 * 1000; // 24h

export async function lineWebhookIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const body = req.body;
  const events = body?.events;
  if (!Array.isArray(events) || events.length === 0) return next();

  // Build deterministic idempotency key per event (userId + timestamp + type)
  for (const ev of events) {
    const key = `line:${ev.source?.userId}:${ev.timestamp}:${ev.type}`;
    const exists = await db.redis?.get(key);
    if (exists) {
      // Already processed — skip this event but continue processing others
      continue;
    }
    // Mark processed with TTL
    await db.redis?.set(key, '1', 'PX', IDEMPOTENCY_TTL_MS);
    // Attach key to event for downstream use
    ev._idemKey = key;
  }
  next();
}
```

Add to webhook route:

```ts
// server/src/routes/webhook/line.ts
import { lineWebhookIdempotency } from '../../middleware/idempotency';
import { handleLineEvent } from '../../services/lineService';

router.post('/webhook/line', lineWebhookIdempotency, async (req, res) => {
  const events = req.body?.events || [];
  const results = await Promise.all(
    events.map((ev) => (ev._idemKey ? handleLineEvent(ev) : Promise.resolve()))
  );
  // LINE expects 200 even if some events skipped
  res.status(200).json({ ok: true });
});

Punch service guard:

```ts
// server/src/services/punchService.ts
export async function createPunch(payload: { userId: number; type: 'IN' | 'OUT'; lat?: number; lng?: number }) {
  const now = new Date();
  const date = now.toISOString().split('T')[0];
  const minuteEpoch = Math.floor(now.getTime() / 60000);

  const result = await db.query(
    `INSERT INTO punches (user_id, date, type, created_at, lat, lng)
     VALUES ($1, $2, $3, $4, $5, $6)
     ON CONFLICT DO NOTHING
     RETURNING *`,
    [payload.userId, date, payload.type, now, payload.lat, payload.lng]
  );

  if (result.rowCount === 0) {
    // Duplicate detected — return existing punch for idempotency
    const existing = await db.query(
      `SELECT * FROM punches
       WHERE user_id = $1 AND date = $2 AND type = $3
         AND EXTRACT(EPOCH FROM created_at)::bigint / 60 = $4
       ORDER BY created_at DESC LIMIT 1`,
      [payload.userId, date, payload.type, minuteEpoch]
    );
    return existing.rows[0] || null;
  }
  return result.rows[0];
}
```

### Frontend — optimistic punch + rollback

File: `src/features/punch/PunchButton.tsx`

```tsx
import { useState, useCallback } from 'react';
import { v4 as uuidv4 } from 'uuid';

type PunchState = 'IDLE' | 'PENDING' | 'COMMITTED' | 'FAILED';

export function PunchButton() {
  const [state, setState] = useState<PunchState>('IDLE');
  const [lastPunch, setLastPunch] = useState<{ id?: string; type?: 'IN' | 'OUT'; ts?: string } | null>(null);

  const punch = useCallback(async (type: 'IN' | 'OUT') => {
    if (state === 'PENDING') return;
    const idempotencyKey = uuidv4();
    const optimistic = { id: `local-${idempotencyKey}`, type, ts: new Date().toISOString() };

    setState('PENDING');
    setLastPunch(optimistic);

    try {
      const res = await fetch('/api/punches', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Idempotency-Key': idempotencyKey,
        },
        body: JSON.stringify({ type }),
      });

      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      const data = await res.json();
      setLastPunch({ id: data.id, type: data.type, ts: data.created_at });
      setState('COMMITTED');
    } catch (err) {
      // Rollback
      setLastPunch(null);
      setState('FAILED');
      // Optionally retry or show toast
      console.error('Punch failed', err);
    }
  }, [state]);

  return (
    <div className="flex gap-4">
      <button
        onClick={() => punch('IN')}
        disabled={state === 'PENDING'}
        className={`px-6 py-2 rounded font-medium ${
          state === 'PENDING'
            ? 'bg-gray-400 cursor-wait'
            : 'bg-green-600 text-white hover:bg-green-700'
        }`}
      >
        {state === 'PENDING' && lastPunch?.type === 'IN' ? 'กำลังบันทึก...' : 'Clock In'}
      </button>
      <button
        onClick={() => punch('OUT')}
        disabled={state === 'PENDING'}

          state === 'PENDING'
            ? 'bg-gray-400 cursor-wait'
            : 'bg-red-600 text-white hover:bg-red-700'
        }`}
