## Deployment
cf workers

## v0.1 scope
v0.1 ships a Linux CLI client (x86_64) that: creates a local SOCKS5 listener, transparently wraps traffic in TLS1.3 lookalike flows to a public HTTPS endpoint via CF Workers tunnel, spawns a TUN interface only when stealth path is confirmed active, and enforces a fail-closed kill-switch that blocks non-stealth traffic; includes local SQLite state, minimal DPI-obfuscation heuristics, and opt-in encrypted telemetry to Supabase for WAU-ZB measurement.

## Open questions
- Which DPI signatures are highest risk in target regions (SNI, packet lengths, timing, TUN interface names)?
- Can we rely on CF Workers for reachability tests and signed config, or do we need a peer relay fallback?
- What is the acceptable false-positive rate for the kill-switch before usability suffers?
- How will we collect WAU-ZB telemetry without deanonymizing users or creating a central log subpoena target?
- Which platforms must v0.1 support (Linux desktop only, or include Android/Windows)?
