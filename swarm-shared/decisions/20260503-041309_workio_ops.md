# workio / ops

## 1. Diagnosis

- No runtime resilience for the LINE webhook processor: crashes on transient LINE API/network errors or malformed events kill the Node process; no graceful shutdown or retry/backoff.
- Missing structured logging and correlation IDs: debugging clock-in/out failures across frontend → webhook → DB is slow; no request traceability.
- No log rotation or size guard: production container logs can fill disk; no stdout/stderr rotation or retention policy.
- Clock endpoint idempotency exists in DB (unique constraint) but webhook handler does not return early on duplicate events; retries from LINE or clients cause unnecessary DB hits and noisy logs.
- Health/liveness probes absent: Kubernetes/PM2/ops tooling cannot detect hung Node processes or unhealthy DB connections.

## 2. Proposed change

Add a lightweight ops-hardened webhook handler wrapper and health endpoints:

- `workio/server/src/lib/logger.ts` — structured logger with correlation IDs
- `workio/server/src/lib/middleware/correlation.ts` — request ID middleware
- `workio/server/src/lib/middleware/health.ts` — `/healthz` and `/readyz` endpoints
- `workio/server/src/routes/line/webhook.ts` — wrap handler with graceful error handling, idempotency short-circuit, retries for transient LINE failures
- `workio/server/src/lib/retry.ts` — small exponential backoff utility
- PM2/ecosystem config or Docker entrypoint log rotation (stdout/stderr) via `pm2` or `docker run` flags

### 3.1 Structured logger (workio/server/src/lib/logger.ts)

```ts
import pino from 'pino';

const transport = pino.transport && process.env.NODE_ENV === 'production'
  ? pino.transport({
      target: 'pino/file',
      options: { destination: 1, mkdir: true } // stdout
    })
  : undefined;

export const logger = pino(
  {
    level: process.env.LOG_LEVEL || 'info',
    timestamp: pino.stdTimeFunctions.isoTime,
    redact: ['req.headers.authorization', 'req.headers.cookie']
  },
  transport
);
```

### 3.2 Correlation middleware (workio/server/src/lib/middleware/correlation.ts)

```ts
import { Request, Response, NextFunction } from 'express';
import { logger } from '../logger';

export function correlationMiddleware(req: Request, res: Response, next: NextFunction) {
  const id = req.headers['x-request-id'] || crypto.randomUUID();
  (req as any).requestId = id;
  logger.child({ requestId: id });
  res.setHeader('x-request-id', id);
  next();
}
```

### 3.4 Retry utility (workio/server/src/lib/retry.ts)

```ts
export async function retry<T>(fn: () => Promise<T>, opts: { retries?: number; delayMs?: number } = {}): Promise<T> {
  const { retries = 3, delayMs = 300 } = opts;
  let lastErr: unknown;
  for (let i = 0; i <= retries; i++) {
    try {
      return await fn();
    } catch (err) {
      lastErr = err;
      if (i < retries) await new Promise((r) => setTimeout(r, delayMs * 2 ** i));
    }
  }
  throw lastErr;
}
```

### 3.5 Idempotent, resilient LINE webhook handler (workio/server/src/routes/line/webhook.ts)

```ts
import { Router } from 'express';
import { logger } from '../../lib/logger';
import { retry } from '../../lib/retry';
import { handleClockEvent } from '../../services/clockService';

export const lineWebhookRouter = Router();

lineWebhookRouter.post('/webhook/line', async (req, res) => {
  const requestId = (req as any).requestId || 'unknown';
  const log = logger.child({ requestId, route: 'line-webhook' });

  try {
    const events = req.body.events || [];
    if (!events.length) return res.status(200).json({ received: true });

    for (const event of events) {
      // Idempotency short-circuit: if already processed, skip
      const exists = await isDuplicateEvent(event);
      if (exists) {
        log.info({ eventId: event?.source?.userId, type: event.type }, 'duplicate event skipped');
        continue;
      }

      // Process with retry for transient failures (e.g., LINE API/network)
      await retry(async () => {
        await handleClockEvent(event);
      }, { retries: 2, delayMs: 500 });
    }

    res.status(200).json({ received: true });
  } catch (err) {
    log.error({ err, requestId }, 'webhook processing failed');
    // Return 200 to prevent LINE redelivery storms for non-retryable errors;
    // rely on idempotency + monitoring instead.
    res.status(200).json({ received: true, warning: 'processing error' });
  }
});

async function isDuplicateEvent(event: any): Promise<boolean> {
  // Minimal duplicate guard: use LINE webhook event id or timestamp+userId+type
  const id = event?.source?.userId && event?.timestamp && event.type
    ? `${event.source.userId}:${event.timestamp}:${event.type}`
    : null;
  if (!id) return false;
  // Use fast cache (Redis/Memory) or DB check; here we use a simple in-memory Set for demo.
  // Replace with persistent store in production.
  return duplicateCache.has(id) || false;
}

// Simple in-memory duplicate cache (replace with Redis or DB in prod)
const duplicateCache = new Set<string>();
// Keep recent entries for 10 minutes to absorb retries
setInterval(() => duplicateCache.clear(), 10 * 60 * 1000);
```

### 3.6 Wire up middleware and routes (workio/server/src/app.ts or index.ts)

```ts
import express from 'express';
import { correlationMiddleware } from './lib/middleware/correlation';
import { healthRouter } from './lib/middleware/health';
import { lineWebhookRouter } from './routes/line/webhook';
import { logger } from './lib/logger';

const app = express();
app.use(express.json());
app.use(correlationMiddleware);

app.use('/healthz', healthRouter);
app.use('/webhook', lineWebhookRouter);

// Graceful shutdown
process.on('SIGTERM', () => {
  logger.info('SIGTERM received, shutting down gracefully');
  // close db pools, http server, etc.
  process.exit(0);
});
process.on('SIGINT', () => {
  logger.info('SIGINT received, shutting down gracefully');
  process.exit(0);
});

export default app;
```

### 3.7 Log rotation (ops-level)

If using PM2 (ecosystem.config.js):

```js
module.exports = {
  apps: [{
    name: 'workio-server',
    script: 'dist/index.js',
    log_file: '/var/log/workio/combined.log',
    out_file: '/var/log/workio/out.log',
    error_file: '/var/log/workio/error.log',
    merge_logs: true,
    max_memory_restart: '500M',
    env_production: { NODE_ENV: 'production' }
  }]
};
```

Or run Docker with log rotation:

```bash
docker run \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=5 \
  your-workio-image
```
