## Deployment

## v0.1 scope
v0.1 in 2 weeks: deterministic workflow engine on Cloudflare Workers with Postgres (Supabase) event sourcing and Redis Streams; 3 guardrail compliance rules (exposure, price band, venue allow-list); 1 broker connector (paper/sandbox) and 1 data vendor connector (market snapshot); immutable audit log via event_log; API to submit research/signal payloads and replay any run; end-to-end latency telemetry and basic dashboard for audit trails.

## Open questions
- What deterministic event schema and versioning strategy will guarantee replayability across connector API changes?
- Which compliance expressions language balances safety, auditability, and performance (e.g., restricted Python eval vs. JSONata vs. custom AST)?
- What latency SLO (p95/p99) and budget per leg (research→signal→risk→execution) are acceptable for v0.1?
- How will secrets/credentials be stored and rotated (Supabase Vault, Workers KV, or external OSS vault) while staying free-tier friendly?
- Which brokers/OMS/EMS and data vendors are in scope for v0.1 connectors and what auth flows do they require?
