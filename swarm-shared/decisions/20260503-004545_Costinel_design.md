# Costinel / design

## Implementation Plan — Costinel “Top-Hub Signal” Card (≤2h, frontend-only)

**Scope & constraints**  
- Pure frontend, zero backend, read-only (Sense + Signal).  
- ≤2h timebox.  
- No new APIs, no infra, no auth.  
- Must render gracefully when graph data is unavailable (graceful fallback).  
- Uses existing knowledge-rag graph export pattern (assumes `/data/knowledge-rag/top-hubs.json` or similar is produced by prior pipeline).  
- Follows “Sense + Signal — ไม่ Execute” philosophy: detect top hub and surface insights; never mutate state.

---

### 1) High-value incremental improvement
Add a **Top-Hub Signal Card** to the dashboard that:
- Reads a lightweight graph summary (top hubs + edge counts) produced by the knowledge-rag pipeline.
- Picks the most-connected hub (highest degree/weight).
- Shows the hub label, connection strength, and 3 contextual signals (short summaries or linked doc titles).
- Provides a “Refresh” button to re-read the local summary (no API calls) and a timestamp of last update.
- Renders a minimal fallback when data is missing.

This directly applies pattern **#knowledge-rag #graph #hub** and gives immediate visibility into the most-connected context without any backend work.

### 3) Implementation steps (≤2h)

1. **Create data directory and sample summary** (if not present)  
   `mkdir -p public/data/knowledge-rag`  
   Place `top-hubs.json` there (CI/knowledge-rag pipeline will overwrite in prod).

2. **Add TopHubSignalCard component**  
   Location: `src/components/cards/TopHubSignalCard.tsx` (or `.jsx` if codebase uses JS).  
   Responsibilities:
   - Fetch `public/data/knowledge-rag/top-hubs.json` (relative path, no auth).
   - Select hub with max `weight` (or `degree` as tiebreaker).
   - Render card with hub label, strength meter, signals list, and timestamp.
   - Graceful fallback UI when fetch fails or data is empty.
   - Manual “Refresh” to re-fetch (client-side).

3. **Integrate into dashboard**  
   - Import and place `TopHubSignalCard` in the dashboard grid near other analytics cards.
   - Ensure responsive layout (Tailwind or existing CSS framework).

4. **Styling & polish**  
   - Use existing design tokens/colors.
   - Small inline strength bar (e.g., width proportional to normalized weight).
   - Truncate long signals with ellipsis; show full on hover/tooltip if available.

5. **Test**  
   - Verify card renders with sample data.
   - Verify fallback renders when file 404s or is malformed.
   - Verify no console errors.

type Hub = {
  id: string;

  degree: number;
  weight: number;
  signals: string[];

  generatedAt: string;
  hubs: Hub[];
};

  const [data, setData] = useState<TopHubsData | null>(null);

    setLoading(true);

        cache: "no-store",

      if (!res.ok) throw new Error(`Failed to load hub data (${res.status})`);

      setData(json);
    } catch (err: any) {
      setError(err.message || "Unknown error");
      setData(null);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    load();
  }, []);

      : 0;
