## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: a single Rust binary client (Linux x86_64) that (1) hides local SOCKS5 proxy ports using portless FUSE-based redirection and ephemeral sockets, (2) obfuscates TUN interface artifacts (MTU/routing rules) and shapes traffic to mimic TLS/HTTPS flows, (3) enforces a strict fail-closed kill-switch that blocks non-stealth traffic, (4) supports a simple meek-like egress via Cloudflare Workers (free-tier) for initial covert transport, (5) records minimal session telemetry to Supabase (blocked=false/true, no PII) and exposes WAU-ZB metric, and (6) provides zero-config UX via a single config file or env flags — no server management required.

## Open questions
- Which DPI fingerprints (SNI, ALPN, packet lengths/timing, TUN MTU artifacts) are most reliable indicators in target regions and must be prioritized for hiding?
- Can we rely solely on client-side portless/TUN-hiding techniques (e.g., FUSE SOCKS5 proxy, traffic morphing) or do we need meek frontends/CDN cover traffic for reliable egress?
- What is the acceptable trade-off between stealth and latency when applying traffic shaping and fragmentation?
- How will we perform safe, automated fail-closed kill-switch across platforms (Linux/Windows/Android) without admin/root in all cases?
- What minimal telemetry schema ensures WAU-ZB measurement without risking user deanonymization or creating logs useful to adversaries?
