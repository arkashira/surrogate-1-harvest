## Deployment
hybrid (Android client + Cloudflare Workers (free tier) + Supabase free tier; optional HuggingFace Spaces for config portal)

## v0.1 scope
v0.1 in 2 weeks: Android app (Kotlin) that establishes a concealed VPN session using VpnService with split-tunnel exclusions; domain-fronted VLESS over TLS on 443 (via Cloudflare Workers) to mask as HTTPS; local obfuscation fallback (shadowsocks-like) when direct blocked; encrypted local SQLite config + Supabase Postgres for minimal auth/telemetry; signed config bundles distributed via Cloudflare Workers; basic WAUS/U counting and kill-switch; no root/MDM required; free-tier infra only (CF Workers + Supabase free).

## Open questions
- Which obfuscation strategy (domain-front/meek/shadowtls/http2-mux) provides best undetectability on Russian DPI with minimal latency on free-tier infra?
- How will we distribute and rotate endpoint configs without centralized blocking (signed bundles vs Workers KV vs Supabase edge functions)?
- What telemetry is strictly necessary for WAUS/U while minimizing user-identifiable data under restrictive-jurisdiction legal risk?
- How do we handle split-tunnel exclusions and per-app routing without root or enterprise MDM on stock Android?
- What fallback path (local bridge mode vs proxy mode) when TUN/VPNService is restricted or detected by Knox/Play Integrity?
