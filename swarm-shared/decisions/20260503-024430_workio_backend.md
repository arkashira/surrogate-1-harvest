**Final synthesized solution (correct + actionable)**

**1. Diagnosis (merged, contradictions resolved)**
- Missing **database-level uniqueness** allows duplicates from LINE retries and client double‑taps.
- Application “last punch” checks are **racy** under concurrency (read‑then‑insert).
- No **idempotency handling** on the webhook endpoint, so retries create duplicate punches.
- No transactional/atomic upsert path for clock‑in/out; current flow is `SELECT → INSERT` without locking.
- No partial index to enforce **one open punch per employee** (`clock_out_at IS NULL`).
- No logging/monitoring to detect retries or duplicates.

**2. Required schema changes (PostgreSQL)**
```sql
-- One open punch per employee (partial unique index)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_employee
ON punches (employee_id)
WHERE clock_out_at IS NULL;

-- Idempotency table for LINE webhook retries (lightweight, FK to punches)
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  punch_id        UUID      NOT NULL REFERENCES punches(id) ON DELETE CASCADE,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_punch_idempotency_created_at
ON punch_idempotency (created_at);
```
Add a unique constraint/index on `punches.idempotency_key` only if you prefer storing it directly on punches; the separate table is preferred to avoid nullable/unique complexities and to preserve history.

**3. Webhook contract (non‑negotiable)**
- Require `X-Idempotency-Key` header on LINE webhook handler.
- If absent: `400 Bad Request`.
- Value must be stable per event (e.g., LINE `deliveryId` or deterministic hash of `timestamp + userId + eventType`).

**4. Implementation — transactional, atomic upsert + idempotency (PostgreSQL + Node/TS)**
```ts
// server/src/services/punchService.ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export async function handlePunchIdempotent(
  idempotencyKey: string,
  employeeId: string,
  action: 'in' | 'out',
  location?: { lat: number; lng: number }
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency check (lock row via SELECT FOR UPDATE to prevent races)
    const idRes = await client.query(
      `SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1 FOR UPDATE`,
      [idempotencyKey]
    );

    if (idRes.rows.length > 0) {
      const punch = await client.query('SELECT * FROM punches WHERE id = $1', [idRes.rows[0].punch_id]);
      await client.query('COMMIT');
      return punch.rows[0]; // return original result
    }

    // 2) Atomic clock-in/out
    let punch;
    if (action === 'in') {
      // Try insert; if open punch exists, return it (idempotent)
      const ins = await client.query(
        `INSERT INTO punches (employee_id, clock_in_at, clock_in_location)
         VALUES ($1, NOW(), $2)
         ON CONFLICT (employee_id)
         WHERE clock_out_at IS NULL
         DO UPDATE SET clock_in_at = EXCLUDED.clock_in_at
         RETURNING *`,
        [employeeId, location ? JSON.stringify(location) : null]
      );
      punch = ins.rows[0];
    } else {
      const upd = await client.query(
        `UPDATE punches
         SET clock_out_at = NOW(), clock_out_location = $1
         WHERE employee_id = $2 AND clock_out_at IS NULL
         RETURNING *`,
        [location ? JSON.stringify(location) : null, employeeId]
      );
      if (upd.rowCount === 0) {
        throw new Error('No open punch found for employee');
      }
      punch = upd.rows[0];
    }

    // 3) Record idempotency key
    await client.query(
      `INSERT INTO punch_idempotency (idempotency_key, punch_id) VALUES ($1, $2)`,
      [idempotencyKey, punch.id]
    );

    await client.query('COMMIT');
    return punch;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

**5. Webhook controller (minimal, safe)**
```ts
// server/src/controllers/WebhookController.js (or .ts)
const { handlePunchIdempotent } = require('../services/punchService');

async function handlePunch(req, res) {
  const idempotencyKey = req.header('X-Idempotency-Key');
  if (!idempotencyKey) {
    return res.status(400).json({ error: 'Idempotency key is required' });
  }

  const { employeeId, action, location } = req.body;
  if (!employeeId || !['in', 'out'].includes(action)) {
    return res.status(400).json({ error: 'Invalid payload' });
  }

  try {
    const punch = await handlePunchIdempotent(idempotencyKey, employeeId, action, location);
    return res.status(200).json(punch);
  } catch (err) {
    console.error('Punch processing failed', { idempotencyKey, err });
    return res.status(500).json({ error: 'Error processing punch' });
  }
}
```

**6. Verification (concrete test plan)**
- Send punch with valid `X-Idempotency-Key` → created (200).
- Replay same key → returns original punch (200), no new row.
- Omit key → 400.
- Concurrent requests with same key → only one punch row (verify via DB count).
- Clock‑in while open punch exists → returns existing open punch (idempotent).
- Clock‑out with no open punch → error.
- Partial index prevents two open punches per employee (DB constraint test).
- Monitor `punch_idempotency` and `punches` counts/logs for retries and duplicates.
