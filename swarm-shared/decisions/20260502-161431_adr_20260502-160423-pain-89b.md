## Deployment
hybrid (local binaries + CF Workers for meek fronting and config/geo fallback)

## v0.1 scope
v0.1 ships a single local CLI binary (Linux x86_64) with: 1) TUN/WireGuard via boringtun hidden behind tls-fragment and meek fronting; 2) SOCKS5-hide mode that binds to localhost-only and randomizes ephemeral ports with no discoverable listeners; 3) fail-closed killswitch that blocks non-VPN egress when stealth path fails; 4) local SQLite session store and WAZB metric export (hashed install id, opt-in); 5) CF Worker providing signed, rotating bridge configs and geo-fallback endpoints; 6) basic integration tests and smoke tests for stealth path + killswitch within 2 weeks.

## Open questions
- Which obfuscation primitives provide best evasion vs performance on low-end devices (TLS fragmentation vs meek vs httpx)?
- How will we distribute and rotate bridge endpoints without centralized infra while resisting enumeration?
- What local artifacts (process names, listening sockets, /proc/net entries) must be masked on Linux/Windows to hide SOCKS5/TUN presence?
- How to implement a reliable kill-switch across platforms that cannot leak during stealth-path failure?
- What minimal telemetry schema preserves privacy while enabling WAZB measurement and rapid iteration?
