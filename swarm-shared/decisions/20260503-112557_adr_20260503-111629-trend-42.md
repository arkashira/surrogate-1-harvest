## v0.1 scope
v0.1 ships in 2 weeks: CF Worker gateway exposing OpenAI-compatible /v1/chat/completions and /v1/completions endpoints; provider registry and model catalog in Supabase Postgres; per-provider API-key auth passthrough; weighted random routing with fallback; in-memory quota guard + Postgres write-behind for usage; idempotency via gateway_request_id; structured request_log writes to Postgres; p95 latency and success-rate dashboard via Workers Analytics; contract tests for OpenAI and Anthropic happy-path; no frontend, no billing, no multi-region.

## Open questions
- Per-provider auth rotation strategy: centralized secrets via CF Workers Secrets vs. Supabase vault with envelope encryption?
- Global idempotency scope and TTL: gateway-level dedupe vs. per-client; storage cost vs. correctness?
- SLA measurement method: synthetic canaries per provider/model vs. observed p95 from request_log?
- Quota synchronization model: optimistic local token-bucket with async Postgres write-behind vs. atomic row locks?
- Streaming response handling: how to normalize SSE/stream formats across providers while preserving backpressure and accurate token counting?
- Billing attribution model: internal cost allocation keys and export format for downstream billing systems?
