## Deployment

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research->signal->risk->execution) with Postgres-backed run state, Redis Streams event log, immutable audit events in Supabase storage, one broker connector (sandbox/paper), one risk rule, and a replay CLI to re-run any workflow_run deterministically from its event log.

## Open questions
- Which brokers/OMS/EMS and data vendors must be supported in v0.1 to prove the NSM?
- What deterministic replay SLA is required (exact bitwise replay vs logical equivalence) and what snapshot granularity is acceptable?
- What compliance event schema and retention policy satisfy near-term regulatory checks?
- What latency budget (p50/p99) from signal ingestion to execution decision is required to capture the hypothesized edge?
- How will idempotency and exactly-once semantics be enforced across connectors given at-least-once Redis Streams delivery?
- Which secrets/credential management approach will be used for broker keys while remaining cloud-free/free-tier friendly?
