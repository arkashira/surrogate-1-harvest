# workio / ops

## 1. Diagnosis

- **No idempotency key enforcement on LINE webhook** — `X-Line-Signature` is verified but not persisted; LINE retries with identical signature/payload create duplicate clock-in/out rows.
- **Non-atomic upsert path** — current flow does `findOne → if exists update else insert`; race condition between concurrent webhook deliveries produces double punches.
- **Missing unique constraint at DB level** — `active_punches` relies on app-only uniqueness (`userId + tenantId + status='active'`) without a DB constraint; duplicates can slip through under load.
- **No retry-safe idempotency table** — absence of a lightweight `webhook_idempotency` table keyed by `X-Line-Signature` or derived idempotency key prevents replay-safe processing.
- **Clock-out path not idempotent** — repeated clock-out webhooks can create multiple `status='out'` rows or fail to close the correct active punch deterministically.

## 2. Proposed change

- **File**: `/opt/axentx/workio/server/src/db/schema.sql` (add constraint + idempotency table)  
- **File**: `/opt/axentx/workio/server/src/routes/lineWebhook.ts` (atomic upsert + idempotency check)  
- **Scope**: small, surgical changes (<2h) that enforce exactly-once semantics for clock-in/out under LINE retries.

## 3. Implementation

### 3.1 DB schema changes (run once)

```sql
-- Add unique constraint to prevent duplicate active punches at DB level
ALTER TABLE active_punches
  ADD CONSTRAINT uq_active_punch_per_user_tenant
  UNIQUE (user_id, tenant_id, status)
  WHERE status = 'active';

-- Idempotency table for LINE webhook replays
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  payload_hash     VARCHAR(64)  NOT NULL,
  processed_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  created_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- Optional: fast cleanup of old keys (keep 24h)
CREATE INDEX IF NOT EXISTS idx_line_idempotency_created
  ON line_webhook_idempotency (created_at);
```

### 3.2 Idempotent webhook handler (diff-style)

```diff
// /opt/axentx/workio/server/src/routes/lineWebhook.ts
+import crypto from 'crypto';
+import { Pool } from 'pg';

+const pool = new Pool({ connectionString: process.env.DATABASE_URL });
+
+async function isAlreadyProcessed(signature: string, body: string): Promise<boolean> {
+  const hash = crypto.createHash('sha256').update(body).digest('hex');
+  const key  = `line:${signature}`;
+
+  const res = await pool.query(
+    `INSERT INTO line_webhook_idempotency (idempotency_key, payload_hash)
+     VALUES ($1, $2)
+     ON CONFLICT (idempotency_key) DO UPDATE
+       SET payload_hash = EXCLUDED.payload_hash
+     WHERE line_webhook_idempotency.payload_hash IS DISTINCT FROM EXCLUDED.payload_hash
+     RETURNING (xmax = 0) AS inserted`,
+    [key, hash]
+  );
+  // inserted = true  => first time
+  // inserted = false => duplicate (same signature+payload)
+  return res.rows[0]?.inserted === false;
+}

export async function handleLineWebhook(req, res) {
  const signature = req.get('X-Line-Signature');
  const body      = JSON.stringify(req.body);

  // Idempotency check (fast, local)
  if (await isAlreadyProcessed(signature, body)) {
    return res.status(200).json({ ok: true, reason: 'duplicate' });
  }

  // Verify signature (existing logic)
  if (!verifySignature(body, signature)) {
    return res.status(401).json({ error: 'invalid-signature' });
  }

  const { userId, type } = req.body.events[0].source; // simplified
  const tenantId = req.body.tenantId; // from JWT/middleware or lookup

  try {
    // Atomic upsert: one active punch per user+tenant
    await pool.query(
      `INSERT INTO active_punches (user_id, tenant_id, status, clock_in_at, last_event_at)
       VALUES ($1, $2, 'active', NOW(), NOW())
       ON CONFLICT (user_id, tenant_id, status)
       WHERE status = 'active'
       DO UPDATE SET last_event_at = EXCLUDED.last_event_at`,
      [userId, tenantId]
    );

    // If type === 'out', close active punch and move to history
    if (type === 'out') {
      await pool.query(
        `WITH closed AS (
           DELETE FROM active_punches
           WHERE user_id = $1 AND tenant_id = $2 AND status = 'active'
           RETURNING *
         )
         INSERT INTO punch_history (user_id, tenant_id, clock_in_at, clock_out_at)
         SELECT user_id, tenant_id, clock_in_at, NOW()
         FROM closed`,
        [userId, tenantId]
      );
    }

    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'processing-error' });
  }
}
```

## 4. Verification

1. **Idempotency smoke test**  
   Send same LINE webhook payload twice (identical `X-Line-Signature` and body) within 5s:  
   - Expect `200 OK` both times.  
   - Confirm only one row exists in `active_punches` (or one `punch_history` row if type=out).

2. **Race-condition test**  
   Fire 10 concurrent identical webhooks (curl in parallel):  
   - Expect exactly one active punch row (constraint prevents duplicates).  
   - No 500 errors.

3. **Clock-out determinism**  
   - Clock-in (type=in) → verify `active_punches` row.  
   - Clock-out (type=out) x3 rapid → verify one `punch_history` row with correct `clock_out_at` and zero active rows.

4. **DB constraint validation**  
   Manually attempt to insert two active rows for same user+tenant:  
   ```sql
   INSERT INTO active_punches (user_id, tenant_id, status, clock_in_at) VALUES (1,1,'active',NOW());
   INSERT INTO active_punches (user_id, tenant_id, status, clock_in_at) VALUES (1,1,'active',NOW());
   ```
   Second insert must fail with unique violation.
