## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on CF Workers that enumerates Lambda (EventBridge rules) and Cloud Functions (GCF) schedules into Supabase Postgres, detects exact-duplicate cron hashes and orphaned entries (target not found), exposes GET /schedules and GET /audit-runs, and produces a simple JSON report of stale schedules to retire.

## Open questions
- Which IAM/GCP service account scopes are acceptable for read-only schedule enumeration?
- How will customers authenticate (CF Access JWT vs Supabase auth) and map to tenant isolation?
- What retention and refresh cadence (daily vs hourly) balances freshness with rate limits?
- How to define 'orphaned' for each platform (no target function vs disabled trigger vs no IAM permissions)?
