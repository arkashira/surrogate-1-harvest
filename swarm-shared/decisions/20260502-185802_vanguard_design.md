# vanguard / design

## 1) Diagnosis

- No canonical discovery workflow exists to surface high-value knowledge (top-hub insights) before planning or training work.
- HF ingestion/training scripts are missing CDN-bypass strategy; they risk 429s by relying on `load_dataset`/`list_repo_files` during training.
- Lightning Studio reuse guard is absent; iterative training will burn quota by recreating studios instead of reusing running ones.
- No single entrypoint script that composes: knowledge-rag top-hub review → file-list snapshot (CDN URLs) → Lightning Studio reuse guard → training stub.
- Missing schema/projection hygiene: ingestion could accidentally persist mixed schema or attribution columns into enriched/ instead of projecting to {prompt,response} only.

## 2) Proposed change

Create `/opt/axentx/vanguard/discovery/run_discovery.sh` (executable) plus `/opt/axentx/vanguard/discovery/train_stub.py`.  
Scope: single workflow file + minimal training stub that composes the proven patterns into one reusable design artifact.

## 3) Implementation

```bash
# /opt/axentx/vanguard/discovery/run_discovery.sh
#!/usr/bin/env bash
set -euo pipefail

# --
# vanguard discovery: top-hub insight -> file-list snapshot -> studio reuse
# --

cd /opt/axentx/vanguard

# 1) Knowledge-rag: surface top-hub insight before planning
echo "== Top-hub insight (knowledge-rag) =="
if command -v knowledge-rag >/dev/null 2>&1; then
  knowledge-rag query top-hub --format md 2>/dev/null || echo "knowledge-rag unavailable; skipping"
else
  echo "knowledge-rag not installed; skipping top-hub review"
fi

# 2) HF snapshot: list one date folder and embed CDN URLs (rate-limit bypass)
HF_REPO=${HF_REPO:-"datasets/example/mirror-merged"}
DATE_DIR=${DATE_DIR:-$(date +%Y-%m-%d)}
OUT_JSON="file_list_${DATE_DIR}.json"

echo "== Listing ${HF_REPO}/${DATE_DIR} (single folder) =="
python3 - <<PY
import os, json, sys
from huggingface_hub import list_repo_tree

repo = os.getenv("HF_REPO", "datasets/example/mirror-merged")
date_dir = os.getenv("DATE_DIR", "$(date +%Y-%m-%d)")
out = "file_list_$(date_dir).json"

try:
    items = list_repo_tree(repo, path=date_dir, recursive=False)
except Exception as e:
    # Fallback: allow empty list for design/dev
    print(f"Note: HF list failed ({e}); using empty snapshot", file=sys.stderr)
    items = []

files = []
for item in items:
    if getattr(item, "type", None) == "file":
        name = getattr(item, "path", "")
        cdn_url = f"https://huggingface.co/{repo}/resolve/main/{date_dir}/{name}"
        files.append({"path": name, "cdn_url": cdn_url})

with open(out, "w") as f:
    json.dump({"repo": repo, "date_dir": date_dir, "files": files}, f, indent=2)
print(f"Wrote {len(files)} files -> {out}")
PY

# 3) Lightning Studio reuse guard
echo "== Checking Lightning Studio reuse =="
python3 - <<PY
import os
try:
    from lightning import Studio, Machine, Teamspace
    teamspace = Teamspace()
    studios = teamspace.studios
    target_name = "vanguard-l40s"
    running = [s for s in studios if s.name == target_name and s.status == "running"]
    if running:
        print(f"Reusing running studio: {running[0].id}")
        os.environ["VANGUARD_STUDIO_ID"] = running[0].id
    else:
        print("No running studio found; train_stub will create one (L40S preferred).")
except Exception as e:
    print(f"Lightning check skipped: {e}")
PY

# 4) Launch training stub (non-blocking; or run directly)
echo "== Launching training stub (CDN-only ingestion) =="
python3 train_stub.py --file-list "$OUT_JSON" --studio-name "vanguard-l40s"

echo "== Discovery run complete =="
```
