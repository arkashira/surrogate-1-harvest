## Deployment
cf workers

## v0.1 scope
CLI-only v0.1 in 2 weeks: scan Lambda (single region/account) and Cloud Functions (single project), persist schedules to Supabase Postgres, detect duplicates (same cron+target), drifted (cron changed since last scan), and orphaned (schedule exists but target function missing), output JSON report and summary counts, no UI, no scheduler, no auth beyond local credentials.

## Open questions
- How will customers grant read-only cross-account access for Lambda discovery (assume-role vs keys)?
- What is the retention window for schedule history vs findings to balance storage and auditability?
- Should scans be continuous (triggered by IaC changes) or periodic (daily/weekly) to start?
- Which identity model will the CF Workers runtime use to write to Supabase (service key vs JWT)?
