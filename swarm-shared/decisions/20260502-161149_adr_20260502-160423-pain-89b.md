## v0.1 scope
v0.1 in 2 weeks: single Rust CLI client (Linux x86_64) with stealth SOCKS5 proxy wrapper, TUN-less mode, chacha20-poly1305 obfuscation, local kill-switch (iptables), CF Workers config endpoint, Supabase telemetry for WAU-ZB, and zero-config onboarding (download-and-run).

## Open questions
- Which cover traffic profiles (TLS-in-TLS, DNS tunneling, HTTP mimicry) are least likely to trigger DPI in target regions?
- Can we distribute config/revocation via CF Workers KV with acceptable latency under throttling?
- How will clients verify integrity of configs without centralized trust (TOFU vs pinned keys)?
- What local port/socket behaviors reliably hide SOCKS5 and TUN artifacts from host-level inspection?
- How to implement fail-closed kill-switch across platforms (Windows Filtering Platform, nftables/iptables, pf) without root/admin at install?
