**Implementation Plan (≤2h)**

### Step 1: Pre-flight snapshot generation

Create a new script `bin/snapshot.sh` that generates a snapshot of the dataset file list for each date folder in the dataset repo. This script will use the `list_repo_tree` API method with `recursive=False` to avoid hitting the rate limit.

```bash
#!/bin/bash

# Set dataset repo and token
REPO="axentx/surrogate-1-training-pairs"
TOKEN="HF_TOKEN"

# Set date folder to snapshot
DATE="2026-04-29"

# Get list of files in date folder
FILES=$(curl -s -X GET \
  https://huggingface.co/api/datasets/${REPO}/resolve/main/${DATE}/ \
  -H 'Authorization: Bearer ${TOKEN}' \
  -H 'Content-Type: application/json')

# Save list to JSON file
echo "${FILES}" > "snapshot-${DATE}.json"
```

### Step 2: Update `bin/dataset-enrich.sh` to use pre-flight snapshot

Modify the `bin/dataset-enrich.sh` script to use the pre-flight snapshot instead of generating the file list at runtime.

```bash
#!/bin/bash

# Set dataset repo and token
REPO="axentx/surrogate-1-training-pairs"
TOKEN="HF_TOKEN"

  # Decode and process file

  # ...
done
```

### Step 4: Update GitHub Actions workflow to use pre-flight snapshot

Modify the GitHub Actions workflow to use the pre-flight snapshot instead of generating the file list at runtime.

```yml
name: Ingest

on:
  workflow_dispatch:
    inputs:
      DATE:
        description: 'Date folder to process'
        required: true

jobs:
  ingest:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'

      - name: Install dependencies
        run: |
          pip install -r requirements.txt

      - name: Generate pre-flight snapshot

          ./bin/snapshot.sh

          ./bin/dataset-enrich.sh
```

### Step 5: Update `bin/dataset-enrich.sh` to handle errors

Modify the `bin/dataset-enrich.sh` script to handle errors when processing the file list from the pre-flight snapshot.

