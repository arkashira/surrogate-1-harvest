## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust agent (single binary) that: opens an ephemeral local SOCKS5 listener; obfuscates egress via a CF Worker endpoint (domain-fronted, TLS with common fingerprints); provides a simple TUN mode that captures and redirects traffic through the obfuscated path; enforces a fail-closed kill-switch that blocks all non-stealth traffic on session failure; stores sessions and WAU-ZB events locally in encrypted SQLite; includes CLI commands to start/stop/status and export anonymized weekly WAU-ZB summaries.

## Open questions
- Which obfuscation primitives are least likely to trigger DPI in target regions while remaining implementable client-side only?
- Can we reliably hide TUN interfaces from local network discovery without root/admin on all target OSes?
- How will we collect WAU-ZB telemetry without deanonymizing users or creating a central point of failure?
- What kill-switch enforcement strategy works across Windows/macOS/Linux without admin/root?
- Do CF Workers free-tier limits (requests/day, CPU) suffice for expected user volume, and what is the fallback if throttled?
