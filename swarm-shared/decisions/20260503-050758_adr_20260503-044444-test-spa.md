## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI endpoints that connect to one AWS account and one GCP project, list schedules, store snapshots in Supabase Postgres, detect duplicates (same cron+owner), drifted schedules (cron change since last snapshot), and orphaned entries (disabled/removed functions), and expose a simple JSON report endpoint.

## Open questions
- Which identity/assume-role pattern will we use to read Lambda schedules without storing long-lived keys?
- How will customers map platform resources to owners (tags vs labels vs external CMDB)?
- What retention window for snapshots balances cost and drift detection accuracy?
- How will customers trigger remediations (manual review vs automated safe retirements)?
