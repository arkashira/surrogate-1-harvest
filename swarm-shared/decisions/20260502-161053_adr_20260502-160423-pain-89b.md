## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust agent (x86_64 Linux/Windows/macOS) that exposes a SOCKS5 listener, applies TLS-based meek fronting via a CF Worker, conceals local port/TUN artifacts, and enforces a fail-closed kill-switch. Includes one StealthProfile (meek + common JA3), local SQLite state, and minimal client-side telemetry reporting WAU-ZB to a CF Worker endpoint. No server-side components beyond the free-tier CF Worker.

## Open questions
- Which DPI signatures are most prevalent in target regions (SNI, JA3, packet length distributions)?
- Can we rely on CF Workers free tier for profile distribution and fronting without logging?
- What TUN setup is feasible on desktop targets (Windows/Wintun, Linux/tun, macOS/utun) without root prompts?
- How will we validate WAU-ZB client-side without leaking identifiers or enabling correlation?
- Which obfuscation primitives can be statically linked to avoid dynamic library detection?
