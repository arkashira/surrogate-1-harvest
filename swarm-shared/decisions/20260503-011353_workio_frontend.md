# workio / frontend

## 1. Diagnosis
- Clock-in button is vulnerable to double-taps and LINE webhook retries; frontend lacks request deduplication (idempotency key), so duplicate open punches can race to the backend despite DB-level fixes.
- No optimistic UI for clock in/out; users see lag and may retry, increasing duplicate risk and poor UX under flaky networks.
- Missing visual state distinction between “ready to clock in” vs “already clocked in today” on the dashboard; employees can’t tell current state at a glance.
- No local timestamp/location preview before submitting; GPS mismatches or wrong date cause rejected punches and support load.
- Error handling is generic toasts; specific failures (GPS off, outside geofence, duplicate) aren’t surfaced with recovery guidance.

## 2. Proposed change
File: `workio/src/components/ClockInOutButton.tsx` (create or update)  
Scope: single component used on dashboard + mobile header. Add idempotency key generation, optimistic UI, state-aware rendering, and inline validation.

## 3. Implementation
```tsx
// workio/src/components/ClockInOutButton.tsx
import { useState, useEffect, useCallback } from 'react';
import { useLocation } from 'react-router-dom';
import { toast } from 'sonner';

type PunchState = 'idle' | 'pending' | 'clockedIn' | 'error';

interface PunchStatus {
  todayOpen: boolean;
  lastClockIn?: string;
  lastClockOut?: string;
}

export function ClockInOutButton() {
  const [status, setStatus] = useState<PunchState>('idle');
  const [serverState, setServerState] = useState<PunchStatus>({ todayOpen: false });
  const location = useLocation();

  // Fetch today's punch state on mount
  const fetchTodayState = useCallback(async () => {
    try {
      const res = await fetch('/api/punches/today', { credentials: 'include' });
      if (!res.ok) throw new Error('Failed to fetch status');
      const data = await res.json();
      setServerState(data);
      setStatus(data.todayOpen ? 'clockedIn' : 'idle');
    } catch {
      setStatus('error');
    }
  }, []);

  useEffect(() => {
    fetchTodayState();
  }, [fetchTodayState]);

  // Generate short idempotency key (date + user derived)
  const makeIdempotencyKey = () => {
    const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
    const random = Math.random().toString(36).slice(2, 9);
    return `punch:${today}:${random}`;
  };

  const handleAction = async () => {
    if (status === 'pending') return;

    // Quick client-side validation
    if (!navigator.geolocation) {
      toast.error('Geolocation not supported. Enable location services.');
      return;
    }

    setStatus('pending');
    const idempotencyKey = makeIdempotencyKey();

    // Optimistic update
    const wasClockedIn = serverState.todayOpen;
    setServerState((s) => ({ ...s, todayOpen: !s.todayOpen }));
    setStatus(!wasClockedIn ? 'clockedIn' : 'idle');

    try {
      const res = await fetch('/api/punches', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Idempotency-Key': idempotencyKey,
        },
        body: JSON.stringify({
          type: !wasClockedIn ? 'clock_in' : 'clock_out',
          // Optional: include approximate lat/lng for client-side pre-check
          clientTime: new Date().toISOString(),
        }),
        credentials: 'include',
      });

      if (!res.ok) {
        const err = await res.json().catch(() => ({}));
        throw new Error(err.message || 'Request failed');
      }

      toast.success(!wasClockedIn ? 'Clocked in' : 'Clocked out');
      // Re-sync to authoritative state
      await fetchTodayState();
    } catch (err: any) {
      // Rollback optimistic update
      setServerState((s) => ({ ...s, todayOpen: wasClockedIn }));
      setStatus(wasClockedIn ? 'clockedIn' : 'idle');

      if (err.message?.includes('duplicate') || err.message?.includes('already open')) {
        toast.error('You already have an open punch today. Contact admin to resolve duplicates.');
      } else if (err.message?.includes('geofence') || err.message?.includes('location')) {
        toast.error('Location not allowed. Please check GPS and try again.');
      } else {
        toast.error('Action failed. Please try again.');
      }
    }
  };

  if (status === 'error') {
    return (
      <button
        onClick={fetchTodayState}
        className="px-4 py-2 rounded bg-gray-200 text-gray-700 hover:bg-gray-300"
      >
        Retry
      </button>
    );
  }

  return (
    <button
      onClick={handleAction}
      disabled={status === 'pending'}
      className={`px-5 py-2.5 rounded font-medium transition-all ${
        status === 'pending'
          ? 'bg-gray-300 cursor-wait'
          : serverState.todayOpen
          ? 'bg-red-600 hover:bg-red-700 text-white'
          : 'bg-green-600 hover:bg-green-700 text-white'
      }`}
      aria-live="polite"
    >
      {status === 'pending'
        ? 'Processing...'
        : serverState.todayOpen
        ? 'Clock Out'
        : 'Clock In'}
    </button>
  );
}
```

Add a tiny backend endpoint to serve today’s state (if not present) and accept `Idempotency-Key` header (backend should enforce uniqueness per key). If the backend already has DB-level uniqueness, this frontend change prevents most duplicates and improves UX.

## 4. Verification
- Manual: On dashboard, click “Clock In” twice quickly — second click should be ignored (disabled) and no duplicate punch created; check `/api/punches/today` returns consistent state.
- Network: Simulate offline or 500 error — optimistic UI should rollback and show error toast with guidance.
- Geolocation: Disable GPS — button should show clear error and not submit.
- Idempotency: Use same `Idempotency-Key` in curl/Postman to backend — second request should return 200 with existing punch (no duplicate).
- Visual: After clock-in, button label changes to “Clock Out” and color to red; after clock-out, returns to green “Clock In”.
