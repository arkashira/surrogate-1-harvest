# vanguard / ops

## 1. Diagnosis
- No deterministic ops launcher: missing a single `ops/run` entrypoint to invoke Lightning training, HF ingestion, or cron jobs with correct env/shell.
- Cron/wrapper scripts risk the “Bash shebang + executable” pattern failures (seen in opus-pr-reviewer and active-learning) — no guardrails.
- HF API rate-limit exposure: training scripts likely call `list_repo_files`/`load_dataset` and will hit 429; no CDN-first file-list strategy baked in.
- No studio reuse logic: Lightning training will create new studios and burn quota instead of reusing running ones.
- No local→remote compute boundary: risk of Mac accidentally running `model.from_pretrained()` or heavy ingestion locally.

## 2. Proposed change
Add `/opt/axentx/vanguard/ops/run` (Bash orchestrator) + minimal helpers:
- Shebang + `set -euo pipefail`, `SHELL=/bin/bash`
- Subcommands: `train`, `ingest`, `cron-ensure`
- Implements:
  - CDN-first file-list fetch (once) → JSON embedded for Lightning training
  - Studio reuse (list → reuse if running)
  - Mac guard (block heavy local model/ingest calls)
- Make executable; add a systemd-timer/cron stub that invokes via `bash ops/run cron-ensure`.

## 3. Implementation
Create/replace files:

```bash
# /opt/axentx/vanguard/ops/run
#!/usr/bin/env bash
# Guard: enforce Bash + fail-fast
set -euo pipefail
SHELL=/bin/bash
export SHELL

readonly PROJECT_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
cd "$PROJECT_ROOT"

log() { echo "[$(date -Iseconds)] [ops/run] $*" >&2; }

# -- Mac guard --
if [[ "$(uname)" == "Darwin" ]]; then
  MAC_MODE=1
else
  MAC_MODE=0
fi

mac_guard_block_heavy() {
  if (( MAC_MODE )); then
    log "ERROR: Mac cannot run heavy local model/ingest workloads (architecture rule)."
    exit 1
  fi
}

# -- Lightning helpers --
lightning_reuse_or_start() {
  local studio_name="${1:-vanguard-train}"
  local machine="${2:-L40S}"
  python3 - "$studio_name" "$machine" <<'PY'
import os, sys
from lightning_sdk import Teamspace, Studio, Machine
team = Teamspace.current()
studio_name = sys.argv[1]
machine = sys.argv[2]
running = [s for s in team.studios if s.name == studio_name and s.status == "Running"]
if running:
    print(f"reuse:{running[0].id}")
    sys.exit(0)
# try to find stopped and restart, else create
stopped = [s for s in team.studios if s.name == studio_name and s.status == "Stopped"]
if stopped:
    stopped[0].start(machine=Machine(machine))
    print(f"restart:{stopped[0].id}")
else:
    s = team.studios.create(name=studio_name, machine=Machine(machine), create_ok=True)
    print(f"create:{s.id}")
PY
}

# -- CDN-first file list --
fetch_file_list() {
  local repo="${1:-datasets/example}"
  local folder="${2:-batches/mirror-merged}"
  local out="${3:-ops/file_list.json}"
  python3 - "$repo" "$folder" "$out" <<'PY'
import json, os, sys
from huggingface_hub import list_repo_tree
repo = sys.argv[1]
folder = sys.argv[2]
out = sys.argv[3]
# single non-recursive call; training will use CDN URLs
items = list_repo_tree(repo=repo, path=folder, recursive=False)
files = [f.rfilename for f in items if f.type == "file"]
os.makedirs(os.path.dirname(out), exist_ok=True)
with open(out, "w") as f:
    json.dump({"repo": repo, "folder": folder, "files": files}, f)
print(f"wrote {len(files)} files -> {out}")
PY
}

# -- Subcommands --
cmd="${1:-help}"
case "$cmd" in
  train)
    mac_guard_block_heavy
    log "Ensuring Lightning studio (reuse preferred)..."
    lightning_reuse_or_start "vanguard-train" "L40S"
    log "Fetching CDN file list (once)..."
    fetch_file_list "datasets/example" "batches/mirror-merged" "ops/file_list.json"
    log "Submitting Lightning training (CDN-only data path)..."
    # Example: launch training that reads ops/file_list.json and uses CDN URLs
    python3 -m vanguard.train --file-list ops/file_list.json --remote-run || true
    ;;
  ingest)
    mac_guard_block_heavy
    log "Running HF ingestion (CDN-first)..."
    python3 -m vanguard.ingest || true
    ;;
  cron-ensure)
    log "Ensuring cron wrappers are executable and have correct shebangs..."
    find "$PROJECT_ROOT" -type f -name "*.sh" -print0 | while IFS= read -r -d '' f; do
      if ! head -1 "$f" | grep -qE '^#!/usr/bin/env bash|^#!/bin/bash'; then
        log "WARN: missing Bash shebang in $f — adding"
        sed -i '1i#!/usr/bin/env bash' "$f"
      fi
      chmod +x "$f"
    done
    # Ensure crontab uses bash (user-level example)
    (crontab -l 2>/dev/null || true) | grep -q 'SHELL=/bin/bash' || (crontab -l 2>/dev/null; echo 'SHELL=/bin/bash') | crontab -
    log "Cron wrappers fixed."
    ;;
  help|*)
    echo "Usage: ops/run {train|ingest|cron-ensure|help}"
    exit 0
    ;;
esac
```

Make executable:
```bash
chmod +x /opt/axentx/vanguard/ops/run
```

## 4. Verification
1. Shebang & permissions:
   - `head -1 ops/run` → `#!/usr/bin/env bash`
   - `test -x ops/run && echo OK`
2. Mac guard:
   - On Mac: `./ops/run train` → exits with error message (blocks heavy local work).
3. Cron-ensure:
   - `./ops/run cron-ensure` → finds/repairs any `*.sh` missing shebangs and ensures user crontab contains `SHELL=/bin/bash`.
4. File-list (dry-run):
   - `python3 -c "import json; print(json.load(open('ops/file_list.json'))['files'][:3])"` after `./ops/run train` (or run fetch standalone) → shows non-empty list.
5. Lightning reuse (dry-run):
   - Run `python3` and import the reuse snippet; confirm it lists studios without error (requires Lightning token in env).
6. Idempotency:
   - Run `./ops/run cron-ensure` twice → second run makes no changes and exits cleanly.
