### 1. Diagnosis (consolidated)
- No client-side idempotency key on punch submissions → double-taps and LINE webhook replays create duplicate punches.
- Clock-in/out button has no optimistic state or debounce → users can spam and queue multiple requests.
- No local validation of valid state transitions (e.g., clock-in while already clocked-in) → invalid requests reach the server and confuse UX.
- Missing visual feedback during submission → users perceive taps as failed and retry.
- No persisted “last punch” client cache → redundant requests and poor offline/slow-network UX.

---

### 2. Proposed change
**File:** `workio/src/components/PunchButton.tsx` (create or replace)  
**Scope:** Single primary punch action component used on dashboard/home.  
**Add:**  
- Idempotency key per logical punch (stable, date+type based).  
- Optimistic UI + disabled/loading state.  
- Debounce + in-flight mutation guard.  
- Local state-transition validation.  
- Persistent last-punch guard in `localStorage`.

### 3. Implementation

```tsx
// workio/src/components/PunchButton.tsx
import { useState, useEffect, useCallback } from 'react';
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { Check, Clock } from 'lucide-react';

// Stable idempotency key: one per logical punch (date + type)
function makeIdempotencyKey(type: 'in' | 'out'): string {
  const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
  return `punch:${today}:${type}`;
}

// Guard: prevent invalid transitions
function canPunch(action: 'in' | 'out', lastPunch: { type: 'in' | 'out'; ts: string } | null): boolean {
  if (!lastPunch) return action === 'in';
  if (action === 'in') return lastPunch.type === 'out';
  return lastPunch.type === 'in';
}

interface PunchPayload {
  type: 'in' | 'out';
  timestamp: string; // ISO
  location?: { lat: number; lng: number };
}

interface Props {
  tenantId: string;
  employeeId: string;
}

export function PunchButton({ tenantId, employeeId }: Props) {
  const queryClient = useQueryClient();
  const [lastPunch, setLastPunch] = useState<{ type: 'in' | 'out'; ts: string } | null>(null);

  // Load last punch from localStorage on mount
  useEffect(() => {
    try {
      const raw = localStorage.getItem(`workio:lastPunch:${employeeId}`);
      if (raw) {
        const parsed = JSON.parse(raw) as { type: 'in' | 'out'; ts: string };
        setLastPunch(parsed);
      }
    } catch {
      // ignore corrupted storage
    }
  }, [employeeId]);

  const punchMutation = useMutation({
    mutationFn: async (punch: PunchPayload) => {
      const res = await fetch('/api/punches', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          // Idempotency header for backend deduplication
          'Idempotency-Key': makeIdempotencyKey(punch.type),
        },
        body: JSON.stringify({ tenantId, employeeId, ...punch }),
      });

      if (!res.ok) {
        const err = await res.text();
        throw new Error(err || 'Punch failed');
      }

      return res.json();
    },
    onSuccess: (data, variables) => {
      const next = { type: variables.type, ts: variables.timestamp };
      setLastPunch(next);
      try {
        localStorage.setItem(`workio:lastPunch:${employeeId}`, JSON.stringify(next));
      } catch {
        // ignore storage failures
      }
      queryClient.invalidateQueries({ queryKey: ['punches'] });
      queryClient.invalidateQueries({ queryKey: ['today-summary'] });
    },
  });

  const attemptPunch = useCallback(
    async (type: 'in' | 'out') => {
      // Guard: prevent rapid re-entry while mutation is in-flight
      if (punchMutation.isPending) return;

      // Local transition validation
      if (!canPunch(type, lastPunch)) {
        // Optionally show toast: "You are already clocked in/out"
        return;
      }

      const now = new Date().toISOString();
      const payload: PunchPayload = {
        type,
        timestamp: now,
        // location: ... (optional)
      };

      // Optimistic update (optional): immediately reflect UI state
      const previous = lastPunch;
      const optimistic = { type, ts: now };
      setLastPunch(optimistic);
      try {
        localStorage.setItem(`workio:lastPunch:${employeeId}`, JSON.stringify(optimistic));
      } catch {
        // ignore
      }

      punchMutation.mutate(payload, {
        onError: () => {
          // Rollback optimistic update on error
          setLastPunch(previous);
          try {
            if (previous) {
              localStorage.setItem(`workio:lastPunch:${employeeId}`, JSON.stringify(previous));
            } else {
              localStorage.removeItem(`workio:lastPunch:${employeeId}`);
            }
          } catch {
            // ignore
          }
        },
      });
    },
    [punchMutation, lastPunch, employeeId]
  );

  const isClockedOut = !lastPunch || lastPunch.type === 'out';
  const isPending = punchMutation.isPending;

  return (
    <button
      onClick={() => attemptPunch(isClockedOut ? 'in' : 'out')}
      disabled={isPending || !canPunch(isClockedOut ? 'in' : 'out', lastPunch)}
      className="inline-flex items-center gap-2 rounded-lg bg-blue-600 px-5 py-3 text-white shadow transition hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed"
    >
      {isPending ? (
        <>
          <Clock className="animate-spin" size={18} />
          Submitting...
        </>
      ) : isClockedOut ? (
        <>
          <Clock size={18} />
          Clock In
        </>
      ) : (
        <>
          <Check size={18} />
          Clock Out
        </>
      )}
    </button>
  );
}
```

### 4. Key correctness + actionability choices (resolved)
- **Idempotency key strategy**  
  Use a stable, date+type key (`punch:YYYY-MM-DD:in`) rather than a random per-mount key. This prevents duplicates across refreshes and replays while allowing a legitimate same-day correction to be handled by the server (if business rules permit) via a new date or explicit override. The key is sent in the `Idempotency-Key` header so the server can dedupe.

- **Optimistic UI + rollback**  
  Apply optimistic update immediately for responsiveness, but rollback on mutation error to keep client state consistent with server truth.

- **Debounce vs in-flight guard**  
  Instead of a time-based debounce that can delay valid taps, use an in-flight guard (`punchMutation.isPending`) plus disabled button state. This prevents spam without introducing artificial latency; combined with idempotency, it is safer and more responsive.

- **Local transition validation**  
  `canPunch` blocks invalid transitions client-side to reduce noise and improve UX. This complements (does not replace) server-side validation.

- **Persisted last punch**  
  Stored under a user-scoped key (`workio:lastPunch:<employeeId>`) to short-circuit redundant requests and restore state across reloads.

- **Query invalidation**  
  Invalidate `punches` and `today-summary` on success so lists and summaries stay correct.
