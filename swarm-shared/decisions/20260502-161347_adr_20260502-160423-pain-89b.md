## Deployment
cf workers (edge config + lightweight API shim) + Supabase (Postgres + auth) — free-tier first; client binary distributed directly (no server hosting costs)

## v0.1 scope
v0.1 ships a minimal Rust CLI agent (single binary) that: creates/masks a local SOCKS5 listener, establishes a TUN interface with artifact masking, applies basic DPI-evasion traffic shaping (padding/timing), enforces a kill-switch on failure, stores local config and short-lived tokens, and emits encrypted local WAU-ZB events; config distribution and user state are stored in Supabase Postgres and distributed via CF Workers edge KV (free-tier). Target: stealthy client-side deployment with zero server hosting costs, usable in 2 weeks.

## Open questions
- Which DPI fingerprints are highest risk in target regions and what innocuous cover protocols are safest to mimic?
- Can TUN interface artifacts be masked reliably on Windows/macOS/Linux without kernel drivers?
- How will client-side telemetry (WAU-ZB) be collected without creating new identifying signals?
- What is the acceptable fail-closed behavior when stealth path degrades (block all traffic vs fallback to local-only)?
- How will config updates be delivered and verified to resist tampering in restrictive networks?
