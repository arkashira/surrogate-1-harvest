# Costinel / design

## Final Implementation — `GET /api/v1/cost-anomaly/signal/top-hub`

**Goal (≤2h)**  
Expose a **read-only** endpoint that returns the top hub(s) in Costinel’s cost-anomaly knowledge graph. Uses existing graph assets; zero side effects; immediately usable by dashboards/alerts.

---

### 2) Architecture & data flow

```
Client
  │
  ▼
Router (GET /top-hub)
  │
  ├─> validate & bind query (for_date, top_n)
  │
  └─> CostAnomalySignalApp.top_hub(...)
        │
        ├─> KnowledgeGraphRepo.top_hubs(for_date, top_n)
        │       └─> Graph query (Cypher/SQL/in-memory)
        │
        └─> Presenter → GraphHubDTO[]
```

- **Read-only**. No writes, no jobs, no cache invalidation required.
- **Date default**: UTC today.
- **top_n clamp**: 1–10 (enforced at API layer).
- **Idempotent & safe**.

```python
from pydantic import BaseModel, Field
from datetime import date
from typing import List

class GraphHubDTO(BaseModel):
    hub_id: str = Field(..., description="Unique hub identifier (e.g., MOC)")
    label: str = Field(..., description="Human-readable label")
    hub_type: str = Field(..., description="Semantic type (team, system, org, region)")
    score: float = Field(..., ge=0, description="Connectivity/signal score")
    for_date: date = Field(..., description="Date this hub is top for")
    related_signals: List[str] = Field(..., max_items=5, description="Related anomaly keys")

class TopHubQueryParams(BaseModel):
    for_date: date | None = Field(
        None, description="Date (YYYY-MM-DD), defaults to today UTC"
    )
    top_n: int = Field(1, ge=1, le=10, description="Number of top hubs to return (1-10)")
```

**b) Repository (`app/repositories/knowledge_graph_repo.py`)**

```python
from datetime import date
from typing import List, Dict
from app.api.v1.cost_anomaly.schemas import GraphHubDTO

class KnowledgeGraphRepo:
    def __init__(self, graph_client):
        self.g = graph_client  # e.g., Neo4j driver / existing ORM / networkx wrapper

    def top_hubs(self, for_date: date, top_n: int = 1) -> List[GraphHubDTO]:
        """
        Returns top hubs by connection strength to cost-anomaly signals on for_date.
        Uses existing knowledge-rag graph.
        """
        rows: List[Dict] = self.g.execute(
            """
            MATCH (h:Hub)-[r:INDICATES|ASSOCIATED_WITH]->(a:Anomaly)
            WHERE date(a.detected_at) = $for_date
            RETURN
              h.id AS hub_id,
              h.label AS label,
              coalesce(h.type, 'unknown') AS hub_type,
              count(r) + coalesce(sum(r.strength), 0.0) AS score,
              collect(a.key)[..5] AS related_signals
            ORDER BY score DESC
            LIMIT $top_n
            """,
            {"for_date": for_date.isoformat(), "top_n": top_n},
        )

        return [
            GraphHubDTO(
                hub_id=r["hub_id"],
                label=r["label"],
                hub_type=r["hub_type"],
                score=float(r["score"]),
                for_date=for_date,
                related_signals=r["related_signals"] or [],
            )
            for r in rows
        ]
```

**c) Service/App layer (`app/services/cost_anomaly_signal.py`)**

```python
from datetime import date
from typing import List
from app.api.v1.cost_anomaly.schemas import GraphHubDTO
from app.repositories.knowledge_graph_repo import KnowledgeGraphRepo

class CostAnomalySignalApp:
    def __init__(self, kg_repo: KnowledgeGraphRepo):
        self.kg_repo = kg_repo

    def top_hub(self, for_date: date | None = None, top_n: int = 1) -> List[GraphHubDTO]:
        effective_date = for_date or date.today()
        return self.kg_repo.top_hubs(for_date=effective_date, top_n=top_n)
```
