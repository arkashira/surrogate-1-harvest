## Deployment
cf workers (free tier; deploy API as worker; Supabase for Postgres; no extra infra)

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP (FastAPI endpoints + worker) that connects to one AWS account and one GCP project, lists Lambda + Cloud Function schedules, detects duplicates (by hash) and orphans (target missing), surfaces results via JSON API and simple static report; supports manual retirement (disable/delete) with audit_run logging; deployable as single CF Worker with Supabase Postgres backend.

## Open questions
- How will we authenticate to customer AWS/GCP accounts (least-privilege read-only role vs org-level service account)?
- What is the acceptable scan cadence (hourly vs daily) and max accounts per customer on free tier?
- Do we store raw cloud metadata (JSONB) or only derived fields to limit PII/size?
- Will orphan detection require historical invocation logs (CloudWatch/Stackdriver) or only current schedule/target existence?
- How do we define 'drifted' (cron expression change, target change, IAM/permission change) and compute hash accordingly?
- Who can retire schedules (self-service approval flow vs manual review) and what audit trail is required?
