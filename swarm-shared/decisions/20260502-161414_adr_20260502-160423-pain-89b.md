## Deployment
cf workers

## v0.1 scope
v0.1 ships a cross-platform CLI daemon (Linux/macOS/Windows) that creates a hidden SOCKS5 listener, tunnels traffic through an obfuscated bridge (shadowsocks-like stream cipher) with a built-in killswitch, stores profiles encrypted in SQLite, and emits privacy-filtered telemetry for WAU-ZB; no server components, no GUI, and no external infra beyond CF Workers for a minimal bootstrap endpoint list.

## Open questions
- Which DPI signatures in our region targets (SNI, packet length/time distributions, TTL anomalies) must we prioritize mimicking?
- Can we rely on CF Workers for domain-fronted bootstrap endpoints without creating centralized chokepoints?
- What is the acceptable trade-off between stealth latency and obfuscation strength for usability in low-bandwidth scenarios?
- How will we validate WAU-ZB telemetry without leaking user identifiers or session metadata?
- Which rootless privilege escalation strategy (if any) is acceptable across desktop platforms to bind low ports or create TUN devices?
