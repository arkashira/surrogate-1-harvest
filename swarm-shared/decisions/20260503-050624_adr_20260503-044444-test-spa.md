## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on CF Workers (or local dev) that discovers Lambda and Cloud Functions schedules via customer-supplied credentials, persists them in Supabase Postgres, detects exact-duplicate cron schedules and orphaned schedules (no matching function), exposes a minimal REST endpoint and JSON report, and provides a read-only view with severity labels and suggested retirement actions.

## Open questions
- Which identity model will the auditor run under (customer-provided keys vs. org-level service account) and how will secrets be stored?
- What is the acceptable discovery cadence (hourly vs. daily) given rate limits and free-tier constraints?
- How will duplicates be grouped (exact cron + tags vs. fuzzy function similarity) to avoid false positives?
- What retention window for schedule history is needed to compute 'stale' (e.g., no invocation in 30/60/90 days)?
- Will remediation be advisory-only or include safe automated disable/delete with dry-run previews?
