# airship / design

## Final Synthesis — CDN-only, Rate-limit-safe Surrogate-1 Training

**Chosen objective**  
Eliminate HF API 429 risk during Surrogate-1 Lightning Studio training by replacing `load_dataset(streaming=True)` with CDN-only parquet loading via a pre-computed file manifest. This is the highest-value change deliverable in <2h and directly unblocks reliable training on Lightning Studio.

---

## Why this wins
- Removes recursive `list_repo_files` and per-file HF API calls during training (rate-limit 1000/5min).  
- Uses HF public CDN (`resolve/main/`) which bypasses auth and has much higher effective throughput.  
- Fits the “Mac orchestration + Lightning Studio training” pattern and reuses running studios.  
- Concrete, minimal, and verifiable (no 429s during data loading).

---

## Implementation Plan (≤2h)

### 1) Manifest generator (run on Mac / orchestration host)
- One `list_repo_tree(..., recursive=False)` call per date folder under `batches/mirror-merged/`.  
- Emit `manifest-{date}.json` with `{path, cdn_url, size, sha256}`.  
- Commit or place alongside training code (or embed as static asset).

### 2) Lightning training script update
- Remove `load_dataset(streaming=True, repo_name, ...)`.  
- Read `manifest-{date}.json`.  
- Use `pyarrow.parquet.read_table(cdn_url, columns=[...])` per file.  
- Stream via an `IterableDataset` that yields rows one-by-one to avoid loading all parquet into RAM.  
- Add robust per-file error handling and deterministic worker-level file sharding.

### 3) Studio reuse + lifecycle guard
- Before `.fit()`, list running studios and reuse a running studio with matching name.  
- If stopped, restart with `target.start(machine=Machine.L40S)` (H200 not in free tier).  
- Short-circuit long-running epochs into checkpointed steps so studio idle-stop doesn’t lose progress.

### 4) Correctness + actionability rules (resolve contradictions)
- Prefer correctness: always validate columns exist in parquet; default to safe subset if schema drift possible.  
- Prefer actionability: manifest is generated once per date and reused; training is fully CDN-only with zero HF dataset API calls during data loading.  
- Avoid speculative complexity (no on-the-fly tree recursion in training). Keep generator simple and training deterministic.

```python
#!/usr/bin/env python3
"""
Generate CDN manifest for a date folder under batches/mirror-merged/.
Run on Mac (or any orchestration host) to avoid HF API usage during training.
"""
import hashlib
import json
import os
import sys
from datetime import datetime
from typing import List, Dict, Any

from huggingface_hub import HfApi

API = HfApi()
REPO_ID = os.getenv("HF_REPO_ID", "your-org/surrogate-dataset-mirror")  # <-- adjust
DATE = os.getenv("MANIFEST_DATE", datetime.utcnow().strftime("%Y-%m-%d"))
FOLDER = f"batches/mirror-merged/{DATE}"
OUT = f"manifest-{DATE}.json"

def sha256_of_url(url: str) -> str | None:
    # Best-effort: if file is local or accessible, compute; otherwise skip.
    # For remote-only usage, omit or store null.
    return None

def build_manifest() -> Dict[str, Any]:
    entries: List[Dict[str, Any]] = []
    try:
        tree = API.list_repo_tree(REPO_ID, path=FOLDER, recursive=False)
    except Exception as exc:
        print(f"Failed to list repo tree for {FOLDER}: {exc}", file=sys.stderr)
        sys.exit(1)

    for item in tree:
        if not item.path.endswith(".parquet"):
            continue
        cdn_url = f"https://huggingface.co/datasets/{REPO_ID}/resolve/main/{item.path}"
        entries.append({
            "path": item.path,
            "cdn_url": cdn_url,
            "size": getattr(item, "size", None),
            "sha256": sha256_of_url(cdn_url),
        })

    manifest = {
        "repo_id": REPO_ID,
        "folder": FOLDER,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "files": entries,
    }
    with open(OUT, "w", encoding="utf-8") as f:
        json.dump(manifest, f, indent=2)
    print(f"Wrote {len(entries)} files to {OUT}")
    return manifest

if __name__ == "__main__":
    build_manifest()
```

### train.py (Lightning Studio) — CDN-only loader
```python
#!/usr/bin/env python3
import json
import pyarrow.parquet as pq
import torch
from torch.utils.data import IterableDataset, DataLoader
from lightning import LightningModule, Trainer
from lightning.pytorch.loggers import CSVLogger
from lightning.pytorch.strategies import DDPStrategy

MANIFEST_PATH = "manifest-2026-04-29.json"  # or injected via env
COLUMNS = ["prompt", "response"]

class ParquetIterable(IterableDataset):
    def __init__(self, manifest_path: str, columns=None):
        super().__init__()
        with open(manifest_path, encoding="utf-8") as f:
            self.manifest = json.load(f)
        self.files = [f["cdn_url"] for f in self.manifest["files"]]
        self.columns = columns or COLUMNS
        if not self.files:
            raise ValueError("No parquet files found in manifest.")

    def __iter__(self):
        worker_info = torch.utils.data.get_worker_info()
        files = self.files
        if worker_info is not None:
            # deterministic sharding across workers
            per_worker = max(1, len(files) // worker_info.num_workers)
            start = worker_info.id * per_worker
            end = start + per_worker if worker_info.id < worker_info.num_workers - 1 else len(files)
            files = files[start:end]

        for url in files:
            try:
                table = pq.read_table(url, columns=self.columns)
                df = table.to_pandas()
                # ensure expected columns exist
                missing = set(self.columns) - set(df.columns)
                if missing:
                    raise ValueError(f"Missing columns {missing} in {url}")
                for _, row in df.iterrows():
                    yield {
                        "prompt": str(row["prompt"]),
                        "response": str(row["response"]),
                    }
            except Exception as exc:
                # log and continue to avoid crashing entire epoch
                print(f"Skipping {url}: {exc}")
                continue

class SurrogateDataModule:
    def train_dataloader(self):
        ds = ParquetIterable(MANIFEST_PATH)
        return DataLoader(ds, batch_size=8, num_workers=2, persistent_workers=True)

class SurrogateModel(LightningModule):
    # minimal stub — replace with your actual model/tokenizer/training_step
    def __init__(self):
        super().__init__()
        self.lm = torch.nn.Linear(1024, 1024)  # placeholder

    def training_step(self, batch, batch_idx):
        # dummy loss — replace with real forward + loss
        loss = torch.tensor(0.0, requires_grad=True)
        self.log("train_loss", loss, prog_bar=True)
        return loss

    def configure_optimizers(self):
        return torch.optim.Adam(self.parameters(), lr=1e-4)

if __name__ == "__main__":
    from lightning.pytorch import seed_everything

    seed_everything(42, workers=True)

    model = SurrogateModel()
    dm = SurrogateDataModule()

    trainer = Trainer(
        max_epochs=1,
        logger=CSVLogger(save_dir="logs/"),
        accelerator="gpu",
        devices=1,
        strategy=DDPStrategy(find_unused_parameters=False),
        log_every_n_steps=10,
        enable_checkpointing=True,
```
