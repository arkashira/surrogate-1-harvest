# vanguard / design

**Most Valuable Next Improvement: Persistent File-List Cache with Enhanced Features**

### Diagnosis

* No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` → HF API 429 (1000 req/5min).
* Lightning Studio lifecycle not reused → quota burn on recreation.
* Training = re-listing repository files on each run.
* File-list embedding in training script not optimized for future changes.

### Proposed change

Enhance the existing file-list cache to persist across sessions and include features such as:

* Automatic refresh of the cache when the repository changes.
* Support for multiple file-lists (e.g., one for each date folder).
* Embedding of the file-list in the training script for easy access.

### Implementation

**Step 1: Create a cache directory**

Create a `cache` directory in the project root to store the file-lists.

```bash
mkdir cache
```

**Step 2: Implement the cache logic**

Create a new file `cache.py` with the following content:

```python
import os
import json
from typing import Dict

CACHE_DIR = 'cache'

def get_file_list(repo: str, path: str) -> Dict:
    cache_file = os.path.join(CACHE_DIR, f'{repo}_{path}.json')
    if os.path.exists(cache_file):
        with open(cache_file, 'r') as f:
            return json.load(f)
    else:
        # Fetch the file-list from the HF API
        file_list = []
        # ...
        with open(cache_file, 'w') as f:
            json.dump(file_list, f)
        return file_list

def refresh_cache(repo: str, path: str):
    cache_file = os.path.join(CACHE_DIR, f'{repo}_{path}.json')
    if os.path.exists(cache_file):
        os.remove(cache_file)
    # Fetch the updated file-list from the HF API
    file_list = []
    # ...
    with open(cache_file, 'w') as f:
        json.dump(file_list, f)
```

**Step 3: Integrate the cache with the training script**

Modify the training script to use the cached file-list.

```python
import cache

# ...

file_list = cache.get_file_list(repo, path)
# ...
```

To confirm that the cache is working correctly, you can:

* Check that the cache directory is populated with the expected file-lists.

* Test that the cache is refreshed correctly when the repository changes.
