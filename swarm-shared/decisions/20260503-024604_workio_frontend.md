# workio / frontend

## 1. Diagnosis

- No optimistic UI on the punch clock button → perceived latency and double-tap risk when users spam the clock-in/out action.
- Missing client-side idempotency key on punch requests → duplicate punches can be created on retries or double-clicks (server sees identical payloads as new requests).
- Punch state is read from server round-trip → UI feels sluggish and can show stale state across tabs/devices.
- No disabled-while-pending guard on the punch control → users can queue multiple requests while one is in-flight.
- Punch timestamp is generated server-side only → no immediate local feedback and harder to reconcile optimistic state.

## 2. Proposed change

File: `/opt/axentx/workio/src/components/PunchClock.tsx` (or equivalent main clock control component)  
Scope: add optimistic UI + idempotency key + disabled-while-pending on the punch action; generate client timestamp for immediate feedback; send `X-Idempotency-Key` header; rollback on error.

## 3. Implementation

```tsx
// /opt/axentx/workio/src/components/PunchClock.tsx
import { useState, useCallback } from 'react';
import { useToast } from '@/hooks/use-toast';

type PunchState = 'in' | 'out' | null;

export function PunchClock() {
  const [status, setStatus] = useState<PunchState>(null);
  const [pending, setPending] = useState(false);
  const { toast } = useToast();

  const punch = useCallback(async () => {
    if (pending) return;

    const nextStatus: PunchState = status === 'in' ? 'out' : 'in';
    const clientTimestamp = new Date().toISOString();
    const idempotencyKey = `punch-${crypto.randomUUID()}`;

    // Optimistic update
    setPending(true);
    setStatus(nextStatus);

    try {
      const res = await fetch('/api/punch', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-Idempotency-Key': idempotencyKey,
        },
        body: JSON.stringify({
          status: nextStatus,
          timestamp: clientTimestamp,
        }),
      });

      if (!res.ok) {
        const err = await res.json().catch(() => ({}));
        throw new Error(err.message || `Punch failed: ${res.status}`);
      }

      const data = await res.json();
      // Optionally reconcile with authoritative server timestamp
      // setStatus(data.status); // keep optimistic unless server disagrees
      toast({
        title: `Clocked ${nextStatus === 'in' ? 'in' : 'out'}`,
        description: new Date(clientTimestamp).toLocaleTimeString(),
      });
    } catch (err: any) {
      // Rollback on error
      setStatus(status); // revert to previous known state
      toast({
        variant: 'destructive',
        title: 'Punch failed',
        description: err.message || 'Please try again.',
      });
    } finally {
      setPending(false);
    }
  }, [status, pending, toast]);

  return (
    <button
      onClick={punch}
      disabled={pending}
      className={`
        inline-flex items-center justify-center rounded-lg px-6 py-3 text-base font-medium
        transition-colors disabled:opacity-60 disabled:cursor-not-allowed
        ${status === 'in'
          ? 'bg-red-600 hover:bg-red-700 text-white'
          : 'bg-green-600 hover:bg-green-700 text-white'
        }
      `}
      aria-busy={pending}
    >
      {pending
        ? 'Processing...'
        : status === 'in'
        ? 'Clock Out'
        : 'Clock In'}
    </button>
  );
}
```

Backend note (small, non-blocking): ensure `/api/punch` respects `X-Idempotency-Key` (store key→response for at least 24h and return cached response on duplicate key). If not yet implemented, add a simple middleware:

```ts
// /opt/axentx/workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { LRUCache } from 'lru-cache';

const cache = new LRUCache<string, { status: number; body: string }>({ max: 5000 });

export function idempotencyMiddleware(req: Request, res: Response, next: NextFunction) {
  const key = req.headers['x-idempotency-key'] as string;
  if (!key || req.method !== 'POST') return next();

  const cached = cache.get(key);
  if (cached) {
    return res.status(cached.status).json(JSON.parse(cached.body));
  }

  const originalSend = res.json;
  // @ts-ignore
  res.json = function (body) {
    cache.set(key, { status: res.statusCode, body: JSON.stringify(body) });
    return originalSend.call(this, body);
  };
  next();
}
```

## 4. Verification

- Manual: open the app, click “Clock In” → UI switches immediately to “Clock Out” and shows success toast; network tab shows one POST with `X-Idempotency-Key`. Rapid double-click does not queue a second request (button disabled while pending).
- Duplicate retry: simulate a network failure (devtools → offline after click, then online + retry) → server should return cached 200 for the same idempotency key and no duplicate punch record in DB.
- Rollback test: force server 500 → UI reverts button state to previous status and shows error toast.
- Cross-tab: optimistic state is local; after successful punch, refresh or poll from server to confirm authoritative state matches.
