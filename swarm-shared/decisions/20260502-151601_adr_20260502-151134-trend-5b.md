## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk guardrail→execution stub) with Postgres event log, Redis Streams queue, compliance rule evaluator, immutable audit events, and mock broker/OMS connectors; edge-deployed API on CF Workers with Supabase backend; includes replay endpoint and basic dashboard-less CLI for runs and audits.

## Open questions
- Deterministic replay strategy: event-sourcing granularity (per-action vs per-workflow) and snapshot cadence?
- Compliance rule engine choice: JSONata/Rego embedded vs custom DSL; how to hot-reload without downtime?
- Broker/OMS connector latency budgets and idempotency key scheme under retries?
- Secrets management on free-tier: CF Workers KV + envelope encryption vs external vault?
- Audit retention and tamper-evidence approach: append-only log + periodic hash chaining?
- SLA targets for sub-second orchestration: p95/p99 budgets per stage (research→signal→risk→execution)?
