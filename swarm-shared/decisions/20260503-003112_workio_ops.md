## Final Synthesized Solution

**Take the strongest parts**:  
- Candidate 1’s **partial unique index** for “one open punch per user” (correct and performant).  
- Candidate 1’s **separate idempotency table** (keeps punches schema clean, supports replay safety).  
- Candidate 2’s **explicit audit columns** (`created_at`, `updated_at`) on punches for traceability.  
- Both agree on **transactional write + idempotency key** and **LINE event deduplication**.

**Resolve contradictions in favor of correctness + actionability**:  
- Do **not** store `idempotency_key` or `line_event_id` directly on `punches` (avoids nullable bloat and keeps uniqueness semantics clear). Keep them in a dedicated idempotency table keyed by the external ID, pointing to the punch.  
- Enforce “one open punch per user” with a **partial unique index** (not application logic).  
- Use **DB transaction** to close previous open punch (if clocking out) and insert new punch atomically.  
- Treat **LINE webhook event IDs** as idempotency keys (same mechanism as generic idempotency key).

### 1) DB schema changes (`server/src/db/schema.sql`)

```sql
-- Audit columns on punches
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ DEFAULT NOW();

-- Partial unique index: only one open punch per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_user
  ON punches (user_id)
  WHERE clock_out_at IS NULL;

-- Fast lookup for punch operations and reporting
CREATE INDEX IF NOT EXISTS idx_punches_user_clock_out
  ON punches (user_id, clock_out_at);

-- Idempotency table (supports LINE event IDs and client idempotency keys)
CREATE TABLE IF NOT EXISTS punch_idempotency (
  idempotency_key TEXT NOT NULL PRIMARY KEY,
  punch_id        INTEGER NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### 2) Idempotency middleware (`server/src/middleware/idempotency.ts`)

```ts
import { Request, Response, NextFunction } from 'express';
import { db } from '../db';

export async function withPunchIdempotency(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // Prefer explicit idempotency key; fallback to LINE webhook event ID
  const key =
    req.body.idempotencyKey ||
    req.header('Idempotency-Key') ||
    (req.body.events?.[0]?.source?.userId && req.body.events[0].source.userId); // simplified fallback example

  if (!key) return next();

  const existing = await db.oneOrNone(
    `SELECT punch_id FROM punch_idempotency WHERE idempotency_key = $1`,
    [key]
  );

  if (existing) {
    const punch = await db.oneOrNone(`SELECT * FROM punches WHERE id = $1`, [
      existing.punch_id,
    ]);
    return res.status(200).json({ punch, idempotent: true });
  }

  (req as any).idempotencyKey = key;
  next();
}
```

### 3) Punches route with transactional upsert (`server/src/routes/punches.ts`)

```ts
import { Router } from 'express';
import { db } from '../db';
import { withPunchIdempotency } from '../middleware/idempotency';

const router = Router();

router.post('/', withPunchIdempotency, async (req, res) => {
  const { userId, type = 'clock_in', latitude, longitude } = req.body;
  const idempotencyKey = (req as any).idempotencyKey;

  if (!userId) return res.status(400).json({ error: 'userId required' });

  const client = await db.connect();
  try {
    await client.query('BEGIN');

    // If clocking out, close any open punch for this user
    if (type === 'clock_out') {
      await client.query(
        `UPDATE punches
         SET clock_out_at = NOW(),
             updated_at = NOW(),
             clock_out_lat = $1,
             clock_out_lng = $2
         WHERE user_id = $3 AND clock_out_at IS NULL`,
        [latitude, longitude, userId]
      );
    }

    // Insert new punch (will fail on partial unique index if another open punch exists)
    const result = await client.query(
      `INSERT INTO punches (
         user_id,
         clock_in_at,
         clock_in_lat,
         clock_in_lng,
         clock_out_at,
         clock_out_lat,
         clock_out_lng
       )
       VALUES ($1, $2, $3, $4, $5, $6, $7)
       RETURNING *`,
      [
        userId,
        type === 'clock_in' ? new Date() : null,
        type === 'clock_in' ? latitude : null,
        type === 'clock_in' ? longitude : null,
        type === 'clock_out' ? new Date() : null,
        type === 'clock_out' ? latitude : null,
        type === 'clock_out' ? longitude : null,
      ]
    );

    const punch = result.rows[0];

    // Record idempotency if key provided
    if (idempotencyKey) {
      await client.query(
        `INSERT INTO punch_idempotency (idempotency_key, punch_id)
         VALUES ($1, $2)`,
        [idempotencyKey, punch.id]
      );
    }

    await client.query('COMMIT');
    return res.status(201).json({ punch, idempotent: false });
  } catch (err: any) {
    await client.query('ROLLBACK');

    // Unique violation from idx_punches_one_open_per_user
    if (err.code === '23505') {
      return res.status(409).json({ error: 'An open punch already exists for this user.' });
    }

    console.error(err);
    return res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

### 5) Verification checklist

1. **DB constraints**  
   - Confirm partial unique index exists:  
     `psql <db> -c "\d punches"` and `\d idx_punches_one_open_per_user`.  
   - Manual test: insert two open punches for same user → second fails with `23505`.

2. **Idempotency**  
   - POST same payload twice with same `idempotencyKey` (or reuse LINE event ID) → second returns `200` with `idempotent: true` and same punch.

3. **Race condition**  
   - Fire 5–10 concurrent identical requests; verify only one punch created and no duplicate open punches.

4. **LINE webhook replay**  
   - Replay past LINE event (same event ID) → no duplicate punch; returns existing punch.

5. **Clock-out behavior**  
   - Create open punch → POST `clock_out` for same user → previous punch `clock_out_at` set; no extra open punch remains.
