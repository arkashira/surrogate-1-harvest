# workio / quality

## Final Synthesis — Correct, Actionable Fix

**Core problem**: LINE retries on 5xx/timeouts; the handler is not idempotent and punch transitions are not atomic. This causes duplicate punches, invalid open/close state, and race conditions under concurrency or retries.

**Chosen approach** (resolve contradictions in favor of correctness + deployability):
- **Idempotency at the webhook event level** using a durable `line_webhook_events` table with `webhook_event_id` as primary key.
- **Atomic punch transitions** enforced by the database (single statement + constraints), not by application read-then-write.
- **Transactional safety**: all side effects (event record + punch) in one DB transaction; commit only on success.
- **Clear HTTP semantics**:
  - `200` only after commit (so LINE stops retrying).
  - `4xx` for invalid payloads (do not retry).
  - `5xx` for transient/server errors (retry with backoff).
- **Observability**: structured logs + persisted webhook events for replay/debug.

### 1. Schema (append to existing migrations/schema)

```sql
-- Idempotency log for LINE webhook events
CREATE TABLE IF NOT EXISTS line_webhook_events (
  webhook_event_id VARCHAR(255) PRIMARY KEY,
  tenant_id        VARCHAR(255) NOT NULL,
  user_id          VARCHAR(255) NOT NULL,
  event_type       VARCHAR(50)  NOT NULL,
  payload          JSONB       NOT NULL,
  processed_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Punches with tenant isolation and uniqueness guard
CREATE TABLE IF NOT EXISTS punches (
  id              SERIAL PRIMARY KEY,
  tenant_id       VARCHAR(255) NOT NULL,
  user_id         VARCHAR(255) NOT NULL,
  clock_at        TIMESTAMPTZ NOT NULL,
  direction       VARCHAR(4)   NOT NULL CHECK (direction IN ('in','out')),
  latitude        DOUBLE PRECISION,
  longitude       DOUBLE PRECISION,
  line_event_id   VARCHAR(255),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  -- Prevent duplicate punches for same tenant/user/time/direction
  UNIQUE(tenant_id, user_id, clock_at, direction)
);

-- Optional helper index for fast last-punch lookup (used by app logic)
CREATE INDEX IF NOT EXISTS idx_punches_user_open ON punches (tenant_id, user_id, direction, clock_at DESC)
WHERE direction IN ('in','out');
```

### 2. Handler (Hono + Zod + Postgres)

File: `/opt/axentx/workio/server/src/routes/line/webhook.ts`

```ts
import { Hono } from 'hono';
import { zValidator } from '@hono/zod-validator';
import { z } from 'zod';
import { pool } from '../../db';

const lineWebhookSchema = z.object({
  destination: z.string(),
  events: z.array(
    z.object({
      type: z.string(),
      mode: z.enum(['active', 'standby']),
      timestamp: z.number(),
      source: z.object({
        type: z.enum(['user', 'group', 'room']),
        userId: z.string().optional(),
        groupId: z.string().optional(),
        roomId: z.string().optional(),
      }),
      webhookEventId: z.string(),
      deliveryContext: z.object({
        isRedelivery: z.boolean(),
      }),
      message: z.object({
        id: z.string(),
        type: z.string(),
        text: z.string().optional(),
      }).optional(),
      postback: z.object({
        data: z.string(),
        params: z.record(z.any()).optional(),
      }).optional(),
    })
  ),
});

const app = new Hono();

app.post(
  '/webhook/line',
  zValidator('json', lineWebhookSchema, (result, c) => {
    if (!result.success) {
      // 400 -> do not retry (client error)
      return c.json({ error: 'invalid_payload', details: result.error.errors }, 400);
    }
    return undefined;
  }),
  async (c) => {
    const body = c.req.valid('json');
    const client = await pool.connect();

    try {
      await client.query('BEGIN');

      for (const ev of body.events) {
        // Idempotency: skip already processed events
        const exists = await client.query(
          `SELECT 1 FROM line_webhook_events WHERE webhook_event_id = $1`,
          [ev.webhookEventId]
        );
        if (exists.rowCount && exists.rowCount > 0) {
          continue;
        }

        // Persist event (idempotency record)
        await client.query(
          `INSERT INTO line_webhook_events (webhook_event_id, tenant_id, user_id, event_type, payload)
           VALUES ($1,$2,$3,$4,$5)`,
          [ev.webhookEventId, 'default-tenant', ev.source.userId ?? '', ev.type, ev]
        );

        // Determine clock action
        const actionRaw = ev.postback?.data || ev.message?.text?.trim().toLowerCase();
        if (!actionRaw || !['in', 'out'].includes(actionRaw)) {
          // Not a clock event; still record but do not create punch
          continue;
        }
        const direction = actionRaw as 'in' | 'out';
        if (!ev.source.userId) continue;

        // Atomic punch insert; unique constraint prevents duplicates
        // We do NOT enforce business rule (in->out->in) at DB level here;
        // app may enforce it before calling, but DB prevents corruption on races/retries.
        const clockAt = new Date(ev.timestamp);
        try {
          await client.query(
            `INSERT INTO punches (tenant_id, user_id, clock_at, direction, line_event_id)
             VALUES ($1,$2,$3,$4,$5)`,
            ['default-tenant', ev.source.userId, clockAt, direction, ev.webhookEventId]
          );
        } catch (err: any) {
          // Unique violation: duplicate (tenant,user,clock_at,direction) — safe to ignore
          if (err.code !== '23505') throw err;
        }
      }

      await client.query('COMMIT');
      return c.json({ status: 'ok' }, 200);
    } catch (err) {
      await client.query('ROLLBACK').catch(() => {});
      console.error('Webhook processing failed', { error: err });
      // 5xx -> LINE will retry (for transient/server errors)
      return c.json({ error: 'processing_failed' }, 500);
    } finally {
      client.release();
    }
  }
);

export default app;
```

### 3. Business-rule guard (optional but recommended)

If you want to enforce valid transitions (no double-in, no double-out), add a lightweight check **inside the same transaction** before insert:

```ts
// Inside the same tx, before INSERT into punches:
const last = await client.query(
  `SELECT direction FROM punches
   WHERE tenant_id = $1 AND user_id = $2
   ORDER BY clock_at DESC, created_at DESC
   LIMIT 1`,
  ['default-tenant', ev.source.userId]
);

if (direction === 'in' && last.rows[0]?.direction === 'in') {
  // Already clocked in — skip or update depending on policy
  continue;
}
if (direction === 'out' && (!last.rows[0] || last.rows[0].direction === 'out')) {
  // No open session to close — skip or handle as needed
  continue;
}
// proceed to INSERT
```

Keep this inside the transaction so it’s race-safe.

### 4. Verification checklist

1. **Schema applied**  
   - `line_webhook_events` exists with primary key on `webhook_event_id`.  
   - `punches` has unique constraint on `(tenant_id, user_id, clock_at, direction)`.

   - Send same payload (same `webhookEventId`) twice → exactly one `line_webhook_events` row and expected punches.

3. **Concurrent/retries**  
   - Fire 10 parallel requests with same event → no duplicate
