# Costinel / ops

## Final Implementation Plan — Costinel “Top-Hub Signal” Card (Read-Only)

**Scope**: Read-only frontend card (≤2h)  
**Principle**: “Sense + Signal — ไม่ Execute” (strictly no writes, no runtime mutations, no self-execution)  
**Goal**: Surface the most-connected hub (e.g., “MOC”) with contextual signals derived from prior knowledge-rag runs, enabling operators to triage cost governance actions without automated execution.

---

### 1) Design Decisions (resolved)

- **Data source**: Serve a lightweight read-only endpoint (`GET /api/knowledge-rag/top-hub`) that returns the latest top-hub signal.  
  - Prefer API over raw static JSON in `public/` to allow server-side validation, caching, and future auth/logging without exposing raw file paths.  
  - Keep implementation cheap: query a cached index or precomputed JSON; no heavy RAG at request time.

- **Component**: `TopHubSignalCard` — purely presentational, no writes, no action triggers.  
  - Props/shape: `{ hubId, hubLabel, rank?, connectionsCount?, signals: { title, summary, href? }[], lastUpdated }`.  
  - Auto-refresh every 60s (configurable) to pick up new signals; manual refresh optional.  
  - Accessibility: `role="region"` and clear aria labels.

- **Fail-safe behavior**:  
  - Graceful empty/skeleton state on error or missing data.  
  - No console errors thrown to user; log only in dev.

### 2) API — Read-Only Endpoint (10–15 min)

```ts
// /api/knowledge-rag/top-hub.ts
import type { NextApiRequest, NextApiResponse } from 'next';
import { getTopHub } from '@/lib/knowledge-rag';

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  if (req.method !== 'GET') {
    res.setHeader('Allow', ['GET']);
    return res.status(405).json({ error: 'Method not allowed' });
  }

  try {
    const hub = await getTopHub(); // { hubId, hubLabel, rank?, connectionsCount?, signals[], lastUpdated }
    return res.status(200).json(hub);
  } catch (err) {
    // Read-only: return empty shape rather than 500 to avoid breaking UI
    return res.status(200).json({
      hubId: null,
      hubLabel: null,
      rank: null,
      connectionsCount: 0,
      signals: [],
      lastUpdated: new Date().toISOString(),
    });
  }
}
```

import { useEffect, useState, useCallback } from 'react';

  connectionsCount: number;
  signals: Signal[];
  lastUpdated: string;

export default function TopHubSignalCard() {
  const [data, setData] = useState<TopHubPayload | null>(null);

      const json = (await res.json()) as TopHubPayload;
      setData(json);

      // Graceful empty state
      setData({
        hubId: null,
        hubLabel: null,
        rank: null,
        connectionsCount: 0,
        signals: [],
        lastUpdated: new Date().toISOString(),
      });
    } finally {
      setLoading(false);
    }
  }, []);

    fetchData();
    const id = setInterval(fetchData, REFRESH_MS);
    return () => clearInterval(id);
  }, [fetchData]);

      <section
        role="region"
        aria-label="Top hub signal"
        className="rounded-lg border bg-card p-4 shadow-sm"
      >
        <div className="h-5 w-24 animate-pulse rounded bg-muted" />
        <div className="mt-3 space-y-2">
          <div className="h-4 w-3/4 animate-pulse rounded bg-muted" />
          <div className="h-4 w-1/2 animate-pulse rounded bg-muted" />

    );
  }

      <section
        role="region"
        aria-label="Top hub signal"
        className="rounded-lg border bg-card p-4 shadow-sm"

      </section>

            </span>

            className="text-xs text-muted-foreground hover:underline"

            Refresh
          </button>
