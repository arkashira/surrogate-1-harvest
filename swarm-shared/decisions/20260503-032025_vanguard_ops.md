# vanguard / ops

## 1. Diagnosis
- No deterministic CDN-first manifest exists; ingestion/training scripts likely still call `list_repo_tree`/`load_dataset` at runtime, risking 429s and non-reproducible runs.
- Missing content-addressed file list keyed by date/slug; training jobs cannot pin exact data versions and may silently drift.
- No ops automation to generate/pin manifests before Lightning Studio starts; quota waste from repeated API calls and studio recreations.
- Scripts probably invoke `load_dataset(streaming=True)` on mixed-schema repos (pyarrow CastError risk) instead of CDN-only parquet fetches.
- No guard to reuse running Lightning Studio instances; each run risks quota burn and idle-stop training death.

## 2. Proposed change
Create a CDN-first manifest generator and Lightning launcher under `/opt/axentx/vanguard/`:
- `scripts/gen_manifest.py` — single API call to `list_repo_tree` for a date folder, emits `manifests/{date}/files.json` with CDN URLs and content-addressed slugs.
- `scripts/run_surrogate1.py` — Lightning launcher that:
  - pins manifest path,
  - reuses running Studio by name,
  - starts L40S (free-tier fallback),
  - passes manifest path to training script,
  - guards against idle-stop by checking status before `.run()`.
- `Makefile` targets: `make manifest DATE=2026-04-29` and `make train DATE=2026-04-29`.

## 3. Implementation

# /opt/axentx/vanguard/Makefile
.PHONY: manifest train clean

DATE ?= $(shell date +%Y-%m-%d)
REPO ?= datasets/your-org/surrogate-1
MANIFEST := manifests/$(DATE)/files.json

manifest: $(MANIFEST)

$(MANIFEST):
	@mkdir -p manifests/$(DATE)
	python3 scripts/gen_manifest.py --repo $(REPO) --date $(DATE) --out $@

train: $(MANIFEST)
	python3 scripts/run_surrogate1.py --manifest $(MANIFEST) --date $(DATE)

clean:
	rm -rf manifests/$(DATE)
```

# /opt/axentx/vanguard/scripts/gen_manifest.py
#!/usr/bin/env python3
"""
Generate CDN-first manifest for a date folder in a HuggingFace dataset repo.
Avoids runtime list_repo_tree calls during training (rate-limit safe).
"""
import argparse
import json
import os
import hashlib
from datetime import datetime

try:
    from huggingface_hub import HfApi
except ImportError:
    HfApi = None

CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

def content_slug(path: str) -> str:
    return hashlib.sha256(path.encode()).hexdigest()[:16]

def main():
    parser = argparse.ArgumentParser(description="Generate CDN manifest for date folder")
    parser.add_argument("--repo", required=True, help="HF dataset repo (org/name)")
    parser.add_argument("--date", required=True, help="Date folder (YYYY-MM-DD)")
    parser.add_argument("--out", required=True, help="Output JSON path")
    parser.add_argument("--tree-depth", type=int, default=1, help="list_repo_tree depth")
    args = parser.parse_args()

    if HfApi is None:
        raise RuntimeError("huggingface_hub required: pip install huggingface_hub")

    api = HfApi()
    folder = f"{args.date}/"
    entries = api.list_repo_tree(repo_id=args.repo, path=folder, recursive=False)

    files = []
    for e in entries:
        if not e.path.endswith((".parquet", ".jsonl")):
            continue
        url = CDN_TEMPLATE.format(repo=args.repo, path=e.path)
        slug = content_slug(e.path)
        files.append({
            "slug": slug,
            "path": e.path,
            "cdn_url": url,
            "size": getattr(e, "size", None),
            "last_modified": getattr(e, "last_modified", None),
        })

    manifest = {
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "repo": args.repo,
        "date": args.date,
        "count": len(files),
        "files": files,
    }

    os.makedirs(os.path.dirname(args.out), exist_ok=True)
    with open(args.out, "w") as f:
        json.dump(manifest, f, indent=2)
    print(f"Wrote {len(files)} files to {args.out}")

if __name__ == "__main__":
    main()
```
