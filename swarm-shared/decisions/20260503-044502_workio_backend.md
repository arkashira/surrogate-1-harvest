# workio / backend

## 1. Diagnosis

- **Missing idempotency key on LINE webhook handler**: duplicate `clock-in/out` events can occur when LINE retries (network timeout or 5xx), causing phantom punches and incorrect daily totals.
- **No signature verification before processing**: `X-Line-Signature` is not validated early, allowing spoofed webhook payloads to mutate attendance state.
- **Race condition on concurrent clock events**: two near-simultaneous webhooks for same user can produce two “in” records without an “out” (or vice-versa) because DB write is not atomic with state check.
- **No outbound LINE notification retry/backoff**: failed notify calls (e.g., punch confirmations, leave approvals) are fire-and-forget; users never see confirmations and admins lack visibility.
- **Missing structured request ID for traceability**: no correlation ID across webhook receipt → DB → notify → client; debugging failures across logs is slow.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/line/webhook.ts` (or equivalent under `server/src/routes/line/`).  
Scope: add middleware for signature verification + idempotency + atomic upsert for clock events + exponential-backoff LINE notify helper + request ID propagation.

// server/src/middleware/lineAuth.ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';

export function lineSignature(secret: string) {
  return (req: Request, res: Response, next: NextFunction) => {
    const sig = req.get('X-Line-Signature');
    const body = JSON.stringify(req.body);
    const expected = crypto
      .createHmac('sha256', secret)
      .update(body)
      .digest('base64');
    if (!sig || !crypto.timingSafeEqual(Buffer.from(sig), Buffer.from(expected))) {
      return res.status(401).json({ error: 'Invalid signature' });
    }
    next();
  };
}

// server/src/utils/idempotency.ts
import { PoolClient } from 'pg';

export async function ensureIdempotent(
  client: PoolClient,
  key: string,
  fn: () => Promise<void>
): Promise<{ alreadyProcessed: boolean }> {
  const q = `
    INSERT INTO idempotency_keys (key, created_at)
    VALUES ($1, NOW())
    ON CONFLICT (key) DO UPDATE SET last_seen_at = NOW()
    RETURNING (xmax = 0) AS inserted;
  `;
  const { rows } = await client.query(q, [key]);
  const inserted = rows[0]?.inserted === true;
  if (!inserted) return { alreadyProcessed: true };
  await fn();
  return { alreadyProcessed: false };
}

// server/src/utils/lineNotify.ts
import axios from 'axios';

export async function sendLineMessage(
  token: string,
  to: string,
  message: { type: string; text: string }
) {
  const url = 'https://api.line.me/v2/bot/message/push';
  const body = { to, messages: [message] };

  const maxRetries = 5;
  let delay = 1000;
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      await axios.post(url, body, {
        headers: { Authorization: `Bearer ${token}` },
        timeout: 10_000,
      });
      return;
    } catch (err: any) {
      if (attempt === maxRetries) throw err;
      const wait = delay * Math.pow(2, attempt) + Math.random() * 200;
      await new Promise((r) => setTimeout(r, wait));
    }
  }
}

```ts
// server/src/routes/line/webhook.ts  (core handler)
import { Router } from 'express';
import { Pool } from 'pg';
import { lineSignature } from '../../middleware/lineAuth';
import { ensureIdempotent } from '../../utils/idempotency';
import { sendLineMessage } from '../../utils/lineNotify';

const router = Router();
const pool = new Pool({ connectionString: process.env.DATABASE_URL });

router.post(
  '/webhook/line',
  lineSignature(process.env.LINE_CHANNEL_SECRET!),
  async (req, res) => {
    const reqId = req.headers['x-request-id'] || crypto.randomUUID();
    const events = req.body.events || [];

    const client = await pool.connect();
    try {
      for (const ev of events) {
        if (ev.type !== 'message' || ev.message?.type !== 'text') continue;

        const userId = ev.source.userId;
        const tenantId = 1; // derive from DB lookup in real impl
        const idempotencyKey = `line:${ev.timestamp}:${userId}:${ev.message.id}`;

        await ensureIdempotent(client, idempotencyKey, async () => {
          // Atomic upsert: latest punch without clock_out gets closed; otherwise insert new clock_in
          const q = `
            WITH latest AS (
              SELECT id, clock_out_at
              FROM attendance
              WHERE tenant_id = $1 AND user_id = $2
              ORDER BY clock_in_at DESC
              LIMIT 1
              FOR UPDATE
            )
            INSERT INTO attendance (tenant_id, user_id, clock_in_at, clock_out_at)
            SELECT $1, $2, NOW(), NULL
            WHERE NOT EXISTS (SELECT 1 FROM latest WHERE clock_out_at IS NULL)
            RETURNING id;
          `;
          const { rows } = await client.query(q, [tenantId, userId]);
          const punchedIn = rows.length > 0;

          const text = punchedIn ? 'Clock in recorded' : 'Clock out recorded (latest open punch updated)';
          await sendLineMessage(process.env.LINE_CHANNEL_ACCESS_TOKEN!, userId, {
            type: 'text',
            text: `[${reqId}] ${text}`,
          });
        });
      }
      res.status(200).json({ status: 'ok' });
    } catch (err) {
      console.error(`[${reqId}] webhook error`, err);
      res.status(500).json({ error: 'Internal server error' });
    } finally {
      client.release();
    }
  }
);

export default router;
```

## 4. Verification

1. **Signature rejection**:  
   ```bash
   curl -X POST http://localhost:3000/webhook/line \
     -H "Content-Type: application/json" \
     -H "X-Line-Signature: invalid" \
     -d '{"events":[]}'
   ```
   Expect `401`.

2. **Idempotency**:  
   - Send same event payload twice within retry window.  
   - Verify second request returns `200` quickly and DB has exactly one new attendance row (or zero if already closed punch).

3. **Race condition**:  
   - Fire two concurrent requests for same user with no open punch.  
   - Verify only one row inserted with `clock_in_at` and second becomes a no-op (or closes punch) depending on timing; no double “in” rows.

4. **LINE notify retry**:  
   - Temporarily block outbound to `api.line.me` (e.g., via firewall) and trigger a punch.  
   - Observe exponential backoff in logs and eventual success when connectivity restored.

5. **Traceability**:  
   - Include `X-Request-ID` header in test request and confirm same ID appears in logs and in LINE message text.
