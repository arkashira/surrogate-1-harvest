## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on Cloudflare Workers (D1/Postgres via Supabase) that discovers Lambda + Cloud Functions schedules, persists metadata to Postgres, detects duplicates (same cron+target) and orphaned schedules (target missing), exposes REST endpoints for audit runs and findings, and outputs a simple JSON report. No UI, no queue, no cache; auth via platform credentials supplied as env vars.

## Open questions
- How will customers authenticate cloud reads (assume IAM roles per platform vs. stored keys)?
- What retention and access model for findings (TTL, export formats)?
- How to define 'drift' thresholds (function hash vs. runtime/env diff)?
- What is the scan cadence and rate-limit/backoff strategy per platform?
- Will customers want webhook/email alerts or only API access?
