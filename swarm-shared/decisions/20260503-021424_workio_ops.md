# workio / ops

Below is the **single, final synthesis** that merges the strongest, most actionable parts of both proposals, removes duplication, resolves contradictions, and favors correctness + concrete execution.

---

## 1. Diagnosis (merged + resolved)

- **Root cause**: LINE webhook retries + non-idempotent ingestion allow duplicate punches when concurrent/retried deliveries arrive for the same tenant+user+day.
- **Race condition**: Clock-in/out operations use read-then-write (find → insert/update) instead of atomic upsert, permitting double clock-ins or clobbered clock-outs.
- **Missing guardrails**:
  - No tenant-scoped uniqueness on `(tenant_id, user_line_uid, punch_date, punch_type)` for punches.
  - No idempotency store keyed by LINE `deliveryId` (or derived key) to cheaply reject retries.
  - No atomicity for leave/OT mutations (mentioned in Candidate 2) — same pattern should apply.
- **Observability gap**: no metrics/alerting for duplicate rate or webhook 4xx/5xx; duplicates discovered only via mismatched reports.

---

## 2. Scope & Strategy (final)

- **Primary target**: `attendanceService.ts` (punch ingestion) + webhook route.
- **Secondary targets** (same patterns): leave/OT mutation endpoints (apply identical idempotency + atomic upsert).
- **Guiding rules**:
  1. **Idempotency first**: reject duplicate LINE deliveries before touching domain data.
  2. **Atomic upsert**: use `INSERT … ON CONFLICT DO UPDATE` for clock-ins; atomic `UPDATE` for clock-outs; avoid read-then-write.
  3. **Tenant-scoped uniqueness**: enforce one punch type per user per day where applicable.
  4. **Safe fallback**: if clock-out arrives without clock-in, preserve record with minimal clock-out row.
  5. **Observability**: emit structured logs + metrics; alert on duplicate spikes.

## 4. Implementation (final)

### 4.1 Core service: `attendanceService.ts`

```typescript
// server/src/services/attendanceService.ts
import { pool } from '../db/index.js';

export async function handleLinePunch(opts: {
  tenantId: string;
  lineUserId: string;
  eventType: 'clock_in' | 'clock_out';
  lineDeliveryId: string;
  timestamp: string; // ISO from LINE
  latitude?: number;
  longitude?: number;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency: reject duplicate LINE delivery
    const deliveryKey = `${opts.timestamp}:${opts.eventType}:${opts.lineUserId}:${opts.lineDeliveryId}`;
    const dedupeRes = await client.query(
      `INSERT INTO line_webhook_deliveries (tenant_id, line_delivery_key, event_type)
       VALUES ($1,$2,$3) ON CONFLICT DO NOTHING RETURNING 1`,
      [opts.tenantId, deliveryKey, opts.eventType]
    );
    if (dedupeRes.rowCount === 0) {
      await client.query('ROLLBACK');
      return { ok: true, deduped: true };
    }

    const punchDate = new Date().toISOString().slice(0, 10); // YYYY-MM-DD (tenant-local date)

    if (opts.eventType === 'clock_in') {
      // Atomic upsert: one clock_in per day
      await client.query(
        `INSERT INTO attendances (tenant_id, user_line_uid, punch_date, punch_type, clock_in_at, latitude, longitude)
         VALUES ($1,$2,$3,$4,now(),$5,$6)
         ON CONFLICT (tenant_id, user_line_uid, punch_date, punch_type)
         DO UPDATE SET clock_in_at = EXCLUDED.clock_in_at, latitude = EXCLUDED.latitude, longitude = EXCLUDED.longitude`,
        [opts.tenantId, opts.lineUserId, punchDate, 'clock_in', opts.latitude ?? null, opts.longitude ?? null]
      );
    } else {
      // Atomic update for existing clock_in row
      await client.query(
        `UPDATE attendances
         SET clock_out_at = now(), latitude_out = $4, longitude_out = $5
         WHERE tenant_id = $1 AND user_line_uid = $2 AND punch_date = $3 AND punch_type = 'clock_in'`,
        [opts.tenantId, opts.lineUserId, punchDate, opts.latitude ?? null, opts.longitude ?? null]
      );

      // If no clock_in row existed, preserve clock_out record
      if ((await client.query(
        `SELECT 1 FROM attendances WHERE tenant_id = $1 AND user_line_uid = $2 AND punch_date = $3 AND punch_type = 'clock_in'`,
        [opts.tenantId, opts.lineUserId, punchDate]
      )).rowCount === 0) {
        await client.query(
          `INSERT INTO attendances (tenant_id, user_line_uid, punch_date, punch_type, clock_out_at, latitude_out, longitude_out)
           VALUES ($1,$2,$3,$4,now(),$5,$6)`,
          [opts.tenantId, opts.lineUserId, punchDate, 'clock_out', opts.latitude ?? null, opts.longitude ?? null]
        );
      }
    }

    await client.query('COMMIT');
    return { ok: true, deduped: false };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 4.2 Webhook handler: `lineWebhook.ts`

```typescript
// server/src/routes/lineWebhook.ts
import { handleLinePunch } from '../services/attendanceService.js';

export async function lineWebhookHandler(req, res) {
  const tenantId = req.tenant?.id;
  if (!tenantId) return res.sendStatus(400);

  const events = req.body.events || [];
  const results = [];

  for (const ev of events) {
    if (ev.type === 'message' && ev.message?.type === 'text') {
      const text = ev.message.text.trim();
      if (text === 'clock_in' || text === 'clock_out') {
        try {
          const result = await handleLinePunch({
            tenantId,
            lineUserId: ev.source.userId,
            eventType: text === 'clock_in' ? 'clock_in' : 'clock_out',
            lineDeliveryId: ev.deliveryId || String(Date.now()),
            timestamp: ev.timestamp ? new Date(Number(ev.timestamp)).toISOString() : new Date().toISOString(),
          });
          results.push({ event: ev, result });
        } catch (err) {
          console.error('punch handling failed', { tenantId, err, event: ev });
          // continue processing other events
        }
      }
    }
  }

  res.json({ ok: true, processed: results.length });
}
```
