# workio / backend

## 1. Diagnosis

- **Non-idempotent LINE webhook handler**: retries on 5xx/timeouts create duplicate punches and invalid open/close state.
- **No atomic punch transition**: concurrent/retried requests can interleave, leaving `punches` without a matching close or double-close.
- **Missing optimistic locking / versioning**: updates to `punches` rely on `WHERE` conditions that can silently fail under race conditions without feedback.
- **No idempotency key storage**: LINE events lack client-side idempotency keys; server must dedupe by event id + tenant to prevent double-processing.
- **No transactional boundary across punch + audit/notification writes**: partial success can persist a punch but fail to emit LINE/audit records, breaking consistency.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/webhook/line.ts` (or equivalent webhook handler).  
Scope: wrap the punch handling logic in a single DB transaction, add idempotency table (`line_event_idempotency`), and enforce atomic punch state transition with row-level locking.

```sql
CREATE TABLE IF NOT EXISTS line_event_idempotency (
  id            SERIAL PRIMARY KEY,
  tenant_id     INTEGER NOT NULL,
  event_id      TEXT    NOT NULL,
  processed_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload_hash  TEXT,
  UNIQUE(tenant_id, event_id)
);
```

### 3.2 Idempotent punch handler (diff-style)

```ts
// server/src/routes/webhook/line.ts
import { Router } from 'express';
import { pool } from '../../db';
import { verifyLineSignature } from '../../utils/line';

const router = Router();

router.post('/line', verifyLineSignature, async (req, res) => {
  const { events } = req.body;
  const tenantId = req.headers['x-tenant-id'] ? Number(req.headers['x-tenant-id']) : null;

  if (!tenantId) {
    return res.status(400).json({ error: 'Missing tenant' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      // Idempotency check
      const idem = await client.query(
        'SELECT 1 FROM line_event_idempotency WHERE tenant_id = $1 AND event_id = $2',
        [tenantId, ev.source.userId + ':' + ev.timestamp + ':' + ev.type]
      );
      if (idem.rowCount > 0) {
        continue; // already processed
      }

      if (ev.type === 'message' && ev.message.type === 'text') {
        const text = ev.message.text.trim().toLowerCase();
        const userId = ev.source.userId;

        // Lock latest open punch for this user/tenant (FOR UPDATE)
        const latest = await client.query(
          `SELECT id, clock_in, clock_out
           FROM punches
           WHERE tenant_id = $1 AND user_id = $2
           ORDER BY clock_in DESC
           LIMIT 1
           FOR UPDATE`,
          [tenantId, userId]
        );

        const now = new Date();

        if (text === 'clock in') {
          // If there's an open punch, close it first (safety)
          if (latest.rowCount > 0 && !latest.rows[0].clock_out) {
            await client.query(
              `UPDATE punches SET clock_out = $1, updated_at = NOW()
               WHERE id = $2`,
              [now, latest.rows[0].id]
            );
          }
          // Create new punch
          await client.query(
            `INSERT INTO punches (tenant_id, user_id, clock_in, created_at, updated_at)
             VALUES ($1, $2, $3, NOW(), NOW())`,
            [tenantId, userId, now]
          );
        } else if (text === 'clock out') {
          if (latest.rowCount === 0 || latest.rows[0].clock_out) {
            // nothing to close
            await client.query(
              `INSERT INTO line_event_idempotency (tenant_id, event_id, payload_hash)
               VALUES ($1, $2, $3)`,
              [tenantId, userId + ':' + ev.timestamp + ':' + ev.type, 'noop']
            );
            continue;
          }
          await client.query(
            `UPDATE punches SET clock_out = $1, updated_at = NOW()
             WHERE id = $2`,
            [now, latest.rows[0].id]
          );
        }

        // Record idempotency
        await client.query(
          `INSERT INTO line_event_idempotency (tenant_id, event_id, payload_hash)
           VALUES ($1, $2, $3)`,
          [tenantId, userId + ':' + ev.timestamp + ':' + ev.type, ev.message.text]
        );
      }
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('LINE webhook error', err);
    res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

## 4. Verification

1. **Schema check**: run `psql workio -c "\d line_event_idempotency"` — table exists.
2. **Idempotency test**: send same LINE event payload twice (within short window) — second request must not create duplicate punch rows and must return 200.
3. **Atomicity test**: simulate concurrent `clock in` requests for same user — only one open punch row should exist; no double inserts.
4. **Locking test**: run two parallel `clock out` against an open punch — one should succeed, one should no-op (idempotent) and not corrupt state.
5. **Rollback test**: force an error mid-handler (e.g., throw after punch insert) — verify no partial punch row persists and no idempotency record is written.
