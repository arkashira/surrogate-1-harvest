## Final Synthesized Solution

Below is the single, authoritative plan that merges the strongest, non-contradictory insights from both proposals and resolves remaining disagreements in favor of **correctness, durability, and concrete actionability**.

## 1. Diagnosis (merged + resolved)

- **Missing DB-level idempotency enforcement**: an `idempotency_key` column exists, but without a **unique index**, concurrent/retried LINE webhooks can still insert duplicate punches.  
  → **Resolution**: add a `UNIQUE` index on `(tenant_id, idempotency_key)` and enforce via `ON CONFLICT DO NOTHING`.

- **Missing business-rule uniqueness**: nothing prevents multiple clock-ins per user per day when retries or double-taps occur.  
  → **Resolution**: add a partial or functional unique index keyed by `(tenant_id, line_user_id, DATE(clock_in_at), type)` **only for active records** (or enforce via deterministic idempotency keys per transition). This is stricter than Candidate 1’s optional index and safer than Candidate 2’s vague “date+punch_type” guard.

- **Unsafe “check-then-insert” under concurrency**: selecting then inserting is racy without proper locking or atomic upsert.  
  → **Resolution**: use a single `INSERT ... ON CONFLICT DO NOTHING` inside a transaction and **never** rely on prior read-only checks for uniqueness.

- **No durable webhook deduplication**: repeated deliveries with same `X-Line-Signature` and body can re-enter the pipeline.  
  → **Resolution**: create a `webhook_deliveries` table with a short TTL and unique `idempotency_key`, and gate processing on atomic insert.

- **Missing retry/backoff for LINE notify**: transient failures cause missed employee confirmations.  
  → **Resolution**: add exponential backoff with jitter and a small, bounded retry count.

- **Missing index for recent-punch lookup**: transition checks (`latest punch is 'in'`) perform full scans under load.  
  → **Resolution**: add a composite index on `(tenant_id, line_user_id, clock_in_at DESC)` to make latest-punch queries efficient.

```sql
-- Idempotency for punches
ALTER TABLE punches ADD COLUMN IF NOT EXISTS idempotency_key TEXT;
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency
  ON punches (tenant_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- Prevent multiple clock-ins or clock-outs per user per calendar day
-- (optional but recommended for business correctness)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_unique_day_type
  ON punches (tenant_id, line_user_id, DATE(clock_in_at), type)
  WHERE type IN ('in', 'out');

-- Fast lookup for latest punch during transition checks
CREATE INDEX IF NOT EXISTS idx_punches_latest_lookup
  ON punches (tenant_id, line_user_id, clock_in_at DESC);

-- Durable webhook deduplication with TTL
CREATE TABLE IF NOT EXISTS webhook_deliveries (
  idempotency_key TEXT PRIMARY KEY,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expires_at TIMESTAMPTZ NOT NULL DEFAULT (NOW() + INTERVAL '10 minutes')
);
CREATE INDEX IF NOT EXISTS idx_webhook_deliveries_expires
  ON webhook_deliveries (expires_at);
```

### 2.2 Middleware: LINE webhook idempotency (`/opt/axentx/workio/server/src/middleware/lineWebhookMiddleware.ts`)

```ts
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';
import crypto from 'crypto';

export async function lineWebhookIdempotency(req: Request, res: Response, next: NextFunction) {
  const signature = req.get('X-Line-Signature') || '';
  const body = JSON.stringify(req.body);
  const idempotencyKey = crypto.createHash('sha256').update(signature + body).digest('hex');

  try {
    const { rows } = await pool.query(
      `INSERT INTO webhook_deliveries (idempotency_key, expires_at)
       VALUES ($1, NOW() + INTERVAL '10 minutes')
       ON CONFLICT (idempotency_key) DO NOTHING
       RETURNING idempotency_key`,
      [idempotencyKey]
    );

    // Duplicate delivery: acknowledge but skip processing
    if (rows.length === 0) {
      return res.status(200).json({ ok: true, duplicate: true });
    }

    (req as any).idempotencyKey = idempotencyKey;
    next();
  } catch (err) {
    // Fail open to avoid dropping legitimate webhooks; log and continue
    console.error('Idempotency check failed', err);
    next();
  }
}
```

### 2.3 Service: atomic punch creation with transition enforcement (`/opt/axentx/workio/server/src/services/punchService.ts`)

```ts
import { pool } from '../db';

export async function createPunch(params: {
  tenant_id: string;
  line_user_id: string;
  type: 'in' | 'out';
  clock_in_at: Date;
  latitude?: number;
  longitude?: number;
  idempotency_key: string;
}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Transition rule enforcement (atomic with insert)
    if (params.type === 'out') {
      const { rows } = await client.query(
        `SELECT type FROM punches
         WHERE tenant_id = $1 AND line_user_id = $2
         ORDER BY clock_in_at DESC LIMIT 1 FOR UPDATE`,
        [params.tenant_id, params.line_user_id]
      );
      const latest = rows[0];
      if (!latest || latest.type !== 'in') {
        throw new Error('Cannot clock out: no active clock-in found');
      }
    }

    // Atomic upsert using idempotency key
    const { rows } = await client.query(
      `INSERT INTO punches (tenant_id, line_user_id, type, clock_in_at, latitude, longitude, idempotency_key)
       VALUES ($1, $2, $3, $4, $5, $6, $7)
       ON CONFLICT (tenant_id, idempotency_key) DO UPDATE
       SET clock_in_at = EXCLUDED.clock_in_at
       RETURNING *`,
      [params.tenant_id, params.line_user_id, params.type, params.clock_in_at, params.latitude, params.longitude, params.idempotency_key]
    );

    await client.query('COMMIT');
    return rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 2.4 Utilities: LINE notify with exponential backoff (`/opt/axentx/workio/server/src/utils/lineNotify.ts`)

```ts
import axios from 'axios';

const MAX_RETRIES = 3;
const BASE_DELAY = 300; // ms

export async function sendLineNotify(token: string, message: string) {
  let lastError: any;
  for (let attempt = 0; attempt < MAX_RETRIES; attempt++) {
    try {
      await axios.post(
        'https://notify-api.line.me/api/notify',
        new URLSearchParams({ message }),
        {
          headers: { Authorization: `Bearer ${token}` },
          timeout: 10000,
        }
      );
      return { ok: true };
    } catch (err: any) {
      lastError = err;
      const delay = BASE_DELAY * Math.pow(2, attempt) + Math.random() * 100;
      await new Promise((r) => setTimeout(r, delay));
    }
  }
  return { ok: false, error: lastError?.message || 'Unknown error' };
}
```
