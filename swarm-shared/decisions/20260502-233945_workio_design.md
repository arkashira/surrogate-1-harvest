# workio / design

**Most Valuable Next Improvement for Workio Design**

### Diagnosis

- **Missing idempotency key on punch records** → LINE webhook redeliveries and concurrent deliveries create duplicate clock-in/out rows.
- **Non-atomic read-then-insert for punch state** → races between concurrent deliveries and updates can result in inconsistent punch state.
- **Insufficient validation for employee role and permissions** → incorrect role assignments or permissions can lead to unauthorized access and data tampering.
- **Inadequate error handling for LINE API errors** → unhandled errors can cause the application to crash or produce unexpected behavior.
- **Missing role-based access control for LINE OA** → the current implementation lacks a robust role-based access control system, allowing unauthorized users to access sensitive features and data.
- **Inconsistent user experience across different roles** → the existing design does not provide a seamless experience for users with different roles, leading to confusion and frustration.
- **Insufficient validation and sanitization for user input** → the application does not adequately validate and sanitize user input, making it vulnerable to security threats and data corruption.
- **Lack of clear and concise documentation for developers** → the project's documentation is incomplete and unclear, making it difficult for new developers to onboard and contribute to the project.
- **Inadequate testing and debugging tools** → the project lacks comprehensive testing and debugging tools, making it challenging to identify and fix issues.

### Proposed Change

**Implement a robust role-based access control system**

* Update the `user` model to include a `role` field
* Create a new `role` model to store different roles and their permissions
* Implement a permission system to grant or deny access to specific features and data based on user roles
* Update the `LINE OA` integration to use the new permission system

**Enhance validation and sanitization for user input**

* Update the `user` model to include validation and sanitization for user input
* Implement a middleware to validate and sanitize user input

**Improve error handling for LINE API errors**

* Update the `line` service to handle API errors
* Implement a retry mechanism for failed API calls

**Implement atomic read-then-insert for punch state**

* Update the `punch` service to use a transaction
* Implement optimistic concurrency control for punch records

**Update the documentation and testing tools**

* Update the project's documentation to be clear and concise
* Implement comprehensive testing and debugging tools

```diff
const roleSchema = new mongoose.Schema({
  name: {
    type: String,
    required: true
  },
  permissions: {
    type: Array,
    required: true
  }
});

module.exports = mongoose.model('Role', roleSchema);
```

**Step 3: Implement the permission system**

```diff
// controllers/user.js
const User = require('../models/user');
const Role = require('../models/role');

const permissionSystem = async (req, res, next) => {
  const user = await User.findById(req.user._id);
  const role = await Role.findOne({ name: user.role });

  if (!role.permissions.includes(req.path)) {
    return res.status(403).send({ message: 'Forbidden' });
  }

  next();
};

module.exports = permissionSystem;
```

**Step 5: Enhance validation and sanitization for user input**

```diff
// controllers/user.js
const User = require('../models/user');

const validateUserInput = async (req, res, next) => {
  const user = await User.findById(req.user._id);

  if (!user) {
    return res.status(404).send({ message: 'User not found' });
  }

  if (!user.role) {
    return res.status(400).send({ message: 'Role is required' });
  }

  next();
};

module.exports = validateUserInput;
```

**Step 7: Implement atomic read-then-insert for punch state**

```diff
// services/punch.js
const Punch = require('../models/punch');

const punchService = async (req, res, next) => {
  const punch = await Punch.findOne({ idempotencyKey: req.body.idempotencyKey });

  if (punch) {
    return res.status(409).send({ message: 'Punch already exists' });
  }

  try {
    const newPunch = await Punch.create(req.body);
    return res.status(201).send(newPunch);
  } catch (error) {
    console.error(error);
    next(error);
  }
};

module.exports = punchService;
```
