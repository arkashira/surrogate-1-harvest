## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP that authenticates to one AWS account and one GCP project, lists Lambda + Cloud Functions with schedules, persists them in Supabase Postgres, marks stale/drifted/orphaned entries, and exposes a minimal FastAPI endpoint to fetch findings. No frontend, no queue, no cache, deploy to CF Workers with local dev via wrangler + dockerized Postgres (Supabase local).

## Open questions

- What retention window for Schedule records balances cost and audit history needs?
- How will duplicates be fingerprinted (expression+region+code hash vs name similarity)?
