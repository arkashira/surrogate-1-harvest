# workio / design

## 1. Diagnosis

- **No idempotency guard on LINE webhook**: duplicate `punch` rows when LINE redelivers or concurrently delivers webhooks for the same clock-in/out event.
- **Non-atomic read-then-insert for punch state**: race between two concurrent webhooks can create two active punches or leave state inconsistent.
- **Missing unique constraint on active punch**: DB allows multiple active (clocked-in) records per user, making reporting and state recovery unreliable.
- **No compensating close on duplicate open punch**: if a duplicate open punch is created, there is no safe way to reconcile/close the stale row automatically.
- **Webhook handler mixes validation + side-effects**: makes idempotency and retries harder to reason about and test.

## 2. Proposed change

- **File**: `workio/server/src/routes/line/webhook.ts` (or equivalent webhook handler)
- **Scope**: add idempotency key (dedupe key) derived from LINE `deliveryId` + `timestamp` (or `webhookEvent.id`), enforce atomic upsert for punch state, and add a partial unique index to prevent multiple active punches per user.

## 3. Implementation

### 3.1 DB schema change (safe, additive)

```sql
-- workio/server/src/db/schema.sql
-- Prevent multiple active (clocked-in) punches per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_user_active
ON punches (user_id)
WHERE status = 'in';
```

### 3.2 Idempotency table (lightweight)

```sql
-- Store processed LINE event ids to guarantee at-most-once handling
CREATE TABLE IF NOT EXISTS line_webhook_events (
  id              TEXT PRIMARY KEY,          -- LINE event id or delivery id
  received_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  processed_at    TIMESTAMPTZ,
  payload_hash    TEXT
);
```

### 3.3 Webhook handler (atomic upsert + idempotency)

```ts
// workio/server/src/routes/line/webhook.ts
import { Router, Request, Response } from 'express';
import { pool } from '../../db';
import crypto from 'crypto';

const router = Router();

function hashPayload(body: any): string {
  return crypto.createHash('sha256').update(JSON.stringify(body)).digest('hex');
}

router.post('/line/webhook', async (req: Request, res: Response) => {
  const body = req.body;
  const events = body?.events;
  if (!Array.isArray(events) || events.length === 0) {
    return res.status(400).json({ error: 'invalid_payload' });
  }

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      const eventId = ev?.source?.userId && ev?.type
        ? `${ev.source.userId}:${ev.type}:${ev.message?.id || ev.timestamp || Date.now()}`
        : null;

      if (!eventId) continue;

      // Idempotency check
      const idempotencyCheck = await client.query(
        `SELECT 1 FROM line_webhook_events WHERE id = $1`,
        [eventId]
      );
      if (idempotencyCheck.rows.length > 0) {
        // Already processed — skip but continue processing other events
        continue;
      }

      // Record event as received (processed_at set after punch handled)
      await client.query(
        `INSERT INTO line_webhook_events (id, payload_hash) VALUES ($1, $2)`,
        [eventId, hashPayload(ev)]
      );

      // Only handle clock in/out via text or specific action
      const userId = ev.source.userId;
      const text = (ev.message?.text || '').trim().toLowerCase();
      const isClockIn = text === 'in' || text === 'clock in' || text === 'เข้างาน';
      const isClockOut = text === 'out' || text === 'clock out' || text === 'ออกงาน';

      if (!isClockIn && !isClockOut) {
        continue;
      }

      const now = new Date();
      const date = now.toISOString().split('T')[0];

      if (isClockIn) {
        // Try insert; if unique constraint violation, fetch existing active punch
        try {
          await client.query(
            `INSERT INTO punches (user_id, date, type, status, clock_in, location)
             VALUES ($1, $2, 'work', 'in', $3, $4)`,
            [userId, date, now.toISOString(), JSON.stringify({ lat: null, lng: null })]
          );
        } catch (err: any) {
          // If duplicate active punch exists, keep it (idempotent)
          if (err.code !== '23505') throw err;
        }
      } else if (isClockOut) {
        // Atomically close the active punch if exists
        const result = await client.query(
          `UPDATE punches
           SET status = 'out', clock_out = $1, updated_at = NOW()
           WHERE user_id = $2 AND status = 'in' AND date = $3
           RETURNING *`,
          [now.toISOString(), userId, date]
        );

        // If no active punch to close, create a closed record for traceability (best-effort)
        if (result.rowCount === 0) {
          await client.query(
            `INSERT INTO punches (user_id, date, type, status, clock_in, clock_out, location)
             VALUES ($1, $2, 'work', 'out', $3, $4, $5)`,
            [userId, date, now.toISOString(), now.toISOString(), JSON.stringify({ lat: null, lng: null })]
          );
        }
      }

      // Mark event processed
      await client.query(
        `UPDATE line_webhook_events SET processed_at = NOW() WHERE id = $1`,
        [eventId]
      );
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'processing_failed' });
  } finally {
    client.release();
  }
});

export default router;
```

## 4. Verification

1. **Apply migrations**  
   ```bash
   psql workio < server/src/db/schema.sql
   ```
   Confirm index exists:
   ```sql
   SELECT indexname, indexdef FROM pg_indexes WHERE tablename = 'punches';
   ```

2. **Idempotency test**  
   Send same LINE webhook payload twice (same event id) and verify:
   - Only one punch row created/updated.
   - `line_webhook_events` contains one entry for the event id.

3. **Concurrency test**  
   Fire two concurrent requests simulating simultaneous clock-in for same user:
   - Expect exactly one active punch (`status='in'`) due to unique partial index.
   - No duplicate rows.

4. **Clock-out behavior**  
   - Clock in → verify active punch.
   - Clock out → verify status updated and `clock_out` set.
   - Clock out again without active punch → verify new closed row created (best-effort) and no crash.

5. **Log + metrics**  
   - Check server logs for `ROLLBACK` or constraint violations.
   - Confirm webhook returns `200` for valid events and `400` for malformed payloads.
