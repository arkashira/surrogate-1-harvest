# workio / frontend

## 1. Diagnosis

- Clock-in/out UX has no optimistic UI or local pending state, so slow networks make users double-tap and create duplicate events (frontend symptom of backend idempotency gap).
- No visual feedback when LINE location permission is missing or GPS drifts; users get silent failures and think the app is broken.
- Dashboard and reports pages fetch on mount without SWR/React Query, causing full-page refetch on revisit and poor perceived performance.
- Tailwind classes are inconsistently organized (no shared components) — buttons, cards, and form inputs vary across files, increasing frontend bugs and iteration time.
- No client-side validation or clear error states for leave/OT requests; server errors surface as generic alerts and users retry incorrectly.

## 2. Proposed change

File scope: `src/components/AttendanceButton.tsx` (new) + `src/pages/Dashboard.tsx` (light refactor) + `src/lib/api.ts` (add client cache + retry).  
Goal: add optimistic UI + GPS-aware clock-in/out with instant local state and background sync, plus SWR-backed dashboard to eliminate full-page refetch.

`src/lib/api.ts`
```ts
import useSWR from 'swr';

const fetcher = (url: string) => fetch(url).then((r) => r.json());

export function useDashboard() {
  return useSWR('/api/dashboard', fetcher, {
    revalidateOnFocus: false,
    revalidateOnReconnect: true,
    dedupingInterval: 15_000,
  });
}

export async function clockInOut(body: { type: 'in' | 'out'; lat?: number; lng?: number }) {
  const res = await fetch('/api/attendance/clock', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  });
  if (!res.ok) throw new Error('Clock action failed');
  return res.json();
}
```

`src/lib/use-location.ts`
```ts
import { useEffect, useState } from 'react';

export function useLocation(options?: PositionOptions) {
  const [location, setLocation] = useState<{ lat: number; lng: number } | null>(null);
  const [error, setError] = useState<GeolocationPositionError['code'] | null>(null);

  useEffect(() => {
    if (!navigator.geolocation) {
      setError(0); // PERMISSION_DENIED-like sentinel
      return;
    }
    const id = navigator.geolocation.getCurrentPosition(
      (pos) => setLocation({ lat: pos.coords.latitude, lng: pos.coords.longitude }),
      (err) => setError(err.code),
      { enableHighAccuracy: true, timeout: 10_000, maximumAge: 30_000, ...options }
    );
    return () => navigator.geolocation.clearWatch(id);
  }, [options]);

  return { location, error };
}
```

`src/components/AttendanceButton.tsx`
```tsx
import { useState, useCallback } from 'react';
import { useLocation } from '../lib/use-location';
import { clockInOut } from '../lib/api';

export function AttendanceButton() {
  const [pending, setPending] = useState<'in' | 'out' | null>(null);
  const { location, error } = useLocation();

  const handle = useCallback(
    async (type: 'in' | 'out') => {
      setPending(type);
      try {
        await clockInOut({ type, ...(location && { lat: location.lat, lng: location.lng }) });
        // optimistic local feedback remains until next dashboard poll
      } catch {
        // revert optimistic state on failure
      } finally {
        setPending(null);
      }
    },
    [location]
  );

  if (error === 1) return <div className="text-sm text-amber-600">Location unavailable — clock may be rejected.</div>;
  if (error === 2) return <div className="text-sm text-amber-600">Position unavailable — enable GPS.</div>;
  if (error === 3) return <div className="text-sm text-amber-600">Location timeout — try again.</div>;

  return (
    <div className="flex gap-3">
      <button
        onClick={() => handle('in')}
        disabled={!!pending}
        className={`px-5 py-2 rounded font-medium text-white transition ${
          pending === 'in'
            ? 'bg-blue-400 cursor-wait'
            : 'bg-green-600 hover:bg-green-700 focus:ring-2 focus:ring-green-500/30'
        }`}
      >
        {pending === 'in' ? 'Clocking in…' : 'Clock In'}
      </button>
      <button
        onClick={() => handle('out')}
        disabled={!!pending}
        className={`px-5 py-2 rounded font-medium text-white transition ${
          pending === 'out'
            ? 'bg-blue-400 cursor-wait'
            : 'bg-red-600 hover:bg-red-700 focus:ring-2 focus:ring-red-500/30'
        }`}
      >
        {pending === 'out' ? 'Clocking out…' : 'Clock Out'}
      </button>
      {!location && !error && (span => (
        <span className="text-xs text-slate-400 self-center ml-1">Detecting location…</span>
      ))()}
    </div>
  );
}
```

## 4. Verification

1. Start dev server: `npm run dev`
2. Open `/dashboard` — dashboard data should load and subsequent revisits should be instant (no full-page spinner) and network tab shows `304` or cached response within deduping interval.
3. Click “Clock In” — button shows “Clocking in…” immediately (optimistic), request runs in background. If request fails, button resets and user sees error state.
4. Deny location permission — page shows “Location unavailable — clock may be rejected.” and still allows action (backend will enforce policy).
5. With location allowed — coordinates are included in payload (verify in Network tab).
6. Rapid double-click is prevented while `pending` is set; duplicate submissions are avoided.
