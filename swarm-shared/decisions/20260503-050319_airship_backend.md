### 1) Manifest generator (run once per date folder)

File: `scripts/generate_cdn_manifest.py`

- Use `huggingface_hub` to list the repo tree for the date folder.  
- Emit a small JSON with repo, date_folder, and per-file CDN URLs.  
- Keep it CLI-friendly and log-friendly for cron/CI.

```python
#!/usr/bin/env python3
"""
Generate CDN-only file manifest for a date folder in a HuggingFace dataset repo.
Run from orchestration host (Mac/CI) when HF API rate-limit window is clear.
"""
import json
import os
import sys
from pathlib import Path

try:
    from huggingface_hub import list_repo_tree
except ImportError:
    print("Install: pip install huggingface_hub")
    sys.exit(1)

HF_REPO = os.getenv("HF_DATASET_REPO", "axentx/surrogate-training-data")
DATE_FOLDER = os.getenv("DATE_FOLDER", "2026-04-29")  # e.g. batches/mirror-merged/2026-04-29
OUTPUT_PATH = os.getenv("MANIFEST_OUT", "file_manifest.json")

def main() -> None:
    items = list_repo_tree(
        repo_id=HF_REPO,
        path=DATE_FOLDER,
        repo_type="dataset",
        recursive=False,
    )

    files = []
    for item in items:
        if item.path.lower().endswith(".parquet"):
            files.append({
                "path": item.path,
                "cdn_url": f"https://huggingface.co/datasets/{HF_REPO}/resolve/main/{item.path}"
            })

    manifest = {
        "repo": HF_REPO,
        "date_folder": DATE_FOLDER,
        "files": files,
        "total_files": len(files),
    }

    out_path = Path(OUTPUT_PATH)
    out_path.write_text(json.dumps(manifest, indent=2))
    print(f"Wrote manifest with {len(files)} files to {out_path}")

if __name__ == "__main__":
    main()
```

Usage (Bash):

```bash
chmod +x scripts/generate_cdn_manifest.py
# Example cron (after rate-limit window clears):
# 0 3 * * * cd /opt/axentx/airship && ./scripts/generate_cdn_manifest.py >> logs/manifest.log 2>&1
```

### 2) CDN parquet loader (training side)

File: `surrogate/training/data/cdn_parquet_loader.py`

- Read manifest JSON.  
- Fetch each parquet via CDN URL with `requests` (no HF auth).  
- Parse with `pyarrow`/`pandas`.  
- Project to `{prompt, response}` at parse time and drop other columns to avoid schema mixing.  
- Provide an iterator (optionally batched) for training loops.

```python
from __future__ import annotations

import json
import os
from pathlib import Path
from typing import Iterator, Dict, Any

import pandas as pd
import pyarrow as pa
import pyarrow.parquet as pq
import requests
from tqdm import tqdm

MANIFEST_PATH = os.getenv("CDN_MANIFEST_PATH", "file_manifest.json")

def load_manifest(manifest_path: str | Path) -> dict:
    with open(manifest_path) as f:
        return json.load(f)

def cdn_parquet_iterator(
    manifest_path: str | Path = MANIFEST_PATH,
    batch_size: int = 512,
) -> Iterator[Dict[str, Any]]:
    """
    Yield {prompt, response} examples from parquet files via CDN URLs.
    No HF API/auth calls during training.
    """
    manifest = load_manifest(manifest_path)
    files = manifest.get("files", [])
    if not files:
        raise ValueError("No parquet files found in manifest.")

    for entry in tqdm(files, desc="Loading parquet via CDN"):
        cdn_url = entry["cdn_url"]
        resp = requests.get(cdn_url, timeout=60)
        resp.raise_for_status()

        table = pq.read_table(pa.BufferReader(resp.content))
        df = table.to_pandas()

        # Project to {prompt, response} at parse time
        if "prompt" not in df.columns or "response" not in df.columns:
            prompt_col = next((c for c in df.columns if "prompt" in c.lower()), None)
            response_col = next((c for c in df.columns if "response" in c.lower() or "completion" in c.lower()), None)
            if prompt_col is None or response_col is None:
                continue
            df = df.rename(columns={prompt_col: "prompt", response_col: "response"})

        df = df[["prompt", "response"]].dropna(subset=["prompt", "response"])

        for _, row in df.iterrows():
            yield {"prompt": str(row["prompt"]), "response": str(row["response"])}
```

Notes:
- If you prefer batched yields, accumulate rows and yield lists of dicts.  
- Keep this loader minimal and schema-strict to avoid mixing incompatible files.

### 3) Lightning Studio launcher (reuse + restart)

File: `surrogate/training/launch_studio.py`

- Detect a running Studio by name and reuse it.  
- If stopped, restart it.  
- Trigger training inside the Studio.  
- Keep launcher lightweight; training logic lives in `SurrogateTrainer.run`.

    def run(self):
        studio = find_running_studio(STUDIO_NAME)
        if studio is None:
            print(f"Starting new studio: {STUDIO_NAME}")
            self.studio = SurrogateTrainer(
                name=STUDIO_NAME,
                cloud_compute=MACHINE,
                cloud_build_config=None,
            )
        else:
            print(f"Reusing running studio: {STUDIO_NAME}")
            self.studio = studio

            print("Studio stopped. Restarting...")
            self.studio.start(cloud_compute=MACHINE)

        # Trigger training run (idempotent)
        self.studio.run()

app = LightningApp(SurrogateFlow())
