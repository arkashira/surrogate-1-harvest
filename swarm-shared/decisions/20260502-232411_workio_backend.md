# workio / backend

## Final Synthesis (Best Parts + Correctness + Actionability)

### 1. Root Cause (resolved)
- **Missing idempotency key**: no durable, unique `line_delivery_id` on punches, so redelivery or concurrent deliveries create duplicates.
- **Non-atomic state transition**: punch insert and employee clock-state are not locked together, allowing double-IN/double-OUT.
- **No durable event log**: raw LINE events are not persisted; retries lose provenance and can re-emit side-effects.
- **No transactional boundary**: partial writes possible between punch insert and state/log updates.
- **Retry hygiene**: handler does not distinguish duplicate (409) from transient (500), causing incorrect retry behavior.

### 2. Correct, Actionable Solution

#### 2.1 DB schema (single migration)
```sql
-- Idempotency key on punches
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS line_delivery_id VARCHAR(255);
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_delivery_id
  ON punches(line_delivery_id)
  WHERE line_delivery_id IS NOT NULL;

-- Durable LINE event log (for replay/audit)
CREATE TABLE IF NOT EXISTS line_events (
  id BIGSERIAL PRIMARY KEY,
  line_delivery_id VARCHAR(255) NOT NULL UNIQUE,
  event_type VARCHAR(64) NOT NULL,
  payload JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  processed_at TIMESTAMPTZ
);
```

#### 2.2 Service: `recordPunch` (idempotent, serialized)
File: `workio/server/src/services/punchService.ts`
```ts
import { pool } from '../db';

export interface PunchInput {
  lineDeliveryId: string;
  employeeId: number;
  tenantId: number;
  type: 'IN' | 'OUT';
  latitude?: number;
  longitude?: number;
  timestamp: Date;
}

export async function recordPunch(input: PunchInput) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency: exact delivery already recorded
    const dup = await client.query(
      `SELECT id, type, created_at FROM punches
       WHERE line_delivery_id = $1`,
      [input.lineDeliveryId]
    );
    if (dup.rows[0]) {
      await client.query('COMMIT');
      return { ...dup.rows[0], _idempotent: true };
    }

    // Serialize state: lock employee's latest punch
    const latest = await client.query(
      `SELECT id, type, created_at FROM punches
       WHERE employee_id = $1 AND tenant_id = $2
       ORDER BY created_at DESC
       LIMIT 1
       FOR UPDATE`,
      [input.employeeId, input.tenantId]
    );

    const last = latest.rows[0];
    if (last && last.type === input.type) {
      // Same-state duplicate (e.g., double-IN) — reject
      await client.query('ROLLBACK');
      const err: any = new Error(`Invalid transition: duplicate ${input.type}`);
      err.code = 'DUPLICATE_TRANSITION';
      throw err;
    }

    // Insert punch
    const res = await client.query(
      `INSERT INTO punches
        (employee_id, tenant_id, type, latitude, longitude, created_at, line_delivery_id)
       VALUES ($1,$2,$3,$4,$5,$6,$7)
       RETURNING id, type, created_at`,
      [
        input.employeeId,
        input.tenantId,
        input.type,
        input.latitude,
        input.longitude,
        input.timestamp,
        input.lineDeliveryId,
      ]
    );

    // Mark event processed (if using line_events)
    await client.query(
      `UPDATE line_events SET processed_at = NOW()
       WHERE line_delivery_id = $1`,
      [input.lineDeliveryId]
    );

    await client.query('COMMIT');
    return res.rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

#### 2.3 Webhook handler (durable + correct retry semantics)
File: `workio/server/src/routes/line/webhook.ts`
```ts
import express from 'express';
import { recordPunch } from '../../services/punchService';
import { pool } from '../../db';

const router = express.Router();

router.post('/line/webhook', async (req, res) => {
  const events = req.body.events || [];
  const results: Array<{ deliveryId: string; status: string }> = [];

  for (const ev of events) {
    const deliveryId = ev.deliveryId || ev.message?.id;
    if (!deliveryId) continue;

    try {
      // 1) Persist raw event (idempotent)
      await pool.query(
        `INSERT INTO line_events(line_delivery_id, event_type, payload)
         VALUES ($1,$2,$3)
         ON CONFLICT (line_delivery_id) DO NOTHING`,
        [deliveryId, ev.type, ev]
      );

      // 2) Process only punch-relevant messages (example: text IN/OUT)
      if (ev.type === 'message' && ev.message.type === 'text') {
        const text = ev.message.text.trim().toUpperCase();
        if (text === 'IN' || text === 'OUT') {
          const emp = await pool.query(
            `SELECT id, tenant_id FROM employees WHERE line_user_id = $1`,
            [ev.source.userId]
          );
          if (!emp.rows[0]) continue;

          await recordPunch({
            lineDeliveryId: deliveryId,
            employeeId: emp.rows[0].id,
            tenantId: emp.rows[0].tenant_id,
            type: text as 'IN' | 'OUT',
            timestamp: new Date(),
          });
        }
      }

      results.push({ deliveryId, status: 'ok' });
    } catch (err: any) {
      // Duplicate delivery or duplicate transition -> 200 (do not retry)
      if (
        err.code === '23505' ||
        err.message?.includes('duplicate') ||
        err.code === 'DUPLICATE_TRANSITION'
      ) {
        results.push({ deliveryId, status: 'duplicate' });
        continue;
      }

      // Transient/server error -> 500 to trigger LINE retry
      console.error('Webhook processing failed', { deliveryId, err });
      return res.status(500).json({ error: err.message });
    }
  }

  return res.json({ results });
});

export default router;
```

### 3. Verification (concrete tests)

1. **Idempotency**
   ```bash
   curl -X POST http://localhost:3000/line/webhook \
     -H "Content-Type: application/json" \
     -d '{"events":[{"deliveryId":"test-123","type":"message","source":{"userId":"U123"},"message":{"type":"text","text":"IN"}}]}'
   ```
   Repeat same request → second call returns 200 with `status: duplicate` and no new punch row.

2. **Constraint enforcement**
   ```sql
   INSERT INTO punches(employee_id, tenant_id, type, created_at, line_delivery_id)
   VALUES (1,1,'IN',NOW(),'dup-test');
   -- Second insert with same line_delivery_id must fail
   INSERT INTO punches(employee_id, tenant_id, type, created_at, line_delivery_id)
   VALUES (1,1,'IN',NOW(),'dup-test');
   -- Expected: ERROR: duplicate key value violates unique constraint
   ```

3. **Race/concurrency**
   Use load tool with same `deliveryId` and concurrent requests:
   ```bash
   ab -n 20 -c 10 -p payload.json -T application/json http://localhost:3000/line/webhook
   ```
   Verify exactly one punch row created and no double-IN/double-OUT rows.

4. **State transition correctness**
   - Sequence IN→OUT→IN should succeed.
   - Sequence IN→IN should be rejected (error `DUPLICATE_TRANSITION`), no second punch inserted.
