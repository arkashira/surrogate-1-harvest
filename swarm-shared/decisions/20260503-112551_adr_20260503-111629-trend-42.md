## Deployment
cf workers

## v0.1 scope
v0.1 ships in 2 weeks: Cloudflare Workers gateway exposing a single /v1/chat/completions endpoint that normalizes OpenAI-format requests, routes to OpenAI or Anthropic (configurable priority), enforces per-provider token-bucket limits stored in Redis, records token ledger and request logs in Supabase Postgres, exposes basic /health and /metrics endpoints, and supports zero-downtime provider failover with configurable timeouts and retry budgets.

## Open questions
- Which provider normalization profiles are required for v0.1 (OpenAI completions vs Anthropic vs Together)?
- What is the authoritative source-of-truth for token counting when provider schemas differ (e.g., embeddings vs chat)?
- How should global rate limits be expressed (tokens/min vs requests/min) and enforced across distributed Workers?
- What p95 latency SLA target (ms) is acceptable for v0.1 and how will it be measured (edge vs origin)?
- Will idempotency be client-supplied keys or gateway-generated, and what is the retention window?
