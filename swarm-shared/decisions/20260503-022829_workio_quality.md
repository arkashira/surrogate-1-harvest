# workio / quality

## 1. Diagnosis

- **Non-idempotent LINE webhook handling**: duplicate deliveries (LINE retries on timeout/5xx) can create multiple punch records for same employee/date because upsert is not atomic and lacks stable dedupe key.
- **Missing tenant-scoped uniqueness**: no unique constraint/index covering `(tenant_id, employee_id, punch_date, punch_type, source)` so duplicates can be inserted under concurrency.
- **No idempotency key in webhook payload processing**: no client-generated or server-stored idempotency key to short-circuit retries within a time window.
- **Race condition between select-then-insert**: current flow likely reads latest punch then inserts/updates; concurrent webhook deliveries can both read “no open punch” then insert two rows.
- **No retry-safe logging**: missing durable dedupe log (or lightweight table) to record processed LINE event IDs so retries are no-ops.

## 2. Proposed change

- **File**: `workio/server/src/services/punchService.ts` (or equivalent) — add atomic upsert with unique constraint and idempotency.
- **File**: `workio/server/src/db/schema.sql` — add unique constraint and optional `processed_webhook_events` table for LINE event idempotency.
- **Scope**: webhook handler `POST /webhook/line` → punch creation/update path.

## 3. Implementation

### Step 2 — Idempotent punch upsert in service

```ts
// workio/server/src/services/punchService.ts
import { db } from '../db';

export async function upsertPunchFromWebhook(params: {
  tenantId: string;
  employeeId: string;
  punchDate: string; // YYYY-MM-DD
  punchType: 'clock_in' | 'clock_out';
  location?: { lat: number; lng: number };
  lineEventId: string;
}) {
  const { tenantId, employeeId, punchDate, punchType, location, lineEventId } = params;

  // Use a transaction to ensure atomicity
  return await db.tx(async (t: any) => {
    // 1) Record processed event idempotently (skip if already processed)
    const inserted = await t.oneOrNone(
      `INSERT INTO processed_webhook_events (tenant_id, event_id, source)
       VALUES ($1, $2, 'line')
       ON CONFLICT (tenant_id, event_id, source) DO NOTHING
       RETURNING event_id`,
      [tenantId, lineEventId]
    );

    // If already processed, return existing punch (or null) — short-circuit
    if (!inserted) {
      return await t.oneOrNone(
        `SELECT * FROM punches
         WHERE tenant_id = $1 AND employee_id = $2 AND punch_date = $3 AND punch_type = $4
         ORDER BY created_at DESC LIMIT 1`,
        [tenantId, employeeId, punchDate, punchType]
      );
    }

    // 2) Atomic upsert: try update latest matching row, if none insert
    const upserted = await t.oneOrNone(
      `UPDATE punches
       SET updated_at = now(), location = $5
       WHERE tenant_id = $1 AND employee_id = $2 AND punch_date = $3 AND punch_type = $4
       RETURNING *`,
      [tenantId, employeeId, punchDate, punchType, location ? JSON.stringify(location) : null]
    );

    if (upserted) return upserted;

    return await t.one(
      `INSERT INTO punches (tenant_id, employee_id, punch_date, punch_type, location)
       VALUES ($1, $2, $3, $4, $5)
       RETURNING *`,
      [tenantId, employeeId, punchDate, punchType, location ? JSON.stringify(location) : null]
    );
  });
}
```

### Step 3 — Webhook handler integration

```ts
// workio/server/src/routes/lineWebhook.ts (or wherever handler lives)
import { Router } from 'express';
import { upsertPunchFromWebhook } from '../services/punchService';

const router = Router();

router.post('/webhook/line', async (req, res) => {
  try {
    const events = req.body.events || [];
    for (const ev of events) {
      if (ev.type === 'message' && ev.message?.type === 'text') {
        // Example: parse clock in/out from text or use payload
        // This mapping depends on your actual LINE command format
        const { tenantId, employeeId } = await resolveEmployeeFromLineSource(ev.source.userId);
        const punchDate = new Date().toISOString().slice(0, 10); // or from payload
        const isClockIn = /^(clock.?in|in|เข้า)$/i.test(ev.message.text);
        const punchType = isClockIn ? 'clock_in' : 'clock_out';

        await upsertPunchFromWebhook({
          tenantId,
          employeeId,
          punchDate,
          punchType,
          location: ev.message?.location || undefined,
          lineEventId: ev.eventId
        });
      }
    }
    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('LINE webhook error', err);
    // Important: return 5xx so LINE will retry (only for transient failures)
    res.status(500).json({ error: 'internal' });
  }
});

export default router;
```

### Step 4 — Ensure handler is idempotent-safe at HTTP level

- Return `200` quickly after dedupe (processed event exists) so LINE stops retrying.
- Only return `5xx` for true transient errors (DB unavailable) to allow safe retry.
