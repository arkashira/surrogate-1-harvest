## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: single-symbol, single-broker pipeline (research→signal→risk→execution) with deterministic Redis Streams pipeline, Postgres audit trail, replay endpoint, and basic risk guardrails (max position size, stop limits). Includes API to submit runs, query status, and replay a PipelineRun end-to-end. No UI; CLI + API only. Deploy on CF Workers + Supabase free tier; Redis on free-tier host.

## Open questions
- Which brokers/data vendors must be supported in v0.1 and what auth flows do they require?
- What deterministic replay fidelity is required (event-level vs state-level) for compliance?
- What is the acceptable consistency model for hot state (Redis) vs durable state (Postgres) under failover?
- Do we need per-pipeline idempotency keys and how will they be garbage-collected?
- What are the exact latency budgets for each stage (research→signal, signal→risk, risk→execution) to meet <500ms median?
