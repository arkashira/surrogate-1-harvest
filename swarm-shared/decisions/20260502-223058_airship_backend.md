# airship / backend

## Decision: Ship `airship discover` (backend orchestrator)

**Scope**: Minimal, high-value backend executable that produces tagged research outputs + CDN-only training manifest with **one HF API call per run** and zero runtime API calls during training.

---

## Implementation Plan (<2h)

### 1. Create executable `/opt/axentx/airship/bin/airship-discover`
- Bash shebang, `chmod +x`
- Cron-safe: all paths absolute, idempotent, no interactive prompts
- Exit codes: 0=ok, 1=business research failed, 2=HF list failed, 3=write failed

### 2. Workflow
1. Run `granite-business-research.sh` if present → capture stdout/stderr → tag outputs `#business-research #knowledge-rag #graph`
2. Query top hub (e.g., "MOC") via knowledge-rag tooling if available → tag `#knowledge-rag #graph #hub`
3. Single HF API call: `list_repo_tree(path=..., recursive=False)` for today’s date folder (YYYY-MM-DD) → save to `manifest/today-file-list.json`
4. Generate training script `train-cdn-only.py` that reads the manifest and downloads via CDN URLs only (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`)
5. Emit summary JSON to `out/airship-discover-<ts>.json` with paths, counts, HF repo, date, tags

### 3. HF Rate-limit strategy
- One `list_repo_tree` call per run (non-recursive per folder)
- If 429: wait 360s and retry once, then abort with clear log
- Use CDN-only downloads in training (zero API calls during data load)

### 4. Outputs
- `out/airship-discover-<ts>.json` — summary + tags
- `manifest/today-file-list.json` — HF file list for training
- `train-cdn-only.py` — generated training script (CDN-only)
- Tagged logs in `logs/` with TSV lines for easy grep

### 5. Cron integration notes
- Shebang `#!/usr/bin/env bash`
- Cron: `SHELL=/bin/bash`
- Example crontab line (user-level):
  ```
  SHELL=/bin/bash
  0 2,8,14,20 * * * /opt/axentx/airship/bin/airship-discover >> /opt/axentx/airship/logs/cron.log 2>&1
  ```

log() {
  local level="$1"
  shift
  local message="$*"
  local line="${TS}\t${level}\tairship-discover\t${message}"
  echo -e "${line}" >> "${LOG_DIR}/airship-discover.log"
  echo "[${level}] ${message}" >&2
}

tag_line() {
  local tags="$1"
  local content="$2"
  echo -e "${TS}\t${tags}\t${content}" >> "${LOG_DIR}/tagged.tsv"
}

ensure_dirs() {
  mkdir -p "${OUT_DIR}" "${MANIFEST_DIR}" "${LOG_DIR}"
}

# ---- Step 1: Business research ----
run_business_research() {
  local script="${BASE_DIR}/granite-business-research.sh"
  if [[ -x "${script}" ]]; then
    log "INFO" "Running granite-business-research.sh"
    if output=$("${script}" 2>&1); then
      log "INFO" "granite-business-research.sh completed"
      tag_line "#business-research #knowledge-rag #graph" "${output}"
      echo "${output}" > "${OUT_DIR}/business-research-${TS}.out"
      return 0
    else
      log "ERROR" "granite-business-research.sh failed: ${output}"
      tag_line "#business-research #knowledge-rag #graph" "FAILED: ${output}"
      return 1
    fi
  else
    log "INFO" "No executable granite-business-research.sh found, skipping"
    return 0
  fi
}

# ---- Step 2: Top hub insight ----
run_top_hub_insight() {
  # Placeholder: if knowledge-rag CLI exists, query top hub (e.g., MOC)
  local kr_cli="${BASE_DIR}/../knowledge-rag/cli.sh"
  if [[ -x "${kr_cli}" ]]; then
    log "INFO" "Querying top hub via knowledge-rag"
    if hub_out=$("${kr_cli}" top-hub 2>&1); then
      tag_line "#knowledge-rag #graph #hub" "${hub_out}"
      log "INFO" "Top hub insight captured"
    else
      log "WARN" "knowledge-rag top-hub failed: ${hub_out}"
    fi
  else
    log "INFO" "knowledge-rag CLI not found, skipping top-hub"
  fi
}
