## v0.1 scope
v0.1 in 2 weeks: CLI-only discovery and audit for a single customer account — reads Lambda + Cloud Functions schedules, persists to local SQLite (migrates to Supabase Postgres), detects duplicates (same function+expression), drifts (expression or target changes), orphans (no target function), and stale schedules (no runs in 30 days); outputs JSON report and retirements checklist.

## Open questions
- How will customers grant read-only cloud credentials (assume IAM roles vs. service keys)?
- Which identity separates multi-tenant data (customer_id from SSO claim or API key)?
- What retention/staleness thresholds define 'orphan' (no function, no logs for X days) and 'stale' (no runs for Y days)?
- Will schedule discovery run continuously (workers cron) or triggered via CLI/CI; what triggers recompute?
