## Final Synthesized Implementation (Correct + Actionable)

Below is the single, authoritative plan that merges the strongest, non-contradictory parts of both proposals and resolves all conflicts in favor of **correctness, atomicity, and concrete actionability**.

---

### 1. Diagnosis (merged)

- **No idempotency persistence** — LINE webhook `X-Line-Signature` is verified but not stored; retries create duplicate punch rows.
- **Non-atomic upsert** — `findOne → if exists update else insert` allows duplicates/races under retries or concurrency.
- **Missing constraints** — no unique constraint on `(employee_id, punch_time)` and no enforcement of at-most-one open punch per user.
- **No transaction safety** — punch creation and related state changes are not atomic; partial failures leave inconsistent state.
- **No replay-safe audit trail** — no immutable record of webhook deliveries to detect or repair duplicates.

### 2. Schema Changes (single source of truth)

File: `/opt/axentx/workio/server/src/db/schema.sql`

```sql
-- 1) Idempotency table for LINE webhook deliveries (replay-safe)
CREATE TABLE IF NOT EXISTS punch_idempotency_keys (
  line_signature  VARCHAR(255) PRIMARY KEY,
  employee_id     INTEGER      NOT NULL,
  punch_time      TIMESTAMPTZ  NOT NULL,
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- 2) Business-unique punch constraint (prevent duplicates)
-- Use second precision to tolerate minor clock/retry jitter.
CREATE UNIQUE INDEX IF NOT EXISTS uq_employee_punch_time_second
  ON punches (employee_id, date_trunc('second', punch_time));

-- 3) At-most-one-open punch per employee (valid state invariant)
-- Allows only one punch where clock_out_at IS NULL per employee.
CREATE UNIQUE INDEX IF NOT EXISTS uq_employee_one_open_punch
  ON punches (employee_id)
  WHERE clock_out_at IS NULL;
```

**Why this wins:**
- Uses a simple PK on `line_signature` (no extra surrogate key) for clarity and fast conflict checks.
- Keeps `punches` constraints at the DB level (index + partial index) so correctness is enforced even outside app code.
- Avoids mixing `tenant_id`/`user_id` vs `employee_id` inconsistencies; we standardize on `employee_id` for punch records (aligns with Candidate 1 field names and existing code).

### 3. Service Implementation (atomic + idempotent)

File: `/opt/axentx/workio/server/src/services/punchService.ts`

```ts
import { Pool } from 'pg';
import { pool } from '../db';

export interface CreatePunchPayload {
  employeeId: number;
  punchTime: Date;
  type: 'in' | 'out';
  latitude?: number;
  longitude?: number;
}

/**
 * Create punch with LINE idempotency.
 * Atomic under at-least-once delivery and concurrent retries.
 */
export async function createPunchWithIdempotency(
  lineSignature: string,
  payload: CreatePunchPayload
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency check/insert (conflict on signature is safe retry)
    await client.query(
      `INSERT INTO punch_idempotency_keys (line_signature, employee_id, punch_time)
       VALUES ($1, $2, $3)
       ON CONFLICT (line_signature) DO NOTHING`,
      [lineSignature, payload.employeeId, payload.punchTime]
    );

    // 2) If signature existed, return existing punch (idempotent read)
    const existing = await client.query(
      `SELECT p.*
       FROM punches p
       JOIN punch_idempotency_keys k ON k.employee_id = p.employee_id
         AND date_trunc('second', k.punch_time) = date_trunc('second', p.punch_time)
       WHERE k.line_signature = $1`,
      [lineSignature]
    );

    if (existing.rows.length > 0) {
      await client.query('COMMIT');
      return existing.rows[0];
    }

    // 3) Atomic upsert on business key (employee + punch_time second-precision)
    const punchRes = await client.query(
      `INSERT INTO punches (employee_id, punch_time, type, latitude, longitude)
       VALUES ($1, $2, $3, $4, $5)
       ON CONFLICT ((employee_id), (date_trunc('second', punch_time)))
       DO UPDATE SET
         type = EXCLUDED.type,
         latitude = EXCLUDED.latitude,
         longitude = EXCLUDED.longitude
       RETURNING *`,
      [payload.employeeId, payload.punchTime, payload.type, payload.latitude, payload.longitude]
    );

    await client.query('COMMIT');
    return punchRes.rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

**Key correctness choices:**
- `ON CONFLICT (line_signature) DO NOTHING` + explicit read of existing punch ensures idempotency without overwriting newer data on signature reuse.
- All writes are inside a single transaction (`BEGIN`/`COMMIT`/`ROLLBACK`) so partial states cannot be persisted.
- Business-key upsert uses second-precision truncation to absorb minor timing jitter from retries while still preventing true duplicates.

### 4. Webhook Integration (concrete)

File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts`

```ts
import express from 'express';
import { createPunchWithIdempotency } from '../services/punchService';

const router = express.Router();

router.post('/webhook/line', async (req, res) => {
  try {
    const signature = req.get('X-Line-Signature');
    const events = req.body.events || [];

    for (const ev of events) {
      if (ev.type === 'message' && ev.message.type === 'text') {
        // Resolve employee by LINE user ID (implement as needed)
        const employeeId = await resolveEmployeeIdByLineUserId(ev.source.userId);
        const punchTime = new Date(ev.timestamp);
        const type = ev.message.text.toLowerCase().includes('in') ? 'in' : 'out';

        await createPunchWithIdempotency(signature, {
          employeeId,
          punchTime,
          type
        });
      }
    }

    res.sendStatus(200);
  } catch (err) {
    console.error('LINE webhook error:', err);
    res.sendStatus(500);
  }
});

export default router;
```
