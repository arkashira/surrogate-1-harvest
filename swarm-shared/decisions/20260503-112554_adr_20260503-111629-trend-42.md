## v0.1 scope
v0.1 in 2 weeks: Cloudflare Workers gateway exposing OpenAI-compatible /v1/chat/completions and /v1/completions endpoints; adapters for OpenAI and Anthropic; Postgres-backed provider configs and quotas; KV-backed model list and rate-limit cache; failover by ordered provider list; basic auth (API key) and row-level audit; contract tests for each provider; latency and token counters in GatewayRequest; free-tier hosting on CF Workers + Supabase free tier.

## Open questions
- Which provider APIs must be supported in v0.1 (OpenAI, Anthropic, Cohere, local OSS)?
- What is the authoritative source for per-provider model lists and pricing (static file vs. live scrape)?
- How will global rate limits and quotas be enforced atomically at high concurrency (Postgres advisory locks vs. KV atomic counters)?
- What latency budget (ms) for p95 gateway overhead vs. provider latency will define SLA compliance?
- Will streaming responses be proxied and how will usage be metered mid-stream?
- What auth scheme will clients use (API keys, JWT) and how will key rotation be handled?
