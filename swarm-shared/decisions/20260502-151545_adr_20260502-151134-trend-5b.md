## Deployment
cf workers (free-tier) + Supabase (free-tier Postgres + edge functions) + HuggingFace Space (optional Redis-backed worker for cache/orchestrator fallback)

## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research→signal→risk check→execution) with Postgres-backed immutable audit log, Redis-backed signal cache, Cloudflare Queue task orchestration, and one broker connector (sandbox/paper trading) plus one sample compliance guardrail (position limit). Includes replay endpoint to re-run a workflow_run deterministically and basic dashboardless API for status/audit retrieval.

## Open questions
- What deterministic replay fidelity is required (full event sourcing vs snapshot+log) and retention window?
- Which broker/OMS/EMS connectors are must-have for v0.1 and what auth/credential model will be supported (OAuth, API keys, HSM)?
- Compliance rule engine: should rules be expressed in JSON/YAML DSL or embedded Python with sandboxing?
- Latency SLO for end-to-end orchestration (sub-100ms, sub-500ms) and acceptable queueing/backpressure strategy?
- How will fee-generating status be computed and surfaced for NSM weighting (broker fee schedule, rebates, or internal attribution)?
