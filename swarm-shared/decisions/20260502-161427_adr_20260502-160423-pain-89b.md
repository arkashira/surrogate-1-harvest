## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust daemon (Linux x86_64) that exposes a SOCKS5 listener on localhost, tunnels traffic through a CF Worker egress using TLS-fragmentation and domain-fronting, implements a strict kill-switch (fail-closed) on TUN/interface changes, stores profiles and session logs in local SQLite, and emits privacy-preserving client-side telemetry (salted hashes, no PII) to measure WAU-ZB; includes CLI setup wizard and basic integration tests for stealth path and block detection fallback.

## Open questions
- Which DPI signatures are most prevalent in target regions (SNI, MTU/TCP fingerprint, packet timing)?
- Can we rely solely on CF Workers free tier for domain-front/obfs egress under expected load?
- How will clients bootstrap initial egress endpoints without a centralized directory that can be blocked?
- What is the acceptable trade-off between obfuscation overhead (latency/CPU) and stealth on low-end devices?
- How will WAU-ZB telemetry be collected and verified client-side without compromising user privacy?
