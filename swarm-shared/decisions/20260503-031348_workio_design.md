# workio / design

## 1. Diagnosis

- Missing **LINE webhook signature verification** (`X-Line-Signature`) on `/webhook/line` allows spoofed/replayed events to be accepted as valid clock-in/out or leave requests.
- No **idempotency protection** — LINE retries on 5xx/timeouts can create duplicate clock-in/out records and leave/OT requests because handlers don’t check `line_event_id` uniqueness.
- Clock-in/out UX lacks immediate feedback and optimistic state; users can double-tap and queue duplicate punches while waiting on LINE/webhook round-trip.
- No local pending queue for offline/weak-network punches; failed requests lose user intent and require manual recovery.
- Backend handlers accept raw event payloads without validating required fields (`userId`, `eventType`, `timestamp`, `location`) before processing.

## 2. Proposed change

Add **LINE signature verification + idempotency by `line_event_id`** in a single, minimal change:

- File: `/opt/axentx/workio/server/src/routes/lineWebhook.ts` (create if missing) or existing webhook handler
- Add middleware `verifyLineSignature()` that validates `X-Line-Signature` using HMAC-SHA256 with `CHANNEL_SECRET`.
- Add DB unique constraint/index on `line_event_id` (or application-level idempotency check) in the events/punches table.
- Wrap handlers so duplicate `line_event_id` returns `200 OK` (idempotent success) instead of creating duplicates or erroring.

`/opt/axentx/workio/server/src/middleware/lineAuth.ts`
```ts
import crypto from 'crypto';

const CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

if (!CHANNEL_SECRET) {
  throw new Error('LINE_CHANNEL_SECRET is required');
}

export function verifyLineSignature(req: any, res: any, next: any) {
  const signature = req.headers['x-line-signature'] as string | undefined;
  if (!signature) {
    return res.status(401).json({ error: 'Missing X-Line-Signature' });
  }

  const body = JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');

  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected))) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  next();
}
```

`/opt/axentx/workio/server/src/routes/lineWebhook.ts`
```ts
import { Router } from 'express';
import { verifyLineSignature } from '../middleware/lineAuth';
import { pool } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = Router();

// Idempotent upsert helper
async function upsertPunch(lineEventId: string, userId: string, eventType: 'in' | 'out', ts: Date, lat?: number, lng?: number) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    // Check for existing by line_event_id
    const existing = await client.query(
      'SELECT id FROM punches WHERE line_event_id = $1',
      [lineEventId]
    );
    if (existing.rows.length > 0) {
      await client.query('COMMIT');
      return { created: false, id: existing.rows[0].id };
    }

    const id = uuidv4();
    await client.query(
      `INSERT INTO punches (id, user_id, event_type, punched_at, lat, lng, line_event_id)
       VALUES ($1, $2, $3, $4, $5, $6, $7)`,
      [id, userId, eventType, ts, lat, lng, lineEventId]
    );
    await client.query('COMMIT');
    return { created: true, id };
  } catch (err: any) {
    await client.query('ROLLBACK');
    // Unique violation from race -> treat as duplicate
    if (err.code === '23505') {
      const dup = await client.query('SELECT id FROM punches WHERE line_event_id = $1', [lineEventId]);
      return { created: false, id: dup.rows[0]?.id || null };
    }
    throw err;
  } finally {
    client.release();
  }
}

router.post('/webhook/line', verifyLineSignature, async (req, res) => {
  try {
    const events = req.body.events || [];
    for (const ev of events) {
      const lineEventId = ev.source?.userId + ':' + ev.timestamp + ':' + ev.type;
      // Normalize to a stable id — use event id from LINE if available, else deterministic
      const stableId = ev.source?.userId && ev.timestamp
        ? `line:${ev.source.userId}:${ev.timestamp}:${ev.type}`
        : `line:${Buffer.from(JSON.stringify(ev)).toString('base64')}`;

      if (ev.type === 'message' && ev.message?.type === 'text') {
        const text = ev.message.text.toLowerCase();
        const userId = ev.source.userId;
        const ts = new Date(ev.timestamp);

        if (text.includes('เข้า') || text.includes('in')) {
          await upsertPunch(stableId, userId, 'in', ts);
        } else if (text.includes('ออก') || text.includes('out')) {
          await upsertPunch(stableId, userId, 'out', ts);
        }
        // Extend for leave/OT similarly with idempotent upserts
      }
    }
    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('LINE webhook error', err);
    res.status(500).json({ error: 'Internal server error' });
  }
});

export default router;
```

## 4. Verification

1. **Signature verification**  
   - Start server and send a POST to `/webhook/line` without `X-Line-Signature` → expect `401`.  
   - Send with a wrong signature → expect `401`.  
   - Compute correct HMAC-SHA256 of the JSON body using `CHANNEL_SECRET` and include as `X-Line-Signature` → expect `200`.

2. **Idempotency**  
   - Simulate a LINE event payload and POST twice with same computed `line_event_id` (or same deterministic stableId).  
   - Check DB: only one punch row created; second request returns `200` without error and does not insert a duplicate.

3. **UX + frontend smoke test**  
   - On the frontend, tap clock-in and observe optimistic state (button disabled/spinner) and confirm punch appears in list after webhook round-trip.  
   - Rapid double-tap should not create duplicate punches (idempotency + optimistic locking prevents duplicates).

4. **Offline/retry simulation**  
   - Disable network, tap clock-in (should queue locally if you add a pending queue later), then re-enable and verify the queued request eventually succeeds once and does not duplicate on retry.
