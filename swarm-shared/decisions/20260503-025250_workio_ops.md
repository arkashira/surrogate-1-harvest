# workio / ops

## 1. Diagnosis

- LINE Messaging API retries on non-2xx or slow responses (default 5 retries over ~20 min) create duplicate clock-in/out events because the webhook handler can be re-entered while an earlier request is still processing or after partial failure.
- No server-side idempotency key or deduplication window for incoming LINE webhook events; tenant+userId+event timestamp collisions are possible across retries.
- Punch writes lack an atomic “first-writer-wins” guard (unique constraint) so concurrent/retried requests can insert two rows for the same logical punch.
- No structured request-id / trace-id propagation between LINE webhook entry and downstream inserts/logs, making duplicate detection and incident triage slow.
- No lightweight operational metric or log line that exposes “duplicate punch attempts” or “webhook re-entrancy” for alerting.

## 2. Proposed change

- **File**: `workio/server/src/routes/line/webhook.ts` (or equivalent route handler)
- **Scope**: Add idempotency key derived from LINE `deliveryId` (or `timestamp+eventType+userId`) and a tenant-scoped unique constraint on punches (`tenantId`, `userId`, `direction`, `dedupKey`). Make handler idempotent and fast-ack (return 200 quickly) after dedup check.
- **Side files**:
  - `workio/server/src/db/schema.sql` — add unique constraint/index.
  - `workio/server/src/middleware/requestId.ts` — add request-id middleware and logging.
  - `workio/server/src/lib/metrics.ts` — lightweight counters for duplicates/errors.

## 3. Implementation

### 3.1 DB schema change

```sql
-- workio/server/src/db/schema.sql
-- Add dedup key column and unique guard for punches
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS dedup_key VARCHAR(128);

CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_tenant_user_dedup
  ON punches (tenant_id, user_id, direction, dedup_key)
  WHERE dedup_key IS NOT NULL;
```

### 3.2 Request-id middleware (logging/tracing)

```ts
// workio/server/src/middleware/requestId.ts
import { Request, Response, NextFunction } from 'express';
import { randomUUID } from 'crypto';

export function requestIdMiddleware(req: Request, res: Response, next: NextFunction) {
  const id = req.headers['x-request-id'] || randomUUID();
  (req as any).requestId = id;
  res.setHeader('X-Request-Id', id as string);
  next();
}
```

Register in your main server file (e.g., `server/src/index.ts`):

```ts
import { requestIdMiddleware } from './middleware/requestId';
app.use(requestIdMiddleware);
```

### 3.4 Idempotent LINE webhook handler

```ts
// workio/server/src/routes/line/webhook.ts
import { Router, Request, Response } from 'express';
import { pool } from '../../db';
import { Metrics } from '../../lib/metrics';

const router = Router();

// Helper: derive stable dedup key from LINE event
function dedupKey(event: any): string | null {
  // LINE provides a deliveryId that is unique per webhook delivery
  if (event?.deliveryId) return `line:${event.deliveryId}`;
  // Fallback: timestamp+type+userId (best-effort)
  const ts = event?.timestamp || Date.now();
  const type = event?.type || 'unknown';
  const uid = event?.source?.userId || 'unknown';
  return `line:fallback:${ts}:${type}:${uid}`;
}

router.post('/webhook/line', async (req: Request, res: Response) => {
  const requestId = (req as any).requestId || 'unknown';
  const body = req.body;

  // LINE may batch events
  const events = Array.isArray(body.events) ? body.events : [];
  if (events.length === 0) return res.sendStatus(200);

  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const event of events) {
      const dkey = dedupKey(event);
      const tenantId = event?.source?.groupId || event?.source?.roomId || 'default'; // adapt to your tenant resolution
      const userId = event?.source?.userId;
      const type = event?.type;

      Metrics.webhook_received(tenantId);

      if (!userId || !type) continue;

      // Map LINE event to punch direction (simplified)
      let direction: 'in' | 'out' | null = null;
      if (type === 'message' && event?.message?.text?.toLowerCase().includes('in')) direction = 'in';
      if (type === 'message' && event?.message?.text?.toLowerCase().includes('out')) direction = 'out';
      // You may instead use a dedicated clock-in command or payload

      if (!direction) continue;

      // Idempotency check + insert in one step (ON CONFLICT DO NOTHING)
      const insertText = `
        INSERT INTO punches (tenant_id, user_id, direction, created_at, dedup_key)
        VALUES ($1, $2, $3, NOW(), $4)
        ON CONFLICT ON CONSTRAINT idx_punches_tenant_user_dedup DO NOTHING
        RETURNING id
      `;
      const result = await client.query(insertText, [tenantId, userId, direction, dkey]);

      if (result.rowCount === 0) {
        Metrics.duplicate_punch(tenantId, userId, dkey || '');
      }
    }

    await client.query('COMMIT');
    // Fast-ack to prevent LINE retries
    res.sendStatus(200);
  } catch (err) {
    await client.query('ROLLBACK');
    Metrics.webhook_error(undefined, err);
    // Return 200 to avoid LINE retries only if you are sure partial failures are handled;
    // otherwise use 5xx carefully (may cause retries). For safety, return 200 after logging.
    console.error('Webhook error', err);
    res.sendStatus(200);
  } finally {
    client.release();
  }
});

export default router;
```
