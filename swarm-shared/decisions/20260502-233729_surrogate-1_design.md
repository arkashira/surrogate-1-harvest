### Step 1: Pre-flight snapshot generation

Create a new script `bin/snapshot.sh` that generates a pre-flight snapshot of the dataset:

```bash
#!/bin/bash

# Define dataset repo and path
REPO="axentx/surrogate-1-training-pairs"
PATH="public"

# Define snapshot directory
SNAPSHOT_DIR="batches/snapshot"

# Create snapshot directory if it doesn't exist
mkdir -p "$SNAPSHOT_DIR"

# Download and process each file in the dataset
for date in $(curl -s https://huggingface.co/datasets/$REPO/resolve/main/$PATH | jq -r '.[] | @base64'); do
  echo "$date"
  file=$(echo "$date" | base64 --decode)
  slug=$(echo "$file" | jq -r '.slug')
  filename=$(echo "$file" | jq -r '.filename')
  jsonl=$(echo "$file" | jq -r '.jsonl')
  slug_hash=$(echo "$slug" | sha256sum | cut -d' ' -f1)
  snapshot_path="$SNAPSHOT_DIR/$slug_hash/$filename"
  mkdir -p "$snapshot_path"
  curl -s https://huggingface.co/datasets/$REPO/resolve/main/$PATH/$jsonl | jq -r '.[] | @base64' | base64 --decode > "$snapshot_path/$filename.jsonl"
done
```

### Step 5: Update the GitHub Actions workflow to use the pre-flight snapshot

Update the GitHub Actions workflow to use the pre-flight snapshot instead of loading the dataset at runtime:

```yml
name: Ingest

on:
  workflow_dispatch:
    inputs:
      SHARD_ID:
        description: Shard ID
        required: true
      ITERATION_TIMESTAMP:
        description: Iteration timestamp
        required: true

jobs:
  ingest:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Hugging Face token
        env:
          HF_TOKEN: ${{ secrets.HF_TOKEN }}
        run: |

          # Set the snapshot directory
          SNAPSHOT_DIR="batches/snapshot"

          # Set the shard ID
          SHARD_ID=${SHARD_ID}

          # Set the iteration timestamp
          ITERATION_TIMESTAMP=${ITERATION_TIMESTAMP}

          # Set the output file path
          OUTPUT_FILE="$SNAPSHOT_DIR/public-merged/$ITERATION_TIMESTAMP-$SHARD_ID-$RANDOM.jsonl"

          # Copy the pre-flight snapshot to the output file
          cp "$SNAPSHOT_DIR/*.parquet" "$OUTPUT_FILE"

          # Dedup the pairs using the central md5 hash store
          dedup.py "$OUTPUT_FILE"
```

This implementation plan should take ≤2h to complete and provides a high-value change by replacing runtime `load_dataset(streaming=True)` + recursive `list_repo_tree` in `bin/dataset-enrich.sh` with a deterministic pre-flight snapshot + CDN download.
