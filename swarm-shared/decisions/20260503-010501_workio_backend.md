### Final Consolidated Solution  
*(Strongest parts from both proposals, resolved for correctness + concrete actionability)*

---

#### 1) Diagnosis (merged, prioritized)
- **Missing DB-level guard**: no partial unique index to enforce *at most one open punch per user per calendar day* (`clock_out_at IS NULL AND deleted_at IS NULL`).  
- **Race-prone app logic**: `find-then-insert` in clock-in handler is unsafe under concurrency and LINE webhook retries.  
- **No idempotency for webhooks**: LINE retries can create duplicates even with an index if app retries after transient errors.  
- **Soft-deletes must be excluded** from uniqueness so cancelled/deleted punches don’t block new ones.  
- **No atomic upsert path**: application must use DB-enforced atomic insert with conflict handling, not app-layer checks.

---

#### 2) Proposed change (minimal, high-leverage)
- **File**: `/opt/axentx/workio/server/src/db/schema.sql`  
  - Add partial unique index covering `deleted_at`.  
- **File**: `server/src/services/punchService.js` (or equivalent data layer)  
  - Replace `find-then-insert` with atomic `INSERT ... ON CONFLICT DO NOTHING` + fetch fallback inside a transaction.  
- **Optional but recommended**: add lightweight `webhook_idempotency` table and check in LINE webhook handler.

---

#### 3) Implementation (concrete, copy-paste ready)

**Step 1 — Schema: partial unique index** (`schema.sql`)
```sql
-- Ensure at most one open, non-deleted punch per user per calendar day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_day
  ON punches (user_id, DATE(clock_in_at))
  WHERE clock_out_at IS NULL AND deleted_at IS NULL;
```

**Step 2 — Atomic clock-in service** (`punchService.js`)
```js
// server/src/services/punchService.js
async function clockIn(userId, clockInAt = new Date(), metadata = {}) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Try atomic insert; rely on partial unique index for conflict detection
    const insertRes = await client.query(
      `INSERT INTO punches (user_id, clock_in_at, clock_out_at, deleted_at, metadata)
       VALUES ($1, $2, NULL, NULL, $3)
       ON CONFLICT DO NOTHING
       RETURNING *`,
      [userId, clockInAt, metadata]
    );

    if (insertRes.rows.length > 0) {
      await client.query('COMMIT');
      return { created: true, punch: insertRes.rows[0] };
    }

    // Conflict or race: fetch the existing open punch
    const existing = await client.query(
      `SELECT * FROM punches
       WHERE user_id = $1
         AND DATE(clock_in_at) = DATE($2)
         AND clock_out_at IS NULL
         AND deleted_at IS NULL
       ORDER BY clock_in_at DESC
       LIMIT 1`,
      [userId, clockInAt]
    );

    await client.query('COMMIT');
    return { created: false, punch: existing.rows[0] || null };
  } catch (err) {
    await client.query('ROLLBACK');
    // Unique violation from partial index or other constraint
    if (err.code === '23505') {
      const dup = await client.query(
        `SELECT * FROM punches
         WHERE user_id = $1
           AND DATE(clock_in_at) = DATE($2)
           AND clock_out_at IS NULL
           AND deleted_at IS NULL
         ORDER BY clock_in_at DESC
         LIMIT 1`,
        [userId, clockInAt]
      );
      return { created: false, punch: dup.rows[0] || null };
    }
    throw err;
  } finally {
    client.release();
  }
}
```

**Step 3 — Idempotency for LINE webhooks** (recommended)

Schema:
```sql
-- Idempotency keys for LINE webhook events (prune older than 24–48h via cron)
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  idempotency_key TEXT PRIMARY KEY,
  event_type TEXT NOT NULL,
  user_id INT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

Handler:
```js
async function handleLineWebhook(event, userId) {
  const idemKey = event.webhookEventId || `${userId}:${event.timestamp}:${event.type}`;
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    const exists = await client.query(
      'SELECT 1 FROM webhook_idempotency WHERE idempotency_key = $1',
      [idemKey]
    );
    if (exists.rows.length > 0) {
      await client.query('ROLLBACK');
      return { processed: false, reason: 'duplicate' };
    }

    const result = await clockIn(userId); // reuse atomic service

    await client.query(
      'INSERT INTO webhook_idempotency (idempotency_key, event_type, user_id) VALUES ($1, $2, $3)',
      [idemKey, event.type, userId]
    );
    await client.query('COMMIT');
    return { processed: true, result };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```
