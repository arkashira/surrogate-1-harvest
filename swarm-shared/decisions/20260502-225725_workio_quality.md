# workio / quality

## Final Synthesis — One Correct, Actionable Plan

**Core problem**: duplicate attendance punches occur because the system relies on client-side retries and app-level checks without server-side idempotency or database-level uniqueness.  
**Resolution priority**: enforce correctness at the database layer first, then add idempotency key support for safe retries, and finally improve observability and client behavior.

---

## 1) Diagnosis (merged, de-duplicated)

- **Missing database uniqueness**: no constraint prevents two punches for the same `(employee_id, punch_type, date, time_window)` when `deleted_at IS NULL`.
- **Race condition**: read-then-insert logic allows concurrent requests to both succeed before uniqueness is checked.
- **No idempotency key**: retries and replays cannot be de-duplicated because requests lack a stable fingerprint and the DB cannot enforce it.
- **No upsert semantics**: backend uses blind insert instead of `INSERT ... ON CONFLICT` keyed by business uniqueness or idempotency key.
- **Observability gap**: no logs/metrics for duplicate attempts or idempotency reuse.

---

## 2) Proposed change (merged scope)

- **DB layer** (`workio/server/src/db/schema.sql`):
  - Add `time_window` (if not present) and backfill sensibly.
  - Add `idempotency_key` column and index.
  - Add a **unique constraint** on `(employee_id, punch_type, DATE(clock_time), time_window)` for non-deleted rows.
- **API layer** (`workio/server/src/routes/attendance.ts`):
  - Accept optional `idempotency_key`.
  - Use `INSERT ... ON CONFLICT` on the business-unique columns to prevent races.
  - If `idempotency_key` is provided, prefer exact match on that key first (true idempotency), then fall back to upsert with conflict handling.
  - Log duplicates and return clear status codes (`409` for duplicates, `200` with `reused: true` for idempotent replays).
- **Client layer** (`workio/src/lib/services/attendance.ts`):
  - Generate a stable idempotency key per punch intent (per employee/punch_type/date) and store it locally.
  - Include the key on every retry so the server can safely de-duplicate.

### Step 2 — Endpoint with idempotency and upsert

```ts
// workio/server/src/routes/attendance.ts
import { Router, Request, Response } from 'express';
import { db } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = Router();

interface PunchBody {
  employee_id: number;
  punch_type: 'clock_in' | 'clock_out';
  clock_time?: string; // ISO
  location?: { lat: number; lng: number };
  idempotency_key?: string;
}

router.post('/punch', async (req: Request<{}, {}, PunchBody>, res: Response) => {
  const { employee_id, punch_type, clock_time = new Date().toISOString(), location, idempotency_key } = req.body;

  if (!employee_id || !punch_type) {
    return res.status(400).json({ error: 'Missing required fields' });
  }

  const client = await db.connect();
  try {
    await client.query('BEGIN');

    // 1) Exact idempotency-key match (strong idempotency)
    if (idempotency_key) {
      const existing = await client.query(
        `SELECT * FROM attendance_punches
         WHERE employee_id = $1 AND idempotency_key = $2 AND deleted_at IS NULL
         LIMIT 1`,
        [employee_id, idempotency_key]
      );
      if (existing.rows.length > 0) {
        await client.query('COMMIT');
        return res.json({ punch: existing.rows[0], reused: true });
      }
    }

    // 2) Determine time_window
    const hour = new Date(clock_time).getHours();
    const time_window = hour < 12 ? 'morning' : 'evening';

    // 3) Upsert on business-unique columns to prevent races
    const effectiveKey = idempotency_key || uuidv4();
    const upsert = await client.query(
      `INSERT INTO attendance_punches (employee_id, punch_type, clock_time, time_window, location, idempotency_key, created_at)
       VALUES ($1, $2, $3, $4, $5, $6, NOW())
       ON CONFLICT (employee_id, punch_type, DATE(clock_time), time_window)
       WHERE deleted_at IS NULL
       DO UPDATE SET clock_time = EXCLUDED.clock_time, location = EXCLUDED.location, updated_at = NOW()
       RETURNING *`,
      [employee_id, punch_type, clock_time, time_window, JSON.stringify(location), effectiveKey]
    );

    await client.query('COMMIT');
    res.json({ punch: upsert.rows[0], reused: false });
  } catch (err: any) {
    await client.query('ROLLBACK');

    // Unique violation from concurrent upsert or constraint
    if (err.code === '23505') {
      console.warn('Duplicate punch prevented', { employee_id, punch_type, clock_time, idempotency_key });
      return res.status(409).json({ error: 'Duplicate punch detected', code: 'DUPLICATE_PUNCH' });
    }

    console.error('Punch failed', err);
    res.status(500).json({ error: 'Internal server error' });
  } finally {
    client.release();
  }
});

export default router;
```

### Step 3 — Client: generate and persist idempotency key

```ts
// workio/src/lib/services/attendance.ts
export async function punch(
  punchType: 'clock_in' | 'clock_out',
  location?: GeolocationCoordinates
): Promise<Response> {
  const employeeId = getEmployeeId(); // implement per app
  const dateStr = new Date().toISOString().split('T')[0];
  const storageKey = `punch_key_${employeeId}_${punchType}_${dateStr}`;

  let idempotencyKey = localStorage.getItem(storageKey);
  if (!idempotencyKey) {
    idempotencyKey = crypto.randomUUID();
    localStorage.setItem(storageKey, idempotencyKey);
  }

  const payload: any = {
    employee_id: employeeId,
    punch_type: punchType,
    location: location ? { lat: location.latitude, lng: location.longitude } : undefined,
    idempotency_key: idempotencyKey,
  };

  return fetch('/api/attendance/punch', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload),
  });
}
```
