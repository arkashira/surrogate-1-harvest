# workio / frontend

## Final Synthesized Solution

**Scope:** `PunchClockButton` + `usePunch` hook (minimal surface).  
**Goal:** Eliminate duplicate punches, provide instant optimistic feedback, guarantee rollback on error, and keep UI consistent with server truth.

---

### 1) Hook: `usePunch` (robust, production-ready)

```tsx
// workio/src/hooks/usePunch.ts
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { v4 as uuidv4 } from 'uuid';

export function usePunch() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: async ({ type }: { type: 'IN' | 'OUT' }) => {
      const idempotencyKey = uuidv4();
      const res = await fetch('/api/punch', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-Idempotency-Key': idempotencyKey,
        },
        body: JSON.stringify({ type }),
      });

      if (!res.ok) {
        const err = await res.json().catch(() => ({}));
        throw new Error(err.message || 'Punch failed');
      }

      return res.json();
    },
    onMutate: async ({ type }) => {
      await queryClient.cancelQueries({ queryKey: ['currentPunch'] });
      await queryClient.cancelQueries({ queryKey: ['todayPunches'] });

      const previous = {
        current: queryClient.getQueryData(['currentPunch']),
        list: queryClient.getQueryData(['todayPunches']),
      };

      const optimistic = {
        id: `optimistic-${Date.now()}`,
        type,
        timestamp: new Date().toISOString(),
        optimistic: true,
      };

      // Optimistic current
      queryClient.setQueryData(['currentPunch'], optimistic);

      // Optimistic list
      queryClient.setQueryData(['todayPunches'], (old: any[] = []) => {
        if (type === 'IN') {
          return [...old, optimistic];
        }
        const copy = [...old];
        const lastInIndex = copy.map((p) => p.type).lastIndexOf('IN');
        if (lastInIndex >= 0) copy.splice(lastInIndex, 1);
        return copy;
      });

      return { previous };
    },
    onError: (err, _variables, context) => {
      if (context?.previous) {
        queryClient.setQueryData(['currentPunch'], context.previous.current);
        queryClient.setQueryData(['todayPunches'], context.previous.list);
      }
    },
    onSettled: () => {
      queryClient.invalidateQueries({ queryKey: ['currentPunch'] });
      queryClient.invalidateQueries({ queryKey: ['todayPunches'] });
    },
  });
}
```

### 2) Component: `PunchClockButton` (actionable + accessible)

```tsx
// workio/src/components/PunchClockButton.tsx
import { useCallback, useMemo } from 'react';
import { usePunch } from '../hooks/usePunch';
import { useQueryClient } from '@tanstack/react-query';

export function PunchClockButton() {
  const queryClient = useQueryClient();
  const { mutateAsync, isPending } = usePunch();

  // Derive next action from cached state (no network roundtrip)
  const { nextType, statusLabel } = useMemo(() => {
    const current = queryClient.getQueryData(['currentPunch']) as { type?: 'IN' | 'OUT' } | undefined;
    if (current?.type === 'IN') {
      return { nextType: 'OUT' as const, statusLabel: 'กำลังปฏิบัติงาน' };
    }
    return { nextType: 'IN' as const, statusLabel: 'พร้อมลงเวลา' };
  }, [queryClient]);

  const handlePunch = useCallback(async () => {
    // Guard: prevent double submit while pending
    if (isPending) return;

    try {
      await mutateAsync({ type: nextType });
    } catch {
      // rollback handled by hook; UI will revert automatically
    }
  }, [isPending, nextType, mutateAsync]);

  return (
    <div className="flex flex-col items-center gap-3">
      <button
        onClick={handlePunch}
        disabled={isPending}
        aria-busy={isPending}
        className={`
          inline-flex items-center justify-center rounded-xl px-8 py-4 text-lg font-semibold
          text-white shadow-lg transition-all active:scale-95 focus:outline-none
          focus:ring-2 focus:ring-blue-500 focus:ring-offset-2
          ${
            isPending
              ? 'cursor-not-allowed bg-gray-400'
              : nextType === 'IN'
              ? 'bg-green-600 hover:bg-green-700'
              : 'bg-red-600 hover:bg-red-700'
          }
        `}
      >
        {isPending ? (
          <span className="flex items-center gap-2">
            <svg className="h-5 w-5 animate-spin" viewBox="0 0 24 24" fill="none" stroke="currentColor">
              <circle cx="12" cy="12" r="10" strokeWidth="4" opacity="0.25" />
              <path d="M12 2a10 10 0 0 1 10 10" strokeWidth="4" strokeLinecap="round" opacity="0.75" />
            </svg>
            กำลังบันทึก...
          </span>
        ) : (
          <>
            {nextType === 'IN' ? 'ลงเวลาเข้า' : 'ลงเวลาออก'}
          </>
        )}
      </button>

      <span className="text-sm text-gray-500" aria-live="polite">
        {statusLabel}
      </span>
    </div>
  );
}
```

### 3) Backend requirement (non-negotiable)

- Accept and **respect** `X-Idempotency-Key` header.
- Return quickly (do not block on heavy work) and use idempotent upsert logic keyed by user+date+type+idempotencyKey (or equivalent unique constraint).
- If idempotency is not yet implemented, add a unique constraint/index to prevent duplicate punches for the same user/date/type within a short time window and make the upsert safe.

---

### 4) Verification checklist

- **Rapid taps:** Button disabled during request; no duplicate optimistic entries.
- **Slow 3G:** Only one in-flight request; optimistic state replaced by server truth after settle.
- **Error case:** Server error → optimistic state rolled back; button re-enabled; user can retry.
- **Query consistency:** After mutation, `currentPunch` and `todayPunches` refetch and match server.
- **Accessibility:** `aria-busy`, `disabled`, and `aria-live` status label present.
