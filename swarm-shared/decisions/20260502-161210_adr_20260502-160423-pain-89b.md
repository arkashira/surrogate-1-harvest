## Deployment
cf workers + Supabase (free-tier first; client distributed as standalone binaries)

## v0.1 scope
v0.1 ships a cross-platform CLI client (Linux x64, Windows, macOS) that: creates a local SOCKS5 proxy with hidden port binding, spawns a TUN interface with obfuscated TLS tunnels (meek-style), enforces a process-level kill-switch on tunnel failure, and supports opt-in anonymized telemetry to Supabase via CF Workers; no server components required — uses public HTTPS endpoints for camouflage.

## Open questions
- Which DPI signatures are most prevalent in target regions and what traffic shapes are safest (meek vs. TLS-fallback vs. QUIC)?
- Can we implement reliable TUN port hiding on Windows/macOS/Linux without root/admin prompts that reveal intent?
- How will we validate fail-closed kill-switch behavior across platforms without leaking DNS or plaintext during transition?
- What minimal, privacy-preserving telemetry schema will satisfy WAU-ZB measurement without deanonymizing users?
- How do we distribute and auto-update client binaries in restrictive regions without relying on blocked CDNs?
