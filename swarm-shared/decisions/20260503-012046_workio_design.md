# workio / design

## 1. Diagnosis

- **No idempotency key on LINE webhook payloads** — LINE retries on 5xx/timeouts can deliver the same event multiple times; without deduplication we create duplicate punches/leave/OT entries.
- **Race condition on punch create** — `find-then-insert` for open punches allows multiple concurrent requests (client double-tap or LINE parallel deliveries) to create multiple open records for the same employee/day.
- **No DB-level uniqueness guard** — schema allows multiple `employee_id, clock_out IS NULL` rows; application-level checks are not safe under concurrency.
- **Missing optimistic concurrency for updates** — leave/OT approval flows use `find-then-update` without versioning; concurrent approvals can overwrite or create inconsistent state.
- **Webhook handler is not idempotent at I/O level** — external side effects (notifications) can be emitted multiple times for the same event.

## 2. Proposed change

- **Scope**: `server/src/db/schema.sql` (add constraints + idempotency table), `server/src/routes/webhook/line.ts` (idempotency + atomic upsert), `server/src/services/punch.service.ts` (use `ON CONFLICT` upsert), `server/src/services/leave-ot.service.ts` (add version check).
- **Goal**: make punch create idempotent and race-safe; make webhook handler safe against LINE retries; add minimal DB constraints to enforce invariants.

## 3. Implementation

### 3.1 DB schema changes (`server/src/db/schema.sql`)

```sql
-- Idempotency table for LINE webhook events
CREATE TABLE IF NOT EXISTS line_webhook_events (
  idempotency_key TEXT PRIMARY KEY,
  event_type      TEXT NOT NULL,
  payload_hash    TEXT NOT NULL,
  processed_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Ensure at most one open punch per employee
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open
ON punches (employee_id)
WHERE clock_out IS NULL;

-- Optional: prevent overlapping punches (defensive)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_no_overlap
ON punches (employee_id, clock_in)
WHERE clock_out IS NULL;

-- Add optimistic lock version for leave/ot rows (if not present)
ALTER TABLE leave_requests ADD COLUMN IF NOT EXISTS version INT NOT NULL DEFAULT 1;
ALTER TABLE ot_requests   ADD COLUMN IF NOT EXISTS version INT NOT NULL DEFAULT 1;
```

### 3.2 Punch service — atomic upsert (`server/src/services/punch.service.ts`)

```ts
import { Pool } from 'pg';

export class PunchService {
  constructor(private pool: Pool) {}

  async clockIn(employeeId: number, clockIn: Date, location?: string): Promise<void> {
    const query = `
      INSERT INTO punches (employee_id, clock_in, location)
      VALUES ($1, $2, $3)
      ON CONFLICT (employee_id)
      DO UPDATE
        SET clock_in = EXCLUDED.clock_in,
            location = EXCLUDED.location,
            updated_at = NOW()
      WHERE punches.clock_out IS NULL;
    `;
    await this.pool.query(query, [employeeId, clockIn, location]);
  }

  async clockOut(employeeId: number, clockOut: Date, location?: string): Promise<void> {
    const query = `
      UPDATE punches
      SET clock_out = $2,
          location = $3,
          updated_at = NOW()
      WHERE employee_id = $1
        AND clock_out IS NULL;
    `;
    const { rowCount } = await this.pool.query(query, [employeeId, clockOut, location]);
    if (rowCount === 0) {
      throw new Error('No open punch to clock out');
    }
  }
}
```

### 3.3 Webhook handler — idempotent processing (`server/src/routes/webhook/line.ts`)

```ts
import express from 'express';
import crypto from 'crypto';
import { Pool } from 'pg';
import { PunchService } from '../../services/punch.service';

const router = express.Router();
const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const punchService = new PunchService(pool);

function hashPayload(payload: any): string {
  return crypto.createHash('sha256').update(JSON.stringify(payload)).digest('hex');
}

async function isProcessed(idempotencyKey: string, payloadHash: string): Promise<boolean> {
  const res = await pool.query(
    `SELECT 1 FROM line_webhook_events WHERE idempotency_key = $1 AND payload_hash = $2`,
    [idempotencyKey, payloadHash]
  );
  return res.rowCount > 0;
}

async function markProcessed(idempotencyKey: string, payloadHash: string, eventType: string) {
  await pool.query(
    `INSERT INTO line_webhook_events (idempotency_key, event_type, payload_hash)
     VALUES ($1, $2, $3)`,
    [idempotencyKey, eventType, payloadHash]
  );
}

router.post('/line', async (req, res) => {
  try {
    const payload = req.body;
    const idempotencyKey = payload.events?.[0]?.webhookEventId || crypto.randomUUID();
    const payloadHash = hashPayload(payload);

    if (await isProcessed(idempotencyKey, payloadHash)) {
      return res.status(200).json({ ok: true, reason: 'duplicate' });
    }

    // Process only clock-in/out intents (simplified)
    const event = payload.events?.[0];
    if (!event) return res.status(400).json({ error: 'no_event' });

    const userId = event.source?.userId;
    if (!userId) return res.status(400).json({ error: 'no_user' });

    // Map LINE userId -> employeeId (assumes mapping exists)
    const employee = await pool.query(
      `SELECT id FROM employees WHERE line_user_id = $1`,
      [userId]
    );

    if (employee.rowCount === 0) {
      return res.status(404).json({ error: 'employee_not_found' });
    }

    const employeeId = employee.rows[0].id;
    const now = new Date();

    // Simple heuristic: if no open punch -> clock in, else clock out
    const openPunch = await pool.query(
      `SELECT id FROM punches WHERE employee_id = $1 AND clock_out IS NULL`,
      [employeeId]
    );

    if (openPunch.rowCount === 0) {
      await punchService.clockIn(employeeId, now);
    } else {
      await punchService.clockOut(employeeId, now);
    }

    await markProcessed(idempotencyKey, payloadHash, 'clock');
    res.status(200).json({ ok: true });
  } catch (err) {
    console.error('LINE webhook error', err);
    // Important: return 5xx so LINE retries if failure is transient
    res.status(500).json({ error: 'internal' });
  }
});

export default router;
```

### 3.4 Leave/OT optimistic concurrency (`server/src/services/leave-ot.service.ts`)

```ts
async function approveLeave(leaveId: number, approverId: number): Promise<void> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const current = await client.query(
      `SELECT version FROM leave_requests WHERE id = $1 FOR UPDATE`,
      [leaveId]
    );

    if (current.rowCount === 0) throw new Error('not_found');

    const result = await client.query(
      `UPDATE leave_requests
       SET status = 'approved',
           approver_id = $1,
           version = version + 1,
           updated_at = NOW()
       WHERE id = $2 AND version = $3`,
      [approverId, leaveId, current.rows[0].version]
    );

    if (result.rowCount === 0) {
      await client.query('ROLLBACK');
      throw new Error('concurrent_update');
    }

    await client.query('COMMIT');
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```
