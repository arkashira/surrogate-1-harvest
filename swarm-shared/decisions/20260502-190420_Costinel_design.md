# Costinel / design

## Final Design — Costinel Discovery Increment (≤2h)

**Goal:** Ship a lightweight, deterministic discovery module that produces machine-readable manifests so onboarding, training, and governance scripts run without repeated API calls or schema surprises.

---

## Directory Layout (adopt Candidate 2 structure + Candidate 1 outputs)
```
Costinel/
├── discovery/
│   ├── __init__.py
│   ├── manifest.py          # main generator
│   ├── cloud_accounts.py    # cloud provider adapters
│   ├── dataset_index.py     # HF CDN bypass + file list
│   ├── readiness.py         # Lightning quota + Kaggle auth probe
│   └── templates/
│       └── manifest.json.j2
├── manifests/               # generated outputs (gitignored)
│   ├── sources.json
│   └── readiness.json
└── scripts/discover.py      # thin CLI entrypoint
```

## Implementation Plan (≤2h)

### 1) Scaffold (5m)
- Create directories above.
- Add `manifests/` to `.gitignore`.

### 2) Core modules (60–75m)

**`discovery/cloud_accounts.py`**
- Read AWS `~/.aws/credentials` (configparser), GCP `~/.config/gcloud/application_default_credentials.json`, Azure `~/.azure/azureProfile.json`, and env vars.
- Normalize to `{provider, account_id, alias, regions, tags}`.
- Fail gracefully (no hard failure if no accounts found).

**`discovery/dataset_index.py`**
- Use `huggingface_hub.HfApi.list_repo_tree(repo, recursive=False)` per date folder.
- Non-recursive to avoid pagination and rate limits.
- Emit `files` and `cdn_urls` per folder.
- If HF API unavailable, emit empty with warning (do not crash).

**`discovery/top_hub_context.py`**
- Prefer `knowledge_rag/top_hub.json` if present.
- Fallback to static `{id: "MOC", type: "hub", related_docs: [...]}`.
- Keep deterministic and small.

**`discovery/readiness.py`**
- **HF CDN bypass**: derive `resolve/...` URLs from discovered files; validate with lightweight `HEAD` (timeout 5s) for a sample (first 3 files).
- **Lightning quota**: call `Teamspace.studios()`; count running; emit available machines and quota.
- **Kaggle auth probe**: POST to `kernels/push` with `isPrivate: True` and a test slug; accept 200/400 as auth_ok (400 means token valid but slug invalid). Do not require phone verification.

**`discovery/manifest.py`**
- Orchestrates above; writes `sources.json` and `readiness.json` atomically (write to temp then rename).
- Deterministic key ordering for stable diffs.

### 3) CLI entrypoint (5m)

**`scripts/discover.py`**
```python
#!/usr/bin/env python3
import argparse
from pathlib import Path
from discovery.manifest import generate_all

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--output-dir", default="manifests", type=Path)
    args = parser.parse_args()
    args.output_dir.mkdir(exist_ok=True)
    generate_all(output_dir=args.output_dir)
```

### 4) Integration hooks (5m)

- **Onboarding Makefile**: add `discover: python scripts/discover.py --output-dir manifests`.
- **Training script (`train.py`)**: load `manifests/sources.json`; use CDN URLs directly (zero HF API calls during data load).
- **CI step**: validate `manifests/readiness.json` schema and require `hf_cdn_bypass.ok == true` and `lightning.quota_remaining_hours > 0` before scheduling jobs.

## Resolved Contradictions (correctness + actionability)

| Conflict | Resolution |
|---------|------------|
| Output location (`manifests/` vs `discovery/`) | Use `manifests/` for generated artifacts (gitignored); keep `discovery/` for source code. Separates generated from source. |
| Single file vs split modules | Use split modules (`cloud_accounts.py`, `dataset_index.py`, `readiness.py`) for maintainability; orchestrate via `manifest.py`. |
| HF CDN bypass: generate URLs vs validate | Generate URLs in `sources.json`; additionally validate a sample via HEAD in `readiness.json` (best of both). |
| Kaggle probe: strict 200 vs 200/400 | Accept 200/400 as auth_ok (400 indicates valid token but invalid slug), which is correct and avoids false negatives. |
| Lightning quota: hardcoded vs real | Use real `Teamspace.studios()` call; fallback to safe defaults only on exception (actionable and truthful). |
| Top-hub: pagerank vs static/export | Prefer `knowledge_rag/top_hub.json` if present; fallback to static MOC. Avoids heavy computation and remains deterministic. |

---

## Acceptance Criteria (≤2h)

- `python scripts/discover.py` produces valid `manifests/sources.json` and `manifests/readiness.json`.
- Training script can consume CDN URLs without HF API calls.
- CI step can validate readiness before Lightning job submission.
- No new required dependencies beyond `huggingface_hub`, `lightning-sdk` (optional), `requests`; graceful degradation if missing.
