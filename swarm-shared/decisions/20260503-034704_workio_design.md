# workio / design

## 1. Diagnosis

- **No real-time state propagation**: frontend relies on local React state/polling; LINE webhook updates are invisible until hard refresh → silent data drift and perceived latency across tabs/devices.
- **Missing optimistic UI + rollback**: clock/leave/OT actions block on round-trip; failures leave UI stuck and require manual refresh to reconcile.
- **No client-side subscription layer**: multi-tab usage causes inconsistent state (e.g., approve in tab A, tab B still shows pending).
- **Backend emits no events**: webhook handlers mutate DB but publish no events for clients to react to; frontend cannot invalidate queries or update cache deterministically.
- **Design-system inconsistency**: no shared loading/optimistic/error states for time-entry components; UX feels disjointed.

## 2. Proposed change

Add a lightweight real-time subscription layer (EventSource / Server-Sent Events) from backend to frontend for tenant-scoped entity changes (`clock`, `leave`, `ot`), plus optimistic UI hooks in the React layer. Scope:

- **Backend**: `workio/server/src/routes/events.ts` (new) + small middleware in `workio/server/src/app.ts` to attach tenant context.
- **Frontend**: `workio/src/lib/hooks/useTenantEvents.ts` + optimistic update helpers in `workio/src/features/time-entry/TimeEntryList.tsx` (or equivalent).
- **Shared types**: `workio/src/types/events.d.ts`.

## 3. Implementation

### Backend: SSE endpoint (tenant-scoped)

```ts
// workio/server/src/routes/events.ts
import { Router, Request, Response } from 'express';
import { db } from '../db';

const router = Router();

router.get('/events', async (req: Request, res: Response) => {
  const tenantId = (req as any).tenant?.id;
  if (!tenantId) {
    return res.status(401).end();
  }

  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');
  res.flushHeaders();

  // Send initial heartbeat
  res.write(`event: connected\ndata: ${JSON.stringify({ tenantId })}\n\n`);

  // Simple pub-sub using an in-memory EventEmitter (replace with Redis in prod)
  const onUpdate = (update: any) => {
    if (update.tenantId === tenantId) {
      res.write(`event: entity-change\ndata: ${JSON.stringify(update)}\n\n`);
    }
  };

  (global as any).eventBus?.on('entity-change', onUpdate);

  // Cleanup on client disconnect
  req.on('close', () => {
    (global as any).eventBus?.off('entity-change', onUpdate);
    res.end();
  });
});

export default router;
```

Wire up in app:

```ts
// workio/server/src/app.ts (excerpt)
import eventsRouter from './routes/events';
app.use('/events', eventsRouter);
```

Emit events in mutation handlers (example for clock):

```ts
// workio/server/src/services/clockService.ts (excerpt)
function emitEntityChange(tenantId: string, entity: string, payload: any) {
  (global as any).eventBus?.emit('entity-change', {
    tenantId,
    entity,
    payload,
    ts: Date.now(),
  });
}

// after successful DB mutation
await db.clockPunch.create(...);
emitEntityChange(tenantId, 'clock', { action: 'punch', ...result });
```

### Frontend: hook for SSE

```ts
// workio/src/lib/hooks/useTenantEvents.ts
import { useEffect } from 'react';
import { useQueryClient } from '@tanstack/react-query';

export function useTenantEvents(tenantId: string | undefined) {
  const queryClient = useQueryClient();

  useEffect(() => {
    if (!tenantId) return;

    const es = new EventSource('/events');
    es.onmessage = (e) => {
      try {
        const data = JSON.parse(e.data);
        if (data.tenantId === tenantId) {
          // Invalidate relevant queries so UI refreshes
          queryClient.invalidateQueries({ queryKey: ['clocks'] });
          queryClient.invalidateQueries({ queryKey: ['leaves'] });
          queryClient.invalidateQueries({ queryKey: ['ot'] });
        }
      } catch {
        // ignore malformed
      }
    };

    return () => {
      es.close();
    };
  }, [tenantId, queryClient]);
}
```

Use in top-level tenant layout:

```tsx
// workio/src/pages/Dashboard.tsx (excerpt)
export default function Dashboard() {
  const { tenant } = useTenant(); // assume provides tenant.id
  useTenantEvents(tenant?.id);
  // ...
}
```

### Frontend: optimistic UI helper (example)

```tsx
// workio/src/features/time-entry/TimeEntryList.tsx (excerpt)
import { useMutation, useQueryClient } from '@tanstack/react-query';

function useClockPunch() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: punch, // your API call
    onMutate: async (newPunch) => {
      await queryClient.cancelQueries({ queryKey: ['clocks'] });
      const previous = queryClient.getQueryData(['clocks']);
      queryClient.setQueryData(['clocks'], (old: any) => [
        { ...newPunch, id: `optimistic-${Date.now()}`, status: 'pending' },
        ...(old || []),
      ]);
      return { previous };
    },
    onError: (err, _newPunch, context) => {
      if (context?.previous) {
        queryClient.setQueryData(['clocks'], context.previous);
      }
    },
    onSettled: () => {
      queryClient.invalidateQueries({ queryKey: ['clocks'] });
    },
  });
}
```

## 4. Verification

1. **Backend**: start server, `curl -N http://localhost:3000/events` (with tenant auth) — should receive `event: connected` then stay open.
2. **Event emission**: perform a clock punch via LINE/webhook; confirm server logs show `emitEntityChange` and the SSE stream emits an `entity-change` event with correct tenantId.
3. **Frontend**: open two browser tabs for same tenant. In tab A, punch in via UI; tab B should show new punch within 1–2s without manual refresh (query invalidation + refetch).
4. **Optimistic path**: simulate a slow/network-failing punch (devtools throttle or mock 500); UI should show pending entry then rollback on error.
5. **Cross-device**: verify React Query cache invalidation works by checking network tab — after event, a refetch to `/clocks` (or equivalent) occurs and UI updates.
