## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: Android app (no root) that connects to a Cloudflare Worker-based egress using shadowsocks over TLS with domain fronting on 443; config fetched from Supabase; per-app VPN mode with split-exclude to conceal TUN; local SOCKS5 listener hidden via port reuse and localhost-only binding; Redis-backed session tracking and basic WAUS/U metric; free-tier limits enforced; no kill-switch and minimal UI (connect/disconnect + status).

## Open questions
- Which tun concealment technique is stable and undetectable on stock Android without root (fd hiding via /proc/self/fd tricks vs userspace-tun vs per-app VPN with split-exclude)?
- Can we reliably use Cloudflare Workers TCP/UDP connect (or Durable Objects + WebSocket) for proxy egress within free-tier limits, or do we need fallback VPSes?
- What cover traffic patterns (domain-fronted TLS, HTTP/2 or QUIC) minimize DPI fingerprintability while remaining performant on high-loss links?
- How will key/cert rotation and endpoint discovery be bootstrapped securely without an app store channel that can be blocked?
- What telemetry can we collect to measure WAUS/U without creating user-identifying logs under legal coercion?
- How do we handle captive portals and DNS hijacking without exposing VPN-related DNS queries?
