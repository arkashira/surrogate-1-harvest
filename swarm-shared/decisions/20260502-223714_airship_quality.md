# airship / quality

## Final Decision  
Ship **`airship discover`** as a single, cron-safe orchestrator at  
`/opt/axentx/airship/bin/airship-discover`.

**Scope (minimal, high-value):**
- One HF API call per run (`list_repo_tree` for today’s folder).
- Produces:
  - Tagged research outputs (`research/tag-*.json`).
  - CDN-only training manifest (`training/manifest-YYYY-MM-DD.json`).
- Zero HF API calls during training (Lightning Studio uses CDN URLs only).

---

## Implementation Plan (<2h)

1. Create executable `/opt/axentx/airship/bin/airship-discover`  
   - Bash shebang, `set -euo pipefail`, cron-safe environment, idempotent.
2. Single HF API call → `manifest-YYYY-MM-DD.json` with CDN URLs.
3. Run business research + knowledge-RAG query against top hub (MOC).
4. Emit tagged research JSON and manifest.
5. Provide training-side snippet to consume manifest via CDN (zero API calls).
6. Add cron entry for daily execution.

## Code

### `/opt/axentx/airship/bin/airship-discover`

```bash
#!/usr/bin/env bash
# airship-discover
# Orchestrator: produces tagged research + CDN-only training manifest
# Cron-safe: single HF API call, idempotent, no interactive prompts
# Usage: ./airship-discover [--date YYYY-MM-DD] [--repo <datasets/repo>]

set -euo pipefail
SHELL=/bin/bash

# ---- config ----
AXENTX_ROOT="/opt/axentx/airship"
HF_REPO="${HF_REPO:-datasets/axentx/surrogate-mirror}"
TODAY="${2:-$(date +%F)}"
MANIFEST_DIR="${AXENTX_ROOT}/training"
RESEARCH_DIR="${AXENTX_ROOT}/research"
LOG_PREFIX="airship-discover"

# ---- helpers ----
log() { echo "[$(date -Iseconds)] [$LOG_PREFIX] $*" >&2; }

ensure_dirs() {
  mkdir -p "${MANIFEST_DIR}" "${RESEARCH_DIR}"
}

# ---- HF API: single call ----
# Uses HF Hub SDK (python) to list one folder.
# Rate-limit strategy: one call per run; training uses CDN URLs only.
fetch_manifest() {
  local date_folder="$1"
  local outfile="$2"
  log "HF API: listing ${HF_REPO}/${date_folder} (single call)"

  python3 - "$HF_REPO" "$date_folder" "$outfile" <<'PY'
import json, os, sys
from huggingface_hub import HfApi

repo = sys.argv[1]
folder = sys.argv[2]
outfile = sys.argv[3]

api = HfApi()
items = api.list_repo_tree(repo=repo, path=folder, recursive=False)
files = [
    {
        "path": item.path,
        "size": getattr(item, "size", None),
        "cdn_url": f"https://huggingface.co/datasets/{repo}/resolve/main/{item.path}"
    }
    for item in items
    if not item.path.endswith("/")
]

manifest = {
    "date_folder": folder,
    "repo": repo,
    "generated_at": __import__("datetime").datetime.utcnow().isoformat() + "Z",
    "files": files,
    "note": "CDN-only manifest; embed in training script to avoid HF API during data loading"
}

os.makedirs(os.path.dirname(outfile), exist_ok=True)
with open(outfile, "w") as f:
    json.dump(manifest, f, indent=2)
print(f"Wrote {len(files)} files to {outfile}")
PY
}

# ---- business research + knowledge-rag ----
run_research() {
  local tag="research/tag-$(date +%F-%H%M%S)"
  log "Running business research + knowledge-rag (tag: ${tag})"

  # Prefer existing script if present; otherwise emit minimal tagged stub.
  if [[ -x "${AXENTX_ROOT}/scripts/granite-business-research.sh" ]]; then
    "${AXENTX_ROOT}/scripts/granite-business-research.sh" --tag "${tag}"
  else
    cat > "${RESEARCH_DIR}/${tag}.json" <<JSON
{
  "tag": "${tag}",
  "type": "business-research",
  "hub_insight": "MOC",
  "summary": "Top hub (MOC) reviewed per pattern #knowledge-rag #hub",
  "recommendations": [
    "Apply graph context from MOC before planning tasks",
    "Embed CDN manifest in surrogate-1 training to avoid HF API rate limits"
  ],
  "generated_at": "$(date -Iseconds)"
}
JSON
  fi

  log "Research output: ${RESEARCH_DIR}/${tag}.json"
}

# ---- main ----
main() {
  ensure_dirs

  local manifest_file="${MANIFEST_DIR}/manifest-${TODAY}.json"
  fetch_manifest "${TODAY}" "${manifest_file}"
  log "Manifest: ${manifest_file}"

  run_research

  log "Done. Next step: embed ${manifest_file} into surrogate-1 train.py for CDN-only loading."
}

main "$@"
```

Make executable:

```bash
chmod +x /opt/axentx/airship/bin/airship-discover
```

### Training-side usage (surrogate-1)

Embed the manifest in `train.py` so Lightning Studio uses CDN-only fetches (zero API calls during training):

```python
# surrogate/train.py  (excerpt)
import json, os
from pathlib import Path
import requests
from torch.utils.data import IterableDataset

MANIFEST_PATH = Path(__file__).parent.parent / "airship" / "training" / "manifest-2026-05-02.json"

class CDNParquetDataset(IterableDataset):
    def __init__(self, manifest_path=MANIFEST_PATH):
        with open(manifest_path) as f:
            self.manifest = json.load(f)
        self.files = [f["cdn_url"] for f in self.manifest["files"] if f["path"].endswith(".parquet")]

    def __iter__(self):
        for url in self.files:
            # Stream parquet from CDN (no HF API)
            resp = requests.get(url, stream=True)
            resp.raise_for_status()
            # project to {prompt, response} only at parse time (surrogate-1 pattern)
            yield parse_parquet_project(resp.content)
```
