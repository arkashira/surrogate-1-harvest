# Costinel / design

## Implementation Plan — Costinel Top-Hub Signal Endpoint

**Highest-value incremental improvement (<2h):**  
Expose a **read-only, side-effect-free** `GET /api/v1/cost-anomaly/signal/top-hub` that returns the most-connected hub (e.g., "MOC") with contextual insights for cost governance. Uses existing knowledge-rag graph to rank hubs by connection strength and returns actionable signal payload without mutations or side effects.

### Design decisions (best-judgement)
- Follows pattern: review most-connected hub before planning (top-hub doc insight 2026-04-27).
- Read-only endpoint aligns with Costinel philosophy: *Sense + Signal — ไม่ Execute*.
- Uses existing knowledge-rag pipeline; no new infra or training required.
- Returns lightweight JSON suitable for dashboard panels and anomaly triage.

### Implementation steps (≤2h)

1. Add FastAPI route in `api/cost_anomaly/signals.py` (or equivalent) for `GET /api/v1/cost-anomaly/signal/top-hub`.
2. Implement service layer `services/top_hub_signal.py` that:
   - Queries knowledge-rag graph for top hub by connection count/weight.
   - Enriches with recent anomaly context (last 7d) and cost impact estimate.
   - Returns deterministic payload keyed by hub slug.
3. Add minimal unit test for service (mock graph).
4. Wire route into main app router.
5. Verify locally with `uvicorn` and curl.

from typing import Dict, Any

router = APIRouter(prefix="/api/v1/cost-anomaly/signal", tags=["signals"])

@router.get("/top-hub", response_model=Dict[str, Any])

    """

#### `services/top_hub_signal.py`
```python
from datetime import datetime, timedelta
from typing import Dict, Any, List
from knowledge_rag.graph import get_top_connected_hubs  # existing graph API
from repositories.anomaly_repo import get_recent_anomalies_by_hub  # existing repo

def get_top_hub_signal(lookback_days: int = 7) -> Dict[str, Any]:
    """
    Returns the top hub by connection strength with contextual signals.
    """
    # 1) Top hub by graph connections (existing knowledge-rag)
    top_hubs: List[Dict[str, Any]] = get_top_connected_hubs(limit=1)
    if not top_hubs:
        return {
            "hub_slug": None,
            "hub_name": "Unknown",
            "connection_count": 0,
            "signal": "no_hubs_found",
            "context": {},
            "generated_at": datetime.utcnow().isoformat() + "Z"
        }

    hub = top_hubs[0]
    hub_slug: str = hub.get("slug", "unknown")
    connection_count: int = hub.get("connection_count", 0)

    # 2) Recent anomalies for this hub
    since = datetime.utcnow() - timedelta(days=lookback_days)
    anomalies = get_recent_anomalies_by_hub(hub_slug, since=since)

    # 3) Build lightweight context
    total_estimated_impact = sum(a.get("estimated_cost_usd", 0) for a in anomalies)
    top_anomalies = sorted(anomalies, key=lambda a: a.get("severity_score", 0), reverse=True)[:3]

    return {
        "hub_slug": hub_slug,
        "hub_name": hub.get("name", hub_slug),
        "connection_count": connection_count,
        "signal": "high_connectivity",
        "context": {
            "lookback_days": lookback_days,
            "anomaly_count": len(anomalies),
            "total_estimated_cost_usd": round(total_estimated_impact, 2),
            "top_anomalies": [
                {
                    "id": a.get("id"),
                    "title": a.get("title"),
                    "severity_score": a.get("severity_score"),
                    "estimated_cost_usd": a.get("estimated_cost_usd"),
                    "detected_at": a.get("detected_at")
                }
                for a in top_anomalies
            ]
        },
        "generated_at": datetime.utcnow().isoformat() + "Z"
    }
```

#### Minimal unit test (`tests/services/test_top_hub_signal.py`)
```python
import pytest
from unittest.mock import patch
from services.top_hub_signal import get_top_hub_signal

@patch("services.top_hub_signal.get_top_connected_hubs")
@patch("services.top_hub_signal.get_recent_anomalies_by_hub")
def test_top_hub_signal_found(mock_anomalies, mock_hubs):
    mock_hubs.return_value = [{"slug": "moc", "name": "MOC", "connection_count": 42}]
    mock_anomalies.return_value = [
        {"id": "a1", "title": "Spike", "severity_score": 8, "estimated_cost_usd": 120.0, "detected_at": "2026-05-02T10:00:00Z"}
    ]

    result = get_top_hub_signal(lookback_days=7)
    assert result["hub_slug"] == "moc"
    assert result["connection_count"] == 42
    assert result["context"]["anomaly_count"] == 1
    assert result["context"]["total_estimated_cost_usd"] == 120.0
```

### Notes & follow-ups

- If knowledge-rag graph isn’t available in this repo yet, stub `get_top_connected_hubs` with a deterministic fallback (e.g., "MOC") to unblock delivery; replace with real graph call once available.
- Consider adding caching (e.g., 5-minute in-memory) if graph queries are expensive; keep it simple for now.
