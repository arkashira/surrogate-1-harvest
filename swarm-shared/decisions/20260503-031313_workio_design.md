# workio / design

## Final Synthesis & Action Plan

**Chosen approach:** Combine the strongest, most concrete elements from both proposals into a single, minimal, high-impact change focused on **backend integrity** (signature verification + idempotency) with a small, concrete frontend guard to prevent double-taps.

---

## 1. Diagnosis (merged, prioritized)

- **Security**: Missing `X-Line-Signature` verification on `/webhook/line` allows spoofed/replayed events to be accepted as valid clock-in/out or leave requests.
- **Integrity**: No idempotency/replay protection — LINE retries on 5xx/timeouts can create duplicate `clock_events` and leave/OT entries.
- **UX**: Frontend clock-in/out has no optimistic state or local debouncing; double-taps during network latency create duplicate user intents before webhook round-trip completes.
- **Architecture**: Webhook handler mixes validation, business logic, and persistence; no clear separation between public ingestion and internal command handling.

---

## 2. Proposed change (single focused scope)

Add **LINE signature verification + idempotency by `line_event_id`** and a **frontend debounce/optimistic state guard** in one coordinated change.

**Files to modify/create:**
- `workio/server/src/middleware/lineAuth.ts` (new) — HMAC-SHA256 signature verification.
- `workio/server/src/middleware/idempotency.ts` (new) — idempotency guard using `line_event_id`.
- `workio/server/src/routes/webhook/line.ts` — apply both middlewares before business logic.
- `workio/server/src/db/schema.sql` — add unique constraint/index on `line_event_id` for `clock_events` and `leave_requests`.
- `workio/server/src/services/clockService.ts` — make punch handling idempotent (UPSERT on `line_event_id`).
- `workio/client/src/components/ClockButton.tsx` (or equivalent) — optimistic state + debounce to prevent double-taps.

---

### 3.2 LINE signature verification middleware

```ts
// workio/server/src/middleware/lineAuth.ts
import crypto from 'crypto';
import { Request, Response, NextFunction } from 'express';

const LINE_CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

export function verifyLineSignature(req: Request, res: Response, next: NextFunction) {
  const signature = req.get('X-Line-Signature');
  if (!signature) {
    return res.status(400).json({ error: 'Missing X-Line-Signature' });
  }

  // LINE sends raw body; ensure raw body is preserved in your express setup
  const rawBody = (req as any).rawBody || JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', LINE_CHANNEL_SECRET)
    .update(rawBody, 'utf8')
    .digest('base64');

  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected))) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  next();
}
```

> **Note**: Ensure your express app captures raw body for this middleware (e.g., use a `verify` function on the LINE route or a dedicated raw-body parser only for this route).

### 3.3 Idempotency middleware (corrected)

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { pool } from '../db';

export async function enforceIdempotency(req: Request, res: Response, next: NextFunction) {
  // Prefer explicit line_event_id from body; fallback to deterministic derivation
  const bodyEvent = req.body?.events?.[0];
  let lineEventId = req.body.line_event_id;

  if (!lineEventId && bodyEvent) {
    // Deterministic, collision-resistant id derived from event content
    const crypto = require('crypto');
    lineEventId = crypto
      .createHash('sha256')
      .update(JSON.stringify(bodyEvent))
      .digest('hex');
  }

  if (!lineEventId) {
    return res.status(400).json({ error: 'Unable to derive line_event_id' });
  }

  try {
    // Quick existence check
    const clockCheck = await pool.query(
      'SELECT 1 FROM clock_events WHERE line_event_id = $1 LIMIT 1',
      [lineEventId]
    );
    const leaveCheck = await pool.query(
      'SELECT 1 FROM leave_requests WHERE line_event_id = $1 LIMIT 1',
      [lineEventId]
    );

    if (clockCheck.rowCount > 0 || leaveCheck.rowCount > 0) {
      // Idempotent success: return 200 so LINE stops retrying
      return res.status(200).json({ ok: true, message: 'duplicate ignored' });
    }

    // Attach for downstream handlers to persist
    (req as any).line_event_id = lineEventId;
    next();
  } catch (err) {
    console.error('Idempotency check failed', err);
    return res.status(500).json({ error: 'idempotency check failed' });
  }
}
```

### 3.5 Make clockService idempotent

```ts
// workio/server/src/services/clockService.ts
import { pool } from '../db';

export async function recordClockEvent(userId: string, lineEventId: string, type: 'in' | 'out', lat?: number, lng?: number) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Upsert by line_event_id to guarantee idempotency
    const result = await client.query(
      `INSERT INTO clock_events (user_id, line_event_id, event_type, latitude, longitude, created_at)
       VALUES ($1, $2, $3, $4, $5, NOW())
       ON CONFLICT (line_event_id) DO NOTHING
       RETURNING id`,
      [userId, lineEventId, type, lat, lng]
    );

    await client.query('COMMIT');
    return result.rowCount > 0;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```
