# workio / backend

## Final Synthesis — One Correct, Actionable Plan

**Core diagnosis (merged, de-duplicated)**
- The app relies on find-then-insert and app-level checks, which are **not safe under concurrency or LINE webhook retries**.
- There is **no database-level guarantee** that a user can have at most one open punch per calendar day.
- There is **no idempotency mechanism** to absorb LINE retries within their 5–30s window.
- Clock-out can orphan multiple open rows if duplicates exist, and there is no efficient constraint to prevent overlapping punches.

**Chosen approach (resolve contradictions in favor of correctness + actionability)**
- Enforce rules **at the database** (single source of truth) with a **partial unique index** for open punches.
- Add **lightweight idempotency** using an `idempotency_key` column on `punches` (no separate table) with a short TTL cleanup; this is simpler, faster, and avoids distributed state while remaining safe under retries.
- Replace find-then-insert with **atomic upsert-style logic** and explicit 409/200 responses so clients and LINE retries are handled deterministically.
- Keep clock-out simple and safe: lock the open row for update in a transaction to prevent races/orphans.

## 1. Database changes (schema.sql)

```sql
-- Prevent multiple open punches per user per calendar day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open_per_day
ON punches (user_id, (date(clock_in_at)))
WHERE clock_out_at IS NULL;

-- Optional future-proof: at most one closed punch per user per calendar day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_closed_per_day
ON punches (user_id, (date(clock_in_at)))
WHERE clock_out_at IS NOT NULL;

-- Idempotency: allow LINE/web retries to be deduplicated quickly
-- Use a deterministic client-provided key (or server-generated) and short TTL.
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key TEXT NULL;

CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_idempotency_key
ON punches (idempotency_key)
WHERE idempotency_key IS NOT NULL;

-- Optional: index to efficiently find open punch for a user
CREATE INDEX IF NOT EXISTS idx_punches_user_open
ON punches (user_id, clock_in_at DESC)
WHERE clock_out_at IS NULL;
```

**One-time cleanup (if duplicates already exist)**
```sql
-- Keep only the earliest open punch per user per day; remove extras.
-- Review before running in production.
WITH ranked AS (
  SELECT id,
         ROW_NUMBER() OVER (PARTITION BY user_id, date(clock_in_at) ORDER BY clock_in_at) AS rn
  FROM punches
  WHERE clock_out_at IS NULL
)
DELETE FROM punches
WHERE id IN (SELECT id FROM ranked WHERE rn > 1);
```

## 2. Idempotency policy (lightweight)

- Client should send `X-Idempotency-Key` for clock-in (UUID or deterministic hash).  
- Server accepts the key, stores it on the punch row, and rejects duplicates with 409 within the TTL (recommended 2–5 minutes).  
- A daily/periodic job deletes punches older than N days with `idempotency_key IS NOT NULL` to keep table small (or use `pg_cron`/application cleanup).

## 3. Route implementation (atomic + safe)

```ts
import express from 'express';
import { pool } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = express.Router();

/**
 * POST /punch/clock-in
 * Body: { userId: number, location?: { lat: number, lng: number } }
 * Headers: X-Idempotency-Key (recommended)
 */
router.post('/clock-in', async (req, res) => {
  const { userId, location } = req.body;
  const idempotencyKey = req.header('X-Idempotency-Key') || uuidv4();
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    // Fast idempotency check
    const idem = await client.query(
      `SELECT id, clock_in_at, clock_out_at, location, location_out
       FROM punches
       WHERE idempotency_key = $1`,
      [idempotencyKey]
    );

    if (idem.rows.length > 0) {
      await client.query('ROLLBACK');
      return res.status(200).json({ ok: true, punch: idem.rows[0], duplicate: true });
    }

    // Try atomic insert; partial index will block second open punch
    const insertResult = await client.query(
      `INSERT INTO punches (user_id, clock_in_at, location, idempotency_key)
       VALUES ($1, NOW(), $2, $3)
       ON CONFLICT DO NOTHING
       RETURNING *`,
      [userId, location ? JSON.stringify(location) : null, idempotencyKey]
    );

    if (insertResult.rowCount === 0) {
      // Conflict: open punch exists (or other constraint)
      const existing = await client.query(
        `SELECT id, clock_in_at, clock_out_at, location, location_out
         FROM punches
         WHERE user_id = $1 AND clock_out_at IS NULL
         ORDER BY clock_in_at DESC LIMIT 1`,
        [userId]
      );
      await client.query('ROLLBACK');
      return res.status(409).json({
        ok: false,
        error: 'already_open',
        punch: existing.rows[0] || null
      });
    }

    const punch = insertResult.rows[0];
    await client.query('COMMIT');
    return res.status(201).json({ ok: true, punch });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Clock-in error:', err);
    return res.status(500).json({ ok: false, error: 'server_error' });
  } finally {
    client.release();
  }
});

/**
 * POST /punch/clock-out
 * Body: { userId: number, location?: { lat: number, lng: number } }
 */
router.post('/clock-out', async (req, res) => {
  const { userId, location } = req.body;
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    // Lock the open row to prevent races/orphans
    const open = await client.query(
      `SELECT id, clock_in_at, clock_out_at, location, location_out
       FROM punches
       WHERE user_id = $1 AND clock_out_at IS NULL
       ORDER BY clock_in_at DESC LIMIT 1
       FOR UPDATE`,
      [userId]
    );

    if (open.rowCount === 0) {
      await client.query('ROLLBACK');
      return res.status(404).json({ ok: false, error: 'no_open_punch' });
    }

    const punch = open.rows[0];
    const updated = await client.query(
      `UPDATE punches
       SET clock_out_at = NOW(), location_out = $1
       WHERE id = $2
       RETURNING *`,
      [location ? JSON.stringify(location) : null, punch.id]
    );

    await client.query('COMMIT');
    return res.status(200).json({ ok: true, punch: updated.rows[0] });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Clock-out error:', err);
    return res.status(500).json({ ok: false, error: 'server_error' });
  } finally {
    client.release();
  }
});

export default router;
```
