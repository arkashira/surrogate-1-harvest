# vanguard / ops

## 1. Diagnosis

- No content-addressed manifest exists → training/frontend hit HF API at runtime, causing 429s and non-reproducible epochs.
- Mixed-schema `enriched/` files include `source`/`ts` columns that break `load_dataset` expectations for surrogate-1 training.
- Lightning Studio is created fresh per run instead of reused → wastes ~80hr/mo quota and risks idle-stop death.
- Data ingestion uses `load_dataset(streaming=True)` on heterogeneous repos → triggers pyarrow `CastError` on schema mismatch.
- HF API rate limits block reproducible snapshots; no CDN-only fallback for training data fetches.

## 2. Proposed change

Create a minimal ops pipeline that:
- Generates a content-addressed manifest (JSON) for a single date folder after ingestion.
- Projects `enriched/` files to `{prompt, response}` only and stores as `batches/mirror-merged/{date}/{slug}.parquet` (no extra cols).
- Reuses a running Lightning Studio for training instead of recreating.
- Embeds the manifest file list so training uses CDN-only downloads (zero API calls).

Scope:
- `/opt/axentx/vanguard/ops/generate_manifest.py` (new)
- `/opt/axentx/vanguard/train/train.py` (patch: accept manifest path, use CDN URLs)
- `/opt/axentx/vanguard/ops/launch_studio.py` (new: reuse running studio)

### 3.1 generate_manifest.py

```python
#!/usr/bin/env python3
"""
Generate content-addressed manifest for a date folder.
Usage:
  python generate_manifest.py <repo> <date> <out_manifest.json>
"""
import json, hashlib, os, sys
from pathlib import Path
from huggingface_hub import list_repo_tree, hf_hub_download

HF_REPO = sys.argv[1]          # e.g. "datasets/myorg/vanguard-ingest"
DATE   = sys.argv[2]           # e.g. "2026-04-29"
OUT    = sys.argv[3]           # e.g. "data/manifest-2026-04-29.json"

FOLDER = f"enriched/{DATE}"

def sha256_bytes(b: bytes) -> str:
    return hashlib.sha256(b).hexdigest()

def main():
    # 1) list top-level folder once (single API call)
    items = list_repo_tree(repo_id=HF_REPO, path=FOLDER, recursive=False)
    files = [it.rfilename for it in items if it.type == "file"]

    manifest = {
        "repo": HF_REPO,
        "date": DATE,
        "generated_by": "generate_manifest.py",
        "files": []
    }

    for rel in files:
        # 2) download each file individually (bypasses streaming/mixed-schema issues)
        local_path = hf_hub_download(repo_id=HF_REPO, filename=f"{FOLDER}/{rel}")
        content = Path(local_path).read_bytes()

        # 3) content-address
        digest = sha256_bytes(content)
        cdn_url = f"https://huggingface.co/datasets/{HF_REPO}/resolve/main/{FOLDER}/{rel}"

        # 4) project to {prompt,response} at parse time (placeholder)
        #    In real usage, parse here and drop extra cols; for now store minimal metadata.
        manifest["files"].append({
            "filename": rel,
            "sha256": digest,
            "cdn_url": cdn_url,
            "size": len(content)
        })

    Path(OUT).write_text(json.dumps(manifest, indent=2))
    print(f"Wrote {len(files)} entries -> {OUT}")

if __name__ == "__main__":
    main()
```

### 3.2 launch_studio.py

```python
#!/usr/bin/env python3
"""
Reuse a running Lightning Studio if available; else start one.
"""
import os
from lightning_sdk import Studio, Teamspace, Machine

TEAMSPACE = os.getenv("LIGHTNING_TEAMSPACE", "default")
STUDIO_NAME = os.getenv("LIGHTNING_STUDIO", "vanguard-train")
MACHINE = Machine.L40S  # fallback; H200 requires lightning-lambda-prod

def get_or_start_studio():
    team = Teamspace(TEAMSPACE)
    running = [s for s in team.studios if s.name == STUDIO_NAME and s.status == "running"]
    if running:
        print(f"Reusing running studio: {STUDIO_NAME}")
        return running[0]

    print(f"Starting studio: {STUDIO_NAME}")
    studio = Studio.create(name=STUDIO_NAME, teamspace=TEAMSPACE, machine=MACHINE)
    return studio

if __name__ == "__main__":
    studio = get_or_start_studio()
    print(f"Studio: {studio.name} | Status: {studio.status} | Machine: {studio.machine}")
```
