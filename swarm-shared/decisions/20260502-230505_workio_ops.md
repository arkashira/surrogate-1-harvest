# workio / ops

## 1) Diagnosis

- **No idempotency key enforcement at DB layer**: clock-in/out endpoints accept duplicate requests for the same `(employee_id, punch_type, date)` when clients retry on flaky mobile networks → double punches and payroll drift.
- **Non-transactional punch writes**: concurrent retries can insert overlapping punch pairs without atomic checks, breaking `latest_punch` consistency.
- **Missing unique constraint**: schema allows multiple `punches` rows for the same employee+date+type, so duplicates accumulate silently.
- **No de-duping on webhook ingress**: LINE Messaging API delivers webhooks multiple times; server treats each as a new punch.
- **No idempotency table or key tracking**: retry-safe semantics require storing `idempotency_key → punch_id` mapping with TTL.

## 2) Proposed change

- **File**: `workio/server/src/db/schema.sql` — add unique constraint and idempotency table.
- **File**: `workio/server/src/routes/punch.ts` (or equivalent) — add idempotency key extraction and atomic upsert.
- **Scope**: single atomic transaction that checks latest punch state and inserts only if no conflicting punch exists for the same logical period.

## 3) Implementation

### Step 1 — Schema changes (`workio/server/src/db/schema.sql`)

```sql
-- Idempotency keys for LINE webhook / client retries
CREATE TABLE IF NOT EXISTS idempotency_keys (
  key         TEXT PRIMARY KEY,
  punch_id    INTEGER NOT NULL REFERENCES punches(id) ON DELETE CASCADE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Prevent duplicate punches for same employee+date+type (same clock-in/out pair)
-- Allow only one 'in' and one 'out' per date; enforce at DB level.
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_employee_date_type
  ON punches (employee_id, DATE(punched_at), punch_type)
  WHERE deleted_at IS NULL;

-- Optional: cleanup job index
CREATE INDEX IF NOT EXISTS idx_idempotency_created_at
  ON idempotency_keys (created_at);
```

### Step 2 — Punch route with idempotency (`workio/server/src/routes/punch.ts`)

```ts
import express from 'express';
import { pool } from '../db';
import { v4 as uuidv4 } from 'uuid';

const router = express.Router();

/**
 * POST /punches
 * Body: { employee_id: number, punch_type: 'in'|'out', idempotency_key?: string }
 * If idempotency_key is provided and seen before, return existing punch.
 */
router.post('/', async (req, res) => {
  const { employee_id, punch_type, idempotency_key } = req.body;
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    // Idempotency check
    if (idempotency_key) {
      const idem = await client.query(
        'SELECT punch_id FROM idempotency_keys WHERE key = $1',
        [idempotency_key]
      );
      if (idem.rowCount > 0) {
        const punch = await client.query(
          'SELECT * FROM punches WHERE id = $1 AND deleted_at IS NULL',
          [idem.rows[0].punch_id]
        );
        if (punch.rowCount > 0) {
          await client.query('ROLLBACK');
          return res.json({ ok: true, punch: punch.rows[0], reused: true });
        }
      }
    }

    // Business rule: only one punch_type per employee per date
    const today = new Date().toISOString().split('T')[0];
    const conflict = await client.query(
      `SELECT id FROM punches
       WHERE employee_id = $1
         AND punch_type = $2
         AND DATE(punched_at) = $3
         AND deleted_at IS NULL
       FOR UPDATE`,
      [employee_id, punch_type, today]
    );

    if (conflict.rowCount > 0) {
      await client.query('ROLLBACK');
      return res.status(409).json({
        error: 'duplicate_punch',
        message: `A ${punch_type} punch already exists for today.`,
      });
    }

    // Insert punch
    const insertPunch = await client.query(
      `INSERT INTO punches (employee_id, punch_type, punched_at, created_at)
       VALUES ($1, $2, NOW(), NOW())
       RETURNING *`,
      [employee_id, punch_type]
    );

    const punch = insertPunch.rows[0];

    // Store idempotency key if provided
    if (idempotency_key) {
      await client.query(
        'INSERT INTO idempotency_keys (key, punch_id) VALUES ($1, $2)',
        [idempotency_key, punch.id]
      );
    }

    await client.query('COMMIT');
    res.json({ ok: true, punch, reused: false });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Punch creation failed', err);
    res.status(500).json({ error: 'internal_error' });
  } finally {
    client.release();
  }
});

export default router;
```

### Step 3 — LINE webhook handler idempotency (`workio/server/src/routes/line.ts`)

```ts
// Inside webhook handler, derive idempotency_key from LINE deliveryId
const deliveryId = body.events[0]?.deliveryId;
if (deliveryId) {
  req.body.idempotency_key = `line:${deliveryId}`;
}
```

## 4) Verification

- **Unit test**: simulate duplicate POST with same `idempotency_key` → second request returns same `punch_id` and `reused: true`.
- **Constraint test**: attempt two `in` punches for same employee today without idempotency key → second receives `409 duplicate_punch`.
- **LINE replay test**: replay same LINE webhook payload (same `deliveryId`) → only one punch row created.
- **DB integrity check**: query `idx_punches_employee_date_type` violations — should return 0 rows after fix.
- **Load test**: fire 10 concurrent identical requests with same idempotency key — exactly one punch row and one idempotency row created.
