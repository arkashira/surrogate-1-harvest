# airship / backend

## Final Implementation Plan  
`airship discover` — zero-config discovery + training orchestration (CDN-only, quota-safe)

### Core behavior (what it does)
- Runs `granite-business-research.sh` (if present) and tags outputs.
- Executes `knowledge-rag` to surface the most-connected hub (e.g., MOC) and related docs.
- Lists one date-scoped folder from a HuggingFace dataset repo (non-recursive) and produces a CDN-only file-list JSON for Lightning training.
- Embeds that JSON into a Lightning-compatible training script template so training fetches via CDN (zero HF API calls during data loading).
- Reuses an existing Lightning Studio session if available; otherwise starts one (prefer L40S, fallback to public tier) and submits the run.
- Provides a safe `kaggle kernel push` helper that uses Bearer auth and `isPrivate=true` to avoid phone-verification blocks.
- Validates wrapper conventions (shebang + executable) and prints cron-safe guidance (`SHELL=/bin/bash`).
- Outputs a concise summary: top hub, file count, next steps, and safe-retry guidance for HF 429s.

### Why this is high-value and correct
- Consolidates scattered patterns into one repeatable command.
- Avoids HF API rate limits by using CDN URLs for training data and a single folder listing at discovery time.
- Enables immediate reuse in Lightning training pipelines with minimal friction.
- Fits within <2h: single script + small templates, no large refactor.
- Resolves contradictions from candidates by:
  - Preferring CDN-only data loading (no auth/API during training) for correctness and quota safety.
  - Using HF folder listing at discovery time; if the API is unavailable or rate-limited, allow a manual file-list fallback rather than failing the entire command.
  - Making Lightning orchestration optional and idempotent (reuse existing session) for robustness.
  - Keeping Kaggle push opt-in and safe (Bearer auth + private by default).

### `/opt/axentx/airship/bin/airship`

```bash
#!/usr/bin/env bash
# airship — zero-config discovery + training orchestration CLI
# Usage: airship discover [--dataset <repo>] [--folder <date-folder>] [--out <json>]
#                        [--lightning] [--kaggle-push] [--help]
#
# Patterns applied:
#  - business research + knowledge-rag + top-hub review
#  - HF CDN bypass + single folder file-list for Lightning training
#  - Lightning Studio orchestration (reuse or start)
#  - Safe Kaggle push (Bearer auth, private)
#  - wrapper shebang + executable checks + cron-safe SHELL guidance

set -euo pipefail

AIRSHIP_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
LOG() { printf '[airship][%s] %s\n' "$(date -u +'%Y-%m-%dT%H:%M:%SZ')" "$*"; }

# ---- helpers ----
require_command() {
  if ! command -v "$1" >/dev/null 2>&1; then
    LOG "ERROR: required command not found: $1"
    exit 1
  fi
}

validate_wrapper() {
  local script="$1"
  if [[ -f "$script" ]]; then
    if ! head -1 "$script" | grep -qE '^#!/usr/bin/env bash|^#!/bin/bash'; then
      LOG "WARN: wrapper missing Bash shebang (add '#!/usr/bin/env bash'): $script"
    fi
    if [[ ! -x "$script" ]]; then
      LOG "WARN: wrapper not executable (chmod +x): $script"
    fi
  fi
}

# ---- HF folder listing (CDN-safe) ----
list_hf_folder() {
  local dataset_repo="$1"
  local folder="$2"
  local api_url="https://huggingface.co/api/datasets/${dataset_repo}/tree/${folder}"
  local tree_response
  tree_response="$(curl -fsSL --retry 3 --retry-delay 5 "$api_url")" || return 1

  local cdn_base="https://huggingface.co/datasets/${dataset_repo}/resolve/main/${folder}"
  echo "$tree_response" | jq -r '.[] | select(.type=="file") | .path' | sort | \
    jq -R -s '
      split("\n") |
      map(select(length > 0)) |
      map({ path: ., cdn_url: ("'"$cdn_base"'/" + .) })
    '
}

# ---- Lightning orchestration ----
run_lightning_training() {
  local filelist_json="$1"
  local template="${AIRSHIP_ROOT}/templates/lightning_train.py.j2"
  local script="${AIRSHIP_ROOT}/bin/lightning_train.py"

  if [[ ! -f "$template" ]]; then
    LOG "WARN: Lightning template not found at $template — skipping Lightning submission"
    return 0
  fi

  # Render training script (lightweight dependency: jq + envsubst; fallback to simple sed)
  if command -v jq >/dev/null 2>&1 && command -v envsubst >/dev/null 2>&1; then
    export AIRSHIP_FILELIST="$(cat "$filelist_json")"
    envsubst < "$template" > "$script"
  else
    # simple fallback: inject filelist path as argument the script can read
    sed "s|__FILELIST_JSON__|${filelist_json}|g" "$template" > "$script"
  fi
  chmod +x "$script"

  LOG "Lightning training script prepared: $script"

  # Try to reuse existing Lightning Studio session; otherwise start one (best-effort)
  # This is advisory and non-blocking if lightning command is unavailable.
  if command -v lightning >/dev/null 2>&1; then
    LOG "Checking Lightning Studio sessions..."
    if lightning session list 2>/dev/null | grep -q 'RUNNING'; then
      LOG "Reusing existing Lightning Studio session."
      lightning run script "$script" --cloud --machine L40S 2>/dev/null || \
        lightning run script "$script" --cloud 2>/dev/null || \
        LOG "WARN: Lightning cloud run failed — run locally instead."
    else
      LOG "Starting Lightning Studio (L40S preferred, fallback to public)."
      lightning run script "$script" --cloud --machine L40S 2>/dev/null || \
        lightning run script "$script" --cloud 2>/dev/null || \
        LOG "WARN: Lightning cloud run failed — run locally instead."
    fi
  else
    LOG "INFO: 'lightning' CLI not available. Run training locally:"
    LOG "  python $script"
  fi
}

# ---- Kaggle push helper ----
run_kaggle_push() {
  local kaggle_script="${AIRSHIP_ROOT}/bin/kaggle_push.sh"
  if [[ -f "$kaggle_script" ]]; then
    LOG "Running Kaggle push helper (safe, private by default)..."
    bash "$kaggle_script"
  else
    LOG "WARN: kaggle_push.sh not found at $kaggle_script — skipping Kaggle push"
  fi
}
