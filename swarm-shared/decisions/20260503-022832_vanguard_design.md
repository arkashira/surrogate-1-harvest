# vanguard / design

## 1. Diagnosis
- No manifest cache: every preview/training launch triggers authenticated HF API calls (`list_repo_tree`, `load_dataset`) → burns quota and risks 429s.
- No CDN bypass: data loads route through authenticated `/api/` endpoints instead of public CDN URLs (`resolve/main/...`) → slower and rate-limited.
- No graceful 429 fallback: training scripts fail immediately on 429 instead of falling back to CDN or waiting/retrying.
- No file-list reuse: training re-enumerates repo files on every run instead of embedding a pre-listed JSON of paths for the target date folder.
- No studio reuse guard: training may recreate Lightning Studio instead of reusing a running one → wastes 80hr/mo quota.

## 2. Proposed change
- File: `/opt/axentx/vanguard/train.py` (or create if missing)
- Scope: add a small manifest/cache layer + CDN-only data loader + studio reuse guard.
- Lines: top of file (imports/config) and data-loading section.

## 3. Implementation
```python
# /opt/axentx/vanguard/train.py
import json, os, time, requests
from pathlib import Path
from huggingface_hub import list_repo_tree, hf_hub_download, HfApi
from lightning import Teamspace, Studio, Machine

HF_REPO = os.getenv("HF_REPO", "datasets/your-org/vanguard-mirror")
DATE_FOLDER = os.getenv("DATE_FOLDER", "2026-04-29")  # e.g. batches/mirror-merged/2026-04-29
MANIFEST_PATH = Path(f"/tmp/vanguard_manifest_{DATE_FOLDER.replace('/', '_')}.json")
CDN_BASE = f"https://huggingface.co/datasets/{HF_REPO}/resolve/main"

# 1) Manifest: list once, cache locally (Mac orchestration step should refresh after rate-limit window)
def get_file_manifest() -> list[str]:
    if MANIFEST_PATH.exists():
        return json.loads(MANIFEST_PATH.read_text())
    # single non-recursive list for the date folder to minimize API calls
    tree = list_repo_tree(repo_id=HF_REPO, path=DATE_FOLDER, recursive=False)
    files = [item.rfilename for item in tree if item.type == "file"]
    MANIFEST_PATH.write_text(json.dumps(files))
    return files

# 2) CDN-only streaming fetch (zero authenticated calls during training)
def stream_cdn_files(file_list: list[str]):
    for f in file_list:
        url = f"{CDN_BASE}/{f}"
        resp = requests.get(url, timeout=30)
        resp.raise_for_status()
        # yield projected {prompt, response} only; attribution via filename pattern
        # implement your parquet/record parsing here; example:
        # data = parse_parquet(resp.content)
        # for row in data:
        #     yield {"prompt": row["prompt"], "response": row["response"]}
        yield resp.content

# 3) Graceful 429 fallback: if CDN fails (rare), fall back to hf_hub_download (authenticated)
def safe_hf_download(file_path: str):
    try:
        return hf_hub_download(repo_id=HF_REPO, filename=file_path, repo_type="dataset")
    except Exception:
        # exponential backoff on 429
        time.sleep(360)
        return hf_hub_download(repo_id=HF_REPO, filename=file_path, repo_type="dataset")

# 4) Lightning Studio reuse (saves quota)
def get_or_create_studio(name: str = "vanguard-l40s"):
    for s in Teamspace.studios:
        if s.name == name and s.status == "Running":
            return s
    # if stopped, restart instead of recreate
    for s in Teamspace.studios:
        if s.name == name and s.status == "Stopped":
            s.start(machine=Machine.L40S)
            return s
    return Studio.create(
        name=name,
        machine=Machine.L40S,
        create_ok=True,
    )

# Example usage in training loop
if __name__ == "__main__":
    studio = get_or_create_studio()
    files = get_file_manifest()
    for content in stream_cdn_files(files):
        # train step using projected {prompt, response}
        pass
```
