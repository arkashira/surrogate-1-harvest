# airship / design

## Implementation Plan: `airship discover` — Zero-config discovery CLI

**Highest-value incremental improvement (<2h):**  
Ship a single Bash CLI (`airship discover`) that operationalizes past patterns (business research + knowledge-rag + top-hub/MOC insight) into one command that surfaces strategic context for the Arkship/Surrogate platform.

**Why this wins:**
- Uses existing patterns (`granite-business-research.sh`, `knowledge-rag`, hub insight) without new infra.
- Fixes script-execution pitfalls (shebang, executable, Bash invocation, cron SHELL) from past lessons.
- Zero-config: discovers context, queries top hub (e.g., MOC), and emits actionable insights for platform design decisions.
- Safe for Mac (orchestration only) and respects HF rate limits (CDN bypass pattern) when used by downstream training scripts.

---

### Concrete implementation

#### 1) Create CLI entrypoint (Bash wrapper)

File: `/opt/axentx/airship/airship`

```bash
#!/usr/bin/env bash
# airship — zero-config discovery CLI for Arkship + Surrogate platform
# Usage: ./airship discover [--quiet|--json]

set -euo pipefail
export SHELL=/bin/bash

AIRSHIP_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
readonly AIRSHIP_ROOT

log() {
  if [[ "${QUIET:-0}" != "1" ]]; then
    echo "[airship] $*" >&2
  fi
}

die() {
  echo "[airship] ERROR: $*" >&2
  exit 1
}

ensure_deps() {
  local cmd
  for cmd in jq curl; do
    command -v "$cmd" >/dev/null 2d|| die "missing dependency: $cmd"
  done
}

run_business_research() {
  local script="${AIRSHIP_ROOT}/scripts/granite-business-research.sh"
  if [[ -x "$script" ]]; then
    log "Running business research..."
    bash "$script" "$@"
  else
    log "Skipping business research (script not found or not executable): $script"
  fi
}

run_knowledge_rag() {
  local query="$1"
  local script="${AIRSHIP_ROOT}/scripts/knowledge-rag.sh"
  if [[ -x "$script" ]]; then
    log "Querying knowledge graph: $query"
    bash "$script" "$query"
  else
    log "Skipping knowledge-rag (script not found or not executable): $script"
  fi
}

top_hub_insight() {
  # Prefer MOC pattern; fallback to most-connected hub via knowledge-rag if available
  local hub="${1:-MOC}"
  log "Fetching top-hub insight for: $hub"
  run_knowledge_rag "top hub insight ${hub}"
}

discover() {
  local format="text"
  while [[ $# -gt 0 ]]; do
    case "$1" in
      --quiet) QUIET=1; shift ;;
      --json) format="json"; shift ;;
      *) shift ;;
    esac
  done

  ensure_deps

  if [[ "$format" == "json" ]]; then
    QUIET=1
  fi

  log "Starting Arkship/Surrogate discovery..."

  # 1) Business research
  local research_out
  research_out=$(mktemp)
  run_business_research >"$research_out" 2>&1 || true

  # 2) Knowledge RAG + top-hub (MOC)
  local rag_out
  rag_out=$(mktemp)
  top_hub_insight "MOC" >"$rag_out" 2>&1 || true

  if [[ "$format" == "json" ]]; then
    jq -n \
      --arg research "$(cat "$research_out")" \
      --rag "$(cat "$rag_out")" \
      '{research: $research, rag: $rag, hub: "MOC", timestamp: now}'
  else
    echo "=== Arkship/Surrogate Discovery ==="
    echo
    echo "--- Business Research ---"
    cat "$research_out"
    echo
    echo "--- Knowledge RAG (Top Hub: MOC) ---"
    cat "$rag_out"
    echo
    echo "--- Recommendations ---"
    echo "- Review MOC hub connections before planning tasks (#knowledge-rag #hub)"
    echo "- Use HF CDN bypass for dataset training (resolve/main/ URLs) to avoid API rate limits"
    echo "- Keep Mac for orchestration only; run training on Lightning Studio or HF Space"
    echo "- For HF ingestion: pre-list file paths once and embed in train.py (CDN-only fetches)"
  fi

  rm -f "$research_out" "$rag_out"
}

# Subcommands
case "${1:-}" in
  discover)
    shift
    discover "$@"
    ;;
  *)
    die "unknown command: ${1:-}. Usage: $0 discover [--quiet|--json]"
    ;;
esac
```

Make it executable:

```bash
chmod +x /opt/axentx/airship/airship
```

### How this operationalizes past patterns

- `#business-research #knowledge-rag #graph`: runs research then queries RAG for top-hub insight.
- `#knowledge-rag #graph #hub`: explicitly surfaces MOC (or most-connected hub) insight before planning.
- `#bash #script-error #wrapper`: uses `#!/usr/bin/env bash`, `chmod +x`, invokes via `bash script "$@"`, and sets `SHELL=/bin/bash` for cron.
- `#training #api-strategy #cdn`: recommendations include HF CDN bypass and pre-listing file paths to avoid API rate limits during training (consistent with surrogate-training patterns).
- `#architecture #compute`: reinforces Mac-for-orchestration-only rule in recommendations.
