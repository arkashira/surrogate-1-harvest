# workio / frontend

## Final synthesized implementation (best of both proposals)

**Chosen filename:** `workio/src/components/PunchButton.tsx`  
**Scope:** optimistic UI, idempotency, disable-on-submit, local rollback on failure, and clear visual states. This keeps the component reusable (accepts `type` and `onPunch`) and avoids hardwiring query logic so it can be used with or without React Query.

```tsx
// workio/src/components/PunchButton.tsx
import { useState, useCallback } from 'react';
import { useToast } from '@/hooks/use-toast';
import { Button } from '@/components/ui/button';
import { Loader2, CheckIn, CheckOut } from 'lucide-react';

type PunchStatus = 'idle' | 'pending' | 'synced' | 'failed';

export function PunchButton({
  type,
  onPunch,
  disabled,
}: {
  type: 'in' | 'out';
  onPunch: (opts: {
    type: 'in' | 'out';
    idempotencyKey: string;
    clientTimestamp: number;
  }) => Promise<void>;
  disabled?: boolean;
}) {
  const [status, setStatus] = useState<PunchStatus>('idle');
  const { toast } = useToast();

  const handlePunch = useCallback(async () => {
    if (status === 'pending' || disabled) return;

    const idempotencyKey = `${Date.now()}-${Math.random().toString(36).slice(2, 10)}`;
    const clientTimestamp = Date.now();
    setStatus('pending');

    try {
      await onPunch({ type, idempotencyKey, clientTimestamp });
      setStatus('synced');
      toast({
        title: `Clock ${type === 'in' ? 'in' : 'out'} recorded`,
        description: 'Your punch was saved successfully.',
      });

      // brief success state then reset to allow next punch
      setTimeout(() => setStatus('idle'), 1800);
    } catch (err) {
      setStatus('failed');
      toast({
        variant: 'destructive',
        title: 'Failed to record punch',
        description:
          err instanceof Error ? err.message : 'Please check your connection and try again.',
      });

      // allow retry after short backoff
      setTimeout(() => setStatus('idle'), 3500);
    }
  }, [type, status, disabled, onPunch, toast]);

  const isPending = status === 'pending';
  const isSynced = status === 'synced';

  return (
    <Button
      size="lg"
      className="w-32 h-32 rounded-full text-lg font-semibold"
      onClick={handlePunch}
      disabled={isPending || isSynced || disabled}
      variant={type === 'in' ? 'default' : 'secondary'}
      aria-busy={isPending}
      aria-live="polite"
    >
      {isPending ? (
        <Loader2 className="h-6 w-6 animate-spin" />
      ) : isSynced ? (
        <CheckIn className="h-6 w-6" />
      ) : type === 'in' ? (
        <CheckIn className="h-6 w-6" />
      ) : (
        <CheckOut className="h-6 w-6" />
      )}
    </Button>
  );
}
```

### Recommended integration (React Query example)

```tsx
// workio/src/pages/Dashboard.tsx
import { PunchButton } from '@/components/PunchButton';
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { postPunch } from '@/lib/api';

export function Dashboard() {
  const queryClient = useQueryClient();

  const punchMutation = useMutation({
    mutationFn: postPunch,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['punches'] });
    },
  });

  const handlePunch = async ({
    type,
    idempotencyKey,
    clientTimestamp,
  }: {
    type: 'in' | 'out';
    idempotencyKey: string;
    clientTimestamp: number;
  }) => {
    await punchMutation.mutateAsync({
      type,
      idempotencyKey,
      clientTimestamp,
    });
  };

  return (
    <div className="flex flex-col items-center gap-6 p-4">
      <PunchButton type="in" onPunch={handlePunch} disabled={punchMutation.isPending} />
      <PunchButton type="out" onPunch={handlePunch} disabled={punchMutation.isPending} />
    </div>
  );
}
```

### Minimal API helper

```ts
// workio/src/lib/api.ts
export async function postPunch(payload: {
  type: 'in' | 'out';
  idempotencyKey: string;
  clientTimestamp: number;
}) {
  const res = await fetch('/api/punches', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      // send idempotency key in header (preferred) and body for compatibility
      'Idempotency-Key': payload.idempotencyKey,
    },
    body: JSON.stringify(payload),
  });

  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    throw new Error(err.message || 'Punch request failed');
  }

  return res.json();
}
```

### Key synthesized decisions (why this is best)

- **Optimistic UI + clear states** (Candidate 1): immediate spinner, success checkmark, disabled during pending, and timed reset. This removes perceived lag and prevents double-taps.
- **Idempotency key on client** (both): generated per action and sent in header/body so server can dedupe retries and race conditions.
- **Local rollback on failure** (Candidate 1): failed state shown via toast, button re-enabled after backoff so user can retry.
- **Reusable component API** (Candidate 1): accepts `type` and `onPunch` so it isn’t coupled to a specific data layer (works with or without React Query).
- **Minimal, focused scope** (Candidate 2): avoids baking query logic into the button; keeps concerns separated and easier to test.
- **Accessibility & UX polish**: `aria-busy`, `aria-live`, and sensible timeouts (1.8s success, 3.5s error) balance feedback without being distracting.
