# surrogate-1 / ops

## Highest-value incremental improvement (≤2h)

**Chosen fix**: Replace `bin/dataset-enrich.sh` with a **manifest-driven, CDN-bypass ingestion pipeline** that eliminates HF API rate limits (429) during training and removes recursive `list_repo_files` calls that hit commit-cap/throughput ceilings.

- **Why this wins**:
  - Removes the 429/1000-5min API throttle during data loading (training no longer calls `/api/`).
  - Avoids recursive `list_repo_files` pagination and 128-commit/hr repo cap by listing once per date folder and embedding the file list in the training job.
  - Fits the existing 16-shard runner layout and dedup store contract.
  - Deliverable is a single script + small train.py patch — <2h.

---

## Implementation plan

1. **Create `bin/build-manifest.sh`**  
   - Runs on Mac (or cron) after rate-limit window clears.
   - Uses `list_repo_tree(path, recursive=False)` per date folder under the dataset repo.
   - Emits `manifests/public-merged/<date>/files.json` with CDN URLs and local paths.
   - Commits manifest JSON to the surrogate-1-runner repo (or keeps it locally for Lightning jobs).

2. **Update `bin/dataset-enrich.sh` → `bin/dataset-enrich-cdn.sh`**  
   - Accepts a manifest file as input (`--manifest manifests/.../files.json`).
   - Each shard (0–15) filters entries by `hash(slug) % 16 == SHARD_ID`.
   - Downloads via CDN (`curl -L <cdn_url>`) with no Authorization header.
   - Projects to `{prompt, response}` only at parse time (avoids pyarrow CastError on mixed schemas).
   - Writes `batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl` (existing convention).
   - Uses central `lib/dedup.py` md5 store for cross-source dedup.

3. **Add/modify `train.py` (Lightning Studio entrypoint)**  
   - Reads the same manifest JSON (bundled in the repo or downloaded once at start).
   - Data loader uses CDN URLs directly (`wget`/`curl` or `requests` streaming) — zero HF API calls during training.
   - Falls back to local files if present.
   - Reuses running Studio if available; restarts only if stopped.

4. **Cron / workflow changes**  
   - Keep `ingest.yml` matrix (16 shards) but pass manifest path via `env`.
   - Add a lightweight “manifest builder” job that runs once per day (or on `workflow_dispatch`) before the 16-shard matrix.

5. **Validation**
   - Dry-run one shard locally: verify CDN downloads, schema projection, and dedup output.
   - Confirm training job starts and streams from CDN without any `/api/` requests (check logs for 429).

### 1) `bin/build-manifest.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
DATE="${1:-$(date +%Y-%m-%d)}"
OUT_DIR="manifests/public-merged/${DATE}"
OUT_FILE="${OUT_DIR}/files.json"

mkdir -p "${OUT_DIR}"

python3 - <<PY
import os, json, sys
from huggingface_hub import HfApi

api = HfApi()
repo = os.environ["REPO"]
date = os.environ["DATE"]

# non-recursive list for the date folder
entries = api.list_repo_tree(repo=repo, path=date, recursive=False)

files = []
for e in entries:
    if e.type != "file":
        continue
    cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{date}/{e.path}"
    files.append({
        "path": e.path,
        "cdn_url": cdn_url,
        "size": getattr(e, "size", None)
    })

out_path = os.environ["OUT_FILE"]
with open(out_path, "w") as f:
    json.dump(files, f, indent=2)
print(f"Wrote {len(files)} entries to {out_path}")
```

### 2) `bin/dataset-enrich-cdn.sh` (replacement worker)

```bash
#!/usr/bin/env bash
set -euo pipefail

# Required env
: "${SHARD_ID:?Need SHARD_ID 0-15}"
: "${MANIFEST_FILE:?Need MANIFEST_FILE path}"
: "${DATE:?Need DATE e.g. 2026-05-03}"

REPO="axentx/surrogate-1-training-pairs"
OUT_DIR="batches/public-merged/${DATE}"
TS=$(date +%H%M%S)
OUT_FILE="${OUT_DIR}/shard${SHARD_ID}-${TS}.jsonl"

mkdir -p "${OUT_DIR}"

python3 - <<PY
import os, json, hashlib, subprocess, sys
from pathlib import Path

shard = int(os.environ["SHARD_ID"])
manifest_path = os.environ["MANIFEST_FILE"]
out_path = os.environ["OUT_FILE"]

with open(manifest_path) as f:
    files = json.load(f)

def assign_shard(slug):
    return int(hashlib.md5(slug.encode()).hexdigest(), 16) % 16

selected = [f for f in files if assign_shard(f["path"]) == shard]
print(f"Shard {shard}: processing {len(selected)} files")

import pyarrow.parquet as pq
import pyarrow as pa
import numpy as np
import io

def project_to_pair(raw_bytes):
    try:
        table = pq.read_table(io.BytesIO(raw_bytes))
    except Exception as exc:
        # fallback: skip malformed
        return None
    # Keep only prompt/response; tolerate extra columns
    cols = set(table.column_names)
    prompt_col = next((c for c in ("prompt", "instruction", "input") if c in cols), None)
    response_col = next((c for c in ("response", "output", "completion") if c in cols), None)
    if not prompt_col or not response_col:
        return None
    df = table.select([prompt_col, response_col]).to_pandas()
    records = []
    for _, row in df.iterrows():
        records.append({"prompt": str(row[prompt_col]), "response": str(row[response_col])})
    return records

from lib.dedup import DedupStore  # existing central store
dedup = DedupStore()

written = 0
with open(out_path, "w") as out_f:
    for item in selected:
        cdn_url = item["cdn_url"]
        # stream download via CDN (no auth)
        proc = subprocess.run(
            ["curl", "-L", "-s", "-f", cdn_url],
            capture_output=True,
            check=False
        )
        if proc.returncode != 0:
            print(f"Download failed: {cdn_url}", file=sys.stderr)
            continue
        pairs = project_to_pair(proc.stdout)
        if not pairs:
            continue
        for p in pairs:
            h = hashlib.md5((p["prompt"] + "\n" + p["response"]).encode()).hexdigest()
            if dedup.seen(h):
                continue
            dedup.add(h)
            out_f.write(json.dumps(p) + "\n")
            written += 1
print(f"Shard {shard}: wrote {written} pairs to {out_path}")
PY
```

Make executable:

```bash
chmod +x bin/dataset-enrich-cdn.sh
```
