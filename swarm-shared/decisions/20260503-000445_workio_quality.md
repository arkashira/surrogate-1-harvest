# workio / quality

## Final Synthesis & Action Plan

**Best consolidated diagnosis**  
- Duplicate punches originate from missing **idempotency** (API + webhook retries/double-taps) and missing **state-machine enforcement** (invalid sequences).  
- **Transactional integrity** between punches and daily summaries is absent, so counts can diverge.  
- Frontend has **no optimistic UI or durable pending queue**, so users see spinners and can lose punches on flaky networks or offline.  
- No **rollback/compensation or reconciliation** when processing fails mid-flow.

**Chosen approach (correct + actionable)**  
1. **Server-side guardrails first** (non-negotiable):  
   - Idempotency key enforcement at the API/webhook boundary.  
   - State-machine validation for punch transitions.  
   - Transactional punch + daily-summary update (or mark summaries stale for async repair).  
   - Return clear, idempotent responses (200/201 for success, 409 for invalid transition, 400/422 for bad requests).

2. **Frontend second (optimistic + resilient)**:  
   - Optimistic UI with local pending queue and exponential-backoff retry.  
   - Use stable client-generated idempotency keys (crypto.randomUUID) persisted to storage so retries across reloads remain idempotent.  
   - Clear UX states: idle → pending → confirmed | error (with retry).

3. **Operational/recovery**:  
   - Background reconciliation job to fix stale summaries and orphaned partial state.  
   - Log and alert on repeated transition violations or idempotency collisions.

### Backend: transactional punch service with idempotency + state machine

```ts
// server/src/services/punchService.ts
import { Pool } from 'pg';
import { v4 as uuidv4 } from 'uuid';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

type PunchType = 'clock_in' | 'clock_out';

interface PunchRow {
  id: string;
  tenant_id: string;
  user_id: string;
  type: PunchType;
  punched_at: Date;
  latitude?: number;
  longitude?: number;
  idempotency_key: string;
  created_at: Date;
}

interface CreatePunchParams {
  tenantId: string;
  userId: string;
  type: PunchType;
  punchedAt?: Date;
  latitude?: number;
  longitude?: number;
  idempotencyKey: string;
}

function isValidTransition(last: PunchRow | null, nextType: PunchType): boolean {
  if (!last) return nextType === 'clock_in';
  if (last.type === 'clock_in') return nextType === 'clock_out';
  return nextType === 'clock_in';
}

async function getLastPunch(client: Pool, tenantId: string, userId: string): Promise<PunchRow | null> {
  const res = await client.query(
    `SELECT * FROM punches
     WHERE tenant_id = $1 AND user_id = $2
     ORDER BY punched_at DESC, created_at DESC LIMIT 1`,
    [tenantId, userId]
  );
  return res.rows[0] || null;
}

/**
 * Creates a punch with idempotency and state-machine enforcement.
 * Updates daily summary in the same transaction (or marks stale for reconciliation).
 */
export async function createPunch(params: CreatePunchParams): Promise<{ punch: PunchRow; created: boolean }> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotency check
    const idem = await client.query<PunchRow>(
      `SELECT * FROM punches
       WHERE tenant_id = $1 AND user_id = $2 AND idempotency_key = $3
       LIMIT 1`,
      [params.tenantId, params.userId, params.idempotencyKey]
    );
    if (idem.rows[0]) {
      await client.query('ROLLBACK');
      return { punch: idem.rows[0], created: false };
    }

    // State-machine guard
    const last = await getLastPunch(client, params.tenantId, params.userId);
    if (!isValidTransition(last, params.type)) {
      await client.query('ROLLBACK');
      throw new Error(`Invalid punch transition: last=${last?.type}, next=${params.type}`);
    }

    const punch: PunchRow = {
      id: uuidv4(),
      tenant_id: params.tenantId,
      user_id: params.userId,
      type: params.type,
      punched_at: params.punchedAt || new Date(),
      latitude: params.latitude,
      longitude: params.longitude,
      idempotency_key: params.idempotencyKey,
      created_at: new Date(),
    };

    await client.query(
      `INSERT INTO punches(id, tenant_id, user_id, type, punched_at, latitude, longitude, idempotency_key, created_at)
       VALUES($1,$2,$3,$4,$5,$6,$7,$8,$9)`,
      [
        punch.id,
        punch.tenant_id,
        punch.user_id,
        punch.type,
        punch.punched_at,
        punch.latitude,
        punch.longitude,
        punch.idempotency_key,
        punch.created_at,
      ]
    );

    // Keep daily summary consistent: mark affected day stale OR update counts transactionally.
    // Here we mark stale for simplicity + safety (async worker recomputes).
    const punchedAtDay = new Date(punch.punched_at);
    punchedAtDay.setHours(0, 0, 0, 0);
    await client.query(
      `INSERT INTO daily_summary_status (tenant_id, user_id, summary_date, stale, updated_at)
       VALUES ($1,$2,$3,$4,now())
       ON CONFLICT (tenant_id, user_id, summary_date)
       DO UPDATE SET stale = true, updated_at = now()`,
      [params.tenantId, params.userId, punchedAtDay]
    );

    await client.query('COMMIT');
    return { punch, created: true };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### API route: require idempotency key and return idempotent responses

```ts
// server/src/routes/punch.ts
import express from 'express';
import { createPunch } from '../services/punchService';

const router = express.Router();

router.post('/:type(clock_in|clock_out)', async (req, res) => {
  try {
    const { type } = req.params;
    const { tenantId, userId, punchedAt, latitude, longitude, idempotencyKey } = req.body;

    if (!tenantId || !userId || !idempotencyKey) {
      return res.status(400).json({ error: 'tenantId, userId, and idempotencyKey are required' });
    }

    const result = await createPunch({
      tenantId,
      userId,
      type: type as 'clock_in' | 'clock_out',
      punchedAt: punchedAt ? new Date(punchedAt) : undefined,
      latitude,
      longitude,
      idempotencyKey,
    });

    // Idempotent success: return existing punch with 200
    return res.status(result.created ? 201 : 200).json({ punch: result.punch, created: result.created });
  } catch (err: any) {
    if (err.message && err.message.includes('Invalid punch transition')) {
      return res.status(409).json({ error: err.message });
    }
    return res.status(500).json({ error: err.message });
  }
});

export default router;
```
