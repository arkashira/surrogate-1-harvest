## Objective
Eliminate HF API 429s during training and reduce Lightning quota/wait time by:
1. **Bypassing the HF API/CDN auth wall** for dataset fetches during training.
2. **Reusing a running Lightning Studio** instead of recreating per job.

Scope: no schema/DB changes; safe fallback to legacy loader; ship <2h.

---

## Architecture (single source of truth)

- **Pre-list step** (run once per date folder):  
  `tools/hf_cdn_list.py` → produces `file_list.json` (repo, folder, files[]).  
  Uses one `list_repo_tree` call; minimal API usage.

- **Training loader** (`surrogate/training/hf_cdn_dataset.py`):  
  Reads `file_list.json`; downloads via public CDN URLs  
  `https://huggingface.co/datasets/{repo}/resolve/main/{path}` (no auth, no rate limit).  
  Caches locally; yields `{local_path, rel_path}` for downstream parsers.

- **Launcher / Lightning integration** (`surrogate/training/launch_studio.py`):  
  - `get_or_create_studio(name)` → reuse running, restart stopped, or create.  
  - Runs training command inside the studio with CDN loader args.

- **Ops workflow / UI**:  
  - CLI: `ops_workflow.sh` drives pre-list + launch.  
  - Optional UI: add pre-launch step to generate `file_list.json` and pass to training command.

### 1) Pre-list utility (run before training)
```python
# tools/hf_cdn_list.py
import json
import argparse
from huggingface_hub import HfApi

def list_date_folder(repo: str, date_folder: str, out_path: str):
    api = HfApi()
    tree = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)
    files = [item.rfilename for item in tree if item.type == "file"]
    payload = {"repo": repo, "folder": date_folder, "files": files}
    with open(out_path, "w") as f:
        json.dump(payload, f, indent=2)
    print(f"Saved {len(files)} files to {out_path}")

if __name__ == "__main__":
    p = argparse.ArgumentParser()
    p.add_argument("--repo", required=True)
    p.add_argument("--folder", required=True)
    p.add_argument("--out", default="file_list.json")
    args = p.parse_args()
    list_date_folder(args.repo, args.folder, args.out)
```

### 2) CDN-only dataset loader
```python
# surrogate/training/hf_cdn_dataset.py
import json
import requests
from pathlib import Path
from torch.utils.data import IterableDataset

class HFCDNDataset(IterableDataset):
    def __init__(self, file_list_path: str, cache_dir: str = ".hf_cdn_cache"):
        super().__init__()
        with open(file_list_path) as f:
            meta = json.load(f)
        self.repo = meta["repo"]
        self.folder = meta["folder"]
        self.files = meta["files"]
        self.cache_dir = Path(cache_dir)
        self.cache_dir.mkdir(parents=True, exist_ok=True)

    def _download(self, rel_path: str) -> Path:
        url = f"https://huggingface.co/datasets/{self.repo}/resolve/main/{rel_path}"
        out = self.cache_dir / Path(rel_path).name
        if out.exists():
            return out
        r = requests.get(url, stream=True, timeout=60)
        r.raise_for_status()
        with open(out, "wb") as f:
            for chunk in r.iter_content(chunk_size=8192):
                f.write(chunk)
        return out

    def __iter__(self):
        for rel in self.files:
            local = self._download(rel)
            # Project to {prompt, response} downstream; keep schema-agnostic here.
            yield {"local_path": str(local), "rel_path": rel}
```

### 3) Lightning Studio reuse + launcher
```python
# surrogate/training/launch_studio.py
from lightning import Lightning, Teamspace, Machine, Studio

def get_or_create_studio(name: str, machine: Machine = Machine.L40S) -> Studio:
    teamspace = Teamspace()
    for s in teamspace.studios:
        if s.name == name:
            if s.status == "Running":
                print(f"Reusing running studio: {name}")
                return s
            else:
                print(f"Restarting stopped studio: {name}")
                s.start(machine=machine)
                return s
    print(f"Creating new studio: {name}")
    return Studio(name=name, machine=machine, create_ok=True)

def run_training_with_cdn(file_list_path: str, repo: str, script: str):
    studio = get_or_create_studio("surrogate-train-l40s")
    if studio.status != "Running":
        studio.start(machine=Machine.L40S)

    cmd = [
        "python", script,
        "--file-list", file_list_path,
        "--repo", repo,
        "--output-dir", "/workspace/output"
    ]
    studio.run(" ".join(cmd), cwd="/workspace")
```

### 4) Ops workflow script
```bash
#!/usr/bin/env bash
# ops_workflow.sh
set -euo pipefail

REPO="huggingface/coherence"
FOLDER="2024-06-01"
FILE_LIST="file_list.json"
TRAIN_SCRIPT="train.py"

# 1) Pre-list (single API call)
python tools/hf_cdn_list.py --repo "$REPO" --folder "$FOLDER" --out "$FILE_LIST"

# 2) Launch training in reused studio with CDN loader
python surrogate/training/launch_studio.py \
  --file-list "$FILE_LIST" \
  --repo "$REPO" \
  --script "$TRAIN_SCRIPT"
```

(If you prefer direct CLI without Python launcher, replace step 2 with:)
```bash
LIGHTNING_STUDIO_NAME="airship-studio"
LIGHTNING_TEAMSPACE_NAME="airship-team"
lightning studio reuse --name "$LIGHTNING_STUDIO_NAME" --team "$LIGHTNING_TEAMSPACE_NAME"
python "$TRAIN_SCRIPT" --dataset-path "https://huggingface.co/datasets/${REPO}/resolve/main/"
```

## Fallback & Safety
- Training script accepts `--file-list`. If missing, falls back to legacy loader.
- CDN downloads are idempotent (local cache).
- Studio reuse checks status; restarts if stopped; creates only if absent.
