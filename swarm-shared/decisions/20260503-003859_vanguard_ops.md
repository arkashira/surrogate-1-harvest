# vanguard / ops

## 1. Diagnosis

- No persisted `(repo, dateFolder)` file manifest → every training run re-enumerates via authenticated HF API → quota burn and 429 risk.
- Data loader likely uses `load_dataset(streaming=True)` or recursive enumeration on heterogeneous repos → amplifies rate-limit exposure and risks pyarrow `CastError`.
- No CDN bypass strategy in place → training jobs perform authenticated API calls during data loading instead of using public CDN URLs.
- Lightning Studio reuse not enforced → each run may recreate studio and waste 80hr/mo quota.
- No guard against idle-stop training death — if studio stops, training dies without automatic restart on an available machine.

## 2. Proposed change

Create `/opt/axentx/vanguard/ops/manifest_and_cdn.py` (single orchestrator script) that:
- Runs on the Mac (or any orchestration node) after rate-limit window clears.
- Calls `list_repo_tree(repo, path=dateFolder, recursive=False)` once.
- Persists `manifest_{repo}_{date}.json` with CDN URLs (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`).
- Embeds this manifest path into the training script so Lightning training does CDN-only fetches with zero API calls.
- Reuses a running Lightning Studio if present; otherwise starts one on L40S (falls back to public tier).
- Checks studio status before `.run()` and restarts if stopped.

Scope: new file + small patch to training launcher (if exists) or add `--manifest` arg to existing train entrypoint.

```python
# /opt/axentx/vanguard/ops/manifest_and_cdn.py
#!/usr/bin/env bash
# Purpose: Generate persisted file manifest for (repo, dateFolder) and launch
# Lightning training with CDN-only fetches and studio reuse.
#
# Usage:
#   bash manifest_and_cdn.sh <repo> <dateFolder> [--train]
#
# Notes:
# - Uses HF API once to list a single folder (non-recursive) to avoid quota burn.
# - CDN URLs do NOT count against API rate limits.
# - Reuses running Lightning Studio to save quota.
# - If studio stopped, restarts on L40S (falls back to public tier).

set -euo pipefail
SHELL=/bin/bash

REPO="${1:-}"
DATEFOLDER="${2:-}"
DO_TRAIN="${3:-}"

if [[ -z "$REPO" || -z "$DATEFOLDER" ]]; then
  echo "Usage: $0 <repo> <dateFolder> [--train]"
  exit 1
fi

MANIFEST_FILE="/opt/axentx/vanguard/ops/manifest_${REPO//[\/]/_}_${DATEFOLDER}.json"
PYTHON_SCRIPT="/opt/axentx/vanguard/train.py"

echo "Generating manifest for ${REPO}/${DATEFOLDER} -> ${MANIFEST_FILE}"

# Use huggingface_hub from Python to list single folder (non-recursive)
python3 - "$REPO" "$DATEFOLDER" "$MANIFEST_FILE" <<'PY'
import json, os, sys
from huggingface_hub import HfApi

repo = sys.argv[1]
date_folder = sys.argv[2].strip("/")
out_path = sys.argv[3]

api = HfApi()
# List only the requested folder (non-recursive) to minimize API calls.
tree = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)

entries = []
for item in tree:
    # Only include files (skip subfolders for this manifest)
    if item.type == "file":
        cdn_url = f"https://huggingface.co/datasets/{repo}/resolve/main/{date_folder}/{item.path}"
        entries.append({
            "path": item.path,
            "cdn_url": cdn_url,
            "size": getattr(item, "size", None)
        })

os.makedirs(os.path.dirname(out_path), exist_ok=True)
with open(out_path, "w") as f:
    json.dump({"repo": repo, "date_folder": date_folder, "files": entries}, f, indent=2)

print(f"Wrote {len(entries)} entries to {out_path}")
PY

echo "Manifest generated."

# If --train requested, launch Lightning training with studio reuse + CDN-only strategy
if [[ "$DO_TRAIN" == "--train" ]]; then
  echo "Launching training with CDN manifest..."

  python3 "$PYTHON_SCRIPT" \
    --manifest "$MANIFEST_FILE" \
    --repo "$REPO" \
    --date-folder "$DATEFOLDER" \
    --use-cdn-only \
    --reuse-studio

  echo "Training command issued."
else
  echo "Skipping training (--train not provided). Manifest ready for use."
fi
```
