# Costinel / ops

### Final Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h, production-ready)

**What we ship**
- A **non-blocking Top-Hub Signal Panel** mounted on the Costinel dashboard (sidebar or top banner area).
- Defaults to hub **MOC** (configurable via `VITE_HUB_NAME`).
- Shows: hub title, short description, top 3 signals (anomalies/opportunities), related documents count, and a “View in Knowledge Graph” link.
- **CDN-first data strategy**: file list generated once on Mac orchestration, saved to `src/data/hub-files.json`, and panel loads content via CDN (`/resolve/main/...`) at runtime — zero Hugging Face API calls during dashboard use.
- **Lightning-aware**: panel is passive (no training); training orchestration remains on Lightning Studio per surrogate-1 patterns.

---

### Why this is highest-value (<2h)
- Directly applies **#knowledge-rag #graph #hub** patterns (top-hub insight).
- Improves ops visibility immediately for Costinel users.
- Reuses existing patterns (CDN bypass, file-list pre-generation) without touching training pipelines.
- Small, safe surface: one component + one config + one build-time asset + one generator script.

---

### Implementation Steps (concrete)

1. **Add config** (`src/config/hub.ts`)
   - Default hub = `MOC`
   - Allow override via `VITE_HUB_NAME`

2. **Create panel component** (`src/components/TopHubSignalPanel.tsx`)
   - Loads `hub-files.json` (generated at build time)
   - Picks latest file for the hub (e.g., `moc-latest.json`)
   - Fetches content via CDN (`https://huggingface.co/datasets/.../resolve/main/...`)
   - Renders title, description, top 3 signals, documents count, and graph link
   - Graceful fallback if CDN fails (skeleton → empty state)

3. **Generate file list on Mac orchestration** (one-time or cron)
   - Script: `scripts/generate-hub-file-list.sh`
   - Uses HF API once (respect rate limits) to list `knowledge-rag/hubs/moc/` (or configured hub)
   - Saves `src/data/hub-files.json`
   - Commits or copies into repo before build

4. **Mount panel in dashboard**
   - Add `<TopHubSignalPanel />` to dashboard layout (sidebar or top banner)
   - Ensure non-blocking (async load, no render-blocking)

5. **Build & verify**
   - Build locally; confirm CDN fetches work and no HF API calls happen at runtime
   - Verify panel appears and degrades gracefully

#### 2) File list generator (`scripts/generate-hub-file-list.sh`)
```bash
#!/usr/bin/env bash
set -euo pipefail

# Generate hub file list for build-time embedding.
# Run from Mac orchestration after rate-limit window is clear.
# Uses HF API once; output committed to src/data/hub-files.json

HUB_NAME="${HUB_NAME:-MOC}"
HF_REPO="knowledge-rag/hubs"
OUT_FILE="src/data/hub-files.json"

echo "Listing files for hub: ${HUB_NAME} in ${HF_REPO}"

# Requires HF_TOKEN in env if private; public repos don't need auth for listing.
# Use non-recursive listing per folder to avoid pagination explosion.
# We list the hub folder only.
curl -s -H "Authorization: Bearer ${HF_TOKEN:-}" \
  "https://huggingface.co/api/datasets/${HF_REPO}/tree/moc" \
  | jq '[.[] | select(.type=="file") | .path]' > "${OUT_FILE}"

echo "Saved file list to ${OUT_FILE}"
```

#### 3) Panel component (`src/components/TopHubSignalPanel.tsx`)
```tsx
import { useEffect, useState } from 'react';
import { HUB_NAME, HUB_DATASET } from '../config/hub';

interface Signal {
  title: string;
  description: string;
  severity: 'high' | 'medium' | 'low';
}

interface HubContent {
  title: string;
  description: string;
  signals: Signal[];
  documentsCount: number;
  updatedAt: string;
}

export function TopHubSignalPanel() {
  const [content, setContent] = useState<HubContent | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    let mounted = true;

    async function load() {
      try {
        // Load pre-generated file list (embedded at build time)
        const fileListResp = await fetch('/data/hub-files.json');
        const files: string[] = await fileListResp.json();

        // Pick latest file for this hub (simple: last by name or by date in name)
        const hubFile = files
          .filter((f) => f.startsWith(`${HUB_NAME.toLowerCase()}/`) || f.includes(HUB_NAME))
          .sort()
          .reverse()[0];

        if (!hubFile) {
          throw new Error('No hub file found');
        }

        // CDN fetch — no Authorization header needed, bypasses API rate limits
        const cdnUrl = `https://huggingface.co/datasets/${HUB_DATASET}/resolve/main/${hubFile}`;
        const data = await fetch(cdnUrl).then((r) => r.json());

        if (!mounted) return;
        setContent({
          title: data.title || HUB_NAME,
          description: data.description || '',
          signals: Array.isArray(data.signals) ? data.signals.slice(0, 3) : [],
          documentsCount: Number(data.documentsCount) || 0,
          updatedAt: data.updatedAt || '',
        });
      } catch (err) {
        console.warn('Failed to load hub signals (non-blocking)', err);
      } finally {
        if (mounted) setLoading(false);
      }
    }

    load();
    return () => {
      mounted = false;
    };
  }, []);

  if (loading) {
    return <div className="skeleton h-32 w-full">Loading hub signals…</div>;
  }

  if (!content) {
    return <div className="text-sm text-muted-foreground">Hub signals unavailable.</div>;
  }

  return (
    <div className="rounded border bg-card p-4 shadow-sm">
      <div className="mb-2 flex items-start justify-between">
        <div>
          <h3 className="text-base font-semibold">{content.title}</h3>
          <p className="text-sm text-muted-foreground">{content.description}</p>
        </div>
        <a
          href={`/knowledge-graph?hub=${encodeURIComponent(HUB_NAME)}`}
          className="text-xs text-primary hover:underline"
        >
          View in Knowledge Graph
        </a>
      </div>

      <div className="mb-2 text-xs text-muted-foreground">
        Related documents: {content.documentsCount}
      </div>

      {content.signals.length > 0 && (
        <ul className="space-y-2">
          {content.signals.map((s, i) => (
            <li key={i} className="rounded bg-muted/50 p-2 text-sm">
              <span className="font-medium">{s.title}</span>
              <p className="text-muted-foreground">{s.description}</p>
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}
```
