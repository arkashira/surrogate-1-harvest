# workio / ops

## 1. Diagnosis

- **No idempotency enforcement on LINE webhook ingestion** — identical `X-Line-Signature` + same event payload causes duplicate clock/leave/OT records when LINE retries.
- **Race-prone upsert in webhook handler** — `findOne` → conditional `update`/`insert` is non-atomic; concurrent replays or double taps corrupt punch state (double clock-in, missed clock-out).
- **No transactional boundary for punch + audit + notification** — partial writes leave inconsistent state if notification or audit insert fails after punch is saved.
- **Missing operational observability** — no structured request-id, trace-id, or idempotency-key logging; debugging duplicates across webhook retries is slow and unreliable.
- **Hard-coded retry/backoff absent for downstream calls** — LINE notification or DB write transient failures can leave user-facing state inconsistent with no safe retry.

## 2. Proposed change

- **File:** `workio/server/src/routes/webhook/line.ts` (or equivalent handler)
- **Scope:** Add idempotency table + atomic upsert for events, wrap punch+audit+notification in transaction, add request-id middleware and structured logs.
- **Minimal surface:** one new table (`webhook_idempotency`), one middleware, one atomic handler change — all <2h.

```sql
CREATE TABLE IF NOT EXISTS webhook_idempotency (
  idempotency_key VARCHAR(255) PRIMARY KEY,
  event_type      VARCHAR(100) NOT NULL,
  payload_hash    VARCHAR(64)  NOT NULL,
  processed_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- Optional: fast expiry for keys (retain 48h)
CREATE INDEX IF NOT EXISTS idx_webhook_idempotency_expiry
ON webhook_idempotency (processed_at)
WHERE processed_at < NOW() - INTERVAL '48 hours';
```

### 3.2 Idempotency middleware

```ts
// workio/server/src/middleware/idempotency.ts
import { Request, Response, NextFunction } from 'express';
import { db } from '../db';
import crypto from 'crypto';

export function idempotencyMiddleware(req: Request, res: Response, next: NextFunction) {
  // Use LINE signature + body hash as idempotency key
  const lineSig = req.get('X-Line-Signature') || '';
  const bodyHash = crypto.createHash('sha256').update(JSON.stringify(req.body)).digest('hex');
  const key = `line:${lineSig}:${bodyHash}`;

  (req as any).idempotencyKey = key;
  (req as any).payloadHash = bodyHash;
  next();
}
```

### 3.3 Atomic handler update (core)

```ts
// workio/server/src/routes/webhook/line.ts
import { Router } from 'express';
import { db } from '../../db';
import { idempotencyMiddleware } from '../../middleware/idempotency';

const router = Router();
router.use(idempotencyMiddleware);

router.post('/line', async (req, res) => {
  const { idempotencyKey, payloadHash } = req as any;
  const events = req.body.events || [];

  const client = await db.connect();
  try {
    await client.query('BEGIN');

    // Idempotency check (atomic)
    const dup = await client.query(
      'SELECT 1 FROM webhook_idempotency WHERE idempotency_key = $1',
      [idempotencyKey]
    );
    if (dup.rows.length > 0) {
      await client.query('ROLLBACK');
      return res.status(200).json({ ok: true, reason: 'duplicate' });
    }

    for (const ev of events) {
      if (ev.type === 'message' && ev.message.type === 'text') {
        const userId = ev.source.userId;
        const text = ev.message.text.trim().toLowerCase();

        // Example: clock in/out via text ("in"/"out")
        const now = new Date();
        const dateKey = now.toISOString().slice(0, 10);

        // Atomic upsert for latest punch per user per date
        await client.query(
          `INSERT INTO punches (user_id, date_key, clock_in, clock_out, updated_at)
           VALUES ($1, $2, $3, $4, $5)
           ON CONFLICT (user_id, date_key)
           DO UPDATE SET
             clock_in = CASE WHEN EXCLUDED.clock_in IS NOT NULL THEN EXCLUDED.clock_in ELSE punches.clock_in END,
             clock_out = CASE WHEN EXCLUDED.clock_out IS NOT NULL THEN EXCLUDED.clock_out ELSE punches.clock_out END,
             updated_at = NOW()`,
          [
            userId,
            dateKey,
            text === 'in' ? now : null,
            text === 'out' ? now : null
          ]
        );

        // Record idempotency (prevents replay)
        await client.query(
          `INSERT INTO webhook_idempotency (idempotency_key, event_type, payload_hash)
           VALUES ($1, $2, $3)`,
          [idempotencyKey, ev.type, payloadHash]
        );

        // Optional: send LINE reply (best-effort; failure does not rollback punch)
        try {
          await sendLineReply(ev.replyToken, `บันทึกเวลา ${text} เรียบร้อย`);
        } catch (e) {
          // log but don't rollback punch
          console.warn('LINE reply failed', e);
        }
      }
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook processing failed', err);
    res.status(500).json({ error: 'processing_failed' });
  } finally {
    client.release();
  }
});
```
