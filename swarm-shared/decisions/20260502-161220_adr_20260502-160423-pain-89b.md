## Deployment
cf workers

## v0.1 scope
v0.1 ships a local Rust CLI that: 1) creates a TUN interface and transparently routes selected traffic through a SOCKS5 proxy; 2) wraps proxy traffic in TLS+HTTP/2 cover on port 443 to mimic HTTPS; 3) enforces a fail-closed kill-switch that blocks non-stealth egress if the stealth path fails; 4) emits minimal, hashed client-side telemetry to a CF Worker queue; 5) stores session and config in local SQLite with optional sync to Supabase Postgres for multi-device restore.

## Open questions
- Which obfuscation primitives are safest and most effective against current DPI in target regions (e.g., meek vs. domain-front vs. QUIC)?
- How will we distribute and rotate cover endpoints without centralized servers while preserving free-tier constraints?
- What minimal client-side telemetry schema preserves privacy yet yields WAU-ZB and blocked-session signals?
- How to implement a reliable cross-platform kill-switch (TUN/socket-level) without elevated privileges on common desktop OSes?
- Which local artifacts (registry, firewall rules, /etc/resolv.conf, networkd configs) must be scrubbed to avoid post-session forensics?
