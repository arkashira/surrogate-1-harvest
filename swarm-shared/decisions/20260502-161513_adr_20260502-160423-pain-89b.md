## Deployment
cf workers

## v0.1 scope
v0.1 ships a minimal Rust CLI client (Linux/macOS) that creates a local SOCKS5 listener, wraps traffic through a TLS tunnel to a public egress (e.g., Cloudflare Worker endpoint or HF Space acting as a simple proxy), applies basic DPI obfuscation (packet length normalization, randomized delays, TLS profile mimicry), conceals TUN artifacts by defaulting to userspace TCP/TLS when TUN cannot be created, and enforces a fail-closed kill-switch that blocks all non-proxied traffic if stealth path fails; includes anonymized session telemetry to Supabase and config distribution via CF Workers KV.

## Open questions
- Which DPI signatures are most common in target regions (SNI, packet length/time, TTL, MTU, TUN device names)?
- Can we rely solely on client-side TUN without elevated privileges on common desktop OSes?
- What is the acceptable trade-off between stealth and throughput (padding vs latency)?
- How will we distribute updated obfuscation profiles without revealing endpoints (staged rollout via CF Workers + KV)?
- What telemetry can we collect without risking user deanonymization (hashing, differential privacy, opt-in)?
