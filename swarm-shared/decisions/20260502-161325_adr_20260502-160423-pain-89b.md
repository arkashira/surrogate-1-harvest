## v0.1 scope
v0.1 ships in 2 weeks: cross-platform CLI client (Linux/Windows/mac) with local SOCKS5 listener, TUN-mode stealth via meek-like HTTPS cover, configurable killswitch (block-all), local SQLite state, encrypted Supabase-backed config fetch + minimal encrypted telemetry via CF Workers Queue/KV, and zero-admin onboarding (single binary + systemd/launchd helpers).

## Open questions
- Which cover protocols (meek/MSS/DoH/mtls-lookalike) maximize throughput while minimizing DPI fingerprint on target networks?
- How will clients discover/rotate cover endpoints without centralized trackers that become blockable?
- What local port-binding strategy avoids conflicts with existing VPN/TUN tools and remains inconspicuous to port scans?
- How do we implement fail-closed killswitch reliably across OSes (Windows/Linux/mac/Android) without admin/root?
- What minimal telemetry schema preserves WAU-ZB measurement while preventing deanonymization or metadata leakage?
- How will clients verify integrity/authenticity of config updates in hostile network conditions (offline-first pinning)?
