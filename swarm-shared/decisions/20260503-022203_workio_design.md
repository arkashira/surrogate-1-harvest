# workio / design

Candidate 3:

**Diagnosis**
- Idempotency is missing at the webhook layer; LINE retries can create duplicate punches.
- The upsert for clock-in/out is not atomic and is vulnerable to race conditions under concurrent requests.
- Tenant isolation is not consistently enforced in queries, risking cross-tenant data access.
- There is no unique constraint or index to enforce one clock-in and one clock-out per employee per day.
- The current code does not store or check a LINE message ID for deduplication.

**Proposed change**
- Add a `line_message_id` column with a unique constraint for idempotency.
- Add a composite unique constraint on `(employee_id, date, event_type)` to enforce one clock-in and one clock-out per day.
- Replace the non-atomic upsert with a single atomic `INSERT ... ON CONFLICT` operation.
- Enforce tenant isolation by including `tenant_id` in all queries and indexes.
- Use database-level constraints and transactions to guarantee correctness.

**Implementation**

`server/src/db/schema.sql`
```sql
-- Add required columns if they do not exist
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS line_message_id UUID,
  ADD COLUMN IF NOT EXISTS tenant_id UUID NOT NULL,
  ADD COLUMN IF NOT EXISTS date DATE NOT NULL,
  ADD COLUMN IF NOT EXISTS event_type VARCHAR(10) NOT NULL CHECK (event_type IN ('clock_in', 'clock_out'));

-- Idempotency: prevent duplicate processing of the same LINE message
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_message_id
  ON punches (line_message_id)
  WHERE line_message_id IS NOT NULL;

-- Business rule: one clock_in and one clock_out per employee per day
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_employee_date_event
  ON punches (employee_id, date, event_type)
  WHERE event_type IN ('clock_in', 'clock_out');

-- Tenant isolation and query performance
CREATE INDEX IF NOT EXISTS idx_punches_tenant_employee_date
  ON punches (tenant_id, employee_id, date);
```

`server/src/services/punchService.ts`
```ts
import { Pool } from 'pg';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

export interface PunchInput {
  tenantId: string;
  employeeId: string;
  lineMessageId: string;
  eventType: 'clock_in' | 'clock_out';
  clockedAt?: Date;
}

export async function upsertPunch(input: PunchInput) {
  const { tenantId, employeeId, lineMessageId, eventType, clockedAt = new Date() } = input;
  const date = clockedAt.toISOString().split('T')[0];

  const query = `
    INSERT INTO punches (tenant_id, employee_id, line_message_id, date, event_type, clocked_at)
    VALUES ($1, $2, $3, $4, $5, $6)
    ON CONFLICT (line_message_id) DO UPDATE
      SET clocked_at = EXCLUDED.clocked_at
      RETURNING *;
  `;
  const values = [tenantId, employeeId, lineMessageId, date, eventType, clockedAt];

  const result = await pool.query(query, values);
  return result.rows[0];
}
```

`server/src/controllers/webhookController.ts`
```ts
import { upsertPunch } from '../services/punchService';

export async function handleLineWebhook(req, res) {
  try {
    const { tenantId, employeeId, messageId, eventType, timestamp } = req.body;

    if (!tenantId || !employeeId || !messageId || !eventType) {
      return res.status(400).json({ error: 'Missing required fields' });
    }

    const punch = await upsertPunch({
      tenantId,
      employeeId,
      lineMessageId: messageId,
      eventType,
      clockedAt: timestamp ? new Date(timestamp) : new Date(),
    });

    return res.status(200).json(punch);
  } catch (error) {
    // Unique violation on line_message_id is expected for retries; treat as success
    if (error.code === '23505') {
      return res.status(200).json({ message: 'Duplicate message ignored' });
    }
    console.error(error);
    return res.status(500).json({ error: 'Internal server error' });
  }
}
```

**Verification**
- Duplicate LINE messages (same `messageId`) do not create extra punches.
- Concurrent clock-in/out requests for the same employee/day/event cannot create logical duplicates.
- Queries always include `tenant_id`; cross-tenant access is prevented.
- Tests cover idempotency, atomic upsert, and tenant isolation.
