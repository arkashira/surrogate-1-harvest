## Summary of Value
- **Eliminates HF API rate-limit failures during training** by using CDN-only downloads.  
- **Avoids `pyarrow.CastError` and mixed-schema failures** by projecting only `{prompt, response}` per file and skipping unreadable files.  
- **Reduces Lightning Studio quota waste** by reusing running studios and restarting stopped ones instead of recreating.  
- **Fully deterministic and reproducible** via committed `file_list.json` manifests.  
- **Ships in <2h** with minimal new code and no heavy dependencies.

---

## Concrete Implementation Plan

1. **Manifest generator (run on Mac or dev box)**  
   - One-time per date folder.  
   - Uses `list_repo_tree(..., recursive=False)` to list files.  
   - Saves deterministic `file_list.json` with `{date, repo, files[]}`.  
   - Committed to repo so training jobs are reproducible and immune to repo drift.

2. **CDN-only iterable dataset loader**  
   - Accepts `file_list.json`.  
   - Downloads via `https://huggingface.co/datasets/{repo}/resolve/main/{path}` (no auth, no HF API).  
   - Projects each file to `{prompt, response}` at parse time.  
   - Skips unreadable/mixed-schema files safely instead of crashing.  
   - Uses `IterableDataset` for low memory and supports multi-worker sharding.

3. **Lightning Studio reuse + idle guard**  
   - Before training, check for a running studio by name; reuse if running.  
   - If stopped, restart on the same machine type (e.g., `Machine.L40S`) instead of recreating.  
   - Avoids quota waste from repeated studio creation and idle-timeout kills.

4. **CLI entrypoints**  
   - `python -m airship.ops.gen_manifest --date 2026-05-03 --repo surrogate-1/mirror --out manifests/2026-05-03/file_list.json`  
   - `python -m airship.surrogate.train --filelist manifests/2026-05-03/file_list.json --studio-name surrogate-1-l40s`

## Resolved Contradictions (Correctness + Actionability)
- **Manifest content**: Use minimal `{date, repo, files[]}` (Candidate 1) rather than adding byte sizes and CDN URLs (Candidate 2) — keeps generator simple and avoids storing derivable/redundant data. URLs are constructed deterministically at load time.
- **Projection robustness**: Keep Candidate 1’s fallback chain (parquet → json) and skip bad files instead of crashing. This is safer and more actionable for ops.
- **Studio reuse**: Use Candidate 1’s explicit `get_or_create_studio` with status check and restart on stopped studios. This is clearer and more actionable than Candidate 2’s implied approach.
- **Loader implementation**: Merge Candidate 1’s `_project_to_pair` logic with Candidate 2’s emphasis on CDN-only, but keep code structure from Candidate 1 for clarity and testability.

```python
# airship/ops/gen_manifest.py
import json
import argparse
from huggingface_hub import HfApi

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--date", required=True, help="Date folder, e.g. 2026-05-03")
    parser.add_argument("--repo", default="surrogate-1/mirror", help="HF dataset repo")
    parser.add_argument("--out", required=True, help="Output path for file_list.json")
    args = parser.parse_args()

    api = HfApi()
    tree = api.list_repo_tree(repo_id=args.repo, path=args.date, recursive=False)
    files = sorted([item.path for item in tree if item.type == "file"])

    payload = {"date": args.date, "repo": args.repo, "files": files}
    with open(args.out, "w") as f:
        json.dump(payload, f, indent=2)
    print(f"Wrote {len(files)} files to {args.out}")

if __name__ == "__main__":
    main()
```

### 2) CDN-only iterable loader

```python
# airship/surrogate/data.py
import io
import json
import torch
import requests
from torch.utils.data import IterableDataset
from datasets import load_dataset

HF_CDN = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

class CDNIterableSurrogateDataset(IterableDataset):
    def __init__(self, file_list_path, repo=None, max_files=None):
        with open(file_list_path) as f:
            manifest = json.load(f)
        self.repo = repo or manifest["repo"]
        self.files = manifest["files"]
        if max_files:
            self.files = self.files[:max_files]

    def _download_file(self, path):
        url = HF_CDN.format(repo=self.repo, path=path)
        r = requests.get(url, timeout=30)
        r.raise_for_status()
        return io.BytesIO(r.content)

    def _project_to_pair(self, file_buffer):
        try:
            ds = load_dataset("parquet", data_files={"train": file_buffer})["train"]
        except Exception:
            file_buffer.seek(0)
            try:
                ds = load_dataset("json", data_files={"train": file_buffer})["train"]
            except Exception:
                return  # skip unreadable/mixed-schema files

        for row in ds:
            prompt = row.get("prompt") or row.get("input") or row.get("question")
            response = row.get("response") or row.get("output") or row.get("answer")
            if prompt is not None and response is not None:
                yield {"prompt": str(prompt), "response": str(response)}

    def __iter__(self):
        worker_info = torch.utils.data.get_worker_info()
        if worker_info is None:
            iter_files = self.files
        else:
            per_worker = max(1, len(self.files) // worker_info.num_workers)
            start = worker_info.id * per_worker
            iter_files = self.files[start : start + per_worker]

        for path in iter_files:
            try:
                buf = self._download_file(path)
                yield from self._project_to_pair(buf)
            except Exception as exc:
                print(f"Skipping {path}: {exc}")
                continue
```

```python
# airship/surrogate/lightning_utils.py
from lightning import LightningWork, Machine, Teamspace

def get_or_create_studio(name: str, machine: Machine = Machine.L40S) -> LightningWork:
    teamspace = Teamspace()
    for s in teamspace.studios:
        if s.name == name:
            if s.status == "running":
                return s
            # stopped/idle: restart on same machine
            s.start(machine=machine)
            return s
    # create new if none exists
    return LightningWork(
        cloud_compute=machine,
        name=name,
        create_ok=True,
    )
```
