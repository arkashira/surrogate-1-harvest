# workio / backend

## 1. Diagnosis

- **Non-idempotent LINE webhook handler**: duplicate deliveries (LINE retries on timeout/5xx) can create multiple punch records for same employee/date because upsert is not atomic and lacks stable dedupe key.
- **Missing tenant-scoped uniqueness**: no DB unique constraint for `(tenant_id, employee_id, punch_date)` or `(tenant_id, request_id)` so race conditions under concurrency produce duplicates.
- **No request-idempotency key**: LINE webhook payloads include `deliveryId`/`timestamp` but the handler does not persist or check a dedupe key, so retries are treated as new punches.
- **Clock-in/out state logic is DB-read-then-write**: gap between SELECT and INSERT/UPDATE allows double-punch when two webhook deliveries interleave.
- **No transactional boundary for punch + audit**: partial writes can leave punch without audit trail, making reconciliation hard.

## 2. Proposed change

File scope:
- `server/src/controllers/lineWebhook.ts` — handler for punch events (clock in/out).
- `server/src/db/schema.sql` — add idempotency table + tenant-scoped uniqueness.
- `server/src/services/punchService.ts` — implement atomic upsert with idempotency key.

## 3. Implementation

### 3.1 Schema: idempotency + uniqueness

```sql
-- server/src/db/schema.sql

-- Idempotency keys for LINE webhook deliveries
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  tenant_id      BIGINT NOT NULL,
  delivery_id    TEXT   NOT NULL,          -- LINE deliveryId or stable id from webhook
  employee_id    BIGINT NOT NULL,
  punch_id       BIGINT NOT NULL,
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (tenant_id, delivery_id)
);

-- Ensure one punch per employee per date (tenant-scoped)
-- Keep existing punches table name if different; adjust accordingly.
CREATE UNIQUE INDEX IF NOT EXISTS uq_tenant_employee_punch_date
ON punches (tenant_id, employee_id, DATE(punched_at));
```

### 3.2 Service: atomic upsert with idempotency

```ts
// server/src/services/punchService.ts
import { pool } from '../db';

export interface ProcessPunchParams {
  tenantId: number;
  employeeId: number;
  deliveryId: string;   // from LINE webhook
  punchedAt: Date;      // event timestamp
  type: 'in' | 'out';
  latitude?: number;
  longitude?: number;
}

export async function processPunch(params: ProcessPunchParams) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // 1) Idempotency check
    const idem = await client.query(
      `SELECT punch_id FROM line_webhook_idempotency WHERE tenant_id = $1 AND delivery_id = $2`,
      [params.tenantId, params.deliveryId]
    );

    if (idem.rows.length > 0) {
      // Already processed — fetch and return existing punch
      const punch = await client.query(
        `SELECT * FROM punches WHERE id = $1`,
        [idem.rows[0].punch_id]
      );
      await client.query('COMMIT');
      return punch.rows[0];
    }

    // 2) Get latest punch for employee to determine next action (optional strictness)
    // 3) Insert punch (conflict on uq_tenant_employee_punch_date will raise)
    const insertPunch = await client.query(
      `INSERT INTO punches (tenant_id, employee_id, punched_at, type, latitude, longitude)
       VALUES ($1, $2, $3, $4, $5, $6)
       RETURNING *`,
      [params.tenantId, params.employeeId, params.punchedAt, params.type, params.latitude, params.longitude]
    );

    const punch = insertPunch.rows[0];

    // 3) Record idempotency
    await client.query(
      `INSERT INTO line_webhook_idempotency (tenant_id, delivery_id, employee_id, punch_id)
       VALUES ($1, $2, $3, $4)`,
      [params.tenantId, params.deliveryId, params.employeeId, punch.id]
    );

    await client.query('COMMIT');
    return punch;
  } catch (err: any) {
    await client.query('ROLLBACK');
    // If unique violation on uq_tenant_employee_punch_date, treat as duplicate and fetch existing
    if (err.code === '23505') {
      const exist = await client.query(
        `SELECT * FROM punches WHERE tenant_id = $1 AND employee_id = $2 AND DATE(punched_at) = DATE($3)`,
        [params.tenantId, params.employeeId, params.punchedAt]
      );
      if (exist.rows.length > 0) return exist.rows[0];
    }
    throw err;
  } finally {
    client.release();
  }
}
```

### 3.3 Controller: consume LINE webhook and call service

```ts
// server/src/controllers/lineWebhook.ts
import { Router } from 'express';
import { processPunch } from '../services/punchService';

const router = Router();

router.post('/line', async (req, res) => {
  try {
    const events = req.body.events || [];
    const results = [];

    for (const ev of events) {
      // Expecting custom payload or message with source.userId + metadata
      // Example: ev.source.userId maps to employee; use mapping table.
      // For now, assume ev.webhookDeliveryId and ev.punch payload exist.
      const deliveryId = ev.deliveryId || ev.message?.id || ev.timestamp?.toString();
      if (!deliveryId) continue;

      // Resolve tenantId & employeeId from mapping (pseudo)
      const tenantId = Number(ev.source?.tenantId || 1); // replace with real lookup
      const employee = await resolveEmployeeFromLineUid(ev.source.userId, tenantId);
      if (!employee) continue;

      const punch = await processPunch({
        tenantId,
        employeeId: employee.id,
        deliveryId,
        punchedAt: ev.timestamp ? new Date(ev.timestamp) : new Date(),
        type: ev.type || 'in',
        latitude: ev.latitude,
        longitude: ev.longitude,
      });

      results.push(punch);
    }

    res.json({ ok: true, results });
  } catch (err) {
    console.error('LINE webhook error', err);
    // LINE will retry on 5xx — return 200 only when processed or explicitly deduped
    res.status(500).json({ error: 'processing_failed' });
  }
});

export default router;
```

### 3.4 Ensure idempotency key is stable

- Use `deliveryId` from LINE (if provided) or a deterministic hash of `(tenantId, userId, timestamp, eventType)` when absent.
- Persist mapping `line_user_id -> employee_id` per tenant to resolve quickly.
