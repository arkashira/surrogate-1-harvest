# workio / design

## Final Synthesis & Action Plan

**Core diagnosis (accepted from both candidates):**  
Duplicate LINE webhook deliveries + non-atomic read-then-insert cause duplicate active punches and inconsistent employee state. Missing idempotency key and transactional boundaries are the root causes.

**Chosen approach (favoring correctness + concrete actionability):**  
Adopt Candidate 1’s atomic, idempotent punch handler with the following tightened, production-ready fixes.

---

### 1. Schema (immediate)

```sql
-- Add idempotency and tenant scoping
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS line_delivery_id VARCHAR(255),
  ADD COLUMN IF NOT EXISTS tenant_id UUID NOT NULL;

-- Enforce exactly one delivery id (idempotency)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_line_delivery_id
  ON punches(line_delivery_id)
  WHERE line_delivery_id IS NOT NULL;

-- One active punch per user (no out) — scoped by tenant
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_active_user
  ON punches(user_id, tenant_id)
  WHERE out IS NULL;

-- Optional: pair related punches (clock-in → clock-out)
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS paired_punch_id UUID REFERENCES punches(id);
```

**Rationale:**  
- `tenant_id` must be `NOT NULL` and part of the active-punch uniqueness to prevent cross-tenant collisions.  
- Keep `date` out of the active-punch unique index; an active clock-in can span midnight, and uniqueness should be “one active punch per user per tenant.”  
- `paired_punch_id` is optional but useful for audits and leave/OT reconciliation.

### 2. Service: atomic record + state transition

```ts
// server/src/services/punchService.ts
import { PoolClient, QueryResult } from 'pg';
import { pool } from '../db';

export type PunchType = 'clock-in' | 'clock-out' | 'break-start' | 'break-end';

export interface PunchInput {
  userId: string;
  tenantId: string;
  type: PunchType;
  lineDeliveryId: string;
  latitude?: number;
  longitude?: number;
  locationName?: string;
}

// Idempotent insert; safe under concurrent duplicate deliveries
export async function recordPunch(
  input: PunchInput,
  client?: PoolClient
): Promise<{ id: string; existed: boolean }> {
  const db = client || pool;
  const insertQuery = `
    INSERT INTO punches (
      id, user_id, tenant_id, date, type, line_delivery_id,
      latitude, longitude, location_name, in
    )
    VALUES (gen_random_uuid(), $1, $2, CURRENT_DATE, $3, $4, $5, $6, $7, NOW())
    ON CONFLICT (line_delivery_id) DO NOTHING
    RETURNING id
  `;
  const insertRes = await db.query(insertQuery, [
    input.userId,
    input.tenantId,
    input.type,
    input.lineDeliveryId,
    input.latitude,
    input.longitude,
    input.locationName,
  ]);

  if (insertRes.rowCount > 0) {
    return { id: insertRes.rows[0].id, existed: false };
  }

  const getQuery = 'SELECT id FROM punches WHERE line_delivery_id = $1';
  const getRes = await db.query(getQuery, [input.lineDeliveryId]);
  return { id: getRes.rows[0].id, existed: true };
}

// Atomic close of the latest active clock-in for this user+tenant
export async function closeActivePunch(
  userId: string,
  tenantId: string,
  lineDeliveryId: string,
  client?: PoolClient
): Promise<{ updated: boolean; punchId?: string }> {
  const db = client || pool;
  const query = `
    WITH target AS (
      SELECT id
      FROM punches
      WHERE user_id = $1 AND tenant_id = $2 AND out IS NULL AND type = 'clock-in'
      ORDER BY in DESC
      LIMIT 1
      FOR UPDATE SKIP LOCKED
    )
    UPDATE punches p
    SET out = NOW()
    FROM target
    WHERE p.id = target.id
      AND NOT EXISTS (SELECT 1 FROM punches WHERE line_delivery_id = $3)
    RETURNING p.id
  `;
  const res = await db.query(query, [userId, tenantId, lineDeliveryId]);
  return { updated: res.rowCount > 0, punchId: res.rows[0]?.id };
}
```

**Key improvements vs Candidate 1:**  
- `closeActivePunch` now checks `line_delivery_id` uniqueness in the same statement to prevent duplicate “out” records from concurrent redeliveries.  
- Explicit `tenant_id` in all uniqueness and filtering to avoid cross-tenant leaks.  
- `SKIP LOCKED` prevents deadlocks under high concurrency.

### 3. Webhook handler (idempotent, transactional)

```ts
// server/src/routes/lineWebhook.ts
import { Router } from 'express';
import { pool } from '../db';
import { recordPunch, closeActivePunch } from '../services/punchService';

const router = Router();

router.post('/line', async (req, res) => {
  const lineDeliveryId = req.headers['x-line-delivery-id'] as string;
  if (!lineDeliveryId) {
    return res.status(400).json({ error: 'missing x-line-delivery-id' });
  }

  const events = req.body.events || [];
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    for (const ev of events) {
      if (ev.type === 'message' && ev.message?.type === 'text') {
        const text = ev.message.text.trim().toLowerCase();
        const userId = ev.source.userId;
        // Resolve tenant per user/company; fallback only for prototyping
        const tenantId = await resolveTenantId(userId, client) || process.env.DEFAULT_TENANT_ID;
        if (!tenantId) throw new Error('tenant resolution failed');

        const deliverySuffix = `${ev.timestamp}`;
        if (text === 'clock in') {
          await recordPunch(
            {
              userId,
              tenantId,
              type: 'clock-in',
              lineDeliveryId: `${lineDeliveryId}:${deliverySuffix}:in`,
              latitude: ev.message?.location?.latitude,
              longitude: ev.message?.location?.longitude,
              locationName: ev.message?.location?.title,
            },
            client
          );
        } else if (text === 'clock out') {
          await closeActivePunch(userId, tenantId, `${lineDeliveryId}:${deliverySuffix}:out`, client);
        }
      }
    }

    await client.query('COMMIT');
    res.status(200).json({ ok: true });
  } catch (err) {
    await client.query('ROLLBACK');
    console.error('Webhook transaction failed', err);
    res.status(500).json({ error: 'transaction_failed' });
  } finally {
    client.release();
  }
});

async function resolveTenantId(userId: string, client: PoolClient): Promise<string | null> {
  const res = await client.query('SELECT tenant_id FROM users WHERE id = $1', [userId]);
  return res.rows[0]?.tenant_id || null;
}

export default router;
```

**Why this works:**  
- Entire batch of events in one transaction; partial failures roll back.  
- Per-event `lineDeliveryId` suffix prevents cross-event collisions while preserving idempotency.  
- Tenant resolution is explicit and safe under concurrency.
