# surrogate-1 / design

## Implementation Plan (≤2h)

**Highest-value improvement**: Add deterministic pre-flight file listing + CDN-only ingestion to eliminate HF API 429s during training and make shard workers resilient.

### What we’ll change
1. Add `bin/list-files.sh` — one-shot Mac/Linux script that calls `list_repo_tree` for a date folder, saves `file-list.json`, and embeds it in the runner so workers never call `list_repo_files` recursively during the cron window.
2. Update `bin/dataset-enrich.sh` to:
   - Accept an optional `FILE_LIST` path (default: fallback to current behavior).
   - When provided, stream only listed files via `hf_hub_download` + CDN URLs (no Authorization header) to bypass `/api/` rate limits.
   - Keep per-schema projection to `{prompt, response}` and dedup via central md5 store.
3. Update `.github/workflows/ingest.yml` to:
   - Add an initial "list" job (or step) that produces `file-list.json` as an artifact.
   - Pass the list to the 16-shard matrix so every worker uses the same deterministic snapshot.
4. Add lightweight retry/backoff for 429 with 360s sleep as per pattern.

### Why this is highest value
- Eliminates the HF API 429 wall during ingestion windows.
- Keeps workers independent, deterministic, and CDN-only (higher rate limits).
- Fits in <2h: small scripts + workflow tweak, no infra changes.

### 1) `bin/list-files.sh`
```bash
#!/usr/bin/env bash
# list-files.sh
# Usage: HF_TOKEN=... ./list-files.sh <date-folder> [output.json]
# Example: ./list-files.sh 2026-05-02 file-list.json
set -euo pipefail

REPO="datasets/axentx/surrogate-1-training-pairs"
DATE_FOLDER="${1:-$(date +%Y-%m-%d)}"
OUTPUT="${2:-file-list.json}"
HF_TOKEN="${HF_TOKEN:-}"

if [ -z "$HF_TOKEN" ]; then
  echo "ERROR: HF_TOKEN is required" >&2
  exit 1
fi

# list_repo_tree non-recursive for the date folder
# We'll use the REST API directly (paginated)
BASE="https://huggingface.co/api"
URL="${BASE}/datasets/${REPO}/tree/${DATE_FOLDER}"

echo "Fetching tree for ${DATE_FOLDER} ..." >&2

items=()
next_url="${URL}"
while [ -n "${next_url}" ]; do
  resp=$(curl -sS -H "Authorization: Bearer ${HF_TOKEN}" "${next_url}")
  # Extract items array
  batch=$(echo "$resp" | python3 -c "import sys,json; data=json.load(sys.stdin); print(json.dumps(data))")
  # If it's an array, append
  items+=($(echo "$batch" | python3 -c "import sys,json; data=json.load(sys.stdin); print(' '.join(json.dumps(i) for i in data))"))
  # Next link (if any)
  next_url=$(echo "$batch" | python3 -c "import sys,json; data=json.load(sys.stdin); links=[l.get('href','') for l in data.get('_links',{}).get('next',[]) if l.get('rel','')=='next']; print(links[0] if links else '')")
done

# Build flat list of file paths under date folder
paths=$(echo "${items[@]}" | python3 -c "
import sys, json, os
items = []
for tok in sys.stdin.read().strip().split(' '):
    if tok:
        items.append(json.loads(tok))
out = []
for it in items:
    if it.get('type') == 'file':
        out.append(it['path'])
    elif it.get('type') == 'directory':
        # We don't recurse; keep flat per pattern. If recursion needed, caller should run per subfolder.
        pass
print(json.dumps(sorted(out)))
")

echo "$paths" > "$OUTPUT"
echo "Wrote ${paths} file paths to ${OUTPUT}" >&2
```

Make executable:
```bash
chmod +x bin/list-files.sh
```
