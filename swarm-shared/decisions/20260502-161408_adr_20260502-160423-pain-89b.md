## Deployment
cf workers

## v0.1 scope
v0.1 ships a single static client binary (Rust) that establishes a meek-style HTTPS tunnel via Cloudflare Workers fronting, exposes a local SOCKS5 listener with innocuous process/tun naming, and enforces a fail-closed kill-switch that blocks all non-tunnel traffic if the stealth path fails; includes client-side SQLite telemetry for WAU-ZB and basic endpoint rotation, all deployable with zero server management.

## Open questions
- Which DPI-evasion transports (meek, shadowsocks, vless over utls, or domain-fronted CF Workers) provide the best reach vs. performance trade-off in target regions?
- How will client-side telemetry (WAU-ZB) be collected without deanonymizing users or creating a metadata honeypot?
- What is the kill-switch granularity (per-app via TUN, system-wide, or split-tunnel) and how to enforce it reliably across desktop OSes?
- How to distribute updated obfuscation fingerprints and endpoint lists to clients without centralized infrastructure that can be blocked?
- What local artifacts (TUN device names, listening sockets, process names) must be masked to avoid on-device detection?
