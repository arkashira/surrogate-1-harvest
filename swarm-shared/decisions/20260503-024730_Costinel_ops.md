# Costinel / ops

## Final Implementation Plan — Top-Hub Signal Panel (CDN-first, <2h, telemetry-aware)

### What we ship (highest-value, production-ready)
- A **non-blocking Top-Hub Signal Panel** on the Costinel dashboard.
- Defaults to hub **MOC** (configurable via `HUB_NAME` / `VITE_HUB_NAME`).
- Shows **exactly 3 actionable signals** (title, snippet/summary, severity, link).
- **CDN-first data strategy**:
  - Build-time: generate `hub-moc-manifest.json` via ops script/cron using `list_repo_tree` → CDN URLs.
  - Runtime: fetch manifest from public CDN (no Authorization, bypass HF API limits).
- **Telemetry-aware**:
  - Respect `DO_NOT_TRACK`.
  - Log minimal non-PII timing + counts (`signal_impression`, `signal_click`) only when allowed.
  - Never block UI on telemetry or fetch failures.
- **Graceful degradation**:
  - API fails open → returns `{ signals: [] }`.
  - UI shows subtle empty state ("Insights unavailable") without errors or console noise.
- **Caching & resilience**:
  - CDN + `s-maxage=300, stale-while-revalidate=60`.
  - 3s timeout on manifest fetch; fallback to yesterday’s manifest if today missing (optional ops note).

1. **Config** (3–5m)
   - Add to `.env.local.example`:
     ```bash
     HUB_NAME=MOC
     # Optional: override manifest CDN URL
     # HUB_MANIFEST_URL=https://huggingface.co/datasets/axentx/costinel-hub/resolve/main/manifests/moc-2026-05-03.json
     DO_NOT_TRACK=false
     # For Vite frontends: VITE_HUB_NAME=MOC
     ```

2. **CDN manifest helper (ops script)** (15–20m)
   - One script to produce `hub-moc-manifest.json`:
     - Use `list_repo_tree` (or repo dataset file index) to list today’s hub files.
     - Output schema:
       ```json
       {
         "hub": "MOC",
         "date": "2026-05-03",
         "files": [
           {
             "slug": "moc-2026-05-03-alert-xyz",
             "cdnUrl": "https://huggingface.co/datasets/axentx/costinel-hub/resolve/main/...",
             "severity": "high",
             "title": "Cost spike in region X",
             "snippet": "Detected 40% increase vs baseline..."
           }
         ]
       }
       ```
   - Cron note: ensure `#!/usr/bin/env bash`, `chmod +x`, and `SHELL=/bin/bash` in crontab.

3. **API route** (`/api/hub-signals`) (20–30m)
   - Accepts `?hub=MOC&limit=3`.
   - Fetches manifest from `HUB_MANIFEST_URL` or built-in CDN path.
   - 3s timeout, fails open → `{ signals: [] }`.
   - Cache headers: `s-maxage=300, stale-while-revalidate=60`.
   - No Authorization header (public CDN).

4. **React component** (`TopHubSignalPanel`) (20–30m)
   - Client fetch to `/api/hub-signals`.
   - Skeleton loader while fetching.
   - Renders up to 3 compact cards (title, snippet, severity badge, link).
   - Empty state: subtle "Insights unavailable" (no errors thrown).
   - Severity colors: critical (red), high (orange), medium (yellow), low (blue).
   - Telemetry (non-PII):
     - On load: `performance.mark` timing; emit `signal_impression` (count) if `!doNotTrack`.
     - On click: emit `signal_click` if `!doNotTrack`.

5. **Wire into dashboard** (10–15m)
   - Insert `<TopHubSignalPanel />` near cost summary or top of sidebar.
   - SSR-safe: client-only or server-side preload with graceful empty fallback.

6. **Ops + CI notes** (5–10m)
   - Add CI step to fetch latest manifest at build time (optional) or rely on runtime CDN.
   - Document cron script location and expected permissions.

#### 2. API route (Next.js) — `/pages/api/hub-signals.ts`
```ts
import type { NextApiRequest, NextApiResponse } from 'next';

type Signal = {
  title: string;
  snippet: string;
  severity: 'low' | 'medium' | 'high' | 'critical';
  link: string;
};

type Manifest = {
  hub: string;
  date: string;
  files: Array<{
    slug: string;
    cdnUrl: string;
    severity: Signal['severity'];
    title: string;
    snippet?: string;
  }>;
};

const DEFAULT_HUB = process.env.HUB_NAME || 'MOC';
const MANIFEST_DATE = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
const CDN_BASE = 'https://huggingface.co/datasets/axentx/costinel-hub/resolve/main';

export default async function handler(
  req: NextApiRequest,
  res: NextApiResponse<{ signals: Signal[] }>
) {
  const hub = (req.query.hub as string) || DEFAULT_HUB;
  const limit = Math.min(parseInt(req.query.limit as string, 10) || 3, 10);

  const manifestUrl =
    process.env.HUB_MANIFEST_URL ||
    `${CDN_BASE}/manifests/${hub.toLowerCase()}-${MANIFEST_DATE}.json`;

  try {
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), 3000);

    const resp = await fetch(manifestUrl, {
      signal: controller.signal,
      headers: { Accept: 'application/json' },
    });
    clearTimeout(timeout);

    if (!resp.ok) throw new Error(`Manifest fetch failed: ${resp.status}`);

    const manifest: Manifest = await resp.json();
    const signals: Signal[] = (manifest.files || [])
      .slice(0, limit)
      .map((f) => ({
        title: f.title || f.slug,
        snippet: f.snippet || '',
        severity: f.severity || 'medium',
        link: f.cdnUrl,
      }));

    res.setHeader('Cache-Control', 's-maxage=300, stale-while-revalidate=60');
    return res.status(200).json({ signals });
  } catch (err) {
    // Fail open — never block UI
    console.warn('Hub signals unavailable:', (err as Error).message);
    res.setHeader('Cache-Control', 's-maxage=60, stale-while-revalidate=30');
    return res.status(200).json({ signals: [] });
  }
}
```
