# workio / frontend

## 1. Diagnosis

- Clock-in/out UI lacks optimistic update and local rollback on network failure → perceived latency and duplicate taps when users retry.
- No client-side idempotency guard against LINE-retry/webhook race; same punch can be applied twice if webhook arrives after optimistic state.
- Punch state is derived from polling or refetch causing unnecessary re-renders and stale reads between sync windows.
- Missing local-first “last punch” cache means offline or flaky connections lose intent and cannot reconcile gracefully.
- No abort/retry/backoff layer for punch mutations leads to unbounded retries and race conditions under poor connectivity.

## 2. Proposed change

File: `workio/src/features/clock/ClockButton.tsx` (create or update)  
Scope: add optimistic update, client-side idempotency key, SWR cache mutation, and exponential backoff with abort for punch mutations.

## 3. Implementation

```tsx
// workio/src/features/clock/ClockButton.tsx
import { useState, useCallback, useRef } from 'react';
import useSWR, { useSWRConfig } from 'swr';

type PunchState = 'IN' | 'OUT' | null;

const CLOCK_ENDPOINT = '/api/clock/punch';

// lightweight fetcher with timeout
const fetcher = (url: string, options: RequestInit & { timeout?: number } = {}) => {
  const { timeout = 8000, ...init } = options;
  const controller = new AbortController();
  const id = setTimeout(() => controller.abort(), timeout);
  return fetch(url, { ...init, signal: controller.signal })
    .then((r) => {
      clearTimeout(id);
      if (!r.ok) throw new Error(`HTTP ${r.status}`);
      return r.json();
    })
    .catch((err) => {
      clearTimeout(id);
      throw err;
    });
};

// client-side idempotency key generator (per-session intent)
function makeIdempotencyKey(userId: string, date: string) {
  return `punch:${userId}:${date}`;
}

export default function ClockButton() {
  const { mutate } = useSWRConfig();
  const userId = 'current-user'; // replace with auth context
  const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
  const idempotencyKey = makeIdempotencyKey(userId, today);

  // local optimistic state
  const [optimisticPunch, setOptimisticPunch] = useState<PunchState>(null);
  const [inflight, setInflight] = useState(false);
  const abortRef = useRef<AbortController | null>(null);

  // derive current state from cache (server truth)
  const { data: serverState } = useSWR<PunchState>(
    `/api/clock/state?userId=${userId}&date=${today}`,
    fetcher,
    { refreshInterval: 30000, revalidateOnFocus: false }
  );

  const currentState = optimisticPunch ?? serverState ?? null;

  const punch = useCallback(
    async (intendedState: PunchState) => {
      if (inflight) return;
      setInflight(true);
      const controller = new AbortController();
      abortRef.current = controller;

      // optimistic update
      setOptimisticPunch(intendedState);

      // exponential backoff config
      const maxAttempts = 4;
      const baseDelay = 600;
      let attempt = 0;

      const doRequest = async (): Promise<void> => {
        attempt++;
        try {
          const result = await fetcher(CLOCK_ENDPOINT, {
            method: 'POST',
            headers: {
              'Content-Type': 'application/json',
              'Idempotency-Key': idempotencyKey,
            },
            body: JSON.stringify({
              userId,
              date: today,
              state: intendedState,
            }),
            signal: controller.signal,
          });

          // commit: update cache and clear optimistic state
          mutate(`/api/clock/state?userId=${userId}&date=${today}`, intendedState, false);
          setOptimisticPunch(null);
          setInflight(false);
          return result;
        } catch (err: any) {
          if (controller.signal.aborted) {
            // rollback optimistic state on abort
            setOptimisticPunch(null);
            setInflight(false);
            return;
          }

          if (attempt >= maxAttempts) {
            // final failure: rollback and surface (toast/logging)
            setOptimisticPunch(null);
            setInflight(false);
            console.error('Punch failed after retries', err);
            return;
          }

          // wait and retry
          const delay = baseDelay * 2 ** (attempt - 1) + Math.random() * 200;
          await new Promise((r) => setTimeout(r, delay));
          return doRequest();
        }
      };

      return doRequest();
    },
    [inflight, userId, today, idempotencyKey, mutate]
  );

  const toggle = useCallback(() => {
    const next = currentState === 'IN' ? 'OUT' : 'IN';
    punch(next);
  }, [currentState, punch]);

  const label = currentState === 'IN' ? 'Clock Out' : 'Clock In';
  const intentLabel = optimisticPunch ? `(${optimisticPunch})` : '';

  return (
    <button
      onClick={toggle}
      disabled={inflight}
      className={`px-6 py-3 rounded-lg font-semibold text-white transition-all ${
        inflight
          ? 'bg-gray-400 cursor-wait'
          : currentState === 'IN'
          ? 'bg-red-600 hover:bg-red-700'
          : 'bg-green-600 hover:bg-green-700'
      }`}
    >
      {inflight ? `Processing ${intentLabel}` : label}
    </button>
  );
}
```

Server-side note (non-frontend but required): ensure `/api/clock/punch` accepts and enforces `Idempotency-Key` header with a unique constraint or cache (e.g., Redis) keyed by `Idempotency-Key` for at least 24h to deduplicate LINE retries/webhook replays.

## 4. Verification

- Manual: tap the button rapidly and while offline (throttle to Slow 3G). Observe optimistic state change immediately and rollback on final failure.
- Network: in DevTools, simulate a 500 response for the punch endpoint; confirm retry/backoff occurs (3–4 attempts) and optimistic state rolls back after final failure.
- Idempotency: replay the same request (curl with same `Idempotency-Key`) and confirm server returns same punch record without duplication.
- SWR cache: after success, confirm `/api/clock/state` cache is updated and other components reading it reflect the new state without full refetch.
