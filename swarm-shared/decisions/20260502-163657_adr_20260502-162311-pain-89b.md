## Deployment
hybrid (CF Workers for rule cache + lightweight egress; HF Spaces for model-based traffic morphing experiments; Android on-device Rust core; Supabase for state)

## v0.1 scope
v0.1 in 2 weeks: Android app with Rust core (NDK) exposing a single 'Stealth Session' that wraps shadowsocks over meek (domain-fronted CDN) via VPNService; hides TUN interface from other apps using fd-namespace tricks and masks local SOCKS5 port; rule bundle compiled-in (no remote config); local SQLite stores session logs; Supabase stores minimal user/session telemetry for WAUS/U; basic traffic morphing (packet split, size padding, jitter); no cover-traffic by default; manual egress region selection; QA pass for DPI lab tests against common app-level VPN detection on stock Android 12+.

## Open questions
- Which TCP/UDP obfuscation primitives remain effective against current DPI in target jurisdictions (SNI/ALPN randomization, packet split vs. record padding)?
- Can we reliably mask TUN interfaces from app-level VPN detection on stock Android without root or Knox (VPNService-based packet routing vs. tun_fd visibility)?
- What is the acceptable tradeoff between cover-traffic overhead and detectability (latency, bandwidth, battery)?
- How will revocation and abuse mitigation work without strong identity (rate-limits, proof-of-work, invite-only tokens)?
- Which CF Workers + Supabase auth flow minimizes PII while supporting per-device config distribution and deniability?
