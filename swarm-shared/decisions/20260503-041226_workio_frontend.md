# workio / frontend

## Final Synthesis (Correct + Actionable)

I merged the strongest, non-contradictory parts from both proposals and resolved conflicts in favor of correctness and concrete actionability.

Key decisions:
- **Idempotency**: use `Idempotency-Key` header + client-side key generation (per punch attempt) and safe retry (retry 5xx + 409, don’t retry other 4xx).
- **Optimistic UI + persisted pending state**: prevents double-taps, survives reloads, and enables explicit retry.
- **Location guard + UX**: mandatory location capture before submit, with clear permission/failure UI and GPS-spinner/accuracy indicator.
- **Server truth for status**: treat `lastPunch`/`nextAllowed` as server-authoritative; poll or refresh after success to avoid stale transitions (e.g., prevent double clock-in).
- **Separation of concerns**: `useClock` orchestrates state/persistence/retry/location; `ClockButton` renders action + pending affordance; `ClockStatus` shows last/next and errors; `ClockFeedback` handles location flow + retry.

### `workio/src/lib/api.ts`
```ts
// Add idempotency and safe retry (network + server errors)
export async function postWithRetry<T = any>(
  url: string,
  body: any,
  { idempotencyKey, retries = 2, signal }: { idempotencyKey?: string; retries?: number; signal?: AbortSignal } = {}
): Promise<T> {
  const headers: Record<string, string> = { 'Content-Type': 'application/json' };
  if (idempotencyKey) headers['Idempotency-Key'] = idempotencyKey;

  let lastError: any;
  for (let i = 0; i <= retries; i++) {
    try {
      const res = await fetch(`${import.meta.env.VITE_API_BASE_URL}${url}`, {
        method: 'POST',
        headers,
        body: JSON.stringify(body),
        signal,
      });

      // 409 (conflict) is safe to retry (e.g., duplicate detected but not finalized)
      if (res.status >= 400 && res.status < 500 && res.status !== 409) {
        throw new Error(`Client error ${res.status}`);
      }
      if (!res.ok) {
        // 5xx or 409 -> retry
        throw new Error(`Server error ${res.status}`);
      }

      return (await res.json()) as T;
    } catch (err: any) {
      if (signal?.aborted) throw err;
      lastError = err;
      if (i < retries) await new Promise((r) => setTimeout(r, 500 * 2 ** i));
    }
  }
  throw lastError;
}

// Typed clock endpoint
export async function clockPunch(
  payload: { tenantId: string; userId: string; type: 'IN' | 'OUT'; location: { lat: number; lng: number } },
  { idempotencyKey, signal }: { idempotencyKey?: string; signal?: AbortSignal } = {}
) {
  return postWithRetry<{ ok: boolean; punch?: { type: 'IN' | 'OUT'; ts: string } }>('/clock/punch', payload, {
    idempotencyKey,
    signal,
  });
}

// Fetch current status (authoritative)
export async function fetchClockStatus({ tenantId, userId }: { tenantId: string; userId: string }) {
  const res = await fetch(
    `${import.meta.env.VITE_API_BASE_URL}/clock/status?tenantId=${encodeURIComponent(tenantId)}&userId=${encodeURIComponent(userId)}`
  );
  if (!res.ok) throw new Error(`Status fetch failed ${res.status}`);
  return res.json() as Promise<{ lastPunch?: 'IN' | 'OUT'; ts?: string; nextAllowed?: 'IN' | 'OUT' }>;
}
```
