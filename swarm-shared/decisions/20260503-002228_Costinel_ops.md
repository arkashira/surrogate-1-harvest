**Final consolidated implementation plan (best of both proposals, corrected + actionable)**

---

## Scope & Value (decided)
- **What**: Read-only “Top-Hub Signal” card for the Costinel ops dashboard.  
- **Why now**: Reuses existing knowledge-rag graph and proven UI patterns; zero backend changes unless you choose to add a thin endpoint.  
- **Outcome**: Ops sees the most-connected hub (e.g., “MOC”), a connection score, a short insight excerpt, and one-click access to full insights + related docs. Enables faster triage of cost anomalies and governance decisions.

---

## Architecture (decided)
- **Component location**: `components/ops/TopHubSignalCard.tsx`  
- **Modal location**: `components/ops/TopHubInsightsModal.tsx`  
- **Data hook**: `hooks/useTopHubInsights.ts`  
- **Data source preference**:  
  1. Call `/api/ops/top-hub` (GET) if it exists.  
  2. If missing, fall back to a local `lib/api/ops.ts` helper that calls the knowledge-rag graph endpoint.  
  3. If graph endpoint is unavailable, use a safe static fallback (ops-friendly).  
- **Data flow**: SWR (or lightweight fetch + polling) → transform → render.  
- **Styling**: Reuse existing Costinel design tokens and primitives (`Card`, `Button`, etc.).  
- **Mutations**: None (read-only).  
- **Polling**: 60s for ops freshness (configurable).

```ts
// lib/api/ops.ts
import type { TopHubData } from '@/types/ops';

const FALLBACK_DATA: TopHubData = {
  hubName: 'MOC',
  hubScore: 87,
  insights: [
    {
      text: 'MOC is the most-connected hub in the knowledge graph, frequently referenced in cost anomaly patterns and governance workflows.',
    },
  ],
  relatedDocs: [
    { title: 'Cost Anomaly Playbook', url: '/docs/ops/playbooks/cost-anomaly' },
    { title: 'Governance Workflows', url: '/docs/ops/workflows/governance' },
  ],
};

export async function fetchTopHub(): Promise<TopHubData> {
  try {
    // Try local API route first
    const localRes = await fetch('/api/ops/top-hub', { cache: 'no-store' });
    if (localRes.ok) {
      const json = await localRes.json();
      return json as TopHubData;
    }
  } catch {
    // continue to graph fallback
  }

  try {
    // Fallback to knowledge-rag graph endpoint (adjust path as needed)
    const graphRes = await fetch('/api/knowledge-rag/top-hub', { cache: 'no-store' });
    if (graphRes.ok) {
      const json = await graphRes.json();
      // Normalize if necessary
      return {
        hubName: json.hubName || json.name || 'Unknown',
        hubScore: Number(json.hubScore || json.score || 0),
        insights: Array.isArray(json.insights)
          ? json.insights.map((i: any) => ({ text: String(i.text || i) }))
          : [{ text: String(json.insight || json.summary || 'No insights available.') }],
        relatedDocs: Array.isArray(json.relatedDocs)
          ? json.relatedDocs.map((d: any) => ({ title: d.title, url: d.url }))
          : [],
      } as TopHubData;
    }
  } catch {
    // continue to static fallback
  }

  return FALLBACK_DATA;
}
```

### 2) Data hook (`hooks/useTopHubInsights.ts`)
```ts
// hooks/useTopHubInsights.ts
import { useState, useEffect, useCallback } from 'react';
import type { TopHubData } from '@/types/ops';
import { fetchTopHub } from '@/lib/api/ops';

const POLL_INTERVAL = 60_000; // 60s

export function useTopHubInsights() {
  const [data, setData] = useState<TopHubData | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  const fetchData = useCallback(async () => {
    try {
      const result = await fetchTopHub();
      setData(result);
      setError(null);
    } catch (err) {
      setError(err as Error);
    } finally {
      setLoading(false);
    }
  }, []);

  useEffect(() => {
    fetchData();
    const id = setInterval(fetchData, POLL_INTERVAL);
    return () => clearInterval(id);
  }, [fetchData]);

  return { data, loading, error, refetch: fetchData };
}
```
