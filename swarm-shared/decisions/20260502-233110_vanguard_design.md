# vanguard / design

## Final Synthesized Solution

**Core diagnosis (agreed):**  
- No persistent file-list cache → repeated `list_repo_tree`/`load_dataset` hits HF 429.  
- Lightning Studio lifecycle not reused → quota burned on recreation; idle-stop kills runs.  
- `load_dataset(streaming=True)` on heterogeneous repos → `pyarrow.CastError`.  
- Heavy `from_pretrained` on Mac → slow, unreliable, burns local resources.  
- No CDN bypass → every fetch consumes `/api/` rate-limit budget.

**Chosen strategy (correct + actionable):**  
1. **Generate once, train many:** a single JSON manifest with CDN URLs created on Mac after rate-limit window.  
2. **CDN-only training:** never call HF API during training; stream parquet via `https://huggingface.co/datasets/{repo}/resolve/main/...`.  
3. **Schema resilience:** project heterogeneous parquet to `{prompt, response}` with safe fallbacks; validate before training.  
4. **Lightning Studio reuse:** detect existing studio; resume or restart cleanly; pin machine type and idle timeout.  
5. **Offload model loading:** run training exclusively on Lightning/Cerebras; Mac only orchestrates.

### 1. Manifest generator (run on Mac)

```bash
# /opt/axentx/vanguard/training/generate_manifest.py
#!/usr/bin/env python3
"""
Generate CDN-only manifest once per (repo, date_folder).
Usage:
  HF_TOKEN=... python3 generate_manifest.py <repo> <date_folder> > file_manifest.json
"""
import json
import sys
from typing import List, Dict

from huggingface_hub import HfApi

CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"


def build_manifest(repo: str, date_folder: str) -> List[Dict]:
    api = HfApi()
    # Non-recursive to avoid pagination explosion; caller can recurse if needed.
    tree = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)
    files = [
        {
            "repo": repo,
            "path": item.rfilename,
            "cdn_url": CDN_TEMPLATE.format(repo=repo, path=item.rfilename),
        }
        for item in tree
        if item.rfilename.endswith(".parquet")
    ]
    if not files:
        # Recurse one level if no parquet at top
        for item in tree:
            if not item.rfilename.endswith("/"):
                continue
            sub = api.list_repo_tree(repo=repo, path=item.rfilename, recursive=False)
            for subitem in sub:
                if subitem.rfilename.endswith(".parquet"):
                    files.append(
                        {
                            "repo": repo,
                            "path": subitem.rfilename,
                            "cdn_url": CDN_TEMPLATE.format(repo=repo, path=subitem.rfilename),
                        }
                    )
    return files


def main() -> None:
    if len(sys.argv) != 3:
        print("Usage: generate_manifest.py <repo> <date_folder>", file=sys.stderr)
        sys.exit(1)
    repo, date_folder = sys.argv[1], sys.argv[2]
    manifest = build_manifest(repo, date_folder)
    json.dump(manifest, sys.stdout, indent=2)


if __name__ == "__main__":
    main()
```

    with open(path) as f:

    if not isinstance(data, list) or not all("cdn_url" in item for item in data):
        raise ValueError("Invalid manifest: expected list of objects with 'cdn_url'")
    return data


# ---------- Dataset ----------
def project_to_prompt_response(example: Dict) -> Dict:
    prompt = (
        example.get("prompt")
        or example.get("instruction")
        or example.get("text")
        or example.get("input")
        or ""
    )
    response = (
        example.get("response")
        or example.get("completion")
        or example.get("output")
        or ""
    )
    return {"prompt": str(prompt), "response": str(response)}


def cdn_only_dataset(manifest: List[Dict], split: str = "train", max_files: Optional[int] = None):
    """
    Build dataset from CDN URLs only (no HF API calls during training).
    Projects heterogeneous parquet to {prompt, response}.
    """
    urls = [f["cdn_url"] for f in manifest]
    if max_files:
        urls = urls[:max_files]

    # Define minimal schema to avoid pyarrow.CastError surprises
    features = Features(
        {
            "prompt": Value("string"),
            "response": Value("string"),
        }
    )

    ds = load_dataset(
        "parquet",
        data_files=urls,
        split=split,
        streaming=True,
        features=features,
    )
    ds = ds.map(project_to_prompt_response)
    return ds

def ensure_or_create_studio():
    """
    Reuse existing Lightning Studio if available; restart if stopped.
    Prefer L40S on lightning-public-prod (free) or H200 on lightning-lambda-prod (paid).
    """
    try:
        from lightning.fabric.plugins import LightningTeam

        team = LightningTeam(team_name=LIGHTNING_TEAM) if LIGHTNING_TEAM else LightningTeam()
        studios = team.list_studios()
        studio = next((s for s in studios if s.name == STUDIO_NAME), None)

        if studio and studio.status == "running":
            print(f"Reusing running studio: {studio.name}")
            return studio

        if studio and studio.status in ("stopped", "failed"):
            print(f"Restarting stopped/failed studio: {studio.name}")
            studio.start()
            studio.wait_until_running()
            return studio

        # Create new
        print(f"Creating studio: {STUDIO_NAME}")
        studio = team.create_studio(
            name=STUDIO_NAME,
            machine_type="L40S-1",
            cloud="lightning-public-prod",
            idle_timeout_min=IDLE_TIMEOUT_MIN,
        )
        studio.wait_until_running()
        return studio
    except Exception as e:
        print(f"Studio lifecycle unavailable ({e}); falling back to local Fabric.")
        return None
