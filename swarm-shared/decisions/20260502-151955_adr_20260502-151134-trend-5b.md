## v0.1 scope
v0.1 in 2 weeks: single deterministic workflow (research → signal → risk check → simulated execution) with immutable audit trail, Redis Streams replayable runs, Postgres (Supabase) persistence, CF Workers API, one broker connector mock, and compliance guardrails as JSON predicates; includes unit/integration tests and replay CLI.

## Open questions
- Which brokers/OMS/EMS and data vendors must be supported in v0.1 to prove NSM?
- What deterministic replay SLA is required (exact bitwise repeat vs logical equivalence) and how will clock-drift and non-deterministic external calls be handled?
- What compliance guardrail format will be enforced (YAML rules, JSON Schema predicates, or WASM policies) and who authors them?
- What latency budget per workflow run (end-to-end) is acceptable for sub-second orchestration on free-tier infra?
- How will secrets/credentials for broker/OMS/EMS connectors be stored and rotated (Supabase Vault, CF Workers KV, or external secret store)?
- What is the minimal audit retention period and export format required for regulatory checks?
