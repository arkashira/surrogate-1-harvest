# workio / backend

## 1. Diagnosis

- **Non-idempotent LINE webhook handler**: retries on 5xx/timeouts create duplicate punches and invalid open/close state.
- **No atomic punch transition**: concurrent/retried requests can interleave, leaving `punches` without a matching close or double-close.
- **Missing request deduplication**: no idempotency key or de-dup store for LINE events (same `webhookEventId` can be processed multiple times).
- **Race condition on `punches` table**: `SELECT ... FOR UPDATE` scope is likely too narrow or missing; concurrent requests can both see "no open punch" and insert two opens.
- **No compensating close on duplicate open**: when duplicate open is detected, there’s no safe rollback/close to preserve valid state.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/line/webhook.ts` (or equivalent handler module).  
Scope: make `handleLineWebhook` idempotent and atomic for clock-in/out events by:
- Adding idempotency check on `webhookEventId` (store processed IDs with short TTL).
- Wrapping punch transition in a single atomic DB transaction with row-level locking (`SELECT ... FOR UPDATE` on employee’s open punch).
- Enforcing exactly-one open punch per employee per tenant at transaction commit.

## 3. Implementation

```ts
// server/src/routes/line/webhook.ts
import { Router, Request, Response } from 'express';
import { pool } from '../../db';
import { redis } from '../../lib/redis'; // assumed ioredis/redis client

const router = Router();
const IDEMPOTENCY_TTL_SECONDS = 60 * 60; // 1h

async function isDuplicateEvent(eventId: string): Promise<boolean> {
  const key = `line:webhook:processed:${eventId}`;
  const set = await redis.set(key, '1', 'EX', IDEMPOTENCY_TTL_SECONDS, 'NX');
  return set === null; // null means key existed -> duplicate
}

router.post('/webhook/line', async (req: Request, res: Response) => {
  const signature = req.headers['x-line-signature'] as string;
  const body = req.body;

  // Basic validation (keep existing signature check if present)
  if (!body.events || !Array.isArray(body.events)) {
    return res.status(400).json({ error: 'invalid payload' });
  }

  for (const event of body.events) {
    // Idempotency guard
    if (await isDuplicateEvent(event.webhookEventId)) {
      console.info(`Duplicate LINE event skipped: ${event.webhookEventId}`);
      continue;
    }

    // Only handle message events with clock-in/out intent (adjust to your command/location logic)
    if (event.type === 'message' && event.message.type === 'text') {
      const text = event.message.text.trim().toLowerCase();
      const userId = event.source.userId;
      const tenantId = 1; // derive from DB mapping in real code

      if (text === 'clock in' || text === 'clock out') {
        const client = await pool.connect();
        try {
          await client.query('BEGIN');

          // Lock employee row and any open punch row to prevent races
          await client.query(
            `SELECT id FROM employees WHERE user_id = $1 AND tenant_id = $2 FOR UPDATE`,
            [userId, tenantId]
          );

          // Find current open punch (if any)
          const openPunch = await client.query(
            `SELECT id FROM punches
             WHERE user_id = $1 AND tenant_id = $2 AND clock_out IS NULL
             ORDER BY clock_in DESC LIMIT 1 FOR UPDATE`,
            [userId, tenantId]
          );

          if (text === 'clock in') {
            // If there's already an open punch, close it first (compensate) then open new
            if (openPunch.rows.length > 0) {
              await client.query(
                `UPDATE punches SET clock_out = NOW() WHERE id = $1`,
                [openPunch.rows[0].id]
              );
            }
            await client.query(
              `INSERT INTO punches (user_id, tenant_id, clock_in, location, source)
               VALUES ($1, $2, NOW(), $3, $4)`,
              [userId, tenantId, event.message.text, 'line']
            );
          } else {
            // clock out
            if (openPunch.rows.length === 0) {
              // No open punch — create a synthetic open then close (or reject).
              // Here we create an open punch so state remains valid.
              await client.query(
                `INSERT INTO punches (user_id, tenant_id, clock_in, clock_out, location, source)
                 VALUES ($1, $2, NOW(), NOW(), $3, $4)`,
                [userId, tenantId, 'auto-open', 'line']
              );
            } else {
              await client.query(
                `UPDATE punches SET clock_out = NOW() WHERE id = $1`,
                [openPunch.rows[0].id]
              );
            }
          }

          await client.query('COMMIT');
        } catch (err) {
          await client.query('ROLLBACK');
          console.error('Failed to process punch:', err);
          // Return 5xx so LINE may retry (idempotency will dedupe)
          return res.status(500).json({ error: 'transaction failed' });
        } finally {
          client.release();
        }
      }
    }
  }

  return res.status(200).json({ message: 'ok' });
});

export default router;
```

Notes:
- Add Redis client setup in `server/src/lib/redis.ts` if not present (or use an in-memory fallback with LRU + TTL for dev).
- Adjust tenant resolution (mapping `userId` → `tenantId`) to your existing logic.
- Keep existing LINE signature verification middleware before this handler if you have it.

## 4. Verification

1. **Idempotency test**: Send the same LINE webhook payload twice (same `webhookEventId`) within 1h. Second request must not create a second punch; logs should show “Duplicate LINE event skipped”.
2. **Concurrent race test**: Simulate two parallel `clock in` requests for same user (e.g., `ab -n 10 -c 5` to local endpoint). After all requests finish, there must be exactly one open punch (or one closed punch if both were clock-out scenarios).
3. **Atomic transition test**: With an open punch present, send `clock in`. Should close previous open and create new open (one row updated, one row inserted). Verify no two open punches exist.
4. **Compensation test**: Send `clock out` with no open punch. Should create a punch with `clock_in` ≈ `clock_out` (synthetic) so state remains consistent.
5. **DB integrity check**: Run `SELECT user_id, count(*) FROM punches WHERE clock_out IS NULL GROUP BY user_id HAVING count(*) > 1;` — must return zero rows.
