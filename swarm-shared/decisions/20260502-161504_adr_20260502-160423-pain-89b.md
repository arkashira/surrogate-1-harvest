## v0.1 scope
v0.1 ships in 2 weeks: local CLI agent (Rust) with TUN/SOCKS5 shim, one stealth profile (QUIC-over-443 mimic), local SQLite session store, opt-in encrypted WAU-ZB telemetry, and fail-closed kill-switch (best-effrict). ## Open questions
- Which DPI signatures (SNI, packet sizes, timing, TUN MTU artifacts) are most reliable to spoof in target regions?
- Can we achieve reliable TUN/tuntap creation without admin/root while remaining undetectable?
- What innocuous cover traffic patterns (ports, TLS fingerprints, QUIC/HTTP3) maximize evasion without harming performance?
- How will we validate zero-blocked sessions (WAU-ZB) without leaking identifiers or telemetry?
- Should we integrate with existing bridges (e.g., obfs4, Snowflake) or build native obfuscation?
- How to implement fail-closed kill-switch across platforms (Windows Filtering Platform, nftables/iptables, pf) without privilege escalation?
