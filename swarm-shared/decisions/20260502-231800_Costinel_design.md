# Costinel / design

## Implementation Plan — `GET /api/v1/cost-anomaly/signal/top-hub`

**Scope & constraints**
- Read-only, no side effects.
- Optional `?for_date=YYYY-MM-DD` (default today UTC) and `?top_n` (1–10, default 1).
- Reuse existing knowledge-graph assets (MOC-style hub insight) to surface the most-connected anomaly/driver hub for the selected date.
- Fast path: if graph is unavailable, fall back to lightweight aggregation on cost-anomaly signals table (no mutations, no external calls).
- Ship in <2h: single endpoint + minimal service/repository layer + tests.

### 2) Endpoint contract

**Request**
```
GET /api/v1/cost-anomaly/signal/top-hub?for_date=2026-05-02&top_n=3
```

**Success response (200)**
```json
{
  "date": "2026-05-02",
  "top_n": 3,
  "hubs": [
    {
      "hub_id": "MOC",
      "hub_type": "service",
      "display_name": "Management-Of-Change",
      "score": 0.92,
      "signal_count": 14,
      "total_impact_usd": 18430.25,
      "primary_tags": ["prod", "us-east-1"],
      "context_snippet": "Top driver for cost anomalies on 2026-05-02"
    }
  ],
  "source": "knowledge_graph"
}
```

**Errors**
- `400` — invalid `for_date` or `top_n` out of range.
- `503` — graph unavailable and fallback data unavailable (graceful degradation preferred; if both unavailable, return empty `hubs` with `source: "none"` and 200).

### 3) Service logic (pseudocode)

```python
# src/services/cost_anomaly/top_hub_service.py
from datetime import datetime, timezone
from typing import List, Dict, Any
from repositories.knowledge_graph_repo import KnowledgeGraphRepo
from repositories.cost_anomaly_repo import CostAnomalyRepo

class TopHubService:
    def __init__(self, graph_repo: KnowledgeGraphRepo, anomaly_repo: CostAnomalyRepo):
        self.graph_repo = graph_repo
        self.anomaly_repo = anomaly_repo

    def get_top_hubs(self, for_date: datetime, top_n: int = 1) -> Dict[str, Any]:
        # Normalize to UTC date
        day = for_date.astimezone(timezone.utc).date()

        # 1) Try knowledge graph first (reuse existing MOC-style hub insight)
        try:
            hubs = self.graph_repo.top_hubs_by_date(day, top_n=top_n)
            if hubs:
                return {
                    "date": str(day),
                    "top_n": top_n,
                    "hubs": hubs,
                    "source": "knowledge_graph",
                }
        except Exception:
            # graceful degrade
            pass

        # 2) Fallback: lightweight aggregation on cost-anomaly signals
        try:
            hubs = self.anomaly_repo.aggregate_top_hubs_by_date(day, top_n=top_n)
            return {
                "date": str(day),
                "top_n": top_n,
                "hubs": hubs,
                "source": "cost_anomaly_agg",
            }
        except Exception:
            return {
                "date": str(day),
                "top_n": top_n,
                "hubs": [],
                "source": "none",
            }
```

class TopHubQuery(BaseModel):
