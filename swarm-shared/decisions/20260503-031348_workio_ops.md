# workio / ops

## 1. Diagnosis

- Missing **LINE webhook signature verification** (`X-Line-Signature`) — any caller can spoof clock-in/out events and corrupt attendance records.
- No **idempotency on `line_event_id`** — LINE retries on 5xx/timeouts create duplicate clock-in/out and leave/OT rows.
- Webhook handler is likely **not atomic** (check-then-insert) — race under retry can double-process the same event.
- No **replay protection window** — old captured events could be replayed after employee leaves or role changes.
- Ops observability gap — no structured logging of verification/idempotency decisions, making incidents hard to triage.

## 2. Proposed change

File: `/opt/axentx/workio/server/src/routes/webhook/line.ts` (or equivalent route handler)  
Scope:
- Add `crypto.timingSafeEqual` signature verification using channel secret from `.env`.
- Add unique constraint/index on `line_event_id` in attendance/leave/OT tables (or upsert with `ON CONFLICT DO NOTHING`).
- Make handler idempotent: upsert by `line_event_id`, return 200 for duplicates.
- Add short replay window check (e.g., reject events older than 5 minutes).
- Add structured logs for verification/idempotency outcomes.

```ts
// /opt/axentx/workio/server/src/routes/webhook/line.ts
import crypto from 'crypto';
import express from 'express';
import { db } from '../../db';
import { logger } from '../../utils/logger';

const router = express.Router();
const CHANNEL_SECRET = process.env.LINE_CHANNEL_SECRET || '';

function verifySignature(body: string, signature: string): boolean {
  if (!CHANNEL_SECRET || !signature) return false;
  const expected = crypto
    .createHmac('sha256', CHANNEL_SECRET)
    .update(body, 'utf8')
    .digest('base64');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}

function isReplay(timestamp: string | undefined, toleranceMs = 5 * 60 * 1000): boolean {
  if (!timestamp) return true;
  const ts = new Date(timestamp).getTime();
  if (Number.isNaN(ts)) return true;
  return Math.abs(Date.now() - ts) > toleranceMs;
}

router.post('/line', express.raw({ type: 'application/json' }), async (req, res) => {
  const signature = req.headers['x-line-signature'] as string;
  const bodyRaw = req.body.toString('utf8');
  const ok = verifySignature(bodyRaw, signature);

  if (!ok) {
    logger.warn({ msg: 'line_webhook_invalid_signature', signature: !!signature });
    return res.status(401).json({ error: 'Invalid signature' });
  }

  let events;
  try {
    events = JSON.parse(bodyRaw).events || [];
  } catch (err) {
    logger.error({ msg: 'line_webhook_invalid_json', err });
    return res.status(400).json({ error: 'Invalid JSON' });
  }

  const results = [];
  for (const ev of events) {
    if (isReplay(ev.timestamp)) {
      logger.warn({ msg: 'line_webhook_replay_rejected', eventId: ev.id, timestamp: ev.timestamp });
      results.push({ status: 'rejected', reason: 'replay' });
      continue;
    }

    try {
      // Idempotent upsert by line_event_id
      const { type, source, timestamp: evTs, ...payload } = ev;
      const result = await db.attendancePunches.upsert({
        where: { line_event_id: ev.id },
        update: {}, // no-op if exists
        create: {
          line_event_id: ev.id,
          user_id: source.userId,
          tenant_id: null, // resolve via user_id in app logic
          event_type: type,
          payload,
          occurred_at: new Date(evTs || Date.now()),
          created_at: new Date(),
        },
      });
      results.push({ status: result.created ? 'inserted' : 'duplicate', eventId: ev.id });
      logger.info({ msg: 'line_webhook_processed', eventId: ev.id, action: result.created ? 'inserted' : 'duplicate' });
    } catch (err: any) {
      // Unique violation or other DB error — still return 200 to stop LINE retries if it's a duplicate
      if (err?.code === '23505' || err?.name === 'UniqueConstraintViolation') {
        logger.info({ msg: 'line_webhook_duplicate', eventId: ev.id });
        results.push({ status: 'duplicate', eventId: ev.id });
      } else {
        logger.error({ msg: 'line_webhook_db_error', eventId: ev.id, err });
        results.push({ status: 'error', eventId: ev.id });
      }
    }
  }

  // Always 200 for signature-valid requests to stop LINE retries
  return res.status(200).json({ results });
});

export default router;
```

## 4. Verification

1. **Signature rejection**  
   ```bash
   curl -X POST http://localhost:3000/webhook/line \
     -H "Content-Type: application/json" \
     -H "X-Line-Signature: bogus" \
     -d '{"events":[{"id":"test-123","type":"message","timestamp":"2026-05-03T12:00:00.000Z","source":{"userId":"U123"},"message":{"text":"in"}}]}'
   ```
   Expect `401` and log line `line_webhook_invalid_signature`.

2. **Valid signature, idempotent insert**  
   - Compute valid signature using your channel secret and request body, then:
   ```bash
   curl -X POST http://localhost:3000/webhook/line \
     -H "Content-Type: application/json" \
     -H "X-Line-Signature: <valid_sig>" \
     -d '{"events":[{"id":"test-123","type":"message","timestamp":"2026-05-03T12:00:00.000Z","source":{"userId":"U123"},"message":{"text":"in"}}]}'
   ```
   Expect `200` and `results[0].status` = `"inserted"`. Repeat same request → `"duplicate"`.

3. **Replay protection**  
   Send same payload with old timestamp (e.g., 10 minutes ago). Expect `200` with `status: "rejected"` and log `line_webhook_replay_rejected`.

4. **DB uniqueness enforcement**  
   Attempt direct duplicate insert into `attendance_punches` with same `line_event_id`; expect unique constraint violation.

5. **Observability**  
   Check logs for `line_webhook_processed`, `line_webhook_duplicate`, and `line_webhook_invalid_signature` entries with structured fields.
