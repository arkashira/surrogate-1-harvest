# workio / design

## Final synthesized answer

### 1. Diagnosis (merged)
- **No DB-level guardrails**: multiple open punches (`clock_out_at IS NULL`) per user are possible under concurrency/retries.
- **Race-prone app logic**: find-then-insert/update allows double-punch on fast taps or LINE webhook retries.
- **Non-idempotent webhook**: repeated deliveries create duplicate punches; no idempotency key or replay protection.
- **Permissive validation**: clock-in allowed while an open punch exists; clock-out allowed when none exists, producing invalid state and confusing UX.
- **Missing concurrency control**: no optimistic locking or atomic upsert to serialize concurrent attempts.

### 2. Scope and goals
- **Scope**: `server/src/db/schema.sql` (constraint + index) and `server/src/routes/webhook/line.ts` (idempotent handler + state-machine validation). Optional: small service helper for atomic upsert.
- **Goal**: enforce “at most one open punch per user” at the database; make webhook processing idempotent and safe under concurrency; add minimal, concrete validation to reject invalid transitions.

### 3. Implementation

#### 3.1 DB schema — enforce single open punch
```sql
-- server/src/db/schema.sql
-- Prevent multiple open punches per user
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_user_open
ON punches(user_id)
WHERE clock_out_at IS NULL;

-- Strengthen schema
ALTER TABLE punches
  ALTER COLUMN user_id   SET NOT NULL,
  ALTER COLUMN clock_in_at SET NOT NULL;
```

#### 3.2 Idempotent webhook handler
```ts
// server/src/routes/webhook/line.ts
import { Router, Request, Response } from 'express';
import { db } from '../../db';
import { verifyLineSignature } from '../../utils/line';

const router = Router();

// Prefer Redis or DB-backed store for multi-instance deployments.
// This in-memory store is safe for single-instance dev/demo.
const IDEMPOTENCY_TTL_MS = 5 * 60 * 1000;
const seen = new Map<string, number>();

function isDuplicate(key: string): boolean {
  const now = Date.now();
  const ts = seen.get(key);
  if (ts && now - ts < IDEMPOTENCY_TTL_MS) return true;
  seen.set(key, now);
  setTimeout(() => seen.delete(key), IDEMPOTENCY_TTL_MS).unref?.();
  return false;
}

router.post('/line', verifyLineSignature, async (req: Request, res: Response) => {
  const { events } = req.body;
  const deliveryId = (req.headers['x-line-delivery-id'] as string) || crypto.randomUUID();

  // Idempotency: fast accept for duplicates to suppress LINE retries
  if (isDuplicate(deliveryId)) {
    return res.status(200).json({ ok: true, reason: 'duplicate' });
  }

  try {
    for (const ev of events) {
      if (ev.type !== 'message' || ev.message.type !== 'text') continue;

      const userId = ev.source.userId;
      const text = ev.message.text.trim().toLowerCase();

      if (text === 'clock in' || text === 'เข้างาน') {
        await handleClockIn(userId);
      } else if (text === 'clock out' || text === 'เลิกงาน') {
        await handleClockOut(userId);
      }
    }

    return res.status(200).json({ ok: true });
  } catch (err: any) {
    // Unique constraint violation => open punch exists. Accept to suppress retries.
    if (err?.code === '23505' || err?.constraint === 'idx_punches_user_open') {
      return res.status(200).json({ ok: true, reason: 'open_punch_exists' });
    }
    console.error('Webhook error', err);
    return res.status(500).json({ error: 'internal' });
  }
});

async function handleClockIn(userId: string) {
  const now = new Date();

  // Prefer atomic insert; conflict means open punch exists -> reject transition.
  try {
    await db.query(
      `INSERT INTO punches (user_id, clock_in_at, created_at)
       VALUES ($1, $2, $3)`,
      [userId, now, now]
    );
  } catch (err: any) {
    if (err?.code === '23505' || err?.constraint === 'idx_punches_user_open') {
      // Reject duplicate clock-in; client should reflect existing open punch.
      return;
    }
    throw err;
  }
}

async function handleClockOut(userId: string) {
  const now = new Date();

  // Atomic update; rowCount === 0 means no open punch -> no-op (idempotent).
  const result = await db.query(
    `UPDATE punches
     SET clock_out_at = $1, updated_at = $2
     WHERE user_id = $3 AND clock_out_at IS NULL
     RETURNING id`,
    [now, now, userId]
  );

  if (result.rowCount === 0) {
    // No open punch; optionally log or notify client of invalid transition.
    return;
  }
}

export default router;
```

#### 3.3 Signature verification (unchanged, required)
```ts
// server/src/utils/line.ts
import crypto from 'crypto';

export function verifyLineSignature(req: Request, res: Response, next: () => void) {
  const channelSecret = process.env.LINE_CHANNEL_SECRET || '';
  const signature = req.headers['x-line-signature'] as string || '';
  const body = JSON.stringify(req.body);
  const expected = crypto
    .createHmac('sha256', channelSecret)
    .update(body)
    .digest('base64');

  if (signature !== expected) {
    return res.status(401).json({ error: 'invalid_signature' });
  }
  next();
}
```

### 4. Verification checklist
1. **Apply schema**: run migration and confirm index exists:
   ```sql
   SELECT indexname, indexdef FROM pg_indexes WHERE tablename = 'punches';
   ```
2. **Concurrent clock-in**: simulate two rapid `clock in` calls for same user. Expect exactly one punch with `clock_out_at IS NULL`.
3. **Duplicate delivery**: replay same `x-line-delivery-id`; second request must return `200 OK` with no state change.
4. **Clock-out idempotency**: multiple `clock out` calls must update the open punch once; subsequent calls are no-ops.
5. **Constraint enforcement**: direct SQL insert of a second open punch for same user must fail with unique constraint violation.
6. **Validation behavior**:
   - Clock-in while open punch exists → rejected (no new row).
   - Clock-out while no open punch → no-op (idempotent).

### 5. Production notes
- Replace in-memory `seen` map with Redis or a DB idempotency table for multi-instance deployments.
- Consider adding `idempotency_key` column to punches/webhook_logs for stronger auditability.
- Optionally add a daily cron to auto-close stale open punches if business rules require it (but prefer explicit clock-out).
