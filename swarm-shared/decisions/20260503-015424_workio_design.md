# workio / design

## Final Synthesized Solution

**Core problems (both candidates agree):**  
- No idempotency protection on LINE webhook retries (same `X-Line-Signature` + body can replay).  
- Non-atomic `findOne → update/insert` is racy and can double-punch or lose updates.  
- Uniqueness for active punches is only in app code, not the database.  
- Missing transactional grouping for punch + audit/event/notification writes.

**Chosen approach:**  
- Add a **DB-level idempotency table** keyed by tenant + external id (signature) + payload hash.  
- Add a **partial unique index** on `active_punch(employee_id, tenant_id) WHERE clocked_out_at IS NULL`.  
- Replace the upsert with a **single atomic SQL statement** inside a transaction that also marks idempotency processed.  
- Keep tenant isolation strict (all queries include `tenant_id`; composite constraints prevent cross-tenant leaks).

### 1) Database schema (run once)

```sql
-- workio/server/src/db/schema.sql

-- Idempotency for LINE webhooks (per-tenant, per-signature)
CREATE TABLE IF NOT EXISTS line_webhook_idempotency (
  id              BIGSERIAL PRIMARY KEY,
  tenant_id       BIGINT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  external_id     TEXT   NOT NULL,            -- X-Line-Signature
  payload_hash    TEXT   NOT NULL,            -- SHA256 of raw body
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  processed_at    TIMESTAMPTZ,
  UNIQUE (tenant_id, external_id)
);

-- Guarantee at most one active punch per employee per tenant
CREATE UNIQUE INDEX IF NOT EXISTS idx_active_punch_employee_tenant_active
  ON active_punch (employee_id, tenant_id)
  WHERE clocked_out_at IS NULL;

-- Index for tenant-isolated queries
CREATE INDEX IF NOT EXISTS idx_punches_tenant_employee
  ON punches (tenant_id, employee_id, clocked_in_at);
```

### 2) Punch service — atomic upsert + idempotency

```ts
// workio/server/src/services/punchService.ts
import { Pool } from 'pg';
import crypto from 'crypto';

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

type PunchInput = {
  tenantId: number;
  employeeId: number;
  clockIn: boolean;
  latitude?: number;
  longitude?: number;
  lineSignature: string;   // X-Line-Signature
  rawBody: string;         // raw request body (for hash)
};

export async function handlePunch(input: PunchInput) {
  const { tenantId, employeeId, clockIn, latitude, longitude, lineSignature, rawBody } = input;
  const payloadHash = crypto.createHash('sha256').update(rawBody).digest('hex');
  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    // Idempotency: insert or detect duplicate
    const idem = await client.query(
      `INSERT INTO line_webhook_idempotency (tenant_id, external_id, payload_hash, created_at)
       VALUES ($1, $2, $3, NOW())
       ON CONFLICT (tenant_id, external_id) DO UPDATE
       SET payload_hash = EXCLUDED.payload_hash
       WHERE line_webhook_idempotency.payload_hash = EXCLUDED.payload_hash
       RETURNING id, processed_at`,
      [tenantId, lineSignature, payloadHash]
    );

    // Already processed -> skip punch logic (idempotent success)
    if (idem.rows[0].processed_at) {
      await client.query('COMMIT');
      return { ok: true, idempotent: true };
    }

    const idemId = idem.rows[0].id;

    // Atomic punch handling
    if (clockIn) {
      // Insert active punch; conflict on same employee+tenant with active punch -> reject duplicate
      await client.query(
        `INSERT INTO active_punch (employee_id, tenant_id, clocked_in_at, latitude, longitude)
         VALUES ($1, $2, NOW(), $3, $4)
         ON CONFLICT (employee_id, tenant_id)
         WHERE clocked_out_at IS NULL
         DO NOTHING`,
        [employeeId, tenantId, latitude, longitude]
      );
    } else {
      // Clock out: move active -> punches atomically
      await client.query(
        `WITH moved AS (
           DELETE FROM active_punch
           WHERE employee_id = $1 AND tenant_id = $2 AND clocked_out_at IS NULL
           RETURNING id, employee_id, tenant_id, clocked_in_at, latitude, longitude
         )
         INSERT INTO punches (employee_id, tenant_id, clocked_in_at, clocked_out_at, latitude, longitude)
         SELECT employee_id, tenant_id, clocked_in_at, NOW(), latitude, longitude
         FROM moved`,
        [employeeId, tenantId]
      );
    }

    // Mark idempotency processed
    await client.query(
      `UPDATE line_webhook_idempotency SET processed_at = NOW() WHERE id = $1`,
      [idemId]
    );

    // Optional: insert audit row or send LINE notification here in same transaction
    // await client.query('INSERT INTO punch_audit(...) VALUES(...)');

    await client.query('COMMIT');
    return { ok: true, idempotent: false };
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

### 4) Verification checklist

1. **Idempotency**  
   - Send identical LINE webhook twice (same signature + body).  
   - Expect: one punch row; second request returns `{ ok: true, idempotent: true }`; no duplicate active_punch or punches.

2. **Race-condition resistance**  
   - Fire 10 concurrent clock-ins for same employee/tenant.  
   - Expect: exactly one `active_punch` row with `clocked_out_at IS NULL`; no constraint violations.

3. **Atomic clock-out**  
   - Clock in once, then fire concurrent clock-outs.  
   - Expect: exactly one row moved to `punches` with non-null `clocked_out_at`; no double clock-outs.

4. **Tenant isolation**  
   - Create two tenants with same employee external IDs.  
   - Perform punches on both; verify no cross-tenant leakage (all queries include `tenant_id`; unique index is tenant-scoped).
