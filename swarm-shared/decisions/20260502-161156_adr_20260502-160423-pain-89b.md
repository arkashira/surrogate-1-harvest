## v0.1 scope
v0.1 ships a cross-platform CLI daemon (Linux-first) that: creates a TUN interface, tunnels traffic through an obfuscated SOCKS5-like transport (shadowsocks-like AEAD), hides local port/TUN artifacts from basic DPI probes, enforces a fail-closed kill-switch (block non-tunnel traffic on TUN loss), stores sessions locally in SQLite, emits minimal encrypted client-side telemetry for WAU-ZB, and runs zero-config with bundled defaults (no server control required).

## Open questions
- Which obfuscation primitives (meek, shadowsocks-like, v2ray-style) minimize DPI signatures while maximizing throughput on low-end devices?
- How will client-side telemetry (WAU-ZB) be collected and verified without deanonymizing users or leaking session metadata?
- What is the kill-switch enforcement model (network namespace isolation, firewall rules, TUN teardown) to guarantee no leaks across platforms (Linux, Windows, macOS)?
- How do we distribute and rotate bridge/obfuscation endpoints without centralized infrastructure while resisting enumeration/blocking?
- Which local artifacts (process names, ports, filesystem paths, packet timing) must be randomized or hidden to defeat on-host detection?
