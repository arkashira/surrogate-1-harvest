# airship / quality

## Highest-value incremental improvement (<2h)

**Goal**: Ship a frontend-safe `airship discover` orchestrator that produces a static, cacheable status page and a CDN-only manifest so the frontend can render health/state without SSR/backend calls.

**Why this wins**:
- Immediate UX value: status/state visible without auth or backend roundtrips.
- Reduces load on Arkship/Surrogate (CDN cacheable).
- Fits existing patterns: CDN bypass, pre-list + embed, reuse running infra.
- Deliverable is a single CLI + static outputs + small frontend hook (~1.5h).

---

## Implementation plan

1. Add `scripts/airship-discover.sh` (Bash, executable, shebang)  
   - Runs discovery (lightweight health/probe) for Arkship (8000) and Surrogate (8001).  
   - Emits:
     - `public/status.json` (canonical status + timestamp + commit/version)
     - `public/manifest.json` (CDN-only asset map: routes, feature flags, last training dataset slug)
   - Uses `curl --max-time 5 --retry 2` for local probes; falls back to cached values if services down.

2. Add `public/status.json` + `public/manifest.json` to repo (committed by script)  
   - `status.json` schema: `{ok, services: {arkship, surrogate}, ts, version, commit}`
   - `manifest.json` schema: `{routes, cdnPath, datasetSlug, trainingFileList}`

3. Embed training file list (CDN-only)  
   - On Mac (or CI), run one `list_repo_tree` for today’s folder, save to `public/manifest.json#trainingFileList`.  
   - Training script (Lightning) will consume this list and fetch via CDN URLs only (zero HF API calls during train).

4. Frontend hook (minimal)  
   - Add small JS module `frontend/lib/status.js` that fetches `/status.json` (cache-first) and exposes reactive state.  
   - Add tiny UI widget (top-bar badge) showing Arkship/Surrogate health (green/yellow/red).

5. Make script cron-friendly  
   - Ensure `SHELL=/bin/bash` in crontab if used.  
   - Script is idempotent and safe to run every 5–15m (or on deploy).

6. Verify locally  
   - `chmod +x scripts/airship-discover.sh`  
   - Run it, confirm `public/status.json` and `public/manifest.json` are valid JSON.  
   - Start both services, confirm probes detect up/down correctly.

## Code snippets

### scripts/airship-discover.sh
```bash
#!/usr/bin/env bash
set -euo pipefail

# airship-discover.sh
# Generates static status + manifest for frontend/CDN consumption.
# Run from repo root: ./scripts/airship-discover.sh

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
PUBLIC_DIR="${ROOT}/public"
STATUS_FILE="${PUBLIC_DIR}/status.json"
MANIFEST_FILE="${PUBLIC_DIR}/manifest.json"

ARKSHIP_URL="http://localhost:8000"
SURROGATE_URL="http://localhost:8001"

VERSION="$(git describe --tags --always --dirty 2>/dev/null || echo 'dev')"
COMMIT="$(git rev-parse HEAD 2>/dev/null || echo 'unknown')"
NOW="$(date -u +"%Y-%m-%dT%H:%M:%SZ")"

probe() {
  local url=$1
  if curl --max-time 5 --retry 2 --fail --silent "${url}/health" >/dev/null 2>&1; then
    echo "up"
  else
    echo "down"
  fi
}

ARKSHIP_STATUS=$(probe "${ARKSHIP_URL}")
SURROGATE_STATUS=$(probe "${SURROGATE_URL}")

OVERALL="ok"
if [ "${ARKSHIP_STATUS}" = "down" ] || [ "${SURROGATE_STATUS}" = "down" ]; then
  OVERALL="degraded"
fi

mkdir -p "${PUBLIC_DIR}"

cat > "${STATUS_FILE}" <<EOF
{
  "ok": $( [ "${OVERALL}" = "ok" ] && echo "true" || echo "false" ),
  "status": "${OVERALL}",
  "services": {
    "arkship": "${ARKSHIP_STATUS}",
    "surrogate": "${SURROGATE_STATUS}"
  },
  "ts": "${NOW}",
  "version": "${VERSION}",
  "commit": "${COMMIT}"
}
EOF

# Minimal manifest (CDN-only asset map).
# trainingFileList can be injected by CI/Mac helper before this script runs.
# If not present, leave empty array.
if [ -f "${PUBLIC_DIR}/.training_filelist.json" ]; then
  TRAINING_FILES=$(cat "${PUBLIC_DIR}/.training_filelist.json")
else
  TRAINING_FILES="[]"
fi

cat > "${MANIFEST_FILE}" <<EOF
{
  "routes": {
    "arkshipUi": "/",
    "arkshipApi": "/api",
    "surrogateApi": "/ai"
  },
  "cdnPath": "/",
  "datasetSlug": "",
  "trainingFileList": ${TRAINING_FILES}
}
EOF

echo "✅ Generated ${STATUS_FILE} and ${MANIFEST_FILE}"
```

### frontend/lib/status.js
```js
// frontend/lib/status.js
// Cache-first fetch for /status.json

const CACHE_KEY = 'airship:status:v1';
const CACHE_TTL_MS = 60 * 1000; // 1m

async function fetchStatus(force = false) {
  const cached = localStorage.getItem(CACHE_KEY);
  if (!force && cached) {
    try {
      const { ts, data } = JSON.parse(cached);
      if (Date.now() - ts < CACHE_TTL_MS) return data;
    } catch {
      // ignore malformed
    }
  }

  try {
    const res = await fetch('/status.json', { cache: 'no-cache' });
    if (!res.ok) throw new Error('status fetch failed');
    const data = await res.json();
    localStorage.setItem(CACHE_KEY, JSON.stringify({ ts: Date.now(), data }));
    return data;
  } catch {
    // fallback to cached if available (even stale)
    if (cached) {
      try {
        return JSON.parse(cached).data;
      } catch {
        return { ok: false, status: 'unknown', services: { arkship: 'down', surrogate: 'down' } };
      }
    }
    throw new Error('status unavailable');
  }
}

function renderBadge(root) {
  fetchStatus().then((s) => {
    const ok = s?.ok === true;
    const color = ok ? '#10b981' : (s?.status === 'degraded' ? '#f59e0b' : '#ef4444');
    root.innerHTML = `
      <span style="display:inline-flex;align-items:center;gap:6px;font-size:12px;color:#fff;padding:2px 8px;border-radius:4px;background:${color}">
        <span style="width:6px;height:6px;border-radius:50%;background:#fff"></span>
        ${ok ? 'Operational' : s?.status || 'Unknown'}
      </span>
    `;
  }).catch(() => {
    root.innerHTML = `<span style="font-size:12px;color:#6b7280">Status unavailable</span>`;
  });
}

export { fetchStatus, renderBadge };
```

### Example usage in frontend HTML
