## Highest-Value Improvement

Add a pre-flight snapshot generator (`bin/snapshot.sh`) that lists all dataset files once per date folder, embeds the file manifest into training, and enables Lightning Studio to do CDN-only fetches with zero HF API calls during data load. This directly implements the **HF CDN Bypass** pattern and prevents 429 rate limits while saving quota.

## Implementation Plan (≤2h)

### 1. Create snapshot generator (`bin/snapshot.sh`)

```bash
#!/usr/bin/env bash
# bin/snapshot.sh
# Lists public dataset files for a date folder and emits file manifest JSON
# Usage: ./bin/snapshot.sh <date> [output.json]
# Example: ./bin/snapshot.sh 2026-04-29 manifest-2026-04-29.json

set -euo pipefail

REPO="datasets/axentx/surrogate-1-training-pairs"
DATE="${1:-$(date +%Y-%m-%d)}"
OUT="${2:-manifest-${DATE}.json}"

echo "📸 Snapshotting ${REPO} for ${DATE} → ${OUT}"

# Single API call: list_repo_tree for the date folder (non-recursive)
# We'll use huggingface_hub CLI if available, else fallback to raw REST
if command -v huggingface-cli &>/dev/null; then
  huggingface-cli repo tree --repo-type dataset "${REPO}" --path "batches/public-merged/${DATE}" --json \
    | jq '[.tree[] | select(.type=="file") | .path]' > "${OUT}"
else
  # CDN-compatible REST (no auth for public repos)
  curl -sL "https://huggingface.co/api/datasets/${REPO}/tree" \
    --get --data-urlencode "path=batches/public-merged/${DATE}" \
    --data-urlencode "recursive=false" \
    | jq '[.[] | select(.type=="file") | .path]' > "${OUT}"
fi

# Also generate a simple newline-delimited path list for easy embedding
jq -r '.[]' "${OUT}" > "${OUT%.json}.paths"

echo "✅ Saved $(jq length "${OUT}") files to ${OUT}"
echo "   CDN paths available in ${OUT%.json}.paths"
```

Make executable:
```bash
chmod +x bin/snapshot.sh
```

### 2. Embed manifest in training script template

Create `train.py` template that reads the manifest and uses CDN-only URLs:

```python
# train.py
# Lightning Studio training with CDN-only dataset fetches
import json
import os
from pathlib import Path
import requests
from datasets import Dataset, Features, Value
import pyarrow as pa

HF_REPO = "datasets/axentx/surrogate-1-training-pairs"
MANIFEST_PATH = os.environ.get("DATASET_MANIFEST", "manifest-latest.json")
CDN_BASE = f"https://huggingface.co/{HF_REPO}/resolve/main"

def load_from_cdn(file_paths):
    """Stream files from CDN without HF API auth/rate limits."""
    rows = []
    for path in file_paths:
        url = f"{CDN_BASE}/{path}"
        # Stream parquet via pyarrow directly from CDN
        import pyarrow.parquet as pq
        try:
            table = pq.read_table(url)
            # Project to {prompt, response} only
            for batch in table.to_batches():
                df = batch.to_pandas()
                for _, row in df.iterrows():
                    rows.append({
                        "prompt": row.get("prompt", ""),
                        "response": row.get("response", "")
                    })
        except Exception as e:
            print(f"⚠️  Failed to fetch {path}: {e}")
            continue
    return Dataset.from_list(rows)

def main():
    # Load manifest generated by bin/snapshot.sh
    with open(MANIFEST_PATH) as f:
        file_paths = json.load(f)

    print(f"📦 Loading {len(file_paths)} files from CDN...")
    dataset = load_from_cdn(file_paths)
    print(f"✅ Dataset size: {len(dataset)}")

    # Continue with Lightning training...
    # Example: use dataset with Trainer
```

### 3. Update GitHub Actions to generate manifest before training

Add a job step in `.github/workflows/ingest.yml` (or create a separate training workflow):

```yaml
# .github/workflows/train.yml
name: surrogate-1-train
on:
  workflow_dispatch:
  schedule:
    - cron: "0 2 * * *"  # daily at 02:00

jobs:
  snapshot-and-train:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Generate dataset manifest
        run: |
          DATE=$(date +%Y-%m-%d)
          ./bin/snapshot.sh "$DATE" manifest-${DATE}.json
          ln -sf manifest-${DATE}.json manifest-latest.json

      - name: Launch Lightning Studio training
        env:
          LIGHTNING_API_KEY: ${{ secrets.LIGHTNING_API_KEY }}
          DATASET_MANIFEST: manifest-latest.json
        run: |
          # Reuse running studio if available
          python -c "
          from lightning import Studio, Machine
          from lightning.fabric.utilities.cloud_io import _load_from_remote_fs
          import os, json

          # Try to reuse running studio
          for s in Studio.list():
              if s.name == 'surrogate-1-train' and s.status == 'Running':
                  print(f'♻️  Reusing running studio: {s.id}')
                  studio = s
                  break
          else:
              print('🚀 Creating new studio')
              studio = Studio.create(
                  name='surrogate-1-train',
                  machine=Machine.L40S,
                  cloud='lightning-public-prod'
              )

          # Upload manifest and train.py
          studio.upload('train.py')
          studio.upload(os.environ['DATASET_MANIFEST'])
          studio.run(['python', 'train.py'])
          "
```
