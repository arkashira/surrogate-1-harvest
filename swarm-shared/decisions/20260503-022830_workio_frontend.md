# workio / frontend

## 1. Diagnosis

- Clock-in/out UI has no optimistic feedback; users can double-tap or retry on slow networks, risking duplicate punches that backend dedupe must absorb.
- No local pending-state or offline queue: if LINE webhook succeeds but UI doesn’t get 2xx (network blip), user may re-punch and create duplicates.
- Missing visual confirmation (toast/modal) after punch; users don’t know whether clock-in/out was accepted.
- GPS acquisition is triggered but UX doesn’t surface lat/long or accuracy; users can’t verify location before submit.
- No loading/idle state on the primary punch button; perceived performance is poor on high-latency mobile connections.

## 2. Proposed change

File: `workio/src/components/PunchButton.tsx` (create if absent) or modify existing punch UI component.  
Scope: single component handling clock-in/out action — add optimistic UI, toast confirmation, GPS readout, and disabled/idle states.

Create/modify component:

```tsx
// workio/src/components/PunchButton.tsx
import { useState, useEffect } from 'react';
import { useToast } from '@/hooks/use-toast';
import { Toaster } from 'sonner';
import { Button } from '@/components/ui/button';
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card';

type PunchType = 'IN' | 'OUT';

export function PunchButton() {
  const [punchType, setPunchType] = useState<PunchType>('IN');
  const [isPunching, setIsPunching] = useState(false);
  const [location, setLocation] = useState<{ lat: number; lng: number; acc: number } | null>(null);
  const [locationError, setLocationError] = useState<string | null>(null);
  const { toast } = useToast();

  // Fetch current punch state (server truth) on mount
  useEffect(() => {
    fetch('/api/punches/today')
      .then((r) => r.json())
      .then((data) => {
        // assume last punch determines next action
        const last = data?.punches?.[0];
        if (last?.type === 'IN') setPunchType('OUT');
        else setPunchType('IN');
      })
      .catch(() => {
        // default to IN on error (safe fallback)
        setPunchType('IN');
      });
  }, []);

  // Acquire GPS once and keep in state
  useEffect(() => {
    if (!navigator.geolocation) {
      setLocationError('Geolocation not supported');
      return;
    }
    navigator.geolocation.getCurrentPosition(
      (pos) => {
        setLocation({
          lat: pos.coords.latitude,
          lng: pos.coords.longitude,
          acc: pos.coords.accuracy,
        });
        setLocationError(null);
      },
      (err) => {
        setLocationError(err.message || 'Location unavailable');
      },
      { enableHighAccuracy: true, timeout: 10000, maximumAge: 30000 }
    );
  }, []);

  const handlePunch = async () => {
    if (isPunching) return;
    if (!location) {
      toast.error('Location unavailable — enable GPS and try again.');
      return;
    }

    setIsPunching(true);
    const optimisticId = `pending-${Date.now()}`;

    // Optimistic local feedback
    const previousType = punchType;
    const nextType = punchType === 'IN' ? 'OUT' : 'IN';
    setPunchType(nextType);

    try {
      const res = await fetch('/api/punches', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          type: punchType,
          lat: location.lat,
          lng: location.lng,
          accuracy: location.acc,
        }),
      });

      if (!res.ok) throw new Error('Punch failed');

      const data = await res.json();
      toast.success(`Clocked ${punchType === 'IN' ? 'in' : 'out'}`, {
        description: `${new Date().toLocaleTimeString()} — ${location.acc.toFixed(0)}m accuracy`,
      });
    } catch (err) {
      // Revert optimistic state on failure
      setPunchType(previousType);
      toast.error('Could not record punch — please try again.');
    } finally {
      setIsPunching(false);
    }
  };

  return (
    <>
      <Toaster position="top-center" />
      <Card className="w-full max-w-sm mx-auto">
        <CardHeader>
          <CardTitle className="text-center">
            {punchType === 'IN' ? 'Clock In' : 'Clock Out'}
          </CardTitle>
        </CardHeader>
        <CardContent className="space-y-4">
          {location && (
            <p className="text-xs text-muted-foreground text-center">
              Accuracy: {location.acc.toFixed(0)} m
            </p>
          )}
          {locationError && (
            <p className="text-xs text-destructive text-center">{locationError}</p>
          )}
          <Button
            className="w-full"
            size="lg"
            onClick={handlePunch}
            disabled={isPunching || !location}
          >
            {isPunching
              ? 'Processing…'
              : punchType === 'IN'
              ? 'Clock In'
              : 'Clock Out'}
          </Button>
        </CardContent>
      </Card>
    </>
  );
}
```

## 4. Verification

1. Start dev server: `npm run dev` and open dashboard on mobile/desktop.
2. Allow geolocation when prompted — confirm accuracy is displayed.
3. Tap “Clock In” — button should immediately show “Clock Out” and display a success toast within ~1s. Network throttling (Slow 3G) should still show immediate optimistic state change.
4. Disable network (Offline) and tap — should show error toast and revert button state; no duplicate punches should be possible while `isPunching` is true.
5. Inspect Network tab: single POST to `/api/punches` with correct payload; no duplicate POSTs on rapid double-tap.
6. Confirm backend idempotency still works: simulate duplicate delivery (curl same payload twice) — should not create two punch rows for same employee/date window.
