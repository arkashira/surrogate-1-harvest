## v0.1 scope
v0.1 ships a cross-platform CLI agent (Linux x86_64, Windows, macOS) that exposes a local SOCKS5 shadow port with TLS-camel camouflage to a single bridge endpoint, enforces a best-effort client-side killswitch (block-all-on-fail), stores sessions locally in encrypted SQLite, and reports opt-in WAU-ZB counters. No TUN in v0.1; zero-config install with reproducible builds and a simple profile JSON for endpoint/transport selection.

## Open questions
- Which pluggable transports provide the best DPI evasion vs. performance trade-off in Russia/Iran/China today (TLS camouflage vs. QUIC mimic vs. obfs4-like)?
- Can we implement a reliable client-side killswitch across Windows/macOS/Linux without admin/root that guarantees no leaks on stealth-path failure?
- How will we distribute bridge/relay endpoints to clients without creating a blocking list (domain fronting, DoH/DoQ bootstrap, or decentralized discovery)?
- What local telemetry granularity is acceptable for WAU-ZB while preserving deniability and minimizing forensic artifacts?
- Will TUN usage be required for v0.1 or can SOCKS5 shadowing + killswitch satisfy NSM without kernel-level code on day one?
