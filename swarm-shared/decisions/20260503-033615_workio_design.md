# workio / design

## Final Synthesized Solution

### Diagnosis (merged, de-duplicated)
- **Silent/Latent LINE-driven state changes**: Clock-in/out, leave, and OT updates from LINE webhooks do not appear in the UI until hard refresh → perceived latency, uncertainty, and silent failures.
- **No optimistic UI or rollback**: Mutations block on network round-trip; failures leave UI stuck with no recovery path.
- **Missing realtime push**: Frontend relies on refresh/polling; no lightweight push (SSE preferred, WS optional) to reflect backend state changes immediately.
- **Idempotency invisible to frontend**: Duplicate LINE deliveries or user double-clicks create retries backend dedupes, but frontend cannot correlate or suppress duplicate optimistic states.
- **No compact status surface**: Employees cannot see last punch or pending approvals at a glance (especially mobile), and there is no accessible skeleton/empty/toast stack.
- **Tenant drift risk on fast switching**: Role/tenant checks live in backend and localStorage; no lightweight design-time guard to prevent cross-tenant UI actions.

---

### Proposed Change (merged + corrected)
**Scope**: `workio/src/features/attendance/`, `workio/src/features/requests/`, global UI layer

- Add **optimistic UI with client-side idempotency keys and rollback** for `clockIn`, `clockOut`, `requestLeave`, `requestOT`.
- Add **global notification center** (toast stack) wired to:
  - optimistic outcomes,
  - rollback errors,
  - realtime events via **SSE** (fallback to long-polling) from backend when LINE webhooks mutate tenant state.
- Add **compact “Last punch” pill** and **“Pending approvals” badge** (mobile-first) in header/sidebar.
- Add **route-level tenant guard** to block UI actions when tenant context missing/mismatched.
- Add **idempotency header** to all state-changing API calls and expose request-id in dev/debug builds for traceability.

**Files to touch**:
- `src/lib/api.ts` (idempotency + request-id header)
- `src/features/attendance/hooks/useClock.ts` (optimistic clock)
- `src/features/requests/hooks/useRequest.ts` (optimistic requests)
- `src/components/NotificationCenter.tsx` (toast stack + realtime listener)
- `src/components/LastPunchPill.tsx` (compact status)

- `src/contexts/TenantContext.tsx` (modify + expose guard)
- `src/App.tsx` (wire provider, notification center, route guard)

### Implementation (merged + corrected)

#### 1) Idempotency + traceability in API layer
```ts
// src/lib/api.ts
import { v4 as uuidv4 } from 'uuid';

export const request = async <T>(
  url: string,
  options: RequestInit & { body?: any } = {}
): Promise<T> => {
  const headers = new Headers(options.headers);

  // Idempotency for mutating methods
  if (!headers.has('Idempotency-Key') && ['POST', 'PUT', 'PATCH', 'DELETE'].includes((options.method || 'GET').toUpperCase())) {
    headers.set('Idempotency-Key', uuidv4());
  }

  // Optional dev aid: expose request-id from response for traceability
  if (!headers.has('X-Request-ID')) {
    headers.set('X-Request-ID', uuidv4());
  }

  if (!headers.has('Content-Type') && options.body) {
    headers.set('Content-Type', 'application/json');
  }

  const res = await fetch(url, {
    ...options,
    headers,
    body: options.body ? JSON.stringify(options.body) : undefined,
  });

  if (!res.ok) {
    const text = await res.text().catch(() => 'Unknown error');
    const err = new Error(text);
    // preserve status for callers
    (err as any).status = res.status;
    throw err;
  }

  // Accept both JSON and empty responses
  const contentType = res.headers.get('content-type') || '';
  if (contentType.includes('application/json')) {
    return res.json();
  }
  return {} as T;
};
```

#### 2) Optimistic clock hook (corrected rollback + dedupe guard)
```ts
// src/features/attendance/hooks/useClock.ts
import { useState, useCallback, useRef } from 'react';
import { request } from '@/lib/api';
import { useNotification } from '@/components/NotificationCenter';

export type Punch = {
  id: string;
  type: 'IN' | 'OUT';
  timestamp: string;
  location?: { lat: number; lng: number };
  _optimistic?: boolean;
  _error?: boolean;
};

export const useClock = (initialPunch?: Punch) => {
  const [punch, setPunch] = useState<Punch | null>(initialPunch || null);
  const [loading, setLoading] = useState(false);
  const { add } = useNotification();
  const lastOptimisticId = useRef<string | null>(null);

  const mutate = useCallback(
    async (type: 'IN' | 'OUT', location?: { lat: number; lng: number }) => {
      const optimisticId = `pending-${Date.now()}-${Math.random().toString(36).slice(2, 7)}`;
      lastOptimisticId.current = optimisticId;

      const optimistic: Punch = {
        id: optimisticId,
        type,
        timestamp: new Date().toISOString(),
        location,
        _optimistic: true,
      };

      setPunch(optimistic);
      setLoading(true);

      try {
        const res = await request<{ punch: Punch }>('/api/attendance/punch', {
          method: 'POST',
          body: { type, location },
        });

        // Accept backend canonical punch; discard stale optimistic if mismatch
        setPunch(res.punch);
        lastOptimisticId.current = null;

        add({
          id: res.punch.id,
          title: `Clocked ${type}`,
          message: new Date(res.punch.timestamp).toLocaleTimeString(),
          type: 'success',
        });
      } catch (err) {
        // Only rollback the optimistic we created
        setPunch((prev) =>
          prev?.id === optimisticId
            ? { ...prev, _optimistic: false, _error: true }
            : prev
        );
        lastOptimisticId.current = null;

        add({
          id: optimisticId,
          title: `Clock ${type} failed`,
          message: err instanceof Error ? err.message : 'Unknown error',
          type: 'error',
          action: {
            label: 'Retry',
            onClick: () => mutate(type, location),
          },
        });
      } finally {
        setLoading(false);
      }
    },
    [add]
  );

  return { punch, loading, mutate };
};
```
