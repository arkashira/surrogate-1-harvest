# workio / quality

## Final Synthesis (Corrected + Actionable)

Below is the single consolidated plan that keeps the strongest, safest ideas from both candidates, removes contradictions, and prioritizes correctness and deployability.

---

## 1. Diagnosis (resolved)

- **No DB-level idempotency**: application checks can race under LINE retries or client double-taps.  
  → Fix: enforce uniqueness at storage level and use `INSERT ... ON CONFLICT DO NOTHING` end-to-end.

- **No DB-level “one open punch” guarantee**: check-then-upsert races allow multiple open punches.  
  → Fix: partial unique index to enforce at most one open punch per user.

- **No optimistic locking on punch rows**: concurrent manager/ops edits can silently overwrite.  
  → Fix: add `row_version` and increment on every update; reject on mismatch.

- **Missing retry/backoff and idempotent webhook handling**: transient failures cause LINE retries to create duplicates.  
  → Fix: derive deterministic idempotency key from webhook (or require client-supplied key), persist it, and make handler idempotent end-to-end.

- **Schema/docs drift**: `.env.example` and README omit required quality settings.  
  → Fix: add defaults and comments; keep names consistent.

- **Contradiction on scope (tenant vs user)**: Candidate 2 emphasizes tenant/day/employee; Candidate 1 uses user-only.  
  → Resolution: enforce constraints at **user** level (since tenant is not modeled in Candidate 1 schema). If multi-tenant support is required, extend with `tenant_id` and include it in unique indexes; otherwise keep user-only to avoid scope creep.

- **Contradiction on idempotency mechanism**: Candidate 2 proposes separate idempotency table; Candidate 1 uses column on punches.  
  → Resolution: use **idempotency key column on punches** (simpler, fewer joins, sufficient for this bounded context). A separate table is unnecessary unless cross-entity idempotency is required.

Schema changes:

```sql
-- Add columns if missing
ALTER TABLE punches
  ADD COLUMN IF NOT EXISTS idempotency_key TEXT,
  ADD COLUMN IF NOT EXISTS row_version INTEGER NOT NULL DEFAULT 1;

-- Idempotency: one non-null key per user (prevents duplicate retries)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_user_idempotency
  ON punches (user_id, idempotency_key)
  WHERE idempotency_key IS NOT NULL;

-- At most one open punch per user (prevents multiple concurrent "in" states)
CREATE UNIQUE INDEX IF NOT EXISTS idx_punches_one_open
  ON punches (user_id)
  WHERE clock_out_ts IS NULL;
```

Backfill (run once, verify in staging):

```sql
-- Remove duplicate idempotency rows (keep newest)
DELETE FROM punches
WHERE ctid NOT IN (
  SELECT DISTINCT ON (user_id, idempotency_key) ctid
  FROM punches
  WHERE idempotency_key IS NOT NULL
  ORDER BY user_id, idempotency_key, created_at DESC
);

-- Close extra open punches per user (keep newest)
WITH ranked AS (
  SELECT ctid,
         ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY created_at DESC) AS rn
  FROM punches
  WHERE clock_out_ts IS NULL
)
UPDATE punches
SET clock_out_ts = COALESCE(clock_out_ts, NOW())
FROM ranked
WHERE punches.ctid = ranked.ctid
  AND ranked.rn > 1;
```

Service contract (TypeScript sketch):

```ts
import { Pool, QueryResult } from 'pg';

export async function createPunch(
  pool: Pool,
  {
    userId,
    lat,
    lng,
    idempotencyKey,
  }: { userId: string; lat: number; lng: number; idempotencyKey?: string }
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    // Idempotent insert: if key exists, return existing row (no-op)
    const insert = `
      INSERT INTO punches (user_id, clock_in_ts, lat, lng, idempotency_key)
      VALUES ($1, NOW(), $2, $3, $4)
      ON CONFLICT (user_id, idempotency_key) DO UPDATE
        SET updated_at = NOW()
      RETURNING *;
    `;
    const { rows }: { rows: PunchRow[] } = await client.query(insert, [
      userId,
      lat,
      lng,
      idempotencyKey || null,
    ]);
    const punch = rows[0];

    await client.query('COMMIT');
    return punch;
  } catch (err: any) {
    await client.query('ROLLBACK');
    // Unique violation on open-punch index
    if (err.code === '23505') {
      const { rows } = await client.query(
        'SELECT * FROM punches WHERE user_id = $1 AND clock_out_ts IS NULL ORDER BY created_at DESC LIMIT 1',
        [userId]
      );
      const existing = rows[0];
      const error: any = new Error('ALREADY_CLOCKED_IN');
      error.existing = existing;
      throw error;
    }
    throw err;
  } finally {
    client.release();
  }
}

export async function updatePunch(
  pool: Pool,
  {
    punchId,
    clockOutTs,
    lat,
    lng,
    expectedVersion,
  }: {
    punchId: string;
    clockOutTs: Date;
    lat?: number;
    lng?: number;
    expectedVersion: number;
  }
) {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');

    const update = `
      UPDATE punches
      SET clock_out_ts = $2,
          lat = COALESCE($3, lat),
          lng = COALESCE($4, lng),
          row_version = row_version + 1
      WHERE id = $1 AND row_version = $5
      RETURNING *;
    `;
    const { rows }: { rows: PunchRow[] } = await client.query(update, [
      punchId,
      clockOutTs,
      lat || null,
      lng || null,
      expectedVersion,
    ]);

    if (rows.length === 0) {
      const err: any = new Error('VERSION_MISMATCH');
      err.code = 'VERSION_MISMATCH';
      throw err;
    }

    await client.query('COMMIT');
    return rows[0];
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

Webhook idempotency guidance:

- Derive a deterministic idempotency key from the LINE event (e.g., stable hash of `userId + eventTimestamp + eventType`) or require clients to supply one.
- Persist the key on punch rows and rely on the unique index to deduplicate retries.
- Implement short retry/backoff in the webhook consumer so transient DB/network errors don’t cause immediate duplicate attempts.
