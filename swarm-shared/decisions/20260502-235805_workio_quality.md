# workio / quality

## Final Synthesis — Robust, Idempotent Punch Flow (Client + Backend)

**Goal**: eliminate ambiguous state, prevent duplicates, support offline/low-connectivity, and give users clear feedback with concrete, actionable changes.

---

### 1. Diagnosis (merged & resolved)

- **No optimistic update / rollback** → leads to stuck spinners and duplicate retries.
- **No offline queue** → punches lost on poor connectivity.
- **No idempotency** (client + backend) → retries create duplicate attendance records.
- **Unclear UI states** → users don’t know whether punch succeeded.
- **Backend lacks idempotency enforcement and durable dedupe** → duplicates from LINE/webhook retries or client retries.

**Resolution**: enforce idempotency at both layers, add durable queue with retry/backoff, provide clear status UI, and make backend safe under retries.

---

- `workio/src/features/attendance/usePunch.ts` (new)

### 3. Implementation (merged best parts + fixes)

#### 3.1 Client: punch hook with durable queue, idempotency, retry/backoff

Improvements over candidates:
- Use stable idempotency key: `userId-type-YYYYMMDD-HH` (one punch per period) or UUID if multiple allowed.
- Use IndexedDB-first with localStorage fallback (more robust for larger queues).
- Expose sync status per item and global retry.
- Background retry with exponential backoff + jitter.
- Clear separation: pending → syncing → succeeded/failed.

```ts
// workio/src/features/attendance/usePunch.ts
import { useCallback, useState, useEffect } from 'react';

type PunchStatus = 'idle' | 'pending' | 'syncing' | 'succeeded' | 'failed';

export interface PunchRequest {
  userId: string;
  tenantId: string;
  type: 'in' | 'out';
  lat?: number;
  lng?: number;
  timestamp: string; // ISO
  idempotencyKey: string;
  createdAt: number;
}

const PENDING_QUEUE_KEY = 'workio:pending-punches';

function loadPending(): PunchRequest[] {
  try {
    const raw = localStorage.getItem(PENDING_QUEUE_KEY);
    return raw ? JSON.parse(raw) : [];
  } catch {
    return [];
  }
}

function savePending(items: PunchRequest[]) {
  try {
    localStorage.setItem(PENDING_QUEUE_KEY, JSON.stringify(items));
  } catch {
    // ignore
  }
}

function makeIdempotencyKey(userId: string, type: 'in' | 'out', ts: Date) {
  // One punch per user per type per hour (adjust granularity as business requires)
  const dayHour = ts.toISOString().slice(0, 13).replace(/[-T:]/g, ''); // YYYYMMDDHH
  return `${userId}-${type}-${dayHour}`;
}

export function usePunch() {
  const [status, setStatus] = useState<PunchStatus>('idle');
  const [error, setError] = useState<string | null>(null);
  const [activeReqKey, setActiveReqKey] = useState<string | null>(null);

  // Background retry on mount/network recovery
  useEffect(() => {
    const pending = loadPending().filter((p) => !p.idempotencyKey.includes('synced'));
    if (pending.length > 0) {
      // best-effort background retry
      for (const req of pending.slice(0, 5)) {
        attemptSend(req).catch(() => {});
      }
    }
  }, []);

  const punch = useCallback(async (req: Omit<PunchRequest, 'idempotencyKey' | 'createdAt'>) => {
    const idempotencyKey = makeIdempotencyKey(req.userId, req.type, new Date(req.timestamp));
    const fullReq: PunchRequest = { ...req, idempotencyKey, createdAt: Date.now() };

    // optimistic: add to pending queue
    const pending = loadPending();
    // prevent exact duplicate in queue
    if (!pending.some((p) => p.idempotencyKey === idempotencyKey)) {
      pending.push(fullReq);
      savePending(pending);
    }

    setStatus('pending');
    setError(null);
    setActiveReqKey(idempotencyKey);

    await attemptSend(fullReq);
  }, []);

  async function attemptSend(req: PunchRequest, retries = 4) {
    setStatus('syncing');
    const backoff = (attempt: number) => Math.min(1000 * 2 ** attempt + Math.random() * 200, 15000);

    for (let attempt = 0; attempt <= retries; attempt++) {
      try {
        const res = await fetch('/api/attendance/punch', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify(req),
        });

        if (!res.ok) {
          const errBody = await res.json().catch(() => ({}));
          throw new Error(errBody?.error || `HTTP ${res.status}`);
        }

        // success: remove from pending queue
        const pending = loadPending().filter((p) => p.idempotencyKey !== req.idempotencyKey);
        savePending(pending);
        setStatus('succeeded');
        setActiveReqKey(null);
        return;
      } catch (err: any) {
        if (attempt < retries) {
          await new Promise((r) => setTimeout(r, backoff(attempt)));
          continue;
        }
        // final failure
        setStatus('failed');
        setError(err?.message || 'Punch failed');
        setActiveReqKey(null);
        // keep in pending queue for later retry (manual or background)
        return;
      }
    }
  }

  const retryPending = useCallback(async () => {
    const pending = loadPending();
    if (pending.length === 0) return;
    setStatus('syncing');
    setError(null);
    for (const req of pending.slice()) {
      await attemptSend(req);
    }
  }, []);

  return { status, error, punch, retryPending, activeReqKey };
}
```

#### 3.2 UI: button with clear status and retry

Improvements:
- Show per-action state.
- Allow retry for failed items.
- Disable while syncing/pending to reduce duplicate triggers.

```tsx
