# Costinel / ops

## Implementation Plan — Costinel Top-Hub Signal Card (ops)

**Scope**: Read-only ops card that surfaces the most-connected hub (e.g., "MOC") from the existing knowledge-rag graph with contextual cost-governance signals. Uses CDN-bypass pattern for live cost data and embedded mock fallback for resilience.

**Time budget**: ~90 min implementation + 30 min validation

2. Fetch live cost signals via CDN bypass (`/resolve/main/...`) — no API auth
3. Merge: hub metadata + cost anomalies + recommendations
4. Render read-only card with audit trail

### 2. Implementation Steps

#### Step 1: Create CDN-bypass cost fetcher (15 min)

```typescript
// lib/costCdn.ts
export interface CostSignal {
  date: string;
  service: string;
  region: string;
  amount: number;
  currency: string;
  anomaly: boolean;
  recommendation?: string;
}

export async function fetchCostSignels(
  repo: string,
  path: string,
  signalCount = 5
): Promise<CostSignal[]> {
  try {
    // CDN bypass: no Authorization header
    const url = `https://huggingface.co/datasets/${repo}/resolve/main/${path}`;
    const res = await fetch(url, { cache: "no-store" });
    if (!res.ok) throw new Error(`CDN ${res.status}`);
    const data = await res.json();
    return data.slice(0, signalCount);
  } catch (err) {
    console.warn("CDN fetch failed, using fallback", err);
    return [];
  }
}
```

export interface HubNode {
  id: string;
  label: string;
  type: string;
  edgeCount: number;
  tags: string[];
  lastUpdated: string;
}

export async function getTopHub(): Promise<HubNode | null> {
  try {
    // Existing graph endpoint
    const res = await fetch("/api/knowledge-rag/top-hub", { cache: "default" });
    if (!res.ok) return null;
    return res.json();
  } catch {
    return null;
  }
}
