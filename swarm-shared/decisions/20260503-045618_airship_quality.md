## Final Synthesis: Resilient Surrogate-1 Training Pipeline (CDN-only, Zero HF API During Training)

**Core insight**: Eliminate HF API rate limits and commit-cap failures by generating a date-scoped manifest on the Mac orchestrator once per day, then using **CDN-only** fetches during training. This is the highest-leverage, lowest-effort change that unblocks stable, long-running training.

## Concrete Implementation (merged + hardened)

### 1) Manifest generator (Mac orchestration)
- Single `list_repo_tree` call per date folder (non-recursive) to avoid pagination.
- Filter to supported extensions (`.parquet`, `.jsonl`).
- Produce deterministic manifest with CDN URLs and content hash for integrity.
- Commit to a sharded sibling repo to distribute write caps; tolerate cap failures gracefully.

**File**: `scripts/generate_training_manifest.py`
```python
#!/usr/bin/env python3
"""
Generate CDN-only training manifest for Surrogate-1.
Run on Mac after rate-limit window clears (e.g., cron 06:00).
"""
import json, os, hashlib, sys
from datetime import datetime, timedelta
from huggingface_hub import HfApi

API = HfApi()
REPO = "axentx/surrogate-1-ingest"
DATE = (datetime.utcnow() - timedelta(days=1)).strftime("%Y-%m-%d")
FOLDER = f"batches/mirror-merged/{DATE}"
SIBLING_REPO = "axentx/surrogate-1-manifests"
SUPPORTED_EXT = {".parquet", ".jsonl"}

def pick_sibling(slug: str, n: int = 5) -> str:
    h = int(hashlib.sha256(slug.encode()).hexdigest(), 16)
    return f"axentx/surrogate-1-manifests-{(h % n) + 1}"

def main():
    print(f"Listing {REPO}/{FOLDER} ...")
    try:
        tree = API.list_repo_tree(REPO, path=FOLDER, recursive=False)
    except Exception as e:
        print(f"Failed to list repo tree: {e}", file=sys.stderr)
        sys.exit(1)

    files = sorted(
        f.rfilename for f in tree
        if any(f.rfilename.endswith(ext) for ext in SUPPORTED_EXT)
    )
    if not files:
        print("No supported files found; exiting.")
        sys.exit(0)

    manifest = {
        "date": DATE,
        "generated_at": datetime.utcnow().isoformat(),
        "repo": REPO,
        "files": [
            {
                "path": f,
                "cdn_url": f"https://huggingface.co/datasets/{REPO}/resolve/main/{f}",
            }
            for f in files
        ],
    }

    out_path = f"training_manifest_{DATE}.json"
    with open(out_path, "w") as f:
        json.dump(manifest, f, indent=2)
    print(f"Wrote {out_path} ({len(files)} files)")

    # Commit to sharded sibling repo; tolerate cap failures
    target_repo = pick_sibling(DATE)
    try:
        API.upload_file(
            path_or_fileobj=out_path,
            path_in_repo=f"{DATE}/{out_path}",
            repo_id=target_repo,
            commit_message=f"Add manifest {DATE}",
        )
        print(f"Committed to {target_repo}")
    except Exception as e:
        print(f"Commit failed (cap or network): {e}")

if __name__ == "__main__":
    main()
```

### 2) CDN-only training data loader
- Accept `--manifest` or default path.
- Stream from CDN URLs with timeout/retry.
- Parse each file individually and project to `{prompt, response}` on-the-fly.
- Avoid `load_dataset` on heterogeneous repos; keep parsing lightweight.

**File**: `surrogate/train.py` (excerpt)
```python
import json, io, torch, time
from torch.utils.data import IterableDataset, DataLoader
import pyarrow.parquet as pq
import requests
from pathlib import Path

class CDNParquetDataset(IterableDataset):
    def __init__(self, manifest_path, timeout=30, max_retries=3):
        super().__init__()
        with open(manifest_path) as f:
            self.manifest = json.load(f)
        self.urls = [f["cdn_url"] for f in self.manifest["files"]]
        self.timeout = timeout
        self.max_retries = max_retries

    def _fetch(self, url):
        for attempt in range(1, self.max_retries + 1):
            try:
                resp = requests.get(url, stream=True, timeout=self.timeout)
                resp.raise_for_status()
                return io.BytesIO(resp.content)
            except Exception as e:
                if attempt == self.max_retries:
                    raise
                time.sleep(2 ** attempt)

    def __iter__(self):
        for url in self.urls:
            buf = self._fetch(url)
            table = pq.read_table(buf, columns=["prompt", "response"])
            for row in table.to_pylist():
                yield {
                    "prompt": row["prompt"],
                    "response": row["response"],
                }

class SurrogateDataModule(DataLoader):
    def __init__(self, manifest_path, batch_size=32, num_workers=4):
        dataset = CDNParquetDataset(manifest_path)
        super().__init__(
            dataset,
            batch_size=batch_size,
            num_workers=num_workers,
            pin_memory=True,
        )
```

### 3) Lightning Studio orchestration (reuse + idle recovery)
- Reuse a running studio if available; restart if stopped.
- Prefer L40S (free tier); fallback to H200 in lambda-prod.
- Pass manifest path into the run so training is fully CDN-driven.

**File**: `scripts/launch_studio.py`
```bash
#!/usr/bin/env bash
set -euo pipefail
export SHELL=/bin/bash

MANIFEST_DATE=$(date -d yesterday +%Y-%m-%d)
MANIFEST="training_manifest_${MANIFEST_DATE}.json"
MANIFEST_PATH="/shared/manifests/${MANIFEST}"

python - <<PY
import lightning as L
from lightning import Studio, Machine

teamspace = L.Teamspace()
studio_name = "surrogate-1-train"

# Reuse running studio; restart if stopped
studio = None
for s in teamspace.studios:
    if s.name == studio_name:
        studio = s
        break

if studio is None:
    print(f"Creating studio: {studio_name}")
    studio = Studio(name=studio_name, machine=Machine.L40S, create_ok=True)

if studio.status != "running":
    print(f"Studio stopped; restarting on L40S")
    studio.start(machine=Machine.L40S)

# Launch training with CDN manifest
studio.run(
    run_name="surrogate-1-cdn-train",
    cloud_build_spec={
        "entry_point": ["python", "train.py", "--manifest", MANIFEST_PATH],
        "requirements": ["torch", "lightning", "pyarrow", "requests"],
    },
    wait=False,
)
PY
```

## Verification checklist
1. Run manifest generator → produces JSON with CDN URLs (no 429).
2. Start training → monitor network: zero calls to `api.huggingface.co`; only CDN downloads.
3. Studio reuse → second run reuses running studio (quota saved).
4. Idle recovery → stop studio manually; next launch restarts automatically.

**Expected outcome**: Stable training through HF rate limits, reduced quota consumption, and immunity to 128-commit caps during training loops.
