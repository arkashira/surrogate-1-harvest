## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI-first MVP (FastAPI backend on CF Workers) that lists Lambda + Cloud Functions schedules, computes next run, flags exact-cron duplicates and missing targets (orphans), and retires schedules via dry-run flag; stores results in Supabase Postgres; no UI, auth via service-account JSON, region-scoped scan.

## Open questions
- Authentication: will customers connect via OAuth (AWS/GCP) or provide scoped service-account keys?
- Granularity of drift detection: compare cron expression only, or include IAM/trigger/env differences?
- Retention window for schedule history and raw cloud_metadata (cost vs. auditability)?
- How to handle cross-account/OU discovery without per-account keys (assume-role vs. org-level reader)?
- Definition of 'orphaned' for each platform (e.g., Cloud Scheduler target deleted vs. missing trigger)?
