## Deployment
cf workers

## v0.1 scope
v0.1 ships a local-only CLI client (Rust/WASM-capable) that: creates and hides a TUN interface, wraps traffic through an obfuscated SOCKS5-like tunnel (TLS-fallback + domain-front options), enforces a fail-closed kill-switch, stores sessions/config in local encrypted SQLite, and emits minimal local telemetry for WAU-ZB — all deployable as a single static binary with no server dependencies; CF Workers host static assets and optional fronting endpoints for testing.

## Open questions
- Which obfuscation primitives minimize CPU/battery while defeating common DPI signatures in target regions?
- Can we rely on CF Workers for domain-fronting endpoints without violating ToS or being takedowned?
- What telemetry schema and consent flow balances WAU-ZB measurement with user privacy and deniability?
- How will we perform safe, automated endpoint health checks without creating detectable probe patterns?
- What is the kill-switch enforcement model (TUN down, firewall drop, or both) per platform (Linux/Windows/macOS)?
