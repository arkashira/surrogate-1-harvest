# vanguard / design

## 1. Diagnosis

- Frontend still likely triggers authenticated HF API calls (`list_repo_tree`, `/api/datasets/*`) at runtime, burning quota and risking 429s on user machines.
- No static file manifest embedded in the frontend bundle; every session re-enumerates the repo instead of using a pre-computed CDN-friendly list.
- Training/Lightning path probably re-lists files via HF API on every run instead of using a persisted manifest + CDN-only fetches.
- No deterministic repo-sharding for HF writes (single repo risks 128 commit/hr cap); ingestion likely writes all files to one repo.
- Missing wrapper-level Bash shebang/execute hygiene for any cron/launcher scripts (pattern from ops) — could cause silent failures in scheduled tasks.

## 2. Proposed change

Create a build-time manifest generator and embed it in the frontend + training entrypoint:

- Add `/opt/axentx/vanguard/scripts/build_manifest.py`
  - Uses HF API **once** (Mac side) to list a date folder via `list_repo_tree(path, recursive=False)` and emits `manifest.json` mapping logical names to CDN URLs (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`).
- Add `/opt/axentx/vanguard/frontend/static/manifest.json` (generated)
  - Included in frontend build so runtime code never calls `/api/...`; only CDN fetches.
- Add `/opt/axentx/vanguard/train/train.py` (or update existing) to accept `--manifest` and perform CDN-only data loading (zero HF API calls during training).
- Add HF write sharding helper: deterministic repo selection by hash-slug across 5 sibling repos (e.g., `vanguard-ds-00`..`04`) to lift 128/hr cap to ~640/hr aggregate.
- Ensure any cron/launcher scripts have `#!/usr/bin/env bash`, `chmod +x`, and `SHELL=/bin/bash` in crontab.

Scope for this 2h slice: implement manifest generator + frontend embedding + training shim (highest leverage, minimal code).

# /opt/axentx/vanguard/scripts/build_manifest.py
#!/usr/bin/env python3
"""
Generate a static manifest for a date folder to avoid HF API calls at runtime.
Usage:
  HF_TOKEN=hf_xxx python build_manifest.py \
    --repo my-org/vanguard-ds \
    --date-folder 2026-05-03 \
    --out ../frontend/static/manifest.json
"""
import argparse
import json
import os
import sys
from pathlib import Path

from huggingface_hub import HfApi, list_repo_tree

CDN_TEMPLATE = "https://huggingface.co/datasets/{repo}/resolve/main/{path}"

def main() -> None:
    parser = argparse.ArgumentParser(description="Build CDN manifest for a date folder.")
    parser.add_argument("--repo", required=True, help="HF dataset repo (org/name)")
    parser.add_argument("--date-folder", required=True, help="Top-level folder (e.g. 2026-05-03)")
    parser.add_argument("--out", required=True, help="Output JSON path")
    parser.add_argument("--token", default=os.getenv("HF_TOKEN"), help="HF token (optional for public repos)")
    args = parser.parse_args()

    api = HfApi(token=args.token or None)

    # List only immediate children in the date folder (avoids recursive API pagination)
    entries = list_repo_tree(
        repo_id=args.repo,
        path=args.date_folder,
        repo_type="dataset",
        recursive=False,
    )

    files = [e for e in entries if e.type == "file"]
    if not files:
        print(f"No files found in {args.repo}/{args.date_folder}", file=sys.stderr)
        sys.exit(1)

    manifest = {
        "repo": args.repo,
        "date_folder": args.date_folder,
        "generated_by": "build_manifest.py",
        "files": [],
    }

    for f in sorted(files, key=lambda x: x.path):
        manifest["files"].append(
            {
                "path": f.path,
                "cdn_url": CDN_TEMPLATE.format(repo=args.repo, path=f.path),
                "size": getattr(f, "size", None),
            }
        )

    out_path = Path(args.out)
    out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_text(json.dumps(manifest, indent=2))
    print(f"Wrote manifest with {len(files)} files -> {out_path}")

if __name__ == "__main__":
    main()

# Accept --manifest and perform CDN-only loading (no HF API during training).
import argparse
import json
from pathlib import Path
from typing import List, Dict

import pyarrow.parquet as pq
import requests
from tqdm import tqdm

def load_cdn_parquets(manifest_path: Path) -> List[Dict]:
    manifest = json.loads(manifest_path.read_text())
    rows = []
    for item in tqdm(manifest["files"], desc="Loading CDN files"):
        url = item["cdn_url"]
        # Stream download; CDN does not require auth for public datasets.
        resp = requests.get(url, stream=True, timeout=60)
        resp.raise_for_status()
        # Write to temp or memory-map; here we read into pyarrow directly from bytes.
        table = pq.read_table(pq.ParquetFile(resp.raw))
        # Project only {prompt, response} at parse time (per surrogate-1 pattern).
        # Adjust column names to your schema.
        if "prompt" in table.column_names and "response" in table.column_names:
            for batch in table.to_batches():
                df = batch.to_pydict()
                for i in range(len(df["prompt"])):
                    rows.append({"prompt": df["prompt"][i], "response": df["response"][i]})
        else:
            # fallback: include all columns but prefer projection
            for batch in table.to_batches():
                df = batch.to_pydict()
                for i in range(len(next(iter(df.values())))):
                    rows.append({k: df[k][i] for k in df})
    return rows

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--manifest", type=Path, required=True, help="Path to manifest.json")
    args = parser.parse_args()

    data = load_cdn_parquets(args.manifest)

if __name__ == "__main__":
