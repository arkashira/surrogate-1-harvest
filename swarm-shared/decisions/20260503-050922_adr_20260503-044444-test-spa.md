## Deployment
cf workers

## v0.1 scope
v0.1 in 2 weeks: CLI + FastAPI service deployable on Cloudflare Workers; scan AWS Lambda and GCP Cloud Functions in one customer-provided region per provider; detect duplicate schedules (identical cron targeting different functions) and orphaned schedules (schedules with no target); store findings in Supabase Postgres; expose GET /findings and POST /scan endpoints; auth via single Supabase-generated service key; no UI, results returned as JSON.

## Open questions
- Which identity model will customers use to grant read-only access (AWS IAM roles vs keys, GCP service accounts)?
- How frequently should scans run to balance freshness vs quota/cost (hourly vs daily)?
- Will orphan detection require historical snapshots or can we rely on current infra state only?
- Do we need RBAC/workspaces now or can we start single-tenant and add multi-tenant later?
- What retention period for schedule snapshots balances auditability vs storage costs?
